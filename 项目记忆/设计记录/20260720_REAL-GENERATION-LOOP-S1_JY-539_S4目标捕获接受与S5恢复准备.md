精确解释：

1. 路由保存的调用前任务快照仍无当前选择，因此进入一次 `召回并提交唯一选择`。
2. 组合器先调用方法召回服务；召回服务在入口内重新读取权威任务。
3. `方法召回状态` 的数值 4 是 `入口拒绝`。召回服务第 90—92 行对任务完整承接、筹办中生命周期和当前选择做入口前置检查；命中 `任务.当前选择.has_value()` 时返回默认入口拒绝。
4. 同轮报告 0 已为同一任务成功形成当前选择。任务服务的选择提交入口只提交任务方法选择，不迁移任务生命周期；因此该并发窗口内使召回入口前置从通过变为拒绝的唯一字段是当前选择。
5. 召回没有形成候选或唯一建议，适配器不生成选择请求，选择结果保持默认 `需求任务方法业务状态::入口拒绝(3)`。
6. 路由的 `映射召回状态` 对入口拒绝走默认分支，返回 `根需求筹办状态::当前性变化(5)`。

因此唯一最窄返回点为：

```text
并发线程甲形成任务当前选择
-> 线程乙进入方法召回服务并重新读取同一任务
-> 当前选择前置条件命中
-> 方法召回返回入口拒绝
-> 路由映射当前性变化并提前返回
```

`report1_post_choice_present=0` 不得解释为竞态结束后的权威任务仍无选择。路由在选择结果非成功时于第 242—244 行提前返回，尚未执行第 246 行调用后任务读回；该诊断字段只是报告默认值。
