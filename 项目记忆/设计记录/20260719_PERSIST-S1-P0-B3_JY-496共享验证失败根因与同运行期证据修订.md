# PERSIST-S1-P0-B3 JY-496 共享验证失败根因与同运行期证据修订

日期：2026-07-19

队列：#301 / DQ-193 / PERSIST-S1-P0-B3 v0.6

事件：JY-496 / E241

## 1. 输入事实

WT-301-R1 在 E240 首轮 Debug 完整自检中自然退出 1。SELF-GOV-S4A、S4B、S4C 各 8 项全部失败；S4D 只有 A06 通过；S5 只有 A01、A05、A08 通过。既有 SERVICE-DATA-S5、TASK-RESULT-S1、PERSIST-S1-P0-A、P0-B0、P0-B3C 仍为 38 / 18 / 18 / 20 / 12 且失败 0。

Release 和连续 20 轮未启动；共享事件段目录已释放，产品进程为 0。WT-301-R1 仍只有三份计划允许代码的未暂存差异，index 空，没有实施记录、提交或推送。

## 2. 生产路径根因

`生产运行期配置.数据.h` 固定 `生产运行期稳定动作键 = 0x4859'0002'0000'0001`，`运行期上下文` 把该值传给 `运行期业务装配`，后者构造唯一任务执行组合器。任务执行冻结入口明确要求动作规格的稳定动作键等于组合器支持键。

#301 草案在路由内仍定义 `安全维护动作稳定键 = 0x0001'0002'0000'0001`，并用该值创建方法动作入口。两键不同，动作阶段冻结请求必然写前拒绝；后继执行、轻量因果、任务完成、需求结算和观察均不可达。

裁决：`安全维护动作稳定键` 必须直接同源于 `生产运行期稳定动作键`。不得改生产配置、扩展 B0 ABI、让路由自己维护第二个支持键或恢复传统执行路径。

## 3. 自检证据根因

两个自检模块已经启动独立生产运行期并把路由构造改为同一租约，但 S4A—S5 的大量断言仍用外部传统夹具的仓库和服务读取新运行期句柄，或用旧仓库统计新运行期写入前后的结构数量。

句柄值不构成跨仓库所有权。新运行期真实写入只属于其上下文拥有的四仓库和领域服务；外部夹具不是这些事实的权威读者。旧夹具读回空值和旧仓库零变化既不能证明生产写失败，也不能证明拒绝零写。

## 4. 修订证据矩阵

| 对象 | 修订后证据 |
| --- | --- |
| 系统固定根和主体材料 | 同一租约上下文的系统角色清单 |
| 需求 / 任务 / 方法 / 轻量因果 | 同一租约 B0 四个值式读回入口 |
| 治理 / 动作 / 结算 / 观察 | 路由四类完整上下文和 B0 读回互证 |
| 节点 / 关系 / 索引数量 | 同一租约上下文仓库的只读计数 |
| 轻量因果无来源动态持久关系 | 同一租约关系仓库只读检查和 B0 因果身份 |
| 缓存 / 日志 / 显示 | 原非权威服务，仅验证引用同一结算材料和不能反写机器事实 |

旧服务不得再读取或写入新运行期句柄。负例继续通过篡改路由值式请求、重复调用路由和比较同运行期结构数量验证，不得删项或把旧夹具零变化当成拒绝证据。

S4B-A2 保留编号，名称改为“任务实际状态由 1 形成 10”。新运行期由任务执行组合器形成实际状态和动态，不能继续宣称旧实例特征槽位被直接覆写。
