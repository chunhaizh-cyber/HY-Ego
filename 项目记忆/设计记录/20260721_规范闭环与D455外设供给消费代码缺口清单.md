## 1. 使用边界

本清单只登记“正式规范已经给出唯一合同，但当前生产代码缺少实现或存在相反接口”的差距。它不是施工计划，不授权修改 C++，也不表示所有缺口可以一次实施。

每个缺口后续必须沿：

```text

-> 详细设计
-> 流程图
-> 函数与结构清单 / 知识图谱
-> 依赖门控施工计划
-> 执行、集成和独立复核
```

推进。

## 3. 非缺口和禁止误修项

下列现有能力应保留，不得因“未接通”而推倒重写：

1. D455 四路同步采样、内外参、时间戳、来源设备和缓存句柄。
2. D455 帧缓存的准备、候选读回、确认发布和精确撤销边界。
3. 运行消息不承载机器事实的约束。
4. 运行期业务装配中的需求、任务、方法、任务执行和结果结算组合器。
5. 自我治理邮箱、任务管理上行桥和固定安全根领域路由的可复用部分。
6. 任务工作线程不得直接轮询外设报告队列。

禁止用以下方式“快速补通”：

- 把彩色、深度、左红外、右红外命名为四类供给产品；
- 让 D455 上行桥直接写任务、状态、特征或场景事实；
- 让自我线程、普通任务工作线程或方法直接读 D455 帧缓存；
- 把 `外部材料` 强行改成 `任务执行请求` 塞入现有任务队列；
- 把自检模块 import 到生产装配充当正式消费者；
- 用日志、摘要、枚举文本或控制面板显示代替产品身份和消费状态；
- 在缺少叶子详细设计、流程图和依赖门控计划时直接施工。

## 4. 建议的详细设计分层顺序

为避免一次设计跨越过多所有权，后续详细设计建议按依赖拆成：

```text
DD-01 D455 材料成熟度与四类产品数据合同
-> DD-02 外设报告队列、分类视图和生命周期
-> DD-03 任务域唯一消费、等待项匹配和已承接材料引用
-> DD-04 冻结工作包与观察、扫描、跟踪方法材料入口
-> DD-05 方法结果领域提交、任务验证和需求复核回流
-> DD-06 生产运行期角色装配、连续调度和停止恢复
-> DD-07 真实 D455 连续样本与端到端验收
```

任务级唯一筹办执行权不属于 D455 产品依赖链，单独形成横向基础包：

```text
TASK-PREP-RIGHT / #328
-> 任务稳定身份原子取得权
-> 召回、选择和冻结同权门禁
-> 被 DD-04 / #327 消费
```

每层先核对是否已有未过期详细设计；能修订则原地修订，语义已过期则移入 `规范/详细设计/过期设计/`，不得让新旧设计并列成为实施依据。
