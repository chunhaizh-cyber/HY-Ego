## 2. 当前函数图

```text
任务工作线程::取出一条任务结果回执
├─ 任务结果治理路由::处理工作线程一条回执
│  └─ 任务结果治理路由::处理显式回执
│     └─ 任务结果结算组合器::处理任务执行结果
│        ├─ 任务业务服务::读取任务
│        ├─ 需求业务服务::读取需求
│        ├─ 方法业务服务::读取登记项 / 读取规格组
│        ├─ 任务业务服务::迁移生命周期
│        ├─ 任务业务服务::提交实际结果并完成
│        └─ 需求业务服务::提交独立结算
└─ 任务管理上行桥::处理下一条上行
   └─ 再次调用 任务工作线程::取出一条任务结果回执
      └─ 原始回执直接转换为自我治理消息
```

当前问题是双消费者和任务域直接触发需求结算；任务状态读取还只以生命周期投影为主。

## 3. DD-05F0 结构与函数清单

| 结构 / 函数 | 用途 | 权限与边界 |
| --- | --- | --- |
| `特征批次初始项规格` | 为一个候选宿主定义一项初始特征槽和值 | 由特征业务服务形成不可变规格；来源使用完整句柄版本；同层数据操作只能消费 |
| `特征批次换代项规格` | 固定既有槽、预期当前关系 / 值完整句柄、原始值版本和新值 | 不原地覆盖旧值；版本漂移在第一写前拒绝 |
| `特征批次变更规格` | 绑定强类型批次身份、受支持规则版本和从 0 连续的确定顺序 | 支持多槽初始、多槽换代及二者混合；调用方不提供可裁决摘要 |
| `特征批次发布记录` | 保存强类型批次身份、非零记录格式版本、规则版本、宿主、完整有序项目、来源和换代写前 / 发布后完整句柄 | 格式版本与业务规则版本分离；4170 专用不可变侧账唯一承载；不是主信息槽、特征值、状态或动态 |
| `特征批次发布记录账` | 按批次身份读取未找到、可读、许可竞争、未知格式或内部不一致 | 事务外预读未找到后，取得核心写入权并在第一结构写前重读；只作幂等历史账，不成为当前值第二权威 |
| `特征批次发布记录参与者` | 与原始材料参与者在同一结构事务准备、确认、逆序撤销和完成发布 | 独占侧账候选；禁止复用特征值原始材料侧表 |
| `特征批次结构参与包` | 把已验证特征候选、原始材料参与者和批次记录参与者加入具名同层数据操作的现行不透明写会话 | 不暴露仓库、执行器、会话、令牌或参与者；只读返回批次结果 |
| `特征批次同层授权桥` | 通过不依赖具体领域类型的静态模板，把私有参与包接入需求任务方法数据操作已经取得的同一事务 | 独立真实模块唯一拥有；不 import 项目领域模块、不拥有批次 / 任务 / 会话 / 令牌 / 仓库；只有两个生产数据操作模块可 import |
| `特征业务服务::形成特征批次变更规格` | 统一校验定义、值域、来源、预期当前性和幂等语义 | 特征服务保持唯一公开业务写入权 |
| `特征体系数据操作::形成特征批次结构参与包` | 写权内重读仍未找到后，形成多槽、新值、旧当前失效和新当前候选 | 只供同层数据操作加入同一会话；普通读取在发布前只见旧完整状态 |
| `需求任务方法数据操作::执行特征批次同层参与` | 在自身私有辅助路径中调用授权桥，把候选宿主写入与特征参与包放入一个事务 | 只接收特征服务已经形成的值式规格；不返回参与包、会话、执行器、参与者或令牌，不解释任务状态语义 |
| `特征体系数据操作::读取特征批次结果` | 按批次身份读取 4170 记录，再按完整句柄权威读回全部槽位、值、关系和来源 | 仅完整同义且结构互证返回历史成功；缺项、双当前、版本不闭合或未知格式均保持独立错误分类 |
| `结构写入执行器::执行` 多参与者重载 | 在一次许可和结构会话中消费非拥有有序参与者组 | 准备 / 确认按登记顺序，失败 / 显式撤销按逆序，全部确认后最后共同发布；既有单参与者重载委托同一算法 |

现有 `特征值原始材料事务参与者` 已支持多项材料，DD-05F0 默认只读复用；批次发布记录由新专用参与者和侧账承载。若 S0 证明既有原始材料参与者无法接入多参与者执行器，退回设计，禁止把批次账扩入原始材料侧表。

DD-05F0 只实现 4170 运行期发布、幂等读回和事务收口。侧账快照导出、恢复候选准入、悬空引用 / 顺序缺口拒绝和恢复发布由具名后继 `DD-05F0R` 在 #329 实际接口进入 main 后另行设计；当前函数清单不得把它们写成已覆盖能力。

命名模块依赖固定为：授权桥不 import 项目领域模块；特征数据操作 import 授权桥；需求任务方法数据操作单向 import 特征数据操作与授权桥。特征数据操作不得 import、前置声明或跨模块 friend 真实需求任务方法数据操作。参与包形成、加入会话、内部参与者和候选结果保持 private，由授权桥作为已 import 的真实实体取得窄访问权；仓库扫描必须证明授权桥 import 只有上述两个生产调用点，自检不得直接 import。

## 4. DD-05F 结构清单

任务域形成 #329 批次身份时使用确定复合键：创建为“任务创建 + 任务幂等主键 + 来源需求完整句柄”，迁移 / 完成 / 失败 / 取消为“具名操作种类 + 任务完整句柄 + 对应任务域幂等材料编号”。裸编号不具有全局批次身份；请求其它主键、前驱、目标阶段、时间、特征、来源、状态段和证据进入完整语义比较。

| 结构 | 用途 | 写入方 | 读取方 | 生命周期 |
| --- | --- | --- | --- | --- |
| `任务固定特征目录项` | 固定特征类型到正式特征定义、值域、来源和阶段规则的稳定映射 | 任务领域初始化 | 任务服务、数据操作、自检 | 运行期稳定；规则版本变更形成新版本 |
| `任务特征值材料` | 一项任务特征的宿主、定义、槽位、当前值、来源、版本和当前性 | 任务数据操作层 | 任务服务和高级只读组合器 | 每次异义变化形成新值，旧值历史保留 |

| `任务状态迁移请求` | 预期当前版本、目标阶段、目标状态段、特征增删改和证据 | 调用方形成，任务域准入 | 任务服务 | 单次请求材料 |

## 6. DD-05A 结构清单

| 结构 | 用途 | 权威性 |
| --- | --- | --- |
| `任务结果回执` | 工作线程产生的执行材料 | 非权威；只作唯一治理入口输入 |
| `任务结果归类事实组` | 每项方法结果的唯一四类归属、来源方法、动作动态、输出场景、规则版本和当前性 | `执行结果类型`、`附带结果路由状态`、`禁止结果状态`等正式任务特征及来源证据的值式读回；与任务迁移共同发布 |

| `世界事实变化触发材料` | 独立世界事实的稳定身份、版本、来源领域、状态 / 动态证据、时间、当前性、去重键和过期条件 | 独立封套；不得伪造任务、需求、方法或执行轮次 |

| `派生需求当前偏差组` | 当前状态、结构化偏差、提交等级和当前性 | 非权威候选材料 |
| `派生需求来源证据组` | 任务、方法、动作运行、动态和结果轮次 | 非权威候选材料 |
| `自我治理回流封套` | 消息身份、去重键、引用、版本、时间和过期条件 | 队列材料；不裁决任务、需求或世界事实 |
| `需求复核请求` | 来源需求、任务结果、实际状态和证据 | 需求域正式入口材料 |
