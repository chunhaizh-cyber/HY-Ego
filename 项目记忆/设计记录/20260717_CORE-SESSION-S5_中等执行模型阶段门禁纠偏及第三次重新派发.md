## 2. 第三次退回的性质

v0.3 已明确授权：

```text
先把入口最终长布尔链等值迁移为唯一具名门禁表。
再运行完整自检。
再按具名假项归属继续或退回。
只有具名表已存在、已编译、已运行后仍退出 1 且表内无假项，才是顶层汇总内部不一致。
```

执行窗口却在“入口具名门禁表尚未实施”时，以旧程序仍无归因为由停止。它把待实施目标当作执行前门禁，把实施前基线套入实施后退回条件。实际接口与 v0.3 一致，`入口.cpp` 已在允许范围内，因此本次不是新的接口漂移，而是阶段顺序误读和执行模型能力不匹配。

任务消息未成功回传只影响设计窗口是否被主动唤醒，不影响仓库内既有执行许可，也不是停止实施的原因。

## 3. 中等执行模型治理裁决

成熟计划交给中等推理模型时，必须把每个阶段编译为四项：

```text
已验证前置事实。
待实施目标。
实施完成证据。
只有前置证据成立后才可判断的完成 / 退回条件。
```

以下行为登记为模型能力不匹配：

```text
误读明确阶段顺序。
跳过已授权步骤。
把待实施目标当作阻塞或接口漂移。
连续两轮不能按检查表推进。
轮次结束时把普通续点写成完成或退回。
```

设计窗口遇到能力不匹配时应收窄同一计划，并可显式改派高推理模型继续执行。模型升级不改变执行角色，不允许修改计划 / 索引 / 队列，不扩大允许文件或代码许可。
