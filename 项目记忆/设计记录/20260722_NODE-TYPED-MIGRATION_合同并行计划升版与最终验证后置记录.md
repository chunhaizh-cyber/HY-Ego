# NODE-TYPED-MIGRATION 合同并行计划升版与最终验证后置记录

日期：2026-07-22

身份：#338 设计治理记录；记录 v0.2 施工组织裁决，不替代正式规范、详细设计、叶子施工合同、任务状态台账或 Git 事实

## 1. 用户裁决

本轮裁决不等待上一个计划执行完成后才制定下一个计划。能够提前精确冻结的提供者—消费者条件，必须先写成预冻结待实现接口合同；提供者负责实现合同，消费者从同一合同基线直接向下形成候选。完整构建、运行、跨计划联调和能力验收可以集中到最终汇合阶段，但不得删除、降低或用静态检查替代。

## 2. v0.1 串行根因与升版边界

v0.1 把两类不同关系混在一起：

1. 业务接口消费关系；
2. 工程 XML、统一自检运行器、入口和默认装配的共享文件所有权。

因此它把 `#339 -> #352` 的最终汇集顺序同时当成设计顺序、开发启动顺序和完整验证顺序，造成后继计划必须等待前一计划实现、集成和验收后才能继续。该组织方式不是业务逻辑本身，也不是正式规范要求。

v0.2 只升版施工组织，不改变节点直接身份、领域类型化记录、自我持久结构、统一冻结恢复和退役验收的机器语义。#339 已按不可变 v0.1 合同执行，本轮不修改其计划、任务身份、文件所有权或执行切片。

## 3. 预冻结待实现接口合同

合同至少冻结：

```text
提供计划与消费计划
模块和文件身份
公开 DTO、完整签名或等价行为
输入输出所有权
逻辑内返回和内部错误分类
规则 / ABI 版本
禁止补造的内部能力
最终真实接线验证和漂移退回点
```

预冻结合同是施工依据，不是当前代码事实。当前基线缺少预期符号属于待实施目标；同名异义实体、模块身份冲突、合同无法满足、循环依赖或实际签名漂移才属于具名 DRIFT。消费者不得复制提供者实现、补造同名接口、修改提供者独占文件、回退旧域或建立临时兼容双轨。
