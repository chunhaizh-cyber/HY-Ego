## 1. 当前事实

1. 管理员终端中的 Windows 沙箱测试通过，但 WT-296 所属 Codex 桌面任务在默认 `elevated` 沙箱中两次执行只读 Git 命令，均返回 `CreateProcessAsUserW failed: 1312`。
2. JY-394 允许一次人工命令授权后，任务能够读取中央材料并完成主要 S0 接口复核；尚未编辑、构建、暂存、提交或推送，阶段 890 未登记。
3. 用户明确拒绝把重复人工授权作为正常执行方式。
4. `C:\Users\zhchh\.codex\config.toml` 已把 `windows.sandbox` 从 `elevated` 改为官方兼容回退 `unelevated`，并设置 `approval_policy = "never"`；现有桌面任务仍缓存旧沙箱进程，必须重启 Codex 桌面后复测。

## 2. 裁决

1. JY-394 的“一次明确命令授权”只保留为历史发生事实，不再是现行执行入口。
2. WT-296 及后续任务默认只使用沙箱权限，禁止主动申请 `require_escalated`，禁止把逐命令人工审批作为平台兼容路径。
3. `approval_policy = "never"` 只取消人工审批弹窗，不取消沙箱边界；越界或权限不足命令必须失败并回报，不得自动提升。
4. 重启 Codex 桌面后，WT-296 先以默认权限合并复测 cwd、分支、HEAD 和 clean 状态。复测成功后从已完成的 S0 续作；仍为 1312 时停止并继续平台排障，不标记接口漂移、代码阻塞或计划失败。
5. #296 文件、结构、接口、任务分支、验证与集成门禁全部不变。

本轮只修订桌面沙箱与审批策略，并同步执行路由。不得声明 #296 已实施、阶段 890 已登记、构建已运行、任务分支已完成或多计划并行已经开始。
