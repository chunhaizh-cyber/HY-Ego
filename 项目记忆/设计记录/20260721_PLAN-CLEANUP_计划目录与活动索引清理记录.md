## 1. 触发原因

用户要求清理计划目录、计划索引和计划文件。清理前 `计划/` 根目录同时存在：

```text
当前活动计划
已完成但未归档的独立文档计划
明确写明不得执行的撤销 / 终止计划
旧依赖图已经失效的 #315—#322
446 KB 的历史流水型计划索引
```

这些文件混在根目录会让“文件存在”被误读为“当前可执行”，也使活动索引无法直接回答当前唯一路线。

## 2. 清理原则

1. 不物理删除历史计划正文，优先使用 Git 可追踪移动。
2. 根目录只保留活动总计划、现行叶子计划和活动索引。
3. 已完成计划移入 `计划/已完成计划/`。
4. 已撤销、终止、被替代或上游三件套失效的计划移入 `计划/过期计划/`。
5. 过期文件不再提供执行许可；能力目标需要时按现行规范重新形成新设计和新计划。
6. 计划索引只保留当前路由、依赖、归档规则和执行门禁，历史流水交由 Git、项目记忆和实施记录保存。
7. 为保持路由一致，同步把队列 #315—#322 改为过期历史 / 非待执行。
