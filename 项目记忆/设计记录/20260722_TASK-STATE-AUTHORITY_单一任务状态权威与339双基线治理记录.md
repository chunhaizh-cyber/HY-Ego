# 单一任务状态权威与 #339 双基线治理记录

日期：2026-07-22

设计治理版本：JY-586/E327

角色边界：设计窗口；不修改 C++、工程或自检，不构建、不运行程序

## 1. 根因

#339 的问题不是一次漏同步，而是同一可变任务状态同时写入叶子计划、计划索引、任务队列、当前状态、工作树登记和窗口交互记录。任何一次派发、S0、退回或计划修订都要求多文件同步，导致不同文件可以各自形成看似完整、实际矛盾的“当前状态”。

根因固定为：

```text
同一可变任务状态存在多个写入所有者
-> 叶子施工合同被当成状态页反复修改
-> worktree 冻结基线与中央新计划正文分叉
-> 执行 S0 无法唯一判断应读取哪个计划 blob 和哪个状态版本
```

治理方法不是再增加一次六文件同步，而是建立一个唯一任务状态台账，其它文件退回不可变合同、导航、依赖、技术身份、投影和事件职责。

## 3. 设计裁决

1. 新建 `项目记忆/任务状态台账.md`，作为每个任务当前生命周期、状态版本、责任角色、执行身份、计划 blob、worktree 引用、派发和 S0 绑定的唯一正式所有者。
2. #339 在 JY-586/E327 中按合法连续迁移链收口：

```text
已派发待执行回执
-> 接口漂移待设计修订
-> 待执行
```

3. WT-339-R1 只读失效。禁止 S0、写入、merge、rebase、reset、cherry-pick、推送、复用和物理删除；保留为双基线失败证据。
4. #339 v0.1 业务范围未改变，可继续使用同一计划版本；动态状态全部迁出后，JY-586 绑定新的不可变计划 blob `69ad59eec5b3cfa7e9b3643063c735543565cbb3`。
5. 预登记 WT-339-R2 路径和分支，但不宣称已经物理创建、形成冻结基线或取得执行许可。台账以结构化 `技术绑定阶段=预登记` 表达该边界，冻结基线版本 / 提交保持空；JY-586 发布后必须从其完整哈希创建并现场核对。
6. #340—#352 各自在台账中占一行，不使用范围聚合状态。#352 v0.1 继续保留前置集成后升 v0.2 冻结最终删除集的静态门禁。
7. 状态提交不在自身正文伪造完整 Git 哈希；使用唯一 `JY-nnn/Ennn` 定位，完整哈希由 Git 解析。

## 6. 自动检查目标

`tools/check_specs.py --strict` 扩展后至少验证：

- #339—#352 在台账一任务一行且状态枚举合法。
- 当前行与同任务最后事件的状态版本、上一状态、计划版本和末端状态一致。
- 复合迁移链逐边合法、同任务事件连续且版本不倒退；已提交事件只能追加，不能删除、改写或重排。
- 台账计划 blob 与当前不可变叶子计划一致。
- 叶子计划只有设计状态和任务状态引用，不含动态生命周期句式。
- `未绑定 / 预登记 / 已冻结` 技术绑定阶段与待执行、依赖门控、派发、S0 和执行状态所需字段按阶段闭合。
- worktree 引用必须在同一技术登记行闭合任务、DQ、计划版本和计划 blob；已冻结时，具名基线版本、完整提交、提交内计划 blob、实际路径 / 分支 / HEAD / clean 必须一致。
- 派发状态版本、派发计划 blob、S0 合同版本、S0 计划 blob、S0 结果版本必须分别解析到 main 祖先提交并按生命周期闭合。
- 待形成状态版本只能是 main 最新 JY/E 的连续下一号；提交后必须由锚定提交标题唯一解析。
- 当前状态按显式投影版本重放事件并逐任务对照，允许滞后但不得领先；checker 不硬编码某次 JY/E。
- 窗口交互只保存与台账同名、同任务、同计划、同迁移链和同操作者的生命周期事件；计划索引和队列不再维护完成声明或调度治理章节。
- #352 v0.2 及以后还必须证明 #347—#351 已完成 / 归档；调用点映射、最终删除集和执行白名单分别使用仓库内唯一不同路径与 blob，产物正文必须声明证据类型、#352、同一计划版本、`PASS` 和正数条目总数，并使用具名非空、无重复结构化表；最新事件只接受以分号分隔的精确 `门禁结果=PASS` 与三项 `证据名=<blob>` 键值。
- 动态生命周期句式在去除 Markdown 空白后扫描，禁止用软换行或“不得写”前缀夹带；执行期 dirty 文件必须属于叶子“精确允许文件”章节，并硬拒绝计划自身及 `AGENTS.md`、`.codex/`、规范、流程图、计划和项目记忆等中央治理文件。
- 任务工作树 `WT-*` 与集成身份 `INT-*` 分型校验；集成中只接受技术登记的精确 `主线发布占用=已取得`，交接 / 退回只接受 `主线发布占用=已释放`，否定自然语言不构成占用事实。
