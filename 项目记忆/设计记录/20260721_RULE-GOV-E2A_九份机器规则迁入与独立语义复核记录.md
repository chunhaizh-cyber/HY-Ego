## 2. 逻辑保留与唯一归属

### 2.1 仓库、事务和错误边界

- 4010 锁定三类完整句柄、记录状态、节点类型 0—15、关系类型 0—18、只追加不重排复用、通用关系和索引候选复判；数值与 `海中鱼巣/核心/句柄.h` 当前 ABI 逐项一致。
- 关系失效保留编号、端点、角色和审计；重复提交刚失效前句柄或当前审计句柄时只幂等读回，不再次推进版本。
- 4020 固定主信息编号永久不复用；恢复只能重新准入历史候选或以新编号形成新身份，不能原地复活已删除记录或旧句柄。
- 4030 只拥有仓库、数据操作、原子 / 聚合 / 高级 / 入口服务的依赖方向和公开接口权力，不复制对象专用写入矩阵；旧文“实例状态 / 动态默认不长期保存”的冲突口径没有迁入。
- 4040—4060 分别唯一拥有事务发布、结果分类和非权威缓存合同。依赖方向保持 `4060 -> 4050 -> 4040 -> 4030 -> 4010 / 4020`，没有形成 E2-A 回边。
- 4050 明确区分首次入口授权无效与请求初始合法后、协议内二次复核发生的允许撤销，避免把入口拒绝和逻辑内结果混成同一时点。

### 2.3 需求结算与方法代际

- 4330 的用途观察身份固定为“完整任务句柄 + 非零任务序号”；任务序号不是方法代际。观察只记录动作前冻结选择、实际动作 / 结果和正式结算，不反写本轮选择。
- 5160 保留同一需求第一轮零或一条正式结算记录、恰一动态证据、零或一因果引用、唯一 `已满足` 结论和三类未达成入口分流；需求服务不得比较裸 `I64`。
- 5340 保持学习链单向：用途观察 -> 非权威候选 -> 独立晋级授权 -> 方法服务发布不可变新代际 -> 后继任务回合重新筹办。来源方法不得原地覆盖，新代际不得进入产生它的同一任务回合。
