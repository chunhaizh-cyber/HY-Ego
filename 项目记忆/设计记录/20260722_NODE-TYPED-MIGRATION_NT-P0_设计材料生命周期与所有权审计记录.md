## 4. 详细设计总体审计结论

### 4.1 核心与恢复

- 旧结构中心设计不能全局替词；节点身份、事务参与者、冻结材料、恢复装载和物理退役必须分别重建。
- 关系、索引、运行期上下文首次发布等独立原则可以窄重绑，但在新接口冻结前不提供代码许可。
- 新设计按六包组织：P1 共享底座；P2A 特征值；P2B 状态动态；P2C 自我；P3 服务与兼容；P4 恢复与退役。

### 4.2 领域归属

| 领域 | 唯一设计所有权 | 禁止越界 |
| --- | --- | --- |
| P2A 特征值 | I64/VecI64/VecU64 类型化原始记录、原始值版本、槽位当前关系消费 | 不拥有状态动态、自我、恢复或 4170 业务语义 |
| P2B 状态动态 | 完整状态/动态事实的关系角色、时间等本域记录、生命周期读取 | 不拥有特征值原始记录、自我列表或下游因果算法 |
| P2C 自我 | 唯一自我、内部世界、成员、实例槽和状态动态读取投影 | 不直接写状态/动态仓库，不拥有根需求或线程生命周期 |
| P3 服务 | 请求适配、需求任务方法读取、删除审计、下游投影消费 | 不重新实现 P2A/P2B/P2C 写算法，不保留可写旧权威 |
| P4 恢复 | 统一冻结、严格解析、隔离候选、索引/投影重建、首次发布、旧材料策略 | 不在线热替换，不以恢复名义恢复已退出通用容器 |
