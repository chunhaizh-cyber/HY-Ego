## 3. JY-382 v0.3 裁决

### 3.1 保留 v0.2 已通过合同

以下合同不再改义：

4. 后继许可序号必须不同，跨会话撤销返回许可拒绝且写后记录零变化。
5. 不新增关系仓库 API 或能力阶段读取接口。

### 3.2 完整接域合同

合法接线形态只有两种：全部字段为空的明确未接域兼容接线，或完整接域。`结构事务接线::已接域()` 必须同时要求：

```text
域编号、运行期纪元、运行期状态。
共享许可取得、独占许可取得。
共享路径令牌验证、独占令牌验证。
撤销失败隔离回调。
```

只提供部分字段的接线是入口非法形态，四仓库、执行器和接收可选接线的特征值服务必须在开始结构访问前拒绝，不能把它当作未接域直写。结构接线可增加 `完全未接域()` / `接线形态有效()` 值式辅助。任何包装接线必须转发全部能力。`自检.动态模式.ixx` 只允许增加一个隔离转发函数和在既有聚合形成处登记该函数；不得修改动态模式生产逻辑、验收项目或其它门禁。

### 3.3 执行器隔离合同

参与者撤销和会话撤销必须都尝试并汇总，不得短路。任一路径只要撤销非成功，执行器必须在当前独占许可对象析构前调用接线隔离回调。调用缺失、回调缺失或回调返回失败均为逻辑错误；不得释放许可后继续运行依赖路径。

### 3.4 最终自检具名归因合同

`入口.cpp` 可以且只可以把现有最终长布尔链等值重排为一份单一值式表：

```text
每项 = 既有门禁名称 + 既有布尔值。
程序通过 = 表内全部值为真。
失败时输出全部为假的门禁名称。
成功时不增加失败输出。
```

不得保留第二份手写长链，不得改变任一叶子表达式、宏条件、执行顺序或退出码，不得新增完整自检正文。名称输出只做人读诊断，不承载机器事实或业务状态。
