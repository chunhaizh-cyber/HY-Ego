对象：#310 / DQ-202 v0.3 / WT-310-R2

## 1. 直接事实

E231 首个 Debug 单轮产品自然退出 0，结束后进程 0。已落盘日志 `C:\Users\zhchh\AppData\Local\Temp\WT-310-R2_Debug_single_E231.log` 的 SHA-256 为 `071F951ED919B44BC9CBDD99824C063577387F4716A5B14C3F7810406B777D39`。

五组行数分别为 38 / 18 / 18 / 20 / 12，失败 0；A33、A34、A14、A18 原始行均以 `: 通过` 结尾。判定脚本错误限定 `=通过`，因此脚本自身返回 1。执行角色按 E231 门禁停止，未启动 Release 单轮或 Debug 20 轮，未补跑，共享目录已释放。三份 WIP 保持，index 为空，未创建记录、提交或推送。

## 2. 裁决

该异常是验证工具合同错误，不是产品失败；但 E231 明确把脚本异常列为停止条件，因此该轮不登记为完整 S3，也不直接从 Release 继续。

预检通过后重新取得共享目录独占，从全新 Debug 单轮开始执行 Debug 单轮、Release 单轮和从第 1 轮全新 Debug 严格 20 / 20。E231 旧轮只作解析样本，不计入新验收。
