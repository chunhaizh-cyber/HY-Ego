## 2. 正式裁决

保留 `#329 / DQ-221`，计划升为 v0.2。R1 和 `f01a370f...` 只作失败与局部实现证据：

```text
禁止 merge / rebase / cherry-pick
禁止 reset / 切基线 / 继续写入
禁止作为 R2 基线或独立集成输入
禁止继承 Debug 构建和超时自检结果
```

v0.2 同时采用五项设计：

1. 新增 4170，以特征领域专用不可变权威侧账承载批次身份、规则版本、完整有序项目、来源和换代前后完整句柄；主信息、原始值侧表、节点和动态均不承载批次账。
2. 扩展核心结构执行器为确定顺序多参与者收口；原始值参与者与批次记录参与者同事务准备 / 确认，失败逆序撤销，最后共同发布；既有单参与者入口委托同一算法。
3. 新增无领域所有权内部模板桥；特征模块不声明需求模块实体，需求模块单向 import 特征与桥；桥 import 只允许两个生产数据操作模块。
4. 事务外幂等预读只作历史快速收口；预读未找到后必须先取得覆盖全部参与结构的核心唯一写入权，并在第一结构写前再次读取侧账。同义零候选历史读回、异义零写冲突、异常保持原分类，只有锁内仍未找到才可形成候选；参与者准备阶段再作独占最终互证。
5. 4170 增加独立非零侧账记录格式版本，并补齐 0500 结构准入证明；格式版本只解释记录形状和恢复兼容性，不与业务规则版本合并。

来源版本直接使用完整句柄版本；规则版本 1 是 v0.2 唯一受支持版本；调用方不提供可裁决摘要；许可竞争、未知格式、记录不完整、同义和异义分别分类。#329 只实现 4170 运行期发布记录、幂等读回、竞争重检和事务参与；快照导出、恢复候选准入和恢复发布拆为 `#333 / DQ-225 / DD-05F0R`，等待 #329 实际侧账接口进入 main 后继续设计，且不阻断 #330。
