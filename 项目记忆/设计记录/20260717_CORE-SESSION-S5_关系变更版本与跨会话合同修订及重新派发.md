
`关系仓库::读取关系审计(关系句柄)` 要求句柄版本与当前记录版本精确相等。协调器令牌验证逐项包含域编号、运行期纪元、许可序号、许可类型和活动线程；许可释放后原许可序号从活动表移除。

## 3. 根因

S4-A10 的同会话分支在确认后撤销成功，随后仍用 `变更.当前关系` 读取恢复记录。该句柄是写后版本；撤销已经恢复写前版本，因此按写后句柄读取必然为空。

失败不是生产接口需要改义，也不是缺少关系仓库 API，而是 v0.1 没有把“撤销后的当前句柄身份”写成精确验收合同，导致自检断言误用写后句柄。

## 4. JY-382 v0.2 裁决

1. 确认不改变关系记录或关系版本，只切能力阶段。
2. 同一原许可撤销逐字恢复写前完整记录和写前版本。
3. 撤销后原写前句柄重新成为当前有效句柄；写后句柄立即过期，写后句柄审计读取必须为空。
4. 撤销不额外递增版本。写后版本在独占许可释放前没有对普通读者发布，不属于已发布历史版本回退。
5. 跨会话按域编号、运行期纪元、许可序号、许可类型和活动线程拒绝；任一不一致均返回 `许可拒绝`。
6. 跨会话拒绝必须保持写后记录、写后句柄可读性和能力阶段零变化。
7. `结构写入会话::完成发布()` 关闭会话层撤销入口；许可释放后原令牌退出协调器活动表，旧能力自然失效。
8. 不新增关系仓库 API，不新增影子版本账，不改变生产函数签名。
