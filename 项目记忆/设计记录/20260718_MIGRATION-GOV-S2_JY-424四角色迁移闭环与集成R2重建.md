## 3. 规则裁决

四角色固定为：

| 角色 | 唯一职责 | 正式产物 | 不得承担 |
| --- | --- | --- | --- |
| 设计 | 规则 / 流程图 / 详细设计 / 计划、依赖和优先级、执行 / 集成 / 复核派发、回执裁决、归档和下一项 | 中央治理提交 | C++ 实施、集成汇集、替只读窗口伪造独立结论 |
| 执行 | 实际接口复核、计划内代码 / 工程 / 自检、任务级验证 | 同名任务分支和执行记录 | 修改中央路由、发布 main、自行转集成 |
| 集成 | 身份 / 范围核对、固定顺序汇集、完整重验、集成记录和 main 非强制发布 | 集成结果、正式 main 提交和集成记录 | 修改业务语义、设计合同或测试口径 |
| 只读复核 | 指定提交与对象上的独立事实核查 | 任务消息报告 | 任何文件写入、构建、入队、依赖解除或完成裁决 |

统一闭环：

```text
设计 -> 真实派发执行 -> 任务分支
-> 设计核对并真实派发集成 -> 集成发布 main
-> 设计归档并重算下一项
-> 里程碑 / 最终完成前独立只读复核
-> 设计逐项裁决后继续或完成
```

