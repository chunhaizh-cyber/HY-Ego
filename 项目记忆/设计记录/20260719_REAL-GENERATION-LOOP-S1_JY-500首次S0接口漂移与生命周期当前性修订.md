# REAL-GENERATION-LOOP-S1 JY-500 首次 S0 接口漂移与生命周期当前性修订

日期：2026-07-19

## 1. 输入

`WT-315-R1` 已按 E244 完成 #315 v0.1 的纯只读 S0。身份、两根、其余 B0 入口、稳定键、工程顺序、阶段 1100 和文件所有权均一致；唯一漂移是计划把“调用方预期旧生命周期版本”写成 `迁移任务生命周期请求` 的输入字段。

## 2. 当前代码事实

正式请求只有：

```text
任务
目标阶段
幂等材料编号
发生时间戳
```

`任务服务::迁移生命周期` 在入口内部读取当前任务，把当时的当前生命周期关系和版本传入 `需求任务方法数据操作::形成任务生命周期写入规格`。数据操作在 `迁移任务生命周期_实现` 写前再次复核该关系和版本，并同时执行合法阶段检查。因此旧关系 / 版本当前性已经由服务专用数据操作边界承载，不属于生产路由公开请求 ABI。

## 3. 根因

v0.1 将数据操作内部不透明写入规格字段误提升为调用方 DTO 字段，形成了不存在的接口假定。若按旧计划实施，只能修改禁止的 B0 / 领域接口，或在路由侧复制当前性判断，都会扩大所有权并重新制造迁移尾账。

## 4. 裁决

1. #315 升级为 v0.2，不新增 B0 字段，不修改领域 / 数据操作实现。
2. 根需求筹办路由只传正式四字段请求，不接收、保存或比较旧生命周期版本。
3. 当前性由任务服务内部读取和数据操作写前复核共同裁决；路由只消费强类型结果并权威读回任务。
4. #315 验收改为合法顺序、非法跨阶段、同阶段同义 / 异义和两线程同义竞争；既有领域专项继续证明内部旧关系 / 版本拒绝。
5. WT-315-R1 因冻结计划 v0.1 已改变而只读失效，不提交断点、不复用。v0.2 从本裁决发布后的新 `origin/main` 建立 WT-315-R2。
6. #316-#322 依赖关系不变，继续门控。

## 5. 同步范围
