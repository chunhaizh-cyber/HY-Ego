## 2. 结论先行

当前代码具备三类可复用基础：

1. 节点、主信息、关系、索引、领域服务、需求任务方法组合器、任务执行组合器和任务结果结算组合器已经进入运行期业务装配。
2. D455 已具备真实相机采集器、四路同步帧协议、短期帧缓存和返回标量运行消息的上行桥。
3. 自我治理邮箱、消息协议、固定安全根领域路由、任务管理上行桥和多轮闭环自检夹具已经存在。

但生产链当前没有接通：

```text
D455 真实采样
-> L0—L3 成熟处理
-> 四类强类型供给产品
-> 外设报告队列
-> 任务域唯一消费和工作项承接
-> 冻结方法读取已承接材料
-> 领域提交
-> 任务验证与需求复核
-> 自我下一轮治理
```

所以当前只能声明“原子能力和自检骨架存在”，不能声明“规范闭环已由生产代码接通”。

## 3. 当前真实 D455 调用链

现有可达链是：

```text
--d455-real-sample-exit
-> 适配/自检.D455真实样本.ixx
-> 创建并打开D455相机采集器
-> 采样彩色、深度、左红外、右红外同步帧
-> D455帧材料缓存
-> D455采样材料上行桥::上行一次
-> 返回 运行消息类型::外部材料 和缓存材料句柄
-> 自检读回与结束
```

证据：

- `海中鱼巣/入口.cpp:441-443` 只把真实 D455 暴露为自检后退出参数。
- `海中鱼巣/入口.cpp:448-451` 在未启用 RealSense 构建时直接拒绝该自检模式。
- `海中鱼巣/适配/自检.D455真实样本.ixx:25,42-43,61` 是真实采集器、缓存和上行桥的唯一真实调用点。
- `海中鱼巣/线程/自检.D455采样材料.ixx` 是其余上行调用点，全部属于模拟自检。
- `海中鱼巣.vcxproj:78` 将 `EnableD455RealSense` 默认设为 `false`。

没有发现生产启动、运行期上下文、运行期业务装配、运行宿主或常驻入口创建 D455 采集器、缓存或上行桥。

## 4. 材料与供给产品符合性

`海中鱼巣/适配/协议.D455采样材料.ixx:126-137` 的 `D455同步帧材料` 当前承载：

```text
彩色平面
深度平面
左红外平面
右红外平面
三组相机外参
批次、时间戳和设备材料
```

`海中鱼巣/线程/缓存.D455帧材料.ixx` 只缓存 `D455同步帧材料`。对生产代码做定向检索后，没有发现下列正式身份或等价强类型结构：

```text
L0 / L1 / L2 / L3 材料成熟度
原始逐簇报告
稳定观察子集报告
扫描变化报告
目标跟踪报告
产品类型
外设报告队列
任务域已承接材料引用
```

因此当前四路相机流不能解释成规范的四类逻辑供给产品；它们只是一个原始同步帧的四个图像平面。

## 5. 上行桥与任务域之间的断点

`海中鱼巣/线程/上行桥.D455采样材料.ixx` 已正确把目标角色限制为任务管理，并形成非事实运行消息：

- 文件头第 1 行明确上行桥“不得入队、写仓库或调用领域服务”。
- 第 54—61 行只接受目标角色为 `任务管理` 的请求。
- 第 103—117 行生成 `运行消息类型::外部材料`，并只携带缓存材料编号、版本、批次、时间和摘要。
- 第 141—146 行把消息和材料返回调用方，没有队列投递。

但 `海中鱼巣/线程/任务管理线程.ixx` 当前没有与之匹配的接收入口：

- 第 155—163 行只建立普通任务调度队列和强类型任务执行请求队列。
- 第 196—321 行只准备、确认和取出 `任务执行冻结请求`。
- 第 369—380 行的公开发送入口只接受 `运行消息类型::任务执行请求`，对 `外部材料` 明确返回“非任务执行请求”。
- 第 470—480 行没有外设报告队列、产品索引、等待项匹配或材料消费窗口结构。

因此当前链实际停在：

```text
D455上行桥返回外部材料消息
-> 没有生产调用方入队
-> 任务管理线程也没有可接收该消息的入口
```

## 6. 外设线程不能替代任务域承接

`海中鱼巣/线程/外设采样材料线程.ixx:1` 明确只承载模拟或空材料，并禁止接真实 D455。该线程：

```text
发送外部材料消息
-> 入自己的有界运行消息队列
-> 自己消费或丢弃过期材料
```

它没有任务、等待项、产品、范围、目标、质量或消费窗口匹配，也没有把报告交给任务域。因此它只是线程壳自检对象，不能作为规范要求的外设报告生产者或任务域消费者。

## 7. 任务工作包和方法读取断点

`海中鱼巣/线程/协议.任务执行请求.ixx:91-130` 的调度请求包含任务、时序、幂等主键和回执材料；第 353—358 行的冻结请求只包含：

```text
调度请求
任务材料
方法材料
动作材料
执行适配器
```

其中没有外设产品身份、成熟度、来源报告、任务域承接关系、工作项、消费窗口或已承接材料引用。

`海中鱼巣/线程/任务工作线程.ixx` 当前消费普通任务消息或强类型任务执行请求并形成结果回执；它没有从任务域取得已承接外设材料的接口。`海中鱼巣/领域/组合.任务执行.ixx` 当前执行适配器只有任务目标状态值变化，也没有按产品类型向观察、扫描或跟踪方法交接材料。

因此即使未来把 D455 消息塞入某个队列，当前冻结方法仍无法按规范取得已承接材料。

## 8. 自我内部治理生产接线

当前自我治理模块并非空白，但生产接线尚未成立：

1. `海中鱼巣/线程/自我线程.ixx:102-109` 的三参数构造不带治理邮箱和处理器。
2. `海中鱼巣/入口.cpp:504` 的当前默认入口自我线程使用该三参数构造；独立 `--runtime-context` 生产模式没有创建自我线程。
3. `海中鱼巣/线程/自我线程.ixx:336-349` 在没有治理邮箱时只进入驻留等待，不执行治理循环。
4. 带邮箱和处理器的构造当前只在 `海中鱼巣/线程/自检.自我治理多轮.ixx` 使用。
5. `任务管理上行桥` 和 `自我治理领域路由` 的生产类已经存在，但构造调用只出现在自检模块。
6. `海中鱼巣/启动.生产运行期.ixx:1` 明确不得启动消费者；`海中鱼巣/入口.cpp:466-476` 的 `--runtime-context` 模式只首发上下文、持有租约后立即返回。

运行期业务装配已经拥有需求、任务、方法、任务执行和结果结算组合器，这是可复用的领域底座；但没有生产组合器创建并持有任务管理线程、任务工作线程、自我治理邮箱、任务管理上行桥和治理处理器，也没有连续调度它们。

因此“自我治理多轮自检通过”只能证明夹具内组合可运行，不能证明生产内部循环已经接通。

## 9. 潜在旁路合同

`海中鱼巣/线程/自我治理消息协议.ixx:448-460` 仍允许：

```text
外部材料适配入口 或 任务管理上行桥
-> 观察材料准入请求
-> 基础服务目标
```

当前没有生产发送者和处理器使该旁路实际运行，但该协议表达能力与 8100、8200 的最新规则不一致：外设报告到达本身不是自我治理消息，必须先由任务域承接并交当前冻结方法处理。后续设计应删除或收窄这条协议能力，不能把它当作补接 D455 的捷径。

