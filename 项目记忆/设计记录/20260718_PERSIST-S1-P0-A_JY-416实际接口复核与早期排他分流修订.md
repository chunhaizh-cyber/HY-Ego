## 2. 漂移判定

原 #297 对运行期上下文能力的假定成立，不需要修改 `启动.运行期上下文.ixx` 或业务装配。两项真实设计漂移为：

1. 阶段 900 的登记所有权应在入口，不应把 `自检.运行器.ixx` 列入允许文件。
2. 显式生产运行期不能沿默认旧栈构造后再分支；必须在首个旧仓库对象之前完成模式冲突拒绝和排他分流。

以上漂移可在原计划边界内通过缩减运行器权限、精确入口职责和会话所有权解决，不需要扩大领域 / 核心 / 线程 / 适配文件范围。

## 3. JY-416 裁决

### 3.1 文件与所有权

#297 v0.2 只允许七个路径：三个新文件、入口、工程及 filters、执行专属断点。`自检.运行器.ixx` 改列禁止文件。

三个新文件固定为：

1. `生产运行期配置.数据.h`：稳定数据单头，唯一拥有生产版本 1 的域、仓库、21 + 3 稳定键、状态值、动态根和动作键；启动请求只含版本。
2. `启动.生产运行期.ixx`：生产真模块，定义强类型启动状态 / 结果和不可复制 `生产运行期会话`；会话唯一拥有 `运行期上下文宿主`。
3. `自检.生产运行期首发.ixx`：Debug 自检真模块，唯一拥有 A01—A18 场景正文，不进入生产依赖方向。

### 3.2 生产启动顺序

`生产运行期会话::启动` 的机械顺序固定为：

```text
校验请求版本
-> 构造候选运行期上下文
-> 初始化系统角色
-> 初始化概念活动
-> 复核完整
-> 尝试首次发布到唯一宿主
-> 读取有效租约并复核同一上下文身份
```

同一会话并发调用只允许一笔返回 `已启动`；其余调用读取同一宿主并返回 `已有当前上下文`，属于合法线性化结果。生产固定配置错误、有效请求后的初始化失败、完整门禁失败、发布异常或发布后租约异身份均属内部不一致，必须追根因。

### 3.3 入口双路径

入口解析参数后、首个旧仓库对象构造前处理：

```text
显式 --runtime-context
-> 先拒绝与 database / performance / D455 / headless 的冲突组合
-> 构造一个生产运行期会话
-> 启动一次并验证租约
-> P0-A 尚无消费者，成功后退出 0
```

默认路径继续构造旧运行期。Debug `--self-test-exit` 只在默认路径向中央运行器登记阶段 900；显式新通道即使同时带 `--self-test-exit`，也只完成真实首发后退出，不执行中央阶段总成。这样生产首发和 900 隔离自检不会在同一进程构造两个上下文。
