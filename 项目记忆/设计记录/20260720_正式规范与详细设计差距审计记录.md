# 正式规范与详细设计差距审计记录

日期：2026-07-20

状态：设计窗口正式差距审计 / 只形成缺口，不构成代码实施或计划入队依据

## 1. 审计目标与权威基准

本次审计以 `规范/规范目录.md` 登记的 85 份编号规范为唯一正式机器逻辑基准，核对 `规范/详细设计/` 中现有详细设计是否：

- 明确承接现行正式规范；
- 保持对象身份、事实来源、写入权、生命周期和相邻边界一致；
- 把根规范原则展开为可机械判断的结构、流程、失败语义和验证合同；
- 不继续使用已删除规范、迁移过渡规范或已归档设计作为当前施工依据。

详细设计只能解释如何实现正式规范，不能凭既有代码、既有计划、实施完成记录或历史确认状态反向覆盖正式规范。

### 2.2 审计方法

对 108 份当前文件执行了以下核对：

1. 逐文件读取标题、状态和依据区；
2. 对照正式目录解析根级规范引用；
3. 扫描已删除规范路径、已归档详细设计引用和迁移过渡依据；
4. 按特征、需求、任务、方法、语言、因果、事务、运行治理、观察和自我循环主题检查关键合同；
5. 对任务筹办、任务状态、需求路径权重、正式特征类型和原始事务令牌命中文件进行人工深读；
6. 把命中分为直接冲突、缺少正式承载、仅引用过期和当前未发现直接冲突四类。

关键词未命中只用于定位缺失候选，不能单独证明语义不存在；直接冲突结论均有正文证据。

## 3. 总结论

当前详细设计体系不能作为一个整体直接进入后续施工计划生成。主要原因不是设计数量不足，而是正式规范体系已经完成重建，详细设计仍主要停留在迁移期、第一轮实现期或旧治理期口径。

最关键的四个断点是：

```text
正式规范权威映射为零
-> 正式特征类型结构没有进入详细设计
-> 需求 OR / AND / 权重没有进入详细设计
-> 任务状态特征集合和任务级唯一筹办执行权没有进入详细设计
```

因此，后续不能只批量替换引用路径。必须先重建特征、需求、任务和方法四个依赖层的详细设计，再处理语言、因果、观察、自我循环和工程兼容路径。

优先级：P0

事实：

- 108 份当前详细设计中，直接引用 `规范/规范目录.md` 登记编号规范的文件为 0；
- 61 份存在根级 `规范/*.md` 引用，但引用对象全部是非正式旧规范或已删除规范；
- 53 份引用当前仍物理存在、但未被正式目录授予规范身份的旧文件；
- 31 份引用 `000_项目规则总纲.md`、`001_规则迁移清单.md` 或 `迁移路线权力分层规范.md` 等已经物理删除的文件；
- 47 份没有根级规范引用；
- 上述 53 和 31 存在 23 份交集，不能相加解释为文件总数。

影响：现有“已确认”“正式详细设计”“施工依据”状态只能证明历史发生事实，不能证明当前仍符合 85 份正式规范。

收口要求：每份保留在主目录的详细设计都必须增加正式规范编号与条款映射；无直接业务规范来源的纯工程设计必须明确写出“不适用原因 + 0050 或项目工程规则替代依据”。仅替换文件名而不重审正文无效。

### DD-GAP-002：任务级唯一筹办执行权没有详细设计承载

优先级：P0

正式要求：`3200`、`5210`、`5230` 规定任务筹办是任务域内建最小方法；同一任务必须按任务稳定身份取得唯一筹办占用，未取得者不得进入召回；不同任务可以并行；筹办可重复并直接形成闭合结果，不建立独立持久化筹办回执。

当前事实：

- 当前 108 份设计中“筹办执行权”命中 0；
- 首批整理已经把所有以独立筹办回执、双路径同义选择或旧状态机筹办桥为核心的 7 份任务设计移入过期目录；
- 主目录因此不再残留独立筹办回执设计，但也没有现行筹办详细设计；
- `通用方法召回登记规格索引复判排序详细设计.md` 对空候选返回无候选材料的口径可以保留，但它不能代替任务域取得执行权、闭合结果和冻结提交。

必须新形成：

```text
任务身份 -> 原子取得筹办占用 -> 权威读取 -> 路径/候选 -> 闭合结果 -> 释放占用或阶段迁移
```

验收必须分别证明同任务排他、不同任务并行、未取得权者零召回、重复筹办保留历史、无候选不创建方法首，以及两路径已同时进入召回时按内部并发规则破坏追根因。
