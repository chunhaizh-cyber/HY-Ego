## 2. 当前代码根因

R4 已排除“任务创建幂等身份错误绑定当前生命周期”的上一层根因。A11 剩余失败来自生命周期迁移的并发收口：

```text
自检两个线程同时运行完整根需求筹办路线
-> 线程甲提交“已承接”
-> 线程乙读取已承接并继续提交“筹办中”
-> 线程甲对“已承接”执行写后读回时，当前已是“筹办中”
-> 服务只识别当前阶段正好等于目标的同阶段幂等
-> 数据操作写后要求当前仍精确等于本次目标阶段和版本
-> 已合法进入完整历史的“已承接”被映射为内部不一致
-> 路由提前返回，A11 两份报告不能同时完整
```

精确锚点：

- `海中鱼巣/线程/自检.根需求筹办.ixx:89-106`：两个线程运行完整筹办路线。
- `海中鱼巣/线程/路由.根需求筹办.ixx:133-155`：任一生命周期调用非成功即返回。
- `海中鱼巣/领域/服务.任务.ixx:96-123`：只识别当前阶段等于请求目标的幂等重入。
- `海中鱼巣/领域/数据操作.需求任务方法.ixx:2875-2946`：提交后要求当前仍精确等于本次目标阶段和版本。

## 3. 上游设计遗漏与裁决

旧目标流程图把“已承接、筹办中”压成一个节点；详细设计只写“并发当前性变化由提交边界收口并重新读回”，没有区分以下三类：

1. 目标历史不存在，且写前关系 / 版本已经漂移：零写入逻辑内返回。
2. 目标迁移已经唯一同义地进入完整历史，但同义线程继续推进当前阶段：按完整历史幂等读回当前任务。
3. 提交已成立后目标历史缺失、重复或异义：内部不一致并追根因。

JY-517 固定四字段请求不变：任务、目标阶段、幂等材料编号、发生时间戳。完整历史同义必须逐项证明目标阶段、状态幂等主键、发生时间戳、状态值、场景、主体以及任务生命周期关系的类型、源、目标和顺序证据。调用方不得新增预期旧版本。

同一幂等材料历史唯一且完全同义时，即使当前任务已经继续前进，也返回当前任务幂等读回；同键异义继续返回幂等冲突且结构不变。历史重复、版本不唯一或关系证据不完整属于既有结构矛盾。数据操作提交后不得要求当前仍停留在本次目标；结构提交已成立但目标历史缺失、重复或异义必须追根因。
