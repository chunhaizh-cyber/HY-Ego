## 3. L1—L4 代码追踪

直接生成点：`海中鱼巣/线程/自检.根需求筹办.ixx`。A09要求同一生产租约内第二次完整筹办返回同一任务 / 方法 / 选择 / 冻结请求；A10先用无效配置验证入口拒绝，再要求有效配置仍完整读回；A11在新生产会话中由两个线程并发执行同一配置。

首个共同失败分支：`海中鱼巣/线程/路由.根需求筹办.ixx` 每次都先调用 `创建已有需求承接任务`。第一次成功后，任务已经合法从“已创建”推进到“已承接 / 筹办中”。

第一个违反契约的位置：`海中鱼巣/领域/数据操作.需求任务方法.ixx` 的 `任务承接匹配`：

```text
读取当前生命周期状态
-> 要求当前生命周期阶段 == 已创建
-> 用该当前状态比较原创建初始状态规格
```

这把不可变创建身份错误绑定到了可推进的当前生命周期。顺序重入时，相同任务主键在任务已筹办中后被误判为幂等冲突；并发时，输家可能在首次创建读回前观察到赢家已经推进的生命周期而命中同一误判。

A10 没有证据表明“无效配置入口拒绝”本身失败；其失败由随后有效配置的同义重入不完整派生。A12 是 A09—A11 的聚合结果，不是独立根因。

## 4. L5 规范与设计裁决

既有目标流程图已经写明“同租约 B0 创建或幂等读回承接任务”；详细设计和输入契约已经要求重复、同键异义和并发单提交。因此控制流和业务语义不变，不新建流程图。

本次补充详细设计的精确合同：

1. 创建幂等匹配复核任务不可变身份、来源需求和虚拟存在。
2. 原始初始状态必须从完整生命周期历史中精确读取第1版，且阶段 / 主键 / 场景 / 主体 / 时间 / 值与原创建规格一致。
3. 当前任务只要求权威材料完整和当前生命周期有效，不要求当前阶段仍为已创建。
4. 同主键异义继续返回幂等冲突，结构不变化。
5. 不新增 B0 公开主键读取、DTO 字段、原始事务能力或调用方生命周期旧版本。
