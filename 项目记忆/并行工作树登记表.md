## 5. 登记规则

1. 本表只由主集成工作树的设计角色修改；集成角色写专属集成记录，由设计角色读取后同步本表。
2. 任务工作树写入前必须先形成登记行；消息或本地目录存在不能替代登记。
3. 每个任务分支必须使用 `codex/<队列号>-<主题>`，每个 worktree 只能承载一个活动计划。
4. 状态只能按 `已预留 -> 已创建待接管 -> 执行中 -> 分支完成待集成 / 接口漂移待设计 / 验证失败待裁决 -> 已集成待设计同步 -> 已完成 / 已归档 -> 可回收 -> 已回收` 推进。
5. 路径、分支、基线、计划版本、文件所有权或窗口所有者任一不一致时停止写入并由设计角色复核。
