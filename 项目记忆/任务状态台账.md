# 任务状态台账

更新时间：2026-07-23

状态版本：JY-605/E346

## 1. 唯一权威职责

本文件是项目任务当前生命周期状态的唯一正式所有者。每个任务在“当前状态唯一表”中只能有一行；状态迁移必须在一个正式治理提交内原子更新该行并向“状态迁移事件”追加一条不可变事件。

其它文件职责固定为：

- 叶子计划冻结目标、依据、依赖合同、允许 / 禁止文件、S0、失败收口和验证矩阵，不维护当前生命周期。
- `计划/计划索引.md` 只维护计划导航、版本、静态依赖、计划激活状态和具名有序执行序列；计划激活状态不等于任务生命周期。
- `项目记忆/Codex任务队列.md` 只维护执行顺序、直接依赖和解除条件。
- `项目记忆/并行工作树登记表.md` 只维护路径、分支、基线、所有权和 Git 技术状态。
- `项目记忆/当前状态.md` 只形成明确标注状态版本的人读投影，可能滞后，不参与 S0 或执行权裁决。
- `项目记忆/窗口交互记录.md` 只追加任务生命周期迁移事件；消息投递、角色交接和一般治理流水只有在构成状态迁移证据时进入事件证据列，不单独形成第二套当前状态或一般流水。

状态版本使用唯一命名提交 `JY-nnn / Ennn`。提交不能在自身正文中保存自身完整 Git 哈希；完整哈希由 Git 对该具名提交唯一解析，不在其它 Markdown 复制维护。

## 2. 生命周期枚举与迁移原则

当前使用以下生命周期枚举：

```text
待执行
依赖门控待执行
已派发待执行回执
执行中
分支完成待集成
已预留待集成派发
已派发待集成回执
集成中
集成身份冲突待设计重建
集成失败退回任务
集成分支验证通过待主线发布
已集成待设计同步
接口漂移待设计修订
验证失败待设计裁决
只读复核中
只读复核完成待设计裁决
已完成 / 已归档
```

`未登记` 只允许作为首条迁移事件的上一状态，不是当前状态。状态迁移必须遵守 `规范/设计执行双窗口交互规范.md`；一项治理提交需要连续完成两个合法迁移时，事件使用 `A > B > C` 记录完整链，不得省略中间失败或退回状态。

## 5. S0 唯一读取合同

执行或集成 S0 必须同时核对：

```text
任务编号和计划版本
台账最新状态版本
计划路径与台账计划 blob
台账派发状态引用
台账 S0 计划 blob
worktree 登记中的路径、分支和冻结基线
Git 实际计划 blob、HEAD、dirty state 与所有者
```

计划 blob、S0 计划 blob、worktree 基线或派发状态任一不一致时固定返回具名 DRIFT，禁止用其它 Markdown 的显示状态或聊天解释跨越。叶子计划正文发生实质变化时，原冻结 worktree 自动只读失效；必须由新的状态版本绑定新 blob，并按正式规则重建技术身份。
