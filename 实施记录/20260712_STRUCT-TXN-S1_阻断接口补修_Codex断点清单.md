## 1. 触发事实

执行窗口在 `9823507` 上只建立 `结构事务接线.数据.h`、`协调.结构事务.ixx`、`自检.结构事务.ixx` 骨架及工程登记，Debug x64 0 警告 0 错误。继续实施前发现：

1. 关系 / 索引的锁内节点读取覆盖查询、统计和父链，不只是创建 / 重挂 / 绑定。
2. 原设计没有给出传统 `.cpp` 可调用的完整函数表、移动许可析构、运行期状态寿命和重入规则。
3. 若节点 / 主信息删除与普通操作都持共享许可，端点仍能在复核期并发消失。

执行窗口未修改四仓库、入口或 `写入事务`，未暂存、未提交。

## 2. 正式裁决

```text
数据桥使用 std::shared_ptr<void> 延长运行期状态寿命，不使用裸协调器指针。
令牌固定为域 + 运行期纪元 + 许可序号 + 许可类型。
移动许可持共享状态、令牌和 noexcept 释放函数；不可复制，析构至多释放一次。
接线固定共享 / 独占取得函数和共享路径 / 独占验证函数。
活动许可验证同时检查状态、域、纪元、序号、类型、线程和活动表。
同线程无令牌重入是实现错误；同步嵌套必须下传令牌，令牌不得跨线程或排队。
节点 / 主信息身份删除取得独占许可；其它普通操作取得共享许可。
四仓库 46 个公开声明入口逐项裁决，41 个入口形成令牌路径。
关系 / 索引本仓库锁内只复制，锁外以同一令牌读取节点。
```
