### 2.2 新接口漂移

| 复核项 | 当前事实 | 对 #269 的影响 |
| --- | --- | --- |
| 虚拟存在规格 | `服务.存在` 只有立即创建请求，要求来源已发布 | 无法表达本会话刚创建的任务 / 方法来源 |
| 虚拟存在写入 | `数据操作.存在场景` 私有 `创建身份` 启动自己的执行器 | 外层会话不能参与，调用即形成嵌套 / 独立事务 |
| 抽象状态规格 | `服务.状态` 的抽象状态只有立即创建 | 需求目标状态和方法角色状态不能加入外层会话 |
| 已发布主体实例状态 | 有业务规格，但 `写入实例状态_会话` 为私有 | 任务生命周期新状态不能加入任务关系换代同一会话 |
| 候选主体实例状态 | 既有规格要求主体已经发布 | 任务 / 方法虚拟存在与初始状态不能同批提交 |
| 会话候选身份 | 提交准备视图可判断，写入阶段无窄入口 | 原子参与者无法验证来源 / 主体确属本次写集 |

若直接实施 #269，只能重复解释存在 / 状态结构、调用原子立即创建形成嵌套事务，或分批发布任务 / 方法半事实。三条路径均不允许。
