## 7. 第一轮建议讨论顺序

建议下一轮讨论不要直接进入字段实现，而按以下顺序确认：

```text
主信息总边界
-> 允许保留的主信息字段类别
-> 必须改写为关系 / 索引 / 特征 / 状态 / 动态 / 因果的类别
-> 语素相关旧字段的处理口径
-> 基础信息名称 / 类型 / 统计字段口径
-> 需求 / 任务 / 方法字段是否拆到各自系统规范
-> 是否建立新项目主信息规范和详细设计
```
