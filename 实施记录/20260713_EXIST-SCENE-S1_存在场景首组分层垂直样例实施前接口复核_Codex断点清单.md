## 2. 当前接口事实

1. `结构写入会话` 不可默认构造、复制或移动；公开窄入口只能创建主信息 / 节点候选、关系、主键绑定，执行四类同令牌读回并请求提交 / 撤销。
2. `结构写入执行器::执行` 每次取得一次独占许可，在当前线程栈内同步传递会话左值引用；失败、异常和无决定均在许可释放前撤销。
3. 会话只提供 `节点可读`，不返回节点类型；无需扩展会话。数据操作层先返回值式类型材料，独占会话内用同一完整版本句柄复核当前性即可闭合。
4. 关系结构化创建拒绝重复关系；索引绑定可区分首次提交、同键同节点幂等和冲突节点拒绝。
5. 当前 `世界服务::创建场景存在(场景)` 只接收已有场景，创建实际存在后接纳；原计划中的“可选场景候选”扩大了实际范围。
6. 既有 `存在服务`、`场景服务`、`世界服务` 仍为兼容实现，本轮没有生产调用迁移依据。

## 4. JY-319 裁决

1. 生产调用固定为 `业务服务 / 组合器 -> 数据操作.存在场景 -> #264 执行器`；只有数据操作模块可导入执行器。
2. `服务.存在.ixx`、`服务.场景.ixx` 分别导出 `存在业务服务`、`场景业务服务`，不重定义既有类。
3. 五条路径固定为实际存在、虚拟存在、场景、接纳已有存在、在已有场景中创建并接纳实际存在；不同时新建场景和存在。
4. 三类身份创建使用非零全局幂等主键。重复关系只有权威读回同一当前关系后才能映射为业务幂等成功。
5. #264 模块、四仓库、事务核心和既有三个服务头全部只读；第一轮只在隔离自检验证新模块，不接既有生产调用。
