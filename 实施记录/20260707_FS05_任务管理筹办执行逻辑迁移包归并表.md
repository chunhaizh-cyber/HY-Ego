## 1. 归并定位

本表把 FS-05 已扫描的旧任务管理、任务筹办、任务执行和任务管理线程事实从 `OF-128` 到 `OF-150` 归并为逻辑迁移包。

本表不确认旧函数体迁移，不新增代码实施切片，不新增旧任务主信息字段。旧函数编号只作为证据锚点；迁移判定单位是逻辑迁移包。

```text
函数 = 证据采集单位
逻辑能力 = 迁移判定单位
领域服务 / 结构闭环 = 代码实施单位
```

## 4. 包级迁移口径

### 第一轮可承接方向

建议确认后可进入后续计划的包：

```text
FS-05-B：任务承接材料与任务头创建。
FS-05-C：任务虚拟存在与任务过程账的读取增强。
FS-05-D：任务生命周期、目标状态与实际结果状态读取 / 记录增强。
FS-05-E：任务选择方法关系的后续读取和候选边界设计。
```

这些包已有任务系统、方法系统、需求目标状态和任务第一轮代码作为结构锚点；但仍必须另建专项计划和代码实施切片，不得直接改代码。

### 后续专项方向

建议转后续专项的包：

```text
FS-05-A：任务角色、阶段和节点种类语义分流。
FS-05-F：任务筹办回执与任务管理提交边界。
FS-05-G：派生需求、因果缺口和安全风险树候选。
FS-05-H：任务控制态、派发准入和运行控制材料。
FS-05-I：任务运行统计、权重和诊断快照。
FS-05-J：任务执行回执与实际结果证据。
FS-05-K：方法调用动作动态。
FS-05-L：运行包、治理协议和事件恢复材料。
```

这些包跨任务、方法、动作、动态、因果、事件日志和非权威统计，不应合并成一个胖切片。

### 排除方向

建议确认排除本轮迁移：

```text
FS-05-X：SQL / 控制面板 / 线程 / 外设排除。
```

旧 SQL 投影、控制面板、工作线程、`std::async`、外设观察报告、D455、体素和显示文本不得进入第一轮任务事实结构。

## 5. 必须删除或改写

```text
不迁移旧任务主信息中的节点种类、创建 / 启动 / 完成时间、对应需求、任务虚拟存在、场景、目标状态、实际结果状态字段。
不迁移旧任务树步骤节点 / 结果节点作为第一轮任务主线。
不迁移旧线程 ID、派发 bool、控制请求 bool、裸指针特征为权威任务事实。
不把任务运行次数、成功失败次数、错误码、界面任务状态、回执 bool 或日志文本作为任务完成、方法成功或需求满足事实。
不让任务筹办直接写需求树，不让线程成为动作来源。
```

## 6. 新增约束

```text
任务必须来源于有效需求。
任务目标状态来自需求目标状态。
任务虚拟存在由任务服务经存在服务创建或读取。
任务生命周期状态必须有发生时间戳。
任务选择方法关系是方法执行权限依据之一。
方法执行和动作动态必须复核动作入口角色节点，不能使用函数名、方法名文本、线程或日志标签作为动作来源。
任务统计和工作线程快照只能作为非权威缓存或诊断材料。
```
