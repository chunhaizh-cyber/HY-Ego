## 4. 新发现一：类型—索引投影不是端到端强类型

会话公开入口：

```text
海中鱼巣/核心/会话.结构写入.ixx:113—150
结构写入会话::读取有效节点与索引投影(节点类型)
```

外层结果区分 `已形成 / 入口拒绝 / 内部不一致`，但两个底层读取把值式复制异常折叠为空集合：

```text
海中鱼巣/核心/节点仓库.cpp:363—369
读取指定类型有效节点记录组：catch (...) -> return {}。

海中鱼巣/核心/索引仓库.cpp:281—285
读取全部主键绑定组：catch (...) -> return {}。
```

会话无法区分“真实没有节点 / 绑定”和“底层复制失败”，仍可能在 `会话.结构写入.ixx:148—149` 返回 `已形成`。这会低估有效节点 15、合法 O1 所有者声明数和 `所有者数 × 2048` 已保留字节，并使内部失败错误进入零记录 / 新建分支。

受影响验收项：A07、A09、A10。

## 5. 新发现二：会话无法证明主信息恰好 15 槽

当前会话只提供：

```text
海中鱼巣/核心/会话.结构写入.ixx:436—438
读取I64值(主信息, 槽位) -> optional<I64>。

海中鱼巣/核心/会话.结构写入.ixx:467—473
主信息可读(主信息) -> bool。
```

逐槽读取 0—14 可以证明预期值存在，却不能证明底层值容器长度恰好为 15，也不能发现槽 15 之后的额外非空值。主信息仓库已有完整值式记录读取，见 `海中鱼巣/核心/主信息仓库.h:105—108`，但 #290 禁止用途专用数据操作取得仓库引用或原始令牌。

因此当前接口不能同时证明：

```text
恰好 15 槽；
无额外非空槽；
请求提交前完整主信息读回；
专用数据操作不穿透仓库 / 原始令牌。
```

受影响验收项：A11。A06 的 15 槽设计表本身仍通过，不代表当前消费接口已经闭合。
