## 2. 当前代码事实

当前代码中身份编号已经使用 `std::uint64_t`：

```text
节点句柄::仓库编号
节点句柄::节点编号
主信息句柄::仓库编号
主信息句柄::主信息编号
关系句柄::仓库编号
关系句柄::关系编号
主信息记录::主信息编号
节点记录::节点编号
关系记录::关系编号
```

`std::int64_t` 当前主要用于：

```text
主信息记录::值容器
写入I64值
读取I64值
```

因此本轮不需要修改 C++ 类型。

## 3. 补充规则

已补充：

```text
编号使用 uint64 语义。
0 保留为无效编号。
有效编号从 1 开始。
同一仓库内编号只递增，不复用。
删除记录只改变状态和版本，不回收编号。
稳定身份由 仓库编号 + 本地编号 + 版本号 构成。
长期容量问题通过仓库编号、分片、空间编号、纪元编号、快照归档和冷热分层解决。
```
