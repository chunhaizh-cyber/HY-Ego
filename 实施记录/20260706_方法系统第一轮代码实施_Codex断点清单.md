# 方法系统第一轮代码实施 Codex 断点清单

日期：2026-07-06

## 1. 执行窗口规则

本窗口按用户新的协作模式执行：

```text
一个窗口负责设计和确认。
当前窗口只执行已经确认的代码实施步骤。
```

本切片不新增方法系统设计口径，只按已确认的 `方法系统规范` 和 `方法系统详细设计` 执行第一轮代码实施。

## 2. 输入入口

已确认依据：

```text
规范/方法系统规范.md
规范/详细设计/方法系统详细设计.md
```

第一轮代码切片范围：

```text
方法首角色节点。
方法虚拟存在关系。
方法条件角色节点创建并自动绑定方法首。
方法结果角色节点创建并自动绑定方法首。

方法结果场景读取。
方法服务继续保持不直接访问特征值服务。
```

## 3. 核心处理

本切片改造 `方法服务`：

```text
新增方法角色状态枚举。
新增创建方法首节点入口。
新增创建方法条件节点入口。
新增创建方法结果节点入口。
新增方法角色状态读取入口。
新增方法虚拟存在读取入口。
新增方法条件 / 结果节点集合读取入口。
新增方法条件 / 结果规格场景读取入口。
```

同时补齐关系仓库入口的完全重复关系拒绝：

```text
同一源节点、同一关系类型、同一目标节点的完全重复关系返回无效关系句柄。
```

该关系仓库变更只拒绝完全重复关系，不禁止已确认多目标语义下的不同目标集合。

## 4. 结构承载

代码修改：

```text
海中鱼巣/领域/方法服务.h
海中鱼巣/核心/关系仓库.cpp
海中鱼巣/入口.cpp
```

结构映射：

```text
方法首节点：节点类型::方法 + 方法首抽象状态 + 方法虚拟存在关系。
方法条件节点：节点类型::方法 + 方法条件抽象状态 + 条件规格场景 + 方法首到条件节点模板关系。
方法结果节点：节点类型::方法 + 方法结果抽象状态 + 结果规格场景 + 方法首到结果节点模板关系。
```

## 6. 入口前置拒绝

本切片验证的拒绝路径：

```text
无效方法首不能创建方法条件节点。
同一方法首到同一方法虚拟存在的完全重复引用关系被拒绝。
缺任务方法关系时方法执行仍被拒绝。
拒绝前后结构数量不变。
```
