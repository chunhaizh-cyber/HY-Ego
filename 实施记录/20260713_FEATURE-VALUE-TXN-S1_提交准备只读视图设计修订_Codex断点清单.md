## 3. 裁决

QR-181 采用推荐方向，但收紧为能力对象：

```text
业务回调完整读回并请求提交
-> 执行器确认当前线程、写入阶段、提交决定、无失败和尚未收口
-> 执行器构造不可默认构造 / 复制 / 移动的结构提交准备只读视图
-> 参与者只经视图复核本会话候选身份、候选节点类型、候选节点主信息和候选 I64 值
-> 参与者完成视图复核后才非阻塞尝试侧表独占锁
-> 会话确认
-> 参与者提交或精确撤销
```

明确不采用：

```text
不放宽可继续写入() 或既有普通读取入口。
不把参与者复核前移到业务回调。
不让执行器冻结或解释特征值领域 DTO。
不把可写会话、原始令牌、许可或仓库引用交给参与者。
```

## 4. 窄视图合同

视图只能由执行器在当前线程栈内同步构造和传递，只暴露：

```text
本会话主信息候选判断
本会话节点候选判断
候选节点类型读取
候选节点主信息锚点读取
候选主信息 I64 槽位读取
```

每次读取都必须复核会话仍为“写入中 + 提交决定 + 无失败 + 当前线程”。节点读取只接受本会话节点候选，I64 读取只接受本会话主信息候选。视图不读取任意已发布结构，不暴露关系读取、写入、确认、撤销或候选对象，不改变写集读回标记。
