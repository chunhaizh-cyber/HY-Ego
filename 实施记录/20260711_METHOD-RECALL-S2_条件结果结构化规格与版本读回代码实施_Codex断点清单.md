## 3. 实际实现

### 3.1 值式材料与固定槽位

新增：

```text
方法条件集合策略：未声明 / 全部满足 / 显式无条件
方法规格来源状态：无效 / 当前 / 已过期
方法条件集合策略材料
方法条件规格材料
方法结果规格材料
方法规格规则版本 1
```

固定结构槽位：

```text
方法首 --模板顺序 3--> 条件集合策略抽象状态
条件节点 --引用顺序 30--> 特征定义
条件节点 --引用顺序 31--> 期望特征值（最终发布关系）
结果节点 --引用顺序 32--> 目标特征
结果节点 --引用顺序 33--> 目标抽象状态（最终发布关系）
```

没有新增或改义节点类型、关系类型，也没有把文本、场景、动作键或日志当作规格事实。

### 3.2 条件与结果规格

`登记方法条件规格` 先验证方法仍活跃、条件归属完整、特征身份、期望值归属、策略未声明和两个固定槽位均为空，再通过特征服务冻结原始类型与原始版本。入口先写特征定义关系，最后写期望值关系并完整读回；只有最终关系存在时规格才可读。

`登记方法结果规格` 先验证方法仍活跃、结果归属完整、目标特征有效、目标状态值可读且目标不是实例状态，再先写目标特征关系、最后写目标状态关系并完整读回。

已发布规格不可原地改义，重复登记或任一规格槽位已占用均在写前拒绝。

### 3.3 条件集合策略

`登记方法条件集合策略` 只接受 `全部满足` 或 `显式无条件`：

```text
全部满足：至少存在一个可完整读回的条件规格。
显式无条件：方法首下没有任何条件节点。
未声明且空条件：读取返回空，保持不可判定。
```

策略发布后禁止再创建条件节点或登记条件规格，防止条件集合与已发布策略互相改义。策略由抽象状态和方法首固定模板关系承载，不使用普通 `bool` 裁决。

### 3.4 来源当前性与审计

条件规格材料冻结条件 / 场景 / 特征 / 期望值、两个完整关系句柄、原始类型、原始版本和规则版本；结果规格材料冻结结果 / 场景 / 目标特征 / 目标状态、两个完整关系句柄、状态值材料和规则版本。

`复核方法条件规格来源` 与 `复核方法结果规格来源` 重新读取当前权威结构：来源关系目标、关系版本或原始值版本变化时，旧材料返回 `已过期`；材料形状、规则版本或当前完整结构无效时返回 `无效`。失效方法不能新增规格，但既有策略和规格仍可通过登记项审计读回。

入口自测分别制造原始值版本推进和关系记录原地重挂，证明旧材料过期且当前读回采用新关系句柄。裸关系重挂只存在于 A11 测试材料，不是方法服务公开改义入口。

### 3.5 并发与诊断

方法服务使用 64 个按完整方法句柄散列的规格写锁。写入口锁序固定为：

```text
登记共享锁 -> 单方法规格条带锁
```

同一方法的条件节点、结果节点、规格和策略写入串行；不同方法在未发生条带碰撞时可并行。生命周期失效持登记独占锁，因此不能与同一时刻的规格发布交错。读取返回值式材料，内部异常先释放规格锁和登记锁，再调用公共 `追根因检查`，不在方法锁内写日志或弹窗。

## 4. 入口拒绝与非成功二分

逻辑内返回：

```text
方法未登记、已失效或生命周期不可读。
条件 / 结果节点不属于方法首，或场景结构不完整。
特征身份错误、期望值归属错误、原始材料不可读或冻结前后版本漂移。
结果目标是实例状态或状态值不可读。
规格槽位重复 / 部分占用，或策略与条件节点 / 条件规格矛盾。
全部满足没有完整条件规格；空条件保持未声明。
旧规格来源关系或原始版本正常推进后，旧材料返回已过期。
```

这些分支不新增规格结构。

追根因解决：

```text
前置通过后第一关系写入失败。
最终发布关系写入失败。
最终发布后规格或策略不能按预期完整读回。
策略登记时既有条件节点结构出现重复或归属异常。
```

追根因分支不把半结构解释为完整规格，也不通过扫描、日志或修复入口继续依赖路径。
