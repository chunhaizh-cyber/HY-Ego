## 3. 核心处理

```text
S1：需求服务新增需求身份复核入口，并保持需求目标状态读取为状态节点。
S2：需求服务新增需求主体、目标宿主、需求场景和统一需求承接材料读取入口。
S3：任务服务按需求创建任务前先经需求服务读取承接材料；缺主体、目标宿主、场景或目标状态时拒绝创建任务承接壳。
S4：默认入口复核需求结算权威记录仍来自结算状态、来源任务、目标状态、实际状态和动态证据；未新增需求结算摘要权威结构。
```

## 4. 结构承载

```text
需求主体：需求节点到存在节点的 引用 关系，按顺序号过滤。
目标宿主：需求节点到存在节点的 引用 关系，按顺序号过滤。
需求场景：需求节点到场景节点的 引用 关系，按顺序号过滤。
目标状态：需求节点到状态节点的 模板 关系。
任务承接：任务服务只在需求承接材料完整后写入任务节点、来源需求、任务虚拟存在、运行场景、目标状态和创建生命周期状态。
需求结算权威记录：仍由结算状态节点和结算相关关系承载，不新增摘要权威字段。
```
