## 3. 实际实现

### 3.1 值类型与公开入口

新增：

```text
事件段配置
事件段状态
事件段拒绝原因
事件段追加结果
事件段读取结果
事件段路径材料
事件日志段服务
```

公开入口：

```text
事件日志段服务::追加结构事件
事件日志段服务::读取事件段
事件日志段服务::读取段路径材料
```

追加和读取共用服务实例私有互斥锁。读取只返回 `事件材料` 值副本；目标不存在时返回空段候选且不创建目录或文件。

### 3.2 规范格式

```text
魔数：HYEVSEG1
格式版本：1
字节序：显式小端，标记 0x01020304
段头：64 字节，含段号、记录数、首末序号、总字节数和 FNV-1a 64 校验
事件帧：显式长度、负载版本、序号、时间戳、结构材料、UTF-8 字节和帧校验
上限：段 4 MiB、入口 512 UTF-8 字节、摘要 4096、帧 8192
```

读取先核对全部边界和校验，再返回完整事件组；任一损坏都返回空事件组，不修补、不跳帧、不返回有效前缀。

### 3.3 发布路径

```text
复核配置、路径、材料、时间戳、调用方序号、UTF-8、既有段和临时段
-> 服务锁内分配末序号 + 1
-> 生成完整候选段
-> CREATE_NEW 写同目录 .tmp
-> FlushFileBuffers + close
-> 临时段完整严格读回
-> MoveFileExW(REPLACE_EXISTING | WRITE_THROUGH)
-> 最终目标完整严格读回
-> 返回追加结果
```

目标被无删除共享方式占用时，实际替换失败被归为内部写入异常；服务复核旧目标原字节和严格读取仍成立，并保留临时候选供诊断，不宣称追加成功。

## 4. 入口拒绝与非成功二分

逻辑内返回：

```text
配置无效、路径越界、材料无效、时间戳 0、调用方预填序号。
UTF-8 无效、字段超限、段已满。
既有段未知版本、损坏、序号断裂或不可严格读取。
残留临时段冲突。
```

这些分支发生在目录创建和文件写入前，目标段原字节不变。

追根因解决：

```text
目录创建、临时段写入、flush / close、临时读回、同卷替换或发布后读回异常。
候选编码或成功结果与最终读回发生内部不一致。
```

内部异常在退出服务锁后进入公共容错检查；锁内不写日志、不弹窗、不调用领域服务。
