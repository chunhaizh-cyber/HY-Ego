## 2. 核心处理

1. 追加数值锁定的 `关系类型::概念生命周期 = 12`，由概念图服务独占解释和写入；同一概念只接受一个已登记生命周期状态目标。
2. 概念图初始化经状态服务创建并幂等登记活跃、冷却、退役三个互异抽象状态；四根和新发布普通概念在进入完整活动快照前均取得活跃关系。
3. 生命周期关系写后立即读回；候选构造失败先清理本轮专用边，再清理本轮生命周期边，最后逆序删除新增概念节点。四根初始化中途失败同样逆序失效本轮关系。
4. 语素服务通过 `语素概念追溯` 反向读取名称入口，按完整句柄排序去重；名称不进入概念签名或上下位裁决。
5. 待命名请求和临时指代只返回活动版本约束的值式材料；已有名称、非活动概念、过期版本、零上下文和零序号均逻辑内拒绝，不写任何仓库或缓存。

## 3. 结构承载

```text
当前生命周期事实：概念 -> 概念生命周期 -> 已登记抽象状态的唯一有效关系。
服务所有权登记：概念图服务会话期关系句柄登记，只用于防止外部关系被误认领，不替代权威关系。
概念名称：继续由语素入口 -> 语素概念追溯 -> 概念关系承载。
待命名请求：调用期 DTO，不是需求节点或名称事实。
临时指代：表达上下文编号、指代序号、概念句柄和活动图版本组成的调用期 DTO，不持久化。
```

## 5. 入口拒绝与失败收口

| 场景 | 分类 | 结果 |
| --- | --- | --- |
| 无效阶段、错误节点类型、实例状态或状态值不匹配 | 逻辑内返回 | 不登记状态，不改变结构 |
| 同阶段同状态重复登记 | 正常幂等 | 返回既有状态句柄 |
| 同阶段冲突状态 | 逻辑内返回 | 不覆盖既有登记 |
| 多生命周期目标、未登记状态目标或服务登记不一致 | 追根因解决 | 不选择第一个，不继续发布 |
| 生命周期关系写后读回异常 | 追根因解决 | 尝试立即删除本轮关系并停止依赖路径 |
| 四根生命周期绑定中途失败 | 追根因解决 | 逆序清理本轮新增关系，初始化保持未发布 |
| 候选创建后续失败 | 追根因解决 | 清理本轮专用关系、生命周期关系和新增节点，不替换活动快照 |
| 已有名称、非活动概念或过期临时指代版本 | 逻辑内返回 | 只返回空值，权威结构不变化 |
