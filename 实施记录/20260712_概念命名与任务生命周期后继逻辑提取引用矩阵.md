## 6. 失败二分

| 场景 | 分类 | 收口 |

| 协议 / 句柄 / 版本无效、候选为空、合法过期、队列满 / 停止、精确重复或正常当前性漂移 | 逻辑内返回 | 在对应写入前拒绝，不新增机器结构或回退权威事实 |

| 前置实施记录、实际函数、模块分类或阶段注册表与假定契约不一致 | 依赖门控返回 | 不改代码，退回设计窗口修订计划 |

每个切片执行前必须读取前置计划正式提交、实施记录、实际导出函数、工程登记和中央运行器注册表，逐项记录：

```text
输入句柄与版本字段
当前关系及其完整句柄 / 版本
正式写入方与只读方
模块 export / import 名称
阶段名称、顺序和回调签名

写后读回与完成材料
```

不得读取其它窗口未提交工作树作为完成证据。若接口漂移需要修改允许清单外文件、复制固定安全根夹具、把入口变成第二实现、让线程直接写机器事实或把普通消息 / bool / 日志升格为权威材料，必须停止并退回设计。
