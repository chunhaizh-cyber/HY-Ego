# 项目级编码规则入口前置拒绝更新记录

日期：2026-07-05

## 1. 输入入口

用户确认口径：

```text
本项目禁止使用校验逻辑，而是要在函数入口的地方就排除所有错误的可能。
所谓高级信息就是任务、方法、需求。
把以上要求落入项目规则中，整个项目编码时通用。
容错检查采用预编译宏的方式进行，并且报错时需要采用日志记录和弹窗提醒。
```

## 2. 核心处理

本次把以上口径提升为项目级通用编码规则，不再只作为语素系统局部规则。

更新规则：

```text
所有仓库、领域服务和业务写入口必须在函数入口排除错误输入。
不得先写入无效结构、悬空关系或错误状态，再靠校验函数、扫描器、日志或显示输出发现和修正。
构建、测试和人工验证只能证明入口拒绝是否生效，不得作为运行期机器结构正确性的兜底机制。
高级信息不是独立节点类型；当前只指需求、任务、方法等既有高级业务节点角色。
容错检查只能作为预编译宏控制的诊断层，默认关闭。
容错检查宏开启后，入口拒绝路径触发时必须同时日志记录和弹窗提醒。
容错检查日志和弹窗只做人读诊断，不得承载机器逻辑、不得修复结构、不得作为业务状态来源。
```
