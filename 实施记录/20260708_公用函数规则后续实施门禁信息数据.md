## 1. 定位

本文件不是计划，不进入 `计划/待确认计划/`，不登记可执行队列，不构成代码实施许可。

本文件只记录 `规范/公用函数规则规范.md` 形成后的后续按需门禁信息，用于未来某个已确认切片确实需要新增或调整公用函数时引用。

关键边界：

```text
规则迁移完成 ≠ 公用函数工具层已实现。
旧函数规则可迁移 ≠ 旧函数体可复制。
门禁信息数据 ≠ 待确认计划。
门禁信息数据 ≠ C++ 实施许可。
公用函数规则存在 ≠ 需要主动迁移公用函数。
公用函数规则存在 ≠ 需要主动建立公共工具层。
```

## 3. 后续按需门禁目录

以下门禁只在具体需求出现时触发，不作为独立迁移路线自动推进。

| 门禁 | 性质 | 触发条件 | 允许产物 | 禁止扩大 |
| --- | --- | --- | --- | --- |
| A0 按需查重 | 具体切片内前置检查 | 某个已确认服务矩阵、流程图迁移包、详细设计或代码实施切片提出新增 / 调整公用函数 | 查重和归属说明 | 不单独发起公用函数迁移，不改 C++ |

## 4. A0 按需查重信息

目标：

```text
在具体新增或调整公用函数需求出现时，检查当前新项目是否已有可复用入口。
避免重复新建公共工具层。
识别纯通用、仓库内部、领域服务、流程编排和禁止候选五类归属。
```

建议只读范围：

```text
海中鱼巣/核心/
海中鱼巣/领域/
海中鱼巣/入口.cpp
说明书/函数功能说明文档.md
规范/公用函数规则规范.md
规范/多线程防锁机制规范.md
规范/错误结果与入口拒绝规范.md
规范/领域服务授权写入规范.md
```

可选后续产物：

```text
具体切片的实施记录或待确认问题记录。
```

A0 通过标准：

```text
列出当前已有公共 / 辅助入口。
按纯通用、仓库内部、领域服务、流程编排、禁止候选五类归属。
标明是否已有同等功能入口。
标明是否存在重复、越权、命名或归属不清风险。
没有 C++ diff。
没有旧函数体复制。
```

## 5. A1 设计补齐信息

只有具体需求明确证明存在以下缺口之一，才需要生成设计：

```text
确有纯通用函数候选，但无工具层承载设计。
多个服务重复需要同一纯通用计算 / 格式化辅助。
仓库内部辅助、领域服务入口和纯通用工具边界不清。
已加锁辅助、只读快照或序列化恢复辅助缺少统一详细设计。
```

可选后续产物：

```text
规范/详细设计/公用函数工具层详细设计.md
实施记录/20260708_公用函数工具层详细设计生成_Codex断点清单.md
```

设计必须回答：

```text
工具层是否需要存在。
工具层只允许承载哪些纯通用函数。
工具层禁止接收哪些领域类型。
哪些辅助必须留在仓库内部。
哪些共享逻辑必须归属领域服务。
是否允许建立公共头文件；若允许，文件名、命名空间和中文命名规则是什么。
入口参数检测如何只做拒绝和来源追溯，不做内部兜底。
只读快照、已加锁辅助和序列化恢复辅助分别如何归属。
```

## 6. A2 代码实施候选信息

只有同时满足以下条件，才允许后续生成 C++ 实施切片候选：

```text
具体需求已确认。
按需查重和归属裁决已完成。
如需工具层，详细设计已生成并经用户确认。
待实现函数均已证明不是现有入口重复。
每个函数候选已有归属方、允许调用方、禁止调用方、读写类型、触碰结构、入口拒绝和结构不变化验收。
已明确允许文件、禁止文件和验证方式。
```

后续如果生成代码实施候选，候选文件必须位于：

```text
计划/待确认计划/
```

代码实施候选必须列明：

```text
允许文件。
禁止文件。
新增 / 修改函数清单。
每个函数的入口拒绝。
每个函数的结构不变化验收。
每个函数的调用方。
验证命令。
完成声明边界。
```
