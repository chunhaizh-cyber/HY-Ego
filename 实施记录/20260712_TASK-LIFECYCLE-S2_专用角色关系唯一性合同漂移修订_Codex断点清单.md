## 1. 漂移事实

```text
关系仓库普通创建唯一性按同关系类型 + 同源 + 同目标拒绝重复，不考虑顺序号。
来源任务与取消授权来源可以是同一任务。
来源方法与来源动作可以是同一方法节点。
JY-299 使用普通引用区分角色时，第二个合法角色会被仓库写前拒绝。
第二次草案曾达到 Debug x64 0/0、完整 self-test 退出 0 和 50 线程竞争，但严格语义复核不通过。
草案已完整撤回，上述运行结果不构成 #222 完成证据。
```

## 2. 裁决

采用专用关系，不采用角色目标强制互异：

```text
关系类型::任务生命周期证据 = 15
未知关系负例 = 16
允许角色顺序号 = 40..46
专用唯一键 = 关系类型 + 完整源状态句柄 + 顺序号
```

同一目标可在不同角色中重复；同一角色无论目标相同或不同都只能有一条有效关系。

## 3. 权力边界

| 层 | 责任 |
| --- | --- |
| 关系仓库 | 验证类型 15 的角色范围和同源同角色唯一；允许同目标跨角色 |
| 任务服务 | 验证源为本次迁移状态、角色目标类型、空 / 非空语义并按类型 + 顺序号读回 |
| 状态服务 | 只管理状态核心和幂等双槽，不解释证据关系 |
| 普通引用 | 保持现行同类型 + 同源 + 同目标重复拒绝，不参与专用角色承载 |
