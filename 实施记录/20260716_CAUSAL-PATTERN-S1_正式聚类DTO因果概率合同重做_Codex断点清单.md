## 2. 当前代码事实

1. `动态聚类结果` 保留全部去重单步实例、正式重复聚类和运动基元；成功结果不保证每个结构键都形成重复聚类。
2. `单步片段实例候选` 保留完整结构键和完整来源步骤；来源步骤携带完整回合、非零片段序号、发生时间戳、来源动态和完整来源批次。
3. `重复动态聚类候选` 只在同一完整结构键获得至少两个不同完整回合支持时形成，保留来源实例组和支持回合组。
4. 同一完整结构键在同一回合可以保留多个不同来源实例，但支持数只加一；因此 #234 不能假定每个支持回合天然只有一个原因实例。
5. `动态模式结构键相同` 和回合身份 / 锚点比较已经公开；结构哈希只分桶，不能裁决相等。
6. #233 容量上界公开为 `动态聚类最大步骤数量`，有效版本 1 输入的 Q10000 乘法可用 64 位计数安全承载。

## 3. JY-365 裁决

1. #234 只消费完整成功的 `动态聚类结果`、原因完整结构键、结果完整结构键和规则版本 1。
2. 原因必须精确命中一个正式重复聚类，确保机会来自至少两个不同完整回合。
3. 结果键允许在全部实例中零次出现，也不要求自身形成重复聚类；这是合法表达 Q10000 为 0 的必要条件。
4. 每个原因支持回合只计一次机会；同回合多原因实例使机会起点歧义，逻辑拒绝，不选择最早 / 最晚实例。
5. 结果只在同一完整回合与唯一原因比较片段序号。结果只在原因前出现时保留前置证据并计负结果，不拒绝整请求。
6. 原因后一个或多个结果实例都只贡献一次正结果，全部后续实例仍保留为证据。
7. 原因与结果片段序号相同则顺序不可判定，逻辑拒绝；不得使用时间戳或遍历顺序兜底。
8. 新增 `材料.因果模式`、`算法.因果模式`、`服务.因果模式`、`自检.因果模式` 四个真模块；DTO 只有材料模块一个声明所有者。
9. 全链纯值同步计算，不读仓库、不取得许可、不调用数据操作、不启动生产线程、不持久化、不接唯一运行期业务装配。
