## 4. 建议进入第一轮计划的逻辑包

建议第一轮只纳入：

```text
FS-09-A：日志输出与人读诊断入口。
FS-09-D：仓库级快照与字段恢复验证。
```

第一轮计划目标不是实现完整持久化系统，而是建立事件日志持久化恢复专项路线：

```text
复核当前事件日志文件入口。
确认事件日志只记录已由结构确认的事件材料。
确认第一轮优先文件事件日志和仓库快照格式，不直接接 SQL Server ADO。
确认旧链表二进制格式不作为新项目快照格式。
确认恢复入口必须先做版本、枚举、句柄和关系一致性拒绝。
```

## 5. 后续或排除逻辑包

以下不进入第一轮计划：

```text
FS-09-B：ADO 存储验证与 SQL 访问适配。
FS-09-C：SQL 控制面板运行态镜像排除边界。
FS-09-E：领域树快照和只读视图材料。
FS-09-F：特征值序列化恢复字段边界。
FS-09-X：线程事件与外设日志排除边界。
```

原因：

```text
SQL / ADO 当前只能作为后续持久化适配输入，不能裁决运行期事实。
控制面板事件流水不是动作动态，也不是权威事件日志。
领域树快照应优先由仓库快照或只读视图派生，不能恢复旧主信息字段。
特征值序列化依赖 Vec 原始值、哈希缓存和特征值系统后续专项。
线程、外设和体素材料必须转 FS-X 或后续外设专项。
```
