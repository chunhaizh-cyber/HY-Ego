## 2. 已形成但未发布的实现草案

本地草案在关系仓库私有结构中增加四类只保存关系编号的候选索引：

```text
完整源节点句柄 + 关系类型 -> 有效正向候选。
完整目标节点句柄 + 关系类型 -> 有效反向候选。
完整节点句柄 -> 有效节点相关候选。
完整源节点句柄 + 关系类型 -> 当前审计正向候选。
```

查询先取候选，再逐项回表复核完整关系句柄、状态、类型、端点、版本和顺序号。关系表继续是唯一权威；逻辑删除继续保留已删除权威记录，只从四类索引移除候选。旧带令牌删除继续验证共享路径令牌，严格删除继续验证独占令牌。

草案已覆盖创建、结构化创建、失效、结构化失效、普通 / 结构化重挂、普通父子挂载 / 重挂、逻辑删除、结构化确认 / 撤销和节点关系批量失效。Debug 专项还形成实际索引桶计数、权威记录诊断、全索引一致性检查和四个固定创建故障点。

这些内容只存在于执行工作树的 4 个未提交文件中，不是正式实现事实，不得由设计窗口或后继计划直接消费。

## 5. 退回原因

详细设计第 9 节固定：正确性是硬门槛；纯写 P95 不得高于 S0 的 150%；小规模稳定回归不满足时停止验收；D-G 不稳定满足时保留实现但不得宣称性能验收通过，必须重复基线或退回设计裁决。

执行窗口无权采用以下处理：

```text
放宽 150% 写入上限或 15% 三轮稳定门槛。
用优化后的测量覆盖 #257 正式 S0 基线。
让 A02-A06 绕过报告完整性，或把 A13 / A14 直接置为通过。
用 Release 结果替代固定 Debug x64 比较。
只提交查询收益而忽略写入回归。
登记 800、归档 #258 或释放 #259-#261。
```

设计窗口需要裁决：

```text
是否保持现有阈值并形成可满足写入门槛的索引结构 / 提交算法修订。
是否把优化后微秒级查询稳定性与 S0 历史报告完整性解耦，同时保留独立、不可弱化的写入回归门禁。
若固定阈值证明与四类索引同锁维护不可兼容，是否按正式“不实施 / 不发布”结果关闭 S1，而不是提交未通过实现。
修订后如何处置执行工作树中的 4 文件草案；未经重新发布不得直接提交。
```
