## 3. 逐签名复核

| #275 需要的能力 | `9f9f880` 正式接口 | 结论 |
| --- | --- | --- |
| 节点完整主键组 | `结构写入会话::读取节点主键组(节点句柄) const` | 一致；受 `可继续写入()` 约束，复用同一独占令牌 |
| 正向关系读取 | `结构写入会话::读取关系记录组(节点句柄, 关系类型) const` | 一致；返回完整关系记录值式组 |
| 反向关系读取 | `结构写入会话::读取来源关系记录组(节点句柄, 关系类型) const` | 一致；可复核根无上位、普通概念直接上位和反向支持 |
| 端点身份复核 | `读取节点类型`、`读取节点主信息`、`主信息可读`、`节点可读` | 一致；许可内重新读取，不信任许可外候选 |
| 关系创建与读回 | `创建关系`、`关系读回匹配` | 一致；关系句柄、类型、端点和顺序号进入会话写集 |
| 稳定键绑定复核 | `读取节点主键组`、`主键绑定匹配` | 一致；可区分写前准入材料和提交前精确读回 |

上述普通读取入口都要求当前线程、写入中、未作提交决定且没有首次失败。#275 不需要放宽会话门禁，也不需要修改四仓库、执行器或事务 ABI。

## 4. 根身份与因果边界

```text
存在根：由存在业务服务以非零幂等主键形成存在节点。
动态根：由动态业务服务创建抽象动态并返回值式材料。
关系根：由二次特征业务服务创建组合二次特征并返回值式材料。
因果根：由轻量因果业务服务基于完整抽象动态或实例动态形成；不存在无来源创建后门。
```

#275 只接收和复核上述已有身份，不登记根、不创建根节点、不新增根侧表。四根句柄和稳定键必须互异，节点类型必须映射到存在 / 动态 / 关系 / 因果对应类别，且根不能作为概念上下位下位端点。

## 5. 概念角色与首次挂载裁决

```text
根节点天然是概念。
普通概念只有沿概念上下位反向路径可达本次同类别已复核根后，才是已归根概念。
首次挂载只允许：已归根上位概念 -> 同类别、非根、且没有概念角色关系的全新下位候选。
关系提交前候选不是概念；提交后才成为可读概念。
实例支持概念只能指向已经归根的概念，不承担首次挂载。
```

首次挂载候选不得已有概念上下位上位 / 下位角色、实例支持概念目标角色或概念定义材料源角色。已归根普通概念可接受同类别多父；无根旧分量、跨类别分量、根作下位、自环和可达环均在第一笔写入前拒绝，不自动修复。

## 7. 故障验收合同

生产业务入口不接收故障参数。仅 `Debug|x64` 在 `数据操作.概念图结构` 编译固定写点故障自检入口：第 N 条关系写入并完成同令牌读回后，不请求提交并退出执行器回调，复用执行器既有无提交决定撤销路径。

每个写点独立运行、独立计数、独立数量快照；撤销后关系数量和三类正反向读取回前态。Release 的故障入口、故障字符串和 755 自检登记必须为 0。
