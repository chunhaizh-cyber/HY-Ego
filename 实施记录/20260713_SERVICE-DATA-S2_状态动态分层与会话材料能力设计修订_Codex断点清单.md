## 2. 当前事实复核

### 2.1 已实现事实

`#264` 会话已具有主信息 / 节点候选、关系、主键、结构读回、提交和精确撤销。`#265` 已证明只有服务专用数据操作模块导入执行器的五模块分层可以编译、运行并完成并发和失败回归。

### 2.2 真实缺口

```text
状态和动态需要写入主信息 I64 值、时间戳和幂等双槽。
当前会话不能写本会话主信息候选的 I64 值。
当前会话不能在同一许可内向数据操作层返回既有节点类型、主信息、I64 值和关系记录组。
已有状态写前无令牌读取不能替代独占许可内复核。
动态服务现有直接创建裸值材料状态路径不符合新分层边界。
```

因此直接实施 `#266` 会迫使数据操作层取得原始令牌或使用同线程无令牌重入路径，均不允许。

## 6. #266 固定功能路径

```text
1. 创建抽象状态。
2. 创建完整实例状态。
3. 创建抽象动态。
4. 使用两个已有完整实例状态记录状态变化动态。
5. 状态业务服务形成前后状态规格，同一会话创建两状态并记录实例动态。
```

实例状态必须有场景、主体、值和合法时间戳。实例动态必须有场景、主体、目标、完整前后状态和合法时间戳。前后状态主体一致、场景一致，前时间戳不得晚于后时间戳；动态时间戳取后状态时间戳。来源动作可空，非空时必须为当前动作入口角色。

## 7. 调用路径

```text
#272：
数据操作层 -> 执行器 -> 一次独占会话 -> 候选值 / 同许可材料 -> 提交或撤销

#266：
调用方 -> 状态业务服务 / 动态业务服务 / 状态动态组合器
-> 数据操作.状态动态

-> 四仓库
-> 值式业务结果
```

业务服务和组合器不暴露、接收、保存、传递或比较原始事务令牌。

## 8. 失败收口

```text
错误输入：第一笔写入前逻辑内返回。
候选值或结构写入不及预期：登记内部不一致，禁止提交。
无提交决定、显式撤销、失败或异常：许可释放前精确逆序撤销。
并发竞争：撤销后权威重读；完全同一事实才映射为幂等。
撤销后有半结构或数量增长：追根因停止，#266 和后继计划不得执行。
```
