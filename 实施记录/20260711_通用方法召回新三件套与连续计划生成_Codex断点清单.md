## 2. 当前事实结论

```text
1. 当前 `读取方法候选材料` 要求调用方先持有方法首，不是通用召回。
2. 来源任务、父方法、前置方法和后续方法已经有绑定 / 读取入口，旧图对此存在漂移。
3. 方法首没有权威登记根、枚举、生命周期和恢复集合。
4. 条件 / 结果及动作输入 / 输出只绑定场景壳，没有结构化规格。
5. 现有索引只支持稳定动作键到单动作入口，不能改义为一结果能力到多方法。
6. 任务承接与需求承接上下文可读，但无独立任务授权与通用当前态选择提交契约。
7. 特征服务已有实例槽位唯一当前值、原始版本和完整内容复核，可作为条件复判正式入口。
```

## 3. 本批裁决

```text
S1：方法服务建立稳定非名称登记根、根到方法首权威归属关系和活跃 / 失效生命周期；最终归属关系是方法首发布边界。
S2：条件规格第一版为特征定义 + 期望特征值；结果规格第一版为目标特征 + 目标抽象状态；规格写后不可原地改义。
S3：新增方法召回专用非权威派生多值索引，不扩展现有索引仓库，不改义稳定动作键。
S4：冻结任务 / 需求目标上下文，索引命中后完整重读登记、生命周期和结果规格，只输出候选。
S5：从需求目标宿主读取实例槽位，经特征服务完整比较条件期望值和当前值；缺失、未知或漂移不视为满足。
S6：语义键排序，完整句柄只做确定性排列；语义并列不自动授权。
S7：因任务授权与通用当前态尚无正式契约，先做更新后 S0，再生成任务服务独占提交三件套。
方法学习后置到任务结果、完成状态、需求结算、动态和轻量因果证据齐备。
```
