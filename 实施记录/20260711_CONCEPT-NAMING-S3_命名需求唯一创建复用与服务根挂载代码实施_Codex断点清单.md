## 1. 实际实施

1. `概念图服务` 与 `语素服务` 增加只读令牌重载；待命名复核在结构独占许可和目标概念分片锁之后，按 `活动图读锁 -> 概念图生命周期 -> 语素 -> 关系 / 节点仓库` 同令牌读取，不保存令牌且不改变普通无令牌入口。
2. `需求服务` 增加值式概念命名需求参数、状态和材料，接入可空结构事务接线；连接态入口只沿 `需求目标概念` 反向关系读取候选，区分零个创建、一个复用、多个追根因，并只把完整结算记录的已满足结果排除在未完成集合之外。
3. 新需求在同一独占许可内创建主信息、需求节点、主体 / 宿主 / 场景 / 目标状态 / 目标概念关系，预发布完整读回后挂到服务根；挂载后再次完整读回。任一步异常均按本轮父子关系、其它关系、节点、主信息逆序撤销。
4. 新增 `路由.概念命名治理` 与 `自检.概念命名需求` 两个真模块。路由许可前只调用通用消息值复核；入口只增加自检模块 import、最终回归 550 一次无捕获登记和总结果汇总。
