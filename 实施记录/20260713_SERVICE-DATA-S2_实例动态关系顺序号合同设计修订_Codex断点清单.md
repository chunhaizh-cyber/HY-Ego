## 2. 核心裁决

JY-323 采用当前代码已经由三组实例动态写入和 `读取动态材料` 共同使用的权威合同：

```text
动态 -> 被改变目标：因果来源，顺序号 1
动态 -> 前状态：因果来源，顺序号 2
动态 -> 后状态：因果来源，顺序号 3
动态 -> 来源动作：因果来源，顺序号 4，可选
```

`#266` 不迁移为 `0 / 1 / 2 / 4`，不修改既有动态读取，不新增兼容解释，也不承担历史结构迁移。JY-321 保留为接口漂移发现记录；QR-180 按本裁决关闭。

## 3. 来源动作同许可合同

来源动作为空时允许继续。非空时，状态动态数据操作层必须在第一笔写入前、同一独占许可内证明：

```text
节点类型 = 方法
模板关系顺序号 0 恰好一个当前状态目标，状态值 = 5
模板关系顺序号 1 恰好一个当前状态目标，状态值在 1..6
两条模板关系、两个状态节点及其主信息均以完整当前句柄读回
```

缺失、重复、目标类型错误、句柄过期或值域错误均属于入口拒绝或许可内版本漂移，第一笔写入前结构不变化。进入写入后关系、节点或值读回不及预期则进入追根因收口，不得静默兼容。

## 4. 结构承载

```text
关系仓库：继续承载实例动态目标、前状态、后状态和来源动作的权威因果来源关系。
节点仓库：继续承载动态、状态、方法节点的完整稳定句柄和节点类型。
主信息仓库：继续承载状态值、时间戳、动作入口角色值和动作入口状态值。
索引仓库：只提供幂等主键候选，不裁决关系角色或来源动作权限。
```

机器逻辑必须按完整关系句柄、顺序号、当前端点和状态值复核；日志、显示、缓存或旧位置回退不能替代权威结构。
