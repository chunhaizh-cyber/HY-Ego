## 3. 核心处理与结构承载

状态与动态继续使用现有主信息、节点、关系和索引结构：

```text
抽象状态：状态节点 + 主信息槽 0 状态值 + 主键索引
实例状态：状态节点 + 槽 0/1/2/3 + 场景运行期临时关系 + 主体 / 场景引用 + 主键索引
抽象动态：动态节点 + 主信息槽 0 动态材料 + 主键索引
实例动态：动态节点 + 主信息槽 1 后状态时间戳 + 场景运行期临时关系 + 主体 / 场景引用
实例动态因果来源：被改变目标 / 前状态 / 后状态 / 来源动作 = 1 / 2 / 3 / 4
```

来源动作允许为空。非空时，数据操作层在同一独占许可内复核方法节点、模板槽 0 唯一状态值 5、模板槽 1 唯一状态值 1..6，以及完整关系、状态节点和主信息句柄。新路径不写入或兼容解释 `0/1/2/4`。

组合路径最初按三个独立共享读取收口，在并发竞争中可能观察到前状态未提交而后状态和动态已提交的混合快照。实施中已在 `数据操作.状态动态` 内改为一次共享许可读取三个主键材料；写前和失败收口均以同一读取隔离判断完整三元事实。未修改仓库、会话、执行器或事务 ABI。

任一写点未完成同会话读回时不请求提交，由 #272 会话按索引、关系、节点候选、主信息候选逆序撤销。并发失败释放独占许可后再权威读取，只有完整同一事实才映射为幂等读回。
