日期：2026-07-15

状态：当前代码事实复核 / 基线 `23238dd`，其中生产代码基线为 `10a10be` / 绑定 #271、DQ-163 / 未改 C++

## 1. 复核目标

本矩阵用于回答三个问题：

1. 哪些入口已经属于新正式运行期业务面，必须保持零原始事务能力公开。
2. 哪些令牌、仓库和传统服务依赖仍是当前真实兼容所有者，不能批量删除。
3. 每类兼容入口由哪个后继计划替代，或为什么必须长期保留在核心内部。

本矩阵不是旧函数体迁移清单，也不是批量删除许可。函数只作为调用点证据；服务边界和结构闭环才是迁移与施工单位。

## 4. 入口拒绝与结构不变化

```text
新正式业务入口：
参数无效、句柄无效、类型不符、权限或前置关系不满足
-> 写入前逻辑内返回
-> 不改写参数
-> 不调用传统服务兜底
-> 结构与数量快照不变化

前置通过后进入结构承载：
执行器、会话、关系写集、读回或发布出现非预期
-> 追根因解决
-> 停止依赖路径
-> 不切回旧无令牌 / 带令牌兼容写
-> 无法证明精确回前态时停止并登记断点
```

#271 的边界自检只验证正式新路径和兼容基线，不给非法数据补默认值，也不在函数内部把非法输入转换成合法材料。

## 5. 删除 / 私有化资格

传统入口只有同时满足以下条件，才可在独立后继计划中裁决私有化、弃用或删除：

1. 所有生产调用点为零。
2. 对应自检已迁移到新服务 / 组合器或明确删除。
3. 新入口覆盖原有入口拒绝、幂等、失败收口和权威读回。
4. 快照、恢复、生命周期、统计、显示和历史兼容计划不再依赖。
5. 删除后没有反向 include / import / 构造依赖。
6. 有明确允许文件、禁止文件和 Debug / Release / 连续回归验收。

当前没有任何一组传统服务头同时满足上述六项。因此 #271 第一轮不删除生产兼容文件。
