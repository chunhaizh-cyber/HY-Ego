## 3. 核心处理与结构承载

1. `材料.用途观察` 唯一拥有 O1-V1 新类型：格式版本、完整任务句柄 + 非零任务序号语义身份、九个业务句柄 + 单一 `因果概率分量` 不可变核心、三种互斥阶段负载、固定形状候选，以及形成 / 同账比较两套独立强类型结果。
2. 格式版本外置于业务身份。候选不保存 `vector`、`string`、完整冻结请求、完整回执、完整结算 DTO、完整用途事件 DTO、机会证据组或待结算提示；历史概率只复用一个已有 `因果概率分量`。
3. `形成用途观察纯值候选` 先验证上游状态—原因—optional 组合，再验证目标格式；五类合法入口拒绝同义映射，规则版本及未知非零格式进入不支持，非法上游组合进入内部不一致。
4. 身份按事件、冻结请求、回执内嵌请求和回执四组来源互证完整任务句柄与非零任务序号；内容按需求、选择记录、方法、动作、场景、主体、目标状态、实际状态、动态证据逐项互证。
5. 内容复核还重新形成 `因果完整结构键` 与 `因果概率分量`，逐项比较机会数、正 / 负结果数和 Q10000；机会数必须非零，正负之和必须等于机会数，且 `Q10000 == 正结果数 * 10000 / 机会数`。
6. 目标裁决同时互证需求、目标状态、实际状态，以及实际状态材料的场景、主体和状态节点，防止只凭事件外层句柄形成候选。
7. 尝试未达成、待结算和最终已结算三种阶段严格互斥。前两种拒绝正式结算与最近结算状态；最终阶段深检正式结算状态、幂等主键、发生时间戳、已满足值、需求、来源任务、目标 / 实际状态、动态证据、场景、主体、可选因果引用与兼容 `最近结算状态`。
8. 构造候选后再次逐字段对照已接受来源；任一不变量破坏返回内部不一致且不输出半候选。
9. `比较用途观察同账` 固定按格式 0、未知格式、两个候选自身合法性、完整身份、不可变核心、阶段负载的顺序执行。完整身份不同只正常分流；同身份同核心同阶段幂等；待结算到合法最终是唯一允许推进；异核心、终态变化、逆向或其它转换均为同账冲突。
10. `自检.用途观察` 独立承载 A01-A16 正反例与固定形状检查，不进入生产依赖方向；入口没有新增夹具、场景正文、循环或逐项输出正文。

## 4. 非成功收口

- 上游合法入口拒绝状态—原因—无事件，映射为同义形成入口拒绝且无候选；目标格式 0 同样入口拒绝。
- 上游合法规则版本不支持或成功上游下非零未知格式，返回不支持，不猜测兼容。
- 上游非法状态—原因—optional、身份 / 内容 / 阶段矛盾、构造后不变量破坏均返回内部不一致，不静默丢弃、不填默认候选。
- 新旧候选格式 0 或自身非法均在身份比较前收口为对应候选非法；非零未知格式只返回不支持。
- 两个合法候选完整身份不同不是冲突；没有既有候选时不调用比较入口，也不伪造“不同身份”。
- 同身份合法候选异核心、终态变化、逆向或其它不允许转换只返回同账冲突，结构变化为零。

提交前只读合同复核发现 `算法.用途观察` 曾为显式命名 `需求正式结算材料` 而直接导入 `数据操作.需求任务方法`；该数据操作模块封闭四仓库并导入结构写入执行器，违反 A15 的生产模块零仓库 / 结构事务依赖。修正限定在同一允许算法文件：删除该直接 import，把因果引用一致性帮助函数改为不显式命名嵌套结算类型的窄模板，只从 `用途事件结果` 已暴露的正式结算对象做只读字段深检。修正不改变公开签名、形成 / 比较语义或候选形状；随后重新执行完整 Rebuild 与全部验证。
