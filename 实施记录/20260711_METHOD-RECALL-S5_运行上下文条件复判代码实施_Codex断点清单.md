## 2. 实施内容

### 2.1 值式复判材料

在 `海中鱼巣/领域/方法召回服务.h` 新增：

```text
条件复判结论
条件复判来源状态
条件项复判材料
方法条件复判材料
复判候选条件
复核条件复判来源
```

复判结论固定区分全部满足、明确不满足、不可判定、材料过期和内部不一致。材料只冻结调用期候选、策略、完整条件规格、实例槽位、当前特征值、当前原始类型 / 版本，以及尚无完整规格的条件节点；全部标记为非权威，不写业务状态。

### 2.2 策略与条件复判

策略分流为：

```text
显式无条件：确认没有条件节点，不读取特征槽位，返回全部满足。
未声明：返回不可判定。
全部满足：至少有一个条件节点，逐项复判；尚无完整规格的节点显式保留并使整体不可判定。
```

每个完整条件项执行：

```text
复核条件规格来源
-> 读取目标宿主对应实例槽位
-> 读取唯一当前特征值
-> 读取期望值和当前值原始版本 / 类型
-> 同类型时经特征服务完整比较
-> 再次复核规格、槽位、当前值和两侧版本
```

明确不等使全部满足策略整体明确不满足；缺槽位、无当前值、无原始材料、类型不同或未形成完整条件规格返回不可判定；任一正常来源变化返回材料过期；当前版本保持却出现比较或结构矛盾时进入公共追根因诊断。

### 2.3 来源复核与并发边界

`复核条件复判来源` 重新检查任务 / 需求冻结请求、方法登记与生命周期、结果规格、条件策略、完整条件节点集合、条件规格、期望值版本、实例槽位、唯一当前值和当前值版本。已发布复判材料可由 #208 在使用前区分当前、已过期或无效。

不同候选可由调用方现有工作线程并行复判；服务内部不创建 `thread`、`async` 或临时线程池。条件项与不可读条件节点分别保持稳定顺序，结构交叉检查和来源节点合并采用线性双指针路径，不让校验退化为条件数平方级扫描。
