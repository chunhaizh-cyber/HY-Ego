## 2. 输入入口与公开结果

结构写入执行器新增通用单参与者生产入口：

```text
执行(业务回调, 事务参与者)
```

业务回调仍只接收不透明 `结构写入会话&`。参与者只通过通用生命周期与执行器交互；生产入口不接收故障参数，不导出原始令牌、许可、仓库引用、候选、确认或撤销能力。

仅在 `HY_EGO_ENABLE_STRUCTURE_COMMIT_FAULT_SELF_TEST` 下编译固定自检入口：

```text
执行确认前故障自检(业务回调, 事务参与者, 无参观察回调)
```

该入口只用于 Debug|x64 隔离自检。Release 不编译此重载，入口程序也不登记阶段 735 隔离自检。

## 3. 核心处理与结构承载

新增提交准备只读视图，由执行器在当前线程、会话已请求提交、无失败且尚未收口时构造。视图不可默认构造、复制或移动，只允许读取本会话新建候选的身份、节点类型、节点主信息和候选 I64 值；既有 `可继续写入()` 和决定后普通读取拒绝语义未放宽。

特征值原始材料参与者按以下结构承载初始值：

```text
I64：四仓库承载特征值身份和主信息值；特征值服务 I64 版本侧表承载版本 1
VecI64 / VecU64：四仓库承载特征值身份；特征值服务 Vec 侧表承载值和版本 1
```

参与者先使用窄视图复核候选，再以非阻塞独占尝试准备侧表记录。普通提交先确认会话四仓库写集，再完成参与者提交；许可竞争、准备拒绝、无决定、显式撤销或异常均先精确撤销参与者写集，再撤销会话写集。共享访问器使用非阻塞共享尝试并返回值式副本，不跨多次读取拼接快照。

固定确认前故障点位于真实参与者准备成功后、第一笔仓库确认前。故障命中后执行器先精确撤销 I64 版本或 Vec 记录，再撤销全部未确认会话写集并释放许可；I64 与 Vec 使用两次独立执行、独立命中计数和独立前后快照。

参与者合同保留在执行器模块内。实施中 MSVC 对跨模块固定重载生成了不一致的模块修饰名，最终把参与者入口和辅助调用改为模板，在不改变生产二参入口、参与者生命周期或禁止文件的前提下消除链接漂移。

## 4. 输出变化与调用路径

新增两个 `ClCompile` 真模块：

```text

海中鱼巣/领域/自检.特征值原始材料事务.ixx
```

生产调用路径：

```text
后续服务专用数据操作层
-> 结构写入执行器二参入口
-> 业务回调通过不透明会话建立候选并请求提交
-> 执行器构造提交准备只读视图
-> 特征值参与者复核候选并非阻塞准备侧表
-> 会话确认四仓库写集
-> 参与者完成提交
-> 值式结构写入结果
```

本切片只在隔离自检调用新生产能力，尚未接入 #267 的特征体系业务路径。`入口.cpp` 只在 Debug|x64 宏内增加自检模块导入、一个值式结果、一次 735 登记和总汇总。
