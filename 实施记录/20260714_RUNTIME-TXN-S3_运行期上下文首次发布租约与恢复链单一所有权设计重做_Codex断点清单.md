## 4. 结构承载与失败边界

1. 发布提交点只有唯一宿主内 `当前上下文_` 的 `shared_ptr` 写入；发布入口不写四仓库或领域侧表。
2. 空候选、结构不完整、装配不完整、系统角色材料缺失、已有上下文和并发落败均为逻辑内返回，宿主不变化。
3. 发布锁内提交前再次复核候选完整性；复核失败不写发布槽。
4. 前置通过后宿主、租约和候选身份不一致属于追根因解决，停止后继消费者。
5. 不新增宿主清空、在线替换、第二宿主、第二租约、第二协调器或恢复专用发布槽。
