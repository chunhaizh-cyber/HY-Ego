## 3. 核心处理与结构承载

`存在场景数据操作` 私有持有结构事务接线、四仓库引用、关系仓库编号和 #264 执行器。值式只读材料在数据操作层内部取得一次共享许可完成；业务服务和组合器不接收或传递原始令牌。写入只经执行器同步回调和不透明会话：

```text
实际存在：主信息候选 -> 存在节点候选 -> 主键绑定 -> 完整读回 -> 提交
虚拟存在：来源当前性复核 -> 身份候选 -> 主键绑定 -> 来源引用最后写入 -> 完整读回 -> 提交
场景：主信息候选 -> 场景节点候选 -> 主键绑定 -> 完整读回 -> 提交
接纳：场景 / 存在当前性复核 -> 归属关系 -> 关系读回 -> 提交
组合：已有场景复核 -> 创建缺失实际存在身份和 / 或归属 -> 完整读回 -> 提交
```

主键竞争或关系竞争先由 #264 在许可释放前精确逆序收口；数据操作层随后重新取得共享读取，只有权威读回同一当前身份或同一当前归属关系才映射为幂等成功。业务准入后出现无法解释的结构拒绝或内部不一致走公共追根因诊断，不降级为普通入口拒绝。

结构事实仍只承载在主信息、节点、关系和索引仓库；未新增显示字段、候选类型、仓库状态或 SQL 投影。
