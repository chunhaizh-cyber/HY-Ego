## 2. 固定样例契约

### 2.1 单组身份和顺序

后续 REAL-LOOP-S2 至 S6 必须沿同一组符号槽位传递，不得在中途替换为另一组需求、任务、方法或动态：

~~~text
样例自我存在
-> 样例现实场景
-> 样例目标特征和值
-> 样例目标状态需求
-> 样例需求承接任务
-> 样例固定方法和动作入口
-> 样例任务选择方法关系
-> 样例动作请求材料
-> 样例特征状态写入
-> 样例改变前后状态、动态和轻量因果引用
-> 样例任务实际结果和完成状态
-> 样例需求实际状态和结算状态
-> 非权威缓存、事件日志、显示与运行消息观察
~~~

时间材料使用符号槽位，要求：

~~~text
样例任务创建时间戳 != 0
样例动态发生时间戳 != 0
样例任务完成时间戳 != 0
样例需求结算时间戳 != 0
样例任务创建时间戳 < 样例动态发生时间戳
样例动态发生时间戳 <= 样例任务完成时间戳
样例任务完成时间戳 <= 样例需求结算时间戳
~~~

所有节点槽位都代表运行时读回的稳定句柄，必须同时复核仓库编号、本地编号和版本号。本文不分配任何节点编号、关系编号、索引键或消息编号。

| 符号槽位 | 当前来源入口 | 角色和约束 |
| --- | --- | --- |

| 样例需求结算状态句柄 | 需求服务::记录需求结算 | 需求结算实例状态；来源任务、目标状态、实际状态和动态证据可读 |

## 3. 权威性分层

| 分层 | 当前材料 | 裁决边界 |
| --- | --- | --- |
| 权威结构 | 需求、任务、方法、特征值、状态、动态、因果引用及其正式关系和索引 | 只能由对应领域服务写入和读回 |
| 请求 / 候选材料 | 动作执行桥请求材料、因果准入动态材料 | 只证明材料可进入后续正式入口，不自动形成执行、完成或满足事实 |
| 运行消息 / 回执 | 有界运行消息队列中的请求、结果回执、停止和观察消息 | 承载机器事实必须为 false；不得裁决动作发生、任务完成或需求满足 |
| 非权威缓存 | 统计服务缓存命名空间、结构统计和二级观察材料 | 可丢失、可清空、可重建；缺失不改变权威结构 |
| 人读审计 | 结构事件材料和事件日志文件 | 写入失败不改变机器事实，不直接恢复运行期结构 |
| 只读显示 | 显示服务生成的显示材料 | 只组装人读材料；禁止写业务事实和文本裁决 |

## 5. 逻辑内返回与追根因解决

### 5.1 逻辑内返回

以下情况在写入口前被拒绝，或只返回空请求 / 候选 / 人读材料，不产生可读半结构：

~~~text
无效句柄或类型错误。
关系缺失、重复唯一关系、稳定键冲突。
时间戳缺失、句柄版本过期、同任务序号倒退。
动作请求材料缺少任务方法关系或主动动作入口。
缓存命名空间不匹配、缓存缺失或日志 I/O 失败。
显示请求越权、请求写事实、文本裁决或请求外部显示通道。
消息重复、满载、停止态或声明承载机器事实。
~~~

后续施工必须在调用前后读取结构数量或正式读回入口，证明这些拒绝没有改变节点、主信息、关系、索引、状态、动态、任务结果或结算事实。

### 5.2 追根因解决

前置条件已经满足并进入创建、绑定、写关系、写状态、记录动态、记录任务结果、结算或正式读回后，任何内部结果不及预期都不是普通失败返回。后续切片必须停止依赖路径，并逐入口选择：

~~~text
事务回滚；
失效隔离并确保公开入口不可读；
结构数量快照加正式读取拒绝证明；
定位并排除逻辑错误后重新验收。
~~~

不得依靠日志、显示、缓存或事后扫描修复半结构。
