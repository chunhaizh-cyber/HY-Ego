# 流程图中途异常口径统计表

日期：2026-07-09

## 1. 二分口径

本表按用户修订后的二分口径统计流程图中的中途返回：

```text
类型一：需要通过追根因来解决的问题。
类型二：属于逻辑内返回。
```

判定标准：

```text
追根因解决：前置条件已经满足，流程进入创建、绑定、写关系、写状态、记录动态、生成结算、读回验证或结构承载后，结果不符合内部预期。
逻辑内返回：该条件本来就是领域协议允许的分支，返回前不产生可读半结构，或返回的是明确的请求材料、候选为空、拒绝材料或人读材料。
```

硬边界：

```text
只要发生在结构写入之后且结果不符合预期，就不能叫逻辑内返回。
只要条件本身是设计内允许返回，且结构不变化或只返回非权威材料，就归为逻辑内返回。
```

## 4. 同步修订路线

后续修订不得只改流程图文字，必须按同一口径同步检查并修正：

```text
流程图：节点命名和分支含义。
详细设计：失败路径、入口拒绝、读回验证、半结构隔离和完成声明边界。
代码：实际返回路径、结构写入顺序、读回验证、逻辑错误处理和验收输出。
```

## 5. 完成声明边界

```text
输入入口：流程图目录下 31 份 Markdown 流程图。
核心处理：按“追根因解决 / 逻辑内返回”二分口径重新归类。
结构承载：本实施记录表。
输出变化：修订统计表；未修改流程图、详细设计或代码正文。
调用路径：设计确认窗口只读审查。
验证方式：人工逐份核查 `流程图/*.md` 异常语义命中点。
```

本表不证明流程图已修订，不证明详细设计已修订，不证明代码已修改，不证明构建通过，不证明旧项目能力迁移完成。
