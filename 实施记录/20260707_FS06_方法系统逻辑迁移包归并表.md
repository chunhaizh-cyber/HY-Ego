## 1. 归并定位

```text
函数 = 证据采集单位
逻辑能力 = 迁移判定单位

```

## 4. 包级迁移口径

### 第一轮可承接方向

建议确认后可进入后续计划的包：

```text
FS-06-A：方法角色身份与角色读取。
FS-06-C：方法条件 / 结果节点与规格场景。
FS-06-D：方法虚拟存在与方法能力基底。
```

这些包已有方法系统第一轮代码作为结构锚点；后续计划应以复核、读取增强和拒绝路径为主，不直接扩大为完整方法候选或方法执行。

### 后续专项方向

建议转后续专项的包：

```text
FS-06-B：方法首注册和稳定读取。
FS-06-E：方法动作入口与方法调用动作动态。
FS-06-F：结果能力、候选召回和结果方向。
FS-06-G：来源任务、父方法、前置 / 后续方法关系。
FS-06-H：方法结构状态、可用性和条件结果对。
FS-06-I：方法统计、练习样本和非权威缓存。
```

这些包跨方法、动作、动态、任务、二次特征和非权威缓存，不应合成一个代码切片。

### 排除方向

建议确认排除本轮迁移：

```text
FS-06-X：SQL / 控制面板 / 旧主信息字段 / 本能兼容排除。
```

旧动作名、函数名、本能函数 ID、旧动作句柄字段、SQL 投影和控制面板显示文本不得作为动作身份或方法事实。

## 5. 必须删除或改写

```text
不迁移旧方法主信息 variant 字段和方法节点种类裸字段。
不迁移旧动作句柄字段、动作名文本、本能函数 ID 作为动作身份。
不把成熟度、允许自动查找、可被查找命中、根写入原语等 bool / I64 裸字段作为权威事实。
不把指针特征、SQL 控制面板投影、日志文本作为方法事实或动作来源。
不直接迁移条件结果对归并函数体或方法学习统计。
```

## 6. 新增约束

```text
方法角色由抽象状态和关系承载。
方法虚拟存在必须经存在服务创建，并通过方法虚拟存在关系绑定到方法节点。
条件 / 结果节点创建时必须绑定有效方法首。
动作身份必须走动作入口角色节点和稳定动作键。
结果能力召回只生成候选，不能直接裁决方法可执行。
统计和练习样本只能进入非权威缓存或后续学习材料。
```

## 7. 建议确认口径

建议确认：

```text
FS-06-A、FS-06-C、FS-06-D 可进入后续方法服务角色 / 条件结果 / 虚拟存在增强专项计划。
FS-06-B、FS-06-E、FS-06-F、FS-06-G、FS-06-H、FS-06-I 转后续专项。
FS-06-X 排除本轮迁移。
```

确认后下一步建议：

```text
优先生成 FS-06 方法服务角色与条件结果读取增强专项计划。
计划范围只覆盖角色读取复核、条件 / 结果节点读取、方法虚拟存在读取和拒绝路径。
方法候选召回、动作动态、前后方法关系、条件结果对和统计缓存不得进入第一轮代码实施。
```
