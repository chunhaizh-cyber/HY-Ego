## 3. 具名接口漂移

### 3.1 复现

为满足“仅 `需求任务方法数据操作` 可取得私有特征批次参与包”，曾在 `数据操作.特征体系.ixx` 的命名模块内前置声明该类并登记 friend。MSVC 将该声明附着为：

```text
需求任务方法数据操作@海中鱼巣.领域.数据操作.特征体系
```

实际定义附着于：

```text
需求任务方法数据操作@海中鱼巣.领域.数据操作.需求任务方法
```

二者是不同实体。第二次 Debug Rebuild 编译阶段通过，但链接阶段产生 13 个错误，包括 `需求业务服务`、`任务业务服务`、`方法业务服务` 构造函数对错误模块实体的未解析引用，最终 `LNK1120`。

### 3.2 裁决

当前允许范围内无法同时满足：

```text
严格只有真实的需求任务方法数据操作可取得私有参与包
+ C++20 命名模块实体唯一
+ 不修改需求任务方法数据操作
+ 不新增双方共同 import 的桥模块 / 模块分区
+ 不修改工程登记
```

把前置声明移入 global module fragment 不能与另一命名模块附着的实际定义合并；由特征模块直接 import 需求任务方法模块会在后继消费时形成循环导入。模板 / 公开工厂只能放宽为非具名调用方，不符合固定合同。

因此命中计划的接口漂移停止条件。已删除错误前置声明和两处跨模块 friend，只保留参与包由特征数据操作自身私有使用的最小可编译状态；不得把该降级解释为 DD05F0-A07 已通过。

### 3.3 设计窗口需要裁决

需要由设计窗口二选一并修订计划、设计、允许范围和工程边界：

1. 新建双方共同 import 的内部授权桥模块 / 模块分区，并登记工程；或
2. 允许修改 `数据操作.需求任务方法.ixx`，由真实模块实体承载授权适配器，同时给出无循环依赖的明确调用方向。

执行窗口不自行选择、不修改禁止文件，也不扩大计划。
