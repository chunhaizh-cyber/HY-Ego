## 2. 核心处理

```text
1. 完整入口不再硬拒绝“主体 == 目标宿主”，改为主体 / 目标宿主角色已明确；是否禁止同一节点承担两个角色，后续由需求规范确认。
2. “只创建需求身份”路径补充主信息创建失败和需求节点创建失败收口。
3. 第一轮简化目标状态需求入口标注为兼容 / 验证入口，不提供完整任务承接材料。
4. 将图中“当前特征值”口径收束为“当前特征状态材料”，并明确必须经特征服务读取，需求服务不得直接依赖特征值服务。
5. 失败收口覆盖本次创建的目标状态、需求节点和关系，不得留下可读半结构。
6. 最小读回验证按路径拆分：需求身份路径、简化入口路径、完整入口路径分别读回对应结构。
7. 将“结算目标状态需求”改为“目标状态满足预判 / 结算预览”，明确不写持久化结算记录。
```
