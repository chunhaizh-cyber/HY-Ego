## 3. 核心处理

```text
S1：任务服务新增统一任务承接材料读取入口，复核来源需求、虚拟存在、运行场景、目标状态和创建状态。
S2：任务虚拟存在、创建状态和完成状态继续通过现有关系和状态服务读取，未新增任务过程账锚点。
S3：任务实际结果状态由任务到状态节点的引用关系承载；记录任务完成状态前必须已有实际结果状态，缺失时入口拒绝。
S4：任务选择方法关系由任务服务封装为选择 / 读取入口；第一轮继续使用引用关系，不实现候选集合、当前方法集合或完整调度。
```

## 4. 结构承载

```text
任务承接材料：任务节点读取来源需求、任务虚拟存在、运行场景、目标状态和创建生命周期状态。
任务实际结果状态：任务节点到实例状态节点的 引用 关系，使用任务服务内部顺序号区分。
任务完成状态：通过状态服务创建实例状态，并由任务生命周期状态读取入口读取。
任务选择方法：任务节点到方法节点的 引用 关系，使用任务服务内部顺序号区分。
拒绝路径：重复任务方法选择、缺实际结果完成、重复实际结果状态均拒绝，拒绝后节点 / 关系 / 索引数量不变。
```
