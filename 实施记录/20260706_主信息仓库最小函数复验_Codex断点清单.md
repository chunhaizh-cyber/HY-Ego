# 主信息仓库最小函数复验 Codex 断点清单

日期：2026-07-06
状态：已实现 / 已验证 / 未迁移旧字段

## 1. 输入入口

计划索引 HR-001 当前下一步为：

```text
进入 S1 主信息仓库最小函数复验。
复验前不得迁移旧函数体。
```

## 2. 本次范围

允许：

```text
复验主信息仓库当前最小函数。
补默认入口的人读复验输出。
修复复验中发现的当前代码缺口。
更新主信息详细设计、计划索引、项目记忆和验收记录。
```

禁止：

```text
新增旧项目主信息类。
新增旧项目主信息字段。
迁移旧项目函数体。
把主信息值容器扩展为需求 / 任务 / 方法通用值容器。
```

## 3. 复验覆盖

```text
创建主信息返回有效句柄。
读取有效主信息成功。
主信息是否有效返回 true。
单槽 I64 可写入和读取。
指定索引 I64 可写入和读取。
未写指定索引返回空。
删除主信息后旧句柄失效。
删除主信息后编号不复用。
无效主信息句柄和已删除主信息句柄不能写入 I64 值。
```

## 4. 发现并修复的问题

复验第一次运行时发现：

```text
写入值索引 2 会扩容值容器。
旧实现使用 std::vector<std::int64_t>。
中间索引 1 被默认填 0，导致读取未写槽返回 0 而不是空。
```

修复：

```text
主信息记录::值容器 改为 std::vector<std::optional<std::int64_t>>。
写入指定槽位时只给目标槽位赋值。
读取指定槽位时，槽位不存在或 optional 为空均返回空。
```
