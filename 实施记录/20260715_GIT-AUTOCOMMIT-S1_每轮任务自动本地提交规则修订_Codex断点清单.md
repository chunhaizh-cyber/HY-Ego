## 1. 用户口径

```text
每完成一轮任务就自动提交。
```

本项目把“提交”解释为本地 `git commit`，不自动扩大为 `git push`。用户明确说“推送”或“同步到云端”，或者目标计划、队列项、正式交互协议明确授权推送时，才执行远端推送。

## 2. 规则裁决

1. 每轮有文件改动的任务在完成授权范围并通过要求的验证后，自动形成一次本地提交，不再等待额外提交指令。
2. 自动提交适用于代码、文档、规则、计划、落盘扫描和正式退回治理，不再只限“计划切片”。
3. 无文件变化不创建空提交；任务未完成、验证失败、范围不明或暂存无法与无关改动分离时不得提前提交。
4. 自动提交只暂存本轮任务文件，必须先核对分支、dirty state、staged diff 和差异检查。
5. 推送不是自动提交的一部分。正式跨窗口交接需要远端可见时，必须先取得明确推送授权并推送成功，再发送正式交接消息。

## 5. 边界

本规则只改变 Git 任务收尾授权，不改变计划入队、代码允许文件、验证门禁、跨窗口所有权或能力完成声明。自动提交不代表自动推送，也不代表任务验证、生产发布或云端同步完成。
