## 2. 输入入口与公开结果

关系仓库新增不可默认构造、不可复制、可移动且析构不写入的 `已发布关系变更能力`。能力由关系仓库私有构造，绑定仓库编号、完整独占令牌、写前完整记录、写后完整记录、变更种类和能力阶段。

结构写入会话新增五个不暴露原始事务能力的值式入口：

```cpp
带值结构写入结果<关系句柄> 失效已发布关系(关系句柄 当前关系);
带值结构写入结果<关系句柄> 重挂已发布关系(
    关系句柄 当前关系, 节点句柄 新源节点, 节点句柄 新目标节点);
节点挂载结果 挂载或重挂节点(节点句柄 节点, 节点句柄 新父节点);
std::optional<关系记录> 读取关系审计(关系句柄 当前关系) const;
std::vector<关系记录> 读取关系审计记录组(
    节点句柄 源节点, 关系类型 类型) const;
```

公开会话面不接收、不返回原始令牌、许可、仓库、锁或可变记录。通用 `创建关系` 和通用已发布关系重挂均拒绝 `普通父子`，普通父子只能通过单父和防环专用入口处理。

## 3. 核心处理与结构承载

关系表继续是权威结构。关系仓库新增独占结构化失效、结构化重挂、普通父子挂载或重挂、确认和精确撤销入口；所有写入都在仓库锁内复核完整关系编号、版本、状态、类型、端点和顺序号。失效与重挂把关系版本精确推进一版，能力保存完整前后态；确认关闭能力，撤销只接受同仓库、同独占许可、仍处于持有阶段且当前记录精确等于写后记录的能力。

普通父子入口在第一笔写入前拒绝无效端点、自身挂载和成环；检测到多父、悬空端点或既有环时归为内部不一致并触发追根因诊断。首次挂载创建一条新普通父子关系；已在目标父返回幂等读回；合法重挂保持关系编号、推进版本并形成可撤销能力。

结构写入会话新增关系变更写集和审计读回标记。撤销顺序固定为已发布关系变更逆序恢复，再依次撤销索引、新关系、节点和主信息；提交在既有候选确认完成后逐项关闭关系变更能力。能力析构不自动写入或回滚。

## 4. 准入、兼容与失败收口

无效句柄、错仓句柄、过期版本、重复端点、共享许可写入、普通父子误走通用重挂和决定后调用均在写前拒绝。关系写后必须通过完整关系句柄和审计记录读回；没有完整读回或没有显式请求提交时，会话按写集逆序恢复。

旧 `删除关系(关系句柄, const 结构事务令牌&)` 共享许可兼容签名与语义保持不变；`严格删除关系` 和本次新增结构化写入口继续只接受独占许可。未修改普通删除 ABI、事务 ABI 或旧无令牌兼容路径。
