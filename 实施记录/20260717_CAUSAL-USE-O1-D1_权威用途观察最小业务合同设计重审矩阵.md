## 4. 阶段主干与 E107 原阻断复核

### 4.1 A06：阶段与推进

三种合法阶段元组保持互斥：

```text
尝试未达成：未达成已转待重筹办 + 未达成 + 未形成 + 无正式结算状态。
待结算：任务已完成待需求结算 + 已达成 + 待结算 + 无正式结算状态。
最终已结算：已完成并结算 + 已达成 + 已满足 + 有完整正式结算状态。
```

唯一允许推进仍为同身份、同不可变核心的 `待结算 -> 合法最终已结算`。终态变化、逆向和其它不允许转换在两个候选各自合法后才进入同账冲突。A06 通过。

### 4.2 E107 原双重收口已经消除

E108 已把顺序统一为：

```text
新候选自身合法
-> 可选既有候选自身合法
-> 身份 / 核心 / 阶段同账比较
```

最终阶段统一沿 `事件.结果来源.需求结算.材料.正式结算->结算状态.状态节点` 深检；`最近结算状态` 只作兼容投影。新候选缺失 / 无效 / 矛盾正式结算材料，或已发布既有候选自身非法，只落内部不一致追根因，不进入同账比较。E107 指出的原 A07 / A10 阻断不再存在。
