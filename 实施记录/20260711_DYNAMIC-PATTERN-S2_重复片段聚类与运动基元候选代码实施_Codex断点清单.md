## 3. 核心处理与结构边界

调用路径固定为：

```text
动态观察组合器::形成动态聚类候选
-> 动态模式业务服务::形成动态聚类候选
-> 动态模式算法::执行动态聚类
-> 动态聚类结果
```

实现合同：

1. `材料.动态模式` 唯一声明公开容量、请求、单步实例、重复聚类、运动基元、处理状态、拒绝原因和结果 DTO。
2. 每个完整 `规范动态步` 只形成一个单步实例；多个步骤 / 窗口仅展开为独立实例，不拼接多步序列或枚举子序列。
3. `计算动态模式结构键稳定哈希` 只定位固定桶；桶内必须由 `动态模式结构键相同` 逐字段裁决，窗口稳定哈希不参与相等、去重或支持计数。
4. 同一来源动态完整句柄且全部来源材料相同只保留一份；同句柄映射不同结构键、回合、片段或来源材料返回内部不一致。
5. 同一回合锚点映射不同完整回合身份返回内部不一致；同一完整回合对同一结构键只贡献一次支持，但不同来源实例仍可保留。
6. 同一完整结构键至少由两个不同完整回合支持才形成重复聚类；版本 1 有效来源动作和空二次角色使运动基元候选与重复聚类一一对应。
7. 输出按完整结构键、回合锚点、片段序号、时间和来源动态稳定排序；输入、窗口、步骤和桶遍历顺序不影响结果。
8. 聚类是同步纯值计算；不 import / 调用数据操作，不读仓库，不取得许可，不保存令牌 / 会话 / 接线 / 锁，不写缓存、文件、SQL 或权威结构。
