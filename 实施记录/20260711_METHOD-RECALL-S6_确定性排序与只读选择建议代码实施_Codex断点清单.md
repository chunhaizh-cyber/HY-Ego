## 2. 实施内容

### 2.1 排序与建议值材料

在 `海中鱼巣/领域/方法召回服务.h` 新增：

```text
结果匹配等级
方法排序策略证据等级
方法排序状态 / 来源状态
方法排序项
方法确定性排序材料
方法选择建议状态 / 材料
生成确定性排序
生成只读选择建议
复核确定性排序来源
复核只读选择建议来源
```

所有材料均为调用期非权威值副本。排序材料保留请求、索引发布版本、完整来源候选数量、精确重复输入数量和每项 #207 复判证据；建议材料只在唯一状态携带建议方法，不写任务选择关系。

### 2.2 全批次准入与来源复核

`生成确定性排序` 要求传入完整 S4 候选组和对应 S5 复判组：

```text
重建当前请求和完整候选组
-> 复核索引发布版本与所有候选来源
-> 按候选完整身份 O(n log n) 排序
-> 精确重复材料去重，冲突重复或缺项拒绝
-> 一一核对候选组与复判组
-> 逐项复核 #207 全部来源
-> 构造语义项并在发布前再次复核完整批次
```

因此调用方不能只提交一个原本属于多候选桶的方法来伪造唯一建议。正常索引、请求、登记、生命周期、规格、条件或特征版本漂移返回材料过期；当前版本结构矛盾进入公共追根因诊断。

### 2.3 语义排序与只读建议

排序键固定为：

```text
完整结果匹配
-> 全部满足 / 明确不满足 / 不可判定
-> 已确认条件策略证据 / 无明确证据
-> 完整方法句柄
-> 同方法多结果时完整结果句柄仅作总序收口
```

完整句柄只提供稳定排列，不进入语义唯一判断。只有全部满足且持有已确认条件策略证据的项可建议；零个可建议项返回无候选，一个返回唯一可建议，多个首项语义相同返回语义并列且不携带建议方法。

当前没有正式选择优先策略入口，`已确认条件策略` 只证明 S5 策略来源可读，所有可建议项保持同级，不生成虚构的策略唯一。缓存命中、展示次数、关系编号、创建顺序和线程完成顺序均不进入排序键。

### 2.4 性能与并发边界

候选归一、冲突重复检查和语义排序均采用 `std::sort` 加线性相邻检查，整体为 O(n log n)。建议扫描采用固定两个指针和计数，不分配临时候选数组；来源复核只排序轻量指针组，不复制完整复判 DTO。

服务内部不创建 `thread`、`async` 或线程池。调用方可并发提交冻结的完整候选组；每次调用只使用局部容器并通过领域服务公开只读入口复核来源，合并结果不依赖线程完成顺序。
