## 4. 包级迁移口径

### FS-03-A 特征身份与语义类型读取

保留行为：

```text
特征节点必须能按稳定身份读取。
按主键或索引命中后必须复核节点类型。
语义类型只作为特征命名、查找或初始化材料。
```

删除行为：

```text
不迁移旧特征服务绑定裸指针。
不迁移旧差异度回调作为权威裁决结构。
不通过旧主信息裸指针读取绕过特征服务。
```

新增约束：

```text
特征身份由节点句柄和索引复核承载。
语素入口到特征的对应关系由关系仓库承载。
高级服务需要特征状态材料时必须经特征服务。
```

验收方式：

```text
有效特征句柄可经特征服务读回。
无效句柄、错误节点类型或索引命中后类型不符时返回空，结构数量不变。
```

### FS-03-B 抽象特征、值域与状态语义分流

保留行为：

```text
抽象特征和值域可作为后续特征增强、方法条件和需求结算规则的材料。
任务 / 方法状态语义必须继续有结构化承载。
```

删除行为：

```text
不迁移旧抽象特征主信息字段和值域变体为当前第一轮权威字段。
不通过旧固定语素初始化壳创建任务状态机或方法状态机事实。
```

新增约束：

```text
任务状态归任务服务和状态服务。
方法角色状态归方法服务和状态服务。
值域系统进入特征 / 特征值增强专项确认。
```

验收方式：

```text
当前不新增抽象特征和值域代码。
后续专项必须证明值域缺失时不会伪造成需求目标或方法条件成立。
```

### FS-03-C 实例特征创建、宿主槽位与当前值读取

保留行为：

```text
可在宿主存在或业务节点下建立特征槽位。
可通过特征服务读取当前特征状态材料。
```

删除行为：

```text
不迁移旧宿主普通子链作为唯一特征归属机制。
不迁移旧特征主信息当前值字段作为所有值类型的权威入口。
不直接复制旧子特征槽位复制逻辑。
```

新增约束：

```text
特征值节点必须通过特征服务封口创建和写入。
特征值必须有特征归属关系，非归属值不得写入目标特征。
删除或重挂必须走仓库或领域服务入口。
```

验收方式：

```text
有效宿主和特征类型可创建特征槽位。
无效宿主、无效特征类型或非归属特征值写入时拒绝，结构数量不变。
```

### FS-03-D I64 特征值写入读取与特征服务封口

保留行为：

```text
I64 可以作为第一轮特征状态材料。
写入后必须能经特征服务读回。
```

删除行为：

```text
不把 VecU、指针句柄、体素值、稳态列表和近帧候选塞入第一轮通用值入口。
不允许需求、任务、方法直接访问特征值服务。
```

新增约束：

```text
特征服务是唯一对外特征值访问入口。
特征值服务只作为特征服务内部依赖。
写入入口必须拒绝无效特征、无效特征值、错误归属和缺少前置关系。
```

验收方式：

```text
I64 写入后可读回。
拒绝路径前后特征值关系和主信息值材料不变。
```

### FS-03-E Vec 原始值容器与特征值身份

保留行为：

```text
旧项目证明 VecU / VecI64 原始值容器、内容去重和子值节点有后续价值。
原始值比较必须优先于内容哈希结论。
```

删除行为：

```text
不迁移旧指针式 VecU 只读指针为长期身份。
不把旧 VecU 句柄直接当新项目稳定句柄。
不在第一轮把 Vec 原始值恢复成需求目标或方法结果事实。
```

新增约束：

```text
Vec 原始值容器必须另行详细设计。
长期身份必须是节点句柄或后续确认的稳定值句柄。
```

验收方式：

```text
本轮只生成候选专项，不新增 Vec 原始值结构。
后续专项需证明哈希碰撞时可复核原始值。
```

### FS-03-F 特征值非权威缓存、内容哈希与命中统计

保留行为：

```text
内容哈希可以缩小候选范围。
命中次数、最后命中和分层索引可作为非权威缓存材料。
```

删除行为：

```text
不把内容哈希、命中次数、最后命中或轮廓分层索引作为业务事实。
不让缓存命中替代原始值比较。
```

新增约束：

```text
缓存必须有命名空间、来源结构、来源版本、规则版本、值类型、可持久化标记和失效条件。
缓存清空不得改变特征值事实。
```

验收方式：

```text
清空缓存后仍能由权威原始值重建候选。
缓存缺失不得改变需求、任务、方法或状态结论。
```

### FS-03-G 特征值域匹配、复合比较与候选判断

保留行为：

```text
值域匹配、复合特征比较和差异度可作为需求结算、方法条件和二次特征候选材料。
```

删除行为：

```text
不把局部 bool、评分、差值或候选命中直接作为需求满足、任务完成或方法可执行事实。
不让 VecU 解释规则绕过领域服务裁决。
```

新增约束：

```text
需求结算事实由需求结算记录承载。
方法条件事实由方法系统和状态 / 特征服务复判。
二次特征候选必须经二次特征服务或后续专项确认。
```

验收方式：

```text
候选比较不新增业务节点或关系。
业务事实必须由需求、任务、方法、状态、动态或因果结构写入。
```

### FS-03-H 稳态、局部区间覆盖与抽象区间升格

保留行为：

```text
稳态、局部区间和抽象区间可作为后续特征学习和二次特征增强候选。
```

删除行为：

```text
不把稳态列表、近帧候选、局部区间覆盖或抽象区间直接作为权威状态事实。
不通过旧升格函数自动创建抽象特征结论。
```

新增约束：

```text
稳态和区间升格必须等待特征增强专项确认。
任何学习或升格都必须有来源样本、评分、版本和回滚 / 复判边界。
```

验收方式：

```text
本轮不新增稳态结构。
后续实现必须证明缓存或候选清空不破坏权威特征值。
```

### FS-03-I 特征值序列化、反序列化与恢复材料

保留行为：

```text
Vec 原始值序列化和恢复可作为后续快照、事件日志或审计材料。
```

删除行为：

```text
不把旧特征值主信息序列化格式直接作为新项目恢复格式。
不序列化缓存字段后再恢复为业务事实。
```

新增约束：

```text
持久化只作为快照、事件日志、恢复和审计适配。
反序列化失败不得写入半结构。
```

验收方式：

```text
本轮不实现恢复代码。
后续恢复专项必须证明失败不写结构、缓存字段不裁决事实。
```

### FS-03-X 体素 / 外设值材料排除

保留行为：

```text
三维体素、轮廓融合和体素链只作为后续外设 / 体素专项材料。
```

删除行为：

```text
不接 D455。
不接体素链。
不接外设观察。
不把 VecU 体素值作为当前世界事实。
```

新增约束：

```text
外设材料只作为材料或请求，不直接成为世界事实。
体素和视觉融合必须等待规范、详细设计、计划和真实样本验收。
```

验收方式：

```text
本轮只登记排除边界，不新增体素、外设或显示代码。
```
