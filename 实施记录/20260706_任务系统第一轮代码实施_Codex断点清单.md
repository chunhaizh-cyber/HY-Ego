## 2. 核心处理

本切片只实现任务系统第一轮承接壳：

```text
按需求创建完整任务承接壳。
任务来源需求关系读取。
任务虚拟存在关系读取。
任务运行场景关系读取。
任务目标状态读取。
任务创建生命周期状态和发生时间戳读取。
```

本切片未实现：

```text
完整任务状态机。
任务筹办。
任务执行。
缺口补全任务。
旧任务主信息字段迁移。
旧任务步骤 / 结果树迁移。
```

## 3. 结构承载

代码修改：

```text
海中鱼巣/领域/任务服务.h
海中鱼巣/入口.cpp
```

任务承接壳当前结构：

```text
节点类型::任务
关系类型::归属：需求节点 -> 任务节点
关系类型::引用：任务节点 -> 来源需求节点
关系类型::引用：任务节点 -> 任务虚拟存在节点
关系类型::引用：任务节点 -> 任务运行场景节点
关系类型::模板：任务节点 -> 需求目标状态节点
关系类型::引用：任务节点 -> 任务创建状态节点
```

任务创建状态由状态服务创建为实例状态，发生时间戳仍由状态服务读取。

## 4. 输出变化

`任务服务` 新增或增强：

```text
任务生命周期状态
按需求创建任务(需求节点, 需求服务, 存在服务, 场景服务, 状态服务, 任务场景, 发生时间戳)
记录任务生命周期状态
读取任务来源需求
读取任务虚拟存在
读取任务运行场景
读取任务目标状态
读取任务创建状态
```

`入口.cpp` 默认自检已改为通过任务服务读回任务承接壳，不再由入口裸写需求-任务关系或任务-创建状态关系。

## 5. 入口前置拒绝

本切片验证的任务入口拒绝：

```text
无效需求创建任务被拒绝。
无效场景创建任务被拒绝。
缺创建发生时间戳创建任务被拒绝。
需求无目标状态时创建任务被拒绝。
拒绝前后结构数量不变。
```

## 6. 调用路径

```text
用户确认进入代码实施
-> 任务系统规范和详细设计
-> 任务服务按需求创建任务
-> 需求服务读取需求目标状态
-> 存在服务创建任务虚拟存在
-> 关系仓库写入任务承接壳关系
-> 状态服务记录任务创建实例状态
-> 入口默认自检读回任务承接壳
```
