## 1. 当前事实

1. JY-292 要求任务服务组合逆序、存在 / 状态服务清理自身候选，但只定义创建 / 读取令牌入口。
2. 后续任务关系失败时，任务服务直接删除虚拟存在、实例状态及内部关系会越过原子服务所有权；不清理会留下半结构。
3. 第三轮第一层实现曾编译 0/0，接顶层组合前已撤回；五个曾编辑代码文件的工作树 blob 与 HEAD 一致，内容 diff 与 cached diff 为空。

## 2. 裁决

存在 / 状态服务分别返回不可默认构造、不可复制、绑定原服务实例和当前独占令牌身份的未发布候选能力。能力只在任务创建调用栈存活，不是机器事实、长期 DTO、消息、许可或日志状态。

每个原子服务独占三个同令牌入口：创建候选、撤销未发布候选、确认候选已发布。任务服务只能读取主句柄并回交能力，不获得候选内部句柄组或任意删除入口。

## 3. 固定顺序

```text
需求同令牌复核
-> 任务节点 / 主信息
-> 存在未发布候选
-> 状态未发布候选
-> 任务自有承接 / 生命周期关系
-> 完整读回
-> 状态 / 存在候选确认
-> 返回成功
```

失败逆序：任务自有关系 -> 状态候选撤销 -> 存在候选撤销 -> 任务节点 / 主信息。

## 4. 幂等与失败语义

同一能力成功撤销后重复撤销返回幂等已撤销且不写结构。首次撤销前候选结构缺失 / 冲突、服务实例或令牌不匹配、候选已确认发布、部分撤销或确认状态矛盾均追根因。错误能力在第一笔撤销写入前拒绝。

## 5. 禁止项

禁止公开裸句柄删除包、可伪造 bool 授权、长期候选登记、任务服务直删原子结构、RAII 析构时写仓库、令牌进入 DTO、修改 #217 ABI 或以日志 / 扫描兜底。

## 6. 执行前复核

执行窗口必须从 clean 内容基线逐项确认候选类型可原地定义在 `存在服务.h` / `状态服务.h`，任务服务已包含对应类型，仓库令牌删除足够让原服务精确撤销。若需要独立数据文件、核心 ABI 或已发布任意删除能力，停止并再次退回设计。
