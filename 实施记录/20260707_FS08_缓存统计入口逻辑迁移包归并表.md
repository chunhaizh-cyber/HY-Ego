## 4. 建议进入第一轮计划的逻辑包

建议第一轮只纳入：

```text
FS-08-E：结构统计运行期快照。
```

同时从已确认的非权威缓存与统计规范中纳入第一轮通用缓存治理壳：

```text
缓存命名空间。
非权威缓存记录结构或统计快照结构。
结构统计运行期快照。
清空结构统计缓存。
缓存清空 / 重建接口壳。
```

第一轮目标是验证“缓存缺失、缓存清空、缓存重建不改变权威节点、关系、索引结构”，而不是实现完整缓存系统。

## 5. 后续或排除逻辑包

以下不进入第一轮计划：

```text
FS-08-A：特征值内容哈希和命中观察缓存。
FS-08-B：特征值哈希候选索引缓存。
FS-08-C：需求观察统计和目标特征缓存。
FS-08-D：需求结算非权威摘要缓存。
FS-08-F：存在观测非权威缓存。
FS-08-G：二次特征运行期签名缓存。
FS-08-X：显示线程体素统计排除边界。
```

原因：

```text
特征值内容哈希和命中观察缓存依赖 VecI64 / VecIU64 原始值能力。
特征值哈希候选索引需要先确认落在索引仓库缓存项还是独立特征值缓存项。
需求观察统计不得替代需求目标状态。
需求结算摘要必须由权威需求结算记录重建，不能从旧主信息、日志或控制台倒推。
存在观测权威事实应进入特征、状态、动态或二次特征结构。
二次特征签名缓存属于二次特征服务后续增强。
显示、线程、体素和外设统计必须转 FS-10 或 FS-X 后续专项。
```
