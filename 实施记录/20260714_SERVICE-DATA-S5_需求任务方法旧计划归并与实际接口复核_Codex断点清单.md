## 2. 当前接口复核矩阵

| 复核项 | 当前事实 | 对 #269 的影响 |
| --- | --- | --- |
| 本次新建主信息、节点、关系、索引 | #264 会话已有写集、读回和逆序撤销 | 可直接作为新分层创建型结构底座 |
| 本次新建候选 I64 值 | #272 已形成同许可写入、读回和撤销 | 可承接第一轮状态值材料 |
| 节点主键组、按源 / 按目标关系记录组 | #276 已形成同许可值式读取 | 可承接业务准入和关系路径复核 |
| 已发布关系失效 | 关系仓库存在兼容入口，但会话没有可逆变更写集 | 不能在 #269 内直接调用，否则失败后无法恢复旧当前关系 |
| 已发布关系重挂 | 既有兼容入口不在会话统一提交 / 撤销边界 | 方法生命周期和普通父子重挂不能闭合 |
| 普通父子首次挂载 / 重挂 | 关系仓库拥有通用结构语义，会话未封装 | 需求父子不能在同一许可内完成写前防环、写后读回和失败恢复 |
| 任务生命周期唯一当前态换代 | 需要先形成新关系，再可逆失效旧关系 | 缺少会话化关系换代时会留下双当前或丢失旧当前 |
| 重筹办失效旧方法选择 | 需要同一会话撤销旧选择 / 投影并形成新筹办状态 | 不能靠旧带令牌入口或事后扫描兜底 |

结论：`#269` 的创建型结构可以设计，但生命周期换代、需求父子重挂和方法生命周期仍缺通用结构能力。不得把该缺口下沉为需求 / 任务 / 方法业务特例，也不得由 `#269` 直接调用旧带令牌兼容入口。
