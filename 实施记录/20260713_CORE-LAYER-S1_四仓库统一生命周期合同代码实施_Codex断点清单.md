## 2. 实际接口

统一值类型：

```text
结构写入状态：已提交、幂等读回、入口拒绝、许可拒绝、版本漂移、候选已撤销、候选已确认、内部不一致
结构写入结果：状态、结构编号、请求版本、当前版本
带值结构写入结果<T>：结构写入结果 + 可选值式载荷
```

新增仓库入口：

```text
主信息仓库：结构化创建主信息未发布候选、结构化确认未发布候选、结构化撤销未发布候选、严格删除主信息
节点仓库：结构化创建节点未发布候选、结构化确认未发布候选、结构化撤销未发布候选、严格删除节点
关系仓库：结构化创建关系、严格删除关系
索引仓库：结构化绑定主键、严格删除主键
```

新结构化写入口统一先验证独占令牌；共享许可返回 `许可拒绝` 且结构不变化。既有无令牌、共享令牌、`bool` 和 `optional` 入口未删除、未改签名。

## 3. 核心处理与结构承载

`核心/结果.结构写入.h` 是稳定数据结构单头文件，不含领域业务枚举或算法。主信息 / 节点候选继续绑定原仓库、完整令牌和精确句柄；候选记录创建后仍为普通有效记录，确认只关闭撤销能力，撤销物理移除精确记录。

关系与索引未新增候选类型。结构化关系创建复核类型和有效端点；结构化索引绑定复核节点、正反向映射、同键幂等和冲突。严格删除复核独占许可、完整当前身份和版本；关系 / 主信息 / 节点删除推进版本，索引删除同时维护正反向映射。旧关系 `删除关系(..., 令牌)` 和索引 `删除主键(..., 令牌)` 继续接受共享路径，等待 #271 按调用点收口。

## 4. 输出变化与调用路径

```text
后继独占写入会话
-> 调用四仓库结构化入口
-> 按状态区分入口拒绝、许可拒绝、版本漂移和内部不一致
-> 保存主信息 / 节点候选、关系句柄和索引键精确写集
-> 发布前失败时：严格删除索引 / 关系 -> 撤销节点候选 -> 撤销主信息候选
-> 发布成功时：确认节点候选 -> 确认主信息候选
```

独立真模块 `海中鱼巣.核心.自检.仓库统一合同` 承载 A01-A14；工程以 `ClCompile` 登记，入口只增加一个 `import`、一次阶段 700 登记和总结果汇总。
