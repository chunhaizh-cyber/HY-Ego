## 4. 总结论

1. 任务创建会同时写 `归属(需求 -> 任务)` 与 `引用(任务 -> 需求)`，双向结构真实存在，可作为第一版承接授权的必要证据。
2. 当前公开承接读取只验证 `任务 -> 需求` 一侧；没有同时返回或复核两条关系句柄 / 版本的任务授权 DTO 或公开入口。因此不能宣称双向授权已经实现。
3. 任务没有唯一当前生命周期关系、版本或合法迁移入口。`选择任务方法` 完全不读取生命周期；当前不存在可由代码证明的“合法选择状态集合”。
4. S6 建议完整携带任务请求、索引发布版本、候选来源、登记 / 生命周期 / 结果规格 / 条件规格和运行上下文材料，并有公开来源复核入口，但任务服务没有任何建议输入字段或调用点。
5. 当前任务方法选择只是一条顺序号 30 的有效 `引用(任务 -> 方法)`；读取只返回方法节点，不返回关系句柄、关系版本、来源版本、状态或审计。
6. 方法失效后，方法服务和动作桥会拒绝继续把该方法视为有效方法首，但旧任务方法关系继续有效、仍阻止重新选择，也没有自动转待重筹办的正式入口。
7. 当前方法动作桥只检查“存在任意 `引用(任务 -> 方法)`”，不限定顺序号 30，不核对任务服务当前选择关系或选择版本。
8. 任务管理线程、任务工作线程和上行桥只搬运非权威请求 / 回执和做只读复核；它们不是授权来源，也没有选择方法写入口。
9. 固定安全根直接创建已知方法并调用旧 `选择任务方法`，不经过 S4-S6、双向授权复核或任务当前态，不能作为通用接线兜底。

第一版权力分层应保持：

```text
选择前承接授权 = 双向需求任务配对 + 当前筹办中关系 / 版本。
执行前完整授权 = 双向配对 + 合法当前生命周期 + 当前任务方法选择 + 方法 / 动作来源当前。
```

这两种授权都必须按调用期权威结构计算，不持久化第二份授权 bool，也不由线程、消息、缓存、日志或显示决定。

### 7.1 `策略唯一` 的实际可达性

协议枚举保留 `策略唯一`，但当前 `方法排序项::可建议` 要求全部可建议项都具有“已确认条件策略”同一证据等级，结果也都为完整匹配、条件也都为全部满足。因此在当前合法数据中，多于一个可建议项会进入语义并列，`策略唯一` 分支没有真实独立优先策略来源。

#223 可以保留该协议值作为未来兼容分支，但不得为验收伪造成功率、成熟度、线程完成顺序或句柄顺序来制造策略唯一。当前真实成功路径是单个可建议项的 `唯一可建议`。

### 7.2 无环接线要求

后续不能让 `任务服务.h` 反向包含 `方法召回服务.h`。设计窗口应在 #223 执行前明确以下一种无环方案：

```text
首选：新增任务服务所有的扁平选择请求 DTO，由方法召回侧把已复核 S6 建议适配为该 DTO；任务服务只解释稳定句柄 / 关系 / 版本并在写边界重读。
备选：新增不依赖任务服务和方法召回服务实现的中立协议头，双方只依赖协议类型。
```

扁平材料至少需要任务、来源需求、预期当前生命周期关系 / 版本、双向承接关系身份、建议状态 / 方法、索引发布版本、方法登记 / 生命周期来源、结果规格来源和必要的条件 / 上下文来源身份。具体字段必须由修订后的流程图、详细设计和 #223 计划固定，执行窗口不得临时发明。

| 已选方法随后失效 | 方法服务不再把它视为有效；旧选择关系仍有效 | 当前选择材料标记来源过期 / 不可执行；不得静默换方法或复活方法 |
| 失效方法的任务尝试执行 | 方法动作桥因 `有效方法首` 失败返回空 | 逻辑内返回，不执行；由正式任务迁移进入待重筹办 |
| 待重筹办后选择新方法 | 旧关系仍阻止当前旧入口 | 正式回到筹办中时先使旧选择失效并保留审计，再按新生命周期版本选择 |
| 旧选择已失效但可审计 | 当前任务服务没有该语义 | 审计读取保留旧关系，不把已失效关系当当前选择 |
| 同任务出现多条有效选择 | 当前唯一目标读取返回空，无法区分无选择和结构损坏 | 追根因解决，不扫描选一条、不按编号或时间猜测 |
| 方法活跃但没有主动入口 | 旧任务选择仍可成功 | #223 写前拒绝；不把“方法节点类型正确”当可执行 |
| 动作入口或规格在选择后漂移 | 旧选择仍有效；执行桥可能返回空 | 冻结执行前重读选择 / 方法 / 动作 / 规格来源，过期不执行并进入正式重筹办路径 |

方法失效不应由方法服务跨域遍历并直接改写所有任务。任务服务保持选择关系所有权，执行前发现来源过期时按任务生命周期和重筹办入口收口。

## 10. 写后读回与执行桥非成功二分

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 无效任务、双向配对缺失、非筹办中、旧生命周期版本 | 逻辑内返回 | 写前不创建选择关系 |
| 无候选、语义并列、建议过期、建议状态不允许 | 逻辑内返回 | 保持筹办中，不按句柄或线程顺序选方法 |
| 方法失效、规格 / 条件 / 索引 / 动作来源漂移 | 逻辑内返回 | 本次不写；必要时由正式生命周期入口形成待重筹办请求 |
| 精确重复同请求且当前结构一致 | 幂等逻辑内返回 | 返回既有当前选择材料，不增长关系 / 版本 |
| 同任务同版本冲突建议 | 逻辑内返回 | 写前拒绝，不覆盖当前选择 |
| 当前选择创建后关系句柄无效 | 追根因解决 | 停止依赖路径，不返回普通失败继续 |
| 创建后正向读回缺失、多条或目标错误 | 追根因解决 | 不由排序器、线程或扫描器补写 |
| 关系版本、来源版本或审计读回与已提交结果矛盾 | 追根因解决 | 公共诊断后停止执行依赖 |
| 重筹办失效旧关系后审计或当前读取不一致 | 追根因解决 | 不创建新选择，不复活旧关系 |
| 当前没有合法选择，任务执行桥读取为空 | 逻辑内返回 | 不形成冻结执行请求 |
| 当前选择来源已过期、方法 / 动作不再可调用 | 逻辑内返回 | 不执行，保留审计并走任务重筹办 |
| 所有冻结版本仍当前但动作桥读回与当前选择矛盾 | 追根因解决 | 停止执行，不把空材料当普通缺口 |

当前 `任务服务::读取执行桥请求材料` 只返回任务和方法；当前 `方法服务::读取动作执行桥请求材料` 只检查任意任务到方法引用。#223 / #224 必须把它们接到“当前选择关系 / 版本”而不是“任意引用存在”。

## 11. 固定安全根与通用选择差异

| 维度 | 固定安全根当前路径 | 通用选择要求 |
| --- | --- | --- |
| 方法来源 | 路由现场创建已知方法和动作入口 | 从正式登记 / 规格 / 索引召回和复判获得建议 |
| 任务状态 | 创建任务后直接选择 | 只接受 #221 / #222 证明的当前筹办中关系 / 版本 |
| 承接授权 | 只读任务承接材料，未复核双向关系 | 复核需求到任务归属和任务到需求引用配对 |
| 选择输入 | 裸任务句柄 + 裸方法句柄 | 冻结筹办请求 + 当前 S6 来源材料 / 扁平版本证据 |
| 并列处理 | 不存在候选集合 | 语义并列保持未选择 |
| 方法失效 | 样例内方法刚创建，未覆盖失效 | 失效 / 规格漂移写前拒绝，旧选择留审计并重筹办 |
| 动作桥 | 使用固定稳定动作键立即读取 | #224 冻结当前选择、动作键、入口和规格来源后才排队 |
| 调度 | 入口隔离自检同步调用专用路由 | 管理 / 工作线程只搬运冻结请求，不裁决授权 |

固定安全根仍可作为限定回归样例，但 #223 不得用它的已知方法、固定稳定键、局部上下文或同步调用替代通用建议和授权链。

## 12. 线程与调用路径结论

| 模块 | 当前与选择的关系 | 权力结论 |
| --- | --- | --- |
| 任务管理线程 | 接收 `任务执行请求` 运行消息并排队 | 不形成授权、不选方法、不写任务关系 |
| 任务工作线程 | 外部触发消费请求并形成非权威回执；强类型回执也由外部发布 | 不执行任意方法、不裁决结果、不写选择 |
| 任务管理上行桥 | 重读既有任务选择、方法 / 动作 / 状态 / 动态后提交非权威治理消息 | 只读；“可请求正式入口复核”不是执行许可 |
| 自我治理领域路由 | 固定安全根直接调用旧选择入口 | 专用样例，不是通用调用方 |
| 控制面板 | 只读显示所选方法 | 不写、不修复、不裁决 |
| `入口.cpp` | 当前唯一 S6 建议生产和大量选择调用均为自检 / 样例 | 不等于生产接线 |

全仓未发现把 `方法选择建议材料` 传给 `任务服务` 的正式生产调用点。
