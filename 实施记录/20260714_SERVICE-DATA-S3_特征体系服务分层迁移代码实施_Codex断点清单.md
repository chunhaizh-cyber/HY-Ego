## 3. 核心处理与结构承载

新增 `数据操作.特征体系` 作为本批唯一生产执行器和特征值参与者导入方。特征业务服务、二次特征业务服务和特征状态组合器只形成业务规格、执行准入和解释值式结果，不接触结构事务能力。

结构继续按既有权威边界承载：

```text
四仓库：特征定义、槽位、特征值、二次特征身份，主键、模板、宿主、当前值和有序组成关系；
主信息：I64 初始值；
特征值服务侧表：I64 版本 1、VecI64 / VecU64 值及版本 1；
二次特征顺序号：按输入顺序固定为 0..n-1。
```

已发布特征值不更新、不覆盖、不删除。已有当前值只有在主键、类型、原始值、版本、槽位关系和宿主材料完整一致时才映射幂等成功；其它情况写前拒绝或按内部结构不一致追根因。

并发复核中发现“宿主 + 定义查询未命中后，另一线程先完成槽位和值提交，而本线程随后两主键命中”的有效交错。实现已改为在该分支权威重读完整槽位、当前值、原始材料和关系；全部一致映射幂等，否则返回冲突或内部不一致，避免把合法并发提交误报为冲突。

## 4. 失败收口与调用路径

入口拒绝发生在第一笔写入前，四仓库和特征值侧表不变化。许可或侧表锁竞争不等待；事务开始后的失败、无决定、显式撤销和异常均由 #264 / #274 在许可释放前先精确撤销参与者侧表写集，再逆序撤销会话写集。

生产调用路径：

```text
特征业务服务 / 二次特征业务服务 / 特征状态组合器
-> 数据操作.特征体系
-> 结构写入执行器二参入口
-> 不透明会话建立候选、关系、主键和 I64 写集
-> 特征值参与者复核并准备版本 1 原始材料
-> 会话确认四仓库写集
-> 参与者完成提交
-> 值式业务结果与权威读回
```

本切片仍只在隔离自检调用这些新生产能力，未迁移既有兼容服务或运行期生产调用点。
