## 2. 输入入口与公开结果

现有 `结构写入执行器::执行` 继续作为唯一执行入口。数据操作层只能在同步回调中取得 `结构写入会话&`，新增公开能力固定为：

```text
写入候选I64值(主信息, 槽位, 预期值) -> 值式结构写入结果
候选I64值匹配(主信息, 槽位, 预期值) -> bool
读取节点类型(节点) -> optional<节点类型>
读取节点主信息(节点) -> optional<主信息句柄>
读取I64值(主信息, 槽位) -> optional<int64>
读取关系记录组(源节点, 关系类型) -> vector<关系记录>
```

公开段不返回原始令牌、许可、仓库引用、锁、未发布候选或可变指针。会话仍不可默认构造、复制或移动，只能在执行器持有一次独占许可期间由当前线程访问。

## 3. 核心处理与结构承载

`写入候选I64值` 先确认完整主信息句柄属于本会话新建候选，再调用现有主信息仓库带令牌写入口。既有主信息、前序会话候选、无效、过期和错仓句柄在写入前拒绝。

值写集保存：

```text
主信息完整句柄
槽位
预期 I64 值
已读回标记
```

同一候选同一槽位首次写入登记一次；重复同值返回幂等读回，重复异值拒绝且不覆盖。`候选I64值匹配` 使用同一会话令牌精确读回后才标记完成。`请求提交` 的完整条件已扩展为主信息、节点、关系、索引和值写集全部读回。

撤销顺序保持：

```text
索引 -> 关系 -> 节点候选 -> 主信息候选
```

不增加单独值删除或补偿日志。候选主信息撤销时，其现有值容器一并物理移除；本切片不允许修改既有主信息。

## 4. 调用路径与输出变化

```text
数据操作层同步回调

-> 提交确认或精确逆序撤销
-> RAII 释放许可

```
