## 2. 正式裁决

不新增平行 S2A 计划。私有端点只有在 #219 唯一编排器正式定义后才能经顶层删除入口做真实运行验收；单独切片会迫使自检取得友元或留下未运行的私有写实现。

JY-285 将 #219 的明确允许范围扩展为：

```text
海中鱼巣/领域/统计服务.h
海中鱼巣/领域/概念图服务.h
海中鱼巣/领域/概念安全删除提交能力.数据.h
```

扩展只允许补以下正式合同：

```text
统计服务：唯一编排器私有缓存冻结 + 目标概念精确无分配提交。
概念图服务：唯一编排器私有图冻结 + 登记提交 + 完整活动快照最后交换。
提交能力：追加概念登记、统计缓存、活动快照阶段。
```

既有整命名空间清空入口保留原用途，但安全删除路径调用次数必须为零。

## 3. 冻结与锁序

```text
结构事务独占许可
-> 活动图锁_ 独占
-> 图写锁_
-> 概念用途缓存锁_ 独占
-> 单个仓库锁
```

唯一编排器在创建提交会话前非阻塞取得三把服务冻结锁，完成实时重读、目标缓存精确重建、登记包和完整活动快照预构造。当前图写路径可能持图锁等待仓库共享许可，因此任一试锁失败必须逆序释放服务锁和结构独占许可，写前返回“提交环境繁忙”；禁止持独占阻塞等待、自旋或睡眠重试。统计服务不得反向调用概念图或仓库；提交完成后按逆序释放。

冻结上下文不可复制、仅在调用栈内移动。现有公开准备入口会自行加锁，编排器持有冻结上下文后必须调用服务私有 `_已加锁` 重读 / 准备辅助，不得递归取得活动图锁、图写锁或概念用途缓存锁。

## 4. 精确缓存合同

```text
准备单位 = 目标概念 + 该目标全部用途观察键 / 值 + 写集身份。
提交前 = 持缓存独占锁重建目标组，与准备包逐项精确相等。
提交时 = 只 erase 准备包中的目标键，不分配、不扫描补写集。
守恒 = 同命名空间其它概念缓存逐项相同。
禁止 = 调用 清空概念用途观察(命名空间)。
```

目标组在会话创建前变化属于候选过期的逻辑内返回；会话创建后不再允许普通失败。

## 5. 完整快照合同

概念图清理包必须在提交前持有可直接交换的新活动快照，至少包含：

```text
新活动版本。
四根。
全部剩余概念节点。
全部剩余活动关系。
全部剩余签名。
删除后的生命周期登记组。
删除后的专用关系登记组。
```

剩余签名与删除后直接边只是重建依据，不等于完整快照。活动快照阶段只做已准备材料交换、版本推进和上一候选图清除，不得提交后扫描重建。
