## 3. A10 证据缺口

详细设计 A10 固定要求：

```text
会话确认故障时，本次 I64 版本或 Vec 记录在许可释放前精确删除。
```

当前自检草稿实际为：

```cpp
验收[7] = 准备失败结果 ...;
验收[8] = 验收[7];
验收[9] = 验收[7];
```

其中 `验收[8]` 对应 A09，`验收[9]` 对应 A10。二者只复用了错误节点类型导致的参与者准备拒绝，参与者没有进入“已准备”阶段，侧表也没有先插入本次记录；因此输出中的 A09 / A10“通过”是重复布尔值形成的伪证据，不能登记。

当前固定执行顺序为：

```text
业务回调
-> 检查会话失败 / 提交决定
-> 参与者准备并插入侧表记录
-> 会话完成提交
-> 参与者完成提交或撤销
```

在实际接口中：

```text
结构写入会话构造、候选对象、原始令牌和完成提交均为执行器私有能力。
提交准备只读视图按 JY-326 明确不暴露写入、确认、撤销、候选对象或令牌。
参与者准备返回后到会话完成提交前没有回调或故障注入点。
四仓库没有确认故障注入入口，且本计划禁止修改四仓库。
通过旧无令牌 / 带令牌兼容写、伪造候选、暴露令牌或并发破坏候选均违反计划边界。
```

因此不能在现有授权范围内真实进入“参与者已准备且会话确认失败”分支。
