## 5. 阶段优化口径

### 5.1 总入口与运行宿主

```text
旧自我线程 / Tick / 控制台命令 / 面板刷新
-> 新项目中只能成为材料、请求、调度说明或人读观察
-> 不能成为动作来源或机器事实写入方
```

### 5.2 材料、基础信息、特征状态

```text
材料准入失败时不创建事实。
外部材料不得直接成为世界事实。
特征值服务只服务特征系统。
高级服务必须经特征服务取得特征状态材料。
```

### 5.3 需求到任务

```text
需求创建流程和需求树后续结构流程分开。
任务承接流程和任务状态机 / 筹办 / 执行桥请求流程分开。
任务执行桥第一轮只设计请求材料，不实现真实执行分派。
```

### 5.4 方法与动作

```text
方法候选只做候选读取和准入。
方法选择必须经任务方法关系。
动作入口只保留身份和来源准入，不实现真实动作分派。
```

### 5.5 动态、结果、因果、结算

```text
方法执行 / 动作入口
-> 输出结果场景 / 动作动态
-> 任务回执 / 实际结果状态
-> 轻量因果引用
-> 需求结算
```

轻量因果只是证据材料，不是稳定因果结论；不得让因果提前裁决需求满足。

### 5.6 缓存、日志、快照

```text
非权威缓存只能读权威结构，不参与业务裁决。
事件日志审计材料需要说明每类写入流程的事件材料字段。
仓库快照 / 恢复拒绝矩阵作为后置恢复能力处理。
```

### 5.7 显示、控制面板、数据库

```text
显示层只读放在主链最后。
控制面板和数据库已经撤出本轮迁移计划。
旧 FS-X 中控制面板、SQL、ADO 事实只保留为排除证据。
控制面板 / 数据库重建候选只能由用户明确指令触发。
```
