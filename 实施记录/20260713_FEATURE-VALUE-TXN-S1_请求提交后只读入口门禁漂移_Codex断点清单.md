## 3. 真实接口漂移

JY-324 固定的参与者顺序为：

```text
业务回调完整读回写集
-> 会话请求提交
-> 参与者准备提交(会话)
-> 参与者通过会话复核候选身份、节点类型、节点主信息和 I64 值
-> 侧表非阻塞准备
-> 会话完成提交
-> 参与者完成提交
```

当前会话实际门禁为：

```cpp
bool 结构写入会话::请求提交() {
    // 完整读回后把决定_ 改为 决定::提交
}

bool 结构写入会话::可继续写入() const {
    return 当前线程可访问()
        && 阶段_ == 阶段::写入中
        && 决定_ == 决定::未决定
        && !首次失败_.has_value();
}
```

而参与者准备阶段必须使用的三个既有只读入口全部先检查 `可继续写入()`：

```cpp
std::optional<节点类型> 读取节点类型(节点句柄) const;
std::optional<主信息句柄> 读取节点主信息(节点句柄) const;
std::optional<std::int64_t> 读取I64值(主信息句柄, std::uint64_t) const;
```

因此回调一旦成功调用 `请求提交()`，`决定_` 已不再是 `未决定`；执行器随后调用参与者准备时，上述三个入口必然返回空。新增“主信息 / 节点是本会话候选”两个判断只能证明候选身份，不能证明节点类型、主信息锚点和候选 I64 值，无法满足详细设计第 6.2 节。

| 编号 | 设计合同 | 当前实际接口 | 影响 |
| --- | --- | --- | --- |
| IF-274-01 | 请求提交后参与者读取节点类型 | `读取节点类型` 要求决定仍为未决定 | 合法特征值节点会被当成材料缺失 |
| IF-274-02 | 请求提交后参与者读取节点主信息 | `读取节点主信息` 要求决定仍为未决定 | 无法证明节点绑定本次主信息候选 |
| IF-274-03 | 请求提交后参与者读取 I64 值 | `读取I64值` 要求决定仍为未决定 | 无法证明 I64 值与待发布版本一致 |
| IF-274-04 | #274 只新增两个候选判断 | 放宽三个 #272 既有入口会改变会话决定后的公开语义 | 执行窗口不能静默扩权 |
