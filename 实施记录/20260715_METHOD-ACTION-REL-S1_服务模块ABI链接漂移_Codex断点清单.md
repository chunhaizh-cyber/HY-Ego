## 3. 真实接口漂移

计划假定新的独立自检真模块可以同时导入需求任务方法数据操作和存在、状态、方法三个正式业务服务模块，并直接构造这些服务。当前模块声明链实际为：

```text
数据操作.存在场景：先导出 class 存在业务服务 前置声明。
数据操作.状态动态：先导出 class 状态业务服务 前置声明。
数据操作.需求任务方法：先导出 class 需求业务服务 / 任务业务服务 / 方法业务服务 前置声明。
服务.存在 / 服务.状态 / 服务.方法：导入对应数据操作模块后，再定义同名正式业务服务。
```

MSVC 在不同模块编译单元中为这些同名类型、构造参数和返回材料形成了不同模块归属修饰名。新自检对象所引用的符号与既有服务对象实际导出的符号不相等。

| 编号 | 新自检引用 | 既有对象定义 | 结果 |
| --- | --- | --- | --- |
| ABI-279-01 | `方法业务服务` 构造参数中的存在 / 状态服务归属正式服务模块 | `服务.方法.ixx.obj` 中对应参数归属数据操作模块前置声明 | 构造函数符号不匹配 |
| ABI-279-02 | `存在业务服务` 构造参数中的存在场景数据操作带正式数据操作模块归属 | `服务.存在.ixx.obj` 导出的构造函数修饰名不同 | 构造函数符号不匹配 |
| ABI-279-03 | `状态业务服务` 构造参数中的状态动态数据操作带正式数据操作模块归属 | `服务.状态.ixx.obj` 导出的构造函数修饰名不同 | 构造函数符号不匹配 |
| ABI-279-04 | `形成抽象状态写入规格` 的请求 / 结果按新自检导入图归属正式状态服务模块 | 既有状态服务对象中的修饰名归属不同 | 方法符号不匹配 |
| ABI-279-05 | `读取主键状态材料` 的返回材料按新自检导入图归属状态动态数据操作模块 | 既有状态服务对象导出的返回类型修饰名不同 | 方法符号不匹配 |

这不是缺少工程登记或旧对象缓存。完整 `/t:Rebuild` 后仍得到相同 5 个未解析符号。

## 5. 允许范围不足

#279 允许修改 `数据操作.需求任务方法.ixx` 和新 / 既有自检，但禁止修改以下真实归属参与者：

```text
海中鱼巣/领域/数据操作.存在场景.ixx
海中鱼巣/领域/数据操作.状态动态.ixx
海中鱼巣/领域/服务.存在.ixx
海中鱼巣/领域/服务.状态.ixx
海中鱼巣/领域/服务.方法.ixx
其它服务声明所有权或装配模块
```

因此执行窗口不能自行采用以下绕过：

```text
删除或搬移服务前置声明。
新建服务声明模块并重排正式模块依赖。
把自检正文移入生产服务或数据操作模块。
文本包含服务实现文件以规避模块 ABI。
只依赖旧 BMI、增量链接或特定导入顺序通过。
跳过独立自检、Release 隔离或完整 Rebuild。
```
