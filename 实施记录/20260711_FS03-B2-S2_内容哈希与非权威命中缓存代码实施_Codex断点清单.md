## 2. 核心处理

1. 特征服务按“哈希规则版本、原始类型、元素数量、完整元素”的固定小端规范字节序列生成规则版本 1 的 64 位 FNV-1a 内容哈希；I64 与 VecI64 使用 ZigZag64，节点身份、原始版本和文本不进入内容哈希。
2. 特征服务提供当前签名复核和完整原始值相等复核；比较前后重读统一版本，过期材料返回逻辑内无结果，同哈希但内容不同返回不相同。
3. 统计服务只接收内容签名 DTO，以共享互斥量保护进程内哈希记录和命中观察；读取同哈希候选、递增命中次数和最近序号、清空缓存均不写权威结构。
4. 缓存记录保留值句柄、原始版本、规则版本和命名空间；旧版本记录必须经特征服务复核后排除，缺缓存时仍可直接计算和完整比较。

## 3. 结构承载

```text
权威原始值与统一版本：继续由特征值服务和既有主信息 / 运行期原始值记录承载。
内容签名：调用期值式 DTO，不是相等结论。
哈希候选与命中观察：统计服务进程内可清空记录，不持久化、不进入节点 / 主信息 / 关系 / 索引。
完整相等结论：仅由特征服务重读完整原始值与版本后给出。
```
