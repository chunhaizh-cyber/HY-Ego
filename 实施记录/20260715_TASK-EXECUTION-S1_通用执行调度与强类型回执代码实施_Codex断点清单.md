## 2. 请求协议、组合器与两阶段队列

新增 `协议.任务执行请求.ixx`，以纯值材料冻结任务双向承接、生命周期、选择记录、方法登记、动作入口和关系 17 输入 / 输出证据。协议统一拒绝无效句柄、错误版本、非法时间、编号冲突、旧引用动作场景和不完整角色关系，不承载仓库、许可、会话或执行器能力。

新增 `组合.任务执行.ixx`，由任务、方法和状态动态正式业务公开面重读权威材料。组合器只支持构造时配置的一个稳定动作键；主体、场景和被改变目标均从任务与方法材料派生，不接受调用方伪造。任务先迁移到排队中并完整读回，之后队列候选才可确认；消费时再迁移执行中并重读动作桥。

任务管理线程新增独立强类型两阶段队列。候选不可默认构造或复制，只可移动；未确认候选析构、显式撤销和移动赋值会精确撤销本次预留。容量、停止、重复、同键异义、乱序和过期均在写前收口，队列锁内不调用业务服务。已确认请求只可取出一次，原运行消息队列继续保持原义。

## 3. 状态动态原子路径与受控消费

状态动态数据操作与组合器新增“已有前状态 + 新后状态 + 实例动态”同会话路径。新后状态和动态在一次结构写入会话中形成，动态关系顺序固定为：

```text
1 = 被改变目标
2 = 前状态
3 = 后状态
4 = 来源动作
```

Debug 固定故障入口只编译两个真实写点。每个写点命中后均由会话精确撤销新后状态、动态、关系和主键写集，数量与可读结构回前态；生产二参入口不接收故障控制。

新增 `路由.任务执行调度.ixx`，串联冻结、队列预留、排队确认、候选确认、取出、执行和过期待重筹办收口。任务工作线程只通过该窄路由受控消费，不持有业务服务、仓库或事务能力，不创建每任务线程。过期且仍精确处于排队中的任务经任务业务服务迁移到待重筹办。

## 4. 强类型非权威回执

任务结果回执协议新增 `非权威任务执行结果证据`，完整携带来源冻结请求。任务工作线程只在状态动态提交和权威读回成功后形成新版回执；内部不一致、过期收口或执行失败均不伪造成功回执。旧回执材料继续按原协议兼容，新回执不得降级为旧材料性质。

本切片不记录任务最终结果、不完成任务、不结算需求，也不把队列或回执写成机器事实。
