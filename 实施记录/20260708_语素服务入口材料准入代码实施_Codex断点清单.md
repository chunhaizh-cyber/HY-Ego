## 2. 核心处理

```text
S1：语素服务拆分“节点是可作为语素对应信息”和“节点是可作为概念追溯目标”两套准入。
S2：创建语素入口增加主键预读、主信息 / 节点 / 关系 / 索引逐步检查、读回验证和本次创建结构失效收口。
S3：创建概念入口改为文本准入、对应信息准入、概念追溯目标准入、创建入口、追加追溯和读回验证的组合流程。
S4：追加语素对应信息和追加语素概念追溯均补重复拒绝、写后读回验证和目标准入分流。
S5：入口验证补 YS-A1 到 YS-A12，覆盖文本拒绝、对应信息准入、概念追溯收窄、主键冲突、重复关系和非语素读取。
```

## 3. 结构承载

```text
语素入口身份：节点类型::语素。
对应信息关系：关系类型::语素对应信息。
概念追溯关系：关系类型::语素概念追溯。
主键绑定：索引仓库主键到语素节点绑定。
失败收口：只删除或失效本次创建的关系、语素节点、主信息和本次绑定主键。
```
