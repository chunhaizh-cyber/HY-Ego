## 2. 总结论

1. #234 的 Q10000 只表达“正式重复原因之后出现结果模式”的发生概率，不表达任务成功、需求效用或方法效果。
2. 正式任务结果治理只有未达成已转待重筹办、任务已完成待需求结算、已完成并结算三种收口。当前没有独立失败或取消领域终态。
3. 任务结果协议具备重复、幂等冲突、序号倒退和过期拒绝合同；工作线程也有进程内去重表。但正式处理显式回执路径只传当前时间戳，没有把同幂等键既有材料或同任务最新序号传入组合器，因此没有形成跨调用、可恢复的权威观察幂等账。
4. 当前没有能同时稳定引用概率来源、任务、需求、所选方法、实际状态、动态证据、需求结算、规则版本和观察时间的一次性用途观察身份。
5. 统计服务的概念用途材料是可清空、不可持久化、不可裁决业务事实的内存缓存；缓存键不能表达动态模式或因果模式身份，相同时间戳重复写入仍会重复计数。
6. 概念图服务只能在活动概念和类别准入后委托统计缓存，并形成保持、冷却、退役复核或重新激活候选；不能定义结果效用，也不能成为方法学习写入口。
7. 新分层方法系统已有登记、规格、生命周期、结构召回、复判、排序和任务授权选择；排序仍是固定结构匹配分值，当前没有结果驱动的方法学习、训练、成功率更新或学习候选生产入口。
8. 轻量因果只把来源动态作为创建准入证据，持久化后不保存来源动态关系；需求结算可带轻量因果引用，但不能据此证明对应某个 #234 概率候选。
9. 唯一运行期上下文和业务装配没有拥有动态模式、因果模式、新版任务结果治理、用途观察或方法学习接线；顶层仍调用既有完成任务并独立结算入口。
10. 当前没有用途观察恢复游标、方法学习恢复材料或待结算重试的正式恢复接线。SQL 审计投影和仓库快照候选均不能替代机器事实恢复。

因此：#235 可以完成并归档；后继必须先建立新的流程图、详细设计和计划，不能让统计缓存、线程或 #234 纯值候选直接修改方法事实。

## 3. 因果发生概率与任务 / 需求结果效用分层

因果概率候选只包含原因结构键、结果结构键、规则版本、机会证据组、机会数、正结果数、负结果数和 Q10000。完整性约束在海中鱼巣/领域/材料.因果模式.ixx:82-105，计算在海中鱼巣/领域/算法.因果模式.ixx:328-338：

    Q10000 = 正结果数 * 10000 / 机会数

正结果只表示某个原因机会之后出现一个或多个结果模式；原因之前出现结果仍计负结果，结果零次也可形成负结果。它不读取任务、需求、方法、实际状态、需求结算或用途效用。

任务 / 需求结果由另一条权威链裁决：

    新版回执
    -> 重读任务、需求、方法、动作、实际状态和实例动态
    -> 需求业务服务判定目标状态达成
    -> 任务完成或转待重筹办
    -> 需求独立结算

需求业务服务比较权威目标状态与实际状态，独立结算重新形成满足 / 未满足关系。因而 Q10000 不能替代目标状态裁决、任务结果、需求结算或方法效用，也不得与结果效用合并为一个分数。

当前正式代码没有结构化“结果效用 DTO”，统计缓存的累计结果效用候选只接收调用方直接传入的 I64，未定义权威裁决来源。

## 11. 一次性用途观察所需身份

后继设计至少需要一个稳定观察主键及完整内容，能够同时关联：

1. 因果概率候选来源：原因结构键、结果结构键、概率规则版本、正式支持回合 / 机会证据身份。
2. 新版结果来源：完整回执身份、来源执行请求、任务序号和幂等键。
3. 权威业务身份：任务、来源需求、所选方法及相关生命周期 / 选择版本。
4. 结果证据：实际状态、实例动态、目标状态裁决。

6. 观察合同：观察规则版本、观察时间、内容哈希或完整值比较材料。

必须显式定义：

    同键同内容 -> 幂等读回
    同键异内容 -> 冲突
    迟到观察 -> 按版本 / 结算状态拒绝或延后
    待结算重试 -> 不生成最终效用，不重复计数
    已完成并结算 -> 只生成一次最终用途观察
