## 3. 核心处理与结构承载

`数据操作.需求任务方法` 是本批唯一导入结构写入执行器并持四仓库的生产模块。写入统一沿以下路径完成：

```text
业务服务准入
-> 存在 / 状态业务服务形成构造权封闭规格
-> 需求任务方法数据操作取得唯一独占写入会话
-> #278 原子同会话参与入口和本批关系 / 主键 / I64 写集
-> #277 可逆关系换代或普通父子重挂
-> 会话内当前版本读回
-> 请求提交或统一逆序撤销
-> 共享许可权威读回形成值式结果。
```

需求由需求节点、主体 / 宿主 / 场景 / 当前特征状态引用、目标抽象状态模板、可选目标概念、普通父子和结算状态关系承载。任务由任务节点、来源需求授权、任务虚拟存在、场景、目标状态、实例生命周期、选择记录 / 方法投影 / 发布关系和实际结果承载。方法由登记根、方法首、共享活跃 / 失效状态、角色状态、条件 / 结果 / 动作入口规格和值材料承载。召回候选、复判、排序和建议保持非权威，只在唯一当前建议成立时形成任务选择请求。

## 4. 弹窗根因与修复

第一次完整 Debug 自检在进入 #269 阶段后触发 MSVC `std::optional::operator->() called on empty optional` 断言。直接生成点是自检在动作桥前置失败后仍无条件解引用动作入口；改为无效句柄输入并保留验收失败后，得到真实上游失败矩阵。

确认并修复三个生产根因：

```text
1. 已失效关系被错误地用“只读有效关系”复核，导致任务生命周期、重筹办和完成路径把正确失效判为内部不一致；改为会话审计读取并核对当前版本状态为已失效。
2. 方法首模板读取只按顺序号识别生命周期，方法条件关系的顺序号 2 与生命周期槽冲突；改为先按目标节点类型区分状态关系与方法规格关系。
3. 重复需求父子挂载进入空写会话后被映射成已提交；改为权威读回确认当前父关系完整且目标父一致后，在第一笔写入前返回幂等读回。
```

自检同时改为从审计记录组按稳定关系编号读取重挂 / 失效后的当前版本，不再用过期句柄伪造审计失败；拒绝后的结构数量均在后续合法写入前立即取快照。原弹窗在后续完整自检及最终连续 20 轮中均未再出现。
