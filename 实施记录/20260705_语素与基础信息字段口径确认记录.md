# 语素与基础信息字段口径确认记录

日期：2026-07-05

## 1. 输入入口

用户在主信息字段功能确认讨论中确认以下口径：

```text
语素入口可以对应多个对应信息节点。
概念追溯可以多级、多路。
概念是基础信息抽象后形成的。
概念节点和概念模板是基础信息的抽象节点。
状态和动态实例信息必须带有时间戳，也就是发生时间。
确认。
```

## 2. 确认结论

### 2.1 语素相关

```text
语素入口到对应信息节点不是单值字段，必须按多对多关系承载。
概念追溯不是单父级字段，必须按多级、多路关系图承载。
语素入口不从词面直接制造概念事实，只绑定已有信息节点或返回候选集合。
```

### 2.2 概念相关

```text
概念是基础信息抽象后形成的长期基础信息节点。
概念节点是基础信息抽象节点。
概念模板不是独立主信息字段，而是抽象基础信息节点在创建、归类或匹配时承担的模板用途。
上级概念由抽象基础信息节点之间的上位抽象关系承载，允许多级、多路。
```

### 2.3 基础信息实例与抽象层

```text
抽象状态和抽象动态不带发生时间戳。
实例状态和实例动态必须带发生时间戳。
时间戳属于场景内实例记录，不属于抽象状态或抽象动态本体。
```

## 3. 结构承载

```text
语素对应信息：关系仓库多对多关系。
概念追溯：关系仓库多级、多路抽象关系。
概念节点 / 概念模板：基础信息抽象节点及其用途关系。
实例状态 / 实例动态发生时间：实例状态 / 实例动态记录的必填机器事实字段。
抽象状态 / 抽象动态：长期基础信息节点，不携带发生时间。
```
