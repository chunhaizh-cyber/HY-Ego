## 3. 核心处理与结构边界

生产调用路径固定为：

~~~text
因果模式业务服务::形成因果概率候选
-> 执行因果模式
-> 因果模式结果 / 因果概率候选
~~~

实现合同：

1. 材料.因果模式唯一声明请求、机会证据、概率候选、处理状态、拒绝原因和完整性 DTO。
2. 原因结构键必须精确命中一个 #233 正式重复聚类；零命中逻辑拒绝，多命中或正式材料矛盾进入内部不一致。
3. 每个原因支持回合只形成一次机会；同一支持回合出现多个原因实例按机会起点歧义拒绝。
4. 结果允许零次出现。结果只在原因前时保留证据并计负结果；原因后一个或多个结果只计一次正结果并保留全部后续证据。
5. 结果与原因片段序号相同按片段顺序歧义拒绝，不用时间戳、最早项或遍历顺序决定先后。
6. Q10000 固定按 floor(正结果数 * 10000 / 机会数) 计算，覆盖零、满量程和中间概率；计数前执行容量与乘法上界检查。
7. 输出按完整回合和来源证据确定排序；哈希碰撞、输入乱序和并发完成顺序不改变结果。
8. 生产链为同步纯值计算，不读取仓库、不取得许可、不调用数据操作、不保存令牌 / 会话 / 接线 / 锁，不写权威结构、缓存、文件或 SQL。
