## 2. 核心处理

1. 先建立最小 `运行消息协议.ixx`，由 MSBuild v145 识别为模块接口并生成 IFC，`入口.cpp` 成功导入中文模块名。
2. 把运行消息类型、线程角色、优先级和队列结果移入 `运行消息协议.ixx`。
3. 把原有有界队列实现移入 `有界运行消息队列.ixx`。
4. 把 6 个 `std::thread` 从混合头文件拆入 6 个独立线程模块。
5. 任务管理线程独占任务调度队列，通过正式 `取出任务执行请求()` 入口向任务工作线程提供材料，不暴露可写队列引用。
6. 缓存统计线程和事件日志线程分别拥有独立队列、状态、停止标记和 join 收口入口。
7. `入口.cpp` 改用 `import`，工程和筛选器登记 8 个模块接口单元。
8. 删除 `海中鱼巣/核心/运行宿主.h` 和 `海中鱼巣/核心/运行消息队列.h`，消除头文件与模块双源。

## 5. 调用路径

```text
入口.cpp
  -> import 运行消息协议
  -> import 有界运行消息队列
  -> import 运行宿主线程 / 任务管理线程 / 任务工作线程
  -> import 缓存统计线程 / 事件日志线程 / 外设采样材料线程
  -> 各线程独立启动
  -> 消息经正式队列入口流转
  -> 各线程独立停止并 join
```

任务请求路径：

```text
任务管理线程.发送任务执行请求
  -> 任务管理线程独占的任务调度队列
  -> 任务工作线程调用管理线程.取出任务执行请求
  -> 任务工作线程生成结果回执材料
  -> 任务工作线程独占的任务回执队列
```
