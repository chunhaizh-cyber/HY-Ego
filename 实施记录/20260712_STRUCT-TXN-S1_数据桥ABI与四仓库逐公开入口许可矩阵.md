## 1. 数据桥稳定 ABI

`结构事务接线.数据.h` 是传统 `.h/.cpp` 与命名模块之间唯一公开桥。正式布局固定为：

```text
结构许可类型：无效 / 共享 / 独占。

结构事务令牌：
  域编号 U64
  运行期纪元 U64
  许可序号 U64
  许可类型

结构事务许可：
  std::shared_ptr<void> 运行期状态
  结构事务令牌
  void (*释放许可)(const std::shared_ptr<void>&, const 结构事务令牌&) noexcept
  不可复制、可移动、析构时至多释放一次

结构事务接线：
  域编号 U64
  运行期纪元 U64
  std::shared_ptr<void> 运行期状态
  结构事务许可 (*取得共享许可)(const std::shared_ptr<void>&)
  结构事务许可 (*取得独占许可)(const std::shared_ptr<void>&)
  bool (*验证共享路径令牌)(const std::shared_ptr<void>&, const 结构事务令牌&)
  bool (*验证独占令牌)(const std::shared_ptr<void>&, const 结构事务令牌&)
```

数据头内联实现移动许可的构造、移动、析构、自身完整性和接线形态判断；无独立 `.cpp`。空接线必须所有字段同时为空。已接域必须域、纪元、共享状态和四个函数入口全部有效。其它组合是半接线，构造器写前拒绝。

`std::shared_ptr<void>` 只延长运行期锁状态寿命，不持久化、不作为机器身份。协调器对象销毁后，既有接线和活动许可仍持有同一运行期状态；最后一个接线 / 许可释放后状态才析构，因此不存在仓库回调悬空。协调器析构不强制早于仓库，但自检仍采用协调器先构造、仓库后构造以便观察所有权。

协调模块为每个状态分配非零运行期纪元和单调许可序号，以活动表记录许可类型与取得线程。令牌字段复制不能建立活动许可；验证必须同时匹配状态、域、纪元、序号、许可类型、活动表和当前线程。令牌只允许同步调用链传递，不得持久化、排队或跨线程使用。

同线程同状态已经持有许可时，再调用无令牌取得入口属于实现重入错误：协调模块触发公共诊断并返回无效许可。嵌套调用必须显式传递原令牌。独占令牌可通过共享路径验证；共享令牌不能通过独占验证。许可跨线程释放或重复释放属于追根因错误。

## 2. 许可分级

```text
共享许可：普通读取、创建、写值、关系创建 / 失效 / 删除 / 重挂、索引绑定 / 删除。
独占许可：节点身份删除、主信息身份删除、后续事务包提交。
无许可：构造期形态检查、不可变仓库编号读取。
```

节点与主信息删除必须独占，因此关系 / 索引持共享许可复制本仓库材料后，可以释放本仓库锁并稳定读取端点。普通关系或索引变化仍由各自仓库锁形成点时快照，不要求结构共享许可互斥。

## 4. 最小实施覆盖

本批必须覆盖上表全部 46 个公开声明入口；其中构造器 4 个、节点不可变仓库编号 1 个不增加令牌重载，其余 41 个入口必须具有“无令牌包装 + 令牌感知实现或明确令牌委托”。不得只修创建、重挂、绑定三个写入口，也不得保留关系 / 索引锁内的任何 `节点_.节点是否有效` 调用。

未接域旧入口继续执行现有单仓库路径，不取得许可、不接受外部令牌。#217 不改变节点删除的领域准入、不新增级联清理、不接入口主装配，也不声明所有运行期调用已经统一隔离。
