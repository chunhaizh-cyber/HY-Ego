## 3. 核心处理与结构承载

1. `材料.用途事件` 是执行前冻结、规范化正式结果、结构化效用和用途事件值式 DTO 的唯一所有者。没有新增节点、主信息、关系、索引、仓库、许可、令牌、会话、锁、线程或持久状态。
2. `算法.用途事件` 在写出任何事件前深度复核完整因果候选：至少两个独立旧回合、严格回合顺序、结果实例去重、原因前 / 后顺序、完整结构键与计数 / Q10000 一致。
3. 冻结键与观察主键由同一字段 `冻结键与观察主键` 承载，不生成第二稳定键。冻结时间必须不早于任务生命周期和调度创建时间、不晚于调度执行时间；结果回执不得早于冻结。
4. 结果返回后按完整 `任务执行冻结请求` 精确比较，不以哈希、碰巧同回合机会或结果后补造候选替代执行前冻结。
5. 组合适配逐项保留完整回执、协议结果、目标裁决、任务提交、需求结算、待结算提示和全部幂等 / 时间参数；没有提前复制 #286 的 `需求正式结算材料`，现有完整 `需求提交结果` 继续保持唯一正式来源。
6. 任务身份、五条任务关系、选择记录、方法、动作、场景、主体、目标 / 实际状态和动态逐项比对；实际状态幂等主键必须等于调度请求中的正式主键。
7. 已结算分支继续复核需求主体、目标宿主、场景、目标特征、当前特征状态与目标状态六条关系语义，并要求最近结算状态句柄有效。
8. 三种正式结果分别映射为：未达成 -> 任务未达成 / 需求结算未形成 / 尝试未达成；待结算 -> 任务已达成 / 需求待结算 / 待结算；已完成并结算 -> 任务已达成 / 需求已满足 / 最终已结算。已达成必须目标值等于实际值，未达成必须不等；未达成和待结算不得夹带已满足需求材料。
9. Q10000、原因 / 结果完整结构键、历史机会组和计数保持独立概率分量，不与当前任务 / 需求结果合成单分数，不形成阈值、置信度、可靠性、稳定因果或学习建议。

## 4. 非成功收口与实施修正

缺冻结、规则不支持、非新版完整回执、结果未正式收口或完整冻结请求不同，按逻辑内返回收口且结构零变化；正式任务 / 需求 / 选择 / 状态 / 动态材料矛盾进入 `内部不一致`，不得用缓存、日志或默认值修补。

实施期间曾由自检函数内过大的局部负例夹具触发 `0xC00000FD` 栈溢出。修正只把重型负例拆入独立内部帮助函数，没有改变生产合同、导出面或允许范围；修正后完整 Rebuild、单轮与连续回归均通过。算法内部帮助函数和自检夹具均未导出，生产算法只导出 `执行用途来源冻结` 与 `执行结构化结果效用` 两个正式入口。
