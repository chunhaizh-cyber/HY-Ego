## 3. 核心处理与结构承载

1. `材料.因果模式` 唯一拥有纯值 `因果完整结构键`：格式版本、原因动态键、结果动态键和因果规则版本。默认格式版本直接初始化为当前格式 1；局部完整、当前版本兼容和原因 / 结果角色不同均可由值本身复核。
2. `因果完整结构键结果` 以强类型状态区分已形成、入口拒绝、不支持和内部不一致。`成功()` 只接受“已形成 + 无拒绝原因 + 有身份 + 当前版本兼容 + 原因结果角色不同”，拒绝版本漂移、同键身份和其它半合法成功态。
3. `算法.因果模式` 新增显式版本兼容、A/B 全字段比较、`std::optional<std::uint64_t>` 稳定哈希和身份形成入口。格式 1 只接受两把动态键规则版本与因果规则版本的当前兼容行；原因 / 结果是固定有序角色。
4. 哈希固定混入格式域、原因角色、结果角色、两把动态键哈希和因果版本，只供召回。无效或不支持身份返回空哈希；身份裁决始终回到 A/B 全字段比较，哈希相同不构成身份相同。
5. `因果概率分量` 删除原因键、结果键和因果版本旧三元组，只持一个 `因果完整结构键 因果身份`，并原样保留机会数、正结果数、负结果数和 Q10000；机会组和四项计数都不进入身份。
6. `算法.用途事件` 新增纯值 `形成因果概率分量`，只调用因果模式身份形成算法并映射四项计数；生产 `形成效用` 与自检使用同一入口，没有复制比较或哈希实现。用途深检已经接受候选后，该入口任一非成功仍统一收口为内部不一致并追根因。
7. 新建 `自检.因果完整结构键.ixx` 真模块。运行时 R01-R14 覆盖身份字段、角色、全字段比较、四类版本漂移、合法结果组合、可选哈希、召回后比较、身份排除字段、生产用途分量入口、既有用途事件回归、禁止依赖声明和进程内 20 轮稳定；外部构建、Release 隔离、完整 20/20、归档、提交与推送仍由正式 A01-A14 门禁结算。
8. 入口只在 Debug 导入自检模块、保存一个结果、以阶段 860 登记一次并纳入总汇总；工程和 filters 各唯一按 `ClCompile` 登记。870 保持零登记，自检模块不进入生产依赖方向。

## 4. 非成功收口与实施修正

- 候选 / 动态键 / 因果版本不完整返回 `入口拒绝 / 候选不完整`；原因键与结果键全字段相同返回 `入口拒绝 / 原因结果相同`；纯值路径不产生结构变化。
- 格式或三类规则版本不兼容返回 `不支持 / 版本不兼容`，不猜测兼容、不降级。
- 无效 / 不支持身份比较为假、哈希为空；哈希碰撞仍按完整字段区分，不以桶位裁决身份。
- 已接受正式用途候选后身份或概率分量形成失败，统一返回 `内部不一致 / 算法不变量破坏`，不静默丢弃、不填默认身份。
- 第一轮 Debug Rebuild 暴露 `节点句柄` 没有可见 `!=`；改用 `==` 后第二轮仍失败，证明根因是 `句柄.h` 的自由比较运算符未经 `材料.动态模式` 模块导出。最终在允许的 `材料.因果模式.ixx` 内直接比较句柄的仓库编号、节点编号和版本号，保持纯值全字段合同，不修改禁止文件 `核心/句柄.h`。最终 Debug / Release Rebuild 和全部回归均通过。
