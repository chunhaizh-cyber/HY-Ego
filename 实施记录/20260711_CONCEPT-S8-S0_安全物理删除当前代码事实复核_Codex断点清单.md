## 4. 并发与 TOCTOU 事实

### 4.1 关系端点

`关系仓库::创建关系` 和 `重挂关系并返回新句柄` 先调用节点仓库确认端点有效，随后才取得关系仓库独占锁。节点删除使用另一把仓库锁。因此存在以下交错：

```text
线程 A：确认节点有效
线程 B：删除节点并推进节点版本
线程 A：取得关系锁并写入指向已删除旧句柄的关系记录
```

关系普通读取会因端点无效而隐藏该记录，但隐藏不等于关系记录已清理。

### 4.2 索引端点

`索引仓库::绑定主键` 同样先检查节点有效，再取得索引锁；可与节点删除形成相同交错。`按主键查节点` 会隐藏失效节点，但主键条目仍留在索引表中，且当前没有按节点反查全部主键的入口。

### 4.3 领域外部引用

概念图 `图写锁_` 不被语素、需求、任务、方法及其它领域服务使用。删除准入即使在图锁内读到零外部引用，其它服务仍可在提交前新增关系。必须增加所有端点写入方共同遵守的冻结 / 事务门禁，不能靠再次扫描兜底。

## 5. 事务、补偿与隔离结论

| 能力 | 当前状态 | 是否足够物理删除 |
| --- | --- | --- |
| 写入事务 | 只有状态机和人读失败阶段；项目中仅入口自检使用 | 否 |
| 仓库原子操作 | 单仓库单记录锁内原子 | 否，不能覆盖跨仓库与服务登记 |
| 候选失败清理 | 只清本轮部分新增节点 / 关系，且不清主信息 | 否 |
| 活动快照 | 只隔离概念图服务会话期读取 | 否 |
| 节点失效读取过滤 | 可让关系 / 索引普通读取隐藏旧端点 | 否，隐藏的关系和索引仍可能残留 |
| 快照恢复 | 只返回隔离候选评估，明确不允许替换运行期仓库 | 否 |

结论：事务、补偿和活动图失效隔离三条路径当前都不足。至少要补齐一条可证明“失败后无公开半结构”的正式路径，并覆盖所有写入方。

## 6. 第一版默认裁决

1. 四根永久拒绝物理删除。
2. 只有当前生命周期为退役的普通概念可生成删除候选。
3. 第一版保守要求零当前有效实例支持、零全部外部有效关系；不在没有协议时猜测“可替代引用”。
4. 删除候选是调用期值式 DTO，至少带基准活动版本、目标生命周期关系与版本、节点 / 主信息句柄、全部相关关系句柄、候选直接边和验证摘要；不持久化为第四生命周期阶段。
5. 下位重挂必须复用现有固定点与传递约简算法，不另写领域局部移动算法。
6. 非权威用途缓存、显示、日志、SQL 和数量快照不得阻止或授权删除；缓存应按活动版本淘汰或显式清理。
7. 节点 `记录状态::已删除` 仍是物理删除表达，不新增“已删除”生命周期状态节点。
8. 普通读取因端点失效返回空，不等于关系、主信息或索引已清理；验收必须读取各自正式清理证据。
9. 等价概念未来重建必须获得新节点句柄，历史审计或缓存不得复活旧句柄或阻止新实例形成当前概念。
10. 在端点冻结、跨仓库提交和失败隔离形成前，不得缩小为“按顺序多次调用删除入口”并宣称安全。
