## 2. 输入入口与公开结果

协议模块导出 SDK 无关的四路流配置、设备身份、内外参、图像平面、同步帧、采集状态和抽象帧来源。默认请求固定为 RGB8 1280x720、Depth Z16 848x480、左右 IR Y8 848x480、30 FPS，并在入口拒绝零尺寸、零帧率、未知格式、零超时和错误流索引。

真实采集只通过 `创建并打开D455相机采集器` 形成 `D455帧来源`；公开接口不暴露 RealSense 类型、SDK 对象、裸缓冲区视图或线程。上行桥返回值式结果、只读材料共享所有权、缓存句柄和既有标量 `运行消息`，消息固定为 `只读材料说明=true`、`承载机器事实=false`。

## 3. 核心处理与结构承载

```text
显式真实相机或模拟帧来源
-> 同一 frameset 的 RGB / Depth / 左 IR / 右 IR
-> SDK 生命周期内复制字节、内参、外参、深度比例和时间材料
-> D455 专用缓存准备未发布候选
-> 候选能力读回
-> 构造标量运行消息
-> 确认发布
-> 普通编号版本读回
-> 返回非权威材料引用。
```

缓存以进程内条目表承载未发布候选和已发布材料，执行条数、总字节、单帧字节和材料年龄四重上限。候选不可默认构造、不可复制；移动时源能力立即清空，缓存同时复核所属实例、完整句柄和能力编号。普通读取只接受已发布材料，精确撤销按编号和版本删除本次条目并扣减精确字节数。

RealSense C++ 头 `rs2.hpp` 在当前 MSVC 真模块中因 `rs2_option_value` 类内 pack 切换触发 C3614。采集器因此改用同版本 SDK 的窄 C API 头完成相同设备、profile、frameset、标定和资源生命周期语义；该变化没有扩大公开接口。工程只在 x64 且 `EnableD455RealSense=true` 时编译真实采集模块，Debug / Release 分别链接并复制 RealSense 与 LZ4 对应运行库。

## 4. 准入与失败收口

无设备、多 D455 未指定序列号、指定设备不存在、非 D455、配置不支持、有限等待超时、设备断开、四路缺失、布局 / 步长 / 字节数 / 时间 / 标定无效、缓存超限 / 满载 / 过期和默认构建未启用真实采集均为逻辑内返回，不留下新可读材料或成功消息。

候选能力不匹配、候选读回与写入材料不一致、确认后普通读回失败、消息引用悬空、字节计数失真和 SDK 内部调用矛盾归为追根因结果。上行桥在发布前异常时按完整句柄精确撤销；发布后读回异常也不返回成功消息并精确撤销。采集器请求停止后拒绝新采样，关闭和析构都停止管线并释放帧、profile、配置、管线、设备和上下文资源。
