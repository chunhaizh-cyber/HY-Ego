## 4. 字段映射矩阵

### 4.1 当前概念签名载体

当前 `概念签名材料` 只有根类别、签名版本和约束组。每个约束含维度、角色序号、定义节点、值类型、节点类型值、I64、VecI64 和 VecU64。通用值类型枚举包含 `稳定句柄`，其值复用 `定义节点`；但当前因果专用维度只有目标类型、改变前值、改变后值和来源动作类型。证据：`海中鱼巣/领域/概念图算法.h:18-97`。

当前 `构造因果概念候选输入` 只从一个因果实例的一条来源动态构造：

```text
因果动态目标类型，角色 0，节点类型值。
因果动态改变前值，角色 0，I64。
因果动态改变后值，角色 0，I64。
可选因果来源动作类型，角色 0，节点类型值。
```

它把被改变目标、前值节点、后值节点和可选来源动作完整句柄放入 `定义材料组`，但这些句柄不进入签名身份，也没有原因 / 结果两组角色。证据：`海中鱼巣/领域/概念图服务.h:1176-1214`。

### 7.1 生命周期转换与活动版本

`转换概念生命周期` 在一把活动图独占锁和图写锁内校验候选、重挂生命周期关系并读回，然后把原 `活动快照_->活动版本` 放入结果；函数内不发布新快照、不递增活动版本。证据：`海中鱼巣/领域/概念图服务.h:2168-2267`。

因此：

```text
活动图版本相同
不推出
概念生命周期未变化
```

只以活动图版本包围多次签名 / 生命周期读取会漏掉生命周期关系变化。

### 7.2 删除候选专用完整当前输入

`概念完整当前输入材料` 是公开值类型，包含活动版本、四根、全部签名、概念内部关系和全部生命周期；它作为 `概念删除影响候选` 的字段公开返回。证据：`概念图服务.h:311-390,444-456`。

当前非自检生产代码中，唯一组装位置是 `读取概念删除影响候选`：

1. 在一把活动图共享锁内复制活动快照，并逐签名读取生命周期，形成完整输入（`:1262-1366`）。
2. 释放锁后执行直接边与根可达重算（`:1368-1458`）。
3. 返回前重新取得共享锁，逐项比较活动版本、根、关系、签名、全部生命周期、目标节点和目标相关关系（`:1460-1466,3658-3714`）。

该路径不能作为通用当前概念视图复用，原因是入口同时要求：

```text
目标是非根活动概念。
目标生命周期为退役。
当前支持数量为 0。
当前外部引用数量为 0。
调用方提交删除候选预算与预期活动版本。
```

双锁 `概念图冻结上下文` 与 `尝试冻结概念图` 位于 private 区，只供友元安全删除编排器使用；不能被普通映射调用方取得。证据：`概念图服务.h:2543-2574`。

### 7.3 分层模块边界

当前 `服务.概念图结构.ixx` / `数据操作.概念图结构.ixx` 只提供四根角色复核、实例支持概念、概念上下位、概念定义材料的创建和关系值式读取；没有活动版本 + 节点 + 签名 + 生命周期的通用原子视图，也没有因果模式到概念签名的匹配入口。

当前性结论：删除专用路径证明“捕获完整输入 + 锁外计算 + 返回前精确复核”的技术骨架可复用，但当前公开语境与所有权不适合概念映射；#284 必须以“缺通用原子只读活动概念视图”收口。
