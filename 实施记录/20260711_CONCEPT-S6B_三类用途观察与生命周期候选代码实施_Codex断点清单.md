## 2. 核心处理

1. 为统计服务新增 `概念用途观察` 命名空间组合和一般、因果预测、因果行动三类用途枚举，以完整命名空间、完整概念句柄和用途类型组成哈希键。
2. 独立共享互斥量保护概念用途缓存；读单项和同概念三类快照使用共享锁，首次写入、原子累计和清空使用独占锁。同键时间倒退、I64 时间上限、次数或有符号效用溢出均在写入前拒绝。
3. 概念图服务重读当前活动版本、活动签名和生命周期后才提交观察；一般用途效用固定为 0，预测和行动用途只接受活动因果概念并保持独立累计。
4. 生命周期复核候选组合当前阶段、三类观察、活动实例支持和外部正式关系。根、支持、外部引用、独立正向效用或近期观察只形成活性依据；规则版本变化返回材料不足。
5. 候选只输出保持活跃、建议冷却、建议退役复核、建议重新激活或材料不足，不写生命周期关系，不创建需求，不删除概念。

## 3. 结构承载

```text
用途观察：统计服务进程内 unordered_map，完整命名空间 + 完整概念句柄 + 用途类型键控。
并发边界：概念用途专用 shared_mutex；同键更新在线性化点内完成。
权威身份与阶段：继续由概念活动快照和概念生命周期关系承载，缓存不复制也不覆盖。
活动支持与外部引用：每次候选读取时从当前活动快照和关系仓库值式重读，不写缓存事实。
复核候选：调用期 DTO，只承载规则版本、计数、最近时间、效用候选、支持 / 引用数量和建议。
```

## 5. 入口拒绝与失败收口

| 场景 | 分类 | 结果 |
| --- | --- | --- |
| 命名空间无效、可持久化、句柄 / 用途 / 时间无效 | 逻辑内返回 | 不写缓存 |
| 一般用途携带非零效用，或非因果概念提交预测 / 行动用途 | 逻辑内返回 | 不写缓存，不生成该用途材料 |
| 时间戳倒退、次数或效用将溢出 | 逻辑内返回 | 原记录保持不变 |
| 规则版本与缓存命名空间不一致 | 逻辑内返回 | 返回材料不足候选，不改变生命周期 |
| 缓存缺失且没有其它活性依据 | 逻辑内返回 | 返回材料不足候选 |
| 前置通过后缓存记录、活动生命周期或候选组装内部不一致 | 追根因解决 | 停止 S6B 依赖路径，不用残缺材料继续裁决 |
