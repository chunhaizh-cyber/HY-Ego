## 4. 关键裁决

```text
1. 第一版不扫描场景自动切段；调用方提交观察回合身份、回合内非零片段序号和动态组。
2. 一个片段内动态同场景 / 主体，按时间戳和完整句柄排序；回合内片段按显式序号排序。
3. 同一来源动态不得跨片段重复；服务不合并片段，也不在片段内部搜索任意子序列。
4. 特征值由特征服务解析唯一语义特征；目标非特征值 / 类型不支持 / 过期逻辑返回。
5. 当前有效特征值归属零 / 多、实例槽位模板定义零 / 多属于内部不变式破坏并追根因。
6. 完整结构键裁决相等，哈希只分桶；实例场景 / 主体 / 时间 / 动态 / 回合身份只保留为证据。
7. 二次特征角色只有 #231 证明稳定时入键，否则版本 1 统一为空。
8. 一个完整片段只形成实例候选；至少两个不同回合各有同键片段才形成重复聚类。
9. 重复聚类至少一步有有效来源动作时才形成运动基元候选。
10. 因果按同回合显式窗口顺序统计：原因出现才计机会，结果在后计正结果，否则计负结果。
11. 概率为 floor(结果数 * 10000 / 机会数)，机会非零即可返回；不设可靠性、置信度或自动升格阈值。
12. 概率与结果效用分开，#235 才复核用途观察幂等和方法学习写入权。
```
