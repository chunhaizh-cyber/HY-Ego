## 2. 输入入口与公开结果

结构写入会话新增只返回布尔值的 `节点是本会话候选(节点句柄)`。入口只在当前线程、写入中、未决定且无首次失败时读取本会话节点候选写集；不返回候选对象，不改变提交准备视图或事务 ABI。

存在和状态原子服务新增三类由业务服务私有构造的规格：

```text
虚拟存在写入规格：幂等主键、预期来源类型、已发布 / 本会话候选来源阶段。
抽象状态写入规格：幂等主键、I64 状态值。
候选主体实例状态写入规格：幂等主键、已发布场景、发生时间戳、I64 状态值，主体类型固定为存在。
```

存在业务服务形成已发布来源或本会话候选来源虚拟存在规格；状态业务服务形成抽象状态、既有已发布主体实例状态和候选主体实例状态规格。公开业务面不接收、返回或比较原始事务令牌、许可、仓库、锁、写入会话或执行器。

## 3. 核心处理与结构承载

存在场景数据操作新增 `写入虚拟存在_会话`。入口复核规格完整性、来源阶段、当前节点类型和主信息，在调用方会话内创建主信息候选、存在节点候选、主键绑定和来源引用关系，并以完整句柄读回；入口不取得许可、不调用执行器、不请求提交、不释放许可。

状态动态数据操作新增 `写入抽象状态_会话`、`写入实例状态_会话` 和 `写入候选主体实例状态_会话`。已发布主体路径要求场景和主体都不是本会话候选；候选主体路径要求场景已发布、主体是本会话新建的存在节点候选。抽象状态继续由状态主信息、状态节点、主键和 I64 槽位承载；实例状态继续由状态主信息、状态节点、主键、状态值 / 时间戳 / 幂等编号槽位，以及运行期临时、主体引用和场景引用三条关系承载。

本次没有新增仓库记录状态、关系类型、索引类型或事务 ABI。所有写集仍由既有不透明结构写入会话记录并由唯一外层执行器统一确认或逆序撤销。

## 4. 准入、兼容与失败收口

无效主键、无效句柄、错误来源类型、已发布 / 候选阶段混用、错误主体类型、候选场景、无效时间戳和版本漂移均在相应第一笔参与写入前拒绝。参与写入后的内部不一致会使会话进入失败状态，外层不能请求提交；外层未请求提交时，本次主信息、节点、主键、I64 和关系写集统一撤销且物理数量与可读结构回到前态。

既有 `创建虚拟存在`、`创建抽象状态` 和 `创建完整实例状态` 立即入口复用相同结构写入核心，保持同键幂等、异义冲突、版本漂移和并发单提交语义。旧无令牌 / 带令牌兼容入口未删除，既有生产调用未迁移。
