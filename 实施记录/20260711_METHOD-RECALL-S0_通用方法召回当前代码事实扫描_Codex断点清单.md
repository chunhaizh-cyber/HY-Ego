## 2. 总结论

1. 当前没有从需求目标状态、任务上下文或结果能力反查方法首的公开入口。`读取方法候选材料` 必须先拿到一个已知方法首句柄，因此它是单方法结构读取，不是通用召回。
2. `登记方法` 只创建裸方法节点，`创建方法首节点` 再绑定方法首角色和虚拟存在；当前没有方法根、正式登记表、方法首枚举、注销入口或可用于恢复重建的权威注册集合。
3. 方法条件节点和结果节点各自只绑定一个新建场景。场景服务没有条件规格、结果能力、目标状态、特征约束、二次特征或初始状态集合的专用写读入口；当前调用点也没有向这些场景写入可判定规格。
4. `读取条件结果对候选` 只复核调用方显式给出的条件节点和结果节点同属一个方法首，并返回调用期 DTO；它不持久化条件结果对、不判断条件、不证明结果能力，也没有实现 `条件结果对` 角色节点。
5. #190 已提供经特征服务封口的精确比较、I64 值域和稳态待复判候选；这些入口当前没有被方法召回调用，且不能直接升格为条件满足、结果能力或排序事实。
6. #192 已提供直接父需求 / 子需求读回；需求层级当前只表达拓扑，不是方法优先级。任务承接材料可提供来源需求、运行场景、目标状态、虚拟存在和创建状态，但当前没有独立可读的任务授权关系。
7. `任务服务::选择任务方法` 只检查任务承接壳、目标节点类型和当前尚无选择方法；它不复核方法首角色、结果目标匹配、条件可满足、动作入口、方法生命周期或显式任务授权。正式召回完成后仍必须由任务服务独占提交任务选择方法关系。
8. 索引仓库是 `uint64 -> 单节点句柄`，无枚举、无多值键、无来源版本和重建接口。现有稳定动作键索引目标是动作入口，不能改义为结果能力到多个方法的索引。
9. 当前没有方法候选缓存、结果能力派生索引、确定性评分 DTO、并列策略、缓存失效版本、清空重建或恢复接线。控制面板方法树也只从已选任务方法取得真实根，不是方法注册或召回入口。
10. 方法服务没有聚合快照版本或内部锁；候选材料由多次仓库读取拼接。各仓库单次读写受锁保护，但跨节点、关系、状态和索引的候选读取不是同一原子快照。
11. 方法首没有活跃 / 失效生命周期。动作入口虽有 `已失效` 枚举，但没有公开状态迁移入口；节点失效后索引只在读取时返回空，旧主键项不会自动删除，也没有方法服务清理和重建路径。
12. 当前仓库快照服务只评估隔离候选，不执行恢复；#191 只覆盖特征值结构化材料和只读恢复候选。方法注册集合、规格、稳定动作键索引和方法派生缓存均没有启动恢复来源。

## 5. 条件、结果与规格事实

### 5.1 条件和结果场景是空规格壳

1. `创建方法条件节点` 位于 `方法服务.h:132`，创建一个方法条件角色节点和一个新场景，只写条件节点到场景的引用及方法首到条件节点的模板关系。
2. `创建方法结果节点` 位于 `方法服务.h:158`，采用相同结构建立结果节点和新场景。
3. 场景服务公开入口只有创建场景、接纳存在、收纳实例状态、收纳实例动态和场景有效性，见 `场景服务.h:16-52`。
4. 当前方法创建和验收调用点只读取条件 / 结果场景句柄，没有写入目标状态、特征约束、二次特征、条件参数、结果初态或能力方向。
5. 动作输入 / 输出规格同样只绑定场景句柄，见 `方法服务.h:505-530`；它们与方法条件 / 结果场景是不同关系，当前也没有规格内容读取集合。

因此，现有场景只能证明“方法结构拥有一个场景句柄”，不能证明场景已成为可判定条件或可召回结果能力。

### 5.2 可复用但尚未接线的材料

1. #190 在特征服务形成 `比较特征值候选`、`生成I64值域候选` 和 `生成稳态候选`，见 `特征服务.h:570-688`。
2. `读取宿主概念特征约束组` 已能返回结构化精确约束，见 `特征服务.h:953`，但它服务当前概念签名，不自动成为方法条件或结果规格。
3. 需求和任务目标均为状态句柄，不是 I64；未来结果能力应先引用目标状态或结构化特征状态规格，再按特征服务读取值材料。
4. #190 的值域和稳态均是待复判候选，不得作为方法可用、条件满足或排序成功事实。

### 6.2 当前授权缺口

当前 `任务服务.h` 没有独立任务授权关系的公开写读入口。`选择任务方法` 也没有检查任务生命周期当前态、方法首角色、候选适配、条件 / 结果规格、动作入口或授权策略。`方法服务::读取动作执行桥请求材料` 只检查任务节点类型、有效方法首、任务到方法的普通引用以及主动动作入口，见 `方法服务.h:476-500`。

后续选择提交必须保持以下权力边界：

```text
方法召回服务：只读输出候选、复判和排序材料。
任务服务：复核任务承接、任务授权、任务当前态和已确认候选后，独占写入任务选择方法关系。
方法服务：在任务方法关系和执行授权链成立后提供动作请求与执行能力。
```

## 7. 索引、排序与生命周期风险

### 7.1 当前索引不能复用为结果能力索引

`索引仓库.h:17-20` 只有绑定单主键、按主键查单节点、删除主键和有效数量。`索引仓库.cpp:10-20` 在同一键已有不同节点时拒绝绑定，因此无法表达“一种结果能力 -> 多个方法”。

稳定动作键由 `方法服务::登记动作入口` 绑定到动作入口节点，作用是找到已知方法的执行入口。复用该键空间会把动作入口身份与结果能力混为一谈，必须禁止。

建议首轮新增方法服务专用的非权威派生多值索引或独立 `方法召回索引`，索引项至少携带：

```text
结构化结果能力签名。
方法首完整句柄。
结果节点完整句柄。
方法登记版本 / 生命周期版本。
结果规格版本。
索引规则版本。
```

命中后必须回到方法服务重读权威登记、结果规格、生命周期和完整签名。哈希只缩小候选，不裁决相等；清空索引不改变任何方法、任务或需求事实。

### 7.2 排序与选择当前为空

当前没有候选评分、匹配等级、并列策略或排序入口。关系仓库按顺序号 / 关系编号或节点编号返回结构集合，只保证结构读取顺序，不是方法优先级。

建议排序固定为调用期值式材料，并按以下顺序逐步引入：

```text
结果能力结构匹配等级
-> 条件复判结论
-> 已确认策略 / 证据材料
-> 完整方法句柄确定性收口
```

没有唯一候选或明确授权策略时不得写任务选择方法关系。最终句柄收口必须比较仓库编号、节点编号和版本号，不只比较节点编号。

### 7.3 生命周期、失效与半结构风险

1. 方法首没有生命周期状态和公开失效入口。
2. 动作入口状态含 `已失效`，但私有绑定入口只在创建时使用，没有公开转换。
3. 节点删除后，索引读取会因目标句柄失效返回空，但旧主键映射仍留在索引表中；后续绑定同键新入口会因旧条目存在而失败。
4. 方法创建、条件 / 结果创建和动作入口登记均跨多个服务逐步写入，没有事务回滚或失效隔离。尤其稳定键旧条目残留时，动作节点、状态和关系可能已创建，最后索引绑定失败。
5. 方法候选读取由虚拟存在、条件组、结果组和动作入口多次读取拼装，没有共同来源版本；并发失效或关系变化时只能返回空或混合时点材料。
6. `索引仓库::按主键查节点` 在持有索引共享锁时调用节点仓库有效性读取。未来新增多值索引、失效和重建前必须按多线程防锁规范重新设计锁顺序，不能复制当前嵌套方式扩展。

这些是后续设计必须先解决的结构问题，本 S0 不修改代码，也不把它们改写为日志或事后扫描修复路径。
