## 2. 当前代码事实

1. `任务业务服务::提交实际结果并完成` 已在一个不透明会话内绑定唯一实际状态并迁移到已完成。
2. `需求业务服务::提交独立结算` 已在独立会话内记录来源任务、目标 / 实际状态、动态和可选因果引用。
3. 当前目标结算口径是目标状态值与实际状态值相等；因果只作证据。
4. `状态业务服务` 和 `动态业务服务` 可返回完整值式状态 / 动态材料；实例动态顺序合同为 `1 / 2 / 3 / 4`。
5. 当前没有跨服务无令牌结果治理组合器，需求服务也没有公开只读目标达成裁决。
6. 当前回执协议仍是版本 1 基础句柄组；#224 新协议和完整来源执行请求只是依赖门控预期。
7. 传统任务管理上行桥仍读取旧服务并生成治理消息，不是新写入所有者。
