## 2. 总结论

1. `任务执行冻结请求` 与新版 `任务结果回执材料` 足以形成非权威观察回合的完整值式身份候选；协议能区分精确重复、同幂等键异义、任务序号倒退和过期材料。
2. 当前没有权威“任务尝试节点”，也没有片段序号字段。任务序号和批次编号是协议排序材料，不能直接替代同一观察回合内的片段顺序。
3. 实例动态可按完整句柄、幂等主键、场景、主体、目标、前后状态、来源动作、时间戳和完整关系句柄读回；来源动作可与冻结请求中的动作入口对齐。
4. 当前没有“动态 -> 任务 / 回执 / 结果”的权威关系。任务、方法、动作入口和动态只能由冻结请求、新版回执及正式服务读回共同复核，不能从单个动态反向推断任务事实。
5. 状态、动态、任务、需求、方法、特征和二次特征的公开业务面不接受原始结构事务令牌；对应数据操作分别取得自己的共享许可。跨服务顺序读取没有统一快照身份，无法严格排除混合快照。
6. 特征服务支持“实例槽位 -> 宿主 + 语义特征定义 + 当前值”和“宿主 + 语义特征定义 -> 唯一槽位 + 当前值”；但没有公开业务入口从特征值节点直接反查唯一槽位和语义特征定义。
7. 二次特征组成关系的连续顺序号 `0..N-1` 只证明结构位置，没有跨实例稳定语义角色证据；#232 版本 1 的二次特征角色部分必须固定为空。
8. 旧 `动态服务.h` / `因果服务.h` 候选只做单动态 I64 差值或单动态完整性判断，不能证明重复片段、运动基元或因果概率；调用点集中在入口和旧自检，不是现行无令牌运行期业务链。
9. 统计服务的概念用途材料是可清空、不可持久化、不可裁决业务事实的缓存。现有任务结果链没有写入该缓存；当前也没有方法学习、训练、成功率或效用更新入口。
10. 唯一运行期业务装配尚未持有 #224 任务执行调度组合器或 #225 任务结果治理组合器；现行第八类路由仍是既有 `完成任务并独立结算`。该接线所有权必须由 #232 重设计裁决，本轮不修复。
11. `服务.动态模式`、`算法.动态模式`、`自检.动态模式`、`自检.动态聚类`、`服务.因果模式`、`算法.因果模式`、`自检.因果模式` 和阶段 630 / 640 / 650 当前均不存在或未登记。

因此：#231 可以完成并归档；旧 #232 不能直接实施，必须先重做流程图、详细设计和计划。

## 5. 特征值到语义特征的实际路径

### 5.1 已有公开能力

`海中鱼巣/领域/服务.特征.ixx:124` 至 `:155` 公开：


`读取实例特征槽位` 返回宿主、特征定义、实例槽位、主信息及宿主 / 模板完整关系证据；`读取实例槽位当前值` 返回特征值节点、主信息、当前值关系、原始类型、原始值版本和 I64 / Vec 原始材料。

`读取宿主特征状态` 在特征体系数据操作的一次共享许可内读取唯一槽位和当前值，是当前已有的同域批量读取边界。

### 5.2 零 / 一 / 多路径

| 起点 | 零路径 | 唯一路径 | 多路径 |
| --- | --- | --- | --- |
| 实例槽位 -> 模板定义 | 无模板按类型不匹配返回 | 返回唯一模板定义 | 多模板为内部不一致 |
| 实例槽位 -> 宿主 | 不等于 1 即内部不一致 | 返回唯一宿主 | 多宿主为内部不一致 |
| 宿主 + 定义 -> 实例槽位 | 未找到 | 返回唯一槽位 | 多槽位为内部不一致 |
| 实例槽位 -> 当前特征值 | 未找到 | 返回完整值与版本 | 多当前值为内部不一致 |

`海中鱼巣/领域/数据操作.特征体系.ixx:929` 的主键特征值反向读取辅助可以在数据操作内部验证唯一来源槽位，但它位于私有实现区，特征业务服务没有对应公开入口。#232 不得直接调用私有数据操作辅助、特征值服务或仓库。

正式缺口是：给定完整特征值节点句柄，公开业务服务不能一次返回唯一实例槽位、宿主、语义特征定义、当前值关系和原始版本。#232 重设计必须新增窄公开读取合同，或先建立服务专用不透明只读会话 / 批量读取前置；零归属不得伪装普通未找到，多归属不得任选第一项。

## 8. 轻量因果、用途和方法所有权

### 8.1 轻量因果

轻量因果业务服务创建前经动态业务服务读取完整动态；数据操作又在自己的许可内复核来源动态和主信息。创建结果只保存因果引用身份和主键，不建立来源动态关系。

因此从轻量因果句柄只能读到因果引用身份，不能反查创建时来源动态；轻量因果不适合作为 #234 原因 / 结果模式证据所有者。

### 8.2 概念用途缓存

`统计服务.h` 的用途类型固定为一般、因果预测、因果行动。缓存键是：

```text
缓存命名空间 + 活动概念完整句柄 + 用途类型
```

材料保存观察次数候选、最近观察时间戳和累计结果效用候选，并显式要求：

```text
不可持久化
不可裁决业务事实
可按命名空间清空
```

概念图服务只在活动概念、活动版本、概念类别和生命周期成立时委托统计服务写缓存。现有调用点位于入口和概念生命周期 / 删除自检；没有 #225 任务结果或需求结算到用途缓存的正式生产接线。

### 8.3 方法系统

方法业务服务当前拥有登记、规格、生命周期和动作桥；方法召回业务服务拥有非权威候选、复判、排序、唯一建议和向任务服务提交选择请求的值式适配。

全仓未发现方法学习、训练、成功率或效用更新入口。现有方法排序分值来自规则化条件 / 结果匹配，不是任务结果驱动学习。未来方法学习必须在 #235 后另建所有权和授权设计，不能让统计缓存或 #234 纯值概率候选直接修改方法事实。

## 10. HYSNAP01 与运行期装配影响

HYSNAP01 当前设计明确：线程、队列、消息、回执、幂等表、缓存、显示、SQL、日志和上一候选图不持久化为机器事实。恢复时派生方法索引、统计缓存、召回临时候选、执行请求和回执队列均清空重建。

因此：

```text
权威动态 / 状态 / 特征值材料 -> 由四仓库和特征值侧表的现有权威段承载
冻结请求 / 新版回执 -> 瞬态非权威材料，不进入 HYSNAP01
#232-#234 窗口、聚类和概率候选 -> 可丢弃纯值候选，不进入 HYSNAP01
统计用途缓存 -> 可清空缓存，不进入 HYSNAP01
```

唯一 `运行期业务装配` 当前拥有七个数据操作、十四个业务服务、六个组合器和一个业务操作组合器，但没有任务执行调度组合器、任务结果结算组合器或两条新版线程路由成员。现行运行期请求路由第八类仍调用 `运行期业务操作组合器::完成任务并独立结算`。

#232 重设计必须明确新版回执如何进入唯一运行期装配，或明确该接线由另一个前置计划拥有；不能在 #232 自检中构造局部对象后宣称生产链已接通。
