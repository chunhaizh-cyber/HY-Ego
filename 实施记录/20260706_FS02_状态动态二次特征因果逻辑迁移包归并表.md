## 4. 包级迁移口径

### FS-02-A 状态身份与状态读取

保留行为：

```text
状态节点必须能按稳定身份读取。
状态读取必须复核节点有效性和状态类型。
实例状态材料必须能读出主体、特征、值和发生时间戳。
```

删除行为：

```text
不迁移旧状态主信息裸字段作为新项目权威状态结构。
不使用运行期裸指针或旧主键解析绕过节点句柄。
```

新增约束：

```text
实例状态必须有存在主体、场景归属和发生时间戳。
高级服务需要特征状态材料时必须通过特征服务。
```

验收方式：

```text
有效实例状态可经状态服务读回主体、场景、值和发生时间戳。
无效句柄、错误节点类型、缺材料状态返回拒绝或空结果，结构数量不变。
```

### FS-02-B 实例状态创建与场景收纳

保留行为：

```text
旧系统“在场景中记录某主体某特征某值”的能力保留。
状态创建后可形成后续动态证据材料。
```

删除行为：

```text
不迁移旧基础信息全链登记、旧场景状态索引裸字段和运行日志阶段文本。
状态服务不得隐式越权创建动态事实。
```

新增约束：

```text
创建实例状态由状态服务入口完成。
场景收纳由场景服务或确认的场景关系入口完成。
动态生成必须走显式动态服务入口。
```

验收方式：

```text
创建后状态节点、状态值、时间戳、存在主体和场景归属可读。
缺场景、缺主体、缺时间戳或缺值时拒绝，状态和关系数量不变。
```

### FS-02-C 状态主体 / 特征 / 值材料读取

保留行为：

```text
读取主体、特征、状态值和排序时间用于后续动态、二次特征和需求结算。
```

删除行为：

```text
不把旧特征值变体、Vec 指针材料或显示变化标记作为通用状态事实。
```

新增约束：

```text
特征材料经特征服务读取。
状态值材料必须能追溯到状态节点或特征服务确认材料。
显示变化缓存只能进入显示只读边界。
```

验收方式：

```text
状态材料可读且来源清楚。
显示缓存缺失或变化不影响需求、任务、方法、动态或因果事实。
```

### FS-02-D 状态生命周期、场景读取与非权威清理

保留行为：

```text
按场景读取状态集合。
状态生命周期删除和活动状态清理需要有边界。
被动态或二次特征引用的状态不得被清理成悬空证据。
```

删除行为：

```text
不使用清理器自动修复权威结构。
不把清理日志、问题快照或计数作为业务事实。
```

新增约束：

```text
删除必须走节点仓库生命周期和领域服务准入。
索引清理、统计和快照属于非权威缓存 / 诊断材料。
```

验收方式：

```text
删除后旧句柄失效且场景状态关系不可读。
被引用状态拒绝清理或进入明确后续治理路径。
```

### FS-02-E 状态变化触发实例动态

保留行为：

```text
同场景、同主体、同特征的状态变化可以形成实例动态证据。
动态必须包含目标、前值、后值和发生时间戳。
```

删除行为：

```text
不由状态服务隐式创建动态。
不迁移 SQL 显示同步、日志阶段文本或旧自动聚合。
```

新增约束：

```text
状态变化生成动态必须是动态服务显式入口或被确认的服务协作入口。
无前状态时不得伪造动态。
```

验收方式：

```text
有效状态变化形成动态节点并可读回前后值、目标和时间戳。
缺前状态、主体 / 特征不一致或缺时间戳时拒绝，动态数量不变。
```

### FS-02-F 动作动态、方法动作动态与来源绑定

保留行为：

```text
方法执行或动作入口可作为动作动态来源候选。
动态可记录来源方法、动作入口角色、输入场景和输出场景。
```

删除行为：

```text
不把旧动作名文本、函数名、日志标签或方法首节点冒充动作身份。
不直接迁移旧方法动作动态函数体。
```

新增约束：

```text
来源动作第一轮必须指向动作入口角色节点。
方法执行权限仍来自任务授权、任务承接和任务选择方法。
```

验收方式：

```text
动作动态来源关系可读。
来源动作无效或不是动作入口角色时拒绝。
显示名称变化不改变动态事实。
```

### FS-02-G 动态聚合和运动基元候选

保留行为：

```text
场景动态集合、位置变化片段和观察关系片段可作为后续候选材料。
```

删除行为：

```text
不把候选直接升级为稳定方法、稳定因果或动作入口事实。
不接外设、体素或控制面板观察链。
```

新增约束：

```text
候选只读；稳定规则必须经二次特征、方法系统、因果评分和确认专项。
```

验收方式：

```text
候选读取不新增节点或关系。
候选输出仅作为后续专项输入，不改变需求、任务、方法或因果事实。
```

### FS-02-H 二次特征组成、来源和值材料

保留行为：

```text
二次特征作为长期基础信息节点。
二次特征可追溯来源状态、来源动态、主体客体和组成材料。
```

删除行为：

```text
不迁移旧二次特征主信息裸字段作为方向、差值、标量、离散语义、满足标记或置信度的权威事实。
不迁移旧概念刻度链、签名缓存和默认刻度为权威结构。
```

新增约束：

```text
二次特征只表达复合前置结构。
判断结论应由状态、特征值、动态、因果引用或后续确认结构承载。
```

验收方式：

```text
二次特征节点和组成关系可读。
缺来源状态或来源动态时不得返回有效状态迁移二次特征。
缓存清空不影响二次特征事实。
```

### FS-02-I 轻量因果引用和动态证据准入

保留行为：

```text
因果入口必须以动态证据作为准入材料。
动作结果验收、目标状态和实际状态可作为后续因果评估证据。
```

删除行为：

```text
不迁移旧因果主信息、因果模板主信息、证据计数或稳定度为第一轮权威因果事实。
不把单个因果引用解释为稳定因果结论。
```

新增约束：

```text
当前第一轮只允许轻量因果引用。
记录因果引用只用来源动态做入口准入证据，不持久化来源动态关系。
```

验收方式：

```text
有效来源动态可创建轻量因果引用。
缺来源动态时拒绝，因果节点数量不变。
```

### FS-02-J 因果模板、任务化和查询候选

保留行为：

```text
因果查询、任务投影、方法候选和自然语言说明可作为后续候选来源。
主动因果任务化必须有可回解动态证据、方法来源和状态变化端点。
```

删除行为：

```text
不把查询命中、自然语言文本、主键列表、证据计数或候选层级当作需求满足、任务生成或方法可执行事实。
不直接迁移旧因果模板生成算法。
```

新增约束：

```text
候选召回必须由需求、任务、方法或因果服务复判。
稳定因果结论、因果模板、任务化规则和事件恢复必须后续详细设计确认。
```

验收方式：

```text
查询只返回候选和缺失证据。
候选查询不得新增需求、任务、方法、结算或因果事实。
```

### FS-02-X SQL / 控制面板 / 显示镜像排除

保留行为：

```text
动态摘要、候选说明、自然语言文本和诊断快照可作为后续显示只读材料。
```

删除行为：

```text
不接 SQL 控制面板实时投影。
不让日志、显示、SQL 同步状态或控制台输出裁决机器事实。
```

新增约束：

```text
显示层只读结构，不写业务事实。
显示只读专项建立前，本轮只登记排除边界。
```

验收方式：

```text
本轮不创建显示或 SQL 代码。
后续显示专项必须证明只读，不参与结构写入或业务裁决。
```
