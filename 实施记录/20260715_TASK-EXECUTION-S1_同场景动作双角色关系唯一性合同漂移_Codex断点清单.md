## 3. 真实合同漂移

详细设计 TE-D6 和计划第 8 节第 5 项固定：

```text
动作入口输入场景 == 任务场景。
动作入口输出场景 == 任务场景。
第一轮不支持跨场景动作或场景迁移。
```

当前动作入口权威创建路径为：

```text
方法业务服务::创建动作入口
-> 需求任务方法数据操作::创建方法规格共用
-> 创建 引用(动作入口, 输入场景, 顺序号 1)
-> 创建 引用(动作入口, 输出场景, 顺序号 2)
```

`关系仓库::创建关系` 对任务生命周期证据和任务方法选择作专用处理；其余有效关系按以下键拒绝重复：

```text
关系类型 + 源节点 + 目标节点
```

该普通关系唯一性判断不包含顺序号。输入场景和输出场景相同时，第二条 `引用` 与第一条具有相同类型、源节点和目标节点，即使顺序号从 1 变为 2 也会被拒绝。`结构化创建关系` 随后把空句柄映射为 `结构写入状态::入口拒绝`，方法动作入口不能发布。

| 编号 | 设计合同 | 当前实际接口 | 影响 |
| --- | --- | --- | --- |
| IF-224-01 | 输入 / 输出场景都等于任务场景 | 动作入口写两条同类型、同源、同目标、不同顺序号的引用 | 第二条关系命中普通重复拒绝 |
| IF-224-02 | 顺序号 1 / 2 分别承载输入 / 输出角色 | 普通关系唯一键忽略顺序号 | 关系仓库不能同时表达同一端点的两个角色 |
| IF-224-03 | 冻结必须读回唯一当前动作及六字段动作桥 | 动作入口创建返回 `需求任务方法业务状态::入口拒绝` | 无动作、无选择，冻结链不能建立 |
| IF-224-04 | #224 在既定允许文件内闭合 | `核心/**` 与 `领域/数据操作.需求任务方法.ixx` 都是禁止文件 | 执行窗口不能改变唯一性或动作入口建模 |
