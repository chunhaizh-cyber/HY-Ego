## 1. 漂移入口

#211 执行窗口在 S0、写代码前确认：

```text
原计划假定：概念图服务::读取待命名请求(概念)
实际唯一入口：概念图服务::读取待命名请求(节点句柄, const 语素服务&)
原桥假定持有：const 概念图服务& + 邮箱&
实际所需持有：const 概念图服务& + const 语素服务& + 邮箱&
```

概念图服务本身不持有语素服务。没有语素服务，桥无法沿正式入口重读名称为空条件，也无法在请求读不到时区分“目标已有名称”和“当前请求不存在”。

```text
协议新增枚举 9 / 7 / 19 仍空闲。

工程模块登记方式未漂移。

```

## 3. 正式裁决

| 编号 | 裁决 |
| --- | --- |
| JY251-D1 | 命名上行桥只读持有 `const 概念图服务&`、`const 语素服务&` 和 `有界自我治理邮箱&`。 |
| JY251-D2 | 新桥模块允许 include `语素服务.h`，但不得修改概念图服务、语素服务或其它领域文件。 |
| JY251-D3 | 桥接收单个请求后调用现有 `读取待命名请求(请求.概念, 语素服务)`，不调用批量读取。 |
| JY251-D4 | 双参读取存在时比较概念、根类别和活动版本；不一致按正常漂移逻辑内返回。 |
| JY251-D5 | 双参读取不存在时调用 `语素服务::读取概念名称入口组`；非空返回目标已有名称，空返回当前请求不存在。 |
| JY251-D6 | 不新增概念图服务无语素参数重载，不让概念图服务保存语素服务，也不修改邮箱或线程所有权。 |

## 4. 非成功二分

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 双参读取无请求且名称入口为空 | 逻辑内返回 | 当前请求不存在，不提交邮箱 |
| 双参读取无请求且名称入口非空 | 逻辑内返回 | 目标已有名称，不提交邮箱 |
| 当前请求存在但根类别 / 活动版本漂移 | 逻辑内返回 | 不提交邮箱，不写结构 |
| 输入过期、重复或邮箱正常拒绝 | 逻辑内返回 | 保持既有队列和领域结构 |
| 当前同一材料下协议映射或邮箱结果自相矛盾 | 追根因解决 | 停止消息依赖路径，不由日志修复 |
