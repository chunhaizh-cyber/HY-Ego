## 2. 输入入口与公开结果

结构写入会话新增两个公开值式只读入口：

```cpp
std::vector<std::uint64_t> 读取节点主键组(节点句柄 节点) const;
std::vector<关系记录> 读取来源关系记录组(
    节点句柄 目标节点,
    关系类型 类型) const;
```

入口不接收、不返回原始令牌、许可、仓库、锁、候选或可变指针。无效句柄、错仓句柄、过期句柄、非法关系类型、跨线程调用和提交决定后调用均返回空值式材料，不改变机器结构。

## 3. 核心处理与结构承载

两个入口均先通过会话现有 `可继续写入()` 和句柄有效门禁，再使用会话已有同一独占令牌调用索引仓库或关系仓库现有带令牌只读入口。节点主键组完整返回全部当前主键，不自动选择、去重、修复或写回；来源关系组按目标节点和关系类型返回完整关系记录，保留关系编号、版本、状态、类型、源端点、目标端点和顺序号。

结构承载保持为索引仓库现有主键映射和关系仓库现有关系记录。本切片不新增事实类型、关系类型、二级索引、概念根、业务 DTO、写集、提交条件、撤销步骤或事务参与者。

自检证明显式撤销后主信息、节点、关系和索引均不可读且数量回前态；正常提交后完整主键组和正反向关系记录可读。只读入口不登记写集，不改变既有精确逆序撤销和提交行为。
