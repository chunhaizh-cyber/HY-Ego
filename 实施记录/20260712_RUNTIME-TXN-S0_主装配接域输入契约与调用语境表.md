# RUNTIME-TXN-S0 主装配接域输入契约与调用语境表

标准检查字段：调用方、输入字段、输入来源、上游是否保证有效、允许逻辑内返回、追根因触发条件、验证方式。下表“语境”对应调用方，“输入”合并表达输入字段与输入来源，“许可所有者 / 允许结构变化 / 非成功收口”共同给出有效性、返回二分和验证边界。

| 语境 | 输入 | 许可所有者 | 允许结构变化 | 非成功收口 |
| --- | --- | --- | --- | --- |
| 生产全新启动 | 参数、配置、初始化材料 | 唯一运行期上下文 | 通过领域服务初始化 | 写前拒绝或追根因，不平行创建上下文 |
| 生产恢复启动 | 已严格验证隔离候选、跨段屏障 | 同一运行期上下文类型 | 一次发布恢复后仓库组 | 屏障前不发布；失败保持旧运行期未建立 |
| 普通生产读写 | 已发布上下文租约 | 顶层调用一次取得共享许可并下传值式令牌 | 领域入口允许的结构变化 | 禁止无令牌重入 |
| 已发布身份删除 | 完整写集、不可伪造业务提交授权、独占结构许可 | 安全删除顶层编排 | 唯一提交点后顺序发布预构造包 | 不允许逐仓库公开删除后门 |
| 创建期补偿 | 本次调用精确创建集合 | 原创建领域服务 | 只撤销本次未发布结构 | 失败追根因并报告残留 |
| 自检夹具 | 隔离仓库与自检材料 | 自检模块 | 只影响隔离仓库 | 不接生产上下文 |
| 快照冻结 | 已发布上下文租约 | 仓库快照生产模块 | 许可内只复制值式材料 | 不冻结 #217 自检组 |
| SQL / 控制面板 | 已发布只读租约或值式快照 | 适配层 | 不写机器事实 | 失败只影响审计 / 显示 |

## 单一所有权裁决

```text
上下文类型只有一个。
全新启动和恢复启动是互斥分支。
每个进程只允许一次发布一个上下文实例和一个宿主租约源。
协调状态由上下文内部唯一拥有，四仓库与服务晚构造、早析构。
快照、线程、SQL 审计、控制面板不得自行创建协调器或仓库组。
```
