## 2. 仓库当前事实

1. 主信息、节点、关系、索引仓库均同时存在无令牌和带令牌入口；主信息 / 节点另有未发布候选、确认和撤销入口。
2. 四仓库各自持有读写锁，仓库带令牌入口负责令牌验证和本仓库读写。
3. 关系表继续是关系权威；关系二级索引尚未实现。
4. 普通删除和未发布候选撤销已形成不同底层入口，但服务层尚未统一使用同一抽象。
5. 结构事务协调模块负责许可签发 / 释放、活动表、结构锁和同线程重入拒绝。

## 4. 存在场景垂直链事实

1. `存在服务` 同时保留无令牌 `创建实际存在 / 创建虚拟存在`、带令牌 `创建虚拟存在候选` 和候选确认 / 撤销。
2. `存在未发布候选` 当前直接保存原始结构事务令牌。
3. 无令牌 `创建实际存在` 先创建主信息再创建节点，内部失败时没有统一精确撤销合同。
4. 无令牌 `创建虚拟存在` 在实际存在创建成功、来源关系失败时直接返回，可能留下已创建存在。
5. `场景服务` 当前只有无令牌创建 / 接纳 / 收纳入口，直接持有主信息、节点、关系仓库。
6. `世界服务::创建场景存在` 先创建实际存在再接纳；接纳失败时没有撤销新存在。
7. 大量入口、自检、初始化和领域服务调用现有无令牌存在 / 场景入口，不能直接删除兼容路径。

## 5. 架构偏差

```text
公开 / 组合层可见原始令牌，事务机制穿透业务层。
领域服务直接持仓库引用并同时承担业务准入、许可验证、候选管理和结构清理。
无令牌、带令牌、候选三种写路径的成功与失败合同不一致。
场景服务没有接域写入路径，存在与场景无法形成统一失败无残留垂直样例。
JY-315 性能基线会测量旧分层路径，不能继续作为当前首个切片。
```

## 6. 当前裁决

保留 `#257-#261` 作为后继性能 / 显示链，但整体增加新架构纠偏前置并重排阶段。先建立仓库合同、服务写入会话和存在场景垂直样例，再分批迁移服务，最后治理兼容路径；期间 `#257` 和 `#214` 均不得执行。
