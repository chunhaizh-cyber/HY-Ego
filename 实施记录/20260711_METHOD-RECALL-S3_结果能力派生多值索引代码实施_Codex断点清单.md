## 2. 实施内容

### 2.1 专用非权威索引

新增 `海中鱼巣/领域/方法召回索引.h`：

```text
方法结果能力键
方法结果索引项
方法结果索引重建候选
方法召回索引状态材料
方法结果索引读取材料
方法召回索引快照材料
方法召回索引实现<能力键哈希器>
```

能力键固定包含：

```text
方法召回索引规则版本
目标特征完整句柄
目标状态值材料
```

索引项冻结以下来源：

```text
方法首与结果节点
登记关系
生命周期关系与版本
结果特征关系
结果目标状态关系
结果规格规则版本
```

完整键相等负责身份裁决，哈希只负责分桶。同一键允许多个方法 / 结果项，桶内按完整方法句柄和结果句柄稳定排序并拒绝重复身份。

### 2.2 一次根扫描的来源聚合

`方法服务` 新增只读材料 `方法结果规格索引来源材料` 和入口：

```text
读取活跃方法结果规格索引来源组(状态, 特征)
```

该入口只取得一次登记共享锁并扫描一次登记根；每个活跃方法只取得对应的 64 条带规格锁，读取其结果节点与完整结果规格。无规格结果节点不进入来源组；只发布一半的规格、错误归属、重复项或内部结构不完整进入追根因收口。

锁序继续固定为：

```text
登记共享锁 -> 单方法规格条带锁
```

没有规格锁到登记锁的反向路径。状态服务与特征服务只被方法服务只读调用，不反向依赖方法服务。

### 2.3 候选构造、发布、读取和清空

候选构造连续读取两次完整来源组并比较全部索引项及来源版本；任一读取失败保持无效，前后不一致返回已过期，完全一致才标记当前。

候选分组、排序、重复检查和完整性检查均在索引锁外完成。发布临界区只执行：

```text
候选桶组 swap
键数量 / 索引项数量替换
发布版本加一
```

读取只在共享锁内复制桶或完整快照，解锁后验证值式副本。清空只交换空桶组、清零派生计数并推进发布版本，不调用领域服务、不修改仓库或方法事实。内部异常在释放索引锁后进入公共容错诊断。
