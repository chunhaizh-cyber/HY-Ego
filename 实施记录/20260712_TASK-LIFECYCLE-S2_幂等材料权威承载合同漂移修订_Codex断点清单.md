## 2. 权威承载裁决

### 2.1 幂等编号

| 项 | 裁决 |
| --- | --- |
| 用途 | 唯一识别一次任务生命周期迁移请求，支持跨后续换代的历史幂等读回 |
| 写入方 | 状态服务的带幂等材料实例状态候选入口 |
| 结构 | 目标实例状态主信息槽位 2 / 3，分别保存 uint64 低 / 高 32 位 |
| 读取方 | 状态服务同令牌只读入口；任务服务只能调用该入口 |
| 生命周期 | 与目标实例状态及生命周期历史项一致；版本 1 初态两槽均不存在 |
| 验证 | 两槽同时存在、组合后非零；单槽、任务内重复编号或读回不一致追根因 |

不得把时间戳、生命周期关系顺序号、关系句柄版本、节点编号、摘要、日志或普通 bool 解释为幂等身份。

### 2.2 完整迁移证据

任务服务以新实例状态为来源、`引用` 为关系类型，固定使用：

| 顺序号 | 角色 | 允许目标节点类型 |
| --- | --- | --- |
| 40 | 来源任务 | 任务 |
| 41 | 来源方法 | 方法 |
| 42 | 来源动作 | 方法 |
| 43 | 来源状态 | 状态 |
| 44 | 来源动态 | 动态 |
| 45 | 取消授权来源 | 任务或方法 |
| 46 | 动作停止证据 | 状态或动态 |

空角色不创建关系；非空角色恰有一条关系。所有证据角色禁止指向存在或场景，避免破坏状态服务既有主体 / 场景唯一读取。任务服务负责创建、读取和提交前删除；状态服务只管理状态核心和幂等双槽，不解释任务证据角色。

## 3. DTO 合同

`任务生命周期当前材料` 与 `任务生命周期历史项` 增加：

```text
可选幂等材料编号
可选前驱生命周期关系
来源任务 / 方法 / 动作 / 状态 / 动态
取消授权来源
动作停止证据
```

版本 1 的可选幂等编号和前驱关系为空；版本 2 以后必须非零且能精确找到版本 N 的前驱关系。旧历史结果和当前投影必须分开返回，不得把幂等命中的旧结果冒充当前态。

## 4. 幂等判定

```text
读取同任务完整生命周期历史
-> 按非零幂等编号查找
-> 无命中：再检查请求预期关系 / 版本是否当前
-> 唯一命中且前驱、目标、时间、七角色逐字段相同：返回原迁移结果 + 当前投影，不写结构
-> 唯一命中但任一字段不同：同键异义，写前拒绝
-> 多项命中：内部结构矛盾，追根因
```

不允许“幂等编号相同即同义”，也不允许只比较摘要。

## 5. 创建、提交与失败清理

```text
状态服务创建带幂等双槽的未发布状态候选
-> 任务服务创建七角色中非空证据关系
-> 任务服务创建版本 N+1 新生命周期关系
-> 完整预读回状态、幂等编号、证据和新关系
-> 唯一提交点：使版本 N 旧关系失效
-> 读回当前 / 历史
-> 状态服务确认候选
```

提交前失败按“新生命周期关系 -> 证据关系 -> 状态候选”逆序清理，旧关系保持有效。旧关系失效后不做版本倒退回滚；任何读回矛盾进入追根因并停止依赖路径。
