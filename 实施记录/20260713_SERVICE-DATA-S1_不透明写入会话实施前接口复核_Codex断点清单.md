## 2. 当前接口事实

`#263` 已形成 `#264` 所需十个独占结构化入口：

```text
主信息：候选创建、确认、撤销
节点：候选创建、确认、撤销
关系：结构化创建、严格删除
索引：结构化绑定、严格删除
```

索引严格删除真实签名为 `严格删除主键(主键, 预期节点, 令牌)`，因此写集必须保存“主键 + 预期节点完整句柄”，不能只保存主键。关系创建成功返回完整关系句柄。主信息 / 节点严格普通删除不属于候选撤销路径。

协调器的独占许可在 RAII 许可对象生存期内持有全域独占结构锁；其它线程取得共享许可必须等待，同线程重复取得许可会被重入门禁拒绝。四仓库构造链要求同时未接域或使用同一结构事务域，各结构化入口再复核独占令牌。因此候选隔离必须由“同域独占许可持续持有 + 会话不外泄”共同证明，不能只依赖句柄不导出。

## 3. 核心处理

1. 将会话从原候选设计的可移动语义收紧为不可默认构造、不可复制、不可移动。
2. 会话只能由执行器在取得独占许可的当前线程栈内直接构造，并以同步左值引用传入回调。
3. 精确写集固定为主信息 / 节点候选、完整关系句柄和“主键 + 预期节点完整句柄”索引绑定对。
4. 最后发布前失败按索引、关系、节点候选、主信息候选逆序收口；最后发布后读回异常或候选确认异常追根因停止。
5. 四仓库、统一结果头、结构事务接线和协调器保持只读；若实现必须修改这些依赖，按接口漂移零代码退回。
6. `#264` 仅允许独立自检提交类型化回调；后续生产调用必须由专项计划登记的 `数据操作.*` 模块承接，领域服务、线程路由和组合器不得直接把执行器作为任意写入口。
