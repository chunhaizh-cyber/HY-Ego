## 2. 核心处理

1. 新增格式版本、节点 / 关系 / 原始类型枚举版本、仓库上下文和材料序号，序列化材料保存特征与特征值完整句柄、原始类型、统一原始值版本、元素数量及 I64 / Vec 完整值副本。
2. `生成特征值序列化材料` 在入口拒绝无效上下文、序号、类型和归属，复制统一原始材料后再次复核当前版本与完整内容；版本漂移返回材料过期，不发布旧材料。
3. `复核特征值恢复候选` 只复核目标仓库上下文、格式与枚举版本、句柄、归属、原始类型和元素数量，输出值式规范化原始材料副本，不调用写入入口。
4. 缓存、命中、稳态、显示或 SQL 材料不得进入权威恢复候选；携带非权威恢复请求时明确拒绝。
5. 候选入口不创建节点、主信息或关系，不修改索引和既有原始值；文件编码、磁盘 / 数据库、恢复提交与启动恢复均不在本切片。

## 4. 输出变化与调用路径

```text
仓库上下文 + 特征 / 特征值完整句柄 + 材料序号
-> 特征服务复核类型、归属并读取统一原始材料
-> 组装格式版本化结构材料
-> 再次复核当前版本与完整内容
-> 可用材料或材料过期 / 结构化拒绝

目标仓库上下文 + 结构化材料
-> 静态字段、身份、归属和既有类型复核
-> 规范化原始材料副本
-> 只读恢复候选或结构化拒绝
```
