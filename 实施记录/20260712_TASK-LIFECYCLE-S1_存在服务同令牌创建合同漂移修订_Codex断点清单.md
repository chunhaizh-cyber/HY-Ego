## 3. 语义裁决

选择“定向补存在服务同令牌路径”，不选择“不创建新虚拟存在”。

原因：

```text
任务虚拟存在承载任务运行特征和过程账。
存在服务是存在节点的原子服务和唯一创建方。
复用需求主体 / 目标宿主会混淆任务过程账与需求主体。
任务服务直造存在节点违反基础信息原子服务边界。
```

## 4. 正式接口合同

存在服务附加默认空值式 `结构事务接线`，并定向增加：

```text
创建实际存在(const 结构事务令牌&)
创建虚拟存在(节点句柄 来源节点, const 结构事务令牌&)
读取存在(节点句柄, const 结构事务令牌&) const
存在是否有效(节点句柄, const 结构事务令牌&) const
```

令牌写入口必须验证当前同域独占许可，再向主信息、节点和关系仓库同步下传。普通无令牌构造和重载保持现有兼容语义；`创建存在概念根` 不属于 #221，不新增令牌重载。

任务服务和状态服务按 JY-275 既定方向附加默认空接线与所需令牌重载。正式调用链为：

```text
自检.任务生命周期初态取得一次独占许可
-> 任务服务验证令牌并创建任务节点 / 主信息
-> 存在服务验证同一令牌并创建虚拟存在 / 来源引用
-> 状态服务验证同一令牌并创建已创建实例状态
-> 任务服务同令牌写承接与生命周期关系
-> 完整读回后返回
```

令牌只同步调用期传递，不保存到任务、存在、状态材料或长期成员许可中。

## 5. 失败收口

进入写入后，各服务必须记录本次创建句柄并在同一独占许可内精确逆序清理：

```text
关系 -> 节点 -> 主信息
```

存在服务负责本次虚拟存在来源关系、存在节点和主信息；状态服务负责本次状态关系、状态节点和主信息；任务服务负责顶层承接 / 生命周期关系以及按组合逆序调用下层清理。不得扫描、按类型猜测、删除既有结构或只靠许可隐藏半结构。清理不及预期继续追根因，释放许可后不得出现半任务、孤立存在或孤立状态。
