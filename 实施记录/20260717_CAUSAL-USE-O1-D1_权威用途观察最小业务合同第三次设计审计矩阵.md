## 4. 阶段主干与正式结算

### 4.1 A06：阶段与推进

三种合法阶段元组保持互斥：

```text
尝试未达成：未达成已转待重筹办 + 未达成 + 未形成 + 无正式结算状态。
待结算：任务已完成待需求结算 + 已达成 + 待结算 + 无正式结算状态。
最终已结算：已完成并结算 + 已达成 + 已满足 + 有完整正式结算状态。
```

唯一允许推进仍为同身份、同不可变核心的 `待结算 -> 合法最终已结算`。终态变化、逆向和其它不允许转换在两个候选各自合法后才进入同账冲突。A06 通过。

```text
事件.结果来源.需求结算.材料.正式结算

-> 结算状态.状态节点
```

## 5. A07 / A10：三层唯一分流复核

### 5.1 第一层：因果键召回

专项规范当前唯一写明：

```text
哈希只选择候选桶。
哈希命中后逐项执行因果完整结构键全字段比较。
当前桶项不同且仍有未比较项 -> 哈希碰撞，保留不同事实并继续扫描，不返回。
哈希未命中，或完整桶扫描结束仍无全字段相同项 -> 逻辑内召回未命中。
本层不形成 O1 身份或同账结论。
```

当前 #287 代码事实与此一致：`计算因果完整结构键稳定哈希` 只返回 optional 哈希；`因果完整结构键相同` 仍逐项比较格式、原因动态键、结果动态键和因果规则版本。哈希无效返回空值，不把默认对象召回为事实。

### 5.2 第二层：O1 身份

```text
完整任务句柄或非零任务序号任一不同
-> 不同 O1 身份 / 新账项
-> 不进入该既有账项的不可变核心或阶段比较
```

O1 详细设计的“不同身份”行、流程图的“新身份或无既有账项”分支和专项规范新正常分流行保持一致。

### 5.3 第三层：O1 同账

只有完整 O1 身份相同，且新候选与可选既有候选各自合法后，才进入：

```text
同核心 + 同阶段 -> 同账幂等。
同核心 + 待结算推进合法最终 -> 同账允许推进。
异核心，或终态 / 逆向 / 其它不允许阶段转换 -> 同账冲突，零结构变化。
候选自身非法、已发布结构非法、同一身份多个当前记录 -> 追根因，不归同账冲突。
```

### 5.4 防止跨层替代

#287 因果完整结构键哈希不得充当 O1 账项索引。尤其是同一 O1 身份但因果完整结构键不同的场景，必须在 O1 同账层按不可变核心不同裁决，不能被召回层隐藏为未命中。

因此：

```text
A07 通过：召回继续、召回未命中、不同身份、幂等、推进、冲突、不支持和内部不一致均有互斥条件。
A10 通过：原“逻辑内未命中 / 冲突”双结果已删除，所有中途非成功返回均有唯一分类。
```
