## 2. 已验证当前事实

1. `任务服务::按需求创建任务` 在外层独占许可内、第一笔任务写入前调用无令牌 `需求服务::需求是否有效` 与 `读取需求承接材料`。
2. 两个无令牌入口会进入已接域节点 / 关系仓库并同线程重复取得结构许可。
3. 需求服务已有可空结构事务接线和私有 `节点类型匹配_令牌`、`读取唯一需求目标_令牌` 等同令牌辅助。
4. 需求服务没有上述两个公开只读令牌重载；#221 原允许范围又禁止修改 `需求服务.h`。
5. 第二次复核未改代码，worktree / index clean。

## 3. 裁决

采用许可内同令牌权威复核，不采用许可前值式快照。后者没有版本复核合同，会在需求失效或承接材料变化时形成 TOCTOU。

`需求服务.h` 定向加入 #221 允许范围，只新增：

```text
bool 需求是否有效(节点句柄, const 结构事务令牌&) const
std::optional<需求承接材料> 读取需求承接材料(节点句柄, const 结构事务令牌&) const
```

实现复用既有私有令牌辅助。不得修改需求创建、命名需求唯一创建复用、需求结算、需求仓库写路径或无令牌兼容语义。

## 4. 固定调用路径

```text
自检.任务生命周期初态取得独占许可
-> 任务服务验证同一令牌
-> 需求服务同令牌重读需求有效性与承接材料
-> 需求材料完整且当前
-> 存在服务同令牌创建任务虚拟存在
-> 状态服务同令牌创建已创建状态
-> 任务服务 / 仓库完成初态提交与读回
```

需求复核失败发生在第一笔任务写入前，属于逻辑内返回。进入写入后内部不及预期继续沿 JY-292 在同一许可内精确逆序清理并追根因。

## 5. 允许与禁止

允许 `海中鱼巣/领域/需求服务.h` 增加两个公开只读令牌重载，允许复用其现有私有辅助。禁止任务服务直接读取需求仓库，禁止令牌进入 DTO，禁止修改 #217 ABI、需求写入口、#213 命名路径或无令牌入口语义。

## 6. 执行前复核

执行窗口必须逐项核对：

```text
两个无令牌需求入口的当前调用点
需求服务可空接线构造形状
节点类型匹配_令牌与读取唯一需求目标_令牌的实际签名
需求承接材料的其余读取是否已有同令牌辅助
任务 -> 需求 -> 存在 -> 状态 -> 仓库全链不嵌套取得许可
最终回归阶段 560 仍空闲
```

任何一项不一致，或必须扩大到需求写入口、#213 路径、仓库 ABI，均停止改代码并再次退回设计。

## 7. 验收语义

1. 需求无效、承接材料为空或材料漂移在第一笔任务写入前返回，结构保持完整前态。
2. #221 连接态路径不调用两个无令牌需求入口。
3. 普通无令牌需求读取保持既有回归。
4. 任务服务不直接访问需求仓库。
5. A09 覆盖任务、需求只读、存在、状态和仓库的同一独占令牌链。
