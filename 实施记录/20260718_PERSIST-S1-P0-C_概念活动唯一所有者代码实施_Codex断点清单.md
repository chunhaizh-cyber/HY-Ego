## 3. 具名接口漂移

当前完整自检的真实调用路径为：

```text
入口.cpp import 专项自检模块
-> 入口.cpp 创建 阶段化自检总成
-> 入口.cpp 调用 登记自检(最终回归, 阶段号, 回调)
-> 入口.cpp 运行最终回归阶段
-> 入口.cpp 把登记结果和专项结果并入最终具名布尔汇总
```

当前证据：

1. `海中鱼巣/入口.cpp:9806` 登记阶段 660 的运行期上下文专项。
2. `海中鱼巣/入口.cpp:9809` 登记阶段 690 的系统角色初始化专项。
3. `海中鱼巣/入口.cpp:9877` 与 `海中鱼巣/入口.cpp:9885` 把上述登记和专项结果并入最终汇总。
4. `海中鱼巣/自检.运行器.ixx` 没有现成的专项 import、默认登记函数或全局登记宿主。

#296 第 4 节只允许修改 `海中鱼巣/自检.运行器.ixx` 并明确禁止 `海中鱼巣/入口.cpp`。因此，即使四个新文件、装配和上下文全部实现，新 `自检.概念活动所有权.ixx` 也无法在现有调用图中被 import、登记为 890、执行并进入完整自检最终汇总。

执行角色不能通过以下方式自行收口：

1. 不能修改禁止文件 `入口.cpp` 增加 import、结果槽、890 回调和最终汇总。
2. 不能让 `阶段化自检总成` 构造函数隐式登记专项；该类型被多处合同自检创建，隐式全局副作用会改变既有运行器语义。
3. 不能用静态初始化、链接副作用或未调用模块伪造阶段登记与通过结果。
4. 不能把专项正文移入 `自检.运行器.ixx`；这违反生产 / 自检模块分离和计划的新文件所有权。
