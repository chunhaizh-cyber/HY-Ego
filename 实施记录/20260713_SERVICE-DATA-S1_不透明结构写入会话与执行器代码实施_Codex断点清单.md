## 2. 输入入口与实际接口

`结构写入执行器::执行` 是本切片唯一执行入口。执行器先拒绝未接域接线、空仓库引用和空回调，再取得一次独占结构许可；许可无效时返回许可拒绝，回调不运行，四仓库结构不变化。

会话只适配 #263 的十个独占结构化入口：

```text
主信息：候选创建、确认、撤销
节点：候选创建、确认、撤销
关系：结构化创建、严格删除
索引：结构化绑定、严格删除
```

旧无令牌写、旧带令牌兼容写以及关系 / 索引旧共享删除入口均未调用。

## 3. 核心处理与结构承载

`海中鱼巣.核心.会话.结构写入` 导出不可默认构造、不可复制、不可移动的 `结构写入会话`。构造函数、原始令牌、仓库引用、写集和收口函数保持私有，只允许 `结构写入执行器` 在当前线程栈内构造并以同步左值引用交给回调。

会话写集分别承载：

```text
主信息未发布候选
节点未发布候选
完整关系句柄
主键 + 预期节点完整句柄
```

每个新写入必须通过会话窄读取完成同令牌读回，才允许请求提交。提交按节点候选、主信息候选顺序确认；显式撤销、无提交决定、仓库操作失败或回调异常均在许可释放前按索引、关系、节点候选、主信息候选逆序收口。会话析构不执行仓库写入。

## 4. 输出变化与调用路径

```text
隔离自检提交同步类型化回调
-> 执行器复核接线、仓库引用和回调
-> 取得一次同域独占许可
-> 当前线程栈内构造不可移动会话
-> 回调只使用会话窄结构操作
-> 写集完成同令牌读回
-> 请求提交：确认节点候选 -> 确认主信息候选
-> 请求撤销 / 失败 / 异常：索引 -> 关系 -> 节点候选 -> 主信息候选
-> 会话结束后由既有许可 RAII 释放
-> 返回值式结构写入结果
```

新增三个真模块并以 `ClCompile` 登记：

```text
海中鱼巣/核心/会话.结构写入.ixx
海中鱼巣/核心/执行器.结构写入.ixx
海中鱼巣/核心/自检.结构写入会话.ixx
```

`入口.cpp` 只增加自检模块导入、一个值式结果、一次阶段 710 登记和总结果汇总。
