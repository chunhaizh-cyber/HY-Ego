## 2. 总结论

1. 当前代码中 `概念图服务`、`实例支持概念`、`概念上下位`、`概念定义材料`、概念签名、固定点候选图、待命名 / 待重算请求和概念生命周期实现的定向扫描均为 0 命中。

3. `关系类型` 当前固定为 0 到 8；三类专用关系只能追加为 9、10、11，且必须同步修改关系类型有效范围。
4. 通用关系创建已经允许非普通父子关系多源、多目标，但没有专用同类校验、概念环检测、按目标枚举来源关系、批量失效或图版本发布。
5. 当前没有“四类根类别 -> 四个稳定句柄”的权威登记结构，也没有真实恢复入口。索引只能帮助找到候选，自我线程快照只在同一线程对象生命周期内复用。
6. 实例状态和实例动态已有明确场景、主体、时间戳及 `运行期临时` 关系，可与长期状态 / 动态区分；现有提升入口每次新建长期节点，不做签名等价复用或固定点。
7. #180 已在本扫描前完成，因此精确 I64、VecI64、VecU64 值和 Vec 容器版本可经特征服务读取；区间、值域、稳态、候选比较和真实内容哈希仍未实现。
8. 语素服务已有多目标对应和概念追溯关系形状，但概念追溯白名单不含存在节点，且当前没有从概念反查名称入口或持久化词面身份。
9. 自我线程的处理器边界可以复用，治理队列也可承载新增协议后的非权威请求；当前协议固定八类消息，没有概念服务角色、待命名、待重算或生命周期请求类型。
10. 统计服务当前材料全部非权威，适合扩展一般 / 预测 / 行动用途候选，但尚无概念使用事件和生命周期统计。
11. 写入事务只有状态壳，节点删除不会自动删除主信息、索引和关系记录；当前不具备安全物理删除或候选图原子发布条件。

## 9. 用途、生命周期和删除

### 9.1 统计与用途

`统计服务.h:72-117` 已有内容哈希、命中、需求观察、存在观测和二次特征签名候选 DTO；所有结果均明确不可裁决业务事实。当前没有概念来源类型、一般使用、因果预测使用、因果行动使用、最近使用时间、冷却候选或退役候选。

建议由统计服务承载三类可重算用途候选，概念图服务重读权威支持和引用后才改变生命周期。预测用途与行动用途必须分开材料和命名空间。

### 9.2 生命周期与安全删除

当前没有活跃、冷却、退役状态结构，也没有重新激活入口。现有状态服务可以创建状态节点，但没有“概念当前生命周期”唯一关系和转换封口。

物理删除还缺：

```text
按目标枚举所有来源关系。
按概念枚举全部入边 / 出边关系句柄。
下位概念兼容重挂和无环预判。
批量关系失效。
概念节点与主信息协同失效。
索引项显式清理与等价概念重建策略。
真实事务、补偿或活动图版本隔离。
删除后固定点和旧句柄 / 孤儿 / 环读回。
```

因此 S8 必须保持真实依赖门控；S6 / S7 可以先实现生命周期状态、候选、冷却、退役和重新激活，不能提前物理删除。
