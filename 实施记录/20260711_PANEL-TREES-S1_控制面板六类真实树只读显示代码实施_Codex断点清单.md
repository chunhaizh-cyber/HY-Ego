## 2. 实施结果

1. `任务服务::读取需求承接任务组` 只按需求目标枚举 `引用` 来源，逐个复核任务类型、完整承接壳和来源需求，确定性排序去重，不扫描节点仓库全表。
2. 控制面板节点 DTO 新增结构化关系角色、可空名称入口组和重复投影；树 DTO 改为真实根节点组、来源版本和投影数量；UI 分类根不进入领域 DTO。
3. 控制面板服务预绑定世界 / 语素值式快照和安全 / 服务两个根需求，只持有领域服务与节点仓库只读引用；六类候选完整后才分配候选序号并返回。
4. 世界树复用既有普通父子 DTO；语素树在既有入口下展开真实对应信息与概念追溯目标；概念树只转换 #186 活动图四根抽象树视图，多父项保留同一句柄和重复投影。
5. 需求树显示两个根需求的目标、主体、目标宿主、场景、当前特征状态、最近结算证据和承接任务；任务树从根需求反向发现真实任务并读回承接、生命周期、结果和选择方法。
6. 方法树只从任务选择关系发现方法首，读取虚拟存在、条件 / 结果、动作入口、输入 / 输出规格和方法关系；任务选择关系同时作为真实来源任务投影，空条件 / 结果不伪造节点。
7. 单类 4096、合计 16384、单路径 128 深度作为完整候选门禁；超预算只返回结构化拒绝，候选数组不携带部分树。
8. 窗口快照改为固定六项树结构材料；TreeView 一次插入世界、概念、语素、需求、任务、方法六个 UI 分类根及全部真实根并展开。分类导航只选中和滚动对应根，不删除其他分类。
9. 刷新先保存旧快照，再取得六类完整候选并重建；逻辑内拒绝保留旧快照，TreeView 插入失败尝试恢复旧视图。重建期间屏蔽同步选择通知，避免旧项引用越过快照生命周期。
