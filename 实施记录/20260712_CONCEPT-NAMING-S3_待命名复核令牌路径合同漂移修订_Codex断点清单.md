## 3. 默认裁决

采用“许可内令牌感知复核”，不采用“许可前值式快照 + 许可内版本复核”。

正式顺序固定为：

```text
复核协议、专用类型和消息值形状
-> 取得 #217 同域独占许可
-> 取得目标概念固定分片锁
-> 调用概念图服务待命名请求令牌重载
-> 取得活动图读锁并读取快照 / 签名
-> 经生命周期令牌辅助读取关系仓库
-> 经语素服务名称入口令牌重载读取关系 / 节点仓库
-> 比较当前请求、来源材料版本和名称为空条件
-> 通过后才进入需求服务同令牌创建 / 复用路径
```

许可前快照方案被否决，因为现有快照入口已经持活动图锁并进入接域仓库，仍无法证明结构许可先于活动图锁。

## 4. 正式接口约束

### 4.1 概念图服务

允许新增：

```text
读取待命名请求(节点句柄, const 语素服务&, const 结构事务令牌&) const
读取概念生命周期_已加锁(节点句柄, const 结构事务令牌&) const
```

第二项是最小私有辅助。它必须调用关系仓库正式令牌重载，不得回落无令牌入口。普通双参公开入口及普通生命周期辅助保持原语义。

### 4.2 语素服务

允许新增：

```text
读取概念名称入口组(节点句柄, const 结构事务令牌&) const
节点是语素入口(节点句柄, const 结构事务令牌&) const
节点是可作为概念追溯目标(节点句柄, const 结构事务令牌&) const
```

后两项是最小私有辅助。关系来源读取和节点类型读取必须使用同一令牌。普通无令牌入口保持原语义，继续服务 S1 等许可外调用方。

### 4.3 通用边界

```text
令牌只同步下传，不保存、不排队、不跨线程。
令牌重载只读并返回值式材料，不新增写入或平行缓存。
不修改核心结构事务 ABI、仓库算法、活动图写入或语素写入。
分片锁内不调用无令牌包装、邮箱、线程等待、用户回调、日志、显示、SQL 或外设。
```

## 5. 非成功二分

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 协议、类型或消息值形状不合法 | 逻辑内返回 | 许可前返回，不读当前活动结构 |
| 许可内重读发现请求消失、版本漂移或名称出现 | 逻辑内返回 | 不进入需求写入，释放分片锁与独占许可 |
| 活动快照声称概念有效但生命周期关系缺失 / 重复 / 状态登记矛盾 | 追根因解决 | 停止依赖路径，不伪装成消息过期 |
| 概念图 / 语素令牌重载回落无令牌路径或令牌验证失败 | 追根因解决 | 停止实施，不扩大核心 ABI |
