## 5. 目标状态与需求语义默认裁决

### D1 信息目标，不复用服务值特征

命名目标默认采用独立“信息目标需求”，不复用初始化的安全值或服务值特征。服务值是根需求的通用 I64 特征，把单个概念是否有名称塞入该槽位会混淆多个概念、当前材料和结算身份。

### D2 主体、宿主和场景

```text
需求主体：自我存在。
目标宿主：自我存在。
需求场景：当前自我所在场景。
目标对象：单独关联的目标概念，不把概念冒充存在宿主。
```

### D3 目标仍是状态

建立一个稳定登记、可读回的共享抽象目标状态，语义为“目标概念已有至少一个有效名称入口”。其 I64 值只能作第一轮状态值材料，不能单独表达语义；需求到目标概念的专用关系和语素名称关系共同确定具体目标。

### D4 目标概念关系

后续详细设计默认新增专用 `需求目标概念` 关系类型，方向为命名需求到目标概念，同一命名需求唯一。该关系：

```text
承载长期目标身份；
阻止 #198 在未结算 / 未撤销需求存在时物理删除目标概念；
不作为 #195 / #197 的一般用途、实例支持或生命周期活性证据；
不得用日志、文本或非权威缓存替代。
```

选择专用关系而不是普通 `引用`，是为了避免自动命名需求被 #195 的外部引用计数误当成概念长期活性，从而让所有无名概念永久无法冷却。

### D5 父需求归属

第一轮命名需求默认挂到已初始化的服务根需求，使用 #192 的 `提交需求父子更新`。不创建第三个根需求，也不先造一个没有独立目标状态的“命名父需求”。只有命名需求完整读回后才提交父子关系。

## 6. 创建、去重和版本默认裁决

### D6 消息协议

后续协议切片默认追加：

```text
自我治理消息类型::概念待命名请求
自我治理来源角色::概念图观察入口
自我治理服务角色::概念图
自我治理只读材料类型::概念待命名材料
自我治理只读材料句柄组::目标概念
自我治理消息::来源材料版本（命名请求中等于活动图版本）
```

来源为概念图观察入口 / 概念图，目标服务为需求，材料性质为非权威候选。协议结构变化应提升协议版本；邮箱和自我线程不增加业务字段或新线程。

### D7 幂等与过期

```text
消息幂等键 = 固定命名用途域 + 完整概念句柄 + 活动图版本的确定性摘要。
完整目标概念和来源版本仍必须逐字段复核，摘要碰撞不得裁决同一请求。
同一活动版本的重复消息由邮箱幂等收口；新活动版本允许重新提交。
消息使用截止时间策略，同时在领域路由重读当前活动图版本和待命名请求。
```

活动图版本只约束请求新鲜度，不进入长期需求身份。概念仍有效且仍无名时，新版本请求必须复用同一未完成需求。

### D8 同概念唯一未完成需求

需求服务或专用命名治理路由必须提供“创建或读取概念命名需求”原子入口：

```text
零个未完成需求 -> 创建完整信息目标需求并读回。
恰好一个 -> 幂等复用，不新增需求或父子关系。
多于一个 -> 追根因解决，不选择第一个。
```

同一目标概念必须串行化扫描与创建；不同概念可使用固定分片锁并发处理，避免全局粗锁。邮箱的单消费者顺序不能替代领域入口的并发正确性。

“未完成”第一轮定义为没有可读的已满足正式结算记录。名称已经存在但正式任务 / 动态结算尚未落账时，需求进入“目标已出现、待正式结算”路径，不再创建第二个需求。

## 7. 名称绑定与结算默认裁决

### D9 名称候选和绑定

自我治理消息不携带名称字符串。未来命名方法应提交候选语素入口句柄作为结构化材料；文本只作外部材料或人读显示，不作为机器身份。

名称绑定必须经语素服务正式入口写入 `语素概念追溯`，并立即由 `读取概念名称入口组` 读回目标概念和语素入口。外部输入、线程、日志或显示层不得直接写关系。

### D10 绑定回执

绑定回执至少需要：

```text
目标命名需求、来源任务、目标概念、名称语素入口、活动图版本、发生时间戳、动作动态证据。
```

回执只作复核材料。概念图服务重读当前概念身份和名称关系、需求服务重读目标概念关系、任务服务重读来源需求后，才能生成实际状态和结算入口材料。

### D11 正式结算

名称关系读回只证明目标条件成立，不等于需求已结算。正式结算继续要求：

```text
来源任务属于该命名需求；
实际状态与共享“已有有效名称入口”目标状态匹配；
命名动作动态可读且来源动作合法；
需求目标概念、名称语素入口和活动版本经当前结构复核；
需求结算记录完整读回。
```

现有 `记录需求结算` 可承接最后一段，但当前没有命名动作、实际状态 / 动态构造和专用回执转换，因此不能直接调用。

### D12 生命周期与删除收口

```text
活动图版本变化：旧消息逻辑内过期；概念仍有效且无名时，新消息复用既有需求。
概念退役：不自动取消命名需求；退役概念按既有规则仍可命名和重新激活。
概念已有名称：停止生成新请求；已有需求等待正式结算，不以空请求直接写“已满足”。
物理删除：存在未完成命名需求目标关系时必须拒绝；撤销 / 失败生命周期另建后续规则。
```
