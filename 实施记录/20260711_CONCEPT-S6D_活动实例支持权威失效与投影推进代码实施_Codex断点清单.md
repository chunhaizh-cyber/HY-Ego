## 2. 核心处理

1. 在 `记录状态` 末尾追加 `已失效 = 3`，保持既有 0 至 2 数值不变。
2. 关系仓库新增值式关系失效结果、当前版本审计读取和按源 / 类型审计组。有效关系失效后编号、类型、端点和顺序号保持，状态转为已失效，版本加一；普通读取继续只看有效关系。
3. 同一失效前句柄或当前审计句柄重复提交幂等；错误仓库编号、过旧版本、已删除状态或端点无效不推进状态。已失效历史不阻止同端点未来创建新的有效关系。
4. 关系仓库同步提供返回新句柄的值式重挂和节点相关有效关系读取，供恢复后的 #197 草案直接复用，不在 #197 重复修改仓库算法。
5. 概念图服务新增实例支持失效请求、状态和结果 DTO，以及实例支持审计组。根登记检查在两把概念图锁之前完成；锁内仍复核当前活动普通概念、精确支持边、关系仓库记录和专用登记。
6. 双锁固定按 `活动图锁_ -> 图写锁_` 获取。写前复制活动快照和专用登记，只移除精确支持边并把活动版本加一；关系正式失效后以无分配交换发布候选，清除上一候选图。
7. 对真实普通概念逐条失效全部有效支持，形成零支持候选；随后用正式固定点候选发布重新建立同端点新支持，旧失效关系继续审计可读。
8. 存在、动态、关系、因果四根逐一提交支持失效请求，均在双锁前拒绝；四组支持关系状态 / 版本、活动版本、候选准入和活跃生命周期不变化。

## 3. 结构承载

```text
关系当前事实：关系仓库中记录状态为有效的当前版本关系。
关系失效历史：同一关系编号、原端点和类型下的已失效当前版本记录；只经显式审计入口读取。
活动支持投影：概念活动快照和专用关系登记只保留当前有效支持关系。
活动版本：每次真实支持失效恰好推进一次；幂等、拒绝和错误句柄不推进。
失效结果：值式 DTO 承载原关系、当前审计关系、原 / 当前活动版本和状态，不成为独立机器事实。
四根支持：继续保持有效，仍是四类实例候选准入基础。
```

## 4. 输出变化与调用路径

```text
显式支持失效请求
-> 句柄 / 类别 / 根登记写前拒绝
-> 活动图锁_ 独占
-> 图写锁_ 独占
-> 当前活动普通概念、预期版本、精确快照边、专用登记和关系记录复核
-> 预构造移除精确边的新快照和登记
-> 关系仓库有效转已失效并返回新审计句柄
-> 普通读取为空、审计读取完整
-> 交换发布活动快照 / 登记并清除上一候选图
-> 活动版本加一回执
```

重复请求先按同一关系编号读取审计历史，再严格复核当前审计句柄或刚失效前句柄；不接受伪仓库编号。未来真实支持重新出现时继续走既有候选发布路径，创建新的有效关系，不复活旧记录。

## 5. 入口拒绝与失败收口

| 场景 | 分类 | 结果 |
| --- | --- | --- |
| 请求、实例、概念或关系句柄无效，类别不一致 | 逻辑内返回 | 不写关系或活动快照 |
| 目标是存在 / 动态 / 关系 / 因果任一根 | 逻辑内返回 | 在概念图双锁前拒绝，根支持和候选准入不变 |
| 非当前活动普通概念、非精确支持或预期活动版本过期 | 逻辑内返回 | 不推进关系或活动版本 |
| 同一支持已经失效且审计完整 | 幂等逻辑内返回 | 返回当前审计句柄，不再次推进版本 |
| 错误仓库编号或非相邻历史版本冒充幂等请求 | 逻辑内返回 | 返回支持不存在，不读取为本仓库审计事实 |
| 快照、登记和关系仓库互相矛盾 | 追根因解决 | 停止依赖路径，不扫描修复或裸删关系 |
| 关系失效后普通 / 审计读回或发布后快照不一致 | 追根因解决 | 停止活动图、生命周期和 #197 依赖 |
