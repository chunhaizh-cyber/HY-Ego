## 4. 包级迁移口径

### FS-04-A 需求身份与目标状态读取

保留行为：

```text
需求必须能按稳定身份读取。
需求目标必须能追溯到目标状态。
按主键或索引命中需求后必须复核节点类型、生命周期和目标状态唯一性。
```

删除行为：

```text
不迁移旧需求主信息中的目标状态、当前状态、目标特征缓存或满足关系裸字段为新项目权威字段。
不使用旧需求树根指针和裸节点指针作为长期身份。
```

新增约束：

```text
需求目标是目标状态，不是 I64。
需求读取必须经需求服务入口复核。
目标状态缺失、多目标冲突或节点类型错误时不得创建任务或结算事实。
```

验收方式：

```text
有效需求可经需求服务读回唯一目标状态。
无效需求、错误节点类型、缺目标状态或多目标状态时返回空或拒绝，结构数量不变。
```

### FS-04-B 需求主体、场景与目标宿主材料读取

保留行为：

```text
任务承接前需要读取需求主体、目标宿主、场景和目标状态材料。
需求目标状态、主体和场景必须可以被后续任务、方法和结算链路追溯。
```

删除行为：

```text
不把旧需求主信息中的需求主体、目标宿主、需求场景字段直接迁移为新主信息字段。
不把材料读取失败降级为默认场景或默认存在。
```

新增约束：

```text
存在和场景材料必须由对应服务或需求服务封装读取。
任务服务按需求创建任务前必须复核需求目标状态和场景材料。
```

验收方式：

```text
有效需求可产出任务初始化所需材料。
缺主体、缺场景或目标宿主无效时任务创建拒绝，需求结构不变化。
```

### FS-04-C 需求承接任务与初始化上下文

保留行为：

```text
需求可以被任务稳定承接。
任务创建后必须可回读来源需求、任务目标状态、任务虚拟存在和生命周期状态。
```

删除行为：

```text
不迁移旧需求主信息对应任务字段和任务列表字段为权威任务关系。
不因 SQL 显示同步成功或失败裁决任务绑定事实。
```

新增约束：

```text
任务承接由任务服务写入任务来源需求关系和需求拥有任务关系。
需求服务只提供需求目标和材料读取，不直接替任务服务写任务结构。
```

验收方式：

```text
按需求创建任务后，可经任务服务读回来源需求和目标状态。
重复承接、缺目标状态或场景无效时拒绝或进入后续确认规则，结构不产生半成品。
```

### FS-04-D 需求结算权威记录与摘要分流

保留行为：

```text
需求结算必须记录目标状态、实际状态、来源任务、动态证据和发生时间戳。
最近结算可作为读取便利，但必须能从权威结算记录重建。
```

删除行为：

```text
不迁移旧累计安全结算、累计服务结算、最近结算任务、最近结算时间为权威主信息字段。
不把父级摘要刷新当作权威结算事实。
```

新增约束：

```text
权威结算记录由需求结算记录结构承载。
摘要、最近记录索引和统计只能作为非权威缓存，清空后不得改变结算事实。
```

验收方式：

```text
结算记录可枚举并可按发生时间戳读取最近记录。
清空非权威摘要后，需求结算事实仍能由权威记录恢复。
```

### FS-04-E 父子需求角色、阻塞与权重读取

保留行为：

```text
父子需求角色、阻塞父任务和权重可作为任务承接或候选排序材料。
父需求和子需求之间的结构关系需要可读。
```

删除行为：

```text
不把旧阻塞字段、权重字段或角色文本直接迁移为任务执行闸门。
不让需求树角色绕过任务授权、任务承接和任务选择方法。
```

新增约束：

```text
父子角色第一轮应作为只读视图或后续专项材料。
任务执行权限仍来自任务授权、任务承接和任务选择方法。
```

验收方式：

```text
父子角色读取只产出候选视图，不创建或阻断任务事实。
任务服务仍复核目标状态、生命周期和任务方法关系。
```

### FS-04-F 逻辑组织需求、获取途径与目标语义

保留行为：

```text
AND / OR / 方法路径组 / 因果子链支撑需求证明旧系统存在逻辑组织需求能力。
获取途径可作为任务筹办和方法候选材料。
```

删除行为：

```text
不把旧获取途径枚举、目标语义文本或逻辑组织类型文本直接作为新项目机器事实。
不在本轮迁移旧逻辑组织需求生成算法。
```

新增约束：

```text
逻辑组织需求必须由需求目标状态、需求父子关系、任务承接和方法 / 因果证据共同承载。
缺少结构设计前，不新增旧枚举字段或显示文本字段。
```

验收方式：

```text
本轮只形成后续专项边界。
后续专项必须证明逻辑组织需求不会绕过需求服务和任务服务写结构。
```

### FS-04-G 需求方向签名、枚举目标和方法候选准入

保留行为：

```text
需求方向、枚举目标合法性和值域 / 生产者材料可作为方法候选和需求结算规则材料。
方向比较可辅助需求去重或冲突候选。
```

删除行为：

```text
不把方向比较 bool、目标语义文本、生产者分级文本或校验结果直接当作方法可执行事实。
不让需求类直接裁决方法候选可执行。
```

新增约束：

```text
方法候选和条件判定必须进入方法系统后续增强。
枚举目标和值域规则必须与特征值系统、需求结算记录和方法条件共同设计。
```

验收方式：

```text
方向和枚举目标只输出候选或拒绝原因。
方法选择仍由任务方法关系和方法服务复核。
```

### FS-04-H 需求树结构化更新指令

保留行为：

```text
需求树多步更新先形成结构化请求，再由统一入口提交，有利于入口拒绝和无半结构。
```

删除行为：

```text
不迁移旧裸指针、旧主键字符串、旧主信息字段名和旧 SQL 显示同步作为更新指令材料。
不提前实现删除需求或复杂逻辑组织需求写入口。
```

新增约束：

```text
新项目如需统一需求更新入口，必须基于稳定句柄、关系、状态和错误结果。
提交入口必须在写入前排除父需求无效、目标状态无效、当前状态无效和关系冲突。
```

验收方式：

```text
构造请求不写结构。
提交入口拒绝后节点、关系、状态和任务数量不变。
```

### FS-04-X SQL / 控制面板 / 旧主信息摘要排除

保留行为：

```text
旧 SQL 投影字段可作为后续只读显示视图参考。
旧摘要字段可作为非权威缓存重建需求的材料参考。
```

删除行为：

```text
不接 SQL 控制面板和投影库。
不把显示阶段文本、运行日志、最近结算摘要、任务列表字段或 SQL 同步结果作为机器事实。
```

新增约束：

```text
显示层只读结构，不写业务事实。
非权威缓存必须能由权威结构重建，清空不影响需求、任务或结算事实。
```

验收方式：

```text
本轮不运行 SQL、不连接控制面板、不新增显示写入。
后续显示专项只能读取结构并生成视图材料。
```
