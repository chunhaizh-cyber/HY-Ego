## 7. 召回、选择、冻结与任务序号边界

### 7.1 现代召回和选择

1. 召回扫描登记根下全部活跃方法，读取条件 / 结果 / 动作规格；要求条件全匹配、结果至少一项匹配、动作恰一，见 `服务.方法召回.ixx:86-143`。
2. 分值固定为 `1000 + 条件数 * 10 + 匹配结果数`；按分值和方法句柄稳定排序，同分前两项返回非唯一建议，见 `:140-162`。
3. 唯一建议只适配任务、方法、两组生命周期快照、排序 / 召回版本和候选数量，见 `:165-189`；没有来源观察、来源方法、方法代际或后继序号下界。
4. `来源方法生命周期关系` 只是所选方法在选择时的生命周期快照，不是父代 / 来源方法关系。

因此，现代召回候选和任务方法选择记录都不是学习候选。

### 7.2 执行冻结

`任务执行冻结请求` 包含完整调度请求、任务材料、方法登记材料、唯一动作入口和适配器，见 `协议.任务执行请求.ixx:241-363`。它不包含方法条件 / 结果规格组，也不包含选择时特征快照、实际结果集合或来源用途观察组。

执行组合器冻结时只重读方法登记和动作组，要求动作恰一并匹配稳定动作键 / 场景，见 `组合.任务执行.ixx:65-95`。因此冻结不能直接计算方法结构差异，也不能证明执行期间方法条件 / 结果没有漂移。

### 7.3 任务序号

调度请求的任务序号由调用方提供，完整比较只保证请求内部一致，见 `协议.任务执行请求.ixx:91-130`。任务管理线程只按同一任务句柄拒绝倒退或相等，并在重建时取该任务最大序号，见 `线程/任务管理线程.ixx:239-260,517-530`。

这不是全局序列、方法代际序列或“第 N 回合产生的新方法只能从 N+1 召回”的正式下界。
