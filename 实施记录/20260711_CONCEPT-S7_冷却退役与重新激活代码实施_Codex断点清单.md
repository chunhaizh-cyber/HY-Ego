## 3. 核心处理

### 3.1 转换材料与入口

```text

```

转换入口固定按 `活动图锁_ -> 图写锁_` 取得双锁，并在锁内重读：

```text
当前生命周期关系与状态

节点相关有效关系中的外部正式引用

```

### 3.2 状态机与关系承载

```text
活跃 -> 冷却
冷却 -> 退役
冷却 / 退役 -> 活跃
```

唯一生命周期关系复用原关系编号，目标状态重挂后版本加一。根概念离开活跃、活跃直接退役、退役转冷却和其它非法跳转均在写前拒绝；同状态请求幂等返回，不推进关系版本。

### 3.3 真实零支持与当前候选

自检选择真实活动普通概念，通过 #210 正式入口逐条失效全部当前实例支持。每条请求独立原子：后续拒绝或并发新增支持不回滚前序合法失效。活动版本推进后，旧 #195 候选过期；只有重算候选的版本、观察、零支持和外部引用材料仍与当前事实一致，才允许进入生命周期转换。

### 3.4 重新激活与读取

冷却或退役概念通过正式固定点候选发布形成新有效支持后，可在当前材料复核通过时重新激活。主动召回只返回活跃概念；活动全图和抽象树不按生命周期删除节点，冷却 / 退役概念继续可读。

## 4. 入口拒绝与失败收口

| 场景 | 收口 |
| --- | --- |
| 无效 / 非活动概念、候选版本或规则不一致 | 写前逻辑内返回，不改变生命周期关系或活动版本 |
| 根离开活跃、非法跳转 | 写前逻辑内返回 |
| 同状态请求 | 幂等返回，不推进关系版本 |
| 当前仍有支持、外部引用、正向用途或近期用途 | 逻辑内返回，保持当前阶段 |
| 观察次数为零 | 返回材料不足，不把缺材料改写为冷却建议 |
| 前置满足后重挂失败 | 追根因解决，不发布成功材料 |
| 重挂后编号、版本、目标或唯一性读回不一致 | 追根因解决，不用扫描或回滚掩盖结构错误 |
| 主动召回含冷却 / 退役概念，或树投影丢失退役概念 | 追根因解决 |
