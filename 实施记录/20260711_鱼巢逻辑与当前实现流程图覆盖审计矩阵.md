## 1. 判定等级

| 等级 | 含义 |
| --- | --- |
| 完整现状包 | 有现状流程图 MD+HTML、逐行映射、输入契约、非成功二分审查、偏差清单，通过机械检查，并且代码基线与审计 HEAD 一致 |
| 批量现状基线 | 有现状图 MD+HTML 和合并核查表，但属于特定历史代码版本的自动证据基线；代码漂移、人工逐节点复核和独立纠偏计划仍需另查 |
| 部分现状包 | 已有现状图或映射，但必需产物不完整，或只覆盖目标的一段 |
| 施工图覆盖 | 有拟实施路径或实施时图，但未按实现后代码回绘现状图 |
| 摘要覆盖 | 只有边界图、代码逻辑摘要、迁移顺序或文档说明 |
| 未覆盖 | 没有能够绑定当前代码入口与调用链的流程图 |
| 排除 / 暂缓 | 明确不迁移或等待独立专项，不等于遗漏实现 |
