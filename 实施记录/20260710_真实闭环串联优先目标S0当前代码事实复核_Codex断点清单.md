## 5. 当前调用路径

`海中鱼巣/入口.cpp` 已存在一条同步样例链：

```text
创建目标状态需求
-> 按需求创建任务
-> 创建方法与动作入口
-> 选择任务方法
-> 读取动作执行桥请求材料
-> 直接调用执行特征状态写入方法
-> 直接记录动作动态证据和轻量因果引用
-> 直接记录任务实际结果状态与完成状态
-> 直接记录需求结算
-> 分别验证缓存、事件日志和只读显示边界
```

该链说明分散领域入口可在同一进程内按固定样例串行调用，但当前仍是 `main` 中的大型自检样例，不是运行宿主驱动的真实闭环编排。

## 6. 已确认缺口

1. `运行宿主` 的线程循环只检查停止标志并 `yield`，不分派调度周期消息，也不调用领域服务。
2. `任务线程壳` 的管理线程和工作线程只 `yield`；`任务工作线程消费一次并生成回执` 仅在调用方显式调用时消费一次，并且只生成结果回执材料。
3. 任务回执未携带可由任务服务复核的实际结果状态、动态证据或执行结果结构，未接入 `记录任务实际结果状态`、`记录任务完成状态` 和需求结算。
4. 缓存统计线程消费一次只返回缓存刷新消息，不调用 `统计服务`。
5. 事件日志线程消费一次只返回成功 / 模拟失败材料，不调用 `记录事件日志` 或 `记录结构事件日志`。
6. 显示服务只按请求边界返回状态，不读取需求、任务、方法、动态或结算结构。
7. 默认常驻入口只执行低频休眠，不创建或托管运行宿主、任务线程壳、缓存事件线程壳。
8. `有界运行消息队列` 当前容器和索引无并发同步保护；既有线程循环没有并发调用队列，因此现有验收不等于队列已可用于真实并发消费。
9. 当前没有一个领域编排入口把需求、任务、方法、动作、动态、回执、结算和观察材料封装为单次可复核闭环。
