## 2. 核心处理

1. `候选图准入状态` 先拆分可发布、未收敛、空候选和内部错误；基准版本过期由发布入口独立写前拒绝。
2. `候选图发布结果` 承载发布状态、请求基准版本、当前活动版本、异常关系句柄和异常节点句柄。
3. 已收敛候选内部错误、创建、生命周期、映射、关系写入和发布前读回异常均在首次发现点调用公共追根因检查。
4. 未发布候选按专用关系、生命周期关系、节点逆序清理；残留对象进入结构化异常句柄组。
5. 活动快照替换并保存上一候选是提交点；提交后旧边异常返回新版本和异常旧关系句柄。
6. 单树详细入口承接任意请求；全部四树详细入口承接正式契约。非活动根、零预算和预算耗尽是逻辑内返回；正式缺快照 / 四根或树内部不一致追根因。

## 4. 输出变化

```text
过期 / 未收敛 / 空候选 -> 独立逻辑返回状态，活动结构不变。
正式收敛候选内部错误 -> 候选内部错误，写前结构不变。
创建 / 生命周期 / 映射 / 关系 / 读回异常 -> 首次异常分类；清理完成后活动快照不变。
清理异常 -> 清理异常 + 残留关系 / 节点句柄。
提交后旧边异常 -> 已提交但旧关系失效异常 + 已生效版本 + 异常关系句柄。
任意单树拒绝 -> 活动图未就绪 / 非活动根 / 零预算 / 预算耗尽。
正式四树异常 -> 正式契约异常或活动图内部错误。
```
