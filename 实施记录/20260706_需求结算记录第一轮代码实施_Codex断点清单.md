## 1. 执行窗口规则

本窗口按用户新的协作模式执行：

```text
一个窗口负责设计和确认。
当前窗口只执行已经确认的代码实施步骤。
```

本切片不新增需求结算记录设计口径，只按已确认的 `需求结算记录规范` 和 `需求结算记录详细设计` 执行第一轮代码实施。


已确认第一轮范围：

```text
不新增需求结算记录节点类型。
以需求结算实例状态作为权威结算记录锚点。
需求节点通过需求结算记录关系挂回每次结算状态节点。
每次结算状态节点绑定来源任务、目标状态、实际状态和动态证据。
需求结算结果来自目标状态值和实际状态值比较。
最近结算记录按发生时间戳、关系顺序号和节点创建序号读取。
```

```text
旧需求结算字段迁移。
主信息结算摘要字段。
非权威需求结算摘要缓存。
复杂结算版本。
多目标结算。
完整因果可靠性判断。
```

## 3. 核心处理

本切片改造 `需求服务`：

```text
新增记录需求结算入口。
新增读取最近结算记录入口。
新增读取结算来源任务入口。
新增读取结算目标状态入口。
新增读取结算实际状态入口。
新增读取结算动态证据入口。
结算状态记录入口补缺发生时间戳拒绝。
```

同时补齐 `关系仓库` 只读关系记录组入口，用于读取需求结算记录关系的顺序号并支持最近结算记录排序。

## 4. 结构承载

代码修改：

```text
海中鱼巣/领域/需求服务.h
海中鱼巣/核心/关系仓库.h
海中鱼巣/核心/关系仓库.cpp
海中鱼巣/入口.cpp
```

结构映射：

```text
需求结算记录锚点：节点类型::状态 的实例状态节点。
需求结算记录关系：关系类型::归属，需求节点 -> 结算状态节点。
结算来源任务：关系类型::引用，结算状态节点 -> 任务节点。
结算目标状态：关系类型::模板，结算状态节点 -> 目标状态节点。
结算实际状态：关系类型::引用，结算状态节点 -> 实际状态节点。
结算动态证据：关系类型::因果来源，结算状态节点 -> 动态节点。
可选因果引用：关系类型::引用，结算状态节点 -> 因果引用节点。
```

## 6. 入口前置拒绝

本切片验证的拒绝路径：

```text
无效来源任务结算被拒绝。
缺实际状态结算被拒绝。
缺发生时间戳结算被拒绝。
缺动态证据结算被拒绝。
结算结果与目标状态 / 实际状态比较不一致时被拒绝。
```
