## 3. 实际实现

### 3.1 登记根与值式材料

新增：

```text
方法角色状态::方法登记根
方法生命周期状态::未定义 / 活跃 / 已失效
方法登记根材料
方法登记项材料
```

`初始化方法登记根` 使用稳定非名称键做同服务实例幂等定位，建立一个无虚拟存在的方法角色根、共享活跃 / 失效抽象状态，以及根到两个状态的固定顺序模板关系。同键重复初始化复用原材料，冲突键写前拒绝。

稳定键和服务内定位材料不独立裁决方法事实；根到方法首的 `归属` 关系才是正式发现发布边界。

### 3.2 方法首最终发布

`创建方法首节点` 保持原调用形状，但现在要求登记根已初始化，并按以下顺序写入：

```text
方法节点
-> 方法首角色状态
-> 方法虚拟存在及引用关系
-> 指向共享活跃状态的生命周期模板关系
-> 最后写登记根到方法首的归属关系
-> 完整读回登记项
```

最终归属关系写成前，裸方法或半结构不会进入登记枚举。现有 `登记方法` 继续保留为底层裸节点入口，不能被通用发现或动作候选入口接受。

### 3.3 生命周期与读取

新增公开入口：

```text
初始化方法登记根
读取方法登记根
读取方法登记项
读取全部方法登记项
读取活跃方法首组
失效方法首
```

失效入口只接受已登记活跃方法首，复用原生命周期关系编号重挂到共享失效状态并推进关系版本。失效项继续由全部登记项和单项审计读取返回，但 `方法首是否有效`、方法候选、动作入口和稳定动作键动作读取均排除失效方法。

批量枚举只从登记根的归属关系发现方法，不扫描全部节点；枚举复用一次根扫描取得的登记记录，避免逐项重复扫描根关系，再按完整节点句柄稳定排序并拒绝重复。

### 3.4 并发与诊断

方法服务新增私有共享互斥锁：根初始化、方法首创建和失效单写串行，根与登记读取返回值副本。逻辑错误分支统一先释放方法服务锁，再调用公共 `追根因检查`，不在登记锁内直接写日志或弹窗。

入口补齐主方法服务和隔离自我治理方法服务的登记根初始化，并新增 8 个并发创建、8 个并发失效样本。既有“多个虚拟存在应拒绝”负例改用未发布裸方法构造，继续验证读取拒绝，同时不污染权威登记根。

## 4. 入口拒绝与非成功二分

逻辑内返回：

```text
稳定非名称键为 0。
同服务实例使用冲突键重复初始化。
登记根未初始化时创建方法首。
目标不是完整已登记活跃方法首。
重复失效或生命周期版本已达上限。
```

这些分支不写登记结构；重复初始化和查询只读取既有材料。

追根因解决：

```text
既有登记根、共享状态或固定关系不完整。
方法节点、角色、虚拟存在、生命周期关系或最终登记关系创建不及预期。
最终发布后登记项不能完整读回。
登记根出现错误目标、重复方法或结构不完整登记项。
生命周期重挂不保留关系编号、不推进版本，或重挂后读回不一致。
```

追根因入口在释放方法服务锁后报告；不通过扫描、日志、删除或恢复逻辑修补结构。
