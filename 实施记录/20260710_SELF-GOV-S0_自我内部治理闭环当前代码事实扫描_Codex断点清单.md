## 7. 根需求与通用闭环之间的结构断点

### 7.1 两组样例未贯通

`自我根需求初始化读数` 在 `入口.cpp:408-508` 之后只用于 SELF-INIT 身份和读回验收。现有 E03-E11 链使用 `入口.cpp:1211` 起另建的自我、场景、特征、需求、任务和方法样例。

当前不存在：

```text
安全根需求 -> 任务。
服务根需求 -> 任务。
根需求 -> 固定方法 / 动作入口。
根需求当前特征槽位 -> 方法执行。
根需求实际结果 -> 任务完成 -> 需求结算。
```

### 7.2 特征槽位与通用结算不一致

根需求初始化在 `初始化.需求.ixx:124-130` 建立：

```text
自我存在 -> 实例特征槽位 -> 当前特征值。
实例特征槽位 -> 特征定义模板。
```

需求关系保存的目标特征是特征定义，当前材料是槽位下的特征值。现有 `需求服务::结算目标状态需求` 在 `需求服务.h:262-278` 使用：

```text
特征.读取I64特征值(目标特征, 当前特征值)
```

该读取要求当前特征值直接归属传入特征节点。对根需求而言，当前值归属实例槽位而不是特征定义，因此现有通用结算预览会返回证据不足。`方法服务::执行特征状态写入方法` 同样要求调用方传入真正拥有特征值的特征节点；当前治理材料尚未携带根需求实例槽位。

SG-S4B / S4C 必须显式贯通“目标宿主 + 特征定义 -> 实例槽位 -> 当前特征值”，不得把定义节点和实例槽位混为同一身份。

## 8. 队列、锁、等待与停止事实

| 对象 | 当前事实 | 后续门禁 |
| --- | --- | --- |
| 有界运行消息队列 | `deque`、两个 `unordered_map` 和停止布尔均无 `mutex` / `shared_mutex` | 不得由多个真实线程并发入队 / 出队；治理队列必须独立实现同步 |
| 运行宿主及五类工作线程 | 线程循环只检查原子停止标志并 `yield` | 当前属于忙等待壳，不得复制为治理等待机制 |
| 自我线程 | 使用一个互斥量和条件变量保护快照并等待停止 | 领域初始化调用发生在锁外；后续治理队列不得在持锁时调用领域服务 |
| 停止消息 | 可入队并把队列置为停止态 | 线程循环并不消费停止消息，实际停止依赖原子标志 |
| 当前任务线程收口 | 入口顺序发送管理 / 工作停止消息，再顺序 join | 只是顺序自检，不是生产者、桥、治理队列、自我线程的完整停止链 |
| 自我线程收口 | 入口返回时析构停止并 join | 没有显式“停止生产 -> 停止桥 -> 停止治理队列 -> join”编排 |

未发现队列锁内调用领域服务的问题，原因是当前运行队列根本没有锁，且线程循环没有调用队列或领域服务。这不等于并发安全或锁顺序验收通过。

## 9. 已确认缺口与风险

1. 自我线程没有治理消息提交、唤醒、冻结、处理或下一轮入口。
2. 运行消息使用原始 `uint64` 目标材料，不足以作为治理协议稳定句柄。
3. 运行队列无并发同步，不能直接作为治理队列复用。
4. 任务结果回执没有公开上行出队，也没有任务实际结果和动态证据的强类型材料。
5. 任务管理 / 工作、缓存和事件线程仅在入口顺序自检中短暂启动并立即停止，不参与正常常驻路径。
6. 安全 / 服务根需求只完成初始化读回，没有进入任务、方法、动作、结果或结算。
7. 根需求当前值归属实例特征槽位，通用需求结算按特征定义读取，当前不兼容。
8. 安全 / 服务根需求尚无固定方法、动作入口、稳定动作键和规格材料。
9. 缓存线程不调用统计服务，事件线程不调用日志入口，显示服务不读取真实闭环结构。
10. 当前证据没有证明多步领域写入的事务回滚或失效隔离；SG-S4 每段必须单独选择失败收口方式。
11. 默认常驻主循环只维持进程生命，不托管任务线程或调度治理批次。
12. SELF-GOV 和 REAL-LOOP 目前只有文档候选重叠，没有代码级单一运行编排所有者。
