## 4. 生产顺序

每个治理循环：

1. 读取唯一自我、内部世界、现实场景、运行代次和单调时间。
2. 冻结本轮事实截止版本；每个领域维护入口最多调用一次，但允许无变化。
3. 把新安全事件、复发、因素变化和因素排除交各自合法领域，把主动安全结算交 6100；SAFETY-C1 只读消费已经发布的主动结算后值，重置维护纪元并跳过受影响层同批被动回归。
4. 新的有效人类需求只经服务合同入口原子建立合同并结算 30% 预支；生产循环把已发布新合同结果、合同完成 / 取消 / 终止 / 到期、准备结果、完整秒和安全事实一次交给 `结算服务生存治理批次`。
5. 聚合入口在单事务内形成 70% 余款 / 关闭、到期未满足事件、准备补回、可审计衰减段和生存安全根回归；`A=0` 保持终止。
6. 从已发布一致快照先形成生存控制态；`A=0/1` 普通任务根权限均为零，只开放具名恢复 / 唤醒最小入口；`A>1` 才形成安全 / 服务根权限，安全来源再组合 6150 五级权限。
7. 根特征自检生成具名缺口；需求入口建立或复用需求。
8. 任务域承接需求并完成六类筹办；只有主动运行态正权限候选参加普通执行竞争，零权限和终止 / 休眠态任务保留事实但不派发。
9. 任务管理从当前候选集合重读权限、授权、状态、方法冻结和来源版本后稳定选择。
10. 方法执行、实际回读、任务完成、需求复核、合同结算、6100 主动安全结算和学习新代际分别落账；SAFETY-C1 不代写主动结算。
11. 下一轮只读取已经发布结果；同轮新学习代际不得反向改变本轮选择。

停止控制消息始终高于业务排序。运行消息四档不成为安全 / 服务权限。

## 5. 休眠与唤醒

```text
服务值 V = 0
+ 生存安全根值 A 不为终止态
+ 本秒无主动安全值变化
-> A 收敛到 1
-> 进入可由有效人类服务重新评估的休眠
```

休眠保留最小人类请求接收、合同准入、单调时间维护和安全事实接收入口。

```text
有效人类需求
-> 唯一合同成立
-> 30% 预支已经原子发布
-> V 大于 0
-> 后续完整秒真实安全门禁通过
-> A 从 1 至少增加到 2
-> 形成主动运行唤醒候选
```

消息、日志、合同记录或线程唤醒本身不得直接写 `A>1`。`A=0` 终止态不走该唤醒链。
