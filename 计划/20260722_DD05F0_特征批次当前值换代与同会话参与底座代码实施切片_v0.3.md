### 5.1 输出形态

临时诊断只在 Debug 自检宏下使用，固定格式：


使用 `std::cerr` 和 `std::endl` 或等义显式 `flush`，每条单独立即落盘。检查点只观察执行位置和耗时，不改变请求、返回值、分支、异常、锁序、许可、线程数量、等待方式、结构或业务结果，不得用日志裁决机器事实。

| --- | --- |
| P0 | 函数入口、夹具仓库 / 执行器 / 服务构造和基础夹具节点 |

| P5 | 新批次漂移、双参与者故障、撤销与分类 / 许可竞争 |
| P6 | A17 事务外预读、抢先发布、写权内重读和结果收口 |

### 5.3 细分规则

- 命中 P3：为每个工作线程记录开始 / 返回，为每次 join 记录开始 / 完成和线程索引。
- 命中 P4、P5 或 P6：只在该段每个同步服务调用、执行器许可、回调、参与者准备 / 确认 / 发布、账重读及竞争回调前后加立即刷新的成对检查点。
- 命中 P0—P2：只围绕该段同步仓库 / 服务 / 执行器调用细分。
- 命中 P7 或所有 P0—P7 均完成：记录函数返回前已完成事实并正式退回，不能在本计划内推断后续运行器 / 入口根因。
