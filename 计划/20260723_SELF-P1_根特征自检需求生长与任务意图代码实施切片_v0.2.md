## 1. 目标与依据

依据 5100—5160、6130—6150、7130 和 8200，实现：

```text
根特征合法自检缺口
+ SAFETY-C1 当前安全层 / 未归层 / 结构缺口
+ SAFETY-C2 当前权限来源
-> 需求目标、OR 路径与 AND 权重
-> 主需求
-> 可承接任务意图
```

安全层、深度、组和权限只读消费 #377 预冻结合同。SELF-P1 不计算或修改安全值、阈值、因果深度和权限。

## 2. 必须冻结

- SELF-C1 / v0.1 的 DTO、签名、版本、所有权、拒绝和内部错误；
- SAFETY-C1 / C2 来源版本在需求候选和任务意图中的值式引用；
- 无安全路径的“未归层”和应有路径缺失的结构缺口不得混淆；
- OR 继承完整父权重，AND 只属于一个 OR 并在组内平分；
- 精确允许文件、禁止文件、专属记录、回滚 / 隔离和验收矩阵。

## 3. 候选步骤

1. S0 读取当前 SELF 与 SAFETY 设计、计划索引和实际接口。
2. 形成根特征缺口候选；自检不直写需求。
3. 只读消费安全层和权限来源，安全值或因果材料漂移时拒绝本轮投影。
4. 经需求领域建立路径、权重、主需求和任务意图。
5. 定向验证未归层、零权限、非法 AND / OR、重复缺口和版本漂移。

## 4. 边界

本切片不承接任务、不筹办、不创建方法、不执行动作、不结算需求，不修改安全事实、共享入口、工程、统一运行器或生产装配。安全权限为零不阻止需求存在、任务意图形成或后续筹办。
