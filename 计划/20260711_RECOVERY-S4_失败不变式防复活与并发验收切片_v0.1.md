## 4. 实施步骤

### S0：实际接口与完成声明复核

读取 #220、#225、#226-#229 的实施记录、正式接口和验收项，建立逐条调用映射。任一格式、状态枚举、墓碑规则、任务终态或宿主发布语义漂移时不得修改代码，退回设计窗口修订 #230。

### S1：格式与失败不变式矩阵

覆盖魔数、版本、长度、段表、重叠、校验、枚举、所有者和跨段冲突。外部坏材料逻辑拒绝并证明正式快照、当前宿主和所有候选均无半结构；当前写入方生成矛盾材料、自读失败或严格验证后导入失败必须进入追根因。

### S2：防复活矩阵

逐项验证删除节点 / 主信息 / 关系墓碑、身份与创建序号高水位、失效关系、历史端点、索引键保留、退役 / 物理删除概念、失效支持、活动版本、任务终态和迟到回执。禁止从最大有效记录重算高水位、从端点有效性重建失效关系或从旧候选 / 回执复活领域事实。

### S3：瞬态材料排除

验证恢复后的线程、队列、消息、回执、幂等表和会话计数均为空或重新构造；HYEVSEG1 存在、缺失或损坏都不触发事件重放、补齐快照或改变当前事实。

### S4：确定性与并发压力

同一冻结材料分别单线程 / 多线程编码，字节与摘要完全一致；同一文件分别单线程 / 多线程解析，值式候选完全一致。持续写入与快照请求交错至少 50 轮，每份快照必须对应一个完整冻结点；端到端生成、严格读回、隔离恢复和屏障至少连续 50 轮。

### S5：最终回归与边界审计

运行 Debug x64、完整自检、最终连续 20 轮以及 PERSIST、RECOVERY、STRUCT-TXN、FEATURE、CONCEPT、METHOD、TASK、DEMAND、SELF-GOV、PANEL 回归。静态扫描确认没有事件重放、在线热替换、跨版本兼容、断电耐久或任务恰好一次的实现 / 声明。

### S6：治理归档

逐项记录 RECOVERY-S4-A01 至 A18，归档计划，更新治理，提交并推送。
