# 海中鱼巣全量重构总计划

编号：HR-001
日期：2026-07-05
状态：v0.1 / 已建立 / 待执行

## 1. 目标

在空 C++ 控制台项目 `海中鱼巣` 中，以全部重构方式重建鱼巢第二代工程内核。

本计划不以旧项目文件搬迁为目标，而以重新建立清晰的内核协议为目标：

```text
节点仓库
稳定句柄
主信息仓库
关系仓库
索引仓库
事务边界
领域服务
最小需求任务方法闭环
```

## 2. 命名硬约束

1. 函数名必须使用中文。
2. 变量名必须使用中文。
3. 类名、结构名、枚举名、命名空间名优先使用中文。
4. 文件名优先使用中文，底层通用文件可使用清晰中文名。
5. 英文只允许用于：
   - C++ 关键字。
   - 标准库标识，例如 `std::vector`、`std::string`。
   - 第三方库 API。
   - Visual Studio / MSBuild 固定字段。
   - 外部协议无法改名的字段。
6. 禁止为了方便临时新增英文业务变量名。

示例方向：

```cpp
struct 节点句柄 {
    std::uint64_t 仓库编号 = 0;
    std::uint64_t 节点编号 = 0;
    std::uint32_t 版本号 = 0;
};

class 节点仓库 {
public:
    节点句柄 创建节点();
    bool 删除节点(节点句柄 节点);
};
```

## 5. 阶段 0：冻结重构总纲

输出文件：

```text
规范/000_项目规则总纲.md
规范/001_规则迁移清单.md
规范/重构总纲.md
规范/最小闭环设计.md
```

必须回答：

| 问题 | 结论要求 |
| --- | --- |
| 海中鱼巣保留哪些鱼巢核心概念 | 写成第一版范围和暂缓范围 |
| 节点身份用什么 | 固定 `节点句柄` / `节点编号` / `版本号` |
| 主信息如何存 | 固定主信息仓库和主信息编号 |
| 关系如何存 | 区分普通父子、归属、引用、索引、模板、运行期临时关系 |
| 哪些逻辑必须结构化 | 所有机器判断必须走节点、特征、特征值、关系和因果引用 |
| 哪些只是显示 | 日志、控制台输出、调试文字、说明文档 |

验收：

```text
项目规则总纲存在
规则迁移清单存在
重构总纲存在
最小闭环设计存在
命名硬约束已写入
暂缓范围已写入
第一版闭环输入 / 输出 / 验证方式已写入
```

## 6. 阶段 1：核心内核

建议目录：

```text
海中鱼巣/
  核心/
    句柄.h
    节点仓库.h
    节点仓库.cpp
    主信息仓库.h
    主信息仓库.cpp
    关系仓库.h
    关系仓库.cpp
    索引仓库.h
    索引仓库.cpp
    写入事务.h
    写入事务.cpp
```

核心结构：

```text
节点句柄
主信息句柄
关系句柄
节点记录
主信息记录
关系记录
```

节点记录至少包含：

```text
节点编号
节点类型
主信息句柄
版本号
有效状态
创建时间
```

关系记录至少包含：

```text
关系编号
关系类型
源节点句柄
目标节点句柄
顺序号
版本号
有效状态
```

第一批函数：

```text
创建节点
读取节点
删除节点
重挂节点
按主键查节点
获取父节点
获取子节点
节点是否有效
校验结构
```

验收：

```text
可创建节点
可删除节点
可重挂节点
可按父节点枚举子节点
删除节点后旧句柄失效
结构校验能发现孤儿关系、重复父关系和失效引用
```

## 7. 阶段 2：领域服务

建议目录：

```text
海中鱼巣/
  领域/
    语素服务.h
    世界服务.h
    特征服务.h
    特征值服务.h
    需求服务.h
    任务服务.h
    方法服务.h
    因果服务.h
```

领域服务职责：

| 服务 | 职责 |
| --- | --- |
| 语素服务 | 自然语言词面到项目身份入口 |
| 世界服务 | 基础信息、存在、场景、世界树事实入口 |
| 特征服务 | 特征类型、实例特征、当前值和历史值入口 |
| 特征值服务 | I64、VecI64、VecIU64、指针句柄等值承载 |
| 需求服务 | 需求创建、目标读取、满足判断 |
| 任务服务 | 由需求承接任务、任务状态推进 |
| 方法服务 | 方法登记、候选召回、执行入口 |
| 因果服务 | 动态来源、结果证据、因果引用链 |

硬约束：

```text
业务层不得直接写节点仓库
业务层不得直接写关系仓库
业务层不得直接操作主信息内部字段完成机器裁决
所有写入必须经过对应领域服务
```

## 8. 阶段 3：最小闭环

第一条闭环：

```text
创建自我存在
创建现实场景
创建一个特征类型
写入自我某个 I64 特征值
创建需求：让该特征达到目标值
由需求生成任务
任务选择一个手写方法
方法修改特征值
记录动作动态
需求根据结果结算
```

验收证据：

```text
输入入口：创建需求
核心处理：任务承接和方法执行
结构承载：存在、场景、特征、特征值、需求、任务、方法、动态、因果引用
输出变化：目标特征值变化
调用路径：需求服务 -> 任务服务 -> 方法服务 -> 世界服务 / 特征服务 -> 因果服务
验证方式：控制台输出只读显示 + 内部结构校验
```

禁止宣称：

```text
不得宣称自我苏醒
不得宣称完整自我循环
不得宣称旧项目能力完成迁移
不得用控制台输出替代结构事实
```

## 9. 阶段 4：旧项目能力迁移

迁移顺序：

```text
语素系统
-> 基础信息 / 世界树
-> 特征 / 特征值
-> 需求树
-> 任务管理
-> 方法系统
-> 状态 / 动态 / 因果
-> 控制台只读显示
-> 持久化
-> 外设观察
-> 体素系统
```

每迁移一项都必须写：

```text
旧项目入口
新项目入口
保留行为
删除行为
新增约束
验证样例
禁止宣称
```

## 12. 反例检查

| 风险 | 反例 | 防线 |
| --- | --- | --- |
| 直接复制旧项目结构 | 把双向链表模板搬进新项目当核心 | 第一版使用节点仓库和关系仓库 |
| 指针继续当长期身份 | 重启后节点引用失效 | 使用节点句柄和版本号 |
| 关系混乱 | 场景成员、存在特征、方法条件都写成普通父子 | 关系类型显式区分 |
| 日志替代事实 | 控制台显示成功但结构未变化 | 验收必须读节点、关系、特征值 |
| 过早接外设 | D455 复杂输入拖垮内核验证 | 第一版只做手写方法闭环 |
| 数据库绑死核心 | SQL 失败导致机器逻辑不可运行 | 数据库先做持久化适配器 |
