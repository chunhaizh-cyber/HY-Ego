## 1. 目标

在空 C++ 控制台项目 `海中鱼巣` 中，以全部重构方式重建鱼巢第二代工程内核。

本计划不以旧项目文件搬迁为目标，而以重新建立清晰的内核协议为目标：

```text
节点仓库
稳定句柄
主信息仓库
关系仓库
索引仓库

领域服务
最小需求任务方法闭环
```

## 2. 命名硬约束

1. 函数名必须使用中文。
2. 变量名必须使用中文。
3. 类名、结构名、枚举名、命名空间名优先使用中文。
4. 文件名优先使用中文，底层通用文件可使用清晰中文名。
5. 英文只允许用于：
   - C++ 关键字。
   - 标准库标识，例如 `std::vector`、`std::string`。
   - 第三方库 API。
   - Visual Studio / MSBuild 固定字段。
   - 外部协议无法改名的字段。
6. 禁止为了方便临时新增英文业务变量名。

示例方向：

```cpp
struct 节点句柄 {
    std::uint64_t 仓库编号 = 0;
    std::uint64_t 节点编号 = 0;
    std::uint32_t 版本号 = 0;
};

class 节点仓库 {
public:
    节点句柄 创建节点();
    bool 删除节点(节点句柄 节点);
};
```

## 7. 阶段 2：领域服务

建议目录：

```text

```

领域服务职责：

| 服务 | 职责 |
| --- | --- |
| 特征值服务 | 只服务于特征系统的当前值、历史值或值句柄承载 |
| 特征服务 | 特征类型、实例特征入口 |
| 存在服务 | 实际存在和虚拟存在入口 |
| 场景服务 | 场景入口，场景内临时实例状态和实例动态收纳 |
| 状态服务 | 实例状态、抽象状态和状态提升边界 |
| 动态服务 | 实例动态、抽象动态和动态提升边界 |
| 二次特征服务 | 关系性、程度性或复合判断前置特征 |
| 因果服务 | 因果引用、证据和动态抽象候选 |
| 世界服务 | 基础信息聚合编排，不绕过原子服务写结构 |
| 需求服务 | 需求创建、目标读取、满足判断 |
| 任务服务 | 由需求承接任务、任务状态推进 |
| 方法服务 | 方法登记、候选召回、执行入口 |
| 语素服务 | 自然语言词面到已存在项目身份入口 |

硬约束：

```text
业务层不得直接写节点仓库
业务层不得直接写关系仓库
业务层不得直接操作主信息内部字段完成机器裁决
所有写入必须经过对应领域服务
世界服务不得绕过基础信息原子服务直接写基础信息结构
特征值服务不得作为需求、任务、方法的通用值容器；只有特征服务可直接访问特征值服务
需求、任务、方法服务必须通过基础信息服务获得存在、场景、特征等能力
```
