# SELF-P3 方法执行实际回读、任务完成与需求结算代码实施切片 v0.2

身份：`#357 / DQ-249 / SELF-P3 v0.2`

日期：2026-07-24

设计状态：范围候选；本文不维护执行过程，精确允许文件和完整 SELF-C4 施工接口尚需后继设计冻结。

## 1. 目标与依据

消费 SELF-C1—C3 预冻结合同，提供 SELF-C4 / v0.1：

```text
方法实际结果
-> 正式状态 / 动态 / 因果回读
-> 任务完成
-> 需求满足
-> 价值结算
-> 下一轮输入
```

任务完成、需求满足和价值结算分别裁决。安全事件、复发、因素变化或验证排除先由各自正式入口发布，再由 `SAFETY-C1 / v1` 在后继维护轮次消费；任务完成不得直接增加、降低或重置安全值。

## 2. 必须冻结

- 方法实际结果、事实回读、任务验证、需求满足、价值结算和下一轮输入 DTO；
- 新安全事件、主动安全结算和因素排除的合法提交边界及 SAFETY-C1 来源版本；
- 材料不足、目标未达成和不可比较的逻辑内结果；
- 前置通过后写后读回、跨域发布或三类裁决混账的内部错误；
- 精确允许 / 禁止文件、专属记录、失败收口和验收矩阵。

## 3. 候选步骤

1. S0 核对 SELF-C1—C4、SAFETY-C1 和实际状态 / 动态 / 因果接口。
2. 执行已冻结方法并回读权威事实。
3. 分别提交任务完成、需求满足和价值结算。
4. 必要时经正式入口提交安全事件 / 因素事实，不在任务结果组合器内维护安全值。
5. 形成下一轮输入并验证重复、失败、禁止结果、版本漂移和原子收口。

本切片不计算安全层、权限或被动回归，不修改线程、工程、入口或统一运行器，不把任务成功、日志或返回码当作安全事实。
