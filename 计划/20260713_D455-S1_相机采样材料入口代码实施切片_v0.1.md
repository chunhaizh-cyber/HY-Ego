## 8. SDK 与构建合同

1. 工程新增 build-system 属性 `EnableD455RealSense`，默认关闭；英文只作为 MSBuild 字段使用。
2. 默认构建仍编译协议、缓存、上行桥和模拟自检；真实采集器与真实样本模块在非 x64 或属性未启用时条件排除。
3. 只有 x64 且 `EnableD455RealSense=true` 时定义源码可读的 D455 真实采集可用宏，加入 `$(VcpkgRoot)` 或 `$(VCPKG_ROOT)` 派生的 include / lib 路径。
4. 工程不得写死 `D:\vcpkg`；当前机器通过命令行属性或环境属性提供根路径。
5. Debug x64 链接并复制 `realsense2d.lib/.dll`；Release x64 链接并复制 `realsense2.lib/.dll`。
6. Win32 不链接 x64 SDK，也不构建真实采集器和真实样本模块；模拟材料链仍不得依赖 SDK。
7. 未启用真实采集时传入 `--d455-real-sample-exit` 必须明确非零返回“当前构建未启用”，不得静默当作通过。

## 9. 实施步骤

### S0 执行前复核

确认分支包含 #265 与 #272 正式结果，读取更新后的入口、运行器、工程和 SDK。确认阶段 840 空闲、允许文件无并行冲突，记录实施前提交和允许 / 禁止文件基线。

### S1 协议与材料

建立 SDK 无关类型、中文业务入口、固定 RGB8 布局、四路内参和三组固定外参。入口先拒绝零尺寸、零帧率、未知格式、零超时、无效身份、编号、版本和批次。

### S2 RealSense 采集器

唯一包含 `librealsense2/rs2.hpp`。按序列号唯一选择 D455，显式配置四路 profile，有限等待同一 frameset，在下一次等待前复制全部字节和标定；任一路缺失不发布。部分启动失败和显式停止均必须释放 SDK 资源。

### S3 专用有界缓存

实现条数、总字节、单帧和年龄四重上限；状态区分未发布候选与已发布。满载拒绝新材料，不静默驱逐仍可读材料；候选、发布、撤销、过期和停止接收均按编号与版本精确处理。

### S4 上行桥

固定顺序为入口拒绝、采样、材料复核、准备候选、候选读回、构造标量消息、确认发布、普通读回和返回。任何内部不一致精确撤销并进入追根因结果；逻辑内拒绝不留下可读半材料。本轮不调用运行消息队列。

### S5 自检与入口最小登记

模拟模块覆盖材料、拒绝、容量、过期、撤销、消息引用和停止；真实模块只在显式参数和启用构建中运行。入口不出现 SDK include、帧构造、采样循环、缓存正文、并发循环或逐项输出，只向中央运行器登记值式自检入口并汇总退出码。

### S6 工程接线与验证

登记六个真模块、条件 SDK 属性和 DLL 复制；先完成无 SDK 默认回归，再完成 Debug / Release x64 SDK 启用构建和模拟自检。只有连接真实 D455 并显式执行后，才能记录真实样本结果。

## 10. 非成功返回与失败收口

逻辑内返回包括配置无效、无设备、多设备歧义、指定设备不存在、profile 不支持、有限等待超时、设备断开、四路候选不完整、缓存超限 / 满载 / 过期、普通错误版本读取和构建未启用真实采集。它们不得留下新可读材料或消息。

前置通过后出现完整候选布局 / 标定 / 来源自相矛盾、候选读回不一致、确认发布异常、普通读回或消息引用悬空、字节计数失真、精确撤销目标不一致、部分启动资源无法释放、停止后有限等待无法收口，均为追根因解决。不得统一映射为普通采集失败后继续验收；公共容错宏开启时按逻辑错误日志和弹窗规则执行，但诊断不得参与修复或成功裁决。
