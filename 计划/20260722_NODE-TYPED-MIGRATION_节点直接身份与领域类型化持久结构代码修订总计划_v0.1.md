1. 当前 main 上有 99 个代码文件引用旧结构名称，其中生产 55、自检 44。
2. 节点仓库当前没有直接稳定主键字段；节点创建依赖旧附属记录。
3. 特征值 I64 和部分领域时间 / 附属材料仍依赖旧值槽或旧句柄。
4. 7 个生产数据操作已经使用结构会话，13 个兼容服务仍存在直接创建旧附属记录的路径；具体入口须在 P0 按最新 main 重扫。
5. 四仓具备值式导出而无指定身份导入；快照服务不允许替换运行期仓库。
6. 运行期上下文宿主只支持空槽首次发布。该边界保留，禁止在线热替换。
7. 4170 侧账无全量冻结和恢复注入，不能单独宣称快照恢复接通。

## 7. 失败收口与回退

每个代码子计划采用新 worktree 和固定基线。入口拒绝必须零写；确认前失败精确撤销；撤销不闭合则隔离事务域。已发布结构不使用候选撤销回滚。

迁移采用阶段性代码兼容而非阶段性双权威：一个入口可以暂时适配旧调用请求，但每个已迁对象在同一运行期只能有一个权威承载。任一阶段验证失败时保留任务分支证据并退回设计，不把旧结构重新解释为规范允许。
