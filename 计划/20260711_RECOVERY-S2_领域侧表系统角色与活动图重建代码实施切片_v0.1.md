## 4. 实施步骤

### S0：实际接口与段 8 明细复核

列出 #191、#220、#203-#225 提交号，确认所有新登记 / 生命周期 / 选择 / 结果事实的真实承载。若存在未列入段 8 的私有服务侧权威状态，停止实施，退回设计并先修订 #226 格式；不得复制私有对象内存。

### S1：特征值侧表导入

在隔离特征值服务中导入 I64 版本与主信息 I64 值、VecI64 / VecU64 完整值 / 类型 / 版本。逐节点复核类型唯一、身份当前、I64 / Vec 不并存，并复用 #191 当前性读回。

### S2：系统角色清单

新增版本 1 角色枚举和值式清单，覆盖语素、世界、自我、场景、两个根需求、目标材料、概念四根 / 名称 / 生命周期，以及实际方法 / 任务固定角色。恢复只注入既有句柄并读回，不调用新建初始化。

### S3：索引所有者复核

按所有者分组，让各初始化 / 领域服务从隔离权威结构重算键、目标和保留策略。候选与重算集合精确相等后才绑定隔离索引，并做正向 / 反向读回；缺所有者、多所有者、未知版本或键冲突拒绝整个候选。

### S4：领域登记

从仓库和显式侧表重建方法登记 / 规格 / 生命周期、任务当前态 / 选择 / 结果、命名 / 删除登记等实际结构。派生召回索引、缓存、执行请求和回执保持空。已验证导入后读回矛盾追根因。

### S5：概念图重建

注入四根和生命周期角色，从当前 / 审计关系重建专用登记，全量重算活动节点 / 关系 / 签名并与快照候选比较；精确保留活动版本，上一候选图为空。退役、失效支持和物理删除边界保持。

### S6：自检与治理

补齐 RECOVERY-S2-A01 至 A18，完成规范、实施记录、计划归档、治理、提交和推送。
