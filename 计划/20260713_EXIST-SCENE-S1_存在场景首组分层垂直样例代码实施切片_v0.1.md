## 2. 已验证当前事实

1. #264 已实现不可默认构造、不可复制、不可移动的 `结构写入会话`；执行器每次只取得一次独占许可，并把会话左值引用同步传给当前线程回调。
2. 会话公开能力固定为主信息 / 节点候选创建、关系创建、主键绑定、四类同令牌读回、请求提交和请求撤销；不提供节点类型读取、任意仓库读取或原始令牌出口。
3. 会话精确写集保存主信息 / 节点候选、完整关系句柄和“主键 + 预期节点完整句柄”；提交前强制写集全部同令牌读回，失败、异常、显式撤销和无提交决定在许可释放前逆序收口。
4. 既有 `存在服务` 已提供实际存在、虚拟存在和手写候选路径；`场景服务` 提供场景创建与接纳；`世界服务::创建场景存在(场景)` 只在已有场景中创建实际存在并接纳，不同时新建场景和存在。
5. 既有三服务仍有无令牌、带令牌和手写候选双路径。本切片不删除、改义或接线迁移这些兼容入口。
6. 关系结构化创建拒绝完全重复关系，不把重复创建直接作为幂等成功。索引结构化绑定可区分首次写入、同键同节点幂等读回和冲突节点拒绝。
7. 当前节点类型在同一节点生命周期版本内不可变。数据操作层可先返回值式节点类型材料，再在独占会话内以同一完整句柄调用 `节点可读` 复核当前性，不需要扩展 #264 会话。

## 3. 实施目标与固定路径

建立以下分层：

```text
调用方
-> 存在业务服务 / 场景业务服务 / 存在场景组合器
-> 数据操作.存在场景
-> 数据操作模块内部调用 #264 结构写入执行器
-> 一次独占许可内的不透明结构写入会话
-> 同会话读回
-> 提交确认或精确逆序撤销
-> 值式领域结果
```

五条固定路径：

1. 创建实际存在：`幂等主键 -> 主信息候选 -> 存在节点候选 -> 主键绑定 -> 全部读回 -> 提交`。
2. 创建虚拟存在：`幂等主键 + 来源需求 / 任务 / 方法 -> 存在身份 -> 主键绑定 -> 来源引用最后形成 -> 全部读回 -> 提交`。
3. 创建场景：`幂等主键 -> 主信息候选 -> 场景节点候选 -> 主键绑定 -> 全部读回 -> 提交`。
4. 场景接纳已有存在：`已有场景 + 已有存在 -> 场景到存在归属 -> 读回 -> 提交`；同一当前关系已存在时幂等读回。
5. 在已有场景中创建并接纳实际存在：`已有场景 + 存在幂等主键 -> 创建缺失的实际存在身份和 / 或归属 -> 全部读回 -> 提交`。

第一轮不同时新建场景和存在，不实现虚拟存在创建并接纳，不收纳状态 / 动态。

## 4. 请求、幂等与结果合同

创建实际存在、虚拟存在和场景均要求非零全局 `uint64` 幂等主键。主键未绑定时允许写入；已绑定到同类型、同语义当前身份时返回幂等读回；绑定到错误类型、过期版本、不同节点或不同虚拟来源时写前冲突拒绝。

实际存在幂等读回必须排除需求 / 任务 / 方法来源语义；虚拟存在幂等读回必须证明来源关系唯一且与请求来源一致。场景接纳先读取权威归属材料：同一当前关系已存在则幂等读回；没有关系才写入；同一端点多条、版本矛盾或不可唯一解释时走追根因路径。

并发窗口导致关系创建或主键绑定被拒绝时，执行器先完整撤销本次写集；许可释放后由数据操作层重读权威结构。只有重读证明同键同语义身份或同一当前归属关系，才映射为幂等成功。

公开领域结果至少区分：

```text
已提交
幂等读回
入口拒绝
幂等冲突
许可拒绝
版本漂移
内部不一致
```

结果只返回必要的存在 / 场景 / 关系完整稳定句柄，不暴露仓库、会话、许可、令牌、锁或候选。

## 8. 实施步骤

### S1 值式材料和数据操作层

建立存在场景数据操作模块。模块私有持有结构写入执行器和必要的只读仓库引用，提供值式节点身份、主键绑定、虚拟来源和场景归属材料，不向上暴露仓库。只有该模块可在生产路径导入执行器并在同步回调中使用会话。

### S2 存在与场景业务服务

建立 `存在业务服务` 和 `场景业务服务`。所有参数先做非零主键、完整句柄、类型、来源、重复和冲突检查；入口拒绝不进入执行器。结构写入结果映射为领域结果，内部不一致走追根因，不用普通失败兜底。

### S3 组合器

建立 `存在场景组合器`，只实现“已有场景中创建并接纳实际存在”。组合器调用类型化业务 / 数据操作入口，不导入执行器，不持有仓库。已有同键实际存在时只补缺失归属；两者均已存在时幂等返回。

### S4 隔离自检和阶段登记

新增独立自检模块。入口仅 import 自检模块、保存一个值式结果、登记最终回归阶段 720 并汇总。完整场景、并发循环、数量快照和逐项输出留在自检模块。

故障与并发验证优先使用隔离仓库预置冲突、过期完整句柄、重复关系竞争和主键竞争，不给生产请求增加故障注入字段。#264 A01-A12 必须作为通用写点撤销回归共同通过。
