## 1. 任务定位

本计划承接 FS-06 方法系统只读扫描和逻辑迁移包归并材料，目标是把可进入第一轮增强的方法服务能力转成后续可执行路线。

本计划是确认对象。`实施记录/20260707_FS06_方法系统逻辑迁移包归并表.md` 只作为生成本计划的证据材料，不是本窗口最终目标。

本计划不迁移旧函数体，不新增旧方法主信息字段，不直接进入 C++ 实施。代码实施必须另建切片，并再次确认允许文件、禁止文件和验证方式。

## 2. 纳入范围

第一轮只纳入：

```text
FS-06-A：方法角色身份与角色读取。
FS-06-C：方法条件 / 结果节点与规格场景。
FS-06-D：方法虚拟存在与方法能力基底。
```

第一轮目标是补齐方法服务角色读取、条件 / 结果节点读取、方法虚拟存在读取和拒绝路径，不实现完整方法查找、候选排序、动作执行或方法学习。

## 3. 排除范围

本计划排除：

```text
FS-06-B：方法首注册和稳定读取。
FS-06-E：方法动作入口与方法调用动作动态。
FS-06-F：方法结果能力、候选召回和结果方向。
FS-06-G：来源任务、父方法、前置 / 后续方法关系。
FS-06-H：方法结构状态、可用性和条件结果对。

FS-06-X：SQL / 控制面板 / 旧主信息字段 / 本能兼容排除。
```

## 5. 设计条款清单

### 必须实现

```text
方法节点身份必须经方法服务复核节点类型。
方法首、方法条件、方法结果角色必须通过角色状态和关系读取。
方法条件 / 结果节点必须能追溯有效方法首。
方法条件 / 结果场景必须通过关系读取，不得写旧主信息字段。
方法虚拟存在必须由方法服务经存在服务创建或读取，并通过关系绑定。
读取入口遇到无效节点、错误角色、多目标或目标类型错误时必须返回空或拒绝。
拒绝路径必须保证节点、关系、状态和场景结构不变化。
```

### 禁止实现

```text
不实现方法候选召回。
不实现结果能力索引。
不实现来源任务、父方法、前置方法和后续方法绑定。
不实现方法调用动作动态。
不实现条件结果对归并。
不实现运行统计或练习样本缓存。
不实现本能方法兼容入口。
```

### 后续待补

```text
方法首注册和稳定索引专项。
动作入口和方法调用动作动态专项。
结果能力和候选召回专项。
方法来源 / 父 / 前置 / 后续关系专项。
条件结果对和方法学习专项。
方法统计非权威缓存专项。
显示层只读视图专项。
```
