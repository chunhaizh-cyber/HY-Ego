### 8.2 公开服务签名


内部提供接口精确为：

```cpp
class 安全直接因果图提供者 {

class 安全维护事实提供者 {
public:

        const 安全维护事实来源请求& 请求) const = 0;

class 安全维护外部事实参与提供者 {
```

## 10. 算法与事务硬合同

1. `D=0` 不成组；`D=1..4` 同号，`D>=5` 进入组 5 且保留真实深度。
2. `A1=M`；仅 `g=1..4` 计算上层收到权限；`Q5=A5`；五组总和精确等于 `M`。
3. 全部乘法使用 checked 128 位或等价安全乘除；禁止 float / double、绕回、钳制和 `I64_MAX` 兜底。
4. 组 5 依次比较权限、后果严重程度、安全不足度、预计 / 时间证据、因果证据、基础优先级、真实深度、规范化最短路径键、来源需求稳定句柄、任务创建序号和任务稳定句柄。
5. 安全不足度固定为 `clamp(floor((Vmax-V)*1,000,000/(Vmax-Vmin)),0,1,000,000)`，使用 checked wide integer；禁止以高位阈值替代 `Vmax`。
6. 因果证据序先要求适用范围、规则和证据截止版本全部当前，再按当前反例数升序、有效基础正样本数降序比较；版本号不作为强度，禁止综合置信度字段。
7. 新事件 / 复发 / 因素变化先进入主动安全事实结算，再读取因素验证排除并重算残余约束，最后裁决被动维护；受影响层同批跳过被动回归并重置维护纪元。
8. 安全值变化事务只发布安全记录、当前关系、状态、动态、因果 / 关系和索引；权限在发布后从一致性快照重算。
9. 任一准备失败逆序撤销；确认后读回不一致、撤销失败或半发布属于内部错误。
10. 同幂等键同义读回，异义拒绝；循环次数、队列压力和无消息时间不进入数值。
