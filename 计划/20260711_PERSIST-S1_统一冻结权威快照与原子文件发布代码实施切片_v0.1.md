## 1. 目标

在一个结构事务冻结点复制全部权威材料，释放冻结后规范编码 HYSNAP01，并以严格读回和同卷原子替换发布固定快照文件：

```text
显式请求
-> 同域独占冻结
-> 四仓库 / 侧表 / 角色 / 登记 / 活动证据快速复制
-> 释放冻结
-> 规范排序与并行编码
-> 临时文件写穿和严格读回
-> 同卷替换正式快照
```

本切片不解析外部快照到仓库、不构造恢复上下文、不修改启动顺序、不启动线程或事件重放。

## 4. 实施步骤

### S0：实际接口冻结

逐项读取 #217、#220、#203-#225 实施记录与代码，生成“权威状态所有者 -> 当前承载 -> 导出入口 -> 段字段”映射。发现计划后新增但本设计未列明的服务侧权威状态时，停止修改并退回设计；不得使用通用字节段。

### S1：格式和值式材料

新增显式小端逐字段格式、固定段类型 / 版本、仓库全状态、高水位、特征原始材料、系统角色、索引所有者、领域登记和概念活动候选 DTO。所有身份使用完整句柄，不持久化指针、互斥量、容器内存或显示文本。

### S2：全状态导出

四仓库新增令牌感知只读导出，覆盖全部记录状态和精确高水位。特征值 / 概念 / 系统角色 / 实际领域所有者新增最小值式导出。索引全键集合必须与各所有者声明集合精确相等。

### S3：冻结编排

仓库快照服务取得一次 #217 同域独占许可，按固定顺序逐所有者复制；一次只持有一个内部锁。冻结内完成同域 / 身份 / 高水位 / 集合一致性复核，随后释放许可。禁止锁内文件 I/O、规范排序、大字节编码、日志或回调。

### S4：冻结外编码

按详细设计 5.4 规范排序。允许独立段并行编码和段校验，最后串行汇总段表、跨段摘要、总长度和总头。单 / 多线程必须生成相同字节。

### S5：文件发布

固定生产目标 `快照/权威状态.hysnap`，自检使用专用临时根。执行 CREATE_NEW、FlushFileBuffers、当前解析器严格读回、MoveFileExW 同卷替换和最终读回。运行文件加入 Git 排除，不能提交测试快照。

### S6：自检与治理

补齐 PERSIST-S1-A01 至 A16、并发冻结和文件拒绝矩阵；完成规范、实施记录、计划归档、治理、提交和推送。

## 8. 失败收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 前置未完成或实际侧表漂移 | 依赖门控返回 | 不改代码或停止修改，退回设计 |
| 请求、路径、版本、预算无效 | 逻辑内返回 | 不冻结、不写临时文件 |
| 当前冻结材料内部矛盾 | 追根因解决 | 释放许可，不编码、不发布 |
| 临时文件创建 / 权限 / 空间 / 写穿失败 | 逻辑内返回 | 清理本次临时候选，旧正式快照保持 |
| 当前写入方临时文件无法被当前解析器读回 | 追根因解决 | 不替换旧文件，停止依赖路径 |
| 同卷替换前候选过期 | 逻辑内返回 | 丢弃候选，旧文件保持 |
| 替换后最终严格读回不一致 | 追根因解决 | 不宣称发布成功，停止恢复依赖 |
