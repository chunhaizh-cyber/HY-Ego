## 1. 目标与依据

依据恢复流程图 v0.3、消费者并行流程图 v0.5、详细设计第 17—19 节和 #295 事实矩阵。本切片只切换默认生产可达图：默认在旧局部仓库 / 服务构造前进入新运行期通道；旧大段继续仅由 Debug 自检 / 兼容证据模式进入，不迁移约 11,000 行入口自检正文，不增加 Release `--legacy-runtime` 逃生口。

来源性质为 MIG-18 重新实现 / 新增约束，不计旧能力迁移。

## 2. 依赖门控

必须先有：

```text
#309 已证明 #299 / #302 两路批次和后继 #301 串行任务均正式集成、920 / 940 / 950 和同租约交叉验收全部通过。
#297 显式新通道实际接口保持有效。
```

#300 未完成只有一个允许例外：静态扫描与运行专项同时证明旧概念命名写路由在默认生产路径不可构造、不可调用、不持有当前事实。任一命中使本计划继续门控；执行角色不得恢复 #214。

## 4. 禁止范围

禁止迁移入口既有自检正文、修改 B0—B4 / #214 / 领域 / 核心 / 线程 / 界面 / 适配、删除旧兼容模块、增加 Release 旧生产通道、快照 / 恢复、O1 接线或双启动。旧正文只能保留为 Debug 自检 / 兼容证据，不再拥有默认生产机器事实。
