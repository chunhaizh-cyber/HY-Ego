## 1. 目标

把当前已授权、已选择方法的任务转换为冻结执行请求，经任务管理 / 工作线程受控执行，并生成强类型非权威结果回执：

```text
任务双向承接授权、当前生命周期与精确方法选择关系 / 版本
-> 完整冻结执行请求
-> 私有有锁有界队列
-> 任务服务排队中 / 执行中迁移
-> 受支持领域执行适配器
-> 状态与动作动态读回
-> 强类型非权威回执
```

本切片不正式记录任务最终结果、已完成状态或需求结算。

## 4. 实施步骤

### S0：实际接口和线程所有权复核

读取 #222、#223、#214 实施记录与实际函数，核对生命周期版本、当前选择、动作规格、强类型回执、阶段 590 和三个线程的启动 / 停止 / 所有权。明确哪一个对象独占现有无锁运行消息队列；若需要任意多生产者直接共享无锁队列，或阶段 600 不可用，先退回设计。

### S1：冻结请求协议

新增 `协议.任务执行请求.ixx` 真模块，导出协议版本 1 和完整值式请求，字段覆盖任务、来源需求、两条承接授权关系完整句柄 / 版本、生命周期关系 / 版本、精确当前选择关系完整句柄 / 版本及所属生命周期版本、方法登记 / 规格、动作入口 / 规格、目标、主体、场景、截止时间和尝试序号。完整句柄逐字段比较；摘要只用于信封和诊断，不能判身份。

### S2：管理线程入队与排队迁移

管理线程持有私有互斥锁和有界完整请求容器。满、停止、重复、过期或无受支持适配器时写前返回，生命周期不变。入队成功后调用任务服务重读双向授权和同一精确当前选择关系 / 版本，再迁移筹办中到排队中；入队成功本身不被写成任务事实。

### S3：工作线程消费与受控执行

工作线程取出完整请求，先经任务服务复核同一选择关系 / 版本仍为当前并迁移排队中到执行中，再调用 `服务.任务执行.ixx`。第一版只接正式登记且当前代码已有领域入口的适配器；不新增任意函数指针表。线程不直接写仓库、状态、动态、任务或需求，也不从任意任务到方法引用重建选择。

### S4：证据读回与强类型回执

方法执行 / 领域服务完成状态和动作动态写入后逐项读回。允许等待或待重筹办走 #222 专用迁移材料；证据完整才形成强类型非权威回执。协议 / 队列拒绝不改权威任务结构，内部同版本矛盾追根因。

### S5：主循环、停止和回归

接通管理 / 工作线程主循环的可停止消费，不按每任务创建线程。新增 `自检.任务执行调度.ixx` 真模块承载 A01-A15、容量 / 顺序 / 重复 / 过期 / 并发测试和逐项输出，入口只以最终回归阶段 600 无捕获回调登记。完成规范、工程登记、实施记录、归档、治理、提交和推送。

## 8. 失败收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 前置接口或线程所有权漂移 | 依赖门控返回 | 不改代码，退回设计 |
| 无适配器、队列满 / 停止、重复、过期 | 逻辑内返回 | 不入队或不迁移生命周期 |
| 入队后请求因并发换代过期 | 逻辑内丢弃 | 不由线程回退 / 修正权威结构 |
| 允许等待 / 待重筹办 | 设计内分支 | 用 #222 专用材料迁移，不生成成功回执 |
| 动作写后状态 / 动态读回不一致 | 追根因解决 | 不生成回执，停止结果依赖路径 |
| 同当前版本请求与任务 / 方法结构矛盾 | 追根因解决 | 不用普通运行消息或日志兜底 |
