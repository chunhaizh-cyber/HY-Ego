## 1. 目标

基于当前无令牌任务、方法和状态动态服务，形成第一条隔离任务执行路径：

```text
筹办中任务
-> 完整冻结请求
-> 两阶段强类型有界队列
-> 排队中 / 执行中生命周期
-> 单一稳定动作键的任务目标状态值变化适配器
-> 实际状态与动作动态同会话提交
-> 强类型非权威回执
```

本切片不记录任务最终结果，不提交已完成生命周期，不结算需求，不把线程或回执升格为机器事实。

## 4. 实施范围

### S0：实际接口和基线复核

1. 读取 #271 正式实施记录、A01-A18 和 780 登记。
2. 逐签名复核任务、方法、状态、动态、状态动态组合器和数据操作。
3. 复核任务管理 / 工作线程现有兼容构造、运行消息路径和强类型回执路径。
4. 复核阶段 600、四个新模块名和允许文件没有占用或其它窗口差异。

### S1：请求协议和任务执行组合器

1. 新建 `领域/组合.任务执行.ixx`。
2. 新建 `线程/协议.任务执行请求.ixx`。
3. 组合器构造时只接收任务业务服务、方法业务服务、状态动态组合器和一个非零稳定动作键。
4. 冻结材料必须覆盖双向承接、筹办中生命周期、选择记录发布、方法投影、方法生命周期、动作角色和场景。
5. 只接受“任务目标状态值变化”适配器；主体、场景和目标由任务权威材料派生。

### S2：已有前状态与新后状态动态原子路径

1. 原地扩展 `组合.状态动态.ixx`，增加“已有前状态 + 新后状态 + 动态”请求和入口。
2. 原地扩展 `数据操作.状态动态.ixx`，在一次执行器 / 会话内重读已有前状态、写新后状态和动态。
3. 关系顺序继续使用目标 / 前 / 后 / 来源动作 `1 / 2 / 3 / 4`。
4. 新路径固定故障自检只在既有 Debug 自检宏下开放；生产入口不接收故障参数。

### S3：两阶段强类型请求队列

1. `任务管理线程` 增加独立有锁有界强类型请求队列。
2. 增加不可复制预留候选能力，支持准备、确认、撤销和快照。
3. 预留候选不可消费；任务完整迁移排队中后才确认。
4. 队列锁内不调用组合器、业务服务、日志、弹窗、文件或用户回调。
5. 旧运行消息队列和旧 `发送任务执行请求` 保持原义兼容。

### S4：调度路由和受控执行

1. 新建 `线程/路由.任务执行调度.ixx`。
2. 提交路径固定为冻结、预留候选、任务迁移排队中、确认候选。
3. 正常漂移撤销候选；内部非预期撤销候选并停止 600。
4. `任务工作线程` 增加强类型受控消费入口，只接收调度路由窄引用，不保存业务服务或事务能力。
5. 消费后迁移执行中，重读动作桥，调用唯一适配器；过期或允许拒绝经任务服务迁移待重筹办。

### S5：新版强类型回执

1. 原地扩展 `任务结果回执协议.ixx`，保留旧材料性质并新增“非权威任务执行结果证据”。
2. 新材料携带完整来源执行请求，且任务、方法、动作、场景和主体必须与句柄组逐字段一致。
3. 工作线程只在实际状态与动态完整读回后发布新回执。
4. 回执准入失败不写任务、需求、状态或动态。

### S6：自检、阶段登记和治理

1. 新建 `线程/自检.任务执行调度.ixx`，承载 A01-A18。
2. 工程按 `ClCompile` 登记四个新模块，生产与自检分离。
3. `入口.cpp` 只增加 Debug import、一次自检登记、阶段 600 和总结果汇总。
4. 通过验证后归档计划，登记实施记录、验收、队列和计划索引。

## 8. 入口拒绝

1. 协议版本、请求编号、幂等键、任务序号、批次、回执编号或任一结构幂等主键为零。
2. 任务句柄无效，时间顺序错误、截止时间错误或时间超过当前 I64 承载范围。
3. 任务不是完整承接的筹办中任务，双向授权或当前选择不完整。
4. 方法未活跃、动作入口不是唯一当前动作、稳定动作键不等于唯一支持键。
5. 动作输入 / 输出场景与任务场景不一致。
6. 队列停止、容量为零 / 已满、请求重复、同键异义、序号倒退或已过期。
7. 排队、执行或收口前权威任务身份与冻结请求不再一致。

入口拒绝不得修改调用方参数、补默认值、降级到运行消息、传统服务或无令牌兼容写路径。

## 9. 失败收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 执行前实际接口与计划漂移 | 依赖门控返回 | C++ 零修改，退回设计 |
| 非法参数或不受支持动作 | 逻辑内返回 | 写前拒绝，结构 / 队列不变 |
| 队列预留失败 | 逻辑内返回 | 任务保持筹办中 |
| 预留后任务正常漂移 | 逻辑内返回 | 精确撤销预留候选 |
| 排队中请求过期 | 逻辑内收口 | 经任务服务迁移待重筹办 |
| 执行适配允许拒绝 / 版本漂移 | 逻辑内收口 | 经任务服务迁移待重筹办，不生成成功回执 |
| 队列候选确认能力失效 | 追根因解决 | 600 不登记，不用运行消息补发 |
| 后状态 / 动态写入或读回非预期 | 追根因解决 | 同会话撤销本次后状态和动态 |
| 同版本授权 / 选择 / 方法 / 动作矛盾 | 追根因解决 | 停止动作路径，不让线程解释 |
| 新回执与来源请求矛盾 | 追根因解决 | 不发布上行回执，不写机器事实 |

若失败后无法用读回和数量快照证明没有可读孤立结果状态、动态或可消费请求，停止并登记待确认问题，阶段 600 不得登记。
