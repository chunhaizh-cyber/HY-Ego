## 8. 实施步骤

### S0 当前代码事实复核

1. 重读允许 C++ 文件头部规则注释。
2. 复核 `特征服务` 当前创建特征类型、读取特征身份、读取特征语义类型、创建实例特征槽位、写入 / 读取宿主 I64 特征状态材料入口。
3. 复核 `状态服务` 抽象状态创建和状态值读取入口。
4. 复核 `需求服务` 目标状态创建、模板关系写入和唯一目标状态读取入口。
5. 记录 TSC-GAP-01 到 TSC-GAP-07 的当前代码事实。

### S1 特征类型身份与语义类型第一轮复核

1. 确认 `创建特征类型` 返回的节点类型为 `节点类型::特征`。
2. 确认 `读取特征身份` 对无效句柄和非特征节点返回空。
3. 确认 `读取特征语义类型` 第一轮等同身份复核，不引入完整语义类型系统。
4. 在默认入口或断点记录中覆盖 TSC-A1、TSC-A2。

### S2 实例特征槽位读回和重复拒绝

1. 确认合法宿主和合法特征类型可以创建槽位。
2. 读回宿主到槽位的 `归属` 关系。
3. 读回槽位到特征类型的 `模板` 关系。
4. 重复创建同一宿主 / 同一特征类型槽位时必须拒绝。
5. 通过数量快照或读回隔离证明拒绝后既有槽位不变。

### S3 宿主 I64 特征状态材料读写

1. 有槽位时写入宿主 I64 特征状态材料。
2. 写入后必须经 `特征服务::读取宿主I64特征状态材料` 读回同值。
3. 无槽位、无效宿主、无效特征类型或非归属特征值必须拒绝。
4. 特征值创建或关系写入失败时，不返回成功结果，半结构不得作为有效业务事实。
5. 本轮不修改 `特征值服务.h`，不实现 Vec / 哈希 / 值域。

### S4 需求目标状态模板关系复核

1. 确认需求目标状态由状态服务创建抽象状态。
2. 确认需求节点通过 `关系类型::模板` 指向唯一目标状态节点。
3. `读取需求目标状态` 必须在目标缺失、多目标或目标类型非状态时返回空。
4. 明确 I64 只是状态值材料，不是需求目标对外语义。

### S5 高级服务封口扫描

1. 扫描需求服务、任务服务、方法服务是否直接包含、接收或调用特征值服务。
2. 扫描新增代码是否绕过特征服务访问特征值节点或主信息值容器。
3. 有越权依赖命中时停止实施并登记待确认问题。

### S6 候选能力排除扫描

1. 扫描本切片是否新增 Vec、哈希、值域、稳态或序列化恢复代码。
2. 扫描是否新增控制面板、SQL、ADO、WebView、D455、体素、外设、`std::thread`、`std::async` 或工作队列。
3. 扫描是否修改禁止文件。
4. 任一命中必须停止并登记断点。

### S7 验证与收口

1. 运行 `git diff --check`。
2. 运行 `python .\tools\check_specs.py`。
3. 运行 Debug x64 构建。
4. 运行默认入口并确认 TSC-A1 到 TSC-A11 对应输出或断点记录。
5. 生成 `实施记录/20260708_BASE-D2_特征与状态材料代码实施_Codex断点清单.md`。
6. 归档本计划并同步项目记忆。
