## 1. v0.3 裁决

P2A、P2B、P2C 的业务语义、对象边界和最终集成顺序不变。v0.3 同时修正两项设计缺口：

1. `NT-C1/v0.2` 曾把计划预期 DTO 和签名写成 #339 提供事实；#339 进入 main 后证明真实导出面并不等同，且旧合同错误地把关系角色和顺序号拆成两个字段。
2. 叶子 v0.2 把 Markdown 实施记录列入执行白名单；该权限已经由项目级角色规则退出。

因此，`NT-C1/v0.3` 直接冻结 #339 已发布的真实模块、类型、签名和生命周期；领域叶只能在自身 DTO 中组合这些值式记录，不得补造一套“核心兼容 DTO”。`NT-C1Q/v0.1` 与 `NT-C2/v0.4` 采用提供者—消费者合同并行，不等待相邻实现完成。

```text
共同计划合同基线
-> #339、#340、#341、#342 各自实现所拥有接口和消费端
-> 四个候选按 #339 -> #340 -> #341 -> #342 顺序汇合
-> #352 唯一完成工程登记、统一运行器接线、双配置构建、完整联调和总验收
```

“冻结接口合同”只表示设计已确定提供者、消费者、模块、DTO、调用语义和失败分类，不表示当前 main、任务分支或任何 C++ 已经实现该接口。候选任务不得把预期符号写成当前代码事实。

`NT-C2/v0.4` 的共同结果类型由 #340 唯一提供，物理文件固定为 `海中鱼巣/领域/节点直接领域合同.数据.h`。#341、#342 和 P3/P4 消费者只能只读包含该文件，不得在各自 DTO 头中复制同名枚举、模板或等价平行结构。该文件只定义值式合同状态、原因和结果形状，不保存仓库、记录容器、关系端点、令牌、锁或其它机器事实。

共同结果统一含字段 `状态: 合同结果状态`、`原因: 合同结果原因` 和与成功状态匹配的值式材料。枚举固定为：

```text
合同结果状态 = 成功 | 幂等读回 | 合法未找到 | 合法空组 | 入口拒绝 | 许可竞争 | 写前漂移 | 读取版本漂移 | 内部错误
合同结果原因 = 无 | 无效身份 | 错命名域 | 错类型 | 错版本 | 未知规则版本 | 关系角色不合法 | 基数不满足 | 同键异义 | 值归属不一致 | 时间不合法 | 自我结构不合法 | 并发许可失败 | 读取期间版本变化 | 权威读回不一致 | 事务阶段不一致
```

全部具名结果都是不同的强类型结果，不得用裸 `bool` 或普通整数互换。其字段形状固定为：

| 结果形状 | 字段与类型 |
| --- | --- |
| `合同无值结果` | `状态: 合同结果状态`；`原因: 合同结果原因` |
| `合同单项结果<T>` | `状态: 合同结果状态`；`原因: 合同结果原因`；`材料: std::optional<T>` |
| `合同组结果<T>` | `状态: 合同结果状态`；`原因: 合同结果原因`；`项目组: std::vector<T>`；`读取结构版本: std::uint64_t` |
| `合同具名组结果<T组>` | `状态: 合同结果状态`；`原因: 合同结果原因`；`材料: std::optional<T组>`；成功或合法空组均携带可识别宿主 / 主体的组材料 |
| `带值读取结果<T>` | `合同单项结果<T>` 的具名读取形状；合法未找到时材料为空 |
| `带值结构写入结果<T>` | `状态: 合同结果状态`；`原因: 合同结果原因`；`材料: std::optional<T>`；`结构版本: std::uint64_t`；仅成功 / 幂等读回有材料 |

`特征值归属读取结果`、各完整事实读取结果、各退役规格结果和各自我单项读取结果使用 `合同单项结果<对应值式材料>`；宿主槽位、主体状态、主体动态和内部世界成员四个具名组读取结果使用 `合同具名组结果<对应组值式材料>`；各写入结果使用 `带值结构写入结果<对应已发布值式材料>`。实现可以采用具名结构而非模板，但字段、类型、可带材料的状态和生命周期必须等价。

只有 `成功 / 幂等读回` 可以携带单项材料，只有 `成功 / 合法空组` 可以携带具名组材料；其它状态材料必须为空。逻辑内结果固定为 `幂等读回 / 合法未找到 / 合法空组 / 入口拒绝 / 许可竞争 / 写前漂移 / 读取版本漂移`，均不得发布半结构。前置通过后的候选、关系、记录、索引、参与者、撤销、发布或权威读回异常一律为 `内部错误`，不得改写成逻辑内拒绝。

所有跨计划请求由调用方拥有，以 `const&` 传入并在第一写前完整复核；提供者不得持有请求引用。所有领域结果、DTO、字符串和组均由结果对象值式拥有，可在仓库锁、事务许可和会话释放后读取；不得返回指针、引用、span、仓库、索引、记录容器、锁、参与者或可写对象。#339 的会话和参与者只在 `NT-C1/v0.3` 数据操作内部同步调用期有效，不得进入 `NT-C2/v0.4` 公开服务。
