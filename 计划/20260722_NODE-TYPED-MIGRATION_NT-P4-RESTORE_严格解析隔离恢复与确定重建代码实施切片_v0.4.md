## 3. `NT-C4/v0.3` 消费与提供合同

### 3.1 只读消费 #349

唯一从 `海中鱼巣/核心/统一权威快照.数据.h` 只读消费 `NT-C4/v0.3` 的 `统一权威快照头`、`统一权威快照段头`、九类强类型段、`规范化快照字节材料`、`规范化权威载荷摘要`、`读取当前必需段声明组`、数值 ABI 和摘要规则。不得在 `恢复材料.数据.h` 或任何 #350 文件复制这些定义、另立第二段清单、平行 DTO、同名替身、通用字节业务记录或兼容解释器；本计划独占实现解析、隔离装载、投影重建和候选验证。

解析器必须逐字段接受下列冻结合同：8 字节魔数 `48 5A 59 43 53 4E 50 31`；容器格式版本 uint16=1；小端标识 uint16=1；容器头 104 字节；段头 64 字节；段清单版本 uint32=1；摘要算法版本 uint32=1 且固定 SHA-256；冻结规则版本 uint32=1；节点稳定主键命名域 ABI uint32=1；九段类型 1—9 各恰一段、按升序出现，合法空段只允许记录数量为 0。

容器头偏移固定为：0 魔数、8 格式版本、10 字节序、12 容器头字节数、16 段头字节数、20 段清单版本、24 摘要算法版本、28 段数量、32 文件总字节数、40 来源运行期域身份、48 来源节点直接仓库身份、56 快照代次、64 冻结规则版本、68 稳定主键命名域 ABI、72—103 容器摘要。段头偏移固定为：0 段类型、2 段格式版本、4 段头字节数、8 段顺序、12 段标志、16 记录数量、24 段载荷字节数、32—63 段载荷摘要。

分配前上限固定为：文件总长 `8,589,934,592` 字节；单段载荷 `2,147,483,648` 字节；每段记录 `4,194,304` 项；单记录 `16,777,216` 字节；UTF-8 字符串 `1,048,576` 字节；单向量 `1,048,576` 项。所有长度、数量、偏移和加法先做溢出检查，再分配或切片。段摘要覆盖段载荷；容器 SHA-256 覆盖全文件，计算时偏移 72—103 视为 32 个零字节。

### 3.2 本计划值式类型

```text
恢复材料包

严格解析结果
隔离运行期建立规格
节点直接运行期上下文候选
候选来源类型（全新 / 恢复）
候选验证报告
已验证运行期上下文候选
候选形成结果
投影重建结果
候选验证结果
```

`恢复材料包` 拥有严格解析后的九段强类型不可变材料，不拥有当前运行期引用。`节点直接运行期上下文候选` 独占隔离仓库、领域记录与待重建投影，只能单次移动；失败时整体销毁。`已验证运行期上下文候选` 是单次移动能力，只能由两个完整候选验证器形成；它不暴露可变仓库、索引、服务内部地址或发布能力，未发布即销毁。

强类型解析和隔离装载顺序固定为：

| --- | --- | --- |
| 节点直接记录段 | #349 的逐项节点 DTO，含节点编号、稳定主键、完整句柄、类型、版本、状态、创建序号 | 全新节点直接身份仓与编号/主键高水位 |

| 状态域记录段 | #349 的状态事实 DTO | 全新状态事实记录仓 |
| 动态域记录段 | #349 的动态事实 DTO | 全新动态事实记录仓 |
| 任务方法选择记录段 | #349 的任务方法选择 DTO | 全新选择与方法生命周期记录仓 |

### 3.3 完整签名与结果分类

```text
严格解析结果 解析统一权威快照(const 规范化快照字节材料&)
候选形成结果 形成隔离恢复候选(恢复材料包&&, const 隔离运行期建立规格&)
投影重建结果 重建节点直接派生投影(节点直接运行期上下文候选&)
候选验证结果 验证完整全新候选(节点直接运行期上下文候选&&)
候选验证结果 验证完整恢复候选(节点直接运行期上下文候选&&, const 规范化权威载荷摘要&)
```

`解析统一权威快照` 只读消费规范化字节并形成独占恢复材料包；`形成隔离恢复候选` 消费材料包，不触碰当前宿主；`重建节点直接派生投影` 只在候选内部建立可重建索引、自我四类投影和概念投影；两个验证函数消费候选，成功后返回单次移动的已验证候选，失败后不得把原候选交还调用方。

解析、候选形成、投影重建和候选验证结果固定区分 `已形成/材料拒绝/资源失败/内部不一致`。未知、损坏、超限、重复、缺段、摘要错、悬空引用或错 ABI 属于材料拒绝；合同已接受后的装载、互证、重建或再冻结不闭合属于内部不一致并按内部逻辑错误收口。消费者为 #351、#352；签名或 ABI 漂移时退回 #350，不由消费者修补。
