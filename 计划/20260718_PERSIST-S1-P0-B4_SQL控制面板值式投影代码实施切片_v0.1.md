## 1. 目标与依据

依据恢复流程图 v0.3、消费者并行流程图 v0.5、详细设计第 17—19 节和 #295 事实矩阵。本切片只让 SQL 审计与只读控制面板消费同身份租约导出的值式投影；数据库、界面、HTML、日志和显示继续只做人读 / 审计材料，不持可写仓库引用、不裁决机器事实、不参与恢复。

来源性质为独立只读重建 + MIG-18 新增约束，不计旧控制面板、旧数据库或旧持久化能力迁移。

## 2. 依赖与假定接口

#298 已提供运行期公共只读查询门面，#297 已提供有效租约。当前 `控制面板服务.h`、窗口和 SQL 适配仍从旧对象 / 结构统计来源取得材料。目标新增一个值式投影模块，把租约查询结果转换为既有控制面板 / SQL 只读 DTO；不得把 DTO 反向写回领域结构。本切片不依赖 #299 的桥实现。

## 4. 禁止范围

禁止入口、统一运行器、B0—B3、仓库 / 领域写服务、线程写路由、旧概念图服务、快照 / 恢复、O1 和 #214。工程与 filters 是本两路并行批次唯一共享代码文件所有权，只允许登记本切片两个新模块；新自检必须由既有 830 自检模块组合调用，不得另增中央登记。禁止 SQL 回写机器事实、界面触发领域提交、从数据库恢复上下文或把显示成功当验证通过。
