1. 完整原因、结果 `动态模式结构键` 的每个字段能否无损落入当前 `概念签名材料`。
2. 原因和结果的有序角色如何编码，当前是否已有正式维度和角色定义。
3. 完整目标、动作句柄如何与当前只含节点类型的概念约束进行身份相等比较。
4. 概念签名版本、活动图版本和概念完整句柄如何只能来自当前概念图读取，禁止调用方提供。

6. 生命周期转换是否递增活动图版本；若不递增，单看版本为何不足。
7. 是否必须新增原子只读活动概念视图，或扩展因果概念签名结构。
8. 未来唯一匹配如何只形成附加标注，完整原因 / 结果键始终作为统计和学习主键。
9. 零匹配、多匹配如何返回材料不足，不选首项、不创建概念、不写关系。
10. 当前 API 不能无损映射时，需要修改哪些上游文件、由哪个后继计划承接。
