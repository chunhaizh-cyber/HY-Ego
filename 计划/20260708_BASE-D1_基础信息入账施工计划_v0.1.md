## 7. BASE-GAP 覆盖

| 差距 | 施工覆盖 | 失败收口 |
| --- | --- | --- |
| BASE-GAP-01 部分单步写入入口未显式读回验证 | S1 为基础信息、存在、场景、抽象状态、抽象动态、因果引用补最小读回 | 读回失败视为逻辑错误，停止验收并登记断点 |
| BASE-GAP-02 多步写入失败收口不完整 | S2 对状态、动态、二次特征建立写前数量快照、写后读回和失败清理 / 读取隔离证明 | 无法证明无可读半结构时停止并登记核心事务边界待确认问题 |
| BASE-GAP-03 实例动态前后值材料当前由动态服务私有辅助创建 | S3 将前值 / 后值状态材料创建边界收束为经状态服务创建 | 若状态服务不能承接所需材料，停止并登记状态 / 动态边界问题 |
| BASE-GAP-04 世界服务兼容壳仍可直接创建基础信息通用节点 | S4 保留兼容壳但补用途读回和禁止替代原子服务的入口拒绝 | 发现兼容壳替代已明确原子服务时拒绝，不写结构 |
| BASE-GAP-05 轻量因果引用和基础身份入口需要分开验收 | S5 拆 BASE-A10 与 BASE-A11，基础身份不需要来源动态，轻量入口只用来源动态准入 | 来源动态无效或材料不完整时拒绝，不创建因果引用 |
| BASE-GAP-06 结构化拒绝原因未统一 | S6 第一轮保留轻量返回，但入口样例和断点清单列明拒绝原因来源 | 若需要机器可读拒绝原因，转后续结构化错误结果专项 |

## 8. 状态 / 动态服务边界

状态服务负责：

```text
创建实例状态。
创建抽象状态。
创建动态前值状态材料。
创建动态后值状态材料。
提供状态材料读回。
拒绝缺场景、缺主体、缺时间戳或无效状态值材料。
```

动态服务负责：

```text
验证实例动态入口材料。
调用状态服务取得前值和后值状态材料。
创建动态主信息和动态节点。
创建动态到主体、场景、目标、前值、后值和可选来源动作的关系。
读回动态材料。
```

动态服务不得绕过状态服务直接创建前后值状态节点或写状态值。状态服务不得创建动态节点、不得表达稳定因果结论。

## 10. 实施步骤

### S0 当前代码事实复核

1. 重读允许 C++ 文件头部规则注释。
2. 复核各服务创建入口、读取入口和失败返回方式。
3. 复核 `写入事务` 当前是否已有可用边界，但不得修改核心文件。
4. 记录 BASE-GAP-01 到 BASE-GAP-06 的当前代码事实。

### S1 单步写入读回

为以下入口补最小读回验证：

```text
世界服务::创建基础信息
存在服务::创建实际存在
场景服务::创建场景
状态服务::创建抽象状态
动态服务::创建抽象动态
二次特征服务::创建二次特征
因果服务::记录因果引用()
```

读回不符合预期时，不返回有效句柄，并按逻辑错误登记断点。

### S2 多步写入失败收口

为以下路径建立写前快照、写后读回和失败收口：

```text
场景服务::接纳存在
状态服务::创建实例状态
动态服务::记录实例动态
二次特征服务::创建组合二次特征
因果服务::记录因果引用(来源动态, 动态服务)
```

失败收口优先使用现有删除 / 失效入口；无法证明无半结构时停止并登记核心事务边界待确认问题。

### S3 状态 / 动态前后值边界

将实例动态前后值状态材料创建收束到状态服务：

```text
动态服务只验证动态入口和编排动态关系。
前值状态材料由状态服务创建。
后值状态材料由状态服务创建。
前后值材料必须可读回。
```

### S4 世界服务兼容壳收窄

`世界服务::创建基础信息` 保留为基础信息通用节点兼容壳，但不得替代存在、场景、状态、动态、二次特征和因果入口。

验收必须证明：

```text
创建基础信息可读回基础信息节点。
已明确类型的基础信息入账走对应原子服务。
兼容壳不新增旧主信息字段。
```

### S5 因果引用双入口拆分

分开实现和验收：

```text
因果引用基础身份：无来源动态，仅创建因果引用节点和主信息。
轻量因果引用：来源动态只作准入证据，不持久化来源动态关系。
```

不得把单个轻量因果引用解释为稳定因果结论。

### S6 拒绝原因追溯

第一轮不强制改 C++ 返回结构。执行窗口必须在入口样例和断点清单中列明以下拒绝来源：

```text
无效句柄。
错误节点类型。
缺场景。
缺存在主体。
缺时间戳。
缺前值或后值。
来源动态无效。
组成项为空或类型不允许。
写入失败。
读回失败。
```

### S7 入口验收样例

在 `海中鱼巣/入口.cpp` 增加 BASE-A1 到 BASE-A12 的最小验收输出。输出只做人读展示，验收布尔值必须来自结构读回、数量快照或有效句柄判断。
