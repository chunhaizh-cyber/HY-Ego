## 1. 当前事实与假定契约

当前四仓库已有版本句柄、读写锁和部分令牌 / 候选 / 删除入口，但结果和候选合同不统一。假定 #262 证明可在不删除现有 ABI 的情况下增加统一结构结果和自检。接口、枚举或删除所有权漂移时不改代码，退回设计。

## 2. 核心处理

统一四仓库对入口拒绝、许可拒绝、版本漂移、内部不一致、确认、撤销和普通删除的结构化结果；固定主信息->节点->关系/索引创建依赖、端点有效性、编号不复用、版本推进和候选幂等。现有兼容 bool / optional 入口保持，内部逐步适配新结果。

不新增业务节点类型，不把需求 / 任务 / 方法准入下沉仓库，不实现关系二级索引或性能计数。
