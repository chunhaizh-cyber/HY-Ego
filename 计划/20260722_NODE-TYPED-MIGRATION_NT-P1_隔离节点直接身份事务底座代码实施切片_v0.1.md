## 2. 目标与完成边界

本切片只建立一个不接默认生产装配的隔离新结构域：

```text
节点直接身份仓库
+ 正式关系仓库
+ 可重建索引仓库
+ 不透明节点直接身份结构写入会话
+ 多强类型参与者结构写入执行器
```

共享 ABI 只追加 `节点稳定主键`、`节点稳定主键命名域`、节点类型 16—19 和关系类型 19—23；旧句柄、旧仓库、旧会话、旧执行器、旧生产装配和旧冻结结构保持原样。新旧域零互查、零关系、零索引、零投影、零幂等账互认、零恢复候选互认、零双写。后继 P2 / P3 只能在该新域建立已迁对象；P4 才拥有默认装配切换、完整冻结恢复、旧域物理退役和全仓零命中权。

## 7. 固定结构与接口合同

### 7.1 节点稳定主键和节点记录

`节点稳定主键` 固定为两个 `std::uint64_t` 分量：`命名域` 和 `键值`。两者均非零才有效，相等必须比较两个分量。命名域必须使用 4010 ABI 版本 1 的强类型固定值；节点仓库复核完整性、域与节点类型匹配、复合键唯一、每域高水位、撤销烧号和历史不复用，领域服务继续裁决完整业务身份与角色关系。禁止运行期注册、哈希推导或借用索引物理命名域。

新域节点记录字段精确为：

```text
节点编号
稳定主键
节点类型
版本号
记录状态
创建序号
```

不得包含主信息句柄、主信息编号、拓扑锚点、无类型值、索引物理键、业务状态或显示材料。新节点仓库内部可以维护由稳定主键到节点编号的可重建定位表，但该表不是第二份身份事实。

### 7.2 节点创建、幂等和读取

新域至少形成下列值式能力；最终签名在 S0 依据实际 C++ 类型可做不改变语义的机械定型：

```cpp
稳定主键当前身份读取结果 读取稳定主键当前身份(节点稳定主键 稳定主键);

带值结构写入结果<节点未发布候选> 结构化创建节点未发布候选(
    节点类型 类型,
    节点稳定主键 稳定主键,
    const 结构事务令牌& 令牌);

std::optional<节点稳定主键> 读取节点稳定主键(节点句柄 节点) const;
```

同键同类型且完整创建语义相同为幂等读回，零写返回原当前身份；同键异类型、异语义或历史身份复用为具名身份冲突，零写。无占用且前置完整才允许单调分配节点编号和创建序号。编号和创建序号出现撤销空洞后不得回收。

### 7.3 关系和索引

关系端点仍为完整节点句柄，稳定主键不得代替关系端点。正式关系仓必须完整支持关系 ABI 0—23；其中 19—23 只是本轮追加的结构 ABI。具体角色、端点、基数、顺序和值域仍由各主题规范唯一裁决，P1 仓库只执行已注入的具名角色准入，不发明业务规则；未知关系类型必须在写前具名拒绝且零写。

术语必须严格分离：

```text
节点稳定主键：节点直接字段，长期身份，历史不复用。
索引物理键：具名所有者命名域内的可重建定位键。
```

索引命中后必须回读新域权威节点 / 关系；清空或重建索引不得改变节点、关系或领域记录。新路径不得新增 `绑定主键`、`按主键查节点`、`主键绑定记录` 等歧义调用名。

```cpp

std::optional<节点稳定主键> 读取候选节点稳定主键(节点句柄 节点) const;
```

不得暴露主信息、候选 I64、仓库、令牌、锁、可写引用或其它领域值。参与者接口只拥有 `准备提交`、`确认待发布`、`完成撤销` 和 `完成发布` 生命周期；具体记录类型、字段、版本、格式和存储归 P2 / P3 各领域模块。P1 自检使用两个编译期不同的本地强类型夹具证明多参与者收口，不得因此建立生产通用记录类型。

### 7.5 固定锁序、确认顺序和逆序撤销

新执行器先取得隔离事务域唯一独占许可。确需取得内部参与者锁时，唯一获取顺序冻结为：

```text
事务域独占许可
-> 节点候选
-> 新关系 / 关系变更
-> 索引候选
-> 领域强类型参与者按登记顺序
```

不得按运行时类型、地址、线程或无序容器决定顺序；不得共享锁升级、反向取得、持锁 I/O、日志、弹窗、外部回调或等待。普通领域服务不得取得这些锁。

提交顺序固定为：

```text
写前准入
-> 取得独占许可并构造会话
-> 同步回调形成节点 / 关系 / 索引候选及参与者本域候选
-> 同一令牌完整读回全部结构写集
-> 会话请求提交
-> 参与者按登记顺序准备
-> 结构会话确认
-> 参与者按登记顺序确认
-> 进入无失败最后发布区
-> 结构会话完成发布 noexcept
-> 参与者按登记顺序完成发布 noexcept
-> 返回值式结果
-> 释放独占许可，形成普通读取唯一可见点
```

任一准备或确认点失败时，固定逆序为：

```text
领域参与者按登记逆序撤销
-> 索引写集逆序撤销
-> 已发布关系变更逆序恢复
-> 新关系逆序删除
-> 节点候选逆序撤销
-> 读回证明前态
```

撤销过程中遇到失败仍继续尝试其余可撤销项；最终不能证明前态时，必须在释放许可前隔离事务域，后继请求不得取得许可。首次内部失败不得被后续撤销结果覆盖。

### 7.6 无失败最后发布区

进入最后发布区前必须完成所有分配、验证、读回、冲突判定和可能失败的确认。最后发布区禁止新分配、新校验、仓库查询、外部回调、日志、显示、I/O 或等待；结构会话和参与者 `完成发布()` 必须为 `noexcept`，只关闭候选 / 撤销能力和提交已确认材料。确认本身不形成普通可见；唯一可见点是最外层独占许可释放。

### 7.7 冻结边界

冻结节点记录与运行期节点记录字段等价并直接含稳定主键；冻结关系保存有效、失效和历史拓扑；索引冻结材料明确为可丢弃加速材料。P1 不建立包含任意领域载荷的通用顶层容器，不形成跨域完整快照，不解析旧快照，不发布恢复候选；这些职责只属于 P4。

## 12. 正式退回条件

```text

7 模块 import 图形成循环，或需要新增未设计共享桥 / 第二实体所有者

新节点必须同时保存稳定主键和主信息句柄
新路径必须调用主信息、通用 I64、旧索引或旧恢复材料
领域参与者必须使用跨领域通用容器、任意槽位、JSON、字节包或可变通用载荷
新旧域必须互查、互建关系 / 索引、共享幂等账 / 恢复候选或双写同一稳定身份
```
