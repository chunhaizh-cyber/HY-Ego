## 1. 目标

在真实任务授权和方法动作接口已经形成的前提下，把结构化候选语素入口绑定为概念名称，并生成值式回执：

```text
命名需求
+ 来源任务与合法当前态
+ 已选方法和动作入口
+ 候选语素入口
-> 语素服务正式绑定
-> 概念侧与语素侧双向读回
-> 方法动作动态 0 -> 1
-> 结构化绑定回执
```

本切片不生成名称字符串，不负责通用方法召回或任务选择，不记录任务结果或需求结算。


## 7. 入口拒绝

```text
候选任一必填句柄或时间戳无效。
命名需求不完整、目标概念不一致或已经正式满足。
来源任务的双向承接授权任一关系 / 版本不当前、当前态不允许动作或需求端点不一致。
精确当前选择关系 / 版本不当前，或选择方法、候选方法和动作入口归属不一致。
方法未登记、已失效、规格 / 召回来源过期或动作入口不可调用。
候选语素入口无效或不是正式语素入口。
活动版本漂移且当前概念身份不可重读。
当前概念已有名称，但候选没有明确同一任务的继续结算依据。
```

所有拒绝发生在新名称关系和动作动态写入之前。

## 8. 失败收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 前置接口未形成 | 逻辑内门控返回 | 不改代码，退回设计修订前置编号 |
| 候选无效、任务未授权、方法未选择、入口失效或材料过期 | 逻辑内返回 | 不写名称关系或动态 |
| 同一语素到同一概念已绑定 | 幂等复用 | 读回既有关系，不新增关系版本 |
| 名称关系创建后正反读回不一致 | 追根因解决 | 不记录动作动态，不生成回执 |
| 名称关系完整但动作动态创建 / 读回不及预期 | 追根因解决 | 保留有效名称事实，需求待结算，停止回执依赖 |
| 当前同版本任务 / 方法 / 动态结构互相矛盾 | 追根因解决 | 停止绑定或回执，不由日志修复 |

## 9. 执行前接口复核与漂移退回

以下任一情况视为漂移：

```text
#223 尚未完成，或没有形成双向授权关系 / 版本、当前生命周期、精确当前选择关系 / 版本、选择审计与动作桥正式接口。
方法登记、召回、复判或排序产物没有可验证的来源版本。
任务选择、方法动作或动作动态接口与假定不同。
语素概念追溯已改变方向、唯一性或返回形状。
候选绑定需要修改任务、方法、动态、需求、核心仓库或自我线程。
#223 不是最终回归阶段 580、590 已占用，或阶段总成 / 自检结果接口与假定不一致。
```

出现漂移不得修改代码，退回设计窗口重写 #214；#215 继续保持门控。
