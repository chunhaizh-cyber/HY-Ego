## 1. 目标与依据

本计划的业务目标保持不变：消费生产运行期已经发布的安全根需求和服务根需求，在同一租约的 B0 正式业务面上形成“未满足根需求 -> 承接任务 -> 筹办中 -> 正式方法基线 -> 唯一选择 -> 纯值执行冻结请求”。

R5 已证明两配置单轮和连续前十轮可以通过，但第十一轮 A11 间歇失败。当前 A11 把会话启动、两份报告完整性、任务 / 方法 / 选择身份和冻结请求相等合并成一个布尔值，现有日志不能唯一证明失败发生在哪一层。v0.5 因此只采集分层证据，不猜测生产修复、不降低 20 / 20 门禁。

同层候选必须在诊断前并列保留：

1. 并发会话启动或租约未成立。
2. 其中一份报告在任务 / 生命周期阶段提前返回。
3. 其中一份报告在方法基线阶段提前返回。
4. 其中一份报告在召回 / 选择阶段提前返回。
5. 其中一份报告在冻结阶段提前返回。
6. 两份报告都完整，但任务、方法、选择或冻结值存在差异。

当前证据不能在上述候选中唯一选中一项。任何生产代码修改都必须等待 R6 首个失败样本给出唯一失败层。
