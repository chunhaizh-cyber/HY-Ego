## 2. 已验证当前事实与假定接口

已验证：生产运行期创建两根，租约公开系统角色、B0 和只读查询；B0 已有需求读回、任务创建/迁移、方法创建/读回、召回选择和执行冻结。`迁移任务生命周期请求` 的正式字段只有任务、目标阶段、幂等材料编号和发生时间戳；任务服务在入口内读取当前任务，把当时的当前生命周期关系和版本封入不透明写入规格，数据操作在写前再次复核该关系和版本。

E256 当前事实：WT-315-R2 在 `0981e8a` 上 Debug x64 Rebuild 0 warning / 0 error；Debug 单轮退出 1，专项 A01—A08 通过、A09 重复同义结构守恒 / A10 同键异义写前拒绝 / A11 并发同义单身份 / A12 聚合守恒失败。执行窗口按首异常停止，未启动 Release 和 20/20；正式回执提交为 `b91d26c9e1a810bc266a027fb404c3b7e8ccd534`。

根因已验证：`需求任务方法数据操作::任务承接匹配` 使用当前生命周期状态复核创建幂等身份，并要求当前阶段仍为“已创建”。首次路由已合法推进到“筹办中”后，完全相同的创建请求会被误判为幂等冲突；并发输家也可能在首次创建读回前观察到已推进阶段并命中同一误判。A10 的入口拒绝本身没有证据表明失败，其后“有效配置再次完整读回”因同一根因失败。

E260 当前事实：R4 按固定顺序带入 `0981e8a -> b91d26c` 并形成 `06fda225742f240e8fbaf0a069bea8aaeb0c8399`。创建幂等领域回归 `SERVICE-DATA-S5-A39` 通过，Debug x64 Rebuild 0 warning / 0 error，Debug 单轮 A01—A10 通过；A11 并发同义单身份和 A12 聚合失败，Release、20/20、strict 和后续静态门禁未启动。R4 clean、远端 0/0、共享目录释放、进程 0，当前不可集成。

R4 下一层根因已验证：两线程同时运行完整筹办路线。线程甲提交“已承接”后，线程乙可以继续提交“筹办中”；线程甲的“已承接”写后读回随即看到当前已为“筹办中”。现有任务服务只识别“当前阶段正好等于请求目标”的同阶段幂等，数据操作写后又要求当前阶段和版本仍精确等于本次目标，因此本次已经合法进入完整历史的“已承接”被映射为内部不一致，路由提前返回。该失败不是创建幂等回归，也不是允许重试掩盖的偶发现象。

假定：现有请求 DTO 足以只用系统角色材料和新值式配置形成固定安全根第一轮方法基线。根需求筹办路由不得接收、保存或比较“调用方预期旧生命周期版本”；生命周期当前性只由任务服务和数据操作提交边界裁决。任务服务现有 `读取生命周期历史` 与状态服务读回足以证明同一幂等材料的目标、时间、值、主体、场景和关系证据；数据操作现有完整历史读回足以在提交后判定本次目标是否唯一同义。执行前仍须逐字段复核 `创建已有需求承接任务`、`迁移任务生命周期`、`读取生命周期历史`、三类方法创建、`召回并提交唯一选择` 和 `冻结任务执行请求`。其它字段、所有权反向依赖或稳定键冲突不一致时保持 clean 退回，不现场发明 ABI。

前置计划：无未完成代码前置；#297、#298、#310、#314 等底座均已独立集成并由 JY-498 只读复核。预期正式产物：生产治理闭环值式配置、根需求筹办路由、阶段 1100 真自检报告和专属实施记录。

接口漂移处理：v0.1 已因错误假定 `迁移任务生命周期请求` 携带旧版本而终止。v0.4 继续保持任务、目标阶段、幂等材料编号和发生时间戳四字段请求；不新增调用方旧版本或 B0 ABI。S0 若发现完整历史无法按同一幂等材料精确证明目标阶段、时间、值、主体、场景和关系证据，或必须暴露原始令牌、公开新主键入口、让路由自行比较生命周期旧版本，保持代码与 index clean，写本计划专属复核断点并退回设计。不得把“当前已继续前进”本身解释为接口漂移。

### S2 生命周期完整历史幂等根因修正

保留 R4 的任务创建幂等修正和 `SERVICE-DATA-S5-A39`。`任务服务::迁移生命周期` 在形成新状态规格前读取完整生命周期历史：同一幂等材料只有一个历史项，且目标阶段、状态幂等主键、发生时间戳、状态值、场景、主体和关系证据完全同义时，直接返回当前任务幂等读回，即使当前阶段已经继续前进；同一幂等材料异义时返回幂等冲突；历史项重复、版本不唯一或关系证据不完整时返回内部不一致。

`数据操作::迁移任务生命周期_实现` 保留写前当前关系 / 版本与合法迁移门禁。目标历史不存在且写前已漂移时零写入返回；提交后不再要求当前仍精确等于本次目标，而是从完整历史核对本次目标证据唯一同义后返回当前任务。若结构结果已提交但目标历史缺失、重复或异义，返回内部不一致并追根因。不得新增请求字段、公开方法、服务依赖或原始事务穿透。

领域自检增加确定性回归：任务以固定“已承接”请求推进后继续到“筹办中”，再次提交原“已承接”请求必须按历史幂等读回当前筹办中任务；同一幂等材料但目标 / 时间 / 值异义仍冲突且结构数量不变；完整历史版本序列唯一且只有一个当前关系。

### S3 值式配置与路由复核

配置只承载版本、幂等主键/材料、规则版本、时间和生产动作键；路由只从租约读系统角色和 B0。先复核根未满足，再创建/读回任务；按正式四字段请求依次迁移到已承接、筹办中，由任务服务内部封存当时关系/版本并由数据操作写前复核。路由不建立第二套生命周期当前性判断。随后登记生产方法基线并提交唯一选择，最后冻结执行请求。

### S4 自检

验证安全根、服务根均为系统角色原句柄；第一轮只消费安全根。覆盖首次、重复、同键异义、根已满足、生命周期合法顺序、非法跨阶段、同阶段同义/异义、目标已在历史且当前继续前进、两线程跨相邻阶段同义竞争、无候选、非唯一和冻结完整性。A11 必须证明两份报告均完整且共享同一任务 / 方法 / 选择 / 冻结请求，不得只证明结构单一。既有领域自检继续证明不透明写入规格的旧关系/版本写前拒绝；#315 不伪造调用方旧版本入口。

### S5 构建与验证

Debug/Release Rebuild，Debug 专项和完整自检，Release 专项零输出，Debug 严格 20/20。共享事件段目录须另行授时。

### S6 任务分支收口
