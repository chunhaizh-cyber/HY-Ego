## 2. 已验证事实与固定合同

已验证事实：

```text
运行期业务操作组合器当前有 13 类线程可路由业务入口。
创建需求并承接任务会同时新建需求，不能承接已经存在的固定根需求。
现有顶层门面不公开任务生命周期迁移、方法首 / 结果 / 动作入口登记或轻量因果创建。
自我治理领域路由仍直接持有十个传统服务 / 仓库引用并执行上述写链。
运行期业务装配已经唯一拥有需求、任务、方法、轻量因果服务和任务执行 / 结果结算组合器。
```

目标只在 `运行期业务操作组合器` 增加下列 11 个直接强类型委托；全部复用现有请求 / 结果 DTO，不新增平行值类型：

```text
创建完整目标状态需求(创建完整目标状态需求请求) -> 需求提交结果
创建已有需求承接任务(创建任务承接请求) -> 任务提交结果
迁移任务生命周期(迁移任务生命周期请求) -> 任务提交结果
创建方法首(创建方法首请求) -> 方法提交结果
创建方法结果(创建方法结果请求) -> 方法提交结果
创建方法动作入口(创建方法动作入口请求) -> 方法提交结果
创建轻量因果(创建轻量因果请求) -> 轻量因果业务结果
读取需求(节点句柄) -> 需求权威材料
读取任务(节点句柄) -> 任务权威材料
读取方法登记项(节点句柄) -> 方法登记项材料
读取轻量因果(节点句柄) -> 轻量因果值式材料
```

`运行期业务操作组合器` 构造增加需求、任务、方法、轻量因果四个业务服务引用，由唯一 `运行期业务装配` 传入。11 个入口不得接触仓库、接线、令牌、许可、执行器或会话；7 个写入口先做值式入口拒绝，再唯一委托对应业务服务；4 个读入口返回值式副本。

外层入口与现有服务方法的机械映射固定为：

| 外层入口 | 现有唯一委托 |
| --- | --- |
| 创建完整目标状态需求 | `需求业务服务::创建完整目标状态需求` |
| 创建已有需求承接任务 | `任务业务服务::创建承接任务` |
| 迁移任务生命周期 | `任务业务服务::迁移生命周期` |
| 创建方法首 | `方法业务服务::创建方法首` |
| 创建方法结果 | `方法业务服务::创建结果` |
| 创建方法动作入口 | `方法业务服务::创建动作入口` |
| 创建轻量因果 | `轻量因果业务服务::创建轻量因果` |
| 读取需求 | `需求业务服务::读取需求` |
| 读取任务 | `任务业务服务::读取任务` |
| 读取方法登记项 | `方法业务服务::读取登记项` |
| 读取轻量因果 | `轻量因果业务服务::读取轻量因果` |

执行窗口只允许按上表转发，不得因外层命名不同而新建第二套服务方法。

## 6. 非成功返回

空句柄、零幂等键、非法生命周期或不完整请求为逻辑内返回，零写。正式业务服务返回许可拒绝、版本漂移或入口拒绝时保持原强类型状态。完整请求已被业务服务接受后，返回材料与请求身份、阶段、方法规格、动态或因果互证矛盾为追根因。必须修改禁止文件才可形成合同属于接口漂移。
