## 6. 冻结接口合同

### 6.1 核心许可

在核心执行器模块按详细设计实现：

```cpp
struct 节点直接统一冻结见证 final;
class 节点直接统一冻结许可 final;

节点直接统一冻结许可 节点直接身份结构事务域::取得统一冻结许可(
    std::uint32_t 预期冻结规则版本) const;

bool 节点直接身份结构事务域::冻结许可属于本域(
    const 节点直接统一冻结许可& 许可) const noexcept;
```

许可 move-only、RAII、只读，只公开 `有效()` 和 `读取见证()`。见证字段宽度固定为 `冻结域身份 u64 / 已发布代次 u64 / 冻结规则版本 u32`。事务域身份构造时非零；已发布代次初值非零，只有真实 `已提交` 在全部参与者完成发布后、独占许可释放前推进一次。幂等读回、拒绝、撤销和内部错误不推进。

### 6.2 特征值全量读取

`特征值类型化记录只读访问器` 增加：

```cpp
enum class 特征值类型化记录全量读取状态 : std::uint32_t {
    已形成 = 1,
    许可竞争 = 2,
    资源失败 = 3,
    内部不一致 = 0x8000
};

struct 特征值类型化记录全量读取结果 final {
    特征值类型化记录全量读取状态 状态;
    std::uint64_t 结构版本;
    std::vector<特征值类型化记录> 记录组;
};

特征值类型化记录全量读取结果 复制全部已发布记录() const;
```

该入口在一次共享锁内复制全部已发布记录。成功时返回真实非零 `结构版本_`；合法空仓返回 `已形成` 和空组。许可竞争、确定分配失败分别返回同名状态；未知格式、不完整记录、重复 `(特征值, 原始值版本)` 或同一特征值存在多个有效当前记录返回 `内部不一致`。所有非成功结果结构版本为 0 且记录组为空；外层统一冻结许可有效时，数据操作必须把子仓许可竞争提升为聚合 `内部不一致`。

### 6.3 新域 4170 全量读取

`节点直接特征批次记录仓` 增加从 1 开始的真实结构版本，只在批次参与者真实完成发布时推进。节点直接特征数据操作只复制本私仓已发布记录，不读取旧默认域账。

数据操作模块唯一声明并实现：

```cpp
struct 节点直接特征批次冻结记录 final;
struct 节点直接特征同代次冻结材料 final;
enum class 节点直接特征冻结状态 : std::uint32_t;
struct 节点直接特征同代次冻结结果 final;

节点直接特征同代次冻结结果 复制同代次特征值与批次全量(
    const 节点直接统一冻结许可& 许可) const;
```

字段、状态值、两仓版本 / 数量、原子 optional 和 P4 段 9 的固定命名域 ABI 映射逐项服从详细设计 v0.2，不得另造请求 DTO、两个独立 optional 或服务工厂。

## 8. 锁、生命周期与结果

锁序唯一为：

```text
节点直接统一冻结许可
-> 特征值类型化记录仓 shared
-> 节点直接特征批次记录仓 shared
```

持子仓锁时只做边界校验和最小值式复制，不做 I/O、日志、上层服务调用或反向取锁。#349 的许可生命周期覆盖 P2Q 调用；返回后材料自有，数据操作不保存许可引用。公开服务不接触许可。

结果分类：

- 许可取得失败由 #349 形成零材料逻辑内结果。
- 错域 / 无效许可为 `入口拒绝`。
- 确定分配失败为 `资源失败`。
- 取得许可后见证、版本或数量变化，坏记录、部分材料、悬空引用、顺序缺口为 `内部不一致`。
- 两仓合法为空为 `已形成`，返回带见证的两个空组。
