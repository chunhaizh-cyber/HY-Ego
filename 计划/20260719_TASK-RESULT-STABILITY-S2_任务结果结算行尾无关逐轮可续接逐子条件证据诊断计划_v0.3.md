## 5. 行尾无关状态协议

### 5.1 正式解析器

状态文件继续使用 UTF-8 `key=value` 文本，但读取只能使用 PowerShell 结构化键值入口：

```powershell
$原文 = Get-Content -Raw -Encoding UTF8 -LiteralPath $状态路径
$状态 = ConvertFrom-StringData -StringData $原文
```

随后只按键精确读取：

```powershell
$状态['冻结候选']
$状态['冻结自检blob']
$状态['历史有效轮次']
$状态['历史摘要SHA256']
$状态['下一轮']
$状态['终止状态']
```

禁止使用 `(?m)^...$`、按 `\n` 拆分、移除 `\r` 后继续或其它依赖具体行尾的字符串正则。缺键、重复键、空值、未知键、轮号非十进制整数或值不一致都在产品进程启动前具名停止。

### 5.2 双行尾预检

S3 必须先在 R3 非 Git 诊断目录建立内容完全相同的两个样本：

```text

```

两份样本都必须包含六个正式键，并由第 5.1 节同一解析器读回。只有两份均得到 `下一轮=43`、`终止状态=未终止` 且全部字段相等，才可建立正式 `resume-state.tsv`。预检使用独立工具调用，不算第 43 轮，不启动产品进程。

## 6. 单轮工具调用协议

第 43—60 轮必须是 18 次独立工具调用。每次只能启动一个：

```powershell
.\x64\Debug\海中鱼巣.exe --self-test-exit
```

禁止在一条工具调用中用循环、递归脚本、后台任务或多进程完成多轮。每轮按顺序：

1. 用 `ConvertFrom-StringData` 读取正式状态，核对六键、当前轮和未终止；
2. 确认无残留进程且共享时隙仍独占；
3. 启动一个全新进程，完整 stdout / stderr 先写 `resume-round-XX.log`；
4. 自然结束后记录真实退出码，解析 TASK / B3C / B0 行数和失败数；
5. 写一行 `resume-summary.tsv` 和日志 SHA-256；
6. 仅当 exit 0、行数 18 / 12 / 20、失败 0、无异常行时推进游标；
7. 轮末等待 2 秒并结束本次工具调用。

工具超时、取消、日志截断、无真实退出码、状态解析异常、行数不符或任何失败叶子都立即停止，不补跑当前轮，不启动后续轮。
