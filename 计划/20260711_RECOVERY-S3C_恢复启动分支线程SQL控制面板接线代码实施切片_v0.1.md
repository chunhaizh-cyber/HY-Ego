## 1. 目标

建立显式全新 / 恢复启动策略，并在 #237 发布同一运行期上下文后按固定顺序接线瞬态运行组件：

```text
显式选择全新或恢复
-> 恢复拒绝绝不回退全新
-> 取得 #237 同身份上下文租约
-> 自我线程
-> 任务管理 / 工作线程
-> 缓存统计 / 事件 / 外设材料线程
-> SQL Server 结构审计投影
-> 只读控制面板
```

UI 和 SQL 只读恢复结果，不参与屏障、发布或机器事实裁决。

## 5. 实施步骤

1. 复核 #237 租约、现有参数解析、四个初始化入口、七个线程模块、SQL 适配和控制面板启动签名。
2. 定义显式启动策略：全新初始化、从指定快照恢复；未定义、同时选择或恢复目标为空写前拒绝。
3. 全新模式发现固定快照时拒绝，不删除 / 覆盖；恢复模式任一拒绝终止，不调用全新初始化兜底。
4. 全新模式调用 #247 全新工厂并发布；恢复模式才消费 #229 屏障许可并由 #237 适配发布。两种互斥分支共用同一空宿主和租约合同。
5. 线程只在发布后构造；每个线程取得同一租约，不缓存候选期引用。按自我、任务、其它线程顺序启动。
6. SQL 审计与控制面板最后读取；任一失败只隔离人读能力，不回滚或改写已发布机器事实。
7. 停止顺序逆向收口，已发布上下文不在线替换或因 UI / SQL 失败销毁。
8. `自检.恢复启动` 承载策略互斥、顺序、身份和 UI / SQL 隔离；恢复总成聚合，入口只调用一次。
9. 完成 A01-A12、治理、归档、提交和推送，解锁 #239。

## 8. 非成功收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| #237 或线程实际接口漂移 | 依赖门控返回 | 不改代码，退回设计 |
| 策略未定义 / 含混、全新模式发现快照、恢复目标无效 | 逻辑内返回 | 不发布、不启动，不删除快照 |
| SQL / 控制面板启动失败 | 逻辑内隔离 | 记录人读故障，已发布上下文和线程事实不变 |
| 发布后租约混合、线程身份漂移或启动顺序违反 | 追根因解决 | 停止后续启动并有序收口已启线程 |
