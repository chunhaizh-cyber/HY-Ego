## 1. 目标

建立命名治理消息到真实需求的领域闭环：

```text
专用消息重读
-> 同目标概念分片串行
-> 读取未完成命名需求
-> 零个创建 / 一个复用 / 多个追根因
-> 完整信息目标关系
-> 服务根普通父子最终发布
-> 完整读回
```

本切片不创建任务、方法、名称关系、实际状态、动态或结算。

### S1：只读材料和未完成判定

```text
新增概念命名需求状态和完整值式材料。
只沿需求目标概念反向关系取得候选，不全节点扫描。
逐项复核需求类型、主体、目标宿主、场景、共享目标状态、目标概念和服务根父子。
最近结算状态值为已满足才算正式完成；其它结果或无记录均算未完成。
历史已满足需求可审计读取，但不参与未完成唯一性计数。
```

### S2：分片锁与创建复用

```text
固定 64 个分片互斥量，以完整概念句柄稳定选择分片。
锁内重新读取并分类未完成需求数量。
零个进入创建；一个完整复用；多个触发追根因且不返回第一个。
新活动版本请求仍复用同一未完成需求，活动版本不持久化为需求身份。
同概念并发只允许一个已创建，不同概念保持可并发。
```

### S3：多步创建和发布

```text
创建需求节点和主信息。
写主体、目标宿主、场景、共享目标状态关系。
写需求目标概念关系。
完整预发布读回，不完整则撤销本次候选。
调用提交需求父子更新挂到服务根，作为需求树最终发布。
发布后完整读回；异常时删除本次父子关系并逆序撤销本次候选，触发追根因。
删除补偿逐项确认目标句柄已经消失，不扫描修复其它结构。
```

### S4：专用领域路由

```text
新增独立 ixx 模块并登记工程文件。
只接收协议版本 2 的概念待命名消息。
```

## 7. 入口拒绝

```text
主体或目标宿主不是存在，场景不是场景。
共享目标状态无效、不是抽象状态或与初始化结果不一致。
目标概念无效、没有当前概念身份或目标关系类型不受支持。
服务根不是完整服务根需求。
主体 / 宿主 / 场景与服务根初始化上下文不一致。
同概念已经存在多个未完成需求。
既有唯一候选缺承接、目标关系、目标状态、父需求或结算材料。
消息协议拒绝、活动版本漂移或当前名称已经存在。
```

所有输入拒绝发生在创建节点和关系之前；多需求或内部残缺归追根因，不得普通拒绝后继续。

## 8. 失败收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 输入类型、初始化上下文、消息或活动版本不满足 | 逻辑内返回 | 不写结构 |
| 一个完整未完成需求 | 幂等复用 | 不推进关系版本，不新增父子 |
| 多个未完成需求或既有当前结构残缺 | 追根因解决 | 不选第一项，不创建新需求 |
| 新候选节点、承接关系或目标关系写入不及预期 | 追根因解决 | 逆序撤销本次未发布候选 |
| 服务根挂载不及预期或发布后完整读回矛盾 | 追根因解决 | 删除本次父子并逆序撤销，停止依赖 |
| 删除补偿本身失败或删除后仍可读 | 追根因解决 | 立即停止，不以扫描器继续修复 |

## 9. 执行前接口复核与漂移退回

以下任一情况视为漂移：

```text
#212 的关系值、共享目标状态或初始化结果形状不同。
需求承接完整性、结算读取或父子提交返回改变。
关系仓库删除 / 反向读取、节点删除或主信息删除接口改变。
已有同功能命名需求入口或并发锁边界形成。
固定安全根路由、自我线程或邮箱已被改成不同的业务分派模型。
完成本切片需要修改核心仓库、概念图服务、语素服务、任务服务或状态服务。
```

出现漂移不得修改代码，退回设计窗口修订 #213-#215。
