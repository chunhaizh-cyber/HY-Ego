本计划把同一任务两条线程同时进入筹办后的“并发同义选择”视为可收口成功，违反“同一任务同一时刻只有一个有效筹办执行权，败者必须在召回前退出”的任务级并发规则；同时，本计划没有先迁移鱼巢正式任务、方法和内部循环业务规范。

因此本计划全文只作历史证据，所有 S0—S6、固定恢复、允许文件和验证授权均失效。不得重派 R8、恢复十个源 blob、修改 / 构建 / 运行 / 写记录、创建 R9 或消费 #316。

## 1. 目标

消费生产运行期已经发布的安全根需求，在同一租约 B0 正式业务面上形成：

```text
未满足根需求
-> 创建或幂等读回承接任务
-> 已承接 -> 筹办中
-> 正式方法基线
-> 唯一任务方法选择
-> 纯值任务执行冻结请求
```

本版只修一个已证明的并发语义：两个同义线程中，线程甲先发布完整当前选择后，线程乙的方法召回因重读任务发现该选择而返回入口拒绝。线程乙不得在权威读回前提前返回；它必须重读当前任务与当前方法，并且只有当前选择完整且与本路由全字段同义时，才把它作为并发同义完成读回继续冻结。

本版不修改召回服务、组合器、任务服务或数据操作，不新增重试、等待、睡眠、第二次召回或第二次选择提交。

## 5. 待实施目标

### 5.1 权威读回顺序

组合调用成功、非成功或调用前已有选择，最终都进入同一个路由级权威裁决：

```text
读取当前任务
-> 读取当前方法
-> 区分读回不完整与真实当前性漂移
-> 检查当前选择存在性和完整性
-> 检查选择与本路由输入全字段同义
-> 只有全同义才把权威选择写入报告并继续冻结
```

组合非成功时，先保留返回状态：召回或选择为内部不一致时立即按内部不一致收口，不得被当前选择掩盖；其它状态才进入权威读回分流。

### 5.2 全字段同义

当前选择必须 `完整()`，并同时满足：

```text
记录身份.幂等主键 == 配置.选择记录幂等主键
任务 == 本路由任务
方法 == 本路由方法
幂等材料编号 == 配置.选择幂等材料编号
来源生命周期关系 == 权威任务.当前生命周期.关系.关系
来源生命周期版本 == 权威任务.当前生命周期.版本
权威任务.当前生命周期.阶段 == 筹办中
来源方法生命周期关系 == 权威方法.生命周期关系.关系
来源方法生命周期版本 == 权威方法.生命周期关系.关系.版本号
排序规则版本 == 配置.排序规则版本
召回请求规则版本 == 配置.召回请求规则版本
来源候选数量 != 0
权威任务.来源需求 == 当前安全根需求
权威方法.身份.节点 == 本路由方法
权威方法.登记根 == 系统角色方法登记根
权威方法.生命周期 == 活跃
```

### 5.3 非成功返回二分

| 事实 | 收口 |
| --- | --- |
| 召回或选择结果内部不一致 | 追根因解决；路由状态内部不一致 |
| 权威任务或方法读回不完整 | 追根因解决；内部不一致 |
| 权威任务来源/生命周期或权威方法活跃关系真实漂移 | 逻辑内返回；当前性变化 |
| 当前选择存在但不完整 | 追根因解决；内部不一致 |
| 当前选择完整但任一同义字段不匹配 | 逻辑内返回；当前性变化 |
| 当前选择完整且全部字段同义 | 并发同义幂等读回；继续冻结 |
| 无选择且召回状态为无候选 | 逻辑内返回；无候选 |
| 无选择且召回状态为非唯一建议 | 逻辑内返回；非唯一 |
| 无选择、任务/方法仍稳定、召回入口拒绝或唯一建议后选择未提交 | 追根因解决；内部不一致 |
