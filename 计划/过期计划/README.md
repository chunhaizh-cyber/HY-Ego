## 1. 进入本目录的条件

计划满足下列任一条件时由设计窗口移入本目录：

```text
正式撤销或终止
被新计划或新架构路线替代
依赖图已经失效
绑定的流程图、详细设计、接口假定或验收矩阵已经失效
当前版本明确写明不得执行
```

本目录文件只保存历史意图、旧接口证据和裁决过程。禁止仅凭文件存在恢复执行、创建 worktree、解除依赖或声明能力已实现。

## 3. 恢复规则

只有用户新裁决和设计窗口完成下列全部步骤后，才可让某项能力重新进入活动路线：

```text
读取现行正式规范
-> 当前代码事实扫描
-> 新详细设计
-> 新流程图 MD + HTML
-> 新函数 / 结构清单和知识图谱
-> 新施工计划与新版本
-> 在计划索引登记精确版本、路径、计划状态和具名依赖
-> 执行窗口按计划状态与真实依赖自主选择
```

恢复能力时应建立新计划文件，不把本目录旧文件移回根目录继续执行。
