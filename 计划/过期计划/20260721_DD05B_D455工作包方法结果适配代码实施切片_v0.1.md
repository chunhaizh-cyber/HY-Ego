## 4. 目标流程

```text
DD-04 工作包和一次调用内只读材料视图
-> 稳定观察 / 扫描变化 / 目标跟踪动作运行
-> D455 实际回读和来源质量材料
-> 对应场景 / 状态 / 动态领域服务正式提交
-> DD-05A 显式任务结果治理入口
-> 权威读回已经持久化的任务治理结果
-> 工作包和材料消费具名结束
```

三类结果：

| 方法入口 | 实际结果 | 提交合同 |
| --- | --- | --- |
| 稳定观察 | 已确认存在 / 状态观察，或空观察 | 按范围和质量提交正常、异常或待复核事实；空观察不伪造存在 |
| 扫描变化 | 前后可比较变化，或材料不足 | 形成状态、动态和结构化残差；材料不足进入待复核 / 合法等待 |
| 目标跟踪 | 当前坐标、暂失或丢失 | 当前时提交跟踪状态；暂失 / 丢失保持空坐标和具名状态 |

每类结果都绑定工作包、任务、工作项、方法运行身份、方法内容版本、动作入口、执行轮次、来源产品、时间、范围、质量、消费窗口和当前性。D455 报告本身不得进入自我邮箱。

## 5. 入口拒绝、逻辑内结果和内部错误

写前拒绝：错工作包 / 任务 / 方法 / 工作项、错产品类型、消费窗口失效、材料过期、内容版本漂移、范围或质量不足、重复结束或无权提交；零机器结构变化。

逻辑内结果：空观察、材料不足、目标暂失 / 丢失、工作包过期或当前外部事实变化；必须形成设计规定的强类型结果，不伪造任务失败或内部错误。

内部错误：完整前置后领域写入 / 读回 / 发布失败、同一产品被第二消费者消费、已确认工作包出现结构缺口、同一执行轮次异义重复、显式治理已接受却不能读回任务治理结果。内部错误停止依赖路径，不包装成等待、空观察或失败。

## 6. 依赖解除后的机械阶段

读取 #327 / #331 的 main 提交、实施 / 集成记录和真实签名；形成精确文件清单、模块命名、阶段号、共享工程 / 入口所有者和验证命令。DRIFT 时退回设计修订，不写代码。

### S1 建立 D455 结果适配值式材料

只携带工作包和只读视图允许的信息；不复制报告队列、帧缓存或任务权威结构，不把日志 / 显示字段变成事实。

### S2 实现稳定观察适配

完成存在 / 状态或空观察的质量、范围、时间和当前性复核，经对应领域服务提交，再调用 DD-05A 显式治理入口。

### S3 实现扫描变化适配

比较前后材料，形成状态 / 动态 / 残差或材料不足结果；保持正常、异常和待复核事实分层。

### S4 实现目标跟踪适配

区分当前坐标、暂失、丢失和材料不足；空坐标必须与具名状态共同成立，不使用魔法坐标或显示文本。

### S5 结束工作包和消费

只有领域提交完成，DD-05A 显式治理返回成功，并且按任务 / 工作包 / 方法运行 / 轮次权威读回同一个已持久化治理结果后，才调用 #327 的具名结束入口。治理与结束不得并行；写前拒绝保持未消费，内部逻辑错误隔离本次工作包，重复同义治理只读回同一个结束结果。

### S6 自检、构建和任务分支收口

逐类覆盖正常、拒绝、逻辑内结果、幂等、异义、过期和内部错误；运行两配置 Rebuild、两配置 `--self-test-exit`、strict、diff 和禁止差异；只提交冻结文件及专属实施记录。

## 7. 最低验收

1. 三类入口各自只消费工作包签发的只读视图，不直接读取报告队列或 D455 全局缓存。
2. 每条结果完整绑定任务、工作项、方法运行、内容版本、动作、执行轮次、产品来源和当前性。
3. 正常、异常、待复核、空观察、材料不足、暂失和丢失均有不同强类型收口。
4. 世界事实只能由对应领域服务最后发布；适配器和报告不能直接写仓库事实。
5. 任务结果只经 DD-05A 显式治理入口；D455 报告、适配材料和工作包不进入自我邮箱。
6. 同义重复幂等，异义重复冲突；过期工作包、消费窗口或版本写前拒绝。
7. 工作包 / 材料消费仅在任务治理结果权威读回后结束；治理与结束不存在并行旁支，异常不伪装正常结束。
