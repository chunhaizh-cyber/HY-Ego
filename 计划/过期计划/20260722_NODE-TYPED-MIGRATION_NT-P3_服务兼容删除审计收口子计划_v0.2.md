## 1. 本版裁决

本版保持 v0.1 与现行 P3 详细设计的业务逻辑、对象边界、错误分类和完成边界不变，只修订施工并发方式：

```text
先冻结提供者—消费者接口合同
-> 提供计划负责按合同实现真实接口
-> 消费计划不等待提供计划实现完成，直接按合同形成隔离源码候选
-> 单任务只做合同、静态、白名单和 diff 卫生检查
-> 按固定顺序汇集全部候选
-> #352 唯一完成工程登记、统一运行器接线、构建、运行、真实联调和最终验收
```

“预冻结待实现接口合同”不是当前代码事实，也不是伪造的可编译接口。提供者不得任意改变合同；消费者不得因当前符号尚不存在而回退旧域、复制算法、建立临时兼容层或自行改写合同。最终接线发现签名、模块身份、返回语义或所有权漂移时，只退回精确提供计划或消费计划。
