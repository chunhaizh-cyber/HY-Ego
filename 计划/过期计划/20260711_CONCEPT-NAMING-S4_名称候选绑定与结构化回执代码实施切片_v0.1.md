## 1. 目标

在真实任务授权和方法动作接口已经形成的前提下，把结构化候选语素入口绑定为概念名称，并生成值式回执：

```text
命名需求
+ 来源任务与合法当前态
+ 已选方法和动作入口
+ 候选语素入口
-> 语素服务正式绑定
-> 概念侧与语素侧双向读回
-> 方法动作动态 0 -> 1
-> 结构化绑定回执
```

本切片不生成名称字符串，不负责通用方法召回或任务选择，不记录任务结果或需求结算。

### S1：候选与授权复核

```text
候选必须包含命名需求、来源任务、目标概念、候选语素入口、双向承接授权关系 / 版本、当前生命周期关系 / 版本、精确当前选择关系 / 版本、已选方法、动作入口、活动图版本和时间戳。
需求服务以同一共享令牌调用 `读取概念命名需求权威材料(需求, 令牌)`；入口只接收需求句柄并自行派生主体、目标宿主、场景、目标状态 / 关系、目标概念 / 关系、服务根需求 / 发布关系和满足状态。候选需求、来源任务承接需求和候选目标概念必须与权威 DTO 一致；后续名称写入只使用 DTO 的权威目标概念。
任务服务以共享路径令牌重读来源需求、双向授权、当前态和精确当前选择关系 / 版本；独占令牌仍可复用同一只读路径。
方法服务以同一令牌和该选择材料重读登记、生命周期、方法投影、动作入口归属和主动可调用状态；方法角色抽象状态值必须经状态服务令牌入口读取，任意任务到方法引用、裸状态解释或无令牌兼容读取不能替代。
语素服务证明候选入口是正式语素入口。
正常过期、未授权、未选择、失效方法和错误候选均写前拒绝。
```


### S3：动作动态与回执

```text
名称关系完整读回后，经方法服务令牌入口记录动作动态；方法服务把同一令牌下传给动态服务令牌入口。
被改变目标为目标概念，改变前值为 0，改变后值为 1，来源动作为已选动作入口。
动态服务必须委托状态服务以同一共享路径令牌和 #256 候选创建前后值实例状态。状态 / 动态服务分别持有自己创建的主信息 / 节点候选，构造全部非发布关系并完成发布前读回；方法服务写来源方法关系。任一步失败按来源方法关系、动态关系 / 候选、前后值状态关系 / 候选逆序撤销。运行期临时最终发布关系成功后立即确认候选，再以同一令牌公开读回完整材料。
全部一致后生成值式绑定回执，不持久化第二份名称事实。
名称关系成功但动态内部失败时，名称事实保持有效、需求保持“目标已出现待正式结算”，并触发追根因；不得伪造回执或结算。
```

| 场景 | 分类 | 收口 |

| #256 未完成、585 未登记或候选签名漂移 | 逻辑内门控返回 | 不改 #214 代码，先完成或修订 #256 |
