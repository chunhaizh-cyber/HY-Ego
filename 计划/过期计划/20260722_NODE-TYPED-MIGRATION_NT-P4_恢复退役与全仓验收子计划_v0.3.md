### 3.2 NT-C4/v0.3 规范化格式 ABI

当前格式固定使用下列数值合同；全部多字节整数按小端编码：

| 项 | 固定值 / 宽度 |
| --- | --- |
| 魔数 | 8 字节 `48 5A 59 43 53 4E 50 31`，ASCII 为 `HZYCSNP1` |
| 容器格式版本 | `uint16 = 1` |
| 字节序标识 | `uint16 = 1`，只表示小端；其它值拒绝 |
| 容器头字节数 | `uint32 = 104` |
| 段头字节数 | `uint32 = 64` |
| 段清单版本 | `uint32 = 1` |
| 摘要算法版本 | `uint32 = 1`，固定 SHA-256，输出 32 字节 |
| 冻结规则版本 | `uint32 = 1` |
| 节点稳定主键命名域 ABI 版本 | `uint32 = 1` |
| 必需段数量 | `uint32 = 9`，每类恰一段并按段类型升序出现 |

104 字节容器头按偏移固定为：

| 偏移 | 字段 | 宽度 |
| --- | --- | --- |
| 0 | 魔数 | 8 字节 |
| 8 | 容器格式版本 | uint16 |
| 10 | 字节序标识 | uint16 |
| 12 | 容器头字节数 | uint32 |
| 16 | 段头字节数 | uint32 |
| 20 | 段清单版本 | uint32 |
| 24 | 摘要算法版本 | uint32 |
| 28 | 段数量 | uint32 |
| 32 | 文件总字节数 | uint64 |
| 40 | 来源运行期域身份 | uint64，非零 |
| 48 | 来源节点直接仓库身份 | uint64，非零 |
| 56 | 快照代次 | uint64，非零且同来源单调 |
| 64 | 冻结规则版本 | uint32 |
| 68 | 节点稳定主键命名域 ABI 版本 | uint32 |
| 72 | 容器摘要 | 32 字节 |

64 字节段头按偏移固定为：

| 偏移 | 字段 | 宽度 / 规则 |
| --- | --- | --- |
| 0 | 段类型 | uint16 |
| 2 | 段格式版本 | uint16，当前均为 1 |
| 4 | 段头字节数 | uint32，必须为 64 |
| 8 | 段顺序 | uint32，从 0 连续到 8 |
| 12 | 段标志 | uint32，当前必须为 0 |
| 16 | 记录数量 | uint64 |
| 24 | 段载荷字节数 | uint64 |
| 32 | 段载荷摘要 | 32 字节 SHA-256 |

段类型 ABI 固定为：`1=节点直接记录`、`2=正式关系记录`、`3=特征值类型化记录`、`4=状态域记录`、`5=动态域记录`、`6=任务方法选择记录`、`7=概念签名记录`、`8=用途观察记录`、`9=特征批次发布记录`。九段均为必需段；合法无记录用记录数量 0 表达，不得省略段。新增段只能提升段清单版本，不能复用或改义 1—9。节点直接记录逐项无损保存节点编号、稳定主键、由仓库身份/节点编号/版本构成的完整稳定句柄、节点类型、记录版本、记录状态和创建序号；节点编号不得从排序位置、稳定主键或句柄外部上下文反推。其余八段逐项保存对应领域规范要求的全部类型化字段、记录版本、记录状态、历史与失效信息，禁止以摘要、数量、索引、投影或通用字节载荷代替原始记录。

规范化上限固定为：文件总长不超过 `8,589,934,592` 字节；单段载荷不超过 `2,147,483,648` 字节；每段记录不超过 `4,194,304` 项；单条规范化记录不超过 `16,777,216` 字节；UTF-8 字符串不超过 `1,048,576` 字节；单个向量不超过 `1,048,576` 项。字符串和向量数量前缀固定为 uint32；单条记录以前置 uint32 记录字节数分帧，但业务字段仍由该具名段的强类型 DTO 唯一解释，不能成为通用字节业务记录。

布尔值固定编码为 uint8 的 0/1；可选值固定编码为 uint8 存在标志 0/1 后接具名值；枚举按正式数值 ABI 的固定宽度编码；节点稳定主键固定编码两个 uint64 分量；完整句柄固定编码为 `uint64 仓库身份 + uint64 对象编号 + uint32 版本号`，与 #339 真实句柄 ABI 一致。字符串只编码 UTF-8 字节且不依赖区域设置；未由对象规范要求时不得擅自正规化文本。各段记录按其冻结 DTO 的 `规范化排序键` 升序编码；排序键至少包含对象完整稳定身份、记录版本和记录内顺序，禁止依赖容器遍历、地址或线程时序。

段摘要只覆盖该段载荷。容器摘要覆盖整个文件，但计算时把容器头偏移 72—103 的摘要字段视为 32 个零字节；摘要比较使用完整 32 字节。恢复候选再冻结相等只比较九段规范化权威载荷，不比较来源运行期域身份、来源仓库身份、快照代次、容器摘要或可重建投影。

### 3.4 NT-C5/v0.3 运行期与发布合同

NT-C5/v0.3 由 #352 唯一实现，固定提供 `生产运行期启动模式`、`生产运行期启动请求`、`生产运行期启动结果`、`运行期上下文`、`运行期上下文租约`、`运行期上下文宿主`、`运行期首次发布结果` 和 `完整自我恢复证据读取结果`。启动模式数值 ABI 为 `1=全新`、`2=恢复`；0 和未知值拒绝。签名固定为：

启动请求使用值式合同版本、模式、候选建立规格和恢复路径；全新模式不得携带恢复路径，恢复模式必须携带唯一非空路径。启动结果区分 `已启动/入口拒绝/材料拒绝/资源失败/已有当前上下文/内部逻辑错误`。宿主生命周期固定为 `空 -> 已发布`，只允许一次迁移；没有清空、替换、恢复回退全新或第二次发布。租约只证明同身份已发布上下文的读取生命期，不暴露可变仓库、锁、令牌或服务内部地址。
