## 1. 目标

把单个值式待命名请求无损转换为协议版本 2 的非权威治理消息：

```text
明确交付一个待命名请求
-> 以概念图服务 + 语素服务调用现有双参读取
-> 请求消失时由语素服务反查区分已有名称
-> 重读当前概念、根类别和活动版本
-> 构造目标概念与来源材料版本
-> 以完整字段约束确定性消息身份
-> 提交唯一治理邮箱
```

本切片不创建需求、不绑定名称、不新增线程，不调用批量待命名读取自动提交。

## 7. 入口拒绝

写前拒绝：

```text
当前时间戳为 0。
截止时间戳为 0、早于或等于当前时间戳。
批次编号为 0。
请求概念句柄无效、活动版本为 0 或原因不属于名称为空。
现有双参入口返回当前单概念待命名请求不存在，且语素名称入口反查为空。
概念、根类别或活动版本与当前读回不一致。
双参入口无当前请求且语素名称入口反查非空，目标已经有名称。
确定性标识构造无法形成非零值。
```

拒绝必须证明邮箱快照、节点、主信息、关系和索引数量不变化。

## 8. 失败收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 输入无效、双参读取无请求且名称入口为空 | 逻辑内返回 | 返回当前请求不存在，不提交邮箱、不写结构 |
| 双参读取无请求且名称入口非空 | 逻辑内返回 | 返回目标已有名称，不提交邮箱、不写结构 |
| 活动版本 / 根类别漂移或过期 | 逻辑内返回 | 不提交邮箱，不写结构 |
| 同完整字段和幂等键重复 | 逻辑内返回 | 返回重复材料，不新增队列项 |
| 同幂等键但完整字段不同 | 逻辑内拒绝冲突 | 不覆盖既有消息，不由摘要裁决 |
| 当前协议映射或邮箱准入出现内部矛盾 | 追根因解决 | 停止该消息依赖路径，不重试修复 |
