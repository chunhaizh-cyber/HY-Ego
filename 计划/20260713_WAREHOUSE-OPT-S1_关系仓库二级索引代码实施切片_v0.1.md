## 3. 核心处理

1. 在关系仓库私有结构中增加有效正向、有效反向、有效节点相关、当前审计正向四类关系编号候选索引。
2. 创建、失效、重挂、物理删除在同一关系仓库独占锁内同步更新关系表和索引。
3. 查询先从索引复制候选，再逐项复核关系表当前版本、状态、关系类型、顺序和有效端点。
4. 替换已确认的正向、反向、存在、节点相关、审计和唯一性全表扫描；未证明等价的路径保留并登记，不临时扩大。
5. 保持确定性排序、普通父子单父和防环、其它关系的多目标 / 多父语义。
6. 扩展 S0 自检，做全表参考模型等价、故障注入、并发原子可见和性能对比。

## 4. 结构承载、输出与调用路径

关系表继续是唯一权威；二级索引只保存关系编号候选。通用索引仓库零改义。

```text
领域服务 / 仓库公开查询 -> 关系键 -> 二级索引候选
-> 关系表当前记录复核 -> 有效端点复核 -> 确定性排序 -> 值式结果
```

写路径：

```text
入口拒绝 -> 唯一性候选复核 -> 预备索引项 -> 同锁提交主表和索引
-> 最小读回；失败则回退本次项且不返回有效句柄
```

## 7. 失败收口与一致性

索引预备分配失败时不得修改主表。提交中任一步失败必须逆序撤销本次索引项和主表项，不返回有效关系句柄；编号可跳号但不可复用。失效、重挂、删除后发现旧索引残项属于逻辑错误，停止后续验收。故障注入必须证明没有可读半索引关系，不能证明时退回设计。
