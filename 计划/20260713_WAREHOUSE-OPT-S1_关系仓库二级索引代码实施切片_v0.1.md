## 3. 核心处理

1. 保留四类候选索引方向，但把桶容器和维护路径收敛为“受影响桶一次定位 + 值式转换计划 + 定向发布 / 移除 + 逆序回退”。
2. 普通热路径不得在同一次状态切换前后各执行通用五项索引成员复核。只检查本次受影响旧成员、目标成员和最小提交读回；Debug 全索引诊断退出正式写入 P95 计时段。
3. 稀疏桶不得继续无证据地叠加每关系编号哈希节点和即时空桶分配 / 回收。可采用紧凑候选桶、连续编号与私有位置 / 附着元数据；如保留首版嵌套哈希集合，必须先证明全部写入门禁。
4. 创建、失效、重挂、逻辑删除、结构化撤销、普通父子挂载 / 重挂和批量失效仍在同一关系仓库独占锁内同步更新关系表和索引；确认入口只复核写后记录与受影响索引后确认能力。
5. 查询先从索引复制候选，再逐项复核关系表当前版本、状态、关系类型、顺序和有效端点；替换已确认的正向、反向、父子、存在、节点相关、审计、唯一性和删除准备包全表扫描。
6. 保持确定性排序、普通父子单父和防环、其它关系的多目标 / 多父语义；`有效关系数量` 等未冻结路径不临时扩大。
7. 自检把材料完整、查询正确、稳定性和性能门禁拆开记录。A02-A06 不再被稳定性布尔伪报失败，但 A13 仍要求所有稳定和性能硬门禁通过；Release 不暴露候选计数或故障控制符号。

## 4. 结构承载、输出与调用路径

关系表继续是唯一权威；二级索引只保存关系编号候选。通用索引仓库零改义。

```text
领域服务 / 仓库公开查询 -> 关系键 -> 二级索引候选
-> 关系表当前记录复核 -> 有效端点复核 -> 确定性排序 -> 值式结果
```

写路径：

```text
入口拒绝 -> 形成受影响桶转换计划 -> 一次定位并预备目标桶
-> 同锁定向发布新候选 / 移除旧候选 / 更新权威记录
-> 最小受影响项读回；失败则逆序回退本次项且不返回有效句柄
```

## 7. 失败收口与一致性

新增桶或候选分配失败时不得修改权威记录。转换计划中任一步失败必须只按已发布步骤逆序撤销本次新索引项并恢复写前关系记录，不返回有效关系句柄；编号可跳号但不可复用。失效、重挂、逻辑删除、撤销或批量失效后发现旧索引残项属于逻辑错误，停止后续验收。固定 Debug 故障注入必须逐点证明没有可读半索引关系，不能证明时以原编号退回设计。

执行窗口不得用清空 WIP、改写 #257 基线、缩短冷却、移出 Debug 配置、忽略 1k / 10k 低绝对耗时门槛或把全量诊断计入业务结构来规避失败。若 JY-356 指定的算法重做后仍有一次正式运行超过写入硬门禁，停止暂存和归档，再次按原编号退回；默认后继裁决为“不实施 / 不发布 S1”。
