# NT-P2 领域载荷与自我投影迁移子计划

日期：2026-07-22

版本：v0.1

上级：`#338 / DQ-230 / NODE-TYPED-MIGRATION`

状态：依赖 NT-P1；A / B / C 仅在所有权互斥与共同基线验证后条件并行，不可派发

## P2A 特征值类型化原始记录

目标：把 I64、VecI64、VecU64 和原始值版本迁入特征值域唯一类型化记录；槽位当前值只由正式关系表达，4170 批次账保持独立。

候选允许：`参与者.特征值原始材料.ixx`、`数据操作.特征体系.ixx`、特征 / 特征值服务、相关组合与自检、新类型化材料文件及工程登记。

禁止：状态动态数据操作、自我内部世界、任务 / 方法、运行期恢复、4170 字段并入特征值。

## P2B 状态与动态完整事实

目标：状态和动态的时间、记录模式、聚合版本等进入各自类型化记录；主体、特征、值、场景、来源、被改变目标、前后状态和来源动作只由正式关系表达。

候选允许：`数据操作.状态动态.ixx`、`服务.状态.ixx`、`服务.动态.ixx`、兼容状态 / 动态服务头、相关组合、自检、新状态 / 动态记录文件及工程登记。

禁止：特征值原始记录、需求任务方法、外设、运行期恢复和自我列表持久数组。

## P2C 自我内部世界和读取投影

目标：建立唯一自我存在、唯一内部世界场景引用、内部世界成员关系、自我实例槽关系，以及由主体关系确定重建的状态 / 动态读取投影。

候选允许：`数据操作.存在场景.ixx`、存在 / 场景 / 特征公开服务的窄入口、自我初始化与聚合读取模块、相关自检及工程登记。

禁止：新增自我专用平行本体、子存在 / 特征 / 状态 / 动态数组、直接仓库写和生产启动恢复。

## 并发与共享所有权

P2A—P2C 只有在 NT-P0 证明文件、类型、公开接口和工程接线互斥时并行。`数据操作.特征体系.ixx`、共享工程文件、运行期装配和统一自检运行器必须指定唯一所有者；存在交集时拆出共享接线切片或具名串行，不以同属 P2 伪造依赖。

## 共同拒绝、失败和验证

错节点类型、错所属领域、关系缺项、记录版本漂移、重复当前关系和列表 / 关系不一致必须在写前拒绝或按内部逻辑错误收口。验证覆盖三种特征值互斥、状态 / 动态完整事实、投影清空重建、自我唯一性、自我非自身成员、关系和记录换代历史、双配置与完整自检。

## 完成边界

只证明三个领域切片迁移；其它对象、兼容服务、快照恢复和旧仓库仍由 NT-P3 / P4 治理。
