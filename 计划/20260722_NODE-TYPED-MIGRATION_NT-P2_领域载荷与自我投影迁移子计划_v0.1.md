# NT-P2 领域载荷与自我投影迁移子计划

日期：2026-07-22

版本：v0.1

上级：`#338 / DQ-230 / NODE-TYPED-MIGRATION`

状态：依赖 NT-P1；现行完整叶子存在真实接口消费链，固定按 P2A -> P2B -> P2C 串行；不可派发

## P2A 特征值类型化原始记录

目标：把 I64、VecI64、VecU64 和原始值版本迁入特征值域唯一类型化记录；槽位当前值只由正式关系表达，4170 批次账保持独立。

候选允许：只新建特征值类型化记录、参与者、节点直接特征数据操作 / 服务 / 组合 / 自检模块；现行 `参与者.特征值原始材料.ixx`、`数据操作.特征体系.ixx` 和现行服务只读。工程登记由本叶在自身串行执行期独占，集成后移交后继。

禁止：状态动态数据操作、自我内部世界、任务 / 方法、运行期恢复、4170 字段并入特征值。

## P2B 状态与动态完整事实

目标：状态和动态的时间、记录模式、聚合版本等进入各自类型化记录；主体、特征、值、场景、来源、被改变目标、前后状态和来源动作只由正式关系表达。

候选允许：只新建状态 / 动态类型化记录、参与者、节点直接状态动态数据操作、服务、组合和自检模块；现行 `数据操作.状态动态.ixx`、现行服务与兼容服务头只读。P2B 必须消费 P2A 已集成的 `复核特征值归属`，工程登记在自身串行执行期独占。

禁止：特征值原始记录、需求任务方法、外设、运行期恢复和自我列表持久数组。

## P2C 自我内部世界和读取投影

目标：建立唯一自我存在、唯一内部世界场景引用、内部世界成员关系、自我实例槽关系，以及由主体关系确定重建的状态 / 动态读取投影。

候选允许：只新建自我值式数据、节点直接存在场景数据操作、节点直接存在 / 场景服务、两个组合器、系统角色回执 / 初始化和独立自检；现行存在 / 场景 / 特征公开服务、系统角色清单、启动、装配和自我线程只读。P2C 消费 P2A 的槽位组及 P2B 的状态 / 动态事实组，工程登记在自身串行执行期独占。

禁止：新增自我专用平行本体、子存在 / 特征 / 状态 / 动态数组、直接仓库写和生产启动恢复。

## 依赖、并发与共享所有权

完整叶子的真实依赖固定为：

```text
NT-P1
-> P2A：形成复核特征值归属、读取宿主实例特征槽位组
-> P2B：消费复核特征值归属，形成读取主体状态 / 动态事实组
-> P2C：消费槽位组和状态 / 动态事实组，形成自我聚合投影
```

因此 P2A—P2C 不并行。若未来要并行，必须另行拆分 P2B 记录底座与完整写入口、P2C 关系 21 底座与聚合投影，并重新完成设计与编号；本计划不预授权该拆分。

`海中鱼巣.vcxproj`、`海中鱼巣.vcxproj.filters` 和 `海中鱼巣/自检.运行器.ixx` 只在当前串行叶子执行期由该叶增量登记自身模块 / 专属自检，集成后所有权移交后继；`入口.cpp` 和默认装配不得由 P2 修改。

错节点类型、错所属领域、关系缺项、记录版本漂移、重复当前关系和列表 / 关系不一致必须在写前拒绝或按内部逻辑错误收口。验证覆盖三种特征值互斥、状态 / 动态完整事实、投影清空重建、自我唯一性、自我非自身成员、关系和记录换代历史、双配置与完整自检。
