## 1. 任务定位

本计划承接 FS-09 事件日志持久化恢复只读扫描和逻辑迁移包归并材料，目标是把可进入第一轮增强的事件日志边界、仓库快照恢复前置拒绝和 SQL / 旧链表格式排除边界转成后续可执行路线。

本计划是确认对象。`实施记录/20260707_FS09_事件日志持久化恢复逻辑迁移包归并表.md` 只作为生成本计划的证据材料，不是本窗口最终确认目标。

本计划不迁移旧函数体，不新增旧主信息字段，不直接进入 C++ 实施。代码实施必须另建切片，并再次确认允许文件、禁止文件和验证方式。


第一轮目标是补齐事件日志持久化恢复专项的计划边界：

```text
事件日志只记录已由结构确认的事件材料。
运行期机器事实仍以内存节点仓库、主信息仓库、关系仓库、索引仓库和领域服务写入结果为准。
仓库快照恢复必须有快照版本、枚举版本、仓库编号、本地编号、句柄版本和关系版本复核。
恢复入口必须先拒绝版本不匹配、枚举不匹配、无效句柄、悬空关系、错误节点类型和越权恢复。
第一轮优先文件事件日志和仓库快照格式，不直接接 SQL Server ADO。
```

## 5. 设计条款清单

### 必须实现

```text
事件日志只做人读审计辅助和后续恢复材料入口，不裁决当前运行期事实。
事件日志记录对象必须来自已由领域服务或仓库正式入口确认的结构事件。
快照恢复必须先验证快照格式版本、节点类型枚举版本、关系类型枚举版本和仓库身份。
恢复材料中的节点、主信息、关系和索引必须经仓库入口或领域服务复核后才能入账。
恢复失败不得留下半结构、悬空关系、可读有效任务、可读有效需求或可读有效方法。
SQL / ADO 不可用不得回滚或改变内存权威结构。
旧链表二进制读取失败或版本不匹配必须拒绝，不得兼容性覆盖新仓库。
```

### 禁止实现

```text
不接 SQL Server ADO。
不实现控制面板事件流水。
不实现旧世界树 / 领域树 SQL 投影。
不迁移旧双向链表模板序列化格式。
不迁移旧特征值主信息序列化字段。
不实现线程事件持久化。
不实现外设 / 体素 / D455 快照恢复。
不把日志、SQL 快照或字段恢复比对作为业务事实。
```

### 后续待补

```text

仓库快照格式详细设计。
恢复入口前置拒绝验收矩阵。
快照版本和枚举版本锁定规范补充。

显示层只读视图专项。

```
