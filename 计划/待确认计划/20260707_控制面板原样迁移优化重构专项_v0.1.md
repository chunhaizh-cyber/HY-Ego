## 2. 策略定位

本专项把控制面板从“长期排除”调整为“独立专项承接”，但不改变机器事实边界：

```text
第一阶段：原样迁移旧控制面板，作为隔离的人读显示 / 操作壳。
第二阶段：在隔离壳内优化功能、补缺口、减少旧依赖。
第三阶段：按海中鱼巣只读视图、DTO / JSON 和领域服务边界重构。
```

控制面板可以迁移 UI 和人读展示链路，但不得直接裁决机器事实，不得绕过领域服务写节点、主信息、关系、索引、需求、任务、方法、状态、动态或因果结构。

## 3. 确认范围

确认本专项后，只允许登记并执行控制面板 S0 当前旧实现与新项目接入点事实扫描。

S0 只读扫描至少覆盖：

```text
旧项目控制面板源码、HTML / WebView2 / SQL / ADO / 线程入口。
旧控制面板读写哪些结构。
旧控制面板哪些能力必须原样迁移才能保留操作体验。
哪些旧写入必须隔离或改为领域服务入口。
新项目当前 FS-10 只读视图、日志、事件、快照计划的可复用边界。
数据库 / SQL / ADO 路线必须标记为主信息迁移完成后的数据库重构专项，不得在控制面板原样迁移阶段抢跑。
```

## 4. 明确排除项

本专项确认前和 S0 阶段禁止：

```text
不直接复制旧控制面板代码进新项目。
不直接接 SQL / ADO / WebView2 / 窗口线程。
不在主信息迁移完成前启动数据库重构或旧 SQL 投影恢复。
不直接让控制面板写运行期机器事实。
不把 HTML、JSON、显示标题、控制台摘要或线程快照作为机器事实。
不把控制面板迁移扩大为 D455、体素或外设接入。
不修改 C++。
```

后续如进入原样迁移代码阶段，必须另建代码实施切片，明确允许文件、禁止文件、隔离方式和验证方式。
