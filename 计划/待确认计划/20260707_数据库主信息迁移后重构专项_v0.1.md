## 2. 策略定位

数据库必须重构，但必须排在主信息迁移完成之后。

当前 `S5 主信息迁移收口验收` 只证明文档收口和门禁一致，不等于旧主信息字段或旧能力迁移完成。因此本专项当前只能作为后置计划登记，不能进入数据库代码实施。

## 3. 顺序门禁

数据库重构必须等待以下条件同时满足：

```text
主信息迁移完成声明具备六类证据：输入入口、核心处理、结构承载、输出变化、调用路径、验证方式。
旧主信息字段的保留 / 改写 / 删除 / 暂缓口径已在规范和实施记录中闭合。
运行期机器事实仍以内存节点仓库、主信息仓库、关系仓库、索引仓库和领域服务为准。
FS-09 事件日志 / 仓库快照格式与恢复拒绝矩阵已有确认产物。
控制面板原样迁移专项不得绕过数据库后置门禁直接恢复旧 SQL 事实入口。
```

## 4. 确认范围

确认本专项后，只允许登记数据库重构后置门禁，不进入 S0 扫描或代码实施。

后续当主信息迁移真正完成后，再另建：

```text
数据库重构 S0 当前旧 SQL / ADO / 快照 / 恢复事实扫描。

```

## 5. 明确排除项

```text

当前不让控制面板直接读写运行期数据库事实。

当前不修改 C++。
```

## 6. 后续目标结构

| 目标 | 口径 |
| --- | --- |
| 数据库快照 | 只作为运行期仓库快照、审计和恢复材料 |
| 事件日志 | 只作为结构变化事件记录和后续恢复材料候选 |
| SQL 只读投影 | 后续可为控制面板提供人读查询材料，但不裁决机器事实 |
| 恢复入口 | 必须先通过完整拒绝矩阵，不得恢复半结构 |
| 主信息字段 | 不恢复旧字段合集，只映射已确认的新主信息结构 |
