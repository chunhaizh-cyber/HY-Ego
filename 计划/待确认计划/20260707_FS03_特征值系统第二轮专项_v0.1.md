## 1. 任务定位

本计划承接已确认的 `实施记录/20260706_FS03_特征特征值逻辑迁移包归并表.md`，目标是把第一轮明确暂缓的特征值系统能力转成后续可执行路线。

本计划是确认对象。它不迁移旧函数体，不新增旧特征主信息、旧抽象特征主信息或旧特征值主信息字段，不直接进入 C++ 实施。代码实施必须另建切片，并再次确认允许文件、禁止文件和验证方式。

FS-03 第一轮已经完成并验证的范围是：


本计划只处理第二轮路线，不改动第一轮已验证边界。

## 5. 设计条款清单

### 必须实现的路线约束

```text
Vec 原始值容器必须有稳定身份，不得用旧指针或旧 VecU 句柄代替。
内容哈希只能缩小候选范围，不能替代原始值比较。
命中次数、最后命中、稳态列表和近帧候选只能作为非权威缓存或候选材料。
值域匹配、复合比较和差异度只能输出候选或局部判断，不直接裁决需求满足、任务完成或方法成功。
序列化和恢复材料必须服务后续事件日志 / 快照恢复专项，不得恢复旧主信息缓存字段为权威事实。
所有对外特征值访问仍必须经特征服务。
```

### 禁止实现

```text
不直接实现 Vec 原始值容器代码。
不直接实现内容哈希、命中统计或哈希候选缓存。
不直接实现值域匹配、复合比较、稳态或区间升格。
不直接实现序列化、反序列化或恢复。
不接 D455 / 体素 / 外设。
不接 SQL / 控制面板 / WebView2。
```

### 后续待补

```text
Vec 原始值容器详细设计。
特征值内容哈希和非权威缓存详细设计。
特征值域与复合比较详细设计。
稳态和区间升格详细设计。
特征值序列化恢复与事件日志 / 快照边界设计。
```
