## 2. 确认范围

本计划只确认一件事：

```text
执行窗口是否可以做一次 S0 只读复核，检查当前新项目是否已有可复用公共函数入口。
```

复核目标：

```text
列出当前已有公共 / 辅助入口。
按纯通用、仓库内部、领域服务、流程编排、禁止候选五类归属。
标明是否已有同等功能入口。
标明是否存在重复、越权、命名或归属不清风险。
形成只读复核断点清单。
```

## 3. 明确排除项

本计划不允许：

```text
修改 C++。
新增公共工具层。
迁移旧 D:\鱼巢 函数体。
复制旧 全局共享函数类.ixx。
新增公用函数文件。
修改工程文件。
登记可执行队列。
把 S0 只读复核说成公用函数工具层已实现。
```

## 6. 实施步骤

### S0-1 读取规则

读取公用函数规则规范、后续实施门禁信息数据和函数功能说明文档。

### S0-2 当前入口扫描

扫描当前新项目已有函数入口，重点识别：

```text
句柄轻量辅助
日志系统辅助
容错检查宏入口
仓库内部已加锁辅助
仓库公开读写入口
领域服务公开入口
领域服务私有辅助
入口.cpp 局部验收辅助
```

### S0-3 分类归属

按以下五类归属：

```text
纯通用候选
仓库内部辅助
领域服务入口
流程编排 / 局部验收辅助
禁止提升 / 排除候选
```

### S0-4 风险判断

判断是否存在：

```text
已有同等功能入口但后续可能重复新建。
领域结构辅助被误认为纯通用工具。
已加锁辅助被外部调用的风险。
只读快照或候选读取返回可写内部引用的风险。
日志 / 显示 / 诊断辅助被误认为机器事实入口的风险。
```

### S0-5 产物生成

生成：

```text
实施记录/20260708_公用函数规则当前入口只读复核_Codex断点清单.md
```

该断点清单只记录事实和分类结论，不生成代码实施许可。
