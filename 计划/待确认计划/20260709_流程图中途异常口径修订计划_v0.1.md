## 2. 确认范围

确认后允许执行以下文档治理动作：

```text
按“中途在某种条件下返回非预期结果”口径修订流程图 Markdown。
同步修订对应 HTML 渲染产物。
生成修订断点清单。
同步计划索引、项目记忆和待确认问题状态。
```

本计划第一轮只处理统计表中 P0 和 P1 项。若修订时发现对应详细设计仍沿用旧“失败返回 / 失败收口”普通分支口径，另建第二轮详细设计同步修订计划。

## 3. 明确排除项

本计划不允许：

```text
修改 C++ 代码。
运行 C++ 构建。
确认或登记代码实施队列。
新增事务层、错误结果结构、日志结构、弹窗结构或机器事实字段。
把日志、回执、控制台输出、显示材料或 HTML 文本作为机器事实。
宣称旧项目能力迁移完成。
宣称任务失败、方法失败、需求不满足、需求结算或稳定因果结论已实现。
接入 SQL、ADO、WebView、控制面板真实 UI、D455、体素或外设。
```

## 4. 目标结构

本计划只修订文档表达，不改变机器结构。

文档层目标结构为：

```text
入口拒绝：写入前拒绝，结构不变化。
只读候选为空：非权威材料缺失，不裁决业务事实。
人读 / 请求回执：只作为材料或诊断，不裁决机器事实。
中途异常：前置通过后返回非预期结果，停止依赖路径。
逻辑错误：入口已排除错误输入后内部结果仍不符合预期，必须追根因。
事务隔离缺口：多步写入未证明回滚、失效隔离或半结构不可读，不得完成声明。
```

## 7. 实施步骤

### S0 状态复核

读取本计划、统计表、计划索引和项目记忆，确认本计划仍为待确认状态。

### S1 术语统一

统一以下术语：

```text
入口拒绝：仅用于写入前拒绝。
中途异常：用于前置通过后返回非预期结果。
逻辑错误：用于写入后读回不符或内部结果不符合预期。
事务隔离缺口：用于未证明完整回滚、失效隔离或半结构不可读的多步写入。
人读 / 请求回执：用于消息或显示材料，不裁决机器事实。
```

### S2 P0 流程图修订

修订统计表中 P0 项，使每个中途失败分支都满足：

```text
不作为正常业务分支。
不继续推进依赖路径。
不形成机器事实。
不得靠日志、回执、显示或控制台文本修复。
必须登记为逻辑错误、事务隔离缺口或待确认问题。
```

### S3 P1 流程图小修

修订统计表中 P1 项，使回执、阻断和后续提交失败节点不被误读为业务事实裁决。

### S4 HTML 同步

同步更新对应 HTML 渲染产物，确保 Markdown 与 HTML 语义一致。

### S5 断点清单和状态同步

生成实施记录：

```text
实施记录/20260709_流程图中途异常口径修订_Codex断点清单.md
```

并同步：

```text

```


人工验收：

```text
P0 项中，前置通过后的非预期返回不得再只写成普通失败返回。
P1 项中，回执 / 阻断 / 后续提交失败不得裁决任务失败、方法失败、需求不满足或机器事实。
可保留项不被误改成中途异常。
```

## 9. 完成声明边界

本计划完成后最多只能声明：

```text
流程图中途异常口径已修订。
Markdown 与 HTML 流程图语义已同步。
对应断点清单已生成。
计划索引和项目记忆状态已同步。
```

不得声明：

```text
代码已修改。
代码已编译。
入口前置拒绝已重新验证。
事务回滚已实现。
半结构不可读已由代码证明。
旧项目能力已迁移。
完整需求 / 任务 / 方法 / 动作 / 因果 / 结算闭环已完成。
```
