## 2. 确认范围

本计划确认后，只确认以下路线：

```text
以结构化消息作为线程通信语义。
以有界队列作为消息交付工程承载。
按运行宿主线程、任务管理线程、任务工作线程、缓存统计线程、事件日志线程和外设采样材料线程拆分后续实施。
线程只做调度、搬运、等待和耗时计算；不作为动作来源或机器事实来源。
权威写入必须回到领域服务或仓库正式入口复核。
```

本计划确认后，第一步只允许登记 THREAD-S0 当前代码事实扫描。任何 C++ 改动仍必须另建独立代码实施切片并确认允许文件、禁止文件和验证方式。

## 3. 明确排除项

```text
不直接实现自我线程。
不宣称自我循环、自我苏醒或初步成熟完成。
不让线程、Tick、锁、等待事件、日志标签、控制台命令成为动作来源。
不让消息或队列承载机器事实。
不让任务工作线程绕过领域服务直接写节点仓库、主信息仓库、关系仓库或索引仓库。
不接 SQL、ADO、WebView、真实控制面板 UI、D455、体素或真实外设。
不把外设采样材料直接写成世界事实。
不实现 lock-free 队列，除非后续性能专项另行确认。
不把本计划合并为一个大代码提交。
```

## 4. 目标服务 / 目标结构

| 目标 | 第一轮角色 |
| --- | --- |
| 运行宿主线程 | 生命周期、停止收口、调度周期消息 |
| 任务管理线程 | 读取待执行任务材料、生成任务执行请求、消费回执候选 |
| 任务工作线程 | 消费任务执行请求、调用领域服务、生成结果回执 |
| 缓存统计线程 | 消费缓存刷新消息、维护非权威缓存材料 |
| 事件日志线程 | 消费事件日志消息、写人读审计和恢复候选材料 |
| 外设采样材料线程 | 第一轮只产出外部材料消息，不接真实外设 |
| 消息结构 | 请求、材料、回执、停止、缓存刷新和事件日志消息 |
| 有界队列 | 队列满拒绝、停止唤醒、无悬挂事实读取 |

### S1 消息类型与有界队列基础壳

建立结构化消息和有界阻塞队列第一轮壳。只验证消息不承载事实、队列满返回局部失败、停止消息可唤醒等待。

### S2 运行宿主线程生命周期壳

建立运行宿主线程启动、停止、join 和调度周期消息，不写业务事实，不声明自我线程。

### S3 任务管理线程和任务工作线程第一轮

任务管理线程只读取任务候选和生成执行请求。任务工作线程只调用领域服务入口和生成回执。任务完成、需求满足和动态因果仍由领域服务复核。

### S4 缓存统计线程和事件日志线程第一轮

缓存统计线程只维护非权威缓存材料。事件日志线程只写审计和恢复候选材料。两者失败不得改变权威结构。

### S5 外设采样材料线程壳

只实现模拟或空采样材料消息，证明外部材料不会直接入事实。不接 D455、体素或真实外设。

### S6 并发验收和收口

验证停止收口、队列满、并发消费、重复消息、消息过期、缓存清空、事件日志失败和外部材料拒绝路径。
