## 5. 目标服务 / 目标结构

| 目标 | 第一轮角色 |
| --- | --- |
| 需求服务 | 提供目标状态需求和结算入口 |
| 任务服务 | 承接需求、读取任务候选、写任务回执和完成状态 |
| 方法服务 | 读取方法候选、写任务选择方法关系、生成动作请求材料 |
| 动态服务 | 记录实例动态和状态变化动态材料 |
| 因果服务 | 记录轻量因果候选，不生成稳定因果结论 |
| 统计服务 | 刷新非权威缓存和统计材料 |
| 日志系统 | 写事件日志审计材料，不裁决运行期事实 |
| 显示服务 | 输出只读显示材料，不写业务事实 |
| 运行宿主 / 队列 | 调度闭环消息和回执，不成为动作来源 |


### S1 闭环样例输入和证据矩阵

建立一份固定闭环样例，列出需求目标状态、任务承接材料、方法候选材料、动作请求材料、动态证据、任务回执、结算状态、缓存观察、事件日志和显示材料的读回证据。

### S2 需求到任务承接到方法选择串联

把需求目标状态、任务承接材料、方法候选材料和任务选择方法关系串为一个可复核路径。不得实现自动选择算法或候选排序。

### S3 方法执行动作请求到动态记录串联

把任务选择方法、动作入口、特征状态材料写入请求和动态证据记录串为一个可复核路径。不得把工作线程或动作名文本作为动作来源。

### S4 动态到任务回执到需求结算串联

把动态证据、任务实际结果状态、任务完成状态和需求结算状态串为一个可复核路径。需求满足必须由需求服务和状态材料复核。

### S5 非权威缓存、事件日志和只读显示材料观察串联

在权威结算完成后刷新非权威缓存、写事件日志审计材料、更新显示服务只读材料。三者清空、失败或关闭不得改变权威事实。

### S6 闭环验收和禁止宣称扫描

执行构建、入口样例、拒绝路径、并发消息边界和排除项扫描，形成完成声明所需六类证据。
