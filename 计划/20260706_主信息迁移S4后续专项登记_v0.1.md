## 0. 定位

本计划只登记主信息字段确认后分流出去的后续专项。

本计划不是任务系统、方法系统、动作入口或缓存统计系统的详细设计，不允许在本计划中代写专项设计。

承接依据：

```text

规范/主信息系统规范.md
规范/详细设计/主信息系统详细设计.md
```

执行边界：

```text
只登记专项、边界、前置文件和禁止迁移项。
不新增主信息字段。
不迁移旧项目主信息类。
不迁移旧函数体。
不把后续专项未确认字段先塞入统一主信息值容器。
```

## 2. 专项门禁

### 2.1 任务系统专项门禁

进入任务系统节点身份专项前必须回答：

```text
任务节点种类由节点类型表达，还是由任务关系表达。
任务虚拟存在、任务场景、目标状态和结果状态分别由哪个服务写入。
任务创建、启动、完成是否形成任务事件、状态节点或事件日志记录。
哪些时间是机器事实，哪些只做人读事件日志。
任务服务可以直接依赖哪些基础信息服务。
```

未确认前禁止：

```text
新增任务主信息字段。
用 I64 或文本字段表示任务节点种类。
把创建 / 启动 / 完成时间直接写入主信息值容器。
绕过存在、场景、特征、状态、动态和因果服务写任务结构。
```

### 2.2 方法系统和动作入口专项门禁

进入方法系统或动作入口专项前必须回答：

```text
方法首节点、条件节点、结果节点是否是独立节点类型或方法关系角色。
方法与任务的授权关系、承接关系和任务方法关系如何追溯。
动作句柄是否为稳定结构身份，谁创建，谁读取，如何失效。
动作执行后如何形成动态来源候选。
方法执行如何通过特征服务请求修改特征状态材料。
```

未确认前禁止：

```text
恢复方法执行前临时授权状态节点。
把动作句柄写成主信息裸字段。
用函数名、字符串、控制台输出或日志作为动作入口身份。
让方法服务直接访问特征值服务。
```

### 2.3 非权威缓存和统计专项门禁

进入缓存统计专项前必须回答：

```text
缓存项用途是什么。
缓存项写入方是谁。
缓存项读取方是谁。
缓存项失效和重建规则是什么。
清空缓存后哪些事实仍必须可由权威结构重建。
缓存是否只服务索引、去重、诊断或性能。
```

未确认前禁止：

```text
把缓存项作为业务事实。
把命中次数、最后命中、内容哈希或结构统计作为需求满足、任务完成、方法成功依据。
把非权威摘要放入主信息字段并被领域服务当作权威判断来源。
```

## 3. 建议执行顺序

后续专项建议按以下顺序推进：

```text
1. 建立任务系统规范和详细设计，覆盖 MI-T2、MI-T3、MI-T4。
2. 建立方法系统规范和详细设计，覆盖 MI-M1、MI-M3、MI-M4、MI-M5。
3. 建立动作入口规范，覆盖 MI-M2，并与动态来源口径对应。
4. 建立非权威缓存和统计专项，覆盖 MI-V2、MI-BI2、MI-D3。
5. 回到主信息迁移 S5 收口验收，证明本轮主信息字段没有被错误迁移。
```

执行顺序只是建议，不代表专项已确认或已实现。
