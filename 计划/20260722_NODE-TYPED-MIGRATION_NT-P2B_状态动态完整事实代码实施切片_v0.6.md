## 3. 状态、动态记录与结果 ABI

`状态动态类型化记录.数据.h` 唯一定义：

```text
状态记录模式 uint32：
  0=未指定，1=观察，2=动作结果，3=治理事实

动态种类 uint32：
  0=未指定，1=状态迁移动能，2=动作致变，
  3=观察动态，4=外部事件动态，5=聚合动态

记录状态 uint32：
  复用4010：0=无效，1=有效，2=已删除，3=已失效
```

`状态域记录` 字段顺序固定为：

```text
节点句柄 所属状态
uint32 格式版本
uint64 记录版本
记录状态 状态
uint64 发生时间
optional<uint64> 接收时间
状态记录模式 记录模式
```

`动态域记录` 字段顺序固定为：

```text
节点句柄 所属动态
uint32 格式版本
uint64 记录版本
记录状态 状态
uint64 发生时间
动态种类 种类
uint64 聚合规则版本
```

格式版本固定为 1。记录身份是“所属节点完整句柄 + 记录版本”；记录不得复制任何关系端点。完整事实请求、具名关系证据、读取组、退役规格 / 材料和服务签名精确读取 P2 第 3.4 节，不得恢复匿名角色 vector 或建立第二结果状态。

状态 / 动态记录仓各自具有非零 `uint64 结构版本`，初值 1；每次本仓真实 `完成发布()` 恰加一。幂等、拒绝、准备、确认、撤销、读取和冻结不推进。达到上限是内部不一致，不得回绕。

## 4. `状态动态记录退役参与包/v1`

### 4.1 类型和签名

`参与者.状态动态类型化记录.ixx` 唯一拥有两个内部记录事务参与者、两个 move-only 包和执行结果；`数据操作.节点直接状态动态.ixx` 唯一拥有窄退役提供者。ABI 固定为：

```text
退役参与包形成状态 uint32：
  1=已形成，2=合法未找到，3=入口拒绝，4=许可竞争，
  5=版本漂移，6=资源失败，0x8000=内部不一致
```

完整签名固定为：

```cpp
状态记录退役参与包形成结果
状态动态记录退役提供者::形成状态记录退役参与包(
    const 状态退役规格&) const;

动态记录退役参与包形成结果
状态动态记录退役提供者::形成动态记录退役参与包(
    const 动态退役规格&) const;

状态记录退役执行结果 状态记录退役参与包::执行一次(
    节点直接身份结构写入执行器&,
    const std::function<void(节点直接身份结构写入会话&)>&);

动态记录退役执行结果 动态记录退役参与包::执行一次(
    节点直接身份结构写入执行器&,
    const std::function<void(节点直接身份结构写入会话&)>&);

状态动态记录退役提供者
节点直接状态动态数据操作::形成记录退役提供者() noexcept;
```

形成结果只在 `已形成` 时携带对应包。包无 public 可写字段，删除复制构造 / 赋值，允许 noexcept 移动；移动后源包无效。`有效()` 只表示内部参与者尚由本包唯一拥有且尚未执行。包不得导出或返回参与者指针、仓、候选、锁、令牌、许可或可写引用。

执行结果精确包含：

```text
节点直接身份结构写入结果 权威结构结果
合同结果状态 发布后记录读回状态
合同结果原因 发布后记录读回原因
optional<状态记录退役审计材料或动态记录退役审计材料> 记录审计
```

审计材料依次包含目标句柄、退役前记录版本 / 状态、退役后记录版本 / 状态、发布后结构版本。只有核心 `权威结构结果=已提交` 且发布后记录精确读回时才携带审计。第二次 `执行一次` 固定返回核心 `入口拒绝`、无审计、零变化。

### 4.2 退役推进、锁序与 #346

形成包只读当前有效记录并冻结写前版本与全部非拓扑字段。无效规格、无当前记录、版本漂移、许可竞争、资源失败和坏项按形成 ABI 分类。

节点退役固定形成新记录版本：

```text
退役后记录版本 = 退役前记录版本 + 1
退役前记录状态 = 有效
退役后记录状态 = 已删除
非拓扑字段 = 精确保持
```

`已失效` 只用于另有正式换代规则的旧记录，不得由节点删除使用。包内部调用核心执行器已有单参与者模板或 span 重载；锁序固定为“核心事务域独占许可 -> 本对象所属状态仓或动态仓独占锁”。一次退役只涉及一个记录仓。

准备阶段复核同一有效写前记录并建立不可见候选；确认只读回候选；发布才追加新记录版本并推进本仓结构版本一次；撤销丢弃候选并保持已发布记录、当前指针、结构版本完全不变。撤销不能证明前态时由核心执行器隔离事务域。

#346 公开重载保持：

```cpp
节点直接对象退役结果 退役节点直接对象(const 状态退役规格&);
节点直接对象退役结果 退役节点直接对象(const 动态退役规格&);
```

#346 私有 `适配.状态动态退役` 只借用 `状态动态记录退役提供者`。它形成包后，把关系失效、索引移除、节点删除和请求提交登记在包的单次回调中；内部参与者与核心会话统一准备、确认、发布 / 撤销。发布后记录读回失败保留核心 `已提交`，另报 `内部错误` 并停止依赖路径，不得重新撤销或否认权威退役。

## 5. `状态动态冻结提供者/v1`

### 5.1 DTO 和签名

```cpp
struct 状态动态同代次冻结材料 final {
    节点直接统一冻结见证 见证;
    std::uint64_t 状态记录结构版本;
    std::uint64_t 动态记录结构版本;
    std::uint64_t 状态记录数量;
    std::uint64_t 动态记录数量;
    std::vector<状态域记录> 状态记录组;
    std::vector<动态域记录> 动态记录组;
};

enum class 状态动态冻结状态 : std::uint32_t {
    已形成 = 1,
    入口拒绝 = 2,
    资源失败 = 3,
    内部不一致 = 0x8000
};

struct 状态动态同代次冻结结果 final {
    状态动态冻结状态 状态;
    std::optional<状态动态同代次冻结材料> 材料;
};

状态动态同代次冻结结果
状态动态冻结提供者::复制同代次状态动态全量(
    const 节点直接统一冻结许可&) const;

状态动态冻结提供者
节点直接状态动态数据操作::形成冻结提供者() const noexcept;
```

提供者删除复制、允许 noexcept 移动，只借用 `const 节点直接状态动态数据操作`；数据操作、事务域和两仓必须长于提供者。返回结果拥有全部 vector 副本；提供者不得保存许可引用、锁或容器视图。

### 5.2 许可、锁序、二次见证和排序

#349 是唯一统一冻结协调方。P2B 不调用 `取得统一冻结许可`，只接受 #349 已持许可，并先复核：

```text
提供者有效
许可有效
事务域.冻结许可属于本域(许可)
冻结域身份非零
已发布代次非零
冻结规则版本 uint32 = 1
```

不满足返回 `入口拒绝`，材料为空。唯一顺序为：

```text
#349 已持统一许可
-> 状态仓 shared 复制全部已发布版本、结构版本、数量，释放
-> 动态仓 shared 复制全部已发布版本、结构版本、数量，释放
-> 再读许可见证并复核属于本域
-> 状态仓 shared 二次读取结构版本 / 数量，释放
-> 动态仓 shared 二次读取结构版本 / 数量，释放
```

不得同时持有两仓锁，不得先取记录仓再取许可，不得二次取得冻结权。合法空仓仍同时返回两个空 vector 和数量 0。数量覆盖当前、历史、已失效和已删除全部已发布记录版本。

排序键固定为：

```text
状态：(所属状态.仓库编号, 所属状态.节点编号, 所属状态.版本号,
      记录版本, uint32(记录状态))
动态：(所属动态.仓库编号, 所属动态.节点编号, 所属动态.版本号,
      记录版本, uint32(记录状态))
```

两次见证、两仓版本和数量必须完全相等。重复记录身份、格式版本非 1、零记录版本、未知状态、字段值域不合法、历史缺项、当前指针不一致、锁序异常、部分材料或二次复核不等均为 `内部不一致`，材料为空；分配失败为 `资源失败`。#349 负责同一许可内的九段跨段互证，#341 不实现统一协调或段编码。

## 10. 逻辑内返回与内部错误

逻辑内返回：写前无效身份 / 类型 / 版本 / 时间 / 来源 / 角色，特征值不相容，同键异义，合法未找到，许可竞争，参与包形成版本漂移，冻结许可无效 / 错域 / 错规则版本。必须在本步骤第一笔写入前闭合。

内部错误：前置通过后的节点 / 关系 / 记录 / 索引候选异常，记录参与者阶段或撤销不闭合，发布版本 / 状态 / 结构版本不精确，统一许可内见证变化，两仓版本 / 数量变化，重复 / 坏记录，部分冻结材料，发布后读回失败。发布前按依赖逆序撤销，不能证明前态则隔离事务域；发布后保留已发布事实并停止依赖路径。
