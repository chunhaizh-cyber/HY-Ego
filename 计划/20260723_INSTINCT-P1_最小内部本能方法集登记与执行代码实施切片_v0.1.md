# INSTINCT-P1 最小内部本能方法集登记与执行代码实施切片

编号：INSTINCT-P1；路由：#356 / DQ-248；版本：v0.1；日期：2026-07-23

设计状态：范围草案。当前计划状态只读计划索引；#353 未冻结 SELF-C3、最小方法清单与文件所有权前不得实施。

## 目标与依据

依据 3300、5300—5340、6130、8210 和 8310，建立能够被正式任务召回、选择、冻结、执行和验证的最小内部本能方法集。

设计下限包括：同步运行态、改变合法自我特征、进入 / 退出待机、重置时序、结算合法任务结果、故障收口、尝试学习、定向学习和练习方法。#353 必须裁决方法粒度，禁止只登记名称、函数指针或成功 bool。

## 冻结前必须补齐

- 每个方法的稳定身份、内容版本、条件、目标结果、动作、来源、生命周期和验证；
- SELF-C3 / v0.1 的登记规格、执行请求 / 结果和扩展合同；
- 内部方法与外设观察方法的文件 / 注册所有权分离；
- 无候选不自动建方法首、方法结构缺口属于筹办漏检。

## 候选步骤

1. S0 核对当前方法服务、动作入口和节点直接方法结构。
2. 实现最小本能方法定义与不可变版本登记候选。
3. 实现条件 / 结果结构完整性读回和重复登记幂等边界。
4. 实现各方法的窄动作入口，不直接裁决任务完成。
5. 定向验证错误条件、版本漂移、动作失败和执行后实际回读。

## 边界

本切片不拥有通用任务筹办、任务结算、学习晋级、外设方法或生产装配。方法执行成功不等于任务完成或需求满足。
