## 5. 非空备份原子替换合同

### 5.1 唯一替换函数

`run-one-round.ps1` 只能有一个正式状态替换辅助。该辅助同时供产品启动前样本预检和真实轮调用，禁止维护第二套替换实现。每次调用固定执行：

```text
读取并逐键核对当前正式六键状态
生成同目录临时状态路径
生成同目录、非空、按轮唯一的备份路径
拒绝临时路径或备份路径已存在
写入临时六键状态并用同一解析器逐键核对
记录替换前正式状态哈希和值
调用 File.Replace(临时路径, 正式路径, 非空备份路径)
核对临时源已经消失
核对正式路径为目标新六键状态
核对备份路径为替换前旧六键状态且哈希一致
保留备份文件，不在本轮删除或覆盖
```

真实轮备份命名必须包含轮号，例如：

```text
resume-state-before-round-43.bak
```

三个路径必须是 `TASK-RESULT-STABILITY-S2-R5` 同一目录下的完整绝对路径。禁止向 `File.Replace` 传 `$null`、空字符串、相对路径或 R4 路径；禁止用 `Move-Item -Force`、先删除正式文件再移动、复制后删除或其它非同一合同替代原子替换。

### 5.2 启动前真实 API 预检

`-ValidateOnly` 除 v0.4 的 AST、脚本哈希、六键、LF / CRLF 和进程 0 门禁外，必须在可丢弃的 R5 样本文件上调用第 5.1 节同一个替换辅助：

```text
样本正式状态=下一轮43
样本临时状态=下一轮44
样本备份路径=非空且同目录
调用后正式样本=下一轮44
调用后备份样本=替换前下一轮43
调用后临时样本不存在
产品启动数=0
```

预检必须输出 `ATOMIC_REPLACE=通过` 和样本正式 / 备份哈希。任何不一致都在产品启动前停止，不得现场改脚本后继续同一 R5 执行轮。

### 5.3 状态推进与异常边界

一个真实轮只有同时满足以下条件才计为有效：

```text
产品自然退出0
TASK / B3C / B0 = 18 / 12 / 20 且失败0
失败证据门禁满足
摘要已落盘
File.Replace 使用非空同目录备份路径成功
正式六键状态推进到下一轮
备份六键状态等于推进前状态
临时状态消失
脚本最终退出0
产品进程0
```

如果 `File.Replace` 抛异常，或调用后正式状态、备份状态、临时文件任一后验不符，立即停止，不补跑当前轮，不启动后继轮。若异常发生在替换调用返回之后，执行窗口必须同时报告正式状态、备份状态和临时文件三方事实；不得假定状态未推进，也不得自动回滚或删除备份。
