## 4. 实施步骤

### S0：接口复核

比对 #206 候选形状、S2 条件规格和特征服务实际读取 / 比较语义。

### S1：条件策略分流

```text
显式无条件：不读取槽位，返回全部满足。
全部满足：要求至少一个完整条件规格。
未声明、空条件或策略矛盾：不可判定。
```

### S2：逐项特征复判

```text
以请求目标宿主和条件特征定义读取实例槽位。
要求唯一当前特征值。
冻结当前原始版本。
经特征服务完整比较期望值与当前值。
再次复核期望值和当前值版本。
任一明确不等 -> 整体不满足。
任一缺失 / 未知 -> 整体不可判定。
全部相同 -> 全部满足。
```

### S3：并发只读和治理收口

候选间可由外部现有任务工作线程并行调用；服务内部不创建临时线程。补齐 METHOD-RECALL-S5-A01 至 A12。

## 7. 入口拒绝

```text
召回候选不完整或来源版本过期。
方法未登记、已失效或条件策略不可读。
条件规格缺特征定义、期望值或版本。
目标宿主无效。
规则版本不支持。
```

缺槽位、缺当前值、类型不可比和正常版本漂移属于不可判定或材料过期，不得视为满足。

## 8. 失败收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 缺槽位、缺当前值、未知类型、空条件未声明 | 逻辑内返回 | 返回不可判定 |
| 完整比较明确不等 | 逻辑内返回 | 返回不满足 |
| 期望值或当前值版本正常漂移 | 逻辑内返回 | 返回材料过期 |
| 当前版本不变但完整内容复核自相矛盾 | 追根因解决 | 停止该候选 |
| 全部满足结果发布前证据项缺失 | 追根因解决 | 不发布满足结论 |
