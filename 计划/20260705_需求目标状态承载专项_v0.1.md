# 需求目标状态承载专项

编号：HR-001-DV
日期：2026-07-05
状态：v0.2 / 已确认 / DV-1 已验证

## 1. 目标

处理 S0 缺口 2：

```text
需求服务当前把需求目标状态降格为需求节点主信息 I64 值容器。
```

本专项目标是把需求目标从需求节点主信息值容器中移出，改为目标状态节点和关系结构承载。

需求目标是状态，不是 I64。I64 只作为第一轮状态值材料，不作为需求目标对外语义。

## 3. 当前缺口

旧代码事实：

```text
需求服务::创建I64目标需求(...)
-> 创建需求节点主信息
-> 主信息_.写入I64值(主信息句柄, 目标材料)
-> 创建需求节点

需求服务::结算I64目标(...)
-> 从需求节点主信息读取 I64 目标材料
-> 从特征值服务读取当前材料
-> 当前材料 >= 目标材料 时返回 已满足
```

这违反已确认口径：

```text
需求目标是目标状态，不是 I64。
MI-V1 值容器语义只保留在特征系统内部值承载范围内。
需求、任务和方法不得把主信息仓库当通用值容器。
只有特征服务可直接访问特征值服务。
高级服务只允许直接依赖存在、场景、特征、二次特征、状态、动态、因果服务。
```

## 4. 目标结构

第一轮目标结构：

```text
需求节点
-> 引用 目标存在节点
-> 引用 目标特征节点
-> 引用 当前特征值节点
-> 模板 目标状态节点
```

目标状态节点：

```text
由状态服务创建。
状态值 = 第一轮目标状态材料。
不携带发生时间戳。
不是需求结算实例状态。
```

比较方式：

```text
第一轮固定为目标状态满足规则。
当前实现暂以 当前特征状态材料 >= 目标状态值 判断。
任意比较方式后续另建二次特征或比较方式专项。
```

## 6. DV-1 代码原则

```text
需求服务必须通过状态服务创建目标状态节点。
需求服务必须通过特征服务读取当前特征状态材料。
需求服务必须通过关系仓库写目标材料关系。
方法服务必须通过特征服务请求改写特征状态材料。
特征值服务只允许由特征服务直接访问。
旧 `创建I64目标需求` 不得继续作为有效写值入口。
旧 `结算I64目标` 不得继续读取需求节点主信息值容器。
```
