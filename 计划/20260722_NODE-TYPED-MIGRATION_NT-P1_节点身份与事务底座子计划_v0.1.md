# NT-P1 节点身份与事务底座子计划

日期：2026-07-22

版本：v0.1

上级：`#338 / DQ-230 / NODE-TYPED-MIGRATION`

状态：依赖 NT-P0 的流程图、详细设计、知识图谱和精确接口；不可派发

## 目标

让节点记录直接承载稳定主键，并形成领域类型化记录参与者、节点 / 关系 / 记录 / 索引同事务发布、精确撤销、值式冻结和旧句柄拒绝的共享底座。该阶段不迁移具体业务服务，不删除旧仓库物理文件。

## 候选允许文件

- `海中鱼巣/核心/句柄.h`
- `海中鱼巣/核心/节点仓库.h`、`节点仓库.cpp`
- `海中鱼巣/核心/关系仓库.h`、`关系仓库.cpp`
- `海中鱼巣/核心/索引仓库.h`、`索引仓库.cpp`
- `海中鱼巣/核心/会话.结构写入.ixx`
- `海中鱼巣/核心/执行器.结构写入.ixx`
- `海中鱼巣/核心/结果.结构写入.h`
- `海中鱼巣/核心/结构事务接线.数据.h`
- `海中鱼巣/核心/权威冻结材料.数据.h`
- NT-P0 允许的新领域记录通用接口文件、对应核心自检和工程登记。

精确执行计划必须从本候选集中收窄；新增文件先通过代码文件归属规范。

## 禁止文件

- 所有旧兼容领域服务头、线程、界面、SQL、D455 和业务入口；
- `海中鱼巣/核心/主信息仓库.h/.cpp` 的删除或复义；
- 具体特征值、状态、动态、自我业务迁移；
- 启动模式、运行期宿主和恢复发布。

## 入口拒绝和失败收口

拒绝零稳定主键、重复主键、错命名域、旧版本、未知记录类型、错所属节点和无冻结锁序。前置通过后的任一参与者异常按 4040 逆序撤销；无法证明前态则隔离事务域。

## 验证

节点直接主键、版本和状态完整读回；同主键异义零写冲突；节点 / 关系 / 领域记录 / 索引共同确认发布或共同撤销；旧节点无直接主键不能冒充新结构；既有单参与者和多参与者回归不恶化；双配置、完整自检、strict、diff 和白名单检查通过。

## 完成边界

只证明共享结构底座可供领域迁移消费；旧仓库和旧领域入口仍存在，不得宣称全仓迁移或物理退役完成。
