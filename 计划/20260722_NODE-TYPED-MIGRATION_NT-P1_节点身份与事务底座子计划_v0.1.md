## 目标

让节点记录直接承载稳定主键，并形成领域类型化记录参与者、节点 / 关系 / 记录 / 索引同事务发布、精确撤销、值式冻结和旧句柄拒绝的共享底座。该阶段不迁移具体业务服务，不删除旧仓库物理文件。

## 已裁决迁移策略

采用“隔离新结构域 -> P2/P3 只在新域形成对象 -> P4 一次切换默认装配并退役旧域”。禁止同一稳定身份双写、跨域关系、跨域索引、跨域投影、生产默认入口提前持有新域，以及在线热替换。

## 禁止文件

- 所有旧兼容领域服务头、线程、界面、SQL、D455 和业务入口；
- `海中鱼巣/核心/主信息仓库.h/.cpp` 的删除或复义；
- 现行 `节点仓库.h/.cpp`、`关系仓库.h/.cpp`、`索引仓库.h/.cpp`、旧会话 / 执行器的改写；
- 具体特征值、状态、动态、自我业务迁移；
- 启动模式、运行期宿主和恢复发布。

## 入口拒绝和失败收口

拒绝零稳定主键、重复主键、错命名域、旧版本、未知记录类型、错所属节点和无冻结锁序。前置通过后的任一参与者异常按 4040 逆序撤销；无法证明前态则隔离事务域。

## 验证

节点直接主键、版本和状态完整读回；同主键异义零写冲突；节点 / 关系 / 领域记录 / 索引共同确认发布或共同撤销；旧节点无直接主键不能冒充新结构；既有单参与者和多参与者回归不恶化；双配置、完整自检、strict、diff 和白名单检查通过。

## 完成边界

只证明共享结构底座可供领域迁移消费；旧仓库和旧领域入口仍存在，不得宣称全仓迁移或物理退役完成。
