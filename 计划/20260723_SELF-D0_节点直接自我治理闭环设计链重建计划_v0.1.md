## 1. 目标

以现行节点直接身份、正式关系、领域类型化记录、自我内部世界和完整恢复合同，重建能够直接生成施工计划的自我内部治理设计链：

```text
根特征自检
-> 需求生长、路径选择与权重
-> 主需求和任务承接
-> 按任务身份取得唯一筹办执行权
-> 六类筹办闭合结果
-> 最小本能方法选择与冻结
-> 方法执行、实际读回和状态 / 动态 / 因果提交
-> 任务目标验证
-> 需求满足与价值结算
-> 用途观察、学习候选、晋级和不可变新代际
-> 后继回合重新筹办
```

本计划只重建设计依据和接口合同，不把旧流程图、归档详细设计、现行固定安全根路由或自检夹具当作生产设计。

## 3. 已验证事实

1. #339—#352 解决的是节点直接基础结构、类型化记录、恢复和旧域切换，不包含完整业务闭环。
2. 当前生产运行期只发布运行期上下文，不启动持续治理消费者。
3. 自我线程的治理处理器是可选接线；默认入口不能证明真实治理处理器已接通。
4. 当前固定安全根路由直接形成方法首和动作入口，不能代替现行通用召回与“无候选不自动创建方法首”规则。
5. 当前代码没有任务级筹办执行权、学习候选、晋级授权、不可变新代际和后继回合可见性的生产符号证据。
6. 旧自我治理、领域回流和真实代际闭环详细设计已经归档或建立在退出结构上，不能直接生成施工计划。
7. 当前有效的任务筹办权详细设计只覆盖唯一占用，不覆盖六类筹办结果和完整闭环。

## 6. 最小本能方法集设计下限

设计至少覆盖以下能力族，并裁决每项是独立方法还是同一方法的强类型动作：

- 同步运行态；
- 改变合法自我特征；
- 进入待机与退出待机；
- 重置时序；
- 结算合法任务结果；
- 故障收口；
- 尝试学习；
- 定向学习；
- 练习方法。

方法不得只有名称或函数；必须有稳定方法身份、适用条件、目标结果、动作入口、内容版本、来源、可执行性、回读和失败边界。
