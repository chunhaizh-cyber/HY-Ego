## 1. 目标

基于 #233 完整模式键和按片段序号排列窗口的显式观察回合，形成第一版因果概率候选：

```text
原因缺席 -> 不计机会
原因出现、之后结果出现 -> 正结果
原因出现、之后结果未出现 -> 负结果
机会 / 结果确定性集合 -> Q10000 概率
```

概率只表达当前证据下的可能性，不要求可靠性证明，不自动升格为稳定因果，也不写任务、需求、方法、统计或概念。

## 4. 实施步骤

### S0：实际接口复核

读取 #233 实施记录和正式模式 / 回合接口，逐项核对结构键、规则版本、来源当前性、回合 / 片段身份、片段顺序和动态跨窗口去重边界。若只能按哈希或动态句柄判断模式出现，停止并退回设计。

### S1：值式请求和服务边界

新增因果模式服务 / 算法，保持现有 `记录因果引用(来源动态)` 语义不变。请求显式携带原因键、结果键、规则版本和非空去重回合组；服务负责当前性 / 返回分类，算法只处理不可变完整材料。

### S2：机会与结果统计

逐回合按片段序号全量比较完整窗口键。原因零次不计机会；原因窗口一次且后续结果窗口一次计正结果；原因一次且无后续结果计负结果。原因 / 结果键相同、多次出现、结果不在原因后、片段序号重复、动态跨窗口重复或回合重复使整个请求逻辑拒绝，不进入动态步内部搜索或任意配对。

### S3：概率候选

使用检查溢出的整数计算 `floor(结果数 * 10000 / 机会数)`。机会为零或规模溢出逻辑返回；安全计数后结果数大于机会数或概率越界追根因。返回正负证据和规则版本，不设阈值。

### S4：模块、自检与收口

建立 `服务.因果模式`、`算法.因果模式` 和独立 `自检.因果模式` 三个真模块。自检承载 A01-A16、无原因、全正、全负、中间概率、结果在前、多次歧义、重复回合、溢出边界、来源过期和并发确定性；入口只登记最终回归阶段 650 无捕获回调。完成工程登记、实施 / 验收记录、归档、治理、精确提交和推送。

## 7. 入口拒绝

```text
原因 / 结果键为空、相同、结构不完整、规则版本为零 / 不支持或两键版本不同。
回合组为空、回合 / 片段身份重复、片段序号无效、动态跨窗口重复、来源过期或与模式键规则版本不一致。
原因 / 结果出现判断需要哈希相等代替完整结构比较。
同回合原因多次、结果多次、结果只在原因前或存在配对歧义。
回合数 / 计数乘法超过明确安全边界。
```

## 8. 失败收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| #233 接口漂移或未完成 | 依赖门控返回 | 不改代码，退回设计 |
| 外部键 / 版本 / 回合无效或来源过期 | 逻辑内返回 | 不形成统计，不写结构 |
| 原因 / 结果键相同、多次出现、结果不在原因后或回合 / 窗口重复 | 逻辑内返回 | 要求重新分段，不进入动态步内部搜索或任意配对 |

| 安全计数后结果数大于机会数或概率越界 | 追根因解决 | 停止返回，不截断或日志兜底 |
