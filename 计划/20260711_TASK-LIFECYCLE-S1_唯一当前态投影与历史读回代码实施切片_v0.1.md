## 4. 实施步骤

### S0：实际接口复核

读取 #217 实施记录和正式接口，核对事务域编号、令牌传递、主信息 / 节点 / 关系创建与删除、关系失效和共享读取调用形状；读取当前关系枚举、任务服务、需求服务、存在服务和状态服务。逐项复核需求服务既有私有令牌辅助可支撑两个公开只读重载，所有构造点可由默认空接线兼容，连接态自检可向四服务注入同一接线。若关系 13 已改变、需求只读重载需要修改需求写链或 #213 命名路径、仓库令牌删除不足以精确清理本次候选、560 已占用或仍需触碰禁止文件，停止并退回设计。

### S1：关系类型与值式 DTO

只在枚举尾部追加 `任务生命周期 = 14` 并同步关系类型准入。准入必须按实际枚举项判断，不得用 `1..14` 连续区间放过尚未实现的保留槽位。新增当前材料与历史项，所有长期身份使用完整稳定句柄；版本使用关系顺序号，不从时间戳或节点编号推导。

### S2：创建发布

任务创建自检取得一次 #217 同域独占许可并把同一令牌传给任务服务。任务服务先验证令牌，再通过需求服务两个只读令牌重载重读需求有效性与承接材料；材料无效、为空或与请求漂移时，在第一笔任务写入前逻辑内返回。复核通过后才创建任务节点 / 主信息并调用存在服务令牌入口；存在服务验证独占令牌，创建实际存在、来源引用并通过令牌读回。随后状态服务验证同一令牌创建已创建实例状态，任务服务继续写承接关系和版本 1 专用关系。发布前完整复核来源需求、任务虚拟存在、场景、状态值和关系端点；读者只能观察创建前无任务或创建后完整当前态。

任一层进入写入后内部不及预期，必须在同一令牌内按本次句柄精确逆序清理：先关系、后节点、再主信息；任务服务负责顶层组合顺序，存在 / 状态服务负责自身候选。清理后公开读取必须回到完整前态；清理失败继续追根因并停止，不扫描修复。

### S3：当前与历史读取

当前读取只接受恰好一条有效专用关系；历史读取合并有效与已失效关系，按生命周期版本确定性排序并逐条读回状态材料。缺失、多条、版本为零 / 重复或材料矛盾进入追根因，不猜测当前态。

### S4：旧入口收口与回归

把任意 I64 状态写辅助收口为任务服务内部实现；新增 `自检.任务生命周期初态.ixx` 真模块承载 A01-A13、并发创建 / 读取和逐项输出，入口只以最终回归阶段 560 无捕获回调登记；不删除既有普通引用历史。完成回归、治理归档、提交和推送。

## JY-292 存在服务同令牌创建修订

1. 任务虚拟存在是任务系统规范确认的正式结构，不采用“不创建新虚拟存在”的替代合同。
2. `存在服务.h` 从原允许范围外定向补入允许范围，新增默认空接线构造兼容和四个最小令牌重载：创建实际存在、创建虚拟存在、读取存在、存在有效性。
3. #221 连接态调用链固定为“自检独占许可 -> 任务服务 -> 存在服务 -> 状态服务 -> 仓库”，全链验证 / 下传同一令牌，不嵌套取得许可。
4. 令牌写链必须记录本次句柄并在失败时许可内精确逆序清理；不得用外部扫描、日志、显示、空读或事后修复兜底。
5. 普通无令牌入口继续兼容既有主装配和隔离夹具，但不用于 A09-A11；#248 后续主装配接域仍负责生产服务同接线。

## JY-294 需求承接材料同令牌复核修订

1. `需求服务.h` 从禁止文件定向移入允许范围，只增加 `需求是否有效(..., const 结构事务令牌&) const` 与 `读取需求承接材料(..., const 结构事务令牌&) const` 两个公开只读重载。
2. 两个重载复用既有私有 `节点类型匹配_令牌`、`读取唯一需求目标_令牌` 和同令牌仓库读取辅助；不得修改需求写入口、#213 命名专用路径或无令牌兼容语义。
3. #221 固定调用链改为“自检独占许可 -> 任务 -> 需求只读 -> 存在 -> 状态 -> 仓库”，全链不嵌套取得许可。
4. 需求失效、承接材料为空或许可内材料漂移必须在第一笔任务写入前逻辑内返回；许可前值式快照不能代替权威复核。
5. 任务服务不得直接读取需求仓库，令牌不得进入需求承接 DTO；如两个只读重载不能仅靠当前允许文件形成，执行窗口再次退回设计。
