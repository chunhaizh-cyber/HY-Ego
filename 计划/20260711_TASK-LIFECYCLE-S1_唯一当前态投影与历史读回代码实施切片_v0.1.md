## 4. 实施步骤

### S0：实际接口复核

读取 #217 实施记录和正式接口，核对事务域编号、令牌传递、关系创建 / 失效和共享读取调用形状；读取当前关系枚举、任务服务和状态服务。若关系 13 已改变、#217 不支持本调用链或任务创建需触碰禁止文件，停止并退回设计。

### S1：关系类型与值式 DTO

只在枚举尾部追加 `任务生命周期 = 14` 并同步关系类型准入。准入必须按实际枚举项判断，不得用 `1..14` 连续区间放过尚未实现的保留槽位。新增当前材料与历史项，所有长期身份使用完整稳定句柄；版本使用关系顺序号，不从时间戳或节点编号推导。

### S2：创建发布

任务创建取得一次 #217 同域独占许可，显式下传令牌，创建已创建实例状态和版本 1 专用关系。发布前完整复核来源需求、任务虚拟存在、场景、状态值和关系端点；读者只能观察创建前无任务或创建后完整当前态。

### S3：当前与历史读取

当前读取只接受恰好一条有效专用关系；历史读取合并有效与已失效关系，按生命周期版本确定性排序并逐条读回状态材料。缺失、多条、版本为零 / 重复或材料矛盾进入追根因，不猜测当前态。

### S4：旧入口收口与回归

把任意 I64 状态写辅助收口为任务服务内部实现；新增 `自检.任务生命周期初态.ixx` 真模块承载 A01-A13、并发创建 / 读取和逐项输出，入口只以最终回归阶段 560 无捕获回调登记；不删除既有普通引用历史。完成回归、治理归档、提交和推送。

## 8. 失败收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| #217 未完成或实际接口漂移 | 依赖门控返回 | 不改代码，退回设计修订 |
| 无效输入、已有当前关系或错误事务域 | 逻辑内返回 | 不写状态或关系 |
| 当前 / 历史只读请求对象无效 | 逻辑内返回 | 返回空值式结果，不补写 |
| 创建后状态或关系读回不及预期 | 追根因解决 | 不发布任务创建结果，停止依赖路径 |
| 当前关系缺失、多条或版本 / 状态矛盾 | 追根因解决 | 不用旧普通引用或时间戳兜底 |
