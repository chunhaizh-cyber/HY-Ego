## 1. 目标

把合法处于筹办中的任务接到 #203-#208 正式方法召回链，并由任务服务独占提交唯一当前方法选择：

```text
任务当前态与双向需求承接授权
-> 两条授权关系完整句柄 / 版本
-> 方法召回 / 复判 / 排序只读建议
-> 方法召回侧复核完整批次并适配任务服务自有扁平请求
-> 任务服务重读全部稳定句柄 / 版本
-> 唯一顺序号 30 当前选择关系 / 版本
-> 精确当前选择动作桥
```

本计划是 #209 扫描要求的明确后继，也是 #214 命名候选绑定所需任务授权 / 当前态 / 选择提交精确前置。本切片不执行方法、不生成回执、不写任务结果或需求结算。

## 4. 实施步骤

### S0：#203-#209 实际接口复核

实施记录必须列出 #203-#209 各提交号、登记 / 规格 / 索引 / 召回 / 复判 / 排序 / 建议函数签名，以及 #209 对授权关系、头文件依赖、任务状态、选择和动作桥的正式结论。再读取 #221 / #222 实际当前态与迁移接口并逐项映射本计划假定字段。任一缺失或漂移时不改代码并退回设计。

### S1：双向授权与任务自有 DTO

任务服务形成 `任务承接授权材料`，同时返回需求到任务归属和任务到需求引用两条关系的完整句柄 / 版本，并证明同一任务 / 需求 / 目标端点。只为精确当前筹办中关系 / 版本形成材料，不持久化第二份授权。

在 `任务服务.h` 定义扁平 `任务方法选择请求` 与 `当前任务方法选择材料`。DTO 只含稳定节点 / 关系句柄、版本、批次身份、幂等材料和当前阶段，不含 `方法选择建议材料`、候选组、复判材料或其它方法召回类型；任务服务不得 include `方法召回服务.h` 或 `方法服务.h`。

### S2：只读建议复核与单向适配

方法召回侧沿既有 `方法召回服务.h -> 任务服务.h` 方向，重读完整 S6 批次、候选身份、索引发布版本、登记 / 生命周期 / 结果规格 / 条件 / 上下文 / 动作来源。只有来源当前的 `唯一可建议` 可适配为任务服务请求；无候选、过期、语义并列和没有独立权威来源的 `策略唯一` 逻辑内返回。不得裁剪批次来源，不得按句柄、容器或线程完成顺序打破并列。

### S3：任务服务独占选择与审计

任务服务只接收自己的扁平请求，在狭窄任务 / 同域写边界内重读当前态、两条授权关系，并对批次、索引、方法登记 / 生命周期 / 规格 / 条件 / 上下文和动作入口的稳定关系句柄 / 版本做结构当前性复核；方法语义复核仍归方法召回侧。首次选择创建唯一有效顺序号 30 引用关系，完整关系句柄 / 版本作为选择身份并绑定当前筹办生命周期版本。

精确同请求幂等；同一生命周期版本的不同方法写前拒绝。只有 #222 正式返回新的筹办中生命周期后，才使旧选择失效并保留审计，再创建新选择；不新增跨多关系回滚。选择成功仍保持筹办中，不自动排队或执行。

### S4：精确动作桥、下游接口与治理收口

任务服务提供双向授权、当前选择、当前性复核和审计读取；方法服务单向依赖任务服务，动作桥只接受精确当前选择关系 / 版本，并重读方法、规格、动作和所属生命周期。提供 #214 / #224 所需只读材料；新增 `自检.任务筹办选择.ixx` 真模块承载 A01-A15、并发选择、来源裁剪、策略唯一、任意引用负例和逐项输出，入口只以最终回归阶段 580 无捕获回调登记。完成规范、实施记录、计划归档、治理、提交和推送。

## 8. 失败收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| #209 缺正式结论或实际接口漂移 | 依赖门控返回 | 不改代码，退回设计并保持 #214 门控 |
| 无候选、建议过期、语义并列或无权威来源的策略唯一 | 逻辑内返回 | 保持筹办中，不写选择 |

| 精确重复选择 | 幂等逻辑内返回 | 读回当前关系，不增长结构 |
| 同一生命周期版本请求不同方法 | 逻辑内返回 | 保持既有当前选择，不覆盖 |
| 任意任务到方法引用或旧选择进入动作桥 | 逻辑内返回 | 不形成动作执行桥材料 |
| 选择写后正反读回不一致 | 追根因解决 | 停止执行依赖，不由排序器或线程补写 |
| 同版本权威材料相互矛盾 | 追根因解决 | 不返回普通失败后继续 |
