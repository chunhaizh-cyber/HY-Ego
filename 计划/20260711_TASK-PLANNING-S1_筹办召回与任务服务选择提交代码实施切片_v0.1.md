## 1. 目标

把合法处于筹办中的任务接到 #203-#208 正式方法召回链，并由任务服务独占提交唯一当前方法选择：

```text
任务当前态与双向需求承接授权
-> 冻结筹办请求
-> 方法召回 / 复判 / 排序只读建议
-> 任务服务重读全部来源版本
-> 唯一任务方法选择关系
```

本计划是 #209 扫描要求的明确后继，也是 #214 命名候选绑定所需任务授权 / 当前态 / 选择提交精确前置。本切片不执行方法、不生成回执、不写任务结果或需求结算。

## 4. 实施步骤

### S0：#203-#209 实际接口复核

实施记录必须列出 #203-#209 各提交号、登记 / 规格 / 索引 / 召回 / 复判 / 排序 / 建议函数签名，以及 #209 对授权关系、任务状态和选择接口的正式结论。任一缺失、建议没有来源版本或授权形状与本设计不一致时，不改代码并退回设计。

### S1：冻结筹办请求

任务服务只为当前筹办中任务形成请求，完整读回任务、生命周期关系 / 版本、来源需求、目标状态、主体、场景和双向承接配对。请求只作值式材料，不持久化第二份授权。

### S2：只读召回建议

调用 #206-#208 正式接口并保留全部来源版本。无候选、候选过期和语义并列逻辑内返回；不得按句柄、容器或线程完成顺序打破并列。内部不一致停止依赖路径。

### S3：任务服务独占选择

任务服务重读当前态、授权、建议、方法登记 / 生命周期 / 规格和动作入口后提交。首次选择只创建一条当前关系；精确重复幂等。待重筹办换代先失效旧选择并保留审计，再创建新选择。选择成功仍保持筹办中，不自动排队或执行。

### S4：#214 接口与治理收口

提供 #214 所需只读材料，补齐 A01-A14、并发选择与并列负例，完成规范、实施记录、计划归档、治理、提交和推送。

## 8. 失败收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| #209 缺正式结论或实际接口漂移 | 依赖门控返回 | 不改代码，退回设计并保持 #214 门控 |
| 无候选、建议过期、语义并列 | 逻辑内返回 | 保持筹办中，不写选择 |
| 未授权、状态漂移、方法失效、动作入口不匹配 | 逻辑内返回 | 不写选择关系 |
| 精确重复选择 | 幂等逻辑内返回 | 读回当前关系，不增长结构 |
| 选择写后正反读回不一致 | 追根因解决 | 停止执行依赖，不由排序器或线程补写 |
| 同版本权威材料相互矛盾 | 追根因解决 | 不返回普通失败后继续 |
