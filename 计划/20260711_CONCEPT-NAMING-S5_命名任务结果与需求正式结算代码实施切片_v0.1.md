### S1：绑定回执与当前结构复核

```text
重读命名需求的目标概念、共享目标状态、主体、宿主、场景和服务根。
重读来源任务的来源需求、当前态、选择方法和动作入口。
重读名称语素入口到概念关系及概念反向名称入口组。
重读动作动态的目标概念、0 -> 1、主体、场景、来源动作 / 方法和时间戳。
活动版本变化只触发当前概念身份复核，不自动否定已经发生的合法绑定。
```

### S2：记录或复用任务结果

```text
若任务尚无结果，调用任务服务正式幂等入口创建匹配共享命名目标的实例实际状态并记录结果 / 完成。
实例状态主体为自我、场景为命名需求场景、值材料与共享目标状态一致、时间不早于动作动态。
若任务已有完全一致结果和完成状态，幂等复用，不创建第二状态。
若任务已有冲突结果、状态不允许完成或关系不完整，追根因并停止结算。
```

### S3：记录或复用需求结算

```text
若需求已有完全一致的已满足结算，返回重复完成材料，不新增记录。
若没有结算，以来源任务、共享目标状态、任务实际状态、已满足、时间戳、场景、主体和动作动态调用正式入口。
任务已经完成而需求结算暂未形成是可重试的合法中间状态；不得回滚名称或任务结果。
若已有冲突结算，追根因，不覆盖。
```

### S4：完整读回与后续隔离

```text
重读任务实际状态、完成状态、需求最近结算、来源任务、目标 / 实际状态和动态证据。
全部一致才返回命名正式完成材料。
正式完成后不自动写统计、方法成功率、因果稳定性或学习权重。
```

## 7. 入口拒绝

```text

命名需求、目标概念、共享目标状态或场景与回执不一致。

名称关系、候选语素入口或概念反查不一致。
动作动态目标、0 -> 1、主体、场景、来源动作 / 方法或时间不一致。
任务已有冲突实际结果 / 完成状态。
需求已有冲突结算。
实例实际状态不能由任务服务原子创建并发布。
```

所有前置拒绝发生在新任务结果或需求结算写入之前。

## 8. 失败收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| #214 或 #225 正式接口未形成 | 逻辑内门控返回 | 不改代码，退回设计修订 |
| 回执无效、正常版本漂移、任务当前态不允许 | 逻辑内返回 | 不写任务结果或结算 |
| 已有完全一致任务结果 / 完成 | 幂等复用 | 不创建第二实例状态 |
| 已有完全一致需求结算 | 幂等复用 | 不新增结算记录 |
| 任务结果完成后需求结算暂时失败 | 可重试合法中间状态 | 保留名称、动态和任务结果，不伪造结算 |
| 当前同版本任务结果、完成或结算结构矛盾 | 追根因解决 | 停止依赖，不覆盖或扫描修复 |
| 任务或需求写后完整读回不一致 | 追根因解决 | 停止方法学习、统计和显示观察 |

## 9. 执行前接口复核与漂移退回

以下任一情况视为漂移：

```text
#214 没有形成真实绑定回执或动作动态。
#225 没有形成幂等结果 / 完成与独立结算接口，或其失败收口与假定不同。
#223 任务授权 / 选择材料和 #225 当前态 / 完成契约与命名回执无法逐项映射。
需求结算入口、最近结算读取或结果状态值语义改变。
完成本切片需要修改核心、状态、方法、动态、语素、概念图或线程承载模块。
```

出现漂移不得修改代码，退回设计窗口重写 #215；不得把固定安全根样例复制为命名结算。
