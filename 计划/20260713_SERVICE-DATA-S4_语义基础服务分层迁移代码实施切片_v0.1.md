## 3. 假定接口契约

本计划假定实际接口继续满足：

```text
结构写入执行器每次取得一次独占许可并同步传入不可移动会话左值引用；
会话可创建主信息候选、节点候选、关系、主键，并可读取节点类型、节点主信息和关系记录组；
无提交决定、失败或异常在许可释放前精确逆序撤销；
动态业务服务::读取动态材料(节点句柄) 返回抽象动态或完整实例动态值式材料；
关系仓库以完整关系记录和有效端点作为权威，不由索引候选裁决事实。
```

任何签名、可见性、决定时序、撤销时序或动态材料语义漂移，都必须在改代码前退回设计。

## 4. 实施范围

1. 创建基础信息通用身份新分层入口。
2. 创建语素入口、概念入口、两类关系追加和集合读取新分层入口。
3. 创建只接受动态业务服务值式证据的轻量因果新分层入口。
4. 所有写入采用非零全局幂等主键、一次独占许可、不透明会话和完整读回。
5. 创建独立自检模块，落实详细设计 `#268 A01-A20`。
6. 入口只增加自检模块导入、一次阶段 750 登记和值式总汇总。

## 6. 固定业务路径

```text
基础信息身份：非零主键 -> 主信息候选 -> 基础信息节点候选 -> 主键 -> 完整读回。
语素入口：非零主键 -> 主信息候选 -> 语素节点候选 -> 主键 -> 对应信息关系。
概念入口：在同一会话中追加唯一概念追溯关系，任一步失败全部收口。
关系追加：已有语素入口只创建一条白名单专用关系。
轻量因果：动态业务服务准入 -> 同许可身份复核 -> 因果引用候选 -> 主键；不写来源动态关系。
```

语素对应信息和概念追溯白名单严格采用详细设计第 5.2 节，不得合并成一个宽白名单。

## 11. 实施步骤

### S1 值式材料和数据操作

建立业务状态、请求规格、身份材料、关系证据和集合读取结果。两个数据操作模块分别封闭仓库与执行器，不向业务层泄露结构能力。

### S2 基础信息身份

实现非零主键、唯一主信息、固定基础信息类型、幂等和冲突拒绝。

### S3 语素入口与多关系

实现两套白名单、文本准入、入口创建、概念入口组合、关系追加和完整集合读取；不得静默选择第一个关系目标。

### S4 轻量因果

通过动态业务服务取得完整值式材料，再由数据操作层复核来源身份并创建因果引用。关系数量不得因来源证据增长。

### S5 自检和入口

独立自检覆盖 A01-A20。入口只在现有隔离自检条件下增加 import、一次调用、一次 750 登记和总汇总。

### S6 收尾

验证通过后登记 750、归档 #268、同步 #275 前置状态；只提交并推送本切片相关文件。
