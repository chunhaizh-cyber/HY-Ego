## 1. 定位

本切片建立任务管理上行桥：读取强类型回执，使用领域服务只读入口复核材料，把通过复核的回执转换为自我治理候选消息并提交唯一 mailbox。

```text
强类型任务结果回执
-> 稳定句柄版本与同任务顺序复核
-> 任务 / 方法 / 状态 / 动态只读复核
-> 执行前许可预判候选
-> 自我治理消息转换
-> SG-S2 mailbox
```

许可预判只说明材料是否具备进入正式执行入口复核的条件，不形成权限事实，不记录任务完成。

## 5. 假定接口契约

假定正式前置产物提供：

```text
S3A 可逐条取出 任务结果回执材料，且同一回执只消费一次。
S1 可构造 任务结果复核请求、执行前许可预判请求 两类治理消息。
S2 提供唯一 mailbox 提交入口并返回准入 / 重复 / 拒绝结果。
```

上行桥预期提供：

```text
复核任务结果回执：只读验证任务、方法、动作、场景、主体、实际状态和动态证据一致性。
生成许可预判材料：只返回候选分类和拒绝原因，不持久化 bool 或状态。
转换并提交治理消息：保留回执幂等键、同任务序号和稳定句柄版本。
停止接收上行：使用局部生命周期状态拒绝停止后的新回执，不形成业务事实。
```

同一任务的上行顺序必须保持，桥不得重排为比回执更早的序号。

## 10. 实施步骤

### S3B-0 执行前接口复核

读取 #171-#173 正式记录和导出接口；逐项确认回执字段、mailbox 返回和协议消息构造。再核对任务 / 方法 / 状态 / 动态公开读取足以完成复核。

### S3B-1 建立上行桥

新增桥模块，只持有公开服务引用和非权威队列入口；不持有核心仓库引用。导出复核结果、许可预判候选和治理消息转换结果。

### S3B-2 回执复核

按任务、方法、动作、状态、动态、场景、主体顺序读取并比较；过期版本、关系不一致、重复或顺序倒退均在提交 mailbox 前返回。

### S3B-3 转换与提交

保留来源回执的幂等键、任务序号和句柄版本，构造对应治理消息，通过唯一 mailbox 提交。桥返回提交材料，不把队列准入等同于领域准入。

桥接停止是运行期生命周期边界：停止后拒绝新上行材料并保持已提交消息不变，不记录任务失败或需求不满足。

### S3B-4 入口验收与收口

覆盖完整回执、版本过期、错误方法、错误动作、错误场景 / 主体、动态缺失、重复和顺序倒退；证明全部路径不改变任务、需求、状态、动态和四仓库数量。完成构建、记录、归档、提交和推送。

## 11. 逻辑内返回与追根因解决

逻辑内返回：回执为空、版本过期、任务不存在、任务方法关系缺失、动作入口不匹配、状态 / 动态材料不完整、场景 / 主体不一致、重复、乱序或 mailbox 拒绝。

追根因解决：公开读取材料完整但桥比较结果不一致、转换后丢失句柄版本 / 幂等 / 顺序、桥直接写领域结构、许可候选变成持久化权限事实，或桥持队列锁调用领域服务。
