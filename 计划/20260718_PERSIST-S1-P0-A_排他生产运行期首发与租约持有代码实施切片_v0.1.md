## 1. 依据与目标

上游为 20260718 恢复流程图 v0.3、同名 HTML、恢复详细设计第 17 节、#295 两份正式事实、#296 `15a7de7` 正式产物和 JY-416 当前接口复核，三层范围一致。来源性质为 MIG-18 重新实现 / 新增约束 / 全新能力，OF-183—OF-190 只作旧证据；不计旧能力迁移。

目标是在不改变默认旧通道的前提下，增加显式 `--runtime-context` 新通道：入口在旧局部仓库 / 服务栈构造前完成排他分流；不可复制的生产运行期会话使用项目版本 1 固定配置构造完整候选，依次初始化系统角色与概念活动，首次发布到会话唯一宿主，并持有与宿主当前上下文同身份的租约直到该早期分支返回。P0-A 尚无生产消费者，显式新通道成功后直接退出 0，不启动常驻循环、线程、SQL 或控制面板。任一通道失败不得回退另一通道。

## 2. 依赖门控与当前事实

### 2.1 前置

```text
#296 / DQ-188 已由 main@15a7de7 完成并归档。
运行期上下文构造签名为 运行期上下文(uint64_t 域编号, uint64_t 仓库编号 = 1)。
系统角色入口为 初始化系统角色(const 系统角色初始化参数&) -> 系统角色初始化结果。
概念活动入口为 初始化概念活动(const 概念活动初始化参数&) -> 概念活动初始化结果。
完整门禁为 完整()；宿主发布入口为 尝试首次发布全新上下文(shared_ptr<运行期上下文>)。
租约入口为 运行期上下文宿主::读取租约()；租约有效且 读取() 返回发布上下文地址时身份一致。
当前入口在参数解析后立即构造旧局部仓库 / 服务栈；尚无 --runtime-context 分支。
中央 Debug 专项登记全部位于入口.cpp；自检.运行器.ixx 只提供通用运行器 / 阶段总成类型，不拥有 900 登记表。
```

### 2.2 假定接口

```text
生产运行期配置版本 1：域 1、仓库 1、稳定前缀 0x4859'0001'0000'0000。
21 个系统角色键 = 前缀 + 1…21；生命周期三角色键 = 前缀 + 22…24。
安全当前 / 目标 = 0 / 1；服务当前 / 目标 = 0 / 1；动态根材料 = 1。
稳定动作键 = 0x4859'0002'0000'0001。
生产运行期启动请求只含请求版本 1，不接收域、仓库、稳定键或状态值覆盖。
生产运行期会话::启动(生产运行期启动请求) -> 生产运行期启动结果。
生产运行期启动结果携带强类型状态、系统角色状态、概念活动状态、发布状态和运行期上下文租约。
生产运行期会话::读取租约() -> 当前宿主同身份租约；会话不可复制并唯一拥有运行期上下文宿主。
```

这些值不得来自自检起始主键、阶段号、时间戳、命令行自由值、SQL、日志或旧对象。入口只识别 `--runtime-context` 模式位，不把命令行值传入固定配置。若上述签名不一致，按逐签名漂移退回，不得修改 `启动.运行期上下文.ixx` 或在入口临建第二宿主。

### 2.3 排他分流与双路径验证

```text
默认 / --self-test-exit：保持现有旧栈路径；Debug 中由入口登记并运行阶段 900 隔离自检。
--runtime-context：在旧栈构造前进入生产会话，真实首发成功后直接退出 0。
--runtime-context --self-test-exit：同样只做真实首发后退出，不运行中央阶段总成。
--runtime-context 与 --database-self-test-exit / --warehouse-performance-self-test-exit / --d455-real-sample-exit / --headless 任一组合：写前退出 2，不构造新旧任一上下文。
```

阶段 900 的自检会话与显式生产会话不会在同一次路径中同时构造。P0-A 不新增未知参数拒绝语义，也不改变现有默认参数组合。

## 6. 非成功返回

无效启动请求版本、与新通道冲突的既有模式组合和已有宿主属于逻辑内返回：不发布、不回退另一通道。生产固定配置不精确、有效请求后的系统角色 / 概念活动失败、完整候选发布异常，或发布成功后租约无效 / 异身份，属于追根因解决。并发同一会话只有一笔 `已启动`，其余 `已有当前上下文` 属于合法线性化结果。实际签名与第 2.1 节不一致且需计划外文件属于接口漂移；目标模块尚不存在不是漂移。
