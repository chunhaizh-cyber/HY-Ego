## 1. 定位

本切片建立自我治理领域路由第一段，固定选用已初始化的安全根需求，串联 E01-E06：

```text
治理候选消息
-> 读取安全根需求同组材料
-> 按需求创建任务
-> 建立固定方法和主动动作入口
-> 绑定固定输入 / 输出规格
-> 任务选择该固定方法
-> 读回动作执行桥请求材料
```

第一轮不做方法召回、排序、自动选择或真实动作分派。

必须证明路由公开读取不能取得不完整上下文；若半结构仍可由正式入口作为完整聚合读取，本切片验收阻塞并另建事务 / 隔离计划。

## 11. 实施步骤

### S4A-0 执行前复核

读取 #174 正式接口、初始化快照和六个领域服务签名；核对固定动作键未占用。任何差异先退回修订。

### S4A-1 建立路由与输入准入

新增路由模块，拒绝非固定安全根需求消息、缺句柄、过期、重复和错误序号；锁内只处理消息值，所有领域调用在冻结批次锁外完成。

### S4A-2 串联 E01-E06

按根需求读回、任务承接、固定方法、动作入口、规格、任务选择方法和动作请求材料顺序调用现有领域入口；每一步立即正式读回。

### S4A-3 负例和结构不变

覆盖错误根需求、错误场景、零时间戳、动作键冲突、重复消息、已有任务方法关系和规格缺失；首写前拒绝前后数量不变。

### S4A-4 收口

完成构建、自检、扫描和正式记录，归档计划并提交推送。

## 12. 逻辑内返回与追根因解决

逻辑内返回：消息类型不支持、固定根需求材料缺失、句柄版本过期、时间戳为零、稳定动作键已占用、重复、乱序或规格材料缺失；均在首写前返回。

追根因解决：前置通过后任务 / 方法 / 动作 / 规格 / 关系 / 索引写入或读回不一致，固定安全根需求被替换，动作请求材料与任务方法关系不一致，或不完整上下文对外可读。
