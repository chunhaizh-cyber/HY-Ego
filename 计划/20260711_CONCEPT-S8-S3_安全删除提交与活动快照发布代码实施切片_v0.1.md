## 6. 实施步骤

1. 保留当前 #219 未暂存草案，从 Debug x64 0/0 与已通过 A 项继续；精确复核关系仓库普通读取、单条审计、组审计及令牌包装，禁止回退或重写已通过的提交链。
2. 在关系仓库 `.h/.cpp` 修订现有显式审计：单条审计只验证完整关系句柄、仓库 / 版本和可审计状态；组审计只验证完整源节点句柄、关系类型及记录源节点精确相等，不再调用 `节点有效_` 过滤历史端点。普通读取 / 查询 / 存在 / 父链 / 数量保持原端点有效检查。
3. 在 `概念安全删除提交能力.数据.h` 追加概念登记、统计缓存、活动快照阶段，保持会话外部不可构造 / 不可复制、唯一编排器友元和连续单调推进。
4. 在统计服务补唯一编排器私有缓存冻结、`_已加锁` 目标重读和目标级提交；只删精确键，不调用整命名空间清空。
5. 在概念图服务补唯一编排器私有图冻结、`_已加锁` 重读 / 准备、登记提交和完整活动快照提交；公开包装与持锁辅助不得递归加锁。
6. 新建生产真模块并定义唯一编排器；它是全部私有安全删除入口和提交会话的唯一友元。
7. 取得结构独占许可后非阻塞试取活动图、图写、概念用途缓存锁；任一失败立即逆序释放并返回提交环境繁忙。
8. 冻结后重读全部准入、写集、登记和缓存；候选过期或材料变化时写前返回。
9. 预构造全部清理包、登记组和完整新活动快照，完成分配、版本、无环和四根可达验证。
10. 创建私有提交会话，依次提交关系、索引、主信息、节点、概念登记、目标缓存和完整活动快照；完成读回后逆序释放。
11. 删除完成后按准备包中每个当前审计关系句柄调用显式审计，证明端点已删除仍可读完整失效记录；同时验证普通单条 / 组查询、存在判断和有效数量均不返回目标关系。
12. 返回值式回执，经节点 / 主信息 / 关系显式审计和仓库 / 服务读回对照；回执不持久化、不含提交会话。
13. 添加同命名空间双概念缓存、活动图公开读取、关系创建 / 重挂、索引绑定、概念发布和删除提交并发线性化自检；发布成功计前态，基准版本过期计后态，两者均为合法结果。
14. 新建 `海中鱼巣/领域/自检.概念安全删除.ixx`，只经公开顶层 `提交安全删除` 承载 A01-A12；不得获得友元或直调单包提交。
15. `入口.cpp` 只导入自检模块并登记 `CONCEPT-S8-S3 / 最终回归阶段 530`，不展开正文。
