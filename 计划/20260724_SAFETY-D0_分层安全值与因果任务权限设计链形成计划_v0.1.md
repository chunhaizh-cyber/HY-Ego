## 1. 目标

在不修改代码的前提下，把用户确认的安全逻辑形成一个范围闭合的设计包：

```text
安全事件、因素、验证排除、场景和有效未复发时间
-> 分层安全值低位回升 / 高位回落
-> 相对于具体安全结果的直接因果最短路径
-> 无上限 I64 精确深度
-> 严格五级任务执行优先级组
-> 1% / 99% 四道递归门控
-> 多需求任务取最高有效权限
-> 任务调度只读消费
```

设计必须区分物理安全根值、分层安全值、需求权重、任务基础优先级和任务执行优先级权限，不把任一值压成另一值或第二份权威事实。

## 4. 现状事实与差距

只读事实扫描固定：

1. 当前轻量因果入口只能形成轻量引用或候选材料，没有稳定直接因果图、环检测、目标特定最短路径和精确深度。
2. 当前自我治理领域路由仍是固定安全根兼容链，不能扩写成动态安全层算法。
3. 当前自我线程只在消息批次中调用可选处理器，没有每个有效治理循环调用一次的安全维护入口。
4. 当前任务执行请求和任务管理线程没有安全需求来源、精确深度、严格五组、权限版本或确定性排序材料，消费仍接近确认顺序。
5. 节点直接身份结构写入底座已有多参与者、确认、逆序撤销和最后发布能力，但安全记录及真实状态 / 动态 / 因果参与者尚未接通。

这些差距只证明需要实施，不证明任何目标接口已经存在。

## 5. 设计冻结

本计划形成的设计包必须同时冻结：

1. 安全层维护族、当前投影、事件、因素、排除、有效未复发证据和维护纪元。
2. 全部相关机器数值使用 I64；权限满刻度固定为 `1,000,000`。
3. 新事件 / 复发优先，随后主动结算、因素排除和被动维护。
4. `D=0` 不形成调度组；`D=1..4` 进入同号组，`D>=5` 进入组 5 并保留真实深度。
5. 五组只经过四道释放门，组 5 是终端汇总组，总权限恒等于满刻度。
6. 安全事实事务和权限值式投影分离；权限投影不得成为第二份持久化事实。
7. #377 只拥有安全协议、纯值算法、记录参与者、数据操作、领域服务和专项自检源码候选。
8. #359 唯一拥有线程路由、任务管理消费、工程、入口、统一运行器、真实提供者适配和生产验收。
