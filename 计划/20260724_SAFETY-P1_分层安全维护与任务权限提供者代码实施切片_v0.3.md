## 5. 结果语义覆盖

1. `安全维护载荷` 在值实际变化时同时给出当前旧值版本、新值版本和维护账版本。
2. `无变化` 与 `精确重复` 必须给出当前值版本、`新值版本=nullopt` 和可回读维护账版本。
3. 不得用同一新值版本字段同时表达新发布、无变化和重放。
4. 被动维护提交方法从 `安全层定义快照` 读取并由定义版本冻结；线程、请求、任务和外部参与提供者不得临时指定或替换。

## 6. 权限缺口覆盖

组 `g` 的当前事实或阈值材料缺失时：

```text
组 g 及全部更高组
-> 权限状态 = 材料缺口
-> 收到权限 = 0
-> 保留权限 = 0
-> 记录具名缺口来源和受影响组

低于 g 的组
-> 保留已经合法形成的权限
```

受影响任务进入 `材料缺口任务组`，不得取得猜测权限。`五组已形成权限总量` 只统计状态为已形成或权限为零的组；不得把材料缺口导致的零解释为完整权限守恒。

`权限读取结果` 的载荷规则固定为：

```text
已形成、权限为零
-> 必须有完整权限读取载荷

材料缺失
-> 必须有部分权限读取载荷
-> 低于首个缺口组的合法权限可消费
-> 首个缺口组及更高组为材料缺口和零权限

版本漂移
-> 必须无载荷
-> 整份权限投影失效，不允许部分消费
```
