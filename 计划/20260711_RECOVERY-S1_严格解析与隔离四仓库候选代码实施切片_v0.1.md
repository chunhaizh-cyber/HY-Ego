-> 总头 / 段表 / 边界 / 校验

-> 索引候选暂存
-> 隔离精确读回

解析结果必须保留段 1 O1 元数据子项和段 5 的最终物理键、目标节点、所有者 / 格式、探测规则版本 / 序号。后续 36 字节规范输入必须由段 1 合同、段 2 的 O1 格式 / 非零任务序号和段 4 角色 1 的完整任务句柄重建，不得从段 5 读取身份副本。失效的角色 31 必须连同新增角色 15、32 原样导入隔离关系候选；不得因失效而过滤，也不得将其恢复为当前有效。

严格解析必须在分配和导入前拒绝：单任务记录数超过 1024、全局记录数超过 65536、单条规范编码超过 2048 字节、总规范编码预算超过 128 MiB、单观察生命周期关系超过 17 条，以及缺槽 / 多槽 / 空槽、坏角色、30/31/32 非恰一当前有效、重复身份或不支持的探测信息。上述外部超限属于逻辑内拒绝，整份隔离候选保持不存在；不得截断或选择第一条。本切片不得自动统计、概念映射、学习、晋级、发布方法、重试待结算或执行动作。

## 4. 实施步骤

### S0：#226 实际格式复核

记录总头、九段、版本、校验、规范字节和文件发布实际接口。若 #226 格式或解析结果与详细设计不同，停止并退回修订 #227；不得自行兼容未知字段。

### S1：严格外部解析

先固定文件总边界，再验证段表唯一性、偏移 / 长度 / 计数、校验和摘要。先解析并验证段 1 的版本、所有者、角色集合和容量合同，边界确认后才允许并行解析段 2—5；全部输出都是不可变值式 DTO。

### S2：恢复专用仓库构造

主信息、节点、关系仓库新增候选构造器，直接接收已验证全记录和精确高水位。不得调用普通创建 / 删除 / 重挂入口逐条拼装，也不得从最大编号推导高水位。

### S3：端点和拓扑复核

有效关系要求精确当前有效端点与合法类型组合；历史关系按身份定位墓碑端点，端点版本不得大于当前墓碑版本。普通父子继续复核单父、自身和成环。历史关系不得进入普通父子 / 当前投影。

### S4：隔离读回

按审计入口逐记录比较指定身份、版本、状态、槽位、创建序号和高水位。索引段只暂存值式候选。外部坏材料逻辑拒绝；已严格验证的导入包无法精确读回追根因。

### S5：自检与治理

补齐 RECOVERY-S1-A01 至 A16、全拒绝矩阵、单 / 多线程解析一致性；完成规范、实施记录、计划归档、治理、提交和推送。

## 7. 入口拒绝

```text
文件路径、魔数、总头 / 段表 / 快照版本或字节序不支持。
文件长度、段数量、段顺序、唯一性、偏移、重叠、边界、计数或校验错误。
记录身份为零 / 重复，版本 / 状态 / 枚举未知或数值溢出。
高水位为零、不高于已用身份或创建序号高水位错误。
节点主信息身份缺失 / 版本不允许。
有效关系端点非当前有效精确句柄或类型组合非法。
历史关系端点身份缺失、端点版本为零 / 大于墓碑版本。
普通父子多父、自身或成环。
```

全部外部材料拒绝发生在候选仓库构造前，或只改变隔离候选；当前运行期不变化。
