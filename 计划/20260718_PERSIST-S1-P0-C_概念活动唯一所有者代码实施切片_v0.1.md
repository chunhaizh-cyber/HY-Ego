## 3. 假定接口契约

以下是待实施目标，不是当前已存在接口：

```text
运行期概念生命周期阶段 { 活跃=1, 冷却=2, 退役=3 }
概念活动初始化参数 { 参数版本=1, 活跃状态稳定主键, 冷却状态稳定主键, 退役状态稳定主键 }
参数有效 = 版本精确为 1、三主键非零且两两不同

运行期上下文::初始化概念活动(概念活动初始化参数)
运行期业务装配::初始化概念活动(系统角色清单, 概念活动初始化参数)
运行期上下文::读取概念活动材料() -> 值式副本

概念活动业务服务：业务准入、状态规格形成、唯一私有拥有三角色 / 签名组 / 活动版本
概念活动数据操作：一个不透明结构写入会话内写三抽象状态、四条根到活跃关系并完整读回
运行期上下文::完整()：系统角色与概念活动材料均完整、同仓库且重建视图一致
```

固定裁决：

1. 新枚举不得与旧 `概念生命周期阶段` 同名，不 import、转换或比较旧类型。
2. 新数据操作不 import `数据操作.概念图结构`；它直接持有同域结构接线所需的最小依赖，并通过 `状态动态数据操作` 参与同一会话。
3. 活动版本初始固定为 1，不由调用方传入；三个生产稳定键只能由调用方显式给出，不能从阶段号、自检主键或算术偏移推导。
4. fresh 基线为系统角色完整、四根已存在、三生命周期角色和非根活动关系不存在。正式 fresh 请求发现预存材料属于内部矛盾，不静默认领。
5. restore 由后续 #228 独立适配完整签名、角色身份和活动版本；本入口不得猜恢复材料或覆盖恢复版本。
6. 旧创建型概念结构内部类型固定机械改名为 `概念结构根类别`；三个所有者文件和 `自检.运行期组合分层.ixx` 的单一消费点同步，底层 `uint8_t`、0—4 数值、字段语义、关系 9 / 10 / 11 以及阶段 755 / 770 保持不变，不与规范 `概念根类别` 转换、比较或合并。

执行前若四文件机械同步、固定模块骨架、状态规格 / 会话接口或三套夹具无法在允许文件内承接上述合同，必须列出精确签名差异并退回设计；不得通过别名 / 宏隐藏 IFC，不得 import 旧概念图服务、扩大传统服务 ABI 或拆成多个已提交事务。

## 7. 非成功返回二分

| 场景 | 分类 | 收口 |

| 参数版本错误、稳定键为零或重复 | 逻辑内返回 | 写前拒绝，结构和私有状态不变 |
