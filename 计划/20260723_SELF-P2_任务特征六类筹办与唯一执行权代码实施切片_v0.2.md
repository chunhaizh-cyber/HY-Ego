# SELF-P2 任务特征、六类筹办与唯一执行权代码实施切片 v0.2

身份：`#355 / DQ-247 / SELF-P2 v0.2`

日期：2026-07-24

设计状态：范围候选；本文不维护执行过程，精确允许文件和完整 SELF-C2 施工接口尚需后继设计冻结。

## 1. 目标与依据

依据 3200、5200—5250、6150 和 SELF-C2 / v0.1，实现完整任务特征、按任务身份唯一筹办占用、六类筹办结果、方法选择和执行冻结。

`SAFETY-C2 / v1` 只作为任务进入正常执行竞争前的值式排序材料：

- 多来源需求取当前最高有效权限并保留全部来源；
- 权限为零保留任务、筹办历史、方法和材料，但不参加正常执行竞争；
- 安全权限不是任务授权、筹办执行权或执行冻结。

## 2. 必须冻结

- 完整任务特征、筹办请求、占用、六类结果和执行冻结 DTO；
- 任务安全来源关系，以及按需形成的精确深度、五级组、权限值和派发前重读合同；当前权限不持久化为任务特征；
- 同任务只有取得占用者进入召回，不同任务可并发；
- 无候选不创建方法首，执行期发现结构缺口属于筹办漏检；
- 精确允许 / 禁止文件、专属记录、失败收口和验收矩阵。

## 3. 候选步骤

1. S0 核对 SELF-C2、SAFETY-C2 和实际任务接口。
2. 形成任务特征与唯一筹办占用。
3. 完成方法召回、选择、冻结和六类结果。
4. 按 SAFETY-C2 从当前来源事实形成非持久权限投影和排序材料；只把实际选择所消费的投影冻结为历史证据，不在任务域重算安全值或保存第二份当前权限。
5. 验证同任务竞争、多任务并发、零权限、版本漂移和恢复旧占用失效。

## 4. 边界

本切片不实现安全算法、不修改自我线程 / 任务管理线程、不执行方法、不写世界事实、不接生产入口。权限变化不自动取消已经运行的任务；停止或重筹办仍走任务正式入口。
