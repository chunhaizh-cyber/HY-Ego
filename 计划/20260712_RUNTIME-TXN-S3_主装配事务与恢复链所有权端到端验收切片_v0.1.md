## 9. 非成功返回与失败收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 执行前接口、调用点或阶段漂移 | 设计漂移 | C++ 零修改，原编号退回设计 |
| 空候选、结构核心不完整、装配不完整、初始化材料缺失 | 逻辑内返回 | 宿主、租约和业务结构不变化 |
| 宿主已有上下文或并发落败 | 逻辑内返回 | 保持原上下文，不替换、不销毁落败候选 |
| 提交点前最终复核失败 | 逻辑内返回 | 宿主保持空，不写发布槽 |
| 前置通过后宿主与租约身份不一致 | 追根因解决 | 停止后继，不登记 680，不用日志 / 显示兜底 |
| 发现恢复路径可调用普通全新发布入口绕过 #229 | 追根因解决 | 撤回草案，修订所有权合同后再实施 |

发布入口本身不写四仓库或领域侧表。失败收口必须通过宿主状态、租约身份和候选结构数量快照证明；无法证明时停止并登记断点。
