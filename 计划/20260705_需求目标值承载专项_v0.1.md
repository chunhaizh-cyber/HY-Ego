# 需求目标值承载专项

编号：HR-001-DV
日期：2026-07-05
状态：v0.1 / 待确认 / 未改代码

## 1. 目标

处理 S0 缺口 2：

```text
需求服务当前直接把 I64 目标值写入需求节点主信息值容器。
```

本专项目标是把需求目标值从需求节点主信息值容器中移出，改为需求系统结构承载。

## 3. 当前缺口

当前代码事实：

```text
需求服务::创建I64目标需求(std::int64_t 目标值)
-> 创建需求节点主信息
-> 主信息_.写入I64值(主信息句柄, 目标值)
-> 创建需求节点

需求服务::结算I64目标(...)
-> 从需求节点主信息读取 I64 目标值
-> 从特征值服务读取当前值
-> 当前值 >= 目标值 时返回 已满足
```

这违反已确认口径：

```text
MI-V1 值容器语义只保留在特征值服务范围内。
需求、任务和方法不得把主信息仓库当通用值容器。
特征值节点不得直接保存需求目标值。
```

## 4. 目标结构

第一轮目标结构：

```text
需求节点
-> 引用 目标存在节点
-> 引用 目标特征节点
-> 引用 当前特征值节点
-> 模板 目标状态节点
```

目标状态节点：

```text
由状态服务创建。
状态值 = 需求目标 I64 值。
不携带发生时间戳。
不是需求结算实例状态。
```

比较方式：

```text
第一轮固定为 I64 下限目标：当前值 >= 目标值。
任意比较方式后续另建二次特征或比较方式专项。
```

## 6. DV-1 代码原则

```text
需求服务必须通过状态服务创建目标状态节点。
需求服务必须通过特征值服务读取当前特征值。
需求服务必须通过关系仓库写目标材料关系。
旧 `创建I64目标需求(std::int64_t)` 不得继续作为有效写值入口。
旧 `结算I64目标` 不得继续读取需求节点主信息值容器。
```
