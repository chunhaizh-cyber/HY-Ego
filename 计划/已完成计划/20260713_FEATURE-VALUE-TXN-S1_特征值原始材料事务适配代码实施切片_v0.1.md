## 2. 实施范围

1. 为结构写入执行器增加不含领域语义的单参与者执行重载。
2. 为结构写入会话模块增加仅执行器可构造的提交准备只读视图；只允许读取本会话候选身份、候选节点类型、候选节点主信息和候选 I64 值。
3. 在特征值服务头中只登记事务参与者和侧表访问器友元。
4. 新建特征值原始材料参与者模块，实现初始 I64 版本、VecI64、VecU64 的准备、完成提交、精确撤销和非阻塞只读。
5. 在执行器增加仅 Debug|x64 宏内可用的固定确认前故障自检重载；故障只位于参与者准备成功后、首次仓库确认前。
6. 新建独立自检模块，覆盖 A01-A16；A09、A10 独立执行，A10 分别使用真实 I64 与至少一种 Vec 记录。
7. 入口只在同一 Debug|x64 宏内增加自检模块导入、一个值式结果、阶段 735 登记和总汇总。

### S1 通用参与者与提交准备只读视图

增加通用参与者生命周期、执行器重载和提交准备只读视图。视图不可默认构造、复制或移动，只能由执行器在当前线程、会话已请求提交、无失败且尚未收口时构造；候选节点 / 主信息限定读取必须使用同一会话令牌。原 `执行(回调)`、`可继续写入()`、既有普通读写入口行为和 ABI 保持。

在同一执行器模块中增加宏控制的 `执行确认前故障自检`。该重载要求无参观察回调非空，在参与者准备成功后调用一次观察回调，随后固定先撤销参与者、再撤销会话并返回内部不一致；不得继续调用 `会话.完成提交()`。普通生产重载不接收或转发故障参数。

### S2 特征值原始材料参与者

实现值式材料暂存、经窄视图进行本会话候选复核、`try_lock` 准备发布、精确提交与撤销。视图复核必须在取得侧表锁前完成；参与者取得侧表锁后禁止调用视图或仓库。

### S3 侧表只读访问器

实现不访问仓库的 `try_lock_shared` 值式副本读取。锁竞争不得等待或拼接多次快照。

### S4 自检与入口

在独立自检模块落实 A01-A16：

A10-I64：真实 I64 参与者准备成功并命中确认前故障一次；返回后 I64 版本未建立、结构数量和可读句柄回前态。
A10-Vec：真实 VecI64 或 VecU64 参与者准备成功并命中确认前故障一次；返回后 Vec 记录未建立、结构数量和可读句柄回前态。

## 8. 失败收口

```text
入口拒绝或侧表锁竞争：参与者不写正式记录；会话在许可释放前撤销。
请求提交后无法构造窄视图，或视图发现线程 / 阶段 / 决定 / 失败状态不符合合同：按内部不一致追根因，参与者和会话在许可释放前撤销，不得退回普通会话入口读取。
参与者准备后结构确认失败：按精确节点句柄撤销本次 I64 版本 / Vec 记录，再撤销或报告会话结果。
隔离自检确认前故障：观察回调只证明固定点被命中；执行器必须先调用参与者完成撤销，再调用会话完成撤销，不得进入任何仓库确认。I64 与 Vec 两条路径分别证明侧表和四仓库回前态。
异常、无提交决定和显式撤销：参与者先收口，四仓库写集再沿既有顺序收口。
收口后必须同时复核四仓库数量、可读结构、I64 版本表和 Vec 表回到前态。
无法证明回前态：停止 #267，登记正式断点，不得用扫描修复。
```
