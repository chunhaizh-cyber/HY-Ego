## 3. 固定闭环样例契约

### 3.1 样例语义

同一组样例必须按以下顺序形成证据：

```text
自我存在
-> 现实场景
-> 目标特征和目标状态
-> 目标状态需求
-> 需求承接任务
-> 固定方法和任务选择方法关系
-> 方法动作入口与动作请求材料
-> 特征状态写入结果
-> 状态变化动态和轻量因果引用
-> 任务实际结果状态与完成状态
-> 需求结算状态
-> 非权威缓存观察材料
-> 事件日志审计材料
-> 显示只读材料
```

### 3.2 句柄槽位规则

证据矩阵必须使用中文符号槽位描述同组身份，例如：

```text
样例自我存在句柄
样例现实场景句柄
样例目标特征句柄
样例目标状态句柄
样例需求句柄
样例任务句柄
样例方法句柄
样例动作入口句柄
样例改变前状态句柄
样例改变后状态句柄
样例动态句柄
样例因果引用句柄
样例任务完成状态句柄
样例需求结算状态句柄
```

文档不得编造运行期节点编号。真实节点编号只能由后续代码运行时创建和读回；计划和证据矩阵固定的是句柄角色、来源入口、关系约束、版本 / 时间戳要求和读回方式。

### 3.3 方法选择边界

第一轮只允许：

```text
固定验收样例中的单一方法。
读取已有且唯一的任务选择方法关系。
由测试入口人工明确选择方法后再读回关系。
```

不得把方法候选材料解释为自动选择结果，不实现召回排序、评分更新、在线学习或自动方法选择。

## 4. 证据矩阵字段

每一行至少记录：

| 字段 | 要求 |
| --- | --- |
| 证据编号 | 使用 `REAL-LOOP-S1-E01` 起的稳定编号 |
| 闭环阶段 | 需求、任务、方法、动作请求、动态、任务结果、结算或观察 |
| 输入槽位 | 本步骤读取的稳定句柄角色、版本和时间戳材料 |
| 当前入口 | 当前代码中的领域服务或只读入口 |
| 写入方 | 唯一允许写正式结构的领域服务 |
| 结构承载 | 节点、主信息、关系、索引、状态、动态或因果引用 |
| 读回入口 | 证明结果可读的当前正式入口 |
| 权威性 | 权威结构、候选材料、非权威缓存、人读审计或只读显示 |
| 前置拒绝 | 句柄无效、类型错误、关系缺失、版本过期或时间戳缺失等拒绝条件 |
| 拒绝后证明 | 节点、关系、索引数量不变，或公开读取入口不可读 |
| 内部异常 | 前置通过后写入、关系、状态、动态、结算或读回不及预期时进入追根因解决 |
| 后续切片 | 由 S2、S3、S4、S5 或 S6 承接 |

矩阵必须区分：

```text
正式结构事实。
运行消息和任务回执材料。
非权威缓存材料。
事件日志审计材料。
显示只读材料。
```

后三类不得裁决需求满足、任务完成、方法成功、动作发生或执行权限。

## 7. 实施步骤

### S0 上游与当前事实复核

只读复核父计划、流程图、详细设计和 #162 S0 记录，确认当前代码入口和缺口没有发生改变。若入口、结构承载或父计划状态已漂移，停止生成矩阵并登记真实阻塞，不用旧记录覆盖当前代码事实。

### S1 固定样例槽位

按第 3 节定义一个闭环样例的中文符号槽位，明确每个槽位的创建 / 读取入口、节点类型、关系角色、版本和时间戳要求。槽位不得填入推测的运行期节点编号。

### S2 建立权威证据矩阵

覆盖需求目标、任务承接、任务选择方法、动作请求、状态变化、动态、轻量因果、任务实际结果、任务完成和需求结算。每一项必须给出写入方、结构承载和读回入口。

### S3 建立非权威观察矩阵

分别记录缓存、事件日志和显示材料的来源、读取入口、可丢失 / 可重建边界及不裁决证明。三类材料不得合并成一个泛化“输出成功”证据。

### S4 建立拒绝与失败收口矩阵

前置材料缺失、句柄无效、类型错误、关系缺失、版本过期、同任务顺序错误、重复消息或队列满载属于逻辑内返回，必须证明结构不变。前置通过后创建、绑定、写状态、记录动态、写任务结果、结算或读回不及预期属于追根因解决，必须停止依赖路径并登记后续施工切片需要采用的事务回滚、失效隔离或数量快照加读取拒绝方式。

### S5 登记后续依赖顺序

只登记以下顺序和每个切片的输入 / 输出边界，不生成代码计划正文：

```text
REAL-LOOP-S2：需求到任务承接到固定方法选择串联。
REAL-LOOP-S3：方法执行、动作请求到动态记录串联。
REAL-LOOP-S4：动态到任务回执到需求结算串联。
REAL-LOOP-S5：缓存、事件日志和显示只读观察串联。
REAL-LOOP-S6：闭环验收、拒绝路径、并发和停止收口验证。
```

S2 依赖 S1 完成；S3 依赖 S2；S4 依赖 S3；S5 依赖 S4；S6 依赖 S2 至 S5 全部完成或形成真实阻塞并被裁决。任何 C++ 切片必须独立列明允许文件、禁止文件、入口拒绝、失败收口、读回验证和构建 / 运行命令。
