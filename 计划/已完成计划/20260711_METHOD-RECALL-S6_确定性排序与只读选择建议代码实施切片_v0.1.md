## 4. 实施步骤

### S0：接口复核

读取 #207 实际 DTO 与来源版本，扫描是否已有正式策略证据入口；没有证据时不得虚构评分。

### S1：语义排序键

```text
结果能力匹配等级。
条件复判结论。
显式策略 / 证据等级（第一版允许为空）。
完整方法句柄确定性排列。
```

不纳入缓存命中、展示次数、关系编号、创建顺序、线程完成顺序或文本名称。

### S2：并列和建议

```text
过滤不满足、不可判定和过期候选。
按语义键排序，再按完整句柄稳定排列。
仅一个可用候选 -> 唯一可建议。
显式策略使首项语义严格优于次项 -> 策略唯一。
语义首项并列 -> 返回有序候选但不建议提交。
```

### S3：自检和治理收口

补齐 METHOD-RECALL-S6-A01 至 A12。

```text

```

空候选和语义并列是正常逻辑内返回。

## 8. 失败收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 无可用候选 | 逻辑内返回 | 返回空有序组 / 无建议 |
| 语义首项并列 | 逻辑内返回 | 返回稳定有序组 / 无建议 |
| 来源版本正常漂移 | 逻辑内返回 | 返回材料过期 |
| 同一输入排序结果不稳定或重复项未消除 | 追根因解决 | 不发布建议 |
| 唯一建议缺来源证据或与首项不一致 | 追根因解决 | 停止 S7 依赖 |

## 9. 执行前复核与漂移退回
