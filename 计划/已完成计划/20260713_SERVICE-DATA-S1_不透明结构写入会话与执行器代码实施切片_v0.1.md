## 2. 已验证当前事实

1. `结构事务接线` 已提供 `取得独占许可`，`结构事务许可` 负责 RAII 释放；协调器保持同线程重入拒绝和线程绑定验证。
2. 主信息仓库已提供结构化候选创建、确认、撤销和严格删除；节点仓库提供同形入口。
3. 关系仓库已提供 `结构化创建关系` 和 `严格删除关系`，创建成功返回精确关系句柄。
4. 索引仓库已提供 `结构化绑定主键` 和 `严格删除主键(主键, 预期节点, 令牌)`；因此索引写集必须保存主键和预期节点完整句柄，不能只保存主键。
5. 四仓库新结构化写入口均只接受独占许可；旧无令牌 / 带令牌兼容入口和关系 / 索引共享删除入口仍存在，但本切片不得调用。
6. 主信息 / 节点候选不可默认构造、不可复制、可移动但不可移动赋值；会话必须直接拥有候选对象，不得返回候选或令牌。
7. 协调器在独占许可对象生存期内持有全域独占结构锁；其它线程取得共享许可必须等待，同线程重复取得许可被拒绝。四仓库构造链要求同时未接域或使用同一结构事务域，结构化入口再按令牌复核实际域。

## 3. 实施目标与调用路径

新增不透明 `结构写入会话` 和 `结构写入执行器`：

```text
隔离自检提交同步类型化回调；后续仅允许已登记数据操作模块提交
-> 执行器检查接线和四仓库引用
-> 取得一次独占许可
-> 在当前线程栈内直接构造不可移动会话
-> 回调只通过会话窄方法创建候选 / 关系 / 索引
-> 会话记录主信息候选、节点候选、关系句柄、索引绑定完整对
-> 回调请求撤销或抛出异常：索引绑定对 -> 关系句柄 -> 节点候选 -> 主信息候选
-> 回调在完整读回后请求提交：节点候选 -> 主信息候选确认
-> 执行器释放许可并返回值式结构结果
```

第一轮只在隔离自检验证，不接业务生产路由，不把会话传入领域服务公开入口。领域服务、线程路由和组合器不得直接导入执行器并提交任意仓库操作回调；后续生产调用必须先由专项计划建立 `数据操作.*` 模块。

## 4. 固定实现合同

1. `结构写入会话` 不可默认构造、不可复制、不可移动；只能由 `结构写入执行器` 在当前线程局部构造，并以左值引用同步交给回调。
2. 会话构造函数、原始令牌、许可、仓库引用、精确写集和收口函数均不得导出；公开能力面只保留窄结构操作和只读结果。
3. 执行器是唯一许可所有者，只取得一次独占许可；不得先取共享再升级，不得嵌套取得许可。
4. 回调不得保存会话指针 / 引用，不得跨线程调用，不得返回候选、仓库引用、许可或原始令牌。
5. 关系写集保存完整当前关系句柄；索引写集保存 `主键 + 预期节点完整句柄`。
6. 回调异常、显式撤销或最后发布前非成功结果必须在许可释放前逆序撤销。撤销须继续尝试本次剩余写集并汇总内部不一致，不得扫描仓库猜测残留。
7. 候选确认只在调用方已完成最后发布和同会话读回后执行。最后发布后读回异常或候选确认异常属于逻辑错误，不得用撤销掩盖已发布事实。
8. 会话析构不得执行仓库写入；执行器必须在会话离开作用域前完成提交或撤销收口，许可仅由既有 RAII 对象释放。
9. 候选隔离由“同域独占许可持续持有 + 会话能力封闭”共同成立，不得只以句柄未导出作为证明。执行器必须把许可持有到提交 / 撤销完成；同会话读回只走内部令牌窄入口。
10. 本切片仅允许独立自检直接提交回调；生产代码中只有后续计划登记的 `数据操作.*` 模块可导入执行器。领域服务、线程路由、消息和组合器不得把执行器当作通用写入逃生口。

## 8. 实施步骤

### S1 会话能力面

建立不可移动会话、私有仓库引用和精确写集；所有窄操作先复核会话线程、阶段和参数，再调用 #263 结构化入口。

### S2 执行与收口

建立执行器的接线拒绝、一次独占许可、同步回调、显式提交 / 撤销、异常捕获、逆序撤销和 RAII 释放。许可取得失败为逻辑内返回且结构不变化；仓库内部不一致或撤销不能回到前态为追根因停止。

### S3 独立自检和登记

新增独立真模块自检；入口只导入模块并登记最终回归顺序 710，不展开场景、循环或逐项输出正文。

## 10. 接口漂移与完成边界

若实际实现必须修改四仓库、结果头、结构事务接线或协调器，无法在一次独占许可内用 #263 入口完成精确逆序撤销，或发现任一公开读取能越过同域独占许可观察中途结构，执行窗口不得改代码，必须退回设计。

完成只证明不透明写入会话和执行器基础可用，不证明领域服务已迁移、多仓库业务事务已接入生产、兼容入口已收口、性能已优化或 `#214` 已恢复。

## 11. 实施结果

已新增 `会话.结构写入`、`执行器.结构写入` 和 `自检.结构写入会话` 三个真模块。会话不可默认构造、复制或移动，只能由执行器在当前线程栈内构造；执行器每次只取得一次独占许可。主信息 / 节点候选、完整关系句柄及“主键 + 预期节点完整句柄”均由会话写集封闭保存，提交前强制完成同令牌读回，失败、异常和未提交决定均在许可释放前精确逆序撤销。

入口只增加自检模块导入、一个值式结果、一次阶段 710 登记和汇总；四仓库、统一结果头、结构事务接线、协调器、领域服务、线程路由、控制面板和 SQL 均未修改。A01-A12、#263 A01-A14、Debug x64 0 警告 0 错误、完整自检退出码 0、严格规范、禁止范围扫描及连续 20 / 20 轮完整自检均通过。详细证据见 `实施记录/20260713_SERVICE-DATA-S1_不透明结构写入会话与执行器代码实施_Codex断点清单.md`。
