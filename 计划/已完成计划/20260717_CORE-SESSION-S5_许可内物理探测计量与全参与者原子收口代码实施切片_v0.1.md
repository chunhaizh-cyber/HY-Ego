以下签名已在 `532e3e5` 形成或保留，v0.6 不授权改名、新增关系仓库 API 或改变候选 / 关系变更生产语义；其完整 A01—A24 验证仍是剩余工作：

```cpp
    const 结构事务令牌& 令牌) const;

    主信息仓库& 主信息,

    结构事务接线 接线 = {});

bool 结构事务接线::已接域() const;

bool 结构事务接线::接线形态有效() const;

    const std::shared_ptr<void>& 运行期状态,
    const 结构事务令牌& 当前独占令牌);
```

会话结果使用强类型状态，不以空 `optional` 混合空闲、拒绝和内部不一致。查询只允许当前线程、写入中、未决定、无首次失败且写集为空。

事务参与者假定生命周期：

```text
准备提交 -> 确认待发布 -> 完成发布 noexcept
未准备 / 已准备 / 已确认待发布 -> 完成撤销
```

会话假定生命周期：

```text
写入中 -> 确认中 -> 已确认待发布 -> 已发布
写入中 / 确认中 / 已确认待发布 -> 撤销中 -> 已撤销
撤销失败 -> 协调器域隔离
```

关系变更逐签名合同：确认不改变关系版本；同一原许可撤销逐字恢复写前记录和写前版本；写前句柄恢复有效，写后句柄过期；跨会话因令牌身份不一致返回 `许可拒绝`，且写后记录、写后句柄和能力阶段零变化。许可释放后原许可退出活动表，会话最终发布关闭会话层撤销入口。

完整接域合同：合法形态只有“全部字段为空的明确未接域兼容接线”和“八项能力全部有效的完整接域”。`已接域()` 必须同时要求域 / 纪元 / 状态、共享与独占许可取得、共享与独占令牌验证、撤销失败隔离回调有效；部分字段接线在四仓库、执行器和接收可选接线的特征值服务入口拒绝，不得落入未接域直写路径。任何包装接线必须逐项转发全部能力；执行器任一撤销失败都必须在当前许可析构前调用隔离回调。

最终总成合同：`入口.cpp` 只允许把现有 `程序通过` 叶子条件等值重排为一份“门禁名称 + 既有布尔值”的单一值式表，由该表同时计算 `程序通过` 并在失败时输出全部假门禁。不得保留第二份长链、改变任一叶子表达式 / 宏条件 / 退出码或新增自检正文。
