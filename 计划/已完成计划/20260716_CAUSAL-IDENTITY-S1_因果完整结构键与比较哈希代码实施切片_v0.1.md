## 4. 假定正式接口契约

### 4.1 材料

在 `海中鱼巣.领域.材料.因果模式` 新增：

```text
因果完整结构键格式版本 = 1。
因果完整结构键：原因动态键、结果动态键、因果规则版本、格式版本。
因果完整结构键处理状态：已形成 / 入口拒绝 / 不支持 / 内部不一致。
因果完整结构键拒绝原因：无 / 候选不完整 / 原因结果相同 / 版本不兼容 / 算法不变量破坏。
因果完整结构键结果：状态、拒绝原因、可选身份和成功判定。
```

`因果完整结构键::局部完整()` 只检查格式非零、两把动态键各自完整和因果版本非零；原因不同及版本兼容由算法入口裁决。

结果 DTO 默认初始化为 `入口拒绝 / 候选不完整 / 无身份`。合法组合仅有：已形成 + 无原因 + 有合法身份；入口拒绝 + 候选不完整 / 原因结果相同 + 无身份；不支持 + 版本不兼容 + 无身份；内部不一致 + 算法不变量破坏 + 无身份。`成功()` 只接受第一种完整组合。

### 4.2 算法

在 `海中鱼巣.领域.算法.因果模式` 新增：

```text
因果完整结构键版本兼容。
因果完整结构键相同。
计算因果完整结构键稳定哈希。
形成因果完整结构键。
```

格式 1 只接受当前 `动态模式规则版本 / 动态模式规则版本 / 因果模式规则版本` 兼容行。原因和结果角色有序；交换角色形成不同身份。完整比较在任一键无效 / 不支持时返回假，不能把两个默认无效对象当同一身份。哈希固定混入格式、原因角色、结果角色和因果版本，只供召回；哈希入口返回 `std::optional<std::uint64_t>`，无效 / 不支持身份返回空值，不进入候选桶。

### 4.3 用途概率分量

`因果概率分量` 改为：

```text
因果完整结构键 因果身份。
机会数、正结果数、负结果数、Q10000。
```

删除重复三元组字段。`算法.用途事件` 只调用因果模式算法形成身份；不得复制比较 / 哈希实现。完整因果候选及机会组继续保留在用途冻结来源。
