## 1. 定位

本切片只把已完成的协议、队列、上行桥和领域路由接入自我线程，验证至少两轮固定安全根治理、幂等和停止收口：

```text
自我线程初始化成功
-> 等待治理队列
-> 冻结一批并释放队列锁
-> 逐条调用领域路由
-> 权威读回与非权威观察
-> 返回等待下一批
-> 停止生产 / 桥接 / 队列
-> join 自我线程
```


## 10. 实施步骤

### S5-0 执行前接口总复核

逐项读取 #171-#178 实施记录和实际导出，建立符号 / 行为比对表。任何不一致先退回对应计划修订，不允许在 S5 兼容性大改。

### S5-1 接入治理生命周期

扩展自我线程构造或启动依赖，初始化成功后等待并冻结治理批次；每条消息在队列锁外调用领域路由。空队列继续条件变量等待。

### S5-2 两轮与幂等验收

第一轮执行固定安全根 E01-E14；返回等待后提交重复幂等材料和第二轮观察 / 回执材料。重复不得新增权威结构，第二轮材料不得混入第一轮。

### S5-3 并发和停止收口

至少两个生产者提交；处理中发出停止，验证停止优先、未落账候选不伪造成任务失败、停止后拒绝新消息，并按生产者 -> 桥 -> 队列 -> 自我线程顺序收口。

### S5-4 系统验证与收口

运行 Debug、自检和 `--headless` 受控停止，执行锁和禁止能力扫描；生成正式实施记录、归档全部 SELF-GOV 活动计划状态，提交并推送。

## 11. 逻辑内返回与追根因解决

逻辑内返回：无治理材料、等待被停止唤醒、重复 / 过期 / 乱序消息、桥或 mailbox 写前拒绝、停止后新消息、只读观察失败。不得生成业务失败事实。

追根因解决：初始化成功却无法进入等待、冻结批次持锁调用领域服务、同一幂等材料重复落账、两轮材料混批、停止后仍写权威结构、join 不返回、死锁、忙轮询，或路由内部异常后继续下一依赖步骤。
