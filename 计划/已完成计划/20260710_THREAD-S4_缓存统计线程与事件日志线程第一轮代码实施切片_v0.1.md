## 7. 实施步骤

### S4-1 当前入口复核

复核统计服务非权威缓存材料入口、日志系统结构化事件入口、运行宿主停止收口入口。

### S4-2 缓存统计线程壳

缓存统计线程只消费缓存刷新消息并维护非权威材料。缓存刷新可合并、丢弃过期或拒绝满载请求，但不得影响权威结构。

### S4-3 事件日志线程壳

事件日志线程只消费事件日志消息并写审计材料。落盘成功不等于事实成立，落盘失败不得改变已经由领域服务确认的结构。

### S4-4 停止收口

停止时先停止缓存刷新生产，再收口缓存统计线程；事件日志线程尽力写出已复制审计材料，失败只产生人读诊断。

### S4-5 默认入口自检

```text
THREAD-S4-A1 缓存统计线程可启动和停止。
THREAD-S4-A2 缓存刷新消息可合并或局部拒绝。
THREAD-S4-A3 清空缓存不改变权威结构。
THREAD-S4-A4 事件日志线程可启动和停止。
THREAD-S4-A5 事件日志落盘失败不裁决机器事实。
THREAD-S4-A6 禁止 SQL / ADO / WebView / 外设接入。
```

## 8. 失败收口策略

```text
缓存刷新满载只返回局部失败或合并材料，不写业务失败事实。
事件日志队列满载或落盘失败只写人读诊断，不回滚权威结构。
持锁期间需要落盘、flush、join 或等待队列时，停止并登记防死锁问题。
```
