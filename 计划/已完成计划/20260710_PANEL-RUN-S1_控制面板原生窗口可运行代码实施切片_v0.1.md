## 7. 新增结构说明

### 7.1 控制面板窗口快照

~~~text
用途：窗口会话的只读 DTO。
写入方：main 和控制面板服务只读入口。
读取方：控制面板窗口。
生命周期：当前窗口会话；关闭即销毁。
验证：来源显示服务、禁止写事实、六类分类完整、操作请求未执行、结构计数不变。
~~~

### 7.2 控制面板窗口运行结果

~~~text
用途：记录窗口是否显示、是否正常关闭及人读失败阶段。
写入方：控制面板窗口运行入口。
读取方：main 的退出码和运行日志。
生命周期：单次窗口运行。
验证：正常关闭与追根因错误明确分离，不参与业务裁决。
~~~

## 8. 实施步骤

### S0 当前事实复核

再次核对允许文件、控制面板服务签名、统计快照和入口末尾常驻逻辑。若同一目标文件出现其他窗口未提交修改，停止并登记脏树风险。

### S1 窗口会话结构与前置拒绝

新增控制面板窗口快照、运行结果和窗口类。先验证总览来源、禁止写事实、六类树候选和操作请求壳边界，再进入窗口资源创建。

### S2 原生窗口和布局

创建主窗口、TreeView、只读详情区、节点 / 关系 / 索引计数、刷新按钮、操作请求按钮、关闭按钮和状态栏。处理 WM_SIZE，保证最小尺寸下中文标签不重叠。

### S3 只读材料显示

把控制面板服务材料和启动结构统计快照映射到控件。TreeView 只插入六类树根，不生成虚假子节点。

### S4 交互

~~~text
选择树根：只更新右侧详情。
刷新：只重读控制面板服务材料。
操作请求：只生成请求壳并显示动作未执行。
关闭：统一销毁窗口。
~~~

### S5 入口与生命周期

入口新增 --headless。无参数默认运行窗口；--self-test-exit 保持最高优先级；--headless 使用原常驻循环。WM_TIMER 检查停止信号，停止时关闭窗口并退出消息循环。

### S6 失败收口

合法只读前置通过后，窗口类、主窗口、必需控件、材料刷新或消息循环不及预期时进入追根因解决：记录失败阶段、销毁已创建资源、返回非零，不显示空白窗口继续运行。

### S7 验收和治理同步

执行 PANEL-RUN-A1 至 A12，生成代码实施断点清单，归档计划并同步计划索引和项目记忆。

## 11. 实施默认决策

| 编号 | 问题 | 本轮决策 |
| --- | --- | --- |
| PANEL-RUN-Q1 | 是否采用原生 Win32 而不是 WebView / 浏览器 | 采用原生 Win32，避免提前引入 HTML / JSON / HTTP 和外部运行通道 |
| PANEL-RUN-Q2 | 是否无参数默认打开控制面板，--headless 保留原常驻模式 | 无参数默认打开窗口，--headless 保留原常驻模式 |
| PANEL-RUN-Q3 | 第一轮是否只展示六类树根、总览和启动结构计数 | 是；真实领域树子节点另建 DTO / 只读聚合计划 |
| PANEL-RUN-Q4 | 操作按钮是否只生成请求壳、不执行领域动作 | 是，保持控制面板不写机器事实 |
| PANEL-RUN-Q5 | 是否修改控制面板服务和其他领域服务 | 不修改，本轮仅新增界面适配和入口生命周期 |

上述决策来自用户“生成控制面板，并使控制面板可以运行”的明确实施目标和当前最小风险路线，不授权扩大到 SQL / WebView / 真实领域树或业务写操作。
