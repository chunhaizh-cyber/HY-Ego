```text

-> 私有强类型回执队列

-> 后续 SG-S3B 领域只读复核
```

## 5. 假定接口契约

假定 #171 的稳定句柄材料可复用于任务结果回执，但本切片不把自我治理消息直接塞入任务工作线程。

预期新增 `任务结果回执材料`，至少包含：

```text
回执协议版本、回执编号、发生时间戳、幂等键、同任务序号。
目标任务、实际结果状态、动态证据、目标方法、目标动作入口、场景、主体。
材料摘要、材料性质和承载机器事实=false。
```

预期任务工作线程提供：

```text
发布强类型任务结果回执：只接收执行方已经形成的证据句柄，不自行伪造结果。
取出一条任务结果回执：返回值副本或移动值，不暴露私有队列引用。
只读回执数量和停止状态。
```

现有原始 `运行消息::结果回执` 继续作为 THREAD 兼容传输材料；本切片不得让原始回执和强类型回执互相裁决。真正接线由 S3B / S5 完成。

## 10. 实施步骤

### S3A-0 执行前复核

读取 #171 正式协议和当前任务工作线程，确认强类型回执模块仍不存在，现有公开 API 未被并行修改；漂移则退回修订。

### S3A-1 建立回执协议

新增值类型、固定句柄槽位、纯值准入、重复 / 冲突 / 顺序拒绝和局部返回枚举。业务成员名使用中文，不包含服务或仓库对象。

### S3A-2 扩展任务工作线程

私有持有强类型回执容器与必要同步；发布入口只接收调用方提供的完整证据，取出入口返回副本或移动值。不得由线程根据日志、文本或原始 uint64 猜测句柄。

### S3A-3 入口验收

覆盖完整回执、缺字段、旧版本、重复、冲突、乱序、停止态、公开取出和不暴露队列引用；证明所有负例不改变四类仓库数量和任务 / 需求读回。

### S3A-4 收口

构建、自检、扫描，生成正式实施记录并归档计划；提交和推送当前分支。

## 11. 逻辑内返回与追根因解决

逻辑内返回：字段缺失、句柄形状无效、版本不匹配、时间戳 / 幂等键 / 序号为零、重复、冲突、乱序、容量满或停止态。

追根因解决：完整回执入队后不可取出、取出内容漂移、同一回执多次上行、线程根据裸数值伪造稳定句柄，或发布 / 取出改变任务、需求、状态、动态结构。
