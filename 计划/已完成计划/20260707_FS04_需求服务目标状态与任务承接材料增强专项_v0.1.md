# FS-04 需求服务目标状态与任务承接材料增强专项 v0.1

日期：2026-07-07
状态：已完成 / 已归档计划 / S0 扫描与 S1-S4 独立代码切片均已完成验证 / 未迁移旧函数

## 1. 任务定位

本计划承接 FS-04 需求树入口只读扫描和逻辑迁移包归并材料，目标是把可进入第一轮增强的需求服务能力转成后续可执行路线。

本计划是确认对象。`实施记录/20260707_FS04_需求树入口逻辑迁移包归并表.md` 只作为生成本计划的证据材料，不是本窗口最终目标。

本计划不迁移旧函数体，不新增旧需求主信息字段，不直接进入 C++ 实施。代码实施必须另建切片，并再次确认允许文件、禁止文件和验证方式。

## 2. 纳入范围

第一轮只纳入：

```text
FS-04-A：需求身份与目标状态读取。
FS-04-B：需求主体、场景与目标宿主材料读取。
FS-04-C：需求承接任务与初始化上下文。
FS-04-D：需求结算权威记录与摘要分流的读取 / 分流边界。
```

第一轮目标不是重建完整需求树，而是补齐需求服务向任务服务提供稳定承接材料的最小读取和拒绝边界。

## 3. 排除范围

本计划排除：

```text
FS-04-E：父子需求角色、阻塞与权重读取。
FS-04-F：逻辑组织需求、获取途径与目标语义。
FS-04-G：需求方向签名、枚举目标和方法候选准入。
FS-04-H：需求树结构化更新指令。
FS-04-X：SQL / 控制面板 / 旧主信息摘要。
```

这些内容进入后续专项，不进入本计划的第一轮代码实施切片。

同时继续禁止：

```text
旧需求类或旧需求主信息类函数体迁移。
旧需求主信息字段迁移。
SQL / 控制面板 / WebView / 显示写入。
D455 / 体素 / 外设。
把方向 bool、目标语义文本、获取途径枚举或结算摘要当作机器事实。
```

## 5. 设计条款清单

### 必须实现

```text
需求身份读取必须经需求服务复核节点类型。
需求目标必须读取为目标状态节点，不得降格为 I64。
需求主体、目标宿主和场景材料必须可由需求服务提供给任务承接路径。
任务承接材料缺失时，任务服务不得创建有效任务承接壳。
需求结算摘要不得作为权威需求事实，只能由需求结算记录或后续非权威缓存重建。
拒绝路径必须保证节点、关系、状态和任务结构不变化。
```

### 禁止实现

```text
不实现父子需求角色视图。
不实现逻辑组织需求。
不实现获取途径枚举。
不实现需求方向签名或方法候选准入。
不实现需求树统一更新指令。
不实现删除需求或删除实例状态。
不接 SQL / 控制面板 / 显示镜像。
```

### 后续待补

```text
父子需求角色和阻塞视图专项。
逻辑组织需求和获取途径专项。
需求方向、枚举目标和方法候选专项。
需求树结构化更新指令专项。
显示层只读视图专项。
非权威需求摘要缓存专项。
```
