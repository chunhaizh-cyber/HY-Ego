## 5. 结构合同

### 5.1 根角色

```text
存在根 -> 已有存在节点；
动态根 -> 已有抽象动态节点；
关系根 -> 已有二次特征节点；
因果根 -> 已有因果引用节点；
四根句柄和稳定键互异；
四根均无作为目标的概念上下位当前关系。
```

根身份只能由对应原子业务服务创建。#275 不得直接创建存在、动态、二次特征或因果引用节点，也不得新增无来源因果入口。

### 5.2 三类关系

```text
实例支持概念：实例和概念归入同一已复核根类别；根不能作为实例端点。
概念上下位：上位和下位归入同一根类别；允许多父和共享节点；拒绝自环、跨类和成环。
概念定义材料：源是已归类概念，目标是允许的长期定义材料；不改变概念类别。
```

关系仓库是唯一权威关系源。索引、缓存、树投影、图投影、控制面板或统计结果不得代替完整关系句柄、版本、状态和端点复核。

### 5.3 概念角色与首次挂载

```text
根节点天然是概念。
普通概念必须通过概念上下位反向路径到达本次已复核同类别根。
首次挂载允许 已归根上位 -> 同类别且没有任何概念角色关系的全新下位候选。
首次关系提交前候选不是概念，提交后才成为可读概念。
实例支持概念只接受已经归根的概念目标，不承担首次挂载。
```

首次挂载候选不得是根，不得已有概念上下位的上位 / 下位、实例支持目标或概念定义材料源角色。已有归根概念可接受同类别多父；不能到达本次根的旧分量不得自动接入或修复。

### 5.4 定义材料白名单

第一轮只允许：

```text
基础信息、存在、场景、特征、抽象状态、抽象动态、二次特征、因果引用。
```

其余类型拒绝。状态和动态必须用运行期临时来源关系在同一许可内区分抽象与实例，不能只看节点枚举。

## 10. 实施步骤

### S1 值式根角色和关系材料

定义根类别、稳定键、根角色组、关系请求、完整关系证据和业务结果；公开类型不得含内部能力。

### S2 根角色复核

以共享许可初读四根完整身份、稳定键、类型、互异性和无上位关系；进入写入时使用 #276 入口在同一独占许可内复核唯一主键组和按目标来源关系组，形成可重复验证的值式材料，不创建根节点或根登记表。

### S3 实例支持与定义材料

实例支持只接受已归根概念目标；定义材料按固定八类白名单准入。实现重复幂等、基于完整关系记录的正向 / 反向读取和关系历史保留。

### S4 上下位、多父和防环

在同一独占许可内区分已归根下位和首次挂载候选，执行反向归根、正向防环和关系创建。首次候选只作叶节点接入；同类多父、共享节点通过；根作下位、无根分量、自环、跨类和多跳环在第一笔写入前拒绝。

### S5 组合器

组合器只接收已由公开业务服务形成的根角色和创建型关系请求，不接生命周期、活动快照或主装配发布。

### S6 自检和入口

独立自检覆盖 A01-A16。仅 Debug|x64 在数据操作模块编译固定写点故障自检入口，生产业务入口不接故障参数；入口只增加 import、一次调用、一次 755 登记和值式总汇总。

### S7 收尾

验证通过后登记 755、归档 #275、解除 #269 的本项依赖；只提交并推送本切片相关文件。#276 只作为已完成前置回归，不得重复实施。
