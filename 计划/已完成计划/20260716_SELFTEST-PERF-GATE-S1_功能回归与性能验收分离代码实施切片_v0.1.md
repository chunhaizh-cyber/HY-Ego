## 2. 目标

把默认确定性功能回归和环境敏感性能验收拆为两个显式入口：

```text
--self-test-exit
    -> 不登记阶段 790
    -> 功能自检决定退出码

--warehouse-performance-self-test-exit
    -> Debug 下按原顺序登记并执行阶段 790
    -> 原功能自检和性能结果共同决定退出码
```

不得修改性能基线算法、门槛、签名、关系仓库或中央运行器语义。

## 5. 实施范围

### S1：运行模式

在 `入口.cpp` 新增显式性能验收模式和参数 `--warehouse-performance-self-test-exit`。该模式自动包含自检后退出；与 `--self-test-exit` 同时出现时按显式性能严格超集执行，参数顺序无关。

Release 收到显式性能参数时在结构装配前退出 2，并输出明确人读原因。

### S2：条件登记

默认功能模式不登记 790；显式 Debug 性能模式继续按最终回归顺序 790 登记现有 `运行关系仓库性能基线自检()`。

必须分别保存“性能模式已请求”“790 实际登记成功”“790 实际执行通过”证据。默认未登记不能映射为登记失败；显式未登记必须失败。

### S3：独立自检

新增 Debug 真模块：

| 文件 | 模块 | 工程项 |
| --- | --- | --- |
| `海中鱼巣/自检.性能门禁分离.ixx` | `海中鱼巣.自检.性能门禁分离` | `ClCompile` |

模块只接收值式模式和登记证据，验证条件登记、两模式互斥边界、退出码接线和旧参数不受影响；阶段 795 唯一登记。完整自检正文不得写入入口。

### S4：验证与收口

运行默认 Debug / Release 功能回归和显式 Debug 性能验收。显式性能验收允许按真实环境退出 0 或 1，但必须证明退出码与 790 结果一致；退出 1 不得写成性能通过。默认 Debug 功能回归必须退出 0 并连续 20/20。
