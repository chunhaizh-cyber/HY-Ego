## 2. 确认范围

确认后只允许生成任务后续设计，范围包括：

```text
任务状态机阶段和状态迁移边界。
任务筹办材料、筹办结果和拒绝矩阵。
任务执行桥的输入 / 输出 / 来源动作边界。
任务运行统计、权重评分、错误码治理、运行包和治理协议的分流设计。
后续代码实施切片候选和依赖条件。
```

## 3. 目标服务和目标结构

```text
目标服务：任务服务、需求服务、方法服务。
目标结构：任务生命周期状态、任务筹办材料、任务执行请求材料、任务实际结果状态、任务方法关系、任务运行统计候选、治理协议候选。
```

第一轮设计必须继续保持：

```text
任务必须来源于有效需求。
任务目标状态必须来自需求目标状态。
任务完成前必须已有实际结果状态。
任务选择方法关系由任务服务封装。
方法执行权限仍来自任务授权、任务承接和任务选择方法。
线程、std::async、派发器、控制台命令、日志标签和动作文本不得成为动作来源。
```

## 4. 明确排除项

```text
不修改 C++。
不迁移旧函数体。
不新增旧任务主信息字段。
不实现工作线程、std::async、派发、取消、超时或外设等待。
不实现真实动作执行分派。
不实现完整任务运行统计或治理协议代码。
不接控制面板 / SQL / ADO / WebView / D455 / 体素 / 外设。
```
