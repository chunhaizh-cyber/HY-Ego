## 1. 定位

本切片把当前原生控制面板从“两棵真实树 + 四个空分类根”升级为单个 TreeView 同时显示六类真实只读结构投影：

```text
世界树真实 DTO
概念活动图四根投影
语素树真实 DTO 和关系目标
两个根需求及真实结构材料
由根需求反向发现的任务
由任务选择关系发现的方法
-> 六类完整候选快照
-> 一次性替换窗口快照
-> 单个 TreeView 同时挂六个分类根并全部展开
```

## 10. 实施步骤

### PANEL-TREES-S1-0 依赖与实际接口复核

逐项读取 #182-#186 实施记录、概念树投影导出、来源关系读取、当前控制面板和需求 / 任务 / 方法只读入口。形成符号比对表；任何漂移按第 12 节退回。

### PANEL-TREES-S1-1 DTO 和服务聚合

调整树节点、树结构和窗口快照 DTO；控制面板服务建立六类完整候选入口和全量复核，不返回部分成功。

### PANEL-TREES-S1-2 需求任务方法真实投影

任务服务增加按需求反向读取任务；控制面板服务按设计第 8 节组装需求、任务、方法真实节点和关系角色，不扫描节点仓库全表。

### PANEL-TREES-S1-3 概念与语素投影

概念只转换 #186 活动树视图；语素读取真实绑定 / 追溯目标。多父和重复引用按路径处理，无名节点用类型 + 句柄回退。

### PANEL-TREES-S1-4 单 TreeView 六分类

窗口重建一次插入六个分类根及真实子根，默认展开；分类导航改为定位项。任一控件插入失败恢复旧快照，恢复失败关闭窗口。

### PANEL-TREES-S1-5 自检、窗口验收与收口

完成 DTO、拒绝、预算、结构不变、自检构建和真实窗口验收；记录截图或等价视觉证据及 TreeView 项读回，生成实施记录、归档、提交和推送。

## 11. 逻辑内返回与追根因解决

逻辑内返回：只读请求越权、请求写事实 / 文本 / SQL 裁决、概念活动图未就绪、世界 / 语素 DTO 或根需求缺失、某类无真实根、预算耗尽。六类候选整体不发布，保留旧快照。

正常成功：无名节点以类型 + 句柄回退；方法等非概念关系重复生成引用叶项；六类完整候选一次发布。

追根因解决：有效根通过后领域读回自相矛盾、概念活动图出现环 / 跨根 / 版本外边、六项 DTO 分类或数量不一致、TreeView 插入失败、节点引用悬空、刷新导致权威结构变化。停止依赖路径，不显示空树伪成功。
