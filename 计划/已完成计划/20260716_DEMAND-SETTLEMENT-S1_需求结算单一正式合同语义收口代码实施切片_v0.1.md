## 1. 目标

收口当前正式模块中的需求结算语义：

```text
值相等才允许写唯一已满足结算。
值不相等不写需求最终结算：#225 新任务结果治理路径继续转待重筹办；#225 之外既有通用组合入口不完成任务且两类结构零变化；已完成任务直接独立结算只拒绝本次结算。
正式结算只绑定上游已创建的一条动态和零或一条因果引用；需求服务只复核，需求数据操作只绑定。
完整同义重复幂等读回；任一来源字段异义冲突。
第二条正式结算或多记录读回进入内部不一致，不选择最近记录。
```

本切片不修改 `需求服务.h` 历史兼容路径，不新增节点、关系、主信息字段、运行期公开签名或自检阶段。

## 4. 实施范围

### S0：执行前复核

重新读取 Git、索引、队列、本计划、JY-370 设计修订断点、E95 实施前复核断点、三份正式代码和两份自检，确认上述符号与行为未漂移、#286 仍为依赖就绪、#283 的 #282 前置已满足且仍等待 #286。若公开签名、关系角色、状态材料或自检所有权漂移，代码零修改退回设计。

### S1：服务写前值相等门禁

在 `需求业务服务::提交独立结算` 形成结算状态规格前拒绝目标值不等；结论固定为 `已满足`。动态必须是上游已有的一条完整动态，可选因果句柄只能全零或一条完整有效句柄，部分句柄写前拒绝。已完成任务直接调用时只拒绝本次结算，不修改任务。需求读取的内部不一致、许可拒绝和版本漂移必须保留分层，不得统一降级成入口拒绝。不新增普通 bool 业务事实，不把未达成写为需求终态。

### S2：写入规格与单一记录读取

把需求结算写入规格收紧为只接受 `已满足`，并拒绝部分可选因果句柄。新增由数据操作模块唯一拥有的 `需求正式结算材料`，挂入 `需求权威材料.正式结算`；既有 `当前结算 / 最近结算状态` 仅由它兼容投影。需求读回对零 / 一 / 多条正式结算明确分支：零条未结算，一条固定关系 1—4 各恰一且完整、关系 5 零或一且若有完整、归属顺序号等于发生时间戳时读回，多条、关系畸形或未满足值进入内部不一致；删除“最近记录”选择算法的正式作用。

### S3：完整结构幂等

在已有唯一结算分支重建并比较 `需求正式结算材料`：结算状态幂等主键、场景、主体、发生时间、状态值，以及来源任务、目标状态、实际状态、动态证据和可选因果引用。全部同义才幂等读回；任一异义均冲突且零写入。首次写入进入独占结构会话后必须再次确认需求的状态型归属为零；并发已占用时本次不写，退出后重读并按完整材料裁决。不得只比结论、哈希或幂等键。

### S4：既有通用组合入口（#225 新结果治理路径之外）完成前裁决

`需求任务方法组合器::完成任务并独立结算` 先读取任务和来源需求，调用正式值相等裁决；只有完整且已达成才提交任务完成和需求结算。未达成或材料不足时两类结构都不变。#225 新任务结果治理路径继续由其既有组合器负责未达成转待重筹办，不在本入口复制该写能力。`组合.运行期业务操作` 只读复核继续转发该入口，公开签名零修改。

### S5：自检与治理收口

扩展既有 760 / 610 自检，覆盖 A01-A16。完成 Debug / Release x64 Rebuild、Release 隔离、完整自检、严格规范和最终 Debug 连续 20 / 20；归档计划并同步治理记录。
