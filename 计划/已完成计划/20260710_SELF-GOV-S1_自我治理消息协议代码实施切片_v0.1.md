## 1. 定位

本切片只建立自我内部治理闭环的非权威消息协议和值级准入复核，不建立治理 mailbox、队列、线程循环或领域落账：

```text
显式治理请求或后续任务上行材料
-> 构造自我治理消息信封
-> 复核协议版本、白名单、公共字段和类型所需句柄
-> 复核过期、幂等冲突和同任务顺序材料
-> 返回准入或拒绝材料
```

消息、复核结果和拒绝原因都只是运行期材料，不是需求、任务、方法、状态、动态、因果或权限事实。

## 4. 实施范围

### 4.1 协议值类型

新增独立 C++20 模块，至少定义：

```text
自我治理协议版本。
自我治理消息类型。
自我治理来源角色。
自我治理服务角色。
自我治理消息优先级。
自我治理只读材料类型。
自我治理材料性质。
自我治理过期策略。
自我治理拒绝原因。
自我治理只读材料句柄组。
自我治理消息。
自我治理准入结果。
```

第一轮消息白名单固定为：

```text
任务结果复核请求。
派生需求请求。
观察材料准入请求。
执行前许可预判请求。
自检缺口候选。
缓存刷新请求。
事件日志请求。
停止请求。
```

第一轮来源角色至少覆盖运行宿主、任务管理上行桥、任务 / 需求分析入口、外部材料适配入口、已确认自检入口和领域落账观察桥。服务角色至少覆盖存在、场景、特征、状态、动态、因果、需求、任务、方法、统计和日志边界；枚举只用于请求路由材料，不裁决服务调用权限。

### 4.2 消息信封

消息信封至少包含：

```text
协议版本。
治理消息编号。
治理消息类型。
来源角色、来源服务和目标服务。
发生时间戳。
优先级。
幂等键。
来源任务序号或同任务顺序号。
批次编号。
非权威材料摘要。
只读材料类型和材料性质。
过期策略及截止时间戳材料。
只读材料句柄组。
```

只读材料句柄组使用固定命名槽位，至少区分：

```text
主体。
场景。
目标宿主。
目标需求。
目标任务。
目标方法。
目标动作入口。
目标材料。
目标特征定义。
实例特征槽位。
当前特征状态材料。
目标状态。
实际结果状态。
动态证据。
```

`目标特征定义` 与 `实例特征槽位` 必须是两个独立句柄槽位，禁止合并。未使用的可选槽位使用空句柄；某消息类型要求的槽位必须通过 `句柄有效` 复核。

### 4.3 纯值复核入口

协议模块只允许提供不持有仓库引用的纯值复核入口，覆盖：

```text
公共信封字段完整性。
消息类型、来源角色、来源服务、目标服务和材料类型白名单。
各消息类型必填句柄矩阵。
稳定句柄的仓库编号、节点编号和版本号非零。
停止请求必须使用停止优先级，普通消息不得冒用停止优先级。
任务相关消息的来源任务序号必须非零。
截止时间已过的材料拒绝。
同一幂等键、不同目标或不同材料摘要的冲突拒绝。
同一任务序号倒退拒绝。
完全重复材料返回重复消息逻辑结果，不生成新事实。
试图承载机器事实或执行许可结论的材料拒绝。
```

协议层只复核句柄结构完整性。节点当前版本、节点类型、关系、任务授权、任务承接和任务选择方法等语义必须由后续生产者适配器、SG-S3B 和领域服务在锁外重新复核；本切片只预留对应拒绝原因，不得假称已经访问仓库完成语义验证。

### 4.4 类型所需材料矩阵

| 消息类型 | 第一轮必填材料 | 本切片边界 |
| --- | --- | --- |
| 任务结果复核请求 | 目标任务、实际结果状态、动态证据、任务序号 | 只复核信封和句柄形状，不裁决任务完成 |
| 派生需求请求 | 目标宿主、目标特征定义、实例特征槽位、当前特征状态材料、目标状态 | 只形成候选，不创建需求 |
| 观察材料准入请求 | 主体、场景、目标材料 | 外部材料不直接成为世界事实 |
| 执行前许可预判请求 | 目标任务、目标方法、目标动作入口、任务序号 | 不新增授权 bool、状态节点或权限事实 |
| 自检缺口候选 | 主体、场景、目标材料 | 第一轮不自动派生需求或修复结构 |
| 缓存刷新请求 | 至少一个目标需求、任务或材料句柄 | 非权威，可拒绝或后续重建 |
| 事件日志请求 | 至少一个目标需求、任务、动态证据或材料句柄 | 只作审计候选 |
| 停止请求 | 无业务句柄要求；来源必须是运行宿主 | 只控制生命周期，优先级最高 |

## 10. 逻辑内返回与追根因解决

逻辑内返回：

```text
协议版本、消息编号、类型、来源、时间戳、幂等键或批次缺失。
必填句柄为空或句柄三元组不完整。
消息类型、来源角色、服务角色或材料类型不在白名单。
停止优先级不匹配、材料过期、幂等冲突、完全重复或同任务序号倒退。
材料性质试图承载机器事实或执行许可结论。
```

这些路径必须在任何机器结构写入前返回，且四类仓库数量保持不变。

追根因解决：

```text
公共字段和类型矩阵已经通过，但复核结果与输入材料不一致。
同一输入在同一协议版本下得到不同准入分类。
拒绝或重复材料导致节点、主信息、关系、索引或业务状态变化。
协议模块出现领域服务调用、仓库写入、线程启动或日志裁决路径。
```

追根因路径必须停止 #171 依赖链并记录真实阻塞，不得继续生成或执行 SG-S2。
