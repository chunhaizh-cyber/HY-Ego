## 6. 默认裁决

| 编号 | 裁决 |
| --- | --- |
| SELF-INIT-Q1 | 自我线程使用独立 `海中鱼巣/线程/自我线程.ixx`，不把初始化塞入运行宿主线程 |

## 7. 实施步骤

### S0 当前入口复核

```text
读取允许代码文件头部规则注释。
确认入口仍在主线程直接调用语素和世界树初始化。
确认需求服务仍拒绝普通入口主体等于目标宿主。
确认特征服务、语素服务和状态服务所需读取入口仍存在。
确认当前分支和 worktree，不覆盖无关修改。
```

### S1 需求初始化服务

```text
新增 海中鱼巣/领域/初始化.需求.ixx 和最小文件头规则。
定义显式初始化参数、单根需求结果和两个根需求初始化结果。
经特征服务创建安全值 / 服务值定义、实例槽位和当前特征值材料。
经语素服务创建概念入口并绑定显式稳定键。
经需求服务和状态服务创建安全根需求、服务根需求及抽象目标状态。
读回两个根需求的主体、目标宿主、场景、目标特征、当前材料和目标状态。
```

### S2 需求服务最小入口补齐

```text
在需求服务新增创建自我目标状态需求入口。
入口只允许主体和目标宿主为同一个有效存在，并要求有效场景、特征和当前值。
复用既有关系槽位和目标状态结构，不新增主信息字段或节点类型。
普通完整需求入口继续拒绝主体等于目标宿主。
前置通过后的写入或读回不及预期进入追根因解决。
```

### S3 自我线程模块

```text
新增独立 C++20 模块 海中鱼巣/线程/自我线程.ixx。
实现启动、线程入口、等待初始化、请求停止、收口等待和只读初始化快照。
线程入口依次调用语素、世界树、需求初始化服务。
初始化快照仅在三个阶段全部成功后一次性发布。
成功后线程驻留等待停止，不调用任务、方法或治理服务。
重复启动复用初始化结果，不重复写结构。
```

### S4 入口和工程接线

```text
在工程文件登记自我线程模块和需求初始化文件。
入口构造需求初始化服务、自我线程和显式参数。
移除主线程对语素 / 世界树初始化的直接调用，改为启动自我线程并等待结果。
入口从自我线程只读快照取得现有控制面板和自检所需初始化结果。
初始化失败立即返回，不进入数据库审计投影、控制面板或无窗口常驻循环。
利用自我线程析构或显式收口保证所有提前返回路径均可 join。
```

### S5 简化验收与记录

```text
补 SELF-INIT-A1 至 A6 默认入口验收。
执行重复启动前后结构数量对照。
执行无效稳定键或重复稳定键的写前拒绝负例。
记录构建、运行、自检、静态边界扫描和完成声明。
```

## 9. 验收方式

| 编号 | 验收条件 |
| --- | --- |
| SELF-INIT-A1 | 自我线程实际进入，并在线程入口完成初始化；主线程不再直接调用两个既有初始化入口 |
| SELF-INIT-A2 | 基础语素、世界根、场景、自我存在和核心关系读回通过 |
| SELF-INIT-A3 | 安全值、服务值语素稳定键唯一并分别指向对应特征定义 |
| SELF-INIT-A4 | 安全根需求、服务根需求各一个，目标状态和完整需求承接材料可读 |
| SELF-INIT-A5 | 重复启动不增加节点、关系或索引 |
| SELF-INIT-A6 | 初始化失败不进入后续运行路径，线程停止与 join 可收口 |
