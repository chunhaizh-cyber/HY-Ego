## 1. 目标

建立方法首的结构化权威发现边界：

```text
稳定非名称方法登记根
-> 共享活跃 / 失效生命周期状态
-> 创建方法首的最后发布关系
-> 全部登记项与活跃方法首只读
-> 原地失效并推进生命周期关系版本
```

裸方法节点、日志、显示、SQL、全节点扫描和稳定动作键均不得承担方法发现。


逐项比对假定接口。接口漂移时不得边改边重设计。

### S1：登记根与共享状态

```text
新增方法登记根角色和活跃 / 失效生命周期枚举。
以稳定非名称键初始化唯一登记根。
根绑定角色状态、共享活跃状态和共享失效状态。
重复同材料初始化幂等；同键异根、异键同根和第二根拒绝。
根不创建虚拟存在，不作为可执行方法首。
```

### S2：方法首发布与发现

```text
创建方法首前要求登记根完整。
先创建角色、虚拟存在和活跃生命周期关系。
最后创建根到方法首归属关系作为发布边界。
有效方法首和现有候选 / 动作读取要求已登记且活跃。
全部登记项保留失效项；活跃组按完整句柄排序并去重。
裸登记方法不进入根，不可由通用发现读取。
```

### S3：生命周期失效

```text
只接受已登记活跃方法首。
原地重挂生命周期关系到共享失效状态。
保持关系编号，版本单调推进。
失效后全部登记可读、活跃发现不可读、动作候选不可读。
不删除方法、动作入口、任务关系或稳定动作键。
```

补齐 METHOD-RECALL-S1-A01 至 A12，完成实施记录、治理更新、计划归档、提交和推送。

## 7. 入口拒绝

写前拒绝：

```text
稳定非名称键为 0。
根节点、角色状态或共享生命周期状态不完整。
登记根冲突。
创建方法首时根未初始化。
失效目标不是已登记活跃方法首。
生命周期关系缺失、不唯一、目标错误或版本异常。
```

拒绝必须证明节点、主信息和关系数量不变化；幂等初始化和幂等查询除外。

## 8. 失败收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 根参数无效、根冲突、根未初始化、失效目标无效或已失效 | 逻辑内返回 | 不写结构 |
| 方法角色、虚拟存在或生命周期创建不及预期 | 追根因解决 | 不写最终登记关系，隐藏半结构 |
| 登记关系写成后完整读回不一致 | 追根因解决 | 不返回有效方法首，停止依赖路径 |
| 生命周期重挂后活跃读取仍命中或关系版本未推进 | 追根因解决 | 停止候选与动作依赖 |

不得通过扫描修复、日志修复或删除半结构继续运行。

## 9. 执行前接口复核与漂移退回

必须记录现有角色顺序号、方法创建调用点、关系重挂返回和状态读取语义。以下任一情况视为漂移：

```text
方法服务已形成同功能登记根或生命周期接口。
方法创建签名或调用顺序改变，入口初始化无法覆盖所有生产调用点。
关系仓库重挂不再保留关系编号并推进版本。
#202 固定了不同的权威登记恢复结构。
需要修改核心仓库、任务服务、线程或控制面板才能完成 S1。
```

出现漂移不得修改代码，退回设计窗口修订 #203 及其后续假定契约。
