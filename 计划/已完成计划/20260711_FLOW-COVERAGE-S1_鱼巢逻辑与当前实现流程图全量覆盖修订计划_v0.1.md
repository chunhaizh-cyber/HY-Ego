## 6. 入口拒绝与失败收口

| 场景 | 口径 | 动作 |
| --- | --- | --- |
| 旧仓库 HEAD 或 dirty 差异在执行前变化 | 逻辑内门控返回 | 更新冻结记录后重新扫描，不沿用旧行号 |
| 被引用旧证据文件不存在 | 追根因解决 | 停止把该路径作为依据，恢复真实证据或修订图 / 设计 |
| 当前工作树与正式 HEAD 不同 | 逻辑内门控返回 | 分栏记录 HEAD 与 dirty 事实，不混写 |
| 施工计划尚未实施 | 逻辑内门控返回 | 保持施工图，不生成当前事实图 |
| 代码已实现但施工图与代码顺序不同 | 追根因解决 | 生成偏差清单，决定修图、修设计或另建代码计划 |
| 正式调用方保证输入有效但代码走普通拒绝 | 追根因解决 | 在契约表拆分语境并登记纠偏 |
| 外部 / 自检 / 候选材料无效且写前拒绝 | 逻辑内返回 | 图中保留明确拒绝节点 |
| 图包缺任一必需文件或机械检查失败 | 逻辑内门控返回 | 不计入覆盖完成度 |
