## 5. 可续接逐轮执行协议

### 5.1 续接状态

执行窗口在非 Git 诊断目录建立 `resume-state.tsv`，初始内容必须包含：

```text
冻结候选=0890464414a8ffd5dfc71dcfc80641d9e4f18181
冻结自检blob=74734d038e3baa0be12a46ee1f275273305b7bf5
历史有效轮次=1-42
历史摘要SHA256=4F5C5D7BE94E547527E30361F40F1C5F9D8767076BD5CC6FE41E427B71EA2EDE
下一轮=43
终止状态=未终止
```

v0.1 的截断第 43 轮永久标记为无效，不得复用其日志或退出状态；v0.2 的第 43 轮必须是新启动的全新进程。

### 5.2 单轮工具调用

每次工具调用只能运行一个：

```powershell
.\x64\Debug\海中鱼巣.exe --self-test-exit
```

禁止在同一工具调用中使用 `for`、`foreach`、`while`、递归脚本、后台任务或其它方式启动多个 `海中鱼巣.exe` 进程。第 43—60 轮必须由 18 次独立工具调用完成。

每次单轮调用必须按顺序：

1. 读取 `resume-state.tsv`，确认当前轮号与文件名一致；
2. 确认没有残留 `海中鱼巣.exe`，共享时隙仍由本任务独占；
3. 启动一个全新进程，完整 stdout / stderr 先写 `resume-round-XX.log`；
4. 进程自然结束后记录真实退出码，再解析 TASK 18、B3C 12、B0 20 和失败后专项证据；
5. 追加一行 `resume-summary.tsv` 并记录本轮日志 SHA-256；
6. 只有退出 0、三组行数正确、三组失败数 0 且没有异常行时，才把 `下一轮` 原子推进为 `XX + 1`；
7. 轮末等待 2 秒，再结束本次工具调用。

工具调用自身超时、被取消、日志截断、进程无真实退出码、行数不符或状态文件不一致时，该轮无效并立即停止；不得推进游标、不得在同一执行轮补跑。

### 5.3 完成口径

只有下列两个集合同时成立，才可写“累计 60 个完整轮”：

```text
v0.1 正式集成证据：1—42
v0.2 新鲜完整证据：43—60
```

不得把 v0.1 截断第 43 轮算入累计，也不得把工具超时或无退出码轮次记为通过或失败。
