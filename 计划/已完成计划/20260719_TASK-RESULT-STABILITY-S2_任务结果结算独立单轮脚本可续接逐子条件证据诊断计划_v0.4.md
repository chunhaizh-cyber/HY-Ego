## 5. 独立单轮脚本合同

### 5.1 固化范围

`run-one-round.ps1` 必须完整封装且只封装一轮：

```text
六键状态读取与精确核对
当前轮号、冻结候选、自检 blob、历史摘要和未终止状态核对
残留进程与共享时隙前置检查
一个全新海中鱼巣.exe --self-test-exit 进程
完整 stdout / stderr 先落 resume-round-XX.log
真实退出码、TASK / B3C / B0 行数和失败数解析
失败后 14 / 11 / 200 证据完整性核对
resume-summary.tsv 单行追加
日志 SHA-256
临时状态文件写入、同解析器六键读回、原子替换正式状态
完整通过后等待 2 秒并退出脚本
```

脚本每次调用最多启动一个产品进程；不得循环执行多个轮次，不得后台运行、递归调用或自行推进后继轮。脚本内容在 AST 预检后冻结并记录 SHA-256；第 43—60 轮期间不得修改。

### 5.2 结构化状态

状态继续使用六键 UTF-8 `key=value`，读取统一为：

```powershell
$原文 = Get-Content -Raw -Encoding UTF8 -LiteralPath $状态路径
$状态 = ConvertFrom-StringData -StringData $原文
```

精确键为 `冻结候选`、`冻结自检blob`、`历史有效轮次`、`历史摘要SHA256`、`下一轮`、`终止状态`。禁止多行正则、按换行拆分或手工去 CR。状态原子更新必须先写临时文件，再用同一解析器核对六键和值，最后替换正式文件。

### 5.3 产品启动前脚本门禁

在任何产品进程启动前必须依次通过：

1. 用 `[System.Management.Automation.Language.Parser]::ParseFile(...)` 解析 `run-one-round.ps1`，语法错误数量必须为 0；
2. 记录脚本 SHA-256；
3. 建立 LF / CRLF 两份六键样本并用脚本同一解析函数逐键比较；
4. 使用与真实轮相同的外层 `powershell.exe -File` 形态调用脚本 `-ValidateOnly -Round 43`；
5. 验证调用退出 0、六键完整、下一轮 43、终止状态未终止、脚本未启动产品、进程数 0；
6. 只有上述全部通过，才可初始化正式 `resume-state.tsv` 并申请真实运行时隙。

`-ValidateOnly` 必须走脚本正式状态读取与前置核对路径，但在进程启动语句前明确返回；禁止维护第二套验证解析器。

## 6. 最小外层调用协议

真实第 43—60 轮仍为 18 次独立工具调用。每次外层命令只允许包含脚本路径、轮号和固定 PowerShell 启动参数，例如：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File "D:/海中鱼巣/日志/诊断/TASK-RESULT-STABILITY-S2-R4/run-one-round.ps1" -Round 43
```

后继调用只把轮号机械改为 44—60。外层工具字符串禁止包含制表符、多行 PowerShell 正文、状态解析、日志解析、字符串模板拼接、循环或 JavaScript 插值。每轮只在脚本退出 0、状态游标推进为下一轮且进程 0 后，才允许设计窗口继续授予下一次调用。

工具超时、取消、脚本非零退出、日志截断、无真实产品退出码、状态解析异常、行数不符、失败叶子或脚本哈希漂移都立即停止，不补跑当前轮，不启动后续轮。
