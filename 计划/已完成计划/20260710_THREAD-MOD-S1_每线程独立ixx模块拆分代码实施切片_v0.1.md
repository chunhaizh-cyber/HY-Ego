## 7. 实施步骤

### S0 最小模块编译探针

1. 新建 `运行消息协议.ixx` 的最小合法模块接口。
2. 在项目文件中登记模块接口单元。
3. 由 `入口.cpp` 使用 `import` 导入并引用一个只读协议类型。
4. 执行 Debug x64 构建。
5. 若失败，记录首个真实错误并追根因；S0 通过前不得继续拆线程。

### S1 共享协议和有界队列模块

1. 把运行消息类型、线程角色、优先级、运行消息和轻量结果结构移入 `运行消息协议.ixx`。
2. 把有界队列和入队 / 出队 / 停止结果移入 `有界运行消息队列.ixx`。
3. 保持停止优先、幂等、版本、同任务顺序和局部失败边界。
4. 共享模块不得出现线程类或 `std::thread` 成员。

### S2 单线程模块第一组

1. 建立 `运行宿主线程.ixx`，只承载运行宿主线程。
2. 建立 `外设采样材料线程.ixx`，只承载外设采样材料线程。
3. 分别保留独立状态、停止标记、启动、停止和 join。
4. 构建并运行对应验收。

### S3 任务线程拆分

1. 删除 `任务线程壳` 同时持有两个线程的结构。
2. 建立 `任务管理线程.ixx` 和 `任务工作线程.ixx`。
3. 明确任务调度队列和回执队列的唯一所有权及引用方向。
4. 保持回执只作为领域复核材料，不裁决任务、需求或方法结果。
5. 构建并运行 THREAD-S3 验收。

### S4 缓存与事件线程拆分

1. 删除 `缓存事件线程壳` 同时持有两个线程的结构。
2. 建立 `缓存统计线程.ixx` 和 `事件日志线程.ixx`。
3. 分离各自队列、状态、停止标记和消费入口。
4. 保持缓存非权威、日志不裁决和写入失败不改结构边界。
5. 构建并运行 THREAD-S4 验收。

### S5 入口与旧文件收口

1. `入口.cpp` 改用模块 `import`。
2. 更新 `.vcxproj` 和 `.filters` 登记全部 8 个模块接口单元。
3. 清除 `运行宿主.h` 和 `运行消息队列.h` 中的权威定义或删除旧文件。
4. 扫描并排除头文件与模块双源、循环导入和同文件多线程。

### S6 完整验证和记录

1. Debug x64 构建。
2. `--self-test-exit` 运行并复验 THREAD-S2 至 THREAD-S6。
3. 复验 6 个线程显式停止并 join。
4. 执行模块归属、禁止能力、规范和 diff 检查。
5. 形成实施记录、同步计划索引和项目记忆。
