## 8. 实施步骤

### S0 当前代码事实复核

1. 重读允许 C++ 文件头部规则注释。
2. 复核 `需求服务` 当前需求身份、目标状态需求、目标状态读取、主体 / 宿主 / 场景读取和承接材料读取入口。
3. 复核 `任务服务::按需求创建任务` 对需求承接材料的只读依赖和拒绝边界。
4. 复核 `特征服务` 当前 I64 特征状态材料读取入口，不直接访问特征值服务。
5. 复核 `状态服务` 当前抽象状态创建和状态值读取入口。
6. 记录 XQ-GAP-01 到 XQ-GAP-07 的当前代码事实。

### S1 需求身份创建和读回

1. `需求服务::创建需求` 返回有效需求节点句柄。
2. 读回需求节点，确认节点类型为 `节点类型::需求`。
3. 主信息或需求节点创建失败时，不返回有效需求句柄。
4. 覆盖 XQ-A1。

### S2 简化目标状态需求入口

1. 合法目标宿主、目标特征、当前特征状态材料和目标状态值可以创建简化目标状态需求。
2. 读回需求节点、目标宿主、目标特征、当前特征状态材料、目标状态和模板关系。
3. 无效目标宿主、无效目标特征、当前材料不可读或目标状态创建失败时拒绝。
4. 简化入口需求读取完整承接材料必须为空，任务服务不得按该需求创建任务。
5. 覆盖 XQ-A2、XQ-A5、XQ-A7、XQ-A9、XQ-A12。

### S3 完整目标状态需求入口

1. 合法主体、目标宿主、场景、目标特征、当前特征状态材料和目标状态值可以创建完整目标状态需求。
2. 读回主体、目标宿主、场景、目标特征、当前材料、目标状态和模板关系。
3. 无效主体、目标宿主、场景、特征或当前材料不可读时拒绝。
4. 默认第一轮保留当前代码对 `需求主体 == 目标宿主` 的硬拒绝，除非用户确认 XQ-IMP-Q1 另行放开。
5. 覆盖 XQ-A3、XQ-A4、XQ-A5、XQ-A6、XQ-A7、XQ-A12。

### S4 目标状态和承接材料只读入口

1. `读取需求目标状态` 必须只返回唯一状态节点。
2. 目标缺失、多目标或目标不是状态节点时返回空。
3. `读取需求承接材料` 必须要求主体、目标宿主、场景和目标状态四项均唯一且类型正确。
4. 任务服务只能基于完整承接材料创建任务承接壳。
5. 覆盖 XQ-A8、XQ-A9、XQ-A10。

### S5 目标状态满足预判

1. 读取需求目标状态值和当前特征状态材料。
2. 当前只按 `当前材料 >= 目标状态值` 返回局部枚举。
3. 证据不足时返回证据不足，不写持久化事实。
4. 不写需求结算记录、任务完成状态、方法成功或世界事实。
5. 覆盖 XQ-A11。

### S6 排除项扫描

1. 扫描本切片是否新增父子需求、逻辑组织、方向签名、枚举目标、方法候选或结构化更新提交实现。
2. 扫描是否新增权威需求结算记录写入。
3. 扫描是否新增控制面板、SQL、ADO、WebView、D455、体素、外设、`std::thread`、`std::async` 或工作队列。
4. 扫描是否修改禁止文件。
5. 任一命中必须停止并登记断点。

### S7 高级服务封口扫描

1. 扫描需求服务、任务服务、方法服务是否直接包含、接收或调用特征值服务。
2. 扫描新增代码是否绕过特征服务访问特征值节点或主信息值容器。
3. 有越权依赖命中时停止实施并登记待确认问题。

### S8 验证与收口

1. 运行 `git diff --check`。
2. 运行 `python .\tools\check_specs.py`。
3. 运行 Debug x64 构建。
4. 运行默认入口并确认 XQ-A1 到 XQ-A13 对应输出或断点记录。
5. 生成 `实施记录/20260708_BASE-D3_需求创建与目标状态代码实施_Codex断点清单.md`。
6. 归档本计划并同步项目记忆。
