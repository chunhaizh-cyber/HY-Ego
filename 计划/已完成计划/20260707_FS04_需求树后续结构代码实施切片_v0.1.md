## 4. 目标服务和目标结构

| 项 | 目标口径 |
| --- | --- |
| 需求服务 | 统一提供父子需求、逻辑组织、方向签名、枚举目标准入和结构化更新请求壳入口 |
| 任务服务 | 只读取需求服务提供的任务承接材料，不直接解释裸关系 |
| 方法服务 | 只读取需求侧候选准入材料，不由需求服务直接执行方法 |
| 关系仓库 | 可承载父子、引用、模板等底层关系，但业务角色必须经需求服务封装 |
| 非权威缓存 / 统计 | 权重和排序材料只作为非权威候选，不裁决需求满足或任务完成 |

## 7. 实施步骤

```text
S0：复核需求服务、任务服务、方法服务当前入口和 FS-04 第一轮验收输出。
S1：实现父子需求角色、阻塞和权重只读入口，拒绝自身父子、环和权重裁决业务事实。

S4：补默认入口最小验收，覆盖拒绝后结构不变化。
```
