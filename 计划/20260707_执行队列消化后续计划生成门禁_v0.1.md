## 2. 确认范围

本计划确认后，只允许登记一个文档治理队列项：

```text
复核 #108 到 #118 执行结果并生成下一批计划候选。
```

该队列项的目标是等执行窗口完成 #108 到 #118 后，读取实际实施记录、详细设计、验收结果和新增待确认问题，再生成下一批待确认计划。

## 3. 生成下一批计划的前置条件

下一批计划必须至少满足：

```text
1. #108 到 #118 已执行完成，或执行窗口明确记录真实阻塞。
2. 每个完成项有对应实施记录或详细设计产物。
3. 若涉及 C++，必须已有独立切片所需的允许文件、禁止文件和验证方式。
4. 若执行中产生待确认问题，必须先写入项目记忆并等待确认窗口裁决。
5. 不以旧函数编号作为迁移单位，只以服务逻辑包、服务专项计划或服务代码实施切片作为计划单位。
```

## 4. 明确排除项

本计划不允许：

```text
不直接修改 C++。
不迁移旧函数体。
不新增旧主信息字段。
不把未执行的详细设计专项预设为已完成。
不跳过 #108 到 #118 执行结果。
不从控制面板或数据库旧实现生成迁移计划。
不接 SQL / ADO / WebView2 / D455 / 体素 / 外设。
```

## 5. 控制面板和数据库口径

控制面板和数据库从本轮迁移计划中撤出：

```text
控制面板：后续根据用户明确指令重建，不参与旧能力迁移。
数据库：后续根据用户明确指令重建，不参与旧能力迁移。
```

旧 FS-X 中的控制面板、SQL、ADO 事实只保留为排除证据，不作为自动生成迁移计划的依据。

本计划自身确认后，只做文档治理验收：

```powershell

```

后续真正生成下一批计划时，必须再次读取 #108 到 #118 的实际产物。
