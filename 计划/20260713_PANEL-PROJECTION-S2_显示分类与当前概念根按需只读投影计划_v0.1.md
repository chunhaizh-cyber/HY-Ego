## 1. 输入入口

依据仓库优化流程图和详细设计，消费 #258 正式关系查询接口及 #259/#260 最终裁决。六项是显示分类，不是六棵物理树；概念结构按多根森林和图处理。

## 2. 核心处理

1. 新增或收紧只读投影请求：显示分类、一个或多个当前概念根、关系类型白名单、深度、页大小和继续游标。
2. 默认按当前分类和当前根按需生成，不再把每次刷新等同于生成六类完整结构。
3. 只通过领域服务 / 关系仓库只读入口取候选，逐关系复核句柄、版本、状态和有效端点。
4. 多父共享节点只生成一个节点展示实体，但保留每条关系边和引用位置；图环用已访问关系集合终止，不丢边。
5. 投影结果是值式人读材料，不写仓库、不回流索引、不缓存为机器事实。

## 3. 结构承载、输出和调用路径

```text
控制面板请求 -> 参数 / 越权 / 限额拒绝 -> 显示分类与根集合
-> 关系候选读取 -> 权威关系复核 -> 限深分页森林 / 图投影 -> UI 展示
```

权威结构仍是统一节点、主信息和关系；显示 DTO 只承载当前请求快照。

## 5. 禁止文件与行为

禁止修改四核心仓库、结构事务、业务领域服务写入口、SQL / ADO、WebView 迁移、线程调度和 `#214`。禁止把六分类固化为六个物理仓库 / 物理根；禁止显示层直接遍历内部表、写业务结构或把缓存命中作为事实。

## 6. 失败收口

无根、非法分类、关系白名单为空、深度 / 页大小超限或越权读取在调用仓库前拒绝。候选为空是正常值式结果。候选命中后同一许可内关系不可复核属于逻辑错误，停止投影且不生成施工完成声明；不得用旧缓存或全表扫描兜底。
