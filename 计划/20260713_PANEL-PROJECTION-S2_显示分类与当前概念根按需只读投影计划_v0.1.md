## 2. 核心处理

1. 在控制面板服务新增值式按需请求、当前页结果、概念根选项和根页游标；不暴露仓库引用、锁、许可、原始令牌或关系白名单。
2. 保留 `生成六类真实树只读材料` 的签名和原义，但生产窗口改用 `生成当前分类真实树只读材料`。
3. 世界、语素和需求先选择当前真实根页再构造；任务只从需求承接关系发现根身份；方法只从任务选择关系发现根身份。根发现不得构造或发布其它分类完整树。
4. 概念分类读取四个活动根选项，但只对当前活动根调用一次正式单根抽象树读取；不得生成四根后裁切。
5. 根分页只在真实根边界切页，不截断单棵递归树。每页独立当前读取并整体替换，禁止跨页拼成机器快照。
6. 控制面板窗口保留六项分类导航；TreeView 只重建当前分类和当前页，概念分类显示根选择下拉，刷新只读取当前分类 / 当前根 / 当前页。
7. 新增独立 Debug 自检真模块，验证单分类隔离、概念单根、分页、旧入口兼容、失败收口、UI 消费边界、结构不变和 Release 隔离。

## 3. 结构承载与输出变化

```text
输入：当前分类、可选当前概念根、节点预算、深度预算、根页大小和可选继续游标。
核心处理：基础准入 -> 根身份发现 -> 根页选择 -> 当前分类值式读取 -> 树结构复核。
结构承载：权威节点 / 主信息 / 关系和业务结构保持不变；按需结果、游标和窗口页快照只做人读会话材料。
输出变化：生产窗口一次只显示当前分类当前页；概念类一次只显示当前活动根；旧六类入口仅保留兼容。
```

非概念分类没有统一活动版本。根集签名只用于拒绝错误游标，不能升级为关系图版本；窗口不得累计多页或把显示缓存作为事实。

## 7. 入口拒绝与失败收口

逻辑内返回：非树分类、无效基础请求、零预算、非法根页大小、非活动概念根、活动图未就绪、允许的空根、游标失效、页越界或预算耗尽。返回不得携带可展示的部分新页，权威结构不变；窗口可保留旧页但必须标记本次未刷新。

追根因解决：前置通过后领域读回矛盾、概念活动版本 / 根类别 / 根句柄不一致、有效根无法构造、树结构复核失败、TreeView 插入或引用读回失败，以及任何只读路径结构变化。命中后停止依赖路径，不得用旧六类入口、缓存、SQL、日志或全表扫描兜底。
