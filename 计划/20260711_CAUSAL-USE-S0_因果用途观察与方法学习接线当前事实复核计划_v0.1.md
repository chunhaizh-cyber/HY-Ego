## 1. 目标

在真实任务结果 / 需求结算和因果概率候选都形成后，复核“因果是否好用”以及方法学习的正式接线：

```text
因果发生概率
+ 真实任务结果 / 完成 / 失败 / 取消
+ 需求结算与结果效用
-> 一次性用途观察证据
-> 统计 / 概念用途 / 方法学习所有权
-> 下一批流程图、详细设计和计划顺序
```

本计划不写用途、统计、概念或方法，不把概率和效用合并成一个分数。

## 5. 执行步骤

### S0：前置和实际接口复核

记录 #225、#234、#231 的提交号、归档计划和正式接口。依赖未满足时保持门控；不得基于计划中的预期 DTO 生成用途结论。

### S1：结果效用证据

逐项读取已完成、未达成、失败、取消、等待 / 待重筹办、待结算和已结算材料，确定哪些结构能代表结果效用，哪些只代表任务生命周期或因果发生概率。

### S2：用途观察所有权

复核统计缓存、概念用途观察和候选生命周期。判断模式用途观察是否需要专用结构、是否只作可清空缓存，以及一次性观察的幂等身份和迟到 / 重复 / 冲突处理。

### S3：方法学习与授权边界

读取方法登记、规格、召回、复判、排序、任务选择和执行授权实际接口。明确学习只能生成候选还是允许登记新方法，以及谁有权提交；不得把线程或统计缓存设为动作来源。

### S4：恢复影响与下一设计链

区分权威结构、可清空缓存和恢复必需侧表，按 JY-277 复核 #246 对唯一运行期上下文的裁决以及 #226-#230 影响。给出无依赖闭环的下一设计顺序；任何未来用途写入 / 方法学习计划必须明确由唯一正式运行期上下文取得许可，纯值概率候选不得反向持有许可。必要时先生成用途观察 S0 / S1，再生成方法学习候选和授权选择计划。

### S5：治理收口

生成正式断点清单，归档本计划，更新计划索引、任务队列、当前状态、实施记录、决策和待确认问题，精确提交并推送。
