只读消费 #349 的 `NT-C4/v0.2`：`统一权威快照.数据.h` 的九段强类型 DTO、规范化格式 ABI、`规范化快照字节材料`、`规范化权威载荷摘要` 和 `序列化统一权威快照`；同时只读消费 #350 按同合同提供的 `解析统一权威快照`、`形成隔离恢复候选` 和 `验证完整恢复候选`。这些均是施工期预冻结合同，实际接线由 #352 验证。#351 不复制 104 字节容器头、64 字节段头、九段数值 ABI、上限或摘要实现，不另立当前格式定义。

本计划固定提供：

```text
旧快照来源类型
旧格式版本身份
旧格式只读材料
旧字段映射项
旧快照迁移拒绝原因
旧快照迁移报告
旧快照迁移结果
旧格式识别结果
识别旧快照格式
迁移旧快照到当前格式
审计旧快照迁移
```

完整签名固定为：

```text
旧格式识别结果 识别旧快照格式(const 旧快照只读材料&)
旧快照迁移结果 迁移旧快照到当前格式(const 旧快照只读材料&, 旧格式版本身份)
旧快照迁移报告 审计旧快照迁移(const 旧快照只读材料&, const 旧快照迁移结果&)
```

`旧快照只读材料` 由调用方拥有，三函数只在调用期读取且不得保存其可变引用、文件句柄或生产仓库引用。`旧格式版本身份` 是识别成功后形成的值式身份，未知或歧义版本不得进入迁移。`旧快照迁移结果` 自有当前格式规范化字节或具名拒绝证据；`旧快照迁移报告` 自有人读审计材料但不裁决机器事实。迁移成功仍须把结果交给 #350 严格解析、隔离恢复和全量互证，不能直接发布运行期。

`迁移旧快照到当前格式` 只在逐版本映射表证明稳定主键、关系、领域记录、历史/失效关系、任务选择/方法生命周期、概念签名和 4170 全部无歧义时输出独立当前格式字节。识别、迁移结果固定区分 `已形成/材料拒绝/资源失败/内部不一致`；报告结果固定包含来源身份、逐字段映射、接受/拒绝结论和证据完整性。旧材料未知、局部、损坏或不可唯一映射属于材料拒绝；映射已证明唯一但输出不能满足 `NT-C4/v0.2` 属于内部不一致。报告只作人读审计，不裁决运行期事实。
