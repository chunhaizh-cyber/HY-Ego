## 5. 必须形成的八类矩阵

### 5.1 字段承载准入矩阵

逐项覆盖完整冻结请求、因果候选、机会证据组、回执、任务结果、结算、状态、动态、效用分量和重试材料：

```text
#282 实际字段
字段来源符号
权威 / 非权威性质
生命周期与可变性
现行规范允许的承载结构
当前读写 API
是否可完整重建
是否违反主信息红线
结论：可复用 / 缺结构 / 缺入口 / 违反规则 / 待设计
```

### 5.2 主信息准入与可行性矩阵

至少复核：当前值容器类型、逐槽写入 / 读回、稀疏槽位、自动扩容、领域容量上限、未知版本 / 标签处理和已发布值不可变性。`主信息可读` 不等于完整值匹配；完整回执、任务结果和因果来源写入主信息当前一律按“未获许可”起步，不得预设例外。

### 5.3 身份与索引矩阵

```text
逻辑观察主键是否严格等于 #282 冻结键。
任务回合身份是否使用完整任务句柄三元组 + 来源任务序号。
索引仓库是否只有全局 uint64 -> 节点句柄映射。
是否存在正式命名域分配、组合键算法和碰撞处理合同。
双索引命中后如何完整比较，恢复后如何保持物理键。
同一物理键命中不同完整身份如何分类。
```

文档候选中的“不同命名域”不能当作当前索引 API 已支持命名域。

### 5.4 关系与阶段承载矩阵

逐项复核候选关系 0—6、模板 1、因果来源 1 / 2 起的端点、顺序号、基数、现有 API、规格形成方、写入方、完整读回、并发单赢家、失效旧阶段和恢复唯一性。

必须回答：当前阶段能否只由有效引用关系顺序号 6 裁决；初始阶段历史材料与当前阶段事实如何区分；待结算转最终是否需要改主信息；结算关系 5 与阶段关系 6 能否由现有会话安全发布。

### 5.5 事务与发布矩阵

```text
写前双重唯一
-> 主信息候选
-> 节点候选
-> 索引绑定
-> 固定关系
-> 当前阶段
-> 逐项读回
-> 请求提交
-> 最后发布
```

每步列明实际函数、入口拒绝、故障撤销、并发结果和发布点。现有 API 无法证明无半结构时只登记缺口。

### 5.6 容量与规范重建矩阵

记录各嵌套向量上限、完整因果候选最坏规模、单观察预计字段 / 嵌套层级、主信息领域容量限制、缺槽 / 多槽 / 空 optional / 未知版本处理和完整往返重建入口。只记录事实，不发明槽位格式、标签、编码器或解码器。

### 5.7 恢复边界矩阵

复核四仓库快照、节点 15 未知类型拒绝、主信息值、双索引、关系历史、待结算定位、恢复后双重唯一 / 阶段唯一 / 结算唯一，以及统计投影如何从权威结构重建。必须证明不与 #226-#230、#237-#239 建第二恢复链；不能证明时登记缺口。

### 5.8 下游依赖矩阵

明确 #284 和 #285 只能消费本 S0 的事实 / 缺口，不得把 #283 S0 当成权威用途账已经实现。未来统计投影、节点 15 或用途观察实施计划必须依赖本 S0 后新修订的三件套与计划。
