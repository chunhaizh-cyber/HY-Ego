## 3. 固定目标合同

### 3.1 装配与稳定动作键

```text
运行期业务装配构造增加 非零稳定动作键，并唯一构造：
  任务执行组合器(任务服务, 方法服务, 状态动态组合器, 稳定动作键)
  任务结果结算组合器(需求服务, 任务服务, 方法服务, 状态服务, 动态服务)
  运行期只读查询组合器(概念活动服务, 语素服务, 任务服务, 方法服务, 状态服务, 动态服务)

运行期上下文增加显式三参数构造(域编号, 仓库编号, 稳定动作键)。
既有二参数构造只作现有隔离自检兼容，委托稳定动作键 1；生产启动不得使用该重载。
生产运行期会话必须把现行正式字段 生产运行期配置.动作键 显式传入三参数构造；该字段的业务语义仍为稳定动作键。
上下文按既有业务操作读取方式增加 读取候选只读查询()；只返回 const 指针，不外露装配、服务或仓库。
```

### 3.2 概念活动与名称值式查询

新增领域真模块 `领域/组合.运行期只读查询.ixx`，唯一声明：

```text
运行期概念命名查询请求：概念句柄、预期活动版本。
运行期概念命名查询状态：待命名、已有名称入口、当前活动中未找到、活动版本漂移、入口拒绝、许可拒绝、版本漂移、内部不一致。
运行期概念命名查询结果：状态、概念、根类别、活动版本、名称语素入口值式材料组。
```

第一轮“当前活动概念”精确限定为 `概念活动材料.根组` 中四个活动根；不得把旧概念图活动快照、旧生命周期或任意节点类型推断成当前活动概念。目标在四根中且名称组为空时为待命名；名称组非空时为已有名称入口；预期版本非零且不等于当前活动版本时为活动版本漂移。

`语素基础数据操作` 增加 `读取概念名称入口组(节点句柄)`：在一次共享许可内用 `语素概念追溯` 反向关系读取来源语素，逐项形成完整 `语素入口值式材料`，按完整句柄确定性排序；空集合是合法值，重复来源、失效关系、类型错误或关系 / 材料互证矛盾为内部不一致。`语素业务服务` 只做句柄入口拒绝并委托，不接触仓库。

### 3.3 任务结果复核值式查询

同一只读查询组合器唯一声明：

```text
运行期任务结果复核请求：任务结果回执材料、当前时间戳。
运行期任务结果复核状态：可准入、入口拒绝、当前性漂移、许可拒绝、版本漂移、内部不一致。
运行期任务结果复核结果：状态、既有任务管理上行拒绝语义的强类型映射、回执副本。
```

复核顺序固定为：协议准入 -> 任务承接 / 生命周期 / 选择 -> 方法与唯一动作规格 -> 实际状态 -> 动态证据 -> 场景 / 主体 / 来源动作 / 值 / 时间互证。全程只调用新分层服务公开值式读取，不写结构、不调用结果结算、不产生许可结论。#299 只消费该结果转换非权威消息，不重复领域复核算法。

### 3.4 顶层业务操作与路由

`运行期业务操作组合器` 从六组合器扩为八组合器，在既有 8 类业务入口外增加 5 类：

```text
冻结任务执行请求(任务执行调度请求) -> 任务执行组合结果
确认任务进入排队中(运行期任务排队请求{冻结请求}) -> 任务执行组合结果
执行任务目标状态值变化(运行期任务执行请求{冻结请求, 当前时间戳}) -> 任务执行组合结果
收口任务执行请求(运行期任务收口请求{冻结请求, 当前阶段}) -> 任务执行组合结果
处理任务执行结果(任务结果治理请求) -> 任务结果治理结果
```

每类新增请求有唯一 `请求完整` 重载并只委托对应既有组合器。业务请求 variant 最终固定 14 个备选项：`monostate + 8 既有 + 5 新增`；业务结果 variant 固定 11 个备选项：`monostate + 8 既有 + 任务执行组合结果 + 任务结果治理结果`。不得因四个任务执行请求共享同一结果类型而复制结果 DTO。

### 3.5 租约查询门面

新增 `适配/查询.运行期只读.ixx`。门面只持 `运行期上下文租约` 值，调用时重新取得上下文和 `读取候选只读查询()`，按值返回 3.2 / 3.3 的结果；不得缓存裸上下文指针、组合器指针或内部引用，不得持仓库、接线、令牌、许可、执行器或生产会话。
