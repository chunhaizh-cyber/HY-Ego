```text

  正式结果、任务达成、需求结算、观察阶段、optional 正式结算状态。

用途观察纯值候选：
  uint32 格式版本。
  用途观察语义身份 身份。
  用途观察不可变核心 不可变核心。
  用途观察阶段负载 阶段负载。
```

候选不得包含 `vector`、`string`、完整冻结请求、完整回执、完整需求结算 DTO、完整用途事件 DTO、完整机会证据组或待结算提示。格式版本不属于业务身份。

### 4.3 两套独立强类型结果

```text
用途观察纯值形成状态：已形成 / 入口拒绝 / 不支持 / 内部不一致。
用途观察纯值形成原因：无 / 请求不完整 / 来源冻结不完整 / 结果尚未正式收口 /
  非新版完整回执 / 来源冻结回合不一致 / 规则版本不支持 / 目标格式不支持 /
  正式结果材料矛盾 / 身份材料矛盾 / 内容材料矛盾 / 阶段材料矛盾 / 算法不变量破坏。
用途观察纯值形成结果：状态、原因、optional 候选。

用途观察同账比较结论：不同身份 / 同账幂等 / 同账允许推进 / 同账冲突 / 不支持 / 内部不一致。
用途观察同账比较原因：无 / 格式版本不支持 / 新候选非法 / 既有候选非法 /
  完整身份不同 / 完整内容相同 / 待结算推进最终 / 不可变核心不同 /
  阶段转换不允许 / 算法不变量破坏。
用途观察同账比较结果：结论、原因。
```

形成结果安全默认值为 `入口拒绝 / 请求不完整 / 无候选`。比较结果安全默认值为 `内部不一致 / 算法不变量破坏`。两套类型不得互相复用，不得用 bool 合并精确结论。形成结果只有 `已形成 + 无原因 + 有合法候选` 成功；其它合法组合必须无候选。比较结果只接受详细设计列出的结论—原因组合。
