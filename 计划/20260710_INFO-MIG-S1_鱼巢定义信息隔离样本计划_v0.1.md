## 2. 确认范围

第一轮只验证两个定义身份：

```text
安全值
服务值
```

每项预期结构：

```text
新项目显式稳定键
-> 概念语素入口
-> 语素对应信息关系
-> 特征节点
-> 语素概念追溯关系回指同一特征节点
-> 主信息锚点
```

只在本切片新建的隔离候选仓库中构建和读回，执行结束后不替换当前运行期仓库。

## 4. 目标服务和结构

| 对象 | 使用入口 | 结构作用 |
| --- | --- | --- |
| 主信息仓库 | 隔离实例 | 提供特征节点和语素节点主信息锚点 |
| 节点仓库 | 隔离实例 | 承载特征和语素节点身份 |
| 关系仓库 | 隔离实例 | 承载语素对应信息关系 |
| 索引仓库 | 隔离实例 | 承载新项目显式稳定键到语素入口的映射 |

| 语素服务 | `创建概念入口` | 校验词面、创建语素入口，把同一特征节点作为对应信息和概念追溯目标，并绑定稳定键 |

## 7. 实施步骤

### S0 代码前置复核

```text
确认两个旧来源行和 SHA256 未变化。
确认当前语素服务仍允许“安全值”和“服务值”作为最小词单元。
确认稳定键不与默认入口现有测试键冲突。
确认运行期结构数量读取入口可用于前后对照。
```

### S1 定义清单和稳定键

```text
在迁移文件中建立只读常量清单。
每项记录中文词面、来源行、来源哈希、目标节点类型和显式稳定键。
稳定键使用 INFO-MIG-Q1 确认后的新项目命名空间。
```

### S2 隔离候选仓库构建

```text
新建隔离主信息、节点、关系和索引仓库。
构造隔离特征服务和语素服务。
所有前置准入在写入前完成。
经领域服务创建两项定义身份。
```

### S3 读回和幂等

```text
按稳定键读取语素入口。
读取唯一语素对应信息目标和唯一概念追溯目标。
确认两个目标相同且都是计划创建的特征节点。
重复执行同一批次，确认节点、关系和索引数量不增加。
```

### S4 拒绝和失败收口

```text
验证空批次、来源指纹错、零稳定键、冲突稳定键和非法词面在写入前拒绝。
构建开始后的非预期结果按追根因处理并丢弃整套隔离仓库。
当前运行期仓库数量始终不变化。
```

### S5 记录和归档

```text
记录 INFO-MIG-S1-A1 至 A8。
计划完成后移入 计划/已完成计划/。
同步计划索引和项目记忆。
不生成 SQL 写入或运行期导入许可。
```
