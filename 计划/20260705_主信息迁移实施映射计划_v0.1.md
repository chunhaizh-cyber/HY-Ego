
## 1. 设计条款清单

### 1.1 必须实现

| 条款 | 来源 | 实施含义 |
| --- | --- | --- |
| 统一主信息仓库 | 主信息系统规范 | 继续使用一个主信息仓库，不拆回旧项目各类主信息类。 |
| 最小主信息记录 | 主信息系统详细设计 | 主信息记录只保留编号、版本、状态、拓扑锚点和值容器。 |
| 入口前置拒绝 | 项目规则总纲 / 主信息详细设计 | 无效句柄、仓库不匹配、版本不匹配、已删除记录、未授权写值必须在入口拒绝。 |
| I64 值容器现有能力复验 | 主信息详细设计 | 复验 `写入I64值`、`读取I64值` 和带值索引重载，不扩大为旧字段迁移。 |
| 语素对应信息关系承载 | 字段功能确认清单 | MI-YS5 继续由 `语素对应信息` 关系承载，支持多对应信息节点。 |
| MI-V1 范围收束 | 字段功能确认清单 | 旧项目特征值 `值` 语义只保留在特征值系统，不扩展为需求 / 任务 / 方法通用值容器。 |
| 字段迁移门禁 | 主信息系统规范 | 任何新字段必须先写字段功能说明、权威性、写入方、读取方和验证方式。 |

### 1.2 禁止实现

| 禁止项 | 原因 |
| --- | --- |
| 新增 `词主信息记录`、`语素入口主信息记录`、`基础信息主信息记录`、`需求主信息记录`、`任务主信息记录`、`方法主信息记录` | 违反统一主信息仓库和旧字段不直接迁移口径。 |
| 把名称、类型、词性、入口类型、对应信息、概念追溯、任务名称或方法名称写成裸主信息字段 | 已确认应走语素关系、索引、入口路由或领域服务。 |
| 把状态、动态、二次特征、因果字段族写成主信息字段 | 已确认拆到对应领域系统。 |
| 把 MI-V1 扩展成需求 / 任务 / 方法通用值容器 | 违反特征值服务只服务特征系统且仅特征服务可直接访问规则。 |
| 为显示、日志、SQL 或控制面板便利新增机器字段 | 显示和日志不得承载机器逻辑。 |
| 直接复制旧项目主信息类或旧函数体 | 阶段 4 门禁禁止未映射迁移。 |
