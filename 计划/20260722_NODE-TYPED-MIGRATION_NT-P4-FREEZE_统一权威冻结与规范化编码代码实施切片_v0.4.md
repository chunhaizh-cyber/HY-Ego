### 3.2 数值 ABI、头布局与上限

当前格式全部多字节整数按小端编码，固定值为：

| 项 | 固定值 / 宽度 |
| --- | --- |
| 魔数 | 8 字节 `48 5A 59 43 53 4E 50 31`，ASCII 为 `HZYCSNP1` |
| 容器格式版本 | `uint16 = 1` |
| 字节序标识 | `uint16 = 1`，其它值拒绝 |
| 容器头字节数 | `uint32 = 104` |
| 段头字节数 | `uint32 = 64` |
| 段清单版本 | `uint32 = 1` |
| 摘要算法版本 | `uint32 = 1`，固定 SHA-256，输出 32 字节 |
| 冻结规则版本 | `uint32 = 1` |
| 节点稳定主键命名域 ABI 版本 | `uint32 = 1` |
| 必需段数量 | `uint32 = 9`，每类恰一段并按段类型升序出现 |

104 字节容器头固定布局：

| 偏移 | 字段 | 宽度 |
| --- | --- | --- |
| 0 | 魔数 | 8 字节 |
| 8 | 容器格式版本 | uint16 |
| 10 | 字节序标识 | uint16 |
| 12 | 容器头字节数 | uint32 |
| 16 | 段头字节数 | uint32 |
| 20 | 段清单版本 | uint32 |
| 24 | 摘要算法版本 | uint32 |
| 28 | 段数量 | uint32 |
| 32 | 文件总字节数 | uint64 |
| 40 | 来源运行期域身份 | uint64，非零 |
| 48 | 来源节点直接仓库身份 | uint64，非零 |
| 56 | 快照代次 | uint64，非零且同来源单调 |
| 64 | 冻结规则版本 | uint32 |
| 68 | 节点稳定主键命名域 ABI 版本 | uint32 |
| 72 | 容器摘要 | 32 字节 |

64 字节段头固定布局：

| 偏移 | 字段 | 宽度 / 规则 |
| --- | --- | --- |
| 0 | 段类型 | uint16 |
| 2 | 段格式版本 | uint16，当前均为 1 |
| 4 | 段头字节数 | uint32，必须为 64 |
| 8 | 段顺序 | uint32，从 0 连续到 8 |
| 12 | 段标志 | uint32，当前必须为 0 |
| 16 | 记录数量 | uint64 |
| 24 | 段载荷字节数 | uint64 |
| 32 | 段载荷摘要 | 32 字节 SHA-256 |

段类型数值 ABI 固定为：`1=节点直接记录`、`2=正式关系记录`、`3=特征值类型化记录`、`4=状态域记录`、`5=动态域记录`、`6=任务方法选择记录`、`7=概念签名记录`、`8=用途观察记录`、`9=特征批次发布记录`。九段均为必需段；合法空段用记录数量 0 表达，不得省略。新增段只能提升段清单版本，不能复用或改义 1—9。

格式上限固定为：文件总长不超过 `8,589,934,592` 字节；单段载荷不超过 `2,147,483,648` 字节；每段记录不超过 `4,194,304` 项；单条规范化记录不超过 `16,777,216` 字节；UTF-8 字符串不超过 `1,048,576` 字节；单个向量不超过 `1,048,576` 项。字符串和向量数量前缀固定为 uint32；单条记录以前置 uint32 记录字节数分帧，但字段仍由具名强类型段唯一解释。

布尔值编码为 uint8 的 0/1；可选值编码为 uint8 存在标志 0/1 后接具名值；枚举按正式数值 ABI 的固定宽度编码；节点稳定主键编码为两个 uint64 分量；完整句柄固定编码为 `uint64 仓库身份 + uint64 对象编号 + uint32 版本号`，与 #339 真实句柄 ABI 一致。字符串只编码 UTF-8 字节，不依赖区域设置。记录按冻结 DTO 的 `规范化排序键` 升序编码，禁止依赖容器遍历、地址或线程时序。

段摘要只覆盖段载荷。容器摘要覆盖整个文件，但计算时将容器头偏移 72—103 的摘要字段视为 32 个零字节。恢复候选再冻结相等只比较九段规范化权威载荷，不比较来源运行期域身份、来源节点直接仓库身份、快照代次、摘要或可重建投影。

### 3.3 值式 DTO

`统一权威快照.数据.h` 必须显式定义并稳定提供：

```text
统一权威快照头
统一权威快照段头
统一权威快照段类型

规范化快照字节材料

必需段声明组读取结果

```

段类型必须逐项具名，第一版不得用 `variant`、JSON、任意键值或通用字节记录表示业务段。`节点直接记录段` 的每一项必须无损承载：

```text
节点编号
稳定主键
完整稳定句柄（仓库身份、节点编号、版本）
节点类型
记录版本
记录状态
创建序号
```

节点编号是独立权威字段，不得从记录顺序、稳定主键或外部上下文反推；稳定句柄中的对象编号必须与节点编号相同，句柄版本必须与记录版本相同。段级材料另带节点稳定主键命名域 ABI、仓库身份、编号高水位和创建序号高水位。关系段覆盖 ABI 0—23 的有效、失效、删除和历史；其它七个领域段必须逐项无损保存所属正式规范要求的全部类型化字段、记录版本、记录状态、历史和失效信息，不得只保存句柄、摘要、数量或显示投影。索引、列表、缓存和投影不进入必需权威段。

### 3.4 完整签名、所有权与生命周期

```text
必需段声明组读取结果 读取当前必需段声明组()
统一冻结结果 形成统一权威冻结材料(const 统一冻结规格&, 统一冻结提供者组&)
规范化编码结果 序列化统一权威快照(const 统一权威冻结材料&)
快照文件保存结果 原子保存快照材料(const 快照文件保存请求&)
```

`统一冻结规格`、`统一权威冻结材料`、`规范化快照字节材料`、`规范化权威载荷摘要` 和 `快照文件保存请求` 均为调用期值式材料。`统一冻结提供者组` 只在 #349 内部适配层有界引用 P1—P3 冻结提供者，不取得其所有权，不得越过调用期保存可变引用。返回结果拥有自身值式材料；任何函数都不得返回仓库引用、锁、令牌、事务会话、参与者、可变段视图或文件句柄。

`读取当前必需段声明组` 返回固定段类型、版本、上限和顺序；`形成统一权威冻结材料` 必须在一个冻结权、一个来源域和一个代次内形成九段；`序列化统一权威快照` 只形成规范化字节，不写文件；`原子保存快照材料` 只消费已编码字节，临时文件未完成刷新、复读和摘要核对时不得替换旧完整文件。

统一冻结与编码结果固定区分 `已形成/入口拒绝/资源失败/内部不一致`；文件保存结果固定区分 `已保存/路径拒绝/资源失败/内部不一致`。路径、资源和调用前材料不合法属于逻辑内返回；合同已经接受后的来源域/代次不一致、缺必需段、跨段互证失败、规范化复读不同属于内部逻辑错误。

消费者固定为 #350、#351、#352。任何消费者不得复制数值 ABI、另立段清单或形成同名替身；实际实现与本合同不一致时退回 #349 升版。
