## 7. 实施步骤

### S0 当前代码事实复核

1. 重读 `海中鱼巣/领域/语素服务.h` 文件头部规则注释。
2. 复核 `创建语素入口`、`创建概念入口`、`追加语素对应信息`、`追加语素概念追溯`、`读取入口绑定目标组`、`读取入口概念追溯组` 当前实现。
3. 复核核心仓库现有入口：`删除主信息`、`删除节点`、`删除关系`、`删除主键`、`有效节点数量`、`有效关系数量`、`有效主键数量`。
4. 记录本切片不修改核心仓库，不引入多仓库同时持锁。

### S1 拆分准入辅助

在 `语素服务` 内部拆分私有辅助：

```text
节点是可作为语素对应信息
节点是可作为概念追溯目标
```

准入规则：

```text
对应信息准入允许基础信息、存在、场景、特征、抽象状态、抽象动态、二次特征、因果引用、需求、任务、方法。
概念追溯目标第一轮只允许基础信息、特征、抽象状态、抽象动态、二次特征、因果引用。
状态 / 动态如果存在运行期临时目标关系，按实例状态 / 实例动态拒绝。
特征值和未确认类型一律拒绝。
```

`节点是可绑定信息` 如需保留，只能作为兼容私有包装，不得继续让概念追溯复用宽口径。

### S2 收紧创建语素入口

`创建语素入口` 必须按顺序处理：

```text
对应信息节点准入。
非零主键预读：已被其他节点占用则入口拒绝，结构不变化。
创建主信息并检查有效。
创建语素节点并检查有效。
创建语素对应信息关系并检查有效。
非零主键绑定并检查成功。
读回验证语素节点、对应信息关系和主键索引。
返回语素入口句柄。
```

失败收口规则：

```text
主信息创建失败 -> 返回无效句柄。
节点创建失败 -> 删除已创建主信息，不返回有效入口。
关系创建失败 -> 删除已创建语素节点和主信息，不返回有效入口。
索引绑定失败 -> 删除已创建关系、语素节点和主信息；若主键已经指向本次语素节点，删除该主键；不得删除原本属于其他节点的主键。
读回验证失败 -> 按逻辑错误记录断点，不返回有效入口；执行中若现有入口无法做到结构不变化，停止并登记待确认问题。
```

失败收口后必须通过读回和数量快照证明没有可读半结构：

```text
失败前后记录有效节点数量、有效关系数量和有效主键数量。
失败后读取入口绑定目标组、入口概念追溯组和主键索引，确认不能读出本次失败创建的有效业务事实。
无法证明结构不变化或无法证明半结构不可读时，停止本切片实施并登记待确认问题，不得继续验收或宣称通过。
```

### S3 收紧创建概念入口

`创建概念入口` 必须作为组合流程：

```text
文本准入。
对应信息节点准入。
概念追溯目标准入。
调用创建语素入口。
调用追加语素概念追溯。
读取入口概念追溯组确认包含目标。
返回有效入口。
```

任一阶段失败都返回无效句柄。概念追溯追加失败、重复拒绝或读回不包含目标时，不得返回有效语素入口；必要时按 S2 的失效收口处理本次创建的入口。

### S4 收紧追加关系与读回验证

追加入口必须保持两条公开路径：

```text
追加语素对应信息
追加语素概念追溯
```

规则：

```text
先确认入口是语素节点。
按关系类型选择 2.2a 或 2.2b 目标准入。
重复关系返回 false，不写新关系，关系数量不增加。
关系创建失败返回 false，结构不变化。
关系创建成功后读回目标组，必须包含目标节点。
读回不符合预期视为逻辑错误，停止本切片验收推进并登记断点。
```

### S5 更新入口验证样例

在 `海中鱼巣/入口.cpp` 增加第一轮验证材料：

```text
YS-A1 空文本拒绝。
YS-A2 组合分隔符拒绝。
YS-A3 长度拒绝。
YS-A4 对应信息允许范围可创建语素入口。
YS-A5 特征值、实例状态、实例动态拒绝且结构不变化。
YS-A6 需求、任务、方法作为概念追溯目标默认拒绝且结构不变化。
YS-A7 合法对应信息 + 未占用主键创建成功并可读回。
YS-A8 主键冲突拒绝，不返回有效入口，不留下可读半结构。
YS-A9 合法概念入口创建成功，绑定目标组和概念追溯组均可读。
YS-A10 重复追加关系拒绝，关系数量不增加。
YS-A11 追加后读回目标组包含目标节点。
YS-A12 非语素入口读取返回空集合或失败，结构不变化。
```

入口输出只作为人读验收结果，不承载机器事实。验收布尔值必须来自结构读回和数量快照。
