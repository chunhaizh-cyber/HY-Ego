## 10. 实施步骤

1. 确认 #241 已完成，读取运行器实际导出、入口登记样本和项目登记，不凭本计划假定签名编码。
2. 重新定位逻辑提取矩阵中的 THREAD / SELF-GOV 稳定变量前缀和 119 项验收编号，生成迁移前编号基线。
3. 新建运行线程自检真模块，迁入 THREAD-S1-S6 场景、夹具、并发收口、报告和 39 项输出。
4. 把 THREAD-S3 与 SELF-GOV-S3A 的共有线程夹具固定为运行线程自检模块唯一所有者，通过执行期强类型单写冻结槽公开最小值式回执；槽位不得进入运行器通用 API 或成为全局状态。
5. 新建自我治理自检真模块，迁入 SELF-GOV-S1-S5 场景、报告和 80 项输出，只读消费已封存回执，并把汇总结果写入供顺序 400 只读消费的同类冻结槽。
6. 在入口建立最小上下文，按 100、200 登记两个单元；保留概念命名 400，不改变运行器单次执行。
7. 从入口删除本批对应场景、局部辅助、并发循环、逐项输出和已迁移局部变量。
8. 比较迁移前后 119 项编号集合与结果，运行完整自检和连续 20 轮回归。

## 11. 入口拒绝与失败收口

| 场景 | 分类 | 动作 |
| --- | --- | --- |
| #241 未完成或实际接口与假定契约不一致 | 逻辑内返回 | 不改代码，退回设计计划窗口修订 |
| 当前代码锚点、验收编号集合或共有夹具边界漂移 | 逻辑内返回 | 更新事实和计划，不机械搬移旧行号 |
| 自检上下文缺少必需服务 / 仓库只读材料 | 逻辑内返回 | 不登记该批次，不进入场景 |
| 运行器拒绝重复顺序 / 编号或空回调 | 逻辑内返回 | 不执行批次，不触碰业务结构 |
| 前置自检失败且强类型交接槽未封存完整材料 | 设计内自检合同失败 | 后继回调仍执行一次，只返回前置材料不完整，不进入依赖场景 |
| 单项按既有语义不通过 | 设计内结果 | 保留原编号，继续其它单元，批次最终非零 |
| 迁移后 119 项编号缺失、重复或结果与基线不一致 | 追根因解决 | 停止依赖路径，定位首次偏差 |
| 共有夹具被复制、并发线程未 join 或回调未恰好一次 | 追根因解决 | 停止 #243，修正所有权和收口 |
| 正式领域写入读回出现内部不一致 | 追根因解决 | 按原领域入口追根因，不用运行器汇总掩盖 |
