## 1. 目标

形成动态模式的第一层只读材料：

```text
特征值 -> 唯一语义特征定义
观察回合内显式片段 -> 完整动态证据
同场景 / 同主体准入 -> 稳定排序
规范动态步 -> 完整片段结构键
返回前来源复核 -> 不可变规范动态窗口
```

本计划不聚类重复片段，不计算因果概率，不创建抽象动态、二次特征、因果、方法或概念。

## 4. 实施步骤

### S0：实际接口复核

读取 #231 正式记录和当前特征、动态、状态、二次特征服务。逐项记录回合身份、片段顺序来源、语义解析、动态当前性、锁边界和二次特征角色结论。任一假定需要越过禁止服务或发明权威字段时停止并退回设计。

### S1：特征语义只读解析

在特征服务内形成特征值到唯一归属特征、实例槽位到唯一模板特征定义和值 / 版本的值式入口。目标非特征值或原始类型不支持逻辑拒绝；当前有效特征值归属零 / 多、实例槽位模板定义零 / 多和同版本矛盾追根因。动态模式服务不包含特征值服务。

### S2：显式回合 / 片段和动态证据

新增值式 DTO 与动态模式服务候选。调用方必须提交非空显式片段组，每个片段携带回合身份和非零唯一片段序号；服务逐条读取完整动态和语义特征，要求片段内及同回合材料同场景 / 主体、动态不跨片段重复，并按片段序号与片段内时间 / 完整句柄稳定排序。

### S3：结构键和当前性

形成版本 1 完整结构键。实例身份只保留为来源证据；二次特征上下文按 #231 结论统一启用或为空。哈希只分桶。窗口形成后逐条重读动态、归属、模板定义和值版本，来源漂移则整体逻辑返回。

### S4：自检与收口

覆盖 A01-A16、当前值零 / 多归属追根因、同时间戳 / 片段序号排序、规则版本、哈希碰撞、过期窗口和内部矛盾诊断。完成规范、工程登记、实施 / 验收记录、计划归档、治理、精确提交和推送。

## 8. 失败收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| #231 接口漂移或未完成 | 依赖门控返回 | 不改代码，退回设计 |
| 外部请求无效、目标非特征值、原始类型不支持、跨主体 / 场景或来源过期 | 逻辑内返回 | 不形成窗口，不写结构 |
| 二次特征角色未证明稳定 | 明确降级 | 全部窗口统一不写二次上下文键 |
| 当前特征值归属零 / 多、槽位模板零 / 多或同版本读回矛盾 | 追根因解决 | 停止返回，不选首项或用缓存兜底 |
| 哈希碰撞但完整结构键不同 | 正常算法分支 | 保留不同键，不合并 |
| 返回前材料漂移 | 逻辑内返回 | 丢弃整个值式窗口，不重试到偶然一致 |
