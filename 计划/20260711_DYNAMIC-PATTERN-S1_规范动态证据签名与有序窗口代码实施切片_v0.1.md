```text

数据操作层一次共享许可 -> 权威动态 / 状态 / 目标 / 动作批量材料
```

## 4. 实施步骤

### S0：接口与范围复核

记录当前 HEAD、#231 提交、正式协议 / 仓库入口签名、阶段占用和工作树状态。确认 C++ dirty 集、stash 和其它窗口文件不进入本切片。

### S1：稳定材料与纯值算法

新增 `材料.动态模式.ixx`，定义版本化请求、回合身份、权威批次、规范动态步、完整结构键、窗口和值式结果。新增 `算法.动态模式.ixx`，只做全字段比较、排序、稳定哈希和窗口形成，不访问仓库、服务、接线、时钟或线程。

### S2：服务专用数据操作

新增 `数据操作.动态模式.ixx`，隐藏结构事务接线和四仓库引用：

```text
纯值前置通过后一次取得共享许可。
同一许可内读取动态、前后实例状态、抽象目标状态、来源动作、节点、主信息和主键。
关系按完整句柄、版本、类型、端点、顺序号及有效 / 历史阶段复核。
形成完整值式批次后释放许可。
```

禁止调用会再次取得许可的既有公开服务 / 数据操作入口；禁止令牌逃逸、跨线程使用、写仓库或返回内部引用。

### S3：业务服务与组合器

新增 `服务.动态模式.ixx` 和 `组合.动态观察.ixx`。组合器接收新版回执、内嵌冻结请求、非零片段序号和规则版本；服务复核目标抽象状态、来源动作、场景、主体、前后状态和回执一致性，再调用纯值算法形成规范窗口。

公开面不得出现原始令牌、许可、接线、仓库、会话或锁。版本 1 对特征值、实例状态、二次特征和其它目标类型逻辑返回，不修改特征服务。

### S4：自检与接线

新增 `自检.动态模式.ixx`，承载 A01-A18。工程和 filters 对六个模块各唯一登记为 `ClCompile`。入口只在 Debug 下 import 自检模块并向最终回归阶段 630 登记一次无捕获回调；不得放入测试正文、线程编排或逐项业务逻辑。

### S5：验证与治理收口

完成 Debug / Release x64 Rebuild、完整自检、专项 A01-A18、并发共享许可阻塞、Release 隔离、最终 Debug 连续 20 / 20、严格规范和范围扫描。通过后归档原计划，登记 630，同步实施 / 验收 / 队列 / 状态记录，只提交本切片允许文件并执行非强制快进推送。

```text

同回合片段序号重复、同一动态跨片段重复或请求数量超上限。
任务、动态、状态、目标、动作、场景、主体句柄无效或版本漂移。
动态目标不是冻结任务目标抽象状态，或来源动作不是冻结动作入口。
动态前后状态、回执实际状态、场景、主体、时间或关系证据不一致。
目标为特征值、实例状态、二次特征或其它版本 1 未支持类型。
```

## 8. 失败收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 实际接口、阶段或允许范围漂移 | 退回设计 | 不改代码、不改号、不扩大范围 |
| 请求前置无效 | 逻辑内返回 | 仓库零写入，尽可能零读取 |
| 共享许可拒绝 | 逻辑内返回 | 不返回半批次，不忙等重试 |
| 句柄自然版本漂移、必需当前关系失效 | 逻辑内返回 | 来源过期，不用旧缓存兜底 |
| 版本 1 不支持目标 | 逻辑内返回 | 不调用特征服务，不降级解释 |
| 历史关系可审计 | 正常历史分支 | 只证明冻结来源，不恢复当前有效性 |
| 同一许可内类型、主信息、关系槽、端点或协议证据矛盾 | 追根因解决 | 停止返回，不选首项或顺序重读 |
| 哈希碰撞但完整键不同 | 正常算法分支 | 保持不同键 |
| 异常 | 内部失败 | 许可 RAII 释放，无结构变化、无能力泄漏 |
