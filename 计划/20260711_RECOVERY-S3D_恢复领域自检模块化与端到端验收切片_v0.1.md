## 1. 目标

把 RECOVERY-S3A 至 S3C 的全部领域自检固定在独立真模块中，清除 `入口.cpp` 中恢复领域场景正文，并执行端到端启动期恢复验收：

```text
恢复任务 / 屏障自检模块
+ 运行期上下文 / 宿主自检模块
+ 启动顺序 / UI / SQL 隔离自检模块
-> 自检.恢复总成
-> 入口一次调用和总结果
-> 端到端从候选到启动 / 停止
```

本切片不修改任务、恢复、上下文、线程、SQL 或控制面板生产语义。

## 4. 实施步骤

1. 复核三个自检模块、恢复总成、入口当前恢复相关代码和 #236 分段矩阵。
2. 将所有 RECOVERY-S3A/S3B/S3C 场景、测试仓库 / 邮箱 / 线程、并发循环、故障注入和逐项输出移入相应自检模块。
3. 恢复总成只聚合值式报告、稳定排序输出并返回总通过；不实现生产逻辑、不修复结构。
4. 入口删除恢复领域 Axx 计算、并发线程和逐项输出，只保留上下文组装、一次套件调用和总结果。
5. 运行端到端全新 / 恢复互斥、任务迁移、屏障、许可、首发、租约、线程顺序、SQL / 面板故障隔离和停止收口。
6. 静态证明生产模块不 import 自检模块，自检不进入正常运行分支。
7. 完成 A01-A12、治理、归档、提交和推送；完成后解锁 #230。
