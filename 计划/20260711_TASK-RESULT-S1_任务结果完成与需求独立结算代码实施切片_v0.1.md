```text

-> 正式重读任务 / 方法 / 动作 / 状态 / 动态

-> 唯一任务结果 + 执行中到已完成
-> 独立需求结算与可重试待结算材料
```

本计划形成 #215 命名任务结果与需求结算可复用的通用契约；不实现命名专用绑定，不修改方法召回或选择。

## 4. 实施步骤

### S0：实际接口复核

读取 #224 强类型回执与最终回归阶段 600、#222 完成边界、#217 同域令牌和当前任务 / 需求 / 状态 / 方法 / 动态服务。记录现有任务结果、完成和需求结算入口是否会产生部分结构；若无法在允许范围内形成提交前完整准备，或阶段 610 不可用，停止并退回设计。

### S1：正式结果路由重读

上行桥保持只读。新增 `路由.任务结果治理.ixx` 和 `服务.任务结果.ixx` 真模块接收强类型回执后，再次逐项重读当前任务版本、选择、方法登记 / 规格、动作入口、实际状态、动作动态、主体和场景。过期、允许拒绝、精确重复在写前返回；同当前版本结构矛盾追根因。

### S2：状态结构化比较

状态服务新增只读比较结果，区分已达成、未达成、材料不足和内部不一致。比较使用目标 / 实际状态定义、主体与场景，不让任务服务通过裸 I64 相等直接裁决业务目标。

### S3：任务结果与完成

未达成时保留状态 / 动态证据，按 #222 迁移等待中或待重筹办，不写最终结果。已达成时取得 #217 同域独占许可，重读执行中当前态，写唯一实际结果并经专用入口迁移到已完成；完整读回后才返回完成材料。重复同结果幂等，冲突结果追根因。

### S4：独立需求结算

任务完成读回后再调用需求服务。结算允许拒绝或暂不可执行时，保留任务已完成，返回可重试待结算材料；不得回滚动作、实际状态、任务结果或生命周期。失败 / 取消任务不得结算为已满足。

### S5：#215 接口与治理收口

固定 #215 可复用的结果 / 完成 / 结算接口，新增 `自检.任务结果结算.ixx` 真模块承载 A01-A15、迟到 / 重复 / 冲突 / 未达成 / 结算重试 / 并发测试和逐项输出，入口只以最终回归阶段 610 无捕获回调登记。完成规范、工程登记、实施记录、归档、治理、提交和推送。

## 8. 失败收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| #224 / #222 / #217 接口漂移 | 依赖门控返回 | 不改代码，退回设计并保持 #215 门控 |
| 回执过期、允许拒绝或精确重复 | 逻辑内返回 | 不新增任务结果或版本 |
| 目标未达成 | 设计内分支 | 保留状态 / 动态，迁移等待 / 待重筹办 |
| 同当前版本回执与权威材料矛盾 | 追根因解决 | 不落账，不由协议兜底 |
| 结果 / 完成写后读回不一致 | 追根因解决 | 停止需求结算，不伪造完成材料 |
| 需求结算允许拒绝 / 暂不可执行 | 逻辑内返回 | 保留任务已完成，返回待结算材料 |
| 需求结算写后结构不一致 | 追根因解决 | 不回滚任务结果或动作事实，停止后续依赖 |

## JY-275 结果与结算许可所有权修订

1. `服务.任务结果.ixx` 或 `路由.任务结果治理.ixx` 的调用期编排负责 import #217 协调模块并取得许可；传统任务 / 状态 / 需求服务只包含数据桥、验证并下传值式令牌。

3. 需求结算暂不可执行时保留任务完成事实，不能通过继续持锁、跨服务回滚或未接线兼容入口补偿。无接线路径不参加 A05 事务验收。

5. 执行前必须核对 #217、#222、#224 的实际令牌重载、许可类型、模块导出和阶段 610。漂移时禁止修改数据桥 / 协调模块、复制协调逻辑或让传统头文件 import 真模块，退回设计。
