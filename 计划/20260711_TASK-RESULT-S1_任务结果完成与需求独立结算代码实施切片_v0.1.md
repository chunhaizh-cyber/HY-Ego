
## 7. 实施步骤

### S0：实际接口复核

读取 #224 正式提交、阶段 600、强类型回执、完整来源执行请求、工作线程取出入口和回执队列合同。逐签名复核本计划假定接口、现有任务 / 需求 / 方法 / 状态 / 动态业务入口及阶段 610 唯一性。

接口不一致时不修改 C++，形成正式断点并退回设计窗口；不得修改 #224 文件来适配旧假定。

### S1：需求目标达成只读裁决

在 `服务.需求.ixx` 增加值式裁决 DTO 和 `判定目标状态达成`。入口先拒绝无效需求 / 实际状态；经需求和状态业务公开面读取完整目标抽象状态与实际实例状态；第一轮以状态值相等返回已达成 / 未达成。

许可拒绝、版本漂移和内部不一致必须保留原状态分类。不得返回可写引用、裸 I64 快捷结论或调用数据操作写入口。

### S2：任务结果结算组合器

新增组合器，接收 #224 回执和本轮幂等 / 时间材料。重读任务、需求、当前选择、方法、动作、实际状态和动态，逐字段复核来源请求。

未达成时调用任务服务迁移待重筹办；达成时调用任务服务现有原子完成入口。任务完成完整读回后调用需求服务独立结算；结算暂不可执行时返回非权威提示。

### S3：任务结果治理路由

新增路由，支持从工作线程锁外取出一条回执和隔离自检显式处理。先调用 #224 协议准入，再调用组合器。路由只持组合器窄引用，不持服务、数据操作、执行器或仓库。

传统任务管理上行桥保持零差异；不从治理消息文本重建回执。

### S4：自检与收口

新增独立自检模块承载 A01-A18：非法参数、过期 / 漂移、未达成、达成、完成幂等、冲突结果、失败 / 取消终态、结算暂不可执行、同回执重试、可选因果、三类并发和 Release 隔离。

入口只在 Debug 最终回归阶段以顺序 610 登记一次。完成工程登记、实施记录、计划归档和项目记忆同步；按项目自动提交规则只提交本切片文件，不自动推送。

## 8. 入口拒绝

```text
回执协议版本 / 材料性质不匹配，或缺完整来源执行请求。
编号为零、用途混用、时间倒序、句柄形状无效。
任务不是精确执行中，也不是同实际结果的已完成幂等重试。
任务来源需求、场景、主体、目标状态与来源请求不一致。
当前选择、方法登记、动作入口或来源版本正常漂移。
实际状态非完整实例状态，或场景 / 主体不属于任务。
动态非完整实例动态，或后状态 / 目标 / 来源动作 / 场景 / 主体不匹配。
已失败 / 取消任务收到迟到回执。
已完成任务收到不同实际结果。
可选因果引用句柄形状无效。
```

所有入口拒绝必须发生在对应写入口调用前，不允许补默认值、切换旧服务或扫描整仓修复。

## 9. 非成功返回和失败收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| #224 实际接口或阶段漂移 | 依赖门控 | 不改代码，按原 #225 / DQ-117 退回设计 |
| 协议、参数、过期、正常版本漂移 | 逻辑内返回 | 结构不变化，返回明确来源问题 |
| 目标未达成 | 逻辑内分支 | 迁移待重筹办；不写结果 / 完成 / 结算 |
| 同版本权威结构矛盾 | 追根因解决 | 停止 610，不用回执或日志兜底 |
| 任务完成入口无写入拒绝 | 逻辑内返回 | 允许同回执再次权威复核 |
| 任务完成写后读回不一致 | 追根因解决 | 停止需求结算，不伪造完成 |
| 需求结算暂不可执行且无写入 | 逻辑内返回 | 保留任务完成，返回非权威提示 |
| 结算写后不一致或权威冲突 | 追根因解决 | 不回滚任务完成，停止学习 / 恢复后继产物 |
