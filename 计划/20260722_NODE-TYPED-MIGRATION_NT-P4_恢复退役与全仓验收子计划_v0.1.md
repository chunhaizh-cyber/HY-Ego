## 禁止文件与行为

禁止在线热替换、逐仓发布、恢复失败自动全新启动、运行时猜测旧槽位、长期双格式双写、通过普通公开服务注入恢复记录，以及用 SQL、日志或显示补齐候选。FREEZE、RESTORE、LEGACY 不得修改默认装配、启动、入口、旧服务头、旧调用方、`核心/句柄.h` 或物理删除旧文件。

## 全新与恢复候选验证

全新候选与恢复候选分型形成，但必须经过等强验证后才能封装为 `已验证运行期上下文候选`：

```text
全新候选
-> 全量身份 / 关系 / 领域记录 / 历史互证
-> 索引、自我投影、概念投影确定重建
-> 形成规范化初始冻结并自洽校验
-> 已验证候选

恢复候选
-> 同一套全量互证与投影重建
-> 再冻结与输入规范化权威载荷相等
-> 已验证候选
```

两类候选共用空宿主首次发布入口。全新候选不得绕过验证直接发布。
