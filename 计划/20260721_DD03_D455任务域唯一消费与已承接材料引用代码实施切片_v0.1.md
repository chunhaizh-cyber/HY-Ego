## 8. 实施步骤

### S0 身份、基线和实际接口复核

按第 2.5 节核对任务 worktree、上游计划、模块名、阶段 844、允许文件和实际签名。任何漂移按第 2.6 节退回。

### S1 建立任务外设材料承接协议

在协议模块中建立详细设计第 6—8 节全部强类型身份、状态、等待特征快照、D455 等待约束、不透明引用和具名结果。禁止裸 bool 裁决、自由文本原因、线程编号、队列地址、裸指针和交接包公开字段。

### S2 建立任务运行态专用数据操作

实现工作项、等待项、窗口和已承接记录侧表及唯一索引。所有记录按身份和版本只增 / 失效，不原地改写历史版本。实现任务侧候选的移动独占能力、预留容量、精确读回和析构撤销。

数据操作可以内部持有 DD-02 交接包，但不得公开其可写对象或原始存储地址。

### S3 建立任务承接业务服务

服务先用 `任务业务服务::读取任务` 复核完整承接、生命周期关系和版本，再登记工作项、等待特征和消费窗口。实现固定匹配顺序、确定性优先级、四类产品用途边界、范围 / 质量 / 时效和重复策略。

仅有生命周期 `等待中` 而缺最小等待特征快照必须拒绝。服务不得迁移任务生命周期或写世界事实。

### S4 建立唯一消费组合器和跨侧提交

组合器移动持有唯一 `D455任务域消费权`，串行编排：

```text
冻结窗口快照
-> DD-02 承接候选
-> 任务 / 等待完整匹配
-> 任务侧候选预留和读回
-> DD-02 正式确认
-> noexcept 安装交接包
-> 原子发布任务记录和完整读回
```

DD-02 确认前失败必须撤销或释放两侧候选。确认后不得返回普通失败；异常进入内部隔离并停止组合器依赖路径。

### S5 建立当前性、领取和窗口收口

实现不透明引用的上下文、任务、工作项、窗口和版本全量复核；实现一个记录一个领取候选、释放恢复和确认进入工作包的通用入口。DD-04 未形成前不得构造真实冻结请求或选择方法。

窗口关闭必须拒绝新候选、撤销未发布候选、失效待领取记录，并等待工作包保留中 / 内部隔离 / 不可舍弃材料收口。

### S6 建立独立自检模块

实现 DD03-A01—A20。夹具必须提供完整任务权威材料、四类 DD-01 产品、DD-02 队列和交接包；不得依赖真实相机、生产上下文、日志文本或控制面板。

### S7 工程、入口、验证和任务分支收口

## 9. 入口拒绝与失败收口

| 入口 | 写前拒绝 | 写后异常 |
| --- | --- | --- |
| 工作项 | 无效身份、任务不完整 / 终态 / 版本错 | 读回不同为内部逻辑错误 |
| 等待项 | 缺特征来源、合法生产者、重触发、时效或产品约束 | 当前特征集合不闭合为内部错误 |
| 窗口 | 错绑定、过期、零数量、非当前等待 | 同版本双绑定为内部错误 |
| 报告匹配 | 无报告、错产品 / 范围 / 质量 / 时效、合法重复 | 同输入确定性裁决歧义为内部错误 |
| 任务侧候选 | 容量不足、确认前当前性变化 | 候选读回不同则撤销并追根因 |
| DD-02 确认后 | 无普通拒绝 | 安装 / 发布 / 读回矛盾进入内部隔离 |
| 引用读取 / 领取 | 错上下文、任务、工作项、窗口、版本或用途 | 当前记录缺交接包 / 双领取为内部错误 |
| 窗口关闭 | 有保留中工作包时等待 | 静默销毁隔离 / 不可舍弃材料为内部错误 |
