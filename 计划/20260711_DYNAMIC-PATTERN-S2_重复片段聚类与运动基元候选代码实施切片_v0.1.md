## 1. 目标

在 #232 规范动态窗口之上形成三层明确候选：

```text
一个显式片段窗口 -> 单实例片段候选
至少两个不同回合各有完整片段键相同的窗口 -> 重复聚类候选
重复聚类且至少一步有有效来源动作 -> 运动基元候选
```

聚类只向上组织实例证据，不执行向下抽象，不创建抽象动态、二次特征、方法、因果或概念。

## 4. 实施步骤

### S0：实际接口复核

读取 #232 实施记录、正式 DTO、当前性入口和规则版本。逐项比较完整结构键、回合 / 片段身份、来源动作和锁边界；任一关键字段缺失或需重新解释实例身份时停止并退回设计。

### S1：纯值片段算法

扩展动态模式算法，只接收完整不可变窗口。每个显式片段窗口形成一个实例片段；按可选哈希分桶后逐字段全量比较，规则版本不同直接分开。

### S2：重复聚类

按回合完整身份去重；至少两个不同回合完整片段键相同才形成重复聚类。来源回合组和输出候选按结构键 / 完整身份稳定排序，线程完成顺序不参与结果。

### S3：运动基元候选

只从重复聚类中检查来源动作；至少一步有有效动作才返回运动基元候选。无动作时保留重复聚类，不把它解释为动作方法。

### S4：自检与收口

继续扩展 `算法.动态模式` 与 `服务.动态模式`，新增独立真模块 `自检.动态聚类` 承载 A01-A14、单回合、两回合、重复身份、版本分离、哈希碰撞、动作有无、来源过期和并发确定性。入口只登记最终回归阶段 640 无捕获回调；不建立第二个动态模式服务。完成工程登记、实施 / 验收记录、归档、治理、精确提交和推送。

## 8. 失败收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| #232 接口漂移或未完成 | 依赖门控返回 | 不改代码，退回设计 |
| 窗口过期、结构不完整、重复回合或版本不一致 | 逻辑内返回 | 不形成聚类，不写结构 |
| 哈希碰撞但完整键不同 | 正常算法分支 | 分开候选，不报告错误 |
| 完整键相同但全量比较结果矛盾 | 追根因解决 | 停止分组，定位算法实现 |
| 只有一个支持回合 | 正常候选分支 | 只返回单实例片段 |
| 重复片段无有效来源动作 | 正常候选分支 | 只返回重复聚类 |
