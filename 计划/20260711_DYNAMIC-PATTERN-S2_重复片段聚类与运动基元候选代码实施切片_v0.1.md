## 1. 目标

在 #232 正式 `规范动态窗口` 之上形成三层非权威值式候选：

```text
一个完整规范动态步 -> 一个单步片段实例候选
同一完整结构键至少由两个不同完整回合支持 -> 一个重复动态聚类候选
重复聚类 + 版本 1 已保证的有效来源动作 -> 一个运动基元候选
```

本轮不把多个步骤或多个窗口拼接为多步运动序列。场景、主体、任务、方法、发生时间戳、回合、片段序号和来源动态只作来源证据；它们不进入跨实例结构相等键。

## 4. JY-364 聚类合同

### 4.1 输入单位与显式容量

新增纯值聚类请求，输入为 `规范动态窗口` 组。窗口数不得超过 `动态模式最大片段数量`，所有窗口的总步骤数不得超过 `动态模式最大片段数量 * 动态模式最大片段数量`；上限必须作为公开常量和入口拒绝合同，不得隐藏在算法内部。

每个完整 `规范动态步` 形成一个单步片段实例。多个步骤即使来自相邻片段序号、相近时间、同一场景或同一主体，也不得在本轮自动拼接为多步模式。

### 4.2 结构键与来源证据

跨回合结构相等只比较 `动态模式结构键` 全部字段：

```text
被改变目标完整句柄
前状态值
后状态值
有来源动作
来源动作完整句柄
规则版本
二次特征角色（版本 1 必须为空）
```

以下字段保留在来源实例中，但不进入结构相等键：

```text
场景、主体、任务、方法
发生时间戳、回合身份、片段序号
来源动态完整句柄、来源批次、关系证据
```

### 4.3 哈希分权

```text
结构键稳定哈希：只用于召回候选桶。
结构键完整比较：唯一相等裁决。
规范动态窗口稳定哈希：只证明窗口材料封装稳定，不参与聚类相等或去重裁决。
```

哈希相同但完整键不同必须分开；不得用容器位置、桶位、窗口哈希或来源动态句柄替代结构语义。

### 4.4 来源去重与冲突

```text
同一来源动态完整句柄重复提交且来源身份 / 结构键完全一致：只保留一份，不增加实例数或支持数。
同一来源动态映射到不同结构键、回合、片段序号或来源材料：内部不一致，追根因。
同一回合锚点映射到不同完整回合身份：内部不一致，追根因。
同一完整回合内的多个同键实例：可以保留去重后的来源证据，但对该聚类只贡献一次支持。
同一窗口重复、输入顺序变化或相同窗口复制：不得改变聚类、支持数和输出顺序。
```

### 4.5 重复聚类与运动基元候选

同一完整结构键至少由两个不同完整回合支持，才形成重复动态聚类候选。只有一个支持回合时正常返回单步实例，不形成重复候选。

#232 版本 1 的完整结构键已经强制 `有来源动作 == true`、来源动作句柄有效且二次角色为空。因此重复聚类成立时运动基元候选条件同步成立。旧计划中的“重复聚类但无来源动作”分支在当前正式 DTO 下不可达，不得为保留该分支而放宽 #232 完整性。

### 4.6 顺序与时间边界

片段序号和发生时间戳只用于来源证据与稳定输出排序：

```text
完整结构键
-> 完整回合排序锚点
-> 片段序号
-> 发生时间戳
-> 来源动态完整句柄
```

本轮不要求跨窗口片段序号连续，不设置最大时间间隔，不按时间相近推断同一模式，不枚举任意子序列。多步连续性、时间约束和序列运动基元必须等正式片段边界 / 回合合同存在后另建专项。

## 5. 目标 DTO 与调用路径

### 5.1 值式材料

`材料.动态模式` 窄幅增加唯一所有权的值式类型：

```text
动态聚类请求
单步片段实例候选
重复动态聚类候选
运动基元候选
动态聚类处理状态 / 拒绝原因
动态聚类结果
显式容量常量
```

这些类型只保存不可变值和来源副本，不保存仓库引用、接线、许可、令牌、会话、锁、指针、回调或线程对象。

### 5.2 调用路径

```text
动态观察组合器::形成动态聚类候选(纯值请求)
-> 动态模式业务服务::形成动态聚类候选(纯值请求)
-> 动态模式算法纯值校验、去重、分桶、完整比较、支持计数和稳定排序
-> 动态聚类结果
```

聚类路径不得调用 `动态模式数据操作`，不得取得第二份共享许可，也不得回到仓库复核“当前性”。需要当前业务裁决时，调用方必须另行重新经过 #232 正式观察入口形成新窗口。
