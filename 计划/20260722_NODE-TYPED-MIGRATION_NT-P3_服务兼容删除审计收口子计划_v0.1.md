# NT-P3 服务兼容删除审计收口子计划

日期：2026-07-22

版本：v0.2

上级：`#338 / DQ-230 / NODE-TYPED-MIGRATION`

状态：依赖 NT-P2C；P3A—P3C 受工程接线所有权串行门控，P3D -> P3E -> P3-SHARED 固定串行；不可派发

## 目标

在隔离节点直接新域形成剩余对象服务、强类型退役删除审计、值式请求适配和新生产候选调用图。P3 不改旧默认服务、调用方、入口或装配；兼容迁移只形成逐调用点 P4 切换映射，不在 P3 提前切换或双写。

## 叶子与唯一所有权

| 叶子 | 唯一范围 | 前置 |
| --- | --- | --- |
| P3A / #343 | 只读消费 P2C 的存在 / 场景接口；新建节点直接系统角色、存在场景增量组合和自检 | P1、P2C |
| P3B / #344 | 新建语素、基础信息、轻量因果、概念结构服务和概念签名参与者 | P1、P2B；P2C 作为共同冻结基线 |
| P3C / #345 | 新建需求、任务、方法、用途观察强类型记录和节点直接服务 | P1、P2B；P2C 作为共同冻结基线 |
| P3D / #346 | 新建退役协调器、独立领域退役适配、审计和投影后验 | P2A—P2C、P3A—P3C |
| P3E / #347 | 新建节点直接请求适配、候选业务 / 只读组合和逐调用点映射自检 | P3A—P3D |
| P3-SHARED / #348 | 隔离新域装配、统一新域自检接线和全图验证 | P3A—P3E |

P3A 不得修改 P2C 独占的 `数据操作.节点直接存在场景.ixx`、`服务.节点直接存在.ixx`、`服务.节点直接场景.ixx`。增量能力进入独立新模块，只能 import P2C 的值式公开接口；接口不足时退回 P2C 原所有者，不得复制算法或重开仓库。

P3D 的各领域退役能力进入独立适配模块，只读消费 P2A—P2C 与 P3A—P3C 固定退役接口；不得回写上游记录或服务模块。

## 禁止文件

禁止修改十四份旧兼容服务头、11 个现行数据操作、17 个现行现代服务、真实旧调用方、`入口.cpp`、默认装配、启动、运行期宿主、P1 / P2 固定文件、D455、SQL / 控制面板和快照恢复。不得为了保持旧函数签名保留可写旧句柄或无类型值槽。

## 步骤、并发与工程所有权

1. P3A—P3C 在互不重叠的新模块中形成剩余领域服务。当前没有独立工程验证路径，因此按工程 XML 增量所有权串行执行，不伪造并行。
2. P3D 通过独立适配模块完成强类型退役；删除 / 失效先冻结精确影响面，关系和领域记录先于节点收口，审计保持历史可读。
3. P3E 形成值式请求适配和完整候选调用图，并冻结旧调用点到 P4 的逐项映射；不修改旧调用点。
4. P3-SHARED 串行接入固定提交，唯一修改隔离装配并最终统一整理 `海中鱼巣/自检.运行器.ixx` 的新域接线；默认入口仍不可到达新域。
5. `海中鱼巣.vcxproj`、`.filters` 和 `海中鱼巣/自检.运行器.ixx` 的新增模块 / 专属自检登记按 P3A -> P3B -> P3C -> P3D -> P3E -> P3-SHARED 固定提交串行移交；任一时刻只有当前叶子取得增量登记所有权。

P3A—P3C 业务语义本可并行，但在形成独立可编译验证路径前，工程 XML 和双配置验证构成不可消除的共享所有权，所以本版本不建立并行执行批次。后续若建立共享预登记或独立编译夹具，必须另行设计并重算依赖，不能仅改状态字段。

## 失败和验证

任一调用点不能无歧义映射、出现循环依赖、公开签名暴露事务能力、需要越过叶子所有权或需要修改旧默认域时正式退回。

每个业务叶完成静态隔离、模块登记、Debug / Release 编译和专属自检；P3-SHARED 再完成两配置完整新域自检、默认入口零可达、11 / 17 / 14 现行入口只读不变、删除后无悬空关系 / 记录 / 索引、审计历史可读、候选请求不双写、`strict`、`diff check` 和白名单检查。

## 完成边界

只证明隔离新域候选服务和逐调用点切换映射已形成；旧默认生产调用点仍未迁移，旧物理文件、默认切换、快照格式和跨重启恢复仍由 NT-P4 收口。
