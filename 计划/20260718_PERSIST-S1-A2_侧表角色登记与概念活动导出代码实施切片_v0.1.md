## 1. 目标与依据

依据恢复流程图 v0.3 当前 JY-422 修订、详细设计第 17 / 20 节、#294 / #295 两份矩阵和 #305 四仓库导出合同。本切片只形成特征值 I64 / Vec 全侧表、系统角色、领域所有者登记和概念活动材料的值式导出，并扩充 `权威冻结材料.数据.h` 的唯一 DTO；不取得统一独占许可、不形成九段冻结点、不编码文件。

来源性质为 MIG-18 重新实现 / 新增约束 / 全新能力，不计旧能力迁移。O1 只登记“权威事实在四仓库、无私有权威侧表、当前未接生产”，不得借登记接入运行期业务装配。

## 2. 依赖与当前事实

#305 假定已提供四仓库全状态 DTO。当前特征值服务私有持有 I64 版本表和 Vec 值 / 类型 / 容器版本；系统角色清单是值式材料；#296 概念活动服务私有持有三角色、完整签名和活动版本；当前没有统一领域所有者登记。

目标内部合同：

```text
特征值权威导出参与者：同域有效令牌下复制全部 I64 / Vec 材料与版本，不暴露服务公开业务令牌入口。
领域权威登记：领域编号、格式版本、是否存在私有权威侧表、所复用快照段和确定性重建声明。
系统角色导出：版本化完整清单和值式参数身份。
概念活动导出：三角色、完整规范签名、活动版本和重建互证视图；上一候选图永不导出。
```

## 6. 非成功返回

无效令牌、登记版本无效、请求材料不完整为逻辑内返回，不返回部分材料。取得有效同域令牌后侧表类型 / 版本矛盾、系统角色与仓库不一致、概念活动重建视图不相等或领域登记重复所有权为追根因。实际接口需计划外文件或无法保持 O1 未接生产为接口漂移。
