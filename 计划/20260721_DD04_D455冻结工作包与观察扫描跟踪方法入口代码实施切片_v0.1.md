1. `协议.任务执行请求.ixx` 的 `任务执行适配器种类` 当前只有 `任务目标状态值变化`。
2. `任务执行冻结请求` 已携带调度、任务、方法、动作材料和适配器，但没有工作项、消费窗口、DD-03 引用、产品、业务前置或完整条件 / 结果快照。

7. 当前方法结构没有单字段通用内容版本。可冻结证据是生命周期关系版本、规格节点 / 关系版本、条件 / 结果来源版本、动作入口关系和稳定动作键。
8. 当前生产代码没有 D455 方法材料入口合同、不可变工作包、观察 / 扫描 / 跟踪只读视图，也没有完整的已确认存在、扫描基准或跟踪目标前置生产接线。

### 2.6 接口漂移退回规则

出现下列任一情况，执行窗口不得修改代码：

```text
#324、#325、#326 或 #328 尚未正式集成和设计归档。
任务筹办执行权不能贯穿到基础冻结，或冻结入口仍可在无权 / 错权时读取方法规格。
DD-01 产品身份、内容版本、产品 variant 或受控解析边界与假定合同不一致。
DD-03 没有移动独占领取候选，或确认进入工作包后仍允许普通失败 / 回滚到待领取。
DD-03 公开接口要求 DD-04 取得 DD-02 消费权、报告候选、交接包、队列地址或裸材料所有权。
任务执行基础冻结不再能独立读取完整任务、当前选择、方法和唯一动作入口。
方法业务服务不能区分并读回完整条件、结果和动作规格，或排序 / 版本材料无法形成确定快照。
实现必须修改任务管理线程、任务工作线程、路由.任务执行调度或生产运行期装配。
实现必须新增核心节点类型、关系类型、通用仓库或原始事务穿透服务的路径。
既有确认排队或目标状态值变化执行入口无法在不改旧语义的前提下显式拒绝 D455 适配器。
阶段 845 或六个模块名已被占用，或者 #327 / DQ-219 已不再唯一指向本计划。
入口不允许最小自检登记，必须展开完整场景到入口。
允许文件与其它活动 worktree 的文件或结构所有权冲突。
```

执行窗口只写正式接口漂移断点并退回设计窗口，不得修改规范、扩大允许文件、平行声明 DD-01—DD-03 类型、把产品或 DD-03 引用塞进通用冻结请求、给识别添加外设入口，或用日志 / bool / 线程编号弥补结构缺口。

## 8. 实施步骤

### S0 身份、基线和实际接口复核

按第 2.5 节核对任务 worktree、上游计划、模块名、阶段 845、允许文件和实际签名。任何漂移按第 2.6 节退回。

### S1 窄扩展通用任务执行冻结

在 `任务执行适配器种类` 追加稳定观察、扫描变化和目标跟踪三个 D455 适配器；新增纯值“可冻结”判断，未知值拒绝。

`任务执行组合器` 构造时新增支持适配器参数，默认保持目标状态值变化；冻结结果使用构造时适配器。公开只读当前性复核，但不暴露业务服务、仓库或事务能力。

`确认任务进入排队中` 和 `执行任务目标状态值变化` 必须显式只接受目标状态值变化。现有构造调用点、阶段 600、状态变化和回执语义不得漂移。

### S2 建立工作包协议

在协议模块建立详细设计第 6 节全部强类型身份、入口种类、状态、结束原因、材料绑定、业务前置 variant、方法内容快照、不可变工作包和值式结果。

协议不得 import 队列、仓库、服务、组合器、线程实现或自检；不得包含产品负载、受控材料组、交接包、裸指针、`shared_ptr`、队列地址、线程编号或自由文本业务状态。

### S3 建立工作包数据操作与业务服务

实现工作包身份、DD-03 承接记录和任务 / 筹办轮次索引；候选必须移动独占、不可复制，预留全部容量和状态槽，支持精确读回及确认前撤销。

服务先完成身份、基础冻结、绑定、前置、方法快照、期限、验证 / 归因 / 禁止项版本准入，再调用数据操作。发布后冻结内容不可原地修改，状态使用独立版本链。

### S4 建立不可变方法材料适配器

构造时完整验证合同集合：方法、动作入口、稳定动作键、方法内容快照版本、任务执行适配器、唯一产品、唯一视图、合同版本和允许前置必须闭合。构造完成后合同集合不可变。

适配器只收窄已有方法的技术输入，不读取候选索引、不选择方法、不创建方法、不写方法长期事实。实现观察、扫描、跟踪三类一次调用内只读视图；原始逐簇和识别入口不存在。

### S5 建立工作包组合器和跨侧确认

组合器固定编排：

```text
DD-03 准备领取候选
-> 读取产品和值式来源
-> 通用基础冻结
-> 方法条件 / 结果 / 动作完整快照
-> 产品 / 适配器 / 合同 / 前置唯一匹配
-> 工作包候选预留和完整读回
-> 再次复核全部当前性
-> DD-03 确认进入工作包
-> noexcept 安装确认标记并原子发布工作包
-> 完整读回
```

确认前任一逻辑内返回必须撤销工作包候选并释放 DD-03 领取候选。确认后不得返回普通失败；安装、发布、索引、状态或读回矛盾必须锁存工作包和材料绑定到内部隔离并停止依赖路径。

### S6 建立当前性、只读视图和具名结束

每次排队、领取执行或形成视图前，复核任务 / 筹办轮次、方法生命周期、全部条件 / 结果、动作、适配合同、DD-03 引用、工作项、窗口、产品、业务前置、期限和禁止项。

冻结后合法事实变化使旧包失效并具名结束材料消费；需要继续时形成新筹办轮次和新工作包。执行期发现冻结前既有结构缺口必须进入内部隔离并标记筹办漏检。

析构只能撤销未确认本地候选；已发布工作包必须经具名结束入口和 DD-03 消费结束入口收口。

### S7 建立独立自检模块

实现 DD04-A01—A20。夹具必须使用 #324—#326 实际协议、#328 实际任务筹办执行权和隔离任务 / 方法结构，不依赖真实相机、生产运行期、日志文本、控制面板或 SQL。

并发至少包含：同一 DD-03 已承接引用 50 线程只形成一个工作包；不同任务、不同工作项使用同一可复用方法时可以并发形成和读取工作包。

### S8 工程、入口、验证和任务分支收口

按依赖顺序登记六个真模块；`入口.cpp` 只增加一个自检 import、阶段 845 的一次登记和既有总结果汇总。运行第 10 节完整验证，写专属实施记录，精确暂存计划允许文件，检查 staged diff，提交并非强制推送同名任务分支。

任务分支完成状态只能是 `分支完成待集成`，不得推送 `origin/main` 或宣称 #327 完成。

## 9. 入口拒绝与失败收口

| 入口 | 写前拒绝 / 逻辑内返回 | 写后异常 / 内部逻辑错误 |
| --- | --- | --- |
| 基础冻结 | 请求无效、任务尚不可冻结、冻结后合法漂移 | 已选择方法在冻结前缺方法 / 动作 / 必需规格 |
| DD-03 领取 | 错任务 / 工作项 / 窗口 / 用途、过期、已有领取者 | 同一承接记录双领取 |
| 产品映射 | 原始逐簇、识别、未知产品、空观察子集、前置未形成 | 同一方法动作存在两个合法产品 / 视图映射 |
| 方法快照 | 生命周期合法变化 | 条件 / 结果 / 动作读回缺失、异义或版本矛盾 |
| 工作包候选 | 容量不足、确认前当前性变化 | 候选读回不同或无法精确撤销 |
| DD-03 确认前 | 撤销工作包候选并释放领取 | 撤销后仍可见半结构 |
| DD-03 确认后 | 无普通拒绝 | 安装 / 发布 / 索引 / 状态 / 读回不一致，内部隔离 |
| 既有排队 / 状态值执行 | 任一 D455 适配器显式拒绝 | D455 绕过工作包进入既有队列 |
| 执行前当前性 | 冻结后事实变化、过期、授权取消 | 冻结前结构缺口到执行期才发现 |
| 材料视图 | 失效、错入口、前置过期 | 产品身份 / 内容版本 / 受控解析错配 |
| 具名结束 | 同义已结束读回 | 静默析构、异义二次结束或丢弃不可舍弃材料 |
