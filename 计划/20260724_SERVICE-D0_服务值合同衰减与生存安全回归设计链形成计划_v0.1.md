## 1. 目标

在不修改代码的前提下，形成以下范围闭合的设计包：

```text
有效人类需求与唯一服务合同
-> 冻结有效期和预算
-> 30% 预支 / 70% 完工余款
-> 服务准备实际衰减补回
-> 单调完整秒服务值维护
-> 最长未满足需求与 30 天门禁
-> 生存安全根值终止 / 休眠 / 主动运行控制
-> 服务值调节根值被动回归
-> 安全 / 服务任务根权限
-> 与分层安全任务权限和自我循环接线边界
```

全部机器数值使用 I64。压力概念不进入本设计。

## 3. 现状差距

只读代码扫描固定：

1. 当前只有安全值 / 服务值根需求初始化，没有服务合同、衰减、回归或任务根权限入口。
2. 自我线程等待超时直接继续，不能用循环次数或超时次数结算物理时间。
3. 固定安全根兼容路由明确拒绝服务根需求。
4. 需求、任务、方法没有合同唯一键、有效期、预算、结算阶段、有效进展或准备补回结构。
5. 任务请求与任务管理是 FIFO 消费，没有安全 / 服务权限来源和稳定排序。
6. 节点直接身份结构写入执行器可作为同事务底座，但不等于服务记录参与者或真实适配已存在。

这些事实只证明差距，不证明任何预冻结接口已经实现。

## 4. 设计冻结

本计划形成并冻结：

1. `SERVICE-C1 / v1`：唯一合同、有效期、预算、30% / 70%、到期未满足事件和准备补回。
2. `SERVICE-C2 / v1`：完整秒、最长未满足需求、30 天门禁、服务衰减段、生存安全根值回归和原子维护。
3. `SERVICE-C3 / v1`：安全 / 服务根权限、6150 组合、多来源取最高和确定性排序。
4. 默认有效期 `2,592,000` 秒是项目版本化默认，不冒充提出者明确表达。
5. `A=0` 是终止态、`A=1` 是可唤醒休眠态、`A>1` 允许主动运行；`1<L<H<=I64_MAX`。
6. 到期先形成终态，到期前仍未满足时发布一次同秒到期未满足事件；事件不恢复合同，也不建立压力概念。
7. 有效服务活动最低 1 只保护已经为正的服务值，不从零生成服务值。
8. 服务值只调节生存安全根值，不改 6140 分层安全值；真实安全事件同秒优先。
9. #379 只拥有十个隔离服务模块和专项源码。
10. #359 唯一拥有真实提供者、线程、任务管理、工程、入口、恢复和生产验收共享面。
