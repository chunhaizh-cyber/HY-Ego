## 1. 目标

在 #221 唯一当前态之上实现固定迁移矩阵、预期版本准入、并发线性化、失败 / 取消专用证据和终态封口：

```text
迁移请求
-> 写前矩阵与证据复核
-> #217 同域独占许可内重读当前态
-> 新状态与版本 N+1 当前关系
-> 旧关系失效并保留审计
-> 唯一当前态 / 连续历史读回
```

本切片不实现方法召回 / 选择、线程调度、任务结果完成入口或需求结算。

## 4. 实施步骤

### S0：接口复核

读取 #221 实施记录和实际函数签名，逐项核对当前关系、版本、历史、令牌和内部状态创建边界。若 #221 仍需最大时间戳推断、历史不可审计或事务域不能覆盖换代，停止并退回设计。

### S1：固定矩阵与请求

按详细设计 6.1 原样实现矩阵；请求必须含预期当前关系、预期版本、目标阶段、时间和幂等材料。通用入口禁止已完成、失败、取消；后二者使用专用入口，完成留给 #225。

### S2：独占换代

写前排除非法请求后取得同域独占许可，重读任务承接壳、当前关系、状态、版本和专用证据。创建版本 N+1 新状态 / 新关系，使版本 N 旧关系失效，完整读回后返回。进入提交点后不得普通失败。

### S3：幂等、并发与终态

精确重复读回既有当前材料；旧版本、同键异义和并发败者稳定拒绝。已完成、失败、取消无出边；执行中取消没有动作停止证据时拒绝，不用 bool 或日志承载证据。

### S4：回归与治理

新增 `自检.任务生命周期迁移.ixx` 真模块承载 A01-A15、并发同版本迁移、终态、非法跳转矩阵和逐项输出；入口只以最终回归阶段 570 无捕获回调登记。完成治理归档、提交和推送。

## 8. 失败收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| #221 / #217 接口漂移 | 依赖门控返回 | 不改代码，退回设计 |
| 非法跳转、旧版本、证据不足、终态复活 | 逻辑内返回 | 本次不写状态、关系或版本 |
| 精确重复请求 | 幂等逻辑内返回 | 读回当前结果，不增长历史 |
| 同旧版本并发败者 | 逻辑内返回 | 返回当前材料，不覆盖胜者 |
| 新状态 / 新关系写后读回不一致 | 追根因解决 | 停止依赖路径，不扫描修复 |
| 当前关系和历史在同版本自相矛盾 | 追根因解决 | 不返回普通失败后继续业务 |
