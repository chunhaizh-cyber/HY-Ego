## 4. 实施步骤

### S0：接口复核

读取 #221 实施记录和实际函数签名，逐项核对当前关系、版本、历史、令牌、状态候选、主信息索引 I64 入口和关系创建唯一性。确认关系 15 未占用、未知负例当前为 15、角色顺序号 40-46 与 570 未占用。若需修改普通引用全局唯一性、主信息仓库 ABI、新增第二种关系类型或事务域不能覆盖换代，停止并退回设计。

### S1：固定矩阵与请求

按详细设计 6.1 原样实现矩阵；请求必须含预期当前关系、预期版本、目标阶段、时间、非零幂等材料编号和固定七角色证据 DTO。空 / 非空角色都参与语义比较。通用入口禁止已完成、失败、取消；后二者使用专用入口，完成留给 #225。

### S2：独占换代

写前排除非法请求后取得同域独占许可，重读任务承接壳、当前关系、完整历史、状态幂等编号和专用证据。状态服务创建版本 N+1 状态候选；任务服务写类型 15 的角色关系和新生命周期关系。同一目标可跨角色复用，同一角色第二关系拒绝。提交前完整预读回，旧关系失效作为唯一提交点。

### S3：幂等、并发与终态

先按完整历史查询幂等编号：唯一命中且前驱、目标、时间和七角色逐字段同义时读回原迁移结果及当前投影；唯一命中但异义时拒绝，多项命中追根因。无命中后才按预期当前关系 / 版本判断旧版本和并发败者。已完成、失败、取消无出边；执行中取消没有动作停止证据时拒绝，不用 bool、摘要或日志承载证据。

### S4：回归与治理

新增 `自检.任务生命周期迁移.ixx` 真模块承载 A01-A15、并发同版本迁移、终态、非法跳转矩阵和逐项输出；入口只以最终回归阶段 570 无捕获回调登记。完成治理归档、提交和推送。

## 8. 失败收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| #221 / #217 接口漂移 | 依赖门控返回 | 不改代码，退回设计 |
| 非法跳转、旧版本、证据不足、终态复活 | 逻辑内返回 | 本次不写状态、关系或版本 |
| 精确重复请求 | 幂等逻辑内返回 | 读回原历史迁移结果及当前投影，不增长历史 |
| 同键异义 | 逻辑内返回 | 完整历史逐字段比较后拒绝，不覆盖既有状态 / 关系 |
| 同一目标承担不同合法角色 | 正常写入 | 用类型 15 的不同顺序号分别承载，完整读回两角色 |
| 同源同角色第二关系 | 逻辑内返回 | 仓库写前拒绝，不覆盖既有角色 |
| 同任务同键多项历史 | 追根因解决 | 停止依赖路径，不选择第一项或按时间排序 |
| 同旧版本并发败者 | 逻辑内返回 | 返回当前材料，不覆盖胜者 |
| 提交前新状态 / 证据 / 新关系预读回失败 | 追根因收口 | 删除新生命周期 / 证据关系并由状态服务撤销候选；旧关系保持有效 |
| 新状态 / 新关系写后读回不一致 | 追根因解决 | 停止依赖路径，不扫描修复 |
| 当前关系和历史在同版本自相矛盾 | 追根因解决 | 不返回普通失败后继续业务 |
