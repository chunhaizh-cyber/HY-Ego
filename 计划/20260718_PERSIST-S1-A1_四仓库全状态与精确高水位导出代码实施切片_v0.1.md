## 1. 目标与依据

依据恢复流程图 v0.3 当前 JY-441 替换子图、消费者流程图 v0.7、详细设计第 17 / 20—22 节、#294 四仓库矩阵和 #304 实际索引合同。本切片只在同一有效结构事务令牌下值式导出四仓库全部当前 / 历史记录、仓库身份、下一编号与节点创建序号精确高水位；不取得统一冻结许可、不导出领域侧表、不编码文件。

来源性质为 MIG-18 重新实现 / 新增约束，不计旧能力迁移。

## 2. 已验证当前事实与假定接口

四仓库已有局部读取能力，但缺统一“全部状态 + 精确高水位”值式合同。索引在 #304 后假定含所有者与保留声明。目标数据所有者 `核心/权威冻结材料.数据.h` 首先声明四仓库导出 DTO；后继 #306 / #307 只扩充 / 组合该唯一值类型，不平行声明。

目标导出必须包含：

```text
主信息：全部有效、失效、删除 / 墓碑记录和下一主信息编号。
节点：全部有效、失效、删除 / 墓碑记录、仓库编号、下一节点编号、下一创建序号。
关系：全部有效、失效、删除 / 墓碑记录、仓库编号、下一关系编号。
索引：全部当前绑定、永久保留绑定、正反向互证、所有者合同和保留状态；当前没有独立自增编号，不得虚构索引高水位。
```

实际记录状态名称若不同，执行前逐枚举映射；不得把当前有效读取冒充全部状态。
