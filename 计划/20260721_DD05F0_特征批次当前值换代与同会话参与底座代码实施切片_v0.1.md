## 2. 已验证当前事实

1. `特征业务服务` 是特征定义、实例槽、模板和当前值关系的唯一公开业务写入方。
2. `特征体系数据操作` 现有入口可以在自己的事务中创建槽位并发布一个初始值，但没有批量换代多个既有槽当前值的公开或同层内部合同。
3. `特征值原始材料事务参与者` 已能封装多项值材料；当前默认只读复用，不预设必须修改。
4. 现有核心结构事务执行器一次可携带一个聚合参与包；通用批次可以在特征数据操作内部聚合，不需要修改核心执行器。
5. 如果由任务数据操作直接创建特征结构，会越过 4030 的特征服务唯一写权；如果先建任务再独立补特征，会违反 4040 的统一确认与最后发布。
6. #330 / DD-05F 必须消费本切片，才能让任务、初始生命周期和初始特征共同发布，并让后续任务状态迁移与特征换代参加同一会话。

## 4. 实施目标与公开边界

```text
特征业务服务校验定义、值域、来源、预期当前关系 / 值 / 版本
-> 形成不可变特征批次变更规格
-> 特征数据操作形成多槽、新值、旧当前失效和新当前候选
-> 聚合特征值原始材料事务参与者
-> 形成供具名同层数据操作加入其现行写会话的不透明参与包
-> 完整确认候选
-> 普通读取仍只见旧完整状态
-> 最后发布一次
-> 普通读取只见新完整状态，旧关系和值保留历史
```

服务公开业务签名不得暴露结构会话、执行器、令牌、参与者、仓库或写锁。跨数据操作协作只允许由特征服务形成完整规格，再由特征数据操作把不透明参与包交给具名同层数据操作；高级服务不得直接取得该包。

## 5. 固定机器合同

### 5.1 批次种类

必须支持：

```text
同一候选宿主的多槽初始批次
多个既有槽的当前值换代批次
初始项和换代项共同存在的混合批次
```

每个初始项固定特征定义、槽位幂等主键、特征值幂等主键、原始类型和值、合法来源身份 / 版本。每个换代项还固定既有槽位、预期当前关系 / 值 / 版本、新值身份和旧当前关系失效依据。批次固定幂等身份、规则版本、确定顺序和完整载荷摘要；摘要不替代逐项语义比较。

### 5.2 当前值换代

换代不得原地覆盖旧值。候选阶段建立新值和新当前关系、使旧当前关系进入待失效候选；最后发布前普通读取只见旧完整状态，发布后只见新完整状态。旧值、旧当前关系和来源材料保留审计，不得出现普通读取可见的双当前、无当前或混合半批次。

### 5.3 幂等、冲突和版本

同一批次键、同一宿主、同一逐项语义、同一预期版本和同一来源是同义重复，读回既有已发布结果。任一项定义、槽位、值、来源、预期当前关系、版本、规则或顺序异义均冲突。预期当前关系或版本漂移在第一写前拒绝，不能以当前数据库 / 仓库值重解释旧请求。

### 5.4 失败收口

入口身份、类型、值域、来源、权限、当前关系或版本不合法时零写入拒绝。前置通过后，候选创建、确认、读回、撤销或最后发布异常属于内部逻辑错误。撤销按参与者逆序执行；任一撤销不完整必须隔离事务域，禁止继续发布或复用该会话，不得包装成普通失败。

## 6. 机械实施阶段

### S0 身份、接口和所有权复核

执行第 3.3 节并返回精确中文符号、调用方向和现有事务参与者能力。PASS 经设计窗口接受后进入 S1；DRIFT 停在专属断点。

### S1 批次值式规格

在特征数据操作 / 服务既有模块内形成初始项、换代项、批次规格和批次结果。分类、身份、版本和值分别强类型表达，不使用文本、裸 I64、普通 bool 或日志承载业务事实。

### S2 多槽初始与当前值换代候选

复用现有槽位、值节点、当前关系和原始材料结构，按确定顺序形成多项候选。旧关系只形成失效候选，不删除历史；完整读回缺项、重复或双当前时停止。

### S3 不透明同层参与包

把全部特征候选和特征值原始材料参与者聚合为单个不透明参与包；提供给具名同层数据操作加入其现行写会话。核心执行器、仓库和特征参与者公开边界零修改。

### S4 服务入口、幂等和撤销

由特征业务服务唯一形成完整规格和解释同义 / 异义；覆盖第一写前版本漂移、候选阶段故障、确认前故障、最后发布前故障、逆序撤销和撤销失败隔离。

### S5 自检和回归

在两份允许自检中加入 DD05F0-A01—A12；不新增工程登记、入口 import、统一阶段或运行器接线。

### S6 完整验证和任务分支收口

运行第 8 节全部命令，只暂存允许文件和专属实施记录，形成任务提交并非强制推送同名 `codex/*` 分支。状态最多为 `分支完成待集成`，不得自行集成或推送 main。
