## 6. 假定接口契约

### 6.1 兼容合同

```text
既有 自检运行器 的类型名、登记、运行、退出码和 #241 A02-A07 语义不改变。
阶段化总成内部复用既有运行器，不复制一套排序 / 汇总实现。
```

### 6.2 新增合同

```text
强类型阶段：初始化闭环、线程协议、领域回执、最终回归。
登记输入：阶段、非零阶段内顺序、全局唯一编号、非空名称和回调。
阶段执行：按阶段单调前进，每阶段一次；同阶段由既有运行器稳定排序并全量执行。
渐进登记：上下文形成后可登记尚未执行的当前 / 后续阶段；已执行阶段拒绝登记。
失败边界：普通单元失败不阻断后续阶段；依赖报告不完整由后继单元返回合同失败。
最终完成：不自动执行遗漏登记；聚合全部已执行阶段，存在已登记未执行项时合同不完整。
```

具体类型名可按当前模块内部命名风格决定，但上述语义不得改义。

## 10. 入口拒绝与失败收口

| 场景 | 分类 | 动作 |
| --- | --- | --- |
| #241 实际接口与本计划事实不一致 | 逻辑内返回 | 不改代码，退回设计窗口 |
| 阶段无效、阶段内顺序为零、字段 / 回调为空 | 逻辑内返回 | 写前拒绝，不执行回调 |
| 全局编号重复或同阶段顺序重复 | 逻辑内返回 | 拒绝本次登记，不改变已登记回调 |
| 向已执行阶段登记或逆序运行阶段 | 逻辑内返回 | 返回合同错误，不执行领域回调 |
| 普通单元失败 | 设计内结果 | 记录失败并允许后续阶段继续 |
| 同一阶段或回调执行两次、漏执行已登记单元 | 追根因解决 | 阶段合同失效，停止后继迁移 |
| 最终聚合数量、失败编号或退出码与阶段结果不一致 | 追根因解决 | 定位总成聚合错误，不解锁 #242 |
