所有准备接口只构造值式或自持有发布材料，不改权威结构。单包提交入口必须是仓库私有成员，只有 #219 `海中鱼巣::领域::概念安全删除编排器` 可调用，并同时要求同域独占令牌与私有提交会话；不得分配、不得提供普通失败返回，也不得暴露给自检、UI、SQL 或普通业务调用方。

#217 正式接口必须包含 JY-272 固定的传统头 `结构事务接线.数据.h`：共享运行期状态、域 / 纪元 / 序号 / 类型令牌、移动许可和四函数表。本切片只读消费该 ABI，在既有仓库 / 服务头中定义各所有者自己的稳定准备包类型；不得修改数据桥、前置声明模块归属类或复制第二套许可。自检模块固定使用 `module; -> include -> export module -> import 协调模块 / 运行器 -> export namespace` 骨架，先编译公开接口再实现 A01-A12。

## 6. 实施步骤

1. 复核 #216 / #217 正式 DTO、域令牌和仓库构造，逐项建立完整写集与现有结构的映射。
2. 复核 `结构事务接线.数据.h` 的令牌 / 写集身份和 #217 隔离已接域构造；若需修改数据桥或协调模块，立即退回设计。
3. 为节点和主信息补显式审计读取，普通读取继续只返回当前有效记录。
4. 为索引补按节点反向主键组，绑定、删除和审计路径同步维护双向映射；旧键真实删除，不靠端点过滤。
5. 为关系、索引、主信息和节点分别形成精确准备包；新增仅以 `海中鱼巣::领域::概念安全删除编排器` 为友元的私有提交入口，要求同域独占令牌、私有提交会话和精确包共同匹配，并预先完成分配、排序、去重、容量和版本上限复核。
6. 在存在、动态、二次特征、因果服务分别新增本类型、非根、退役目标的值式所有者清理授权；授权不直接写仓库。
7. 在概念图服务准备生命周期登记、专用关系登记、不含目标的新活动快照、活动版本和上一候选图清理材料；本切片不发布。
8. 在统计服务准备仅按目标概念清理的非权威缓存键，不参与准入与授权。
9. 汇总为 `结构写集已补全` 候选；缺任一关系、主键、所有者、登记或缓存身份都保持不完整。
10. 新建纯数据头 `海中鱼巣/领域/概念安全删除提交能力.数据.h`：会话外部不可构造、不可复制、仅可移动；构造和阶段推进仅向唯一编排器开放，不加入 #217 事务令牌业务语义。
11. 验证准备包可丢弃且结构不变化；以编译期断言和静态扫描验证提交会话不可外部构造 / 不可复制、四仓库不存在公开单包提交面。#218 自检不得成为仓库友元或直接提交单包。
12. 新建 `海中鱼巣/领域/自检.概念清理适配.ixx`，承载 A01-A12、隔离仓库、令牌伪造、准备包丢弃、并发压力、私有能力静态约束、报告和逐项输出；导出无参运行入口并返回正式 `自检单元结果`。
13. `入口.cpp` 只增加模块 `import` 和一次 `CONCEPT-S8-S2 / 最终回归阶段 520` 无捕获回调登记，不展开场景、循环或逐项输出。
