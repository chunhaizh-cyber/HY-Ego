# 2210 根规范 语素入口

编号：2210

日期：20260720

状态：正式根规范

## 1. 根定位

语素入口是已确认词身份与项目内部信息对象之间的正式路由桥。它不是词本身、分词结果、纯词性标签、业务对象本体或世界事实。

语素入口同时服务两类不同材料：

```text
人类词性：自然语言分类和解析候选
信息入口类型：项目内部对象角色、存储结构和上下文路由
```

二者必须分别承载、分别读回，不得继续以“词性”一词混称。

## 2. 正式结构承载

正式语素入口至少由以下结构共同成立：

```text
语素入口节点及稳定句柄
+ 明确的语素入口角色材料
+ 唯一所属词关系
+ 规范化人类词性材料
+ 信息入口类型及路由版本
+ 一个或多个语素对应信息关系
+ 零个或多个语素概念追溯关系
+ 关系目标角色、版本和当前性
+ 主键、反向查找索引和完整发布事实
```

允许复用 `节点类型::语素`，但入口角色必须由正式材料证明。入口不得根据父节点显示名、节点深度、对应目标类型或只读投影分组反推角色。

## 3. 词、人类词性与入口多样性

1. 每个正式语素入口必须且只能属于一个当前可读词身份。
2. 一个词可以拥有多个人类词性，也可以在同一人类词性下拥有多个信息入口类型或多个上下文含义。
3. 人类词性只提供语言标注和候选，不直接裁决对应信息类型、存储树、现实存在、任务满足或方法可执行。
4. 外部分词编码只能作为解析别名；正式入口必须归一到受控人类词性身份及规范名称，不得把外部编码作为并列正式词性。
5. 同一词性下存在多个入口时，机器身份仍按完整入口句柄、信息入口类型和关系材料区分；任何只读分组都不得合并机器事实。

## 4. 理解准入与对应范围

正式入口必须至少绑定一个当前可读且角色允许的项目内部信息对象。只有分词、人类词性或人工标注而没有内部对象身份时，只能保留为语言标注或待理解候选，不得进入正式语素入口查找。

对应信息可以是规范允许的基础信息类对象以及需求、任务、方法等既有高级业务对象，但必须使用稳定句柄和具名关系：

```text
允许：基础信息、存在、场景、特征、抽象状态、抽象动态、二次特征、因果引用、需求、任务、方法
禁止：特征值、实例状态、实例动态、显示节点、日志节点、自由文本节点和不存在的“高级信息”类型
```

概念追溯目标必须是允许的长期概念或概念模板身份。实例入口应绑定实例对象，并通过概念模板、概念集合或正式领域关系追溯到概念；任务、运行实例和临时状态不得伪装成概念。

对应信息和概念追溯均允许多对多关系，不得压缩为单个裸指针或单值主信息字段。

## 5. 信息入口类型与路由

信息入口类型只负责路由已理解入口，不等于人类词性。它至少应明确：

```text
目标对象角色
目标存储树或领域入口
是否要求主体 / 场景 / 时间 / 任务 / 方法上下文
路由规则版本
允许的对应信息与概念追溯角色
```

人类词性可以形成信息入口候选排序，但不得自动发布信息入口类型。没有足够理解证据时，信息入口类型保持未定义，材料继续停留在语言候选层。

## 6. 写入、读取与事务边界

语素领域服务负责业务准入和成功 / 失败语义；语素专用数据操作层负责不透明结构写入会话、词与入口关系、对应关系、概念追溯、索引、精确撤销和最后发布。公开业务入口不得暴露原始结构事务令牌、仓库引用或候选对象。

同一规范身份的重复请求必须幂等读回；同一稳定键对应不同词、词性、入口类型或目标结构时必须拒绝。写入完成后必须在同一正式读回中互证节点角色、所属词、人类词性、入口类型、全部关系端点、主键和版本。

## 7. 只读聚合原则

语素服务必须能够从词身份完整读回该词的人类词性集合及各词性下的语素入口集合。聚合只形成不含写入能力的值式投影，不合并入口身份、对应信息或概念追溯事实；具体投影合同由 7200、7210 规定。

## 8. 禁止事项

- 禁止用词面、词性或显示文本直接裁决业务语义。
- 禁止把语素入口当作其对应对象本体。
- 禁止把未理解文本、实例状态、实例动态或自由文本写成长期入口。
- 禁止把人类词性与信息入口类型合并成一个枚举或一个显示字段。
- 禁止外部领域和显示层绕过语素服务直接写关系仓库。

## 9. 子规范路由

7200 规定语素与信息多关系整合；7210 规定信息入口类型、存储树和上下文路由。入口结构和路由只能来自这两份子规范及其上级合同，不得由实现自行补造。
