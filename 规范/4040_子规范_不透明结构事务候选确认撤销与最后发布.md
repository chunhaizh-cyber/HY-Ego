# 4040 子规范 不透明结构事务候选确认撤销与最后发布

编号：4040
日期：20260724
状态：正式子规范

## 1. 定位

本规范规定跨仓库或跨内部材料的一次结构写入，如何通过不透明事务形成未发布候选、完成可逆确认、在失败时精确撤销，并在最后一步统一发布。它只定义机器事务语义，不规定具体模块、类、函数、字段、锁实现或故障注入方式。

直接上级规范：0050。

直接依赖规范：4010、4030。

相关边界规范：0500、4050、7140。

## 2. 事务身份与职责

一次结构事务必须具有不可与其它事务混淆的稳定运行期身份，并明确：

```text
非零运行期域身份、非零已发布代次与受支持冻结规则版本
本次写入身份
唯一活动写入权
参与仓库与内部材料参与者
精确写集和撤销材料
领域最后发布事实
完成、撤销或隔离结论
```

事务身份、写入权和候选能力不得由裸线程编号、锁地址、日志标签或显示文本承担。线程只是执行载体；事务结构中的唯一有效写入权负责裁决谁可以推进本次写入。

职责分离固定为：

- 协调层只签发、验证和释放同一事务域的读写权，并在撤销无法闭合时隔离事务域；
- 仓库只执行本仓库通用结构读写和结构不变量检查；
- 服务专用数据操作层隐藏仓库、写入权和撤销细节，组织同一事务中的结构写集；
- 领域服务决定业务准入、业务不变量、幂等语义和最后发布事实；
- 内部材料参与者只提交本次值式材料，不解释领域业务，不取得仓库、锁或原始事务能力。

领域公开入口、外部调用方、线程和消息不得取得或传递原始事务能力、仓库引用或内部可变写会话。

## 3. 候选与参与者阶段

候选和参与者至少具有以下互斥阶段：

```text
写入中
-> 已确认待发布
-> 已发布

写入中或已确认待发布
-> 已撤销
```

候选必须绑定创建事务、创建方、完整身份、当前版本和精确反向材料；不得默认构造为有效候选，不得复制为第二份写能力，移动后原能力必须失效。

确认只证明本次候选或参与者已经完成完整读回和可逆复核。确认不得单独生成业务事实，不得清除精确身份、版本、写前材料或撤销能力，也不得使普通读取者提前看到候选。

已发布表示本次事务已经越过统一可见点。发布后候选能力永久关闭，不得再用候选撤销替代已发布结构的正式删除、失效、补偿或新版本流程。

## 4. 候选隔离与完整读回

事务从第一笔写入到发布或完整撤销结束，必须持有覆盖全部参与结构的同一个唯一写入权。普通读取路径在统一可见点之前不得观察候选、即时关系、索引绑定或内部材料的中间组合；事务内读回只能使用本次受控能力。

未发布候选不得进入：

```text
公开索引或普通查询结果
领域服务返回值
线程消息或外设报告
日志、显示投影或序列化输出
其它事务的候选或事实来源
```

若任一参与仓库、侧表或读取入口可以绕过同一事务域观察或修改中间结构，必须在第一笔写入前拒绝；不得依赖“通常不会并发”或事后扫描作为隔离证明。

### 4.1 统一冻结读取见证

完整快照或跨领域全量读取只能由统一协调方从事务域取得一次不透明冻结读取许可。许可必须在取得覆盖全部参与结构的共享读取权后，由事务域自身签发下列值式见证：

```text
非零运行期域身份
非零已发布代次
受支持冻结规则版本
```

运行期域身份在一个事务域生命周期内不变。已发布代次只在一次真实写事务的会话和全部参与者完成最后发布后单调推进；入口拒绝、逻辑内零写、幂等读回、候选撤销和内部错误均不得推进。冻结规则版本由事务域实现固定，不得由调用方传入任意数值后原样回显。

统一协调方在许可存续期间把同一个不可写许可交给具名内部只读提供者。提供者只能验证许可属于其绑定事务域、读取见证并复制自有值式材料；不得再次取得事务域冻结权，不得返回锁、仓库、事务令牌或可写引用。许可取得失败属于形成快照前的具名逻辑内结果；提供者收到无效、错域或规则版本不受支持的许可时，必须在复制第一项材料前入口拒绝。许可归属与版本复核通过后，见证变化、部分材料或必需提供者绕过许可属于内部逻辑错误。

请求确认前必须在同一事务内读回并证明：

```text
全部候选身份和版本有效
全部必需关系、索引和内部材料齐全
正向与反向结构一致
唯一性、基数和领域前置仍成立
写集与本次请求语义一致
全部参与者具备精确撤销材料
```

## 5. 失败撤销

统一可见点之前任一步骤不能继续时，必须停止新增写入，并按依赖逆序撤销本次精确写集。撤销只依据本次事务保存的完整身份、版本、写前值和写后值，不得扫描仓库猜测残留，不得用普通删除冒充候选撤销，不得由候选析构隐藏写入。

撤销顺序必须先处理依赖者，再处理其被依赖结构；领域类型化记录与其它内部材料参与者、关系变更、索引与新关系、节点候选都必须纳入同一闭合证明。某一项撤销失败不得阻止其余可撤销项继续尝试。

撤销完成必须以可读结构证明事务前态已经恢复。身份编号、创建序号或其它不可复用高水位可以前进，但不得因此留下可读半结构、悬空索引、错误当前版本或已确认却未发布的参与者。

任一参与结构无法证明恢复前态时，属于内部逻辑错误。协调层必须在释放本次唯一写入权之前隔离受影响事务域；隔离后不得向新请求或尚未取得写入权的等待请求开放该域。不得在撤销失败后普通返回并让读取者观察半结构。

## 6. 确认与最后发布

只有本次写集完整读回、会话和全部参与者均处于“已确认待发布”，并且领域最后发布事实已经明确时，事务才能进入最后发布区。

最后发布区必须无失败：

```text
不得新增结构或分配新的业务资源
不得调用外部回调或跨域服务
不得执行仍可能拒绝、抛错或改变裁决的复核
不得记录决定机器结果的日志或显示材料
只允许关闭候选撤销能力、完成参与者发布并释放唯一写入权
```

释放覆盖全部参与结构的唯一写入权，是普通读取者观察本次完整结构的唯一统一可见点。不得把某个仓库先提交、某条关系先可见、日志写出或返回值形成解释为提前发布。

已发布代次必须在最后发布区内、全部参与者完成发布之后、释放唯一写入权之前推进一次。推进失败或发生零值 / 回绕属于内部逻辑错误，事务域不得继续向新冻结或写请求开放。幂等读回没有产生新的发布事实，不推进已发布代次。

最后发布后的读回与已确认结构不一致，属于内部逻辑错误。此时不得重新打开候选撤销能力，不得静默返回失败，也不得把错误降级为缓存失效、普通等待或业务缺口。

## 7. 内部材料参与者

当一次业务操作同时改变通用仓库结构和服务内部权威材料时，内部材料必须成为同一事务的正式参与者。参与者只能接收值式材料和本次提交准备所需的只读视图，并遵守：

1. 只复核与本次候选绑定的材料，不任意读取或修改其它已发布身份；
2. 准备和确认后继续保持精确反向材料，直到发布或撤销完成；
3. 完成发布只关闭阶段、清除撤销能力并释放内部占用，不再执行可失败工作；
4. 参与者失败时先撤销参与者写集，再撤销其依赖的通用结构；
5. 不向领域服务或外部调用方暴露事务参与能力。

不属于本次事务的统计缓存、显示快照、日志和诊断材料不得伪装成事务参与者。

## 8. 幂等、冲突与已发布变更

幂等必须同时比较显式幂等身份和完整请求语义。相同身份且语义完全相同可以返回已发布结果；相同身份但载荷、目标、版本或参与写集不同，必须在写前返回具名冲突或追查身份复用根因，不得返回旧成功。

未发布候选撤销、已发布事实删除、已发布事实失效和已发布事实换代是不同合同。已发布结构的改变必须重新取得正式领域授权并形成新的事务或专用补偿流程，不得复用旧候选能力或把版本回退伪装成新发布。

## 9. 验证合同

验证至少覆盖：

| 验证项 | 必须证明 |
| --- | --- |
| 候选不可见 | 统一可见点前普通读取、索引、消息和投影均不能取得候选 |
| 确认可逆 | 确认后、发布前仍保有完整精确撤销能力 |
| 全参与者一致 | 仓库结构和内部材料在同一事务中共同确认、共同发布或共同撤销 |
| 逆序撤销 | 每个确认点之前失败均能按精确写集恢复可读前态 |
| 撤销失败隔离 | 无法恢复时事务域在写入权释放前被隔离，后继请求不能进入 |
| 最后发布 | 全部参与者确认后才进入无失败区，普通读取只在唯一可见点后见到完整结果 |
| 幂等冲突 | 同键同义零写收敛；同键异义不返回旧成功 |
| 发布后异常 | 发布后不一致进入内部逻辑错误，不重新启用候选撤销 |
| 冻结见证 | 见证由事务域在一次共享许可内签发；域身份稳定、真实发布才推进代次、提供者不得二次取得冻结权 |
