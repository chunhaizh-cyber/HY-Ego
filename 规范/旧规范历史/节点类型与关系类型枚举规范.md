# 节点类型与关系类型枚举规范

编号：015
日期：2026-07-06
状态：已确认规范 / 节点 0—15 与关系 0—18 已进入代码 / #293、JY-388 已完成节点 15 与关系 18 的追加、专用准入和回归验证

## 1. 定位

本规范定义第一版允许的节点类型和关系类型边界。枚举只表达结构分类，不表达业务完成、状态结论或显示文本。

枚举值一经进入代码、快照、事件日志、索引、持久化材料或跨版本恢复材料，不得重排、复用或改名复义。新增节点类型或关系类型只能追加新值，不得插入到既有枚举中间。删除旧类型只能废弃不用，不得把旧数值改作新语义。

## 2. 节点类型

第一版允许以下节点类型。数值 0—14 已进入当前代码；数值 14 由 #223 / JY-305 正式实现。节点 15 已由 #293 按追加规则实现为 `用途观察记录`；节点仓库继续在第一笔写入前拒绝 16 及更大值。

JY-304 根据 #223 在 clean HEAD `4a8bdd4` 的实际接口复核补充准入边界；#223 / JY-305 当时只把 `节点仓库.cpp` 私有上限同步到 `任务方法选择记录 = 14`。节点仓库公开 ABI、锁、编号、版本、创建序号和其它创建 / 删除语义未改。JY-371 当时确认用途观察的身份、容量、关系阶段、专用数据操作和恢复合同不足；JY-376 因此在 #287 时点继续保持节点 15 与关系 18 未分配，只登记不写仓库的纯值因果完整结构键切片。后续追加事实由本规范第 7 节承载。

| 2 | 基础信息 | 基础信息通用节点 | 只作为尚未拆入具体基础信息类型时的通用承载 |

| 5 | 特征 | 特征类型或特征身份 | 由特征服务写入 |

### 2.2 基础信息类长期节点

`基础信息` 在语义上是基础信息类信息节点集合，不只等于 `基础信息` 单一节点类型。

第一轮基础信息类长期节点包括：

```text
基础信息通用节点
存在
场景
特征
抽象状态
抽象动态
二次特征
因果引用
```

其中 `状态` 和 `动态` 是底层节点枚举名。是否为抽象状态 / 抽象动态，不由枚举值单独裁决，必须由状态服务、动态服务、运行期临时关系和对应领域入口共同判定。

场景内实例状态 / 实例动态是临时零散节点，必须有存在主体、场景归属和发生时间戳，不得作为长期语素入口对应信息节点。抽象状态 / 抽象动态才可作为长期基础信息类节点绑定语素入口或跨场景复用。

`特征值` 是值承载节点，第一轮不得默认作为语素入口对应信息节点。若后续需要让语素直接指向某个值，必须先确认特征值系统规范和入口边界。

### 2.3 禁止节点类型

禁止新增或使用以下节点类型：

```text
高级信息
名称字段节点
显示标题节点
SQL 投影节点
控制面板节点
临时日志节点
任务完成节点
需求满足节点
方法成功节点
```

`高级信息` 不是节点类型，只是讨论中对需求、任务、方法等既有高级业务节点角色的简称。

### 2.4 枚举不裁决业务事实

以下判断不得只由节点类型得出：

```text
节点类型::任务 不代表任务已创建完整承接壳。
节点类型::方法 不代表方法可执行。
节点类型::需求 不代表需求已满足。
节点类型::状态 不代表状态已被确认长期有效。
节点类型::动态 不代表动态已进入长期抽象层。
节点类型::因果引用 不代表因果结论稳定可靠。
候选名称“用途观察记录”不得被解释为节点 15 已锁定、因果稳定、方法好用、方法已经学习或可以自动执行。
```

业务事实必须由领域服务、关系、状态、动态、因果引用、结算记录或对应结构共同裁决。

## 3. 关系类型

本表锁定允许的关系类型。数值 0-17 已进入当前代码；数值 17 已由 #279 按 JY-347、JY-349 实现并验证。

### 3.1 关系类型数值锁定表

| 数值 | 关系类型 | 第一轮用途 | 多目标 / 唯一性口径 |
| --- | --- | --- | --- |
| 0 | 普通父子 | 表达树形或层级父子 | 同一子节点同时只能有一个有效父节点 |
| 1 | 归属 | 表达对象归属于场景、存在、需求或容器 | 是否多目标由对应领域规范确认 |
| 2 | 引用 | 表达非拥有引用 | 默认由领域规范确认是否多目标；未确认前不得模拟列表字段 |
| 3 | 索引 | 表达索引项和目标之间的结构连接 | 第一轮优先由索引仓库管理，不裁决业务事实 |
| 4 | 模板 | 表达方法、任务、特征、需求目标等模板关系 | 多目标和唯一性由对应服务专用读取入口裁决 |
| 5 | 运行期临时 | 表达实例状态 / 实例动态等临时归属 | 不进入长期事实裁决，不作为长期语素绑定依据 |
| 6 | 因果来源 | 表达动态、前后值、目标、证据等来源材料 | 因果服务或动态相关服务写入和读取 |
| 7 | 语素对应信息 | 语素入口绑定对应信息节点 | 已确认同一语素入口可多目标 |
| 8 | 语素概念追溯 | 语素入口追溯概念节点 | 已确认可多级、多路目标 |
| 9 | 实例支持概念 | 实例节点支持一个或多个同类概念 | 同一实例可支持多概念，同一概念可被多实例支持；完全重复关系幂等 |
| 10 | 概念上下位 | 同类概念间的直接上位到下位边 | 允许下位多父；禁止跨类、自环和已知成环 |
| 11 | 概念定义材料 | 概念节点引用结构化定义材料 | 允许多目标；目标必须进入定义材料白名单 |
| 12 | 概念生命周期 | 概念节点指向已登记的抽象生命周期状态 | 同一概念必须且只能有一个有效目标；目标仅限活跃、冷却或退役状态 |
| 13 | 需求目标概念 | 需求节点引用待创建或复用名称的目标概念 | 允许同一需求在后续专用服务约束下引用目标概念；不得参与概念用途或生命周期活性计数 |
| 14 | 任务生命周期 | 任务指向当前或历史实例状态 | 同一任务恰有一条有效当前关系；顺序号为生命周期版本，失效旧关系保留审计 |
| 15 | 任务生命周期证据 | 生命周期实例状态指向迁移证据角色 | 顺序号仅允许 40-46；同源同角色唯一，同一目标可跨不同角色复用 |
| 16 | 任务方法选择 | 任务指向不可变任务方法选择记录 | 顺序号固定 30；同一任务至多一条有效关系，失效历史保留审计 |
| 17 | 方法动作场景 | 动作入口方法节点指向输入 / 输出场景角色 | 顺序号仅允许 1 / 2；同源同角色唯一，同一场景可跨输入 / 输出角色复用；#279 已实现验证 |

### 3.2 当前关系类型边界

```text
普通父子：只表达普通层级和通用移动重挂，不表达所有业务关系。
归属：用于拥有、承接或容器关系时，必须由领域规范说明源 / 目标类型和唯一性。
引用：用于非拥有引用时，外部调用方不得直接把裸引用裁决为业务事实。
索引：索引只服务定位、候选或加速；索引缺失不得改变权威结构事实。
模板：用于方法、任务、特征、需求目标等模板关系；读取必须由领域服务复核目标类型和唯一性。
运行期临时：用于实例状态 / 实例动态等临时归属；不得被长期语素入口当作抽象信息。
因果来源：用于动态证据、前值、后值、被改变目标和结算证据；轻量因果引用本身不写来源动态关系。
语素对应信息：已确认多目标，读取必须返回集合。
语素概念追溯：已确认多级、多路目标，读取必须返回集合或追溯图，不得静默选第一个。
实例支持概念：只由概念图服务写入；实例和概念必须类别兼容，概念根不得作为实例端。
概念上下位：只由概念图服务写入；源为直接上位、目标为直接下位，允许多父但必须保持同类无环。
概念定义材料：只由概念图服务写入；不得连接名称文本、日志、控制面板材料、SQL 行或线程消息。
概念生命周期：只由概念图服务写入；目标必须是经状态服务创建并登记的活跃、冷却或退役抽象状态，同一概念不得并存多个生命周期目标。
需求目标概念：数值固定为 13；只作为需求到目标概念的权威结构引用。它不增加概念用途或生命周期活性引用，不构成重新激活依据，但同一有效关系仍属于安全删除候选必须观察的当前关系，不得被全局忽略。
任务生命周期：数值固定为 14；源必须是任务、目标必须是实例状态。同一任务恰有一条有效当前关系，顺序号从 1 连续递增并只表达生命周期版本，旧关系失效后只从审计入口读取。
任务生命周期证据：数值固定为 15；源必须是对应迁移实例状态，目标与顺序号角色由任务服务验证。关系仓库只对本类型按同源 + 同顺序号判定角色唯一；同一目标跨不同角色允许，普通引用唯一性不因此改变。
任务方法选择：数值固定为 16；源必须是任务，目标必须是任务方法选择记录，顺序号固定 30。关系仓库按完整任务源句柄 + 类型 + 顺序号拒绝第二条有效当前记录；旧关系失效后仅由审计读取。顺序号 30 普通引用只作方法投影，不能替代本专用关系与记录节点。
方法动作场景：数值固定为 17；源必须是动作入口方法节点，目标必须是场景，顺序号 1 / 2 分别表示输入 / 输出场景。关系仓库按完整源句柄 + 类型 + 顺序号拒绝第二条有效角色关系；同一目标可跨两个角色复用。新动作入口只写本类型，旧普通引用槽 1 / 2 只读兼容；#279 已实现并通过 A01-A16 验证。
```

### 3.3 记录状态数值与关系读取边界

`记录状态` 数值已进入句柄、仓库记录和后续快照 / 恢复兼容边界，只能追加，不得重排或复用：

| 数值 | 记录状态 | 当前关系语义 |
| --- | --- | --- |
| 0 | 无效 | 未形成可读记录或入口拒绝默认值 |
| 1 | 有效 | 普通业务读取、关系存在判断和有效数量可见 |
| 2 | 已删除 | 已进入删除语义；普通读取不可见，恢复口径等待恢复专项 |
| 3 | 已失效 | 原关系历史仍保留，但不再参与当前业务关系或活动投影 |

当前已实现的 `已失效` 业务写入口包含概念图支持失效、任务生命周期换代和任务方法选择换代。失效保持关系编号、类型、源端点、目标端点和顺序号不变，关系版本加一；普通读取继续只返回 `有效`，显式审计读取才允许返回当前版本的 `有效` 或 `已失效` 记录。关系 17 已复用现有失效、删除和审计合同，但禁止通用原地重挂；#279 已完成对应回归验证。JY-371 只证明这些底层语义局部可复用，未形成用途观察角色 / 基数 / 阶段合同；JY-376 继续禁止预设关系 18 或用途观察阶段关系。

同一实例 / 普通概念端点后续重新形成真实支持时，应创建新的有效关系，不得把旧失效记录原地恢复。四根支持不进入本切片失效或同端点再建语义，其失效时机等待实例退役 / 删除事务专项整体裁决。

关系仓库详细设计必须与本规范同步。若 `规范/详细设计/关系仓库详细设计.md` 对某关系类型的唯一性、多目标或读取规则有更具体约束，代码实施必须同时满足本规范和关系仓库详细设计；二者冲突时不得编码，必须先修正规范。

## 4. 新增类型准入流程

### 4.1 新增节点类型前必须说明

```text
用途
写入方
读取方
生命周期
是否可由已有节点类型 + 关系 / 状态 / 特征表达
是否允许作为语素入口对应信息节点
是否属于基础信息类、高级业务节点角色、运行期实例类或证据类
入口拒绝规则
验证方式
禁止宣称
```

不得为显示、日志、控制面板、SQL 投影、临时兼容字段或自然语言标题新增节点类型。

### 4.2 新增关系类型前必须说明

```text
用途
写入方
读取方
生命周期
是否允许多目标
是否允许重复
源节点类型和目标节点类型
是否可由已有关系表达
验证方式
禁止宣称
```

新增关系类型不得绕过关系仓库详细设计、领域服务授权写入规范和入口前置拒绝规则。

## 5. 禁止行为

```text
不得用普通父子关系表达所有关系。
不得为显示方便新增节点类型或关系类型。
不得把需求满足、任务完成、方法成功做成节点类型。
不得把高级信息做成独立枚举。
不得重排、复用或改名复义已经进入代码和持久化材料的枚举数值。
不得用枚举类型替代领域服务的业务裁决。
不得用运行期临时关系把实例状态 / 实例动态提升为长期基础信息。
```
