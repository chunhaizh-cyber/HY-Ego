# 非权威缓存与统计规范

编号：非权威缓存-001
日期：2026-07-06
状态：已确认项目规范 / 概念用途缓存第一轮已实现 / #283-#284 S0 已完成 / JY-376 固定完整因果身份先行、统计与概念标注继续后置

## 1. 定位

非权威缓存和统计只服务性能、索引、去重、诊断和人读观察，不承载机器事实。

本规范覆盖主信息迁移 S4 中登记的：

```text
MI-V2 内容哈希 / 命中次数 / 最后命中
MI-BI2 结构_统计
MI-D3 安全 / 服务权重 / 累计结算 / 最近结算任务和时间中的非权威摘要部分
```

本规范不允许把这些缓存项回流为主信息权威字段。

## 1.1 第一轮确认口径

第一轮非权威缓存与统计确认以下口径：

```text
缓存不进入主信息权威字段。
第一轮可实施范围收窄为缓存命名空间、缓存记录结构、结构统计运行期快照、缓存清空 / 重建接口壳。
特征值内容哈希、VecI64 / VecIU64 原始值缓存和需求结算摘要缓存必须等待对应权威结构确认并实现后再进入代码。
缓存缺失、缓存清空或缓存命中变化不得改变需求满足、任务完成、方法成功或世界事实。
```

上述“第一轮可实施范围”保留为 2026-07-06 起步边界。后续正式计划已经实现结构统计、特征值内容哈希 / 命中观察和概念用途观察等独立切片；每类缓存仍必须逐项证明可清空、不持久化且不裁决权威结构。

## 3. 非权威缓存类型

| 缓存类型 | 用途 | 权威来源 | 清空后影响 |
| --- | --- | --- | --- |
| 特征值内容哈希 | 去重、索引和快速比较 | 特征值原始 Vec 数据 | 清空后可由原始值重建，不改变特征值事实 |
| 特征值命中次数 | 观察值复用次数 | 索引命中事件或读取事件 | 清空后不改变任何业务事实 |
| 特征值最后命中 | 观察最近复用时间 | 索引命中事件时间 | 不是状态发生时间，不改变业务事实 |
| 结构统计 | 诊断、调度、观察 | 节点、关系、索引、状态、动态等权威结构 | 清空后可重新扫描或重建，不改变结构事实 |
| 需求结算非权威摘要 | 人读显示和兼容摘要 | 需求结算记录、状态、动态和因果证据 | 清空后可由结算记录重建，不改变结算事实 |
| 概念用途观察 | 分开记录一般、因果预测和因果行动用途的次数、最近时间与效用候选 | 概念图服务复核后的活动概念身份、活动版本和结构化调用材料 | 清空后不改变概念、活动图、支持关系或生命周期；不得直接触发转换 |

## 5. 缓存命名空间

缓存命名空间用于定位、清空、失效和重建缓存项，不表达业务事实。

缓存命名空间至少包含：

```text
缓存类型
来源结构类型
来源句柄或来源仓库
来源版本
生成规则版本
缓存值类型
是否可持久化
失效条件
```

第一轮命名空间示例：

```text
特征值内容哈希:v1:特征值句柄
特征值命中次数:v1:特征值句柄
结构统计:v1:节点仓库
结构统计:v1:关系仓库
结构统计:v1:索引仓库
需求结算摘要:v1:需求句柄
概念用途观察:活动图版本2:规则版本5:概念句柄:一般
概念用途观察:活动图版本2:规则版本5:因果概念句柄:因果预测
```

命名空间只用于缓存治理。不得把命名空间字符串、显示标题、函数名或日志标签作为权威身份。

## 6. 权威结构和缓存关系

缓存不得替代以下权威结构：

```text
节点身份
关系
索引项
特征值原始值
状态节点
动态节点
因果引用
需求目标状态
任务目标 / 实际结果状态
方法条件 / 结果结构
需求结算记录
```

缓存读取方必须能接受缓存缺失。缓存缺失只能导致性能下降、需要重建或诊断信息减少，不得导致业务事实丢失。

## 7. 写入和重建规则

缓存写入必须满足：

```text
写入方明确。
读取方明确。
失效条件明确。
重建来源明确。
缓存关闭或清空时，不改变机器事实。
```

缓存不得由显示层、日志层或控制台输出反向写入。

缓存重建只能读取权威结构，不得凭旧缓存互相推导出新事实。

命中次数只记录候选召回、索引读取或特征复用观察；命中次数不代表方法好用、特征重要、需求满足或任务完成。

最后命中是缓存观察时间，不是状态发生时间、动态发生时间或需求结算时间。

孤立关系诊断只做人读诊断，不得由统计缓存自动删除、修复或补写关系。结构修复必须另走正式仓库或领域服务入口。

需求结算摘要缓存依赖需求结算记录规范和详细设计确认。权威结算记录未实现前，不实现需求结算摘要刷新；不得从需求状态节点、控制台输出或日志倒推摘要缓存。

### 7.1 概念用途观察专用规则

```text
概念用途缓存键必须包含完整缓存命名空间、完整概念句柄和用途类型。
来源版本必须等于活动图版本；规则版本变化形成独立命名空间，旧缓存不得冒充当前规则材料。
一般用途只累计次数和最近观察时间，结果效用必须保持 0。
因果预测和因果行动只允许概念图服务在重读活动因果概念类别后提交，两个用途独立累计有符号效用候选。
同键时间戳允许相等但不得倒退；时间戳必须落在正 I64 范围，次数和有符号效用溢出必须写前拒绝。
统计服务只维护运行期缓存；概念图服务组合当前生命周期、活动实例支持和外部正式引用后才可输出复核候选。
保持活跃、建议冷却、建议退役复核、建议重新激活和材料不足均为非权威候选，不是生命周期转换许可。
清空指定概念用途命名空间或全部二级缓存后，节点、主信息、关系、索引、活动图版本和生命周期关系不得变化。
```
