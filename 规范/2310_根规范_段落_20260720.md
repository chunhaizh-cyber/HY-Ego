# 2310 根规范 段落

编号：2310

日期：20260720

状态：正式根规范

## 1. 根定位

段落是文章版本内的有序语言材料容器，承载段落原始文本、文章归属、段落顺序和有序自然句组成。段落不是业务状态容器。

## 2. 正式结构承载

正式段落至少由以下结构共同成立：

```text
段落稳定身份与明确语言对象角色
+ 原始文本和语言归属
+ 唯一文章版本归属关系
+ 在文章中的顺序号
+ 段落到自然句的有序组成关系
+ 解析版本、当前发布和来源证据
```

不得因缺少段落专用节点类型而用普通父子指针链、语素节点或显示树项冒充段落身份。

## 3. 归属、顺序与版本

1. 一个段落版本必须且只能归属一个文章版本。
2. 同一文章版本中的段落顺序号必须唯一；顺序号变化属于组成版本变化，不得只移动容器位置而不更新正式关系。
3. 一个自然句版本必须且只能归属一个段落版本，并保留句序。
4. 相同段落文本在同一或不同文章位置出现时仍保留各自组成身份，不按文本去重。
5. 段落修订必须形成新材料版本或具名当前性变化，保留原始来源；不得静默覆盖历史段落。

## 4. 写入方、读取方与验证

语言领域服务负责段落身份、文章归属、原始文本、句子组成、顺序和发布版本。正式读回应互证段落角色、唯一文章归属、段落序号、自然句顺序和材料版本。

解析出的概念、需求候选、方法候选和事实只能作为带来源版本的解析材料提交给对应领域；对应领域的正式结果不得回写为段落业务字段。

## 5. 禁止事项

- 禁止把段落文本、位置、摘要或解析标签解释为业务状态、对象身份或治理结论。
- 禁止依赖内存地址、容器顺序或显示行号承担长期段落顺序。
- 禁止把解析出的需求、任务、方法、状态、动态或因果复制进段落主信息。
