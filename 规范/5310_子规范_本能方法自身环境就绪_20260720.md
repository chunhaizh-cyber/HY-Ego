# 5310 子规范 本能方法自身环境就绪

编号：5310
日期：20260720
状态：正式子规范

## 1. 定位

本规范定义本能方法执行前必须具备的自身方法环境，以及学习方法自身环境与目标方法环境的严格分离。

上级规范：3300、5300、5320。

## 2. 自身环境合同

本能方法进入核心逻辑前必须权威读取并确认：

```text
当前方法首及当前生命周期
唯一治理虚拟存在
当前动作入口
输入与输出场景合同
至少一组适用条件规格
明确结果规格
本轮运行身份
内容版本和当前性
```

这些结构应在方法登记和筹办冻结前形成。执行入口只作复核，不得在执行期自动补造方法首、条件、结果、动作入口或治理虚拟存在。

## 3. 条件权威

1. 方法首和能力签名只能用于召回与预筛选。
2. 执行参数设置、参数匹配、作用对象绑定和值域判断必须以当前条件规格为直接权威。
3. 任务输入场景只是本轮实参来源，不能替代方法条件场景或条件节点。
4. 条件匹配、禁止项和版本复核全部通过后，才允许进入核心逻辑。
5. 规格差异应收口到方法领域的强类型规格解析入口，不分散到每个本能函数中以名称或文本判断。

## 4. 学习方法与目标方法分离

学习或练习方法 A 处理目标方法 B 时，必须分别读取两套方法身份、治理虚拟存在、条件结果规格、运行事实和场景。

A 不得把 B 的治理虚拟存在当作自身环境，不得把 A 的条件/结果场景写入 B，也不得在“确保自身环境”阶段改写 B 的结构或长期账。

B 的运行次数、成功次数、连续成功、状态、稳定性和最近运行事实只能由 B 自身执行或方法领域提交入口更新。

## 5. 运行实例

每次执行必须先形成独立运行身份及其输入、输出和时间关系，再进行绑定和核心动作。方法规格场景、任务输入场景、本轮可执行参数场景、临时结果场景和输出回执场景必须可区分。

## 6. 非成功分类

| 情况 | 分类 |
| --- | --- |
| 入口材料本就不满足公开前置条件 | 入口拒绝，不产生半结构 |
| 当前事实变化导致条件不再满足或参数不可绑定 | 合法运行分流，形成运行结果 |
| 冻结前已存在的自身规格、动作入口或治理结构缺失 | 筹办漏检和内部逻辑错误 |
| 写入后内部结构不符合预期 | 内部逻辑错误并追根因 |

内部错误必须记录逻辑错误日志并按诊断宏弹窗，但日志和弹窗不承担机器状态，也不得以“条件不足”掩盖。

## 7. 禁止事项

- 禁止只因方法首存在就跳过规格确认。
- 禁止仅凭能力签名或稳定动作键直接执行。
- 禁止绕过条件规格从输入包猜参数。
- 禁止把目标方法环境与学习方法自身环境混用。
- 禁止把任务筹办、任务执行、线程主循环或回执同步登记成本能方法。
- 禁止用方法名、动作名或显示文本判断语义。
