# 5250 子规范 任务筹办按方法能力查找到就绪因果链

编号：5250
日期：20260720
状态：正式子规范

## 1. 定位

本规范定义普通可任务化原子任务如何按方法结果能力召回候选、读取就绪因果链、定位缺口并形成闭合筹办结果。

直接上级规范：3200、3300。

直接依赖规范：1190、3100、5230。

相关边界规范：4310、5240、5300。

## 2. 适用闸门

本规范只适用于目标完整、没有应先推进的活动子需求、目标特征定义允许普通任务化的原子任务。

管理、汇总、AND、OR、方法路径、因果支撑和结构治理节点即使当前没有子节点，也不得进入普通方法召回；必须先按对应展开与聚合规则处理。

## 3. 召回合同

1. 从任务来源需求读取目标宿主、目标特征、目标状态和满足关系。
2. 方法结果索引只提供候选起点，必须回读方法当前登记项、生命周期和结果规格复判。
3. 候选方法必须至少有一个当前结果能力可命中任务目标。
4. 方法名称、动作名称、函数名、分类标签和固定编号不得参与语义命中。
5. 通用特征写入、状态创建、动态创建、任务筹办和其它底层原语不得作为语义候选方法。

## 4. 就绪链

候选方法就绪必须同时满足：

```text
方法生命周期和内容版本当前
结果能力命中任务目标
真实动作入口当前
条件和条件结果对完整
输入与参数绑定闭合
禁止项未命中
授权、安全、验证和归因合同成立
相关因果与状态证据仍当前
```

方法首只用于召回；执行必须回到条件、结果、动作入口和本轮参数。

## 5. 因果链边界

同特征前置、跨特征联动和状态变化线索可帮助发现条件或间接路径，但不能单独证明某方法能产生目标。方法能力必须由动作动态、条件结果对和实际复现证明。

安全、服务、观察、外设和学习前置只能通过正式因果、需求条件、方法条件或已入树子需求进入，不得由筹办器硬编码。

## 6. 缺口分类

| 缺口 | 合法结果 |
| --- | --- |
| 无既有候选方法 | 路径查找状态可以记为无确认路径；本轮闭合结果只能是 5230 的合法等待，或已经定位具体能力缺口后的能力缺口承接；不得自动建方法首 |
| 候选方法失效或不可用 | 指向该方法及方法状态的能力缺口 |
| 缺动作入口 | 指向目标方法动作入口的结构化缺口 |
| 缺条件结果对 | 指向目标方法条件结果契约的结构化缺口 |
| 缺输入或参数 | 指向具体参数宿主、特征和取得/绑定状态的缺口 |
| 条件事实未满足 | 指向正式条件状态及合法生产者的等待或子目标 |
| 证据不足 | 指向观察、验证、归因或因果证据的具名缺口 |

只有既有候选已明确暴露具体缺口时，才能形成对应方法补齐目标。单纯无候选不得伪装为“已有方法不可用”，不得自动派生泛化学习需求。

“无确认路径”只描述本轮召回尚未形成可确认方法路径，不是第七类筹办闭合结果、任务终态或上层回退。筹办结束前必须把它收束为带等待对象、合法生产者、重新触发条件和当前版本的合法等待；只有缺口宿主、缺口角色和合法处理方已经确定时，才可收束为能力缺口承接。

## 7. 直接达成与补齐推进

可直接达成目标且就绪的方法优先。只推进方法结构、参数或状态的补齐方法属于另一层候选，必须通过结果能力说明其补齐目标，不能冒充业务目标方法。

方法状态、执行入口、条件结果对、参数和前置条件是不同缺口，不得压成一个“无方法”布尔值。

## 8. 回流和验证

具体缺口经需求领域入树、被子任务承接并形成结果后，触发父任务新筹办轮次。父任务必须重新召回和复判，不能复用旧就绪结论。

验证覆盖空索引、索引陈旧、候选失效、结果不匹配、多个候选、参数缺失、条件突变、学习候选以及无候选不建方法首。
