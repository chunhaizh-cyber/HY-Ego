# 4140 子规范 枚举型实例特征值合法来源

编号：4140
日期：20260720
状态：正式子规范

## 1. 定位

本规范区分枚举值域定义和枚举实例值生产，规定每个枚举状态必须有合法生产者和证据。

直接上级规范：1130、2100。

直接依赖规范：4160。

相关边界规范：1120、1160、5200、5300。

## 2. 两层结构

1. 枚举值域是抽象特征定义的一部分，声明允许的 I64 点值及含义；建立值域不需要为每个值创建本能方法。
2. 枚举实例值是某宿主某实例特征在某时刻的当前事实，必须由合法生产者提交，并关联状态、动态或规则证据。
3. 值域初始化不得被误写成实例状态产生。

## 3. 合法生产者

| 枚举事实类型 | 合法生产者 |
| --- | --- |
| 结构或生命周期状态 | 对应领域创建、迁移或提交入口 |
| 直接观察/检查结果 | 具有明确输入输出的观察、检查或外设方法 |
| 综合判断结果 | 已登记输入特征族和收束规则的判断方法/服务 |
| 数值比较归类 | 特征比较与枚举映射规则 |
| 外部反馈状态 | 外部材料承接、验证和世界事实提交链 |
| 初始化事实 | 具名初始化入口及初始化来源证据 |

线程内部局部标记若不进入机器事实可留在局部结构；一旦参与需求、因果、调度或结算，就必须进入上述正式生产链。

## 4. 专用方法判定

运行时必须通过观察、控制、验证或综合判断才能成立的枚举值，需要具名方法或判断服务。由生命周期、结构初始化或已有规则直接决定的枚举，不为每个值另建专用本能方法。

方法状态、任务状态等由各自领域统一提交，不允许任意线程裸写。

## 5. 需求目标

枚举型需求目标使用等于或等于集合；目标值必须属于定义值域并存在合法生产者。缺生产者时只能形成方法就绪、判定规则、观察或学习缺口，不得进入普通执行路径。

## 6. 评估缓存边界

质量、风险、可用性等评估结果若需要持久化，必须带来源方法、输入场景、证据、生成时间、规则版本和有效期；不得作为存在或场景的无来源原始事实常驻。

## 7. 禁止事项

- 禁止由说明文字、日志、UI 或随手赋值产生枚举事实。
- 禁止无动作动态、来源动态或规则证据的枚举修改。
- 禁止把多个层级状态压成一个过重枚举。
- 禁止为结构型枚举每值盲目建立本能方法。
