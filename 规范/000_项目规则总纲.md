# 000 项目规则总纲

编号：000
日期：2026-07-05
状态：正式起步规则

## 1. 定位

本文件是 `海中鱼巣` 的第一版项目规则总纲，用于承接旧项目中已经证明有效的工程治理口径，并按新项目“全量重构、重新建核”的目标收束。

本文件不要求新项目复制旧项目结构；只迁移适合新内核的硬规则。

## 2. 核心结构规则

### 2.1 结构闭环

系统内部逻辑必须通过项目结构表达和流转。

第一版允许的核心结构为：

```text
节点
节点句柄
主信息
关系
索引
特征
特征值
需求
任务
方法
状态
动态
因果引用
语素入口
```

禁止用以下材料承载机器逻辑：

```text
日志
控制台输出
注释
说明文档
临时字符串
临时 JSON
显示标题
调试文本
```

### 2.2 节点身份

节点长期身份必须使用稳定身份：

```text
节点编号
节点句柄
主键
版本号
```

指针只能作为运行期缓存或临时访问材料，不得作为长期事实身份。

### 2.3 主信息边界

主信息只保存身份、拓扑锚点、值容器或明确声明的非权威缓存。

凡是表达对象属性、观测事实、统计事实、治理状态、判断结论、任务结果、方法结果或因果解释的内容，应优先进入：

```text
特征
特征值
状态
动态
二次判断结果
因果引用
需求
任务
方法
```

### 2.4 关系边界

新项目不得把所有关系混成普通父子关系。第一版至少区分：

```text
普通父子
归属关系
引用关系
索引关系
模板关系
运行期临时关系
因果来源关系
```

关系变更必须通过仓库或领域服务入口完成。

## 3. 领域服务规则

所有业务写入必须经过领域服务。

第一版领域服务包括：

```text
语素服务
世界服务
特征服务
特征值服务
需求服务
任务服务
方法服务
因果服务
```

禁止外部模块直接写：

```text
节点仓库
主信息仓库
关系仓库
索引仓库
```

只读查询可以通过领域服务返回只读视图，不暴露可写内部对象。

## 4. 特征和值规则

一阶机器事实优先使用：

```text
I64 状态值
I64 Q10000 评分
计数
版本号
时间戳
节点句柄
主信息句柄
VecI64
VecIU64
特征值句柄
```

普通 bool 不得作为一阶机器事实扩散。需要二态判断时，应由状态值、评分、计数、版本、来源证据或复合裁决生成。

## 5. 动作和因果规则

线程不是动作来源。

外部输入不是动作来源。

动作来源必须来自明确方法、领域服务写入入口或后续正式定义的动作提交入口。

因果引用必须能回查：

```text
来源动作
输入材料
被改变目标
改变前值
改变后值
验证方式
```

## 6. 显示和日志规则

显示层、日志层和控制台输出只供人读。

显示层可以读取结构：

```text
节点
主信息
关系
特征
特征值
需求
任务
方法
动态
因果引用
```

显示层不得写业务事实，不得作为任务完成、需求满足或方法成功的机器判断来源。

## 7. 持久化规则

第一版运行期核心以内存仓库为准。

数据库和文件持久化只作为：

```text
快照
事件日志
恢复材料
审计材料
只读查询材料
```

数据库不得直接裁决机器事实，不得绕过领域服务写运行期结构。

## 8. 命名规则

函数名、变量名、类内业务成员名必须使用中文。

允许英文的范围：

```text
C++ 关键字
标准库标识
第三方 API
编译系统字段
外部协议固定字段
```

禁止为了方便新增英文业务变量名、英文业务函数名或英文业务枚举名。

## 9. 完成声明规则

任何“完成 / 已实现 / 已支持 / 已接入 / 已闭环”声明必须给出：

```text
输入入口
核心处理
结构承载
输出变化
调用路径
验证方式
```

缺少结构承载或验证方式时，只能声明为：

```text
文档草案
设计方向
代码壳
部分实现
待验证
```
