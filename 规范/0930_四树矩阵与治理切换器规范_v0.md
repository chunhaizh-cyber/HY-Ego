# 0930 四树矩阵与治理切换器规范 v0

## 1. 四树矩阵

### 1.1 总定义

本规范中的“四树”固定为：

- `筹办方法树`
- `维护方法树`
- `纠偏方法树`
- `收束方法树`

必须固定：

- 四树是域方法树，不是具体任务树，也不是四套并行扩张器
- 治理切换器不是第五棵树，只负责判断“现在该激活哪棵域方法树”
- 域方法树只有在作用于某个具体任务虚拟存在时，才会实例化为该任务的一段任务子树
- 四树之间不得互相串岗

### 1.2 四树矩阵

| 域方法树 | 入口条件 | 退出条件 | 允许动作 | 禁止动作 | 输出结果 | 因果登记要求 |
| --- | --- | --- | --- | --- | --- | --- |
| 筹办树 | 新需求进入；根任务未建立；大回退后需重组 | 根任务已建立且进入待执行、待处理、就绪、排队中之一 | 创建任务、初始细分、绑定初始方法、写初始控制快照、插入初始检查点 | 直接终止任务、无因果创建大量分支 | 根任务、初始子任务、初始方法绑定 | 必须登记“为什么立任务、为什么这样细分、为什么先绑这个方法” |
| 维护树 | 任务存在且未终态；无高优先级纠偏；主链仍可推进 | 步前检查通过并继续；或转纠偏；或转收束 | 步前检查、步后检查、更新预算、有限细分、有限延长、调整方法优先级 | 硬中断式终止、无限细分、越过安全边界扩展 | 更新后的控制快照、必要的新子任务或补充步骤 | 必须登记“为什么继续维持、为什么补检查、为什么只做局部修补” |
| 纠偏树 | 失败反馈；安全跌落；超时；外部取消；检查失败 | 已完成回退、替换、收缩、终止之一 | 挂起、回退、替换方法、插恢复任务、收缩、终止、登记中断因果 | 正常大规模扩展、无证据直接继续主路径 | 纠偏结果、恢复路径或终止结果 | 必须登记“偏差来源、触发信号、采取了哪种纠偏动作、为什么允许恢复或终止” |
| 收束树 | 正常完成；明确失败；取消；预算耗尽；不可恢复 | 已写反馈、已写经验、已归档状态链和因果链 | 汇总结果、写反馈、写经验、写回适配关系、清理无效分支、归档 | 再次细分、再进入筹办、无条件重试 | 闭环结果、经验写回、终态任务 | 必须登记“为什么现在收束、最终结果是什么、哪些经验被沉淀、哪些分支被关闭” |

### 1.3 四树 v0 最小动作范围

筹办树 v0 只做：

- 从需求创建根任务
- 初始绑定方法

维护树 v0 只做：

- 步前检查
- 步后更新控制快照

纠偏树 v0 只做：

- 方法失败后切换备用方法
- 安全跌落后收缩

收束树 v0 只做：

- 写反馈
- 写经验
- 置终态


## 2. 治理上下文

### 2.1 设计目标

治理上下文只服务于一件事：

- `给轻量治理切换器提供最小判定输入`

v0 不追求把所有控制细节都塞进去，只保留最小足够字段。

### 2.2 推荐结构

```cpp
enum class 枚举_治理阶段
{
    筹办 = 0,
    维护 = 1,
    纠偏 = 2,
    收束 = 3
};

struct 结构_控制快照
{
    I64 安全值_Q10000 = 0;
    I64 服务值_Q10000 = 0;
    I64 需求值_Q10000 = 0;
};

struct 结构_治理上下文
{
    基础信息节点类* 被控任务 = nullptr;
    基础信息节点类* 当前需求 = nullptr;

    bool 根任务已建立 = false;
    bool 存在纠偏请求 = false;
    bool 满足收束条件 = false;
    bool 当前步骤完成 = false;
    bool 当前步骤失败 = false;

    结构_控制快照 当前控制快照{};
};
```

### 2.3 字段解释

| 字段 | 作用 |
| --- | --- |
| 被控任务 | 当前总控要判域的任务对象 |
| 当前需求 | 当前任务所服务的需求来源 |
| 根任务已建立 | 判断是否还停留在筹办入口 |
| 存在纠偏请求 | 判断是否应切到纠偏树 |
| 满足收束条件 | 判断是否应切到收束树 |
| 当前步骤完成 | 给维护树、收束树提供局部完成信号 |
| 当前步骤失败 | 给纠偏树提供直接失败信号 |
| 当前控制快照 | 提供当前安全、服务、需求三值快照 |

必须固定：

- 治理上下文用于“判域”，不直接替代执行模块的完整业务上下文
- 若缺少可判域的关键信息，应保持保守，不凭空推断大规模扩张


## 3. 切换规则

### 3.1 最小切换规则

轻量治理切换器 v0 先固定为：

```text
未建根任务 -> 筹办
已建根任务且满足收束条件 -> 收束
存在纠偏请求 -> 纠偏
其余情况 -> 维护
```

这是 v0 的最小换挡规则。

### 3.2 推荐接口

```cpp
枚举_治理阶段 计算当前治理阶段(
    const 结构_治理上下文& 上下文);
```

如果需要同时返回原因，可扩为：

```cpp
struct 结构_治理切换结果
{
    枚举_治理阶段 阶段 = 枚举_治理阶段::筹办;
    std::string 原因说明;
};
```

### 3.3 切换纪律

必须固定：

- 切换器只判当前该进哪棵树，不直接执行树内复杂动作
- 每棵树动作完成后，必须重新计算治理阶段
- 不允许绕过切换器直接从 `维护树` 偷跳到大规模扩张
- 不允许在 `收束树` 内无条件重启主流程

### 3.4 第一条完整验收链

v0 最优先验收这条链：

```text
需求进入
-> 切到筹办树
-> 建根任务并绑定初始方法
-> 切到维护树
-> 执行一步
-> 方法失败，提交纠偏请求
-> 切到纠偏树
-> 切备用方法
-> 回到维护树
-> 执行完成
-> 切到收束树
-> 写反馈和经验
```


## 4. 收束

本文定义的是：

- `四树矩阵`
- `治理上下文`
- `轻量治理切换器 v0 的最小切换规则`
