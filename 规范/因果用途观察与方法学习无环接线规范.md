## 1. 定位

本规范固定从非权威因果概率候选和正式任务结果出发的单向守卫，并列出未来可能采用的权威用途材料、可重建统计、方法学习候选、独立晋级和后继代际必须满足的语义不变量。权威结构承载、概念映射、学习算法与代际结构分别须经 #283、#284、#285 S0 复核，不能由本规范直接生成后继结构或代码许可。

```text
因果发生概率候选
+ 正式任务结果与需求结算
-> 结构化结果效用
-> 经 #283 证明合法的权威用途材料
-> 经后继设计确认的非权威统计投影
-> 非权威方法学习候选
-> 独立晋级
-> 方法业务服务登记新代际
-> 后续任务召回、选择、执行
```

本链不证明稳定因果，不把概率等同于效用，不把统计或候选等同于方法事实，不允许本轮结果反写本轮选择。

## 3. 需求结算第一版目标口径

JY-369 以当前正式实现收敛第一版结算口径：

```text
目标状态值 == 实际状态值 -> 已达成。
目标状态值 != 实际状态值 -> 未达成。
第一版没有独立“证据不足”正式终态；材料不完整在写前拒绝或当前性返回收口。
同一需求第一版只允许一个正式结算结论：同结论、同完整结构幂等读回；异结论冲突。
未达成任务迁移待重筹办，不写需求最终结算。
任务完成但需求结算暂不可执行时，形成待结算账项，不伪造最终效用。
```

旧规范中的 `实际值 >= 目标值`、同一需求多次正式结算和独立“证据不足”结论不再作为第一版目标合同。当前 #225 新任务结果治理路径只部分符合；公开正式服务、既有通用组合入口和完整结构幂等由 #286 收口，#286 完成前不得把本节写成当前全量实现事实。未来若需区间比较、多次结算或证据不足终态，必须另立规范、迁移和恢复计划，不得在用途学习链中临时扩展。

## 4. 用途事件与结构化效用

当前 `任务执行冻结请求` 只冻结任务、选择记录、方法、动作和执行请求，没有“本回合实际采用哪个因果概率候选”的字段或权威映射。因此第一版增加独立的执行前用途来源冻结材料：

```text
因果用途来源规则版本和非零冻结键；该键是 #283 必须复核的未来观察身份候选，不在 #282 中生成第二观察键。
完整因果概率候选及其规则版本。
完整任务执行冻结请求。
任务、选择记录、方法、动作入口和来源版本。
形成冻结的任务序号与时间戳。
```

该材料只能由后继任务选择 / 执行组合路径在实际决定采用候选之后、动作执行之前形成。它在 #282 中仍是值式来源证据；未来如何进入权威观察必须先经 #283 S0 证明结构承载符合现行规则。没有该冻结材料、冻结内容与结果回执不一致，或调用方在结果产生后临时补造候选时，必须逻辑返回；禁止把任意历史 Q10000 候选与任意已收口任务结果事后配对。

用途事件必须同时保留：

```text
完整新版回执和冻结请求。
任务完整句柄与任务序号组成的回合身份。
来源需求、任务、选择记录、方法、动作入口及版本化来源。
目标状态、实际状态和实际动态证据。
执行前用途来源冻结中的因果原因 / 结果完整结构键、规则版本、历史机会证据和 Q10000。
任务结果治理三种正式状态之一。
任务达成分量、需求结算分量和观察阶段。
```

三种正式结果固定映射：

| 正式结果 | 任务达成分量 | 需求结算分量 | 观察阶段 | 后继 |
| --- | --- | --- | --- | --- |
| 未达成已转待重筹办 | 未达成 | 未形成最终结算 | 尝试未达成 | 可形成尝试用途材料；不得当作最终需求负效用或因果负样本 |
| 任务已完成待需求结算 | 已达成 | 待结算 | 待结算 | 保留完整待结算材料；权威承载 / 恢复待 #283 复核，不得进入最终统计和学习 |
| 已完成并结算 | 已达成 | 已满足 | 最终已结算 | 未来结构必须保证同一任务回合最终用途材料唯一，当前不预设承载 |

冻结候选中的历史机会分类和 `Q10000` 始终作为独立概率材料保留。它们不是当前任务回合的结果分类，与任务达成或需求结算可能同向也可能不同向，任何服务不得把二者压缩成单一“好 / 坏”分数。

## 5. 候选权威用途观察结构与 #283 S0

### 5.1 候选节点角色

JY-369 暂时保留以下候选编号，供 #283 S0 检查；它不是当前代码值，也不构成追加枚举许可：

```text
节点类型::用途观察记录 = 15
```

若后继设计继续采用用途观察记录，它只能是长期证据账项，不属于基础信息语素入口，不是高级业务节点角色，也不替代状态、动态、因果引用、需求、任务或方法。#283 完成前不得声明节点 15、用途观察服务或写入路径存在。

第一版候选设计不新增关系类型，并提出以下关系角色。#283 必须逐项复核现有关系 API、端点、顺序号、基数、规格形成方、写入方、读回、并发和恢复；不能证明的项目标为缺口，不得由执行角色直接实现：

| 关系 | 方向 | 顺序号 | 基数 | 角色 |
| --- | --- | --- | --- | --- |
| 归属 | 任务 -> 用途观察记录 | 0 | 同一任务回合恰有一项 | 观察所属任务 |
| 引用 | 用途观察记录 -> 需求 | 1 | 恰一 | 来源需求 |
| 引用 | 用途观察记录 -> 方法 | 2 | 恰一 | 本回合所选方法 |
| 引用 | 用途观察记录 -> 任务方法选择记录 | 3 | 恰一 | 选择审计 |
| 引用 | 用途观察记录 -> 实际状态 | 4 | 恰一 | 任务实际结果状态 |
| 引用 | 用途观察记录 -> 结算状态 | 5 | 待结算时零，最终已结算时恰一 | 正式结算证据 |

若 #283 证明上述候选角色无法在不改变现有关系通用语义的前提下保证同任务回合唯一发布，必须退回设计；S0 本身不得追加专用关系或批量增加用途、方法、状态、统计关系类型。

### 5.3 主信息红线与字段承载门禁

现行主信息只允许身份、拓扑锚点、值容器或明确声明的非权威缓存，因果证据、任务运行和业务状态不得直接作为通用主信息字段。当前没有用途观察专用格式、容量上限、严格解码器或完整往返重建入口，因此不得预设把完整回执、冻结请求、机会证据和结构化效用序列化进主信息。

#283 必须把以下材料逐字段分流到允许的结构，分类为可复用、缺结构、缺入口、违反规则或待设计：

```text
观察主键与规则版本；身份候选固定为观察主键原样等于执行前冻结键。
任务完整句柄、任务序号、回执身份和幂等材料。
冻结执行请求、选择 / 方法来源版本和完整因果候选。
任务结果收口、结构化效用各分量、状态、动态、结算和重试材料。
历史机会证据组、原因 / 结果完整结构键、因果规则版本和 Q10000。
```

当前观察阶段不得预写成可变主信息字段。顺序号 6 阶段关系、待结算定位和恢复顺序只是候选，必须经 #283 对现有会话与 #226-#230、#237-#239 的事实复核后再由设计角色裁决；当前不预设游标、格式或平行恢复链。

哈希只允许加速召回。幂等、冲突和相等裁决最终必须比较完整规范元组，不得只比较哈希、摘要、日志、时间戳、缓存键或节点编号。

## 6. 幂等语义与发布 S0 门禁

```text
同观察主键 + 同完整内容 -> 幂等读回。
同观察主键 + 异内容 -> 幂等冲突。
同任务完整句柄 + 同任务序号 + 不同观察主键 -> 冲突。
任务序号倒退 -> 写前拒绝。
迟到但仍待结算 -> 只恢复同一账项。
已形成尝试或最终观察 -> 不另建、不重复计数。
```

上述是未来必须保持的业务语义，不证明当前结构可以承载。#283 必须对以下候选发布链逐函数复核实际入口、入口拒绝、精确撤销、并发单赢家和最后发布点：

```text
写前完整准入
-> 未发布节点 15 候选
-> 主信息规范元组候选
-> 初始阶段抽象状态与固定关系候选
-> 全量读回
-> 确认
-> 最后发布观察记录
```

若现有 API 无法证明无半结构、完整读回、双重唯一或同账换代，只登记设计缺口。S0 不创建候选、不登记阶段、不运行构建。未来任一写后结果不符合已确认不变量时仍必须追根因并精确撤销，不得扫描修复或仅记录日志后继续。

## 7. 概念映射、非权威统计和方法学习 S0

若 #283 证明未来权威用途结构可合法承载，后继统计设计只能从该已发布权威结构读取并重建，并至少保持：

```text
尝试未达成与最终已结算分栏统计。
待结算不进入统计。
Q10000、任务达成、需求结算分别投影。
完整原因 / 结果结构键始终保留，不得被概念句柄或签名缩写替代。
清空投影不得改变用途账、方法或任务事实。
```

当前因果概念签名只表达单个动态的目标类型、前后 I64 和来源动作类型，无法预设无损表达有序原因 / 结果完整结构键及句柄身份；生命周期转换也不保证活动图版本递增。#284 只做字段映射与当前性 S0，不实现概念映射或统计投影。

方法学习候选的概念边界是引用确定排序的来源观察组、统计规则版本、目标方法当前版本和候选变更类型；候选不得创建节点、关系、索引或任务选择，不得修改现有方法。结构化条件 / 结果 / 动作差异的正式来源和确定推导算法尚未裁决，必须由 #285 S0 形成事实矩阵后再修订本节。

后继独立晋级设计至少必须重新复核：

```text
来源观察仍有效且阶段允许。
统计投影可由来源观察重建。
目标方法与规格当前性成立。
存在明确任务授权和晋级规则版本。
候选未被重复晋级或冲突晋级。
```

通过后也只能调用方法业务服务登记新代际方法。统计服务、线程、因果模式服务、学习候选服务和晋级服务均不得绕过方法服务写仓库。来源方法、授权来源、来源材料、候选幂等材料和后继任务序号下界的节点 / 主信息 / 关系 / 索引映射必须等 #283-#285 三项 S0 后重新设计，当前不得留给执行角色发明。

## 8. 代际与无环约束

本回合冻结请求中的选择记录、方法、动作和来源版本一经执行不得被本回合结果修改。新代际方法只能参加后续任务的召回、复判、排序、任务选择和执行。

```text
第 N 回合选择并执行方法 Vn
-> 第 N 回合结果形成观察
-> 学习候选与独立晋级
-> 登记方法 Vn+1
-> 仅第 N+1 或更后任务回合可召回 Vn+1
```

禁止同轮反馈闭环、原地改写 Vn、替换已发布选择记录、让新候选重跑当前动作或改变当前任务结果。

## 9. 运行期与恢复

唯一运行期未来只可有一个用途材料所有者和一个恢复入口；是否需要用途观察服务、重试消费者或其它组件须由 S0 后设计裁决。当前没有恢复影响执行计划；必须先完成 #283-#285 三项 S0，由设计角色补齐合法结构、映射、学习算法、方法代际和后继调用后，才可裁决恢复影响后续。

未来恢复影响 S0 只能形成接口矩阵、原地修订既有恢复计划或生成必要的后继实施计划，不修改 C++ / 工程，不建立平行宿主、快照或恢复链。后继恢复实现合同至少必须保持：

```text
恢复四仓库和经后继设计确认的权威用途结构
-> 校验经 #283 确认的身份、回合和阶段不变量
-> 恢复或重建经 S0 确认的待结算定位状态，但不重复写用途材料
-> 从权威结构重建经后继设计确认的统计投影
-> 从投影重新形成经后继设计确认的学习候选
-> 不在恢复过程中自动晋级或执行方法
```

非权威统计投影和方法学习候选不得持久化为恢复权威事实。

## 10. 非成功返回二分

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 执行前用途来源冻结缺失、版本不支持或与结果回执不一致 | 逻辑内返回 | 结构零变化，不事后配对任意候选 |
| 待结算尚未形成正式结算 | 逻辑内返回 / 可恢复阶段 | 保持同一账项，不进入统计或学习 |
| 候选材料不足、模式概念映射不唯一、晋级未通过 | 逻辑内返回 | 保持非权威或返回空，不写方法事实 |
| 正式 DTO 同回合身份、选择、状态、动态或结算互相矛盾 | 追根因解决 | 停止后继，不选择首项或补默认值 |
| 写后读回、关系基数、主信息元组或发布状态不一致 | 追根因解决 | 精确撤销未发布候选，不继续统计 |
| 恢复产生同回合双记录、游标倒退或重复计数 | 追根因解决 | 停止恢复发布，不用缓存覆盖权威账 |
