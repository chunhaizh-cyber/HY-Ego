# 非权威缓存与统计规范

编号：非权威缓存-001
日期：2026-07-06
状态：已确认项目规范 / 第一轮非权威缓存和统计边界方案 / 未改代码

## 1. 定位

非权威缓存和统计只服务性能、索引、去重、诊断和人读观察，不承载机器事实。

本规范覆盖主信息迁移 S4 中登记的：

```text
MI-V2 内容哈希 / 命中次数 / 最后命中
MI-BI2 结构_统计
MI-D3 安全 / 服务权重 / 累计结算 / 最近结算任务和时间中的非权威摘要部分
```

本规范不允许把这些缓存项回流为主信息权威字段。

## 1.1 第一轮确认口径

第一轮非权威缓存与统计确认以下口径：

```text
缓存不进入主信息权威字段。
第一轮可实施范围收窄为缓存命名空间、缓存记录结构、结构统计运行期快照、缓存清空 / 重建接口壳。
特征值内容哈希、VecI64 / VecIU64 原始值缓存和需求结算摘要缓存必须等待对应权威结构确认并实现后再进入代码。
缓存缺失、缓存清空或缓存命中变化不得改变需求满足、任务完成、方法成功或世界事实。
```

## 3. 非权威缓存类型

| 缓存类型 | 用途 | 权威来源 | 清空后影响 |
| --- | --- | --- | --- |
| 特征值内容哈希 | 去重、索引和快速比较 | 特征值原始 Vec 数据 | 清空后可由原始值重建，不改变特征值事实 |
| 特征值命中次数 | 观察值复用次数 | 索引命中事件或读取事件 | 清空后不改变任何业务事实 |
| 特征值最后命中 | 观察最近复用时间 | 索引命中事件时间 | 不是状态发生时间，不改变业务事实 |
| 结构统计 | 诊断、调度、观察 | 节点、关系、索引、状态、动态等权威结构 | 清空后可重新扫描或重建，不改变结构事实 |
| 需求结算非权威摘要 | 人读显示和兼容摘要 | 需求结算记录、状态、动态和因果证据 | 清空后可由结算记录重建，不改变结算事实 |

## 4. 缓存承载结构

第一轮允许的缓存承载位置必须显式登记，不得隐式塞入主信息权威字段。

| 承载位置 | 用途 | 第一轮状态 | 边界 |
| --- | --- | --- | --- |
| 索引仓库缓存项 | 特征值哈希、命中次数、最后命中等索引辅助 | 设计占位 | 只定位候选或记录观察，不裁决业务事实 |
| 统计服务运行期快照 | 节点、关系、索引统计 | 第一轮可实施 | 可清空、可重建，不改变权威结构 |
| 需求结算摘要缓存 | 由权威结算记录重建的人读摘要 | 设计占位 | 需求结算记录未确认并实现前不得编码 |
| 主信息非权威缓存区 | 经字段功能确认后的兼容缓存 | 后续确认后才可使用 | 必须标记非权威，不得进入主信息权威值容器 |

第一轮代码若进入实施，优先实现统计服务运行期快照和缓存清空 / 重建接口壳；不得提前实现依赖 Vec 原始值或需求结算记录的缓存事实。

## 5. 缓存命名空间

缓存命名空间用于定位、清空、失效和重建缓存项，不表达业务事实。

缓存命名空间至少包含：

```text
缓存类型
来源结构类型
来源句柄或来源仓库
来源版本
生成规则版本
缓存值类型
是否可持久化
失效条件
```

第一轮命名空间示例：

```text
特征值内容哈希:v1:特征值句柄
特征值命中次数:v1:特征值句柄
结构统计:v1:节点仓库
结构统计:v1:关系仓库
结构统计:v1:索引仓库
需求结算摘要:v1:需求句柄
```

命名空间只用于缓存治理。不得把命名空间字符串、显示标题、函数名或日志标签作为权威身份。

## 6. 权威结构和缓存关系

缓存不得替代以下权威结构：

```text
节点身份
关系
索引项
特征值原始值
状态节点
动态节点
因果引用
需求目标状态
任务目标 / 实际结果状态
方法条件 / 结果结构
需求结算记录
```

缓存读取方必须能接受缓存缺失。缓存缺失只能导致性能下降、需要重建或诊断信息减少，不得导致业务事实丢失。

## 7. 写入和重建规则

缓存写入必须满足：

```text
写入方明确。
读取方明确。
失效条件明确。
重建来源明确。
缓存关闭或清空时，不改变机器事实。
```

缓存不得由显示层、日志层或控制台输出反向写入。

缓存重建只能读取权威结构，不得凭旧缓存互相推导出新事实。

命中次数只记录候选召回、索引读取或特征复用观察；命中次数不代表方法好用、特征重要、需求满足或任务完成。

最后命中是缓存观察时间，不是状态发生时间、动态发生时间或需求结算时间。

孤立关系诊断只做人读诊断，不得由统计缓存自动删除、修复或补写关系。结构修复必须另走正式仓库或领域服务入口。

需求结算摘要缓存依赖需求结算记录规范和详细设计确认。权威结算记录未实现前，不实现需求结算摘要刷新；不得从需求状态节点、控制台输出或日志倒推摘要缓存。
