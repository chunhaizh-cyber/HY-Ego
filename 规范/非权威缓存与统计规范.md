# 非权威缓存与统计规范

编号：非权威缓存-001
日期：2026-07-06
状态：待确认项目规范 / 第一轮非权威缓存和统计边界方案 / 未改代码

## 1. 定位

非权威缓存和统计只服务性能、索引、去重、诊断和人读观察，不承载机器事实。

本规范覆盖主信息迁移 S4 中登记的：

```text
MI-V2 内容哈希 / 命中次数 / 最后命中
MI-BI2 结构_统计
MI-D3 安全 / 服务权重 / 累计结算 / 最近结算任务和时间中的非权威摘要部分
```

本规范不允许把这些缓存项回流为主信息权威字段。

参照材料：

```text

D:\鱼巢\规范\3100_根规范_需求主信息类_20260701.md
D:\鱼巢\特征值主信息类.h
D:\鱼巢\需求主信息类.h
D:\鱼巢\基础信息主信息类.h
```

旧项目可迁移口径：

```text
特征值原始值可有内容哈希用于去重和快速查找。
命中次数和最后命中只能表达复用观察，不表达业务事实。
结构统计可以帮助诊断、调度和观察。
需求结算摘要可以做人读显示或兼容缓存。
```

旧项目不可直接迁移内容：

```text
内容哈希作为业务类型或特征类型。
命中次数作为需求满足、任务完成或方法成功依据。
最后命中作为事实发生时间。
结构统计作为结构事实权威。
累计结算摘要作为需求满足或任务完成依据。
最近结算任务主键和最近结算时间作为结算事实权威。
```

## 3. 非权威缓存类型

| 缓存类型 | 用途 | 权威来源 | 清空后影响 |
| --- | --- | --- | --- |
| 特征值内容哈希 | 去重、索引和快速比较 | 特征值原始 Vec 数据 | 清空后可由原始值重建，不改变特征值事实 |
| 特征值命中次数 | 观察值复用次数 | 索引命中事件或读取事件 | 清空后不改变任何业务事实 |
| 特征值最后命中 | 观察最近复用时间 | 索引命中事件时间 | 不是状态发生时间，不改变业务事实 |
| 结构统计 | 诊断、调度、观察 | 节点、关系、索引、状态、动态等权威结构 | 清空后可重新扫描或重建，不改变结构事实 |
| 需求结算非权威摘要 | 人读显示和兼容摘要 | 需求结算记录、状态、动态和因果证据 | 清空后可由结算记录重建，不改变结算事实 |

## 4. 权威结构和缓存关系

缓存不得替代以下权威结构：

```text
节点身份
关系
索引项
特征值原始值
状态节点
动态节点
因果引用
需求目标状态
任务目标 / 实际结果状态
方法条件 / 结果结构
需求结算记录
```

缓存读取方必须能接受缓存缺失。缓存缺失只能导致性能下降、需要重建或诊断信息减少，不得导致业务事实丢失。

## 5. 写入和重建规则

缓存写入必须满足：

```text
写入方明确。
读取方明确。
失效条件明确。
重建来源明确。
缓存关闭或清空时，不改变机器事实。
```

缓存不得由显示层、日志层或控制台输出反向写入。

缓存重建只能读取权威结构，不得凭旧缓存互相推导出新事实。

## 6. 禁止扩大声明

本规范建立后只能声明：

```text
非权威缓存和统计边界方案已建立。
MI-V2、MI-BI2、MI-D3 的缓存摘要部分已有新项目承载方案。
这些字段不得直接迁移为主信息权威字段。
```

不能声明：

```text
缓存统计规范已确认。
缓存统计代码已实现。
缓存系统已完成。
需求结算系统已完成。
旧统计字段已迁移。
```
