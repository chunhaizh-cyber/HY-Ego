# 公用函数规则规范

日期：2026-07-08
状态：正式规范 / 从旧鱼巢规则提取后按海中鱼巣口径改写

## 1. 定位

本规范用于约束 `海中鱼巣` 中“公用函数、公共辅助函数、已加锁辅助函数、只读快照函数、序列化 / 恢复辅助函数”的归属和准入。

本规范只迁移旧 `鱼巢` 中已经稳定的工程规则，不复制旧项目全局共享函数实现，不迁移旧双向链表模板，不把旧函数体作为新项目代码入口。

旧项目证据入口：

```text

D:\鱼巢\说明书\函数功能说明文档.md
D:\鱼巢\说明书\双向链表模板核心说明.md

D:\鱼巢\实施记录\20260704_世界树操作门面整改_Codex断点清单.md
```

## 2. 公用函数分类

公用函数候选必须先分类，再决定落点。

| 类型 | 定义 | 新项目落点 |
| --- | --- | --- |
| 纯通用函数 | 不依赖项目领域类型，不读写机器事实，只做通用计算、格式化、边界转换或轻量辅助 | 可进入后续确认的公共工具文件 |
| 仓库内部辅助函数 | 只服务同一仓库内部结构维护、查找、重挂、快照或序列化 | 对应仓库内部私有 / 受控辅助入口 |
| 领域服务公共入口 | 涉及节点、主信息、关系、索引、特征、状态、动态、需求、任务、方法、因果等业务结构 | 对应领域服务或仓库正式入口 |
| 流程编排候选 | 跨多个服务形成应用逻辑步骤 | 服务矩阵和流程图迁移包，不落成全局公用函数 |
| 禁止候选 | 试图用工具函数绕过领域服务、仓库锁、入口拒绝、确认门禁或第一版禁止接入范围 | 拒绝并登记缺口或待确认项 |

## 3. 允许和禁止落点

允许进入纯通用公用函数的内容：

```text
字符串或路径格式化的人读辅助。
不依赖项目结构的数值计算。
不读写节点、主信息、关系、索引的临时转换。
不承载机器事实的诊断文本整理。
```

禁止进入纯通用公用函数的内容：

```text
创建、修改或删除节点。
写主信息、关系或索引。
读取可写内部引用。
裁决需求满足、任务完成、方法成功或世界事实。
直接访问特征值服务给高级服务使用。
隐藏旧 SQL、控制面板、线程、外设或体素接入。
绕过领域服务组合多个仓库写入。
```

只要函数候选接触以下结构，就不得作为全局纯通用函数处理：

```text
节点
节点句柄
主信息
关系
索引
特征
特征值
存在
场景
状态
动态
二次特征
因果引用
语素入口
需求
任务
方法
动作入口
非权威缓存
事件日志
仓库快照
```

## 4. 新增前门禁

新增任何公用函数、公共辅助函数或跨文件辅助入口前，必须先完成以下检查：

```text
1. 查当前代码和函数说明，确认是否已有同等功能入口。
2. 说明函数用途、归属方、允许调用方和禁止调用方。
3. 标明读写类型：纯计算、只读、候选读取、写结构、序列化、恢复。
4. 标明触碰结构和结构权威来源。
5. 标明入口前置拒绝条件。
6. 标明拒绝后结构不变化验收方式。
7. 标明是否需要锁、由谁持锁、是否存在已加锁内部辅助。
8. 标明生命周期：临时材料、运行期结构、持久化快照、审计材料。
```

已有仓库或领域服务可以承载时，优先扩展既有入口；不能因为复用方便而新建全局工具函数。

## 5. 入口拒绝和非法数据来源追溯

公用函数不得靠内部兜底掩盖非法输入。

入口检查的目的不是“让函数吞掉坏数据继续运行”，而是：

```text
发现非法数据来源。
拒绝本次写入或候选输出。
返回上游材料生成、服务调用或流程编排处消除来源。
证明拒绝后结构不变化。
```

涉及结构写入的入口至少必须拒绝：

```text
无效句柄。
已删除句柄。
错误节点类型。
错误关系类型。
缺少必需前置关系。
生命周期不满足。
越权写入方。
材料来源不可信。
矩阵归属不唯一。
服务入口不明确。
```

入口拒绝已经排除错误输入后，内部写入结果、关系目标、参数值或结构状态仍不符合预期时，必须视为逻辑错误追根因；不得把公共辅助函数设计成静默修复器。

## 6. 已加锁辅助函数规则

`已加锁` 风格函数只能作为同一仓库或同一对象内部的持锁路径辅助函数。

必须遵守：

```text
公开入口负责加锁和入口前置拒绝。
已加锁辅助函数不再次获取同一把锁。
已加锁辅助函数不得暴露给领域服务、显示层、工具层或外部调用方。
已加锁辅助函数不得调用可能反向进入本仓库公开入口的函数。
已加锁辅助函数不得在持锁期间落日志、弹窗、持久化 I/O、外设等待或调用上层领域服务。
```

命名为 `已加锁` 的辅助函数如果需要跨仓库访问，必须先有详细设计说明锁顺序和拒绝路径；否则应拆回公开入口或服务编排。

## 7. 只读快照和材料函数规则

只读函数、快照函数和候选读取函数必须返回：

```text
记录副本。
句柄组。
只读材料。
非权威候选。
快照结构。
```

不得返回：

```text
可写内部引用。
长期裸指针。
可绕过仓库入口修改的容器引用。
会被上层误认为权威事实的临时缓存。
```

快照和候选结果只作为读取材料、审计材料或缩短锁持有时间的运行期副本，不得变成机器事实来源，也不得用于绕过领域服务写入。

只读访问不得隐式创建结构；需要创建或确保存在时，必须走明确写入口，并重新执行入口拒绝。

## 8. 序列化、快照和恢复辅助规则

序列化、快照和恢复类公共辅助函数必须明确：

```text
格式版本。
数量上限。
字段边界。
读取失败清理。
恢复前校验。
恢复拒绝矩阵。
恢复后读回验证。
```

序列化和快照只作为持久化、恢复、审计或人读排查材料，不裁决运行期事实。

恢复入口不得先替换当前结构再靠扫描器发现坏数据；必须在替换或提交前完成版本、数量、句柄、关系端点、索引边界和结构一致性准入。

## 9. 服务矩阵和流程图迁移应用

流程图迁移过程中出现“公用函数”候选时，必须先执行公用函数归属判断：

```text
纯通用 -> 后续公共工具候选。
仓库内部 -> 对应仓库内部辅助。
领域结构 -> 对应领域服务或仓库正式入口。
跨服务编排 -> 服务逻辑包和流程图迁移包。
绕过归属服务 -> 拒绝路径。
```

服务操作函数矩阵中不得用“公用函数”替代以下字段：

```text
新项目承接服务。
机器事实写入方。
只读方 / 显示方。
拒绝路径。
结构不变化验收。
后续产物。
```

如果一个旧项目公共函数候选无法说明归属服务、允许调用方、读写类型、结构不变化验收和后续产物，则不得进入批次级代码实施包。

## 10. 验收方式

文档层验收：

```text
已列出旧项目证据入口。
已明确规则迁移口径而非复制旧实现。
已同步到规则总纲、规则迁移清单、迁移路线权力分层、服务矩阵和流程图模板。
```

代码层验收在未来具体切片中另行声明，至少包括：

```text
新增函数前已查重。
入口拒绝覆盖无效句柄、错误类型、越权来源和必需材料缺失。
拒绝后结构不变化。
只读入口不暴露可写内部引用。
已加锁辅助只在内部持锁路径调用。
序列化 / 恢复有版本、边界和拒绝矩阵。
```

## 11. 完成声明边界

本规范只能证明：

```text
旧鱼巢公用函数相关规则已被提取并按海中鱼巣口径改写。
后续公用函数候选已有归属、准入、拒绝、锁、快照和恢复边界。
```

不能证明：

```text
新项目已经新增公共工具实现。
旧全局共享函数类已经迁移。
旧函数体已经迁移。
旧项目能力已经迁移。
多线程调度、控制面板、SQL、D455、体素或外设已经接入。
```
