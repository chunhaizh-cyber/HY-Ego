# 2100 根规范 特征值

编号：2100
日期：20260720
状态：正式根规范

## 1. 根定位

特征值是特征系统的原始值容器。它只承载原始值材料、原始类型、版本和必要的比较、序列化及索引统计，不承载业务解释。

## 2. 本项目承载

```text
特征值节点及稳定句柄
+ 唯一主键和主信息身份
+ 实例特征槽位到当前值的唯一关系
+ I64、VecI64 或 VecU64 之一的原始值材料
+ 原始值版本和容器版本
+ 非权威序列化、哈希和候选统计材料
```

向量原始材料可以由特征值服务的受控侧表承载，但侧表身份、版本和特征值节点必须一一对应；侧表不得成为绕过节点和关系的独立业务事实来源。

## 3. 事实与边界

1. 特征值可以保存原始值、必要序列化材料和底层精确比较能力。
2. 特征值不得保存业务类型、编码解释、坐标维度语义、轮廓解释、需求满足或任务完成结论。
3. 坐标、轮廓、颜色、体素及其它业务含义必须由特征定义、二次特征、状态或因果证据解释。
4. 哈希、序列化、恢复材料和值域候选都不是权威原始值本体；使用前必须回读节点、关系、原始值和版本。
5. 一个特征值节点同一版本只能有一种原始类型和一组自洽原始材料。

## 4. 写入、读取与验证

特征值服务只服务于特征系统，只有特征领域服务可以直接访问。需求、任务、方法及其它高级服务不得包含、接收或调用特征值服务。

特征领域服务负责实例槽位、值类型和值域准入；特征值服务及其事务参与者在同一写入会话中写入原始材料。有效读回必须证明值节点、主信息、唯一槽位来源、原始类型、原始版本和原始材料完全一致。

## 5. 生命周期与当前性

原始值变化必须推进原始值版本或形成新值节点，不得静默覆写后继续沿用旧版本。序列化和恢复候选必须带仓库上下文、格式版本和值版本；恢复材料必须经特征服务重新准入并发布，不能直接成为运行期事实。
