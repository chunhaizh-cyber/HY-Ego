# 5200 子规范 任务根据需求初始化

编号：5200
日期：20260720
状态：正式子规范
权威等级：正式规范

## 1. 定位

本规范定义任务承接需求时必须形成的结构锚点、任务特征来源、固定任务特征类型、方法开放特征角色和初始化顺序。

直接上级规范：3200。

直接依赖规范：0500、3100、3300。

相关边界规范：5210、5230、5240。

## 2. 结构锚点不是任务特征

以下事实由本项目节点和关系结构承载，不得再复制成并列权威特征：

| 结构事实 | 权威承载 |
| --- | --- |
| 任务身份 | 任务节点及稳定句柄 |
| 来源需求 | 任务—需求正式关系 |
| 运行态治理存在 | 任务—唯一虚拟存在关系 |
| 运行场景 | 任务—场景关系 |
| 目标宿主、目标特征和目标状态 | 通过来源需求及任务目标关系回读 |
| 当前生命周期 | 任务生命周期状态关系及版本 |
| 候选方法 | 本轮候选集合及其来源记录 |
| 当前方法 | 任务方法选择记录及当前发布关系 |
| 实际结果状态 | 任务—实际结果状态关系 |

这些结构事实参与任务状态判断，但不属于第 4 章的可特征化任务特征类型。

## 3. 任务特征集合来源

```text
任务特征集合
= 任务领域固定治理特征集合
∪ 来源需求锚定特征集合
∪ 当前激活方法声明的任务业务特征集合
∪ 当前激活组合方法声明的协作特征集合
```

任务领域固定治理特征只能由任务领域规范定义。来源需求锚定特征必须能够回到来源需求、活动路径或授权事实。方法业务特征和组合协作特征必须由对应方法正式声明。

任务不得因为“本轮需要”而直接创建新业务特征类型。需要新增特征时，必须先由合法方法或任务上级规范声明，再按 0500 完成特征类型准入。

## 4. 固定任务特征类型总表

### 4.1 承接、路径、权重和授权

| 特征类型 | 值语义 | 合法来源 |
| --- | --- | --- |
| 任务承接角色 | 管理、执行、聚合、逻辑组织等具名角色；不得由有无子节点单独决定 | 来源需求语义、活动路径和任务领域裁决 |
| 目标当前性 | 当前、已过期、待复核等强类型值 | 来源需求目标及相关版本 |
| 授权当前性 | 有效、失效、撤销中等强类型值 | 任务授权关系及授权版本 |
| 活动 OR 路径状态 | 所属替代路径及其当前性 | 来源需求的活动 OR 路径 |
| 所属 AND 组状态 | 所属共同成立组及其当前性 | 当前 OR 路径内的 AND 组 |
| 继承父权重 | 当前 OR 路径继承的完整父权重 | 来源需求权重版本 |
| AND 任务份额 | 当前 AND 组内分配给本任务的份额 | AND 成员数量和分配版本 |
| 冻结结算预算 | 本任务本轮最多可结算预算 | 路径权重、AND 份额和冻结版本 |
| 调度优先级 | 可排序的强类型优先级值 | 来源需求优先级和任务治理规则 |
| 时限状态 | 未到期、临近、到期、无时限等强类型值 | 来源需求时限与当前时间事实 |
| 当前业务状态段 | 当前完整状态特征集合的状态段身份 | 任务领域状态裁决 |
| 状态段当前性 | 当前、历史、失效 | 状态段版本与当前发布关系 |

### 4.2 筹办治理

| 特征类型 | 值语义 | 合法来源 |
| --- | --- | --- |
| 筹办轮次 | 单调递增的本任务筹办轮次 | 任务领域开始筹办入口 |
| 筹办触发类型 | 初次承接、子目标回流、等待解除、执行后未达成、方法或事实版本变化等 | 正式触发事实 |
| 筹办占用状态 | 未占用、有效占用、释放、失效；占用身份不得是裸线程编号 | 任务级筹办执行权入口 |
| 筹办读取当前性 | 本轮读取的任务、需求、路径、授权和方法版本是否仍当前 | 权威重读与版本比较 |
| 路径查找状态 | 已形成路径、需子目标、需因果补齐、无确认路径等强类型结果 | 需求、因果和方法能力读取 |
| 候选集合完备状态 | 未开始、形成中、已完整、版本失效 | 方法召回结果及登记版本 |
| 方法选择当前性 | 未选择、当前、已失效、待切换 | 任务方法选择记录及方法版本 |
| 方法特征契约当前性 | 未挂载、已挂载、已验证、已失效 | 方法声明和挂载记录 |
| 筹办结果类型 | 可执行冻结、子目标承接、能力缺口承接、合法等待、目标已完成、授权失效 | 本轮筹办闭合裁决 |

### 4.3 就绪、条件和输入闭合

| 特征类型 | 值语义 | 合法来源 |
| --- | --- | --- |
| 前置条件满足状态 | 完整满足、部分满足、未满足、证据不足 | 当前方法条件与世界状态比较 |
| 输入闭合状态 | 已闭合、缺输入、输入过期、输入冲突 | 输入特征集合及当前值 |
| 参数绑定状态 | 已完整绑定、部分绑定、不可绑定、版本失效 | 方法参数契约与任务上下文 |
| 禁止项命中状态 | 未命中、已命中、待复核 | 方法禁止项、安全约束和当前事实 |
| 验证合同状态 | 已形成、缺验证特征、版本失效 | 方法验证特征声明 |
| 归因合同状态 | 已形成、缺归因特征、版本失效 | 方法归因特征声明 |
| 执行就绪裁决 | 未裁决、不可就绪、已就绪、当前性失效 | 条件、输入、参数、禁止项和验证合同联合裁决 |
| 执行冻结当前性 | 未冻结、当前、已失效 | 任务、方法、输入、授权和路径冻结版本 |

### 4.4 等待与重新触发

| 特征类型 | 值语义 | 合法来源 |
| --- | --- | --- |
| 等待类型 | 等待子目标、能力补齐、条件、输入、外部事实、授权或时间 | 筹办闭合结果或执行后合法变化 |
| 等待条件当前性 | 未满足、已满足、已失效、待复核 | 等待条件及其合法生产者 |
| 重新触发状态 | 未触发、已触发、触发失效 | 正式子结果、状态动态或授权变化 |
| 等待版本 | 单调变化的等待合同版本 | 任务领域等待提交入口 |

等待对象、缺失特征集合、合法生产者、阻塞来源和重新触发条件由正式节点、关系和特征集合承载，不压成自由文本原因。

### 4.5 执行、结果和完成

| 特征类型 | 值语义 | 合法来源 |
| --- | --- | --- |
| 执行轮次 | 单调递增的本任务执行轮次 | 合法执行开始入口 |
| 执行输入当前性 | 当前、过期、冲突、待复核 | 冻结输入和执行前重读 |
| 方法运行当前性 | 未开始、运行中、已结束、已失效 | 当前方法运行事实 |
| 执行结果类型 | 产生可比较结果、仅有动作事实、需要附带结果、不需要附带结果、禁止结果、条件变化、输入过期等 | 方法输出、状态和动态事实 |
| 目标达成裁决 | 未裁决、未达成、已达成、证据不足 | 实际结果与任务目标比较 |
| 本轮推进裁决 | 未裁决、无推进、有推进、方向相反、不可比较 | 本轮前后状态与目标方向比较 |
| 本轮推进量 | 与目标值域一致的有量纲变化量 | 前后状态比较 |
| 剩余差距 | 与目标值域一致的有量纲差距 | 当前实际状态与目标状态比较 |
| 验证裁决 | 未验证、验证通过、验证不通过、证据不足 | 方法验证特征和实际证据 |
| 归因状态 | 已归因、待归因、多来源、证据不足 | 方法、动作动态和因果证据 |
| 禁止结果状态 | 未出现、已出现、待复核 | 禁止结果集合与实际结果比较 |
| 附带结果路由状态 | 无附带结果、待路由、已路由、路由拒绝 | 非当前目标的有效结果及对应需求 |
| 结果当前性 | 当前、已被后续事实替代、已失效 | 实际结果版本和世界状态版本 |
| 目标达成比例 | 有边界的目标满足比例，不得替代逐项证据 | 目标特征集合与实际结果集合比较 |
| 完成当前性 | 未完成、当前完成、完成证据失效 | 目标达成、验证和结果当前性联合裁决 |
| 结算候选状态 | 未形成、已形成、已提交、已拒绝 | 合法完成事实和路径预算 |
| 需求满足记录投影 | 未形成、已形成、来源失效 | 5160 已发布需求正式结算记录及其当前性 |
| 价值结算投影 | 未入账、部分入账、全部入账、价值证据失效 | 4320 的冻结预算、同源归因、价值结算事实及其当前性 |

`需求满足记录投影` 和 `价值结算投影` 都是任务领域按上游权威事实形成的读取投影，不是任务专有的第二份结算记录。需求正式结算记录只允许 5160 定义的 `已满足` 结论；“部分入账”只描述 4320 的独立价值预算消费，不得写成部分满足、部分完成或另一种需求正式结论。投影清空或重建不得改变需求满足记录、价值结算事实、冻结预算或防重复领取事实，任务领域也不得反向改写这些上游结构。

### 4.6 失败、取消和终止

| 特征类型 | 值语义 | 合法来源 |
| --- | --- | --- |
| 业务失败分类 | 合法失败条件的具名强类型分类 | 方法失败条件、禁止结果或授权治理规则 |
| 失败证据当前性 | 当前、失效、待复核 | 实际结果、动态和失败条件版本 |
| 取消当前性 | 未取消、已授权取消、取消完成 | 取消授权关系及版本 |
| 动作停止确认 | 无动作需停止、停止中、已停止、停止证据不足 | 执行动作和停止动态 |
| 终止版本 | 失败或取消终止事实版本 | 任务领域终止提交入口 |

内部逻辑错误不属于业务失败分类，禁止通过这些特征写成任务失败。

## 5. 方法声明的开放任务特征

方法可以声明任务专有业务特征，但必须给每个特征指定以下角色之一：

```text
输入
输出
验证
归因
安全与权限
失败处理
结算支撑
组合协调
```

开放特征不进入固定全局枚举。任务只挂载当前选择方法或组合方法正式声明的特征类型。

每条方法特征挂载记录至少包含：

```text
任务身份
特征类型身份
来源方法身份
来源方法内容版本
特征角色
激活原因
筹办轮次
是否必需
是否已填值
是否已验证
当前性
```

## 6. 初始化顺序

任务根据需求初始化时按以下顺序形成：

1. 创建或幂等复用任务节点。
2. 建立来源需求、唯一运行态治理存在、场景和目标状态关系。
3. 建立初始生命周期状态及版本。
4. 初始化第 4.1 节中能够由来源需求确定的固定治理特征。
5. 初始化来源需求锚定特征，且不得用任务默认值覆盖来源需求明确值。
6. 当前方法尚未选择前，不挂载方法开放特征，不创建无来源的独有特征槽。
7. 方法选择后，按第 5 章建立挂载记录并激活方法声明特征。

## 7. 方法切换

方法切换必须保留旧方法选择记录、旧挂载记录、旧特征和值、方法运行引用、验证和归因证据。旧特征标记为历史或失效，新方法特征完整挂载并验证后才成为当前集合。

不得原地覆盖方法版本，不得把旧方法业务特征继续当作新方法的无来源当前特征。
