# 4010 子规范 统一仓库稳定句柄与通用关系索引边界

编号：4010
日期：20260721
状态：正式子规范

## 1. 定位

本规范统一定义节点、关系和索引所依赖的仓库身份、稳定句柄、记录状态、节点 / 关系数值 ABI，以及逻辑树图在关系仓库和索引仓库之间的职责边界。各领域对象的业务身份、专用关系角色、基数、状态和写入规则仍由对应对象根规范与主题子规范唯一规定。

直接上级规范：0050。

直接依赖规范：0500。

相关边界规范：4020、4030。

## 2. 统一仓库与逻辑空间

运行期核心共享以下底层结构职责：

```text
主信息仓库 -> 主信息身份、生命周期和受控值容器
节点仓库 -> 节点身份、节点类型和主信息锚点
关系仓库 -> 稳定节点之间的树图关系
索引仓库 -> 命名域内键到节点候选的定位
```

世界、语素、需求、任务、方法、特征、状态、动态和因果等逻辑空间不得为自身建立不兼容的句柄体系或平行事实仓库。逻辑空间由节点类型、正式根 / 入口关系、关系类型、索引所有者与命名域以及领域服务入口共同定位；空间标签、文件目录、线程、日志、显示名或索引命中不得单独裁决空间归属。

容量、分片、冷热分层、纪元或归档需要扩展时，可以增加仓库编号或物理分区，但不得改变稳定句柄含义，也不得让任一逻辑空间取得独立于统一结构的事实裁决权。

## 3. 稳定句柄与编号

### 3.1 句柄组成

长期结构身份统一使用完整值式句柄：

```text
节点句柄 = 仓库编号 + 节点编号 + 版本号
主信息句柄 = 仓库编号 + 主信息编号 + 版本号
关系句柄 = 仓库编号 + 关系编号 + 版本号
```

规则固定为：

1. 仓库编号和本地编号使用无符号 64 位身份语义，`0` 为无效编号。
2. 版本号使用无符号 32 位身份语义，`0` 为无效版本。
3. 同类句柄只有三个组成部分全部相等时才表示同一版本身份；不得只比较本地编号。
4. 同一仓库内本地编号从非零值单调分配，不得因删除、失效、恢复或重建而复用。
5. 版本只表达生命周期当前性或同一结构记录的合法换代，不承载普通业务值变化、评分、阶段或统计。
6. 裸指针、容器位置、线程编号、锁地址、文本键和显示名称只能作短生命周期辅助，不得替代稳定句柄。
7. 编号空间接近工程上限时必须先形成分仓、分片、纪元或归档设计；禁止回绕、截断或复用编号。

### 3.2 记录状态 ABI

记录状态数值是快照、事件、恢复和跨版本读取的兼容边界：

| 数值 | 记录状态 | 通用含义 |
| --- | --- | --- |
| 0 | 无效 | 尚未形成合法记录，或结构化结果的拒绝默认值 |
| 1 | 有效 | 当前普通业务读取可见 |
| 2 | 已删除 | 身份已进入删除生命周期；普通读取不可见 |
| 3 | 已失效 | 历史关系或换代记录仍可审计，但不参与当前业务裁决 |

这些数值只能在末尾追加新值，不得重排、复用或改名复义。恢复材料携带未知状态值时必须在发布前拒绝。

## 4. 节点类型数值 ABI

节点类型只分类节点身份，不证明对象已经具备完整业务结构、处于某个状态或完成某项能力。

| 数值 | 节点类型 | 分类边界 |
| --- | --- | --- |
| 0 | 未分类 | 无合法业务类型；不得作为正常业务节点发布目标 |
| 1 | 语素 | 语言入口类节点 |
| 2 | 基础信息 | 尚未拆入更具体类型的通用基础信息身份 |
| 3 | 存在 | 存在对象身份 |
| 4 | 场景 | 场景对象身份 |
| 5 | 特征 | 特征定义或特征槽身份 |
| 6 | 特征值 | 特征系统内部值承载身份 |
| 7 | 需求 | 需求对象身份 |
| 8 | 任务 | 任务对象身份 |
| 9 | 方法 | 方法及其正式角色身份 |
| 10 | 动态 | 实例或抽象动态身份 |
| 11 | 因果引用 | 轻量因果引用或相应因果身份入口 |
| 12 | 状态 | 实例或抽象状态身份 |
| 13 | 二次特征 | 二次特征身份 |
| 14 | 任务方法选择记录 | 一次任务方法选择的专用记录身份 |
| 15 | 用途观察记录 | 一次用途观察证据账的专用记录身份 |

节点类型数值一经进入代码、序列化、快照、事件或恢复材料即永久锁定，只能在末尾追加。废弃类型只能停止产生新记录，不能把原数值改作其它语义。

类型本身不得被解释为需求满足、任务完成、方法可执行、因果稳定或用途观察已经晋级。类型 14 的完整身份与选择语义由任务 / 方法规范负责；类型 15 的完整证据账与阶段语义由用途观察规范负责。

## 5. 关系类型数值 ABI

关系类型只规定结构分类；端点类型、角色、基数、顺序号和值域必须同时服从该业务关系的唯一主题规范。

| 数值 | 关系类型 | 通用结构用途 |
| --- | --- | --- |
| 0 | 普通父子 | 普通树形层级与通用重挂 |
| 1 | 归属 | 对象对场景、存在、需求或容器的归属 |
| 2 | 引用 | 非拥有的稳定对象引用 |
| 3 | 索引 | 索引项与目标的结构连接 |
| 4 | 模板 | 模板、定义或规格引用 |
| 5 | 运行期临时 | 实例状态、实例动态等运行期归属 |
| 6 | 因果来源 | 原因、结果、动态和值材料的来源联系 |
| 7 | 语素对应信息 | 语素入口到对应信息的多目标联系 |
| 8 | 语素概念追溯 | 语素入口到概念的多级、多路联系 |
| 9 | 实例支持概念 | 实例对兼容概念的支持联系 |
| 10 | 概念上下位 | 同类概念的直接上位到下位联系 |
| 11 | 概念定义材料 | 概念到结构化定义材料的联系 |
| 12 | 概念生命周期 | 概念到当前生命周期状态的联系 |
| 13 | 需求目标概念 | 需求到目标概念的联系 |
| 14 | 任务生命周期 | 任务到生命周期实例状态的联系 |
| 15 | 任务生命周期证据 | 生命周期状态到迁移证据角色的联系 |
| 16 | 任务方法选择 | 任务到任务方法选择记录的联系 |
| 17 | 方法动作场景 | 动作入口方法到输入 / 输出场景角色的联系 |
| 18 | 用途观察证据角色 | 用途观察记录到具名证据与阶段角色的联系 |

关系类型数值遵守与节点类型相同的只追加、不重排、不复用和不改名复义规则。关系 14—18 的顺序号属于专用角色合同；本规范只锁定类型数值，具体角色和值域分别由任务、方法和用途观察主题规范唯一拥有。

## 6. 节点与关系通用结构边界

### 6.1 节点边界

节点记录只承载节点编号、节点类型、主信息锚点、版本、记录状态和创建顺序。节点发布前必须复核主信息句柄当前有效；节点类型不得替代领域对象的完整结构准入。

节点删除必须推进生命周期版本并使旧句柄退出普通读取。已删除记录只能由显式审计或恢复治理入口读取，不得重新成为当前业务事实，也不得回收原节点编号。

### 6.2 关系创建

关系创建必须在任何写入前复核：

```text
关系类型已知
源、目标为完整当前节点句柄
端点记录有效且类型符合业务关系合同
写入方具有该关系类型的领域授权
顺序号或角色符合专用规范
重复、唯一性、多目标和成环约束均成立
```

完全重复关系不得产生第二条有效当前记录。某关系是否允许同源同类型多目标、同目标多角色或下位多父，只能由具名关系规范允许；未明确允许时不得用多条关系模拟列表字段。

关系 7 和 8 已确认是集合 / 图读取语义，读取不得静默选择第一个目标。其它专用关系即使物理上可形成多条记录，也必须先满足其端点、角色和基数合同。

### 6.3 普通读取、审计与失效

普通关系读取、存在判断、按源 / 目标枚举、节点相关关系枚举和有效计数只返回当前 `有效` 记录，并复核完整关系句柄、版本、类型和有效端点。按源读取目标、按目标读取来源和按节点读取相关记录均返回值式副本，不暴露仓库内部容器。

显式审计入口可以读取当前版本的有效、已失效或已删除记录，但审计结果不得进入普通业务裁决。关系失效时保持关系编号、类型、端点和顺序号，状态由有效推进为已失效，版本单调增加并返回新句柄；旧句柄随版本推进失效。已失效记录不得原地恢复为有效，同一端点后续再次形成真实关系时应创建新的有效关系。

重复提交刚失效前的关系句柄或该次失效形成的当前审计句柄时，必须幂等读回同一当前审计结果，不得再次推进关系版本、生成第二条失效历史或改变普通读取结果。其它旧版本句柄不得越过当前版本复核取得失效写权。

### 6.4 普通父子重挂

节点移动统一解释为 `普通父子` 关系的创建或重挂。通用入口必须保证：

1. 节点和新父节点当前有效且不相同。
2. 同一子节点同一时刻至多一个有效普通父节点。
3. 重挂后父链不形成自环或已知环。
4. 重挂复用原关系编号、推进版本并返回当前关系句柄；旧版本句柄退出普通读取。
5. 领域服务只执行本领域前置拒绝并调用通用入口，不得复制另一套移动算法。

专用关系不得默认使用通用重挂。只有其唯一主题规范明确允许并给出版本、基数、审计和撤销合同后，才能进入专用值式变更。

## 7. 索引边界

索引只保存命名域内物理键到节点候选的定位记录，不承载节点存在、关系拓扑、业务完成或领域结论。索引命中后必须回读并复核：

```text
索引所有者和命名域
键格式与探测规则版本
目标完整节点句柄
节点类型、版本和记录状态
必要关系与领域当前性
正向绑定与反向绑定一致性
```

不同语义的词面、需求编号、任务编号、方法键、特征键或观察身份不得混入无类型的共享键空间。正式索引写入必须具有具名所有者、非零命名域、键格式 / 探测版本、冲突处理和删除保留策略；未知兼容索引只能作为待迁兼容路径，不能成为新领域索引的准入方式。

索引缺失、清空或重建不得改变权威结构事实。悬空候选、正反绑定不一致、错类型、错版本或同一唯一键指向异义目标属于结构不一致，不能以“未命中”掩盖。索引的哈希、探测次数、二级候选或缓存布局属于实现设计；只有所有者、命名域、候选复判和事实非权威边界属于本规范。

## 8. 持久化兼容与恢复

快照、事件和恢复材料必须保存完整句柄、记录状态、节点 / 关系数值类型及其格式版本。恢复准入必须先拒绝：

```text
零仓库编号、零本地编号或零版本
未知或被复义的记录状态
未知节点类型或关系类型
编号重复、回退或超出下一编号边界
节点主信息锚点无效
关系端点无效或版本不一致
索引候选无法通过权威结构复判
```

恢复材料只是候选；只有经当前结构不变量和领域不变量复核并通过正式发布入口后，才可进入运行期当前事实。序列化格式、物理容量和恢复算法由实现设计规定，不得反向改变本规范的数值 ABI。

## 9. 验证合同

实现或恢复验证至少覆盖：

1. 三类句柄的零值拒绝、完整比较、编号单调和旧版本失效。
2. 记录状态、节点类型 0—15、关系类型 0—18 的精确数值和未知值拒绝。
3. 节点主信息锚点、关系有效端点、普通父唯一性与无环重挂。
4. 多目标关系返回完整集合，未授权多目标在写前拒绝。
5. 普通读取不返回已删除 / 已失效记录，审计读取不反写当前事实。
6. 索引命中后的正反绑定、句柄、类型、版本和领域关系复判。
7. 快照或恢复遇到未知枚举、悬空关系和索引不一致时不发布半结构。

验证必须读回结构记录与完整句柄，不得以日志、显示文本、索引命中或函数成功码替代。
