## 2. 语言材料与文章层级

1. 词、语素、子句、自然句、段落和文章分别承载对应语言层级，不得用一段文本同时替代全部层级身份。
2. 文章由有序段落组成，段落由有序自然句组成，自然句由有序子句组成；组成、顺序和来源必须由正式关系承载，不得依赖数组地址、文件行号或显示缩进作为长期身份。
3. 词表达语言单位材料；语素入口连接外部语言表达与内部已理解信息。二者都不直接成为存在、世界状态、需求或因果事实。
4. 文本内容、编码位置、标点和语言层级是材料事实；概念、指代、语义角色和内部信息对应必须经过理解准入后另行建立。
5. 同一语言材料可以对应多个内部信息节点，同一内部信息也可以有多个语言入口；不得迁回单目标字段。
6. 语言层级的删除、重排或修订不得自动删除已经独立成立的内部事实；其引用关系必须按生命周期显式失效或重挂。

## 3. 语素入口与理解准入

正式入口顺序保持：

```text
接收语言 / 外部材料
-> 建立或复用语言层级材料
-> 完成分词、指代、上下文和语义候选处理
-> 由理解入口验证对应内部信息已存在或具备合法创建路径
-> 建立语素入口到一个或多个内部信息节点的正式关系
-> 返回可追溯的理解结果或拒绝材料
```

1. 未理解文本只能保留为材料，不得进入正式语素对应关系。
2. 字符串相同、词形相同或名称相同不等于概念、存在或信息身份相同；复用必须由正式身份和上下文证据裁决。
3. 指代解析结果在上下文不足时只能保留候选或空，不得强制绑定最近节点。
4. 语素入口不得直接绑定特征值节点作为长期通用语义；需要表达值时必须先经特征系统形成有定义、有主体、有场景的正式信息结构。
5. 语素到对应信息、概念追溯、上位概念和概念模板用途都是多对多、多级、多路关系，不得压成单值主信息字段。
6. 文本、显示名和说明只能用于人读表达或候选检索，不得成为机器路由、类型或事实判断依据。

## 4. 存在概念与实例创建闭环

1. 存在概念表达可复用类别，存在实例表达可持续引用的具体主体；概念与实例必须有不同稳定身份和显式归属关系。
2. 创建实例前必须具有合法概念、主体材料、来源证据和必要初始特征；缺材料时返回候选不足或拒绝，不得创建空壳后补写。
3. 创建成功必须能从概念读到实例、从实例回到概念，并能读取实例稳定句柄和来源；单向插入或只写显示名不构成闭环。
4. 同一实例的重复创建请求必须按正式稳定键、概念和来源材料复核后复用；名称相等不构成幂等键。
5. 存在根概念的元特征定义“属于存在概念体系”的正式边界，不是普通实例名称，也不得由用户界面分类根替代。
6. 根概念、普通概念和实例的层级、支持和生命周期必须由关系与服务入口治理；不得直接改节点父指针或依赖树形显示。

## 5. 存在作为特征值中的身份材料

1. 当某个特征的合法值域是“一个存在”时，值必须承载可验证的存在稳定句柄，而不是裸指针、地址、名称、临时编号或复制的存在对象。
2. 该句柄值仍属于对应特征值域；创建、比较和读取必须同时验证特征定义与被引用存在的当前生命周期。
3. 存在句柄相等只说明身份相同，不说明两个存在的全部特征、状态或语义相同。
4. 被引用存在失效、退役或删除时，引用必须按正式生命周期处理；不得让悬空句柄继续成为合法当前值。
5. 海中鱼巣使用 `节点句柄` 及正式关系表达该语义，不复制鱼巢裸指针字段。

## 6. 信息入口类型与存储路由

1. 信息入口首先区分材料类型和理解状态，再裁决进入语言材料、基础信息、需求、任务、方法或其它已存在的正式角色；不得先按目录树或旧主信息枚举硬路由。
2. “基础信息”是特征、存在、场景、抽象状态、抽象动态、二次特征、因果引用等基础信息类节点集合，不等于单一 `基础信息` 节点类型。
3. “高级信息”不是新节点类型，只指需求、任务、方法等既有高级业务角色；禁止新增同名通用节点容器。
4. 实例状态和实例动态是场景内临时零散事实，不得直接作为长期语素入口对应信息；需要长期表达时必须对应抽象状态、抽象动态或其它长期信息节点。
5. 已理解入口可以绑定已有基础信息类节点、需求、任务或方法节点；任何新建仍必须调用对应领域服务，不得由语素服务直接写目标仓库。
6. 信息存储树只作领域关系与查询视图，不得成为统一物理容器、地址所有者或跨领域裸写入口。
