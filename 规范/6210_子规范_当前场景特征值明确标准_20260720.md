# 6210 子规范 当前场景特征值明确标准

编号：6210
日期：20260720
状态：正式子规范

## 1. 定位

本规范规定“当前场景特征值明确”的准确含义、对应闭合标准、局部扫描启动、首次基准和后续变化边界。

上级规范：0050、1130、1140、1150、1160、1170、6200、6300、6370。

## 2. 正式含义

历史名称“当前场景特征值明确”在本层只表示：

```text
当前限定观察范围内的全部有效可观测单位，
已经稳定对应到存在。
```

它不是特征具有安全意义、全世界对象已识别、全场风险明确或所有存在状态已更新。

## 3. 对应结果

每个有效可观测单位必须形成下列过程结果之一：

- 稳定对应已有存在；
- 稳定对应可提交的新存在候选；
- 多重对应冲突；
- 证据不足，尚未稳定对应。

冲突和证据不足只能是缺口，不能作为最终闭合归属。可观测的地面、墙面、桌面、介质和固定支架应对应静止、环境或低动态存在，不得用“背景”或“未知区域”替代归属。无效、不可观测材料在材料层保留原因，不进入有效单位完成计数。

## 4. 闭合条件

```text
已稳定对应单位数 == 有效可观测单位数
AND 未稳定对应单位数 == 0
AND 多重对应冲突单位数 == 0
AND 稳定对应存在数 > 0
```

有效单位数为零时不能直接写全场明确；只能记录无有效材料或当前范围为空，是否为空场景明确由观察范围、观察能力和覆盖事实另行裁决。

对应事实由正式提交入口消费识别输出、观察材料和过程证据后写入。识别方法本身只返回已有存在稳定身份或空/候选材料，不写对应闭合事实。

## 5. 局部启动与全量闭合分离

不需要等待全部有效单位闭合后才启动扫描。只要至少一个存在已经稳定对应，就可以按范围、ROI、外设视野、来源目标、频率、跟踪接管和优先级生成本轮扫描关注集合。

局部扫描成功只证明该存在和本轮材料的当前状态可处理，不得扩大为全场特征明确。未稳定部分继续走缺口解释或识别补充，已稳定部分可以并行扫描。

## 6. 扫描输入与未覆盖解释

扫描最低输入是当前自我场景稳定身份、已稳定对应存在集合、内部场景基准和已合法进入系统的外设当前数据集。

```text
本轮未覆盖存在 = 本轮应覆盖存在 - 本轮外设覆盖存在
```

范围外、视野外、低频跳过、跟踪排除、优先级不足、缺坐标、缺基准、缺值域或材料不可回查的存在，应分别记录原因，不得直接写消失或遮挡。

扫描只能使用已经由观察方法或提交入口形成的合法外设数据项。原始外设材料尚未合法承接时，只能输出结构化材料缺口。

## 7. 首次基准与后续变化

首次扫描没有历史可比材料时，只建立已确认存在的当前特征值和状态基准，不生成发现语义或变化动态。

只有存在上一轮、历史基准或其它可比观察材料，且值发生可验证变化时，才可提交特征变化事实和存在状态动态。

## 8. 写入边界

对应事实、当前状态事实、扫描变化事实和跟踪事实由自我侧对应提交入口分别裁决。它们不得写需求满足、安全值、服务值或风险明确。

任务管理只按结构化目标和已入树需求形成等待项，不按目标文本或路径名称猜测外设动作。

## 9. 验证

验证覆盖范围限定、四类对应结果、零有效材料、局部扫描、首次基准、未覆盖差集、识别不写对应事实和场景明确不等于风险明确。
