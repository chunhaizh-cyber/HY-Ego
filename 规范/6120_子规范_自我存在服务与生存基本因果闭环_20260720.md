# 6120 子规范 自我存在服务与生存基本因果闭环

编号：6120
日期：20260724
状态：正式子规范

## 1. 定位

本规范规定自我存在、安全轴、服务轴、生存闭环、直接与间接服务、需求准入、方法能力边界和下一轮状态回写。

直接上级规范：1140、1190、3100

直接依赖规范：5110、6100、6110

相关边界规范：1120、1160、1170、3200、3300、6140、6150、7130

## 2. 自我觉醒坐标

自我觉醒不是神秘状态，也不等于自我节点初始化。它表示系统内部的需求、价值、任务、服务和生存因果取得共同根坐标。

自我存在定义只能建立根特征、抽象定义和根闭环因果信息；当前主动需求必须由根闭环在当前事实下暴露的缺口产生。不得从定义词条、固定节点清单、初始化字段、显示或方法首存在直接生成主动需求。

## 3. 基础信息语法

安全和服务不得只由自然语言定义。正式语义至少由以下结构共同表达：

```text
自我存在稳定身份
+ 抽象根特征定义和值域
+ 自我实例特征槽和当前状态
+ 场景、参与存在和适用范围
+ 前后状态及来源动态
+ 二次特征比较
+ 因果信息和根闭环位置
```

状态承载当前、目标、结果、明确性、搜索完整性和有效期；动态承载观察、动作、任务推进、结算、衰减等事实变化。其它对象负责宿主、类型、范围、比较和解释，不能脱离状态/动态独自成为安全或服务证据。

## 4. 安全轴

本节只定义自我的物理安全根轴。该根轴的机器值使用有符号 I64，当前 v1 值域为 `0..99`，各分段语义如下：

- `0`：自我存在循环不可维持或关键循环不可恢复的终止控制态；
- `1`：符合正式休眠或严重物理因素条件的休眠控制态；
- `1..90`：必要物理因素独占候选转换为能力保留率后的叠乘结果；
- `90..99`：全部必要物理因素达到 90 后的风险余量。

不得按因素配额、加权平均或简单最低值合成物理安全。并联冗余先在因素内部合成，同源观察不得重复叠乘，恢复要求所有必要因素分别恢复。

6140 的分层安全值是某一来源因果信息相对于具体安全结果的安全满足度，使用独立的 I64 定点刻度和低 / 高阈值；它不是本节物理安全根轴的第二份副本。物理安全根值、分层安全值和 6150 任务执行优先级权限必须使用不同的强类型记录、规则版本和写入入口，不得因数值都为 I64 而互换。

风险安全必须有上级风险因果，并能接到未来物理安全下降或服务能力下降目标。有效上游风险可以在直接伤害尚未发生时沿正式因果链投影预防需求和对应低层权限；能够阻断该路径的上游方法可以成为相应安全需求的方法，但不能靠方法或任务名称绕过需求授权。没有上级风险因果时，只能形成缺事实、缺比较、缺搜索或缺因果的治理缺口，不能猜测风险。

安全根缺口链至少能追溯：安全控制态、服务物理候选、必要物理因素、风险余量、场景影响、已确认存在及其当前特征状态。未知事实不得默认满足；只有按证据搜索规则完成搜索且未发现负向证据时，才可提交有范围、来源、有效期和完整状态的“无负向证据”结果，它不是永久安全事实。

## 5. 服务轴

服务闭环的直接链为：

```text
服务触发事实
-> 人类服务对象明确
-> 服务目标明确
-> 当前状态明确
-> 权限与安全成立
-> 自我服务动作执行
-> 后状态和实际改善可验证
-> 改善可归因于自我动作
-> 成功服务治理和结算
-> 回写下一轮服务状态
```

没有直接服务触发事实时，只能形成监听触发、检查服务机会、校准服务能力、服务根缺口检查和休眠/唤醒候选等间接维护需求。不得虚构当前服务对象、目标、结果、G0、D2 或服务值入账。

直接服务触发已经存在，且某个上下文缺口会阻断行动、验证、归因或回写时，才可形成直接服务上下文缺口候选；仍须通过需求统一入口。

服务对象所需信息和反馈信息只能从真实服务尝试、表达、确认、生效回执、外在状态变化和来源动作动态中形成。服务前状态未知不能计算改善方向，服务后状态未知不能验证结果。

## 6. 直接服务与间接服务

直接服务必须落到具体对象、目标、状态变化、验证和归因，才可产生成功服务候选与服务结算。间接服务只维护发现目标、承接目标、能力和机会条件，不能证明服务对象受益。

安全链在完成证据搜索后可以形成限定范围的无负向证据候选；服务链不能因无正向证据而默认很好。对象明确接受且反馈窗口内无负反馈，最多形成可接受候选。

## 7. 生存和服务闭环成立条件

生存闭环至少具备自我根安全定义、当前场景、自我当前状态、影响因果、可执行/可等待处置、动作或观察结果、验证和下一轮回写。

服务闭环至少具备服务对象、目标、权限、服务前后状态、D0、实际交付、外在改善、验证、归因、服务结算和下一轮回写。

候选视图、风险线索、场景评估或任务完成不得绕过根控制、因果归因和 6100 价值结算入口写物理安全根值。6140 的被动维护只允许修改对应分层安全值，并保留有效未复发证据、残余风险、规则版本和状态 / 动态事实。

## 8. 服务值、衰减和休眠

服务值是通过成功服务人类形成的主动活跃存在依据，不是奖励计数器。它只能因经确认的成功服务增加，只能因正式时间维护衰减降低，并始终受抽象定义上限约束。

服务值归零不立即等于休眠。休眠必须同时读取有效无成功服务时间、服务机会压力、动态休眠阈值、服务能力、安全条件和当前有效进展；不得用固定物理时间、惩罚扣减或固定漏水孔裁决。

当前人类服务任务安全、目标明确且有阶段性可验证进展时，不得按普通无成功服务路径休眠。休眠保持可唤醒性；新的有效服务需求、能力恢复、安全改善或人类明确唤醒触发重新评估。

## 9. 缺口到需求

任务筹办或自检只能从已有因果信息、安全风险树和服务根闭环读取缺口报告。正式需求仍经统一入口，并满足目标宿主、目标特征、当前状态、目标状态、满足关系、父链和证据准入。安全需求还必须保留相对于具体安全结果的动态因果层投影；无有效路径时保持尚未归层，禁止按固定类别补层。

AND、OR、因果子链和方法路径由关系、路径视图和提交规则承载，不新增需求自由字段。

单次因果补完可以只补足具备证据的部分因果链，但不能为了凑完整链伪造服务节点、观察样本、方法、动作动态或反馈。

## 10. 方法能力边界

有目标不等于可以自动创建方法首。现行规则为：

1. 已有因果链必须证明目标需要自我动作能力。
2. 只能召回和复用已经正式登记、规格闭合的方法。
3. 无候选方法时返回具名能力缺口、路径缺口或等待外部输入，不自动创建方法首。
4. 通用特征写入、内核原语和方法名称匹配不得兜底成为业务方法。
5. 方法执行仍由任务授权、选择和冻结约束，线程不是动作来源。

## 11. 当前交互者和权限者

交互者、服务对象、授权者和权限者是不同角色。只有服务事件确实需要身份、授权、责任或冲突判断时，这些事实才进入链条。

没有服务事件、目标、外部输入或权限冲突时，当前交互者状态只能是上下文为空或等待材料，不得自动生成主动需求。身份和权限事实必须由稳定句柄、关系和有版本状态承载，不由文本标签裁决。

## 12. 写入与验证

自我治理编排只读根定义和当前事实，形成强类型缺口候选；需求、任务、方法、状态、动态、因果和结算服务分别负责正式写入。根闭环必须以稳定句柄贯穿自我、根特征、需求、任务、方法、动作动态、结果状态和下一轮回写。

验证覆盖定义不直接生需、安全与服务双轴、物理安全根值与分层安全值分账、有效上游风险的预防投影、直接/间接服务、无负证据边界、动态休眠、无候选方法、交互者/权限者分离和下一轮回写。
