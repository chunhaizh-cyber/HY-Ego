# 6370 子规范 识别本能方法唯一性收敛

编号：6370
日期：20260720
状态：正式子规范

## 1. 定位

本规范规定识别方法在确定范围内用已有特征把候选集合收敛到唯一已有存在，或返回空句柄的完整规则。

上级规范：0050、1130、1140、1180、2100、3100—3300、6300、6360。

## 2. 识别语义

识别只回答：

```text
当前存在材料在确定范围内是否对应一个已有存在？
是 -> 返回该已有存在稳定句柄
否或证据不足 -> 返回空句柄
```

识别不是比较完所有特征，不创建存在，也不补写一阶特征。具体存在和抽象存在都可以成为已有候选，但必须具有合法稳定身份和当前可读特征。

同一性评分、候选数量、硬冲突数量、证据不足原因和补观察建议只属于本轮运行证据，不是识别的语义输出。

## 3. 当前识别范围

唯一性只能表达“当前识别范围内唯一”，不得表达“全世界唯一”。范围可以是自我所在场景、观察材料可见范围、ROI、服务目标区域、跟踪窗口或安全关注区域。

识别范围必须关联：

- 来源场景或观察材料集；
- 可选 ROI、掩码、空间范围或跟踪窗口；
- 有效期和版本；
- 范围内候选存在集合生成规则。

没有明确范围或范围已经失效时，只能返回空句柄和结构化范围缺口。

## 4. 合法输入

识别只读取：

- 当前存在材料及其已有特征；
- 当前范围内已有存在集合；
- 候选存在的已有特征；
- 合法比较值域和阈值；
- 当前材料可复验状态；
- 场景、范围和来源版本。

缺少特征时不得即时观察或补写。识别仍返回空句柄，并以枚举、掩码、状态或二次特征表达缺平面轮廓、缺掩码、缺稳定材料、缺候选特征或缺空间范围等原因。

## 5. 候选集合和唯一公式

```text
C0 = 当前识别范围内所有可比较已有存在

Cf = { e ∈ C0 | 当前存在材料的特征 f 与 e 的特征 f 在合法值域内 }

C = C0 ∩ Cf1 ∩ Cf2 ... ∩ Cfn

若 |C| = 1
且 硬冲突数量 = 0
且 用于唯一识别特征数量 > 0
则返回唯一候选稳定句柄
否则返回空句柄
```

候选过滤必须由精确特征类型、比较规则、阈值和范围形成，不能使用“看起来像”或自由文本评分裁决。

## 6. 单特征和组合特征唯一

一个合法特征足以在当前范围筛出唯一候选时，可以停止继续比较，但仍须完成最低反证检查。平面轮廓、可观测物理标记等可以成为单特征唯一依据；外设簇号、帧号、任务号和内部节点 ID 不可以。

一个特征不足时，可以组合平面轮廓、空间尺寸、子轮廓、掩码重合、深度或颜色摘要、可观测物理标记和空间上下文。最小唯一特征组至少保留：

- 特征类型集合；
- 每类比较结果和阈值来源；
- 候选集合收敛过程；
- 最终候选；
- 硬冲突验算。

该特征组是本轮识别证据，不得反写成目标存在的新一阶特征。

## 7. 比较值域

比较阈值只能来源于：

- 特征类型默认比较值域；
- 候选存在历史稳定值域；
- 外设测量不确定度；
- 当前任务容差；
- 合法外设学习结果；
- 安全或服务要求。

识别必须能够回读每个阈值的来源、版本和可信状态。无来源阈值不得参与候选过滤或硬冲突判断。

## 8. 候选召回顺序

候选召回应优先使用低成本、强区分的已有特征。通常按平面轮廓、空间尺寸、子轮廓、掩码重合、深度或颜色摘要、可观测物理标记、空间上下文或局部三维模型逐步收敛。

顺序可以按历史区分度、计算成本、材料质量和当前任务严格度调整，但必须留下强类型选择依据。空间位置主要用于范围过滤、反证和跟踪辅助，不应替代轮廓、尺寸或子结构成为普通唯一依据。

## 9. 唯一后反证

候选收敛为一后，至少检查：

1. 平面轮廓是否超过硬误差；
2. 掩码重合率是否低于硬阈值；
3. 空间中心或范围是否发生硬冲突；
4. 关键类别特征是否互斥；
5. 同一材料是否强匹配多个存在；
6. 同一存在是否被多个互斥材料强匹配。

有任一硬冲突时返回空句柄，形成冲突状态和复验缺口。反证只读取已有特征，不现场生成新观察材料。

## 10. 分支收束

| 情况 | 识别输出 | 后续材料 |
| --- | --- | --- |
| 候选数量为零，材料稳定可复验 | 空句柄 | 新存在承接或复验需求依据 |
| 候选数量为零，材料不稳定 | 空句柄 | 补观察依据 |
| 候选数量大于一且仍有特征 | 继续过滤 | 候选收敛证据 |
| 候选数量大于一且无更多特征 | 空句柄 | 补观察依据 |
| 候选数量为一且有硬冲突 | 空句柄 | 冲突复验依据 |
| 候选数量为一且无硬冲突 | 唯一候选稳定句柄 | 最小唯一特征组和验算证据 |

识别本身不创建后续需求。结构化依据经需求统一入口、任务承接和筹办后，才能形成观察、复验或新存在承接任务。

## 11. 完成条件

返回非空稳定句柄当且仅当：

```text
识别范围当前且已确定
当前材料可复验
候选召回完成
唯一候选数量 = 1
硬冲突数量 = 0
用于唯一识别特征数量 > 0
候选是当前可用的已有具体或抽象存在
```

强唯一物理标记可以减少多轮证据要求，但不能免除范围、可复验、唯一候选和无硬冲突。

## 12. 与扫描和跟踪的关系

扫描必须读取已建立的材料到存在对应，或取得非空识别结果。未归属材料不得由扫描直接生成状态变化；已归属部分可以继续，但不能把局部结果扩大为全范围闭合。

跟踪目标必须是已经唯一识别、已归属或具有可复验稳定种子的存在。候选不再唯一时，跟踪返回冲突或空信息，并形成复验依据；不得用局部跟踪稳定替代全场识别完成。

## 13. 禁止规则

1. 不得把识别写成全特征比对完成。
2. 不得在范围未确定时宣称唯一。
3. 不得把当前范围唯一扩张成全世界唯一。
4. 不得由识别补写颜色、坐标、轮廓或深度等一阶特征。
5. 不得把唯一但有硬冲突的候选写成已归属。
6. 不得把多候选写成弱归属或默认归属。
7. 不得把报告、ROI、掩码、点集或材料页当成存在真值。
8. 不得由识别结果反推扫描完成、跟踪完成、场景明确、安全明确、需求满足或价值可结算。
9. 不得用说明文本承载冲突、阈值或收敛过程。
10. 控制面板不得反写识别结论。

## 14. 验证边界

验证至少覆盖范围门禁、单特征唯一、组合唯一、阈值来源、硬冲突、空候选、多候选、结构化缺口以及扫描跟踪隔离。
