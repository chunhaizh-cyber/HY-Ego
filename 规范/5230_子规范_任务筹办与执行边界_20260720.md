# 5230 子规范 任务筹办与执行边界

编号：5230
日期：20260720
状态：正式子规范

## 1. 定位

本规范定义任务领域内建筹办最小方法、任务级唯一筹办执行权、闭合筹办结果、重复筹办、执行冻结和执行期错误边界。

上级规范：0050、0300、3100、3200、3300。

## 2. 筹办身份与执行权

1. 任务筹办是任务领域内建最小方法，由待筹办任务直接触发，不进入普通方法召回。
2. 并发隔离键是任务稳定身份。同一任务同一时刻只能有一个有效筹办占用和执行权。
3. 执行权必须由任务结构中的唯一有效占用事实、轮次、版本和当前性裁决，不持久化裸线程编号。
4. 未取得执行权的路径必须在读取方法候选前正常拒绝，不得参与召回、选择或冻结。
5. 两条路径同时进入同一任务召回、选择或冻结，属于内部并发规则破坏；不得以同义选择幂等成功掩盖。
6. 不同任务可以并发筹办，即使它们来源于同一根需求或选择同一方法。

## 3. 每轮筹办输入

每轮必须权威重读：

```text
任务身份与运行态治理存在
来源需求及目标
任务角色与活动路径
授权、权重和预算版本
活动子需求与等待合同
候选方法、方法内容版本和生命周期
方法条件、结果能力、动作入口和任务特征契约
当前世界状态、输入和禁止项
```

筹办可以重复进行。触发来源、轮次和读取版本必须单调、可追溯；旧冻结在相关事实变化后失效。

## 4. 闭合结果

任务筹办直接返回并提交本轮闭合结果，不建立独立持久化筹办回执。结果只能是：

| 结果 | 必须同时形成的结构 |
| --- | --- |
| 可执行冻结 | 当前方法、条件与输入绑定、授权、路径、方法版本、验证和归因合同 |
| 子目标承接 | 完整子目标草案、合法生产者、等待合同和回流条件 |
| 能力缺口承接 | 已定位的方法/动作/条件/参数/证据缺口及合法处理方 |
| 合法等待 | 等待对象、等待事实生产者、重新触发条件和当前状态 |
| 目标已完成 | 当前目标比较、验证和结果证据 |
| 授权失效或取消 | 授权证据、动作停止要求和终止边界 |

禁止返回无结构“无法解决、退回上层”。无候选方法时不得自动创建方法首，不得用通用写入逻辑兜底；必须形成合法等待或其它可定位闭合结果。

## 5. 路径和候选边界

1. 筹办先读取需求目标语义和活动子链，不能按叶子/非叶子或名称决定任务形态。
2. 逻辑组织、管理和路径节点先执行展开、聚合或路径选择，不直接进入普通方法候选。
3. 普通可任务化原子任务才按方法结果能力完整填充候选集，并从候选集中选择。
4. 筹办不直接写需求树；子目标和缺口草案交需求领域统一入口。
5. 被动、提交、结算和内部治理动作不进入主动候选集。

## 6. 执行冻结

冻结必须绑定任务、筹办轮次、方法、方法内容版本、动作入口、条件、输入、路径、授权、禁止项、验证、归因和失效条件。

执行开始前必须复核冻结当前性。任一绑定事实变化时，旧冻结失效并进入新筹办轮次，不得在旧冻结上就地修补。

真实外部动作开始前如需治理许可，许可只表达本次动作是否可开始，不替代筹办结果、任务状态或完成裁决。

## 7. 执行边界

执行期允许的正常分流包括：条件突然变化、输入过期、外部事实变化、输出未达目标、验证证据不足或副作用约束命中。它们必须形成强类型运行结果并按任务状态合同收束。

执行期才发现下列冻结前既有缺口，属于筹办漏检和内部逻辑错误：

```text
缺方法或方法已失效
缺动作入口
缺条件结果对或必要规格
目标语义不完整
任务结构、授权或提交协议缺失
运行态治理存在或写回入口损坏
```

不得将这些错误包装成普通需求、任务失败或合法等待，不得在执行期补造后继续。

## 8. 状态与动作来源

任务状态迁移由任务领域正式提交入口裁决。回执同步只属于非业务派生整理，不得改变任务状态、进入候选集或生成主链动作动态。

被动任务状态提交可生成动态证据，但动作语义是被动动作；线程和任务管理器不得裸生成动作动态。

## 9. 完成边界

方法运行成功、动作发生、来源需求当前满足、子任务完成和输出存在都不能单独证明任务完成。完成必须由任务自身目标结果、验证、禁止结果、归因和当前性联合裁决。
