# 6110 子规范 生存深层需求投影

编号：6110
日期：20260724
状态：正式子规范

## 1. 定位

本规范规定如何从物理不安全事件的上游深层原因投影出生存深层需求候选，以及候选的类型、层级、门控、优先级和入树边界。

直接上级规范：3100

直接依赖规范：5110、5130、5140、6100

相关边界规范：1190、4320、6140、6150

## 2. 定义

```text
生存深层需求 =
为了降低未来物理不安全事件发生概率，
从该事件的上游深层因果原因投影出的需求。
```

直接导致具体安全结果的一项有效直接原因处于精确因果深度 1；沿有效直接因果关系每增加一条边，精确深度增加 1。深层需求不是因为名称高级或预置类别，而是因为它的来源因果信息相对于该具体安全结果具有更长的有效直接因果路径。

## 3. 合法事实和结构承载

物理不安全事件、原因节点、时间顺序和变化证据由状态、动态和因果结构承载。持久层只保存方向明确、版本有效的直接因果关系；间接因果、相对具体安全结果的路径和精确深度都由当前有限因果图按需计算，不复制为固定层级事实。

层级投影身份固定为“来源因果信息 + 目标安全结果 + 因果图版本”。同一来源到同一结果存在多条有效路径时，以最短有效路径裁决精确深度和调度组，但全部有效路径及其精确距离仍可由权威直接因果结构重算；同一来源对不同安全结果可以具有不同深度。无有效路径时标记尚未归层，禁止按名称、任务类型或预置类别猜测。

精确因果深度使用非负 I64，业务上不设最大层数。深层投影只形成带来源因果、目标宿主、目标特征、候选类型、精确深度、图版本、可控性、证据状态和调度策略版本的值式候选。因果服务不得直接创建需求或任务；正式需求仍经需求统一提交入口。

派生置信分由因果基础计数、证据类型和查询参数计算，不得新增为因果、需求或任务的持久字段。

## 4. 投影方向

对会增加物理不安全事件概率或影响的深层原因，可以形成以下候选：

- 降低其发生概率；
- 降低其影响强度；
- 提高观测和确认能力；
- 提高控制和恢复能力；
- 建立隔离、预警、备份或替代路径。

协作、信誉、解释能力、维护关系、备份和治理等显示名称不能直接决定需求身份；只有它们能经因果链降低未来物理风险时，才属于生存深层需求候选。

## 5. 两类账

生存安全根值的主动变化只在物理状态真实改善或恶化并通过 6100 结算闸门时更新；6170 只允许在主动安全事实优先后形成服务调节的控制回归。深层原因风险下降通常只形成潜在生存贡献、未来风险下降、应急能力、恢复能力或可维护性改善。6140 的分层安全值可以按有效未复发证据和残余风险规则被动维护，但它不是生存安全根值，也不得反向证明现实改善。

禁止把完成解释、协作、维护或能力任务直接换算成物理安全增量。必须继续回查它是否降低了上游原因、未来故障、宕机或损坏风险。

## 6. 生成顺序

```text
发现物理不安全事件或风险上升
-> 反向追溯上游原因
-> 只读取当前版本有效的有向直接因果边
-> 针对具体安全结果计算全部有效路径和最短精确距离
-> 无路径则形成尚未归层结果
-> 判断可观察、可验证和可控制性
-> 可干预节点形成控制候选
-> 不确定节点形成确认候选
-> 不可控制节点形成隔离、预警、备份或替代候选
-> 形成需求候选并附带层级投影版本
-> 经需求统一入口入树
```

总顺序是先止损，再查因，再控因，最后治理深层结构。

## 7. 候选类型

| 类型 | 触发条件 | 目标方向 |
| --- | --- | --- |
| 确认型 | 原因仅是怀疑或证据不足 | 补齐当前事实和因果判断条件 |
| 控制型 | 原因已确认且可控制 | 降低原因发生或影响 |
| 预防型 | 同类原因长期或重复出现 | 建立稳定预防和监测机制 |
| 备份型 | 原因不可控或短期不可消除 | 隔离、备份、恢复、替代和预警 |
| 能力型 | 深层原因是自我能力不足 | 补齐解释、预测、停止、治理或学习能力 |

类型只决定候选目标方向和验证合同，不绕过目标状态、父需求和入树准入。

## 8. 精确层级和五级调度映射

因果链没有五层上限。精确深度始终保留，调度策略只把它映射到五个任务执行优先级组：

```text
精确深度 1 -> 组 1
精确深度 2 -> 组 2
精确深度 3 -> 组 3
精确深度 4 -> 组 4
精确深度 >= 5 -> 组 5
```

组号、精确深度和任务执行优先级权限是三个分别可审计的 I64 值。组 5 是调度终端汇总组，不是因果存储或推理截断；深度 6、20 或更深的需求均保留其精确深度。五级映射、层间权限释放、组内比较和版本合同由 6150 唯一规定，不再使用固定 `L0—L5` 类别或十进衰减刻度。

## 9. 门控

1. 物理不安全事件正在发生时，应急止损和能够阻断当前有效风险路径的手段取得近层权限；不得等待直接伤害再次发生才允许预防。
2. 低层安全值未高于低位阈值时，上层相关任务的正常执行优先级权限为零，但上层需求、任务、观察、筹办和材料接收可以继续存在。
3. 权限为零不取消需求或任务，也不允许上层任务靠名称、类别、固定表或任务自身字段绕过来源需求授权。
4. 某层达到明确饱和标准后，可以停止主动生成同义需求，但仍保留有有效期的监测和历史事实。
5. 深层原因不可控不等于无事可做；应转换为确认、隔离、备份、预警、恢复或替代候选。

## 10. 正式入树

候选进入需求树前必须具备目标宿主、目标特征、当前状态、目标状态或区间、满足关系、父需求和派生归因。

应急止损和近层安全原因挂安全根需求或对应安全父需求；深层确认、预防、备份和能力候选挂触发它的安全父需求或因果投影父需求。每条安全需求必须保留来源因果信息、目标安全结果、图版本、最短路径精确深度及全部来源路径引用，父链必须能追溯到具体安全结果。

## 11. 执行后验收

深层需求完成后，因果领域重新读取上游原因、路径状态和物理风险变化。成功证据可以增强路径基础计数，失败或竞争原因可以削弱或分化候选；不得直接改写因果结论、生存安全根值或分层安全值。上游方法阻断有效风险路径后，可以由正式事实、因果和 6140 维护入口重新结算对应层，而不是把任务成功直接换成安全值。

## 12. 禁止项

- 禁止因果服务直接写正式需求或任务。
- 禁止深层需求绕过低层执行权限门控进入正常任务竞争。
- 禁止把显示名称、自然语言标签或固定类型表当作深层因果证据。
- 禁止持久化间接因果或固定层号替代直接因果图。
- 禁止把五个调度组解释为因果深度上限。
- 禁止把深层需求完成直接结算为生存安全根值。
- 禁止把派生置信分持久化为因果节点通用字段或平行权威记录。

## 13. 验证

验证覆盖有向直接因果图、目标特定最短路径、无路径尚未归层、环与错向拒绝、非负 I64 精确深度、深度不截断、五级调度映射、证据不足转确认、不可控转备份、低层权限门控、饱和停止、权限与结算分离、父链可追溯和需求入口唯一写入权。
