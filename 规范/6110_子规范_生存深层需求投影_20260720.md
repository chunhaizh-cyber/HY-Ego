# 6110 子规范 生存深层需求投影

编号：6110
日期：20260720
状态：正式子规范

## 1. 定位

本规范规定如何从物理不安全事件的上游深层原因投影出生存深层需求候选，以及候选的类型、层级、门控、优先级和入树边界。

上级规范：0050、1190、3100、5110、5130、5140、6100。

## 2. 定义

```text
生存深层需求 =
为了降低未来物理不安全事件发生概率，
从该事件的上游深层因果原因投影出的需求。
```

直接导致物理不安全事件的原因是近端原因；距离事件超过一层的上游节点是深层原因。深层需求不是因为名称高级，而是因为它对物理安全的作用更远、更长期、更间接。

## 3. 合法事实和结构承载

物理不安全事件、原因节点、时间顺序和变化证据由状态、动态和因果结构承载；因果解释层按查询参数计算路径距离、发生概率、影响强度和派生置信分。

深层投影只形成带来源因果、目标宿主、目标特征、候选类型、层级、可控性、证据状态和优先级的值式候选。因果服务不得直接创建需求或任务；正式需求仍经需求统一提交入口。

派生置信分由因果基础计数、证据类型和查询参数计算，不得新增为因果、需求或任务的持久字段。

## 4. 投影方向

对会增加物理不安全事件概率或影响的深层原因，可以形成以下候选：

- 降低其发生概率；
- 降低其影响强度；
- 提高观测和确认能力；
- 提高控制和恢复能力；
- 建立隔离、预警、备份或替代路径。

协作、信誉、解释能力、维护关系、备份和治理等显示名称不能直接决定需求身份；只有它们能经因果链降低未来物理风险时，才属于生存深层需求候选。

## 5. 两类账

真实物理安全值只在物理状态真实改善并通过 6100 结算闸门时更新。深层原因风险下降通常只形成潜在生存贡献、未来风险下降、应急能力、恢复能力或可维护性改善。

禁止把完成解释、协作、维护或能力任务直接换算成物理安全增量。必须继续回查它是否降低了上游原因、未来故障、宕机或损坏风险。

## 6. 生成顺序

```text
发现物理不安全事件或风险上升
-> 反向追溯上游原因
-> 标记因果距离
-> 判断可观察、可验证和可控制性
-> 可干预节点形成控制候选
-> 不确定节点形成确认候选
-> 不可控制节点形成隔离、预警、备份或替代候选
-> 应用低层门控和饱和门控
-> 经需求统一入口入树
```

总顺序是先止损，再查因，再控因，最后治理深层结构。

## 7. 候选类型

| 类型 | 触发条件 | 目标方向 |
| --- | --- | --- |
| 确认型 | 原因仅是怀疑或证据不足 | 补齐当前事实和因果判断条件 |
| 控制型 | 原因已确认且可控制 | 降低原因发生或影响 |
| 预防型 | 同类原因长期或重复出现 | 建立稳定预防和监测机制 |
| 备份型 | 原因不可控或短期不可消除 | 隔离、备份、恢复、替代和预警 |
| 能力型 | 深层原因是自我能力不足 | 补齐解释、预测、停止、治理或学习能力 |

类型只决定候选目标方向和验证合同，不绕过目标状态、父需求和入树准入。

## 8. 层级和优先级衰减

离物理损害越近，优先级越高；越深层越长期，只有近层稳定后才进入独立竞争。第一轮投影可按十进衰减表达距离影响，例如 L0 到 L5 使用 `900000、90000、9000、900、90、10` 的同构整数刻度。

该刻度只用于候选优先级和因果投影衰减，不是任务结算预算、安全值或服务值。

概念公式为：

```text
Need(X) =
严重度(P)
* 路径发生概率
* 因果影响强度
* 派生置信分
* 距离衰减
* 可干预程度
```

值超过正式生成阈值时才形成相应候选；证据不足时形成确认候选，不形成修复候选。具体计算必须使用版本化整数规则，不新增自由浮点字段。

## 9. 门控

1. 物理不安全事件正在发生时，只生成应急止损及其直接手段。
2. 直接原因未控制时，深层治理只能作为解决近因的手段，不作为独立高层需求进入主竞争。
3. 近因未稳定时，不生成永久安全或长期治理的独立需求。
4. 某层达到明确饱和标准后，该层停止主动生成需求，只保留有有效期的监测。
5. 深层原因不可控不等于无事可做；应转换为确认、隔离、备份、预警、恢复或替代候选。

## 10. 正式入树

候选进入需求树前必须具备目标宿主、目标特征、当前状态、目标状态或区间、满足关系、父需求和派生归因。

应急止损和近层安全原因挂安全根需求或对应安全父需求；深层确认、预防、备份和能力候选挂触发它的安全父需求或因果投影父需求。父链必须保留到物理风险根目标的追溯。

## 11. 执行后验收

深层需求完成后，因果领域重新读取上游原因、路径状态和物理风险变化。成功证据可以增强路径基础计数，失败或竞争原因可以削弱或分化候选；不得直接改写因果结论或核心安全值。

## 12. 禁止项

- 禁止因果服务直接写正式需求或任务。
- 禁止深层需求绕过低层门控进入主竞争。
- 禁止把显示名称、自然语言标签或固定类型表当作深层因果证据。
- 禁止把深层需求完成直接结算为真实物理安全值。
- 禁止把派生置信分持久化为因果主信息字段。

## 13. 验证

验证覆盖因果距离、候选类型、证据不足转确认、不可控转备份、低层门控、饱和停止、优先级与结算分离、父链可追溯和需求入口唯一写入权。
