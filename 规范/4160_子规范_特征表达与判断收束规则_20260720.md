# 4160 子规范 特征表达与判断收束规则

编号：4160
日期：20260721
状态：正式子规范

直接上级规范：1120、1130、1160、1180、2100。

直接依赖规范：无。

相关边界规范：4110—4150、6300—6320。

## 1. 定位

本规范规定材料、测量、状态和判断在特征系统中的基础表达语法。核心原则是：判断结果不得伪装成直接事实。

## 2. 基础分类

1. 一阶特征只表达可直接取得、测量、计数、评分、引用或追溯的材料值。
2. 复合特征表达具名子特征的组成关系，本体不承载判定日志、错误码、界面状态、临时缓存或说明文本。
3. 二次特征表达关系性、比较性、方向性和聚合性判断，必须能够回查输入事实和比较规则。
4. 状态或治理收束结果表达合法流程或方法执行后的阶段性结论，必须保留来源、输入、规则、适用范围和有效期。

凡特征值依赖其它特征、评分、数量、版本、来源、阈值、任务上下文或治理裁决推导，无论值类型是枚举、整数还是二态值，都属于判断型结果。

## 3. 本项目承载

```text
一阶材料 -> 特征定义 + 实例槽位 + 特征值节点及原始值版本
复合组成 -> 抽象特征或二次特征的有序关系
比较与聚合 -> 二次特征及其输入、规则和版本
阶段结论 -> 状态节点或治理对象特征集合
来源证据 -> 动态、因果、规则身份和版本关系
```

普通函数返回值、日志、错误码、控制台和界面只可作为人读或调用结果，不得直接写成上述机器事实。

## 4. 判断收束原则

判断型结果必须同时声明并可回读：输入特征族、收束规则、写入方、读取方、失败分流、来源证据、适用范围、有效期和规则版本。输入不足、冲突、过期、不可回查或阈值不满足必须形成具名结果，不得退化为裸 `false`。

单个评分超过阈值、错误码为零、外设报告“可用”、界面显示成功或日志写入成功均不能单独形成判断结论。

## 5. 值类型边界

直接分类码和由唯一生命周期入口推进的阶段码可以作为一阶材料，但只表达分类或生命周期。质量、可用性、完成度、归属、稳定性、安全、服务结算、需求满足和任务完成属于判断型结果，必须经过正式收束。

句柄有效、版本一致、评分较高或候选唯一都只可作为收束输入，不自动等于材料可用或判断完成。普通 `bool` 不得持久化为业务判断事实。

## 6. 外部材料边界

外设、网络、文件、数据库和用户输入只能直接形成可追溯原始材料、测量、计数、评分、版本、来源和低层候选。它们不得直接裁决材料质量、正式提交、存在归属、需求满足、任务完成或安全与服务结算。

外部材料必须经观察、中间层收束、自我认知或指定治理方法，结合当前任务、场景、阈值和证据回查后，才能形成判断型特征或状态。
