# 0920 四域切换判定规则

## 1. 目标

本文用于把任务控制从“有四域”推进到“能判域、能切域、能进入域内动作”。

本文回答的是：

- 总控先看什么输入
- 先判哪个域，后判哪个域
- 每个域的最小进入规则是什么
- 动作做完后如何重新判域
- `任务控制总控模块_v0` 应提供什么最小接口

## 3. 总原则

必须固定：

- `控制不是直接控制所有细节，而是先决定当前任务应处在哪个功能域。`

因此总控第一判断不是：

- 要不要细分
- 要不要回退
- 要不要延长

而是：

- 现在该进入 `筹办 / 维护 / 纠偏 / 收束` 的哪一域

总控主链必须固定为：

```text
先判当前功能域
-> 再进入该域执行域内动作
-> 动作完成后重新判域
```


## 4. 判定输入最小集合

总控至少应读取以下输入：

| 输入项 | 作用 |
| --- | --- |
| 当前任务状态 | 判断任务当前处在哪个运行段 |
| 当前是否有可用方法 | 判断是否需要重回筹办 |
| 方法结果是否稳定 | 判断是否需要纠偏或可继续维护 |
| 最近反馈 | 判断失败、漂移、风险变化 |
| 安全值 | 判断是否允许继续或必须收束 |
| 服务值 | 判断是否值得继续投入 |
| 需求紧迫度 | 判断是否需要尽快形成或恢复路径 |
| 当前预算 | 判断是否允许继续扩张或修补 |
| 当前是否已满足完成判据 | 判断是否应进入收束 |

对齐到 `任务控制总控模块_v0` 的最小字段时，至少应承载：

- `来源需求`
- `当前任务`
- `当前场景`
- `当前任务状态`
- `安全值_Q10000`
- `服务值_Q10000`
- `需求值_Q10000`
- `已有可尝试方法`
- `最近结果稳定`
- `最近执行失败`
- `风险变坏`
- `检查点通过`
- `任务链已成形`
- `依赖缺失`
- `需要新子任务或新步骤`
- `关键方法不再可用`
- `服务收益不足以继续`
- `安全要求必须停止`
- `已满足完成判据`
- `已确定无法继续`


## 5. 四域切换判定顺序

判定顺序必须固定为：

```text
先判收束
再判纠偏
再判筹办
最后落到维护
```

原因：

- `收束` 负责处理“已经够了”与“必须停下”
- `纠偏` 负责处理“已经跑偏”
- `筹办` 负责处理“还没搭起来”
- `维护` 是默认稳定推进域


## 6. 四域进入规则

### 6.1 进入收束

满足任一即可进入：

- 已满足完成判据
- 已判定无法继续且无需再纠偏
- 服务收益不足以支撑继续
- 安全要求必须停止
- 当前任务已处于 `完成 / 失败 / 取消 / 超时 / 无法执行` 终结态

### 6.2 进入纠偏

在未命中收束前提下，满足任一即可进入：

- 最近执行失败
- 最近结果不稳定
- 风险变坏
- 检查点未通过
- 关键方法不再可用

### 6.3 进入筹办

在未命中收束与纠偏前提下，满足任一即可进入：

- 任务未成形
- 无可尝试方法
- 依赖缺失
- 需要新子任务或新步骤
- 当前任务状态为 `未定义 / 未启动 / 筹办中 / 待重筹办`

### 6.4 进入维护

在前述三域均未命中时，进入维护。

维护成立时至少应满足：

- 任务链已成形
- 至少有一个方法可尝试
- 无明显偏差
- 未达到结束条件


## 7. 域间切换表

| 当前域 | 触发 | 下一域 |
| --- | --- | --- |
| 筹办 | 已形成可执行任务链 + 已绑定可尝试方法 + 必要检查点已插入 | 维护 |
| 筹办 | 筹办后仍无路可走且不值得继续 | 收束 |
| 维护 | 最近执行失败 / 检查失败 / 风险变坏 / 结果不稳定 | 纠偏 |
| 维护 | 已满足完成判据 / 必须停止 / 收益不足 | 收束 |
| 维护 | 当前链断裂 / 无方法可尝试 / 依赖缺失 | 筹办 |
| 纠偏 | 已恢复到可执行状态 | 维护 |
| 纠偏 | 需要重新搭建任务链 | 筹办 |
| 纠偏 | 已不值得继续或必须停止 | 收束 |
| 收束 | 已完成收口动作 | 终结 |

必须固定：

- `收束` 结束后不允许在同一轮任务里偷偷重启主流程
- 如需再次启动，应由新输入重新触发 `筹办`


## 8. 域内动作执行后的重判规则

总控不应假设“做了某动作就一定留在当前域”。

必须固定为：

- 每次域内动作完成后，都应重新计算输入快照
- 用最新快照再次调用“判定当前功能域”
- 由新判定结果决定下一步继续留域、切域还是终结

最小闭环应写成：

```text
判域
-> 执行动作
-> 更新状态/反馈/预算
-> 重判
```


## 9. `任务控制总控模块_v0` 最小接口要求

总控模块最少应提供：

```cpp
判定当前功能域(...)
执行筹办任务(...)
执行维护任务(...)
执行纠偏任务(...)
执行收束任务(...)
```

其中：

- `判定当前功能域` 只负责判域，不直接做复杂业务动作
- 四个执行入口只做域内最小分发，不替代现有 `任务执行类 / 任务失败恢复模块_v0 / 学习写回模块_v0`
- 学习不应成为总控第一入口，而应作为 `纠偏` 或 `收束` 内的局部动作


## 10. 最小验收链

至少应打通以下 4 条最小链：

### 10.1 新需求进入

```text
无任务 -> 判定进入筹办 -> 创建任务 -> 细分或绑定方法
```

### 10.2 任务正常运行

```text
有可用方法 + 结果稳定 -> 判定进入维护 -> 刷新状态或继续推进
```

### 10.3 方法失败

```text
最近执行失败 -> 判定进入纠偏 -> 替换方法或回退或补条件
```

### 10.4 任务结束

```text
满足完成判据 或 无法继续 -> 判定进入收束 -> 写反馈或归档或学习写回
```


## 11. 收束

本文定义的是：

- `任务控制总控在什么输入下进入哪一域、域间如何切换、动作后如何重新判域。`
