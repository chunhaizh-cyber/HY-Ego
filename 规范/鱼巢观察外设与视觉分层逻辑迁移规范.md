## 2. 三层权力边界

```text
外设 / D455 层：采集、校正、滤波、分割、掩码、局部图、报告
观察材料层：观察帧、像素簇、质量、时间、范围和对应候选
自我认知层：存在候选、稳定对应、状态变化、内部细分和世界事实提交
```

1. 外设只供应材料、报告和质量证据；不得宣布存在、像素归属、场景真值、风险结论或需求满足。
2. 自我认知只能读取已经通过材料准入的外设结果，执行识别、验证、比较和正式提交；不得把预览画面、控制面板复现或外设缓存当世界事实。
3. 线程是采集和调度载体，不是观察动作、特征变化、状态动态或因果来源。需要写动作动态或实例特征变化时必须绑定正式方法。
4. 外设私有的校正、滤波、分割、重捕获、配置学习和调度不逐项注册为自我本能方法；对自我可见的是正式外设能力、请求 / 订阅和结构化报告。

## 3. 观察事实、质量与风险判断

1. 基础观察事实必须带来源外设、帧 / 材料集身份、时间、场景范围、质量、可观测区域和版本；缺一不得扩大为全场事实。
2. 材料可用、特征明确、存在确认、风险判断和世界提交是不同阶段。前一阶段成功不能直接宣布后一阶段成功。
3. 清晰、完整、稳定、可信、已生成、可用、已确认等判断型枚举必须有合法生产者、输入证据、判定规则、输出宿主、输出特征和来源动态。
4. 深度无效、遮挡、反光、缺失、过期和冲突必须显式成为材料质量 / 不可观测状态；不得填零、补背景或沿用旧值伪装当前观察。
5. 低质量或低新鲜度材料不得提交为正常事实，但失败材料仍是可审计材料，可以触发补观察或治理候选。
6. 风险判断读取已经成立的观察事实与因果证据；风险候选不得直接改安全值、需求满足或动作权限。

## 4. 当前场景明确与认知工作区

1. “当前场景特征值明确”必须限定主体、场景、材料集、可观测单位集合、目标特征集合、时间和完成标准；局部明确不能扩大为全场明确。
2. 有效可观测单位只有稳定对应已有存在、稳定对应新存在候选、多重对应冲突或证据不足等正式结果；冲突和证据不足只能作为缺口，不能当最终闭合归属。
3. 无效 / 不可观测材料不进入“有效单位全部对应”的完成计数，但必须保留原因和范围。
4. 自我所在场景认知工作区是当前场景、观察材料、存在候选、已确认存在、状态和任务关注的运行期组合视图，不是第二套权威世界仓库。
5. 工作区可以缓存索引和投影，但所有稳定身份与事实必须回查存在、场景、特征、状态、动态和关系仓库。

## 5. 识别、扫描与跟踪

1. 识别从观察像素簇和已有正式特征中寻找对应存在，只返回已有存在稳定身份或空 / 候选材料；不得补写一阶特征、创建世界事实或把当前范围唯一扩大为全局唯一。
2. 识别唯一性必须在明确候选范围、完整值复核、质量和多帧 / 多证据一致性满足后收敛。哈希、名称、单帧相似或局部唯一不能单独确认身份。
3. 扫描只针对至少一个稳定对应存在及其本轮应覆盖材料，更新其当前观察特征并在有历史可比材料时输出状态变化；首次扫描只建立当前基准，不宣称变化。
4. 扫描发现新区域只能输出未稳定对应特征或需重识别区域，不得直接创建新存在。
5. 跟踪第一版只承接已确认存在的指定目标，返回新位置 / 特征材料或空；不得当作全场扫描，也不得把候选跟踪稳定自动升格为存在确认。
6. 识别、扫描、跟踪报告可以同一周期并存，但必须各带范围和目标，不能互相替代；自我侧方法之间通过结果、缺口、需求、任务和方法选择衔接，不直接函数调用越权。
7. 只有观察入口可以从合法外设材料生产新的平面轮廓等一阶观察特征候选；识别只读已有特征，扫描只更新已确认存在的当前观察值，跟踪只读唯一凭证并返回目标材料。

## 6. 已确认存在的内部细分与体素

1. 只有存在身份、外部轮廓和材料质量已确认后，才可递归细分其内部结构；候选存在和未闭合像素簇不得直接建立权威内部世界。
2. 子结构必须能回溯父存在、局部材料、分解方法、空间关系、时间和版本；递归停止条件、冲突和证据不足必须显式。
3. 体素融合只融合时空标定兼容、质量合格且归属已确认的材料；融合结果是版本化几何 / 占据候选，不自动成为存在身份、因果或语义事实。
4. 体素分解产生子结构候选，需要经过稳定性、边界、归属和多帧验证后才能提交正式结构。
5. 场景体素服务负责几何材料和候选，自我视觉先验只提供预测 / 排序；先验不得覆盖当前实际观察，也不得在冲突时强写世界事实。
6. 体素、点云、掩码和彩色 / 深度局部图使用句柄、缓冲引用、索引或场景引用承载，不得塞入 I64 或主信息自由字段。

## 7. 外设作用与二次特征

1. 外设自身可直接测得或被正式动作改变的属性按一阶特征承载；外设与主体、场景或其它对象之间的相对位置、连接、可达和作用关系按二次特征承载。
2. 动作请求、外设执行报告、实际回读、状态变化和关系变化必须分开；执行报告成功不能替代实际回读。
3. 作用外设的一阶改变必须经正式方法 / 动作入口、权限和验证；外设线程不得裸改世界结构。
4. 关系性变化只有在端点身份、前后关系和证据均明确时，才可由二次特征或关系服务提交。

## 8. 外设独占线程与综合流程

1. 每个需独占访问的外设由独立控制 / 观察线程串行拥有物理句柄；其它线程通过有界请求、订阅和结构化回执交互，不直接调用驱动。
2. 双目相机线程独占采集、时间同步和设备配置，保持有界缓存、帧年龄、来源编号和停止 / 失效状态；不得持消息锁等待回调或持帧锁执行长时认知。
3. 综合流程保持：请求 / 订阅准入 -> 采集与校正 -> 质量 / 掩码 -> 逐簇识别报告、扫描变化报告或跟踪报告 -> 自我侧验证 -> 正式事实提交或缺口治理。
4. 报告必须区分最大允许年龄、是否允许缓存、范围、目标、质量和失败原因。超时、设备断开和材料不足走结构化拒绝 / 等待，不伪造空世界。
5. 重计算、视觉解析、推理、匹配和验证在锁外完成；多锁遵守项目正式锁序，禁止持外设锁进入领域写入或持仓库锁等待外设。
