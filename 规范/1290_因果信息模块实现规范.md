# 1290 因果信息模块实现规范

## 1. 目标

本文用于固定当前 `因果信息类.ixx` 的实现边界，回答以下问题：

- 当前因果模块已经实现了哪些最小能力
- 因果如何创建、累计、排序和查询
- 因果稳定度 Q 的语义是什么，不是什么
- 因果如何服务于学习与反推，又不越权替代方法验证

本文对应当前工程实现：

- `因果信息类.ixx`
- [0980_原生学习兜底函数集规范（现有实现对齐版）.md](D:\鱼巢\规范\0980_原生学习兜底函数集规范（现有实现对齐版）.md)

同时必须固定：

- 本文描述的是 `当前实现现状`
- 下一阶段因果重构的目标边界，以 [1285_因果分层建模与实例生成规范.md](D:\鱼巢\规范\1285_因果分层建模与实例生成规范.md) 为准
- 当“当前实现现状”和“下一阶段建模目标”不一致时，不得把现状误写成长期上位原则


## 2. 定位

当前因果模块的定位应固定为：

`运行时因果证据仓 + 反向检索索引 + 反推学习原料层`

但必须额外固定：

- 该定位对应的是当前未完成分层重构前的运行态定位
- 后续若重构到 `实例因果信息 / 抽象因果模板 / 因果链` 体系，本文需要按新实现继续修订

它不是：

- 已验证方法库
- 任务调度器
- 跨进程持久化因果数据库


## 3. 当前因果模型

当前实现中的因果模型，最少由以下部分构成：

- `条件状态列表`
- `动作信息`
- `过程动态列表`
- `结果状态列表`
- `观察次数`
- `首次发生时间`
- `最近发生时间`
- `稳定度Q`

当前因果键由以下内容组合：

- `动作信息指针`
- `条件签名`
- `过程签名`
- `结果签名`

因此当前因果应理解为：

- 同一进程内的运行时聚合证据
- 不是跨进程稳定 ID


## 4. 创建与累计规则

当前 `创建或累计因果信息` 的前提应固定为：

1. 条件状态列表非空
2. 结果状态列表非空
3. 过程动态列表非空
4. 动作信息非空
5. 条件签名、过程签名、结果签名都非零

若这些前提不满足，当前实现应理解为：

- 直接不创建
- 返回空结果

若命中相同因果键，则当前实现会：

- 复用已有因果节点
- 刷新因、果和锚点动态
- 刷新条件、过程、结果列表
- 更新最近发生时间
- 累加观察次数
- 重算稳定度 Q

若未命中相同因果键，则当前实现会：

- 新建因果节点
- 初始化首发时间和最近时间
- 初始化观察次数为 1
- 记录到正向索引和反向索引


## 5. 稳定度 Q 的当前语义

当前 `稳定度Q` 的实现是启发式排序分，不是概率真值。

当前构成至少包括：

- 观察次数分
- 是否有动作信息
- 是否有条件签名
- 是否有过程签名
- 是否有结果签名

因此必须固定：

1. `稳定度Q` 只用于当前实现中的排序和粗筛。
2. `稳定度Q` 不能直接当成“因果正确概率”。
3. 后续若要升级稳定度算法，必须保证“同样本更稳定优先、更新样本更近优先”的排序语义不被破坏。


## 6. 查询与排序规则

当前模块已经实现：

- 按结果签名查询因果
- 按动作信息查询因果

两类查询的排序语义都应固定为：

1. 先按 `稳定度Q` 降序
2. 再按 `最近发生时间` 降序
3. 最后按上限截断

因此必须固定：

- 查询接口返回的是“候选因果证据”
- 不是“已经可直接执行的方法”


## 7. 与反推学习的边界

当前因果模块与反推学习的关系应固定为：

- 因果模块负责沉淀和检索证据
- 反推学习负责把因果证据提炼为候选方法

因此必须固定：

1. 因果命中只能产出候选，不得直接产出“已验证方法”。
2. 因果模块不直接决定方法升格。
3. 任何因果到方法的转化，都必须经过学习或验证链路。


## 8. 线程与身份边界

当前因果键与反向索引使用了动作信息指针和内存态索引，因此必须固定：

1. 当前因果模块默认是进程内运行时结构，不是可直接持久化协议。
2. 任何跨进程、跨重启复用，都不能直接依赖当前指针键。
3. 若要引入持久化或跨进程因果 ID，必须新增稳定标识层，不能篡改现有运行时键语义。

另外，当前模块没有独立暴露细粒度内部线程同步，因此必须固定：

- 因果模块的写入与查询，应由上层调用方保证时序串行或更外层同步


## 9. 硬约束

1. 因果模块的职责是证据累计和候选检索，不是最终决策。
2. 因果必须同时具备因、动作、过程、果的最小闭环才能成立。
3. `稳定度Q` 只是一阶排序分，不得误读成概率。
4. 当前因果键是运行时键，不得误当成跨进程稳定主键。
5. 因果到方法的转化必须继续经过学习和验证，不得跳过。
