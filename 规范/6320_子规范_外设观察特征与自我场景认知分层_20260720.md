# 6320 子规范 外设观察特征与自我场景认知分层

编号：6320
日期：20260720
状态：正式子规范

## 1. 定位

本规范规定外设材料层、任务承接层、自我认知层和场景事实层之间的权力边界，以及观察、识别、扫描、跟踪的治理衔接。

上级规范：0050、1130—1190、2100、3200、3300、6200—6310。

## 2. 四层结构

```text
外设私有采集、校正、滤波和稳定化
-> 外设报告、材料页或合法当前数据集
-> 任务管理按等待项、订阅项、需求和时效承接
-> 自我方法形成观察候选、当前状态或正式场景提交
```

帧号、上一帧和来源外设帧身份只用于外设与接口层。进入自我层后必须转换为观察材料集、观察范围、可观测单位及其对应关系。

外设线程、报告、材料页、队列项、ROI 和目标观察约束均不是事实主体。只有领域服务的正式提交结果可以进入场景事实。

### 2.1 队列消费权

外设报告队列的正式消费者只能是任务域承接层。自我线程、需求根、普通任务工作线程和观察、识别、扫描、跟踪方法均不得直接取得高频外设流或报告队列的读取权。

任务域承接必须先完成：

```text
读取强类型供给产品
-> 校验外设身份、产品类型、来源链、版本和时效
-> 匹配当前任务、等待项、范围、目标、质量和优先级
-> 形成绑定任务与工作项的已承接外设材料引用
-> 交给当前冻结方法作为任务运行材料
```

已承接外设材料引用只在绑定任务、工作项和消费窗口内有效。它不是新的事实主体，也不得把外设队列项复制成长期世界事实。

## 3. 外设公开能力

外设对自我只公开控制、检查和使用三类边界能力。

- 控制类可以改变外设自身状态或采集配置，必须产生明确动作动态；
- 检查类原则上不启动、修复、切换模式或长期占用外设，若必须改变主要工作状态，应返回控制类缺口；
- 使用类按等待项或订阅项取得结构化观察报告，不直接写自我事实。

采集、校正、滤波、分割、匹配、SLAM、TSDF、重捕获和内部配置学习属于外设私有过程，不登记为自我层本能方法。

## 4. 扫描的治理位置

当前阶段由内部治理过程主动产生的主要场景维护入口是扫描。安全或服务治理为了确认当前状态、验证任务结果或维护当前场景，可以生成扫描需求。

扫描最小输入是：

- 自我内部场景稳定句柄；
- 已合法进入系统的外设当前数据集；
- 本轮扫描关注集合和应覆盖存在集合；
- 已有对应关系、当前特征值基准和比较规则。

扫描只比较本轮应覆盖的已确认存在。范围外、视野外、低频跳过、跟踪排除、优先级不足和条件不足的存在不得解释为消失或遮挡。

扫描可以更新当前特征值和当前状态并记录状态变化；不得从原始外设材料生成新的长期观察特征。需要补充一阶特征时，必须形成观察需求。

## 5. 四方法不得直接互调

观察负责补充存在的一阶观察特征；识别只读已有特征并返回已有存在稳定句柄或空句柄；扫描维护当前特征值和状态；跟踪只返回指定目标的新空间坐标或空信息。

四者不能成为彼此的子函数。扫描过程中发现对应、坐标、当前值、质量或未归属缺口时，只能输出结构化派生需求材料：

```text
派生需求类型
派生需求目标
派生需求来源证据
```

该材料经需求统一入口、任务承接和筹办后，才可能再次选择观察、识别、扫描或跟踪。

## 6. 高层观察运行模式

相机外部可见的高层观察运行模式只包括逐簇识别、扫描变化和目标跟踪。它们由任务管理等待项或订阅项表达，不是相机唯一全局状态。

ROI、补观察、质量诊断、局部细分、重捕获、稳定特征取值和配置学习只能作为等待项参数、内部策略或报告字段，不新增外显模式。

等待项至少携带需求或任务引用、目标范围或目标存在、期望特征、最大报告年龄、质量要求、优先级和是否允许缓存。

## 7. 事实提交与动作动态

每个对自我可见的方法必须保持：

```text
输入特征和来源材料
-> 方法执行
-> 输出特征、状态或缺口
-> 动作动态或来源动态
```

线程不是动作来源。动作动态必须关联执行方法、来源任务、输入版本、输出范围和发生时间。

空间坐标、轮廓、颜色和内部子结构只在其合法生产者、阈值和量化规则明确后提交。子存在、子结构、叶子特征块、内部未解释区域和冲突区域通过关系及场景零散事实目录承载，不堆入父存在普通字段。

## 8. 显示和验收边界

控制面板中的自我场景视图只能读取自我所在场景的正式快照及只读已承接观察材料，不得把原始相机画面、未被需求领走的外设消息或外设内部候选当作自我识别结果，也不得反写世界事实。

独立标识的相机预览或外设诊断视图可以只读观察原始画面或原始逐簇产品副本，但它不是报告队列的正式消费者，不改变消费状态，不参与任务匹配，也不得与自我场景结果混合显示或作为业务验收证据。

缺少真实外设、真实画面、任务承接、方法动态和正式场景提交时，只能记录等待验证，不得用夹具或显示成功宣布观察认知闭环完成。
