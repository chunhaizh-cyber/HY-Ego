# 6330 子规范 作用外设一阶特征改变与关系二次特征边界

编号：6330
日期：20260720
状态：正式子规范

## 1. 定位

本规范规定作用外设的直接输出、外界实际回读、关系二次特征和因果确认之间的边界。

上级规范：0050、1130、1160—1195、2100、3200、3300、4220、4320。

## 2. 直接改变边界

作用外设只能直接改变目标存在或外设自身的一阶特征，例如位置、姿态、速度、开合程度、末端状态、输出强度或工作状态。

“靠近、接触、遮挡、包含、夹持稳定、支撑、改善”等关系不是外设直接写入的物理字段。它们必须在动作后由自我读取一个或多个存在的一阶特征，通过合法计算形成二次特征、状态或关系判断。

不得为了表达关系变化而给作用方法新增“关系直接输出”字段。

## 3. 计划、执行和事实分账

必须分开：

| 层级 | 含义 |
| --- | --- |
| 动作计划或预测 | 期望改变、允许误差、风险和副作用预算 |
| 方法执行报告 | 外设命令是否按协议执行以及执行范围 |
| 实际观察回读 | 目标和外设一阶特征实际发生的变化 |
| 二次特征与因果判断 | 由前后事实、时序和作用范围计算的关系与归因 |

方法执行成功只表示作用动作完成，不表示任务目标已达成。预测不是事实，不能覆盖实际回读。

## 4. 动作动态

作用方法的动作动态至少关联：

- 来源任务和执行方法；
- 作用外设、目标存在和作用范围；
- 动作前目标与外设状态；
- 计划输出、允许误差和副作用门限；
- 执行时间范围和方法结果；
- 需要进行的感知验证。

动作动态不能只记录“成功”或“失败”。

感知方法只能记录观察行为产生的材料和事实刷新，不能把外界对象自身变化伪装为自我造成的变化。

## 5. 动作后验证

动作后必须重新观察并比较：

- 目标一阶特征是否达到目标范围；
- 外设末端误差是否在允许范围；
- 关系性二次特征是否满足；
- 副作用风险是否可接受；
- 观察材料是否新鲜、稳定且来源一致。

所有比较只使用明确的特征类型、比较符、目标值和有来源阈值。业务词不能代替比较方向。

作用前预测、执行报告和作用后实际结果必须分账。目标未达成、读回冲突或副作用不可接受时，任务按任务规范结算或形成结构化缺口，不得伪造成功状态。

## 6. 关系二次特征

距离、姿态偏差、中心偏差、重合率、接触状态、遮挡程度、包含关系和夹持稳定度等关系，必须由相关存在的一阶特征、比较规则、阈值、来源动态和版本计算。

二次特征的参与者通过稳定句柄和关系引用承载。只有上级二次特征规范允许时，多输入关系才可以沉淀为可复用二次特征；否则只保留为当前运行视图或短期状态。

## 7. 因果确认

相机观察到目标变化，不足以证明作用外设导致该变化。因果判断至少核对：

- 动作时间与变化时间；
- 作用范围和目标身份；
- 动作前后特征；
- 外设传感器与外部观察的一致性；
- 其它可能来源和硬冲突；
- 结果是否在预测误差范围内。

未完成上述验证时，只能记录同时发生或待归因候选，不能写稳定因果结论。

## 8. 任务筹办边界

作用任务的候选方法必须同时具备作用能力和必要的结果验证能力。筹办只选择已登记的方法结构，不现场执行感知、不生成关系事实，也不因缺少候选而创建方法首。

作用方法执行期若发现冻结前应具备的结构缺口，属于筹办漏检和内部逻辑错误；执行期新出现的外部材料不足或目标变化按结构化结果返回。

## 9. 验证

验证覆盖一阶直接改变、关系二次计算、计划与事实分账、动作后回读、阈值来源、副作用门限、因果反证以及筹办不写关系事实。

没有真实外设动作、实际回读、状态动态和因果证据时，不得宣称作用路径、关系改变或任务目标闭环成立。
