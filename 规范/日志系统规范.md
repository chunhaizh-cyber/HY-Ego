# 日志系统规范

编号：LOG-001
日期：2026-07-05
状态：项目规范 / 已确认 / 第一轮代码已实现并验证

## 1. 定位

本规范定义 `海中鱼巣` 的日志边界。

日志只做人读诊断、运行观察、审计辅助和调试定位，不承载机器逻辑，不参与需求满足、任务完成、方法成功、世界事实、结构修复或业务状态裁决。

## 2. 日志分类

第一版日志分为：

```text
逻辑错误日志
运行日志
调试日志
事件日志
```

所有日志都放在 `D:\海中鱼巣\日志` 目录下。

### 2.1 逻辑错误日志

逻辑错误报错必须写入独立日志文件。

逻辑错误日志用于记录违反项目结构规则、入口前置条件、仓库写入边界、领域服务授权边界或不可继续执行的逻辑错误。

逻辑错误日志必须独立于运行日志和调试日志，不得与普通运行情况或临时调试输出混写。

逻辑错误日志记录内容只允许包含人读诊断材料：

```text
发生时间
所属模块
入口名称
错误分类
拒绝原因
相关句柄或编号
触发路径摘要
```

逻辑错误日志不得写入或反推机器事实，不得作为自动修复、任务结算、需求满足或方法成功的依据。

若逻辑错误同时触发弹窗，弹窗与逻辑错误日志必须表达同一诊断原因；弹窗只做人读提醒。

弹窗出现即表示逻辑错误已经暴露。该逻辑错误未排除前，不得继续推进依赖该错误路径的实现、验收或完成声明。

### 2.2 运行日志

运行日志必须使用独立日志文件。

运行日志主要记录系统运行情况，用于人读观察系统启动、关闭、阶段推进、关键入口调用、外部材料接收、构建或运行环境提示、持久化适配状态等运行态信息。

运行日志不得与逻辑错误日志混写。运行日志可以记录运行过程中的普通状态说明，但不得把这些说明当作机器事实、业务状态、任务完成或需求满足依据。

运行日志记录内容应保持克制：

```text
启动和关闭
主要模块初始化
关键阶段开始和结束
外部请求或材料接收摘要
持久化适配读写摘要
非错误级运行提示
```

运行日志不得替代结构读取、验收输出、事件日志或逻辑错误日志。

### 2.3 调试日志

调试信息可以根据具体排查需要创建日志。

调试日志只允许服务于明确调试切片，例如入口拒绝排查、关系写入排查、索引候选排查、最小闭环验收排查。

调试日志必须独立于逻辑错误日志和运行日志，不得长期默认开启，不得作为功能输出，不得参与机器判断。

每个调试日志切片必须写明：

```text
调试目的
启用宏
影响文件
输出位置
关闭条件
验证方式
```

### 2.4 事件日志

事件日志属于后续持久化和审计适配范围。

事件日志可以记录已经由结构确认的事件材料，但不得反向裁决运行期事实。运行期核心仍以内存节点仓库、主信息仓库、关系仓库、索引仓库和领域服务写入结果为准。

## 3. 调试日志宏分片规则

调试日志代码必须按切片使用预编译宏控制启用和关闭。

宏命名应能定位切片范围：

```text
HY_EGO_DEBUG_LOG_<切片名>
```

示例：

```text
HY_EGO_DEBUG_LOG_ENTRY_REJECT
HY_EGO_DEBUG_LOG_RELATION_WRITE
HY_EGO_DEBUG_LOG_INDEX_QUERY
HY_EGO_DEBUG_LOG_MIN_LOOP
```

默认构建不得启用任何调试日志切片宏。

宏关闭时：

```text
不得写调试日志
不得改变运行行为
不得改变返回值
不得新增机器事实
不得影响验收结果
```

宏开启时：

```text
只允许增加人读调试输出
不得替代入口前置拒绝
不得修复结构
不得补写业务状态
不得作为测试通过的唯一依据
弹窗只提醒人处理逻辑错误，不能改变结构结果
```

## 4. 调试完成关闭规则

调试完成后必须关闭对应切片调试日志代码。

关闭要求：

```text
移除默认启用宏或构建参数
确认默认构建不产生该切片调试日志
保留必要的人读说明或实施记录
不得保留长期高频无用日志
```

若调试日志代码需要暂时保留，必须满足：

```text
默认宏关闭
切片边界清晰
输出路径独立
不会污染逻辑错误日志
不会污染运行日志
不会导致无用日志泛滥
后续关闭或删除条件明确
```

## 5. 禁止事项

禁止以下做法：

```text
用日志承载机器事实
用日志修复结构
用日志替代入口前置拒绝
用日志结果判断任务完成
用日志结果判断需求满足
用日志结果判断方法成功
长期默认开启调试日志
多个调试切片共用一个无法关闭的总宏
逻辑错误日志、运行日志和调试日志混写
调试结束后遗留高频无用日志
```

## 6. 验收方式

文档验收：

```text
规范文件存在
项目规则总纲引用本规范
计划索引列出本规范
项目记忆记录本规范状态
```

第一轮代码验收要求：

```text
逻辑错误报错写入独立逻辑错误日志文件
运行情况写入独立运行日志文件
默认构建不产生调试日志
指定调试宏开启后只产生对应切片日志
调试宏关闭后运行行为不变
日志不参与结构写入和机器裁决
```

当前第一轮代码已建立公共日志入口、运行日志入口、逻辑错误日志入口、调试日志切片宏入口和事件日志独立文件入口。默认运行只写运行日志；容错检查逻辑错误日志仍受 `HY_EGO_ENABLE_FAULT_TOLERANCE_CHECK` 控制；调试日志仍受对应切片宏控制。

```text

公共日志写入入口已建立。
```
