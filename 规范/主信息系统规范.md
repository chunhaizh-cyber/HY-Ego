# 主信息系统规范

编号：005
日期：2026-07-05
状态：待确认项目规范 / 第一轮边界草案

## 1. 定位

本规范用于定义 `海中鱼巣` 第一版主信息系统的项目级边界。

主信息系统不是旧项目各类 `*主信息类` 的字段合集，也不是业务字段表。第一版主信息系统只提供统一主信息仓库、稳定主信息句柄、最小记录生命周期和值容器能力。

旧项目主信息字段只能作为候选材料进入讨论，不得直接决定新项目字段布局。

## 2. 主要用途

主信息系统的主要用途：

```text
为节点提供统一主信息身份
保存主信息编号、版本号和有效状态
保存最小拓扑锚点
保存明确允许的值容器
为后续非权威缓存提供受控位置
阻止业务语义字段在主信息中膨胀
```

主信息系统不负责裁决世界事实、任务状态、方法可执行、需求满足、语素理解、显示标题或因果结论。

## 2.1 编号长期运行规则

主信息系统中的编号必须按长期运行设计，不得使用有符号整数表达身份编号。

以下身份编号统一使用 `uint64` 语义：

```text
仓库编号
主信息编号
拓扑锚点编号
后续可能新增的主信息本地索引编号
```

编号规则：

```text
0 保留为无效编号。
有效编号从 1 开始。
同一主信息仓库内主信息编号只递增，不复用。
删除主信息只改变记录状态并递增版本，不回收编号。
主信息稳定身份由 仓库编号 + 主信息编号 + 版本号 共同构成。
```

`I64` / `int64` 只用于值容器中的业务数值、状态值、评分或差值，不得用于主信息编号。

若长期运行导致单仓库编号、索引、快照或持久化体量接近工程上限，不得临时复用编号或改变编号类型。必须先进入规范和详细设计，采用：

```text
新增仓库编号
分片仓库
空间编号
纪元编号
快照归档
冷热数据分层
```

第一版只确认编号口径，不实现分片、纪元或归档机制。

## 3. 功能能力分层

### 3.1 总功能能力

主信息系统总能力包括：

```text
统一主信息仓库
主信息句柄
主信息记录生命周期
最小值容器
拓扑锚点
非权威缓存准入
字段新增门禁
跨系统承载分流
```

### 3.2 当前已实现能力

当前代码已实现：

```text
统一主信息仓库
主信息记录
主信息句柄
创建主信息
读取主信息
删除主信息
主信息有效性判断
写入单个 I64 值
读取单个 I64 值
```

当前代码没有实现：

```text
旧项目各类型主信息类
语素入口主信息字段
基础信息名称 / 类型字段
需求 / 任务 / 方法主信息字段族
非权威缓存准入登记
主信息字段迁移
```

### 3.3 第一轮待确认能力

第一轮只确认主信息边界和字段准入：

```text
是否保持一个统一主信息仓库
是否禁止迁移旧项目主信息字段族
哪些内容允许进入值容器
哪些内容必须改写到关系、索引、特征、状态、动态、因果、需求、任务或方法
是否暂缓所有非权威缓存字段
```

### 3.4 后续待实现能力

后续能力必须在字段功能逐项确认后再实现：

```text
VecI64 / VecIU64 值容器
主信息拓扑锚点写入入口
非权威缓存登记入口
主信息快照适配
主信息审计适配
```

### 3.5 暂缓能力

以下能力暂缓：

```text
旧项目词主信息类字段
旧项目语素入口主信息类字段
旧项目基础信息主信息字段族
旧项目需求主信息字段族
旧项目任务主信息字段族
旧项目方法主信息字段族
旧项目 SQL 主信息投影
旧项目控制面板字段显示回写
```

## 4. 实现方法

第一版实现方法：

```text
节点身份 -> 节点仓库
主信息身份和值容器 -> 主信息仓库
节点之间的含义连接 -> 关系仓库
查找入口 -> 索引仓库
业务写入规则 -> 领域服务
状态事实 -> 状态 / 特征 / 特征值
变化事实 -> 动态
解释和证据 -> 因果引用
需求目标 -> 需求服务和关系
任务运行 -> 任务服务和关系
方法能力 -> 方法服务和关系
```

具体信息不得简单塞入节点仓库或主信息仓库。节点仓库只保存节点身份、类型和生命周期；主信息仓库只保存最小主信息和值容器；业务语义必须结构化分流。

## 5. 对外服务

主信息系统只通过 `主信息仓库` 和领域服务授权路径提供服务。

当前服务：

```text
创建主信息
读取主信息
删除主信息
主信息是否有效
写入 I64 值
读取 I64 值
```

外部业务模块不得绕过领域服务，把业务事实直接写成主信息字段。

## 6. 结构承载

| 业务概念 | 项目承载结构 | 说明 |
| --- | --- | --- |
| 主信息身份 | 主信息句柄 | 长期身份，包含仓库编号、主信息编号、版本号 |
| 主信息生命周期 | 主信息记录状态和版本号 | 创建、读取、删除和旧句柄失效 |
| 主信息编号 | `uint64` 语义编号 | `0` 无效，仓库内只递增不复用 |
| 节点类型 | 节点仓库 | 不由主信息字段重复保存 |
| 语素对应信息 | 关系仓库 | 使用 `语素对应信息` 关系，不恢复旧字段 |
| 名称 / 类型 | 语素入口关系或后续只读显示派生 | 不保存为裸文本字段 |
| 特征值原始值 | 主信息值容器或后续特征值记录 | 只保存值，不保存业务判断 |
| 状态事实 | 状态节点 + 关系 + 特征值 | 不作为主信息字段 |
| 动态事实 | 动态节点 + 因果来源关系 | 不作为主信息字段 |
| 因果证据 | 因果引用 + 动态样本 | 不作为主信息字段 |
| 需求目标 | 需求节点 + 状态 / 特征值关系 | 不作为通用主信息字段 |
| 任务运行 | 任务节点 + 关系 + 状态 / 动态 | 不作为通用主信息字段 |
| 方法能力 | 方法节点 + 条件 / 结果结构 | 不作为通用主信息字段 |

## 7. 字段准入规则

允许优先进入主信息的内容：

```text
主信息编号
版本号
有效状态
拓扑锚点编号
值容器
明确声明的非权威缓存
```

不得作为主信息权威字段的内容：

```text
对象属性
观测事实
统计事实
治理状态
判断结论
任务结果
方法结果
需求满足结论
因果解释
显示标题
日志摘要
SQL 投影行
裸文本语义
```

新增字段前必须先回答：

```text
它是否是身份、拓扑、值容器或非权威缓存。
它是否可以由节点、关系、索引、特征、状态、动态、因果、需求、任务或方法表达。
它是否会成为第二套业务事实账。
它的写入方、读取方、生命周期、是否权威和验证方式是什么。
```

无法回答时不得新增字段。

## 8. 分流规则

具体信息按以下规则分流：

```text
是节点身份 -> 节点仓库
是原始值 -> 主信息值容器或特征值系统
是两个对象的连接 -> 关系仓库
是查找入口 -> 索引仓库
是对象属性或状态 -> 特征 / 状态 / 特征值
是变化过程 -> 动态
是来源、原因、证据 -> 因果引用
是目标 -> 需求
是运行承接 -> 任务
是可执行能力 -> 方法
是人读说明 -> 文档 / 显示 / 日志，不入机器事实
```

## 9. 旧项目字段处理口径

第一轮确认以下处理口径：

```text
旧项目主信息字段不直接迁移。
旧项目语素对应基础信息字段已改写为关系。
旧项目名称 / 类型字段暂不进入主信息。
旧项目需求 / 任务 / 方法字段归入各自系统规范，不作为通用主信息字段迁移。
旧项目特征 / 状态 / 动态 / 因果字段归入对应系统规范，不作为主信息迁移。
旧项目非权威缓存字段第一轮暂缓。
```

## 10. 验证方式

规范确认后，主信息代码实施或调整至少需要验证：

```text
可创建主信息。
可读取有效主信息。
删除主信息后旧句柄失效。
主信息编号使用 `uint64` 语义，`0` 不作为有效主信息编号。
删除主信息后不复用旧编号。
无效主信息句柄不能写入值。
I64 值可写入和读取。
语素对应信息仍通过关系读取，不依赖主信息字段。
需求、任务、方法、状态、动态、因果事实不通过通用主信息字段裁决。
```

## 11. 禁止宣称

本规范建立或确认后仍不得宣称：

```text
旧项目主信息字段已迁移
旧项目主信息类已移植
需求 / 任务 / 方法主信息已完成
特征 / 状态 / 动态 / 因果主信息已完成
主信息字段功能已全部确认
旧项目能力已迁移
```
