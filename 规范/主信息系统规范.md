# 主信息系统规范

编号：005
日期：2026-07-05
状态：已确认项目规范 / 第一轮主信息边界已确认 / 主信息字段功能第一轮已确认

## 1. 定位

本规范用于定义 `海中鱼巣` 第一版主信息系统的项目级边界。

主信息系统不是旧项目各类 `*主信息类` 的字段合集，也不是业务字段表。第一版主信息系统只提供统一主信息仓库、稳定主信息句柄、最小记录生命周期和值容器能力。

旧项目主信息字段只能作为候选材料进入讨论，不得直接决定新项目字段布局。

本规范确认的是主信息系统的边界和字段准入规则，不证明旧项目主信息字段已经迁移，不证明需求、任务、方法、特征、状态、动态或因果系统已经完成。

## 2. 主要用途

主信息系统的主要用途：

```text
为节点提供统一主信息身份
保存主信息编号、版本号和有效状态
保存最小拓扑锚点
保存明确允许的值容器
为后续非权威缓存提供受控位置
阻止业务语义字段在主信息中膨胀
```

主信息系统不负责裁决世界事实、任务状态、方法可执行、需求满足、语素理解、显示标题或因果结论。

## 2.1 编号长期运行规则

主信息系统中的编号必须按长期运行设计，不得使用有符号整数表达身份编号。

以下身份编号统一使用 `uint64` 语义：

```text
仓库编号
主信息编号
拓扑锚点编号
后续可能新增的主信息本地索引编号
```

编号规则：

```text
0 保留为无效编号。
有效编号从 1 开始。
同一主信息仓库内主信息编号只递增，不复用。
删除主信息只改变记录状态并递增版本，不回收编号。
主信息稳定身份由 仓库编号 + 主信息编号 + 版本号 共同构成。
```

`I64` / `int64` 只用于值容器中的业务数值、状态值、评分或差值，不得用于主信息编号。

若长期运行导致单仓库编号、索引、快照或持久化体量接近工程上限，不得临时复用编号或改变编号类型。必须先进入规范和详细设计，采用：

```text
新增仓库编号
分片仓库
空间编号
纪元编号
快照归档
冷热数据分层
```

第一版只确认编号口径，不实现分片、纪元或归档机制。

## 3. 功能能力分层

### 3.1 总功能能力

主信息系统总能力包括：

```text
统一主信息仓库
主信息句柄
主信息记录生命周期
最小值容器
拓扑锚点
非权威缓存准入
字段新增门禁
跨系统承载分流
```

### 3.2 当前代码已实现能力

当前代码已实现能力按文档声明记录如下，后续仍需通过构建和验收用例复验：

```text
统一主信息仓库
主信息记录
主信息句柄
创建主信息
读取主信息
删除主信息
主信息有效性判断
写入单个 I64 值
读取单个 I64 值
写入指定索引 I64 值
读取指定索引 I64 值
```

当前代码没有实现：

```text
旧项目各类型主信息类
语素入口主信息字段
基础信息名称 / 类型字段
需求 / 任务 / 方法主信息字段族
非权威缓存准入登记
主信息字段迁移
```

### 3.3 第一轮已确认能力

第一轮确认以下主信息边界和字段准入结论：

```text
保持一个统一主信息仓库。
主信息记录保持最小化，不拆回旧项目各类主信息类。
旧项目主信息字段族不直接迁移。
值容器只承载明确被授权的原始值、状态值、评分、差值或向量值。
对象属性、观测事实、统计事实、治理状态、判断结论、任务结果、方法结果、需求满足结论和因果解释必须分流到对应结构。
第一轮暂缓所有非权威缓存字段，后续必须逐项登记用途、写入方、读取方、生命周期、是否权威和验证方式。
主信息字段功能确认清单已全部确认；该确认只决定旧字段处理口径，不等于字段已经迁移。
```

### 3.4 后续待实现能力

后续能力必须在字段功能逐项确认后再实现：

```text
VecI64 / VecIU64 值容器
主信息拓扑锚点写入入口
非权威缓存登记入口
主信息快照适配
主信息审计适配
```

### 3.5 暂缓能力

以下能力暂缓：

```text
旧项目词主信息类字段
旧项目语素入口主信息类字段
旧项目基础信息主信息字段族
旧项目需求主信息字段族
旧项目任务主信息字段族
旧项目方法主信息字段族
旧项目 SQL 主信息投影
旧项目控制面板字段显示回写
```

### 3.6 确认结论

本规范确认后，主信息系统只允许作为以下结构角色使用：

```text
稳定主信息身份
主信息生命周期记录
最小值容器
后续明确登记的拓扑锚点
后续明确登记的非权威缓存
```

不得把本规范解释为允许恢复旧项目主信息字段族，也不得把主信息值容器当成第二套业务事实账。

## 4. 实现方法

第一版实现方法：

```text
节点身份 -> 节点仓库
主信息身份和值容器 -> 主信息仓库
节点之间的含义连接 -> 关系仓库
查找入口 -> 索引仓库
业务写入规则 -> 领域服务
状态事实 -> 状态 / 特征 / 特征值
变化事实 -> 动态
解释和证据 -> 因果引用
需求目标 -> 需求服务和关系
任务运行 -> 任务服务和关系
方法能力 -> 方法服务和关系
```

具体信息不得简单塞入节点仓库或主信息仓库。节点仓库只保存节点身份、类型和生命周期；主信息仓库只保存最小主信息和值容器；业务语义必须结构化分流。

## 5. 对外服务

主信息系统只通过 `主信息仓库` 和领域服务授权路径提供服务。

当前服务：

```text
创建主信息
读取主信息
删除主信息
主信息是否有效
写入 I64 值
读取 I64 值
写入指定索引 I64 值
读取指定索引 I64 值
```

外部业务模块不得绕过领域服务，把业务事实直接写成主信息字段。

## 6. 结构承载

| 业务概念 | 项目承载结构 | 说明 |
| --- | --- | --- |
| 主信息身份 | 主信息句柄 | 长期身份，包含仓库编号、主信息编号、版本号 |
| 主信息生命周期 | 主信息记录状态和版本号 | 创建、读取、删除和旧句柄失效 |
| 主信息编号 | `uint64` 语义编号 | `0` 无效，仓库内只递增不复用 |
| 节点类型 | 节点仓库 | 不由主信息字段重复保存 |
| 语素对应信息 | 关系仓库 | 使用 `语素对应信息` 关系，不恢复旧字段 |
| 名称 / 类型 | 语素入口关系或后续只读显示派生 | 不保存为裸文本字段 |
| 特征值原始值 | 主信息值容器或后续特征值记录 | 只保存值，不保存业务判断 |
| 状态事实 | 状态节点 + 关系 + 特征值 | 不作为主信息字段 |
| 动态事实 | 动态节点 + 被改变目标 / 改变前值 / 改变后值 / 发生时间戳 | 不作为主信息字段 |
| 因果证据 | 因果引用 + 动态样本 | 不作为主信息字段 |
| 需求目标 | 需求节点 + 状态 / 特征值关系 | 不作为通用主信息字段 |
| 任务运行 | 任务节点 + 关系 + 状态 / 动态 | 不作为通用主信息字段 |
| 方法能力 | 方法节点 + 条件 / 结果结构 | 不作为通用主信息字段 |

## 7. 字段准入规则

允许优先进入主信息的内容：

```text
主信息编号
版本号
有效状态
拓扑锚点编号
值容器
明确声明的非权威缓存
```

不得作为主信息权威字段的内容：

```text
对象属性
观测事实
统计事实
治理状态
判断结论
任务结果
方法结果
需求满足结论
因果解释
显示标题
日志摘要
SQL 投影行
裸文本语义
```

新增字段前必须先回答：

```text
它是否是身份、拓扑、值容器或非权威缓存。
它是否可以由节点、关系、索引、特征、状态、动态、因果、需求、任务或方法表达。
它是否会成为第二套业务事实账。
它的写入方、读取方、生命周期、是否权威和验证方式是什么。
```

无法回答时不得新增字段。

主信息字段迁移红线：

```text
未确认字段功能说明，不新增主信息字段。
未确认字段是否权威，不允许进入值容器。
未确认字段能否由关系、索引、特征、状态、动态、因果替代，不允许编码。
未确认读取方和写入方，不允许作为领域服务对外契约。
```

## 8. 分流规则

具体信息按以下规则分流：

```text
是节点身份 -> 节点仓库
是原始值 -> 主信息值容器或特征值系统
是两个对象的连接 -> 关系仓库
是查找入口 -> 索引仓库
是对象属性或状态 -> 特征 / 状态 / 特征值
是变化过程 -> 动态
是来源、原因、证据 -> 因果引用
是目标 -> 需求
是运行承接 -> 任务
是可执行能力 -> 方法
是人读说明 -> 文档 / 显示 / 日志，不入机器事实
```

## 9. 旧项目字段处理口径

第一轮确认以下处理口径：

```text
旧项目主信息字段不直接迁移。
旧项目语素对应基础信息字段已改写为关系。
旧项目名称 / 类型字段暂不进入主信息。
旧项目需求 / 任务 / 方法字段归入各自系统规范，不作为通用主信息字段迁移。
旧项目特征 / 状态 / 动态 / 因果字段归入对应系统规范，不作为主信息迁移。
旧项目非权威缓存字段第一轮暂缓。
```

## 9.1 主信息字段功能第一轮确认结论

`实施记录/20260705_主信息字段功能确认待确认清单.md` 已按用户“全部确认”形成第一轮结论。该结论只允许更新规范和后续迁移计划，不允许直接迁移旧字段或旧代码。

语素相关字段确认如下：

| 编号 | 确认结论 |
| --- | --- |
| MI-YS1 `词` | 作为注册种子、索引键或显示来源，不进入主信息权威字段。 |
| MI-YS2 `词性` | 暂缓到语言候选材料或候选查询，不参与入口绑定目标裁决。 |
| MI-YS3 `信息入口类型` | 改写为语素入口关系、索引或入口路由规范，不作为主信息字段。 |
| MI-YS4 `对应基础信息类型` | 废弃主信息字段，由节点类型和关系入口前置拒绝承载。 |
| MI-YS5 `对应基础信息` | 继续由 `关系类型::语素对应信息` 承载，支持多对应信息节点。 |
| MI-YS6 `结构_信息入口定义` | 暂缓到入口路由规范。 |
| MI-YS7 `默认根名` | 只做人读显示或初始化材料，废弃为机器字段。 |
| MI-YS8 入口策略字段 | 暂缓并改写为状态值或策略规则，不使用 bool 裁决机器事实。 |

基础信息相关字段确认如下：

```text
名称、类型、任务名称、方法名称：走语素入口关系或候选查询，不进入裸主信息字段。
统计、命中、缓存：第一轮暂缓，后续只能作为非权威缓存或诊断材料逐项登记。
语言词性、指代解析、外设、体素、SQL、控制面板：全部暂缓到对应系统规范。
特征形态和值域：拆到特征 / 特征值规范。
当前值、稳态、近帧候选：拆到特征值 / 状态 / 动态 / 因果系统。
概念模板、概念集：改写为多级、多路概念追溯关系。
场景状态索引、动态索引、二次特征索引：改写为索引仓库或关系仓库，索引不作为事实权威。
状态、动态、二次特征、因果字段族：拆到对应领域系统，不作为主信息第一轮字段。
```

特征值、需求、任务和方法相关字段确认如下：

```text
MI-V1 值容器语义只保留在特征系统内部范围内。
特征值服务只允许由特征服务直接访问，不得扩展成需求 / 任务 / 方法通用值容器。
需求目标是目标状态，不得直接写入需求节点主信息值容器。
MI-V2 哈希、命中次数、最后命中改写为索引 / 非权威缓存或暂缓。
需求主体、目标、满足关系和需求任务关系拆到需求系统、状态系统和关系仓库。
任务名称 / 类型走语素入口关系。
MI-T2、MI-T3 登记为任务系统专项，不进入本轮主信息字段。
方法名称走语素入口关系。
MI-M1、MI-M2 登记为方法系统和动作入口规范专项，不进入本轮主信息字段。
方法关系、条件场景、结果场景、治理状态和查找策略拆到方法系统、关系仓库、状态值或策略规则。
```

后续任何字段若要进入主信息，仍必须重新提供字段功能说明，并证明它属于身份、拓扑锚点、最小值容器或明确声明的非权威缓存。

## 10. 验证方式

规范确认后，主信息代码实施或调整至少需要验证：

```text
可创建主信息。
可读取有效主信息。
删除主信息后旧句柄失效。
主信息编号使用 `uint64` 语义，`0` 不作为有效主信息编号。
删除主信息后不复用旧编号。
无效主信息句柄不能写入值。
I64 值可写入和读取。
指定索引 I64 值可写入和读取。
语素对应信息仍通过关系读取，不依赖主信息字段。
需求、任务、方法、状态、动态、因果事实不通过通用主信息字段裁决。
新增主信息字段前必须存在字段功能说明。
```

## 12. JY-381 用途观察专用主信息候选（待 #290 审计）

用途观察节点 15 必须关联一条主信息，但不得把完整回执、结算 DTO、用途事件、机会证据组或节点句柄副本写入通用值容器。格式 1 恰好使用 15 个非空 I64 槽：

```text
0 O1 格式版本
1 / 2 任务序号高 / 低 32 位
3 / 4 原因动态前 / 后状态值
5 原因动态规则版本
6 / 7 结果动态前 / 后状态值
8 结果动态规则版本
9 因果规则版本
10 因果完整结构键格式版本
11 机会数
12 正结果数
13 负结果数
14 Q10000
```

格式、三个规则版本均必须为当前非零版本；高 / 低位只能是非负 32 位值，组合后任务序号必须非零；机会数为 1—4096，正数 + 负数 = 机会数，Q10000 必须不大于 10000 且等于 `正数 * 10000 / 机会数`。槽数不等于 15、任一空槽、整数越界或未知格式均在第一笔结构写入前拒绝。

主信息一经发布不得原地覆盖；待结算推进最终只更换关系 18 的阶段 / 结算状态关系。该专用布局仍须 #290 审计，当前主信息仓库代码未新增领域格式裁决。
