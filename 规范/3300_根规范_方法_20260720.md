# 3300 根规范 方法

编号：3300
日期：20260720
状态：正式根规范

## 1. 根定位

方法是带稳定身份、适用条件、预期结果和可调用动作入口的能力契约。方法不是名称、函数指针、任务、运行回执或日志摘要。

方法首只提供方法身份和大范围召回入口。具体能力和执行语义必须回到条件、结果、条件结果对、动作入口及其当前版本。

## 2. 本项目承载

方法在海中鱼巣中由以下结构共同承载：

```text
方法登记根
+ 方法首节点及稳定句柄
+ 方法唯一治理虚拟存在
+ 方法条件节点及规格关系
+ 方法结果节点及规格关系
+ 条件结果对和样本证据关系
+ 动作入口节点、输入场景、输出场景和稳定动作键
+ 生命周期状态、内容版本和当前性关系
+ 运行事实、统计和学习证据关系
```

“方法虚拟存在”是方法内部唯一长期能力基底，不是世界事实存在。方法运行实例必须另有本轮运行身份和场景，不得污染长期能力基底。

## 3. 条件、结果和能力原则

方法条件声明执行前必须满足或绑定的输入事实；方法结果声明方法可以产生的目标状态；条件结果对表达在特定条件下产生特定结果的可复现能力样本。

条件和结果必须关联明确场景、目标特征、抽象状态、来源版本和作用对象。名称、动作文本、函数名、语素文本、显示标签和方法首本身不得替代这些规格。

条件场景和结果场景是方法规格或样本容器，不是任务本轮真实输入、输出或世界事实场景。运行期输入、临时结果和输出回执必须使用独立场景与当前性关系。

## 4. 召回、选择与执行原则

方法召回按结果能力形成非权威候选和建议；任务领域在取得该任务唯一筹办执行权后，才可进行完整候选填充、复判、选择和冻结。

无候选方法时不得自动创建方法首，也不得用通用特征写入逻辑、底层原语或分类标签伪装成语义方法。

执行前必须重新验证方法生命周期、内容版本、条件、参数、禁止项、动作入口和任务冻结。方法首存在或能力粗命中不能单独证明可执行。

## 5. 方法动作与运行事实

主动方法真实执行和被动状态提交必须以具体方法或正式提交方法作为动作来源，并形成相应动态证据。线程、消息和任务调度器不是动作来源。

纯派生整理处理不属于方法，不进入候选集，也不生成主链动作动态。方法运行成功只说明本轮动作运行，不等于任务完成或需求满足。

方法运行事实、输出结果、条件结果样本、长期统计和方法状态必须分账。只有方法自身执行路径或方法领域正式提交入口可以更新该方法的长期运行账和稳定能力。

## 6. 写入权与生命周期

方法登记、规格、条件结果对、动作入口、生命周期、运行账和稳定性只能由方法领域服务及其专用数据操作层写入。任务、学习动作、世界服务、显示层和外部调用方不得绕过方法服务直接改写。

方法生命周期、规格版本和动作入口当前性必须可回读。失效方法不得被当前召回；旧版本事实可以保留，但必须与当前发布关系隔离。

方法结构在执行冻结前必须完整复核。执行期才发现冻结前已经存在的结构缺口，属于筹办漏检和内部逻辑错误，不得补造结构后继续执行。

## 7. 相邻对象边界

任务可以选择并冻结方法，方法不能自行创建任务或需求。学习能力只能提交候选结构、运行证据或结构化缺口，正式需求与任务仍由各自领域建立。

一个方法可以被多个任务复用；每个任务的参数、授权、运行态、结果和结算必须留在各自任务或运行实例中，不得写入共享方法虚拟存在。
