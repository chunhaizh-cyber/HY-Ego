# 3300 根规范 方法

编号：3300
日期：20260721
状态：正式根规范

直接上级规范：0050。

直接依赖规范：无。

相关边界规范：1170、1190、3100、3200、4220、5230—5250、5300—5340。

## 1. 根定位

方法是带稳定身份、适用条件、预期结果和可调用动作入口的能力契约。方法不是名称、函数指针、任务、运行回执或日志摘要。

方法首只提供方法身份和大范围召回入口。具体能力和执行语义必须回到条件、结果、条件结果对、动作入口及其当前版本。

## 2. 本项目承载

方法在海中鱼巣中由以下结构共同承载：

```text
方法登记根
+ 方法首节点及稳定句柄
+ 方法唯一治理虚拟存在
+ 方法条件节点及规格关系
+ 方法结果节点及规格关系
+ 条件结果对和样本证据关系
+ 动作入口节点、输入场景、输出场景和稳定动作键
+ 生命周期状态、内容版本和当前性关系
+ 运行事实、统计和学习证据关系
```

“方法虚拟存在”是方法内部唯一长期能力基底，不是世界事实存在。方法运行实例必须另有本轮运行身份和场景，不得污染长期能力基底。

## 3. 事实来源与能力原则

方法条件声明执行前必须满足或绑定的输入事实；方法结果声明方法可以产生的目标状态；条件结果对表达在特定条件下产生特定结果的可复现能力样本。条件、结果、动作入口和能力事实只能来自方法领域正式登记、真实运行证据及其合法升格过程。

条件和结果必须关联明确场景、目标特征、抽象状态、来源版本和作用对象。名称、动作文本、函数名、语素文本、显示标签、候选命中和方法首本身不得替代这些规格或证明可执行。

规格场景、任务本轮场景、运行期临时场景和输出事实必须保持身份分离；具体登记、召回、选择、运行账与证据升格合同由 5300 唯一规定。

## 4. 写入权与生命周期

方法登记、规格、条件结果对、动作入口、生命周期、运行账和稳定能力只能由方法领域服务及其专用数据操作层写入。任务、学习动作、世界服务、显示层和外部调用方不得绕过方法服务直接改写。

方法生命周期、内容版本、规格版本和动作入口当前性必须可回读。失效方法不得被当前召回；旧版本事实可以保留，但必须与当前发布关系隔离。方法结构在执行冻结前必须完整复核，执行期才发现冻结前既有结构缺口属于筹办漏检和内部逻辑错误。

## 5. 相邻对象与执行边界

任务可以在取得自身筹办执行权后选择并冻结方法；方法不能自行创建任务或需求。无候选方法不得授权自动创建方法首，也不得让通用写入、底层原语或分类标签冒充语义方法。

一个方法可以被多个任务复用；每个任务的参数、授权、运行态、结果和结算必须留在各自任务或运行实例中，不得写入共享方法治理虚拟存在。方法运行成功只证明本轮运行事实，不等于任务完成、需求满足或价值结算。

主动方法、正式被动提交和纯派生整理的动作边界由 4220、5300 规定；本能方法自身环境、函数执行和学习能力边界由 5310—5330 规定，学习候选晋级、新代际发布和任务回合同轮隔离由 5340 唯一规定。
