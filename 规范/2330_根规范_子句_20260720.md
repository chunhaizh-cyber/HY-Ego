# 2330 根规范 子句

编号：2330

日期：20260720

状态：正式根规范

## 1. 根定位

子句是自然句版本内可独立解析的结构片段和候选语义锚点。子句可以关联解析候选，但子句本身不是场景、状态、动态、因果、需求或方法事实。

## 2. 正式结构承载

正式子句至少由以下结构共同成立：

```text
4010 节点类型 19 与子句身份命名域形成的稳定身份
+ 唯一自然句版本归属关系
+ 原子结构类型
+ 起始词位置和结束词位置
+ 与范围一致的局部词出现序列
+ 解析候选关系及其来源版本
+ 当前发布和历史版本
```

词范围统一采用“起始位置包含、结束位置不包含”的半开区间。起始位置必须小于结束位置，结束位置不得超过所属自然句的词出现数量。范围规则属于结构合同，不能由不同调用方自行解释。

## 3. 归属、范围与解析候选

1. 一个子句版本必须且只能归属一个自然句版本。
2. 局部词出现序列必须与所属自然句的区间材料逐项一致，并保留重复词和顺序。
3. 同一自然句可以有相邻、包含或重叠子句，但每个子句必须保留解析器身份、解析版本和独立范围；不得用覆盖顺序隐式裁决谁有效。
4. 子句可以关联场景、状态、动态和因果等解析候选。关系只表示语言解析关联，目标领域对象自身及其正式关系才承载业务事实。
5. 需求候选、方法候选、动态候选或因果候选必须经对应领域入口验证和发布；候选置信度、模型解释和语法标签不得直接升级为稳定项目事实。

## 4. 写入方、读取方与验证

语言领域服务负责子句身份、自然句归属、原子结构、词范围、局部词序、解析版本和候选关系。对应领域服务只接收带完整来源的候选请求，不回写子句业务状态。

正式读回应互证子句角色、唯一自然句归属、半开区间合法性、局部词序与自然句一致、解析候选端点和来源版本。范围或候选在发布后不能完整读回时属于逻辑错误。

## 5. 禁止事项

- 禁止把子句文本、语法标签、模型推断或候选置信度直接升级为业务事实。
- 禁止用字符串字节偏移代替词位置，或在不同编码下复用同一范围。
- 禁止让子句领域记录复制对应场景、状态、动态或因果的内部事实。
- 禁止把控制面板分组、缩进或选中状态作为子句归属和范围来源。
