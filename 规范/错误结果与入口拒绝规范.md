# 错误结果与入口拒绝规范

编号：016
日期：2026-07-05
状态：待确认规范 / 核心内核 P1

## 1. 定位

本规范定义函数入口拒绝的结果口径。项目禁止把错误结构先写入再由校验逻辑修复。

## 2. 入口拒绝类型

第一版入口拒绝原因：

```text
无效句柄
已删除句柄
版本不匹配
错误节点类型
错误关系类型
缺少前置节点
缺少前置关系
越权写入方
生命周期不满足
重复写入冲突
未确认字段功能
```

## 3. 返回模型

当前代码可以使用：

```text
无效句柄
std::nullopt
局部 bool 成功 / 失败
领域枚举结果
```

要求：

```text
bool 只表示局部函数执行结果，不持久化为机器事实。
领域状态必须用枚举、状态值、计数、版本或结构关系表达。
拒绝路径不得写业务事实。
拒绝路径不得依赖日志、弹窗或控制台输出。
```

## 4. 容错诊断

容错检查只允许在宏开启时记录和弹窗：

```text
HY_EGO_ENABLE_FAULT_TOLERANCE_CHECK
```

宏关闭时不得写日志、不得弹窗、不得改变运行行为。

## 5. 验收方式

```text
无效输入在函数入口被拒绝。
拒绝后仓库结构不变化。
默认构建不产生容错日志。
宏开启构建能编译通过。
```
