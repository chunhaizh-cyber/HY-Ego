编号：需求结算-001

## 1. 定位

需求结算记录负责表达“某个需求在某次结算中，依据哪些目标状态、实际状态、任务和证据，被判定为已满足、未满足或证据不足”。

需求结算记录不是需求目标本身，也不是任务完成本身。需求目标是目标状态；任务完成需要任务目标状态和实际结果状态比较；需求结算记录保存结算过程和证据链。

本规范覆盖主信息迁移 S4 中 MI-D3 的权威结算记录部分。MI-D3 中的安全 / 服务权重、累计结算、最近结算任务和时间若只是摘要，归入 `规范/非权威缓存与统计规范.md`。

## 1.1 第一轮确认口径

第一轮需求结算记录确认以下口径：

```text
第一轮不新增 `需求结算记录` 节点类型，使用需求结算实例状态作为权威结算记录锚点。
需求节点必须通过需求结算记录关系挂回每次结算状态节点。
每次结算状态节点必须绑定本次结算采用的目标状态和实际状态。
实际状态节点第一轮必须由调用方传入已有状态节点，可来自任务服务的实际结果状态或状态服务在任务结果阶段创建的状态节点；需求服务只复核，不在结算入口临时发明实际状态。
最近结算记录按结算状态发生时间戳最大判定，时间戳相同再按关系顺序号或结算状态节点创建序号较大判定；仍无法唯一时不得静默选择。
第一轮 I64 比较规则为：实际状态值大于等于目标状态值为已满足，实际状态值小于目标状态值为未满足，任一值缺失为证据不足。
```

## 3. 结算记录承载结论

| 信息 | 新项目承载 | 写入方 | 读取方 | 状态 |
| --- | --- | --- | --- | --- |
| 结算结果 | 需求结算实例状态或后续结算记录节点 | 需求服务经状态服务 | 需求服务、任务服务、诊断层 | 已确认 |
| 结算发生时间 | 需求结算实例状态发生时间戳 | 需求服务经状态服务 | 需求服务、诊断层 | 已确认 |
| 需求结算记录归属 | 需求节点 -> 结算状态节点关系 | 需求服务 | 需求服务、诊断层 | 已确认 |
| 结算来源任务 | 结算状态节点 -> 任务节点关系 | 需求服务 / 任务服务 | 需求服务、任务服务 | 已确认 |
| 结算目标状态 | 结算状态节点 -> 状态节点关系 | 需求服务 | 需求服务、任务服务 | 已确认 |
| 结算实际状态 | 结算状态节点 -> 状态节点关系 | 任务服务或状态服务先创建，需求服务复核并绑定 | 需求服务 | 已确认 |
| 结算动态 | 需求结算导致的状态变化或结算事件动态 | 需求服务经动态服务 | 因果服务、诊断层 | 已确认 |
| 结算因果引用 | 轻量因果引用或后续可靠因果 | 因果服务 | 需求服务、诊断层 | 已确认 |

第一轮不新增 `需求结算记录` 节点类型。若后续需要独立结算记录节点，必须先修订节点类型与关系类型枚举规范。

## 3.1 结算记录关系边界

第一轮权威结算记录至少包含以下关系：

| 关系角色 | 源节点 | 目标节点 | 唯一性 | 用途 |
| --- | --- | --- | --- | --- |
| 需求结算记录 | 需求节点 | 结算状态节点 | 可多 | 枚举该需求全部权威结算记录，并据发生时间戳读取最近记录 |
| 结算来源任务 | 结算状态节点 | 任务节点 | 每次结算唯一 | 追溯本次结算来自哪个任务 |
| 结算目标状态 | 结算状态节点 | 状态节点 | 每次结算唯一 | 固化本次结算采用的目标状态 |
| 结算实际状态 | 结算状态节点 | 状态节点 | 每次结算唯一 | 固化本次结算采用的实际状态 |
| 结算动态证据 | 结算状态节点 | 动态节点 | 第一轮至少一个 | 追溯本次结算的动态证据 |
| 结算因果引用 | 结算状态节点 | 因果引用节点 | 可选，可多 | 追溯轻量因果引用证据 |

`需求承接任务` 和 `结算来源任务` 必须拆开：

```text
需求承接任务：需求节点 -> 任务节点，表达需求有哪些任务承接。
结算来源任务：结算状态节点 -> 任务节点，表达本次结算来自哪个任务。
```

外部调用方不得直接用裸 `引用` 关系混读需求任务事实；需求承接任务、结算来源任务、结算目标状态、结算实际状态必须通过需求服务或任务服务的专用读取入口返回。

## 3.2 实际状态来源

第一轮 `记录需求结算` 必须接收已有实际状态节点。

实际状态节点允许来源：

```text
任务服务记录的任务实际结果状态。
状态服务在任务结果阶段创建的实例状态。
后续已确认的状态提升或结果状态入口。
```

需求服务在结算入口只复核实际状态节点有效、类型正确，并绑定为本次结算实际状态；不得在结算入口根据日志、控制台输出、非权威摘要或临时特征值材料直接发明实际状态节点。

## 3.3 最近结算记录规则

最近结算记录定义为：

```text
同一需求的全部需求结算记录关系目标中，结算实例状态发生时间戳最大的记录。
若发生时间戳相同，选择需求结算记录关系顺序号较大的记录。
若关系顺序号仍无法区分，选择结算状态节点创建序号较大的记录。
若仍无法唯一，返回空或冲突结果，不得静默选择第一个。
```

最近结算记录是权威结算记录读取规则，不等同于非权威最近结算摘要缓存。

## 3.4 第一轮比较规则

第一轮只确认 I64 目标状态比较规则：

```text
实际状态值 >= 目标状态值 -> 已满足。
实际状态值 < 目标状态值 -> 未满足。
目标状态值缺失或实际状态值缺失 -> 证据不足。
```

后续比较方式扩展必须进入需求系统或需求结算规则专项，不得临时代码硬扩。

## 4. 结算记录和非权威摘要边界

权威结算记录必须能回答：

```text
结算的是哪个需求。
该需求目标状态是什么。
结算时实际状态是什么。
来源任务是什么。
结算结果是什么。
结算发生时间戳是什么。
结算证据来自哪些状态、动态或因果引用。
```

非权威摘要只能回答：

```text
最近一次大致结算展示。
累计安全 / 服务结算展示。
诊断或兼容观察。
```

非权威摘要缺失、清空或重建，不得改变需求结算事实。
