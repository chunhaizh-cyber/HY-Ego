编号：动作入口-001
日期：2026-07-06
状态：已确认项目规范 / 第一轮动作入口角色方案 / 未改代码

## 1. 定位

动作入口负责回答“某个方法是否具备可调用的真实执行入口，以及该入口执行后如何成为动态来源候选”。

动作入口不是函数名、文本说明、日志标签或控制台命令。动作入口必须是稳定结构身份，并由方法服务或后续动作入口服务创建、读取、失效和追溯。

本规范覆盖主信息迁移 S4 中登记的 MI-M2 动作句柄。

## 4. 动作入口类型

第一轮动作入口类型建议：

| 类型 | 含义 | 是否可被普通任务主动选择 |
| --- | --- | --- |
| 未设置 | 无有效动作入口 | 否 |
| 本能方法入口 | 已有本能方法或内部方法可调用入口 | 是，需满足方法条件和任务方法关系 |
| 外部实现入口 | 外部实现主键对应的可调用入口 | 后续确认，第一轮不接外设和外部实现 |
| 动作序列入口 | 多步动作组合入口 | 后续确认，第一轮不压扁为单步入口 |
| 被动状态提交入口 | 生命周期状态成立后的被动提交入口 | 否，不进入普通候选方法集 |
| 被动整理入口 | 普通整理代码或诊断同步 | 否，不生成主链动作动态 |

动作入口类型不得由字符串自由扩展。新增类型必须先进入规范和详细设计。

方法候选召回入口必须默认排除被动状态提交入口和被动整理入口。只有明确进入被动证据提交流程时，才允许读取被动状态提交入口；被动整理入口不得生成主链动作动态。

## 5. 动作入口和动态来源

方法执行或领域服务写入可以成为动作动态来源候选，但不是所有写入都形成动作动态。

规则：

```text
主动方法真实调用并造成可记录事实时，应生成动作动态。
被动状态提交可以生成被动动作动态证据，但不能变成可主动选择动作。
被动整理动作不得进入普通候选方法集，不生成主链动作动态。
外部输入和线程不是动作来源。
通用特征写入逻辑不是动作来源；它只能接受外层语义方法作为来源。
```

动作动态应优先记录：

```text
来源方法
动作入口
发生场景
动作主体
被改变目标
改变前状态
改变后状态
发生时间戳
输入材料或参数场景
输出状态或结果场景
```

第一轮动态来源指向：

```text
`动态服务::记录实例动态` 的 `来源动作` 第一轮指向动作入口角色节点。
动作动态还应能追溯来源方法；来源方法与动作入口角色之间通过方法动作入口关系确认。
来源动作不得指向线程、函数名、文本动作名、控制台命令或日志标签。
若后续新增独立动作入口节点类型，必须先修订动作入口规范、动态服务详细设计和节点类型枚举规范。
```

动作动态和轻量因果引用边界：

```text
动作动态负责保存具体动作证据。
因果引用只以来源动态作为准入证据，不负责保存全部动作细节。
不得把因果引用节点当成动作动态的替代物。
```

## 6. 动作入口和方法执行权限

方法执行权限来自：

```text
任务授权
任务承接
任务选择方法
```

动作入口不恢复方法执行前授权状态壳。动作入口只证明方法具备可调用入口，不证明本轮任务已经允许执行。

执行前至少必须有：

```text
有效任务节点。
有效方法节点。
任务到方法关系。
有效动作入口。
方法条件满足或当前切片明确暂缓条件检查。
需要写特征状态材料时，必须经特征服务。
```
