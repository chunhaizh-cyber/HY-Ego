# 6360 子规范 相机外设综合工作流程

编号：6360
日期：20260720
状态：正式子规范

## 1. 定位

本规范收束双目相机类感知外设从治理需求、观察约束、外设供料、任务承接到观察、识别、扫描、跟踪和场景事实提交的综合流程。

上级规范：0050、1130—1190、2100、3100—3300、6200—6350、6370。

## 2. 综合主链

```text
内部治理、任务验证或场景维护需要当前事实
-> 形成扫描、观察、识别或跟踪需求
-> 任务承接并筹办，冻结合法方法和观察约束
-> 外设等待项进入相机独立控制域
-> 相机形成原始逐簇、稳定观察子集、扫描变化或目标跟踪逻辑供给产品
-> 任务域从强类型报告队列按产品、任务、范围、时效和质量承接
-> 当前冻结方法读取本工作项已经承接的任务运行材料
-> 领域服务提交特征、状态、动态、存在对应或场景事实
-> 任务验证实际结果并决定完成、等待或重新筹办
```

外设线程、报告、队列项、材料页和观察约束均不是事实主体。相机中间层不得被自我侧方法省略，也不得内联到观察、识别、扫描或跟踪执行体中现场制造材料。

外设报告队列的正式消费者只能是任务域承接层。自我线程、需求根、普通任务工作线程和业务方法不得直接轮询相机高频流或物理报告队列。业务方法取得的是绑定任务、工作项、产品类型和消费窗口的已承接材料引用。

## 3. 自我侧与外设侧

自我侧负责为什么看、看什么、采用哪种方法、结果是否成为正式事实。外设侧负责按照等待项稳定供给材料、说明质量并隔离高频工程过程。

相机外设内部可以使用深度稳定化、跨轮轨迹、SLAM、TSDF、OctoMap、VOS 等增强模块，但必须保持：

- 原始深度不被滤波结果覆盖；
- 外设轨迹 ID 不成为存在 ID；
- 地图、位姿和表面模型只作外设材料；
- 指定目标的高成本模型不成为默认全场路径；
- 可选模块缺失不改变主链事实权力。

## 3.1 成熟度与供给产品二轴

相机材料成熟度和逻辑供给产品是两个独立维度：

```text
成熟度轴：L0 原始 -> L1 单轮稳定化 -> L2 跨轮关联 -> L3 稳定观察子集

产品轴：原始逐簇报告 / 稳定观察子集报告 / 扫描变化报告 / 目标跟踪报告
```

原始逐簇和稳定观察子集属于逐簇识别外显模式的两个供给阶段；扫描变化和目标跟踪分别对应另两种外显模式。因此对外仍只有三种观察运行模式，但可以提供四类逻辑数据产品。

四类产品不等于四个物理容器。一个强类型报告队列加产品索引和分类视图可以满足四类供给；分队列实现也必须保持相同的来源继承、时效、一次消费、任务绑定和禁止跨类替代规则。

## 4. 扫描的主动治理位置

当前治理主动产生的主要感知需求是扫描：安全判断、服务增加、任务结果验证或当前场景维护需要读取已确认存在的当前状态时，生成扫描需求。

扫描输入必须包括：

- 自我内部场景稳定句柄；
- 本轮应覆盖存在集合和扫描关注集合；
- 已合法承接的外设当前数据集；
- 当前对应关系和特征值基准；
- 比较规则、阈值来源和可选观察约束。

扫描只处理已归属、可比较、优先级达标且与本轮任务、安全或服务相关的存在。范围外、视野外、低频跳过、跟踪排除、优先级不足和条件不足不得解释为消失或遮挡。

没有可比历史时，扫描只能建立当前特征值和状态基准，不得生成状态变化。扫描发现未归属材料、缺特征或对应冲突时，不得直接调用其它方法。

## 5. 四类方法职责

| 方法 | 职责 | 语义输出 |
| --- | --- | --- |
| 观察 | 从合法外设材料形成观察像素簇、候选材料并补充合法一阶观察特征 | 新观察材料、特征或结构化缺口 |
| 识别 | 在确定范围内用已有特征收敛已有存在 | 已有存在稳定句柄或空句柄 |
| 扫描 | 比较已确认存在的当前值与基准，维护当前状态 | 当前值、状态、变化或未覆盖解释 |
| 跟踪 | 在已有目标身份下取得目标的新空间坐标 | 新坐标、跟踪状态或空信息 |

四者不是函数直连关系，也不能成为彼此的子步骤。缺口只能形成：

```text
派生需求类型
派生需求目标
派生需求来源证据
```

三件套经需求统一入口、任务承接和新一轮筹办后，才可能选择后续方法。无候选方法时不得自动创建方法首。

## 6. 观察材料到正式存在

```text
单轮原始材料
-> 跨轮稳定外设观察材料
-> 观察像素簇
-> 存在候选
-> 识别返回已有存在，或新存在承接与复验
-> 已确认观察存在
```

稳定观察子集报告只允许进入任务承接和观察方法，不得直接进入识别、扫描或跟踪的正式判断。识别只消费已经由自我侧合法形成的存在材料、特征和候选范围；扫描和跟踪除各自定向外设产品外，还必须读取自我侧已确认存在、特征、对应关系以及可比基准或目标约束。

扫描变化报告和目标跟踪报告是基于 L3 来源材料、已确认存在及各自基准或目标约束形成的定向产品。它们可以由任务域分别交给扫描和跟踪方法，但报告本身仍不等于状态变化或新空间事实；方法必须完成业务准入、实际比较和领域提交。识别没有独立外设报告直通权，只读取观察链已经形成的已有特征和候选范围。

视觉工程中的背景必须归为静止存在、环境存在、低动态存在、待识别材料或低质量材料，不能成为自我认知中的“背景存在”。过滤只降低高成本处理频率，不得把存在从认知链中删除。

## 7. 观察特征优先级和轮廓

普通观察优先使用平面轮廓；只有平面轮廓不够唯一，或任务涉及空间操作、碰撞、支撑、抓取、距离和三维占位时，才升级空间极值轮廓或局部三维模型。

平面轮廓和空间极值轮廓由特征类型解释坐标维数、中心点和边界点顺序。底层向量编码不得进入业务语义。匹配必须由平均误差、最大误差、重合率、尺寸偏差和综合评分等量化二次特征推出。

空间位置只用于范围过滤、反证和跟踪辅助，不能代替轮廓、尺寸或子结构成为普通识别唯一依据。

## 8. 扫描写入边界

扫描可以更新已确认存在的当前特征值和当前状态，但不能从原始外设材料新建长期一阶观察特征。需要把新值加入长期特征集合时，必须形成观察需求并由观察方法提交。

扫描未覆盖集合按以下方式计算：

```text
本轮应覆盖存在集合 - 本轮合法外设覆盖存在集合
```

未覆盖分类只进入本轮扫描运行事实，不直接改变存在长期状态。图像相对不动也不能直接证明存在静止；必须在场景坐标、空间坐标或相机位姿补偿后的坐标中满足量化稳定条件。

## 9. 目标观察约束

目标观察约束可以规定范围、目标存在、期望特征、质量、时效和误差预算，用于指导相机供料和扫描优先级。它不能：

- 代替需求、任务或执行冻结；
- 直接确认存在或状态；
- 要求相机反向创建任务；
- 变成新的事实主体；
- 让相机为某一固定方法永久运行。

若约束缺失或材料不足，自我侧方法返回结构化缺口，不能直接调用中间层补造。

## 10. 合法来源

所有特征值、阈值、枚举和状态变化必须有可回查来源。阈值可来源于特征类型默认值域、外设不确定度、存在历史稳定值域、任务容差、安全或服务约束以及合法外设学习结果。

枚举状态必须由量化规则推出。状态变化必须关联输入状态、输出状态、来源方法、来源任务和发生时间；外设消息或说明文本不能替代这些事实。

## 11. 实时性

已经观察完成的区域不重复完整观察；静止存在由低频扫描维护；扫描只处理本轮候选；跟踪只处理指定目标；无需求命中的材料只缓存、摘要或丢弃。

安全事件、当前任务回执、状态跃迁、跟踪目标丢失、外设错误和影响目标判断的结构化缺口不得舍弃。

## 12. 禁止规则

1. 不得让外设线程直接写需求、任务、场景事实或价值。
2. 不得让自我侧直接消费高频原始流。
3. 不得把报告、材料页、约束、ROI、掩码或外设轨迹作为事实主体。
4. 不得把扫描或跟踪扩张成新一阶特征裸写入口。
5. 不得把识别扩张成全特征比对或新存在创建。
6. 不得把当前范围唯一扩张成全世界唯一。
7. 不得把控制面板显示、模拟样本或夹具结果当作真实链路验收。
8. 不得把无需求命中的消息上报成正式认知。
9. 不得把图像相对稳定直接写成存在静止。
10. 执行期发现冻结前应具备的方法结构缺口，必须按筹办漏检追根因。

## 13. 验证与完成边界

验证必须覆盖需求到等待项、L0—L3 成熟度、四类逻辑供给产品、任务域唯一队列消费、产品到冻结方法的类型匹配、四方法隔离、派生需求三件套、正式场景提交、阈值来源和真实样本。

没有真实日志、材料快照、方法动态、状态变化和领域提交证据时，不得宣称相机综合认知链已经实现。
