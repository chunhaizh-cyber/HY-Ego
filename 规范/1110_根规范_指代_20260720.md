# 1110 根规范 指代

编号：1110
日期：20260720
状态：正式根规范

## 1. 根定位

指代是在确定表达上下文和解析版本内，把一个语言位置或语素入口指向一个当前可读项目对象的引用事实。指代文本不是对象身份。

## 2. 本项目承载

持久指代由来源语言身份、目标稳定句柄、引用关系、表达上下文、指代序号、活动解析版本、作用域和当前性共同承载。短生命周期指代可以使用值式材料，但不得脱离活动版本成为长期事实。

## 3. 成立边界

1. 来源、目标、上下文、序号和活动版本必须完整。
2. 目标必须是当前可读且角色允许的项目对象。
3. 同一文本在不同上下文可以指向不同对象；不得建立全局“文本等于对象”规则。
4. 指代冲突、多候选、变化和消解必须由关系、状态、二次特征、动态或解析证据表达，不压成自由文本或 bool。

## 4. 写入和生命周期

指代只能由语言/概念解析领域在当前解析上下文中建立或发布。过期活动图版本、零上下文、零序号和失效目标必须拒绝。

解析更新时旧指代保留历史但退出当前发布；临时指代不得自动持久化。

## 5. 禁止事项

禁止按代词、显示名、词面或日志直接取得存在身份；禁止把指代成立解释为目标对象本体状态或业务结论。
