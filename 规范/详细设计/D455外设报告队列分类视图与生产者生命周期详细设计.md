# D455 外设报告队列分类视图与生产者生命周期详细设计

日期：2026-07-21

版本：v0.1

状态：正式详细设计 / DD-02 / 队列与生产控制域设计完成 / 代码未实现 / 不授权生产接线

## 1. 设计目标

本设计承接 DD-01 的强类型 D455 观察供给产品，固定从“合法产品值”到“任务域可唯一承接报告”之间的实现合同：

```text
一个物理强类型报告队列
+ 四类产品索引
+ 四类非消费只读视图
+ 受控材料保留域
+ 每设备唯一生产控制域和生产者世代
+ 任务域唯一消费权
```

本设计必须回答：

1. 四类逻辑供给产品如何共用一个物理队列而不互相替代。
2. 报告、队列项、材料保留权和生产者世代分别由谁拥有。
3. 发布如何保证“产品可见”和“队列项存在”是同一个确认点。
4. 任务域如何成为唯一正式消费者，而只读视图不取得消费权。
5. 队列满、材料过期、重复、断流、目标丢失和停止时如何具名收口。
6. 同一 D455 实例如何只有一个有效生产控制域，同时不持久化裸线程编号。

本设计不实现 L0—L3 算法、不启动真实 RealSense 常驻线程、不匹配任务、不建立冻结工作包，也不提交世界事实。

## 4. 总体架构裁决

### 4.1 物理队列数量

固定采用：

```text
一个 `D455观察供给报告队列`
-> 一个按发布顺序维护的物理活动报告序列
-> 四个按产品类型维护的身份索引
-> 四个只读分类视图
```

不建立四个独立物理队列。理由是四类产品共享同一设备世代、容量、材料保留、发布原子点、一次消费状态和停止边界；拆成四个物理队列会额外产生跨队列容量、停止和唯一消费一致性问题。

四类索引只是同一报告记录的身份目录，不复制产品、不复制材料保留权，也不形成第二消费位置。

### 4.2 模块分层

```text
DD-01 观察供给协议
-> DD-02 受控材料域
-> DD-02 报告流转协议
-> DD-02 强类型报告队列
-> DD-02 观察供给生产者
-> DD-03 任务域唯一承接
```

报告队列不 import 任务、需求、方法、仓库或领域服务。任务身份匹配、等待项匹配和已承接材料引用由 DD-03 负责。

## 6. 报告和生产者强类型

### 6.1 报告身份

```text
D455报告身份
- 来源设备身份摘要
- 生产者世代
- 报告编号
- 报告版本
```

报告编号在同一生产者世代内从 1 单调增长且不复用。新生产者世代可以从 1 重新编号，但完整身份仍不同。报告身份不成为世界存在身份、任务身份或方法身份。

### 6.2 报告状态

```text
enum class D455报告状态
未知
未发布候选
已发布待承接
任务域保留中
已正式承接
仅缓存
降级处理
摘要保留
已丢弃
已过期
已撤销
```

状态属于队列流转记录，不写入节点仓库。活动队列只保留 `未发布候选 / 已发布待承接 / 任务域保留中`；终结状态只形成有界值式回执和统计，不继续占用活动容量。

固定状态迁移：

```text
未发布候选 -> 已发布待承接
未发布候选 -> 已撤销
已发布待承接 -> 任务域保留中
已发布待承接 -> 已过期（仅允许可舍弃产品）
任务域保留中 -> 已发布待承接（释放且仍当前）
任务域保留中 -> 已过期（释放时已经过期且允许舍弃）
任务域保留中 -> 已正式承接 / 仅缓存 / 降级处理 / 摘要保留 / 已丢弃 / 已过期
```

其它迁移属于内部逻辑错误。

### 6.3 投递策略

```text
enum class D455报告投递策略
可过期舍弃
只允许摘要收口
不可舍弃
```

`D455报告投递约束` 至少包含产品类型、优先级、策略、最大排队年龄和强类型原因。目标丢失产品必须是不可舍弃；普通过期扫描变化、重复中间材料和无等待项原始逐簇可以是可舍弃或摘要收口。

外设错误不伪装成四类观察供给产品。DD-02 把错误锁存在生产者强类型故障快照中并阻断继续生产；DD-06 后续决定控制域如何把故障报告交给任务 / 安全治理。故障不得仅写日志后丢失。

### 6.4 生产者身份与状态

```text
D455生产者身份
- 来源设备身份摘要
- 生产控制域编号
- 生产者世代

enum class D455生产者状态
未启动
启动中
运行中
背压等待投递
断流恢复中
正在停止
已停止
故障
```

生产者快照至少包含身份、状态、当前队列世代、待投递报告身份、背压原因、断流原因、恢复次数、故障原因、最近状态迁移时间和是否仍持有设备占用权。不得包含裸线程编号。

## 7. 受控材料域

### 7.1 材料域路由

`D455受控材料引用` 中的材料域编号只负责选择唯一来源适配器。材料域路由至少执行：

```text
按材料域编号定位唯一适配器
-> 验证材料种类、编号、版本、批次和摘要
-> 请求来源适配器取得不可伪造的只读保留权
-> 精确读回材料元数据
-> 返回保留权和元数据，不返回裸缓存对象
```

同一材料域编号只能登记一个活动适配器。重复登记、材料域代次不一致或适配器返回不同元数据必须拒绝或追根因。

### 7.2 材料保留权

`D455材料保留权` 是移动独占、不可复制、由来源适配器私有构造的运行期能力。它至少绑定：

```text
材料域编号和代次
材料编号和版本
材料种类
来源批次和摘要
只读寿命载体
```

报告候选持有全部保留权，才允许进入确认发布。保留权析构只释放运行期只读材料寿命，不改变世界事实。

### 7.3 当前帧缓存适配

`D455帧缓存材料域适配器` 对当前 `D455帧材料缓存` 执行：

```text
接收 D455帧材料句柄
-> 按编号版本读取已发布同步帧
-> 验证设备、批次、时间和摘要
-> 形成材料种类=同步帧的 D455受控材料引用
-> 以缓存返回的只读共享快照作为内部寿命载体
-> 返回不透明材料保留权
```

即使原缓存随后精确撤销，只要报告或已承接材料仍持有保留权，该只读快照仍可在消费窗口内存活。该适配不修改现有缓存，不把缓存本身升级为正式报告队列，也不向业务方法暴露原缓存入口。

### 7.4 全引用保留

发布前必须从产品头和负载收集全部 `D455受控材料引用`，按完整引用去重后逐个保留。出现任一情况不得准备报告候选：

- 引用无效、域不存在、版本过期或种类不匹配；
- 引用元数据与产品来源链不一致；
- 同一完整引用读回两个不同摘要；
- 产品声明的引用没有对应保留权；
- 保留权存在但产品没有该引用。

部分保留成功后出现后续失败，已取得的保留权全部自动释放。

## 8. 一个物理队列、四类索引和四类视图

### 8.1 物理记录

每个活动报告记录至少包含：

```text
报告身份
不可变 D455观察供给产品值
内容摘要
报告状态
投递约束
发布时间和最大年龄
全部材料保留权
候选能力代次或承接能力代次
```

物理记录由报告队列唯一写入。候选、报告身份、内容摘要和材料保留权必须在同一互斥域内核对。

### 8.2 四类索引

四个索引分别保存活动报告身份：

```text
原始逐簇报告索引
稳定观察子集报告索引
扫描变化报告索引
目标跟踪报告索引
```

确认发布时在同一锁内完成物理发布序列和唯一对应索引的插入。终结时在同一锁内从物理活动序列及其产品索引移除。一个报告不得同时进入两个产品索引。

### 8.3 非消费只读视图

四类视图只返回值式视图快照：报告身份、产品头、状态、发布时间、时效、质量摘要、投递策略和产品值式副本。视图：

- 不改变报告状态；
- 不取得材料保留权；
- 不延长任务消费窗口；
- 不分配消费游标；
- 不形成任务匹配或世界事实；
- 不保证视图返回后受控引用仍可解析。

控制面板或外设诊断可以读取视图，但不得调用认领、承接或终结入口。

## 9. 报告候选和原子发布

### 9.1 固定发布顺序

```text
验证 DD-01 产品值式合同
-> 收集并保留全部受控材料引用
-> 验证投递约束和生产者身份
-> 原子预留队列容量和报告身份
-> 建立未发布候选
-> 精确读取候选
-> 原子确认发布并写入唯一产品索引
-> 按身份读取已发布报告
```

报告身份由队列绑定的生产者世代分配。外部调用方不能自选或复用报告编号。

### 9.2 候选能力

`D455报告候选` 是移动独占能力，绑定队列地址、队列世代、报告身份和候选代次。候选不可复制；析构时若尚未确认，必须在队列锁内精确撤销候选、释放容量和材料保留权。

候选不进入物理发布序列和四类索引，任何只读视图和任务域消费入口都不可见。

### 9.3 确认点

`确认发布D455报告` 在同一锁内验证：

```text
候选能力仍属于本队列和本世代
候选状态仍为未发布
产品值、摘要、身份和材料保留权与候选读回一致
物理序列和四类索引当前没有同身份项
```

全部成立后才同时改变状态为 `已发布待承接`、写入物理序列和唯一产品索引。调用返回后才对任务域可见，因此不存在“产品已发布但没有队列项”的合法中间状态。

确认前任一步失败都必须精确撤销。确认后普通读回出现身份、产品、状态、索引或材料保留不一致属于内部逻辑错误：生产者进入故障或隔离状态并停止依赖路径，不得把另一份读回当成功，也不得用日志修复。

## 10. 任务域唯一消费权和精确认领

### 10.1 唯一消费权

报告队列只允许在尚未发布任何报告且尚未签发时调用一次 `签发D455任务域唯一消费权`。返回的 `D455任务域消费权`：

- 移动独占、不可复制；
- 绑定队列地址、队列世代和消费权代次；
- 不记录裸任务线程编号；
- 只允许 DD-03 任务域所有者长期持有；
- 析构不重新开放第二次签发，队列必须停止并建立新世代才能更换正式消费者。

只读视图不需要消费权，但任何认领、释放、正式承接或具名终结都必须同时验证消费权和报告候选能力。

### 10.2 精确认领

任务域从分类视图或自身等待项匹配结果取得报告身份后，调用：

```text
准备按身份承接D455报告
```

成功后报告从 `已发布待承接` 原子迁移到 `任务域保留中`，并返回移动独占 `D455报告承接候选`。同一报告同时只允许一个有效承接候选；其它路径不能读走或终结该报告。

候选允许 DD-03 只读产品值、材料元数据和时效，然后三选一：

```text
确认正式承接
释放回已发布待承接
按强类型处置原因终结
```

### 10.3 正式承接交接包

确认正式承接时，队列必须先构造不可失败移动的 `D455报告承接交接包`，再在同一锁内：

```text
从活动序列和产品索引移除报告
-> 把全部材料保留权移动到交接包
-> 形成已正式承接回执
-> 释放活动容量
```

交接包由 DD-03 放入绑定任务和工作项的已承接材料引用。交接包析构表示该消费窗口结束并释放材料寿命，不回滚为未消费报告。

DD-02 不接收任务句柄、工作项身份或方法身份；它只保证报告和材料保留权被唯一交给任务域一次。

## 11. 生产控制域和生产者生命周期

### 11.1 每设备唯一占用

运行期组合器以后必须显式拥有一个 `D455生产控制域登记器`。`取得D455生产者占用` 在登记器互斥域内按稳定设备身份裁决：

```text
同一设备没有活动占用 -> 分配新生产控制域编号和生产者世代，返回移动独占占用权
同一设备已有活动占用 -> 逻辑内拒绝，不进入启动
不同设备 -> 可以分别取得占用并并发生产
```

占用事实只存在于登记器和移动能力中。线程只是持有生产者对象的执行载体，不参与唯一性比较。

### 11.2 启动

```text
未启动
-> 验证设备占用权、队列配置、材料域和唯一消费权已经就绪
-> 启动中
-> 绑定队列世代和生产者世代
-> 运行中
```

DD-02 的启动不打开 RealSense、不创建常驻 OS 线程；它只建立 SDK 无关生产控制结构。真实采集线程和驱动打开顺序由 DD-06 负责。

### 11.3 正常投递和背压

运行中接收一个已经形成的产品和投递约束，按第 9 节发布。队列满时：

- `可过期舍弃`：返回具名 `队列积压 / 已丢弃` 结果，释放本次材料保留，不改变世界事实；
- `只允许摘要收口`：形成强类型摘要回执后释放完整材料，不能只写说明文本；
- `不可舍弃`：保存唯一待投递报告包，状态转为 `背压等待投递`，停止接受新的普通产品，不覆盖待投递包。

目标丢失产品必须走不可舍弃分支。背压解除后只能重试同一报告包；成功后返回运行中。若待投递包身份或内容在重试间变化，属于内部逻辑错误。

### 11.4 断流恢复

未来采样适配器发现超时、设备断开或来源失效时调用具名断流入口：

```text
运行中 / 背压等待投递
-> 断流恢复中
-> 记录强类型原因、发生时间和恢复次数
-> 外部采样控制确认来源重新打开且设备身份相同
-> 运行中
```

设备身份不同、生产者世代变化或恢复次数耗尽时进入 `故障`。DD-02 不自行重开驱动，不用休眠循环伪造恢复。

### 11.5 停止

停止顺序固定为：

```text
任一非终态
-> 正在停止
-> 拒绝新产品和新候选
-> 撤销全部未发布候选
-> 等待任务域已认领候选完成或释放
-> 对剩余可舍弃报告具名过期 / 丢弃
-> 不可舍弃报告必须已承接，或由宿主提供强类型强制终结原因
-> 停止队列并释放设备占用权
-> 已停止
```

宿主关闭可以成为强制终结原因，但必须形成强类型终结回执；不得静默清空不可舍弃报告。停止失败或队列 / 材料计数不一致进入 `故障` 并追根因。

## 12. 过期、重复和处置

### 12.1 过期

可过期舍弃报告在未认领且超过最大排队年龄时，可以在队列锁内迁移为 `已过期`、从活动序列和产品索引移除并释放材料。不可舍弃报告即使超过业务新鲜度也不自动删除；任务域必须读取其“已过新鲜窗口”材料并具名承接、降级或终结。

### 12.2 重复

同一完整报告身份：

- 内容摘要和全部产品值相同：只返回 `精确重复` 读回，不新增队列项、不延长材料寿命；
- 内容或材料来源不同：内部逻辑错误，生产者故障；
- 不同报告身份但产品内容相同：允许作为不同采样时点产品存在，由 DD-03 的等待项和时效规则决定是否重复处置。

幂等不能给第二个生产者或第二个消费者授予执行权。

### 12.3 具名终结

任务域终结至少区分：

```text
仅缓存
降级处理
摘要保留
已丢弃：过期 / 重复 / 无命中 / 队列积压 / 低优先级
已过期：承接前过期 / 保留期间过期
```

处置状态和原因是强类型值；日志和显示只解释，不参与裁决。

## 13. 并发和锁边界

1. 材料域路由有独立互斥域，只保护适配器登记和材料保留元数据。
2. 报告队列有一个互斥域，保护容量、候选、物理活动序列、四类索引、消费权签发、承接候选和统计。
3. 生产控制域登记器有独立互斥域，只保护设备占用和生产者世代。
4. 不在持有队列锁时调用外部材料适配器、任务域回调、日志、控制面板或领域服务。
5. 发布先在锁外完成产品验证和材料保留，再在队列锁内做候选和确认；失败时按能力精确撤销。
6. 同一报告的候选发布和承接能力均为移动独占；不同报告可以由同一生产者串行发布，DD-06 是否多算法并发后置。
7. 四类只读视图在锁内形成值式快照后立即释放锁，不持锁渲染或匹配。

## 14. 入口拒绝、逻辑内返回和内部逻辑错误

| 阶段 | 逻辑内拒绝 / 返回 | 内部逻辑错误 |
| --- | --- | --- |
| 生产者占用 | 设备身份无效、同设备已有生产者 | 同设备出现两个有效占用权、世代倒退或复用 |
| 材料保留 | 域不存在、引用过期、种类不符、材料已释放 | 同引用读回两个摘要、保留权与元数据不一致 |
| 候选准备前 | 产品合同无效、生产者非运行、普通报告队列满、产品已过期 | 同报告身份不同内容、计数或身份分配器溢出 |
| 候选准备后 | 调用方主动撤销 | 候选读回与输入产品、材料保留或投递约束不一致 |
| 确认发布 | 候选能力过期或属于其它队列 | 确认成功后物理序列、索引或普通读回缺失 / 不一致 |
| 分类视图 | 无该类型活动报告返回空集合 | 一个报告进入两个索引、索引指向不存在记录 |
| 精确认领 | 报告不存在、已被认领、已过期或消费权无效 | 同一报告出现两个有效承接候选 |
| 释放 / 终结 | 释放时已过期按规则转过期 | 候选身份、状态、材料交接或容量计数矛盾 |
| 生命周期 | 重复启动、未运行投递、恢复条件不足 | 不允许状态跳转、不可舍弃待投递包被覆盖或静默清除 |

内部逻辑错误必须停止依赖路径并进入故障 / 隔离状态。公共诊断宏开启时写逻辑错误日志并弹窗；宏关闭不改变业务行为。不得由扫描器、日志或显示自动修复队列。

## 17. 后继边界

### 17.1 DD-03 承接

DD-03 负责：

```text
用唯一消费权读取报告候选
-> 重新验证等待项、任务、工作项、产品类型、范围、质量和时效
-> 形成绑定任务和工作项的已承接外设材料引用
-> 接收并持有 D455报告承接交接包
```

DD-02 不判断哪个任务应取得报告，不把报告身份当任务身份，也不签发方法执行权。

### 17.2 DD-04—DD-06 承接

- DD-04 决定冻结工作包如何只携带 DD-03 已承接材料引用。
- DD-05 决定方法结果如何经领域服务提交并回流任务验证。
- DD-06 负责真实生产宿主、OS 线程、采集器、算法生产者、任务域消费者和停止顺序装配。

### 17.3 真实设备边界

DD-02 自检只使用 SDK 无关值式产品、受控假材料来源和当前帧缓存适配器。真实 D455 连续生产、断开重连和三模式供料由 DD-07 验收；不能用 DD-02 自检宣称外设已接入。
