## 1. 目标与边界

本设计冻结从已结算方法用途观察到不可变新代际的唯一合法路径。用途观察、统计和学习候选均不是方法事实；候选不能自证晋级，方法内容不能原地修改，新代际不能进入产生它的同一任务回合。

## 2. 结构职责

| 结构 | 身份 | 写入 / 读取 | 生命周期、并发、恢复 |
| --- | --- | --- | --- |
| 用途观察 | 方法代际、任务运行、确定观察组、实际结果、目标比较、场景版本 | 用途观察服务写；统计读取 | 正式观察持久；按观察身份幂等 |
| 观察统计 | 完整因果结构键、适用范围、统计规则版本、尝试未达成样本数 / 正负结果数 / 派生比例、最终已结算样本数 / 正负结果数 / 派生比例 | 统计服务写；候选服务读 | 可清空重建；待结算排除；不承担方法事实 |
| 学习候选 | 来源方法代际、观察组、候选稳定键、候选规则版本、证据范围、新增 / 保留 / 移除 / 替换 / 未知 / 冲突六类确定差异 | 候选服务写；晋级服务读 | 非权威材料；同键异义拒绝 |
| 晋级授权 | 候选键、授权规则、授权方、有效期、目标代际 | 独立晋级服务写；方法域读 | 正式授权持久；过期不可使用 |
| 新代际 | 稳定方法身份、新内容版本、父代、条件、结果、动作、来源、发布代次、来源任务回合 | 方法发布入口唯一发布；各任务取得筹办权后权威重读 | 不可变持久；发布前已冻结回合本轮不可见；发布后无活动旧冻结的新任务首次筹办可见；同一来源任务回合不可见 |

观察统计严格使用 4330 的分组与分栏字段；六类差异只属于 5340 的学习候选合同。未知或冲突差异只返回具名拒绝，不能被默认归入可晋级。

## 3. 晋级算法

```text
4330 提供已发布、结构合法的用途观察
-> 按完整因果结构键、适用范围和统计规则版本分组
-> 尝试未达成与最终已结算分别统计样本数、正负结果数和派生比例；待结算排除
-> 候选服务基于完整方法快照和确定观察组形成非权威学习候选
-> 对候选变更形成新增 / 保留 / 移除 / 替换 / 未知 / 冲突六类确定差异
-> 独立晋级服务校验规则、证据、授权和目标代际
-> 未授权 / 证据不足 / 冲突：逻辑内返回
-> 授权通过：方法领域建立不可变新代际
-> 完整读回父代、条件、结果、动作和版本
-> 记录新代际发布代次及来源任务回合
-> 每个任务按当前冻结快照、发布代次与来源任务回合分别裁决
-> 发布前已冻结回合及来源任务产生代际的同一回合继续不可见
-> 发布后建立或没有活动旧冻结的新任务，在首次取得筹办权并权威重读时可按条件召回
-> 旧回合结束后重新进入后继新筹办轮次的任务可按条件召回
```

同一父代和目标版本出现两个异义候选时拒绝发布。发布前置通过后新代际读回缺项、父代被覆盖或出现两个“当前”代际，属于内部错误。

## 5. 恢复与验证

- 统计和候选可从正式用途观察重建；清空不改变方法事实。
- 晋级授权和新代际作为正式记录恢复；恢复过程不得自动重新晋级。
- 后继可见性记录新代际发布代次和来源任务回合，并逐任务读取当前冻结快照：发布前已冻结回合及来源任务产生代际的同一回合本轮不可见；旧回合结束后必须重新取得筹办权并权威重读；发布后建立或没有活动旧冻结的新任务，在首次取得筹办权并权威重读时可按条件召回。不得用一个共享“最早可见筹办轮次”、线程时间或缓存命中裁决全部任务。
- 验证至少覆盖候选非权威、授权拒绝、授权过期、同义重放、异义冲突、父代保留、新代际不可变、同轮不可见、后继轮可见。
