## 4. 服务边界

### 4.1 特征服务

第三批只允许新增只读语义解析：

```text
由特征值节点反查唯一归属特征。
若归属特征是实例槽位，则读取唯一模板特征定义。
返回归属特征、语义特征定义、特征值和当前原始材料。
再次解析可判断值式材料是否仍为当前。
```

特征值服务继续只被特征服务直接访问。动态模式服务不得包含或调用特征值服务。

### 4.2 动态服务

动态服务继续拥有实例动态记录、完整动态材料和轻量引用阻断读取。第三批可以补只读当前性材料，但不得让动态服务自动扫描场景并裁决聚类事实，也不得写抽象动态。

### 4.3 动态模式服务 / 算法

建议新增独立值式编排层：

```text
动态模式服务：组合动态服务、特征服务和二次特征服务的只读材料，负责入口准入、当前性和返回分类。
动态模式算法：只接收完整不可变 DTO，负责规范结构键、稳定排序、片段全量比较和重复分组。
```

算法不得持有仓库 / 服务引用，不得读日志、时间源或线程完成顺序，不得通过哈希相等替代完整结构比较。

### 4.4 因果服务 / 因果模式服务

现有因果服务继续保持轻量因果引用入口。第三批建议把跨回合机会 / 结果计算放在独立因果模式服务 / 算法，不改义 `记录因果引用(来源动态)`，也不把单动态候选别名冒充概率模板。

### 4.5 统计、任务、需求和方法

本批代码切片不修改统计、任务、需求或方法服务。#235 只读扫描后才能裁决：

```text
谁重读任务结果、完成和需求结算证据。
谁生成一次性用途观察幂等键。
概率候选和结果效用如何分开。
统计服务是否只保存可清空观察缓存。
方法学习是否需要新的领域编排服务。
```

## 5. 输入与签名边界

### 5.1 显式观察回合与片段

候选输入不使用“扫描场景内全部动态后自行切段”。第一版由正式调用方提交属于某个观察回合的显式片段：

```text
回合编号 / 尝试编号。
回合内非零片段序号。
来源任务和执行尝试材料，若 #231 复核后存在。
非空动态句柄组。
回合规则版本。
```

同一回合可以提交多个片段，但片段序号必须唯一并确定先后，同一动态不得跨片段重复。每个片段内部动态必须同场景、同主体，按 `发生时间戳 -> 完整动态句柄` 确定性排序。相同时间戳允许存在，但完整句柄负责稳定次序。

### 5.2 规范动态步

第一版结构键至少包含：

```text
语义特征定义完整句柄。
前 I64 值。
后 I64 值。
来源动作是否存在及其完整句柄。
可明确复核的二次特征上下文结构键。
规则版本。
```

场景、主体、发生时间戳和来源动态句柄保留在证据中，但不进入跨实例语义相等键。二次特征上下文只有在 #231 证明角色顺序稳定时才能进入键；否则保持空并由 #232 实施记录明确，不得按容器遍历顺序猜角色。

### 5.3 结构键与哈希

完整结构键裁决相等，哈希只用于候选分桶：

```text
结构键全字段相同 -> 可进入同组。
哈希相同但结构键不同 -> 分开候选。
规则版本不同 -> 不可比较。
来源材料过期 -> 本次候选逻辑内返回，不用缓存兜底。
```

## 6. 聚类与运动基元边界

```text
一个完整显式片段可形成单实例片段候选。
至少两个不同观察回合各有一个完全相同的规范片段键，才形成重复聚类候选。
运动基元候选要求重复聚类成立，且片段至少一个步骤具有有效来源动作。
同一回合不能重复计为两个支持回合。
第一版只比较调用方显式提交的完整片段，不从片段内部做任意子序列搜索，不使用隐藏时间阈值。
```

聚类向上只形成候选材料；本批不执行向下抽象，不创建抽象动态、二次特征、方法、概念或因果节点。

## 7. 因果概率候选边界

因果请求显式提供不同的原因模式键、结果模式键和因果观察回合组。每个因果观察回合由同一回合身份下、按片段序号排列的 #232 当前窗口组成；服务只按完整窗口键匹配，不在动态步内部搜索。每个回合只允许：

```text
原因模式在窗口组中出现零次或一次。
结果模式在原因片段之后出现零次或一次。
原因零次的回合不计机会。
原因一次且后续结果一次：计 1 次机会、1 次结果。
原因一次且后续结果零次：计 1 次机会、0 次结果。
原因 / 结果多次、相同动态跨窗口重复或片段序号重复导致配对歧义：逻辑内拒绝该请求，要求调用方重新分段。
```

概率采用：

```text
Q10000概率 = floor(结果回合数 * 10000 / 机会回合数)
```

机会数非零即可返回候选。第一版不设置最低概率、置信度、可靠性或自动升格阈值。概率只表示当前证据下的可能性，不是稳定世界事实；“是否好用”由 #235 更新后 S0 另行裁决接线。

| #225 未完成、#231 未完成或假定接口漂移 | 依赖门控返回 | 不改代码，退回设计或等待前置 |

| 当前有效特征值读取零 / 多归属，或实例槽位读取零 / 多模板定义 | 追根因解决 | 停止候选路径，不选择第一个 |

| 计数无溢出且机会非零但概率越界 | 追根因解决 | 不返回概率候选 |
