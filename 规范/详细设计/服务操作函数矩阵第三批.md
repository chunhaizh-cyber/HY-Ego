## 4. 服务边界

### 4.1 特征服务

第三批只允许新增只读语义解析：

```text
由特征值节点反查唯一归属特征。
若归属特征是实例槽位，则读取唯一模板特征定义。
返回归属特征、语义特征定义、特征值和当前原始材料。
再次解析可判断值式材料是否仍为当前。
```

特征值服务继续只被特征服务直接访问。动态模式服务不得包含或调用特征值服务。

### 4.2 动态服务

动态服务继续拥有实例动态记录、完整动态材料和轻量引用阻断读取。第三批可以补只读当前性材料，但不得让动态服务自动扫描场景并裁决聚类事实，也不得写抽象动态。

### 4.3 动态模式服务 / 算法

当前已形成独立值式编排层：

```text
动态模式数据操作：在一次共享许可内读取完整节点、主信息、关系和状态值材料。
动态模式服务：复核正式任务动态链，负责入口准入、当前性和返回分类。
动态模式算法：只接收完整不可变 DTO，负责规范结构键、稳定排序、完整比较、去重和重复分组。
```

算法不得持有仓库 / 服务引用，不得读日志、时间源或线程完成顺序，不得通过哈希相等替代完整结构比较。

### 4.4 因果服务 / 因果模式服务

现有因果服务继续保持轻量因果引用入口。第三批建议把跨回合机会 / 结果计算放在独立因果模式服务 / 算法，不改义 `记录因果引用(来源动态)`，也不把单动态候选别名冒充概率模板。

### 4.5 统计、任务、需求和方法

JY-369 已基于 #235 和后续代码事实完成以下保守裁决：

```text
只有用途事件适配组合器直接重读正式任务结果 DTO；材料 / 算法 / 服务不反向依赖上层组合器。
任务选择 / 执行路径在动作前显式冻结实际采用的因果候选；结果产生后不得事后配对。
Q10000、任务达成、需求结算和观察阶段分栏，不形成单一好坏分数。
节点 15、主信息元组、索引、关系阶段和专用数据操作层均为 #283 待复核候选，不是实施合同。
完整因果键不能被当前活动概念签名有损替代；#284 只做字段映射与原子当前性 S0，不实现统计或概念映射。
方法学习候选算法、晋级、方法代际、后继回合接线、运行期和恢复计划必须在 #283-#285 三项 S0 后重新设计。
需求结算由 #286 收口值相等、单一已满足和完整结构幂等。
```

## 5. 输入与签名边界

### 5.1 显式观察回合与片段

候选输入不使用“扫描场景内全部动态后自行切段”。第一版由正式调用方提交属于某个观察回合的显式片段：

```text
回合编号 / 尝试编号。
回合内非零片段序号。
来源任务和执行尝试材料，若 #231 复核后存在。
非空动态句柄组。
回合规则版本。
```

同一回合可以提交多个片段，但片段序号必须唯一并确定先后，同一动态不得跨片段重复。每个片段内部动态必须同场景、同主体，按 `发生时间戳 -> 完整动态句柄` 确定性排序。相同时间戳允许存在，但完整句柄负责稳定次序。

### 5.2 规范动态步

第一版结构键至少包含：

```text
语义特征定义完整句柄。
前 I64 值。
后 I64 值。
来源动作是否存在及其完整句柄。
可明确复核的二次特征上下文结构键。
规则版本。
```

场景、主体、发生时间戳和来源动态句柄保留在证据中，但不进入跨实例语义相等键。二次特征上下文只有在 #231 证明角色顺序稳定时才能进入键；否则保持空并由 #232 实施记录明确，不得按容器遍历顺序猜角色。

### 5.3 结构键与哈希

完整结构键裁决相等，哈希只用于候选分桶：

```text
结构键全字段相同 -> 可进入同组。
哈希相同但结构键不同 -> 分开候选。
规则版本不同 -> 不可比较。
来源材料过期 -> 本次候选逻辑内返回，不用缓存兜底。
```

```text

至少两个不同完整回合各有相同完整结构键，才形成重复聚类候选。

第一版只比较调用方显式提交的完整片段，不从片段内部做任意子序列搜索，不使用隐藏时间阈值。
```

聚类向上只形成候选材料；本批不执行向下抽象，不创建抽象动态、二次特征、方法、概念或因果节点。

## 7. 因果概率候选边界

因果请求显式提供 #233 成功的完整 `动态聚类结果`、不同的原因 / 结果完整结构键和规则版本。原因键必须精确命中一个正式重复聚类；结果键允许在全部单步实例中零次出现。服务按完整回合和片段序号匹配，不在动态步内部搜索。每个原因支持回合固定为：

```text
原因实例必须恰有一个；多于一个时机会起点歧义，逻辑拒绝整个请求。
结果实例只在同一完整回合匹配；跨回合不配对。
结果只在原因前出现：保留前置证据，当前机会计负结果。
结果与原因片段序号相同：顺序不可判定，逻辑拒绝整个请求。
原因后结果出现一次或多次：保留全部后续证据，当前机会只计一次正结果。
原因后结果零次：当前机会计一次负结果。
不在原因支持回合组中的回合不计机会，也不计负结果。
```

概率采用：

```text
Q10000概率 = floor(结果回合数 * 10000 / 机会回合数)
```

机会数固定等于原因重复聚类支持回合数，因此至少为 2。结果键零次出现时合法返回 Q10000 为 0。第一版不设置最低概率、置信度、可靠性或自动升格阈值。概率只表示当前证据下的可能性，不是稳定世界事实；JY-369 已把它与结构化任务 / 需求效用分栏，并要求经过权威用途账、统计、学习候选和独立晋级后才可能形成新代际方法。
