```text

-> 唯一服务合同
-> 30% 预支 / 70% 完工余款
-> 服务准备衰减补回

-> 服务值调节生存安全根值被动回归
-> 安全 / 服务任务根权限
-> 与 6150 分层安全权限组合
```

## 2. 当前代码事实

当前 `main` 的只读事实固定为：

1. `初始化.需求.ixx` 能建立安全值 / 服务值语素、实例槽、I64 初值和两个根需求；它没有服务合同、时间维护或任务权限入口。
2. `自我线程.ixx` 等待消息超时后直接继续；循环次数、等待超时和消息批次都不是完整物理秒。
3. `自我治理领域路由.ixx` 是固定安全根兼容链，并明确拒绝服务根需求；不得扩写为本设计的服务领域实现。
4. 需求、任务、方法服务没有服务合同唯一键、预算、30% / 70%、有效进展、准备补回或服务任务分类。
5. 任务执行请求不含安全 / 服务权限材料，任务管理按确认顺序 `push_back` / `pop_front`，没有候选集合权限排序。
6. 当前轻量因果只形成轻量引用，不能裁决任务是安全来源还是服务来源。
7. 节点直接身份结构写入执行器已有多参与者准备、确认、最后发布和逆序撤销底座，可由本设计的数据操作层消费。
8. 长期执行通道中的未集成候选不是 `main` 当前接口，不得作为本设计已实现前置。

## 3. 合同分层

### 3.1 `SERVICE-C1 / v1` 服务合同与结算

唯一职责：

```text
读取有效人类需求和准备事实
-> 建立或读回唯一合同
-> 冻结有效期和预算
-> 结算 30% 预支
-> 裁决完成 / 取消 / 到期 / 终止
-> 结算 70% 余款或关闭余款
-> 发布到期未满足事件
-> 结算准备衰减补回
```

合同建立 + 30% 预支可以由合同入口独立原子发布；合同终态、准备补回和同秒时间维护在生产循环中由 `SERVICE-C2` 聚合入口共同提交。`SERVICE-C1` 不计算时间衰减、不计算生存安全根回归、不排序任务。

合同建立入口必须通过 `服务生存事实提供者` 回读当前生存安全根值和版本。`A=0` 时返回 `入口拒绝`、无载荷、零写入；`A=1` 或 `A>1` 才继续合同准入。安全值或定义版本漂移时返回 `版本漂移`，不得用请求、线程状态或旧缓存解释终止态。

### 3.2 `SERVICE-C2 / v1` 服务值与生存安全维护

唯一职责：

```text
读取已发布新合同 / 预支、合同终态、准备、进展、安全和时间事实
-> 聚合调用 SERVICE-C1 纯值终态 / 准备结算合同
-> 计算跨过的完整秒
-> 形成服务衰减段
-> 处理 30 天门禁和有效活动最低 1
-> 新安全事实优先
-> 形成生存安全根值被动回归候选
-> 同事务发布值、状态、动态、游标和幂等账
```

它是生产治理循环中合同终态 / 准备结算与时间维护的唯一聚合提交入口。它不改变 6140 分层安全值，不从任务等待、调度资源紧张或任务数量反写服务值或安全值。

### 3.3 `SERVICE-C3 / v1` 安全 / 服务任务根权限

唯一职责：

```text
读取已发布生存安全根值和阈值
-> 形成安全根权限与服务根权限
-> 安全来源组合 6150 当前五级安全权限
-> 服务来源使用服务根权限
-> 多来源任务取最高
-> 形成确定性排序投影
```

权限是可重建值式投影。只有实际选择 / 派发历史冻结来源与规则版本；不得把当前权限复制为任务第二份权威状态。

载荷出现条件：

| 结果 | 必须有载荷 | 必须无载荷 |
| --- | --- | --- |
| `服务合同结果` | `已结算、精确重复` | 其它分类 |
| `服务生存治理结果` | `已维护、无变化、精确重复` | 其它分类 |
| `任务根权限结果` | `已形成、权限为零` | 其它分类 |
| `服务生存任务排序结果` | `已形成、当前无可执行任务` | 其它分类 |

逐入口结果语义固定为：

| 入口与分支 | 唯一结果分类 | 载荷语义 |
| --- | --- | --- |
| `建立或读取服务合同` 新合同与 30% 预支同事务发布 | `已结算` | 合同记录 + 预支结算记录 + 新服务值版本 |
| `建立或读取服务合同` 同键同义且既有原子结果完整 | `精确重复` | 读回既有合同、预支结算和当前服务值版本 |
| `建立或读取服务合同` 当前生存安全根值为 0 | `入口拒绝` | 无载荷、零写入 |
| `结算服务生存治理批次` 完整完成并关闭 / 支付余款阶段 | `已维护` | 治理载荷内含终态合同 + 余款结算记录；即使因饱和实际写入为 0 也关闭阶段 |
| `结算服务生存治理批次` 取消、到期或正式终止且关闭余款 | `已维护` | 治理载荷内含终态合同 + 可选到期未满足事件；结算记录不适用 |
| `结算服务生存治理批次` 准备新结果完成并关闭补回阶段 | `已维护` | 治理载荷内含准备、唯一衰减段和补回结算；饱和实际写入为 0 仍关闭阶段 |
| `结算服务生存治理批次` 跨代次无连续时长证明但可建立当前新纪元 | `已维护` | 新时间纪元 + 连续性缺口记录 + 当前值 / 合同版本；不补算停机区间 |
| `结算服务生存治理批次` 没有跨秒、终态、准备结算或值变化 | `无变化` | 当前合同 / 准备 / 值版本和可回读维护账 |
| `结算服务生存治理批次` 当前运行代次 / 单调时间本身无法成立 | `时间证据缺口` | 无载荷、零写入 |
| 任一入口精确同键同义重放 | `精确重复` | 读回该入口既有不可变结果 |
| 任一入口材料 / 版本 / 结构拒绝 | 对应拒绝分类 | 无载荷、零写入 |

原子合同不允许出现“合同已建立但预支阶段未发布”的可读状态；若提供者读到该组合，属于内部不一致，不得用 `已形成` 修补。

## 9. 算术合同

### 9.1 预算与结算

```text
M = I64_MAX
T = 2,592,000
D0 = floor(M / T) = 3,558,399,705,576
B = min(M, D0 * 冻结有效总秒)
P0 = floor(B / 10) * 3 + floor((B mod 10) * 3 / 10)
P0 = max(1, P0)
P1 = B - P0
```

乘法使用至少 128 位中间值或等价检查算法。服务值增加先计算 `min(请求增加, M-V)`，减少先计算 `min(请求减少, V)`；禁止先溢出后钳制。

### 9.2 完整秒衰减

```text
无有效未满足需求:
  D = 1

最长等待 t 小于 T:
  D = max(1, D0 + floor(D0 * t / T))

达到 T 或出现到期未满足事件:
  无有效服务活动 -> 目标 V = 0
  有有效服务活动且 V 大于 0 -> 目标 V = 1
  有有效服务活动但 V 等于 0 -> 保持 0
```

普通衰减也执行有效活动最低 1 保护。多个需求只读取最长等待 `t`，并列按提出边界、提出者、需求稳定身份排序，不叠加步长。

批量结算按事实秒、合同终态、30 天边界、服务比例档位、活动保护和饱和边界分段。每段保存请求地板和、实际减少、前后值和跳跃证明版本；结果必须与逐秒参考顺序完全相同。

### 9.3 服务比例与安全根回归

```text
U = floor(I64_MAX / 98)

V = 0:
  A = 0 -> 保持 0
  A 大于等于 1 -> Anew = 1

V 大于 0:
  P = min(99, 1 + floor(V / U))
```

全部普通回归分支前先裁决 `A=0` 保持终止。

```text
A 小于 L:
  距离 = L - A
  步长 = max(1, floor(距离 / 100) * P)
  Anew = min(L, A + 步长)

L 小于等于 A 且 A 小于等于 H:
  Anew = A

A 大于 H:
  距离 = A - H
  比例 = 100 - P
  步长 = max(1, floor(距离 / 100) * 比例)
  Anew = max(H, A - 步长)
```

同秒主动安全结算改变 `A` 时，跳过该秒服务被动回归。安全门禁材料缺失时不允许被动上升。

### 9.4 根权限

```text
Q = 1,000,000

A 等于 0:
  生存控制态 = 终止
  普通任务竞争状态 = 不适用
  服务根权限 = 0
  安全根权限 = 0

A 等于 1:
  生存控制态 = 休眠
  普通任务竞争状态 = 不适用
  服务根权限 = 0
  安全根权限 = 0

A 大于 1 且 A 小于等于 L:
  生存控制态 = 主动运行
  普通任务竞争状态 = 适用
  服务根权限 = 0
  安全根权限 = Q

L 小于 A 且 A 小于 H:
  生存控制态 = 主动运行
  普通任务竞争状态 = 适用
  服务根权限 = max(1, floor((A-L) * 990,000 / (H-L)))
  安全根权限 = Q - 服务根权限

A 大于等于 H:
  生存控制态 = 主动运行
  普通任务竞争状态 = 适用
  服务根权限 = 990,000
  安全根权限 = 10,000
```

`A=0/1` 时不进入 6150 组合和普通任务排序；终止只保留正式恢复 / 重建与停止控制入口，休眠只保留人类请求接收、合同准入、完整秒维护、安全事实接收和停止控制入口。`A>1` 时安全来源权限为 `floor(安全根权限 * 6150当前来源权限 / Q)`；独立服务来源权限为服务根权限。多来源任务取最高，不求和。安全兼服务来源只按安全来源计算权限。

任务有效执行优先级为：

```text
floor(任务基础优先级 * 来源权限 / Q)
```

同值比较依次使用来源权限、任务基础优先级、来源分类、安全具体排序材料、任务创建序号和任务稳定句柄。#359 可以在不改变本公式的前提下把 6150 已冻结的安全组内比较用于安全来源。

## 10. 同秒处理和事务

生产聚合入口 `结算服务生存治理批次` 的逻辑顺序：

```text
冻结事实截止版本
-> 新安全事件、复发和主动安全结算
-> 读取已经原子发布的新合同 / 预支结果
-> 裁决合同完成、取消和正式终止
-> 到期判定及到期未满足事件
-> 预支 / 余款 / 关闭余款
-> 准备完成、失败和此前衰减补回
-> 重算未满足集合和有效活动
-> 完整秒衰减或 30 天门禁
-> 若同秒主动安全值未变化，计算安全根被动回归
-> 形成合同、结算、值、状态、动态、衰减段、游标和幂等候选
-> 加入记录参与者与外部参与包
-> 按执行器协议逐参与者准备；每个参与者在准备阶段完成自身候选写入与写后读回
-> 全部准备与读回通过后按协议确认
-> 最后原子发布
-> 从已发布快照重建权限投影
```

准备或确认失败按加入逆序撤销。禁止在全部确认后才做首次写后读回；确认后新增写入、撤销失败、合同已支付但阶段未发布、游标推进但衰减段缺失或值改变但动态缺失都属于内部逻辑错误。

合同终态、准备补回、服务值衰减和生存安全回归在生产循环中不得分别提交。`建立或读取服务合同` 的合同 + 预支原子事务是唯一例外：它用于接收新的有效人类需求，且安全根被动回归只从后续完整秒进入聚合治理。

## 11. 幂等与恢复

合同建立键至少包含自我、提出者、需求、首次提出事件、合同代次、冻结有效期和规则版本。结算键追加合同版本、结算阶段、服务值版本和事实截止版本。维护键追加运行代次、时间纪元、起止完整秒边界、服务 / 安全值版本、合同 / 进展集合版本、主动安全事实截止版本和全部规则版本。

同键同义读回既有结果；同键异义拒绝。

恢复材料必须保存：

```text
服务值和生存安全根值当前 / 历史状态
合同、预算、结算阶段、实际写入和舍弃值
准备归属、到期未满足事件和进展事实
聚合衰减段、完整秒边界、时间纪元和连续性证明
主动安全事实游标、阈值、方法、公式和格式版本
状态、动态、关系、因果和幂等账
```

跨代次没有可信连续时长证明时不补算停机区间。恢复在隔离候选中完成全部结构和业务不变量检查、索引重建及权限投影重算后一次发布。
