# 控制面板数据库重建候选代码逻辑详细设计

更新时间：2026-07-08

状态：已生成 / 后置候选 / 仅可在用户明确指令后生成待确认专项计划 / 未生成施工计划 / 未入队 / 未改 C++ / 未构建 / 未迁移旧函数

## 1. 设计定位

本设计承接第 20 项 `流程图/20260708_控制面板数据库重建候选代码逻辑流程图_v0.1.md`，只记录控制面板、SQL / ADO、WebView2、D455、体素和外设作为排除证据后的重建候选边界。

本设计不自动生成控制面板或数据库实施计划，不接入旧控制面板、SQL、ADO、WebView、D455、体素或外设。

## 2. 输入入口

```text
旧控制面板事实
SQL / ADO 投影事实
WebView2 事实
D455 / 体素 / 外设事实
用户明确的重建专项请求
```

没有用户明确要求重建专项时，必须拒绝自动迁移、自动计划、自动入队和自动代码修改。

## 3. 核心处理

```text
识别候选来源
-> 判断是否有用户明确重建专项请求
-> 区分只读显示和数据库审计适配
-> 只读显示依附显示层只读设计
-> 数据库只作为快照 / 事件日志 / 恢复 / 审计适配
-> D455 / 体素 / 外设转独立专项
-> 所有候选仍需待确认计划和用户确认
```

控制面板不得写事实，SQL 不得裁决运行期事实。

## 4. 结构承载

```text
排除证据：FS-X 控制面板 / SQL / D455 / 体素 / 外设事实
只读显示候选：显示层只读专项后续材料
数据库审计适配候选：快照、事件日志、恢复和审计材料
独立外设候选：D455、体素、外设专项材料
```

当前运行期核心仍以内存仓库为准。

## 5. 输出变化

默认输出为拒绝自动迁移和保持排除项。只有用户明确要求时，才可生成问题定义、详细设计或待确认专项计划。

本设计不登记可执行队列，不修改 C++，不运行构建。

## 6. 当前代码差距

```text
无 DTO / JSON / WebView / SQL / ADO / 控制面板 / D455 / 体素 / 外设接入口。
无数据库审计适配服务。
无控制面板只读服务。
旧项目事实不能作为新项目代码事实。
```

## 7. 验收矩阵

| 验收项 | 验证方式 | 通过口径 |
| --- | --- | --- |
| 未明确请求拒绝 | 无用户专项指令时出现迁移请求 | 不生成计划、不入队、不改代码 |
| 控制面板写事实拒绝 | 面板按钮或 HTML 请求写业务事实 | 拒绝 |
| SQL 裁决拒绝 | SQL 结果试图裁决运行期事实 | 拒绝 |
| 外设转专项 | D455 / 体素 / 外设请求 | 不借本设计接入 |
| 计划门禁 | 用户明确要求重建 | 先生成待确认专项计划 |

## 8. 后续边界

后续只有用户明确要求时，才可生成控制面板重建专项或数据库审计适配专项；专项仍必须先走详细设计、待确认计划、允许文件、禁止文件和验证方式。
