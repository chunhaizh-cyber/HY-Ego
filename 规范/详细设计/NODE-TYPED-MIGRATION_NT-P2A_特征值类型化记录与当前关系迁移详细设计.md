## 1. 目标与完成边界

本设计把特征值域的 `I64`、`VecI64`、`VecU64` 原始材料统一迁移为一类受特征值领域约束的类型化记录，并保持槽位、模板、当前值和来源全部由正式关系表达。

目标结构固定为：

```text
特征值节点直接身份
+ 关系 22 角色 1 的宿主到实例槽关系
+ 关系 22 角色 2 的实例槽到特征定义关系
+ 关系 22 角色 3 的实例槽到当前特征值唯一关系
+ 特征值到合法来源的来源关系（业务需要时）
+ 特征值领域唯一类型化记录
+ 独立 4170 不可变批次发布记录
+ 可重建索引和组合读取投影
```

本设计完成只证明 P2A 的可实现结构、流程、接口候选、差距和验证合同已定义。它不证明：

```text
NT-P1 已经实现或接口已经稳定
任何 C++ 已修改
旧特征服务兼容入口已经退役
主信息仓库已经全仓删除
4170 已具备完整快照恢复
完整运行期上下文或自我已经跨重启恢复
```

## 3. 当前代码事实与差距

### 3.1 当前数据操作路径

`数据操作.特征体系.ixx` 已具备：

1. 特征定义、实例槽位、初始值和批次的强类型规格；
2. 事务外预读、锁内重读、幂等同义 / 异义分类；
3. 节点、关系、索引和内部材料的统一结构执行器；
4. `特征值原始材料事务参与者` 与 `特征批次发布记录事务参与者` 的同事务登记；
5. 宿主到槽位、槽位到定义模板、槽位到当前值和值到来源的关系；前三项当前仍使用通用归属 / 模板，尚未实现关系 22；
6. 换代时旧当前关系失效、新当前关系发布和历史审计；
7. 发布前完整读回、确认、逆序撤销和最后发布。

这些事务和关系语义是迁移时必须保留的已实现能力，但其结构承载仍不符合 4020。

### 3.2 当前原始值分裂

当前 `参与者.特征值原始材料.ixx` 和 `特征值服务.h` 把一项特征值拆为：

| 原始类型 | 当前值本体 | 当前版本 | 当前读取 |
| --- | --- | --- | --- |
| I64 | 特征值节点所绑定主信息的槽 0 | `I64版本记录表_` | 主信息槽 0 + I64 版本侧表组合 |
| VecI64 | `Vec记录表_` | 同记录 `容器版本` | Vec 侧表 |
| VecU64 | `Vec记录表_` | 同记录 `容器版本` | Vec 侧表 |

因此当前实现存在三个结构问题：

1. I64 值本体仍在已被规范退出的无类型主信息槽；
2. I64 与 Vec 使用不同权威承载和不同版本记录形状；
3. 值节点有效性、读取投影和事务参与者仍依赖主信息句柄。

### 3.3 当前双入口

当前至少有两类写入口：

- 新入口：`特征体系数据操作` 经统一结构执行器和正式参与者提交；
- 兼容入口：`特征服务.h`、`特征值服务.h` 直接分步创建节点 / 关系并写原始材料。

兼容入口的局部锁、逐步写入和读回不能证明与数据操作路径具有同一全参与者事务语义。P2A 允许阶段性保留调用签名，但只能把它们路由到同一特征值类型化记录权威；不得继续保留主信息值与类型化记录双写。最终公开写入口封口由 NT-P3 完成。

### 3.4 当前可保留结构

以下事务和值对象语义与新规范一致，迁移时保留；槽位拓扑只保留端点、基数和换代语义，关系类型必须改为 22：

- 实例槽位和特征定义均为稳定节点；
- 宿主到实例槽、实例槽到定义模板、实例槽到当前值已经由关系表达，但当前通用 `归属 / 模板` 必须迁为关系 22 角色 1 / 2 / 3；
- 当前关系唯一，多条当前值属于内部结构矛盾，不选择第一条；
- 换代保留失效旧关系审计；
- 原始类型三者互斥；
- 4170 批次账独立、不可变、完整有序并与值本体同事务发布；
- 候选、确认、撤销、最后发布和发布后读回的分类边界。

## 4. 目标结构

### 4.1 特征值类型化记录

计划新增特征值域专用记录数据结构。候选逻辑形状如下，实际 C++ 声明需在 P1 接口复核后固定：

```text
特征值类型化记录
  所属特征值节点：完整当前节点句柄
  记录格式版本：非零、当前受支持版本
  原始值版本：非零、单调、同时作为本域记录版本
  记录状态：有效 / 已删除 / 已失效中的合法值
  原始载荷：I64载荷 | VecI64载荷 | VecU64载荷（三者严格互斥）
```

载荷为特征值领域内部的强类型封闭联合，不是跨领域通用值容器。它不得接收：

```text
任意槽位编号
拆分句柄
宿主、定义、当前关系或来源端点
批次身份、业务规则版本、项目顺序或写前证据
坐标、颜色、轮廓、任务完成等业务解释
JSON、自由文本或不可解释字节包
```

记录身份由：

```text
所属特征值节点完整句柄
+ 特征值记录类型
+ 原始值版本
```

共同限定，不增加跨领域记录编号、通用附属句柄或槽位索引。

### 4.2 三种载荷不变量

| 类型 | 必需字段 | 必须为空 | 额外约束 |
| --- | --- | --- | --- |
| I64 | 一个 `int64` | VecI64、VecU64 | 元素数隐含为 1 |
| VecI64 | 非空 `vector<int64>` | I64、VecU64 | 数量 `1..最大序列元素数量` |
| VecU64 | 非空 `vector<uint64>` | I64、VecI64 | 数量 `1..最大序列元素数量` |

原始类型由强类型载荷本身确定，不再需要一份可能与载荷漂移的平行业务字段。若实现为枚举加封闭联合，确认阶段必须证明枚举和实际分支一致；该枚举只属于特征值记录格式，不得扩成通用值类型 ABI。

### 4.3 状态与当前性

特征值记录的 `有效` 表示该值节点的原始材料合法可读，不表示它仍是某槽位当前值。槽位当前性只由唯一有效的“实例槽位到值节点”关系裁决。

一次槽位换代后：

- 旧当前关系进入已失效审计；
- 旧值节点及其类型化记录继续可由审计和 4170 历史重放读取；
- 新值节点、新类型化记录和新当前关系成为当前组合；
- 不把旧值记录标成删除来模拟当前关系失效。

值节点正式删除时，节点、必要关系和本域记录必须按后继删除计划共同推进，不由 P2A 顺手建立普通业务删除入口。

### 4.4 值变化策略

P2A 的生产数据操作默认采用“新值新节点”：

```text
旧值节点 + 旧有效记录 + 旧当前关系
-> 失效旧当前关系
-> 新值节点 + 原始值版本 1 的新记录 + 新当前关系
```

这与当前批次换代和 4170 的结果引用一致。当前 `特征值服务` 对同一值节点原地重写并推进版本的兼容行为不得作为新生产路径继续扩展。

若 NT-P3 为迁移旧调用暂时保留同节点换代，必须同时满足：

1. 新记录版本单调增加；
2. 旧版本保留审计；
3. 同一节点只有一个普通读取当前记录；
4. 写前预期版本参与拒绝；
5. 4170 已引用版本不被复义；
6. 不产生主信息值或第二侧表双写。

未满足以上六项时直接拒绝旧调用，不得静默原地覆盖。

## 5. 关系与索引合同

### 5.1 关系唯一职责

P2A 消费 4010 已追加的关系类型 22，不再用通用 `归属 / 模板` 表达实例特征拓扑。4110 的角色合同固定为：

| 顺序号 | 角色 | 源节点类型 | 目标节点类型 | 当前基数 / 唯一性 |
| --- | --- | --- | --- | --- |
| 1 | 宿主槽 | 存在、场景、需求、任务或方法 | 特征（实例槽） | 宿主零到多；槽位反向恰一宿主；唯一键含完整宿主和槽位 |
| 2 | 定义模板 | 特征（实例槽） | 特征（抽象定义） | 每槽恰一；唯一键为类型 + 源槽 + 角色 |
| 3 | 当前值 | 特征（实例槽） | 特征值 | 每槽恰一当前关系；唯一键为类型 + 源槽 + 角色 |

同一宿主和同一抽象定义同一当前版本只能有一个有效实例槽，必须联合回读角色 1、2。当前值换代创建新角色 3 并正式失效旧角色 3；不得原地改目标。

值的合法来源继续使用正式 `因果来源(特征值节点 -> 来源)` 关系，并按请求和 4170 顺序角色复核。类型化记录不得保存关系 22 或来源关系的端点副本。

### 5.2 索引

稳定主键由 P1 直接进入节点记录。索引只负责由主键定位值节点候选，命中后必须回读：

```text
节点稳定主键与句柄
节点类型和记录状态
槽位当前关系或历史审计关系
特征值类型化记录与版本
批次重放需要的 4170 引用
```

索引缺失或清空不得改变值本体。索引命中但上述材料不一致属于内部不一致，不得降级成未找到。

## 6. 输入契约和调用语境

### 6.1 唯一业务解释方

隔离新域的节点直接特征服务是实例槽、值域、合法来源、批次语义和结果解释的唯一公开写入方。节点直接特征值记录服务只向该服务和节点直接特征体系数据操作提供内部能力，不向旧默认生产模块、需求、任务、方法、世界、概念图、语素、线程、SQL 或显示开放写入口。

公开请求只能携带：

```text
稳定宿主 / 定义 / 槽位 / 当前值 / 来源句柄
稳定主键或具名幂等身份
强类型 I64 / VecI64 / VecU64 值式载荷
预期当前关系、预期当前值和预期原始值版本（换代时）
受支持业务规则版本和连续项目顺序（批次时）
```

不得携带仓库引用、锁、原始事务令牌、写会话、参与者、撤销控制权、可变内部记录或决定裁决的摘要。

### 6.2 写前拒绝

第一笔写入前至少复核：

1. 数据操作、接线和运行期事务域有效；
2. 所有完整句柄非零且为当前版本；
3. 宿主类型允许持有实例特征；
4. 特征定义是抽象定义，不是另一个实例槽；
5. 实例槽的关系 22 角色 1 宿主槽与角色 2 定义模板唯一完整；
6. 初始值事务形成角色 3 当前值；换代的角色 3 当前关系、当前值和原始值版本与请求一致；
7. 新值稳定主键空闲或只命中完整同义已发布结构；
8. 三种原始载荷严格互斥且值域 / 元素数合法；
9. 来源当前有效且属于已发布来源或同一事务内具名候选；
10. 批次身份、规则版本、顺序、项目唯一性和 4170 预读分类完整。

## 7. 写入流程

### 7.1 非批次初始发布

```text
节点直接特征服务形成业务请求
-> 节点直接特征体系数据操作形成不可变规格
-> 事务外读取槽位当前值和新值主键
-> 取得唯一写入权
-> 锁内重读前置
-> 创建带稳定主键的值节点候选
-> 创建槽位到新值当前关系候选
-> 登记类型化记录候选
-> 事务内读回节点、关系和记录
-> 结构会话与记录参与者确认
-> 无失败最后发布
-> 释放写入权后组合读取并返回
```

如果调用同时创建实例槽，则关系 22 角色 1、2、3 都属于同一结构事务。4110 已规定每个实例槽恰一角色 3 当前关系，因此不得先发布空槽，再用另一个事务补当前值。

### 7.2 单槽换代

单槽换代请求必须携带旧当前关系、旧值节点和旧原始值版本。锁内复核成立后：

```text
失效旧当前关系候选
-> 读回失效审计身份和端点
-> 创建新值节点
-> 登记新类型化记录
-> 创建新当前关系
-> 完整读回
-> 共同发布
```

写前版本漂移属于逻辑内返回；旧关系已经成功形成失效候选后，任何新值、记录或新关系异常均是内部逻辑错误，必须精确撤销到旧关系仍有效的前态。

### 7.3 特征批次

特征批次复用当前 4170 语义：

```text
事务外批次账预读
-> 唯一写入权内再次读取
-> 为每项形成槽位 / 换代关系、值节点和类型化记录候选
-> 为每个新值形成来源关系
-> 形成独立 4170 候选记录
-> 类型化记录参与者和批次账参与者按登记顺序准备
-> 结构会话确认
-> 参与者依序确认待发布
-> 无失败最后发布
```

4170 记录只保存新值句柄、关系句柄、写前证据和来源引用，不复制 I64 / Vec 值。历史重放必须按记录中的新值句柄读取类型化记录，并复核发布后关系审计。

## 8. 读取与组合投影

### 8.1 当前值读取

```text
读取实例槽节点直接身份
-> 读取唯一宿主归属和唯一模板关系
-> 读取唯一有效当前值关系
-> 读取当前值节点直接身份与稳定主键
-> 读取该值节点唯一当前有效特征值类型化记录
-> 复核所属节点、格式版本、原始值版本、记录状态和载荷互斥
-> 组合不可变值式结果
```

投影可以继续返回槽位、定义、值节点、关系 22 角色 1 / 2 / 3、原始类型、原始值版本和具体值，但不再返回主信息句柄。投影本身不持久化，也不能反向写入。

### 8.2 跨域值式只读归属复核

P2A 必须由隔离新域的节点直接特征服务提供值式只读 `复核特征值归属` 合同，供 P2B 状态 / 动态事实在不读取特征值记录容器的前提下验证值引用。

候选输入：

```text
目标特征值完整句柄
可选预期抽象特征定义完整句柄
读取口径：当前关系或显式历史审计
```

候选输出 `特征值归属值式材料` 至少包含：

```text
结构化读取状态
特征值节点完整句柄和节点版本
原始值版本
实例槽完整句柄
抽象特征定义完整句柄
关系 22 角色 2 定义模板完整证据
关系 22 角色 3 当前或历史审计完整证据及其状态
```

当前事实写入只接受有效角色 3；历史状态 / 动态读取可以显式消费已失效角色 3 审计，但不得把它重新解释为当前。该接口不返回可变记录、记录容器、仓库、令牌、参与者或内部锁，也不允许 P2B 直接访问旧侧表或新类型化记录仓。

### 8.3 按宿主读取实例特征槽位组

P2A 必须由隔离新域的节点直接特征服务提供值式只读 `读取宿主实例特征槽位组` 合同，供 P2C 从自我等合法宿主读取当前实例特征投影，而不直接接触关系、索引或特征值记录容器。

候选输入只包含目标宿主完整节点句柄。处理顺序固定为：

```text
复核宿主节点直接身份和关系 22 合法宿主类型
-> 按关系 22 角色 1 枚举以该宿主为源的实例槽候选
-> 对每个槽位读回唯一反向宿主、角色 2 抽象定义和角色 3 有效当前值
-> 读取当前值节点直接身份及其唯一当前类型化记录
-> 复核端点、基数、版本、记录所属节点和载荷互斥
-> 组合调用期不可变槽位组
```

候选输出 `宿主实例特征槽位组值式材料` 由零到多项 `实例特征槽位值式材料` 组成；每项至少包含实例槽、抽象定义、当前特征值、原始类型、原始值版本和具体值。合法宿主没有实例槽时返回空组；宿主身份无效或类型不允许时属于入口拒绝。枚举后发现多宿主、多模板、多当前值、悬空端点或记录不一致属于内部逻辑错误，不得跳过坏项后返回部分组。

该组只在本次调用期存在，不持久化、不反向写入，也不成为自我对象的第二份槽位数组。接口不得返回关系仓库、索引、可变记录、记录容器、事务令牌、参与者或内部锁；P2C 不得绕过它直接拼装自我实例特征投影。

### 8.4 历史值读取

批次历史或关系审计读取不要求目标仍是当前值，但必须证明：

1. 值节点及记录仍合法审计可读；
2. 4170 的发布后关系句柄能在关系审计中找到同端点、同版本或合法后继审计记录；
3. 记录原始值版本符合该批次发布合同；
4. 当前关系后续失效不会改变历史值内容。

### 8.5 比较、序列化和缓存

三态比较、内容哈希、值域候选和序列化统一从类型化记录的值式投影读取。哈希、序列化和候选统计仍是非权威派生材料；清空或重建不得改变值本体。

P2A 只改读取来源，不改变 4130 的三态语义，不给 Vec 增加坐标、句柄组、轮廓或其它业务解释。

## 9. 事务参与者设计

### 9.1 类型化记录参与者

候选参与者唯一职责：

```text
接收本次事务的强类型记录候选
-> 拒绝同节点重复候选和字段不自洽
-> 在准备阶段复核值节点是本会话候选或合法换代目标
-> 检查目标记录不存在异义当前版本
-> 写入未发布记录并保存精确反向材料
-> 确认后保持可逆
-> 发布时只关闭撤销能力和释放占用
```

参与者不得解释值域、来源、批次同义或槽位当前性；这些属于特征服务、数据操作和正式关系。

### 9.2 撤销顺序

依赖顺序按具体写集固定，至少满足：

```text
4170 批次账候选
-> 特征值类型化记录候选
-> 来源关系 / 新当前关系 / 失效关系候选
-> 索引绑定
-> 新值节点
-> 新实例槽及其宿主 / 模板关系（若本次创建）
```

执行器可以采用等价的内部顺序，但必须证明先撤依赖者、后撤被依赖结构。撤销失败必须隔离事务域，不得扫描猜测、普通删除或析构隐藏写入。

### 9.3 最后发布

最后发布区不得再：

- 分配节点、关系、记录或容器；
- 检查值域、来源或幂等语义；
- 调用外部服务或回调；
- 生成决定结果的日志；
- 执行可能失败的读回。

只允许关闭候选撤销能力、完成各参与者发布并释放唯一写入权。

## 10. 非成功返回二分

### 10.1 逻辑内返回

| 情形 | 结果 | 写入 |
| --- | --- | --- |
| 无效句柄、零主键、未知类型 / 格式、空向量、值域越界 | 入口拒绝 | 零写入 |
| 宿主、定义、槽位、来源或当前关系前置不满足 | 入口拒绝或版本漂移 | 零写入 |
| 同键同义且全部结构互证 | 幂等读回 | 零写入 |
| 同键异义、同槽多定义请求或批次语义冲突 | 幂等冲突 | 零写入 |
| 唯一写入权 / 侧账占用竞争 | 许可拒绝 | 零写入或未形成候选 |
| 第一笔写入前版本变化 | 版本漂移 | 零写入 |

### 10.2 内部逻辑错误

下列情形不得返回普通失败继续：

- 前置通过后节点、关系、记录或索引创建结果不及预期；
- 同一值节点出现多个当前记录、多个原始类型或版本断裂；
- 类型化记录复制了关系端点、批次字段或跨域载荷；
- 候选准备、确认、撤销或最后发布阶段不一致；
- 普通读取看到候选或半结构；
- 发布后节点、当前关系和记录无法完整互证；
- 4170 历史记录与新值记录或关系审计不一致；
- 撤销无法证明恢复事务前态。

处理固定为：停止新增写入、精确逆序撤销；撤销不闭合则隔离事务域；保留证据并追根因，不改成普通需求、学习缺口、缓存失效或旧主信息回退。

## 14. 验证矩阵

### 14.1 结构与类型

1. I64、VecI64、VecU64 分别形成一条完整类型化记录；任意两种同时有值写前拒绝。
2. 记录所属节点必须是当前可读特征值节点；错节点类型、错句柄版本、零格式或零原始值版本拒绝。
3. I64 不再读取或写入主信息槽；三种类型都从同一记录访问器读取。
4. 类型化记录中宿主、定义、槽位、当前关系、来源和批次字段精确零新增。
5. 组合投影清空、复制或重建不改变节点、关系和记录事实。

### 14.2 关系与换代

1. 实例槽只有一个关系 22 角色 1 宿主、一个角色 2 模板和恰一个角色 3 有效当前值关系。
2. 初始发布形成新值节点、记录和当前关系。
3. 换代保留旧值记录和失效旧关系审计，新关系唯一当前。
4. 写前预期关系、值或原始值版本漂移均零写入。
5. 多当前值、悬空关系、错端点和索引 / 权威读回不一致进入内部逻辑错误。
6. 关系 22 未知角色、非法宿主、角色 1 多宿主、角色 2 多模板和角色 3 多当前值按 4110 分类拒绝或追根因。

### 14.3 批次与事务

1. 多槽初始、多槽换代和混合批次共同发布一条 4170 记录和逐值类型化记录。
2. 同键同义重放读回相同结果；同键异义、顺序变化、来源变化和写前证据变化零写冲突。
3. 每个结构写点、记录参与者准备 / 确认点和批次账参与者准备 / 确认点故障均逆序恢复前态。
4. 撤销失败在释放唯一写入权前隔离事务域。
5. 普通读取在最后发布前看不到值节点、关系、记录或批次账半结构。
6. 发布后历史重放经 4170、新值记录和关系审计逐项互证。

### 14.4 服务与依赖

1. 高级服务不直接访问特征值记录服务。
2. 新生产写路径只经特征服务 -> 特征体系数据操作 -> 执行器 / 参与者。
3. 旧兼容入口保持旧默认域且不得调用新域；新域入口不读取或写入旧主信息和旧侧表，两域句柄、关系、索引和幂等账零互通。
4. P2A 生产文件不 import 自检，不形成服务循环。
5. P2B / P2C 禁止文件和共享装配文件相对冻结基线零差异。
6. P2B 只通过特征服务值式 `复核特征值归属` 读取值归属；静态扫描证明 P2B 不 include / import / 访问特征值记录参与者、记录容器或旧侧表。
7. P2C 只通过特征服务值式 `读取宿主实例特征槽位组` 读取宿主槽位；合法无槽宿主返回空组，任一槽位权威结构异常不得返回部分组；静态扫描证明 P2C 不访问关系仓库、特征值记录参与者、记录容器或旧侧表。

缺任一项时不可派发 P2A 代码。

### 15.2 与 P2B / P2C 的固定串行依赖和所有权移交

P2B 对 P2A 新增真实消费依赖：P2B 写入关系 19 角色 3（特征）和角色 4（值）前，必须消费已落地的节点直接特征服务值式 `复核特征值归属`，确认二者归属一致，不得直接读取特征值侧表。因此 P2B 的相关新域写入口不能与 P2A 接口实现并行派发；它必须基于 P2A 固定提交完成实际接口 S0 后再实施。

P2C 对 P2A 也新增真实消费依赖：P2C 形成自我实例特征读取投影时必须消费已落地的节点直接特征服务值式 `读取宿主实例特征槽位组`，不得直接读取关系仓库、特征值记录容器或旧侧表。因此 P2C 的相关新域读取入口不能与 P2A 接口实现并行派发；它必须基于 P2A 固定提交完成实际接口 S0 后再实施。

执行路线固定为 `#340 P2A -> #341 P2B -> #342 P2C`，不得因业务文件可隔离、设计事实可以并行审阅或由同角色子智能体并行核对而授权三个执行叶并行。所有权移交条件固定为：

1. #340 独占特征值记录、特征体系数据操作、对应自检、工程 XML 和统一自检运行器；
2. #340 集成并经设计接受后，#341 才能只读消费其值式接口并接管工程 XML / 统一自检运行器；
3. #341 集成并经设计接受后，#342 才能消费 P2A / P2B 固定接口并接管工程 XML / 统一自检运行器；
4. 运行期装配和入口在三个叶子中始终禁止修改，不另立共享接线叶子；
5. 三者消费同一已接受 NT-P1 固定提交，后继只读前序生产文件，不反向修改接口。

### 15.3 立即停止并退回设计

- NT-P1 实际接口不能表达记录候选、精确撤销或最后发布；
- 需要把关系端点复制进类型化记录才能实现读取；
- 需要把 4170 字段合并进值记录或复制原始值进批次账；
- 旧兼容入口无法在不双写的情况下阶段迁移；
- P2A 与 P2B / P2C 出现同文件、同接口或工程接线重叠；
- `复核特征值归属` 需要向 P2B 暴露记录容器、侧表、仓库或令牌才能实现；
- `读取宿主实例特征槽位组` 需要向 P2C 暴露关系仓库、索引、记录容器、侧表、仓库或令牌才能实现；
- 构建或自检连续出现无具名根因的失败；
- 验证只能依赖日志、显示、索引命中或返回 bool，而不能读回权威结构。
