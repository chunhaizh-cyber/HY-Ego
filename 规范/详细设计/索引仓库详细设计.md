# 索引仓库详细设计

编号：012-D1
日期：2026-07-05
状态：已确认详细设计 / 核心内核 P1 / 可作为第一轮代码实施依据

## 1. 设计目标

索引仓库负责查找入口，不裁决机器事实。索引命中只能返回候选节点，候选必须由领域服务按节点类型、关系、状态和权限复判。

第一轮索引仓库只确认主键索引和候选查询边界，不把索引提升为事实源，不用索引替代关系、特征、状态、动态或因果引用。

## 2. 规范承接

| 上层规则 | 本设计承接 |
| --- | --- |
| 结构闭环 | 索引只作为查找结构，不用临时字符串、显示标题或 SQL 行裁决事实 |
| 节点身份 | 索引命中后必须回到节点句柄和节点有效性判断 |
| 关系边界 | 索引不替代关系语义，关系仍由关系仓库承载 |
| 语素查询规则 | 查询能力返回候选集合，不静默选择第一个事实 |
| 入口前置拒绝 | 绑定索引前拒绝无效主键、无效节点和冲突策略未确认 |

## 3. 索引边界

```text
索引不是事实源。
索引可以重建。
索引失效不得改变机器事实。
索引命中不得静默选第一个作为业务事实。
索引只能返回句柄、候选集合或命中来源。
领域服务必须复判候选是否可用。
```

## 4. 结构设计

| 结构 | 用途 | 写入方 | 读取方 | 生命周期 | 验证方式 |
| --- | --- | --- | --- | --- | --- |
| `索引句柄` | 后续索引项身份 | 索引仓库 | 索引仓库、持久化适配 | 随索引项生命周期 | `0` 无效，删除后不复用 |
| `主键索引` | 从主键定位节点 | 领域服务 / 索引仓库 | 领域服务 | 可重建 | 主键命中后仍需节点有效 |
| `多值索引` | 后续候选查询 | 待确认 | 待确认 | 可重建 | 多候选返回集合 |
| `索引项` | 查找材料 | 索引仓库 | 索引仓库 | 随索引生命周期 | 失效节点不应作为有效结果 |
| `命中来源` | 说明候选为何命中 | 索引仓库 | 领域服务、显示层只读入口 | 运行期只读 | 只能做人读和复判材料 |

第一版主键索引可先实现为：

```text
主键 -> 节点句柄
```

后续名称、类型、别名、语素入口等查询必须返回候选集合或冲突候选包，不得恢复旧 SQL 投影或语素树遍历作为事实入口。

## 5. 主键范围

第一版主键使用 `uint64` 语义：

```text
0 保留无效。
主键只作为查找入口。
主键不是节点身份本体。
节点身份仍以节点句柄为准。
主键绑定前必须已有有效节点。
主键命中后必须再次校验节点当前有效性和版本。
```

主键冲突规则：

```text
同一主键第一轮只允许绑定一个当前有效节点。
重复绑定同一主键到同一节点可以返回已有绑定或成功，具体由函数实现固定。
重复绑定同一主键到不同节点必须返回冲突或失败。
若旧节点已删除，是否允许重绑该主键必须由具体领域服务确认，不得由索引仓库自行解释为事实替换。
```

## 6. 函数设计

| 函数 | 功能 | 入口拒绝 | 读写结构 | 验证方式 | 状态 |
| --- | --- | --- | --- | --- | --- |
| `绑定主键` | 绑定主键到节点句柄 | 主键为 0、节点句柄无效、节点已删除、主键冲突策略未确认 | 写主键索引 | 可按主键查回节点 | 已实现（文档声明）/ 设计已确认 / 待复验 |
| `按主键查节点` | 返回主键候选节点 | 主键为 0 | 读主键索引并复查节点有效性 | 未绑定返回空，已删除节点不作为有效结果 | 已实现（文档声明）/ 设计已确认 / 待复验 |
| `移除主键` | 移除索引项或使索引项失效 | 主键无效、节点不匹配、索引项不存在 | 写主键索引 | 删除后不再命中 | 后续 |
| `重建索引` | 从权威结构重建索引 | 来源结构未确认、重建范围未确认 | 写索引表 | 重建后候选一致 | 后续 |
| `按键返回候选集合` | 返回多值索引候选 | 键无效、索引类型未确认 | 读多值索引 | 同键多候选全部返回 | 后续 |
| `生成冲突候选包` | 返回候选和命中来源 | 候选来源缺失、候选结构未确认 | 运行期只读候选包 | 不静默选择事实 | 后续 |

## 7. 权威性和重建规则

```text
索引仓库不是权威账本。
主键绑定属于查找入口，不等于业务事实成立。
若索引丢失，可以由节点、关系、语素入口或领域服务确认的来源结构重建。
重建过程不得创造新业务事实。
重建过程发现冲突时必须返回冲突候选或失败，不得静默覆盖。
```

索引仓库可以保存非权威缓存，但任何非权威缓存必须明确：

```text
来源结构
重建方法
失效条件
读取方
是否允许持久化
禁止作为机器事实来源
```

## 8. 入口前置拒绝

索引仓库必须在入口拒绝：

```text
主键为 0。
节点句柄无效。
节点已删除。
节点版本失效。
主键已绑定到不同有效节点且冲突策略未确认。
索引类型未确认。
候选查询要求返回单一事实。
显示文本或 SQL 行反向写入索引。
```

## 9. 与其他系统的边界

| 对象 | 交互方式 | 边界 |
| --- | --- | --- |
| 节点仓库 | 绑定和查询时校验节点有效性 | 索引不代替节点身份 |
| 关系仓库 | 候选复判时读取关系 | 索引不代替关系事实 |
| 语素服务 | 语素入口可绑定主键或候选查询 | 语素服务不得让索引静默裁决事实 |
| 领域服务 | 负责候选复判和业务语义解释 | 索引仓库只返回候选和命中来源 |
| 显示层 | 可以读取候选用于人读显示 | 显示层不得写索引事实 |

## 10. 禁止行为

```text
不得把索引命中当作事实成立。
不得通过索引绕过领域服务。
不得让显示文本反向写索引事实。
不得用索引替代关系。
不得静默选择候选集合中的第一个结果。
不得用 SQL 投影作为运行期事实入口。
```

## 11. 验收条件

设计确认后，代码复验至少覆盖：

```text
有效节点可绑定主键。
无效节点不能绑定主键。
已删除节点不能作为有效索引结果。
主键可查回节点句柄。
未绑定主键返回空。
主键为 0 时查询和绑定均被拒绝。
重复主键绑定到不同有效节点时返回冲突或失败。
后续查询能力必须返回候选集合，不静默选中事实。
```

```text
语素冲突候选包已完成。

旧项目查询能力已迁移。
最小闭环已跑通。
```
