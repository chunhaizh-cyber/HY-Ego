## 2. 定位

因果模板候选是由动态证据集合、状态变化端点、来源动作或来源方法材料归集出的可复核模式候选。查询候选是基于已有动态证据、轻量因果引用、二次特征候选或运动基元候选返回的候选材料。

第一轮候选只能回答：

```text
有哪些动态证据可用于后续因果评估。
候选缺少哪些证据。
候选可能关联哪些来源动作或来源方法。
候选可能指向哪些目标状态、实际状态或状态变化端点。
候选是否需要后续需求、任务、方法或因果服务复判。
```

第一轮候选不得回答：

```text
因果已经稳定成立。
需求已经满足。
任务应该自动创建。
方法已经可执行。
结算事实已经形成。
自然语言说明就是机器事实。
```

## 3. 输入材料

| 输入材料 | 来源服务 | 第一轮准入 |
| --- | --- | --- |
| 动态证据集合 | 动态服务 | 动态有效、类型正确、材料完整 |
| 动态聚合候选 | 动态服务后续候选入口 | 只作候选材料，不裁决事实 |
| 运动基元候选 | 动态服务后续候选入口 | 只作变化片段材料 |
| 轻量因果引用 | 因果服务 | 只证明已有准入引用，不证明稳定因果 |
| 状态材料 | 状态服务 | 主体、场景、值、发生时间戳完整 |
| 二次特征组成材料 | 二次特征服务 | 组成项可追溯到状态或动态 |
| 来源动作入口 | 动态服务 / 方法服务确认材料 | 必须为空或为动作入口角色节点 |
| 来源方法 | 方法服务 | 必须为有效方法节点，且只作来源候选 |

第一轮禁止输入：

```text
SQL 行。
控制面板事件。
WebView2 显示文本。
D455 帧。
体素材料。
外设队列消息。
线程事件。
日志文本。
自然语言说明。
```

## 4. 候选结构

| 字段 | 含义 | 第一轮要求 |
| --- | --- | --- |
| 候选编号 | 本轮查询内局部候选身份 | 可丢弃、可重建，不作为长期事实 |
| 候选类型 | 因果模板候选 / 查询候选 / 缺失证据候选 | 必填 |
| 动态证据集合 | 候选依据的动态节点集合 | 可为空但为空时只能返回缺失证据候选 |
| 来源动作候选 | 可能的动作入口角色节点 | 可选，必须通过动作入口身份复核 |
| 来源方法候选 | 可能的来源方法节点 | 可选，必须由方法服务复核 |
| 目标状态候选 | 候选要解释或达成的目标状态 | 可选，必须为状态服务材料 |
| 实际状态候选 | 候选观察到的实际状态 | 可选，必须为状态服务材料 |
| 变化端点 | 前状态、后状态或动态前后值 | 必须能追溯到状态或动态 |
| 缺失证据 | 缺少的动态、状态、来源动作或来源方法材料 | 只作候选诊断，不写业务事实 |
| 候选评分 | Q10000 或后续评分占位 | 只作排序材料，不裁决事实 |
| 候选版本 | 候选规则版本 | 规则变化时旧候选失效 |
| 人读说明 | 调试或显示说明 | 不承载机器逻辑 |

候选结构第一轮默认不持久化。若后续需要持久化因果模板、候选评分或证据样本集合，必须另建详细设计和代码实施切片。

## 5. 因果模板候选边界

允许：

```text
按同一目标状态归集动态证据集合。
按同一来源动作或来源方法归集状态变化端点。
按动态聚合候选形成候选模板材料。
按二次特征组成项形成复合候选材料。
返回缺失证据清单。
```

禁止：

```text
自动创建稳定因果结论。
自动创建因果模板长期节点。
自动创建需求、任务或方法。
自动写需求结算记录。
把候选评分写入状态、任务、方法或需求事实。
把自然语言模板说明写成机器事实。
```

## 6. 查询候选边界

查询候选入口第一轮只返回候选和缺失证据。查询不得写结构，不得补造证据。

允许查询：

```text
按目标状态查找候选动态证据集合。
按来源动作查找候选状态变化。
按来源方法查找候选动态材料。
按轻量因果引用查找可回读动态证据。
按二次特征或运动基元候选查找后续因果评估材料。
```

禁止查询：

```text
创建需求。
创建任务。
选择方法。
写结算事实。
写稳定因果结论。
触发动作执行。
触发外设或线程。
```

## 7. 任务化候选边界

因果任务化第一轮只允许形成“是否缺少任务材料”的候选判断，不创建任务。

若候选缺少证据，只能返回：

```text
缺目标状态。
缺实际状态。
缺来源动态。
缺来源动作。
缺来源方法。
缺二次特征组成项。
```

任务化候选进入任务系统前必须另建专项，明确：

```text
写入方。
读取方。
生命周期。
任务授权来源。
任务承接关系。
拒绝后结构不变化检查方式。
```

## 8. 入口拒绝矩阵

| 编号 | 拒绝项 | 处理 |
| --- | --- | --- |
| FS02-CAUSE-01 | 查询请求为空 | 返回空候选，不写结构 |
| FS02-CAUSE-02 | 动态证据无效或错误节点类型 | 拒绝该证据，不写结构 |
| FS02-CAUSE-03 | 动态材料缺主体、场景、目标、前值、后值或时间戳 | 返回缺失证据候选 |
| FS02-CAUSE-04 | 状态材料无效或缺主体 / 场景 / 值 / 时间戳 | 返回缺失证据候选 |
| FS02-CAUSE-05 | 来源动作不是动作入口角色节点 | 拒绝动作来源候选 |
| FS02-CAUSE-06 | 来源方法无效或错误节点类型 | 拒绝方法来源候选 |
| FS02-CAUSE-07 | 候选跨不兼容场景且无明确跨场景设计 | 拒绝单一模板候选 |
| FS02-CAUSE-08 | 查询请求要求自动创建需求、任务、方法或结算 | 拒绝写入请求 |
| FS02-CAUSE-09 | 查询请求包含 SQL、控制面板、D455、体素、外设、线程或日志材料 | 拒绝外部材料 |
| FS02-CAUSE-10 | 候选要求把轻量因果引用解释为稳定结论 | 拒绝升级 |

拒绝后必须保持节点、主信息、关系、索引、状态、动态、二次特征、因果、需求、任务、方法和结算记录不变化。

## 9. 与后续服务关系

| 后续方向 | 可接收内容 | 仍需门禁 |
| --- | --- | --- |
| 因果服务 | 动态证据集合、缺失证据、来源动作 / 方法候选 | 稳定因果结论和模板持久化专项 |
| 任务服务 | 缺失证据候选、目标状态候选 | 任务化专项和任务授权门禁 |
| 方法服务 | 来源方法候选、运动基元候选 | 方法候选召回和条件结果对设计 |
| 需求结算记录 | 目标状态、实际状态、动态证据候选 | 结算写入规则和验收专项 |
| 显示只读视图 | 候选摘要和缺失证据说明 | 只读 DTO / 视图边界 |

候选材料不得自动推进到上述服务。任何写入都必须由对应服务专项确认。
