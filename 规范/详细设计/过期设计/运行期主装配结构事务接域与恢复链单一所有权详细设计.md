# 运行期主装配结构事务接域与恢复链单一所有权详细设计

日期：2026-07-12

状态：历史设计证据 / JY-290、JY-295 旧装配假定已由 JY-336-JY-338 替代 / #249 当前实施依据改为 `规范/详细设计/运行期上下文首次发布租约与恢复链单一所有权详细设计.md`

## 1. 定位与依据

本设计承接 #217 隔离事务合同和 #246 主装配 S0，建立唯一生产运行期上下文、四仓库接域、领域服务接线、失败清理和恢复链同所有者验收。流程图为 `流程图/20260712_运行期主装配结构事务接域与恢复链单一所有权流程图_v0.1.md` 及同名 HTML。

## 2. 当前事实与假定合同

当前事实：#217、#246、#247 已正式完成；当前入口主装配仍未切换，恢复链代码未实现。#248 首次执行前复核发现没有最终服务成员 / 构造 / 接线 / 初始化 / 清理矩阵，现由 JY-295 修订。

## 3. 唯一运行期上下文

最终 `运行期上下文` 持有唯一协调状态、接线、四仓库和领域服务装配；同一 `启动.运行期上下文` 模块中的宿主持有发布状态和租约源。写入方仅为启动装配模块；线程、SQL、控制面板、快照和恢复只持只读租约。生命周期从启动候选构造开始，#248 完成服务装配且完整读回后才一次发布；停止时先停消费者并释放租约，再由宿主释放上下文，按服务、索引、关系、节点、主信息、接线、协调器逆序析构。

## 4. #247 主装配接域

新建生产真模块 `启动.运行期上下文` 与自检真模块 `自检.运行期上下文`。同一协调器只生成一份正式接线，按主信息、节点、关系、索引顺序构造四仓库，并建立宿主 / 租约骨架。#247 阶段没有正式领域服务装配，结构核心必须不可生产发布；660 验证同域身份、构造 / 析构骨架和不完整发布拒绝。任何仓库未接域、重复接域、不同状态混装或结构核心被当作完整上下文发布均写前拒绝；构造后身份矛盾归追根因。

## 5. #248 服务接线与失败清理

新建生产真模块 `装配.领域服务` 与自检真模块 `自检.领域接线`，并允许回改 `启动.运行期上下文.ixx` / `自检.运行期上下文.ixx`。#248 硬前置改为 #214，传递包含 #221-#223。`装配.领域服务` 只接收接线和四仓库引用，不 import 或拥有上下文；`启动.运行期上下文` 单向 import 服务装配模块。

长期成员按 JY-295 矩阵固定为存在至显示 16 项。四个初始化服务只在全新候选工厂局部构造，生成值式系统角色材料后销毁。全新候选必须逐项读回世界 / 语素 / 根需求 / 概念根 / 方法登记根材料，才允许 `完整()` 为真并由同一空宿主首次发布。仓库快照、控制面板、SQL、线程及算法 / 路由模块不计入完整性，只作发布后消费者。

#246 的删除 / 失败点继续按六类裁决：测试隔离、写前无结构、创建期补偿、已发布删除、候选丢弃、内部不一致清理。已发布结构删除必须走正式所有者 / 事务入口；内部不一致不能静默补偿。

## 6. #249 端到端验收

新建 `自检.主装配事务` 真模块，证明全新启动、正常读写、拒绝不变式、失败清理、快照冻结许可来源和恢复发布许可均引用同一种正式上下文身份。该切片只验收和窄幅修正 #247/#248 模块，不实现 HYSNAP01 或恢复算法。

## 8. 非成功二分

外部模式无效、接线材料缺失、#247 结构核心因缺服务拒绝发布、候选未发布属于逻辑内返回；已经完成接域、创建、删除、清理或完整发布后身份、读回、计数不一致属于追根因解决。

## 9. 依赖与漂移

`#217 -> #246 -> #247` 与 `#221 -> #222 -> #223 -> #214` 汇合到 `#248 -> #249`。#248 执行前必须逐项复核 JY-295 矩阵；实际成员、构造、初始化或恢复裁决不一致时不得改代码，退回设计修订。#249 通过后才允许按 JY-277 解锁或正式修订恢复链。

## 10. 完成边界

只可声明单进程生产主装配已纳入统一结构事务合同。不得声明 ACID、跨进程、断电恢复、数据库恢复、自我苏醒或旧能力迁移完成。
