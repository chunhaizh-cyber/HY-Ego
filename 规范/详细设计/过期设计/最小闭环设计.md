# 最小闭环设计

编号：003
日期：2026-07-05
状态：阶段 0 正式规范

## 1. 定位

最小闭环用于验证第一版内核能以结构方式完成一次需求、任务、方法和结果结算。

本设计不证明自我苏醒，不证明完整自我循环，不证明旧项目能力迁移。

## 2. 闭环目标

第一条闭环固定为：

```text
创建自我存在
创建现实场景
创建一个特征类型
写入自我某个 I64 特征值
创建目标状态需求：让该特征状态材料满足目标状态
由需求生成任务
任务选择一个手写方法
方法修改特征值
记录动作动态
需求根据结果结算
```

## 3. 输入入口

第一版闭环分为种子准备阶段和闭环触发阶段。

### 3.1 种子准备阶段

种子准备阶段只建立闭环所需前置结构，不等于闭环触发：

```text
世界服务创建自我存在。
世界服务创建现实场景。
特征服务创建目标特征类型。
特征服务写入当前特征状态材料。
```

### 3.2 闭环触发阶段

闭环触发入口为：

```text
创建需求
```

创建需求时必须给出：

```text
目标存在
目标特征
目标状态
满足方式
验证读取路径
```

外部输入只作为请求材料，不直接成为世界事实。

第一轮 DV-1 口径中：

```text
目标值落为目标状态材料，不作为需求目标的对外语义。
满足方式固定为：当前特征状态材料 >= 目标状态值。
验证读取路径固定为：读取目标特征状态材料、任务状态、需求结算状态、动态和因果引用结构。
比较方式 / 验证方式属于后续通用结算规则，必须进入需求或结算规则专项后才能扩展。
```

需求服务不得在 `创建需求` 内偷偷补建目标存在、目标特征、目标状态或当前特征状态材料。目标存在无效、目标特征无效、目标状态非法、当前特征状态材料不存在、满足方式未确认或验证读取路径缺失时，必须在入口前置拒绝。

## 4. 结构承载

闭环必须至少落入以下结构：

```text
存在
场景
特征
特征值
需求
任务
方法
动态
因果引用
```

各结构职责如下：

| 结构 | 职责 |
| --- | --- |
| 存在 | 承载自我对象身份 |
| 场景 | 承载现实场景身份 |
| 特征 | 承载目标特征类型和实例特征 |
| 特征值 | 承载 I64 当前值和更新后的值 |
| 需求 | 承载目标状态引用、满足判断和结算结果 |
| 任务 | 承接需求并推进状态 |
| 方法 | 承载手写方法入口和执行结果 |
| 动态 | 记录方法执行造成的动作动态 |
| 因果引用 | 以来源动态作为准入证据创建轻量因果引用 |

日志、控制台输出、说明文字不得替代上述结构。

## 5. 值类型

第一版特征状态材料使用：

```text
I64 状态值
```

需求目标是目标状态，不是 I64。I64 只作为第一轮状态值材料或特征状态材料。

二态判断不得直接扩散为普通 bool 机器事实。

需求满足判断应由以下材料生成：

```text
当前特征状态材料
目标状态
第一轮固定满足规则：当前特征状态材料 >= 目标状态值
来源证据
结算状态发生时间戳
```

## 6. 调用路径

最小闭环调用路径固定为：

```text
需求服务
-> 任务服务
-> 方法服务
-> 世界服务 / 特征服务
-> 因果服务
-> 需求服务结算
```

业务层不得直接写节点仓库、主信息仓库、关系仓库或索引仓库完成闭环。

## 7. 核心处理

闭环处理步骤：

```text
种子准备：
1. 世界服务创建自我存在和现实场景。
2. 特征服务创建目标特征类型。
3. 特征服务写入自我目标特征当前状态材料。

闭环触发：
4. 需求服务接收已有目标存在、目标特征、目标状态、满足方式和验证读取路径。
5. 需求服务在入口拒绝无效目标存在、无效目标特征、非法目标状态、缺少当前特征状态材料、未确认满足方式或缺少验证读取路径。
6. 需求服务创建目标状态需求并记录目标状态关系。
7. 任务服务根据需求创建任务。
8. 方法服务选择手写方法。
9. 方法服务执行方法并请求特征服务修改目标特征状态材料。
10. 动态服务记录实例动态，因果服务以来源动态作为准入证据创建轻量因果引用。
11. 需求服务读取结果并结算需求状态。
12. 写入入口前置拒绝无效节点、无效关系和不允许的业务写入。
```

## 8. 输出变化

最小闭环必须产生以下输出变化：

```text
目标特征值发生变化
任务状态发生变化
需求结算状态发生变化
动作动态新增
因果引用新增
```

控制台输出只能展示这些结构变化，不能作为输出变化本身。

## 9. 验证方式

最小闭环验证分级：

```text
文档已生成：本设计和重构总纲存在。
代码已编译：项目 Debug x64 构建通过。
入口前置拒绝通过：仓库和领域服务入口拒绝无效节点、无效关系和不允许的业务写入。
最小闭环通过：创建需求后，目标特征值、任务状态、需求结算、动态和因果引用均发生结构变化。
```

未完成入口前置拒绝前，不得声明最小闭环通过。

未接入真实外设样本前，不得声明真实样本验证通过。

## 10. 禁止宣称

最小闭环通过后仍不得宣称：

```text
自我苏醒
完整自我循环
旧项目能力已迁移
D455 外设已接入
体素系统已接入
控制面板已接入
真实外设样本通过
```
