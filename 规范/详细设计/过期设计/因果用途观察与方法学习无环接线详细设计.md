## 5. O0 数据合同

### 5.1 `因果完整结构键`

在现有 `海中鱼巣.领域.材料.因果模式` 中新增：

```cpp
inline constexpr std::uint32_t 因果完整结构键格式版本 = 1;

struct 因果完整结构键 {
    动态模式结构键 原因动态键;
    动态模式结构键 结果动态键;
    std::uint32_t 因果规则版本 = 0;
    std::uint32_t 格式版本 = 因果完整结构键格式版本;

    bool 局部完整() const noexcept;
};
```

字段顺序即角色顺序。`局部完整()` 只检查格式非零、两把动态键局部完整和因果版本非零；原因不等于结果及版本兼容由算法入口裁决，避免材料模块反向依赖算法模块。

### 5.2 处理结果

同一材料模块新增强类型结果，禁止普通 bool 模糊“无效 / 不支持 / 内部错误”：

```cpp
enum class 因果完整结构键处理状态 : std::uint8_t {
    已形成 = 1,
    入口拒绝 = 2,
    不支持 = 3,
    内部不一致 = 4
};

enum class 因果完整结构键拒绝原因 : std::uint8_t {
    无 = 0,
    候选不完整 = 1,
    原因结果相同 = 2,
    版本不兼容 = 3,
    算法不变量破坏 = 4
};

struct 因果完整结构键结果 {
    因果完整结构键处理状态 状态 = 因果完整结构键处理状态::入口拒绝;
    因果完整结构键拒绝原因 拒绝原因 = 因果完整结构键拒绝原因::候选不完整;
    std::optional<因果完整结构键> 身份;
    bool 成功() const noexcept;
};
```

合法组合固定为：

| 状态 | 原因 | 身份 |
| --- | --- | --- |
| 已形成 | 无 | 有值，且身份局部完整、版本兼容、原因与结果不同 |
| 入口拒绝 | 候选不完整 / 原因结果相同 | 无值 |
| 不支持 | 版本不兼容 | 无值 |
| 内部不一致 | 算法不变量破坏 | 无值 |

`成功()` 只有在第一行全部成立时返回真；默认对象必须是确定的入口拒绝态。算法不得构造其它状态—原因—身份组合；若内部路径产生其它组合，归为算法不变量破坏，不向调用方传播半合法结果。

### 5.3 算法公开入口

在现有 `海中鱼巣.领域.算法.因果模式` 中新增：

```cpp
bool 因果完整结构键版本兼容(const 因果完整结构键&) noexcept;
bool 因果完整结构键相同(const 因果完整结构键&, const 因果完整结构键&) noexcept;
std::optional<std::uint64_t> 计算因果完整结构键稳定哈希(const 因果完整结构键&) noexcept;
因果完整结构键结果 形成因果完整结构键(const 因果概率候选&);
```

`形成因果完整结构键` 的顺序固定：

```text
候选局部完整
-> 两把动态键局部完整
-> 原因与结果全字段不相同
-> 格式 1 兼容表复核
-> 按原因 / 结果固定角色复制字段
-> 用 A/B 比较复核输出与来源三元组一致
-> 返回已形成
```

候选机会组、计数和 Q10000 仍由现有 `因果概率候选::完整()` 与用途深检负责；完整结构键不复制机会组，也不把概率计数变成身份字段。

### 5.4 完整比较

```text
先比较格式版本。
再调用现有动态模式结构键全字段比较比较原因角色。
再比较结果角色。
最后比较因果规则版本。
```

`因果完整结构键相同` 只在左右两项均局部完整、原因与结果不同且版本兼容时执行字段比较；任一项无效或不支持时返回假，包括两个相同的默认无效对象，禁止把无效键归并成同一身份。不得先比哈希再把哈希相等直接当身份相同。调用方可用哈希筛桶，但最终结果必须来自 `因果完整结构键相同`。

### 5.5 稳定哈希

稳定哈希必须以固定常量区分字段和角色：

```text
格式域标记
-> 格式版本
-> 原因角色标记
-> 原因动态键稳定哈希
-> 结果角色标记
-> 结果动态键稳定哈希
-> 因果版本域标记
-> 因果规则版本
```

原因 / 结果交换必须得到不同的混入序列。哈希函数不接收时间源、进程随机种子、地址或容器迭代顺序。

哈希入口只接受局部完整、原因与结果不同且版本兼容的身份；其它输入返回 `std::nullopt`，不得返回可进入候选桶的默认哈希。即使两个合法不同身份产生相同哈希，仍必须保留为两个候选并做完整比较。

### 5.6 版本兼容

格式 1 的唯一兼容行：

| 身份格式 | 原因动态版本 | 结果动态版本 | 因果版本 | 结果 |
| --- | --- | --- | --- | --- |
| 1 | `动态模式规则版本` | `动态模式规则版本` | `因果模式规则版本` | 支持 |
| 其它 | 任意 | 任意 | 任意 | 不支持 |

未来扩展只能新增显式行；不能把 `原因版本 == 结果版本 == 因果版本` 写成永久规则。

### 6.2 效用形成

```text

-> 调用 `形成因果完整结构键`
```
