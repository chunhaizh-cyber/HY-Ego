## 1. 定位

本详细设计对应：


领域服务负责把业务请求转为允许写入的结构变化。仓库只负责最小结构，领域服务负责业务准入、节点类型检查、生命周期检查和必要前置结构检查。

本文件只描述第一轮最小入口，不证明完整领域系统完成，不进入旧项目能力迁移。

## 2. 总体规则

第一轮领域服务入口必须满足：

```text
函数入口先排除无效句柄、错误节点类型、已删除节点和缺少前置结构。
拒绝路径只返回无效句柄、false、std::nullopt 或领域枚举结果。
拒绝路径不得写业务事实。
日志、弹窗和控制台输出只做人读诊断。
业务事实必须落到节点、关系、已确认领域值容器、状态节点、动态节点或因果引用节点。
```

目标领域服务分层：

```text
特征值服务
特征服务
存在服务
场景服务
状态服务
动态服务
二次特征服务
因果服务
世界服务
需求服务
任务服务
方法服务
语素服务
```

其中 `世界服务` 是基础信息聚合服务；特征值、特征、存在、场景、状态、动态、二次特征、因果是基础信息原子服务；需求、任务、方法是高级信息服务；语素服务只负责文本入口绑定。

当前第一轮代码已包含以下服务入口：


当前第一轮拆分不代表目标服务分层已经完整完成；抽象状态 / 抽象动态治理、完整依赖注入治理和旧项目能力迁移仍未完成。

## 2.1 基础信息服务目标边界

| 特征服务 | 管理特征入口和特征状态材料读写入口 | 第一轮已有 `创建特征类型`，并已封装特征值创建 / 读取 / 写入 |
| 存在服务 | 管理实际存在和虚拟存在 | 第一轮已拆出，世界服务创建存在时调用存在服务 |

| 状态服务 | 区分实例状态和抽象状态 | 第一轮已拆出；实例状态发生时间戳入口拒绝已实现；任务状态和需求结算状态经状态服务创建 |
| 动态服务 | 区分实例动态和抽象动态 | 第一轮已拆出，动态记录经动态服务创建，因果服务不再直接创建动态 |
| 二次特征服务 | 管理关系性、程度性或复合判断前置特征 | 第一轮已拆出，特征服务创建二次特征时调用二次特征服务 |
| 因果服务 | 管理因果引用、证据和抽象候选 | 第一轮已有最小因果引用入口 |
| 世界服务 | 编排基础信息原子服务 | 第一轮已改为存在 / 场景聚合编排；基础信息通用节点仍保留兼容壳 |

高级信息服务必须通过基础信息服务取得基础能力：

```text
需求服务通过存在服务、场景服务、特征服务和状态服务工作。
任务服务通过存在服务、场景服务、特征服务、状态服务和动态服务工作。
方法服务通过存在服务、场景服务、特征服务、动态服务和因果服务工作。
```

高级服务不得直接依赖特征值服务；特征值节点的创建、读取和写入必须由特征服务对外封口。

语素服务只绑定已存在信息节点，不创建基础信息、高级信息或临时实例状态 / 动态。

## 4. 当前实现边界

### 4.1 因果引用轻量化口径

因果引用不是单条动态的证明书。

动态是因果判断的证据材料。因果可靠性由后续因果聚合、样本积累、评分、版本和复判机制形成。

第一轮 `记录因果引用(来源动态)` 只用来源动态做入口准入检查，不持久化来源动态关系。

不得把单个因果引用节点解释为稳定因果结论。

### 4.2 方法执行权限口径

方法执行权限来自任务授权、任务承接和任务选择方法。

方法服务不单独生成执行前授权状态。

任务一旦通过任务服务承接需求并选择方法，方法执行即具备通过特征服务请求修改目标特征状态材料的权限。

第一轮方法执行入口检查：

```text
任务节点有效。
方法节点有效。
任务到方法关系有效。
目标特征节点有效。
特征值节点归属于目标特征。
特征服务允许写入该状态材料。
```

本轮只声明任务方法关系入口拒绝已实现，不声明完整方法权限系统完成。

当前可以声明：

```text
第一轮领域服务最小入口已存在。
基础信息服务分层第一轮代码重构已验证。
入口前置拒绝第一轮样例已通过。
需求、任务、方法、动态、因果引用可组成控制台入口的最小结构样例。
```

当前不得声明：

```text
完整领域服务已完成。
候选方法选择已完成。
任务状态机已完成。
需求树已完成。
因果解释系统已完成。
旧项目能力已迁移。
```
