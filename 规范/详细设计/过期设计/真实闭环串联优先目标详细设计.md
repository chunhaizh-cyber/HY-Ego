## 1. 定位

本详细设计承接用户口径：当运行宿主与多线程消息队列专项计划完成后，下一批优先目标应把已完成的分散能力串成更真实的闭环。

目标链路为：

```text
需求
-> 任务承接
-> 方法候选 / 选择
-> 动作请求
-> 动态记录
-> 任务回执
-> 需求结算
-> 非权威缓存 / 事件日志 / 只读显示材料
```

本设计不是代码实施许可，不声明真实闭环、运行宿主线程、自我循环、自我苏醒、旧能力迁移、完整任务状态机、完整方法执行系统或完整缓存 / 事件日志 / 显示系统完成。

## 4. 串联原则

1. 需求目标仍由目标状态承载。
2. 任务承接必须复核需求、主体、目标宿主、场景和目标状态。
3. 方法候选 / 选择第一轮只允许固定验收样例、读取既有任务选择方法关系或人工明确选择；不得实现自动召回排序、自动选择、在线学习或评分更新，也不得把候选材料当成功事实。
4. 动作请求来自任务选择方法和方法动作入口，不来自线程、Tick、日志标签或控制台命令。
5. 任务工作线程只能调用领域服务入口，不能裸写仓库。
6. 动态记录必须有存在主体、场景归属和发生时间戳。
7. 任务回执和需求结算必须回到任务服务、需求服务、状态服务、动态服务和因果服务复核。
8. 非权威缓存、事件日志和只读显示材料只能观察，不裁决。

## 5. 目标结构

| 环节 | 权威结构 | 非权威 / 人读材料 | 第一轮目标 |
| --- | --- | --- | --- |
| 需求 | 需求节点、目标状态关系 | 显示只读材料 | 读取或创建一个目标状态需求 |
| 任务承接 | 任务节点、任务到需求关系、任务状态 | 任务候选材料 | 生成可调度任务材料 |
| 方法候选 / 选择 | 任务选择方法关系 | 候选只读材料、缓存候选 | 通过固定样例、既有关系或人工明确选择得到一个保守方法入口 |
| 动作请求 | 方法动作入口、动作规格、执行桥请求材料 | 请求消息 | 生成动作请求但不让消息成事实 |
| 动态记录 | 实例动态节点、状态变化材料 | 动态候选材料 | 记录实际动作结果动态 |
| 任务回执 | 实际结果状态关系、完成状态关系 | 回执消息 | 回写任务实际结果状态 |
| 需求结算 | 结算状态、来源任务、动态证据、可选因果引用 | 结算观察材料 | 复核目标状态与实际状态 |
| 缓存 / 日志 / 显示 | 无权威裁决 | 非权威缓存、事件日志、只读显示材料 | 观察闭环结果，不反向改事实 |

## 中途非成功返回二分口径

本文件按 2026-07-09 硬规则修订：中途非成功返回只分为 `追根因解决` 和 `逻辑内返回`。

- `追根因解决`：前置条件已经满足，并进入创建、绑定、写关系、写状态、记录动态、结算、读回或结构承载后，结果不符合内部预期；必须停止依赖路径，定位根因，当前未证明完整回滚时登记事务隔离缺口或半结构隔离缺口。
- `逻辑内返回`：领域协议允许的拒绝、候选为空、请求材料返回或人读材料返回；必须保证结构不变化，且返回材料、日志、回执、显示或控制台输出不裁决机器事实。
