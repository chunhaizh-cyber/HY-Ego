## 2. 定位

特征值序列化材料是后续仓库快照、事件日志或恢复候选材料的一部分。它只描述特征值系统内部可恢复材料，不裁决当前运行期事实。

第一轮只允许设计：

```text
I64 原始值材料。
VecI64 原始值材料。
VecU64 原始值材料。
特征值节点身份。
特征归属复核材料。
容器版本或值版本。
非权威缓存材料的丢弃或候选边界。
```

第一轮不允许设计成：

```text
直接恢复当前事实。
绕过特征服务写特征值。
把缓存字段恢复为业务事实。
把旧 VecU / VecI64 二进制格式作为新格式。
把体素、外设或 SQL 字段纳入特征值恢复。
```

## 3. 序列化格式版本

特征值序列化材料必须包含独立格式头：

| 字段 | 含义 | 第一轮要求 |
| --- | --- | --- |
| 特征值序列化格式版本 | 特征值材料格式版本 | 必填；未知版本拒绝 |
| 节点类型枚举版本 | 特征值节点类型数值版本 | 必填；必须匹配当前枚举锁定口径 |
| 值类型枚举版本 | I64 / VecI64 / VecU64 值类型版本 | 必填；未知类型拒绝 |
| 仓库集合编号 | 所属快照或恢复上下文 | 必填；必须与目标恢复上下文匹配 |
| 材料序号 | 序列化材料内部顺序 | 只作审计和排序材料 |
| 人读摘要 | 审计说明 | 可选；不得承载机器逻辑 |

格式版本只说明材料解释方式，不证明材料可恢复为当前事实。

## 4. 原始值材料字段

| 值类型 | 必填字段 | 禁止字段 |
| --- | --- | --- |
| I64 | 特征值节点句柄、特征节点句柄、I64 值、值版本 | 命中次数、控制面板显示字段、旧主信息字段 |
| VecI64 | 特征值节点句柄、特征节点句柄、元素数量、I64 元素序列、容器版本 | 裸指针、旧 VecU 二进制块、体素编码 |
| VecU64 | 特征值节点句柄、特征节点句柄、元素数量、U64 元素序列、容器版本 | 裸指针、旧 VecU 二进制块、体素编码 |

Vec 材料必须显式记录元素数量。元素数量与实际元素序列不一致时拒绝恢复。空序列是否允许，必须由后续代码实施切片确认；未确认前默认拒绝空 Vec 作为可恢复材料。

## 5. 身份和归属复核

反序列化前必须先复核：

```text
特征值节点句柄有效。
特征值节点类型为特征值。
特征节点句柄有效。
特征节点类型为特征。
特征值归属于目标特征。
句柄仓库编号匹配目标恢复上下文。
句柄版本与快照或候选区材料一致。
值类型与既有特征值材料类型不冲突。
```

身份复核失败时，不得创建特征值节点，不得写主信息值容器，不得补关系，不得写缓存。

## 6. 与仓库快照和事件日志的关系

特征值序列化材料不得绕过 FS-09 快照恢复拒绝矩阵。

```text
仓库快照先复核节点、主信息、关系和索引段。
特征值序列化材料必须在节点身份、主信息身份和特征归属关系均已通过候选复核后才能进入特征值恢复候选。
事件日志只能作为事件审计和候选补充，不得修补不完整快照后直接写当前结构。
```

若快照段缺失或关系端点复核失败，特征值序列化材料只能被丢弃或记录为人读诊断，不能被解释为当前特征值事实。

## 7. 非权威缓存恢复边界

以下材料不得恢复为业务事实：

```text
内容哈希。
命中次数。
最近命中序号。
候选热度。
稳态候选。
值域候选评分。
复合比较候选。
```

第一轮恢复策略：

| 材料 | 恢复口径 |
| --- | --- |
| 内容哈希 | 可作为候选审计材料；恢复后必须重新由原始值生成，不直接采信 |
| 命中次数 | 默认丢弃或作为人读诊断；不得影响事实读取 |
| 稳态 / 值域 / 比较候选 | 不随特征值原始值恢复；后续另建二次特征或候选治理专项 |
| 可持久化标记 | 只说明缓存是否可进入候选材料，不说明可恢复为事实 |

缓存字段缺失或损坏不得阻止权威原始值恢复候选；缓存字段存在也不得让无效原始值通过。

## 8. 反序列化拒绝矩阵

| 编号 | 拒绝项 | 拒绝原因 | 结构变化要求 |
| --- | --- | --- | --- |
| FS03-SER-R01 | 特征值序列化格式版本缺失或未知 | 不能解释材料字段 | 不写任何结构 |
| FS03-SER-R02 | 节点类型枚举版本不匹配 | 特征值节点类型可能复义 | 不写任何结构 |
| FS03-SER-R03 | 值类型未知 | I64 / VecI64 / VecU64 边界不明 | 不写值材料 |
| FS03-SER-R04 | 特征值句柄无效或版本不匹配 | 稳定身份不可信 | 不创建或修改特征值 |
| FS03-SER-R05 | 特征句柄无效或类型错误 | 归属目标不可信 | 不创建归属关系 |
| FS03-SER-R06 | 特征值不归属目标特征 | 越权恢复 | 不写值材料 |
| FS03-SER-R07 | Vec 元素数量与实际序列不一致 | 原始值材料损坏 | 不写 Vec 材料 |
| FS03-SER-R08 | 空 Vec 未经后续切片确认 | 容器语义未确认 | 不写 Vec 材料 |
| FS03-SER-R09 | 既有值类型与恢复值类型冲突 | 类型复义 | 不覆盖既有材料 |
| FS03-SER-R10 | 缓存字段要求恢复为事实 | 非权威材料越权 | 丢弃缓存字段，不写业务事实 |
| FS03-SER-R11 | 旧 VecU 二进制块要求直接解释 | 旧格式未迁移 | 拒绝该材料 |
| FS03-SER-R12 | SQL / 控制面板 / 外设字段混入 | 禁止材料混入 | 不解释、不迁移、不恢复 |

## 9. 恢复失败结构不变化

恢复失败必须保证：

```text
节点数量不变。
主信息数量不变。
关系数量不变。
索引数量不变。
既有特征值 I64 材料不变。
既有 Vec 容器材料不变。
既有特征到特征值归属关系不变。
需求、任务、方法、状态、动态、因果引用不变化。
缓存清空或缓存损坏不改变权威结构。
```

不得先写入半个特征值材料，再通过扫描器、日志、控制台输出、显示层或缓存重建修复。
