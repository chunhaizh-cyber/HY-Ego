## 1. 定位

本详细设计承接第 10 项 `方法候选召回与选择代码逻辑流程图`，用于把需求侧候选准入材料、任务侧选择上下文、方法结构候选读取、稳定动作键动作入口读取、候选材料汇合和任务选择方法关系，收束为可审查的第一轮方法候选召回与选择边界。

本设计不等于完整方法查找系统，不等于候选排序、条件判定、条件结果对归并、方法学习或自动选择策略已经落地。本设计不提供代码实施许可，不自动登记 `项目记忆/Codex任务队列.md`。后续若进入 C++ 实施，必须另行生成并确认施工计划，明确允许文件、禁止文件、入口拒绝、追根因解决收口、读回验证和完成声明边界。

## 4. 结构承载

| 业务概念 | 项目承载结构 | 值类型 / 身份结构 | 写入方 | 读取方 | 验证方式 |
| --- | --- | --- | --- | --- | --- |
| 需求候选准入材料 | 需求节点、主体、目标宿主、场景、目标状态 | 需求节点句柄、存在节点、场景节点、状态节点 | 需求服务既有入口 | 任务服务、方法候选读取方 | `读取需求承接材料` 完整可读 |
| 任务候选选择上下文 | 任务承接壳、筹办回执请求材料、执行桥请求材料 | 任务节点、来源需求、运行场景、目标状态、任务选择方法 | 任务服务 | 方法候选读取方、方法执行流程 | 任务承接壳完整，筹办回执状态为可作为请求材料 |
| 方法结构候选材料 | 方法首、方法虚拟存在、条件节点、结果节点、动作入口 | 方法节点句柄、存在节点、场景节点、动作入口角色节点 | 方法服务 | 任务服务、方法候选读取方 | 方法首有效，条件 / 结果集合和动作入口可读 |
| 稳定动作键入口材料 | 索引仓库主键到动作入口节点映射 | `uint64` 稳定动作键、动作入口节点句柄 | 方法服务登记动作入口时写入 | 方法服务 | 稳定动作键非 0，命中入口归属有效方法首 |
| 候选材料汇合结果 | 只读候选材料集合 | 句柄组、材料状态、拒绝原因 | 当前第一轮不写权威候选事实 | 方法候选读取方、显示层只读 | 不排序、不自动选择、不写任务方法关系 |
| 任务选择方法关系 | 任务到方法引用关系，顺序号为任务选择方法 | 任务节点句柄、方法节点句柄、关系句柄 | 任务服务 | 任务服务、方法执行流程 | 任务承接壳完整，目标节点为方法，当前无选择方法，读回唯一 |

候选材料不是方法可执行事实。候选召回结果、缓存命中、统计次数、显示文本和 SQL 投影都不能成为任务选择方法的权威依据。

## 5. 请求分流

```text
需求侧候选准入
-> 需求服务读取需求承接材料
-> 主体、目标宿主、场景、目标状态完整
-> 输出需求候选准入材料
-> 不裁决候选方法。

任务侧选择上下文
-> 任务服务读取任务承接材料
-> 读取筹办回执请求材料
-> 材料状态为可作为请求材料
-> 输出任务候选选择上下文
-> 不自动选择方法。

方法结构候选读取
-> 复核方法首有效
-> 读取方法虚拟存在、条件节点集合、结果节点集合和动作入口
-> 动作入口必须归属有效方法首
-> 输出候选方法结构材料
-> 不证明满足需求目标。

稳定动作键入口读取
-> 稳定动作键非 0
-> 索引仓库命中动作入口
-> 方法服务复核动作入口归属方法首
-> 输出动作入口材料
-> 索引只做入口召回。

候选材料汇合
-> 若后续候选规则未确认，只输出只读材料
-> 若后续规则确认，再复核目标状态、条件场景、结果场景、动作入口、来源任务和策略状态
-> 候选唯一或选择策略授权后，才允许请求任务服务选择任务方法。
```

## 6. 入口前置拒绝

后续若进入代码实施，必须先拒绝：

```text
无效需求节点。
需求承接材料缺主体、目标宿主、场景或目标状态。
无效任务节点。
任务承接壳不完整。
筹办回执请求材料状态不是可作为请求材料。
无效方法节点或方法首无效。
方法首缺唯一虚拟存在。
方法条件节点或方法结果节点角色错误。
条件 / 结果场景缺失、不唯一或目标不是场景节点。
动作入口缺失、失效或不归属该方法首。
稳定动作键为 0 或索引命中目标不是有效动作入口。
多候选且无授权选择策略。
候选算法未确认却试图写任务选择方法关系。
任务已有当前选择方法却重复写入。
方法服务或任务服务试图直接依赖特征值服务。
日志、显示、SQL、控制台文本、旧函数名或动作名文本被当作候选裁决依据。
```

拒绝后不得写任务选择方法关系、候选裁决事实、方法状态、缓存命中或统计事实。若入口前置拒绝已经排除错误输入，内部写入结果仍不符合预期，必须按逻辑错误追根因。

## 7. 服务边界

| 服务 | 本设计允许 | 禁止越界 |
| --- | --- | --- |
| 需求服务 | 提供需求承接材料和方法候选准入材料 | 不召回方法，不写任务选择方法关系 |
| 任务服务 | 提供承接壳、筹办回执请求、执行桥请求；在授权时写任务选择方法关系 | 不计算候选排序，不证明方法满足需求目标 |
| 方法服务 | 读取方法首、条件、结果、动作入口和稳定动作键入口材料 | 不直接依赖特征值服务，不写任务事实 |
| 索引仓库 | 根据稳定动作键读取动作入口候选 | 不裁决方法可执行事实 |
| 特征服务 / 状态服务 | 后续候选规则需要特征状态材料时提供正式读取入口 | 不由方法 / 任务服务绕过调用特征值服务 |
| 显示层 | 只读展示候选材料 | 不写候选事实、不裁决选择结果 |

## 中途非成功返回二分口径

本文件按 2026-07-09 硬规则修订：中途非成功返回只分为 `追根因解决` 和 `逻辑内返回`。

- `追根因解决`：前置条件已经满足，并进入创建、绑定、写关系、写状态、记录动态、结算、读回或结构承载后，结果不符合内部预期；必须停止依赖路径，定位根因，当前未证明完整回滚时登记事务隔离缺口或半结构隔离缺口。
- `逻辑内返回`：领域协议允许的拒绝、候选为空、请求材料返回或人读材料返回；必须保证结构不变化，且返回材料、日志、回执、显示或控制台输出不裁决机器事实。
