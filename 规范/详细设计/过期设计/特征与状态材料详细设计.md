## 5. 请求分流

特征与状态材料入口必须先把上游材料降权为请求，再按请求类型分流。

```text
创建特征类型 -> 特征服务创建主信息和特征节点。
读取特征身份 / 语义类型 -> 特征服务复核节点类型。
创建实例特征槽位 -> 特征服务验证宿主、特征类型和重复槽位后写关系。
读取宿主特征集合 / 槽位 -> 特征服务只返回已绑定特征类型的槽位材料。
写入宿主 I64 特征状态材料 -> 特征服务找到槽位，找或创建槽位归属特征值，再经特征值服务内部入口写 I64。
读取宿主 I64 特征状态材料 -> 特征服务复核槽位、特征值归属和 I64 值。
需求目标状态 -> 状态服务创建抽象状态，需求服务以模板关系承载唯一目标状态。
高级服务需要特征状态材料 -> 必须经特征服务，不得直接访问特征值服务。
Vec / 哈希 / 值域 / 稳态 / 序列化恢复 -> 后续候选门禁，不在本设计中落代码。
```

## 6. 入口前置拒绝

所有写入口必须在写结构前拒绝：

```text
无效特征节点。
错误特征节点类型。
无效宿主节点。
宿主类型不允许挂接特征槽位。
无效特征类型。
重复实例特征槽位。
关系仓库或特征值服务内部依赖未注入。
无效特征值节点。
特征值不归属目标槽位或目标特征。
需求目标状态缺失、不唯一或不是状态节点。
高级服务直接包含、接收或调用特征值服务。
试图把 I64、Vec、哈希、缓存命中或候选评分直接当作需求目标、任务完成、方法成功或世界事实。
```

拒绝后必须保持：

```text
节点数量不因本次错误输入产生有效业务事实。
主信息数量不因本次错误输入产生有效业务事实。
关系数量不因本次错误输入产生有效业务事实。
索引数量不因本次错误输入产生有效业务事实。
既有特征状态材料不变。
既有需求目标状态不变。
```

若当前代码暂不能证明物理数量完全不变，后续施工计划必须采用读回隔离、失效清理或停止登记事务边界待确认问题，不得扩大声明。

## 8. I64 和目标状态边界

I64 只允许作为第一轮状态值材料或特征状态材料：

```text
特征服务可经内部特征值服务写入 I64 特征状态材料。
状态服务可创建状态值为 I64 的抽象状态。
需求目标对外语义必须是目标状态节点，不是 I64。
需求节点到目标状态节点当前由 `关系类型::模板` 承载；“目标状态关系”只是语义口径，不表示已新增独立关系枚举。
需求结算或方法条件如需数值比较，必须先读取目标状态和当前状态材料，再由对应服务复判。
```

禁止：

```text
高级服务直接访问特征值服务。
把 I64 直接写成需求目标主信息字段。
把特征值节点直接作为语素入口对应信息节点。
把哈希、缓存命中、候选评分或显示文本当成当前状态。
```

## 中途非成功返回二分口径

本文件按 2026-07-09 硬规则修订：中途非成功返回只分为 `追根因解决` 和 `逻辑内返回`。

- `追根因解决`：前置条件已经满足，并进入创建、绑定、写关系、写状态、记录动态、结算、读回或结构承载后，结果不符合内部预期；必须停止依赖路径，定位根因，当前未证明完整回滚时登记事务隔离缺口或半结构隔离缺口。
- `逻辑内返回`：领域协议允许的拒绝、候选为空、请求材料返回或人读材料返回；必须保证结构不变化，且返回材料、日志、回执、显示或控制台输出不裁决机器事实。
