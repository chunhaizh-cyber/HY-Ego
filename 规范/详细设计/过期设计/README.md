# 过期详细设计归档说明

日期：2026-07-20

状态：历史设计归档 / 不作为当前施工依据

## 1. 目录用途

本目录保存已经撤回、被后继详细设计替代，或核心前提已经被正式规范否定的详细设计。

归档文件只用于追溯历史设计、既有计划和实施事实，不得直接生成新计划、恢复旧合同或作为当前代码实施依据。当前机器逻辑以 `规范/规范目录.md` 登记的正式规范为准；需要继续实现的能力必须重新形成符合现行规范的流程图、详细设计和计划。

## 2. 归档判定

满足下列任一条件才进入本目录：

1. 文件正文明确声明已撤回、只保留历史证据或不再作为施工依据；
2. 文件明确给出已经替代它的后继详细设计；
3. 文件的核心业务合同已经被现行正式规范直接否定，无法通过局部勘误继续作为实施依据。

仅存在旧规范路径引用、已经完成实施、等待增强或需要局部修订，不自动判定为过期。
