## 1. 定位

本设计定义 `海中鱼巣` 后续自我治理循环的承载边界。自我治理循环的职责是接收非权威请求和回执材料，冻结本轮材料批次，完成规范化、分类、顺序裁决和治理动作候选生成，再把需要改变机器结构的事项逐条交回领域服务复核。

本设计不是自我线程函数迁移说明，不复制旧 `鱼巢` 自我线程函数体，不把旧治理 mailbox、旧消息结构、旧指针字段或旧执行前许可结论直接改写为新项目代码。

本设计生成不代表：

```text
自我治理 mailbox 已实现。
自我治理消息协议已实现。
自我循环或自我苏醒已实现。
运行宿主已驱动完整领域闭环。
旧项目能力已迁移。
```

## 3. 当前事实、旧证据与后续候选

### 3.1 当前海中鱼巣事实

```text
运行消息队列已经能够承载请求、材料、回执和停止信号，并拒绝消息承载机器事实。
运行宿主、任务线程壳、缓存事件线程壳和外设采样材料线程壳已经具备第一轮启动、停止和收口材料。
任务工作线程当前只在调用方显式调用时消费一次并生成回执材料，不调用领域服务。
缓存线程消费材料不调用统计服务；事件线程消费材料不调用日志入口。
默认入口已有同步固定闭环样例，但运行宿主尚未编排领域服务。
默认常驻入口只保持进程生命周期，不等于自我治理循环。
当前没有自我治理消息、治理 mailbox、有界治理队列或冻结批次结构。
```

### 3.2 旧鱼巢只读证据

旧项目可保留为逻辑证据的部分：

```text
任务管理上行材料需要转换为治理材料。
每轮处理前冻结 mailbox，避免本轮输入集合在处理中漂移。
治理材料需要规范化、分类、优先级和幂等裁决。
任务结果、派生需求、观察承接和执行前许可请求存在抢占处理需求。
纯线程状态不应落成业务事实。
```

旧项目明确不得迁移的部分：

```text
指针、文本主键、摘要或 bool 直接承载长期身份、权限或机器事实。
治理线程直接修改旧世界结构。
旧审批函数无条件设置“已审批=true、允许执行=true”。
控制面板刷新请求进入治理主干。
旧双向链表、SQL / ADO、D455、体素、外设或旧线程函数体。
```

## 4. 核心定义

| 名称 | 定义 | 权威性 |
| --- | --- | --- |
| 自我治理请求材料 | 外部调用方、任务管理上行、领域服务或自检候选提交的治理请求 | 非事实 |
| 自我治理消息 | 请求材料经协议规范化后的只读消息信封 | 非事实 |
| 治理 mailbox | 接收自我治理消息的入口缓冲区 | 非事实、非长期仓库 |
| 有界治理队列 | 负责并发入队、容量、优先级、幂等和停止收口的工程承载 | 非事实 |
| 冻结治理批次 | 某轮开始时从 mailbox 取得的不可变消息引用集合 | 非事实、轮次局部材料 |
| 治理动作候选 | 对一条治理消息建议执行的领域操作类型和输入材料 | 非事实 |
| 治理决议材料 | 规范化、分类和准入后的领域服务调用候选 | 非事实，不等于已执行 |
| 领域落账结果 | 领域服务正式入口完成写入并读回后的结构结果 | 可成为机器事实，来源是领域服务而非消息 |

## 5. 承载边界矩阵

| 承载对象 | 写入方 | 读取方 | 权威性 | 生命周期 | 验证方式 |
| --- | --- | --- | --- | --- | --- |
| 运行消息队列 | 运行宿主、任务 / 缓存 / 日志 / 外设线程壳 | 对应运行线程壳 | 请求与回执材料，不是事实 | 运行宿主生命周期 | 清空、重放或丢失不改变权威结构 |
| 自我治理消息 | 任务桥、领域请求适配器、自检候选适配器 | 治理 mailbox 准入入口 | 规范化材料，不是事实 | 从构造到完成 / 拒绝 / 过期 | 任何消息字段不得单独裁决权限或业务状态 |
| 治理 mailbox | 自我治理消息准入入口 | 冻结批次入口 | 非权威工程缓冲 | 运行期，停止后清空或审计 | 满载、重复、过期拒绝后结构不变 |
| 有界治理队列 | mailbox 内部或独立并发适配层 | 治理循环 | 非权威工程承载 | 运行期 | 并发入队、停止优先、顺序和幂等验证 |
| 冻结治理批次 | 治理循环冻结入口 | 规范化、分类和逐条归并入口 | 非权威轮次快照 | 单轮；完成后释放 | 冻结失败不产生可读半批次，处理中新增消息进入下一轮 |

裁决：后续默认采用独立有界治理队列，不直接复用 `有界运行消息队列` 作为治理事实容器。运行消息队列可以把任务上行材料送到桥接入口；桥接入口重新复核并构造自我治理消息，再进入治理 mailbox。`计划/20260710_SELF-GOV_自我内部治理闭环专项计划_v0.1.md` 已采用该默认裁决；编码仍必须由后续独立代码切片授权。

## 6. 自我治理消息协议候选

### 6.1 消息信封

后续协议候选至少包含：

```text
治理消息编号。
治理消息类型。
来源角色和来源服务。
目标需求、目标任务、目标方法、目标动作入口或目标材料的稳定句柄。
目标概念稳定句柄和来源材料版本。
每个稳定句柄的版本。
发生时间戳。
优先级。
幂等键。
来源任务序号或同任务顺序号。
批次编号。
只读材料类型与只读材料句柄组。
过期策略和拒绝原因候选。
```

协议不得包含：

```text
把指针当长期身份。
把摘要、日志、控制台文本或显示文本当机器事实。
把 `允许执行`、`任务完成`、`方法成功`、`需求满足` bool 当权威结果。
把运行线程状态当需求、任务、方法、状态或动态事实。
```

### 6.2 消息类型候选

| 类型 | 来源 | 目标 | 处理边界 |
| --- | --- | --- | --- |
| 任务结果复核请求 | 任务管理上行桥 | 任务服务 | 只提交实际结果和动态证据候选 |
| 派生需求请求 | 任务 / 需求分析入口 | 需求服务 | 只提交目标状态和当前材料候选 |
| 观察材料准入请求 | 外部材料或任务上行桥 | 对应基础信息服务 | 外部材料不直接成为世界事实 |
| 执行前许可预判请求 | 任务管理上行桥 | 任务 / 方法服务 | 只生成候选，不形成权限事实 |
| 自检缺口候选 | 已确认自检入口 | 后续治理服务 | 不自动创建需求或修复结构 |
| 缓存刷新请求 | 领域落账后观察桥 | 统计服务 | 非权威，可丢弃或重建 |
| 事件日志请求 | 领域落账后审计桥 | 日志系统 | 只作审计材料 |
| 停止请求 | 运行宿主 | 治理循环 | 最高优先级，只控制生命周期 |
| 概念待命名请求 | `上行桥.概念命名治理` 真模块 | 需求服务后继路由 | 只承载单个目标概念和当前活动图版本，不创建需求或绑定名称 |

控制面板刷新请求不进入自我治理协议主干；后续真实 UI 仍需独立专项。

### 6.3 协议版本 2 的概念待命名扩展

#211 把既有协议从版本 1 提升为版本 2，旧八类消息数值和语义保持不变，并锁定：

```text
自我治理消息类型::概念待命名请求 = 9
自我治理来源角色::概念图观察入口 = 7
自我治理服务角色::概念图 = 19
自我治理只读材料类型::概念待命名材料 = 9
自我治理只读材料句柄组::目标概念
自我治理消息::来源材料版本
```

概念待命名消息只允许 `目标概念` 和非零来源活动图版本，来源任务序号必须为零，任务、方法、动作、状态、动态等句柄必须为空。单请求上行桥只读持有概念图服务、语素服务和唯一治理邮箱，调用正式双参入口重读请求；读不到时再由语素名称入口反查区分“已有名称”和“当前请求不存在”。桥不调用批量待命名读取，不保存候选集合，不创建线程，也不写领域结构。

## 7. 规范化与结构准入

### 7.1 入队前置拒绝

以下情况属于逻辑内返回，且不得写机器结构：

```text
消息编号、类型、来源、时间戳、幂等键或批次材料缺失。
目标稳定句柄无效、版本过期或节点类型不匹配。
同一幂等键对应不同目标或不同材料摘要。
同一任务序号倒退。
消息试图承载机器事实或权限 bool。
队列容量为零、队列满载、队列停止或生产者已停止。
消息类型不在当前已确认白名单。
```

### 7.2 冻结准入

冻结批次必须遵守：

```text
只在治理队列短锁内移动或复制消息引用。
冻结成功后释放队列锁，再做规范化、领域读取或耗时计算。
冻结期间新消息留在 mailbox，进入下一轮。
冻结批次按批次编号和消息编号形成稳定顺序。
冻结失败不得形成部分可读批次；原 mailbox 保持可重试或整体拒绝状态。
```

### 7.3 结构准入

消息进入冻结批次仍不代表可以写结构。每条消息必须先转换为治理决议材料，并在领域服务入口重新验证：

```text
稳定句柄和版本。
节点类型和关系类型。
任务授权、任务承接和任务选择方法关系。
动作入口和规格材料。
状态 / 动态主体、场景和发生时间戳。
需求目标状态和实际状态材料。
```

## 8. 批次分类与抢占顺序

同一任务内的权威生命周期候选必须串行；不同任务和只读观察材料可在证明无共享写入后并行。

建议优先级：

| 顺序 | 类别 | 原因 |
| --- | --- | --- |
| 1 | 停止请求 | 生命周期最高优先级 |
| 2 | 已取出执行的任务结果复核、冲突和过期材料 | 防止已执行结果长期悬空 |
| 3 | 派生需求和观察材料准入 | 可能影响下一轮任务候选，但不直接写事实 |
| 4 | 执行前许可预判请求 | 只作候选；不得越过既有任务生命周期写入 |
| 5 | 自检缺口候选 | 不自动修复，不抢占已确认任务结果复核 |
| 6 | 缓存刷新和事件日志请求 | 非权威观察材料，允许合并或延后 |

抢占只改变材料处理顺序，不改变任务、需求、方法、状态或动态事实。

## 9. 治理动作候选与治理决议材料

治理循环只能产生以下决议候选：

```text
拒绝当前材料。
等待缺失材料。
请求领域服务重新读取。
请求任务结果复核。
请求派生需求准入。
请求观察材料准入。
请求执行前许可预判。
请求非权威缓存刷新。
请求事件日志审计。
```

治理决议材料必须记录目标领域服务、稳定句柄、版本、发生时间戳、幂等键、同任务序号和只读证据句柄。治理决议材料不得直接记录“已完成”“已满足”“允许执行”或“成功”为机器事实。

## 10. 逐条归并与领域服务落账

| 治理消息 | 领域服务复核 | 允许输出 | 禁止输出 |
| --- | --- | --- | --- |
| 任务结果复核请求 | 任务服务、状态服务、动态服务、方法服务 | 实际结果状态关系、任务完成状态候选的正式写入结果 | 回执消息直接裁决任务完成 |
| 派生需求请求 | 需求服务、状态服务、特征服务 | 经目标状态和当前材料复核后的需求节点 | 摘要或线程决定直接创建需求 |
| 观察材料准入请求 | 存在 / 场景 / 特征 / 状态 / 动态等原子服务 | 经准入后的正式结构 | 外部材料直接写世界事实 |
| 执行前许可预判请求 | 任务服务、方法服务 | 当前任务授权关系、承接状态、任务选择方法关系的只读复核材料 | 新建授权 bool / 状态节点或消息许可事实 |
| 需求结算预览请求 | 需求服务、状态服务、动态服务、因果服务 | 结算候选或正式结算入口结果 | 缓存、日志或显示文本裁决需求满足 |
| 缓存刷新请求 | 统计服务 | 非权威缓存材料 | 业务状态 |
| 事件日志请求 | 日志系统 | 审计材料 | 当前机器事实或恢复裁决 |

每条归并必须满足：

```text
领域服务入口前置拒绝先完成。
多步写入在可读前完成全部结构写入和读回。
任何内部写入或读回不符合预期都进入追根因解决。
失败后必须回滚、失效隔离，或由数量快照和读取拒绝证明半结构不可读。
```

## 11. 任务管理上行桥

候选调用路径：

```text
任务工作线程生成结果回执材料
-> 任务管理线程读取回执材料
-> 任务上行桥复核消息类型、目标任务、版本、时间戳、幂等键和任务序号
-> 构造自我治理消息
-> 治理 mailbox 入队
-> 冻结治理批次
-> 生成任务结果复核决议材料
-> 任务服务 / 方法服务 / 状态服务 / 动态服务正式入口复核
-> 需求服务结算候选或正式结算入口
```

桥接入口不得：

```text
把结果回执当任务完成。
把消息来源线程当动作来源。
把材料摘要当实际结果状态。
绕过任务服务直接写任务状态。
在持有任务队列或治理队列锁时调用领域服务。
```

## 12. 执行前许可预判边界

旧项目无条件设置 `已审批=true`、`允许执行=true` 的路径明确禁止迁移。

新项目许可链固定为：

```text
执行前许可预判请求
-> 只读检查材料是否完整
-> 任务服务读取任务授权关系和任务生命周期
-> 任务服务读取任务承接材料
-> 任务服务读取任务选择方法关系
-> 方法服务复核方法首、动作入口和规格材料
-> 返回“可进入执行入口复核”或拒绝材料
-> 真正执行入口再次复核全部关系
```

预判结果只回答“材料是否足以进入正式执行入口复核”，不回答“机器已经拥有执行权限”。不得把预判结果持久化为 bool、日志、摘要、线程状态、消息状态或独立授权状态节点。

## 13. 权威裁决边界

| 对象 | 唯一允许裁决方 | 必须读回的结构 |
| --- | --- | --- |
| 执行许可 | 任务 / 方法正式入口基于既有授权、承接和选择方法关系 | 任务授权关系、任务状态、任务方法关系、动作入口和规格 |
| 任务实际结果 | 任务服务与状态 / 动态服务 | 实际结果状态、来源动态和任务关系 |
| 任务完成 | 任务服务 | 实际结果状态、完成状态和发生时间戳 |
| 方法成功候选 | 方法 / 任务 / 状态 / 动态服务联合复核 | 方法身份、动作入口、实际结果状态和动态证据 |
| 需求满足 | 需求服务 | 目标状态、实际状态、来源任务和动态证据 |
| 需求结算 | 需求服务 | 结算状态、来源任务、目标状态、实际状态和动态证据 |

治理消息、mailbox、冻结批次、缓存、日志和显示均不在裁决方列表中。

## 14. 逻辑内返回与追根因解决

### 14.1 逻辑内返回

```text
准入前材料为空或字段缺失。
句柄无效、版本过期、幂等冲突、同任务序号倒退。
队列满、队列停止、重复消息或已过期材料。
候选为空、等待材料、只读预判拒绝或停止请求。
不产生可读半结构的人读审计、缓存刷新或显示请求拒绝。
```

### 14.2 追根因解决

```text
前置条件通过后，冻结批次只移动部分消息而出现可读半批次。
领域服务已经开始创建节点、绑定关系、写状态、记录动态或结算后结果不符合预期。
回滚、失效隔离或数量快照证明失败。
任务结果已经正式写入，但任务 / 需求读回链不完整。
停止收口后仍存在被读成已完成事实的悬挂候选。
```

追根因路径必须停止依赖链，不得只写日志、返回 false 或继续下一条消息。

## 15. 事务、失效隔离与数量快照

后续每一种多步落账路径必须在施工计划中选择并验证以下一种收口方式：

```text
事务回滚：全部写入在同一已确认事务边界内提交或回滚。
失效隔离：失败结构被标记为不可读，所有公开读取入口拒绝。
数量快照加读取拒绝：证明节点、关系、索引数量未增加，或新增候选无法从任何公开入口读取。
```

当前不能用“最终验证函数”修复半结构；数量快照只作验收证据，不替代入口前置拒绝和事务边界。

## 16. 锁、等待与停止收口

```text
队列 / mailbox 锁内只复制、移动或排序消息材料。
释放队列锁后再读取领域结构、调用领域服务、写日志或执行耗时计算。
不得在仓库锁、领域服务锁或队列锁内等待、join、flush、弹窗或反向调用回调。
停止消息最高优先级；停止后拒绝新治理请求。
已冻结但未落账的消息回到未完成候选材料或失效隔离，不得伪造成任务失败。
先停止生产，再停止任务桥，再收口治理队列，最后 join 治理线程候选。
```

## 17. 验收矩阵

| 编号 | 验收项 | 通过标准 |
| --- | --- | --- |
| SG-A1 | 拒绝入队后结构不变 | 无节点、关系、索引或业务状态变化 |
| SG-A2 | 冻结批次失败无半结构 | mailbox 保持原状或整批不可读，不出现部分冻结可读状态 |
| SG-A3 | 单条归并失败收口 | 已写结构回滚、失效隔离，或数量快照和读取拒绝证明不可读 |
| SG-A4 | 上行回执不裁决 | 回执转治理消息后仍需任务 / 需求 / 方法 / 状态 / 动态服务复核 |
| SG-A5 | 人读材料不承载事实 | 日志、摘要、控制台、显示和控制面板刷新请求不改变机器结构 |
| SG-A6 | 队列边界分离 | 运行消息队列清空不影响治理领域事实，治理队列不承载机器事实 |
| SG-A7 | 同任务顺序 | 创建、选择方法、执行结果、完成和结算不得重排 |
| SG-A8 | 许可预判降权 | 预判不新增授权 bool / 状态节点，不绕过任务授权关系 |
| SG-A9 | 锁外领域调用 | 扫描不出现持队列锁调用领域服务、日志或 join |
| SG-A10 | 停止收口 | 停止后无悬挂候选被读成任务完成、方法成功或需求满足 |
