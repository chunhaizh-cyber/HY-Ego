## 1. 目标

把当前由系统角色清单、稳定键索引、普通父子、通用归属关系和旧领域材料拼出的“自我”迁移为：

```text
唯一当前自我存在
+ 自我持有的唯一内部世界场景
+ 内部世界场景到子存在的正式成员关系
+ 关系 22 承载的自我实例特征槽
+ 关系 19 主体角色确定的状态读取投影
+ 关系 20 主体角色确定的动态读取投影
```

子存在、特征、状态和动态的“列表”均为调用期不可变读取投影，不持久化、不反向写入，也不成为第二份权威事实。

`自我线程::初始化` 仍直接调用 `世界树初始化器::初始化自我世界树`，该路径会再创建世界树根、自我所在场景和自我存在，写普通父子和归属，并把自我相对坐标写入主信息 I64 槽。它没有消费生产运行期系统角色清单，也没有共享唯一自我结构裁决。

后继代码计划必须使所有生产和仍可到达的自我初始化语境汇合到一个正式入口。仅迁移生产启动而保留另一条可写自我路径，不构成 P2C 完成。

### 3.4 不能从现状推出的事实

当前代码存在一个名为“自我存在”的稳定键，不等于结构上已经证明同一运行期自我域只有一个当前自我。当前“自我场景”也不能仅凭名称改解释为自我内部世界，因为当前自我正是该场景成员，而 7130 明确禁止自我成为自身内部世界成员。

## 4. 目标结构

### 4.1 自我身份

自我继续使用节点类型为 `存在` 的普通存在节点，不新增 `自我` 节点类型、自我专用平行本体或自我领域持久记录。当前自我候选必须同时满足：

```text
存在节点直接稳定主键非零
完整当前节点句柄、版本和记录状态有效
作为关系 21 角色 1 的源，且全运行期自我域恰有一个当前源
角色 1 恰好指向一个当前有效内部世界场景
不作为该内部世界关系 21 角色 2 的目标
```

关系 21 角色 1 是识别当前自我结构候选的必要结构证据。系统角色稳定键只负责从配置定位预期身份；索引命中和清单缓存均须回读关系 21 才能形成完整自我结论。

### 4.2 自我所在场景与内部世界场景

完整自我结构一次冻结五项互异身份：

| 角色 | 节点类型 | 稳定主键命名域 |
| --- | --- | --- |
| 世界根 | 基础信息 | 基础信息身份 |
| 世界根内部世界场景 | 场景 | 场景身份 |
| 自我所在场景 | 场景 | 场景身份 |
| 自我存在 | 存在 | 存在身份 |
| 自我内部世界场景 | 场景 | 场景身份 |

必须区分自我所在场景与自我内部世界场景：自我通过归属关系进入前者，绝不能成为后者的成员。固定拓扑不再由 S0 或执行窗口选择：

```text
世界根 --普通父子/0--> 世界根内部世界场景
世界根内部世界场景 --普通父子/0--> 自我所在场景
自我所在场景 --归属/0--> 自我存在
世界根内部世界场景 --普通父子/0--> 自我内部世界场景
自我存在 --关系21/1--> 自我内部世界场景
```

上述五条是固定世界 / 自我关系；关系 21 角色 2 的成员关系是额外零到多集合，不计入五条。任一身份或固定关系缺失、复用、错命名域、错端点、错版本或非当前时，完整结构整体不成立。

### 4.3 关系 21

| 顺序号 | 角色 | 源 | 目标 | 当前基数与唯一键 |
| --- | --- | --- | --- | --- |
| 1 | 内部世界 | 当前自我存在 | 内部世界场景 | 每个当前自我恰一；唯一键为类型 + 完整源 + 顺序号 |
| 2 | 场景成员 | 该自我的内部世界场景 | 子存在 | 每场景零到多，同一成员至多一条；唯一键追加完整目标 |

存在领域只授权角色 1，场景领域只授权角色 2。统一 `存在场景数据操作` 可以在同一结构会话中接受由两个领域服务形成的不可变规格，但不得自行解释自我业务语义。

角色 2 写前必须证明：目标存在当前有效、目标不等于自我、源场景确为该自我的角色 1 目标、同一成员没有重复当前关系、不会通过成员/内部世界/父场景链形成自包含环。

### 4.4 自我实例槽

不新增自我专用槽位关系。自我作为合法 `存在` 宿主，统一使用关系 22：

| 顺序号 | 角色 | 方向 | 基数 |
| --- | --- | --- | --- |
| 1 | 宿主槽 | 自我存在 -> 实例槽 | 零到多；每槽反向恰一宿主 |
| 2 | 定义模板 | 实例槽 -> 抽象特征定义 | 恰一 |
| 3 | 当前值 | 实例槽 -> 特征值 | 恰一当前关系 |

P2C 不写特征结构。新域自我初始化需要安全、服务等槽位时，只调用节点直接特征领域公开入口；完整自我聚合读取只消费 P2A 固定的 `节点直接特征业务服务::读取宿主实例特征槽位组`。

### 4.5 状态与动态投影

自我状态投影只由关系 19 角色 1 的目标为当前自我的完整状态事实形成。P2C 消费 P2B 固定接口：

```text
节点直接状态业务服务::读取主体状态事实组(节点句柄 主体)
```

自我动态投影只由关系 20 角色 1 的目标为当前自我的完整动态事实形成。P2C 消费：

```text
节点直接动态业务服务::读取主体动态事实组(节点句柄 主体)
```

两个接口均只用主体角色反向索引缩小候选，再逐项完整读回关系角色、节点和领域类型化记录；返回调用期不可变结果组。合法无事实返回空组，任一候选出现结构异常则整体返回内部不一致，不返回“部分成功列表”。

## 5. 输入契约与调用语境

### 5.1 全新候选上下文初始化

`完整自我初始化请求` 规则版本固定为 2，字段顺序固定为：

```text
自我结构规则版本
世界根稳定主键
世界根内部世界场景稳定主键
自我所在场景稳定主键
自我存在稳定主键
自我内部世界场景稳定主键
出生期内部世界成员完整句柄组
```

全部稳定主键必须非零、两两不同并处于第 4.2 节固定命名域。出生期成员为空是合法输入；非空成员只接收已经由存在服务形成的当前稳定存在身份，按完整节点句柄去重。初始化器不得用文本、语素、显示名或索引键临时创建不明成员。

五项身份和五条固定关系必须在同一事务中形成或同义读回，避免发布局部世界拓扑或“有自我而无内部世界”的半结构。已经存在的同键节点必须逐项证明同义；任一同键异义或另一当前自我在第一笔写入前拒绝。

### 5.2 后续增加成员

建议请求：

```text
当前自我完整句柄
预期内部世界完整句柄
候选成员存在完整句柄
可选幂等请求身份
```

场景服务先验证目标类型和成员资格，存在服务只读确认当前自我/内部世界对应关系，组合器调用数据操作提交角色 2。成员已存在且语义相同返回幂等读回；自我自身、错场景、旧版本、重复异义和环风险写前拒绝。

### 5.3 聚合读取

调用方只能提交自我稳定身份或完整当前句柄，不提交列表、不提交仓库/索引/许可，也不能指定“以缓存为准”。读取顺序固定为：

```text
唯一自我及角色 1
-> 角色 2 成员组
-> P2A 宿主实例槽位组
-> P2B 主体状态事实组
-> P2B 主体动态事实组
-> 组合不可变自我读取投影
```

四个结果组可以为空，但身份和内部世界不可为空。调用中任一结构版本漂移时整次结果无效，由调用方重新读取；不得拼接不同版本批次的局部列表。

### 5.4 旧自我线程语境

`自我线程` 只能从已发布运行期上下文租约取得当前自我聚合入口，不得再次调用旧世界树初始化器创建第二套世界根、自我场景或自我存在。旧 `初始化.世界树.ixx` 若仍被测试使用，必须转为只读适配/退役证据，不得继续拥有生产写权。

## 6. 写入过程

### 6.1 创建完整自我结构

```text
系统角色初始化器接收固定配置
-> 存在/场景服务分别形成类型化不可变规格
-> 事务外读取稳定主键、节点和关系 21 当前结构
-> 同键同义完整结构：幂等读回
-> 同键异义或已有另一自我：零写冲突
-> 未找到：取得覆盖节点、关系、索引和初始化回执的唯一写入权
-> 锁内重读唯一性、场景身份和父场景
-> 创建或同义复用世界根、世界根内部世界场景、自我所在场景、自我存在与自我内部世界场景候选
-> 建立三条普通父子和自我所在场景到自我存在的归属关系
-> 建立关系 21 角色 1
-> 按请求建立角色 2，禁止自我成员和自包含环
-> 同会话按固定字段顺序读回五项身份、五条固定关系、全部出生期成员、主键、端点、版本和基数
-> 确认待发布
-> 无失败最后发布
-> 事务外通过领域服务读回完整自我结构
```

系统角色清单只在最后读回成功后形成。清单版本必须推进；旧版本没有内部世界字段时不能被新完整性入口接纳，也不得通过默认“自我场景即内部世界”兼容。

### 6.2 增加成员

成员关系是独立后继事务：先读回当前自我、角色 1 内部世界和候选成员，写前检查非自我/无环/无重复，再创建一条角色 2。确认前失败精确撤销本次关系；已发布成员移除必须走正式关系失效，不复用候选撤销。

## 7. 组合读取投影

建议值式结构：

| 结构 | 内容 | 持久化 |
| --- | --- | --- |
| `完整自我结构值式材料` | 五项节点直接身份和五条固定关系证据 | 否 |
| `内部世界成员组值式材料` | 内部世界完整身份、角色 2 关系证据和成员完整身份组 | 否 |
| `自我实例特征组值式材料` | P2A 返回的完整槽位组 | 否 |
| `自我状态事实组值式材料` | P2B 返回的完整状态事实组 | 否 |
| `自我动态事实组值式材料` | P2B 返回的完整动态事实组 | 否 |
| `自我聚合读取值式材料` | 上述五项的双遍逐字段一致组合 | 否 |

投影可以在调用期间排序、筛选和裁剪显示结果，但任何投影字段丢失、缓存清空或显示裁剪都不能改变节点、关系或领域记录。不得把结果组写回 `系统角色清单`、自我节点、场景节点或新建“自我记录表”。

普通聚合不能取得 #349 的统一冻结许可，也不能用某一仓版本、节点最大版本、关系数量或本地计数伪造顶层结构版本。组合器按第 5.3 节顺序连续采集两遍；两遍值式材料逐字段相等才返回成功。任一不同返回 `合同结果状态::读取版本漂移(8)` 与 `合同结果原因::读取期间版本变化(13)`，材料为空。

成员按成员完整节点句柄 `(仓库编号、节点编号、版本号)`，再按角色 2 完整关系句柄排序；槽位、状态和动态保持提供者合同的规范化顺序。重复稳定键必须整体拒绝，不能依赖排序消歧。

## 8. 非成功返回与失败收口

### 8.1 逻辑内返回

| 分支 | 时点 | 结构变化 |
| --- | --- | --- |
| 零主键、重复主键、无效句柄、错类型、旧版本 | 第一笔写入前 | 零变化 |
| 请求把自我加入自身内部世界或明确形成环 | 第一笔写入前 | 零变化，入口拒绝 |
| 同键同义且唯一自我结构完整 | 权威预读后 | 零变化，幂等读回 |
| 已有另一完整当前自我、同键异义或场景语义冲突 | 第一笔写入前 | 零变化，唯一性/幂等冲突 |
| 许可竞争或写前版本漂移 | 第一笔候选前 | 零变化，具名返回 |
| 合法没有内部成员、槽位、状态或动态 | 读取阶段 | 返回空结果组，不是错误 |
| 读取期间版本变化 | 组合结果发布前 | 丢弃本次投影并返回版本漂移 |

### 8.2 内部逻辑错误

以下情况不能降级为普通空列表、缺口、等待或幂等成功：

- 权威结构中已有零个或多个竞争当前自我，而调用语境要求读取已发布完整自我；
- 一个自我有零个或多个当前角色 1；
- 自我是角色 2 目标、成员关系成环、同一成员重复；
- 角色 21/22/19/20 的端点、顺序号、基数或版本不一致；
- 候选创建、完整读回、参与者确认、逆序撤销或最后发布不闭合；
- 发布后正式服务读回与已确认结构不一致；
- P2A/P2B 返回部分列表或把结构异常静默过滤。

统一收口：

```text
停止新增写入
-> 统一可见点前按成员关系、内部世界关系、场景/存在、索引和回执材料逆序撤销
-> 读回证明事务前态
-> 无法证明则在释放唯一写入权前隔离事务域
-> 已发布异常停止依赖路径并保留证据，不重开候选撤销
```

## 9. 生命周期、并发与恢复边界

1. 当前自我身份的并发隔离键是自我域稳定身份及关系 21 角色 1 唯一键，不是初始化线程编号或锁地址。
2. 两个线程同时初始化同一自我时，只能有一个取得写入权；另一条路径锁内重读后同义幂等读回。不同语义占用返回冲突。
3. 内部世界成员可以在不同目标身份上并发准备，但同一场景/成员唯一键只能有一个有效关系。
4. 内部世界关系和成员关系失效必须按关系版本形成历史，不原地改目标，不删除被引用事实。
5. 自我聚合投影不进入快照。P4 保存节点、正式关系 0—23、领域记录、失效关系和历史事实，并在隔离候选中用固定关系 0 / 1 / 21、成员关系 21 / 2、P2A 关系 22 与 P2B 关系 19 / 20 材料重建自我投影。
6. 本设计不授权生产恢复、旧快照猜测、自我上下文在线热替换或恢复失败回退全新启动。

## 10. 分阶段迁移策略

1. NT-P1 先发布隔离新域的节点直接稳定主键、正式关系 0—23 ABI 和关系事务能力；P2C 写固定关系 0 / 1 / 21 与成员关系 21 / 2，只读消费 P2A 关系 22 和 P2B 关系 19 / 20。
2. P2A 在同一隔离新域发布关系 22 与 `读取宿主实例特征槽位组`。
3. P2B 在同一隔离新域发布关系 19/20 与两个主体事实组读取接口。
4. P2C 只在节点直接新域的候选初始化器中新增独立内部世界稳定键和关系 21；不把旧 `自我场景` 静默当内部世界。
5. P2C 新建节点直接系统角色回执，不修改现行系统角色清单或默认运行期上下文；旧清单不默认补字段。
6. P3 在隔离新域形成全部新生产候选调用图，并列出旧世界树、自我线程和现行系统角色调用点的切换清单，但不切默认装配。
7. P4 在完整恢复与总验收后一次切换所有可到达自我初始化路径，随后处理快照、恢复和旧域物理退役。

阶段迁移可以保留旧调用签名的显式拒绝或只读适配，但同一运行期不能同时存在两套当前自我权威，也不得同时写关系 21 和旧普通父子自我列表。

### 16.4 普通读取一致性

唯一自我读取使用 `NT-C1Q/v0.1::读取有效类型角色关系组(关系21, 1)` 形成候选，再逐项读回五身份和五条固定关系。成员和退役影响面分别使用按源、按目标和相关节点读取。名称、索引、缓存和旧普通父子路径不能单独裁决唯一自我。

聚合读取不取得 #349 统一冻结许可。组合器连续采集两遍完整自我、成员、P2A 槽位、P2B 状态和 P2B 动态，逐字段相等才返回；不同返回 `读取版本漂移(8)/读取期间版本变化(13)` 且材料为空。成员按完整节点句柄后接完整角色 2 关系句柄排序；其它组保持提供者规范化顺序。

### 16.5 自我结构退役参与包

退役包可移动不可复制、没有 public 可写字段、移动后源包失效，并且只允许执行一次。形成入口只接受 `非当前历史自我`，必须在第一写前证明目标不是当前唯一自我、目标版本精确、当前唯一自我仍完整。当前自我、永久世界根和世界根内部世界场景普通退役固定拒绝。

P2C 没有私有领域记录仓，不构造空的核心事务参与者。包在执行器回调中先登记冻结的关系 21 失效候选，再调用 #346 提供的回调登记其它当前关系失效、索引移除、目标自我节点删除、完整读回和请求提交，最后使用执行器无额外参与者重载完成统一确认 / 撤销 / 发布。内部世界场景及其它四项固定身份都不是删除目标。

第二次执行、移动后执行、空回调或无效执行器返回核心入口拒绝和合同 `入口拒绝/无效身份`，不调用回调且零变化。提交前失败由核心执行器逆序撤销。真实提交后验证自我节点版本加一且已删除、全部冻结关系版本加一且已失效、索引已移除；发布后读回失败保留核心 `已提交`，另报 `内部错误/权威读回不一致`，不得重新撤销。

### 16.6 恢复投影验证

值式验证请求的自我结构候选组必须恰有一项。它同时消费五项节点记录、五条固定关系 0 / 1 / 21、成员关系 21 / 2、P2A 关系 22 材料以及 P2B 关系 19 / 20 材料；“只消费关系 19—22”不是完整合同。

未知规则版本返回 `入口拒绝`；候选零 / 多自我、固定身份或关系缺失 / 重复 / 坏端点、错命名域、自我成为角色 2 目标、成员重复 / 成环、槽位 / 状态 / 动态主体错配或任一部分材料返回 `材料不一致`。合法空成员 / 槽位 / 状态 / 动态仍 `已通过`。资源分配失败返回 `资源失败`；验证自身出现不可能分支返回 `内部不一致`。所有非通过状态材料为空。

该入口不接收 #350 私有候选仓或公开可写服务，不新建持久列表或第二本体，不把验证结果升级为机器事实。P4 仍须验证九段、全部历史 / 失效关系和索引后才能发布。
