# 显示分类与当前概念根按需只读投影详细设计

日期：2026-07-16

决策：JY-358

状态：正式详细设计 / #261 可重新登记 / 830 未实施 / 未改 C++ / 未构建

## 1. 设计定位

本设计承接 `流程图/20260716_显示分类与当前概念根按需只读投影流程图_v0.1.md` 和当前接口 S0，把控制面板生产刷新从“每次生成六类完整结构”收窄为“每次只生成当前分类，概念分类只生成当前活动概念根”。

六项继续只是显示分类。权威身份、节点和关系仍由统一仓库与领域服务承载；概念结构继续允许四个活动根、多父共享节点和图关系。显示 DTO、窗口快照、根页游标和耗时记录都不是机器事实。

#258 已正式不实施、不发布。本设计只消费当前正式全表查询之上的领域服务值式入口，不要求关系二级索引，不读取失败 WIP，不修改关系仓库或概念图服务。

## 2. 当前事实与改造原则

| 当前事实 | 本轮处理 |
| --- | --- |
| `生成六类真实树只读材料` 固定构造六类并整体发布 | 保留公开原义作兼容回归；生产窗口不再调用 |
| 概念类读取四根并逐根生成视图 | 新入口读取四根选项，但只对当前根调用一次正式单根视图 |
| 任务根在需求完整构造时发现 | 抽取只读根身份发现辅助，不构造需求树 |
| 方法根在任务完整构造时发现 | 抽取只读根身份发现辅助，不构造任务树 |
| 窗口快照固定保存六项数组 | 改为当前分类页、当前概念根、根选项和页游标 |
| TreeView 一次重建六个分类根 | 六项导航继续存在；TreeView 只重建当前分类根和当前页真实根 |
| 非概念分类没有统一活动版本 | 每页独立读取并整体替换；不得宣称跨页同版本快照 |

新内部辅助必须同时被旧六类入口和新按需入口复用，避免两套业务解释。不得为了按需读取复制需求、任务、方法或概念关系语义。

## 3. 输入入口与值式合同

### 3.1 按需请求

在 `控制面板服务.h` 中新增值式请求，字段语义冻结为：

```text
控制面板按需树请求
- 基础请求：复用现有 控制面板请求，只允许服务只读来源。
- 当前分类：只允许世界、概念、语素、需求、任务、方法。
- 当前概念根：仅概念分类使用；未指定时按根类别稳定顺序选择第一根。
- 节点预算：当前页全部真实根合计的节点上限，必须大于 0。
- 深度预算：单棵递归投影的最大深度，必须大于 0。
- 根页大小：当前页最多包含的真实根数量，必须在 1..256。
- 继续游标：可空；非空时必须与分类、当前概念根、根集签名和下一根偏移一致。
```

建议公开入口名固定为：

```cpp
控制面板按需树候选材料 生成当前分类真实树只读材料(
    const 控制面板按需树请求& 请求) const;
```

入口不得接收关系类型白名单。每类可读关系角色继续由控制面板服务现有领域语义固定，调用方不能借请求扩大关系读取范围。

### 3.2 概念根选项

按需结果中的概念根选项是值式材料，每项至少包含根类别、完整根句柄和人读显示名称。选项必须来自 `概念图服务::读取全部概念根`，按根类别稳定排序，数量必须恰好为 4，且都绑定同一非零活动版本。

窗口不直接持有概念图服务引用。概念分类首次进入时若没有当前根，服务确定选择第一根并在结果中返回；后续选择必须复核仍是当前活动根。

### 3.3 根页游标

游标是值式显示协议，不是仓库索引或权威版本。字段至少包含：

```text
分类
可选当前概念根
概念活动版本（非概念类为 0）
根集签名
下一根偏移
```

根集签名由当前分类、确定排序后的完整根句柄组和可选概念活动版本计算；只用于拒绝错误分类、错误根、根身份组变化和越界偏移。不得只用本地编号，不得把签名解释成关系图版本。

根页只在真实根组边界切分。单棵根下递归结构超过节点或深度预算时整页逻辑拒绝，不返回截断树冒充完整结果。

### 3.4 按需结果

```text
控制面板按需树候选材料
- 状态
- 当前分类
- 当前概念根
- 概念活动版本
- 概念根选项组
- 当前页树结构材料
- 可选下一页游标
- 候选序号
- 完整
- 追根因错误
```

成功结果只包含一个分类。`当前页树结构材料.分类` 必须等于请求分类，根组不能为空，稳定句柄、只读标志、来源版本和节点计数必须完整。逻辑拒绝结果不得携带可展示的部分树；追根因结果必须显式标记，窗口不得保留为成功新页。

## 4. 六类按需构造

### 4.1 世界

从构造时保存的世界值式 DTO 读取确定根身份组，先选择根页，再只复制和复核当前页根。不得访问其它分类。来源版本沿用世界 DTO 的正式值。

### 4.2 概念

1. 读取活动版本和四根登记。
2. 未指定当前根时选择稳定排序第一根；已指定时复核句柄和根类别属于本次四根登记。
3. 只调用一次 `读取抽象树视图并返回结果(当前根, 节点预算, 深度预算)`。
4. `已生成` 转为成功页；零预算、非活动根和预算耗尽属于逻辑内返回；活动图内部错误属于追根因。
5. 转换后只允许一个真实根，活动版本、根类别和根句柄必须逐项一致。

不得先生成四根再裁切当前根。

### 4.3 语素

从构造时保存的语素值式 DTO 读取根身份组，先选页再构造。语素对应信息和概念追溯仍沿用现有控制面板服务解释，不增加新关系角色。

### 4.4 需求

根身份只来自构造时保存的两个根需求。排序、去重和有效性复核通过后先选页，再对当前页逐根调用现有需求根构造辅助。根组重复、缺失或失效按现有合同逻辑拒绝，不用其它需求补位。

### 4.5 任务

新增私有只读辅助从两个根需求分别调用现有 `任务服务::读取需求承接任务组`，合并后按完整句柄排序去重。该辅助只发现任务根身份，不生成需求节点材料，不读取需求详情，不发布需求树。

选定根页后才逐根生成任务材料。任务材料中发现的选择方法只服务当前任务节点详情或兼容辅助，不触发方法分类构造。

### 4.6 方法

先复用任务根身份发现辅助，再对任务根调用现有任务选择方法只读入口形成方法根身份与来源任务对。按完整方法句柄排序去重后选择根页，再生成当前页方法递归投影。

同一方法被多个任务选择时只作为一个方法真实根，但来源任务关系仍按现有方法材料展示；多父、前置、后续和路径环继续使用现有递归与重复投影合同。不得构造任务树作为方法根发现的副产品。

## 5. 排序、分页与快照语义

1. 根身份排序使用完整稳定句柄的现有确定顺序；概念根先按根类别，再按完整句柄兜底。
2. 当前页范围为 `[偏移, min(偏移 + 根页大小, 根数量))`。空页、越界偏移和不匹配游标逻辑拒绝。
3. 有后续根时返回下一页游标；最后一页不返回游标。
4. 窗口翻页时整体替换当前页，不追加旧页节点。返回首页时重新读取当前材料。
5. 概念页以活动版本检测版本漂移。世界和语素沿用正式 DTO 来源版本；需求、任务、方法没有统一活动版本，只保证单次结果内部完整。
6. 根集签名只能证明根身份组未变化，不能证明非概念分类的递归关系在多次请求间未变化。因此不得把多页合并为同一时间点的权威图。

## 6. 窗口消费

### 6.1 快照

`控制面板窗口快照` 从固定六项树结构数组收窄为：

```text
六项树视图分类材料（继续服务导航说明）
当前分类按需候选
当前分类
当前概念根
概念根选项组
当前页游标和返回首页所需请求状态
总览、操作请求、结构统计和数据库审计材料
```

旧六树候选序号不再用于生产窗口；按需候选序号只标识会话期显示候选。

### 6.2 导航与控件

六项分类导航继续保留。用户选择新分类时立即请求该分类首页，成功后才改变当前 TreeView；逻辑拒绝保留上一份有效页并显示人读状态，追根因错误停止依赖路径。

概念分类显示下拉选项，选项只来自按需结果。切换概念根后请求该根首页。非概念分类隐藏概念根下拉。根组存在后续页时显示上一页 / 下一页命令；命令只改变会话期页状态。

TreeView 每次只插入一个当前分类 UI 根和当前页真实根。分类 UI 根不是领域节点，不计入真实节点数。节点详情继续显示完整句柄、类型、关系角色、显示名称、来源版本和重复投影；显示名称不参与身份裁决。

### 6.3 刷新与失败恢复

刷新只重新请求当前分类、当前概念根和当前页入口。游标失效时逻辑内返回并从当前分类首页重新读取，不自动生成六类完整候选。

新页构造成功后才整体替换旧页和重建 TreeView。控件插入或引用读回失败属于追根因；先恢复上一份有效页，恢复失败则关闭窗口，不显示空树伪成功。

数据库审计页仍独立刷新，不参与当前树事实裁决。#261 不改变 SQL、日志或操作请求壳合同。

## 7. 旧六类兼容入口

`生成六类真实树只读材料` 的签名、六类顺序、整体候选和失败原义保持。其内部可复用新抽取的根发现与单类构造辅助，但不得反向调用按需窗口状态。

新自检必须在同一静态夹具和无并发变化条件下逐类比较：按需首页完整覆盖该分类全部根时，其树结构与旧六类候选对应分类逐节点、句柄、角色、来源版本、重复投影和计数一致。

生产窗口、刷新、分类切换、概念根切换和翻页均禁止调用旧六类入口。按需失败不得回退到旧六类入口。

## 8. 失败收口

### 8.1 逻辑内返回

```text
基础只读请求拒绝
状态 / 动态详情等非树分类
零预算、过大根页或越界页
非活动概念根、活动图未就绪
允许的真实根为空或游标失效
单根预算耗尽
```

逻辑内返回不发布部分新页，不改变权威结构。窗口可以保留上一份有效页，但必须显示本次未刷新，不得把旧页标成新结果。

### 8.2 追根因解决

```text
前置通过后领域读回互相矛盾
概念单根结果与活动版本、类别或根句柄不一致
根身份有效但节点材料无法构造
树分类、计数、稳定句柄、关系角色或重复投影复核失败
TreeView 插入、引用绑定或旧页恢复失败
只读路径导致权威结构数量或句柄组变化
```

命中后停止当前依赖路径，不用缓存、SQL、日志、旧六类入口或全表扫描兜底。

## 9. 并发与性能口径

按需结果是一次调用内的值式只读材料，不持有仓库引用或锁。窗口不跨线程传递游标、树节点引用或领域对象。现有 UI 线程模型保持。

结构性能验收优先于抖动耗时：

```text
当前分类成功时，即使其它五类材料不可生成，也必须成功。
概念当前根输出只能包含该根，不能生成其它三根后裁切。
TreeView 项数量只对应当前分类当前页，不包含其它分类结构节点。
翻页只替换当前页，不累计所有页。
```

专项可记录单分类和旧六类入口耗时，但不得设置依赖微秒抖动的硬门槛，也不得仅凭耗时宣称性能提升。若结构证据不满足，即使耗时下降也不得登记 830。

## 13. 完成声明边界

实施并验证后最多声明：控制面板生产窗口已按当前显示分类生成只读页，概念分类可按当前活动根切换，旧六类入口保留兼容，显示路径不改变权威结构。

不得声明：关系仓库二级索引或性能提升已经发布、六分类是六棵物理树、跨页形成同版本权威快照、完整业务操作控制面板完成、旧控制面板 / 数据库 / 外设能力迁移完成，或项目迁移全部完成。
