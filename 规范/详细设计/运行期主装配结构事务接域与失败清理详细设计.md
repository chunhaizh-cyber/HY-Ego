# 运行期主装配结构事务接域与失败清理详细设计

## 1. 目标与边界

建立唯一生产运行期上下文，使四仓库、领域服务、快照、线程、SQL 审计和只读控制面板消费同一事务域与租约。创建期补偿、已发布删除和内部不一致清理分别治理。本设计不实现崩溃恢复、跨进程 ACID、安全删除或旧能力迁移。

## 2. 当前事实

- 入口生产四仓库使用默认空接线，服务保存这些未接域仓库引用。
- #217 只在自检模块构造一个接域隔离仓库组。
- 生产清理分布在世界树初始化、语素初始化、语素服务和概念图服务。
- 当前快照服务只有拒绝矩阵；没有生产上下文、宿主、租约或恢复发布。

## 3. 唯一运行期上下文

生产真模块 `启动.运行期上下文` 唯一拥有：

```text
结构事务协调器
同一结构事务接线
主信息、节点、关系、索引仓库
领域服务装配
一次发布状态
只读 / 调用期租约源
```

最终对象边界分为同一模块内两个角色：`运行期上下文` 拥有协调器、接线、四仓库和领域服务装配；`运行期上下文宿主` 拥有发布互斥 / 一次状态、已发布上下文和租约源。宿主不反向成为上下文成员，避免自持有。

最终成员声明顺序固定为：

```text
结构事务协调器
结构事务接线
主信息仓库
节点仓库
关系仓库
索引仓库
领域服务装配
```

该顺序满足节点仓库依赖主信息仓库、关系 / 索引依赖节点仓库；析构逆序保证领域服务先于索引 / 关系 / 节点 / 主信息仓库，仓库先于接线和协调器。入口不得分别构造第二组生产仓库或领域服务。

### 3.1 分阶段装配合同

#247 只建立协调器、接线、四仓库形成的结构核心，以及宿主 / 租约骨架。结构核心缺少领域服务装配时 `完整()` 必须为假，生产发布入口必须写前拒绝；660 只验证结构身份、生命周期骨架和“不完整不可发布”，不得把该阶段称为完整生产上下文首发。

#248 新建 `装配.领域服务` 后，允许回改 `启动.运行期上下文.ixx` 和对应 #247 自检模块。最终依赖方向固定为：

```text
启动.运行期上下文 import 装配.领域服务
装配.领域服务 接收结构事务接线与四仓库引用
装配.领域服务 不 import 启动.运行期上下文
```

#248 把领域服务装配加入上下文最后一个成员，并在全部服务同域、初始化与必要读回完整后才开放全新 / 恢复共用的一次发布。670 首次验证完整上下文成功发布、重复发布拒绝和逆序析构；#249 再做端到端所有权验收。不得预造未来服务类型、使用 `void*` / 无类型槽、在 #247 发布半上下文，或让 #248 独立持有服务生命周期。

## 4. 启动分支

```text
全新启动 -> #247 结构核心 -> #248 领域服务装配 -> 完整读回 -> 一次发布
恢复启动 -> 严格隔离候选 -> #229 屏障许可 -> 适配 #247/#248 最终同类型上下文 -> 完整读回 -> 一次发布
```

两个分支互斥，共用同一发布槽和租约类型。发布前失败不暴露候选；发布后不得覆盖实例。

## 5. 许可与调用链

- 顶层生产入口取得一次共享或独占许可并下传值式令牌。
- 传统服务只保存值式接线，不保存移动许可。
- 下层仓库令牌入口只验证，不重新取得许可。
- 创建期补偿沿同一调用期令牌逆序执行。
- 已发布删除除结构独占许可外，还必须持有不可伪造的领域提交授权；结构令牌不等于业务授权。
- 任何逐仓库提交能力必须保持私有，只能由正式顶层编排调用。

## 6. 清理分类

| 分类 | 所有者 | 事务要求 |
| --- | --- | --- |
| 测试隔离材料 | 自检模块 | 不进入生产上下文 |
| 写前拒绝前无结构 | 原入口 | 不创建补偿 |
| 创建期可定位补偿 | 原创建服务 | 同一许可、精确本轮集合、逆序补偿 |
| 已发布权威结构删除 | 领域所有者 / 专项编排 | 独占结构许可 + 领域提交授权 |
| 候选 / 非权威材料丢弃 | 候选所有者 | 不写权威结构 |
| 内部不一致后清理 | 原写入服务 | 追根因、诊断、精确残留，不作普通失败 |

## 7. 快照与恢复归属

仓库快照只从 #248 后完整发布的上下文租约取得共享许可，许可内快速复制值式材料，许可外编码和文件发布。#229 只拥有恢复发布屏障；#237 使用 #247/#248 最终同一上下文类型的恢复工厂；#238/#239 只消费同一租约合同。禁止任何恢复计划创建第二协调器、第二宿主或平行租约源。

## 8. 锁序

```text
结构事务许可
-> 活动图锁
-> 图写锁
-> 单个仓库锁
```

持有仓库锁时不得请求活动图锁或另一仓库锁。关系 / 索引锁内复制、锁外同令牌读取节点。补偿和安全删除编排不得通过全局许可放宽单仓库锁规则。

## 9. 分段实施

1. #247：建立唯一上下文结构核心、四仓库主装配接域、宿主 / 租约骨架和 660 自检；缺服务时生产发布必须拒绝。
2. #248：建立领域服务装配，回改唯一上下文加入最终服务成员并开放完整首发；完成逐调用点清理改接、创建期补偿与已发布删除分流和 670 自检。
3. #249：验证主装配接域、快照许可来源、恢复单一所有权、入口无平行构造和 680 自检。
4. #226-#230、#237-#239：消费 #249 正式合同后再修订 / 执行。

## 10. 验收

- 生产只有一个上下文类型、一个发布槽、一个协调状态和一个租约源。
- 四仓库同域、同纪元、同运行期状态；服务不保存移动许可。
- #247 缺服务结构核心不能生产发布；#248 后完整上下文才可首次发布。
- 最终上下文拥有服务装配，服务装配模块不拥有仓库、宿主或租约，模块依赖无环。
- #246 全部清理点均有分类和所有者，不以扫描修复。
- 全新 / 恢复分支互斥且共用发布合同。
- 快照、线程、SQL 审计、控制面板不创建仓库或协调器。
- 660 / 670 / 680 唯一登记，入口只作最小调用和汇总。

## 11. 禁止声明

详细设计和计划完成不等于代码已接域、恢复已实现、安全删除已完成、完整 ACID 或旧能力迁移完成。
