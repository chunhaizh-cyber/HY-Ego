## 4. 结构承载

| 业务概念 | 项目承载结构 | 值类型 / 身份结构 | 写入方 | 读取方 | 验证方式 |
| --- | --- | --- | --- | --- | --- |
| 生命周期迁移请求材料 | 任务承接壳 + 目标生命周期状态材料 | 任务节点句柄、状态值材料、当前生命周期状态节点候选 | 任务服务只读组合 | 后续状态机提交入口 | 任务有效、承接壳完整、目标状态值不是未定义 |
| 生命周期状态记录辅助 | 任务到实例状态节点引用关系 | 状态节点句柄、发生时间戳、I64 生命周期状态值 | 任务服务经状态服务创建实例状态后写关系 | 任务服务、显示层只读、后续状态机复核 | 任务 / 虚拟存在 / 场景类型正确，状态值有效，关系写入后可读 |
| 筹办回执请求材料 | 任务承接材料只读组合 | 任务、来源需求、运行场景、目标状态 | 任务服务只读返回 | 后续筹办设计、方法候选设计 | 承接材料完整，不写候选方法、不写派生需求 |
| 执行桥请求材料 | 任务承接壳 + 任务选择方法关系 | 任务节点句柄、方法节点句柄 | 任务服务只读返回 | 后续执行桥设计、方法服务、动作入口服务 | 承接材料完整且唯一任务选择方法关系可读 |
| 完成状态请求 | 实际结果状态 + 生命周期状态记录辅助 | 实际结果状态节点、完成时间戳、已完成状态值 | 任务服务经状态服务写完成状态 | 任务服务、需求结算候选流程 | 已有实际结果状态且时间戳有效；不单独裁决目标达成 |
| 非权威运行统计材料 | 任务引用状态节点的只读统计 | 状态节点数量、局部计数 | 任务服务只读计算 | 显示层只读、后续缓存统计设计 | 只统计可读状态节点数，不裁决任务完成、方法成功或需求满足 |

普通 `bool` 和局部计数在本设计中只能表达当前读取路径是否可用或材料数量，不得持久化为任务完成、方法成功、需求满足或世界事实。

## 5. 请求分流

```text
生命周期迁移请求材料
-> 复核任务节点、任务承接壳和目标生命周期状态材料
-> 返回可作为请求材料 / 无效任务 / 承接壳不完整
-> 后续状态迁移合法性矩阵确认后才允许生成提交入口。

生命周期状态记录辅助
-> 复核任务、虚拟存在、场景、状态值和时间戳
-> 经状态服务创建实例状态
-> 写任务到生命周期状态节点引用关系
-> 写入不及预期不得声明状态迁移成功。

筹办回执请求材料
-> 读取任务承接材料
-> 承接壳完整时返回任务、来源需求、运行场景和目标状态材料
-> 不形成候选方法集合、方法缺口、派生需求或任务状态写入。

执行桥请求材料
-> 读取任务承接材料
-> 读取任务选择方法
-> 存在唯一任务选择方法关系时返回执行桥请求材料
-> 不启动线程、不 std::async、不派发动作、不记录动作动态。

完成状态请求
-> 读取任务承接材料和实际结果状态
-> 时间戳有效后记录已完成生命周期状态
-> 目标状态与实际结果状态比较仍转后续任务结果 / 需求结算流程。

非权威运行统计材料
-> 读取任务承接材料
-> 遍历任务引用目标中的状态节点
-> 返回状态节点数
-> 不裁决任务完成、方法成功或需求满足。
```

## 6. 入口前置拒绝

后续若进入代码实施，必须在入口先拒绝：

```text
任务句柄无效。
任务节点类型不是任务。
任务承接壳缺来源需求、虚拟存在、运行场景或目标状态。
目标生命周期状态值未定义。
生命周期状态记录缺虚拟存在、场景、有效时间戳或状态值。
状态服务创建实例状态失败。
生命周期状态关系写入不及预期。
筹办请求试图直接写候选方法、任务状态或派生需求。
执行桥请求缺唯一任务选择方法关系。
执行桥请求试图启动线程、派发动作或调用外设。
完成状态请求缺实际结果状态或有效完成时间戳。
运行统计请求把统计数当作任务完成或方法成功依据。
```

入口拒绝后不得写任务状态、方法关系、需求树、动作动态或统计事实。若入口前置拒绝已经排除错误输入，内部写入结果仍不符合预期，必须按逻辑错误追根因，不能用日志、弹窗、扫描器或显示层兜底修复。

## 7. 服务边界

| 服务 | 本设计允许依赖 | 禁止越界 |
| --- | --- | --- |
| 任务服务 | 读取任务承接材料、读取 / 写入任务生命周期状态辅助、读取任务选择方法、读取请求材料 | 不实现完整状态机，不直接执行方法，不写需求树，不启动线程 |
| 状态服务 | 创建生命周期实例状态、实际结果状态和完成状态材料 | 不裁决任务目标是否达成 |
| 方法服务 | 后续读取方法结构、条件 / 结果和动作入口材料 | 本设计不召回候选方法、不排序、不执行 |
| 动态服务 / 因果服务 | 后续动作动态和因果证据入口 | 本设计不记录动作动态，不生成因果结论 |
| 显示层 / 日志 | 只读诊断和人读展示 | 不承载机器事实，不裁决完成、成功或满足 |

线程、工作队列、`std::async`、运行日志、控制台输出、运行包和治理协议均不得成为动作来源或任务事实来源。

## 中途非成功返回二分口径

本文件按 2026-07-09 硬规则修订：中途非成功返回只分为 `追根因解决` 和 `逻辑内返回`。

- `追根因解决`：前置条件已经满足，并进入创建、绑定、写关系、写状态、记录动态、结算、读回或结构承载后，结果不符合内部预期；必须停止依赖路径，定位根因，当前未证明完整回滚时登记事务隔离缺口或半结构隔离缺口。
- `逻辑内返回`：领域协议允许的拒绝、候选为空、请求材料返回或人读材料返回；必须保证结构不变化，且返回材料、日志、回执、显示或控制台输出不裁决机器事实。
