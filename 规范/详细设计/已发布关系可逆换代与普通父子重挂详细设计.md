# 已发布关系可逆换代与普通父子重挂详细设计

日期：2026-07-14

状态：正式详细设计 / JY-333 / #277 施工依据 / 代码未实施

## 1. 目标

在不暴露原始结构事务令牌、不修改协调器许可强度、不改变普通删除合同的前提下，为关系仓库和不透明结构写入会话增加三类通用能力：

```text
已发布关系可逆失效。
已发布关系可逆重挂。
普通父子首次挂载 / 幂等 / 防环重挂的会话化承载。
```

同时补充会话内关系审计读取，使后继数据操作层可以在同一独占许可内复核旧版本、当前版本、状态和端点。该能力是 `#269` 的结构前置，不承载需求、任务、方法业务语义。

## 2. 当前代码事实

### 2.1 已有会话能力

`#264、#272、#276` 已实现并验证：

```text
主信息 / 节点未发布候选创建、读回、确认和撤销。
关系创建与严格删除撤销。
索引绑定与严格删除撤销。
候选 I64 写入与读回。
节点类型、节点主信息、I64、按源 / 按目标关系记录组和节点主键组读取。
```

这些能力只覆盖本次新建写集，不覆盖已经发布关系的状态或端点变更。

### 2.2 关系仓库已有兼容入口

当前关系仓库已有：

```cpp
关系状态变更材料 失效关系(关系句柄, const 结构事务令牌&);
std::optional<关系句柄> 重挂关系并返回新句柄(
    关系句柄, 节点句柄, 节点句柄, const 结构事务令牌&);
节点挂载结果 挂载或重挂节点(
    节点句柄, 节点句柄, const 结构事务令牌&);
```

这些入口执行后直接改变已发布记录，没有调用期撤销能力，也没有被 `结构写入会话` 记录。新服务层不能直接持令牌调用它们；无令牌重入又会被同线程许可门禁拒绝。

### 2.3 后继真实需求

```text
需求父子：首次挂载或把已有普通父关系原子重挂到新父需求。
任务生命周期：先创建新状态和新当前关系，再失效旧当前关系。
任务重筹办：失效旧方法投影和旧选择发布关系。
方法生命周期：原地把同一生命周期关系重挂到活跃或失效状态。
```

若任何后继写入或读回失败，必须在独占许可释放前恢复旧关系并撤销本次新写集；当前会话无法做到。

## 3. 可逆变更能力

### 3.1 类型

在 `关系仓库.h` 增加移动型 `已发布关系变更能力`。最小内部材料：

```text
来源仓库编号。
精确写前关系记录。
精确写后关系记录。
变更种类：失效 / 重挂。
阶段：持有 / 已确认 / 已撤销 / 已移动。
```

硬约束：

```text
不可默认构造、不可复制。
允许移动，移动后源对象失效。
构造函数只对关系仓库开放。
析构不写仓库、不确认、不撤销。
不导出令牌、仓库指针、锁或可变关系记录。
```

该能力不是新的关系节点、关系记录状态、业务授权或可长期保存的恢复材料。它只存在于一次独占写入会话栈内。

### 3.2 失效

新增独占结构化入口，名称以实施时现有命名风格为准，语义固定为：

```text
输入：完整当前有效关系句柄。
写前：复核仓库、版本、状态、类型、端点和版本上限。
锁内：保存完整写前记录；状态改为已失效；版本加一；形成写后记录。
输出：结构化结果、写后完整句柄和已发布关系变更能力。
```

无效、过期或已非有效关系是逻辑内拒绝。完整当前句柄与仓库记录同版本矛盾是内部不一致。

### 3.3 重挂

重挂入口必须继续复核：

```text
当前关系有效且完整。
新源 / 新目标节点当前有效。
新端点不形成同类型重复关系。
普通父子不得绕过单父和父链防环入口。
版本未耗尽。
```

锁内保存写前记录，更新端点并推进版本，形成能力。通用重挂入口不允许用于普通父子；普通父子只能走第 3.4 节入口。

### 3.4 普通父子挂载 / 重挂

关系仓库继续唯一拥有单父和防环算法。会话入口不得先在领域层读父链后调用普通关系创建冒充原子挂载。

结构化结果分支：

| 分支 | 结构变化 | 会话写集 |
| --- | --- | --- |
| 首次挂载 | 新建一条 `普通父子`，方向为父到子 | 登记为本次关系创建写集 |
| 已在目标父 | 无 | 不登记写集，返回幂等材料 |
| 合法重挂 | 原关系源端点改为新父，版本加一 | 登记可逆变更能力 |
| 无效、自身、成环 | 无 | 逻辑内返回 |
| 多父、悬空或父链矛盾 | 无 | 追根因停止 |

普通父子关系顺序号保持当前合同，不借本批新增权重、逻辑组织、脱离父节点或删除。

## 4. 确认与撤销

### 4.1 确认

确认只在同仓库、同完整独占令牌下复核写后记录仍精确匹配，然后把能力阶段改为已确认。确认不再次修改关系记录。

### 4.2 撤销

撤销必须：

```text
验证能力仍处于持有阶段。
验证仓库编号和完整令牌一致。
验证当前记录精确等于写后记录。
在关系仓库锁内恢复写前状态、端点、顺序号和版本。
读回当前记录精确等于写前记录。
把能力阶段改为已撤销。
```

不得通过再次调用 `重挂关系`、创建反向补丁关系、删除后重建或扫描整仓猜测恢复。撤销只恢复本能力记录的单条精确关系。

### 4.3 会话顺序

`结构写入会话` 增加关系变更写集。顺序固定：

```text
写入阶段：本次新身份 / 新关系 -> 已发布关系变更 -> 同许可完整读回。
撤销阶段：已发布关系变更逆序恢复 -> 索引 -> 本次新关系 -> 节点候选 -> 主信息候选。
提交阶段：沿既有合同确认节点 / 主信息候选 -> 逐项确认关系变更能力 -> 结束会话。
```

关系变更能力关闭前不得释放许可。任一变更撤销失败都返回内部不一致并停止后继验收。

## 5. 会话公开面

新增窄入口，最终中文函数名可按现有风格小调，但语义不可扩大：

```cpp
带值结构写入结果<关系句柄> 失效已发布关系(关系句柄 当前关系);
带值结构写入结果<关系句柄> 重挂已发布关系(
    关系句柄 当前关系,
    节点句柄 新源节点,
    节点句柄 新目标节点);
节点挂载结果 挂载或重挂节点(节点句柄 子节点, 节点句柄 新父节点);
std::optional<关系记录> 读取关系审计(关系句柄 当前或刚失效关系) const;
std::vector<关系记录> 读取关系审计记录组(
    节点句柄 源节点,
    关系类型 类型) const;
```

所有入口继续受当前线程、写入阶段、未作决定和无首次失败门禁。审计入口只返回值副本，不登记写集；业务解释留给后继数据操作层。

## 6. 锁与并发

```text
外层仍只有一次结构事务独占许可。
关系仓库内部仍使用现有仓库锁，不增加业务锁或第二把关系锁。
同一域其它共享读必须等待许可释放。
同线程无令牌重入继续拒绝。
能力不能跨线程、跨会话或跨域使用。
```

可逆恢复期间短暂出现写前 / 写后切换只发生在独占许可内，不得被遵守接线的外部读取观察到。
