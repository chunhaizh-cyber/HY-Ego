# 自我线程系统初始化详细设计

日期：2026-07-10

状态：已生成 / 可作为 SELF-INIT-S1 待执行计划依据 / 未改 C++ / 未构建 / 未运行程序

## 1. 定位

本设计把第一轮系统初始化放入独立自我线程的线程入口，并把范围收窄为：

```text
基础语素信息
世界树根、现实场景和自我存在
安全根需求
服务根需求
```

自我线程完成初始化后只进入驻留等待态。本设计不实现自我治理消息、治理 mailbox、冻结批次、任务调度、方法执行、本能方法、需求结算、自我循环或自我苏醒。

## 2. 当前代码事实

```text
海中鱼巣/领域/初始化.语素.ixx 已能建立第一轮基础显示语素及其结构关系。
海中鱼巣/领域/初始化.世界树.ixx 已能建立世界根、现实场景、自我存在、父子 / 归属关系和零点坐标。
海中鱼巣/领域/需求服务.h 已能创建完整目标状态需求，但完整入口当前拒绝需求主体等于目标宿主。
海中鱼巣/领域/特征服务.h 已能创建特征定义、实例特征槽位、特征值并写入 / 读回 I64 状态值材料。
海中鱼巣/入口.cpp 当前在主线程同步调用语素和世界树初始化；没有独立自我线程模块。
现有六类线程模块均不承担本设计的系统初始化职责。
```

## 3. 文件与职责

| 文件 | 职责 |
| --- | --- |
| `海中鱼巣/线程/自我线程.ixx` | 独占自我线程生命周期；在线程入口顺序调用三个初始化服务；发布只读初始化快照；成功后驻留等待停止 |
| `海中鱼巣/领域/初始化.需求.ixx` | 编排安全值 / 服务值定义、概念语素、自我实例特征材料和两个根需求；不得直接写核心仓库 |
| `海中鱼巣/领域/需求服务.h` | 补充自我维护型完整目标状态需求入口，允许需求主体与目标宿主同为自我，但继续拒绝无效句柄、错误类型、缺场景、缺当前值等输入 |
| `海中鱼巣/入口.cpp` | 构造依赖和显式参数；启动自我线程；等待初始化完成；读取初始化快照；失败时退出而不进入控制面板或常驻态 |
| `海中鱼巣.vcxproj` / `.filters` | 登记新自我线程模块和需求初始化文件 |

## 4. 自我线程生命周期

第一轮使用以下工程状态：

```text
未启动
初始化中
运行中
正在停止
已停止
初始化失败
```

状态只描述线程和启动工程生命周期，不是需求、任务、方法、状态或动态事实。

### 4.1 启动

```text
入口完成参数前置检查
-> 创建 std::thread
-> 线程入口标记初始化中
-> 初始化基础语素信息
-> 初始化世界树
-> 初始化两个根需求
-> 一次性发布初始化快照
-> 标记运行中
-> 驻留等待停止
```

自我线程不得把自身启动、Tick、线程状态或初始化成功解释为动作动态。

### 4.2 等待与停止

初始化开始前已收到停止请求时，线程逻辑内返回且结构不变化。第一笔结构写入开始后，停止请求只锁存，不中断三个初始化阶段；初始化完成后立即停止。这样不会向入口发布只有语素或只有世界树的半初始化快照。

等待和 join 不得在持有仓库锁、领域服务锁或初始化快照锁时执行。

## 5. 初始化参数与结果

### 5.1 显式参数

```text
安全值语素稳定键
服务值语素稳定键
安全当前值
安全目标值
服务当前值
服务目标值
```

第一轮默认裁决：

```text
安全值语素稳定键 = 0x0001'0001'0000'0001
服务值语素稳定键 = 0x0001'0001'0000'0002
安全当前值 = 1
安全目标值 = 10
服务当前值 = 1
服务目标值 = 10
```

这些数值只作为第一轮显式启动材料，不定义长期生命值上限、权重、满足程度或稳定阈值。需求对外语义仍是目标状态，不是 I64。

### 5.2 初始化快照

快照至少包含：

```text
语素初始化结果
世界树初始化结果
安全值特征定义
安全值语素入口
安全值实例特征槽位
安全值当前特征值
安全根需求
安全根需求目标状态
服务值特征定义
服务值语素入口
服务值实例特征槽位
服务值当前特征值
服务根需求
服务根需求目标状态
初始化代次
```

快照只引用仓库中已经由领域服务写入并读回的稳定句柄，不复制业务值为第二套权威事实。

## 6. 基础语素初始化

自我线程首先调用现有 `语素初始化服务::初始化基础显示语素()`。该阶段继续保持现有边界：

```text
基础显示语素只提供结构化名称材料。
显示文本不裁决节点类型、需求状态或线程状态。
语素入口通过语素服务写入和读回。
```

安全值、服务值属于根需求的语义前置，由 `初始化.需求.ixx` 使用语素服务建立概念入口，并分别绑定到对应特征定义；不得把字符串本身当稳定身份。

## 7. 世界树初始化

自我线程第二步调用现有 `世界树初始化服务::初始化自我世界树()`，取得：

```text
世界树根节点
自我所在场景
自我存在节点
根到场景父子关系
场景到自我父子关系
场景接纳自我归属关系
自我零点坐标
```

两个根需求必须使用该结果中的同一个自我存在和现实场景，不再创建第二套自我锚点。

## 8. `初始化.需求.ixx` 设计

### 8.1 定位

新增 `需求初始化服务`，只通过特征服务、语素服务、需求服务和状态服务完成编排。它不得包含节点仓库、主信息仓库、关系仓库或索引仓库的可写引用。

### 8.2 单个根需求结构

```text
特征定义
-> 概念语素入口和显式稳定键
-> 自我实例特征槽位
-> 当前特征值材料
-> 根需求节点
-> 需求主体引用：自我存在
-> 目标宿主引用：同一自我存在
-> 需求场景引用：自我所在场景
-> 目标特征引用：对应特征定义
-> 当前材料引用：对应当前特征值
-> 抽象目标状态模板关系
```

### 8.3 自我需求自我口径

当前完整需求入口把 `需求主体 == 目标宿主` 作为普通拒绝。第一轮新增专用入口 `创建自我目标状态需求`：

```text
只允许主体与目标宿主是同一个有效存在。
必须提供有效现实场景、有效特征定义和可读当前特征值。
必须经状态服务创建抽象目标状态。
必须写入完整主体、目标宿主、场景、特征、当前材料和目标状态关系。
必须能由 读取需求承接材料 完整读回。
```

普通完整需求入口的既有“主体不得等于目标宿主”规则保持不变，避免扩大其他需求入口语义。

### 8.4 两个根需求

```text
安全根需求：目标特征为“安全值”，目标状态值材料为 10。
服务根需求：目标特征为“服务值”，目标状态值材料为 10。
```

第一轮不新增根需求专用节点类型、不新增需求名称字段、不用显示文本判断根需求身份。根需求角色由自我存在、目标特征和初始化结果槽位共同限定。

## 9. 重复启动与可见性

同一个自我线程对象只执行一次系统初始化。初始化成功后再次调用启动，逻辑内返回重复启动结果并复用原初始化快照，不再次调用三个初始化服务，不增加节点、关系或索引。

初始化快照只在三个阶段全部成功后一次性发布。任何阶段发生追根因错误时：

```text
自我线程标记初始化失败并退出。
入口等待结果后返回启动失败。
程序不得进入控制面板、SQL 审计投影或常驻循环。
当前运行期仓库随进程退出释放，不把部分初始化结构继续作为运行事实使用。
```

本切片不实现跨进程恢复，也不把 SQL Server 当初始化来源。

## 10. 逻辑内返回与追根因解决

### 10.1 逻辑内返回

```text
依赖为空。
稳定键为零或相同。
当前值或目标值参数超出入口允许范围。
重复启动。
首笔结构写入前收到停止请求。
```

这些分支必须发生在写入前，结构不变化。

### 10.2 追根因解决

```text
语素服务开始写入后结果或读回不及预期。
世界树服务开始写入后句柄、关系或坐标读回不及预期。
需求初始化开始创建特征、语素、特征值、需求或目标状态后结果不及预期。
两个根需求未同时形成完整承接材料。
初始化快照发布后读回身份与仓库当前版本不一致。
```

追根因路径必须停止依赖流程，不得降级为“少一个根需求也继续运行”。

## 11. 简化验收

| 编号 | 验收条件 |
| --- | --- |
| SELF-INIT-A1 | 自我线程实际进入，并在线程入口完成初始化；主线程不再直接调用语素和世界树初始化 |
| SELF-INIT-A2 | 基础语素结果成功，世界根、场景、自我存在和三条核心关系可读 |
| SELF-INIT-A3 | 安全值、服务值概念语素分别按非零唯一稳定键读回并指向对应特征定义 |
| SELF-INIT-A4 | 安全根需求、服务根需求各一个，目标状态值材料均为 10，完整需求承接材料可读 |
| SELF-INIT-A5 | 重复启动不重复初始化，结构数量不增加 |
| SELF-INIT-A6 | 初始化失败时入口不进入控制面板、SQL 审计投影或常驻循环；线程可收口 |

## 12. 完成声明边界

计划执行并验证后只能声明：

```text
自我线程已在自身线程入口完成第一轮基础语素、世界树和安全 / 服务两个根需求初始化。
初始化结果已经通过简化读回验收，重复启动不重复写入。
```

不得声明：

```text
自我治理循环完成。
自我苏醒或初步成熟完成。
任务、方法、本能动作或需求结算已运行。
旧鱼巢初始化能力已迁移完成。
数据库恢复、外设、D455 或体素已接入。
```
