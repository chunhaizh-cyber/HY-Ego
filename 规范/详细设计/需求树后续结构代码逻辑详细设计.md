## 5. 请求分流

```text
读取需求树后续结构材料
-> 复核需求节点、可选父需求、目标状态和承接材料完整性
-> 返回请求状态、目标状态和第一轮阻塞权重材料。

结构化更新请求判定
-> 复用需求树后续结构材料读取
-> 当前只判断请求是否应拒绝
-> 不写父子关系、不写索引、不创建需求节点。

枚举目标准入材料
-> 复用需求树后续结构材料读取
-> 只证明目标状态材料可读
-> 不实现枚举目标值域、生产者分级或合法性算法。

方法候选准入材料
-> 读取需求承接材料
-> 只证明主体、目标宿主、场景和目标状态齐全
-> 不召回方法、不选择方法、不绕过任务方法关系。
```

## 6. 入口前置拒绝

### 6.1 读取需求树后续结构材料

必须在入口或材料读取前拒绝：

```text
需求节点无效。
需求节点类型不是需求。
父需求非空但无效。
父需求非空但类型不是需求。
父需求等于当前需求。
需求目标状态缺失。
需求目标状态多目标或类型错误。
```

当前第一轮只拒绝父需求等于当前需求，不证明完整祖先 / 子孙环路检测已经实现。后续若允许写父子关系，施工计划必须补完整环路检测和关系数量不变化验收。

### 6.2 结构化更新请求判定

当前入口只允许返回拒绝判断，不允许提交结构变化。即使材料状态为“可作为请求材料”，也只能解释为“后续提交候选材料具备”，不能解释为父子关系可以直接写入。

### 6.3 枚举目标准入材料

当前入口必须保持只读。返回 true 只表示目标状态材料可读，不表示枚举目标值域已验证、目标值合法性已裁决、生产者分级已建立、方法候选可执行或需求满足已成立。

### 6.4 方法候选准入材料

当前入口只读取需求承接材料完整性。返回 true 不得绕过任务服务按需求创建任务、任务授权关系、任务承接关系、任务选择方法关系、方法服务条件 / 结果判定或动作入口来源准入。

## 7. 后续候选门禁

### 7.1 父子需求角色、阻塞和权重

后续若进入施工计划，必须先裁决：

```text
父子关系采用既有关系类型还是新增专用关系类型。
一个子需求第一轮是否只允许一个当前父需求。
父子成环检测放在需求服务还是关系仓库通用入口。
阻塞材料是否仅作为任务筹办材料，不作为任务执行权限。
权重材料是非权威缓存、评分材料，还是暂不持久化。
```

第一轮建议：只生成只读视图和拒绝矩阵，不直接实现权威权重系统。

### 7.2 逻辑组织需求和获取途径

逻辑组织需求仍必须是需求节点及其关系 / 角色材料，不新增独立“逻辑组织需求”节点类型。AND、OR、方法路径组和因果子链支撑只能作为角色或候选材料，不能由文本或旧枚举裁决机器事实。

### 7.3 方向签名、枚举目标和方法候选

方向签名必须来自目标状态、目标特征、二次特征或已确认值域材料，不能来自旧方向 bool 或显示文本。枚举目标和值域规则必须与特征值系统、需求结算记录和方法服务候选入口共同设计。

方法候选准入只允许输出候选材料或拒绝原因，不得选择方法，不得执行方法，不得写任务方法关系。

### 7.4 结构化更新提交

后续若新增提交入口，必须采用“两段式”：

```text
构造请求壳：只做材料收集，不写结构。
统一提交入口：在写入前完成全部前置拒绝，再一次性写父子关系和必要索引。
```

提交失败时必须不返回成功，不留下可读有效半结构。不能证明物理回滚时，必须采用失效标记、读取隔离或登记事务边界待确认问题。

## 8. 服务边界

需求树后续结构的写入方只能是需求服务或后续确认的需求服务统一提交入口。调用方不得绕过需求服务直接写关系仓库、索引仓库或需求主信息字段。

允许读取方：

```text
需求服务：读取材料、拒绝原因和只读视图。
任务服务：读取承接材料和后续筹办材料。
方法服务：读取后续方法候选材料。
显示层只读：读取结构化材料后显示，不写业务事实。
```

禁止：

```text
需求服务直接访问特征值服务。
任务服务或方法服务直接访问特征值服务。
显示层写需求树关系。
SQL / 控制面板 / 日志 / 文本视图裁决需求树事实。
线程、Tick、外部输入或控制台命令成为动作来源。
```

## 中途非成功返回二分口径

本文件按 2026-07-09 硬规则修订：中途非成功返回只分为 `追根因解决` 和 `逻辑内返回`。

- `追根因解决`：前置条件已经满足，并进入创建、绑定、写关系、写状态、记录动态、结算、读回或结构承载后，结果不符合内部预期；必须停止依赖路径，定位根因，当前未证明完整回滚时登记事务隔离缺口或半结构隔离缺口。
- `逻辑内返回`：领域协议允许的拒绝、候选为空、请求材料返回或人读材料返回；必须保证结构不变化，且返回材料、日志、回执、显示或控制台输出不裁决机器事实。
