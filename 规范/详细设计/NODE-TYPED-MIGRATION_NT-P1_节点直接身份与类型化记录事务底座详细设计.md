## 1. 目标与完成边界

NT-P1 的目标不是把 `主信息` 改一个名字，而是建立以后所有领域迁移共同消费的结构底座：

```text
节点记录直接稳定身份
+ 关系仓库承担全部正式拓扑
+ 领域自有强类型记录参与同一事务
+ 索引只承担可重建定位
+ 同一独占许可内完整读回、确认、逆序撤销与最后发布
```

本设计只冻结共享结构、函数、调用语境、失败收口、文件所有权候选和验证合同。它不迁移特征值、状态、动态、自我、需求、任务、方法或 D455 业务，不授权 C++ 修改，不证明代码已接通。

## 4. 新的节点直接身份合同

### 4.1 稳定主键值类型

实现应新增值类型，名称固定为 `节点稳定主键`，最小形态为：

```cpp
struct 节点稳定主键 {
    std::uint64_t 命名域 = 0;
    std::uint64_t 键值 = 0;
};
```

规则：

1. 两个分量均非零才有效；相等必须比较两个分量。
2. 命名域常量、允许节点类型和签发规则统一使用 4010 冻结的节点稳定主键命名域 ABI 版本 1；代码以强类型 `节点稳定主键命名域` 和固定 `constexpr` 声明表表达，不建立运行期注册表。
3. 节点仓库执行值完整、复合键唯一、域与节点类型匹配、每域高水位、撤销烧号和历史不复用；领域服务 / 专用数据操作仍在第一笔写入前裁决完整业务身份与角色关系。
4. 该值随节点历史保存，不因节点编号、节点版本或索引布局变化而改变。
5. 索引仓库的 `最终物理键` 仍是定位键；不得把它复制到 `节点稳定主键.键值` 后宣称完成迁移。

若实现评审发现请求命名域不在 4010 固定表、域与节点类型错配、固定表不能覆盖目标对象或必须运行期注册新域，必须退回规范治理；不允许 P1 用哈希、节点类型数值、旧主信息编号或索引物理域猜测。

### 4.2 节点记录

目标 `节点记录` 字段顺序和职责固定为：

```cpp
struct 节点记录 {
    std::uint64_t 节点编号 = 0;
    节点稳定主键 稳定主键;
    节点类型 类型 = 节点类型::未分类;
    std::uint32_t 版本号 = 0;
    记录状态 状态 = 记录状态::无效;
    std::uint64_t 创建序号 = 0;
};
```

| 字段 | 写入方 | 生命周期 | 不得承担 |
| --- | --- | --- | --- |
| 节点编号 | 节点仓库单调分配 | 永不复用 | 业务名称、命名域或值 |
| 稳定主键 | 领域请求提供，节点仓库复核 | 节点历史内不可改 | 索引物理布局、显示键、拆分旧句柄 |
| 类型 | 领域请求提供 | 不得原地复义 | 完整业务状态 |
| 版本号 | 节点仓库 | 生命周期换代单调增加 | 普通领域值版本 |
| 状态 | 节点仓库 | 服从 4010 ABI | 业务完成或满足 |
| 创建序号 | 节点仓库单调分配 | 可留高水位空洞 | 稳定身份或业务顺序 |

节点仓库内部可以维护 `节点稳定主键 -> 节点编号` 的可重建定位表，以避免全表扫描；该表不进入独立事实合同。删除、恢复或重建不得使同一复合稳定主键指向另一节点。

### 4.3 创建与幂等

规范化创建入口：

```cpp
带值结构写入结果<节点未发布候选> 结构化创建节点未发布候选(
    节点类型 类型,
    节点稳定主键 稳定主键,
    const 结构事务令牌& 令牌);
```

结果分支：

| 条件 | 分类 | 结果 |
| --- | --- | --- |
| 无效许可、零主键、未知类型、领域层已判错命名域 | 逻辑内返回 | 具名拒绝，零写 |
| 同稳定主键已有当前有效节点，且节点类型和完整创建语义相同 | 逻辑内返回 | `幂等读回`，返回既有当前句柄，零写 |
| 同稳定主键已有当前或历史身份，但类型 / 创建语义不同 | 逻辑内返回 | 新增的具名 `身份冲突` 状态，零写 |
| 无占用且前置完整 | 正常候选 | 单调分配节点编号与创建序号，版本 1，状态有效 |
| 前置通过后插入、登记或读回失败 | 内部逻辑错误 | 登记首次失败，撤销或隔离 |

`身份冲突` 应追加到 `结构写入状态` 末尾，不得重排既有数值。最终数值由叶子计划在 ABI 审查时冻结。

## 5. 领域类型化记录参与者

### 5.1 不建立通用记录仓库

P1 不新增下列结构：

```text
通用领域记录仓库
跨领域记录编号
variant / any / void* / 字节数组形式的业务载荷
自由记录类型数值表
按槽位索引的可选值数组
```

每个领域定义自己的记录类型、字段、记录版本、格式版本和存储。P1 只复用现有 `结构写入事务参与者` 生命周期，使这些强类型记录和节点 / 关系 / 索引在一个事务中共同发布。

### 5.2 参与者最小状态机

```text
空闲
-> 已登记候选
-> 已准备
-> 已确认待发布
-> 已发布

已登记候选 / 已准备 / 已确认待发布
-> 已撤销
```

每个具体参与者至少保存：

```text
所属节点完整当前句柄
本领域具体记录类型
记录版本、格式版本和记录状态
强类型写前值与写后值
本次候选身份
阶段
精确撤销材料
```

参与者的 `准备提交` 只取得 `结构提交准备只读视图`，不得取得仓库、令牌或锁。其 `完成发布()` 必须 `noexcept` 且只关闭候选能力，不分配资源、不做新校验、不调用外部服务。

### 5.3 提交准备只读视图

新路径保留或新增：

```cpp
bool 节点是本会话候选(节点句柄 节点) const;
std::optional<节点类型> 读取候选节点类型(节点句柄 节点) const;
std::optional<节点稳定主键> 读取候选节点稳定主键(节点句柄 节点) const;
```

新路径删除：

```cpp
主信息是本会话候选
读取候选节点主信息
读取候选I64值
```

具体领域参与者验证其自身强类型材料，不通过该视图读取其它领域值。若参与者需要关系材料，必须在业务回调阶段通过结构会话的具名值式读取完成并把必要结果固化到参与者候选；不得在最后发布区重新查询。

## 6. 结构写入会话与执行器

### 6.1 新会话持有边界

规范化 `结构写入会话` 只持有：

```text
节点仓库
关系仓库
索引仓库
本次结构事务令牌
所有线程身份
节点 / 关系 / 索引精确写集
阶段、决定和首次失败
```

它不持有主信息仓库，不管理领域业务值，不保存参与者指针。参与者集合仍由执行器拥有，避免会话取得领域语义。

### 6.2 新会话函数

```cpp
struct 稳定主键当前身份读取结果 {
    enum class 状态 { 空闲, 已占用, 入口拒绝, 内部不一致 };
    状态 结果状态;
    std::optional<节点句柄> 节点;
};

稳定主键当前身份读取结果 读取稳定主键当前身份(节点稳定主键 稳定主键);

带值结构写入结果<节点句柄> 创建节点候选(
    节点类型 类型,
    节点稳定主键 稳定主键);

std::optional<节点稳定主键> 读取节点稳定主键(节点句柄 节点) const;
```

旧 `读取物理主键当前绑定` 应改名为 `读取索引物理键当前绑定`，仅表达索引定位；兼容旧名若暂存，只能转发同一非权威读取，禁止新调用。

### 6.3 执行顺序

```text
验证执行器接线、回调和参与者集合
-> 取得独占许可
-> 构造新会话
-> 同步执行业务回调
-> 回调内完成节点 / 关系 / 索引候选和全部读回
-> 会话请求提交
-> 按登记顺序准备全部强类型参与者
-> 确认结构会话
-> 按登记顺序确认全部参与者
-> 进入无失败区
-> 完成结构会话发布
-> 完成全部参与者发布
-> 返回
-> 许可析构释放，形成普通读取唯一可见点
```

发布顺序本身不表示中间可见；在独占许可释放前，普通读取均不得观察候选组合。

### 6.4 逆序撤销

```text
全部领域记录参与者按登记逆序撤销
-> 索引写集逆序删除
-> 已发布关系变更逆序恢复
-> 新关系逆序删除
-> 节点候选逆序撤销
-> 读回证明前态
```

任何一项失败仍继续尝试其余可撤销项。最终无法证明前态时，执行器必须在释放独占许可前调用事务接线的隔离入口。编号与创建序号高水位允许前进，不得复用。

## 7. 关系、索引与冻结边界

### 7.1 关系

关系仓库现有值式端点、版本、状态、失效和重挂能力原则上保留。NT-P1 只修正它对节点仓库的构造依赖和节点有效性读回，不新增业务关系角色，不让稳定主键代替节点句柄成为关系端点。

### 7.2 索引

为消除“两个主键”歧义，设计术语固定为：

```text
节点稳定主键：节点记录直接字段，长期身份，命名域内不可复用。
索引物理键：索引所有者命名域内的定位键，可由业务规范形成，可重建。
```

新函数和字段优先使用 `索引物理键`。现有 `主键绑定记录`、`绑定主键`、`按主键查节点` 可在迁移期保留兼容名称，但新路径不得新增调用；叶子计划必须登记退役命中和最终零命中门禁。

### 7.3 值式冻结

目标 `冻结节点记录` 与运行期节点记录字段等价：

```cpp
struct 冻结节点记录 {
    std::uint64_t 节点编号;
    节点稳定主键 稳定主键;
    节点类型 类型;
    std::uint32_t 版本号;
    记录状态 状态;
    std::uint64_t 创建序号;
};
```

P1 可以提供节点、关系、索引各自的值式导出和领域参与者的强类型导出接口，但不得创建包含任意领域载荷的通用顶层容器。完整快照聚合、隔离装载和运行期上下文发布由 NT-P4 设计拥有。

索引冻结材料属于可丢弃加速材料；删除索引材料后必须能从节点、关系和领域记录重建，不得用索引补齐缺失稳定主键或业务事实。

## 8. 输入契约与调用语境

### 8.1 允许调用方

```text
领域服务完成业务准入
-> 服务专用数据操作形成值式结构请求
-> 结构写入执行器同步回调
-> 不透明结构会话和本域强类型参与者
```

外部调用方、线程、消息、SQL、持久化材料、显示和日志不得直接调用节点候选、参与者或执行器。

### 8.2 创建请求最小输入

```text
节点类型
节点稳定主键（命名域 + 键值）
领域幂等身份和完整创建语义
后继关系请求
可选索引请求（具名所有者、命名域、探测版本）
零个或多个本域强类型记录候选
```

领域幂等身份不进入节点通用字段；由领域服务比较完整语义。同键同义才允许幂等读回，同键异义必须具名冲突。

## 9. 非成功返回二分

### 9.1 逻辑内返回

以下必须在第一笔写入前零写返回：

```text
无效稳定主键或对象规范未登记命名域
未知节点类型、无效句柄、错版本或越权调用
同稳定主键同义请求的幂等读回
同稳定主键异义请求的身份冲突
索引所有者、命名域或请求形态无效
参与者集合为空规则不适用、重复指针或回调为空
显式撤销或未作提交决定
```

### 9.2 内部逻辑错误

以下发生在准入通过后，必须停止新增写入并追根因：

```text
候选插入后无法按完整句柄和稳定主键读回
同一稳定主键出现两个可读当前节点
关系端点、版本、基数或正反材料不一致
索引写后正反绑定不一致
参与者写后记录、版本或所属节点不一致
会话或任一参与者确认失败
逆序撤销无法恢复可读前态
发布后通过正式服务读回不一致
```

不得把这些错误降级为普通拒绝、缓存失效、日志提醒或旧路径回退。

## 10. 迁移策略与已发现范围漂移

### 10.1 禁止的伪兼容

下列“兼容”都会重建已退出权威，禁止采用：

```text
把主信息编号直接复制成节点稳定主键
节点记录同时保存稳定主键和主信息句柄
把主信息句柄移动到一个新命名的“权威材料”字段
同一次领域写同时写主信息值槽和类型化记录
用通用 variant / JSON / 字节包接收所有领域记录
让新节点通过旧索引物理键证明稳定身份
```

### 10.2 可接受的阶段迁移

两种候选中，本路线正式选择第二种：

1. **一次性切换**：不采用。它会把 99 个旧结构命中文件和全部生产调用方压入一个不可安全审查的施工批次。
2. **隔离新结构域**：采用。先在不接入现行生产装配、与旧运行期身份完全隔离的新结构域建立底座；后继 P2/P3 只在该域形成新对象，禁止同一稳定身份双写；待全部消费者迁移后由 P4 一次切换默认装配。旧域仅作待退役实现事实，不能与新域建立关系、互查或提供新结构准入材料。

隔离新结构域不是长期双轨：

```text
旧域：只维持当前生产和回归事实，不取得新规范符合性
新域：创建节点直接身份、完整支持正式关系 ABI 0—23，并承载领域类型化记录对象；关系 19—23 只是本轮追加 ABI / 新增主题角色，不是仓库支持上限；新域不接默认生产入口
跨域：句柄、关系、索引、读取投影、幂等账和恢复候选全部禁止互通
切换：P4 在完整冻结、隔离恢复、索引 / 投影重建和全量验证后一次改变默认装配
退役：切换后删除旧仓库、旧句柄、旧字段和旧工程登记
```

不允许在同一运行期仓库中让一部分当前节点缺稳定主键、另一部分节点有稳定主键并由普通业务共同读取，因为这会使 4010 的节点身份不变量无法成立。

### 10.3 当前 NT-P1 候选白名单修正

`节点记录.主信息` 被多份生产文件中的创建 / 读取路径消费；当前只允许核心候选文件时，无法同时做到：

```text
删除节点对主信息的依赖
保持工程可编译和现有生产入口可运行
不建立主信息到稳定主键的伪映射
不产生同对象双权威
```

因此旧 NT-P1 候选白名单退出。新叶子计划只允许追加 ABI 和建立隔离新结构域，不直接修改现行 `节点仓库.h/.cpp`、`关系仓库.h/.cpp`、`索引仓库.h/.cpp`、旧会话 / 执行器或生产装配。叶子计划必须新增独立模块、隔离自检、禁止跨域关系和最终切换门禁；P4 才取得旧核心文件的物理退役权。

### 10.4 隔离域代码文件候选

| 文件 | 分类 / 模块 | 唯一职责 |
| --- | --- | --- |
| `海中鱼巣/核心/句柄.h` | 既有数据结构头 | 只追加 `节点稳定主键`、`节点稳定主键命名域`、节点类型 16—19 和关系类型 19—23 数值 ABI；不删除旧符号 |

| 编号 | 必须证明 |
