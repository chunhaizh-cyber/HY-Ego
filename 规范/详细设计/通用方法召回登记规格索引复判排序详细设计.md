本轮目标是把“调用方已知方法首后读取结构”推进为：

```text
方法首权威登记发现
-> 条件 / 结果结构化规格
-> 可清空、可重建的派生多值索引
-> 按任务目标只读召回
-> 运行上下文条件复判
-> 确定性排序与只读选择建议
-> 更新后复核任务授权选择接线
```

本设计不直接实现代码，不把候选、缓存、哈希、排序或显示材料升格为业务事实，不实现方法学习、物理删除、跨重启恢复或自动执行。

## 2. 当前代码事实

### 2.1 已实现

```text
方法节点、方法首、条件节点、结果节点和动作入口均使用节点类型::方法并以角色抽象状态区分。
方法首可绑定唯一虚拟存在、条件节点组、结果节点组和唯一未失效动作入口。
来源任务、父方法、前置方法和后续方法已经有绑定与读取入口。
动作入口可用稳定动作键通过现有单值索引读取。

特征服务可读实例槽位唯一当前值、原始材料、原始版本、内容签名和跨特征完整内容相等结果。
关系仓库、节点仓库和现有索引仓库分别有自己的锁边界。
```

## 3. 权力边界

| 对象 | 权威写入方 | 允许读取方 | 权力边界 |
| --- | --- | --- | --- |
| 方法登记根与登记关系 | 方法服务 | 方法服务、后续只读召回服务 | 根和关系裁决方法是否进入正式发现集合 |
| 方法生命周期 | 方法服务 | 方法服务、召回服务、任务服务后续接线 | 活跃 / 失效由结构关系承载；统计和索引不得转换 |
| 条件 / 结果规格 | 方法服务编排状态与特征服务公开入口 | 方法服务、召回服务 | 规格是权威结构；文本和动作键不参与 |
| 结果能力派生索引 | 方法召回索引 | 方法召回服务 | 只加速候选，可清空、可重建、命中后必须重读权威结构 |
| 召回 / 复判 / 排序 | 方法召回服务 | 任务筹办或治理调用方 | 只输出值式材料，不写任务方法关系 |
| 任务选择方法关系 | 任务服务 | 方法执行桥、控制面板只读层 | 只有任务服务可在授权与当前态契约满足后提交 |
| 方法学习 | 后续独立领域计划 | 后续方法服务 | 必须依赖任务结果、需求结算、动态和因果证据，不使用命中次数裁决 |

高级服务不得直接调用特征值服务。方法服务和方法召回服务需要特征值材料时，只能经特征服务。

## 4. 默认裁决

### 4.1 沿用 #193 MR-Q1 至 MR-Q10

```text
方法发现由方法服务权威登记根和登记关系提供，禁止全节点扫描。
结果能力使用目标状态和结构化特征 / 状态材料，不使用文本、函数名或稳定动作键。
多值索引首轮归方法召回域，保持非权威；不改义现有索引仓库。
召回上下文包含任务来源需求、任务目标状态、运行场景和需求目标状态；父需求只作上下文。
条件复判重读带版本特征材料；缺失、未知或漂移不视为满足。
排序先结果匹配、再条件复判、再显式策略证据，最后完整句柄只做确定性排列。
登记、生命周期、规格或规则变化使候选索引失效。
恢复后丢弃派生索引与排序结果，从权威登记和规格重建。
任务方法关系只由任务服务提交。
方法学习后置。
```

### 4.2 本设计补充裁决

| 编号 | 问题 | 裁决 |
| --- | --- | --- |
| MR-D1 | 方法登记根形状 | 使用一个 `节点类型::方法` 节点并新增 `方法角色状态::方法登记根`；根不代表可执行方法，不创建方法虚拟存在 |
| MR-D2 | 登记关系 | 根通过 `关系类型::归属` 指向方法首；该关系是方法发现发布边界 |
| MR-D3 | 生命周期 | 根持有共享活跃 / 失效抽象状态，方法首用固定顺序号模板关系指向其中一个；关系版本是该方法生命周期版本 |
| MR-D4 | 失效语义 | 失效只把方法移出活跃发现、动作读取和派生索引；登记项继续审计可读，不物理删除 |
| MR-D5 | 条件规格第一版 | `特征定义 + 期望特征值`，期望值必须经特征服务证明属于该特征定义 |
| MR-D6 | 结果规格第一版 | `目标特征 + 目标抽象状态`，目标状态必须是非实例状态并可读状态材料 |
| MR-D7 | 规格修改 | 第一版规格写后不可原地改义；需要改变规格时失效旧方法或另建版本化修改专项 |
| MR-D8 | 无条件方法 | 必须显式登记 `显式无条件` 策略；空条件集合且无显式策略一律不可判定 |
| MR-D9 | 语义并列 | 完整句柄可稳定排序，但不能把语义并列改写成授权唯一 |
| MR-D10 | S7 收口 | 因当前无任务授权和通用当前态契约，先执行更新后 S0，再生成任务服务选择提交专用三件套 |

上述裁决没有未决用户门禁。执行时若实际接口无法满足，必须按对应计划退回修订，不得自行改义。

## 5. S1 方法首权威登记与生命周期

### 5.1 目标结构

新增值类型候选：

```text
方法生命周期状态
  未定义
  活跃
  已失效

方法登记根材料
  稳定非名称键
  登记根
  活跃状态
  失效状态

方法登记项材料
  登记根
  方法首
  登记关系完整句柄
  生命周期状态
  生命周期关系完整句柄
  生命周期版本
```

固定关系槽位建议：

```text
方法角色顺序号 = 0（现有）
动作入口状态顺序号 = 1（现有）
方法生命周期顺序号 = 2
方法条件集合策略顺序号 = 3
根活跃状态顺序号 = 40
根失效状态顺序号 = 41
```

根到方法首使用 `归属`，不与方法首到条件 / 结果的 `模板`、组织追溯 `引用` 或稳定动作键 `索引` 混用。

### 5.2 公开入口假定

```text
方法服务::初始化方法登记根(稳定非名称键, 状态服务)
方法服务::读取方法登记根()
方法服务::读取全部方法登记项(状态服务)
方法服务::读取活跃方法首组(状态服务)
方法服务::读取方法登记项(方法首, 状态服务)
方法服务::失效方法首(方法首, 状态服务)
```

现有 `创建方法首节点` 保持调用形状，但要求登记根已经初始化，并在最后写入根到方法首的归属关系。现有 `登记方法` 继续只是底层方法节点分配入口；裸方法节点不进入登记根，不得被通用发现。

### 5.3 发布与隔离

```text
创建方法节点、角色状态、虚拟存在和活跃生命周期关系。
这些步骤完成前，方法首不在登记根下，通用发现不可见。
最后写根到方法首归属关系，作为单一发布边界。
发布后重读登记项、角色、虚拟存在和生命周期。
读回不一致归追根因；未发布半结构不得由方法服务返回为有效方法首。
```

失效使用关系仓库 `重挂关系` 原地把生命周期目标从共享活跃状态改为共享失效状态。关系编号保持、版本单调推进。失效后：

```text
读取全部方法登记项：仍返回失效项。
读取活跃方法首组：排除。
读取方法候选材料和动作执行材料：排除。
派生索引重建：排除。
已有任务方法关系：不自动删除，由 S7 后续契约处理。
```

### 5.4 并发

方法服务新增私有登记写串行边界和根定位读写保护。创建、失效和根初始化按方法服务单写有序；只读返回值副本。不得持有方法服务锁调用日志、弹窗、SQL、显示、外设或用户回调。

## 6. S2 条件与结果结构化规格

### 6.1 条件规格

```text
方法条件规格材料
  方法首
  条件节点
  条件场景
  特征定义
  期望特征值
  特征关系完整句柄
  期望值关系完整句柄
  期望值原始类型
  期望值原始版本
  规格规则版本 = 1
```

提交前必须证明：

```text
方法首已登记且活跃。
条件节点属于该方法首并有唯一条件场景。
特征定义是特征节点。
期望特征值经特征服务证明属于该特征定义。
期望值原始材料和版本可读。
该条件节点尚无结构化规格。
```

先写特征定义关系，再写期望值关系作为规格发布完成点；读取入口只有在两者唯一且版本一致时返回完整材料。前置通过后写入或读回异常归追根因。

### 6.2 结果规格

```text
方法结果规格材料
  方法首
  结果节点
  结果场景
  目标特征
  目标状态
  特征关系完整句柄
  目标状态关系完整句柄
  目标状态值材料
  规格规则版本 = 1
```

目标状态必须是抽象状态，不得把实例状态、裸 I64、文本或动作输出场景当结果能力。目标状态值只作为该状态节点的结构化材料参与能力键，不改变“需求目标是目标状态”的对外语义。

### 6.3 条件集合策略

```text
方法条件集合策略
  未声明
  全部满足
  显式无条件
```

`全部满足` 要求至少一个完整条件规格；`显式无条件` 要求没有条件节点或条件规格。空集合且未声明不得被解释为满足。

### 6.4 版本

第一版规格不可原地修改。规格版本由：

```text
规则版本
关系完整句柄与关系版本
目标节点完整句柄
期望特征值原始版本（条件）
```

共同组成，不新增一个可被遗漏推进的裸计数器。任一来源变化都会使旧候选材料过期。

## 7. S3 结果能力派生多值索引

### 7.1 文件与结构

建议新增：

```text
海中鱼巣/领域/方法召回索引.h
```

该文件只包含值类型、私有互斥量和非权威索引，不直接写任何仓库。

```text
方法结果能力键
  规则版本
  目标特征完整句柄
  目标状态值材料

方法结果索引项
  结果能力键
  方法首
  结果节点
  登记关系完整句柄
  生命周期关系完整句柄和版本
  结果规格关系完整句柄和版本
  目标状态完整句柄

方法召回索引快照
  发布版本（仅索引实例内观察）
  规则版本
  多值桶
```

目标状态值材料来自目标状态节点，仅用于同一目标特征下缩小候选；命中后必须重读目标状态节点、状态值和完整规格。它不是裸 I64 需求目标。

### 7.2 入口

```text
方法召回索引::发布重建候选(完整索引项组)
方法召回索引::按结果能力键读取候选(键)
方法召回索引::清空()
方法召回索引::读取发布版本()
```

重建先在锁外或私有临时候选中完成完整校验，再在独占锁内单次替换。读取返回值副本。哈希碰撞由完整键相等复核，不把哈希值当身份。

### 7.3 失效和恢复

```text
方法失效、规格来源版本变化或规则版本变化：命中后判过期，要求重建。
清空：不改变方法、规格、需求、任务或动作入口。
进程恢复：丢弃索引，从权威方法登记和规格重建。
当前无真实恢复入口：不得声明跨重启可用。
```

现有 `索引仓库` 继续只服务单键单节点，不扩展、不改义稳定动作键。

## 8. S4 按任务目标只读召回

### 8.1 服务边界

建议新增：

```text
海中鱼巣/领域/方法召回服务.h
```

该服务只组合方法服务、方法召回索引、需求服务、任务服务、特征服务和状态服务的公开只读入口，不拥有仓库写权限。

### 8.2 请求材料

```text
方法召回请求材料
  任务
  来源需求
  需求主体
  目标宿主
  运行场景
  目标特征
  目标状态
  目标状态值材料
  请求规则版本
```

构造请求时必须交叉复核：

```text
任务承接材料的来源需求等于请求来源需求。
任务运行场景等于需求场景。
任务目标状态等于需求目标状态。
需求目标宿主、目标特征和目标状态均可读。
目标状态值材料在冻结前后未漂移。
```

### 8.3 候选材料

```text
方法召回候选材料
  请求材料值副本
  方法首
  结果节点
  结果能力键
  登记 / 生命周期 / 结果规格来源版本
  召回状态
```

索引桶命中后，逐项重读当前登记、活跃生命周期、结果规格和能力键。正常生命周期或规格并发变化属于候选过期逻辑内返回；在同一当前版本下出现关系、目标或内容矛盾，归追根因。

父需求只可附加为上下文值材料，不参与第一版候选权重。S4 不读取显示、日志、SQL 或控制面板材料。

## 9. S5 运行上下文条件复判

### 9.1 复判对象

对每个 S4 当前候选：

```text
读取方法条件集合策略。
显式无条件：可进入已复判候选。
未声明或空条件：不可判定。
全部满足：逐个读取完整条件规格。
```

### 9.2 特征复判

每个条件规格按以下路径：

```text
需求目标宿主
-> 特征服务读取同一特征定义的实例槽位
-> 读取唯一当前特征值
-> 读取当前原始材料与版本
-> 特征服务复核期望值与当前值完整内容相同
-> 再次复核来源版本
```

复判结论：

```text
全部满足
明确不满足
不可判定
材料过期
内部不一致
```

缺槽位、缺当前值、类型不支持、材料过期或未知都不得视为满足。复判结果只在调用期有效，不写方法状态、任务状态、需求结算或统计事实。

### 9.3 并行边界

冻结请求和候选值材料后，不同候选可由现有任务工作线程并行复判；不得为短小候选组在服务内部创建临时线程。合并顺序不得依赖线程完成顺序。线程不是选择事实或动作来源。

## 10. S6 确定性排序与只读选择建议

### 10.1 排序层级

```text
1. 结果能力匹配等级。
2. 条件复判结论。
3. 已确认的显式策略 / 证据等级。
4. 完整方法句柄确定性排列。
```

第一版没有持久成功率、成熟度或学习评分。缓存命中次数、展示次数、关系编号、节点创建顺序和线程完成顺序不得进入语义评分。

### 10.2 并列规则

完整方法句柄比较顺序：

```text
仓库编号
-> 节点编号
-> 版本号
```

它只保证同一输入得到稳定列表。若前 3 层语义键相同，仍是语义并列；不得因为完整句柄排在第一就自动生成授权唯一。

### 10.3 输出

```text
方法排序项
  方法首
  结果匹配等级
  条件复判结论
  策略证据材料
  来源版本

方法选择建议
  请求
  有序候选组
  状态：无候选 / 语义并列 / 唯一可建议 / 策略唯一 / 材料过期
  建议方法（仅唯一状态可有值）
```

S6 不调用 `任务服务::选择任务方法`。

## 11. S7 任务授权选择接线更新后 S0

#193 已确认当前缺少独立任务授权入口，`选择任务方法` 也没有复核通用当前态、方法登记、规格、召回或建议材料。#201 将先扫描任务生命周期和通用执行。

## 13. 非成功返回二分

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 根参数无效、根未初始化、方法未登记、重复失效 | 逻辑内返回 | 不写或不推进结构 |
| 方法创建发布前内部节点 / 状态 / 关系失败 | 追根因解决 | 不写最终登记关系，半结构不得进入发现 |
| 登记关系写成后完整读回不一致 | 追根因解决 | 停止依赖路径，不返回有效方法首 |
| 失效关系重挂后活跃读取仍命中 | 追根因解决 | 停止候选和执行依赖 |
| 条件 / 结果规格输入无效、重复绑定、实例状态误入 | 逻辑内返回 | 不写规格 |
| 规格写入后唯一读回不一致 | 追根因解决 | 不发布完整规格材料 |
| 索引重建输入为空 | 逻辑内返回 | 可发布空候选快照 |
| 索引候选内部不完整却准备发布 | 追根因解决 | 拒绝替换现有快照 |
| 召回请求不完整、候选桶为空 | 逻辑内返回 | 返回请求拒绝或无候选 |
| 登记 / 生命周期 / 规格正常并发漂移 | 逻辑内返回 | 剔除过期候选并要求重建 |
| 当前来源版本不变但关系或内容矛盾 | 追根因解决 | 停止该候选和依赖路径 |
| 条件缺槽位、缺当前值、未知或不满足 | 逻辑内返回 | 不视为满足，不写事实 |
| 排序无候选或语义并列 | 逻辑内返回 | 返回有序材料但无提交建议 |
| 未来任务选择写后唯一读回不一致 | 追根因解决 | 停止执行桥 |

日志和弹窗只做人读诊断，不修复结构、不裁决候选。

## 14. 并发与锁顺序

```text
方法登记与生命周期写入：方法服务私有写串行边界。
方法规格写入：同一方法 / 规格节点串行，发布前不可读。
派生索引：候选在私有临时结构构造，单次锁内替换。
召回、复判、排序：只读值材料；不持有索引锁调用领域服务。
```

任何读取索引后复核权威结构的入口必须先复制候选、释放索引锁，再调用方法、需求、任务、特征或状态服务。不得形成“索引锁 -> 仓库锁 -> 回调 -> 索引锁”的反向路径。

## 15. 恢复边界

```text
权威恢复候选：方法登记根、根到方法首关系、生命周期关系、条件 / 结果规格关系。
必须丢弃并重建：派生多值索引、召回候选、复判结果、排序结果、选择建议。
当前方法根运行期定位、规格持久化和恢复重建尚未实现。
```

#202 将统一盘点恢复边界。S1-S6 实施不得提前写文件、SQL 或启动恢复。

## 19. JY-303 任务选择适配边界

第一版不新增独立批次身份字段。一次调用期排序批次由 `方法确定性排序材料` 的规则版本、召回请求、索引发布版本、来源候选数量、重复输入数量和完整有序候选组共同构成；`方法选择建议材料` 再给出建议状态与建议方法。

`方法召回服务::适配任务方法选择请求` 必须先调用现有 `复核只读选择建议来源`。该入口已经重读索引发布版本、完整候选组，并逐项复核登记、生命周期、结果规格、条件规格和运行上下文来源；只有来源当前的 `唯一可建议` 才能展开为任务服务自有扁平请求。

方法召回索引保持非权威。索引发布版本、候选数量、重复数量和完整有序候选组进入选择记录只作来源审计与幂等比较，不成为任务提交授权。任务服务不得包含、持有或调用方法召回索引；提交期只复核请求携带的权威结构身份。索引版本单独变化不失效已提交选择；权威来源不变却重建出不同候选集属于索引 / 召回内部不一致，必须在本领域追根因。
