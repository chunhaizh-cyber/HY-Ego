# 需求结算代码逻辑详细设计

更新时间：2026-07-08

状态：已生成 / 可直接生成待确认施工计划 / 未生成施工计划 / 未入队 / 未改 C++ / 未构建 / 未迁移旧函数

## 1. 设计定位

本设计承接第 15 项 `流程图/20260708_需求结算代码逻辑流程图_v0.1.md`，在已确认的 `需求结算记录详细设计` 基础上，按当前代码逻辑明确目标状态、实际状态、来源任务、动态证据和可选轻量因果引用的结算边界。

本设计不声明完整需求结算系统完成，不生成代码实施许可。

## 2. 输入入口

```text
需求节点
来源任务节点
目标状态节点
实际状态节点
场景节点
结算主体
动态证据
可选因果引用
状态服务
因果服务材料
```

入口必须拒绝无效需求、来源任务无效、目标状态与需求目标不一致、目标值或实际值不可读、未结算结果写入、动态证据无效、可选因果引用节点类型错误。

## 3. 核心处理

```text
复核需求目标状态
-> 读取目标状态值和实际状态值
-> 判定证据不足 / 已满足 / 未满足
-> 经状态服务创建结算状态实例状态
-> 写需求到结算状态关系
-> 写结算状态到来源任务、目标状态、实际状态、动态证据的引用关系
-> 可选写结算状态到因果引用的引用关系
-> 读取最近结算记录和各项来源材料
```

I64 只作为状态值材料，不能作为需求目标对外语义。因果引用只作为可选证据材料，不提前裁决需求满足。

## 4. 结构承载

```text
需求节点：结算所属需求
状态节点：目标状态、实际状态、结算状态
任务节点：来源任务
动态节点：动态证据
因果引用节点：可选轻量因果材料
关系仓库：需求结算记录关系、目标状态关系、实际状态关系、来源任务关系、动态证据关系、可选因果引用关系
```

非权威缓存、日志、控制台输出和显示文本不得承载结算事实。

## 5. 输出变化

成功路径输出结算状态句柄，并允许显示层和非权威缓存只读观察。

失败路径不得写结算记录，不得用外部状态替换需求目标，不得留下可读半结算结构。

## 6. 当前代码差距

```text
多步写入已有入口拒绝和追根因解决，但未证明完整事务回滚。
尚缺数量快照级半结构不可读验证。
因果服务参数当前未参与稳定因果裁决。
最近结算冲突样例和追根因解决收口样例仍需施工计划补齐。
```

## 7. 验收矩阵

| 验收项 | 验证方式 | 通过口径 |
| --- | --- | --- |
| 目标状态复核 | 传入非需求目标状态 | 拒绝且不写结算 |
| 状态值读取 | 目标或实际状态缺值 | 只能输出证据不足或拒绝写入 |
| 结算状态写入 | 有效目标、实际和来源任务 | 读回结算状态及来源关系 |
| 因果可选边界 | 传入非因果引用节点 | 拒绝，不让因果裁决结算 |
| 半结构防漏 | 模拟关系写入不及预期 | 不得读到有效结算记录 |

## 8. 后续边界

后续施工计划必须覆盖写前数量快照、每段关系写入不及预期停止门禁、失败后读回不可见验证和最近结算冲突样例。
## 中途非成功返回二分口径

本文件按 2026-07-09 硬规则修订：中途非成功返回只分为 `追根因解决` 和 `逻辑内返回`。

- `追根因解决`：前置条件已经满足，并进入创建、绑定、写关系、写状态、记录动态、结算、读回或结构承载后，结果不符合内部预期；必须停止依赖路径，定位根因，当前未证明完整回滚时登记事务隔离缺口或半结构隔离缺口。
- `逻辑内返回`：领域协议允许的拒绝、候选为空、请求材料返回或人读材料返回；必须保证结构不变化，且返回材料、日志、回执、显示或控制台输出不裁决机器事实。
