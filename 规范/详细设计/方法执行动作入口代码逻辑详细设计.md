## 4. 结构承载

| 业务概念 | 项目承载结构 | 值类型 / 身份结构 | 写入方 | 读取方 | 验证方式 |
| --- | --- | --- | --- | --- | --- |
| 执行桥请求材料 | 任务节点 + 当前任务选择方法关系 | 任务节点、方法节点、请求状态 | 任务服务读取，不新增事实 | 方法服务、后续执行桥 | 任务承接壳完整且唯一任务选择方法可读 |
| 动作入口角色 | 方法节点的动作入口角色节点 | 方法节点句柄、方法角色状态、动作入口状态 | 方法服务 | 方法服务、任务服务、动态服务 | 有效方法首、动作入口角色有效、状态非失效 |
| 稳定动作键 | 索引仓库主键到动作入口节点 | `uint64` 稳定动作键 | 方法服务登记动作入口 | 方法服务 | 非 0、不冲突、命中入口归属该方法首 |
| 输入 / 输出规格 | 动作入口到场景节点引用关系 | 场景节点句柄 | 方法服务 | 方法服务、任务服务 | 规格场景有效且同类规格唯一 |
| 特征状态材料写入 | 特征服务写 I64 特征状态材料 | 特征节点、特征值节点、I64 状态值 | 方法服务经特征服务请求 | 特征服务、任务回执后续读取方 | 任务方法关系和主动动作入口成立后，经特征服务读回 |
| 动作动态证据 | 实例动态节点和来源动作 / 来源方法关系 | 动态节点、动作入口节点、方法首节点 | 方法服务经动态服务写入 | 动态服务、因果服务、任务结果后续读取方 | 动态材料完整，来源动作有效，来源方法关系写入成功 |

稳定动作键只用于定位动作入口候选，不裁决动作事实。动作入口只证明方法具备入口材料，不证明任务已经授权执行；执行权限仍来自任务授权、任务承接和任务选择方法。

## 5. 请求分流

```text
动作入口登记与读取
-> 复核有效方法首、动作入口类型、稳定动作键、索引冲突和重复入口
-> 创建动作入口角色节点
-> 绑定动作入口角色状态和动作入口状态
-> 写方法首到动作入口引用关系
-> 绑定稳定动作键索引
-> 读回有效动作入口。

动作规格与注册材料复核
-> 输入 / 输出规格场景有效
-> 同类规格未重复
-> 写动作入口到规格场景引用关系
-> 读回输入 / 输出规格
-> 本能动作注册材料只作为候选材料。

任务执行桥请求
-> 任务承接壳完整
-> 唯一任务选择方法可读
-> 输出任务节点 + 方法节点
-> 不代表真实分派已发生。

特征状态材料写入
-> 复核任务节点、方法节点、任务到方法关系和主动动作入口
-> 经特征服务写 I64 特征状态材料
-> 后续必须经特征服务读回当前材料
-> 不直接写特征值服务。

动作动态证据记录
-> 复核动作入口归属方法首和主动入口
-> 动态服务记录实例动态
-> 写动态到来源方法首引用关系
-> 读回动态材料和来源方法关系
-> 不声明任务完成或需求满足。
```

## 6. 入口前置拒绝

后续若进入代码实施，必须先拒绝：

```text
无效任务节点。
任务承接壳不完整。
缺少任务选择方法关系。
无效方法节点或目标节点不是方法。
方法首无效或缺主动动作入口。
动作入口类型未设置。
稳定动作键为 0 或已被占用。
重复当前动作入口。
动作入口节点等于方法首节点。
输入规格或输出规格场景无效。
重复绑定输入规格或输出规格。
特征值节点不归属目标特征。
方法服务试图直接调用特征值服务。
来源动作不是动作入口角色节点。
动态缺场景、存在主体、发生时间戳、被改变目标、前值或后值。
线程、函数名、动作名文本、日志标签、控制台命令被当作动作来源。
```

拒绝后不得写动作入口关系、稳定动作键索引、输入 / 输出规格关系、特征状态材料、实例动态或来源方法关系。内部写入结果不符合预期时，必须按逻辑错误追根因。

## 7. 服务边界

| 服务 | 本设计允许 | 禁止越界 |
| --- | --- | --- |
| 任务服务 | 提供执行桥请求材料和任务选择方法读取 | 不分派线程，不写动作动态 |
| 方法服务 | 管理动作入口、规格材料、特征状态写入请求和动作动态证据入口 | 不直接依赖特征值服务，不裁决需求满足 |
| 特征服务 | 写入和读回第一轮 I64 特征状态材料 | 不由任务 / 方法服务绕过特征服务访问特征值服务 |
| 动态服务 | 记录实例动态和读取动态材料 | 不写任务实际结果状态，不生成稳定因果结论 |
| 索引仓库 | 根据稳定动作键定位动作入口 | 不裁决动作事实 |
| 显示层 / 日志 | 只做人读观察和诊断 | 不写机器事实，不证明方法成功 |

## 中途非成功返回二分口径

本文件按 2026-07-09 硬规则修订：中途非成功返回只分为 `追根因解决` 和 `逻辑内返回`。

- `追根因解决`：前置条件已经满足，并进入创建、绑定、写关系、写状态、记录动态、结算、读回或结构承载后，结果不符合内部预期；必须停止依赖路径，定位根因，当前未证明完整回滚时登记事务隔离缺口或半结构隔离缺口。
- `逻辑内返回`：领域协议允许的拒绝、候选为空、请求材料返回或人读材料返回；必须保证结构不变化，且返回材料、日志、回执、显示或控制台输出不裁决机器事实。
