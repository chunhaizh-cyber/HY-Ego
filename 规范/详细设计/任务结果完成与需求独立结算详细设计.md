# 任务结果完成与需求独立结算详细设计

日期：2026-07-15

状态：JY-351 / #224、DQ-116、600 已完成 / 正式接口复核通过 / #225 当前首个待执行 / 610 未实施

## 1. 定位

本文依据 `规范/5160_子规范_需求正式结算记录与唯一结论.md`、`规范/5230_子规范_任务筹办与执行边界_20260720.md` 与 `规范/详细设计/任务执行调度与强类型回执详细设计.md`，替代已经退出的 `TASK-RESULT-S1` 旧许可下传口径，只设计以下闭环：

```text
#224 强类型非权威执行回执
-> 协议准入和完整来源请求复核
-> 任务 / 需求 / 方法 / 动作 / 状态 / 动态权威重读
-> 需求业务服务目标达成只读裁决
-> 未达成：执行中任务迁移待重筹办
-> 已达成：任务实际结果与已完成生命周期原子提交
-> 需求独立结算
```

本文不设计真实动作执行、方法学习、稳定因果结论、持久化待结算队列、恢复发布、概念命名绑定或生产常驻线程主循环。

## 2. 依据和当前代码事实

### 2.1 已验证当前事实

1. `任务业务服务` 的公开面不含原始事务能力，已经提供 `读取任务`、`读取动作桥请求`、`迁移生命周期` 和 `提交实际结果并完成`。
2. `提交实际结果并完成` 在一次结构写入会话内创建完成实例状态、绑定任务唯一实际结果、发布新生命周期并失效旧执行中生命周期；完成后重新读取任务材料。
3. `需求业务服务` 已提供 `读取需求` 和 `提交独立结算`。结算独立取得自己的结构写入会话，不与任务完成共享回滚边界。
4. 当前需求结算复核来源任务已完成、任务实际结果、目标状态、场景和主体，并以目标状态值与实际状态值相等形成第一轮结论。
5. `状态业务服务::读取状态材料` 返回抽象状态或实例状态的值式完整材料。
6. `动态业务服务::读取动态材料` 返回实例动态完整材料，目标 / 前状态 / 后状态 / 来源动作顺序合同为 `1 / 2 / 3 / 4`。
7. `方法业务服务` 可读取方法登记项和角色规格，当前动作桥只由任务服务在执行中任务上返回。
8. `需求任务方法组合器` 已有“完成任务并独立结算”隔离基础入口，但它不接收线程回执、不重读方法 / 动态来源，也不处理目标未达成到待重筹办。
9. `#224` 已新增 `任务结果回执材料性质::非权威任务执行结果证据`，回执同时携带实际状态、动态、方法、动作、场景、主体句柄和完整 `任务执行冻结请求`；协议逐项复核外层回执编号、时间、幂等键、任务序号、批次、任务、方法、动作、场景和主体。
10. 完整冻结请求已包含任务来源需求、虚拟存在、场景、目标状态、生命周期、选择、方法和动作的完整关系证据；实际状态和动态由回执外层句柄提供并由组合器重新读取权威材料。
11. `任务工作线程::取出一条任务结果回执()` 在内部锁内移动并弹出一条值式回执，返回后不持队列锁；停止锁存后拒绝新发布，但允许排空停止前已入队回执。
12. 传统 `任务管理上行桥` 直接依赖旧服务头，只读生成治理消息。本轮不得把它改为新结果写入所有者。

### 2.2 当前缺口

```text
没有接收 #224 新回执并重读新分层权威服务的结果治理路由。
没有把任务、需求、方法、动作、实际状态和动态组合复核放在一个无令牌组合器中。
需求服务缺少公开只读“目标状态是否达成”值式裁决，组合器不能复制裸 I64 比较。
未达成执行结果还没有正式迁移待重筹办的结果治理路径。
结算暂不可执行时，缺少明确的非权威待结算提示和权威重试条件。
```

## 4. 类型和公开入口

### 4.1 任务结果治理请求

请求至少包含：

```text
#224 强类型任务结果回执。
当前时间戳。
未达成转待重筹办的幂等材料编号和发生时间戳。
任务完成状态幂等主键和发生时间戳。
需求结算状态幂等主键和发生时间戳。
可选轻量因果引用句柄。
```

入口先检查：

```text
协议版本和材料性质属于 #224 新执行结果证据。
回执包含完整来源执行请求，且外层句柄与来源请求逐项一致。
全部幂等编号非零且用途不混用。
执行发生时间 <= 回执时间 <= 完成 / 结算时间。
待重筹办时间不早于执行发生时间。
可选句柄形状有效；空因果引用保持空，不补默认节点。
```

非法输入在读取业务结构前返回，并保留明确拒绝原因以追查问题来源。

### 4.2 需求目标状态达成材料

`需求业务服务` 新增只读入口候选：

```text
判定目标状态达成(需求节点, 实际状态节点)
```

返回值式材料至少包含：

```text
读取状态：已找到 / 未找到 / 入口拒绝 / 类型不匹配 / 许可拒绝 / 版本漂移 / 内部不一致。
目标裁决：未定义 / 已达成 / 未达成。
需求、目标状态和实际状态完整句柄。
目标状态与实际状态的值式材料快照。
```

入口实现：

```text
读取完整目标状态需求。
经状态业务服务分别读取需求目标抽象状态和实际实例状态。
复核实际状态属于可读场景内实例材料。
第一轮只比较状态值相等。
```

该 DTO 是本次读取结果，不是持久化机器事实；任务结果组合器只消费裁决枚举，不直接比较两个 I64。

### 4.3 任务结果结算组合器

组合器构造依赖：

```text
需求业务服务。
任务业务服务。
方法业务服务。
状态业务服务。
动态业务服务。
```

最小公开入口：

```text
处理任务执行结果(任务结果治理请求)
```

组合器不依赖线程、队列、运行消息、数据操作、执行器或仓库。

### 4.4 权威重读

组合器必须重新取得并复核：

```text
任务双向承接、来源需求、虚拟存在、场景和目标状态。
执行中或已完成的唯一当前生命周期。
当前选择记录、发布关系、方法投影和来源生命周期身份。
方法登记项、方法生命周期和动作入口角色 / 状态 / 场景规格。
实际状态的实例身份、场景、主体、时间和状态值。
实例动态的目标、前状态、后状态、来源动作、场景、主体和时间。
```

精确一致性：

```text
回执任务、来源请求任务和权威任务一致。
任务来源需求、场景、虚拟存在和目标状态与来源请求一致。
选择方法和动作入口与来源请求一致。
实际状态场景 / 主体等于任务场景 / 虚拟存在。
动态场景 / 主体等于任务场景 / 虚拟存在。
动态后状态等于实际状态，被改变目标等于任务目标状态，来源动作等于回执动作入口。
动态和实际状态发生时间不晚于回执发生时间。
```

已完成幂等重试允许当前生命周期已经是已完成，但必须有唯一实际结果且与回执一致；失败、取消或不同实际结果不得复活或覆盖。

### 4.5 未达成分支

目标裁决为未达成且任务仍是本次精确执行中身份时，组合器调用：

```text
任务业务服务::迁移生命周期(执行中 -> 待重筹办)
```

实际状态和动态已经由 #224 发布，属于场景证据，不撤销。未达成分支不得调用 `提交实际结果并完成`，不得写需求结算，也不得把非权威回执记为任务失败。

### 4.6 达成、完成与结算

目标达成后调用：

```text
任务业务服务::提交实际结果并完成
```

现有入口必须保证：

```text
任务仍是执行中，或已完成且同实际结果幂等。
唯一实际结果关系和已完成生命周期在一次不透明会话中提交。
新完成状态、当前生命周期和实际结果完整读回。
```

只有任务完成读回成立，组合器才调用：

```text
需求业务服务::提交独立结算
```

结算再次复核来源任务、实际状态、动态和可选因果引用。目标裁决仍由需求服务内部权威规则形成；组合器不传 `bool 已满足`。

### 4.7 待结算提示与重试

结算出现入口拒绝、许可拒绝或版本漂移且没有形成结算结构时，组合器返回值式提示：

```text
任务、需求、实际状态、动态证据、可选因果引用。
完成结果和结算尝试状态。
本次结算幂等主键与时间材料。
提示性质：非权威待结算请求材料。
```

提示不得写仓库、日志、事件段、快照或缓存作为业务状态。再次提交同一回执时，组合器必须先读回：

```text
任务已完成。
任务实际结果仍等于回执实际状态。
需求仍未结算。
动态证据仍完整并与实际状态一致。
```

只有上述权威条件成立才可再次调用需求结算。第一轮不承诺提示丢失后的自动重建或跨重启恢复；这属于恢复链后继合同。

### 4.8 任务结果治理路由

路由最小职责：

```text
从任务工作线程取出一条 #224 强类型回执，或接收隔离自检显式回执。
调用回执协议进行值式准入。
协议可准入后调用任务结果结算组合器。
返回强类型处理结果，不生成自我治理文本消息。
```

路由不持业务服务、数据操作、仓库或事务能力，只持任务结果结算组合器窄引用。传统 `任务管理上行桥` 保持兼容只读路径，不参与本轮写入。

## 5. 调用路径

### 5.1 正常达成

```text
任务工作线程回执队列
-> 任务结果治理路由
-> 任务结果回执协议
-> 任务结果结算组合器
-> 任务 / 需求 / 方法 / 状态 / 动态业务服务重读
-> 需求业务服务目标达成裁决
-> 任务业务服务提交实际结果并完成
-> 任务完成权威读回
-> 需求业务服务独立结算
```

### 5.2 未达成

```text
回执准入和权威重读
-> 需求业务服务裁决未达成
-> 任务业务服务迁移待重筹办
-> 返回未达成值式结果
```

### 5.3 结算重试

```text
同一回执或非权威待结算提示重新提交
-> 协议和权威材料重新读取
-> 任务完成同结果幂等读回
-> 需求仍未结算
-> 再次调用独立需求结算
```

## 6. 并发和幂等

1. 路由不持队列锁调用组合器；取出回执后释放线程内部锁。
2. 同一回执并发处理时，任务服务只允许一个完成提交，其余返回同实际结果幂等读回。
3. 需求结算并发时只允许一个提交；相同结论返回幂等读回，不同权威结论视为冲突或内部不一致。
4. 未达成并发使用同一待重筹办幂等材料；一个迁移，其余精确幂等或正常版本漂移。
5. 已完成任务收到不同实际结果时写前拒绝，不回退生命周期、不覆盖实际结果。
6. 不新增线程、`std::async`、`std::future`、工作队列或持久化重试器。

## 7. 非成功返回二分

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 协议、编号、时间或句柄形状无效 | 逻辑内返回 | 写前拒绝，定位问题材料来源 |
| 回执过期、任务正常漂移、失败 / 取消终态 | 逻辑内返回 | 不复活、不写结果或结算 |
| 已完成任务收到不同实际结果 | 逻辑内返回 | 拒绝冲突来源，不覆盖权威结果 |
| 目标材料正常缺失或许可暂不可得 | 逻辑内返回 | 结构不变化，允许重新权威复核 |
| 目标未达成 | 逻辑内分支 | 迁移待重筹办，不写最终结果 |
| 同一当前版本的任务 / 需求 / 动态结构矛盾 | 追根因解决 | 停止结果路径，不由回执补全 |
| 任务完成写后读回不一致 | 追根因解决 | 停止需求结算，不伪造完成材料 |
| 需求结算暂不可执行且未写入 | 逻辑内返回 | 保留任务完成，返回非权威提示 |
| 需求结算写后不一致或既有权威结算冲突 | 追根因解决 | 不回滚任务完成，停止后继学习和恢复产物 |
