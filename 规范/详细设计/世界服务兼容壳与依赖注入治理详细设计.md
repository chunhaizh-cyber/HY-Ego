## 2. 当前事实

S0 扫描已经确认：

```text
世界服务::创建基础信息() 兼容壳仍存在。
入口样例仍通过 世界.创建基础信息() 创建语素对应信息和概念节点。
世界服务构造函数仍持有主信息仓库、节点仓库和多个基础信息原子服务引用。
世界服务创建存在和创建场景已经改为调用存在服务和场景服务。
```

因此本轮治理重点不是立即删除兼容壳，而是先明确保留、收窄、替代和最终移除的门禁。

## 3. 兼容壳定位

`创建基础信息()` 第一轮只允许作为未拆入具体基础信息类型时的兼容入口。

允许用途：

```text
保留第一轮语素样例所需的基础信息通用节点。
承接尚未拆分到存在、场景、特征、抽象状态、抽象动态、二次特征、因果引用的过渡材料。
作为后续替代路线的扫描锚点。
```

禁止用途：

```text
作为万能基础信息写入口。
替代存在服务、场景服务、状态服务、动态服务、二次特征服务、因果服务或特征服务。
为需求、任务、方法创建未定义业务事实。
承载显示标题、日志说明、控制台文本或 SQL 行。
承载外设、体素、D455 或控制面板材料。
```

## 4. 治理路线

| 阶段 | 目标 | 代码前门禁 |
| --- | --- | --- |
| G1 保留兼容壳 | 明确当前兼容用途和调用点 | 只读扫描和详细设计 |
| G2 收窄兼容壳 | 限制调用方和节点用途 | 独立代码实施切片 |
| G3 替代调用点 | 将语素样例改为明确原子服务节点 | 独立代码实施切片 |
| G4 移除仓库裸引用 | 世界服务不再因兼容壳持有可写仓库 | 独立代码实施切片 |
| G5 废止兼容壳 | 删除或隐藏 `创建基础信息()` public 入口 | 独立代码实施切片和回归验证 |

任何阶段都不得绕过基础信息原子服务写入长期基础信息结构。

## 5. 依赖注入边界

世界服务应依赖基础信息原子服务，而不是直接依赖仓库完成业务写入。

| 依赖 | 允许用途 | 禁止用途 |
| --- | --- | --- |
| 存在服务 | 创建和读取存在 | 裸写存在节点 |
| 场景服务 | 创建场景、接纳存在、收纳实例材料 | 直接改关系仓库 |
| 特征服务 | 挂接特征、读取特征状态材料 | 直接调用特征值服务 |
| 状态服务 | 创建或读取实例 / 抽象状态 | 世界服务直接创建状态节点 |
| 动态服务 | 创建或读取实例 / 抽象动态 | 世界服务直接创建动态节点 |
| 二次特征服务 | 创建和读取二次特征组成 | 世界服务直接写组成关系 |
| 因果服务 | 记录轻量因果引用和候选材料 | 世界服务直接写因果节点 |

主信息仓库、节点仓库、关系仓库和索引仓库只允许在世界服务兼容壳尚未替代前作为受控过渡依赖；后续移除必须按独立切片验证。

## 6. 入口拒绝矩阵

| 编号 | 拒绝项 | 处理 |
| --- | --- | --- |
| BASE-WORLD-01 | 调用兼容壳创建已明确的存在、场景、特征、状态、动态、二次特征或因果节点 | 拒绝或迁移到对应原子服务入口 |
| BASE-WORLD-02 | 高级服务请求世界服务裸写业务事实 | 拒绝，要求走对应领域服务 |
| BASE-WORLD-03 | 兼容壳请求写显示、日志、SQL、控制台文本 | 拒绝，不写机器结构 |
| BASE-WORLD-04 | 兼容壳请求写 D455、体素、外设或控制面板材料 | 拒绝外部材料 |
| BASE-WORLD-05 | 世界服务依赖缺少必要原子服务 | 构造或调用拒绝，不回退到仓库裸写 |
| BASE-WORLD-06 | 原子服务写入失败 | 返回失败，不补写半结构 |

拒绝后节点、主信息、关系、索引和各领域服务结构不变化。

## 7. 验收矩阵

| 编号 | 验收项 | 通过标准 |
| --- | --- | --- |
| BASE-WORLD-V01 | 兼容用途可枚举 | 所有 `创建基础信息()` 调用点均有用途说明 |
| BASE-WORLD-V02 | 原子服务优先 | 已明确类型的基础信息写入经原子服务 |
| BASE-WORLD-V03 | 高级服务不裸写 | 需求、任务、方法不得通过世界服务写基础结构 |
| BASE-WORLD-V04 | 特征值封口 | 世界服务不得直接访问特征值服务 |
| BASE-WORLD-V05 | 依赖缺失拒绝 | 原子服务缺失时不退化为仓库裸写 |
| BASE-WORLD-V06 | 拒绝不改结构 | 兼容壳拒绝后核心结构数量不变 |
