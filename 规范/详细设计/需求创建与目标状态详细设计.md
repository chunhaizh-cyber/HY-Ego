本设计是详细设计候选，不等于代码实施许可，不自动登记 `项目记忆/Codex任务队列.md`。后续进入 C++ 实施前，必须另行生成并确认施工计划，明确允许文件、禁止文件、入口拒绝、追根因解决收口、读回验证和完成声明边界。

## 3. 设计对象

本设计覆盖以下第一轮入口和只读材料：

```text
需求服务::创建需求。
需求服务::创建目标状态需求(目标宿主, 目标特征, 特征服务, 当前特征状态材料, 状态服务, 目标状态值)。
需求服务::创建目标状态需求(需求主体, 目标宿主, 需求场景, 目标特征, 特征服务, 当前特征状态材料, 状态服务, 目标状态值)。
需求服务::读取需求目标状态。
需求服务::读取需求主体。
需求服务::读取需求目标宿主。
需求服务::读取需求场景。
需求服务::读取需求承接材料。
需求服务::结算目标状态需求，按本设计称为目标状态满足预判 / 结算预览。
任务服务::按需求创建任务 对需求承接材料的只读依赖。
```

本设计排除：

```text
父子需求、阻塞权重、逻辑组织需求、方向签名、枚举目标、方法候选准入。
结构化需求树更新指令。
权威需求结算记录写入。
完整事务回滚框架或仓库级事务实现。
旧需求主信息字段、旧任务列表字段、SQL 投影、控制面板显示、D455、体素和外设。
```

## 4. 结构承载

| 业务概念 | 项目承载结构 | 值类型 / 身份结构 | 写入方 | 读取方 | 验证方式 |
| --- | --- | --- | --- | --- | --- |
| 需求身份 | 主信息 + `节点类型::需求` | 需求节点句柄、主信息句柄 | 需求服务 | 需求服务、任务服务、显示层只读 | 需求节点可读且类型为需求 |
| 目标宿主 | 需求到存在节点的引用关系 | 存在节点句柄 | 需求服务 | 需求服务、任务服务 | 关系目标唯一且类型为存在 |
| 需求主体 | 带主体顺序号的需求到存在节点引用关系 | 存在节点句柄 | 需求服务 | 需求服务、任务服务 | 关系目标唯一且类型为存在 |
| 需求场景 | 带场景顺序号的需求到场景节点引用关系 | 场景节点句柄 | 需求服务 | 需求服务、任务服务 | 关系目标唯一且类型为场景 |
| 目标特征 | 带目标特征顺序号或简化引用的需求到特征节点引用关系 | 特征节点句柄 | 需求服务 | 需求服务、目标状态满足预判 | 关系目标存在且类型为特征 |
| 当前特征状态材料 | 需求到当前材料节点的引用关系，材料必须经特征服务读取 | 特征值节点句柄或后续确认材料 | 特征服务提供材料，需求服务写引用 | 需求服务经特征服务复读 | 特征服务可读 I64 状态材料 |
| 目标状态 | 需求到状态节点的模板关系 | 状态节点句柄、第一轮 I64 状态值材料 | 状态服务创建，需求服务写模板关系 | 需求服务、任务服务、结算预览 | 唯一模板目标为状态节点 |
| 需求承接材料 | 主体、目标宿主、场景、目标状态四项只读组合 | 四个节点句柄 | 需求服务读取组合 | 任务服务 | 四项均唯一且类型正确 |
| 目标状态满足预判 | 局部领域枚举结果 | 已满足 / 未满足 / 证据不足 | 需求服务只读计算 | 调用方只作预览 | 不写持久化结算记录 |

## 5. 请求分流

需求创建入口必须先把上游材料降权为需求创建请求，再按请求类型分流：

```text
只创建需求身份 -> 只创建需求节点和主信息，不承诺目标状态或任务承接材料。
简化目标状态需求 -> 验证目标宿主、目标特征和当前特征状态材料，创建目标状态和需求目标关系，作为兼容 / 验证入口。
完整目标状态需求 -> 验证主体、目标宿主、场景、目标特征和当前特征状态材料，创建目标状态和完整承接材料关系。
读取需求目标状态 -> 只接受有效需求节点和唯一状态模板目标。
读取需求承接材料 -> 只接受主体、目标宿主、场景和目标状态均唯一且类型正确的需求。
目标状态满足预判 -> 读取目标状态值和当前特征状态材料，只返回局部枚举，不写权威结算记录。
旧需求树 / SQL / 显示材料 -> 拒绝或转后续专项。
```

简化入口不得被任务服务当作完整承接材料来源。任务服务只能读取 `读取需求承接材料` 返回的完整材料；缺主体、目标宿主、场景或目标状态时，不得创建任务承接壳。

## 6. 入口前置拒绝

### 6.1 创建需求身份

入口和内部写入必须拒绝或收口：

```text
主信息创建不及预期。
需求节点创建不及预期。
需求节点创建后类型不可读或不是需求。
```

失败时不返回有效需求句柄，不允许调用方把主信息、日志、控制台输出或临时文本当作需求事实。

### 6.2 简化目标状态需求

写结构前必须拒绝：

```text
目标宿主无效或不是存在节点。
目标特征无效或不是特征节点。
当前特征状态材料不能经特征服务读取。
状态服务不能创建目标状态节点。
目标状态节点不可读或不是状态节点。
需求节点创建不及预期。
需求到目标宿主、目标特征、当前特征状态材料或目标状态关系写入不及预期。
```

简化入口只作为兼容 / 验证入口，不提供主体和场景，不满足任务承接完整材料。

### 6.3 完整目标状态需求

写结构前必须拒绝：

```text
需求主体无效或不是存在节点。
目标宿主无效或不是存在节点。
需求场景无效或不是场景节点。
目标特征无效或不是特征节点。
主体 / 目标宿主角色未明确。
当前特征状态材料不能经特征服务读取。
状态服务不能创建目标状态节点。
需求节点创建不及预期。
任一承接材料关系或目标状态模板关系写入不及预期。
```

流程图确认口径是“主体 / 目标宿主角色已明确”，不在设计层硬拒绝同一节点承担两个角色；如果后续仍要禁止同一节点承担两个角色，必须作为需求规范或施工计划待确认项单独裁决。

### 6.4 读取和预判

只读入口必须拒绝：

```text
需求节点无效或不是需求节点。
目标状态缺失、多目标或目标不是状态节点。
主体、目标宿主或场景缺失、多目标或类型错误。
目标特征和当前特征状态材料关系缺失。
状态服务无法读取目标状态值。
特征服务无法读取当前特征状态材料。
```

目标状态满足预判只能返回局部领域枚举，不写需求结算状态、结算记录关系、任务完成、方法成功或世界事实。

## 8. I64 和目标状态边界

需求目标的对外语义必须是目标状态节点，不是 I64。

```text
I64 可作为状态服务创建抽象状态时的第一轮状态值材料。
I64 可作为特征服务读取当前特征状态材料时的第一轮数值材料。
需求节点不得把裸 I64 写入主信息值容器来表示目标。
需求到目标状态当前使用 `关系类型::模板` 承载；不得在未确认枚举前新增专用关系类型。
目标状态满足预判中的 `当前材料 >= 目标状态值` 只是第一轮固定比较规则，不代表任意比较表达式、枚举目标或方法候选裁决已实现。
```

禁止：

```text
需求服务、任务服务或方法服务直接访问特征值服务。
调用方绕过需求服务直接写需求目标状态关系。
把目标状态满足预判结果持久化为普通 bool、任务完成、方法成功或世界事实。
把日志、控制台输出、显示标题或流程图文字当作需求状态。
```

## 9. 追根因解决收口和最小读回

所有写入路径必须进入最小读回验证。

| 路径 | 最小读回验证 |
| --- | --- |
| 创建需求身份 | 需求节点可读，节点类型为需求 |
| 简化目标状态需求 | 需求节点、目标宿主、目标特征、当前特征状态材料、目标状态和模板关系可读回 |
| 完整目标状态需求 | 需求节点、主体、目标宿主、场景、目标特征、当前特征状态材料、目标状态和模板关系可读回 |
| 读取需求目标状态 | 只读回唯一状态节点 |
| 读取需求承接材料 | 主体、目标宿主、场景和目标状态四项均可读回 |

多步写入任一环节失败时：

```text
不返回有效需求句柄或成功结果。
本次创建的目标状态、需求节点和关系不得作为有效业务事实被读取。
当前如果不能证明物理回滚，后续施工计划必须明确采用回滚、失效标记、读取隔离或停止登记事务边界问题。
不得靠日志、显示、扫描器或事后校验修复半结构。
读回不符合预期时视为逻辑错误，停止后续施工计划推进并登记断点。
```

## 12. 后续施工计划生成门禁

生成施工计划前必须先确认本详细设计，并裁决以下事项：

```text
是否在本轮处理 `需求主体 == 目标宿主` 的当前代码差距。
追根因解决收口采用回滚、失效标记、读取隔离，还是登记仓库事务边界待确认问题。
本轮是否只覆盖创建 / 读取 / 预判，不覆盖权威结算记录。
允许文件和禁止文件。
最小读回验证和拒绝后结构不变化的检查方式。
高级服务对特征值服务直接依赖扫描范围。
不修改核心仓库、工程文件、旧源码、控制面板、SQL、ADO、D455、体素或外设相关文件，除非另行确认。
```

## 13. 完成声明边界

可以声明：

```text
需求创建与目标状态详细设计已生成。
```

```text

```
## 中途非成功返回二分口径

本文件按 2026-07-09 硬规则修订：中途非成功返回只分为 `追根因解决` 和 `逻辑内返回`。

- `追根因解决`：前置条件已经满足，并进入创建、绑定、写关系、写状态、记录动态、结算、读回或结构承载后，结果不符合内部预期；必须停止依赖路径，定位根因，当前未证明完整回滚时登记事务隔离缺口或半结构隔离缺口。
- `逻辑内返回`：领域协议允许的拒绝、候选为空、请求材料返回或人读材料返回；必须保证结构不变化，且返回材料、日志、回执、显示或控制台输出不裁决机器事实。
