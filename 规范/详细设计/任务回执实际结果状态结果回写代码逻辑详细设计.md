# 任务回执实际结果状态结果回写代码逻辑详细设计

更新时间：2026-07-08

状态：已生成 / 可直接生成待确认施工计划 / 未生成施工计划 / 未入队 / 未改 C++ / 未构建 / 未迁移旧函数

## 1. 设计定位

本设计承接第 13 项 `流程图/20260708_任务回执实际结果状态结果回写代码逻辑流程图_v0.1.md`，只把当前任务服务中已经存在或可保守表达的结果回写边界整理为后续施工计划依据。

本设计不声明完整任务执行回执系统完成，不声明需求满足或需求结算完成，不生成代码实施许可。

## 2. 输入入口

```text
任务节点
任务承接材料读取入口
实际结果状态节点
发生时间戳
状态服务
动态服务相邻证据材料
任务运行统计只读请求
```

入口必须先排除无效任务、承接材料不完整、目标状态缺失、实际结果状态非状态节点、重复实际结果状态关系、缺少非零发生时间戳和把统计材料当业务裁决的请求。

## 3. 核心处理

```text
读取任务承接材料
-> 复核来源需求、任务虚拟存在、运行场景、目标状态和创建状态
-> 记录任务实际结果状态关系
-> 读取任务实际结果状态并检查唯一引用关系
-> 在已有实际结果状态和非零发生时间戳后记录任务完成生命周期状态
-> 读取动态证据相邻材料
-> 读取运行统计材料但保持非权威
```

当前设计不比较任务目标状态和实际结果状态是否满足。目标达成、需求结算和因果裁决必须转后续需求结算或因果专项。

## 4. 结构承载

```text
任务节点：任务身份和承接壳
状态节点：目标状态、实际结果状态、完成生命周期状态
关系仓库：任务到实际结果状态的引用关系、任务到生命周期状态的引用关系
动态节点：只作为相邻证据材料，不由本入口强制写入
统计材料：非权威只读计数，不承载任务完成事实
```

日志、控制台、显示文本、运行包布尔值和统计缓存不得承载任务结果事实。

## 5. 输出变化

成功路径只能输出：

```text
任务实际结果状态材料
任务完成状态材料
动态相邻证据材料
非权威运行统计材料
```

失败路径不得写实际结果状态关系、不得写完成状态、不得把半结构读成有效回执。

## 6. 当前代码差距

```text
未证明目标状态满足判定。
未证明需求结算记录写入。
未证明专门持久化回执记录结构。
动态证据尚未成为结果回写入口的强制参数。
多步写入还缺数量快照级半结构不可读验证。
```

## 7. 验收矩阵

| 验收项 | 验证方式 | 通过口径 |
| --- | --- | --- |
| 承接材料准入 | 传入无效任务或缺目标状态 | 拒绝且不写结果关系 |
| 实际结果状态写入 | 传入有效任务和状态节点 | 读回唯一顺序号 20 引用关系 |
| 重复结果拒绝 | 已有实际结果状态后再次写入 | 拒绝且原关系不变 |
| 完成状态门禁 | 缺实际结果状态或时间戳为 0 | 拒绝写完成状态 |
| 统计非权威 | 读取运行统计材料 | 不裁决任务完成、方法成功或需求满足 |
