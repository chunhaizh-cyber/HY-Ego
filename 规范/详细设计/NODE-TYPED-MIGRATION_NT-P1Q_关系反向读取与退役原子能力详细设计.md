## 2. `NT-C1Q/v0.1` 公开读取合同

仓库和值式会话视图分别提供同名只读入口：

```cpp
std::vector<正式关系记录> 读取指向有效关系组(
    节点句柄 目标节点, 关系类型 类型) const;
std::vector<正式关系记录> 读取相关有效关系组(
    节点句柄 节点, 关系类型 类型) const;
std::vector<正式关系记录> 读取有效类型角色关系组(
    关系类型 类型, std::int64_t 顺序号) const;
```

规则固定：

- 只返回当前 `记录状态::有效` 且两端节点当前有效的值式记录；
- 按目标读取要求 `目标节点` 完整相等；
- 相关读取返回源或目标命中的并集，同一自关系按关系编号只出现一次；
- 类型角色读取同时匹配关系类型与 `顺序号`，顺序号是唯一业务角色字段；
- 结果统一按 `关系编号` 升序；领域服务再按自身角色、端点、基数和当前性复判；
- 无结果返回合法空组，不用裸失败或异常表达；
- 节点无效、异域、错版本或未知关系类型在读取前拒绝；一致读取期间版本变化由领域结果映射为读取版本漂移；
- 普通调用方不得以 `读取全部关系审计` 或全仓扫描替代本合同。

会话视图必须把本会话已创建、已失效和已重挂候选叠加到取得独占许可时的已发布基线；同一查询不得同时返回同一关系的旧版本与候选当前版本。未发布候选仍只对本会话可见。

## 3. `NT-C1Q/v0.1` 原子退役合同

结构写入会话新增：

```cpp
节点直接身份带值结构写入结果<节点句柄> 删除节点候选(
    节点句柄 节点);
节点直接身份带值结构写入结果<可重建索引记录> 移除索引候选(
    const 索引物理键& 键);
```

删除节点候选只把当前有效节点推进为 `已删除` 新版本，不回收节点编号、稳定主键或版本；存在任何未纳入同一会话失效集的当前有效相关关系时写前拒绝。移除索引候选只移除可重建定位记录，不删除节点或关系事实；不存在时允许幂等读回，错所有者、错命名域或指向异义目标时拒绝。

两项候选与关系失效、领域记录退役参与者一起进入同一个既有执行器：

```text
冻结全引用
-> 关系失效候选
-> 领域记录退役参与者登记
-> 索引移除候选
-> 节点删除候选
-> 全参与者准备 / 确认
-> 最后发布
-> 权威读回
```

本设计不定义领域记录退役参与者；每个领域提供者必须向退役计划提供强类型、可移动不可复制且生命周期覆盖执行器调用的参与包。退役编排方不得直接打开私有领域记录仓。

## 4. 生命周期、错误与验证

所有返回值均值式拥有，不暴露仓库容器、锁、令牌、会话或引用。仓库对象和会话不可复制；只读查询不延长会话生命周期。

逻辑内结果：无效身份、异域、错版本、未知类型、关系仍被引用、索引不存在的幂等读回、许可竞争和第一写前漂移。

内部错误：前置通过后枚举遗漏 / 重复、候选视图同时暴露旧新版本、索引移除后仍可命中、节点删除发布后普通读取仍可见、确认 / 撤销 / 发布或权威读回不一致。

验证至少覆盖按目标、相关、自关系去重、类型角色、空组、会话候选叠加、关系未清时删除拒绝、同事务关系失效 + 索引移除 + 节点删除、逆序撤销和历史审计连续性。
