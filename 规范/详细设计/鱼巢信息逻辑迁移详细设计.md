# 鱼巢信息逻辑迁移详细设计

日期：2026-07-10

状态：需按正式规范母本重审 / 鱼巢正式规范业务逻辑不再是候选 / 既有隔离样本设计只作历史候选 / 未改 C++ / 未写 SQL Server

## 1. 目标

在没有旧数据库和可恢复旧快照的前提下，把 `D:/鱼巢` 的正式规范业务逻辑和其它信息 / 实现证据分别处理：正式业务逻辑必须迁入，旧数据结构和静态定义只转成可审查、可分批、可读回的结构候选。

本设计解决的是：

```text
鱼巢规范目录登记的正式业务逻辑
-> 必须继承并完成海中鱼巣结构映射
鱼巢初始化定义、静态定义表和旧服务实现证据
-> 结构迁移候选
-> 新项目口径分类
-> 隔离候选仓库结构样本
-> 证据矩阵
-> 后续独立确认的运行期导入、快照和数据库持久化
```

本设计不解决：

```text
旧数据库导入
旧运行实例恢复
旧指针或旧节点编号复用
旧函数体复制
旧字段原样落库
运行期仓库替换
SQL Server 全量快照写入或恢复
```

## 3. 双轴权力顺序

业务逻辑按以下顺序裁决：

```text
用户明确硬裁决
-> D:/鱼巢/规范/规范目录.md
-> 0050 总纲
-> 对应根规范
-> 对应正式子规范
-> 0300 已登记冲突裁决
```

海中鱼巣 AGENTS.md、正式规范和当前代码负责结构、工程和实施边界，只能决定如何承载上述业务逻辑，不能改变流程、顺序、条件、状态和失败语义。

鱼巢当前代码定义和调用事实、说明书、工程文档、实施记录、计划和日志，以及海中鱼巣既有 FS 扫描、流程图和详细设计，只能补充结构映射和当前事实。它们不得把鱼巢正式规范降为候选，也不得反向改写正式业务逻辑。

同级鱼巢正式规范相互冲突且 0300 未裁决时，必须登记源冲突并冻结受影响分支，不得用海中鱼巣当前实现静默选边。

## 5. 分类规则

| 来源类别 | 默认处理 | 说明 |
| --- | --- | --- |
| 节点和关系语义 | 改写 | 映射到新节点类型、关系类型和领域服务，不复制旧链表结构 |
| 静态特征定义词 | 待裁决后保留或改写 | 先判断是长期特征类型、状态、运行材料还是人读标签 |
| 自检、线程、消息和运行包字段 | 默认暂缓或改写 | 优先归入运行消息、验收证据或非权威材料，不批量建立特征节点 |
| 默认本能方法种子 | 服务级拆分 | 只迁移已确认的方法结构和动作入口语义，不迁移函数指针表 |
| 初始因果模板 | 默认待裁决 | 必须降为候选或轻量因果证据，不导入稳定因果结论 |
| D455、体素、外设定义 | 暂缓 | 保留证据，不进入第一轮迁移 |
| 日志、缓存、显示文本 | 拒绝成为事实 | 只作人读证据 |
| 旧运行实例和旧编号 | 拒绝 | 当前无可信快照，也不复用旧身份 |

## 6. 第一轮最小样本

`INFO-MIG-S1` 只选择两个来源稳定、无外设依赖且符合当前语素服务三字符上限的特征定义候选：

| 候选 | 旧证据 | 旧入口类别 | 目标候选 |
| --- | --- | --- | --- |
| 安全值 | `D:/鱼巢/自我类.特征定义.cpp:170` | 特征模板入口 | 特征节点 + 概念语素入口 + 对应信息关系 + 概念追溯关系 + 新项目稳定主键 |
| 服务值 | `D:/鱼巢/自我类.特征定义.cpp:171` | 特征模板入口 | 特征节点 + 概念语素入口 + 对应信息关系 + 概念追溯关系 + 新项目稳定主键 |

本批只迁移“定义身份”，不迁移任何当前值、历史值、需求满足结论、任务状态或因果结论。`物理安全` 等四字符以上词面不绕过当前语素准入上限，转入后续语素长度治理。

### 6.1 构建顺序

```text
读取已确认的两项清单
-> 检查来源指纹和显式稳定键
-> 新建隔离主信息仓库、节点仓库、关系仓库和索引仓库
-> 构造特征服务、语素服务及必要依赖
-> 创建特征类型
-> 通过语素服务创建概念入口，以同一特征节点作为对应信息和概念追溯目标，并绑定显式稳定键
-> 从索引读取语素入口
-> 从语素入口读取唯一特征目标
-> 从语素入口读取唯一概念追溯目标并确认仍为同一特征节点
-> 核对节点类型、关系和结构数量
-> 生成证据矩阵
-> 丢弃隔离仓库，不替换运行期仓库
```

### 6.2 稳定键边界

第一轮稳定键必须满足：

```text
由新项目显式分配，不复用旧枚举值、指针或旧数据库编号。
不得只依赖文本哈希生成。
同一清单版本内唯一。
重复执行时按稳定键先读回并核对类型和关系，不重复创建。
稳定键命名空间需在 INFO-MIG-Q1 确认后才可进入代码。
```

## 7. 隔离和失败收口

### 7.1 逻辑内返回

以下情况在任何结构写入前返回：

```text
迁移批次为空
候选仍为待裁决
来源指纹不匹配
稳定键为零或批次内冲突
文本不符合语素最小词单元准入
目标服务或依赖未提供
```

### 7.2 追根因解决

入口前置条件已通过并开始构建隔离候选仓库后，出现以下情况必须视为逻辑错误：

```text
特征节点创建不成功
语素入口、对应信息关系或概念追溯关系创建不成功
稳定键绑定后无法读回
读回节点类型或目标关系不符合计划
结构数量与预期不一致
重复执行产生额外结构
```

处理方式：

```text
记录独立逻辑错误
整套丢弃隔离候选仓库
不继续后续候选
不修改运行期仓库
不写 SQL Server
修复根因后重新执行完整批次
```

## 8. 后续运行期导入边界

隔离样本通过不等于已经完成迁移。后续运行期导入必须另建计划，至少补齐：

```text
已确认定义清单
稳定键命名空间
完整批次预检
原子导入或隔离仓库替换策略
幂等、版本和冲突拒绝
失败后运行期仓库不变化
快照材料化
恢复拒绝矩阵
数据库快照和事件日志适配
```

数据库只能接收新项目已经读回验证的快照或事件审计材料，不得用 SQL 记录反向裁决运行期机器事实。
