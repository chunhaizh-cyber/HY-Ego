## 2. 定位

实例状态是场景内临时零散节点，必须有存在主体、场景归属、发生时间戳和值材料。实例状态生命周期治理的目标是防止失效状态继续作为有效状态材料、动态证据、二次特征组成项或因果候选证据被读取。

生命周期治理不是：

```text
事后扫描修复器。
日志驱动清理器。
显示层过滤器。
SQL 删除同步。
旧状态删除函数体迁移。
```

## 3. 生命周期状态口径

第一轮设计只区分读取准入，不新增状态机字段。

| 口径 | 含义 | 第一轮处理 |
| --- | --- | --- |
| 有效实例状态 | 节点有效、类型正确、主体 / 场景 / 值 / 时间戳完整 | 可作为状态材料读取 |
| 失效实例状态候选 | 后续删除或失效入口准备处理的状态 | 写入口未实现，只登记拒绝规则 |
| 被引用实例状态 | 已被动态、二次特征、因果候选或结算证据引用 | 不允许直接删除成悬空引用 |
| 不完整实例状态 | 缺主体、场景、值或时间戳 | 读取拒绝，不作为有效材料 |
| 已删除旧句柄 | 节点仓库生命周期已失效 | 读取拒绝，旧句柄不得复用 |

若后续需要持久化生命周期状态、删除原因或失效版本，必须另建结构设计，不得把普通 bool 或日志文本作为一阶机器事实。

## 4. 引用阻断边界

删除或失效实例状态前必须复核引用关系。

| 引用来源 | 阻断原因 | 第一轮处理 |
| --- | --- | --- |
| 实例动态 | 动态前后值或目标状态仍需可追溯 | 有引用时拒绝删除写入口 |
| 二次特征 | 组成项需要可追溯状态材料 | 有组成引用时拒绝删除写入口 |
| 轻量因果引用或因果候选 | 动态证据和状态端点需要可追溯 | 有候选引用时拒绝删除写入口 |
| 需求结算记录 | 目标 / 实际状态证据需要可追溯 | 有结算引用时拒绝删除写入口 |
| 任务目标或结果状态 | 任务生命周期和结果读取需要可追溯 | 有任务引用时拒绝删除写入口 |
| 方法条件或结果状态 | 方法条件 / 结果对需要可追溯 | 有方法引用时拒绝删除写入口 |

第一轮不自动级联删除。任何级联清理必须另建专项，明确关系类型、清理顺序、失败回滚和结构不变化验收。

## 5. 读取拒绝边界

状态服务读取实例状态材料时必须先判断：

```text
状态句柄有效。
节点类型是实例状态。
节点未删除。
存在主体有效。
场景归属有效。
状态值存在。
发生时间戳存在。
未处于后续确认的失效状态。
```

拒绝后：

```text
返回空材料或明确拒绝结果。
不补写主体关系。
不补写场景关系。
不补写状态值。
不补写发生时间戳。
不写日志或显示文本作为状态事实。
```

## 6. 关系清理边界

后续如果确认删除写入口，关系清理只能处理本状态作为源或目标的已确认关系类型，并且必须在入口前置复核所有阻断引用。

允许清理的候选关系：

```text
状态到场景的临时归属关系。
状态到存在主体的引用关系。
状态到抽象状态的提升 / 引用关系，前提是未作为长期证据使用。
状态服务明确拥有的状态辅助关系。
```

禁止清理：

```text
动态证据关系，除非动态删除专项确认。
二次特征组成关系，除非二次特征生命周期专项确认。
因果证据候选关系，除非因果候选专项确认。
需求、任务、方法、结算记录关系，除非对应高级服务专项确认。
核心仓库中不明来源关系。
```

## 8. 结构不变化要求

任何读取拒绝或删除拒绝后，以下结构必须不变化：

```text
节点仓库数量。
主信息仓库数量。
关系仓库数量。
索引仓库数量。
状态节点有效性。
动态节点集合。
二次特征组成关系。
因果引用节点。
需求、任务、方法和结算关系。
```

若后续删除写入口确认，成功删除的结构变化必须限定在已确认允许清理的状态自身和状态服务拥有关系内，并记录旧句柄失效，不得让旧句柄读成新状态。
