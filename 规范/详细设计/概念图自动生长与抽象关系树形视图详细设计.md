## 2. 设计目标

本设计建立统一概念图协议，正式覆盖：

```text
存在概念自动生成。
动态概念自动生成。
关系概念自动生成，关系由二次特征承载。
因果概念自动生成。
向上聚类和向下抽象的双向固定点计算。
概念图到抽象关系树形视图的只读投影。
类型必填、名称可空、命名请求和临时指代。
一般用途、因果预测用途和因果行动用途的分开观察。
活跃、冷却、退役和物理删除生命周期。
新增或删除概念后的上下层重新收敛。
```

本设计不直接许可 C++ 修改。第一份后续计划只执行当前代码事实扫描，代码实施必须根据扫描结果拆分独立计划。

## 3. 核心定义

### 3.1 概念

概念是一个可被结构化描述、比较、复用和表达的信息团。自然语言名称只是概念的表达入口，不是概念本体，也不参与概念身份判断。

概念节点必须满足：

```text
具有有效稳定节点句柄。
属于存在、动态、关系、因果四类之一。
除四个根概念外，至少具有一个同类上位概念。
定义材料由结构化特征、抽象状态、二次特征或概念关系承载。
名称可以为空，也可以有多个语素名称。
```

### 3.2 概念图

概念图是由概念节点、实例支持关系、概念定义材料关系和概念上下位关系组成的权威逻辑图。概念图由关系仓库承载，索引只加速候选召回。

概念图不是：

```text
普通父子树。
语素树。
名称字典。
控制面板树形控件。
内容哈希表。
日志或 SQL 投影。
```

### 3.3 抽象树

“抽象树”固定解释为“抽象关系树形视图”。它从选定根概念沿概念上下位关系展开：

```text
图中一个概念允许有多个上位概念。
树形视图可以在不同路径重复投影同一概念句柄。
树形视图只读，不创建、删除、重挂或裁决概念关系。
```

### 3.4 实例信息团

实例信息团是已经由对应原子服务完成准入、写入和读回的实例及其结构化材料。裸节点、日志文本、自然语言说明、控制台输出、SQL 行、线程事件或外设原始材料均不构成已完成实例信息团。

### 3.5 种子概念

种子概念是实例完成后首先确保命中的概念。等价签名已存在时必须复用，不得为每个实例无条件复制概念节点。

当实例没有可区分定义材料时，直接命中对应根概念，不额外创建与根等价的空定义子概念。

## 5. 四根与现有节点类型映射

本设计不新增统一的 `概念` 节点类型，也不新增 `高级信息` 节点类型。

| 概念根 | 概念节点使用的现有节点类型 | 根节点来源 | 普通概念来源 |
| --- | --- | --- | --- |
| 存在 | `节点类型::存在` | 初始化阶段由存在服务创建并永久保留 | 存在实例的特征和关系材料 |
| 动态 | `节点类型::动态`，且不是运行期临时动态 | 初始化阶段由动态服务创建并永久保留 | 完整实例动态及其抽象状态、路径和时间类别材料 |
| 关系 | `节点类型::二次特征` | 初始化阶段由二次特征服务创建并永久保留 | 具体关系二次特征的角色、参与者和关系值材料 |
| 因果 | `节点类型::因果引用` | 初始化阶段由因果服务创建并永久保留 | 因果实例和来源动态的临时准入材料 |

四个根概念是唯一允许无普通实例支持的概念种子。它们必须预置名称、永久有效且禁止淘汰。

四个根概念还必须各自具有不依赖自然语言名称的幂等初始化身份。逻辑承载固定为“四类根类别到四个稳定节点句柄”的机器结构，由概念图服务独占登记和读取、由对应原子服务创建节点；名称只能在根句柄成立后经语素关系绑定。重复初始化同一根类别必须复用当前有效根句柄，不得因为名称相同、名称变化或名称缺失重新创建根节点。

CONCEPT-S0 必须先确认当前初始化、稳定句柄恢复和唯一性门禁中可复用的正式入口；CONCEPT-S1 再把实际承载位置、写入方、读取方、恢复方式和读回规则固定到规范与接口。无论最终复用哪一现有入口，都不得用显示文本、日志、内容哈希或名称查重冒充根身份。

### 5.1 场景、特征和状态的归属

```text
场景：实例发生和适用上下文，不是第五个概念根。
特征：概念定义维度和匹配材料，不是独立概念根。
特征值：只经特征服务读取，不能被概念图服务直接访问特征值服务。
实例状态：场景内临时证据，不进入概念图作为概念节点。
抽象状态：长期可复用的定义或条件材料，不是第五个概念根。
```

抽象状态和抽象动态仍保持现有长期基础信息身份。动态概念属于动态根；抽象状态作为存在、动态、关系和因果概念可引用的结构化材料。

## 6. 类型与名称

### 6.1 类型不能为空

概念类型分两层：

```text
结构类别：由节点类型确定属于存在、动态、关系或因果。
概念类型：由一个或多个直接上位概念句柄确定。
```

除根概念外，任何概念发布前都必须至少有一个同类上位概念。上位概念的节点身份就是下位概念的概念类型；不得把上位概念的自然语言名称复制为类型字符串。

### 6.2 名称可以为空

名称继续由语素服务和既有语素关系承载：

```text
语素对应信息：语素入口绑定已存在概念节点。
语素概念追溯：语素入口追溯一个或多个概念节点。
```

不得新增概念名称字段，不得要求概念创建函数接收非空名称，也不得用名称查重代替概念签名比较。

### 6.3 预置名称

预置名称分为两类：

```text
根名称：存在、动态、关系、因果，随根概念初始化正式绑定。
普通名称候选：只保存签名匹配规则或命名材料；等实例概念真实生成后再绑定。
```

普通名称候选不能单独创建世界概念。虚构、计划、假设或语言学习材料必须先在明确的虚拟场景形成实例信息团。

### 6.4 无名概念与指代

概念无名称时，概念图服务只输出结构化待命名请求材料，不直接创建需求节点。自我治理读取该材料后，才可经需求服务创建命名需求，避免基础信息服务反向依赖高级服务。

表达时由语言上下文生成临时指代：

```text
输入：目标概念句柄、当前表达上下文、可见候选概念组。
输出：只在本次表达上下文有效的代词或指示表达。
禁止：把临时代词持久化为概念名称、稳定指代对象或世界事实。
```

## 7. 概念图专用关系

当前 `关系类型` 只有 0 到 8，没有概念图权威关系。本设计要求后续规范和代码只能追加新枚举值，不得复用、改义或插入既有值。

| 建议数值 | 关系类型 | 方向 | 多目标 | 生命周期 | 写入方 |
| --- | --- | --- | --- | --- | --- |
| 9 | `实例支持概念` | 实例节点 -> 概念节点 | 是 | 从实例支持成立到实例失效或关系撤销 | 概念图服务 |
| 10 | `概念上下位` | 上位概念 -> 下位概念 | 是，允许下位多父 | 与概念图版本一致 | 概念图服务 |
| 11 | `概念定义材料` | 概念节点 -> 特征、抽象状态、二次特征或相关概念材料 | 是 | 与概念定义版本一致 | 概念图服务经对应服务复核 |

### 7.1 `实例支持概念`

```text
同一实例可以支持多个概念。
同一概念可以由多个实例支持。
完全相同的源、目标关系禁止重复。
实例与概念必须属于兼容类别。
实例失效后不得继续作为有效支持证据读取。
```

旧项目 `概念集` 只作为多概念行为证据，不迁移为主信息向量或裸指针集合。

### 7.2 `概念上下位`

```text
源节点必须比目标节点更一般。
源、目标必须属于同一根类别。
允许一个下位概念有多个直接上位概念。
禁止自环和任何间接环。
只保存直接覆盖关系；存在中间概念时不得同时保留传递冗余边。
```

该关系不能复用 `普通父子`。普通父子当前强制单父，而概念图已确认必须支持多父。

### 7.3 `概念定义材料`

定义材料关系只连接结构化材料，不连接名称、日志、显示或 SQL：

```text
允许目标：特征、抽象状态、长期抽象动态、二次特征、因果引用或同类概念材料。
禁止目标：实例时间、原始场景句柄、日志节点、控制面板材料、SQL 行、线程消息文本。
```

内容哈希和索引命中只作候选召回，不替代该关系。

## 8. 概念签名

概念签名是从权威定义关系和对应值材料派生的规范化只读材料，用于等价判断、偏序比较和候选召回。

建议运行期结构：

```text
结构_概念签名材料
  概念类别
  签名版本
  特征约束组
  抽象状态约束组
  二次特征约束组
  角色约束组
  上下文约束组
```

签名字段必须按稳定句柄、约束类型和值域规范排序。名称、实例句柄、原始场景句柄、单次时间戳、概率、命中次数、稳定度、活跃度、好用度和内容哈希均不得进入概念身份。

### 8.1 权威性

```text
权威：概念节点、概念定义材料关系、概念上下位关系及对应领域值结构。
派生：规范化签名材料。
非权威：签名内容哈希、候选索引、命中缓存和相似度排序。
```

索引未命中时必须允许回退到领域服务的图读取和签名比较；不得因缓存丢失改变概念事实。

### 8.2 等价

两个概念签名只有在以下条件全部相同时才等价：

```text
概念类别相同。
签名版本兼容。
规范化特征约束相同。
抽象状态、二次特征、角色和上下文约束相同。
```

自然语言名称相同不构成概念等价；名称不同也不阻止概念等价合并。

## 9. 四类概念形成材料

### 9.1 存在概念

| 输入 | 进入签名 | 不进入签名 |
| --- | --- | --- |
| 存在实例 | 节点类别、经特征服务读取的特征约束、关系二次特征约束、抽象上下文 | 实例句柄、名称、原始场景句柄、观察时间 |

没有可区分材料时，存在实例只支持“存在”根概念。后续特征或关系材料补齐后，可再次触发更细种子概念生成。

### 9.2 动态概念

| 输入 | 进入签名 | 不进入签名 |
| --- | --- | --- |
| 完整实例动态 | 主体所属概念、被改变目标类型、前后抽象状态、过程路径、时间类别、来源动作概念材料 | 具体主体、原始场景、单次发生时间、函数名、线程名 |

实例时间只能转为持续时间、频率、节奏等抽象二次特征，不得原样写入动态概念。

### 9.3 关系概念

| 输入 | 进入签名 | 不进入签名 |
| --- | --- | --- |
| 具体二次特征 | 关系角色、参与者概念、方向、关系值或区间、适用抽象上下文 | 具体参与实例句柄、显示名称、单次时间 |

结构组成、物理部分关系和普通父子关系不得因为都叫“关系”而自动进入关系概念。关系概念只承接二次特征服务明确准入的语义关系。

### 9.4 因果概念

| 输入 | 进入签名 | 不进入签名 |
| --- | --- | --- |
| 因果实例和来源动态临时材料 | 条件概念或抽象状态、动态概念、结果概念或抽象状态、适用上下文 | 来源动态持久关系、单次实例时间、可靠性门槛、成功率、名称 |

来源动态只作为因果实例和因果概念生成入口的临时准入证据，不新增 `因果概念 -> 来源动态` 或 `因果实例 -> 来源动态` 持久关系。因果实例可以通过 `实例支持概念` 支持因果概念。

概率、预测命中率和行动收益属于使用材料，不参与因果概念身份，也不阻止因果概念创建。

## 10. 第一版匹配规则

| 材料 | 身份匹配 | 候选排序 |
| --- | --- | --- |
| 离散值 | 精确相等 | 可按命中维度数量排序 |
| 数值 | 已确认区间的包含、相等或相交规则 | 可按区间重叠度排序 |
| 关系 | 关系角色、方向、参与概念签名精确匹配 | 可按已匹配角色数量排序 |
| 上下文 | 抽象上下文约束匹配 | 可按适用范围排序 |
| 模糊相似度 | 不裁决身份 | 只生成候选顺序 |

若特征系统尚未提供已确认区间，第一版退化为精确数值约束；概念图服务不得自行发明区间阈值。

## 11. 聚类、抽象与偏序

### 11.1 一般性顺序

对同类概念签名定义“更一般”偏序：

```text
约束更少的签名更一般。
数值区间包含另一区间时，包含方更一般。
参与概念更一般、其他角色相同的关系签名更一般。
条件更少或条件概念更一般的因果签名更一般。
不同根类别之间不可比较。
```

根概念的定义约束为空，是该类别最一般节点。

### 11.2 向上聚类

向上聚类从新种子概念与同一局部候选组提取最小共同一般签名：

```text
离散约束相同则保留，不同则移除该维度。
数值区间使用已有合法区间中的最小共同包含区间。
关系角色相同则保留，参与概念提升到共同上位概念。
没有共同可区分约束时只回到根，不创建空定义中间概念。
```

概念创建不要求最小样本数。样本数只影响后续召回排序和生命周期，不作为创建门槛。

### 11.3 向下抽象

向下抽象根据支持实例之间可区分的特征或二次特征约束生成更细签名：

```text
优先使用已确认、可比较、可追溯的定义维度。
维度选择按可区分程度排序；并列时按特征或关系稳定句柄排序。
每个生成签名必须至少由一个有效实例支持。
与已有签名等价时复用，不复制节点。
```

第一版不使用不可解释的向量距离直接创建概念身份。模糊相似度只能把候选送入上述确定性复核。

### 11.4 直接上下位关系

若概念 A 比概念 B 更一般，并且不存在概念 C 满足 `A 比 C 更一般` 且 `C 比 B 更一般`，则写入 `A -> B` 的直接 `概念上下位` 关系。

这等价于对当前签名偏序图保存传递约简。多父关系自然保留，自环和间接环由偏序入口拒绝。

## 12. 双向固定点算法

### 12.1 受影响分量

新增、合并、退役或删除概念后，只沿同类 `概念上下位` 关系向上和向下读取连通分量。以下关系不扩张重算边界：

```text
实例支持概念。
概念定义材料。
普通引用。
语素对应信息。
语素概念追溯。
需求、任务、方法关系。
```

它们可以作为材料复核或删除引用门禁，但不把不同概念类别合并成一个重算分量。

### 12.2 迭代顺序

```text
1. 规范化本轮新增或变化签名。
2. 合并等价签名候选。
3. 向上生成共同一般签名。
4. 向下生成可区分签名。
5. 重新计算直接上下位关系。
6. 比较本轮前后的规范签名集合和直接边集合。
7. 未稳定则按稳定句柄顺序继续下一轮。
```

### 12.3 收敛条件

固定点成立必须同时满足：

```text
本轮未生成新的规范签名。
本轮未合并新的等价概念。
直接上下位边集合不再变化。
所有非根概念至少具有一个直接上位概念。
所有概念可追溯到且只属于一个根类别。
不存在自环或间接环。
```

### 12.4 终止和预算

签名生成只能使用当前有限实例材料、已有合法区间和已有概念关系，因此算法输入域必须有限。实现仍应设置单轮候选数量和迭代预算，防止异常数据导致无界计算。

预算耗尽时：

```text
不得发布未收敛的候选图版本。
输出结构化待重算请求材料。
线程只负责调度该材料，概念图服务仍是实际计算和写入方。
该路径属于逻辑内返回，不得写日志文本代替待重算材料。
```

## 13. 服务边界

### 13.1 概念图服务

后续建议新增 `概念图服务`，定位为基础信息聚合服务，不新增节点仓库或独立身份体系。

职责：

```text
接收四类实例完成材料。
经对应原子服务读取实例和定义材料。
构造、比较和读取概念签名。
调用对应原子服务创建长期概念节点。
独占写入实例支持概念、概念上下位和概念定义材料关系。
计算受影响分量和固定点候选图。
在已确认失败收口边界内发布候选图版本。
返回抽象关系树形只读视图。
输出待命名、待重算和生命周期复核请求材料。
```

禁止：

```text
直接访问特征值服务。
直接创建需求、任务或方法。
用线程、日志、控制台或显示作为概念来源。
绕过原子服务直接伪造存在、动态、二次特征或因果节点。
```

新增该服务前必须同步修订基础信息服务分层规范、仓库逻辑空间规范和领域服务授权写入规范。

### 13.2 原子服务

| 服务 | 新增职责候选 | 保持边界 |
| --- | --- | --- |
| 存在服务 | 创建存在根和存在概念节点；读取存在实例材料 | 不按名称创建概念 |
| 特征服务 | 返回概念签名所需的结构化特征约束 | 概念图服务不得直接访问特征值服务 |
| 状态服务 | 提供抽象状态和实例状态材料读取 | 实例时间不进入概念本体 |
| 动态服务 | 创建动态根和长期动态概念；返回完整实例动态材料 | 线程和函数名不是动作来源 |
| 二次特征服务 | 创建关系根和关系概念；返回角色及组成材料 | 不把普通结构关系混为二次特征 |
| 因果服务 | 创建因果根和因果概念；复核来源动态临时准入材料 | 不持久化来源动态关系，不产生绝对因果结论 |

### 13.3 世界服务与自我治理

世界服务可负责聚合创建实例信息团，但不得直接写概念图专用关系。实例完成材料由明确聚合入口提交给概念图服务。

自我治理可以承接：

```text
待命名请求材料。
待重算请求材料。
周期生命周期复核请求。
```

自我线程和消息队列只搬运请求。命名需求必须由自我治理经需求服务正式创建，不能由概念图服务反向写高级结构。

### 13.4 语素和显示

语素服务只绑定已存在概念节点，并返回多名称、多对应和多路追溯结果。显示服务和控制面板只读取抽象树投影，不反写概念图。

## 14. 实例完成触发

四类自动触发点：

| 类型 | 完成条件 | 概念生成请求 |
| --- | --- | --- |
| 存在 | 存在有效，当前可用特征和二次特征材料已形成一致快照 | 存在实例完成材料 |
| 动态 | 主体、场景、目标、前后值、发生时间戳和可选来源动作完整 | 动态实例完成材料 |
| 关系 | 二次特征角色、组成项和可选关系值完整 | 关系实例完成材料 |
| 因果 | 因果实例有效，来源动态仅在调用期通过完整性复核 | 因果实例完成材料 |

实例节点必须先生成，概念图请求随后生成。概念图发布失败不得把无效概念半结构解释为实例事实，也不得删除已经有效的世界实例冒充回滚。

当前代码没有完整跨仓库回滚。首轮代码计划必须在以下方式中选择并验证一种：

```text
真实事务回滚。
候选图版本整体发布。
显式失效隔离并由公开读取入口拒绝半结构。
```

在没有证明之前，只允许读取和计算候选，不允许宣称完整概念图自动发布完成。

## 15. 因果的好用度

因果概念不设置“可靠世界事实”创建门槛。使用评价分为：

| 用途 | 输入 | 结果材料 | 评价方向 |
| --- | --- | --- | --- |
| 预测用途 | 条件和因果概念 | 预测结果与后续实际结果 | 是否提高预测收益、降低遗漏或误判成本 |
| 行动用途 | 条件、因果概念和候选动作 | 选择动作后的实际结果与副作用 | 是否提高目标推进、降低风险和代价 |

预测好用不自动等于行动好用，观察相关模式也可以作为预测工具，但不能未经行动证据直接取得同等行动排序。

概率、次数和评分应先作为可重算的非权威统计材料。真正的任务选择、方法选择和行动请求仍由任务、方法和自我治理结构裁决。

## 16. 生命周期与删除

### 16.1 生命周期状态

概念生命周期至少包括：

```text
活跃。
冷却。
退役。
已删除，由节点生命周期失效表达。
```

不得使用持久化 bool 表示概念是否有效或是否应删除。活跃、冷却和退役必须由正式状态或特征状态材料承载；命中次数和最近使用时间可作为可重算统计候选。

既有 BASE-D2 的“候选、稳定、冷却、待淘汰”在本设计中不再作为概念能否存在的升格门槛，而解释为召回和生命周期治理阶段。

### 16.2 有用性复核

以下材料说明概念仍有用：

```text
仍有有效实例支持。
仍被其他概念作为不可替代定义材料使用。
仍被需求、任务、方法、语素或表达入口引用。
仍有独立区分价值。
因果概念仍在预测或行动候选中产生正向用途。
```

非权威统计只能提出冷却或退役候选。生命周期服务必须重新读取权威支持关系和外部引用后才能改变生命周期状态。

### 16.3 冷却与退役

```text
长期未命中或无独立价值 -> 冷却候选。
冷却期内重新获得实例支持或有效使用 -> 重新激活。
冷却期结束且权威复核仍无用 -> 退役。
退役节点不进入主动召回，但仍可读、可被重新激活。
```

保留退役阶段本身就是重生抖动缓冲。节点物理删除后，新的有效实例允许重新创建等价概念；历史审计记录不得阻止正常重建。

### 16.4 物理删除前置条件

必须全部满足：

```text
不是四个根概念。
当前生命周期为退役。
没有有效实例支持关系。
没有需求、任务、方法、语素或其他概念的不可替代引用。
所有直接下位概念均可重挂到兼容的直接上位概念。
候选重挂后仍同类、根可达且无环。
已有可证明的事务、补偿或失效隔离边界。
```

### 16.5 删除顺序

```text
1. 在候选图版本中计算子节点重挂。
2. 运行上下双向固定点计算并验证候选图。
3. 失效实例支持、定义材料和上下位关系。
4. 失效概念节点句柄。
5. 发布删除后的概念图版本。
6. 读回旧句柄失效、无孤儿、无环和上下层稳定。
```

当前事务壳没有真实回滚，故首轮实现不得直接进入物理删除。可以先实现冷却、退役和删除候选读取；物理删除必须由后置依赖门控计划承接。

## 17. 只读抽象树视图

建议返回结构：

```text
结构_抽象树视图项
  概念句柄
  根类别
  路径上位概念组
  当前深度
  是否在其他路径重复投影
```

```text
从一个已确认根概念开始。

多父节点按每条有效路径投影，不复制概念事实。
名称为空时返回空名称组，不伪造名称。
```

| 编号 | 场景 | 分类 | 结构结果 |
| --- | --- | --- | --- |
| CONCEPT-R01 | 实例句柄无效或类型不符 | 逻辑内返回 | 不创建概念和关系 |
| CONCEPT-R02 | 实例来源材料不完整 | 逻辑内返回 | 不创建概念和关系 |
| CONCEPT-R03 | 没有可区分定义材料 | 正常成功 | 只支持对应根概念 |
| CONCEPT-R04 | 等价签名已存在 | 正常成功 | 复用概念，只确保支持关系 |
| CONCEPT-R05 | 请求用名称、哈希或相似度直接裁决身份 | 逻辑内返回 | 不写概念图 |
| CONCEPT-R06 | 上下位关系跨类别、自环或成环 | 逻辑内返回，写入前拒绝 | 当前图不变化 |
| CONCEPT-R07 | 固定点预算耗尽 | 逻辑内返回 | 活动图不发布候选变化，只返回待重算材料 |
| CONCEPT-R08 | 前置通过后节点或关系写入不及预期 | 追根因解决 | 停止依赖路径，回滚、失效或读取隔离半结构 |
| CONCEPT-R09 | 候选图内部出现环、孤儿或跨根 | 追根因解决 | 禁止发布候选图 |
| CONCEPT-R10 | 概念仍有支持或外部引用 | 逻辑内返回 | 保持退役，不物理删除 |
| CONCEPT-R11 | 物理删除缺少事务或隔离证明 | 逻辑内返回 | 保持退役，不冒充删除成功 |
| CONCEPT-R12 | 删除后读回不一致 | 追根因解决 | 停止依赖路径并恢复或隔离候选版本 |

## 20. 当前代码事实与缺口

| 当前事实 | 设计影响 |
| --- | --- |
| 当前没有 `概念图服务` | 必须先做服务归属和依赖扫描 |

| 关系仓库缺少按目标枚举来源关系的通用读取 | 上下双向遍历和删除引用复核需要补只读能力或专用服务入口 |
| 存在服务没有概念 / 实例区分和概念生成入口 | 存在概念路径尚未实现 |
| 状态、动态提升每次创建新节点并使用普通 `引用` | 没有签名复用、显式支持关系或固定点重算 |
| 二次特征服务只支持组合和状态迁移组成项 | 没有关系概念签名、上下位和区间抽象 |
| 因果服务只创建轻量因果引用，不保存来源动态关系 | 因果概念生成必须在调用期保留临时来源动态材料 |
| 语素服务已有多对应和多路概念追溯 | 可复用为命名绑定，不应新增名称字段 |
| 写入事务只有状态壳，没有真实回滚 | 候选图整体发布和物理删除必须后置门控 |
| 当前没有概念用途和生命周期正式结构 | 先设计候选读取和退役，再进入物理删除 |

```text
CONCEPT-S0：当前代码事实扫描，锁定关系、服务、事务、索引、语素和自我治理接口。
CONCEPT-S1：修订逻辑空间、服务分层、授权写入和枚举规范；设计三类关系与概念图服务入口。
CONCEPT-S2：四根初始化、存在种子概念和实例支持关系第一轮。
CONCEPT-S3：动态、关系、因果种子概念路径；继续保持来源动态不持久化。
CONCEPT-S4：规范签名、等价复用、上下位偏序、受影响分量和固定点只读候选。
CONCEPT-S5：候选图版本发布和抽象关系树形只读视图。
CONCEPT-S6：待命名请求、上下文指代、预测 / 行动用途候选和生命周期状态。
CONCEPT-S7：冷却、退役和重新激活。
CONCEPT-S8：在事务、补偿或失效隔离正式产物满足后，实现安全物理删除和删除后固定点。
```

## 22. 验收矩阵

| 编号 | 验收项 | 通过标准 |
| --- | --- | --- |
| CONCEPT-A01 | 四根唯一 | 四根可读、类型正确、名称已绑定、禁止淘汰 |
| CONCEPT-A02 | 类型必填 | 每个非根概念至少一个同类直接上位概念 |
| CONCEPT-A03 | 名称可空 | 无名概念可有效存在，创建入口不要求名称 |
| CONCEPT-A04 | 实例优先 | 实例信息团先完成，随后自动提交概念生成请求 |
| CONCEPT-A05 | 等价复用 | 等价签名不新增概念节点，只增加或复用支持关系 |
| CONCEPT-A06 | 多概念支持 | 一个实例可支持多个概念，一个概念可由多个实例支持 |
| CONCEPT-A07 | 多父无环 | 下位概念可有多个直接上位，任何写入均不能产生环 |
| CONCEPT-A08 | 双向固定点 | 新增概念后受影响分量上下遍历，签名和直接边集合稳定 |
| CONCEPT-A09 | 树图分离 | 树形视图重复投影不复制节点、不反写概念图 |
| CONCEPT-A10 | 特征值封口 | 概念图服务只经特征服务取得特征值材料 |
| CONCEPT-A11 | 因果用途 | 因果概念无可靠性创建门槛，预测和行动用途分开观察 |
| CONCEPT-A12 | 来源动态边界 | 因果生成可用来源动态准入，但不持久化来源动态关系 |
| CONCEPT-A13 | 命名与指代 | 待命名材料结构化，临时代词不成为名称或世界事实 |
| CONCEPT-A14 | 生命周期 | 活跃、冷却、退役可读且重新命中可激活 |
| CONCEPT-A15 | 删除门禁 | 根、有支持、有引用、不可重挂或无事务隔离时拒绝物理删除 |
