## 1. 定位

本详细设计承接第 3 项“基础信息入账代码逻辑流程图”。目标是定义外部材料或上游服务请求如何被降权为基础信息入账请求，并经基础信息原子服务或世界聚合服务写入基础信息类节点、主信息和关系结构。

本设计不证明当前 C++ 已按本设计全部实现，不形成施工计划，不允许直接进入代码实施。

## 3. 设计对象

基础信息入账第一轮覆盖以下请求：

```text
基础信息通用节点。
实际存在。
场景。
场景接纳存在。
实例状态。
抽象状态。
实例动态。
抽象动态。
二次特征。
因果引用基础身份。
轻量因果引用(来源动态)。
```

以下请求不在本详细设计中展开：

```text
特征 / 特征值 / 特征状态材料：转第 4 项。
需求 / 任务 / 方法业务节点：转高级服务流程。
稳定因果结论：转后续因果聚合或轻量因果引用流程。
缓存统计、事件日志、显示层：只读或治理材料，不参与本入账写入。
```

## 4. 结构承载

| 入账对象 | 结构承载 | 写入方 | 只读方 | 权威性 |
| --- | --- | --- | --- | --- |
| 基础信息通用节点 | 主信息 + `节点类型::基础信息` | 世界服务兼容壳 | 语素服务 / 高级服务 / 显示层只读 | 机器事实 |
| 实际存在 | 主信息 + `节点类型::存在` | 存在服务 | 世界服务 / 状态服务 / 动态服务 / 高级服务 | 机器事实 |
| 场景 | 主信息 + `节点类型::场景` | 场景服务 | 世界服务 / 状态服务 / 动态服务 / 高级服务 | 机器事实 |
| 场景接纳存在 | `关系类型::归属` | 场景服务 / 世界服务编排 | 状态服务 / 动态服务 / 高级服务 | 机器事实 |
| 实例状态 | 主信息值容器 + `节点类型::状态` + 运行期临时 / 引用关系 | 状态服务 | 动态服务 / 特征服务 / 高级服务 | 场景内临时材料 |
| 抽象状态 | 主信息值容器 + `节点类型::状态` | 状态服务 | 语素服务 / 高级服务 / 显示层只读 | 长期基础信息 |
| 实例动态 | 主信息时间戳 + `节点类型::动态` + 运行期临时 / 引用 / 因果来源关系 | 动态服务，前后值材料经状态服务 | 因果服务 / 二次特征服务 / 高级服务 | 场景内动态证据 |
| 抽象动态 | 主信息动态材料 + `节点类型::动态` | 动态服务 | 语素服务 / 因果服务 / 高级服务 | 长期基础信息 |
| 二次特征 | 主信息 + `节点类型::二次特征` + 可选组成项引用关系 | 二次特征服务 | 特征服务 / 高级服务 / 显示层只读 | 长期基础信息 |
| 因果引用基础身份 | 主信息 + `节点类型::因果引用` | 因果服务 | 语素服务 / 高级服务 / 显示层只读 | 轻量证据入口 |
| 轻量因果引用 | 主信息 + `节点类型::因果引用`，来源动态只作准入证据 | 因果服务 | 高级服务 / 因果后续流程 | 轻量证据入口 |

## 5. 服务入口矩阵

| 入口 | 类型 | 输入 | 输出 | 写结构 | 入口拒绝 |
| --- | --- | --- | --- | --- | --- |
| `世界服务::创建基础信息` | 兼容写入口 | 无 | 基础信息节点句柄 | 主信息、基础信息节点 | 主信息或节点创建失败 |
| `存在服务::创建实际存在` | 写入口 | 无 | 存在节点句柄 | 主信息、存在节点 | 主信息或节点创建失败 |
| `场景服务::创建场景` | 写入口 | 无 | 场景节点句柄 | 主信息、场景节点 | 主信息或节点创建失败 |
| `场景服务::接纳存在` | 写入口 | 场景、存在 | 归属关系句柄 | 归属关系 | 场景无效、存在无效、关系重复或写入失败 |
| `状态服务::创建实例状态` | 写入口 | 场景、存在、发生时间戳、状态值 | 状态节点句柄 | 主信息、状态节点、运行期临时和引用关系 | 场景无效、存在无效、时间戳无效、任一写入失败 |
| `状态服务::创建抽象状态` | 写入口 | 状态值 | 状态节点句柄 | 主信息、状态节点 | 主信息写值或节点创建失败 |
| `动态服务::记录实例动态` | 写入口 | 场景、存在、时间戳、目标、前值、后值、可选来源动作 | 动态记录结果 | 动态节点、状态材料、运行期临时 / 引用 / 因果来源关系 | 输入无效、前后值缺失、来源动作无效、任一写入失败 |
| `动态服务::创建抽象动态` | 写入口 | 动态材料 | 动态节点句柄 | 主信息、动态节点 | 主信息写值或节点创建失败 |
| `二次特征服务::创建二次特征` | 写入口 | 无 | 二次特征节点句柄 | 主信息、二次特征节点 | 主信息或节点创建失败 |
| `二次特征服务::创建组合二次特征` | 组合写入口 | 组成项组 | 二次特征节点句柄 | 主信息、二次特征节点、组成项引用关系 | 组成项为空、组成项类型不允许、任一写入失败 |
| `因果服务::记录因果引用()` | 写入口 | 无 | 因果引用节点句柄 | 主信息、因果引用节点 | 主信息或节点创建失败 |
| `因果服务::记录因果引用(来源动态, 动态服务)` | 准入写入口 | 来源动态 | 因果引用节点句柄 | 主信息、因果引用节点 | 来源动态无效、动态材料不完整、任一写入失败 |

## 6. 统一入口拒绝

基础信息入账入口必须先排除以下输入：

```text
无效句柄。
错误节点类型。
无效或缺失发生时间戳。
缺少场景。
缺少存在主体。
缺少被改变目标。
缺少动态前值或后值。
来源动作不为空但不是动作入口角色。
二次特征组成项为空或类型不允许。
来源动态不是动态节点或动态材料不完整。
```

入口拒绝只用于发现非法材料来源并追溯上游，不允许在当前函数内部裁剪、补默认值、临时造节点或兜底修复后继续写入。

## 7. 写入失败收口

任一主信息、节点、关系或索引写入失败，必须进入统一失败收口：

```text
不返回有效句柄。
不允许留下可被读取为有效业务事实的半结构。
已经创建的结构必须回滚、失效或被读取隔离。
必要时登记逻辑错误或待确认问题。
不得依赖日志、显示、扫描器或后续校验修复半结构。
```

第一轮详细设计不授权修改核心仓库事务模型。后续施工计划必须明确采用以下哪一种收口方式：

```text
调用现有删除 / 失效入口清理本次创建结构。
通过读取隔离证明半结构不可作为有效业务事实读取。
停止实施并登记核心事务边界待确认问题。
```

## 8. 最小读回验证

所有写入路径都必须进入最小读回验证。

| 路径 | 最小读回验证 |
| --- | --- |
| 基础信息通用节点 | 节点可读，类型为基础信息，主信息可读 |
| 实际存在 | 节点可读，类型为存在，主信息可读 |
| 场景 | 节点可读，类型为场景，主信息可读 |
| 场景接纳存在 | 场景和存在可读，归属关系可读或场景目标组包含存在 |
| 实例状态 | 状态可读，值可读，发生时间戳可读，主体和场景材料唯一可读，场景临时关系可读 |
| 抽象状态 | 状态可读，值可读，不存在运行期临时目标关系 |
| 实例动态 | 动态可读，发生时间戳可读，主体、场景、目标、前值、后值材料可读，来源动作为空或有效 |
| 抽象动态 | 动态可读，动态材料可读，不存在运行期临时目标关系 |
| 二次特征 | 二次特征节点可读，主信息可读，组合二次特征的组成项组可读 |
| 因果引用基础身份 | 因果引用节点可读，主信息可读 |
| 轻量因果引用 | 来源动态准入通过，因果引用节点可读，主信息可读 |

读回不符合预期时，视为逻辑错误：

```text
停止生成施工计划。
登记断点问题。
追溯仓库写入结果、调用参数和入口服务。
不得继续推进代码实施候选。
```

## 9. 各路径详细设计

### 9.1 基础信息通用节点

`世界服务::创建基础信息` 是兼容壳，只用于尚未拆入具体基础信息类型的通用承载入口。

流程：

```text
创建主信息。
创建基础信息节点。
读回主信息和节点。
返回基础信息节点句柄。
```

边界：

```text
不得替代已经明确的存在、场景、特征、抽象状态、抽象动态、二次特征或因果引用入口。
不得承载显示标题、日志或临时文本。
```

### 9.2 实际存在

`存在服务::创建实际存在` 写入实际存在身份。

流程：

```text
创建主信息。
创建存在节点。
读回节点类型和主信息。
返回存在节点句柄。
```

存在服务不得从文本、日志、控制台输出或显示材料自动创建存在事实。

### 9.3 场景

`场景服务::创建场景` 写入场景身份。

流程：

```text
创建主信息。
创建场景节点。
读回节点类型和主信息。
返回场景节点句柄。
```

场景服务不得把显示窗口、控制面板页面或 SQL 投影自动写成场景事实。

### 9.4 场景接纳存在

场景接纳存在只写场景到存在的归属关系。

流程：

```text
验证场景节点有效。
验证存在节点有效。
创建归属关系。
读回关系或目标组。
返回关系句柄。
```

场景无效、存在无效、关系重复或关系写入失败时，返回无效关系句柄，结构不变化。

### 9.5 实例状态

实例状态是场景内临时零散节点，必须有：

```text
存在主体。
场景归属。
发生时间戳。
状态值。
```

流程：

```text
入口验证场景、存在和发生时间戳。
创建主信息并写状态值。
写发生时间戳槽位。
创建状态节点。
创建场景到状态的运行期临时关系。
创建状态到存在的引用关系。
创建状态到场景的引用关系。
读回状态材料。
```

实例状态不得直接作为长期语素入口对应信息；只有抽象状态可作为长期基础信息节点。

### 9.6 抽象状态

抽象状态是长期基础信息节点，本体不携带发生时间戳。

流程：

```text
创建主信息并写状态值。
创建状态节点。
读回状态值。
确认不存在运行期临时目标关系。
返回状态节点句柄。
```

### 9.7 实例动态

实例动态是场景内动态证据，必须有：

```text
存在主体。
场景归属。
发生时间戳。
被改变目标。
改变前值状态材料。
改变后值状态材料。
可选来源动作。
```

关键边界：

```text
改变前值和改变后值状态材料必须经状态服务创建。
动态服务不得绕过状态服务直接创建状态节点或写状态值。
动态服务只编排动态节点和动态到状态材料的关系。
```

流程：

```text
入口验证场景、存在、时间戳、被改变目标、前值、后值和可选来源动作。
请求状态服务创建前值状态材料。
请求状态服务创建后值状态材料。
创建动态主信息并写动态发生时间戳。
创建动态节点。
创建场景到动态的运行期临时关系。
创建动态到主体和场景的引用关系。
创建动态到目标、前值、后值和可选来源动作的因果来源关系。
读回动态材料。
```

### 9.8 抽象动态

抽象动态是长期基础信息节点，本体不携带发生时间戳。

流程：

```text
创建主信息并写抽象动态材料。
创建动态节点。
读回动态节点和主信息材料。
确认不存在运行期临时目标关系。
返回动态节点句柄。
```

### 9.9 二次特征

二次特征只维护复合或关系特征结构，判断结果必须落到状态、动态或因果结构。

当前代码事实：

```text
二次特征服务::创建二次特征() 调用主信息仓库创建主信息，再创建二次特征节点。
```

流程：

```text
创建主信息。
创建二次特征节点。
如是组合二次特征，验证组成项非空且类型允许。
逐项创建二次特征到组成项的引用关系。
读回二次特征节点、主信息和组成项组。
```

组成项允许范围以当前代码事实为准：

```text
基础信息
存在
场景
特征
状态
动态
二次特征
因果引用
```

### 9.10 因果引用基础身份

因果引用基础身份只创建因果引用节点，不表达稳定因果结论。

流程：

```text
创建主信息。
创建因果引用节点。
读回节点类型和主信息。
返回因果引用节点句柄。
```

### 9.11 轻量因果引用

轻量因果引用以来源动态作为入口准入证据。

流程：

```text
验证来源动态为动态节点。
经动态服务读取动态材料。
动态材料完整则创建因果引用基础身份。
读回因果引用节点和主信息。
```

边界：

```text
不持久化因果引用到来源动态的关系。
不得把单个因果引用解释为稳定因果结论。
后续因果可靠性由动态样本、评分、版本和复判机制形成。
```

## 10. 当前代码差距

| 编号 | 差距 | 当前事实 | 后续要求 |
| --- | --- | --- | --- |
| BASE-GAP-01 | 部分单步写入入口未显式读回验证 | 存在、场景、抽象状态、抽象动态、因果引用等返回句柄后主要由调用方使用 | 后续施工计划必须补最小读回验证 |
| BASE-GAP-02 | 多步写入失败收口不完整 | 状态、动态、二次特征等多步写入失败时返回无效结果，但可能已有中间结构 | 必须设计回滚、失效或读取隔离，不允许可读半结构 |
| BASE-GAP-03 | 实例动态前后值材料当前由动态服务私有辅助创建 | `动态服务::创建值材料状态` 直接创建状态节点和主信息 | 后续应改为经状态服务创建状态材料，或先生成独立施工切片确认 |
| BASE-GAP-04 | 世界服务兼容壳仍可直接创建基础信息通用节点 | 当前 `世界服务::创建基础信息()` 存在 | 后续需确认兼容壳保留边界，不得扩大替代原子服务 |
| BASE-GAP-05 | 轻量因果引用和基础身份入口需要在验收中分开 | 当前 `记录因果引用()` 与多个重载共存 | 详细设计和施工计划必须分别验收 |
| BASE-GAP-06 | 结构化拒绝原因未统一 | 当前多用无效句柄、空 optional、空集合或局部枚举 | 第一轮可保留轻量返回，但施工计划需列明拒绝原因追溯方式 |

## 11. 锁边界和公共函数边界

基础信息服务应顺序调用仓库公开入口，不在服务层同时持有多个仓库锁。

本详细设计不授权新增全局公用函数工具层。若后续施工切片需要新增共享辅助，应先按 `规范/公用函数规则规范.md` 判断归属：

```text
领域结构读写辅助 -> 对应领域服务内部。
仓库内部结构辅助 -> 对应仓库内部。
纯通用计算 -> 后续公共工具候选。
```

## 12. 验收矩阵

| 验收项 | 输入 | 期望 |
| --- | --- | --- |
| BASE-A1 基础信息通用节点读回 | 创建基础信息 | 节点类型为基础信息，主信息可读 |
| BASE-A2 存在读回 | 创建实际存在 | 节点类型为存在，主信息可读 |
| BASE-A3 场景读回 | 创建场景 | 节点类型为场景，主信息可读 |
| BASE-A4 场景接纳存在读回 | 合法场景 + 合法存在 | 归属关系可读，目标组包含存在 |
| BASE-A5 实例状态读回 | 合法场景、存在、时间戳、状态值 | 状态材料完整，场景 / 主体 / 值 / 时间戳可读 |
| BASE-A6 抽象状态读回 | 状态值 | 状态值可读，不是实例状态 |
| BASE-A7 实例动态读回 | 合法场景、存在、时间戳、目标、前后值 | 动态材料完整，前后值状态材料可读 |
| BASE-A8 抽象动态读回 | 动态材料 | 动态节点和主信息材料可读，不是实例动态 |
| BASE-A9 二次特征读回 | 合法组成项组 | 二次特征节点、主信息和组成项组可读 |
| BASE-A10 因果引用基础身份读回 | 无来源动态 | 因果引用节点和主信息可读 |
| BASE-A11 轻量因果引用读回 | 来源动态材料完整 | 因果引用节点可读，不写来源动态关系 |
| BASE-A12 拒绝后结构不变化 | 无效句柄、错误类型、缺时间戳、缺前后值、无效来源动态 | 不返回有效句柄，不留下可读业务事实半结构 |

## 13. 后续施工计划生成门禁

生成施工计划前必须确认本详细设计，并完成以下门禁：

```text
明确允许文件。
明确禁止文件。
明确是否允许修改状态服务和动态服务以收束前后值状态材料创建边界。
明确是否允许调用现有删除 / 失效入口处理失败收口。
明确若现有核心仓库无法证明结构不变化时，是否停止并登记核心事务增强待确认问题。
明确不修改核心仓库、工程文件、旧源码、控制面板、SQL、D455、体素或外设相关文件，除非另行确认。
```
