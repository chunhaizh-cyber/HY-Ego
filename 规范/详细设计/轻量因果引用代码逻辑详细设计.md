# 轻量因果引用代码逻辑详细设计

更新时间：2026-07-08

状态：已生成 / 可直接生成待确认施工计划 / 未生成施工计划 / 未入队 / 未改 C++ / 未构建 / 未迁移旧函数

## 1. 设计定位

本设计承接第 14 项 `流程图/20260708_轻量因果引用代码逻辑流程图_v0.1.md`，用于明确轻量因果引用身份创建、来源动态准入、动态材料准入和因果候选只读边界。

本设计不生成稳定因果结论，不裁决需求满足，不自动创建需求、任务、方法或结算记录。

## 2. 输入入口

```text
来源动态节点
动态服务
因果服务
可选需求结算入口传入的因果引用节点
```

入口必须拒绝无效来源动态、非动态节点、动态材料不完整、把轻量因果引用解释为稳定因果结论、以及用因果候选写业务事实的请求。

## 3. 核心处理

```text
创建因果引用基础身份
-> 读回因果引用节点类型和主信息有效性
-> 对来源动态做轻量节点类型准入
-> 可选通过动态服务读取动态材料并做更强准入
-> 读取因果模板候选和查询候选
-> 读取缺失证据候选占位
-> 作为需求结算的可选证据材料
```

当前 `记录因果引用(来源动态)` 只验证动态节点类型；`记录因果引用(来源动态, 动态服务)` 才能复核动态材料完整性。后续计划必须明确推荐入口和禁用误用边界。

## 4. 结构承载

```text
因果引用节点：轻量因果引用身份
动态节点：来源动态准入证据
动态材料：发生场景、主体、目标、前状态、后状态、发生时间戳、来源动作
候选材料：因果模板 / 查询候选 / 缺失证据候选，只读
```

当前不持久化“因果引用 -> 来源动态”的关系，不保存证据集合、评分、版本或可靠性结论。

## 5. 输出变化

成功路径输出轻量因果引用节点或只读候选材料。只读候选路径不得改变节点、主信息、关系或索引数量。

失败路径返回空材料，不创建因果引用，不写因果关系，不创建任务、方法、需求或结算。

## 6. 当前代码差距

```text
没有专门因果引用材料读取入口。
没有证据集合、来源动态集合、评分、版本或可靠性状态。
缺失证据候选仍是保守占位。
没有稳定因果结论、因果模板持久化、因果任务化或可靠性评分。
多步创建还缺数量快照级半结构不可读验证。
```

## 7. 验收矩阵

| 验收项 | 验证方式 | 通过口径 |
| --- | --- | --- |
| 身份创建 | 创建后读回节点 | 类型为因果引用且主信息有效 |
| 来源动态准入 | 传入非动态节点 | 拒绝且不创建因果引用 |
| 动态材料准入 | 传入材料不完整动态 | 拒绝更强准入路径 |
| 关系不写边界 | 来源动态准入前后统计关系数量 | 不因轻量准入新增来源动态关系 |
| 候选只读 | 读取模板 / 查询 / 缺证据候选 | 节点、主信息、关系、索引数量不变 |

## 8. 后续边界

后续若生成施工计划，必须把轻量因果、可靠因果结论和需求结算因果材料分开，禁止让单个因果引用裁决需求满足。
## 中途非成功返回二分口径

本文件按 2026-07-09 硬规则修订：中途非成功返回只分为 `追根因解决` 和 `逻辑内返回`。

- `追根因解决`：前置条件已经满足，并进入创建、绑定、写关系、写状态、记录动态、结算、读回或结构承载后，结果不符合内部预期；必须停止依赖路径，定位根因，当前未证明完整回滚时登记事务隔离缺口或半结构隔离缺口。
- `逻辑内返回`：领域协议允许的拒绝、候选为空、请求材料返回或人读材料返回；必须保证结构不变化，且返回材料、日志、回执、显示或控制台输出不裁决机器事实。
