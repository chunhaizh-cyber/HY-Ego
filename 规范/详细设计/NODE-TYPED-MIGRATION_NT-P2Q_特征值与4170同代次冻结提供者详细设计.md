## 2. 已确认实际接口

1. `节点直接身份结构事务域` 当前只有共享互斥、隔离标记和下一事务序号，没有运行期域身份、已发布代次或冻结规则版本。
2. `特征值类型化记录只读访问器` 只有单项当前 / 审计读取和一次独立加锁的结构版本读取，没有全量副本。
3. #340 的节点直接新域实际写入 `数据操作.节点直接特征体系.ixx` 私有 `节点直接特征批次记录仓`；旧 `参与者.特征批次发布记录.ixx` 的账只被旧 `数据操作.特征体系.ixx` 持有。
4. `节点直接特征服务` 只持有 `节点直接特征体系数据操作*`。公开业务服务不得接收或返回结构许可，因此本冻结提供面不加入该服务。
5. 既有 `节点直接领域合同.数据.h` 在完整领域类型导入前包含，只能承载通用结果形状，不能承载 `vector<特征值类型化记录>` 或 4170 完整记录。

## 3. 单一冻结权和所有权

所有权固定为：

```text
#349：统一冻结协调算法唯一所有者；一次取得、持有并释放全域冻结许可
节点直接身份结构事务域：锁、运行期域身份、已发布代次和冻结规则版本唯一状态所有者
#375：实现许可见证和 P2A 有界值式复制；只消费 #349 已持有的许可
#352：最终运行期对象装配、真实提供者接线和完整集成验证所有者
```

#375 不得自行形成第二冻结权。#349 也不得把调用方构造的域身份、代次或规则版本交给提供者回显；快照来源只能读取事务域签发的真实见证。

节点直接统一冻结许可 节点直接身份结构事务域::取得统一冻结许可(
    std::uint32_t 预期冻结规则版本) const;

    const 节点直接统一冻结许可& 许可) const noexcept;

许可内部以 RAII 持有事务域共享读取权，但公开面不返回互斥、锁守卫、事务令牌、域指针或写能力。事务域构造时签发非零运行期域身份；已发布代次初值非零，只在会话和全部参与者真实完成发布后、独占许可释放前推进一次。入口拒绝、幂等读回、撤销、内部错误和资源失败不推进代次。域身份或代次零值 / 回绕属于内部逻辑错误并隔离该事务域。

当前冻结规则版本固定为 `uint32 1`。预期版本不符、事务域隔离或共享许可竞争时返回无效许可；#349 把它分类为形成冻结材料前的逻辑内结果。

结果是一个原子聚合；两组记录和见证只能共同存在或共同为空。不得保留 v0.1 的两个独立 `optional`，也不得把 DTO 放入 `节点直接领域合同.数据.h`。

`特征值类型化记录只读访问器::复制全部已发布记录()` 在一次共享锁内复制全部已发布记录。成功时状态为 `已形成`、结构版本非零，合法空仓返回空组；许可竞争、确定分配失败分别返回同名状态，未知格式、不完整记录、重复 `(特征值, 原始值版本)` 或同一特征值存在多个有效当前记录返回 `内部不一致`，所有非成功结果结构版本为 0 且记录组为空。新域批次私仓增加从 1 开始、仅在真实批次发布后推进的结构版本。数据操作在许可属于自身事务域后，按固定顺序复制两仓并再次核对见证、结构版本和数量；统一冻结许可已经有效时出现子仓许可竞争必须映射为聚合 `内部不一致`，不得降级为普通等待。

## 6. 读取、互证和锁顺序

顺序固定为：

```text
#349 取得一次节点直接统一冻结许可
-> 从许可读取第一份真实见证
-> #375 数据操作验证许可属于本域
-> 特征值类型化记录仓 shared：读取版本 / 数量并复制全部已发布记录
-> 新域 4170 批次记录仓 shared：读取版本 / 数量并复制全部已发布批次
-> 在同一许可内再次读取见证、两仓版本和数量
-> 复核每条记录完整、批次项目连续、写前可选组三项全空或全有
-> 复核新值和写前原始值版本能在特征值记录组中找到
-> 按稳定键排序并一次返回原子值式材料
-> #349 继续形成其余段并完成节点 / 关系 / 跨段互证
```

唯一锁序是：

```text
事务域统一冻结许可
-> 特征值类型化记录仓共享锁
-> 节点直接特征批次记录仓共享锁
```

不得反序、共享锁升级、持锁 I/O、回调上层服务或二次取得事务域许可。由于外层冻结许可阻止全部新域写事务，两个子仓可以按上述顺序形成最小副本后释放；排序和跨材料互证在子仓锁外、统一冻结许可内完成。

## 7. 新旧 4170 边界

新域第 9 段唯一读取 #340 私有 `节点直接特征批次记录仓`。旧 `特征批次发布记录账` 只属于旧默认域，#375 禁止读取、合并、迁移或用其补齐新域记录。

P4 现有段 9 字段顺序保持不变时，映射固定为：

```text
批次身份第一 u64 = 节点直接特征批次身份命名域 ABI（固定为 1）
批次身份第二 u64 = 新域账内 特征批次身份.编号
业务规则版本 u64 = 新域记录.业务规则版本
账记录格式编码值 u64 = 新域记录.记录格式版本 的零扩展
状态 u8 = 1（仅已发布记录进入冻结材料）
项目顺序 u64 = 新域项目.项目顺序号
项目种类 = 写前三个可选字段全空为初始、全有为换代
```

该映射不改变账内单分量批次身份，不建立第二权威；第一分量是跨恢复稳定的格式命名域，不得写入冻结见证的运行期域身份。来源运行期域身份只保存在快照头 / 段来源并来自同一冻结许可。恢复时账记录格式编码值必须非零且不大于 `uint32` 上限，状态只能为 1，项目种类必须可唯一还原；恢复再冻结时两分量批次身份保持相等。

## 8. 生命周期和装配

运行期上下文先拥有事务域和两仓，再构造 `节点直接特征体系数据操作`；#349 的内部适配器只持有该数据操作的只读引用。调用期由 #349 在栈上持有冻结许可，数据操作返回自有值式材料后不再保留许可引用。

对象析构顺序固定为：

```text
#349 调用期许可
-> #349 冻结适配器
-> 节点直接特征体系数据操作
-> 特征值记录仓 / 新域批次仓
-> 节点直接身份结构事务域
```

公开 `节点直接特征服务` 无需修改，不接触许可；最终对象装配和生命周期实证由 #352 完成。
