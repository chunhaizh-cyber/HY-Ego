## 2. 目标与边界

P2Q 只为统一冻结提供两类完整值式材料：

```text
特征值类型化记录：当前、历史、失效、删除及原始值版本
4170 特征批次发布记录：全部不可变批次和项目顺序
```

它不解释快照格式，不持有冻结文件，不返回记录仓、批次账、锁、令牌、参与者或引用，不修改 P4 文件。#349 只适配返回值，不得进入 P2A 私仓补事实。

## 3. 接口

```cpp
struct 特征值与批次冻结请求 final {
    std::uint64_t 冻结域身份;
    std::uint64_t 冻结代次;
    std::uint64_t 冻结规则版本;
};

struct 特征值全量冻结材料 final {
    std::uint64_t 冻结域身份;
    std::uint64_t 冻结代次;
    std::vector<特征值类型化记录> 记录组;
};

struct 特征批次全量冻结材料 final {
    std::uint64_t 冻结域身份;
    std::uint64_t 冻结代次;
    std::vector<特征批次发布记录> 批次组;
};

struct 特征值与批次冻结结果 final {
    节点直接领域结果头 结果;
    std::optional<特征值全量冻结材料> 特征值;
    std::optional<特征批次全量冻结材料> 批次;
};

特征值与批次冻结结果 读取同代次特征值与批次全量(
    const 特征值与批次冻结请求& 请求) const;
```

全部 DTO 值式拥有。服务可移动不可复制，由具名工厂从 P2A 只读访问器和 4170 只读访问器形成；依赖生命周期长于服务。

## 4. 顺序与结果

```text
复核非零冻结域 / 代次 / 规则版本
-> 在同一冻结权内读取特征值全量
-> 读取 4170 全量
-> 再读冻结身份与数量
-> 复核批次引用的值、关系、原始值版本和项目顺序
-> 分别按稳定排序键排序
-> 返回两个同域同代次自有值式材料
```

入口无效、冻结权不可取得或资源不足是逻辑内返回；前置通过后两次冻结身份不等、记录缺历史、批次悬空、顺序重复、读回坏项或部分材料返回属于内部逻辑错误。禁止过滤坏项。
