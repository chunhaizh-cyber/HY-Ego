## 1. 目标与完成边界

NT-P3 负责按 `NT-C1/v0.2`、`NT-C2/v0.2` 和 `NT-C3/v0.2` 预冻结接口合同形成完整的服务层候选调用图；P3 唯一合同正文位于 `计划/20260722_NODE-TYPED-MIGRATION_NT-P3_服务兼容删除审计收口子计划_v0.2.md` 第 2 节。本详细设计只引用 `C3-A0—C3-F2`，不复制 DTO、签名或 ABI 定义。提供者实现可以尚未汇合，真实接线和完整验证由最终切换计划完成：

```text
隔离新域值式调用请求
-> 具名节点直接请求适配
-> 节点直接领域服务解释业务规则
-> 节点直接专用数据操作编排
-> P1 节点 / 关系 / 索引事务底座
-> P2A / P2B / P2C 类型化记录与值式读取合同
-> 完整读回、退役审计和候选调用图验证
```

P3 同时重建节点退役路径，使权威删除只由节点、关系、具名领域类型化记录和可重建索引共同裁决，并在发布后使缓存 / 投影失效和确定重建。

本设计只证明目标结构、接口候选、叶子拆分、失败收口与验证合同已经定义。它不证明：

```text
P1—P2C 已经完成或其实际接口已经稳定
任何节点直接服务、数据操作或请求适配已经存在
当前默认运行期已经接入新域
旧服务头、旧装配、旧生产调用方或旧仓库已经删除
完整快照恢复、跨重启恢复或旧快照迁移已经完成
4010 / 5210 / 5230 / 5300 / 7140 已经在代码中实现
```

默认入口、旧默认装配、旧服务头、旧生产调用方最终迁移、旧仓库 / 句柄 / 字段物理删除和统一恢复由 P4 唯一拥有。P3 不得提前完成这些动作。

### 4.4 新旧域句柄机械隔离

P1 新域和旧域都可能产生相同形状的 `节点句柄`。仅凭调用约定不能拒绝跨域句柄。

开工条件至少满足一种可编译、可自检机制：

```text
独立仓库类型使旧句柄无法进入新域接口
或句柄携带不可伪造且互异的仓库 / 运行域身份并由每个入口校验
或具名新域值类型在适配边界完成旧句柄到新句柄的禁止性隔离
```

禁止静态强制转换、字段拷贝、同数值即同身份、查新域失败后回读旧域。

## 5. 隔离新域总结构

### 5.1 分层

目标层级固定为：

```text
隔离新域调用方 / 隔离自检
-> 值式请求适配
-> 节点直接领域服务
-> 节点直接专用数据操作
-> P1 节点 / 关系 / 索引共同事务
-> P2A / P2B / P2C 类型化记录参与者与值式读取
```

职责不得倒置：

| 层 | 唯一职责 | 禁止 |
| --- | --- | --- |
| 请求适配 | 规范化值式请求、校验域身份和命名域、映射结果 | 访问仓库、持有令牌、补业务规则、旧域回退 |
| 领域服务 | 解释业务语义、值域、权限、幂等身份、前置与结果 | 暴露会话 / 参与者、直接写仓库、复制数据操作算法 |
| 专用数据操作 | 预读、第一写前重读、形成共同事务、完整读回 | 发明业务语义、暴露仓库、跨域互查 |
| P1 底座 | 节点直接身份、正式关系、索引候选、事务阶段 | 保存领域值、把索引升级为事实 |
| P2 领域记录 | 保存所属领域的强类型非拓扑事实 | 复制关系端点、形成通用无类型容器 |

## 7. 请求适配合同

### 7.1 允许的适配

P3E 按 `C3-E1—C3-E3` 的强类型重载集合适配每一种 `C3-A1—C3-D2` 具名请求。不得建立通用请求体、`variant`、字节载荷或兼容重载。请求适配只允许：

1. 将线程、组合器或隔离自检的值式请求转换为具名节点直接业务请求；
2. 校验请求版本、对象类型、命名域、稳定句柄域身份和必填材料；
3. 把节点直接服务的结构化业务结果映射为调用方值式结果；
4. 保留同键同义、冲突、许可拒绝、第一写前版本漂移、只读投影形成期间的读取版本漂移和内部不一致的可区分状态。

### 7.2 禁止的适配

不得：

- 接收裸 `uint64_t` 后由节点类型或查找结果猜测命名域；
- 接收旧主信息句柄、旧物理主键或旧仓库引用；
- 新域未找到时调用旧服务补值；
- 同时向新旧域双写或比较同义后选择一边；
- 暴露结构写入会话、执行器、事务令牌、参与者、仓库、锁或可变内部引用；
- 把内部逻辑错误压成 `bool false`、空句柄或普通等待。

### 7.3 候选调用点登记

每个候选调用点必须登记：

| 字段 | 内容 |
| --- | --- |
| 调用方 | 精确文件、模块、类和函数 |
| 当前入口 | 当前旧服务 / 现代服务与签名 |
| 目标入口 | 节点直接适配 / 服务与值式签名 |
| 身份映射 | 命名域、稳定主键、句柄域身份来源 |
| 结构依赖 | P1 关系 / 索引和 P2 记录 / 值式服务 |
| 非成功映射 | 每个逻辑内结果和内部错误保持方式 |
| P4 动作 | 默认切换时替换、删除或保留的精确动作 |

P3 只能证明目标调用点在隔离候选装配中可到达。P4 才修改默认调用方。

## 8. 统一写入与读取合同

### 8.1 创建 / 更新

```text
领域服务校验值式请求和业务前置
-> 专用数据操作事务外预读主键、节点、关系、领域记录和业务账
-> 分类同义、冲突、许可拒绝、版本漂移或可创建
-> 取得覆盖全部参与结构的唯一写入权
-> 第一笔写入前锁内重读
-> P1 创建 / 更新节点、关系和索引候选
-> P2 具名领域记录参与者登记候选
-> 完整读回并复核业务语义
-> 全部参与者确认待发布
-> 无失败最后发布
-> 领域服务值式读回
```

若对象只有节点和关系而无非拓扑载荷，不得为了统一接口创建空领域记录。若对象存在必要非拓扑值，必须有该对象具名强类型记录，不能写入主信息替代。

### 8.2 普通读取

读取顺序固定：

```text
稳定主键 / 完整句柄输入
-> 校验命名域和运行域
-> 索引缩小候选
-> 节点直接身份复判
-> 正式关系复判
-> 读取对象所属类型化记录
-> 领域服务组合调用期不可变投影
```

索引未命中可按该入口的正式协议返回未找到；索引命中但节点、关系或记录不成立属于内部不一致。组合投影不持久化，不生成第二份完整读取材料仓库。

### 8.3 幂等

同义只在全部权威结构互证时成立。只比较稳定主键、节点句柄或请求哈希不足以裁决幂等。同键异义必须拒绝；不能读取另一线程 / 另一域的结果后把并发规则破坏伪装为成功。

## 9. 强类型退役、删除与审计

### 9.1 退役规格

删除由对象领域服务形成 `对象强类型退役规格`，至少引用：

```text
目标节点完整句柄与预期版本
全部当前关系句柄及版本
所属领域记录的具名身份与预期版本
需要保留的历史 / 失效事实
索引所有者、命名域和候选绑定
领域业务账中的不可变引用
```

禁止通用 `记录退役容器`、任意 `variant` 或记录类型 + 字节数组。每个领域记录参与者自己验证和退役本域记录。

### 9.2 提交顺序

```text
冻结目标权威影响面
-> 取得统一写入权并锁内重读
-> 失效全部当前关系
-> 删除中间概念时创建或复用全部替代直接关系 10
-> 各具名领域记录参与者登记退役 / 删除历史
-> 移除索引候选绑定
-> 标记节点删除并保存节点审计
-> 完整读回关系历史、记录历史、索引不存在和节点审计
-> 全部参与者确认
-> 无失败最后发布
```

具体候选登记顺序由 P1 / P2 实际参与者依赖决定；撤销严格逆序。节点不可在关系或领域记录仍引用时先删除。

概念删除还必须登记概念签名类型化记录参与者，并在同一事务内读回：

```text
概念节点及本域记录历史
概念签名记录及版本
关系 9—12 当前、失效和历史
删除后新增或复用的关系 10
全部外部关系
索引候选不存在
四根永久活跃、剩余概念同类无环且唯一根可达
```

概念签名记录不得保存定义节点句柄、稳定目标句柄、关系句柄或活动版本；句柄型约束由关系 11 和签名记录中的角色顺序号互证。

### 9.3 发布后投影处理

权威退役发布成功后：

```text
使概念活动图、召回组、用途统计和其它缓存失效
-> 按权威节点 / 关系 / 记录确定重建
-> 复核投影不再包含退役节点
```

投影清理失败是发布后内部逻辑错误，应停止依赖路径并追根因；不得回滚已发布权威事务，也不得把投影命中纳入节点是否删除的机器裁决。

### 9.4 回执分账

权威退役回执只包含可审计的节点、关系、记录和索引结果。缓存 / 投影失效与重建形成独立后验报告。两者可由上层组合显示，但组合显示不是新的机器事实。

## 10. 逻辑内返回与内部错误

### 10.1 逻辑内返回

- 请求缺失、类型错误、命名域未知、旧域 / 跨域句柄；
- 同键同义完整读回；
- 同键异义；
- 业务前置不满足；
- 第一笔写入前版本漂移；
- 未形成候选前许可竞争；
- 删除候选过期；
- 协议允许的未找到或合法空组。

这些结果必须零写入或在形成候选前结束。

### 10.2 内部逻辑错误

- 前置通过后节点、关系、记录或索引候选创建异常；
- 完整读回无法互证；
- 参与者阶段、确认、撤销或最后发布不闭合；
- 新域句柄被旧域接受或反之；
- 领域记录复制关系端点或回读旧主信息补项；
- 发布后权威读回或退役审计不一致；
- 缓存 / 投影无法按权威结构确定失效和重建。

发布前内部错误停止新增写入并逆序精确撤销；不能证明前态时隔离事务域。发布后错误停止依赖路径并追根因，不重开候选撤销。


若实际模块依赖无法在新增隔离文件中成立，正式退回设计；不得以“只改一行旧装配”扩大范围。

## 15. 停止与漂移门禁

出现以下任一情况，停止受影响叶子并退回设计：

1. P1 / P2 实际接口与本设计假定的事务、值式读取或隔离域身份不同；
2. P1 实际接口没有实现 4010 已冻结的命名域 ABI、高水位和跨域拒绝；
3. 新旧域句柄无法机械拒绝跨域；
4. 任务方法选择仍只能通过 16 个无类型槽保存关系证据，或缺关系 16 / 23 的正式角色；
5. 概念签名参与者、关系 9—12、替代关系 10、权威回执和投影后验报告无法按 7140 分账；
6. 任何对象需要通用记录容器、主信息兼容双写或旧域回退；
7. 新服务必须直接 import 旧默认数据操作或旧服务头；
8. 需要修改入口、默认装配、旧生产调用方、旧服务头或物理删除旧仓；
9. 需要扩改 P1 / P2 已固定文件而没有新的上游设计路由；
10. P3A—P3E 对同一生产文件、公开接口、自检或工程文件出现所有权重叠；
11. 删除完成只能由缓存、日志、计数或 bool 证明，不能读回节点、关系、记录和索引审计；
12. 完整候选调用图只能通过在线热替换或双域同时装配验证。
