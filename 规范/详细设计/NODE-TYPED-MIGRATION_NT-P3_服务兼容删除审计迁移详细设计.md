## 1. 目标与完成边界

NT-P3 负责按 #339 已实现的 `NT-C1/v0.3`、预冻结的 `NT-C2/v0.4` 和 `NT-C3/v0.5` 形成完整服务层候选调用图；P3 唯一合同正文位于 `计划/20260722_NODE-TYPED-MIGRATION_NT-P3_服务兼容删除审计收口子计划_v0.5.md` 第 2 节。本详细设计只引用具名合同，不复制 DTO、签名或 ABI 定义。预冻结提供者可以尚未汇合，真实接线和完整验证由最终切换计划完成：

```text
隔离新域值式调用请求
-> 具名节点直接请求适配
-> 节点直接领域服务解释业务规则
-> 节点直接专用数据操作编排
-> P1 节点 / 关系 / 索引事务底座
-> P2A / P2B / P2C 类型化记录与值式读取合同
-> 完整读回、退役审计和候选调用图验证
```

P3 同时重建节点退役路径，使权威删除只由节点、关系、具名领域类型化记录和可重建索引共同裁决，并在发布后使缓存 / 投影失效和确定重建。

本设计只证明目标结构、接口候选、叶子拆分、失败收口与验证合同已经定义。它不证明：

```text
P2A—P2C 已经完成或其预冻结接口已经实现
任何节点直接服务、数据操作或请求适配已经存在
当前默认运行期已经接入新域
旧服务头、旧装配、旧生产调用方或旧仓库已经删除
完整快照恢复、跨重启恢复或旧快照迁移已经完成
4010 / 5210 / 5230 / 5300 / 7140 已经在代码中实现
```

默认入口、旧默认装配、旧服务头、旧生产调用方最终迁移、旧仓库 / 句柄 / 字段物理删除和统一恢复由 P4 唯一拥有。P3 不得提前完成这些动作。

### 4.4 新旧域句柄机械隔离

P1 新域和旧域都可能产生相同形状的 `节点句柄`。仅凭调用约定不能拒绝跨域句柄。

开工条件至少满足一种可编译、可自检机制：

```text
独立仓库类型使旧句柄无法进入新域接口
或句柄携带不可伪造且互异的仓库 / 运行域身份并由每个入口校验
或具名新域值类型在适配边界完成旧句柄到新句柄的禁止性隔离
```

禁止静态强制转换、字段拷贝、同数值即同身份、查新域失败后回读旧域。

## 5. 隔离新域总结构

### 5.1 分层

目标层级固定为：

```text
隔离新域调用方 / 隔离自检
-> 值式请求适配
-> 节点直接领域服务
-> 节点直接专用数据操作
-> P1 节点 / 关系 / 索引共同事务
-> P2A / P2B / P2C 类型化记录参与者与值式读取
```

职责不得倒置：

| 层 | 唯一职责 | 禁止 |
| --- | --- | --- |
| 请求适配 | 规范化值式请求、校验域身份和命名域、映射结果 | 访问仓库、持有令牌、补业务规则、旧域回退 |
| 领域服务 | 解释业务语义、值域、权限、幂等身份、前置与结果 | 暴露会话 / 参与者、直接写仓库、复制数据操作算法 |
| 专用数据操作 | 预读、第一写前重读、形成共同事务、完整读回 | 发明业务语义、暴露仓库、跨域互查 |
| P1 底座 | 节点直接身份、正式关系、索引候选、事务阶段 | 保存领域值、把索引升级为事实 |
| P2 领域记录 | 保存所属领域的强类型非拓扑事实 | 复制关系端点、形成通用无类型容器 |

## 10. 逻辑内返回与内部错误

### 10.1 逻辑内返回

- 同键同义完整读回；
- 同键异义；
- 业务前置不满足；
- 第一笔写入前版本漂移；

- 删除候选过期；
- 协议允许的未找到或合法空组。

### 10.2 内部逻辑错误

- 完整读回无法互证；
- 参与者阶段、确认、撤销或最后发布不闭合；
- 新域句柄被旧域接受或反之；
- 领域记录复制关系端点或回读旧主信息补项；
- 发布后权威读回或退役审计不一致；
- 缓存 / 投影无法按权威结构确定失效和重建。

发布前内部错误停止新增写入并逆序精确撤销；不能证明前态时隔离事务域。发布后错误停止依赖路径并追根因，不重开候选撤销。

#340—#345 各对象族提供具名 `形成……退役参与包`。参与包可移动不可复制，内部独占领域记录参与者，生命周期覆盖一次执行器调用。#346 对每个对象族使用具名重载，固定顺序为：
