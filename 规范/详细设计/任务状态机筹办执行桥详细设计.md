## 1. 定位

本详细设计承接 FS-05 第一轮任务服务承接材料与生命周期代码实施后的后续排除项，拆分任务状态机、任务筹办、任务执行桥、运行统计、运行包和治理协议边界。

本设计只定义任务服务后续结构和门禁，不实现线程、派发、`std::async`、真实执行桥、外设等待或旧任务模块函数迁移。

## 3. 总体分层

后续任务系统拆为五层：

```text
任务生命周期状态机。
任务筹办材料和筹办回执。
任务执行桥边界。
任务运行统计和权重评分。
运行包、治理协议和恢复材料分流。
```

第一轮不得让线程、日志、控制台输出、界面状态、运行包 bool、外设报告或 SQL 投影承载任务事实。

## 4. 任务生命周期状态机

### 4.1 状态阶段

| 阶段 | 建议状态 | 结构承载 | 写入方 | 读取方 | 完成边界 |
| --- | --- | --- | --- | --- | --- |
| 承接 | 已创建、已承接 | 任务生命周期实例状态 | 任务服务 | 任务服务、需求服务 | 已由第一轮承接壳覆盖 |
| 筹办 | 筹办中、待重筹办 | 任务生命周期实例状态 + 筹办回执 | 后续任务服务 | 任务服务、方法服务 | 只形成候选和缺口，不执行方法 |
| 等待 | 等待中、排队中 | 任务生命周期实例状态 | 后续任务服务 | 任务服务 | 不由线程快照裁决 |
| 执行 | 执行中 | 任务生命周期实例状态 + 执行请求材料 | 后续任务服务 | 任务服务、方法服务、动作入口服务 | 不实现真实派发 |
| 收束 | 已完成、失败、取消 | 任务生命周期实例状态 + 实际结果状态 | 任务服务 | 需求服务、任务服务 | 完成必须比较目标状态和实际结果状态 |

任务完成不能只看状态值。`已完成` 必须同时满足：

```text
任务有效。
任务来源需求有效。
任务目标状态唯一可读。
任务实际结果状态唯一可读或满足后续最新规则。
实际结果状态满足任务目标状态。
完成发生时间戳有效。
```

### 4.2 状态迁移拒绝

| 迁移 | 拒绝条件 |
| --- | --- |
| 已创建 -> 筹办中 | 缺来源需求、缺目标状态、缺任务虚拟存在、缺运行场景 |
| 筹办中 -> 排队中 | 无候选方法、候选方法未复核、存在未处理缺口 |
| 排队中 -> 执行中 | 缺任务选择方法关系、方法无动作入口、条件 / 结果结构未确认 |
| 执行中 -> 已完成 | 缺实际结果状态、实际结果未达目标、缺发生时间戳 |
| 任意 -> 失败 | 缺结构化失败原因或错误码来源不明 |
| 任意 -> 取消 | 来源需求失效或取消原因未确认 |

拒绝路径不得创建半任务、半状态、半方法关系或日志事实。

## 5. 任务筹办材料和回执

任务筹办只读取结构、形成候选和缺口回执，不直接执行方法，不直接写需求树。

| 材料 | 来源 | 用途 | 权威性 |
| --- | --- | --- | --- |
| 任务承接材料 | 任务服务 `读取任务承接材料` | 判断任务是否可筹办 | 权威结构 |
| 来源需求目标状态 | 需求服务 / 任务服务 | 方法候选目标 | 权威结构 |
| 需求主体、目标宿主、场景 | 需求服务 | 任务上下文 | 权威结构 |
| 候选方法集合 | 后续方法服务 | 召回候选 | 候选材料 |
| 方法结构缺口 | 方法服务、动作入口服务 | 形成缺口回执 | 候选材料 |
| 派生需求消息 | 需求服务后续入口 | 需求树生长候选 | 候选材料 |

筹办回执建议字段：

```text
任务节点。
来源需求。
目标状态。
筹办阶段。
候选方法集合。
选中方法候选。
方法结构缺口。
派生需求候选。
拒绝原因。
发生时间戳。
```

筹办回执不是任务事实。只有任务服务后续提交入口复核回执后，才可写任务状态、任务方法关系或派生需求请求。

入口拒绝：

```text
任务无效。
来源需求缺失。
目标状态缺失。
任务虚拟存在缺失。
运行场景缺失。
候选方法被直接当作选中方法。
派生需求消息绕过需求服务直接入树。
回执缺发生时间戳。
```

## 6. 任务执行桥边界

执行桥第一轮只定义服务边界和依赖，不实现真实执行分派。

执行请求材料：

```text
任务节点。
任务选择方法关系。
方法首节点。
动作入口角色节点。
输入场景。
输出场景或结果场景。
目标状态。
发生时间戳。
```

执行桥依赖：

| 依赖 | 当前口径 |
| --- | --- |
| 任务服务 | 提供任务、目标状态、实际结果状态和任务方法关系 |
| 方法服务 | 提供方法首、条件节点、结果节点和方法虚拟存在 |
| 动作入口服务或方法服务动作入口入口 | 提供动作入口角色和稳定动作键 |
| 动态服务 | 后续记录方法调用动作动态 |
| 状态服务 | 记录执行中、完成、失败等生命周期状态 |

执行桥不得：

```text
把线程作为动作来源。
把函数名、动作文本或日志标签作为动作身份。
绕过任务方法关系执行方法。
缺动作入口时执行。
缺条件 / 结果结构时声明方法可用。
把执行成功 bool 直接持久化为任务完成。
```

## 7. 运行统计、权重评分和错误码

| 材料 | 处理口径 |
| --- | --- |
| 运行次数、成功次数、失败次数 | 非权威统计，清空不影响任务事实 |
| 连续失败次数 | 非权威治理材料，不直接取消任务 |
| 最近错误码 | 诊断材料，不能成为机器事实来源 |
| 优先级、权重评分、权重版本号 | 后续任务管理排序材料，不裁决任务完成 |
| 特征使用统计 | 非权威缓存或特征统计专项 |

这些材料若落结构，必须进入非权威缓存 / 统计服务或任务管理后续专项。任务权威状态仍由生命周期状态、目标状态、实际结果状态和任务方法关系承载。

## 8. 运行包和治理协议分流

| 旧材料 | 新项目分流 |
| --- | --- |
| 一步治理任务包 | 后续任务管理请求材料 |
| 回流结算包 | 需求结算记录或事件日志材料，不直接裁决当前事实 |
| 治理恢复事件镜像 | FS-09 事件日志和恢复专项 |
| 恢复快照 | 仓库快照格式与恢复拒绝矩阵专项 |
| 原语判定和二次特征证据 | 二次特征服务和方法候选后续专项 |
| 工作线程快照 | 非权威运行观察材料 |
| 外设消息承接 | FS-X 排除，后续外设材料专项 |

治理协议可以作为后续设计参考，但不能把协议 bool、线程触发来源、控制请求或恢复快照直接裁决为当前任务事实。
