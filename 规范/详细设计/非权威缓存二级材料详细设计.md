## 4. 二级材料结构

| 材料 | 来源结构 | 写入方 | 读取方 | 失效条件 | 验证方式 |
| --- | --- | --- | --- | --- | --- |
| 特征值内容哈希材料 | 特征节点、特征值节点、来源值版本 | 后续特征服务请求统计服务记录候选 | 特征服务、索引候选读取方 | 原始值变化、规则版本变化、缓存清空 | 哈希命中后仍需原始值或特征服务复核 |
| 特征值命中观察材料 | 索引命中候选、特征值节点、发生序号 | 特征服务或索引候选路径 | 诊断层、候选策略 | 手动清空、规则版本变化 | 命中次数不参与需求、任务、方法裁决 |
| 需求观察统计材料 | 需求节点、目标状态、观察窗口 | 需求服务只读请求或统计服务运行期缓存 | 需求服务、显示层 | 需求目标或结算记录变化 | 统计缺失不影响需求目标读取 |
| 需求结算摘要材料 | 权威需求结算记录、来源任务、实际状态、动态证据 | 需求服务根据权威结算记录请求刷新 | 显示层、诊断层 | 结算记录新增、失效、恢复版本变化 | 摘要清空后权威结算记录仍可读 |
| 存在观测缓存材料 | 存在节点、场景、观察来源版本 | 存在服务或世界服务后续只读材料请求 | 显示层、观察候选读取方 | 存在生命周期变化、场景关系变化 | 缓存缺失不改变存在事实 |
| 二次特征签名缓存材料 | 二次特征节点、组成项组、组成版本 | 二次特征服务请求统计服务记录候选 | 特征服务、二次特征服务 | 组成项变化、规则版本变化 | 签名命中只缩小候选，不裁决复合事实 |

## 5. 权威复核顺序

```text
特征值内容哈希
-> 先复核特征节点和特征值节点有效
-> 复核特征值归属目标特征
-> 复核来源值版本或原始值材料
-> 哈希只缩小候选
-> 最终仍由特征服务和原始材料裁决。

需求结算摘要
-> 先复核需求节点、目标状态和权威结算记录
-> 摘要只从结算记录重建
-> 摘要缺失或过期只触发候选重建
-> 不写需求满足、任务完成或方法成功。

存在观测缓存
-> 先复核存在节点和场景归属
-> 缓存只记录观察候选
-> 外部输入或显示文本不得直接写入缓存事实
-> 不接外设、D455、体素或控制面板。

二次特征签名
-> 先复核二次特征节点和组成项
-> 签名只辅助候选召回
-> 命中后仍读取组成项关系
-> 不把签名写成二次特征事实。
```

## 7. 失效、清空和重建

| 场景 | 规则 |
| --- | --- |
| 清空缓存 | 只能清空运行期非权威材料，不改变节点、主信息、关系、索引、特征、需求、任务、方法、状态、动态或因果结构 |
| 重建特征值哈希 | 只能从特征服务提供的权威特征值材料或后续确认的原始值容器重建 |
| 重建需求摘要 | 只能从权威需求结算记录、目标状态、实际状态、来源任务和动态证据重建 |
| 重建存在观测 | 只能从存在服务、场景服务和已确认观察材料重建 |
| 重建二次特征签名 | 只能从二次特征组成项关系和组成项当前有效性重建 |
| 快照恢复后 | 缓存版本不匹配时默认失效，不得恢复为业务事实 |
