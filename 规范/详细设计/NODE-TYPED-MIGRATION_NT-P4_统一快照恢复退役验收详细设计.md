## 1. 目标与完成边界

NT-P4 负责按 P1—P3 具名“节点直接”预冻结接口合同形成冻结、恢复和旧格式候选，并在全部实际实现汇合后完成最后切换闭环：

```text
同代次统一冻结
+ 规范化序列化与严格解析
+ 全新隔离恢复候选装载
+ 节点 / 关系 / 全部领域记录 / 4170 全量互证
+ 索引与四类自我读取投影确定重建
+ 空宿主首次原子发布
+ 全新 / 恢复互斥启动
+ 旧快照离线迁移或明确拒绝
+ 默认装配一次切换
+ 旧仓库、旧句柄、旧字段、旧文件和工程登记物理退役
+ 完整自我跨重启与全仓验收
```

本设计形成只证明目标结构、接口候选、失败收口、所有权和验证合同已经定义。它不证明：

```text
P2—P3 已经完成或其预冻结接口已经实现
任何 C++、工程或自检已经修改
当前程序已经能保存或恢复快照
旧快照可以无损迁移
默认生产装配已经切换
主信息或旧域已经物理删除
完整自我跨重启恢复已经通过
```

## 4. 隔离新域与最终切换原则

### 4.1 P1—P3 前置形状

P4 只接受以下前置：

```text
P1：具名节点直接新域仓库、命名域 ABI、高水位、关系 0—23、可重建索引、事务和冻结导出
P2A：具名节点直接特征值记录、关系 22、4170 共同事务和值式读取
P2B：具名节点直接状态/动态记录、关系 19/20 和主体事实读取
P2C：具名节点直接自我关系 21、内部世界、成员和聚合读取投影
P3：关系 16 选择角色、关系 23 方法生命周期、概念签名记录、关系 9—12、全部新生产候选调用图和退役审计只指向上述新域；旧兼容入口的逐调用点切换映射已冻结，旧入口仍保持旧默认只读并等待 P4-CUTOVER 处理
```

P2A、P2B、P2C 不直接改写旧默认生产模块；P3 也不提前改变默认入口。P4 不得接受“同名旧文件已经就地变成新域”的口头假设，必须按固定提交和实际模块身份复核。

### 4.2 切换前零互通

切换前必须机械证明：

| 隔离面 | 必须零互通的内容 |
| --- | --- |
| 身份 | 同一稳定主键不同时在旧域和新域创建；句柄不能跨域通过有效性检查 |
| 关系 | 新域关系端点只指向新域节点；旧域关系不能成为新域候选材料 |
| 索引 | 索引所有者、命名域和物理键不跨域查找或回退 |
| 领域记录 | 新域类型化记录不读取主信息值槽或旧侧表补项 |
| 4170 | 新域批次账只引用新域槽位、值节点和关系 |
| 投影 | P2A/P2B/P2C 值式服务不在新域缺项时回读旧域 |
| 恢复 | 新域恢复候选不读取当前旧运行期结构补齐缺段 |
| 装配 | 默认生产仍只指向旧域；P3 新生产候选图只能由隔离自检或 P4 候选装配到达 |

任一互通命中都冻结最终切换，不得以“只读”“临时兼容”或“切换后会删除”放行。

### 4.3 最终切换提交原子性

默认装配改指新域、旧生产入口不可达、旧文件 / 字段 / 工程项删除必须形成同一个最终切换提交。该提交进入主线前必须在任务工作树完整构建、运行和静态验证；主线不得出现以下中间状态：

```text
默认同时可到达旧域和新域
新默认已启用但旧入口仍可写
旧文件已删除但新恢复/启动装配尚未接通
工程登记先删后补或先补后留双模块
```

## 5. 统一冻结模型

### 5.1 快照身份

一次统一冻结至少具备：

```text
当前快照格式版本
快照代次
来源运行期域身份
来源节点直接仓库身份
冻结规则版本
段清单版本
完整性校验算法版本
规范化权威载荷摘要
```

快照代次在同一来源运行期内单调且非零。来源域、代次或格式不同的段不得拼接。

### 5.2 必需权威段

目标 `统一权威冻结材料` 不是任意 `variant`、JSON 或通用字节记录容器。九段的记录级字段、数值 ABI、排序键、历史表达、恢复 DTO、候选单次移动和旧格式拒绝唯一读取 `NT-C4/v0.4` 详细设计；本节只保留段职责摘要：

```text
1 节点直接记录段：节点编号、完整节点句柄、稳定主键、类型、版本、状态、创建序号、节点稳定主键命名域 ABI 版本和各域签发高水位
2 正式关系记录段：关系类型 ABI 0—23 的全部有效、已失效、已删除及历史版本、端点、顺序号和高水位；业务角色只由顺序号主题合同表达
3 特征值类型化记录段：I64 / VecI64 / VecU64、原始值版本和历史
4 状态域记录段：状态记录身份、发生时间、接收时间、记录模式、记录版本、记录状态及历史
5 动态域记录段：动态记录身份、发生时间、动态种类、聚合规则版本、记录版本、记录状态及历史
6 语义 / 因果 / 概念记录包：词、语素入口、基础信息、轻量因果、完整因果和概念签名的强类型非拓扑原始值
7 需求 / 任务 / 方法治理记录包：需求、需求结算、任务治理、任务方法选择和方法的强类型非拓扑原始值
8 用途观察记录段：观察记录身份、机会数、正数、负数、Q10000、观察规则版本、阶段、记录版本、记录状态及历史
9 特征批次发布记录段：4170 不可变特征批次发布事实
```

九段均为必需段，每类恰一段并按段类型升序出现；合法无记录用记录数量 0 表达，不得省略。状态、动态、任务方法选择、用途观察及其它对象的全部拓扑端点与业务角色只进入第 2 正式关系记录段，业务角色只由 `正式关系记录.顺序号` 主题合同解释；领域记录段不得复制主体、场景、特征、值、来源、方法、动作、状态、动态或观察对象端点。每段有固定段类型 ABI、段格式版本、记录类型、数量上限、值域校验器、冻结方、装载方和互证方。新增领域段只能显式追加并提升段清单版本；不能复用或改义 1—9，未知段、重复段或缺必需段必须拒绝。

### 5.3 不进入权威段的材料

以下内容不作为必需权威段：

```text
索引当前绑定和反向绑定
子存在、特征、状态和动态列表
组合读取投影、显示树、摘要和缓存
概念根 / 生命周期 / 专用关系登记数组、活动快照、上一候选图、抽象树、主动召回组和用途统计
线程、锁、许可、仓库地址和可变服务引用
日志、SQL 审计记录、控制台输出和人读报告
```

可以保存明确标注为可丢弃的加速校验提示，但恢复装载必须在完全删除这些提示后仍得到等价结果。P4 第一版优先不写索引段，减少双权威风险。

### 5.4 统一冻结顺序

```text
运行期上下文签发唯一冻结身份
-> 取得覆盖新域全部写入参与结构的单一冻结权
-> 复核没有未完成候选、已确认未发布参与者或隔离事务域
-> 按固定段顺序复制节点、关系、各领域记录、历史事实和 4170
-> 在同一冻结权内复核数量、高水位、端点、所属节点和跨段引用
-> 形成不可变统一冻结材料和规范化摘要
-> 释放冻结权
```

冻结权只阻止写入推进，不把线程或锁状态写入快照。任一段导出失败、资源失败或互证失败时不产生可被命名为“完整快照”的输出。

## 6. 规范化序列化与严格解析

### 6.1 `NT-C4/v0.4` 容器编码

当前格式使用显式规范化二进制编码，不直接写入 C++ 结构内存。数值合同固定为：魔数 8 字节 `48 5A 59 43 53 4E 50 32`（ASCII `HZYCSNP2`）；容器格式版本 uint16=2；字节序 uint16=1 且固定小端；容器头 104 字节；段头 64 字节；段清单版本 uint32=2；九段记录 ABI 版本 uint32=1；摘要算法版本 uint32=1 且固定 SHA-256 32 字节；冻结规则版本 uint32=1；节点稳定主键命名域 ABI uint32=1；必需段数量 uint32=9。完整记录布局只读 `NT-C4/v0.4` 详细设计，不得从本节摘要补造。

104 字节容器头固定布局：

| 偏移 | 字段 | 宽度 |
| --- | --- | --- |
| 0 | 魔数 | 8 字节 |
| 8 | 容器格式版本 | uint16 |
| 10 | 字节序标识 | uint16 |
| 12 | 容器头字节数 | uint32 |
| 16 | 段头字节数 | uint32 |
| 20 | 段清单版本 | uint32 |
| 24 | 摘要算法版本 | uint32 |
| 28 | 段数量 | uint32 |
| 32 | 文件总字节数 | uint64 |
| 40 | 来源运行期域身份 | uint64，非零 |
| 48 | 来源节点直接仓库身份 | uint64，非零 |
| 56 | 快照代次 | uint64，非零且同来源单调 |
| 64 | 冻结规则版本 | uint32 |
| 68 | 节点稳定主键命名域 ABI 版本 | uint32 |
| 72 | 容器摘要 | 32 字节 |

64 字节段头固定布局：

| 偏移 | 字段 | 宽度 / 规则 |
| --- | --- | --- |
| 0 | 段类型 | uint16 |
| 2 | 段格式版本 | uint16，当前均为 1 |
| 4 | 段头字节数 | uint32，必须为 64 |
| 8 | 段顺序 | uint32，从 0 连续到 8 |
| 12 | 段标志 | uint32，当前必须为 0 |
| 16 | 记录数量 | uint64 |
| 24 | 段载荷字节数 | uint64 |
| 32 | 段载荷摘要 | 32 字节 SHA-256 |

格式上限固定为：文件总长 `8,589,934,592` 字节；单段载荷 `2,147,483,648` 字节；每段记录 `4,194,304` 项；单记录 `16,777,216` 字节；UTF-8 字符串 `1,048,576` 字节；单向量 `1,048,576` 项。字符串和向量数量前缀使用 uint32；单记录以前置 uint32 字节数分帧，但仍由具名强类型段解释。

布尔值编码为 uint8 的 0/1；可选值编码为 uint8 存在标志后接具名值；稳定主键编码为两个 uint64；完整句柄固定编码为 `uint64 仓库身份 + uint64 对象编号 + uint32 版本号`，与 #339 真实句柄 ABI 一致。不得序列化指针、`size_t`、本机填充、虚表、锁、`optional` 内存布局或区域设置相关文本。

段摘要只覆盖段载荷。容器摘要覆盖整个文件，但计算时将容器头偏移 72—103 视为 32 个零字节。恢复再冻结相等只比较九段规范化权威载荷，不比较来源域、来源仓库、快照代次、摘要或可重建投影。

### 6.2 写文件

序列化先写入同目录具名临时文件，完整写入、刷新和复读摘要通过后再原子替换目标快照材料。失败时删除临时材料或保留为具名诊断文件，但不得覆盖上一份完整快照。文件原子替换只保护材料完整性，不把文件升级为运行期事实。

### 6.3 分配前严格解析

解析顺序固定为：

```text
打开只读输入
-> 检查总文件大小硬上限
-> 读取固定最小头并验证魔数、版本、长度和摘要算法
-> 检查段数量、每段长度、记录数量、偏移累加和总长度
-> 拒绝重叠、重复、缺必需段、未知段、截断、尾随和摘要不符
-> 仅在所有边界成立后按段上限分配不可变值式材料
-> 逐字段解码并验证枚举、句柄、状态、版本和值域
```

未知节点 / 关系 ABI、未知记录状态、零或回退高水位、数值截断、超限向量和重复身份均在构造仓库候选前拒绝。

## 7. 隔离恢复候选与全量互证

### 7.1 恢复候选身份

解析结果先形成不可变 `恢复材料包`，再由恢复协调器创建全新的 `节点直接运行期上下文候选`。候选拥有新的运行期域身份和独立容器；不得引用当前宿主、当前仓库、当前索引或当前服务内部地址。

候选阶段至少区分：

```text
已解析
-> 装载中
-> 已装载待互证
-> 已验证待发布
-> 已发布

任一发布前阶段 -> 已销毁
```

候选能力不进入公开领域服务，不由普通调用方持有，不提供逐条公共注入接口。

### 7.2 固定装载顺序

```text

7. 正式服务完整读回
8. 候选再次冻结与规范化载荷相等
```

不得先发布节点后补关系，也不得把当前运行期结构、索引命中、日志、SQL 或默认值用于补齐材料。

## 8. 索引、列表和自我投影重建

索引由节点稳定主键、索引所有者/命名域规则、正式关系和领域记录确定重建。构建后逐项完成正向、反向、句柄、类型、版本和领域当前性复判。输入快照携带的旧索引或加速提示不得直接注入当前索引。

### 8.2 四类自我读取投影

恢复候选必须在隔离域内通过 P2 固定值式接口重建：

```text

场景服务::读取内部世界成员组
特征业务服务::读取宿主实例特征槽位组
```

除自我身份和唯一内部世界必须存在外，成员、槽位、状态、动态可以是合法空组。任何候选结构异常必须让整个聚合结果失败，不得返回部分成功列表。

### 8.3 概念投影确定重建

恢复候选必须只从四根固定身份、三阶段状态身份、概念节点、概念签名类型化记录、关系 9—12 和对象本域记录确定重建：

```text
专用关系反向登记
活动快照 / 抽象树 / 主动召回组

```

同一权威输入必须得到等价投影。用途统计没有正式事件账时恢复为空并重新积累；不得从旧统计缓存、活动版本或召回组补造历史。投影重建失败使整个恢复候选拒绝，不能把已有登记数组或快照直接注入。

### 8.4 完整自我跨重启证明

跨重启验收至少建立一个确定场景：唯一自我、唯一内部世界、两个成员、两个实例槽及换代历史、至少两条状态事实、至少一条动态事实、失效关系和 4170 批次记录。进程甲通过正式服务创建并冻结后退出；进程乙仅以恢复模式启动，发布前重建全部索引 / 投影，发布后通过同一正式服务按稳定身份读回。

“再冻结值式相等”比较的是规范化权威载荷，不比较来源进程身份、快照代次、文件摘要或可重建索引。任何稳定身份、关系历史、领域值、4170 顺序或自我投影差异均失败。

## 9. 运行期宿主与启动路由

### 9.1 `NT-C5/v0.4` 单一首次发布入口

运行期值式 DTO 固定为：`生产运行期启动模式`、`生产运行期启动请求`、`全新运行期启动请求`、`恢复运行期启动请求`、`生产运行期启动结果`、`运行期上下文`、`运行期上下文租约`、`运行期上下文宿主`、`运行期首次发布结果` 和 `完整自我恢复证据读取结果`。启动模式数值 ABI 为 `1=全新`、`2=恢复`；0 和未知值拒绝。精确签名为：

```text
运行期首次发布结果 尝试首次发布已验证上下文(已验证运行期上下文候选&&)
生产运行期启动结果 启动全新上下文(const 全新运行期启动请求&)
生产运行期启动结果 启动恢复上下文(const 恢复运行期启动请求&)
完整自我恢复证据读取结果 读取完整自我恢复证据(const 运行期上下文租约&)
```

启动结果固定区分 `已启动/入口拒绝/材料拒绝/资源失败/已有当前上下文/内部逻辑错误`。候选和宿主间以单次移动转移所有权；未发布候选销毁，发布后调用方不得继续访问。租约只提供同身份已发布上下文的读取生命周期，不暴露可变仓库、锁、令牌或服务内部地址。

全新和恢复只决定候选如何形成，不改变宿主发布语义。宿主继续满足：

1. 只接受完整、已验证、尚未发布且与宿主目标域一致的候选；
2. 锁内再次确认宿主为空；
3. 已有当前上下文一律返回 `已有当前上下文`，不比较同义、不清空、不替换；
4. 发布只移动一个完整上下文引用并形成同身份租约；
5. 不提供 `尝试恢复替换`、`清空当前上下文` 或 `替换当前上下文`。

### 9.2 全新 / 恢复互斥

入口路由必须显式区分：

```text
全新模式：读取固定新域配置并形成全新候选
恢复模式：读取唯一指定当前格式快照，严格恢复
```

两个模式同时出现、恢复模式缺路径、路径重复、未知参数或恢复文件不可读均在创建候选前拒绝。恢复失败直接返回非零启动结果；不得改走全新模式。

全新候选与恢复候选必须分型形成但等强验证：全新候选完成稳定身份、关系 0—23、领域记录、历史事实和业务不变量全量互证，确定重建索引、自我投影与概念投影，并形成规范化初始冻结证明；恢复候选完成同一套互证与重建后，还必须再冻结并与输入规范化权威载荷值式相等。两类只有验证全部通过后才能封装为 `已验证运行期上下文候选`，全新候选不得直接越过验证进入宿主。

### 9.3 发布后读回

宿主首次发布后、启动任何任务线程、外设消费者或业务请求路由前，启动会话必须通过正式服务读回关键稳定身份、关系 19—23、关系 16 的选择角色、概念签名 / 关系 9—12、领域记录、4170、自我聚合投影和概念派生投影。读回不一致属于内部逻辑错误：停止启动并退出进程；不得回滚为半旧半新上下文，也不得重新启用候选撤销。

## 10. 旧快照策略

### 10.1 只允许离线迁移或拒绝

生产恢复入口只接受当前格式。旧格式由独立离线工具只读处理：


旧输入保持不变。迁移器不链接生产启动和宿主发布能力。

映射表必须逐版本、逐节点类型、逐旧槽位和逐旧关系语义明确。至少证明：

1. 节点稳定主键能从受规范约束的旧身份材料唯一确定；
2. 旧拓扑锚点能唯一展开为具名正式关系；
3. 旧无类型 I64 槽能按所属对象和槽位合同唯一进入一个领域记录字段；

5. 任务方法旧 16 槽能无歧义形成选择记录非拓扑字段、关系 16 角色和关系 23 历史；
6. 概念签名、关系 9—12、四根身份和三阶段身份能无歧义恢复；
7. 缺失的历史、失效关系、领域记录或 4170 不被默认值补造。

任何一项无法证明时明确拒绝，并只提供只读结构审计；不得运行时猜测或恢复已退出容器。

## 12. 候选结构与函数职责

### 12.1 值式结构候选

精确独占 `海中鱼巣/工具/旧格式快照.数据.h`、`海中鱼巣/工具/迁移.旧快照到当前格式.ixx`、`海中鱼巣/工具/审计.旧快照迁移.ixx`、`海中鱼巣/工具/自检.旧快照迁移.ixx` 四个文件。不 import 生产宿主、线程、外设或默认装配。LEGACY 可以与 FREEZE / RESTORE 并行形成候选，最终输出必须交真实 RESTORE 接口验证。

### 13.4 P4-CUTOVER：完整恢复、一次切换与物理退役

依赖 P3、P4-FREEZE、P4-RESTORE 和旧格式裁决全部固定。#352 是共享接线、默认切换、物理退役和完整验证的唯一所有者；现有 v0.4 白名单和删除集只作后继升版输入，必须在 `CUTOVER-GATE/v2` 修订中重新闭合新合同，不得由执行 S0 动态扩大或替换。

P4-FREEZE、P4-RESTORE、P4-LEGACY 不得写默认装配、入口、句柄旧 ABI、工程 XML、统一运行器或物理删除旧文件；三者业务文件互斥并按同一 P4 合同并行形成候选。CUTOVER 是工程、运行器、默认装配、入口、旧文件删除和完整验证的唯一所有者，不存在无编号 P4-SHARED。

### 13.5 真实依赖

```text
P1—P3 + P4 公共待实现合同
-> P4-FREEZE / P4-RESTORE / P4-LEGACY 并行候选

P3 实际候选 + P4-FREEZE + P4-RESTORE + P4-LEGACY 实际候选
-> P4-CUTOVER 真实接线、完整恢复、默认切换、旧域删除和全仓验收
```

前三叶只依赖预冻结 `NT-C4/v0.4`，可以在提供者实现尚未出现时并行形成候选；最终真实接线仍由后继切换计划的门禁裁决。门禁不靠计划中的动态状态文本，而由以下六份内容寻址证据闭合：

```text
CUTOVER-PROVIDERS/v1：#339—#351 固定提交、文件和提供合同
CUTOVER-ACCEPT/v2：NT-C1/v0.3、NT-C1Q/v0.1、NT-C2/v0.4、NT-C3/v0.5、NT-C4/v0.4 逐签名核对矩阵
CUTOVER-MAP/v2：#347 可编译 DTO、完整签名、字段 / 结果映射和逐调用点唯一新入口映射
CUTOVER-DELETE/v1：旧生产对象逐路径删除/保留裁决
CUTOVER-WHITELIST/v1：最终修改/删除白名单
CUTOVER-GATE/v2：共同集成基线及上述证据 blob 的闭合引用
```

#352 计划只冻结门禁证据格式，不填写尚未发生的 PASS、提交或 blob。实际共同基线和新技术身份未形成前，候选全集、计划路径和聊天回执均不能赋予 #352 写权；精确修改 / 删除白名单由设计计划预先冻结，执行 S0 只核对，不生成、扩大或等待设计接受。

## 14. 非成功结果与失败收口

### 14.1 逻辑内返回

| 分支 | 时点 | 结构变化 |
| --- | --- | --- |
| 模式同时出现、恢复缺路径、未知参数 | 候选前 | 零变化，入口拒绝 |
| 文件不存在、不可读、超限或资源不足 | 发布前 | 宿主不变，返回材料 / 资源失败 |
| 未知格式 / 段 / 枚举、重复、缺段、摘要错、尾随 | 解析阶段 | 销毁材料，不创建仓库候选 |
| 身份重复、悬空关系、错角色、错版本、缺领域记录 | 隔离互证 | 销毁候选，宿主不变 |
| 旧快照没有无歧义映射 | 离线迁移 | 明确拒绝，只保留只读审计 |
| 合法宿主已经有当前上下文 | 发布前锁内 | 拒绝二次发布，现有租约不变 |

这些结果不得触发“恢复失败后全新启动”。

### 14.2 内部逻辑错误

以下必须追根因，不得包装成普通材料拒绝后继续：

- 同一冻结权下导出的段彼此不一致；
- 经过严格解析且逐字段有效的材料无法被内部装载器精确建立；
- 索引 / 投影重建产生输入权威结构不存在的新事实；
- 候选再次冻结与输入规范化权威载荷不相等；
- `已验证候选` 仍缺 P2/P3 正式服务所需结构；
- 最终切换后旧默认入口仍可达或新旧域出现互通；
- 发布后正式服务读回不一致。

发布前内部错误销毁候选并保持宿主不变。发布后错误不重开候选撤销、不清空宿主、不切回旧域；启动进程必须停止消费者并失败退出。

## 15. 验证矩阵

### 15.1 统一冻结与格式

1. 同一快照代次覆盖全部必需段，任一段失败不产出完整快照。
2. 节点冻结含稳定主键、命名域 ABI 版本和各域签发高水位，不含主信息句柄、拓扑锚点或无类型值容器。
3. 关系段覆盖 ABI 0—23，保留有效、失效、删除和历史版本，端点及角色完整。
4. 每个领域记录段强类型且无关系端点副本；任务选择记录、概念签名记录和 4170 独立保留。
5. 规范化字节输出不依赖 C++ 填充、指针、`size_t` 或区域设置。
6. 临时写失败、刷新失败、摘要失败和原子替换失败不覆盖旧完整文件。

### 15.2 严格解析与隔离装载

7. 空文件、截断头、未知版本、未知段、重复段、缺段、尾随字节、摘要错全部拒绝。
8. 超大总长、段长、数量、向量长度及整数溢出在分配前拒绝。
9. 零 / 重复稳定主键、编号回退、高水位不闭合和未知状态 ABI 拒绝。
10. 悬空关系、错端点、错角色、错基数、未知关系 23、关系 16 角色缺失 / 重复和跨域句柄拒绝。
11. 缺领域记录、错所属节点、错值域、任务选择拆分句柄槽、概念签名与关系 11 不一致、缺历史 / 失效关系或 4170 引用不一致拒绝。
12. 每个失败点候选销毁，宿主为空或既有上下文保持逐项值式不变。

### 15.3 重建与自我恢复

13. 完全不读取快照索引也能确定重建正反索引。
14. 成员、槽位、状态和动态列表清空后能从关系 19—22 与领域记录重建；方法生命周期由关系 23 重建。
15. 唯一自我、内部世界、自我非成员和无环全部成立。
16. P2A 槽位组、P2B 状态 / 动态组和 P2C 聚合投影全部通过正式值式服务读取。
17. 合法空组保持空；任一坏项使整体投影失败，不返回部分列表。
18. 清空概念登记、活动快照、抽象树、召回组和统计缓存后，只从概念签名与关系 9—12 确定重建等价投影；无事件账统计恢复为空。
19. 进程甲冻结退出、进程乙只走恢复模式首发并再冻结；规范化权威载荷相等。
20. 原始值、时间、关系 16 / 23、概念签名、关系历史、4170 顺序和稳定身份跨进程精确相等。

### 15.4 宿主与启动

21. 全新 / 恢复参数互斥；恢复失败返回非零且不启动全新候选。
22. 全新与恢复共用一个已验证候选首次发布入口。
23. 宿主已有上下文时同一候选和不同候选均拒绝，不替换租约。
24. 不存在恢复替换、清空或在线热切换 API。
25. 发布前普通服务和线程看不到候选；发布后消费者只取得同一完整租约。
26. 发布后读回失败使进程停止，不重新发布旧域。

### 15.5 隔离、切换与退役

27. 切换前旧默认与新候选在八类隔离面零互通。
28. P3 新生产候选调用图在 P4 前不被默认入口到达。
29. 最终切换暂存集同时包含新默认接线、旧入口不可达、旧文件、影子权威读取和工程项删除。

32. 离线迁移器不 import 生产宿主，无法无歧义映射时拒绝。
33. Debug / Release 当轮重建、两配置完整自检、跨进程恢复、故障矩阵、`strict`、`diff check` 和暂存白名单全部通过。
34. 独立只读覆盖审计逐项确认后，设计窗口只据正式证据归档完整恢复和退役声明。

## 16. 停止与漂移门禁

出现以下任一情况，停止受影响叶子并发送结构化修订请求：

1. P1—P3 实际文件或接口仍直接修改旧默认域，不能证明隔离新域；
2. P3 没有形成新生产候选调用图或仍需要旧主信息补值；
3. 任一领域权威记录没有有界强类型冻结 / 装载合同；
4. 统一冻结需要分时读取不同代次或允许未完成候选存在；
5. 严格解析必须依赖通用 JSON / 任意字节记录恢复业务语义；
6. 恢复候选需要调用公开写服务逐条注入或读取当前上下文补项；
7. 旧快照只有运行时猜测才能映射；
8. 需要在线热替换、恢复失败回退全新或第二次宿主发布；
9. 默认切换和旧域删除无法形成同一提交；
10. P4 叶子对工程、入口、上下文或自检运行器出现并发所有权重叠；
11. 验证只能证明文件读写、日志、数量或索引相等，不能通过正式服务读回权威结构。
