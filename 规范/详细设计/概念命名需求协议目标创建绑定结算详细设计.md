## 3. 权力边界

| 对象 | 权威写入方 | 允许读取方 | 边界 |
| --- | --- | --- | --- |
| 待命名请求 | 概念图服务只读生成 | 上行桥、领域路由 | 值式候选，不是需求或名称事实 |
| 治理消息 | 概念命名治理上行桥构造，邮箱只承载 | 自我线程、专用领域路由 | 非权威请求；摘要和顺序不裁决业务身份 |
| 命名目标状态 | 需求初始化经状态服务创建 | 需求服务、任务服务、结算路由 | 共享抽象状态；I64 只作状态材料 |
| 需求目标概念关系 | 需求服务 | 需求、概念、任务和删除治理只读入口 | 需求到概念的长期目标身份；不计入概念活性引用 |
| 命名需求 | 需求服务 | 任务服务、控制面板只读层 | 同概念最多一个未完成需求；服务根父子是需求树发布边界 |
| 名称关系 | 语素服务 | 概念图服务、命名路由、显示层 | `语素概念追溯` 是名称事实；文本和 UI 不裁决 |
| 绑定回执 | 命名治理领域路由值式生成 | 结算路由 | 只作复核材料，不等于结算 |
| 任务结果与完成 | 任务服务 | 需求结算、显示、后续学习 | 必须来自授权任务和合法方法动作 |
| 需求结算 | 需求服务 | 需求树、后续学习、只读显示 | 需要任务、实际状态、动态和完整读回 |

邮箱、自我线程、上行桥和显示层均不得直接写节点、主信息、关系、状态、需求、任务、方法、动态或结算。

## 4. 默认裁决

### 4.1 沿用 #196 D1 至 D12

```text
命名需求采用独立信息目标，不复用安全值或服务值特征。
主体和目标宿主均为自我存在；场景为当前自我所在场景；目标概念单独关联。
需求目标是共享抽象状态，不是 I64。
需求目标概念使用专用关系，不使用普通引用。
命名需求挂到服务根，不创建第三个根需求。
协议增加目标概念、活动版本和专用类型，版本必须提升。
同活动版本消息幂等；活动版本变化可重发，但长期需求身份不变化。
同概念最多一个未完成需求；多于一个归追根因。
名称候选是语素入口句柄，不是字符串。
绑定回执必须包含需求、任务、概念、语素、版本、时间和动作动态。
名称关系存在不等于正式结算。
未完成命名需求阻止目标概念物理删除，但不作为生命周期活性依据。
```

## 6. S2 信息目标状态与目标概念关系

### 6.1 共享目标状态

`自我根需求初始化结果` 增加：

```text
节点句柄 概念命名目标状态
```

`需求初始化服务::初始化两个根需求` 在两个根需求均完整后，经状态服务创建一次共享抽象状态并读回状态材料。建议第一版状态值为 `1`，只表示该抽象状态内部的已达成值材料；对外语义必须由：

```text
命名需求
+ 概念命名目标状态
+ 需求目标概念关系
+ 语素概念追溯关系
```

共同确定。不得把值 `1` 当成“任意概念已经命名”。

初始化失败必须沿既有初始化撤销路径清理本轮新建状态；两个根需求数量仍为 2。入口和控制面板只接收初始化结果中的稳定句柄，不按名称或全节点扫描重新发现。

### 6.2 关系类型

```text
关系类型::需求目标概念 = 13
方向：需求 -> 概念
同一需求：唯一目标概念
同一概念：可有历史已结算需求，但最多一个未完成需求
```

关系仓库只增加类型有效性白名单，不承载需求语义。创建、唯一性、端点类型、未完成判定和读回全部由需求服务封口。

概念图生命周期复核必须显式排除该关系；安全删除复核必须把该关系列为未完成需求阻断证据。这两个方向不能混为一个“外部引用”布尔值。

### 6.3 只读材料

```text
概念命名目标基线材料
  服务根需求
  自我存在
  当前自我场景
  共享命名目标状态
  目标概念关系类型版本 = 1
```

S2 只建立基线，不创建普通命名需求，不提交治理消息，不写名称关系。

## 7. S3 创建、复用与服务根发布

### 7.1 需求服务值类型

```text
概念命名需求状态
  已创建
  已复用
  目标已出现待结算
  请求已过期
  无效材料
  内部不一致

概念命名需求材料
  需求
  主体
  目标宿主
  场景
  目标状态
  目标概念
  服务根需求
  目标概念关系
  父子关系
  最近结算记录（可空）
  是否正式满足
```

公开入口假定：

```text
需求服务::读取概念命名需求(需求, 共享目标状态, 服务根需求, 状态服务)
需求服务::读取目标概念的命名需求组(目标概念, 共享目标状态, 服务根需求, 状态服务)
需求服务::创建或读取概念命名需求(
    主体,
    目标宿主,
    场景,
    共享目标状态,
    目标概念,
    服务根需求,
    状态服务)
```

`读取目标概念的命名需求组` 只沿 `需求目标概念` 反向关系取得候选，再逐个完整读回；禁止全节点扫描。历史已正式满足需求可审计返回，但唯一创建只统计未完成项。

### 7.2 线性化与去重

固定分片数作为实现常量，建议 64。分片选择必须由完整句柄三个字段的稳定组合得到；同一完整句柄总落入同一锁，不使用非权威消息摘要。

锁内顺序：

```text
1. 再次读取目标概念的全部命名需求关系。
2. 完整复核每个来源需求、目标状态、父需求和结算。
3. 未完成数量 > 1：追根因，不返回第一项。
4. 未完成数量 = 1：返回复用材料，不写结构。
5. 未完成数量 = 0：进入新建候选。
```

同一概念并发请求只能有一个 `已创建`，其余为 `已复用`。不同概念只在哈希碰到同一分片时串行，不建立全局粗锁。

### 7.3 多步写入与发布

新建顺序：

```text
1. 创建需求节点及主信息。
2. 写主体、目标宿主、场景和共享目标状态关系。
3. 写需求目标概念关系。
4. 完整预发布读回承接材料和目标概念。
5. 调用 #192 正式入口挂到服务根。
6. 完整读回目标概念、目标状态、父需求和两类关系。
7. 返回已创建材料。
```

第 5 步的服务根普通父子关系是需求树最终发布边界。正式 `读取概念命名需求` 只有在第 6 步全部成立时才返回。

任何创建后、最终发布前的内部不及预期，按本次调用记录的句柄逆序撤销：目标概念关系、承接关系、需求节点、主信息。父子发布后读回若出现内部矛盾，先删除本次新建父子关系，再逆序撤销并触发追根因诊断。不得撤销既有复用需求，不得扫描猜测“多余节点”，不得把日志当事务记录。

### 7.4 专用领域路由

新增模块候选：

```text
海中鱼巣/线程/概念命名治理领域路由.ixx
```

S3 只实现“消息 -> 创建或复用需求”的路线。路由先调用通用协议复核，再验证专用类型，重读当前待命名请求并比较 `来源材料版本`。然后调用需求服务，不直接写仓库。

既有 `自我治理领域路由.ixx` 保持固定安全根职责，不加入命名分支。自我线程继续通过外部处理器分派消息类型，不新增线程或第二邮箱。

## 8. S4 名称候选绑定与回执

### 8.1 依赖门控

当前代码和 #196 尚不足以证明以下接口：

```text
命名需求如何形成任务。
任务当前态与授权如何允许选择方法。
通用方法登记、规格、召回、复判和排序如何产出命名候选方法。
已选方法如何产出候选语素入口。
动作入口如何记录“绑定名称”的动作动态。
```

因此 #214 依赖 #201、#203-#209 的正式产物，并要求 #209 之后生成的任务选择提交计划已经完成。#214 执行前必须把这些后继编号补入正式实施记录；若 #209 仍只证明缺口，#214 不得实施代码，必须退回设计窗口修订前置编号和假定接口。

### 8.2 候选与回执

假定结构化候选：

```text
概念名称候选材料
  命名需求
  来源任务
  目标概念
  候选语素入口
  已选方法
  动作入口
  活动图版本
  发生时间戳
  动作动态
```

候选语素入口必须是已存在、可由语素服务读取的正式语素入口。名称字符串只能作为该入口的外部输入来源或人读显示，不进入治理消息、关系身份或幂等裁决。

绑定入口归属 `语素服务`，建议在现有 `追加语素概念追溯` 上增加领域授权参数或新建专用封口入口，而不是让路由直接调用关系仓库。入口前复核：

```text
命名需求的目标概念等于候选目标概念。
来源任务承接该命名需求。
任务已合法选择该方法。
动作入口属于该方法且当前可执行。
活动版本与当前概念身份可接受。
概念当前没有同一候选关系；重复绑定幂等。
```

写后必须通过 `读取概念名称入口组` 和入口侧追溯双向读回。成功后生成：

```text
概念名称绑定回执
  命名需求
  来源任务
  目标概念
  名称语素入口
  已选方法
  动作入口
  名称关系完整句柄或可复核关系材料
  活动图版本
  发生时间戳
  动作动态
```

回执是值材料，不持久化为第二份名称事实，不自动结算需求。

## 9. S5 任务结果与正式结算

### 9.1 前置

#215 依赖 #214 实际完成，且任务服务已经具备通用实际结果与完成提交契约。若 #214 只完成扫描或接口退回，#215 保持门控。

### 9.2 结算复核

专用领域路由重读：

```text
命名需求仍指向同一目标概念和共享命名目标状态。
来源任务承接该命名需求。
任务选择方法与回执一致。
动作入口属于已选方法。
语素概念追溯关系和反查结果均包含候选入口。
动作动态的场景、主体、来源动作和发生时间戳一致。
需求尚无已满足结算，任务尚无冲突结果。
```

活动图版本变化不自动否定已经发生的合法绑定；它只要求当前概念身份和生命周期仍可读。概念退役仍允许完成已有命名需求；概念已安全删除则应被更早的删除门禁阻止。

### 9.3 落账顺序

```text
1. 状态服务创建以自我为主体、当前场景和发生时间戳承载的实例实际状态。
2. 实际状态值与共享目标状态值匹配，但实例状态不替代共享目标身份。
3. 任务服务记录实际结果和完成状态，并完整读回。
4. 需求服务调用正式记录需求结算入口，结果为已满足。
5. 重读最近结算记录、来源任务、目标状态、实际状态和动态证据。
6. 只有全部一致才输出正式完成材料。
```

步骤 1-4 前必须确认任务服务多步提交的实际失败收口。若现有入口不能防止部分任务结果，#215 必须先退回生成任务结果事务前置，不得在命名路由中复制一套状态机。

## 10. 非成功返回二分

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 无明确单概念请求、概念已命名、活动版本正常漂移、消息过期或重复 | 逻辑内返回 | 不提交或不写结构 |
| 协议当前版本字段完整却映射互相矛盾 | 追根因解决 | 停止消息依赖路径 |
| 命名目标状态初始化参数无效 | 逻辑内返回 | 不创建状态 |
| 状态创建后读回不完整 | 追根因解决 | 逆序撤销本轮初始化并停止依赖 |
| 命名需求输入类型、场景、根或状态不匹配 | 逻辑内返回 | 不创建需求 |
| 同概念恰好一个未完成需求 | 逻辑内返回中的幂等复用 | 返回既有完整材料 |
| 同概念多于一个未完成需求 | 追根因解决 | 不选择第一个，不创建新需求 |
| 创建候选任一步写入或读回不及预期 | 追根因解决 | 只撤销本次候选，停止依赖 |
| 消息活动版本过期或名称已经出现 | 逻辑内返回 | 不创建第二需求；既有需求待正式结算 |
| S4 真实任务 / 方法接口尚未形成 | 逻辑内门控返回 | 不改代码，退回设计计划补前置 |
| 候选材料无效、任务未授权、方法未选择、活动材料过期 | 逻辑内返回 | 不写名称关系 |
| 名称关系写入后双向读回不一致 | 追根因解决 | 不生成绑定回执，不进入结算 |
| 回执重复且既有正式结算一致 | 逻辑内返回 | 返回重复完成材料，不新增结构 |
| 回执字段与当前同版本结构矛盾 | 追根因解决 | 停止结算路径 |
| 任务结果或需求结算写后读回不一致 | 追根因解决 | 停止后续学习、统计和显示观察 |

日志与弹窗只做人读诊断，不参与重试、修复、去重或结算。

## 11. 并发与锁顺序

```text
上行桥：无业务写锁；重读值材料后提交邮箱。
治理邮箱：保持现有独立锁，不持锁调用概念图或需求服务。
自我线程：冻结后锁外调用专用领域路由。
需求服务：按目标概念分片锁串行同概念扫描、创建和发布。
仓库：每次正式入口只持自身锁；服务不得同时长期持多个仓库锁。
语素绑定：由语素服务封口，同一概念 / 语素关系重复幂等。
结算：遵守任务服务和需求服务各自正式写入口，不在路由中跨服务裸持锁。
```

分片锁内可以顺序调用仓库正式入口，但不得调用邮箱、线程等待、日志、显示、SQL、外设或用户回调。删除补偿只处理本次候选句柄并按逆序调用正式仓库入口。

## 12. 生命周期、删除与恢复

```text
活动版本变化：旧消息过期；概念仍无名时新消息复用既有未完成需求。
概念冷却或退役：不取消需求；已有命名任务可继续，退役概念仍可命名和重新激活。
概念重新激活：不新建第二需求。
名称出现：停止新消息和新需求；既有需求等待正式任务 / 动态结算。
物理删除：任何未完成命名需求目标关系都必须阻断；已满足历史关系如何归档由安全删除设计裁决。
恢复：协议队列、值式请求和回执可丢弃；命名需求、目标概念关系、父子关系、名称关系和正式结算属于恢复候选。
```

#202 将统一盘点恢复。S1-S5 不写文件、SQL 或启动恢复。
