## 2. 当前代码事实

### 2.1 已实现

```text
概念图服务可按单个概念或活动图读取值式待命名请求，字段为概念、根类别、活动图版本和原因。
概念图服务会重读活动快照和生命周期；存在有效名称入口后不再返回待命名请求。
语素服务可沿语素概念追溯反查名称入口，也有低层正式关系写入口。

治理邮箱支持有界并发提交、幂等冲突、重复、过期和整批冻结；不解释业务语义。
自我线程锁外调用治理处理器；不需要为命名新增线程。
需求服务可读取主体、目标宿主、场景和目标状态组成的完整承接材料。
需求父子普通关系支持首次挂载、同父幂等、原地重挂、单父和防环。
需求结算要求来源任务、目标状态、实际状态、动作动态和发生时间戳。
节点、主信息和关系仓库存在正式删除入口，可用于同一次未发布多步写入的逆序撤销。
```

## 4. 默认裁决

### 4.1 沿用 #196 D1 至 D12

```text
命名需求采用独立信息目标，不复用安全值或服务值特征。
主体和目标宿主均为自我存在；场景为当前自我所在场景；目标概念单独关联。
需求目标是共享抽象状态，不是 I64。
需求目标概念使用专用关系，不使用普通引用。
命名需求挂到服务根，不创建第三个根需求。
协议增加目标概念、活动版本和专用类型，版本必须提升。
同活动版本消息幂等；活动版本变化可重发，但长期需求身份不变化。
同概念最多一个未完成需求；多于一个归追根因。
名称候选是语素入口句柄，不是字符串。
绑定回执必须包含需求、任务、概念、语素、版本、时间和动作动态。
名称关系存在不等于正式结算。
未完成命名需求阻止目标概念物理删除，但不作为生命周期活性依据。
```

```text
概念命名治理上行桥::提交待命名请求(
    概念待命名请求 请求,
    std::uint64_t 当前时间戳,
```

```text

```

新桥模块允许 include `语素服务.h`，但 `概念图服务.h`、`语素服务.h` 和其它领域文件仍禁止修改。不得为了隐藏桥依赖，在概念图服务内保存语素服务或新增无语素参数的待命名读取。

入口按以下顺序复核：

```text
1. 调用现有 `读取待命名请求(请求.概念, 语素服务)`。
2. 当前请求存在：逐字段比较概念、根类别和活动图版本。
3. 当前请求不存在：调用 `语素服务::读取概念名称入口组(请求.概念)`。
4. 名称入口组非空：返回目标已有名称。
5. 名称入口组为空：返回当前请求不存在。
```

活动版本或根类别不一致、已有名称、请求消失、过期或重复均逻辑内返回。两次读取都是只读副本；桥不持业务锁，也不要求跨服务快照。若当前同一材料下协议复核或邮箱映射自相矛盾，归追根因。

桥不得保存待命名集合、创建需求、绑定名称、管理线程或自动轮询。

## 16. JY-276 概念命名事务调用语境补充

本节覆盖第 7-9 节中对“取得结构许可”和“调用正式结果 / 结算接口”的早期笼统描述。#217、#223、#225 未形成正式提交前，以下属于依赖门控假定契约。

### 16.1 传统服务与真模块边界

`需求服务.h`、`语素服务.h`、`任务服务.h`、`方法服务.h`、`状态服务.h`、`动态服务.h` 均为传统头文件。它们不得 import 或持有协调模块公开类，只允许包含 `核心/结构事务接线.数据.h`、保存可空接线，并在令牌感知重载中验证 / 下传 `const 结构事务令牌&`。移动 `结构事务许可` 只存在于生产真模块或隔离自检的调用栈，不进入服务成员、治理消息、绑定回执或结算 DTO。

### 16.2 S3 唯一创建许可与锁序

`路由.概念命名治理.ixx` 是 S3 顶层独占许可所有者：

```text
路由完成协议 / 类型 / 消息值形状复核；不读取当前活动状态
-> 取得 #217 独占许可
-> 取得目标概念固定分片锁
-> 概念图令牌重载取得活动图读锁并重读快照 / 生命周期
-> 语素服务令牌重载下传同一令牌并复核名称入口
-> 当前请求与消息版本一致且名称仍为空
-> 调用需求服务令牌感知入口
-> 需求服务向仓库下传同一令牌
-> 成功完整读回或失败逆序撤销
-> 释放分片锁
-> 释放结构许可
```

固定锁序只能是“结构许可 -> 分片锁 -> 活动图读锁 -> 概念图 / 语素令牌重载 -> 仓库本地锁”。许可前快照方案不采用，因为当前无令牌 `读取待命名请求` 会先持 `活动图锁_` 再进入接域仓库，形成 `活动图锁 -> 结构许可` 的逆序风险；许可内也不得调用该无令牌包装，否则会触发同状态无令牌重入。分片锁内同样不得调用邮箱、线程等待、用户回调、日志、显示、SQL 或外设。S3 完成只证明统一命名路由的连接态创建路径，不证明入口现有主装配全部服务已接域。

### 16.3 S4 名称绑定与动作动态

S4 由同一命名治理路由取得一次共享许可，并把同一值式令牌传给语素、任务、方法、状态和动态服务的令牌感知入口。共享许可用于保持参与身份在调用期不被独占删除，并允许各仓库继续以本地锁保护单次写入；它不把名称关系和动作动态改写为单一原子事务，也不允许调用任务 / 状态的独占写入口。

JY-312 覆盖 JY-307 / JY-310 后的活动调用链为：

```text
统一路由取得共享许可
-> 任务服务共享路径只读入口重读选择组合身份、生命周期与审计
-> 方法服务经状态服务共享路径只读入口重读方法投影、角色状态、动作入口归属、当前状态与规格
-> 语素服务同令牌绑定并双向读回名称关系
-> 方法服务同令牌复核动作入口并委托动态服务
-> 状态 / 动态服务消费 #256 未发布主信息与节点候选，构造前后值状态、动态和全部非发布关系
-> 方法服务同令牌创建来源方法关系
-> 发布前完整读回；失败时关系 -> 节点候选 -> 主信息候选逆序撤销
-> 创建最终发布关系并立即确认候选
-> 动态服务同令牌公开读回材料并复核完整性
-> 方法服务组合读回动作动态证据
-> 释放共享许可后生成值式回执
```

名称关系完整成立后，动作动态写入 / 读回发生内部不一致时，名称事实继续有效，需求保持待正式结算，路由释放许可并追根因停止，不生成绑定回执。不得为了伪造原子性删除已发布名称、复制动态或增加跨服务回滚。

```text
读取当前任务方法选择材料
```

有效独占令牌仍可通过共享路径验证，因此 #221-#223 的既有独占调用语义不变。`按需求创建任务`、生命周期迁移、任务方法选择提交、失效当前选择和所有其它写入口继续验证独占令牌。不得增加任务写入共享许可，不得拆解或复制选择记录。

### 18.2 状态服务共享读与实例材料发布合同

`状态服务.h` 只允许：

```text
新增 读取状态值(状态, const 结构事务令牌&) 共享路径只读重载。
把 读取状态材料(..., 令牌) 与 读取状态幂等编号(..., 令牌) 改为验证共享路径令牌。
新增供动态服务调用的实例状态令牌发布入口；该入口接受有效共享路径令牌，但必须由状态服务完整拥有写入和清理。
```

实例状态令牌发布必须先构造主信息、状态节点、主体 / 场景引用并完成发布前读回，最后创建 `运行期临时` 关系使其成为可读实例状态。发布前任一步失败必须只清理本次句柄；发布后内部读回不一致属于追根因。现有 `创建实例状态候选`、候选确认 / 撤销、幂等迁移状态和其它事务写入口继续只验证独占令牌，不得放宽。

### 18.3 方法与动态链

方法服务的令牌版动作桥不得再调用任何无令牌关系、索引、状态或动作桥入口。动态服务必须通过状态服务令牌入口创建前后值实例状态，不得直接创建状态节点；动态本体也以引用 / 因果关系先构造、`运行期临时` 关系最后发布。名称关系和动态各自形成完整事实，但二者不冒充原子事务。

### 18.4 验收边界

共享令牌正例必须贯穿任务选择、状态值 / 材料、方法动作桥、语素绑定、状态前后值、动态及完整读回；伪造、过期、错线程和错域令牌必须写前拒绝。独占令牌仍可读取上述只读材料，任务 / 状态既有独占写入口必须继续拒绝共享令牌。自检还必须证明无令牌入口零调用、发布前失败无可读实例状态、名称已发布而动态失败时不伪造回执。

## 19. JY-312 未发布身份候选前置裁决

本节覆盖第 18.2 节中“状态服务直接用共享令牌创建后按裸句柄清理”的不完整合同。JY-311 已证明主信息 / 节点普通删除只接受独占令牌，现有 ABI 下不能完成共享写入失败清理。

### 19.1 方案裁决

采用独立前置 #256 `CORE-UNPUBLISHED-S1`，不把核心修改塞入 #214，也不把 S4 改为全域独占：

```text
#256 在主信息仓库和节点仓库增加不可伪造、不可复制、仅调用栈持有的未发布候选。
共享撤销只消费创建入口返回的同仓库、同完整令牌候选，不接受裸句柄。
普通删除主信息 / 节点继续验证独占令牌。
结构事务接线、协调器、许可类型和活动表不变。
#214 等待 #256 实际完成后，由状态 / 动态服务组合候选能力。
```

否决“直接把 #214 改成独占许可”：当前方法服务存在内部登记 / 规格锁后进入仓库的历史路径；在未完成全方法服务锁序专项前，路由持全域独占再进入方法内部锁会形成新的锁序风险。否决“当前 ABI 原子发布”：当前没有同时创建主信息、节点和关系的单调用原子入口。

### 19.2 S4 候选组合

状态服务和动态服务分别拥有自己的候选组合，不允许方法服务直接撤销状态 / 动态内部身份：

```text
创建主信息未发布候选
-> 创建节点未发布候选
-> 创建本服务非发布关系
-> 发布前读回
-> 创建运行期临时最终发布关系
-> 立即确认节点 / 主信息候选
-> 公开完整读回
```

方法服务创建来源方法关系后，若后续发布前检查失败，先删除自己创建的来源方法关系，再请求动态服务撤销动态候选；动态服务先撤动态关系和动态身份，再请求状态服务撤销前后值状态候选。每层只撤销自己持有的候选和精确关系。

最终发布关系成功后必须立即确认候选。确认后公开读回不一致属于追根因，不得再消费候选撤销，也不得删除已发布名称关系。名称关系已完整而动态候选失败时，动态候选数量回前态，名称事实继续有效，需求保持待结算，不生成绑定回执。

