# 显示层只读代码逻辑详细设计

更新时间：2026-07-08

状态：已生成 / 可直接生成待确认施工计划 / 未生成施工计划 / 未入队 / 未改 C++ / 未构建 / 未迁移旧函数

## 1. 设计定位

本设计承接第 19 项 `流程图/20260708_显示层只读代码逻辑流程图_v0.1.md`，用于把显示层只读候选从控制台自检输出、服务只读材料和后续 DTO / JSON / 视图候选中分离出来。

本设计不声明独立显示层只读视图服务已实现，不接 WebView、控制面板、截图或窗口。

## 2. 输入入口

```text
人读查看请求
调试观察请求
服务只读材料
需求 / 任务 / 方法 / 状态 / 动态 / 因果 / 统计快照 / 日志审计材料
```

入口必须拒绝绕过领域服务读取、显示层写业务事实、标题 / JSON / HTML / 控制台文本裁决业务状态的请求。

## 3. 核心处理

```text
读取领域服务只读材料
-> 组装只读展示材料
-> 检查展示材料不承载机器逻辑
-> 输出到控制台或后续只读视图
-> 检查显示层不写业务事实
-> 对 WebView / DTO / JSON / 截图 / 控制面板请求保持专项门禁
```

显示层只读材料只能来自服务只读入口，不得裸读或裸写节点仓库、主信息仓库、关系仓库或索引仓库。

## 4. 结构承载

```text
服务只读材料：权威结构的只读视图或副本
展示材料：人读输出，不承载机器事实
控制台输出：诊断显示，不是显示层能力完成证明
后续 DTO / JSON / WebView：未立项候选
```

显示层没有机器事实写入方。

## 5. 输出变化

成功路径只输出人读展示材料。权威结构不变。

失败路径拒绝越权读取、写入或事实裁决请求，不修改任何业务结构。

## 6. 当前代码差距

```text
没有独立显示层只读视图服务。
没有跨服务只读 DTO。
没有稳定显示契约。
没有显示层验收入口。
当前控制台自检输出不能声明显示层只读视图完成。
```

## 7. 验收矩阵

| 验收项 | 验证方式 | 通过口径 |
| --- | --- | --- |
| 服务只读来源 | 展示材料来源检查 | 只来自服务只读入口 |
| 写入拒绝 | 显示请求试图写业务事实 | 拒绝且结构不变 |
| 文本隔离 | 标题 / JSON / 控制台文本试图裁决状态 | 拒绝作为机器事实 |
| 通道门禁 | 请求 WebView / DTO / 控制面板 | 转后续专项 |
| 结构不变 | 显示前后统计权威结构 | 数量和关系不变 |

## 8. 后续边界

后续若进入显示层实现，必须另建待确认施工计划，明确只读入口、禁止写入、展示字段来源和结构不变验证。
