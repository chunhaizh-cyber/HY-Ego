# 语义基础服务分层迁移详细设计

日期：2026-07-14

状态：JY-332 已修订 / #268、#276 已完成 / #275 实际接口复核通过待执行 / #275 代码未实施

## 1. 目标

本设计把原 `#268` 中混合的世界余项、语素、概念图、因果和统计拆成两个可独立验收的服务层切片：

```text
#268 / SERVICE-DATA-S4：基础信息通用身份、语素入口与多关系、轻量因果引用。
#275 / CONCEPT-DATA-S1：四类概念根角色复核、实例支持概念、概念上下位、概念定义材料。
```

两项都遵守：领域服务公开业务入口不接触原始结构事务令牌；只有数据操作层导入不透明结构写入执行器；关系仓库继续承载权威关系；索引只召回候选并由完整句柄和权威记录复核。

## 3. 已验证当前事实

### 3.1 前置能力

`#264` 已形成一次独占许可内的不透明结构写入会话和执行器；`#272` 已补同许可节点类型、节点主信息、I64 和按源节点 / 关系类型关系组读取；`#265-#267` 已分别形成存在场景、状态动态和特征体系新分层服务。

`动态业务服务` 当前公开 `读取动态材料(节点句柄)`，返回可判定抽象动态或完整实例动态的值式材料。轻量因果服务可调用该公开业务入口，不需要原始令牌穿透。

`#268` 已于提交 `3fcf903` 完成并验证。存在、动态、二次特征和轻量因果四类原子业务服务均可形成根身份所需节点句柄；轻量因果必须先取得有效动态值式材料，不存在无来源因果创建后门。

### 3.2 现有兼容路径

```text
世界服务.h：创建基础信息仍直接创建主信息和基础信息节点。
语素服务.h：直接创建主信息、语素节点、两类语素关系和可选主键索引，并保留带令牌只读路径。
因果服务.h：直接创建因果引用；来源动态只作准入证据，不持久化来源关系。
概念图服务.h：超过 4400 行，同时包含根登记、专用关系、生命周期、活动快照、重挂、失效和删除。
统计服务.h：非权威缓存及概念清理参与逻辑，不是结构事实写入方。
```

上述文件仍是当前生产兼容事实。本设计不修改或删除它们，只建立隔离新路径。

### 3.3 当前能力缺口

不透明会话支持候选创建、主键绑定、关系创建、材料读取和逆序撤销，但不支持既有关系重挂、关系失效、已发布结构删除或概念活动快照替换。因此生命周期、重挂、失效、删除和统计清理不能夹入 `#268` 或 `#275`。

JY-331 实际接口复核还确认：会话只有按源节点读取关系记录组，没有按目标节点读取来源关系记录组；会话只能复核指定主键绑定，没有非破坏性的节点完整主键组读取。关系仓库和索引仓库已经有对应带令牌只读入口，因此新增 `#276 / CORE-SESSION-S3 / 752` 只做会话窄封装，完成后再复核 `#275`。

`#276` 已于提交 `9f9f880` 完成并验证。结构写入会话现已正式具备 `读取节点主键组(节点句柄)` 和 `读取来源关系记录组(节点句柄, 关系类型)`；两个入口继续受 `可继续写入()` 约束并使用会话同一独占令牌。四仓库、执行器、事务 ABI 和领域文件未修改。

## 5. #268 第一轮范围

### 5.1 固定路径

```text
1. 以非零幂等主键创建基础信息通用身份。
2. 以非零幂等主键创建语素入口并绑定一个已有对应信息节点。
3. 创建语素概念入口：文本准入 + 对应信息关系 + 概念追溯关系在一次会话内发布。
4. 为已有语素入口追加一条语素对应信息关系。
5. 为已有语素入口追加一条语素概念追溯关系。
6. 读取语素入口身份、主键、全部对应信息和全部概念追溯目标。
7. 以非零幂等主键和来源动态材料创建轻量因果引用。
8. 读取基础信息、语素和轻量因果的值式材料。
```

### 5.2 语素目标白名单

语素对应信息目标固定允许：

```text
基础信息、存在、场景、特征、抽象状态、抽象动态、二次特征、因果引用、需求、任务、方法。
```

语素概念追溯目标固定允许：

```text
基础信息、存在、特征、抽象状态、抽象动态、二次特征、因果引用。
```

两类入口都拒绝特征值、实例状态和实例动态。需求、任务、方法只可作为对应信息目标，第一轮不得作为概念追溯目标。

### 5.3 文本边界

文本只用于调用入口判断是否为当前允许的最小词单元。分隔符集合和长度规则继续以已确认语素准入设计及当前兼容事实为依据。文本不得写入主信息名称、节点字段、索引显示键、日志或控制面板，也不得自动由语言命名结果回流。

### 5.4 轻量因果边界

轻量因果请求必须先调用 `动态业务服务::读取动态材料`。只有材料当前可读，且为抽象动态或完整实例动态时才能继续。来源动态只作入口准入证据：

```text
不创建 因果引用 -> 来源动态 关系；
不把单条引用解释为稳定因果结论；
不由因果服务绕过动态业务服务解释状态或动态关系；
不把来源动态句柄保存为因果引用内部字段。
```

## 6. #268 分层职责

### 6.1 数据操作.语素基础

负责基础信息身份和语素结构的共享读取、独占写入、完整读回和并发幂等收口。它是本路径唯一可导入结构写入执行器和持有四仓库引用的生产模块。

### 6.2 基础信息业务服务

负责非零主键、请求角色和基础信息通用节点语义。它不得退化为任意节点类型工厂，不替代存在、场景、状态、动态、特征、二次特征或因果服务。

### 6.3 语素业务服务

负责文本准入、两类目标白名单、关系角色白名单、重复关系拒绝和业务结果映射。它不创建目标节点，不裁决需求、任务、方法或世界事实。

### 6.4 语素概念入口组合器

只把文本准入、对应信息关系和概念追溯关系组合成一次业务请求；失败时不允许留下只有对应信息关系、没有追溯关系的半入口。

### 6.5 数据操作.轻量因果与轻量因果业务服务

业务服务通过动态业务服务取得来源动态值式材料；数据操作层只负责在独占许可内复核动态节点仍当前可读并创建因果引用身份。两个模块不得持久化来源动态关系。

## 8. #268 结构不变量

### 8.1 基础信息身份

```text
非零主键 -> 唯一当前基础信息节点；
基础信息节点 -> 唯一当前主信息；
节点类型固定为 基础信息；
同主键不同类型或不同节点是冲突，不得选第一个。
```

### 8.2 语素入口

```text
非零主键 -> 唯一当前语素节点；
语素节点 -> 唯一当前主信息；
创建入口时至少有一条语素对应信息关系；
同一入口可有多个互异对应信息目标和多个互异概念追溯目标；
相同类型、相同源、相同目标的重复关系映射为幂等，不新增第二条；
集合读取返回全部权威当前关系，不静默选第一个。
```

### 8.3 轻量因果引用

```text
非零主键 -> 唯一当前因果引用节点；
因果引用节点 -> 唯一当前主信息；
来源动态准入不形成持久化关系；
同主键不同因果身份是冲突。
```

## 9. #275 第一轮范围

### 9.1 根角色复核

四类根角色固定映射：

| 根类别 | 已有原子身份类型 | 创建所有者 |
| --- | --- | --- |
| 存在根 | `节点类型::存在` | 存在业务服务 |
| 动态根 | `节点类型::动态` 且为抽象动态 | 动态业务服务 |
| 关系根 | `节点类型::二次特征` | 二次特征业务服务 |
| 因果根 | `节点类型::因果引用` | 轻量因果业务服务；必须有有效来源动态 |

`#275` 不绕过上述原子服务创建根身份。它接收已有根句柄和稳定非名称键，在共享许可内初读索引、完整节点、主信息、类型、四项互异以及不存在上位概念关系，形成可重复验证的根角色值式材料。根角色不是独立仓库事实；权威证据是稳定键、完整节点身份和关系仓库中的无上位关系，任何读取都必须重新复核。

因果根若无法由 `#268` 正式公开面形成满足上述合同的已有身份，`#275` 必须在实施前退回设计，不得自行增加无来源因果创建后门。

JY-332 已按 `9f9f880` 复核确认该退回条件未触发：四类原子业务服务均可形成节点句柄和幂等键材料，轻量因果入口没有无来源创建路径。

`#276` 完成后，`#275` 的每条关系写入必须在同一独占许可内使用以下会话材料再次复核根和端点，不得把许可外初读结果当作写入授权：

```text
读取节点类型。
读取节点主信息。
读取节点主键组，并用主键绑定匹配复核唯一稳定键。
按源节点读取关系记录组。
按目标节点读取来源关系记录组。
```

### 9.2 创建型专用关系

第一轮只允许：

```text
实例支持概念：实例 -> 概念；
概念上下位：上位概念 -> 下位概念；
概念定义材料：概念 -> 长期定义材料。
```

每条关系都由关系仓库权威承载。主键索引不替代关系；显示树、森林和图只读投影不参与写入裁决。

### 9.3 多根、多父和图关系

```text
四类根互异，分别作为四个概念森林入口。
同一概念可有多个同类别上位概念。
共享下位节点只保存一份节点身份，由多条概念上下位关系表达多父。
同一关系不得重复；跨类别父子、自环和形成有向环的关系拒绝。
图投影重复出现同一节点时，身份仍以完整节点句柄为准，不复制机器事实。
反向查询必须从关系仓库读取来源关系，不从显示缓存推断。
历史审计继续由关系记录及其版本 / 状态承载，不在 #275 删除历史关系。
```

### 9.4 概念角色与首次挂载

第一轮不新增 `概念` 节点类型，也不建立根登记表或概念角色侧表。概念角色只由当前权威关系解释：

```text
已复核根节点：天然是本次根角色中的概念。
普通概念节点：沿概念上下位反向路径至少可达本次一个同类别已复核根。
首次挂载候选：类型可映射到当前根类别、不是任一已复核根，且当前没有概念上下位、实例支持概念或概念定义材料角色关系的节点。
```

首次挂载只能由一条 `已归根上位概念 -> 首次挂载候选` 的概念上下位关系完成。关系提交前候选不是概念，关系提交后才能作为概念被读取或承接实例支持 / 定义材料。不得先形成孤立概念标记、缓存事实或显示投影，再补上下位关系。

同一节点已经归入本次同类别根后，可继续接受同类别多父；如果反向路径到达不同类别根、出现无根终点、已有概念角色但不能到达本次根，或已有互相矛盾的关系，归为入口拒绝或内部不一致，不自动选一个根。

## 11. 概念关系准入

### 11.1 实例支持概念

实例必须是当前可读长期节点，且不能是本次任一已复核根。概念目标必须是本次已复核根，或已经沿概念上下位反向路径到达本次同类别根的普通概念；未挂载候选不能由实例支持关系直接升格为概念。实例结构类型必须与目标根类别映射一致；跨类别支持、自指和重复矛盾拒绝。

### 11.2 概念上下位

上位必须是本次已复核根或已经归入本次根的普通概念。下位只能是：

```text
已经归入本次同类别根的普通概念；或
类型映射到同一类别、不是根、且当前没有三类概念角色关系的首次挂载候选。
```

已有下位可接受同类别多父。首次挂载候选只允许作为叶节点接入，不能携带未归根的下位分量。新增边前从下位沿概念下位方向查找上位；可达即成环。根作为下位、跨类别、无根分量、自环、直接环和多跳环均在第一笔写入前拒绝。

### 11.3 概念定义材料

源必须是本次已归根概念。第一轮目标白名单固定为：

```text

```

## 12. 幂等与并发

所有创建请求先用共享许可读取候选并按完整句柄、节点版本、关系状态和端点复核。不存在时由数据操作层取得一次独占许可并全部重读：

```text
完全相同事实已存在 -> 幂等读回；
主键或关系角色冲突 -> 入口拒绝；
写前材料漂移 -> 版本漂移；
写入后并发形成同一完整事实 -> 释放许可后权威重读映射为幂等；
写入后结构不完整 -> 追根因，不得选第一个或自动修复。
```

防环遍历和关系创建必须在同一独占许可内完成，不能先无锁判环后写入。第一轮不增加线程、异步任务或工作队列。
