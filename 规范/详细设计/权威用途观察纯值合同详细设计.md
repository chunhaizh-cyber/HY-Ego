## 5. 固定形状材料合同

### 5.1 格式

```text
用途观察合同格式版本 = 1。
```

格式版本裁决解释兼容性，不属于业务身份。第一版只接受格式 1，不猜测兼容，也不实现规范编码布局或持久化版本升级。目标格式 0 属于请求畸形，按 `入口拒绝 / 请求不完整` 收口；非零未知格式才按 `不支持 / 目标格式不支持` 收口。

### 5.2 用途观察身份

```text
用途观察身份：
  完整任务句柄。
  非零任务序号。
```

完整任务句柄保留仓库编号、节点编号和版本号。身份相等必须比较三个完整句柄字段与任务序号。格式版本属于候选解释门禁，不属于业务身份；格式先做兼容门禁，不把两个不支持格式对象比较为同一身份。

以下字段不进入身份：

```text
冻结键、请求编号、幂等键、回执编号、批次编号和时间戳。
因果完整结构键及其哈希。
九个不可变核心业务引用。
机会数、正结果数、负结果数和 Q10000。
阶段负载。
```

### 5.3 不可变核心

```text
用途观察不可变核心：
  需求。
  选择记录。
  方法。
  动作。
  场景。
  主体。
  目标状态。
  实际状态。
  动态证据。
  因果概率分量 历史概率。
```

九个业务引用和任务身份均为完整 `节点句柄`。历史概率复用 `材料.用途事件` 已有固定形状 `因果概率分量`，其内部只保存一个 `因果完整结构键` 与机会数、正结果数、负结果数、Q10000；不得再平行复制五个字段，不保存机会证据组，也不保存第二套原因 / 结果动态键。

### 5.4 阶段负载

```text
用途观察阶段负载：
  正式结果。
  任务达成。
  需求结算。
  观察阶段。
  可选正式结算状态句柄。
```

合法元组仅有：

| 观察阶段 | 正式结果 | 任务达成 | 需求结算 | 正式结算状态 |
| --- | --- | --- | --- | --- |
| 尝试未达成 | 未达成已转待重筹办 | 未达成 | 未形成 | 无 |
| 待结算 | 任务已完成待需求结算 | 已达成 | 待结算 | 无 |
| 最终已结算 | 已完成并结算 | 已达成 | 已满足 | 有且完整有效 |

阶段负载不保存待结算提示、结算 DTO、结算时间戳或关系证据。

### 5.5 候选

```text
用途观察候选：
  格式版本。
  身份。
  不可变核心。
  阶段负载。
```

候选不能包含 `vector`、`string`、完整 `用途来源冻结材料`、完整 `规范化任务结果来源`、完整 `用途事件材料`、完整 `需求正式结算材料` 或完整因果机会证据组。该约束由类型所有权和自检静态形状门禁共同证明，不用运行后扫描修补。

## 6. 强类型形成结果

形成合同独立命名为：

```text
用途观察纯值形成状态。
用途观察纯值形成原因。
用途观察纯值形成结果：状态、原因、optional 用途观察纯值候选。
```

不得复用同账比较的结论 / 原因类型，也不得用一个 bool 同时承载入口拒绝、不支持和内部不一致。

### 6.1 处理状态

```text
已形成。
入口拒绝。
不支持。
内部不一致。
```

### 6.2 原因

```text
无。
请求不完整。
来源冻结不完整。
结果尚未正式收口。
非新版完整回执。
来源冻结回合不一致。
规则版本不支持。
目标格式不支持。
正式结果材料矛盾。
身份材料矛盾。
内容材料矛盾。
阶段材料矛盾。
算法不变量破坏。
```

### 6.3 合法组合

| 状态 | 合法原因 | 候选 |
| --- | --- | --- |
| 已形成 | 无 | 必须有且自身合法 |
| 入口拒绝 | 请求不完整、来源冻结不完整、结果尚未正式收口、非新版完整回执、来源冻结回合不一致 | 必须无 |
| 不支持 | 规则版本不支持、目标格式不支持 | 必须无 |
| 内部不一致 | 正式结果材料矛盾、身份材料矛盾、内容材料矛盾、阶段材料矛盾、算法不变量破坏 | 必须无 |

默认结果为 `入口拒绝 / 请求不完整 / 无候选`。`成功()` 只接受 `已形成 / 无 / 有合法候选`。未列组合、非成功携带候选或成功缺候选均属于结果对象内部不一致，不能用 bool 掩盖。

## 7. 强类型同账比较结果

比较合同独立命名为：

```text
用途观察同账比较结论。
用途观察同账比较原因。
用途观察同账比较结果：结论、原因。
```

比较结果不携带候选 `optional`，不选择待发布值，不定义 O1-S2 的发布语义。

### 7.1 结论

```text
不同身份。
同账幂等。
同账允许推进。
同账冲突。
不支持。
内部不一致。
```

### 7.2 原因

```text
无。
格式版本不支持。
新候选非法。
既有候选非法。
完整身份不同。
完整内容相同。
待结算推进最终。
不可变核心不同。
阶段转换不允许。
算法不变量破坏。
```

合法结论—原因组合固定为：

| 结论 | 原因 |
| --- | --- |
| 不同身份 | 完整身份不同 |
| 同账幂等 | 完整内容相同 |
| 同账允许推进 | 待结算推进最终 |
| 同账冲突 | 不可变核心不同、阶段转换不允许 |
| 不支持 | 格式版本不支持 |
| 内部不一致 | 新候选非法、既有候选非法、算法不变量破坏 |

比较结果的安全默认值固定为 `内部不一致 / 算法不变量破坏`。只有上表列出的结论—原因配对合法；任一未列配对都属于结果对象内部不一致。比较入口不接收候选集合，不扫描多记录，不选优，也不执行合并或发布。没有既有候选时不调用双候选比较入口，由调用方保留新候选作为调用期材料；不得伪造“不同身份 / 完整身份不同”。已发布层同一身份多记录仍属于 O1-S2 未来结构审计，不在 O1-V1 中伪造集合输入。

## 8. 形成算法

公开入口：

```text
形成用途观察纯值候选(const 用途事件结果&, std::uint32_t 目标格式版本)
-> 用途观察纯值形成结果。
```

执行顺序不可交换：

### G1 上游结果矩阵

1. 上游为合法入口拒绝状态—原因—无事件：映射为同义入口拒绝。
2. 上游为合法不支持状态—原因—无事件：`不支持 / 规则版本不支持`。
3. 上游为内部不一致或任一未列状态—原因—optional 组合：`内部不一致 / 正式结果材料矛盾`。
4. 只有 `已形成 + 无原因 + 有事件` 才继续检查目标格式。
5. 成功上游下目标格式为 0：`入口拒绝 / 请求不完整`；目标格式非零但不是 1：`不支持 / 目标格式不支持`；格式 1 才进入 G2。

顺序不得交换：非法上游组合必须先追根因，不能被目标格式不支持遮蔽。

不得把上游 `因果候选不完整` 当作 `用途事件结果` 的合法入口拒绝原因；它只属于更早的冻结结果入口。

### G2 完整身份互证

必须逐项相等：

```text
事件.任务
= 冻结来源.任务执行请求.调度请求.任务
= 冻结来源.任务执行请求.任务材料.身份.节点
= 结果来源.完整回执.来源执行请求->调度请求.任务
= 结果来源.完整回执.来源执行请求->任务材料.身份.节点
= 结果来源.完整回执.句柄组.目标任务。

冻结来源.冻结任务序号
= 冻结来源.任务执行请求.调度请求.任务序号
= 结果来源.完整回执.来源执行请求->调度请求.任务序号
= 结果来源.完整回执.来源任务序号。
```

回执 `来源执行请求` 必须存在，且通过既有完整请求比较与冻结来源的 `任务执行请求` 逐字段相同。任务句柄无效、序号为零或任一来源不一致均为 `内部不一致 / 身份材料矛盾`。

### G3 不可变核心互证

九个事件业务引用必须完整有效，并与冻结任务、选择、方法、动作、场景、主体、目标、回执实际状态和动态证据逐项一致。

历史概率必须：

```text
从冻结来源的完整因果概率候选重新调用 #287 正式入口形成因果完整结构键。
形成成功后用 #287 全字段比较与事件效用中 `因果概率分量.因果身份` 相同。
通过既有 `形成因果概率分量` 重新形成固定形状分量，并与事件效用的整个 `因果概率分量` 逐项相同。
机会数、正结果数、负结果数和 Q10000 与冻结候选逐项相同。
四项计数满足机会数大于 0、机会 = 正 + 负、正 <= 机会、负 <= 机会、Q10000 <= 10000，且 `Q10000 == 正结果数 * 10000 / 机会数`。
```

任一失败在成功事件路径中统一为 `内部不一致 / 内容材料矛盾`，不得把 #287 入口拒绝或不支持向下游降格为普通来源拒绝。

### G4V 阶段合法性与正式结算深检

先验证事件效用的正式结果、任务达成、需求结算和观察阶段属于三种合法元组。

最终阶段还必须按以下顺序深检：

```text
结果来源.需求结算.成功()。
结果来源.需求结算.材料.正式结算 存在。
正式结算.完整()。
正式结算.结算状态.状态节点完整有效。
正式结算中的需求、来源任务、目标状态、实际状态和动态证据与事件逐项相同。
正式结算.结算状态.幂等主键 = 事件.结果来源.参数.结算状态幂等主键。
正式结算.结算状态.发生时间戳 = 事件.结果来源.参数.结算发生时间戳。
正式结算.结算状态.状态值 = 已满足，场景 = 事件.场景，主体 = 事件.主体。
正式结算的可选因果引用与事件结果参数中的有效 / 空因果引用逐项一致。
正式结算.结算状态.状态节点与兼容投影最近结算状态互证相同；裁决来源仍是正式结算材料。
```

尝试未达成和待结算阶段必须同时没有 `正式结算` 与 `最近结算状态`，也不得把待结算提示写入候选。任一矛盾为 `内部不一致 / 阶段材料矛盾`，停止形成。

### G5 固定形状投影与后验

只有 G1-G4V 全部通过后才构造候选。固定形状由类型声明和编译期 / 自检形状门禁保证；运行期不设置不可达的“投影合同破坏”原因。构造后立即在同一纯值函数内复核输出的格式版本、身份、不可变核心和阶段负载与已接受来源逐项相同；后验只验证算法构造不变量，不修补输入或输出。后验失败返回 `内部不一致 / 算法不变量破坏`，不得输出半候选。

## 9. 同账比较算法

公开入口：

```text
比较用途观察同账(
    const 用途观察纯值候选& 新候选,
    const 用途观察纯值候选& 既有候选)
-> 用途观察同账比较结果。
```

没有既有候选时不调用本入口；形成结果仍只表示新候选可作为调用期材料，不表示已发布。

顺序固定：

1. 分别检查两个候选的格式版本非零；格式 0 属于候选自身非法，新候选或既有候选分别返回内部不一致。
2. 两个格式均非零后再判兼容；任一非零未知格式即 `不支持 / 格式版本不支持`。
3. 分别检查两个候选自身属于 5.2—5.5 合法合同；新候选非法或既有候选非法分别返回内部不一致，不进入身份或阶段比较。
4. 完整任务句柄或任务序号不同：`不同身份 / 完整身份不同`。
5. 身份相同且不可变核心、阶段负载全部相同：`同账幂等 / 完整内容相同`。
6. 身份与不可变核心相同、既有为待结算、新值为合法最终已结算：`同账允许推进 / 待结算推进最终`。
7. 身份相同但不可变核心不同：`同账冲突 / 不可变核心不同`。
8. 身份与核心相同但发生终态变化、逆向或其它不允许阶段转换：`同账冲突 / 阶段转换不允许`。

允许推进方向只有 `待结算 -> 最终已结算`。尝试未达成和最终已结算均为终态。哈希不参与 O1-V1 比较；不得新增 O1 哈希或用 #287 因果哈希代替完整 O1 身份 / 内容比较。

## 10. 非成功返回唯一分类

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 上游合法入口拒绝组合 | 逻辑内返回 | 强类型入口拒绝，无候选、零结构变化 |
| 上游合法不支持或目标格式不支持 | 逻辑内不支持 | 不猜测兼容 |
| 两个合法候选身份不同 | 正常分流 | 不同账项，不进入同账内容比较 |
| 两个合法候选同身份但核心不同或阶段转换不允许 | 逻辑内冲突 | 同账冲突，零结构变化 |
| 上游组合非法，成功事件身份 / 内容 / 阶段矛盾 | 追根因解决 | 内部不一致，停止后继 |
| 新候选或既有候选自身非法 | 追根因解决 | 候选合法性先行，不进入同账比较 |
| 最终阶段缺正式结算材料、材料不完整或与来源矛盾 | 追根因解决 | 阶段材料矛盾；不得降格为冲突 |
| 形成输出与已接受来源不一致 | 追根因解决 | 算法不变量破坏，不输出半候选 |

O1-V1 不开始结构写入，因此不存在写后失败、撤销或发布成功返回。任何文档或代码若把 O1-V1 描述为写入、发布、撤销或恢复，属于范围越界。
