## 2. 定位

实例状态和实例动态是场景内临时零散节点，必须由状态服务和动态服务管理生命周期。场景服务负责收纳，但不得替代状态服务或动态服务删除、失效或修复材料。

生命周期治理目标：

```text
失效后不再作为有效材料读取。
被引用时阻断删除或失效。
拒绝后不改变结构。
不靠日志、显示或扫描器修复结构。
```

## 3. 实例状态生命周期

实例状态有效读取必须满足：

```text
状态句柄有效。
节点类型为实例状态。
存在主体有效。
场景归属有效。
状态值存在。
发生时间戳存在。
未被后续确认的失效结构标记为失效。
```

实例状态删除或失效必须阻断：

```text
被实例动态引用。
被二次特征组成引用。
被因果候选或需求结算引用。
被任务目标 / 结果或方法条件 / 结果引用。
```

第一轮不实现 `删除实例状态(状态)`。

## 4. 实例动态生命周期

实例动态有效读取必须满足：

```text
动态句柄有效。
节点类型为实例动态。
存在主体有效。
场景归属有效。
被改变目标有效。
改变前值存在。
改变后值存在。
发生时间戳存在。
来源动作为空或为动作入口角色节点。
```

实例动态删除或失效必须阻断：

```text
被轻量因果引用用作准入证据。
被因果模板候选用作动态证据。
被二次特征组成引用。
被抽象动态候选引用。
被需求结算记录引用。
被方法执行结果或任务推进证据引用。
```

第一轮不实现 `删除实例动态(动态)`。

## 5. 关系清理边界

后续若确认删除或失效写入口，只能清理该服务明确拥有的关系。

允许候选：

```text
状态服务拥有的状态到存在 / 场景 / 抽象状态关系。
动态服务拥有的动态到存在 / 场景 / 目标 / 来源动作关系。
场景服务拥有的临时收纳关系，且必须经对应服务确认。
```

禁止候选：

```text
二次特征组成关系，除非二次特征生命周期专项确认。
因果证据关系，除非因果治理专项确认。
需求、任务、方法和结算记录关系，除非对应高级服务专项确认。
未知来源关系。
SQL、显示、日志或控制面板镜像材料。
```

## 6. 入口拒绝矩阵

| 编号 | 拒绝项 | 处理 |
| --- | --- | --- |
| BASE-LIFE-01 | 状态或动态句柄无效 | 拒绝，不写结构 |
| BASE-LIFE-02 | 节点类型错误 | 拒绝，不写结构 |
| BASE-LIFE-03 | 缺主体、场景或发生时间戳 | 读取拒绝，不补结构 |
| BASE-LIFE-04 | 状态缺值 | 读取拒绝，不补状态值 |
| BASE-LIFE-05 | 动态缺目标、前值或后值 | 读取拒绝，不补动态材料 |
| BASE-LIFE-06 | 来源动作非法 | 拒绝动作动态材料 |
| BASE-LIFE-07 | 存在二次特征、因果、需求、任务、方法或结算引用 | 拒绝删除或失效 |
| BASE-LIFE-08 | 请求级联删除未确认结构 | 拒绝级联 |
| BASE-LIFE-09 | 请求从日志、显示、SQL、控制面板、外设或线程触发生命周期变更 | 拒绝越权来源 |

拒绝后节点、主信息、关系、索引、状态、动态、二次特征、因果、需求、任务、方法和结算记录不变化。

## 7. 验收矩阵

| 编号 | 验收项 | 通过标准 |
| --- | --- | --- |
| BASE-LIFE-V01 | 状态有效读取 | 完整实例状态可读出主体、场景、值、时间戳 |
| BASE-LIFE-V02 | 动态有效读取 | 完整实例动态可读出主体、场景、目标、前后值、时间戳 |
| BASE-LIFE-V03 | 不完整材料拒绝 | 缺必填材料时返回空或拒绝，不补结构 |
| BASE-LIFE-V04 | 引用阻断 | 被引用状态 / 动态不得直接删除或失效 |
| BASE-LIFE-V05 | 级联拒绝 | 未确认级联删除请求被拒绝 |
| BASE-LIFE-V06 | 拒绝后结构不变化 | 拒绝后核心结构数量不变 |
| BASE-LIFE-V07 | 外部来源排除 | 日志、显示、SQL、控制面板、D455、体素、外设和线程不能触发生命周期变更 |
