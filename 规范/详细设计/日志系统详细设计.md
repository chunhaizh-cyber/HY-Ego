# 日志系统详细设计

编号：LOG-001-D1
日期：2026-07-05
状态：已建立详细设计 / 第一轮代码已实现并验证

## 1. 对应规范

本详细设计对应：

```text
规范/日志系统规范.md
规范/0050_项目通用机器逻辑与禁止性规则总纲_20260721.md
规范/4050_子规范_入口拒绝逻辑内结果与内部逻辑错误.md
```

日志系统只提供人读诊断、运行观察、调试定位和后续审计材料入口，不承载机器事实，不参与需求满足、任务完成、方法成功、世界事实、结构修复或业务状态裁决。

## 2. 文件边界

第一轮日志文件固定放在：

```text
D:\海中鱼巣\日志
```

第一轮独立文件：

| 日志 | 文件名 | 启用条件 | 边界 |
| --- | --- | --- | --- |
| 逻辑错误日志 | `逻辑错误.log` | 容错检查宏开启并触发逻辑错误诊断 | 只记录人读错误原因，不改变返回值 |
| 运行日志 | `运行.log` | 运行日志入口被显式调用 | 只记录启动、关闭和阶段摘要，不作为结构事实 |
| 调试日志 | `调试_<切片名>.log` | 对应 `HY_EGO_DEBUG_LOG_<切片名>` 宏开启 | 默认关闭，只服务具体排查切片 |
| 事件日志 | `事件.log` | 事件日志入口被显式调用 | 第一轮只作审计材料入口，不作恢复和裁决 |

## 3. 公共代码入口

| 函数 / 宏 | 功能 | 主要用途 | 输入 | 输出 | 入口拒绝 | 验证方式 |
| --- | --- | --- | --- | --- | --- | --- |
| `日志目录路径()` | 返回日志目录 | 统一路径 | 无 | 路径 | 无 | 路径为 `D:\海中鱼巣\日志` |
| `写入日志(类别, 模块, 入口, 内容, 切片)` | 写入指定类别日志 | 公共底层写入 | 日志类别、模块、入口、内容、可选切片 | 局部 bool | 未知类别、调试切片为空、目录或文件无法打开 | 文件存在且类别隔离 |
| `记录逻辑错误日志(模块, 入口, 内容)` | 写逻辑错误日志 | 容错检查诊断 | 模块、入口、内容 | 局部 bool | 内容为空时拒绝 | `逻辑错误.log` 有记录 |
| `记录运行日志(模块, 入口, 内容)` | 写运行日志 | 启动、结束、阶段摘要 | 模块、入口、内容 | 局部 bool | 内容为空时拒绝 | `运行.log` 有记录 |
| `记录事件日志(模块, 入口, 内容)` | 写事件日志 | 后续审计材料入口 | 模块、入口、内容 | 局部 bool | 内容为空时拒绝 | `事件.log` 有记录 |
| `HY_EGO_DEBUG_LOG_MIN_LOOP` | 最小闭环调试日志切片宏 | 验收排查 | 编译宏 | 无 | 默认关闭 | 默认运行不生成调试日志，宏开启编译通过 |
| `海中鱼巣记录最小闭环调试日志(模块, 入口, 内容)` | 宏控制调试日志入口 | 最小闭环切片调试 | 模块、入口、内容 | 无 | 宏关闭时空操作 | 宏关闭不写文件 |

## 4. 容错检查接入

容错检查仍由以下宏控制：

```text
HY_EGO_ENABLE_FAULT_TOLERANCE_CHECK
```

宏关闭时：

```text
海中鱼巣容错检查 不写日志。
海中鱼巣容错检查 不弹窗。
海中鱼巣容错检查 不改变返回值和结构。
```

宏开启且条件不满足时：

```text
调用 记录逻辑错误日志。
调用弹窗提醒。
输出调试字符串。
不改变调用方返回值，不修复结构。
```

## 5. 实现方法

第一轮用头文件内联入口实现，避免引入持久化层或后台线程：

```text
创建日志目录。
按日志类别选择独立文件。
追加写入时间、模块、入口和内容。
文件写入结果只作为局部 bool 返回，不进入机器事实。
```

日志写入失败不得改变业务路径，不得尝试用备用机器状态补偿。
