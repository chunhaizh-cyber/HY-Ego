## 1. 定位

本设计承接：

```text
流程图/20260711_概念活动实例支持权威失效与投影推进流程图_v0.1.md

```

目标是在不删除原支持关系、不放宽候选完整性、不伪造无支持样本的前提下，建立：

```text
有效实例支持
-> 关系记录原地失效并保留审计
-> 活动快照移除有效投影
-> 活动版本推进
-> 生命周期候选重新计算
-> #197 活跃到冷却前置真实可达
```

本设计不自动判断实例何时失效，不接场景销毁、外设、时间轮或恢复，不实现概念生命周期转换本身；生命周期转换仍由修订后的 #197 负责。

## 2. 漂移事实

### 2.1 已证明

```text
概念候选项::完整() 对普通创建候选强制来源实例组非空。
候选图发布要求每个来源实例在候选图中恰有一条指向该候选的实例支持边。
发布后实例支持概念关系进入活动快照。
当前没有支持失效、解绑或审计状态入口。
#195 生命周期候选把任一有效实例支持视为活性依据。
原 #195 自检找不到无支持普通概念时回退到存在根，A06 只检查候选非未定义，因此不能证明无支持普通概念路径。
```

### 2.2 结论

原 #197 设计的生命周期规则不需要改义：任一有效实例支持仍应阻止冷却 / 退役。不可达的根因是支持关系只有创建和删除，没有“保留历史但不再有效”的生命周期状态。

不能采用：

```text
放宽概念候选来源实例非空约束。
测试代码直接删除支持关系。
从活动快照裸移除边但保持关系有效。
把存在根或其它根节点冒充无支持普通概念。
让统计缓存或测试 bool 覆盖支持事实。
```

## 3. 权威模型

### 3.1 记录状态扩展

```text
记录状态::无效 = 0
记录状态::有效 = 1
记录状态::已删除 = 2
记录状态::已失效 = 3
```

`已失效` 与 `已删除` 不同：

| 状态 | 普通读取 | 审计读取 | 可作为当前结构 | 关系编号 |
| --- | --- | --- | --- | --- |
| 有效 | 可读 | 可读 | 是 | 保持 |
| 已失效 | 不可读 | 可读 | 否 | 保持 |
| 已删除 | 不可读 | 后续恢复专项裁决 | 否 | 保持 |

本切片只允许把 `实例支持概念` 的有效关系转换为已失效。通用仓库入口提供值式状态变更能力，但其它领域不得未经专项计划调用。

### 3.2 关系版本

失效操作：

```text
关系编号不变。
源节点、目标节点、类型和顺序号不变。
版本号加一。
状态从有效变为已失效。
旧句柄变陈旧；新句柄只可通过审计读取。
```

旧失效关系不原地重新激活。未来同一实例重新支持同一概念时，由正式概念发布路径创建新的有效关系；历史失效关系继续审计可读。

## 4. 关系仓库接口

### 4.1 值式结果

```text
关系状态变更状态
  已变更
  已在目标状态
  无效句柄
  状态转换不允许
  版本已耗尽
  内部不一致

关系状态变更材料
  原关系句柄
  当前关系句柄
  类型
  源节点
  目标节点
  原状态
  当前状态
```

### 4.2 入口

```text
关系仓库::失效关系(关系)
关系仓库::读取关系审计(当前关系句柄)
关系仓库::获取关系审计记录组(源节点, 类型)
关系仓库::获取节点相关关系记录组(节点)
关系仓库::重挂关系并返回新句柄(关系, 新源节点, 新目标节点)
```

后两个入口吸收 #197 草案中已经形成、但尚未提交的可复用仓库能力，使修订后的 #197 不再修改关系仓库。

普通 `读取关系`、正向 / 反向节点组、关系记录组、存在关系和有效关系数量继续只返回 `有效`。审计接口必须显式命名，禁止普通业务读取误把失效历史当当前关系。

### 4.3 创建与重复

`创建关系` 的重复判断只针对当前有效关系。某对实例 / 概念存在已失效历史时，未来允许创建新的有效支持关系；同一时刻仍最多一个完全相同的有效关系。

## 5. 支持失效请求

```text
实例支持失效请求
  实例
  概念
  当前支持关系
  预期活动版本

实例支持失效状态
  已失效
  已经失效
  支持不存在
  请求已过期
  已拒绝
  内部不一致

实例支持失效材料
  实例
  概念
  原支持关系
  当前审计关系
  原活动版本
  当前活动版本
  状态
```

第一版请求是显式领域命令。它必须指向当前活动快照中的精确有效支持关系，并携带预期活动版本。它不自动从时间、日志、统计或外部文本推断失效原因。

入口候选：

```text
概念图服务::失效活动实例支持(实例支持失效请求)
概念图服务::读取实例支持审计组(实例, 概念)
```

## 6. 活动快照推进

### 6.1 写前候选

在 `图写锁_ -> 活动图锁_` 的既有顺序内：

```text
重读请求活动版本。
重读快照中的精确支持边。
重读关系仓库有效关系和概念图专用登记。
确认实例与概念类别一致，概念不是实例自身。
复制当前活动快照和有效专用登记组。
从副本中移除精确支持边和登记项。
活动版本加一。
清除上一候选图幂等材料，防止旧候选跨版本复用。
规范化并验证副本。
```

活动快照完整性不要求每个普通概念仍有支持；概念形成时的候选完整性继续要求来源实例非空。两者是不同阶段，不能把生命周期后的零有效支持反向解释为概念从未合法形成。

### 6.2 提交

```text
关系仓库失效精确支持关系。
读回新版本审计记录，并证明普通读取已消失。
用预先构造的副本做无分配交换，发布新活动快照和有效登记组。
重置上一候选图。
重读活动版本、活动关系组、正反支持投影和审计历史。
```

活动版本推进使 #195 缓存命名空间和生命周期候选自然过期。任何旧候选不得跨支持失效版本提交 #197 转换。

### 6.3 幂等

请求的活动版本已经过期时普通返回请求过期。若按同端点审计读取确认该关系已经失效，允许返回 `已经失效`，但不得再次推进关系或活动版本。

## 7. #197 接线

修订后的 #197 必须：

```text
依赖 #210 正式完成。
不再修改关系仓库。
只以当前活动快照中的有效实例支持数量判定活性。
自检对真实普通概念逐条调用正式支持失效入口，直到有效支持为零。
重新生成 #195 候选，确认活动版本和缓存来源版本均已更新。
在无其它外部引用 / 有效用途时验证活跃到冷却。
新支持关系重新出现后再验证重新激活。
```

原 #197 未提交草案中的生命周期转换代码不能进入 #210 提交。关系新句柄和相关关系读取部分只有在符合本设计接口、通过 #210 验收时才可归入 #210。

## 8. 非成功返回二分

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 无效句柄、类型不匹配、非当前活动边或活动版本过期 | 逻辑内返回 | 不改关系或快照 |
| 同一关系已经失效且审计完整 | 幂等逻辑内返回 | 不推进版本 |
| 请求关系不存在且没有审计历史 | 逻辑内返回 | 返回支持不存在 |
| 当前快照、登记和关系仓库对同一边互相矛盾 | 追根因解决 | 停止依赖路径 |
| 写前新快照不能通过结构验证 | 追根因解决 | 尚未写仓库，保持旧事实 |
| 关系失效后审计、版本或普通读取不一致 | 追根因解决 | 停止活动图和生命周期依赖 |
| 无分配发布后活动版本或投影不一致 | 追根因解决 | 停止候选、转换和显示依赖 |
| #197 目标概念仍有其它有效支持 | 逻辑内返回 | 继续阻止冷却 / 退役 |

## 9. 并发与锁

```text
概念支持创建、候选发布、支持失效和生命周期转换继续共用图写串行边界。
活动快照候选在写前完整构造。
关系仓库独占锁内只做单关系状态转换，不回调概念图服务。
概念图锁内不得调用日志、显示、SQL、外设或用户回调。
只读关系接口返回值副本，不暴露状态可写引用。
```

同一支持的并发失效只有一个 `已失效`，其余为过期或 `已经失效`；不同支持按图写边界串行，第一版不做分片写。

## 10. 恢复边界

```text
权威恢复候选：有效 / 已失效关系记录、版本、端点和活动快照版本。
可重建：有效专用登记、活动支持正反投影。
必须丢弃：调用期失效请求和返回材料。
```

当前没有关系状态或活动快照恢复，本切片不声明跨重启保留。#202 必须纳入 `已失效` 状态和防复活规则。
