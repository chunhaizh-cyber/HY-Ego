# 不透明结构写入会话概念图只读材料扩展详细设计

日期：2026-07-14

状态：正式详细设计 / JY-331 / #276 施工依据 / 代码未实施

## 1. 目标

在不改变四仓库权威边界、结构事务 ABI、独占许可强度和既有撤销顺序的前提下，为 `结构写入会话` 增加 `#275` 所需的两个通用只读材料入口：

```text
同令牌读取节点的完整主键组。
同令牌按目标节点和关系类型读取来源关系记录组。
```

本设计不建立概念根、概念类别、反向索引、防环算法或任何领域语义，不把仓库或原始令牌暴露给数据操作层。

## 2. 当前代码事实

### 2.1 已有会话能力

`#272 / CORE-SESSION-S2` 已实现并验证：

```text
读取节点类型。
读取节点主信息。
读取主信息 I64 值。
按源节点和关系类型读取当前关系记录组。
复核指定主键到预期节点的绑定。
```

`结构写入会话` 只在执行器持有一次独占许可时、当前线程栈内同步使用；普通读取入口在请求提交或撤销后继续拒绝。

### 2.2 实际接口缺口

当前没有：

```cpp
std::vector<std::uint64_t> 读取节点主键组(节点句柄 节点) const;
std::vector<关系记录> 读取来源关系记录组(
    节点句柄 目标节点,
    关系类型 类型) const;
```

现有 `主键绑定匹配` 只回答一个指定键是否映射到节点，且不匹配会登记会话内部失败；它不能表达“当前节点全部主键组”的写前准入材料。现有 `读取关系记录组` 只按源节点读取；它不能证明根没有上位关系，也不能返回概念直接上位或实例反向支持关系。

### 2.3 仓库已有窄入口

当前仓库已经提供带 `结构事务令牌` 的只读入口：

```cpp
索引仓库::读取节点主键组(节点句柄, const 结构事务令牌&) const;
关系仓库::获取来源关系记录组(节点句柄, 关系类型, const 结构事务令牌&) const;
```

因此本切片只需在会话内封装这两个入口，不修改仓库 ABI、仓库锁或事务协调器。

## 3. 公开能力合同

### 3.1 节点主键组

新增：

```cpp
std::vector<std::uint64_t> 读取节点主键组(节点句柄 节点) const;
```

入口只在当前线程、写入阶段、未请求提交 / 撤销、无首次失败且节点句柄有效时调用索引仓库带同令牌入口。返回普通 `std::vector<std::uint64_t>` 值副本，不返回索引记录、仓库引用或令牌。

会话只返回材料，不解释唯一性。调用数据操作层必须继续复核：

```text
预期稳定键非零。
主键组恰好一个且等于预期稳定键。
该键按主键查询仍精确映射到同一完整节点句柄。
```

当前会话已有 `主键绑定匹配` 可完成最后一项，但数据操作层只能在确认主键组唯一后调用；不能用它替代主键组读取。

### 3.2 按目标来源关系记录组

新增：

```cpp
std::vector<关系记录> 读取来源关系记录组(
    节点句柄 目标节点,
    关系类型 类型) const;
```

入口在同一会话令牌下调用关系仓库现有 `获取来源关系记录组`，沿用仓库稳定顺序并返回完整记录值副本。调用数据操作层必须逐项复核：

```text
关系编号和版本可构成有效完整关系句柄。
记录状态为有效。
记录类型等于请求类型。
记录目标节点等于请求目标。
源节点和目标节点完整且当前可读。
同一完整关系不得重复出现。
```

该入口与既有按源 `读取关系记录组` 组合后，可以在同一独占许可内完成正向遍历、反向读取、根无上位关系复核和完整关系互证。

## 4. 非成功语义

```text
无效句柄、跨线程、错误阶段、已有首次失败：逻辑内返回空材料，结构不变化。
当前没有主键或没有指定来源关系：逻辑内返回空材料，由数据操作层按业务上下文解释。
调用方已掌握应存在的完整结构，但正反向记录、主键映射、版本、状态或端点不一致：追根因，禁止继续请求提交。
```

会话层不得把空材料自动解释为根、概念、无父或不存在；领域语义只留在后继数据操作层和业务服务。

## 5. 写集、提交与撤销

两个新入口都是只读入口：

```text
不新增写集。
不改变 写集已完整读回() 条件。
不把读取结果保存为许可释放后的写入授权。
不改变请求提交、完成提交和完成撤销顺序。
```

若执行器回调未请求提交、显式撤销、发生异常或其它写入失败，仍按现有索引、关系、节点候选、主信息候选顺序收口。只读材料不能参与补偿或修复结构。

## 6. 文件与模块

本切片不新增生产模块，只修改现有真模块和自检：

| 文件 | 分类 | 变化 |
| --- | --- | --- |
| `海中鱼巣/核心/会话.结构写入.ixx` | 现有生产真模块 | 增加两个同许可值式只读入口 |
| `海中鱼巣/核心/自检.结构写入会话.ixx` | 现有自检真模块 | 增加正反向、主键组、拒绝、并发隔离和回归验收 |
| `海中鱼巣/入口.cpp` | 最小入口 | 只增加一次阶段 752 登记和总结果汇总 |

不修改关系仓库、索引仓库、节点仓库、主信息仓库、执行器、事务接线、协调器、工程文件或任何领域模块。

## 7. 调用路径

```text
后继数据操作层
-> 结构写入执行器
-> 一次独占许可
-> 结构写入会话
-> 节点主键组 / 按目标来源关系记录组值式读取
-> 与既有节点、主信息、按源关系材料组合复核
-> 请求提交或沿现有路径撤销
-> 释放许可
-> 值式业务结果
```

领域服务、组合器、线程、消息处理器和显示层不得直接导入会话或执行器。

## 8. 并发与锁

```text
许可强度继续为独占。
不增加仓库锁、会话锁或业务锁。
不释放许可后再执行防环或反向复核。
其它线程共享读取继续在独占许可释放前等待。
同线程无令牌仓库重入继续拒绝。
```

本批不通过缓存、二级索引或投影加速关系遍历。后续仓库二级索引只能提供候选，仍须由关系记录完整复核。

## 9. 验收矩阵

| 编号 | 验收 |

| A01 | 两个新入口公开面不含原始令牌、许可、仓库引用、锁、候选或可变指针 |

完成后只能声明：

```text
结构写入会话已具备同一独占许可内的节点主键组和按目标来源关系记录组值式读取能力。
#275 所需的根无上位、稳定键唯一性、反向查询和防环材料入口已经具备，仍须由 #275 实际接口复核后才能实施领域路径。
```

```text
概念根已经建立或登记。
概念上下位、防环或三类概念关系已经实现。
关系仓库二级索引、生命周期、统计、删除或生产调用迁移已经完成。
#275、#269-#271、#214 或 #257 已恢复或完成。
```
