## 3. 目标结构

### 3.1 文件类型二分

| 类型 | 文件形态 | 主要职责 | 工程登记 | 调用方式 |
| --- | --- | --- | --- | --- |
| 数据结构 | `功能.分类.h/.cpp` | 稳定类型、记录 / 容器布局和结构自身不变式 | `.h` 为 `ClInclude`，`.cpp` 为 `ClCompile` | `#include` |
| 模块 | `功能.分类.ixx` | 服务、协议、初始化、线程、桥、适配、显示、自检或编排 | `ClCompile` | `import` |

判定以文件主要职责为准，不以是否出现 `struct`、是否只有一个类、是否需要模板为准。

### 3.2 模块身份

模块身份由路径确定：

```text
海中鱼巣/<目录>/<功能>.<分类>.ixx
-> export module 海中鱼巣.<目录>.<功能>.<分类>;
```

工程项目、筛选器、模块声明和调用 `import` 必须使用同一路径语义。不得同时保留新旧两个模块名作为兼容壳，除非专用迁移计划明确证明兼容需求和移除时点。

### 3.3 生产与验收二分

每个新功能至少先判定两种职责：

```text
生产职责：正式入口、值类型、协议和运行行为。
验收职责：测试材料、并发压力、断言、逐项输出和回归聚合。
```

两者必须落入不同模块。自检模块只通过生产公开入口验证，不得成为生产模块的反向依赖。

依赖方向固定为：

```text
数据结构 / 领域公开头
-> 生产模块
-> 自检模块
-> 自检运行器或入口过渡调用
```

禁止：

```text
生产模块 import 自检模块。
领域服务 include 自检模块。
入口.cpp 重新展开自检实现。
为了测试导出可写生产内部状态。
```

## 4. 入口边界

### 4.1 长期目标

`入口.cpp` 的长期职责收束为：

```text
参数解析。
运行模式选择。
运行期上下文或顶层组合模块构造。
调用统一自检运行器或正常运行入口。
停止、资源回收和退出码映射。
```

领域自检模块通过统一的 `自检.运行器` 聚合；入口只调用一次并接收总报告。

### 4.2 过渡边界

统一运行器尚未形成时，允许新切片在入口增加：

```text
一个或少量模块 import。
一个小型值式 / 引用式自检上下文组装。
一次自检模块调用。
一个总通过布尔值并入既有总退出条件。
```

禁止增加：

```text
测试邮箱、测试仓库或测试线程的逐步构造正文。
A01-Axx 每项计算正文。
并发线程创建 / join 循环正文。
逐项 std::cout 或 输出验收项 调用正文。
碰撞、过期、漂移等场景展开。
```

### 5.2 自检上下文

`自检.概念命名治理` 可以定义仅供自检使用的上下文和报告类型。上下文只接收 A01-A12 已经需要的服务引用、仓库只读观察引用、固定样本句柄和既有回归结果；不得让入口传入测试步骤函数或业务判定回调。

建议形状：

```text
概念命名治理自检上下文
  概念图服务引用
  语素服务引用
  节点 / 关系 / 索引只读观察引用
  请求消失样本句柄
  已有名称样本句柄
  旧八类消息、邮箱、自我线程和任务桥回归证据

概念命名治理自检报告
  A01 至 A12
  总通过
```

精确字段以执行时现有变量可达性复核为准，但入口增量必须保持“上下文组装 + 一次调用 + 总结果汇总”，不得因为上下文字段不便而把测试正文留在入口。

### 5.3 自检模块行为

自检模块内部负责：

```text
读取当前两个或以上待命名请求。
构造邮箱和上行桥测试实例。
验证协议、专用白名单、旧八类兼容、单请求提交和批量不自动提交。
验证请求消失、已有名称、版本 / 根漂移、过期、重复和完整字段碰撞。
完成并发同请求去重和不同概念不丢失测试。
输出 A01-A12 人读结果并返回总通过。
```

自检模块不得写节点、主信息、关系、索引或概念活动图；它只允许写自检私有邮箱。结构数量和活动版本前后必须一致。

### 5.4 工程与入口

工程登记：

```text
两份新 .ixx 均为 ClCompile。
filters 同步到线程分类。
旧概念命名治理上行桥路径不得保留。
```

入口过渡调用：

```text
import 海中鱼巣.线程.上行桥.概念命名治理;
import 海中鱼巣.线程.自检.概念命名治理;
```

生产桥可由自检模块直接 import。入口若只为自检使用生产桥，可以只 import 自检模块；执行时以 MSVC 模块可见性和现有调用点为准，禁止重复 import 之外的兼容包装。

## 8. 非成功收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 新文件无法明确归为数据结构或模块 | 逻辑内返回 | 不建文件，退回计划补职责 |
| 文件名、模块名、工程登记或 import 不一致 | 逻辑内返回 | 不进入实现或构建，先修订四处映射 |
| 旧计划要求入口新增自检正文 | 逻辑内返回 | 不执行该范围，退回设计修订 |
| 模块已进入工程后出现声明重复、依赖环或 BMI 身份矛盾 | 追根因解决 | 停止依赖路径，修复真实模块图，不用头文件兼容壳掩盖 |
| 自检通过但生产 / 自检职责混写 | 逻辑内返回 | 验收不通过，拆分后重验 |
