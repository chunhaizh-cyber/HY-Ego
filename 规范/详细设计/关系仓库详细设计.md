# 关系仓库详细设计

编号：011-D1
日期：2026-07-05
状态：已确认详细设计 / 核心内核 P0 / 语素多目标关系口径已确认 / 可作为第一轮代码实施依据

## 1. 设计目标

关系仓库负责节点之间的结构连接。关系仓库必须在写入口拒绝悬空关系、无效句柄、已删除节点、错误关系类型和不允许的重挂路径。

关系不是业务字段，也不是显示层快捷索引。业务含义必须通过明确关系类型、领域服务写入入口和读取规则表达。

第一轮关系仓库的目标是提供最小关系记录、关系句柄、关系创建、普通父子重挂、按关系类型读取单个兼容目标、按关系类型读取目标集合、枚举父子和端点有效性校验能力。

## 2. 规范承接

| 上层规则 | 本设计承接 |
| --- | --- |
| 关系边界 | 关系类型显式，不把所有关系混成普通父子 |
| 入口前置拒绝 | 写入前拒绝无效源节点、无效目标节点、旧版本、已删除节点和非法关系类型 |
| 结构闭环 | 语义连接使用关系记录承载，不用主信息字段、显示文本或日志承载 |
| 领域服务封口 | 业务关系由领域服务或正式仓库入口写入，外部模块不得裸写关系表 |
| 版本语义 | 关系保存端点句柄版本，读取和写入必须校验当前有效性 |

## 3. 关系类型

第一版关系类型：

```text
普通父子
归属
引用
索引
模板
运行期临时
因果来源
语素对应信息
语素概念追溯
```

关系类型含义：

| 关系类型 | 第一轮用途 | 多目标口径 |
| --- | --- | --- |
| 普通父子 | 表达树形或层级父子 | 同一子节点同时只能有一个有效父节点 |
| 归属 | 表达对象归属于场景、存在或容器 | 是否多目标由对应领域规范确认 |
| 引用 | 表达非拥有引用 | 允许多目标前必须由具体服务确认 |
| 索引 | 表达索引项和目标之间的结构连接 | 第一轮优先由索引仓库管理 |
| 模板 | 表达方法、任务或特征模板关系 | 暂缓复杂模板推导 |
| 运行期临时 | 表达运行期临时连接 | 不进入持久事实裁决 |
| 因果来源 | 表达动态、方法、输入材料和前后值来源 | 因果服务写入和读取 |
| 语素对应信息 | 语素入口绑定内部信息节点 | 已确认支持同一语素入口多目标 |
| 语素概念追溯 | 语素入口追溯概念节点 | 已确认支持多级、多路目标 |

`高级信息` 不得成为关系类型或节点类型。需求、任务、方法只作为已有节点类型参与关系。

## 4. 结构设计

| 结构 | 用途 | 写入方 | 读取方 | 生命周期 | 验证方式 |
| --- | --- | --- | --- | --- | --- |
| `关系句柄` | 关系长期身份 | 关系仓库 | 领域服务、关系仓库 | 随关系记录生命周期 | 仓库编号、关系编号、版本号有效 |
| `关系记录` | 关系最小记录 | 关系仓库 | 关系仓库、授权领域服务 | 创建到删除 | 源和目标节点当前有效 |
| `关系编号` | 仓库内本地身份编号 | 关系仓库 | 关系仓库、持久化适配 | 仓库内只递增不复用 | `0` 无效，删除后不回收 |
| `关系类型` | 显式语义边界 | 领域服务 | 关系仓库 | 创建后稳定，重挂不改类型 | 非法关系类型入口拒绝 |
| `源节点` | 关系起点 | 领域服务 | 关系仓库 | 随关系记录生命周期 | 写入时校验当前版本 |
| `目标节点` | 关系终点 | 领域服务 | 关系仓库 | 随关系记录生命周期 | 写入时校验当前版本 |
| `顺序号` | 同源同类型枚举排序 | 关系仓库 | 关系仓库 | 随关系记录生命周期 | 枚举结果顺序稳定 |
| `有效状态` | 关系生命周期状态 | 关系仓库 | 关系仓库、领域服务 | 创建到删除 / 失效 | 失效后不作为当前关系返回 |

关系记录第一版至少表达：

```text
关系编号
关系类型
源节点句柄
目标节点句柄
顺序号
版本号
有效状态
```

## 5. 编号和版本语义

```text
关系编号使用 uint64 语义。
0 保留为无效关系编号。
有效编号从 1 开始，仓库内单调递增，不复用。
创建关系时版本号 = 1。
删除或失效关系时递增关系版本号。
重挂普通父子关系时，旧父关系失效，新父关系新建或以明确重挂入口更新。
关系端点保存节点句柄，写入和读取时必须校验节点当前版本。
```

普通业务状态变化不递增关系版本。关系版本只用于关系记录生命周期和旧关系句柄失效。

## 6. 函数设计

| 函数 | 功能 | 入口拒绝 | 读写结构 | 验证方式 | 状态 |
| --- | --- | --- | --- | --- | --- |
| `创建关系` | 创建源到目标的指定关系 | 源节点无效、目标节点无效、关系类型非法、源 / 目标版本失效、唯一性冲突 | 写关系表 | 无效端点返回无效关系句柄 | 已实现（文档声明）/ 设计已确认 / 待复验 |
| `读取关系` | 按关系句柄读取有效关系记录 | 关系句柄无效、仓库编号不匹配、版本不匹配、已失效 | 读关系表 | 删除或失效后返回空 | 第一轮应具备 / 待复验 |
| `删除关系` | 使关系记录失效并递增版本 | 无效关系句柄、版本不匹配、已失效 | 写关系状态和版本 | 旧关系句柄不可读 | 第一轮应具备 / 待复验 |
| `获取父节点` | 读取普通父子关系的父节点 | 子节点无效 | 读关系表 | 重挂后返回新父节点 | 已实现（文档声明）/ 设计已确认 / 待复验 |
| `获取子节点` | 枚举普通父子子节点 | 父节点无效 | 读关系表 | 子节点数量和顺序符合预期 | 已实现（文档声明）/ 设计已确认 / 待复验 |
| `重挂节点` | 改变节点父关系 | 子节点无效、新父节点无效、会产生悬空关系、会违反唯一父节点约束 | 写关系表 | 当前父节点更新且旧父关系失效 | 已实现（文档声明）/ 设计已确认 / 待复验 |
| `获取目标节点` | 按源节点和关系类型读取第一个兼容目标 | 源节点无效、关系类型非法 | 读关系表 | 单目标旧调用仍可读到第一个已绑定目标；不得用于裁决多目标事实 | 已实现 / 设计已确认 / 本切片复验 |
| `获取目标节点组` | 按源节点和关系类型返回目标集合 | 源节点无效、关系类型非法 | 读关系表 | 多目标全部返回，不静默选第一个 | 本切片新增 / 已验证 |
| `枚举来源节点` | 按目标节点和关系类型反查来源集合 | 目标节点无效、关系类型非法 | 读关系表 | 别名和因果来源可反查 | 后续 / 查询切片 |

## 7. 唯一性和冲突

第一版确认以下最小口径：

```text
普通父子关系同一子节点同时只能有一个有效父节点。
语素对应信息已由语素系统规范和主信息迁移口径确认可作为同一语素入口多目标关系。
语素概念追溯已由语素系统规范和主信息迁移口径确认可作为多级、多路目标关系。
其他同一源节点同一关系类型如果需要多目标，必须由具体系统规范确认。
未确认多目标语义前，不得用重复关系模拟列表字段。
```

冲突处理规则：

```text
若唯一关系已存在，再次创建同类型有效关系必须返回冲突或失败。
若允许多目标，读取函数必须返回集合或候选包，不得静默选第一个。
兼容单目标读取函数只能服务旧调用路径或单目标语义，不得作为语素多目标事实裁决入口。
若旧关系已失效，新关系可创建，但旧关系句柄不得继续命中。
```

## 8. 重挂规则

普通父子重挂必须满足：

```text
子节点有效。
新父节点有效。
子节点和新父节点不能违反领域服务禁止的结构约束。
旧父关系必须明确失效，不能保留为当前父关系。
重挂结果必须能由 `获取父节点` 和 `获取子节点` 读出。
```

第一版暂不实现复杂环检测。如果某类父子关系可能形成环，必须在对应领域服务入口先拒绝。

## 9. 入口前置拒绝

关系仓库必须在入口拒绝：

```text
源节点句柄无效。
目标节点句柄无效。
源节点或目标节点已删除。
源节点或目标节点版本不匹配。
关系类型未定义。
关系类型不允许当前源 / 目标节点类型组合。
唯一性关系已存在。
未确认多目标语义却尝试创建同源同类型多目标关系。
```

不得先写入悬空关系，再由扫描器修复。

## 10. 禁止行为

```text
不得创建悬空关系。
不得先写错误关系再扫描修复。
不得绕过领域服务裸写业务关系。
不得用普通父子关系混装所有语义。
不得让显示层写关系。
不得用关系替代索引候选包。
不得把多目标关系静默读成单一事实。
```

## 11. 验收条件

设计确认后，代码复验至少覆盖：

```text
无效源节点创建关系被拒绝。
无效目标节点创建关系被拒绝。
已删除节点不能作为关系端点。
版本失效节点不能作为关系端点。
普通父子同一子节点只能有一个当前父节点。
重挂后旧父关系不再作为当前父节点。
语素对应信息和语素概念追溯可按关系类型读取目标集合。
同一语素入口下多个语素对应信息目标都可读回。
同一语素入口下多个语素概念追溯目标都可读回。
多候选语义未确认时不得静默选择第一个目标。
```

## 12. 禁止宣称

本设计确认后仍不得宣称：

```text
关系仓库代码已验证通过。
复杂关系推导已完成。
旧项目世界树关系已迁移。
因果系统已完成。
最小闭环已跑通。
```
