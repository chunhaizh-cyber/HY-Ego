## 3. #231 已验证当前事实

1. 新版任务执行冻结请求和新版任务结果回执可共同形成非权威观察回合值式身份候选。
2. 当前没有权威任务尝试节点，也没有同一回合内的片段序号字段。
3. 动态不能反向权威推导任务、回执或结果；必须联合完整冻结请求、完整新版回执和结构读回复核。
4. 现有业务服务 / 数据操作读取各自取得共享许可，顺序调用不能排除混合快照。
5. 特征业务服务没有“特征值节点 -> 唯一槽位、宿主、语义定义、当前关系、原始版本”的公开反向入口。
6. 二次特征顺序号只表示单个结构内的位置，不能推断跨实例语义角色。
7. 正式任务执行链把实例动态的 `被改变目标` 写为任务目标抽象状态，不是特征值节点。
8. 实例动态关系槽固定为 `1=被改变目标、2=前状态、3=后状态、4=来源动作`。
9. 唯一运行期业务装配尚未持有新版任务执行调度组合器和任务结果治理组合器。
10. `服务.动态模式`、`算法.动态模式` 等目标模块和阶段 630 / 640 / 650 当前均不存在或未登记。

因此旧设计中的以下假定全部撤回：

```text
公开动态模式服务取得许可并向多个服务下传原始令牌。
动态目标必须是特征值并在 #232 内补特征反向语义解析。
二次特征角色可以由组成项顺序号推断。

新版任务执行 / 结果治理已经进入唯一运行期业务装配。
```

## 6. 输入与观察回合身份

### 6.1 输入请求

版本 1 请求至少包含：

```text
完整新版任务结果回执。
回执内嵌的完整任务执行冻结请求。
同回合非零片段序号。
非零且受支持的规则版本。
可选的同一回合多片段请求组；每个片段当前只绑定一个正式回执动态证据。
```

第一版不扫描场景、不猜测时间窗口，也不允许调用方只提交裸动态句柄冒充任务结果来源。

### 6.2 回合身份

回合身份完整比较使用新版回执及其完整冻结请求的值式字段，包括但不限于：

```text
请求编号、回执编号、幂等键、任务序号、批次编号。
任务、需求、场景、虚拟存在、目标状态完整句柄。
方法、动作入口、稳定动作键和冻结关系证据。
实际结果状态、动态证据及发生时间戳。
```

哈希可以用于分桶，但回合相等必须全字段比较。该身份只用于候选去重和排序，不成为权威节点、关系、索引或持久化身份。

### 6.3 片段序号

同一回合内片段序号必须：

```text
非零。
请求组内唯一。
由调用方显式给出。
只表示候选中的先后，不反向修改回执、动态或任务。
```

重复片段序号、同回合材料矛盾或同一动态跨片段重复，整体逻辑返回。

## 7. 单许可权威批量读取

### 7.1 许可所有权

`数据操作.动态模式` 构造时隐藏结构事务接线和四仓库引用。公开组合器 / 服务只持有该数据操作的窄业务依赖，不可访问内部接线或仓库。

每次读取顺序固定为：

```text
纯值协议前置检查
-> 一次取得共享许可
-> 获取私有 const 令牌引用
-> 批量读取全部节点、主信息、关系和主键材料
-> 在许可内形成完整值式批次
-> 释放许可
-> 服务与算法只消费值式批次
```

不得在许可内调用 `状态动态数据操作::读取动态材料` 等会再次取得许可的公开入口，不得把令牌保存到 DTO、成员、lambda 逃逸对象或其它线程。

### 7.2 批量读取范围

数据操作层至少复核：

```text
回执动态节点、动态主信息和动态主键。
动态关系槽 1 / 2 / 3 / 4 的完整关系句柄、版本、类型、源端点、目标端点和顺序号。
前 / 后实例状态节点、主信息、场景、主体、发生时间戳和值材料。
被改变目标节点、主信息和抽象状态值式材料。
来源动作节点、主信息、方法角色和动作入口材料。
冻结请求与回执携带的任务、需求、场景、主体、方法、动作及关系证据。
```

已经按正式生命周期失效的冻结关系，必须使用关系历史审计入口按完整句柄复核其原记录；仍要求当前有效的关系使用有效读取。设计不得把“已失效但可审计”误判为“从未存在”，也不得把历史关系重新解释为当前有效。

### 7.3 批次状态

```text
已找到：全部必需材料在同一许可期内完整且互相一致。
入口拒绝：协议、编号、句柄或数量前置无效，仓库零写入。
许可拒绝：无法取得共享许可，不产生半批次。
来源过期：句柄版本自然漂移、必需当前关系已失效或当前节点已删除。
不支持：目标不是版本 1 抽象状态正式链。
内部不一致：同一许可内出现类型、主信息、关系槽、端点、审计或协议证据矛盾。
```

`来源过期` 和 `不支持` 是逻辑内返回；`内部不一致` 必须追根因，不得选择第一个关系、使用旧缓存或顺序重读到偶然一致。

## 8. 业务复核与规范窗口

### 8.1 正式链复核

服务层要求：

```text
动态被改变目标 == 冻结任务目标状态。
目标节点为完整抽象状态，不是特征值、实例状态或二次特征。
动态来源动作 == 冻结动作入口。
动态场景 / 主体 == 冻结任务场景 / 虚拟存在。
动态后状态 == 回执实际结果状态。
动态前 / 后状态与同许可读取的实例状态材料一致。
回执字段、冻结请求字段和权威结构句柄逐项一致。
```

版本 1 不读取特征服务，不修改特征体系，也不把抽象状态值误写为特征值语义。

### 8.2 规范动态步

结构键版本 1 包含：

```text
被改变目标抽象状态完整句柄。
前实例状态值。
后实例状态值。
来源动作是否存在；正式任务执行链要求存在并保存完整动作句柄。
规则版本。
二次特征角色固定为空。
```

场景、主体、发生时间戳、来源动态句柄、回合身份、片段序号、任务、方法和关系证据保留为来源证据，不进入跨实例结构相等键。

### 8.3 排序与哈希

片段内部排序固定为：

```text
发生时间戳升序
-> 动态完整句柄的仓库编号、本地编号、版本升序
```

回合内片段只按显式片段序号升序。完整结构键全字段比较裁决相等；稳定哈希只用于分桶，碰撞不同键必须分开。

### 8.4 当前性声明

规范窗口保存“形成该窗口时同一共享许可期内读到的完整材料”。许可释放后：

```text
窗口仍可作为非权威历史候选参与 #233 / #234 纯值计算。
窗口不证明仓库此刻仍未变化。
需要当前业务裁决时必须重新进入正式数据操作入口。
不得为窗口发明全局快照编号、仓库版本或持续租约。
```

## 10. 并发、锁与一致性

1. 一次请求只取得一份共享许可，并在许可释放前完成全部权威读取和批次复制。
2. 许可持有期间其它线程的独占结构写入不得完成；自检必须证明写线程只在许可释放后完成。
3. 数据操作不得启动线程、`std::async`、后台任务或用户回调。
4. 许可释放后算法可以无锁处理不可变副本；输出排序不得依赖线程完成顺序。
5. 许可失败时不返回半批次；异常不得泄漏许可、令牌或仓库引用。
6. 本设计不宣称全局 MVCC、跨请求同版本快照或持续当前性。

## 11. 非成功返回二分

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| #231 正式接口或阶段 630 漂移 | 设计退回 | 不改代码、不改号、不扩大文件 |
| 协议版本错误、编号为零、回执与冻结请求不一致、片段序号重复 | 逻辑内返回 | 仓库零写入，尽可能仓库零读取 |
| 无法取得共享许可 | 逻辑内返回 | 无半批次、无重试忙等 |
| 句柄自然版本漂移、当前来源已失效 | 逻辑内返回 | 返回来源过期，不用旧缓存兜底 |
| 目标不是抽象状态正式链 | 逻辑内返回 | 版本 1 不支持，不调用特征服务 |
| 同一许可内关系端点 / 槽位、节点类型、主信息或协议证据矛盾 | 追根因解决 | 停止返回，不选首项、不伪造关系 |
| 历史关系可审计但不再有效 | 正常历史分支 | 只证明冻结时来源，不恢复为当前有效 |
| 哈希碰撞但完整键不同 | 正常算法分支 | 保持分离 |
| 算法确定性或计数不变量破坏 | 追根因解决 | 停止返回，不截断或日志掩盖 |

## 12. 后继 #233 / #234

## 13. 运行期、持久化和迁移边界


## 17. JY-364 #233 正式聚类设计

### 17.1 当前 DTO 纠偏

`规范动态窗口` 的稳定哈希混入回合、片段序号、发生时间戳和来源动态。该哈希用于证明窗口材料的确定封装，不能表达跨回合结构语义，也不能替代 `动态模式结构键相同`。

当前正式公开路径中，最小可独立复核的片段单位是一个完整 `规范动态步`。步骤携带一个完整回合身份、一个片段序号、一个结构键和一个来源动态。#233 版本 1 因此固定：

```text
一个规范动态步 = 一个单步片段实例。
一个规范动态窗口 = 一个或多个单步实例的值式容器。
多个步骤 / 窗口不自动拼接为多步片段。
不从时间接近、片段序号相邻或容器相邻推断序列边界。
```

这不是对多步运动基元的完成声明。多步连续性需要独立的片段边界、序列身份和时间合同，后续必须另建专项。

### 17.2 输入请求与容量

聚类请求只包含 `规范动态窗口` 组。输入必须满足：

```text
窗口组非空。
窗口数量 <= 动态模式最大片段数量。
全部步骤总数 <= 动态模式最大片段数量 * 动态模式最大片段数量。
每个窗口完整，每个步骤完整，规则版本一致且为版本 1。
```

窗口数和总步骤数上限必须由 `材料.动态模式` 以公开常量定义。容量拒绝是逻辑内返回，不允许算法静默截断、部分聚类或在内部使用未公开阈值。

### 17.3 候选材料

`材料.动态模式` 是聚类值式实体的唯一声明所有者，至少增加：

| 类型 | 内容 | 性质 |
| --- | --- | --- |
| 动态聚类请求 | 规范动态窗口组 | 纯值输入 |
| 单步片段实例候选 | 结构键哈希、完整结构键、完整来源步骤 | 非权威实例 |
| 重复动态聚类候选 | 完整结构键、去重来源实例、去重支持回合组 | 非权威重复候选 |
| 运动基元候选 | 重复候选身份、有效来源动作、来源实例组 | 非权威动作相关候选 |
| 动态聚类结果 | 处理状态、拒绝原因、三类候选组 | 纯值结果 |

候选不分配长期节点编号，不保存仓库、许可、令牌、会话、接线、锁、裸指针、回调或线程对象。局部容器位置也不得成为候选身份。

### 17.4 完整结构键

跨回合相等必须逐字段比较：

```text
被改变目标完整句柄
前状态值
后状态值
有来源动作
来源动作完整句柄
规则版本
二次特征角色
```

版本 1 要求 `有来源动作 == true`、来源动作句柄有效、二次特征角色为空。以下字段不进入结构键，但必须保留在来源实例中用于追溯和确定排序：

```text
回合、片段序号、发生时间戳、来源动态
场景、主体、任务、方法
来源批次及其节点 / 主信息 / 关系证据
```

场景、主体、任务和方法不入键不表示它们不重要，而是表示本轮聚类比较的是跨实例结构变化语义；来源证据仍完整保留，任何后续业务裁决必须重新进入正式服务。

### 17.5 哈希、分桶与完整比较

算法只允许：

```text
使用 计算动态模式结构键稳定哈希 召回候选桶。
在桶内使用 动态模式结构键相同 裁决相等。
哈希碰撞但完整键不同的实例保持分离。
规范动态窗口.稳定哈希 不参与模式相等、支持计数或来源去重。
```

不得使用模糊距离、数值容差、可配置相似阈值、容器地址、桶位或首次出现顺序裁决模式。

### 17.6 来源去重与支持数

来源去重分两层：

1. 来源动态层：同一 `来源动态` 完整句柄重复且所有来源材料一致，只保留一份；映射到不同结构键、回合、片段序号或来源材料时为内部不一致。
2. 支持回合层：同一完整结构键内，同一 `动态模式回合身份` 只贡献一次支持；同回合存在多个去重实例时可以保留来源，但不能虚增支持数。

若 `动态模式回合锚点相同` 为真而 `动态模式回合身份相同` 为假，说明同一回合锚点出现矛盾回执，必须返回内部不一致，不得选第一份或最后一份。

重复聚类门槛固定为两个不同完整回合。单窗口复制、同一动态复制、同一回合同键重复、输入排列变化和并发自检完成顺序均不得改变支持数。

### 17.7 运动基元候选

版本 1 的 `动态模式结构键::完整()` 已强制有效来源动作，因此：

```text
重复聚类不成立 -> 不形成运动基元候选。
重复聚类成立 -> 有效来源动作已由全部来源步骤正式保证，形成一一对应的运动基元候选。
```

不得新增“无来源动作但保留重复聚类”的兼容 DTO，也不得为了恢复旧分支放宽 #232 完整性。运动基元候选只表示该单步变化在不同回合重复出现且具有同一正式动作来源，不表示方法有效、动作可靠、因果成立或可自动执行。

### 17.8 顺序、时间和确定性

输出排序固定为：

```text
完整结构键字段顺序
-> 回合排序锚点
-> 片段序号
-> 发生时间戳
-> 来源动态完整句柄
```

片段序号和时间戳只参与来源排序。本轮不要求不同窗口之间片段序号连续，不设置最大时间间隔，不把时间相邻解释为序列连续，不枚举子序列。输入顺序、哈希桶遍历顺序或并发自检完成顺序不得改变输出。

### 17.9 模块与调用方向

```text
组合.动态观察::形成动态聚类候选
-> 服务.动态模式::形成动态聚类候选
-> 算法.动态模式 纯值去重 / 分桶 / 比较 / 计数 / 排序
-> 材料.动态模式 中的值式结果
```

`数据操作.动态模式` 不参与 #233 聚类路径。聚类不得取得共享或独占许可，不得重读仓库，不得回到 #232 服务补当前性，不得把窗口保存为缓存。需要当前业务裁决时必须重新调用 #232 正式观察入口形成新窗口。

### 17.10 并发与线程边界

生产聚类算法是同步纯值计算，不启动线程、`std::async`、后台任务或用户回调。自检可以在 Debug 隔离场景中并发打乱相同输入，但只能证明纯值结果不依赖完成顺序，不能据此宣称生产并行聚类或常驻观察线程已经实现。


仍保留的歧义只有：同一原因支持回合存在多个原因实例，无法确定唯一机会起点；或结果与唯一原因片段序号相同，无法确定先后。两者均不得使用发生时间戳、容器顺序或首次遍历项猜测。

| 模块 | 责任 | 禁止能力 |

第一版逐回合机会证据固定保留：

```text
完整回合身份。
唯一原因单步实例。
原因前结果实例组。
原因后结果实例组。
正结果 / 负结果分类。
```

第一版概率候选固定保留：

```text
原因 / 结果完整结构键。
规则版本。
确定排序的机会证据组。
机会数、正结果数、负结果数。
Q10000 概率。
```

结果状态至少区分 `已形成 / 入口拒绝 / 不支持 / 内部不一致`，拒绝原因至少区分请求不完整、版本不支持、原因结果相同、原因不是重复聚类、机会起点歧义、片段顺序歧义、正式聚类材料矛盾和算法不变量破坏。状态和拒绝原因必须一致；不得用空 optional 或单个 bool 隐藏分类。

### 19.4 入口与正式聚类复核

入口先验证：

```text
动态聚类结果.成功() 为真。
单步实例总数不超过 动态聚类最大步骤数量。
原因 / 结果结构键均完整、互不相同、规则版本等于请求版本 1。
全部相关单步实例完整且结构键版本一致。
原因结构键按完整比较精确命中一个重复动态聚类候选。
```

原因必须是正式重复聚类，以确保机会集合来自至少两个不同完整支持回合。零个匹配是逻辑拒绝；多于一个相同完整键的重复聚类是 #233 正式结果不应产生的内部不一致，必须追根因。结果键只需完整并与原因键不同，可以在单步实例组中零次、一次或多次出现，也不要求自身是重复聚类。

服务不得只信任调用方填写的计数。算法必须复核原因聚类的来源实例、支持回合和 `单步实例组` 中匹配原因键的实例一致；同一来源动态、回合锚点、完整回合或来源实例出现 #233 正式输出不可能产生的矛盾时，返回内部不一致并追根因，不得静默重算一个“修正后聚类”继续。

### 19.5 机会和结果语义

机会集合只遍历原因重复聚类的完整支持回合：

```text
每个支持回合必须至少有一个原因实例。
恰有一个原因实例 -> 形成一个唯一机会起点。
多于一个原因实例 -> 机会起点歧义，逻辑拒绝整个请求。
不在原因支持回合组中的回合 -> 不计机会，不计负结果。
```

对每个唯一机会，只收集相同完整回合且完整结构键等于结果键的单步实例：

```text
结果片段序号 < 原因片段序号 -> 原因前结果证据，不贡献正结果。
结果片段序号 == 原因片段序号 -> 顺序不可判定，逻辑拒绝整个请求。
结果片段序号 > 原因片段序号 -> 原因后结果证据。
原因后结果证据组非空 -> 当前机会为正结果，计数加 1。
原因后结果证据组为空 -> 当前机会为负结果，计数加 1。
```

同一机会存在多个原因后结果实例仍只计一次正结果；所有实例完整保留用于追溯。原因前已有结果、原因后又出现结果时，当前机会仍为正结果。不得跨回合配对，不得用时间戳替代片段序号，也不得从动态步内部搜索新的隐藏子序列。

### 19.6 计数与 Q10000

固定不变量：

```text
机会数 == 原因重复聚类支持回合数 == 机会证据组大小。
正结果数 + 负结果数 == 机会数。
0 <= 正结果数 <= 机会数。
Q10000 = floor(正结果数 * 10000 / 机会数)。
0 <= Q10000 <= 10000。
```

原因重复聚类至少包含两个支持回合，因此有效请求的机会数不为零。#233 又把单步实例总数限制为 `动态聚类最大步骤数量`，版本 1 使用 64 位计数时 `正结果数 * 10000` 有可证明上界；入口仍复核容量，算法仍保留乘法前检查，但不得用构造超出 #233 公开容量的无效 DTO 冒充有效溢出验收。

第一版不设置最低机会数之外的阈值，不输出可靠性、置信度、成熟度或自动升格建议。`Q10000 = 0` 和 `Q10000 = 10000` 都只是当前输入证据的非权威比值。

### 19.7 确定排序

机会证据按 #233 正式回合排序锚点确定排序。每个机会内，原因前 / 后结果实例沿 `回合 -> 片段序号 -> 发生时间戳 -> 来源动态完整句柄` 排序。输入容器排列、哈希碰撞、桶遍历顺序和 Debug 并发自检完成顺序不得改变候选字段、证据顺序或计数。

### 19.8 调用与运行期边界

```text
因果模式业务服务::形成因果概率候选
-> 因果模式算法纯值计算
-> 材料.因果模式 中的值式结果
```

#234 不新增数据操作模块，不调用 `服务.动态模式` 重读当前性，不读取仓库或取得许可。生产算法同步执行，不启动线程、`std::async`、后台任务或用户回调。自检可以并发打乱输入副本，只证明纯值结果不依赖完成顺序，不证明生产并行计算或运行期常驻观察已经实现。
