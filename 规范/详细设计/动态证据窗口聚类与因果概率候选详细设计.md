## 1. 定位

本设计把 FS-02 第三批拆成五段：

```text
DYNAMIC-CAUSAL-S0：在 #225 完成后复核真实任务结果、动态、特征、用途和学习接线。
DYNAMIC-PATTERN-S1：形成特征语义归一、观察回合内显式片段、规范动态步和有序窗口。
DYNAMIC-PATTERN-S2：对完整片段做确定性聚类，返回重复片段与运动基元候选。
CAUSAL-PATTERN-S1：按显式回合内片段窗口序列统计原因模式的机会 / 结果并返回概率候选。
CAUSAL-USE-S0：复核概率、结果效用、用途观察和方法学习的真实写入方。
```

本批落实“聚类向上、抽象向下”中的聚类侧：多个实例回合可以向上形成非权威模式候选，但不直接创建抽象动态、二次特征、因果、方法或概念节点。抽象节点的生成仍必须在实例证据形成后，由后续专用抽象流程裁决。

## 6. DYNAMIC-PATTERN-S1 规范动态证据

### 6.1 目标值式类型

以下为假定契约，不是当前实现事实：

```text
语义特征解析材料
  来源特征值完整句柄
  唯一归属特征完整句柄
  语义特征定义完整句柄
  是否实例槽位
  原始值类型、I64 当前值和原始版本
  归属 / 定义关系当前版本

动态证据材料
  来源动态完整句柄
  场景、主体、被改变目标
  前 / 后状态和前 / 后 I64
  来源动作可选完整句柄
  发生时间戳
  动态来源版本
  语义特征解析材料

观察片段请求
  回合完整身份
  回合内非零片段序号
  可选任务尝试 / 结果上下文
  非空动态完整句柄组
  规则版本

规范动态步
  结构键字段
  来源动态证据
  规范序号

规范动态窗口
  回合完整身份、片段序号、场景、主体、规则版本
  非空有序规范动态步组
  完整片段键、可选哈希和来源版本集合
```

### 6.2 特征语义解析

特征服务新增只读入口候选：

```text
读取特征值语义材料(特征值完整句柄)
复核特征值语义材料当前(值式材料)
```

处理顺序：

```text
1. 写前式入口检查：句柄有效、节点类型为特征值。
2. 目标不是特征值时逻辑返回；当前有效特征值读取唯一归属特征，零或多属于内部不变式破坏并追根因。
3. 若归属特征是实例槽位，读取唯一模板特征定义；零或多同样属于内部结构矛盾并追根因。
4. 读取特征值当前原始类型、I64 值与版本；第一版非 I64 明确返回不支持。
5. 返回完整值式材料；再次复核必须逐项重读关系、值和版本。
```

动态模式服务不得包含、接收或调用特征值服务，也不得自行解释特征归属关系。

### 6.3 显式观察回合与片段

第一版不提供“给定场景自动发现窗口”入口。调用方提交一个或多个显式片段，每个片段必须：

```text
回合身份非零；同一请求可以有多个属于同一回合的片段。
回合内片段序号非零且唯一，用于表示显式先后。
规则版本为当前支持值。
动态句柄组非空且数量在实现明确上限内。
全部动态同场景、同主体且句柄不重复。
每条动态材料完整，前 / 后 I64 与特征值当前语义可复核。
```

同一回合的全部片段必须同场景、同主体；同一来源动态不能跨片段重复。调用方负责给出片段边界，服务只验证，不把相邻片段合并或从一个片段再自动切分。

排序固定为：

```text
发生时间戳升序
-> 完整动态句柄的仓库编号、本地编号、版本升序
```

相同时间戳允许存在，完整句柄消除片段内部次序歧义；回合内片段只按显式片段序号排序。容器遍历、关系读取、线程完成和哈希桶顺序均不能影响任何规范序号。

### 6.4 规范动态步结构键

结构键版本 1 至少包含：

```text
语义特征定义完整句柄。
前 I64 值。
后 I64 值。
来源动作是否存在；存在时写完整动作句柄。
可选稳定二次特征上下文完整结构键。
规则版本。
```

场景、主体、发生时间戳、来源动态句柄、任务、回合身份和片段序号保留为证据，不进入跨实例语义相等键。原因是这些字段用于证明来源和顺序，却不应阻止不同实例对同一语义变化的聚类。

二次特征上下文若启用，必须使用稳定角色身份和每个组成项的语义特征定义，不得使用读取数组下标。#231 无法证明稳定角色时，版本 1 上下文字段统一为空且明确记录，不允许有的请求猜测、有的请求省略。

### 6.5 当前性与返回

全部片段窗口形成后再次逐条复核动态、特征归属、模板定义、原始值和规则版本。任一来源过期时整个请求逻辑内返回，不写缓存或仓库。已经通过唯一性前置、同一当前版本却读取矛盾时追根因。

## 7. DYNAMIC-PATTERN-S2 重复聚类与运动基元

### 7.1 服务与算法分层

```text
动态模式服务：复核 #232 值式窗口当前性、请求边界和返回分类。
动态模式算法：只接收不可变完整窗口，形成结构键、分桶、全量比较、稳定分组和候选 DTO。
```

算法不持有仓库或服务引用，不读取日志、系统时钟、线程身份或全局缓存。可以计算稳定哈希用于分桶，但相等只由完整结构键逐字段、逐步比较决定。

### 7.2 候选层次

```text
一个完整显式片段窗口 -> 一个单实例片段候选。
至少两个不同回合各有一个相同规则版本、完整片段键相同的窗口 -> 一个重复聚类候选。
重复聚类成立且至少一条动态步有有效来源动作 -> 一个运动基元候选。
```

同一回合身份对同一个完整片段键只计一次支持，即使该回合提交多个同键片段也不重复计数。来源回合按完整身份稳定排序；输出候选按完整片段键、来源回合首身份稳定排序。哈希碰撞时结构键不同的候选必须分组，不能合并。

第一版不做：

```text
在显式片段的动态步内部做任意子序列搜索或滑动窗口枚举。
编辑距离、模糊相似、数值容差或隐藏阈值。
跨规则版本比较。
根据单次来源动作推断运动基元。
写抽象动态、二次特征、因果、方法或概念。
```

### 7.3 值式返回候选

```text
实例片段候选：完整片段键、规则版本、来源回合 / 片段序号和来源动态证据。
重复聚类候选：完整片段键、规则版本、不同支持回合组和支持数。
运动基元候选：重复聚类材料、具有来源动作的步骤位置和动作证据。
```

候选无服务内持久生命周期，调用结束后可以丢弃。后续调用必须由来源重新形成，不得把旧候选缓存当前性当事实。

## 8. CAUSAL-PATTERN-S1 概率候选

### 8.1 请求与结果

```text
因果概率请求
  原因完整片段键
  结果完整片段键
  相同规则版本
  非空因果观察回合组

因果观察回合材料
  回合完整身份
  按片段序号严格递增的规范动态窗口组
  同场景、同主体且来源动态跨窗口不重复

因果概率候选
  原因键、结果键、规则版本
  机会回合组
  正结果回合组
  负结果回合组
  机会数、结果数、Q10000 概率
  完整来源版本集合
```

原因与结果可以是 #233 完整片段键，不要求已经成为权威概念。第一版要求原因键与结果键不同。请求材料必须可由 #232 / #233 当前接口重读和全量比较；每次模式出现只表示某个完整窗口键相等，不在窗口内部搜索动态步子序列。

### 8.2 回合计数

每个回合先按片段序号稳定排列完整窗口，再只接受以下无歧义情况：

| 原因出现 | 原因后结果出现 | 处理 |
| --- | --- | --- |
| 0 | 任意 | 不计机会 |
| 1 | 1 | 机会 +1，结果 +1，记入正结果回合 |
| 1 | 0 | 机会 +1，结果不变，记入负结果回合 |
| 多次 | 任意 | 整个请求逻辑拒绝，要求拆分回合 |
| 1 | 多次 | 整个请求逻辑拒绝，要求拆分回合 |
| 1 | 结果只在原因片段前 | 整个请求逻辑拒绝，不反向配对 |

同一回合不得重复提交或重复计数；片段序号不得重复，同一来源动态不得跨窗口重复。第一版不进行动态步子序列搜索、最近邻配对、最佳匹配或跨回合因果连接。

### 8.3 概率

使用检查溢出的整数运算：

```text
机会数 > 0
结果数 <= 机会数
Q10000 = floor(结果数 * 10000 / 机会数)
0 <= Q10000 <= 10000
```

没有机会时逻辑内返回空候选；输入规模导致安全乘法无法表达时逻辑拒绝。已完成安全计数却出现结果数大于机会数或概率越界，属于算法内部矛盾并追根因。

### 8.4 概率不是稳定因果

本项目不要求因果候选先成为“可靠世界事实”。概率只表达当前证据下的可能性，可以很低，也可能在新证据加入后变化。第一版不设置：

```text
最低概率。
置信度或显著性阈值。
自动升格稳定因果。
自动创建因果节点 / 引用。
自动选择方法或执行动作。
```

“是否好用”需要把概率候选和真实任务结果 / 需求结算效用放在一起观察，由 #235 事实复核后另建用途计划。

## 12. 非成功返回二分

| 场景 | 分类 | 收口 |
| --- | --- | --- |

| 只有一个支持回合或重复片段无来源动作 | 正常候选分支 | 分别只返回实例片段或重复聚类 |
| 原因缺席或机会数为零 | 正常 / 逻辑内返回 | 不形成概率候选 |
| 原因 / 结果键相同、多次出现、结果不在原因后、窗口来源重复或回合重复 | 逻辑内返回 | 要求调用方重新分段，不进入动态步内部搜索 |
| 安全计数后结果数大于机会数或概率越界 | 追根因解决 | 不截断、不返回错误概率 |
