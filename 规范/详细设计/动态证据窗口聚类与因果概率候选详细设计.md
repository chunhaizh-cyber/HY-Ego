## 3. #231 已验证当前事实

1. 新版任务执行冻结请求和新版任务结果回执可共同形成非权威观察回合值式身份候选。
2. 当前没有权威任务尝试节点，也没有同一回合内的片段序号字段。
3. 动态不能反向权威推导任务、回执或结果；必须联合完整冻结请求、完整新版回执和结构读回复核。
4. 现有业务服务 / 数据操作读取各自取得共享许可，顺序调用不能排除混合快照。
5. 特征业务服务没有“特征值节点 -> 唯一槽位、宿主、语义定义、当前关系、原始版本”的公开反向入口。
6. 二次特征顺序号只表示单个结构内的位置，不能推断跨实例语义角色。
7. 正式任务执行链把实例动态的 `被改变目标` 写为任务目标抽象状态，不是特征值节点。
8. 实例动态关系槽固定为 `1=被改变目标、2=前状态、3=后状态、4=来源动作`。
9. 唯一运行期业务装配尚未持有新版任务执行调度组合器和任务结果治理组合器。
10. `服务.动态模式`、`算法.动态模式` 等目标模块和阶段 630 / 640 / 650 当前均不存在或未登记。

## 6. 输入与观察回合身份

### 6.1 输入请求

版本 1 请求至少包含：

```text
完整新版任务结果回执。
回执内嵌的完整任务执行冻结请求。
同回合非零片段序号。
非零且受支持的规则版本。
可选的同一回合多片段请求组；每个片段当前只绑定一个正式回执动态证据。
```

第一版不扫描场景、不猜测时间窗口，也不允许调用方只提交裸动态句柄冒充任务结果来源。

### 6.2 回合身份

回合身份完整比较使用新版回执及其完整冻结请求的值式字段，包括但不限于：

```text
请求编号、回执编号、幂等键、任务序号、批次编号。
任务、需求、场景、虚拟存在、目标状态完整句柄。
方法、动作入口、稳定动作键和冻结关系证据。
实际结果状态、动态证据及发生时间戳。
```

哈希可以用于分桶，但回合相等必须全字段比较。该身份只用于候选去重和排序，不成为权威节点、关系、索引或持久化身份。

### 6.3 片段序号

同一回合内片段序号必须：

```text
非零。
请求组内唯一。
由调用方显式给出。
只表示候选中的先后，不反向修改回执、动态或任务。
```

重复片段序号、同回合材料矛盾或同一动态跨片段重复，整体逻辑返回。

## 7. 单许可权威批量读取

### 7.1 许可所有权

`数据操作.动态模式` 构造时隐藏结构事务接线和四仓库引用。公开组合器 / 服务只持有该数据操作的窄业务依赖，不可访问内部接线或仓库。

每次读取顺序固定为：

```text
纯值协议前置检查
-> 一次取得共享许可
-> 获取私有 const 令牌引用
-> 批量读取全部节点、主信息、关系和主键材料
-> 在许可内形成完整值式批次
-> 释放许可
-> 服务与算法只消费值式批次
```

不得在许可内调用 `状态动态数据操作::读取动态材料` 等会再次取得许可的公开入口，不得把令牌保存到 DTO、成员、lambda 逃逸对象或其它线程。

### 7.2 批量读取范围

数据操作层至少复核：

```text
回执动态节点、动态主信息和动态主键。
动态关系槽 1 / 2 / 3 / 4 的完整关系句柄、版本、类型、源端点、目标端点和顺序号。
前 / 后实例状态节点、主信息、场景、主体、发生时间戳和值材料。
被改变目标节点、主信息和抽象状态值式材料。
来源动作节点、主信息、方法角色和动作入口材料。
冻结请求与回执携带的任务、需求、场景、主体、方法、动作及关系证据。
```

已经按正式生命周期失效的冻结关系，必须使用关系历史审计入口按完整句柄复核其原记录；仍要求当前有效的关系使用有效读取。设计不得把“已失效但可审计”误判为“从未存在”，也不得把历史关系重新解释为当前有效。

### 7.3 批次状态

```text
已找到：全部必需材料在同一许可期内完整且互相一致。
入口拒绝：协议、编号、句柄或数量前置无效，仓库零写入。
许可拒绝：无法取得共享许可，不产生半批次。
来源过期：句柄版本自然漂移、必需当前关系已失效或当前节点已删除。
不支持：目标不是版本 1 抽象状态正式链。
内部不一致：同一许可内出现类型、主信息、关系槽、端点、审计或协议证据矛盾。
```

`来源过期` 和 `不支持` 是逻辑内返回；`内部不一致` 必须追根因，不得选择第一个关系、使用旧缓存或顺序重读到偶然一致。

## 8. 业务复核与规范窗口

### 8.1 正式链复核

服务层要求：

```text
动态被改变目标 == 冻结任务目标状态。
目标节点为完整抽象状态，不是特征值、实例状态或二次特征。
动态来源动作 == 冻结动作入口。
动态场景 / 主体 == 冻结任务场景 / 虚拟存在。
动态后状态 == 回执实际结果状态。
动态前 / 后状态与同许可读取的实例状态材料一致。
回执字段、冻结请求字段和权威结构句柄逐项一致。
```

版本 1 不读取特征服务，不修改特征体系，也不把抽象状态值误写为特征值语义。

### 8.2 规范动态步

结构键版本 1 包含：

```text
被改变目标抽象状态完整句柄。
前实例状态值。
后实例状态值。
来源动作是否存在；正式任务执行链要求存在并保存完整动作句柄。
规则版本。
二次特征角色固定为空。
```

场景、主体、发生时间戳、来源动态句柄、回合身份、片段序号、任务、方法和关系证据保留为来源证据，不进入跨实例结构相等键。

### 8.3 排序与哈希

片段内部排序固定为：

```text
发生时间戳升序
-> 动态完整句柄的仓库编号、本地编号、版本升序
```

回合内片段只按显式片段序号升序。完整结构键全字段比较裁决相等；稳定哈希只用于分桶，碰撞不同键必须分开。

### 8.4 当前性声明

规范窗口保存“形成该窗口时同一共享许可期内读到的完整材料”。许可释放后：

```text
窗口仍可作为非权威历史候选参与 #233 / #234 纯值计算。
窗口不证明仓库此刻仍未变化。
需要当前业务裁决时必须重新进入正式数据操作入口。
不得为窗口发明全局快照编号、仓库版本或持续租约。
```

## 10. 并发、锁与一致性

1. 一次请求只取得一份共享许可，并在许可释放前完成全部权威读取和批次复制。
2. 许可持有期间其它线程的独占结构写入不得完成；自检必须证明写线程只在许可释放后完成。
3. 数据操作不得启动线程、`std::async`、后台任务或用户回调。
4. 许可释放后算法可以无锁处理不可变副本；输出排序不得依赖线程完成顺序。
5. 许可失败时不返回半批次；异常不得泄漏许可、令牌或仓库引用。
6. 本设计不宣称全局 MVCC、跨请求同版本快照或持续当前性。

## 11. 非成功返回二分

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| #231 正式接口或阶段 630 漂移 | 设计退回 | 不改代码、不改号、不扩大文件 |
| 协议版本错误、编号为零、回执与冻结请求不一致、片段序号重复 | 逻辑内返回 | 仓库零写入，尽可能仓库零读取 |
| 无法取得共享许可 | 逻辑内返回 | 无半批次、无重试忙等 |
| 句柄自然版本漂移、当前来源已失效 | 逻辑内返回 | 返回来源过期，不用旧缓存兜底 |
| 目标不是抽象状态正式链 | 逻辑内返回 | 版本 1 不支持，不调用特征服务 |
| 同一许可内关系端点 / 槽位、节点类型、主信息或协议证据矛盾 | 追根因解决 | 停止返回，不选首项、不伪造关系 |
| 历史关系可审计但不再有效 | 正常历史分支 | 只证明冻结时来源，不恢复为当前有效 |
| 哈希碰撞但完整键不同 | 正常算法分支 | 保持分离 |
| 算法确定性或计数不变量破坏 | 追根因解决 | 停止返回，不截断或日志掩盖 |
