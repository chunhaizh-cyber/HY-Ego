## 5. 许可内物理主键点查

### 5.1 结果状态

`结构物理主键读取结果` 至少使用强类型状态：

```text
空闲：当前物理键无绑定；正常探测结果。
已绑定：返回一个当前有效完整节点句柄。
入口拒绝：主键为零、线程 / 阶段 / 决定不允许或会话已有失败。
内部不一致：索引命中悬空节点、句柄版本 / 仓库不一致或正反记录矛盾。
```

只有 `已绑定` 可以携带节点；`空闲` 不携带节点且不得登记首次失败。`内部不一致` 必须登记会话首次失败并禁止后继写入。不得继续用一个空 `optional` 同时表示空闲、许可异常和悬空目标。

### 5.2 查询门禁

点查只允许同时满足：

```text
调用线程等于会话创建线程。
会话阶段为写入中。
决定仍为未决定。
不存在首次失败。
主信息、节点、关系、索引、关系变更和候选 I64 写集均为空。
主键非零。
```

任一门禁不满足时零仓库写入。查询不登记写集、不取得第二许可、不缓存仓库引用，也不允许请求提交后继续读取。

### 5.3 完整性复核

索引命中后，会话必须用同一令牌：

```text
读取目标完整节点记录。
确认句柄仓库编号、节点编号和版本精确匹配。
读取该节点反向主键组。
确认当前物理键恰在反向组中且无重复。
```

索引空闲为正常结果；命中但目标不可读、版本漂移或反向缺键均为内部不一致。

## 6. 类型—索引值式投影

### 6.1 仓库原语

节点仓库增加指定类型当前有效记录的值式副本；索引仓库增加全部正向主键绑定的值式副本。两者均只接受会话内部原始令牌，返回后不保留锁或引用。

仓库锁规则：

```text
每次只持有一个仓库锁。
先复制节点记录组并释放节点锁。
再复制索引绑定组并释放索引锁。
最后由仍持结构独占许可的会话做跨仓库比较。
```

不得同时持有节点仓库锁和索引仓库锁，不得把跨仓库一致性判断下沉到任一仓库。

### 6.2 会话投影

`结构类型索引投影结果` 至少承载：

```text
强类型状态：已形成 / 入口拒绝 / 内部不一致。
指定类型当前有效完整节点组。
指向这些节点的物理主键—完整节点句柄绑定组。
```

会话还必须用每个节点现有反向主键组逐一复核全部正向绑定：

```text
每个正向绑定目标都是当前有效完整节点。
每个正向绑定恰在目标节点反向主键组中。
每个反向主键恰有一条正向绑定且指回同一完整节点。
指定类型投影不混入其它类型节点。
主键、节点或完整绑定不重复。
```

任一悬空、重复、错类型或正反不对称都返回内部不一致并登记首次失败；不得把不一致记录过滤后继续形成“干净投影”。

### 6.3 后继 O1 使用边界

#291 只形成通用投影。后继 O1 专用数据操作在同一会话中才能：

```text
按关系 18 角色 1 完整反查同任务记录。
对节点 15 读回 15 槽和角色结构。
重建 36 字节身份、探测序号和最终物理键。

```

固定预留计算是 O1 业务设计，不进入通用会话，也不新增字节账或第二索引。

## 7. 会话状态机

会话状态固定为：

```text
写入中
-> 确认中
-> 已确认待发布
-> 已发布

写入中 / 确认中 / 已确认待发布
-> 撤销中
-> 已撤销

撤销中
-> 撤销失败隔离
```

当前“确认失败直接进入已结束”必须删除。确认失败只记录首次失败、确认进度和“需要撤销”；写集、候选能力和原令牌必须保留到撤销完成或域隔离。

请求提交只表达业务回调已经完成写集和读回，不是发布。`完成确认` 只把全部写集推进到“已确认待发布”，也不是发布。`完成发布` 只在执行器已确认所有参与者后调用，必须是无分配、无回调、无日志、无故障点、无可失败返回的阶段封口。

## 8. 仓库可逆确认合同

### 8.1 节点与主信息候选

同一候选能力绑定：创建仓库、原独占令牌和完整句柄。候选撤销必须接受：

```text
未发布：擦除候选记录，进入已撤销。
已确认待发布：按同一完整句柄精确擦除，进入已撤销。
已撤销：幂等返回候选已撤销。
已发布、已移动、令牌失效或版本不一致：拒绝或内部不一致。
```

确认只改变能力阶段，不清除反向材料。撤销不得调用普通删除，不得扫描编号，不得回拨 `下个编号` 或创建序号高水位。失败候选烧掉的编号允许形成空洞；完成标准是记录不可读且无关系、索引、侧表或容量占用。

`CORE-UNPUBLISHED-S1` 的第 3 项必须使用两组互不混用的候选机械验证该合同：

```text
候选甲：在同一原共享许可内创建节点 / 主信息候选并完成确认；随后按节点 -> 主信息顺序使用原能力和原令牌撤销，两项均返回已完成，两个完整句柄均不可读。
候选乙：在同一原共享许可内创建并确认，但不撤销；把两项能力、两个完整句柄和原许可序号保留到许可作用域外。
后继许可：取得新的共享许可并证明许可序号不同；用后继令牌调用候选乙两项撤销必须均返回已拒绝，两个记录仍逐句柄可读且内容不变。
最终清理：释放后继共享许可，再取得独占许可，按节点 -> 主信息普通删除候选乙并证明两个句柄不可读。
```

本次只允许改写第 3 项场景的局部材料和断言。其它 11 项验收表达式、`std::array<bool, 12>` 总数、模块名、中央编号 `CORE-UNPUBLISHED-S1` 和生产代码全部保持不变。不得通过删除中央登记、强制门禁为真、弱化第 8.1 节合同或新增能力阶段读取接口让自检通过。

### 8.2 新关系与索引

新关系继续由完整关系句柄逆序严格删除；索引继续由“主键 + 预期完整节点句柄”严格删除。每一逆操作都必须执行，不能因前一个失败短路。

### 8.3 已发布关系变更

逐签名合同固定如下：

1. `结构化确认已发布关系变更(能力, 原令牌)` 只在能力仍为持有、令牌与能力保存的域编号 / 运行期纪元 / 许可序号 / 许可类型 / 活动线程一致且当前记录等于写后记录时成功；成功只把能力切为已确认，不改变关系记录或关系版本。
2. `结构化撤销已发布关系变更(能力, 原令牌)` 同时接受持有和已确认阶段；成功前必须逐字段匹配写后完整记录，成功后逐字恢复写前完整记录和写前版本，并把能力切为已撤销。
3. 撤销成功后，原写前句柄重新成为当前有效句柄；写后句柄立即过期，按写后句柄审计读取必须为空。撤销不额外递增版本，因为写后版本在独占许可释放前未对普通读者发布。
4. 用后继会话令牌调用确认或撤销必须返回 `许可拒绝`。拒绝必须保持写后记录、写后句柄可读性和能力阶段不变；第二次许可的许可序号必须与第一次不同，不能只靠“看起来是新对象”证明跨会话。
5. 会话 `完成发布()` 关闭会话层撤销入口；原许可释放后从协调器活动表移除，旧能力自然无法通过令牌验证。不得增加仓库“手工使能力失效”API，也不得新增影子版本账。

S4-A10 自检必须用原写前句柄读取撤销后的恢复记录，并同时断言写后句柄读取为空；跨会话分支必须保存第一次许可序号，确认第二次许可序号不同，并断言跨会话撤销为 `许可拒绝` 且写后记录、写后句柄和能力的公开可观察语义均零变化。能力阶段是私有实现细节，不得为自检新增读取接口；必须以静态复核确认许可拒绝在阶段切换之前返回，并以重复跨会话调用保持同一拒绝与记录不变作为运行证据。不得继续用写后句柄读取恢复后的写前记录。

## 9. 事务参与者四段协议

`结构写入事务参与者` 统一为：

```text
准备提交(结构提交准备只读视图)
确认待发布()
完成发布() noexcept
完成撤销()
```

参与者内部状态至少为：

```text
未准备 -> 已准备 -> 已确认待发布 -> 已发布
未准备 / 已准备 / 已确认待发布 -> 已撤销
撤销失败 -> 撤销失败隔离
```

约束：

1. 准备阶段可分配、写入未发布侧表材料并取得内部独占锁。
2. 确认待发布可以失败，但失败前后仍持有锁和精确反向材料。
3. 完成发布只清理撤销材料、切换阶段并释放内部锁，必须 `noexcept`，不得包含分配、业务回调、日志、故障注入或任何可能失败的结构复核。
4. 完成撤销从已准备或已确认待发布精确恢复侧表前态并释放内部锁；已撤销幂等。
5. 参与者不得取得仓库、原始令牌、结构许可或可写会话，也不得调用仓库入口。

准备阶段的非成功必须使用强类型来源：只有资源不足或许可 / 侧表竞争等设计允许拒绝，且尚未产生任何可读半结构时，才保留为逻辑内返回；抛出、参与者内部状态不一致、准备返回“成功”但锁 / 写集 / 反向材料不完整，或准备后读回异常，均归为追根因解决。两类非成功都必须先撤销参与者，再撤销会话；任一撤销失败进入域隔离。

真实适配至少覆盖当前特征值原始材料参与者；只用假参与者不能形成 #291 完成声明。

## 10. 锁序与唯一可见点

写锁顺序固定为：

```text
结构独占许可
-> 参与者侧表独占锁
```

接域生产侧表读写固定为：

```text
结构共享许可
-> 参与者侧表共享锁
-> 值式读取
```

`特征值服务` 必须保存可选结构事务接线；唯一 `运行期业务装配` 必须传入与四仓库相同的正式接线。接域模式下，创建、写入和七个公开读取都先取得对应结构许可，再使用带令牌仓库入口并取得 `原始值锁_`；不得在持有侧表锁后调用无令牌仓库入口。既有两参数构造和无接线行为只保留给当前隔离自检 / 未接域兼容上下文，不计入 #291 原子可见性完成声明，也不得出现在唯一运行期装配。

不得保留“直接取侧表锁而不经过结构许可”的接域生产旁路。参与者可以在结构许可释放前先完成发布并解锁侧表，因为普通读者仍无法取得结构共享许可；结构独占许可最后释放才是四仓库和侧表统一可见点。

## 11. 执行器顺序

生产入口固定为：

```text
取得结构独占许可
-> 构造会话
-> 同步业务回调、查询、写入和读回
-> 请求提交
-> 参与者准备提交
-> 会话完成可逆确认
-> 参与者确认待发布
-> 进入无失败最终发布区
-> 会话完成发布
-> 参与者完成发布 noexcept
-> 结构独占许可最后释放
-> 返回成功
```

任一最终发布前失败固定为：

```text
保存原失败结果
-> 参与者完成撤销
-> 会话逆序撤销全部写集
-> 读回前态
-> 全部成功才释放许可并返回原失败
```

执行器必须继续尝试所有撤销项并汇总结果；不得使用会掩盖剩余逆操作的提前返回或短路表达式。

## 12. 撤销失败隔离

### 12.1 完整接域门禁

`结构事务接线::已接域()` 必须同时验证：

```text
域编号非零。
运行期纪元非零。
运行期状态有效。
共享许可取得回调有效。
独占许可取得回调有效。
共享路径令牌验证回调有效。
独占令牌验证回调有效。
撤销失败隔离回调有效。
```

合法接线形态只有：全部字段为空的明确未接域兼容接线，或上述八项全部成立的完整接域。只提供部分字段的接线是入口非法形态，不能因 `已接域() == false` 就落入未接域直写兼容路径。`结构事务接线.数据.h` 可以增加不承载业务状态的 `完全未接域()` 与 `接线形态有效()` 值式辅助；四仓库构造、执行器有效性和接收可选接线的特征值服务必须在开始访问前拒绝部分接线。`领域/自检.动态模式.ixx` 的读取门禁包装只改变共享许可取得时机，必须把基础接线的撤销失败隔离回调经包装状态原样转发；不得通过删除 `已接域()` 的隔离项使七字段旧包装继续通过。执行窗口还须静态扫描全部 `结构事务接线` 形成入口，证明不存在其它漏转发包装。

### 12.2 执行器隔离调用点

任一参与者或会话逆操作失败时：

```text
继续尝试其它逆操作并记录全部失败位。
在结构许可析构前通知协调层把当前事务域置为撤销失败隔离。
协调层从此拒绝该域所有共享和独占许可。
隔离前已经登记但尚未取得结构锁的请求，在获锁后、返回许可对象前必须在活动锁内二次复核隔离标志；若已隔离，则移除自身活动记录、释放刚取得的结构锁并返回无效许可。
触发公共逻辑错误诊断：写独立逻辑错误日志并弹窗，仅供人读。
停止依赖该域的运行期路径；只能走重启和正式恢复，不允许在线扫描修补。
```

物理互斥量可以在隔离状态登记后由当前许可对象正常析构，但协调层不得再签发有效许可；新请求在登记前拒绝，已登记等待者在获锁后二次复核并自清理。因此普通读者不能在半结构上继续运行。隔离标志、活动表登记和二次复核必须受同一活动锁保护；不得以 `std::terminate`、死锁或永久持有线程栈对象冒充隔离合同。

执行器必须把“参与者撤销结果 + 会话撤销结果”汇总后再判断是否隔离，不能因第一项失败短路第二项。任一异常回调、首次失败、未请求提交、准备失败、会话确认失败、参与者确认失败或 Debug 撤销故障路径只要出现撤销非成功，都必须在当前独占许可对象仍存活时调用接线隔离回调。隔离回调缺失或返回失败时仍返回内部不一致并触发逻辑错误诊断；不得把 `已接域()` 弱化为绕过方式。

## 16. 验收

### 16.1 查询

1. 空键、有效绑定、悬空目标和正反不对称严格分流。
2. 同一会话连续 1024 次点查只取得一次独占许可，写集始终为空。
3. 指定类型投影覆盖零、一、多节点；零键、一键、多键、错类型和悬空绑定有唯一结果。
4. 第一笔候选后、请求提交后、错误线程和已有首次失败时查询均拒绝。
5. 静态扫描证明核心查询无 O1 业务常量，领域公开面无仓库、令牌或许可。

### 16.2 原子收口

1. F01—F09 每一项失败后，可读权威记录、有效计数、关系、索引、侧表和容量占用等于前态；失败候选完整句柄不可读。
2. 允许编号 / 创建序号高水位前进，不要求回拨或复用句柄。
3. 成功路径中接域 `特征值服务` 的普通读者在许可释放前不得完成，释放后一次看到完整四仓库与侧表；唯一运行期装配不得保留两参数无接线构造。
4. R01—R06 每一项使事务域进入隔离；隔离后新请求在登记前拒绝，隔离前已登记等待者获锁后二次复核后返回无效，逻辑错误诊断触发。
5. 最终发布区静态和运行审计证明无分配、回调、日志、故障点和可失败返回。
6. 当前特征值原始材料参与者完成真实适配，`特征值服务` 接域生产读写按结构许可 -> 侧表锁收口；既有结构写入、仓库、关系换代、特征体系和运行期事务自检全部回归。
7. `CORE-UNPUBLISHED-S1` 第 3 项以候选甲证明节点 / 主信息同一原许可确认后可撤销且记录不可读，以候选乙证明原许可释放后的后继许可序号不同、撤销拒绝且记录不变，再由独占许可清理；其它 11 项与 12 项总数不变。
8. S4-A10 明确证明：同一原许可确认后仍可撤销；写前句柄恢复有效且记录逐字段等于写前记录；写后句柄过期；后继会话许可序号不同、撤销返回许可拒绝且写后记录零变化。静态复核证明许可拒绝先于私有阶段切换；不得为读取能力阶段增加生产接口。聚合 A14 必须随完整 S4 回归通过。

### 16.3 构建与治理

完整自检退出码必须由一份单一顶层门禁表计算。每项只承载“既有门禁名称 + 既有布尔值”，`程序通过` 对该表做全通过汇总；失败时只输出实际为假的门禁名称。不得同时保留另一份手写长布尔链形成双重真相，不得改变任一叶子表达式、宏条件、执行顺序或退出码。该输出是人读诊断，不写节点、关系、索引、日志业务状态或阶段登记。
