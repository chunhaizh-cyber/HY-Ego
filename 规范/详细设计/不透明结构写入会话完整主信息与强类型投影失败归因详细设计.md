# 不透明结构写入会话完整主信息与强类型投影失败归因详细设计

编号：CORE-SESSION-S6

日期：2026-07-17

状态：JY-384 设计完成 / #292、DQ-184 待执行 / 阶段 761 未登记 / #290 v0.3 依赖门控 / 880 未登记

## 1. 目标与完成边界

本设计只修复 #290 第二次审计确认的两个通用接口缺口：

```text
节点 / 索引批量值式复制把资源失败折叠为空集合，导致会话可能把内部失败标为“已形成零记录”。
会话只有逐槽 I64 读取和可读布尔，不能返回完整主信息值容器，调用方无法证明无第 15 槽之后的额外非空槽。
```

完成后只允许声明：结构写入会话能够区分真实空投影、入口拒绝、资源失败和内部不一致，并能在原许可内返回完整主信息值式副本。不得声明 O1-S2、节点 15、关系 18、15 槽业务格式、容量、HYSNAP01 或阶段 880 已实现。

## 2. 当前事实

1. `节点仓库::读取指定类型有效节点记录组` 当前返回 `std::vector<节点记录>`，复制异常被 `catch (...) -> {}` 折叠为空集合。
2. `索引仓库::读取全部主键绑定组` 当前返回 `std::vector<主键绑定记录>`，复制异常同样折叠为空集合。
3. `结构写入会话::读取有效节点与索引投影` 只看到两个向量，可能把资源失败误标为 `已形成`。
4. `主信息仓库` 已有令牌内 `读取主信息(主信息句柄, 结构事务令牌)` 并返回完整 `主信息记录`，无需新增主信息仓库公开接口。
5. 会话当前只有 `读取I64值` 与 `主信息可读`；专用数据操作不能合法取得主信息仓库引用或原始令牌。
6. #291 已完成可逆确认、全量撤销、无失败发布和撤销失败域隔离；#292 只补读取结果语义，不改这些合同。

## 3. 仓库强类型批量结果

### 3.1 节点仓库

将现有返回类型改为专用值式结果，至少承载：

```cpp
enum class 节点记录组读取状态 : std::uint8_t {
    已形成,
    入口拒绝,
    资源失败,
    内部不一致
};

struct 节点记录组读取结果 {
    节点记录组读取状态 状态 = 节点记录组读取状态::入口拒绝;
    std::vector<节点记录> 记录组;
};
```

语义固定：

- `已形成` 可以携带真实空集合；只有该状态的 `记录组` 可被消费。
- 令牌无效或节点类型未定义返回 `入口拒绝`，记录组为空但不得解释为零记录。
- `reserve`、复制或排序阶段资源异常返回 `资源失败`。
- 仓库表中命中“有效但节点编号为零、类型未定义、主信息句柄无效、版本为零或创建序号为零”的记录返回 `内部不一致`。
- 读取只在节点仓库锁内复制本仓记录；返回后不保留锁、引用或迭代器。

### 3.2 索引仓库

对应结果至少承载：

```cpp
enum class 主键绑定组读取状态 : std::uint8_t {
    已形成,
    入口拒绝,
    资源失败,
    内部不一致
};

struct 主键绑定组读取结果 {
    主键绑定组读取状态 状态 = 主键绑定组读取状态::入口拒绝;
    std::vector<主键绑定记录> 记录组;
};
```

绑定主键为零、目标句柄不完整或正向表内出现自身重复构造矛盾属于 `内部不一致`；复制资源异常属于 `资源失败`。`已形成 + 空集合` 才表示当前确实没有主键绑定。

### 3.3 Debug 故障入口

既有 `HY_EGO_ENABLE_STRUCTURE_COMMIT_FAULT_SELF_TEST` 宏内允许为节点 / 索引批量读取各增加一次性资源失败注入。生产签名不得携带故障参数；Release 不得出现故障入口或状态存储。静态扫描必须证明所有复制异常路径都返回 `资源失败`，不能再次 `return {}`。

## 4. 会话类型—索引投影

`类型索引投影状态` 扩展为：

```text
已形成 / 入口拒绝 / 资源失败 / 内部不一致
```

调用顺序固定：

```text
会话查询门禁
-> 节点仓库强类型读取并释放节点锁
-> 索引仓库强类型读取并释放索引锁
-> 会话逐完整句柄完成跨仓库互证
-> 返回值式投影
```

映射规则：

| 底层结果 | 会话结果 | 后继 |
| --- | --- | --- |
| 节点、索引均 `已形成` 且互证一致 | `已形成` | 可消费，包括真实空投影 |
| 会话自身线程 / 阶段 / 类型门禁失败 | `入口拒绝` | 零写返回 |
| 任一仓库 `资源失败` | `资源失败` | 登记会话失败并禁止后继写入 |
| 会话门禁已通过后仓库仍 `入口拒绝` | `内部不一致` | 追根因 |
| 任一仓库 `内部不一致` 或跨仓库互证失败 | `内部不一致` | 追根因 |

投影仍只允许当前线程、写入中、未决定、无首次失败且写集为空时调用，不取得第二许可。

## 5. 完整主信息值式读取

### 5.1 结果类型

会话新增：

```cpp
enum class 完整主信息读取状态 : std::uint8_t {
    已读取,
    入口拒绝,
    资源失败,
    内部不一致
};

struct 结构完整主信息读取结果 {
    完整主信息读取状态 状态 = 完整主信息读取状态::入口拒绝;
    std::optional<主信息记录> 记录;
};

结构完整主信息读取结果 读取完整主信息(主信息句柄 主信息);
```

只有 `已读取` 可以携带记录，其余状态必须不携带可消费记录。

### 5.2 门禁与生命周期

读取允许：当前线程、会话写入中、未决定、无首次失败、句柄完整，并发生在 `请求提交` 之前。它不要求写集为空，因此既可读取既有主信息，也可在本会话写完候选槽位后读回完整值容器。

成功读取本会话候选时必须把对应候选标记为已读回，继续满足既有提交准备门禁。读取不取得第二许可，不返回仓库引用、原始令牌、可变引用或跨度对象。

### 5.3 失败语义

- 句柄无效、错误线程、决定后或会话已有失败：`入口拒绝`。
- 会话门禁已通过，但完整句柄在主信息仓库中不存在、版本不符或状态不可读：`内部不一致`，登记首次失败。
- 完整记录或值容器复制发生资源异常：`资源失败`，登记首次失败。
- 若失败前已有候选写集，执行器必须沿 #291 完整撤销；若为空写集，则零结构变化结束。

## 6. 业务解释边界

核心会话只返回完整 `主信息记录`，不判断：

```text
容器是否恰好 15 槽；
槽 0—14 是否全部非空；
第 15 槽以后是否存在额外非空值；
值域、格式版本、O1 身份或容量是否合法。
```

这些判断由后继 O1 专用数据操作在同一会话内完成。完整值式副本使其能够检查 `值容器.size()`、每个 optional 和额外槽，而无需穿透主信息仓库或原始令牌。

## 7. 文件与模块边界

本设计不新建代码文件。所有变更原地进入既有传统仓库头 / 源文件、既有生产真模块 `会话.结构写入.ixx`、既有自检真模块和入口最小登记。工程与 filters 无需修改。

## 8. 非成功返回

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 会话线程、阶段、类型或句柄门禁失败 | 逻辑内返回 | `入口拒绝`，零新增写入 |
| 写前批量复制资源失败 | 逻辑内失败 | `资源失败`，锁死后继写入，空写集不变 |
| 会话门禁通过后仓库入口拒绝 | 追根因解决 | `内部不一致` |
| 仓库记录自身非法或跨仓库互证失败 | 追根因解决 | `内部不一致`，停止后继 |
| 写后完整主信息复制资源失败 | 追根因解决 | 保留 `资源失败` 归因并撤销既有写集 |
| 完整主信息句柄对应记录缺失 / 版本状态矛盾 | 追根因解决 | `内部不一致` 并撤销 |
| 撤销失败 | 追根因解决 | 许可析构前隔离事务域 |

## 9. 验收

1. 真实空节点 / 空索引只以 `已形成 + 空集合` 返回。
2. 节点复制和索引复制资源失败各自可被 Debug 一次性注入，并在会话层保持 `资源失败`，不得形成零记录成功。
3. 仓库入口拒绝、资源失败、内部不一致和真实空集合四类互斥。
4. 投影跨仓库悬空、重复、错类型和正反不对称继续为内部不一致。
5. 完整主信息成功返回整个值容器；15 个非空槽可读，追加第 15 槽后的额外非空值可被完整副本发现。
6. 本会话候选完整读回后可正常请求提交；错误线程、决定后、无效句柄和版本矛盾按状态拒绝。
7. 资源失败后不能继续候选写入；已有写集会完整撤销。
8. 核心与仓库零 O1 / FNV / 业务容量常量，领域公开面零仓库 / 令牌 / 许可穿透。
9. Debug / Release x64 Rebuild、两配置完整自检、Release 隔离、最终 Debug 20 / 20、strict 与范围门禁通过。
10. 只登记 Debug 最终回归阶段 761；不改变 759、760、770，不登记 880。
