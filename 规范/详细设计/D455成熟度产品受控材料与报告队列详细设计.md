## 1. 设计目标与权威边界

本设计承接现有 DD-01、DD-02 的可复用业务合同，补齐从同步帧材料到四类 D455 供给产品的生产链：

```text
同步帧材料
-> 独立判定 L0—L3、外显模式和产品类型
-> 受控材料与一个物理报告队列
-> 对齐 / 滤波 / 点云 / 分割 / 原始逐簇
-> 跨轮关联 / 局部质量 / 稳定门
-> 稳定观察子集 / 扫描变化 / 目标跟踪
-> 四类索引或只读视图
```

L0—L3 是材料成熟度，逐簇识别 / 扫描变化 / 目标跟踪是外显模式，原始逐簇 / 稳定观察子集 / 扫描变化 / 目标跟踪是供给产品。三轴必须分别写入和验证，不能互相推导或替代。

正式依据包括 0050、0500、4010—4070、6200—6250、6300—6360，以及对象根规范 1140、1150、1160、1170、3200、3300。6210 的当前值完整性、6220 的场景工作区、6230 的递归细分和 6330 的一阶改变边界是后继产品准入约束。报告、队列项、材料页、质量分数、轨迹号和索引命中都不是世界事实。

`D455受控材料域` 构造只接 `std::uint64_t 材料域代次`；`D455观察供给报告队列` 构造只接 `std::uint64_t 容量, std::uint64_t 消费代次`；`D455观察供给生产者` 构造注入 `D455受控材料域&` 与 `D455观察供给报告队列&`。内部候选是不可复制、不可持久的不透明对象；只允许 C6 组合层通过内部提供者取得。

`D455报告消费参与包` 由 C2 唯一声明为可移动不可复制的内部能力；它在自身生命周期内同时持有消费候选、候选队列项和一个 `节点直接身份结构写入事务参与者`。C6 只能经下列端口取得该能力：

```cpp
class D455报告消费参与包 {
public:
    virtual ~D455报告消费参与包() = default;
    virtual const D455报告队列项& 读取候选队列项() const noexcept = 0;
    virtual 节点直接身份结构写入事务参与者& 读取事务参与者() noexcept = 0;
    virtual D455报告队列结果 读取发布后结果() const = 0;
};
struct D455报告消费准备结果 {
    D455报告队列结果状态 状态;
    std::unique_ptr<D455报告消费参与包> 参与包;
    bool 追根因;
};
class D455报告消费提供者 {
public:
    virtual ~D455报告消费提供者() = default;
    virtual D455报告消费准备结果 准备D455报告消费(
        const D455报告消费规格& 规格) = 0;
};
```

`准备D455报告消费` 只准备候选，不确认、不发布。C6 把 `读取事务参与者()` 返回的同一实体与任务侧参与者按“报告 -> 任务侧”顺序交给节点直接执行器；执行器统一准备、确认、逆序撤销和最后发布。发布后 `读取发布后结果()` 必须是 `已确认` 或同义 `幂等读回`，否则属于内部错误。调用方不得复制候选、绕过执行器单独确认、持久化参与包或从队列位置补造报告身份。

公开及内部行为固定为：

## 4. PER-C1—PER-C5 / v0.2 预冻结接口合同

### 4.1 PER-C1 / v0.2：分类和值式验证

- 模块 / 文件：第 3.2 节 #361 两个精确文件。
- ABI / 规则版本：导出 `inline constexpr std::uint32_t D455观察供给协议ABI版本 = 2`；三轴枚举、产品身份、来源范围和分类规则版本必须完整。
- 提供 / 消费：#361 提供，#362—#365 消费。
- 输入：完整产品身份、来源范围、时间、成熟度证据、模式证据、产品负载摘要、规则版本；不输入或拥有 C3—C5 的具体负载实体。
- 输出：不可变 `D455材料分类` 或具名拒绝。
- 写读方：协议层形成值；队列、逐簇、稳定化、变化跟踪只读。
- 生命周期：与产品版本一致；重放同一身份和同一规则必须同义。
- 并发键：产品身份；并发只读无占用。
- 逻辑内返回：字段缺失、组合非法、规则版本不支持。
- 内部错误：同一产品 / 规则得到不同分类，或值式冻结后变化。
- 恢复：从完整产品记录重新验证，不从索引或显示文本恢复。
- 漂移退回：字段、枚举、ABI 或三轴关系改变。

### 4.2 PER-C2 / v0.2：受控材料与单一队列

- 模块 / 文件：第 3.2 节 #362 五个精确文件。
- ABI / 规则版本：导出 `inline constexpr std::uint32_t D455报告流转ABI版本 = 2`；受控引用、报告稳定身份、队列项、生产 / 消费代次和四类索引版本分别冻结。
- 提供 / 消费：#362 提供，#366、#373 消费。
- 输入：PER-C1 通过的描述、C3—C5 所有者形成的强类型产品分支和生产者有效占用；C2 必须在保留材料前核对 `产品类型` 与 `variant` 分支唯一匹配。
- 输出：受控引用、单一队列项身份、四类索引可见性。
- 写读方：唯一生产控制域保留并发布；任务域唯一消费；控制面板只读。
- 生命周期：保留权覆盖排队、承接和方法读取窗口；索引可清空重建。
- 并发键：设备身份控制生产占用；队列项身份控制发布 / 消费；两者不混用。
- 逻辑内返回：队列满、重复产品、过期材料、生产代次失效、产品类型与强类型负载不匹配。
- 内部错误：产品已发布但队列项不可读，或队列项存在但受控引用不可读。
- 恢复：物理队列和四索引不保存为权威队列快照；关键报告与消费治理事实按稳定身份持久为恢复候选。新运行期隔离复核后，已承接项只恢复任务侧引用且不得重新消费；尚未承接且当前不可舍弃项以原报告身份进入新生产代次投递候选；过期、已消费、失效项收束；重建空物理队列和四索引后随新生产 / 消费代次原子发布。任何恢复材料都不能升级为世界事实。
- 漂移退回：物理队列数量、消费权、保留生命周期或恢复语义改变。

### 4.3 PER-C3 / v0.2：原始逐簇

- 提供 / 消费：#363 提供，#364 消费。
- 输入：L0 / L1 合法同步帧、内外参、对齐 / 滤波 / 点云 / 分割规格。
- 输出：原始逐簇产品和逐项质量；不能输出存在句柄。
- 写读方：#363 纯处理模块形成；#364 只读消费；队列、任务和领域服务不得写入。
- 生命周期：派生物绑定设备、生产代次、来源帧范围和规则版本；失效后可丢弃重算。
- ABI / 规则版本：导出 `inline constexpr std::uint32_t D455逐簇处理ABI版本 = 2`；坐标、单位、掩码、来源和质量字段必须完整。
- 并发键：设备身份 + 生产代次 + 帧范围。
- 逻辑内返回：空帧、标定不匹配、深度无效、无合法簇。
- 内部错误：前置通过后坐标、来源或产品版本读回不一致。
- 恢复：确定性重算或按完整受控产品记录恢复，不从算法日志恢复。
- 漂移退回：坐标系、点云所有权、簇身份或质量字段改变。

### 4.4 PER-C4 / v0.2：跨轮稳定门

- 提供 / 消费：#364 提供，#365 消费。
- 输入：同观察范围、连续生产代次内的原始逐簇窗口和稳定门规格。
- 输出：稳定观察子集或合法未稳定结果。
- 写读方：#364 关联 / 稳定门模块形成；#365 只读消费；不得发布报告或确认存在。
- 生命周期：绑定关联窗口和稳定门版本；窗口重置、来源失效或规则改变时整体失效。
- ABI / 规则版本：导出 `inline constexpr std::uint32_t D455稳定观察ABI版本 = 2`；来源链、局部质量、遮挡 / 暂失和稳定状态必须强类型承载。
- 并发键：设备身份 + 观察范围 + 关联窗口。
- 逻辑内返回：证据轮数不足、局部质量不足、关联断裂。
- 内部错误：同一窗口 / 规则生成互相矛盾的稳定结论。
- 恢复：只从完整跨轮材料及规则重算；非权威关联缓存可重建。
- 漂移退回：稳定门、关联键或跨轮窗口语义改变。

### 4.5 PER-C5 / v0.2：稳定、扫描和跟踪产品

- 提供 / 消费：#365 提供，#366 消费。
- 输入：PER-C4 稳定观察子集、扫描 / 跟踪模式规格。
- 输出：稳定观察子集产品、扫描变化产品或目标跟踪产品。扫描请求没有可比基准时不得新增第五产品，只返回“缺少可比基准”的结构化结果，并把已经成立的稳定观察子集按其原产品身份交给观察任务链建立合法当前数据项。
- 写读方：#365 变化 / 跟踪处理模块形成；#366 只读消费并承接；不得执行方法或写场景事实。
- 生命周期：产品绑定稳定子集、基准 / 跟踪窗口和模式规则版本；过期、目标丢失或基准切换后失效。
- ABI / 规则版本：导出 `inline constexpr std::uint32_t D455变化跟踪ABI版本 = 2`；产品类型、负载、来源和基准版本必须一致。
- 并发键：稳定子集身份 + 模式规格版本。
- 逻辑内返回：首次扫描无可比基准时不形成扫描变化产品；稳定观察子集仍按原产品身份发布，由观察方法和提交入口形成合法外设当前数据项，再经结构化扫描需求、新任务和新筹办轮次进入首次扫描基准工作包。无变化、目标丢失、时效超窗均不伪造变化产品。
- 内部错误：模式输出引用未继承稳定子集来源，或产品类型与负载不一致。
- 恢复：由完整稳定材料和模式规则重算；轨迹号不能恢复存在身份。
- 漂移退回：产品映射、负载字段、模式输出或来源继承改变。
