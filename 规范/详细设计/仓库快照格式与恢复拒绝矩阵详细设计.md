## 2. 定位

仓库快照是运行期核心仓库的持久化候选材料，用于后续恢复入口读取、复核和重放前置检查。

第一轮快照只覆盖：

```text
节点仓库。
主信息仓库。
关系仓库。
索引仓库。
```

第一轮快照不覆盖：

```text
特征值 Vec 原始值容器。
内容哈希。
命中统计。
稳态缓存。
值域候选。
事件日志正文。
运行日志、逻辑错误日志、调试日志。
SQL / ADO 投影。
控制面板、WebView、D455、体素、外设材料。
```

快照材料不得直接裁决当前运行期事实。恢复入口后续即使实现，也必须先完成版本、枚举、仓库编号、句柄版本、节点类型、关系端点、索引候选和授权写入边界复核，再通过正式恢复入口形成候选结构。

## 3. 快照头格式

| 字段 | 含义 | 第一轮要求 |
| --- | --- | --- |
| 快照格式版本 | 快照文件结构版本 | 必填；第一轮建议 `1`；未知版本拒绝恢复 |
| 节点类型枚举版本 | 节点类型数值锁定版本 | 必填；必须匹配 `规范/节点类型与关系类型枚举规范.md` 当前锁定口径 |
| 关系类型枚举版本 | 关系类型数值锁定版本 | 必填；必须匹配当前关系类型锁定口径 |
| 快照序号 | 快照生成序号 | 必填；只作审计和排序材料，不作为业务事实 |
| 快照时间戳 | 快照生成时间 | 可选审计材料，不替代状态 / 动态发生时间戳 |
| 仓库集合编号 | 本快照覆盖的仓库集合身份 | 必填；恢复时必须与目标恢复上下文匹配 |
| 段清单 | 节点段、主信息段、关系段、索引段是否存在 | 必填；缺失必需段拒绝恢复 |
| 段计数 | 各段记录数 | 必填；段计数与实际记录数不一致拒绝恢复 |
| 人读摘要 | 说明材料 | 可选；不得承载机器逻辑 |

快照头只说明快照材料自身，不证明快照内容已经可恢复为当前事实。

## 4. 节点段

节点段记录第一轮只保存节点仓库最小结构：

| 字段 | 来源结构 | 恢复复核 |
| --- | --- | --- |
| 仓库编号 | 节点仓库 | 必须匹配节点句柄仓库编号 |
| 节点编号 | 节点记录 | `0` 拒绝；不得重复 |
| 节点版本 | 节点记录 | `0` 拒绝；必须与关系端点记录一致 |
| 节点类型 | 节点记录 | 必须属于已锁定节点类型枚举 |
| 主信息仓库编号 | 主信息句柄 | 必须指向快照主信息段中的有效主信息 |
| 主信息编号 | 主信息句柄 | 必须存在于主信息段 |
| 主信息版本 | 主信息句柄 | 必须与主信息段版本一致 |
| 有效状态 | 节点生命周期 | 已删除节点不得作为有效关系端点或有效索引目标 |
| 创建序号 | 节点记录 | 只作审计排序材料 |

节点段恢复拒绝要点：

```text
节点编号为 0。
节点编号重复。
节点类型未知或与枚举版本不匹配。
节点绑定的主信息不存在。
节点绑定的主信息版本不匹配。
已删除节点仍被关系段作为有效端点引用。
已删除节点仍被索引段作为有效命中返回。
```

## 5. 主信息段

主信息段记录第一轮只保存主信息仓库最小结构和值容器材料：

| 字段 | 来源结构 | 恢复复核 |
| --- | --- | --- |
| 仓库编号 | 主信息仓库 | 必须匹配主信息句柄仓库编号 |
| 主信息编号 | 主信息记录 | `0` 拒绝；不得重复 |
| 主信息版本 | 主信息记录 | `0` 拒绝；必须与节点段引用一致 |
| 有效状态 | 主信息生命周期 | 已删除主信息不得被有效节点绑定 |
| I64 槽位表 | 当前最小值容器 | 只作为已确认领域入口材料，不得扩大为旧主信息字段 |
| 拓扑锚点编号 | 预留拓扑锚点 | 第一轮可空；不得裁决业务事实 |

主信息段不得包含旧项目各类主信息类字段，不得新增名称、类型、词性、任务时间、动作句柄、需求结算摘要、SQL 字段或控制面板显示字段。

## 6. 关系段

关系段记录第一轮保存关系仓库最小结构：

| 字段 | 来源结构 | 恢复复核 |
| --- | --- | --- |
| 仓库编号 | 关系仓库 | 必须匹配关系句柄仓库编号 |
| 关系编号 | 关系记录 | `0` 拒绝；不得重复 |
| 关系版本 | 关系记录 | `0` 拒绝 |
| 关系类型 | 关系记录 | 必须属于已锁定关系类型枚举 |
| 源节点仓库编号 | 源节点句柄 | 必须匹配节点段记录 |
| 源节点编号 | 源节点句柄 | 必须存在且有效 |
| 源节点版本 | 源节点句柄 | 必须与节点段版本一致 |
| 目标节点仓库编号 | 目标节点句柄 | 必须匹配节点段记录 |
| 目标节点编号 | 目标节点句柄 | 必须存在且有效 |
| 目标节点版本 | 目标节点句柄 | 必须与节点段版本一致 |
| 顺序号 | 关系记录 | 只作枚举顺序材料 |
| 有效状态 | 关系生命周期 | 已失效关系不得作为当前关系恢复 |

关系源 / 目标复核必须在恢复入口前置完成。任何悬空源节点、悬空目标节点、端点版本不匹配、端点已删除或关系类型未知，都必须拒绝恢复该快照。

普通父子关系还必须复核：

```text
同一子节点同时只有一个有效父节点。
父节点不是子节点自身。
父节点不是子节点子孙。
不存在缺当前父关系却被解释为重挂成功的记录。
```

多目标关系必须按已确认关系类型口径复核。未确认多目标语义的同源同类型重复关系不得在恢复时静默保留。

## 7. 索引段

索引段第一轮只保存主键索引候选材料：

| 字段 | 来源结构 | 恢复复核 |
| --- | --- | --- |
| 索引命名空间 | 索引仓库 | 第一轮主键索引必填；未知命名空间拒绝 |
| 主键 | 主键索引 | `0` 拒绝 |
| 目标节点仓库编号 | 节点句柄 | 必须匹配节点段记录 |
| 目标节点编号 | 节点句柄 | 必须存在且有效 |
| 目标节点版本 | 节点句柄 | 必须与节点段版本一致 |
| 命中来源 | 索引材料 | 可选人读复判材料，不裁决事实 |

索引段不作为事实源。恢复时索引可丢弃重建；若索引段存在冲突、指向无效节点或指向已删除节点，必须拒绝恢复该索引项，并按后续恢复入口设计决定是拒绝整份快照还是进入只读候选诊断。

第一轮完整恢复入口未确认前，默认保守口径是：索引段结构冲突导致快照恢复拒绝，不允许静默覆盖。

## 9. 恢复失败结构不变化

恢复入口后续实现时，必须先在内存候选区完成完整复核，通过后才能进入正式仓库写入或受控替换流程。

恢复失败时必须保证：

```text
节点数量不变。
主信息数量不变。
关系数量不变。
索引数量不变。
已有有效句柄不失效。
已有关系目标不改变。
已有索引命中不改变。
需求、任务、方法、状态、动态、因果引用等领域结构不因恢复失败变化。
```

不得先写入一部分节点、主信息、关系或索引，再通过事后扫描器、日志、控制台输出或显示层修复。

## 10. 与事件日志的边界

事件日志和仓库快照是两类材料：

| 材料 | 用途 | 禁止 |
| --- | --- | --- |
| 事件日志 | 记录已确认结构事件和入口结果，作为审计与恢复候选材料 | 不替代仓库快照，不裁决当前事实 |
| 仓库快照 | 记录某一时刻核心仓库结构候选材料 | 不绕过恢复拒绝矩阵，不接 SQL / 旧格式 |

恢复设计可读取事件日志做候选补充，但不得用事件日志修补不完整快照后直接写当前结构。快照和事件日志的合并恢复必须另建详细设计或代码实施切片。
