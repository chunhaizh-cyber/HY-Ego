日期：2026-07-12

状态：正式设计 / NEW-05 / 未改 C++ / 未构建 / 未运行程序

## 1. 目标

建立唯一中央自检运行器，把当前入口中的线程、自我治理、概念生命周期和删除候选自检分批迁入独立真模块，并给修订后的 STRUCT-TXN-S1 提供统一自检登记入口。

本设计只改变自检承载、登记、执行顺序和汇总方式，不改变生产领域逻辑、机器事实、现有验收语义或旧能力迁移状态。

## 5. 中央运行器接口

### 5.1 值式类型

```text
自检单元结果
  编号
  名称
  总通过
  验收项数量
  失败项数量

自检批次结果
  登记数量
  执行数量
  通过数量
  失败数量
  失败编号组
  运行器契约是否完整
```

这些字段只服务测试与退出码映射，不得持久化为业务状态。

### 5.2 登记入口

假定公开入口：

```text
自检运行器::登记自检(
  std::uint32_t 顺序,
  std::string 编号,
  std::string 名称,
  自检执行回调 回调)
```

登记前拒绝：

```text
顺序为零。
编号或名称为空。
编号重复。
顺序重复。
回调为空。
登记集合已经冻结或执行已经开始。
```

拒绝只改变测试控制材料，不接触机器结构。

### 5.3 执行入口

```text
自检运行器::运行全部()
自检运行器::转换退出码(自检批次结果)
```

执行规则：

1. 第一次执行前冻结登记集合并按顺序稳定排序。
2. 每个登记单元恰好执行一次。
3. 普通自检失败不短路后续单元，保持当前“全部场景后汇总”的行为。
4. 单元报告必须保留原验收编号和总通过语义。
5. 同一运行器实例不允许二次执行；重复调用返回运行器契约错误材料。
6. 运行器本身不创建线程。领域自检内部需要的并发仍由对应自检模块控制和收口。
7. 退出码只由入口根据批次结果映射，运行器不启动 SQL、窗口或常驻分支。

## 6. 登记顺序与依赖

| 顺序 | 单元 | 前置材料 |
| ---: | --- | --- |
| 100 | 运行线程 | 线程模块和隔离夹具 |
| 200 | 自我治理 | THREAD-S3 强类型回执材料、当前治理协议和隔离领域服务 |
| 300 | 概念生命周期 | 当前概念图、生命周期和支持材料 |
| 310 | 概念删除候选 | 顺序 300 后的当前活动图材料 |
| 400 | 概念命名治理 | SELF-GOV 汇总回归、概念图和语素材料 |
| 500 及以后 | 后续领域或结构事务 | 对应计划正式登记 |

跨单元只传值式报告或显式只读上下文，不共享可写仓库裸引用作为隐式全局状态。THREAD-S3 和 SELF-GOV-S3A 的共有夹具由运行线程自检模块拥有，自我治理模块只消费值式回执材料。

跨单元依赖统一使用执行期强类型单写冻结槽：在登记前由本批自检装配创建，来源单元在自身回调内写入一次并封存，后继单元只取得常量读取能力。槽位不得进入中央运行器通用 API，不得持久化，不得成为全局变量或业务状态。若前置单元失败且没有形成完整封存材料，后继回调仍须被运行器调用，但只返回“前置自检材料不完整”的自检合同失败，不重复执行前置单元，也不进入依赖该材料的领域场景。THREAD-S3 到 SELF-GOV-S3A、SELF-GOV 汇总到概念命名、CONCEPT-S7 到 CONCEPT-S8-S1 均遵守此合同。

## 7. 分批迁移

### 7.1 #241 中央运行器基础

1. 建立 `自检.运行器` 真模块。
2. 用当前 `概念命名治理自检上下文` 登记顺序 400。
3. 入口从直接调用改为“登记、运行一次、合并批次结果”。
4. 不迁移其它场景正文，不改变现有概念命名 A01-A12。

### 7.2 #242 线程与自我治理

1. 按逻辑提取矩阵迁移 THREAD-S1-S6 共 39 项。
2. 迁移 SELF-GOV-S1-S5 共 80 项。
3. 把共有线程夹具的所有权固定到运行线程自检模块。
4. 入口删除对应场景、局部辅助、并发循环和逐项输出，改为最小上下文与两项登记。
5. 概念命名单元仍以顺序 400 消费 SELF-GOV 汇总回归。

### 7.3 #243 概念生命周期与删除候选

1. 迁移 CONCEPT-S7 共 8 项。
2. 迁移 CONCEPT-S8-S1 共 10 项。
3. 入口删除对应场景和输出，登记顺序 300 / 310。
4. 候选仍停在 `领域影响已形成`，不得借自检迁移调用删除提交。

## 8. 入口边界

迁移后入口只允许保留：

```text
import 自检运行器和本批自检模块。
构造最小只读上下文或值式夹具输入。
向运行器登记。
调用一次运行全部。
把批次结果合并到总自检结果和退出码。
```

不得继续保留已迁移逻辑包的：

```text
完整场景正文。
并发循环和等待条件。
局部验收结构。
逐项输出正文。
只为自检存在的服务和仓库装配。
```

入口行数减少只是辅助信号，不是完成判定。完成必须由原编号集合、行为结果、结构读回和连续回归共同证明。

## 9. 非成功返回二分

| 场景 | 分类 | 处理 |
| --- | --- | --- |
| 执行前代码锚点、模块签名或工程登记漂移 | 逻辑内返回 | 不改代码，退回计划修订 |
| 登记参数无效、编号 / 顺序重复或登记已冻结 | 逻辑内返回 | 不执行该批次，不触碰业务结构 |
| 自检单元按既有负例得到预期拒绝 | 逻辑内返回 | 计入对应验收项，不改变原分类 |
| 已进入正式领域创建 / 写入 / 读回后内部结果异常 | 追根因解决 | 保留原入口语境，定位首次异常，禁止用运行器汇总掩盖 |
| 迁移后原验收编号缺失、重复或结果不同 | 追根因解决 | 不删除旧承载，不进入后续批次 |
| 回调执行中修改登记集合或同一单元执行两次 | 追根因解决 | 运行器合同失效，停止依赖路径 |
| 概念删除候选意外进入提交或改变活动结构 | 追根因解决 | 禁止解锁 #217，追查迁移越权 |

## 10. 验收

### 10.1 运行器

```text

总数、失败编号组和退出码一致。
运行器不写任何业务仓库，不启动生产线程或外部适配。
```
