## 1. 定位

本详细设计承接第 7 项“任务承接需求代码逻辑流程图”，用于把需求节点、需求承接材料、任务创建入口、任务承接壳、任务实际结果状态、任务完成状态、任务方法关系和后续请求材料，收束为当前项目可审查的任务服务边界。

本设计不是 `规范/详细设计/任务状态机筹办执行桥详细设计.md` 的确认升级。旧候选设计继续作为任务状态机、筹办和执行桥证据材料；本设计只覆盖第 7 项流程图之后的“任务承接需求”第一轮代码逻辑边界。

本设计不等于代码实施许可，不自动登记 `项目记忆/Codex任务队列.md`。后续若进入 C++ 实施，必须另行生成并确认施工计划，明确允许文件、禁止文件、入口拒绝、失败收口、读回验证和完成声明边界。

## 4. 结构承载

| 业务概念 | 项目承载结构 | 值类型 / 身份结构 | 写入方 | 读取方 | 验证方式 |
| --- | --- | --- | --- | --- | --- |
| 任务主体 | 任务节点 | 任务节点句柄 | 任务服务 | 任务服务、方法服务、需求服务、显示层只读 | 节点有效且类型为任务 |
| 来源需求 | 任务到需求节点的引用关系 | 需求节点句柄 | 任务服务按需求创建任务 | 任务服务、需求服务、显示层只读 | 唯一来源需求可读且类型为需求 |
| 任务虚拟存在 | 任务到存在节点的引用关系 | 存在节点句柄 | 任务服务经存在服务创建 | 任务服务、方法服务、显示层只读 | 唯一虚拟存在可读且类型为存在 |
| 运行场景 | 任务到场景节点的引用关系 | 场景节点句柄 | 任务服务来自需求承接材料或显式场景请求 | 任务服务、方法服务、显示层只读 | 唯一运行场景可读且类型为场景 |
| 目标状态 | 任务到状态节点的模板关系 | 状态节点句柄 | 任务服务来自需求目标状态 | 任务服务、需求服务、方法服务 | 唯一目标状态可读且类型为状态 |
| 创建状态 | 任务到实例状态节点的引用关系 | 状态节点句柄，带发生时间戳 | 任务服务经状态服务创建 | 任务服务、显示层只读 | 唯一创建状态可读且类型为状态 |
| 任务承接材料 | 来源需求、虚拟存在、运行场景、目标状态、创建状态组合 | 节点句柄组合 | 任务服务读回计算 | 任务服务、后续筹办和执行请求读取方 | 五项均唯一可读 |
| 实际结果状态 | 任务到状态节点的引用关系 | 状态节点句柄 | 任务服务 | 任务服务、需求结算读取方 | 第一轮只允许一次记录，状态节点类型正确 |
| 完成状态 | 任务生命周期实例状态 | 状态节点句柄，带发生时间戳 | 任务服务经状态服务创建 | 任务服务、需求结算读取方 | 已有实际结果状态且发生时间戳有效 |
| 任务选择方法 | 任务到方法节点的引用关系 | 方法节点句柄 | 任务服务 | 方法服务、任务服务 | 第一轮只允许一次选择，方法节点类型正确 |
| 后续请求材料 | 生命周期迁移、筹办回执、执行桥、运行统计局部材料 | 局部结构 / 非权威材料 | 任务服务只读生成 | 后续设计或施工计划 | 只作请求材料，不裁决任务完成或方法成功 |

普通 `bool` 在本设计中只能作为局部入口拒绝或读取成功控制流，不得持久化为任务完成、方法成功、需求满足或世界事实。

## 5. 请求分流

```text
按需求创建任务
-> 复核需求有效、发生时间戳有效、需求承接材料完整。
-> 创建任务节点、任务虚拟存在、来源需求关系、虚拟存在关系、运行场景关系、目标状态关系和创建状态关系。
-> 读回验证五类承接材料。
-> 返回有效任务节点或无效任务。

读取任务承接材料
-> 复核任务节点和任务承接壳完整性。
-> 读取唯一来源需求、唯一虚拟存在、唯一运行场景、唯一目标状态和唯一创建状态。
-> 返回组合材料或空。

记录实际结果状态
-> 复核任务承接材料完整、实际结果状态有效且类型为状态。
-> 第一轮拒绝重复记录。
-> 写任务到实际结果状态的引用关系。

记录完成状态
-> 复核任务承接材料完整、实际结果状态已存在、发生时间戳有效。
-> 经状态服务创建任务完成实例状态。
-> 不在本入口裁决需求满足。

选择任务方法
-> 复核任务承接材料完整、方法节点有效且类型为方法。
-> 第一轮拒绝重复选择。
-> 写任务到方法节点的引用关系。

读取后续请求材料
-> 读取生命周期迁移、筹办回执、执行桥和运行统计材料。
-> 只返回后续候选材料，不写任务状态机事实、不派发执行。
```

## 6. 入口前置拒绝

### 6.1 按需求创建任务

必须拒绝：

```text
需求节点无效。
需求节点类型不是需求。
发生时间戳为 0。
需求承接材料缺主体、目标宿主、场景或目标状态。
显式任务场景请求中，任务场景无效或与需求承接材料场景不一致。
任务节点创建失败。
任务虚拟存在创建失败。
来源需求、虚拟存在、运行场景、目标状态或创建状态任一关系写入失败。
创建状态无效或读回不唯一。
```

拒绝后不得把返回值解释为有效任务承接壳。当前多步写入失败只证明读取入口不得把半结构读成完整承接材料，不证明物理数量完全回滚。

### 6.2 读取任务承接材料

必须拒绝：

```text
任务节点无效。
任务节点类型不是任务。
来源需求缺失或多目标。
任务虚拟存在缺失或多目标。
运行场景缺失或多目标。
目标状态缺失或多目标。
创建状态缺失或多目标。
任一目标节点类型错误。
```

### 6.3 实际结果、完成状态和任务方法

必须拒绝：

```text
任务承接材料不完整。
实际结果状态无效或类型不是状态。
重复记录实际结果状态。
缺少实际结果状态时记录完成状态。
完成状态发生时间戳为 0。
方法节点无效或类型不是方法。
重复选择任务方法。
```

## 7. 失败收口和半结构边界

当前 `按需求创建任务` 是多步写入路径。第一轮设计只允许声明：

```text
失败时返回无效任务。
任务承接材料读取入口不会把不完整关系组读成完整任务。
后续依赖必须先通过任务承接材料读回验证。
```

不能声明：

```text
失败后节点、关系、状态物理数量一定完全回滚。
失败节点一定已删除或失效。
任务仓库事务边界已经完成。
```

后续若把任务创建失败收口提升为施工门禁，必须在施工计划中选择并验证至少一种策略：

```text
写入前数量快照和失败后数量不变。
显式事务边界。
失败节点失效标记。
读取隔离：半结构无法被任何有效读取入口返回。
```

## 8. 后续候选门禁

### 8.1 任务状态机

后续若进入任务状态机设计或施工计划，必须先裁决状态迁移表、允许迁移、拒绝原因、发生时间戳规则和状态读回唯一性。本设计只允许读取生命周期迁移请求材料，不允许把请求材料直接写成权威状态迁移。

### 8.2 任务筹办

筹办回执只作为候选和缺口材料。候选方法集合、方法结构缺口、派生需求候选和拒绝原因不得直接写任务方法关系、需求树关系或任务完成状态。

### 8.3 执行桥

执行桥请求材料必须依赖任务选择方法、方法结构、动作入口和发生时间戳。缺任务方法关系、缺动作入口或缺条件 / 结果结构时，不得执行方法，不得创建动作动态，不得把线程或函数名当作动作来源。

### 8.4 运行统计

运行统计、权重、错误码、运行包和治理协议只能作为非权威材料或审计材料，不得裁决任务完成、方法成功、需求满足或世界事实。

## 9. 服务边界

任务承接需求的写入方只能是任务服务，以及任务服务经存在服务、状态服务等领域服务完成的受控写入。调用方不得绕过任务服务直接写任务节点、任务关系、生命周期状态、实际结果状态或任务方法关系。

允许读取方：

```text
任务服务：读取承接材料、生命周期材料和后续请求材料。
需求服务：读取任务结果或需求结算所需任务材料。
方法服务：读取任务选择方法和执行桥候选材料。
显示层只读：读取结构化材料后显示，不写业务事实。
```

禁止：

```text
任务服务直接访问特征值服务。
显示层写任务事实。
SQL / 控制面板 / 日志 / 文本视图裁决任务事实。
线程、Tick、外部输入或控制台命令成为动作来源。
任务执行桥绕过任务方法关系执行方法。
```

## 13. 完成声明边界

可以声明：

```text
任务承接需求代码逻辑详细设计已生成。
该设计承接第 7 项流程图和当前任务服务入口事实。
该设计区分当前已落任务承接材料、实际结果状态、完成状态、任务方法关系、后续请求材料、当前代码差距和验收矩阵。
```

不能声明：

```text
该设计已确认。
代码已实现。
Debug x64 构建已运行。
完整任务状态机已实现。
任务筹办、真实执行桥、线程派发、工作队列、取消、超时或重试已实现。
方法候选算法、动作入口后续能力或旧任务模块能力已迁移。
旧任务主信息字段、SQL、控制面板、D455、体素或外设已迁移或接入。
```
