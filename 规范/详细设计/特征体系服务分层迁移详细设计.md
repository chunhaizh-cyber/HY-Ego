# 特征体系服务分层迁移详细设计

日期：2026-07-14

状态：JY-329 实际接口复核通过 / #267 正式施工依据 / 前置 #274、DQ-166、735 已满足 / 740 未实施

## 1. 目标

在已验证的服务专用数据操作层模式上，为特征、特征值初始原始材料和二次特征建立新的无原始令牌业务服务面：

```text
调用方
-> 特征业务服务 / 二次特征业务服务 / 特征状态组合器
-> 数据操作.特征体系
-> #274 特征值原始材料参与者
-> #272 结构写入执行器和不透明会话
-> 四仓库 + 特征值服务内部侧表
```

本设计只形成隔离新路径，不修改既有 `特征服务.h`、`特征值服务.h`、`二次特征服务.h` 生产调用，不删除兼容入口。

## 2. 已验证当前事实

### 2.1 既有结构语义

当前结构使用：

```text
特征定义：节点类型::特征
实例特征槽位：节点类型::特征
宿主 -> 槽位：关系类型::归属
槽位 -> 特征定义：关系类型::模板
槽位 / 特征 -> 特征值：关系类型::归属
特征值：节点类型::特征值
二次特征：节点类型::二次特征
二次特征 -> 组成项：关系类型::引用
```

当前 `特征服务` 已有定义、槽位、I64 / Vec 原始材料和候选比较等兼容入口；`特征值服务` 是特征值原始材料的直接内部入口；`二次特征服务` 直接写四仓库。当前这些服务仍包含无令牌仓库路径，不满足新分层公开面。

### 2.2 原始材料承载

```text
I64 值：特征值主信息槽位 0。
I64 原始版本：特征值服务 I64 版本侧表。
VecI64 / VecU64 值和版本：特征值服务 Vec 侧表。
```

`#272` 只覆盖本会话主信息候选 I64 值，不覆盖两个侧表。`#274` 已在提交 `906d971` 形成并验证通用事务参与者、提交准备候选身份判断、初始原始材料参与者和非阻塞侧表访问器；A01-A16、阶段 735、Debug / Release x64 和正式实施记录均已完成。#267 只能消费这些正式公开入口，不得修改或复制参与者实现。

### 2.3 二次特征边界

当前兼容服务仅按节点类型接纳状态和动态，不能区分抽象长期身份与实例临时材料。新第一轮路径不得仅凭 `节点类型::状态 / 动态` 把实例临时材料组成长期二次特征。

### 2.4 JY-329 正式接口复核

`#267` 所需合同已按 `906d971` 逐签名确认：

```text
结构写入执行器::执行(业务回调, 参与者&)：一次独占许可内准备、确认、提交或撤销。
结构提交准备只读视图：只读本会话主信息 / 节点候选、候选节点类型、主信息锚点和候选 I64 值。
特征值原始材料事务参与者(特征值服务&)：登记初始材料；只接受版本 1、本会话新建特征值候选。
特征值原始材料侧表访问器(特征值服务&)：读取(特征值节点)；try_lock_shared 值式复制，不访问四仓库。
结构写入会话：创建主信息候选、创建节点候选、创建带顺序号关系、绑定主键、写入 / 匹配候选 I64，以及完整读回和请求提交入口均存在。
```

参与者模块不包含四仓库、许可、线程或工作队列依赖；执行器不包含特征值领域类型。阶段 740 尚未登记，#267 五个计划模块均不存在，允许范围可闭合且未发现接口漂移。

## 3. 第一轮范围

### 3.1 写入路径

```text
1. 创建特征定义。
2. 为已有宿主和已有特征定义创建唯一实例特征槽位。
3. 为已有无当前值槽位发布一个初始 I64、VecI64 或 VecU64 特征值。
4. 在一次会话内创建实例槽位并发布一个初始特征值。
5. 创建有序长期组成项的组合二次特征。
```

### 3.2 只读路径

```text
读取特征定义身份。
读取宿主和特征定义对应的唯一实例槽位。
读取实例槽位唯一当前特征值。
读取宿主特征状态值式材料，包括 I64 / Vec 类型、值和原始版本。
读取二次特征有序组成项及完整关系证据。
```

### 3.3 排除

```text
不更新、覆盖、删除已发布特征值，不推进版本 2 及以上。
不切换当前值，不保留历史值，不失效或删除槽位。
不实现值域、稳态、哈希、缓存、序列化、恢复或非权威统计。
不把状态、动态加入第一轮二次特征组成项。
不迁移生产调用，不删除兼容入口。
```

## 4. 分层职责

### 4.1 特征体系数据操作层

`特征体系数据操作` 是本批唯一允许导入结构写入执行器和 `#274` 参与者模块的生产类型。职责：

```text
持有事务接线、四仓库和特征值服务内部引用；
取得共享许可并返回值式身份、关系和原始材料；
把类型化写入规格提交给一次独占会话；
在同一许可内复核句柄、端点、类型、顺序、主键和唯一性；
协调四仓库写集与特征值原始材料参与者；
提交前完整读回，失败后精确收口；
只返回结构化结果，不做高级业务裁决。
```

仓库引用、特征值服务引用、原始令牌、许可、会话、候选和参与者均不得出现在导出请求或结果中。

### 4.2 特征业务服务

`特征业务服务` 负责：

```text
创建特征定义；
创建实例特征槽位；
为无当前值槽位发布初始特征状态；
读取特征定义、槽位和宿主特征状态材料；
裁决宿主类型、定义 / 槽位角色、原始值字段和幂等语义。
```

高级服务后续只能依赖该业务服务的值式读取入口，不得直接访问 `特征值服务`、参与者或数据操作层。

### 4.3 特征状态组合器

组合器只承接“一次创建槽位并发布初始状态”。它先让特征业务服务形成经过入口检测的值式槽位和原始材料规格，再提交给数据操作层在一次会话内权威复核并完成。数据操作层不得信任调用方 DTO 代替仓库事实；组合器不导入执行器，不接触仓库或参与者。

### 4.4 二次特征业务服务

`二次特征业务服务` 负责有序组成项准入和幂等语义。第一轮允许组成项节点类型：

```text
基础信息、存在、场景、特征、二次特征、因果引用
```

状态和动态第一轮拒绝。后续只有通过状态 / 动态业务服务证明为抽象长期身份后才能扩展，不得在本计划中按节点类型放宽。

## 6. 公开请求与结果

### 6.1 原始值请求

```cpp
struct 初始特征原始值请求 {
    特征值原始类型 原始类型;
    std::optional<std::int64_t> I64值;
    std::vector<std::int64_t> VecI64值;
    std::vector<std::uint64_t> VecU64值;
};
```

字段必须互斥。I64 只携带一个值；Vec 不能为空且最多 4096 项。调用方不提供原始版本，数据操作层第一轮固定形成版本 1。

### 6.2 写请求

```text
创建特征定义请求：特征定义幂等主键。
创建实例特征槽位请求：槽位幂等主键、宿主、特征定义。
发布初始特征状态请求：特征值幂等主键、实例槽位、初始原始值请求。
创建槽位并发布初始状态请求：互异槽位 / 特征值主键、宿主、特征定义、初始原始值请求。
创建组合二次特征请求：二次特征幂等主键、有序组成项组。
```

### 6.3 结果

结果至少区分：

```text
已提交、幂等读回、未找到、入口拒绝、幂等冲突、许可拒绝、版本漂移、内部不一致
```

成功结果只返回稳定句柄、原始值式副本、原始版本和完整关系值式证据。不得返回仓库、特征值服务、令牌、许可、会话、候选、参与者、锁或内部指针。

## 7. 结构不变量

### 7.1 特征定义

特征定义为带非零主键的 `节点类型::特征`。第一轮定义不得携带模板关系，不把语言名称、显示标题或日志作为身份。

### 7.2 实例特征槽位

槽位必须同时满足：

```text
节点类型为特征；
有唯一非零主键；
恰好一个允许宿主通过归属关系指向它；
它通过恰好一个模板关系指向特征定义；
宿主 + 特征定义组合下最多一个当前槽位。
```

第一轮宿主类型按当前 `特征服务` 正式白名单冻结为：

```text
存在、场景、需求、任务、方法
```

其余节点类型一律拒绝。数据操作层不得把“任意有效节点”当宿主，也不得把本轮二次特征组成项白名单误用为特征宿主白名单。

### 7.3 当前特征值

实例槽位的 `归属` 目标中：

```text
0 个特征值：无当前值，逻辑内返回。
1 个特征值：唯一当前值。
大于 1 个：内部结构歧义，追根因，不选择第一个。
```

第一轮只在 0 个当前值时创建一个新特征值。已有当前值时，同一特征值主键和完全相同材料可幂等读回；任何新主键、不同值或不同类型均拒绝，不覆盖旧值。

### 7.4 二次特征

二次特征有唯一非零主键。组成关系使用 `关系类型::引用`，顺序号固定为输入位置 `0..n-1`。组成项不能为空、不得重复、不得包含自身、不得包含未允许类型。读取必须按完整关系句柄和顺序号复核，不能依赖容器返回偶然顺序。

## 8. 幂等合同

索引只召回候选。每个主键命中后必须复核完整节点句柄、节点类型、主信息、关系、原始类型、值和版本。

```text
未绑定：允许进入写入。
绑定到完全相同事实：幂等读回。
绑定到错误类型、过期句柄、不同宿主、不同定义、不同组成项或不同原始材料：幂等冲突。
```

并发失败后必须先由参与者和会话完成撤销、释放许可，再权威重读。只有重读到完全相同事实才映射为幂等成功。

## 9. 写入路径

### 9.1 创建特征定义

```text
主键写前读取 -> 主信息候选 -> 特征节点候选 -> 主键 -> 同令牌读回 -> 提交
```

### 9.2 创建实例槽位

```text
写前复核宿主、定义和唯一性
-> 主信息候选 -> 特征槽位节点候选 -> 主键
-> 宿主到槽位归属 -> 槽位到定义模板
-> 完整关系句柄读回 -> 提交
```

### 9.3 发布初始特征状态

```text
写前复核槽位身份、唯一定义和无当前值
-> 主信息候选 -> 特征值节点候选 -> 主键 -> 槽位到值归属
-> I64 由会话写主信息候选；Vec 不写 I64
-> #274 参与者登记版本 1 或 Vec 值
-> 会话和参与者共同读回 -> 一次提交
```

### 9.4 创建槽位并发布初始状态

在一次会话内执行 9.2 和 9.3。两个主键必须非零且互不相同；槽位、值、四条必要结构和原始材料必须全部读回后才能提交。

### 9.5 创建组合二次特征

```text
写前复核有序长期组成项
-> 主信息候选 -> 二次特征节点候选 -> 主键
-> 按 0..n-1 写引用关系
-> 逐关系完整句柄和端点读回 -> 提交
```

## 10. 读取合同

宿主特征状态读取必须在一次共享结构许可内完成：

```text
复核宿主和特征定义
-> 读取唯一槽位和完整宿主 / 模板关系
-> 读取唯一当前特征值和归属关系
-> 读取特征值节点与主信息
-> #274 访问器 try_lock_shared 复制侧表材料
-> I64 从同许可主信息读取，Vec 从侧表副本读取
-> 复核唯一原始类型和版本
-> 返回值式材料
```

不得先释放结构许可再读取侧表并拼接。侧表锁竞争返回许可拒绝，不把旧值或空值当当前事实。

## 11. 入口检测

所有写入口必须在第一笔写入前拒绝：

```text
主键为 0，组合主键相同；
句柄无效、错误仓库、错误节点类型或过期版本；
宿主类型不在明确白名单；
特征定义实际是已有实例槽位或模板结构歧义；
同宿主和定义已有槽位；
槽位无定义、多个定义或已有当前值；
原始类型未知、字段不互斥、Vec 为空或超过 4096；
二次特征组成项为空、重复、包含自身、错误类型或状态 / 动态；
高级服务试图直接传入特征值服务、参与者或结构能力。
```

入口检测用于定位问题来源，不在内部填默认值、选择第一个或修复旧结构。

## 12. 失败收口

```text
入口拒绝：第一笔写入前返回，四仓库和侧表不变化。
侧表锁竞争或版本漂移：参与者不正式发布；会话在许可释放前撤销。
写入或读回内部不一致：禁止提交，参与者和会话精确收口。
结构确认失败：#274 先撤销本次侧表记录，再完成会话失败收口。
释放许可后重读：仅完全相同事实可映射为并发幂等；否则必须证明全部回前态。
无法证明回前态：停止 #268-#271，登记正式断点，不生成兼容收口或性能计划执行许可。
```

## 13. 并发与锁

数据操作层共享读取使用 `结构共享许可 -> 侧表 try_lock_shared`。写入使用 `结构独占许可 -> 侧表 try_lock`，参与者持有侧表锁到结构确认或撤销结束。任何路径不得在持有结构许可时等待侧表锁。

并发验收至少覆盖：

```text
同一特征定义主键竞争；
同一宿主 + 定义槽位竞争；
同一槽位初始值竞争；
组合槽位和值双主键竞争；
同一二次特征主键竞争；
兼容侧表写锁占用时新读写快速返回且无残留。
```
