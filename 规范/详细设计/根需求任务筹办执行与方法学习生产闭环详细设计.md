# 根需求任务筹办执行与方法学习生产闭环详细设计

日期：2026-07-19

状态：v0.2 / JY-517 / 生命周期完整历史幂等并发收口修订 / #315 代码尚未验证通过

## 1. 目标

本设计建立第一轮真实生产生成闭环：

```text
既有固定根需求
-> 承接任务
-> 生命周期筹办
-> 正式方法选择
-> 任务管理线程排队
-> 任务工作线程执行
-> 强类型回执上行
-> 生产自我治理线程处理结果和结算
-> O1 权威用途观察
-> O2 非权威统计投影
-> M0 方法内容版本与执行快照
-> M1 确定性差异候选
-> M2 独立授权后形成不可变新方法代际
-> 后继任务筹办可见新代际
```

“真实”至少同时满足：同一生产运行期身份、真实后台线程自然消费、全部权威写入经过正式业务面、回执只作非权威材料、学习不会自动改写当前方法事实。

## 3. 当前事实与目标分界

### 3.1 已实现事实

1. 生产运行期初始化并发布安全根需求和服务根需求。
2. 同身份租约可取得系统角色材料、运行期业务操作和只读查询。
3. 需求、任务生命周期、方法登记/召回/选择、执行、强类型回执、任务结果和需求结算均有正式入口。
4. 任务管理线程有两阶段请求队列；任务工作线程有窄执行入口和强类型回执队列；上行桥有同租约只读复核能力。
5. O1 权威用途观察结构和业务服务已经存在。
6. #315 v0.3 已在隔离任务分支形成创建幂等修正；R4 Debug Rebuild 通过且 A01—A10 通过，但 A11 并发同义和 A12 聚合失败，当前不可集成。

### 3.2 待实施目标

1. 同一生产宿主持有会话、租约和所有消费者线程。
2. 两个现有根需求被真实消费，不再创建平行根。
3. 后台线程自然排队、执行、回执和治理，不由测试线程直接代跑。
4. O1 进入生产结算后链。
5. O2、M0、M1、M2 分层实现。

已实现事实不得因为本设计出现而被扩大为目标已完成。

## 4. 根需求语义

“建立根需求”固定为生产运行期系统角色初始化的组成部分。安全根需求和服务根需求是唯一固定根；治理链只能从 `运行期上下文::读取系统角色材料()` 取得它们，并经 B0 复核需求当前状态。

禁止：

```text
生产消费者再次调用旧需求初始化器；
为测试方便创建第三个根；
用文本名称或固定数组下标识别根；
把根需求已存在误写成任务已经建立。
```

根已满足时，本轮为逻辑内返回；清单声明完整但结构读回矛盾时追根因并停止消费者。

## 5. 生产治理宿主

新增生产治理宿主，只拥有瞬态运行组件：

```text
生产运行期会话
同身份运行期租约
生产自我治理线程
任务管理线程
任务工作线程
任务管理上行桥泵
有界治理邮箱
停止协调状态
```

宿主不拥有第二份仓库、领域服务或索引，不复制 B0 业务组合器。所有能力在调用时从同一租约只读取得。

启动顺序：

```text
生产运行期首发
-> 根需求和业务面完整复核
-> 治理邮箱
-> 生产自我治理线程
-> 任务管理线程
-> 任务工作线程
-> 上行桥泵
-> 提交根需求治理唤醒
```

停止顺序反向：停止新治理输入，停止管理请求准入，唤醒并收口工作线程，上行桥排空已形成回执，自我治理线程排空已接收消息，最后析构会话。任何已接受请求不得因停止直接丢失。

## 6. #315 根需求承接与任务筹办

### 6.1 生产配置

建立固定形状的根需求治理配置，至少包含：配置版本、目标根角色、任务幂等主键、生命周期幂等材料、方法登记/规格稳定主键、选择幂等材料、时间材料和既有生产动作键。稳定键必须与 `生产运行期配置` 同源并在 S0 做全字段冲突扫描。

该配置只提供值式请求材料，不是权威状态。根需求、任务、方法和选择必须由 B0 读回。

承接任务的创建幂等身份不随任务生命周期推进而失效。相同任务主键再次进入时，数据操作必须按任务不可变身份、来源需求、虚拟存在和历史第 1 版“已创建”状态复核原创建材料，并返回当前任务权威材料；不得要求任务当前阶段仍为“已创建”。相同任务主键但来源需求、虚拟存在、初始状态主键 / 时间 / 值任一异义时继续返回幂等冲突且结构不变化。该修订不新增 B0 公开入口，不允许调用方按主键绕过领域服务读取任务，也不放宽生命周期迁移的当前关系 / 版本复核。

生命周期迁移请求沿用当前正式四字段合同：任务、目标阶段、幂等材料编号和发生时间戳。生产路由不得新增“预期旧版本”字段，也不得在调用侧复制当前性裁决。任务服务和数据操作必须按以下顺序共同收口：

```text
读取当前任务与完整生命周期历史
-> 以幂等材料编号定位历史候选
-> 候选唯一且目标阶段、发生时间、状态值、主体、场景和关系证据完全同义：
   返回当前任务幂等读回；允许当前生命周期已经继续前进
-> 同一幂等材料存在但字段异义：
   返回幂等冲突且结构不变化
-> 历史候选重复、关系证据不完整或同一版本不唯一：
   内部不一致并追根因
-> 目标历史不存在：
   任务服务把当时的当前生命周期关系和版本封入不透明写入规格
-> 数据操作在提交前复核关系、版本和合法迁移
-> 写前已经漂移且目标历史仍不存在：
   逻辑内返回，零新结构
-> 本次提交成立或并发同义提交已经成立：
   从完整历史再次核对唯一同义目标证据并返回当前任务
-> 已提交后目标历史缺失、重复或异义：
   内部不一致并追根因
```

“完全同义”的关系证据至少包括：关系类型为任务生命周期、源节点为同一任务、目标节点为本历史状态、顺序号等于该生命周期版本；状态材料必须是同一任务虚拟存在在同一场景中的完整实例状态，幂等主键、发生时间戳和值与四字段请求及目标阶段一致。历史项可以因后继迁移成为已失效审计记录，但完整历史中必须只有一个同义项，并且仍只有一个当前生命周期关系。

数据操作写后不得再要求“当前阶段和当前版本仍精确等于本次目标”。若本次目标已经唯一同义地进入完整历史，而同义线程已经把当前阶段继续推进到后继合法阶段，本次调用仍返回当前任务幂等读回或成功读回。只有结构提交结果已声明提交而上述历史证据缺失、重复或异义时，才映射为内部不一致。

### 6.2 筹办状态机

```text
读系统角色清单
-> 选定未满足根需求
-> 读当前需求
-> 创建或幂等读回承接任务
-> 按正式四字段请求迁移到已承接/筹办中
-> 创建或幂等读回生产方法基线
-> 召回、复判、排序
-> 提交唯一任务方法选择
-> 冻结任务执行请求
```

无候选或非唯一时保持筹办中并逻辑内返回，不造固定成功。方法基线是生产固定能力登记，不得借用自检局部夹具。

生命周期验收必须覆盖合法顺序、非法跨阶段、同阶段同义/异义、历史同义重入和两线程同义竞争。确定性领域回归至少证明：同一“已承接”请求在任务已继续推进到“筹办中”后再次调用，按完整历史幂等读回当前任务；同一幂等材料但目标、时间或值异义继续冲突且结构不变化；历史中只有一个当前关系和唯一版本序列。根需求筹办并发回归必须证明两个线程跨“已承接 -> 筹办中”相邻阶段推进后，两份报告均完整并指向同一任务、方法、选择和冻结请求。任务创建幂等验收继续覆盖“首次创建后已推进到筹办中，再以完全相同创建材料读回当前任务”，并证明同主键异义仍拒绝、结构数量不变化。旧关系/版本写前拒绝继续由既有任务服务/数据操作专项证明；#315 不通过伪造请求字段重复测试内部 ABI。

## 7. #316 真实任务线程执行

任务管理线程把自旋改为条件等待，只有两阶段确认后的请求才可出队。任务工作线程的线程体自然调用现有窄执行入口；测试不得直接调用 `受控消费一条任务执行请求` 作为最终验收。

执行适配器从同一租约取得 B0，线程本身不写仓库、不成为动作来源。每个请求至多形成一个新版强类型回执；请求过期或当前性漂移按现有业务合同收口。

必须提供：线程启动/停止、等待唤醒、队列容量、排空策略、异常状态快照和真实线程 ID 证据。线程 ID 只作验收材料，不承载机器逻辑。

## 8. #317 回执治理、结果结算与 O1

旧 `自我线程` 绑定语素/世界/需求初始化器，会创建另一套根，不得作为生产消费者。新增生产自我治理线程，只接收已经发布上下文和治理处理器，不拥有初始化器。

上行桥泵从任务工作线程取强类型回执，以同租约只读查询复核后写治理邮箱。生产自我治理线程消费消息并调用 B0 `处理任务执行结果`，形成任务完成或待重筹办以及独立需求结算。

正式结果和结算形成后，构造纯值用途事件，再调用同域唯一用途观察服务发布 O1。O1 是权威观察账项，但不直接修改方法。

## 9. #318 O2 用途统计投影

O2 只从已发布 O1 完整身份重建。统计主键使用完整 O1 身份；待结算与最终结算分栏，成功/失败/取消不得混算。投影可清空、可重建、不可裁决业务事实。

第一轮不要求概念映射。概念映射缺失不得阻断完整身份统计。

## 10. #319 M0 方法内容版本与执行快照

方法内容版本必须覆盖方法首、条件规格、结果规格和动作规格的规范化有序内容。既有来源版本、选择关系版本、任务序号或时间戳均不得代用。

执行前冻结所选方法内容版本；执行后把选择、执行、结果、结算、O1 身份与该版本形成不可变执行快照。旧快照不随方法后继代际变化。

## 11. #320 M1 确定性差异候选

M1 输入仅为 M0、按完整身份排序去重的 O1 观察组、O2 值式投影和学习规则版本。输出固定分为新增、保留、移除、替换、无差异、未知和冲突。

候选是非权威材料；相同输入必须得到相同规范结果，输入顺序不得影响结果。缺槽、多映射或材料不足不得猜测。

## 12. #321 M2 晋级授权与新方法代际

M2 必须消费独立、不可伪造的晋级授权，复核来源方法内容版本、观察组和候选当前性。未授权只保留候选，不改变方法事实。

授权通过后由方法领域服务/专用数据操作创建不可变新方法代际，发布代际族、代际号、来源版本、观察组、规范差异、授权和生命周期。失败不得留下半代际。

## 13. #322 后继筹办可见性与端到端验收

第一轮任务的选择和执行快照永远保持旧代际。只有后继任务在新的召回快照中可以看到新代际。验收至少跑两轮：

```text
第一轮：根需求 -> 旧代际 -> 执行 -> 结算 -> 学习 -> 新代际
第二轮：新的待办/重筹办任务 -> 召回新代际 -> 选择可见性成立
```

必须使用真实生产自我治理线程、任务管理线程、任务工作线程和上行桥，禁止主测试线程直接调用消费入口。

首轮可通过显式 `--generation-loop` 生产模式验收。默认无参数入口切换、旧所有者退出和跨重启恢复不在 #322 内；#322 完成后按真实接口另建激活与耐久化计划。

## 14. 非成功二分

入口前形状错误、队列容量不足、停止、过期、目标历史不存在的写前当前性漂移、同一生命周期幂等材料的异义冲突、无候选、非唯一、材料不足、未授权和无差异均可逻辑内返回，前提是没有可读半结构。

完整生命周期历史中的同一幂等材料重复、同一版本重复、关系证据不完整均属于既有结构矛盾，必须追根因。一旦进入创建、生命周期换代、选择发布、执行写入、结果结算、O1 发布或方法代际发布，成功状态与权威读回矛盾、不可逆半结构、同一身份异义或所有者不一致都必须追根因解决，停止后继线程并保留值式证据。尤其是生命周期提交已经成立后，目标历史缺失、重复或异义不得降级成写前当前性漂移。

## 16. 迁移阶段终止与后继

## 17. 完成声明边界

只有 #315-#322 全部经任务分支、独立集成、设计归档和独立只读覆盖审计后，才可声明“固定生产根需求到方法新代际可见性的真实生成闭环第一轮形成”。

不得扩大为：默认入口已切换、完整自我苏醒、通用自主学习、全部根需求、所有动作类型、持久化恢复、控制面板业务操作、外部设备接入或旧能力迁移完成。
