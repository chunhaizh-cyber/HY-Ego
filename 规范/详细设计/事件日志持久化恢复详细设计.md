本详细设计只定义事件日志作为后续持久化和恢复材料的结构边界，不实现 C++，不接 SQL / ADO / 控制面板 / D455 / 体素 / 外设。

## 2. 定位

事件日志是已经由仓库正式入口或领域服务确认后的结构事件审计材料。

第一轮事件日志只允许记录：

```text
入口名。
事件类别。
仓库编号。
节点句柄。
主信息句柄。
关系句柄。
索引键。
结构事件发生顺序号。
结构事件时间戳。
写入结果。
```

事件日志不得反向裁决运行期事实。运行期事实仍以内存节点仓库、主信息仓库、关系仓库、索引仓库和领域服务当前结构为准。

## 3. 结构化事件材料字段

| 字段 | 含义 | 第一轮要求 |
| --- | --- | --- |
| 事件序号 | 事件日志文件内递增序号 | 后续 C++ 实施时由事件日志入口生成，不复用节点编号 |
| 事件时间戳 | 人读审计时间 | 只作审计材料，不作为状态 / 动态发生时间戳 |
| 入口名称 | 产生事件的仓库入口或领域服务入口 | 必填，空入口拒绝 |
| 事件类别 | 节点、主信息、关系、索引或领域结构事件 | 必填，未知类别拒绝 |
| 仓库编号 | 结构所属仓库编号 | 可选，但有句柄时必须一致 |
| 节点句柄 | 相关节点身份 | 可选，必须是结构入口确认后的句柄 |
| 主信息句柄 | 相关主信息身份 | 可选，必须是结构入口确认后的句柄 |
| 关系句柄 | 相关关系身份 | 可选，必须是结构入口确认后的句柄 |
| 索引键 | 相关索引候选或主键 | 可选，不裁决事实命中 |
| 写入结果 | 已确认写入、入口拒绝或业务失败 | 只记录调用结果，不修复结构 |
| 摘要 | 人读摘要 | 可选，不承载机器逻辑 |

第一轮不记录旧 SQL 字段、旧链表二进制字段、特征值 Vec 原始值、内容哈希、命中统计、控制面板显示字段、线程编号或外设快照。

## 4. 事件来源准入

事件日志只允许由以下已确认入口写入：

```text
节点仓库正式入口。
主信息仓库正式入口。
关系仓库正式入口。
索引仓库正式入口。
领域服务完成结构写入后的专用事件记录点。
```

禁止来源：

```text
控制台输出。
运行日志。
逻辑错误日志。
调试日志。
SQL 快照。
控制面板事件流水。
线程推进事件。
外设快照。
旧项目字段恢复比对。
```

事件日志写入必须发生在业务结构已经由正式入口确认之后。入口拒绝路径可以记录“拒绝事件”，但不得为了记录事件而补写节点、主信息、关系、索引、状态、动态、需求、任务或方法结构。

## 5. 写入方和读取方

| 角色 | 权限 |
| --- | --- |
| 仓库正式入口 | 后续可在写入成功后调用事件日志入口记录结构事件 |
| 领域服务 | 后续可在领域结构写入完成后记录领域结构事件 |
| 持久化适配器 | 后续只读取事件日志作为恢复候选材料，不直接裁决当前事实 |
| 验收工具 | 只读事件日志，检查格式和边界 |
| 运行期业务入口 | 不得从事件日志读取业务事实 |
| 显示层 | 只可作为审计材料只读展示 |

## 6. 生命周期

第一轮生命周期：

```text
结构入口确认。
事件材料组装。
事件日志入口前置拒绝。
追加写入事件日志。
写入结果只作为局部返回。
后续恢复设计读取事件日志作为候选材料。
```

事件日志可以归档、切分或重建索引，但不得替代仓库快照。事件日志文件损坏、缺失或写入失败时，不得改变业务结构，不得回滚已经成功的权威结构写入。

## 7. 与其它日志的边界

| 日志 | 用途 | 与事件日志边界 |
| --- | --- | --- |
| 运行日志 | 人读运行态观察 | 不记录结构恢复材料，不作为事件恢复来源 |
| 逻辑错误日志 | 人读逻辑错误诊断 | 不修复结构，不替代事件日志 |
| 调试日志 | 切片调试材料 | 默认关闭，不参与恢复 |
| 事件日志 | 后续审计和恢复候选材料 | 不裁决当前事实，不替代仓库快照 |

事件日志写入失败可以写运行日志或逻辑错误日志作为人读诊断，但诊断日志不得反向成为事件材料。

## 8. 写入失败和空内容拒绝

事件日志入口第一轮必须拒绝：

```text
空入口名称。
未知事件类别。
完全空结构材料。
未确认结构写入结果。
句柄字段与事件类别明显不匹配。
事件日志目录或文件无法写入。
```

写入失败处理：

```text
返回局部失败结果。
不补写业务结构。
不删除已确认业务结构。
不把失败写入主信息、关系、索引或状态节点。
必要时写人读运行日志或逻辑错误日志。
```

## 9. 不裁决运行期事实声明

事件日志不得用于：

```text
判断节点当前是否存在。
判断主信息当前是否有效。
判断关系当前是否成立。
判断索引当前是否命中。
判断需求是否满足。
判断任务是否完成。
判断方法是否成功。
判断动态或因果是否可靠。
修复悬空关系。
补写缺失结构。
恢复为当前事实时绕过领域服务。
```

恢复入口若后续实现，必须先通过快照版本、枚举版本、仓库编号、句柄版本、节点类型、关系目标和领域服务授权矩阵复核，再形成候选恢复材料。候选恢复材料仍不得直接覆盖当前运行期事实。

## 10. 后续 C++ 实施切片入口

后续若进入 C++，必须另建代码实施切片，并至少明确：

```text
允许文件。
禁止文件。
事件材料结构名称。
事件类别枚举。
事件日志写入入口函数名。
空内容拒绝方式。
写入失败返回方式。
默认入口验收输出。
日志写入失败不改变结构的检查方式。
```

第一轮建议入口：

```text
记录结构事件日志(事件材料)
事件材料是否可写入(事件材料)
```

上述名称只是后续候选，不构成本次代码许可。
