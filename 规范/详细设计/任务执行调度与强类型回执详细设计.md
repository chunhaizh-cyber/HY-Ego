# 任务执行调度与强类型回执详细设计

日期：2026-07-15

状态：JY-341、JY-344 / #224 专项详细设计 / #271、DQ-163、780 已完成且实际接口复核通过 / 代码未实现

## 1. 定位

本文替代 `规范/详细设计/任务生命周期筹办通用执行与结果结算详细设计.md` 中 `TASK-EXECUTION-S1` 的旧许可下传口径，只设计以下闭环：

```text
筹办中任务和当前方法选择
-> 冻结完整执行请求
-> 两阶段强类型请求队列
-> 排队中 / 执行中任务生命周期迁移
-> 单一受支持状态值变化适配器
-> 实际状态与动作动态同会话提交
-> 强类型非权威回执
```

本文不设计任务最终结果、已完成生命周期、需求结算、任意方法分派、外部执行器、外设动作或生产常驻线程主循环。任务结果与结算继续归 `#225 / DQ-117 / 610`。

## 2. 依据和当前代码事实

### 2.1 已验证当前事实

1. `任务业务服务` 公开面不含原始事务能力，已经提供任务读取、生命周期迁移、方法选择提交和实际结果完成入口。
2. `任务业务服务::读取动作桥请求` 只在任务处于 `执行中` 时返回六字段值式材料：任务、方法首、动作入口、稳定动作键、输入场景和输出场景。
3. `任务权威材料` 已包含双向承接关系、虚拟存在、场景、目标状态、当前生命周期和当前选择；`任务方法选择材料` 已包含选择记录、发布关系、方法投影、来源任务生命周期和来源方法生命周期身份。
4. `方法业务服务` 可读取方法登记项和按角色读取方法规格组，不执行任意方法。
5. `状态动态组合器` 当前只能在一次会话中同时新建前状态、后状态和动态；`动态业务服务` 可对两个已有状态记录动态，但“先创建后状态，再创建动态”是两个提交边界。
6. `状态动态数据操作` 已具备一次不透明写入会话、现有状态读回、实例状态写入、来源动作角色复核和实例动态写入能力，当前实例动态顺序合同为目标 / 前状态 / 后状态 / 来源动作 `1 / 2 / 3 / 4`。
7. `任务管理线程` 当前只持有运行消息队列；`发送任务执行请求` 不携带完整权威请求。
8. `任务工作线程` 当前的 `消费一次并生成回执` 只把运行消息转换为模拟普通回执，不调用业务服务或写状态 / 动态。
9. `任务工作线程` 已有独立强类型非权威回执队列；`任务结果回执协议` 已有版本、幂等、顺序、过期和句柄形状准入，但没有完整来源执行请求。
10. `运行期业务操作组合器` 和 `运行期业务请求路由` 固定为八类同步业务操作；`运行期业务装配` 尚未拥有任务执行组合器或任务线程对象。

### 2.2 当前缺口

```text
现有动作桥材料不足以排队后证明完整授权与选择当前性。
现有运行消息不能承载完整执行请求。
现有队列没有“先预留、后迁移权威任务、再确认可消费”的提交合同。
现有状态动态公开面不能原子完成“已有前状态 + 新后状态 + 动态”。
现有工作线程没有受控领域执行入口。
现有强类型回执不能携带完整来源执行请求供 #225 再复核。
```

## 4. 类型和公开入口

### 4.1 任务执行调度请求

提交方输入只允许包含：

```text
协议版本
请求编号、幂等键、任务序号、批次编号
任务句柄
创建时间、截止时间
排队生命周期幂等主键和时间戳
执行生命周期幂等主键和时间戳
待重筹办收口幂等主键和时间戳
实际状态幂等主键、发生时间戳和状态值
动态幂等主键
回执编号、回执幂等键和回执时间戳
```

入口必须先检查：

```text
所有编号非零且互不冲突。
句柄形状有效。
创建时间 <= 排队时间 <= 执行时间 <= 实际状态时间 <= 回执时间。
截止时间不早于创建时间。
收口时间不早于排队时间。
时间可转换为当前 I64 时间材料。
```

非法输入只返回来源错误，不读取队列、不调用服务、不补默认值。

### 4.2 冻结任务执行材料

`任务执行组合器::冻结` 必须从权威读取结果派生以下材料：

```text
任务身份、来源需求、虚拟存在、场景、目标状态。
需求拥有任务关系和任务来源需求关系的完整句柄 / 版本 / 端点。
筹办中生命周期关系、状态节点、版本和阶段。
选择记录身份、选择发布关系、方法投影关系、来源生命周期关系 / 版本。
方法身份、方法生命周期关系 / 版本和活跃状态。
动作入口身份、方法角色关系、角色状态关系、入口状态关系。
稳定动作键、输入场景和输出场景。
适配器种类：任务目标状态值变化。
```

冻结材料是可复制值式快照，不是授权能力。每次排队和执行前都必须重读权威结构并逐字段比较。

### 4.3 两阶段强类型请求候选

`任务管理线程` 新增：

```text
准备强类型任务执行请求(请求) -> 请求候选能力
确认强类型任务执行请求(候选能力)
撤销强类型任务执行请求(候选能力)
取出强类型任务执行请求()
读取强类型任务执行队列快照()
```

候选能力必须：

```text
不可默认构造、不可复制。
只绑定原任务管理线程、请求编号和内部预留代次。
移动后源对象失效。
未确认析构只撤销队列预留，不写业务结构。
确认后不再可撤销，且请求恰好一次进入可消费队列。
停止锁存后拒绝新预留；已成功预留的候选仍允许完成确认或撤销。
```

队列内部维护请求编号、幂等键和同任务最新序号。重复同完整请求返回精确重复；同键异义、序号倒退和容量不足写前拒绝。

### 4.4 任务执行组合器

组合器最小入口：

```text
冻结任务执行请求(调度请求)
确认任务进入排队中(冻结请求)
执行任务目标状态值变化(冻结请求, 当前时间)
收口过期或允许拒绝请求(冻结请求, 当前阶段)
```

组合器只依赖：

```text
任务业务服务
方法业务服务
状态动态组合器
```

组合器不依赖任务管理线程、工作线程、运行消息、仓库或数据操作。

### 4.5 状态动态原子扩展

`状态动态组合器` 新增请求：

```text
已有前状态
新后状态幂等主键、发生时间戳和值
动态幂等主键
被改变目标
来源动作
```

组合器先经状态服务读取前状态，派生场景和主体，形成后状态写入规格，再调用数据操作。

`状态动态数据操作` 在一次独占不透明会话中：

```text
重读已有前状态完整身份
-> 重读被改变目标
-> 非空来源动作角色复核
-> 写入新后状态
-> 写入实例动态，目标 / 前 / 后 / 来源动作顺序 1 / 2 / 3 / 4
-> 读回后状态和动态
-> 请求提交
```

动态写入失败、读回不完整或固定故障命中时，会话逆序撤销动态关系、动态节点 / 主信息和新后状态节点 / 主信息。已有前状态不得进入撤销写集。

### 4.6 任务执行调度路由

路由只组合队列候选和任务执行组合器：

```text
冻结
-> 准备队列候选
-> 组合器迁移任务到排队中并完整读回
-> 确认候选
```

路由不得把任务管理线程的队列锁带入组合器调用。正常漂移时撤销候选；内部不一致时撤销候选并返回追根因结果。

工作线程调用路由的消费入口：

```text
取出已确认请求
-> 过期 / 当前性分类
-> 组合器迁移到执行中
-> 唯一适配器执行
-> 生成强类型回执
-> 发布到既有强类型回执队列
```

### 4.7 强类型非权威回执

`任务结果回执协议` 保留现有兼容材料性质，并新增“非权威任务执行结果证据”。新材料必须包含：

```text
完整来源执行请求
实际结果状态
动态证据
目标方法和动作入口
场景和主体
回执编号、幂等键、来源任务序号、批次和时间
```

协议只检查值式一致性、幂等、顺序和过期，不裁决任务结果或需求满足。来源请求中的任务、方法、动作、场景和主体必须与回执句柄组一致。

## 5. 调用路径

### 5.1 提交路径

```text
调用方
-> 任务执行调度路由
-> 任务执行组合器冻结
-> 任务业务服务 / 方法业务服务只读重读
-> 任务管理线程准备队列候选
-> 任务执行组合器确认排队
-> 任务业务服务迁移生命周期
-> 任务管理线程确认队列候选
```

### 5.2 执行路径

```text
任务工作线程受控消费入口
-> 任务执行调度路由
-> 任务执行组合器复核并迁移执行中
-> 任务业务服务读取执行中动作桥
-> 状态动态组合器
-> 状态动态数据操作
-> 结构写入执行器 / 不透明会话
-> 实际状态与动态值式结果
-> 任务工作线程
-> 强类型非权威回执队列
```

## 6. 并发和锁边界

1. 强类型请求队列只使用自身互斥锁；锁内不得调用组合器、业务服务、日志、弹窗、文件、数据库或用户回调。
2. 队列候选预留后必须释放队列锁，再迁移任务生命周期。
3. 任务生命周期和动作写入分别由现有数据操作层取得结构许可；队列等待期间不持有结构许可。
4. 工作线程不得保存服务、数据操作、执行器、仓库或原始令牌；受控消费入口只接收任务执行调度路由。
5. 同一请求只能由一个已确认队列项消费；重复请求按完整值式材料幂等或拒绝。
6. 不新增每任务线程、`std::async`、`std::future`、工作窃取或外设等待。

## 7. 非成功返回二分

| 场景 | 分类 | 结构收口 |
| --- | --- | --- |
| 参数、句柄、编号、时间无效 | 逻辑内返回 | 写前拒绝，结构和队列不变化 |
| 任务不是筹办中或没有当前选择 | 逻辑内返回 | 不冻结、不入队 |
| 稳定动作键不受支持或场景不匹配 | 逻辑内返回 | 不注册临时适配器 |
| 队列停止、已满、重复、过期 | 逻辑内返回 | 不迁移任务 |
| 队列候选后任务发生正常并发漂移 | 逻辑内返回 | 精确撤销队列候选 |
| 请求取出前已合法离开对应排队状态 | 逻辑内返回 | 丢弃非权威请求，不回退任务 |
| 请求过期且任务仍为精确排队中 | 逻辑内收口 | 经任务服务迁移待重筹办 |
| 动作适配器发生允许拒绝或版本漂移 | 逻辑内收口 | 经任务服务迁移待重筹办，不生成成功回执 |
| 同一当前版本授权 / 选择 / 方法 / 动作结构矛盾 | 追根因解决 | 停止动作写入 |
| 任务已排队但候选确认失败 | 追根因解决 | 停止 600，不用运行消息补请求 |
| 后状态或动态写入 / 读回非预期 | 追根因解决 | 同会话精确撤销本次后状态和动态 |
| 新回执与来源执行请求同版本矛盾 | 追根因解决 | 不发布上行回执，不改机器事实 |

## 9. 验收设计

1. 无效参数在首次服务读取前拒绝，数量与队列快照不变。
2. 完整冻结材料覆盖双向授权、生命周期、选择记录、方法投影、方法生命周期和动作来源。
3. 不受支持稳定动作键、跨场景动作和多动作入口不入队。
4. 队列候选未确认不可消费；析构撤销预留；移动后源能力失效。
5. 停止、容量、重复、同键异义、乱序和过期均不改变筹办中任务。
6. 任务排队中读回完成后才确认候选；候选确认不调用领域服务。
7. 过期已确认请求只经任务服务迁移待重筹办。
8. 排队中到执行中使用请求内独立幂等材料并完整读回。
9. 执行前逐字段重读冻结授权、选择、方法和动作。
10. 唯一适配器只能处理构造时配置的稳定动作键。
11. 新后状态和动态同会话提交，关系顺序保持 `1 / 2 / 3 / 4`。
12. 每个新写点固定故障后，后状态、动态、关系和数量回到前态。
13. 成功回执附完整来源执行请求，仍标记非权威且不写任务结果。
14. 旧强类型回执兼容路径原义保持；新回执不能降级为旧材料性质。
15. 工作线程不直接持业务服务、仓库、许可、令牌、会话或执行器。
16. 新生产文件中 `结构事务令牌|结构事务许可|结构写入会话|结构写入执行器|主信息仓库|节点仓库|关系仓库|索引仓库` 公开能力命中为 0。
17. Debug / Release x64、完整自检、Release 零登记和最终连续 20/20 通过。
18. 阶段 600 唯一登记；#225 只因 #224 正式完成解除直接前置，不自动实施 610。
