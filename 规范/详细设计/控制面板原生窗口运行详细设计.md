# 控制面板原生窗口运行详细设计

更新时间：2026-07-10

状态：已生成 / 已作为 PANEL-RUN-S1 上游材料 / 待计划索引同步 / 未改 C++ / 未构建 / 未运行程序

## 1. 设计定位

本设计把已验证的控制面板只读服务壳落到可见、可关闭、可刷新的人读原生窗口。第一轮目标是“运行程序即可看到控制面板”，不是完整领域数据浏览器。

技术路线固定为：

~~~text
同一海中鱼巣可执行程序。
原生 Win32 窗口和 Windows 公共控件。
main 自检通过后进入窗口消息循环。
默认启动控制面板。
--headless 保留无窗口常驻模式。
--self-test-exit 保持现有自检后退出语义。
~~~

不使用 WebView、HTML 运行通道、JSON 服务、HTTP 服务、SQL、ADO 或数据库。

## 2. 当前事实

~~~text
控制面板服务.h 已提供只读总览、六类树候选、操作请求壳和后续门禁。
显示服务.h 已提供只读来源和越权拒绝。
统计服务.h 已提供节点、关系、索引数量快照。
入口.cpp 已完成 CONSOLE-PANEL-A1 至 A9 自检，但默认只 sleep 常驻。
vcxproj 是 C++20 / UTF-8 / Console 应用，当前无原生窗口源文件。
~~~

## 3. 启动参数

| 参数 | 语义 | 优先级 |
| --- | --- | --- |
| --self-test-exit | 自检通过后直接退出，不创建窗口 | 最高 |
| --headless | 自检通过后沿用现有常驻循环，不创建窗口 | 次高 |
| 无参数 | 自检通过后创建并运行原生控制面板 | 默认 |

未知参数继续按当前保守策略忽略，不将参数文本作为机器事实。

## 4. 窗口布局

第一轮窗口采用安静、操作型布局：

~~~text
顶部标题区：海中鱼巣控制面板、只读状态、运行模式。
左侧导航区：固定六类树根。
右侧总览区：节点数、关系数、索引数、选中分类、材料来源和权威边界。
底部命令区：刷新、生成操作请求、关闭。
底部状态栏：最近一次人读操作结果。
~~~

控件使用稳定尺寸和 WM_SIZE 响应式重排，窗口最小尺寸必须保证中文标签不重叠、不截断关键状态。

六类树根严格为：

~~~text
世界树
概念树
语素树
需求树
任务树
方法树
~~~

状态和动态只出现在右侧详情说明，不作为第七、第八棵树。

## 5. 新增会话结构

### 5.1 控制面板窗口快照

用途：

~~~text
承载当前窗口一次刷新所需的人读只读材料。
~~~

字段候选：

~~~text
控制面板材料 总览材料
array<控制面板树视图材料, 6> 树视图材料组
控制面板操作请求材料 操作请求材料
结构统计快照 启动结构统计
wstring 状态文本
~~~

写入方：

~~~text
main 编排入口创建启动结构统计。
控制面板窗口通过控制面板服务刷新只读总览、树候选和操作请求壳。
~~~

读取方：

~~~text
控制面板窗口过程和控件刷新函数。
~~~

生命周期：

~~~text
只存在于当前进程和当前窗口会话；不持久化，不写回领域结构，关闭窗口后销毁。
~~~

验证：

~~~text
来源显示服务为 true。
允许写业务事实为 false。
六类树候选均为可展示分类。
操作请求壳已生成但已执行动作为 false。
窗口操作前后节点、关系、索引数量不变。
~~~

### 5.2 控制面板窗口运行结果

用途：区分正常关闭、入口拒绝和内部启动错误的人读结果，不承载业务状态。

字段候选：

~~~text
bool 已显示窗口
bool 正常关闭
bool 逻辑内返回
bool 追根因错误
wstring 失败阶段
~~~

## 6. 代码边界

### 6.1 控制面板窗口

新建：

~~~text
海中鱼巣/界面/控制面板窗口.h
海中鱼巣/界面/控制面板窗口.cpp
~~~

职责：

~~~text
初始化公共控件。
注册窗口类。
创建主窗口、树视图、只读文本区、命令按钮和状态栏。
处理 WM_SIZE、WM_COMMAND、WM_NOTIFY、WM_TIMER、WM_CLOSE 和 WM_DESTROY。
从控制面板服务刷新人读材料。
运行和退出 Win32 消息循环。
~~~

不得：

~~~text
直接持有节点、主信息、关系或索引仓库可写引用。
调用需求、任务、方法、状态、动态或因果写入口。
根据显示文本裁决机器事实。
创建独立 UI 工作线程。
~~~

### 6.2 main 编排入口

入口.cpp 只负责：

~~~text
解析 --headless。
自检通过后准备启动结构统计快照。
默认调用控制面板窗口运行入口。
headless 时沿用现有常驻循环。
把窗口正常关闭转换为程序正常退出。
~~~

不得把窗口线程、按钮或消息解释为动作来源。

### 6.3 工程文件

海中鱼巣.vcxproj 只增加窗口 .cpp / .h 编译登记。Windows 公共控件库通过明确链接配置或受控编译指令接入，不改变 Console 子系统。

## 7. 交互边界

### 7.1 选择树根

选择只改变右侧人读详情：

~~~text
分类名称。
是否可作为树视图分类。
状态 / 动态只能作为详情的说明。
当前第一轮尚未接入真实领域树子节点。
~~~

### 7.2 刷新

刷新只重新调用：

~~~text
控制面板服务::生成只读总览材料
控制面板服务::生成六类树视图候选材料
~~~

第一轮结构统计是窗口启动快照。刷新不得直接重新读取仓库；后续如需实时统计，必须新增显示 / 统计只读聚合计划。

### 7.3 生成操作请求

按钮只调用：

~~~text
控制面板服务::生成操作请求壳
~~~

窗口显示“请求已生成、动作未执行”，不得调用任何领域写入口。

### 7.4 关闭

关闭窗口、Alt+F4、标题栏关闭按钮和停止信号均走统一销毁与消息循环退出路径。

## 8. 非成功路径

逻辑内返回：

~~~text
--self-test-exit。
--headless。
用户关闭窗口。
控制面板服务拒绝写事实、文本裁决、数据库裁决或外部通道请求。
~~~

拒绝后证明：

~~~text
不创建新的领域节点、主信息、关系或索引。
窗口只更新人读状态栏。
结构统计启动快照保持不变。
~~~

追根因解决：

~~~text
合法只读请求却返回非显示服务来源或允许写事实。
窗口类注册后主窗口创建失败。
任一必需控件创建失败。
消息循环返回错误。
刷新后只读材料违反已确认边界。
~~~

追根因路径必须记录失败阶段、销毁已创建窗口资源并退出，不得显示空白窗口后宣称成功。

## 9. 线程和锁边界

~~~text
不新增 UI 线程。
窗口消息循环运行在 main 线程。
WM_TIMER 只轮询停止信号，不写业务结构。
窗口不持有仓库锁，不在消息处理期间执行持久化 I/O、日志扫描、外设等待或用户可写回调。
控制面板服务和统计快照只提供值材料。
~~~

## 11. 验证命令

代码实施计划登记可执行队列后至少执行：

~~~powershell
git diff --check
python .\tools\check_specs.py
msbuild .\海中鱼巣.vcxproj /p:Configuration=Debug /p:Platform=x64 /m
.\x64\Debug\海中鱼巣.exe --self-test-exit
~~~

窗口验收必须额外完成：

~~~text
启动默认程序。
确认进程持续运行并找到“海中鱼巣控制面板”顶层窗口。
确认六类树根、三个结构计数、刷新、操作请求和关闭控件存在。
改变窗口大小后检查无重叠。
发送 WM_CLOSE 并确认进程正常退出。
启动 --headless 并确认不出现窗口、进程保持常驻，随后正常停止。
~~~

## 12. 明确排除

~~~text
不接 SQL、ADO、数据库、WebView、HTML、JSON、HTTP、本机服务端口。
不接 D455、体素、外设或真实控制面板写操作。
不展示真实六类领域树子节点。
不新增业务字段、节点类型、关系类型或索引类型。
不把窗口状态、控件文本或按钮状态作为机器事实。
不声明旧控制面板能力迁移完成。
~~~
