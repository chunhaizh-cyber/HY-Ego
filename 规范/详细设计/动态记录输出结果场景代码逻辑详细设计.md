## 5. 请求分流

```text
状态服务路径记录实例动态
-> 校验场景、存在、发生时间戳、被改变目标、前后值和来源动作
-> 状态服务创建前后实例状态
-> 读回前后状态材料
-> 创建动态节点和发生时间戳
-> 写场景临时、主体、发生场景、目标、前值、后值和可选来源动作关系
-> 读回动态材料。

无状态服务重载记录实例动态
-> 校验基础入参
-> 创建值材料状态
-> 写同组动态关系
-> 读回前后 I64 值
-> 明确该路径不等于完整实例状态材料路径。

记录状态变化动态
-> 读取前后状态材料
-> 要求主体一致、场景一致、被改变目标有效、来源动作有效或为空
-> 使用后状态时间戳创建动态
-> 写同组动态关系
-> 读回动态材料。

输出结果场景材料边界
-> 读取动态发生场景
-> 读取方法结果场景
-> 读取动作输出规格场景
-> 当前只作为相邻材料
-> 不写任务实际结果状态。

动态候选读取
-> 读取动态材料
-> 读取动态聚合候选或运动基元候选
-> 候选只表达变化量和材料可用性
-> 不裁决任务完成、方法成功或稳定因果。
```

## 7. 服务边界

| 服务 | 本设计允许 | 禁止越界 |
| --- | --- | --- |
| 动态服务 | 写实例动态、状态变化动态、值材料型动态；读取动态材料和候选材料 | 不写任务实际结果状态，不生成稳定因果结论 |
| 状态服务 | 创建完整实例状态和读取状态材料 | 不把抽象状态误当实例状态 |
| 场景服务 | 收纳实例动态和读取场景临时材料 | 不裁决任务结果 |
| 方法服务 | 提供方法结果场景、动作输出规格和动作动态证据来源 | 不把规格场景直接写成任务结果 |
| 任务服务 | 后续第 13 项写任务实际结果状态 | 本设计不写任务结果 |
| 因果服务 | 后续以来源动态作为轻量因果准入证据 | 不把单条动态升级为稳定因果结论 |
| 显示层 / 日志 | 只做人读观察和诊断 | 不写动态事实或结果场景事实 |

## 中途非成功返回二分口径

本文件按 2026-07-09 硬规则修订：中途非成功返回只分为 `追根因解决` 和 `逻辑内返回`。

- `追根因解决`：前置条件已经满足，并进入创建、绑定、写关系、写状态、记录动态、结算、读回或结构承载后，结果不符合内部预期；必须停止依赖路径，定位根因，当前未证明完整回滚时登记事务隔离缺口或半结构隔离缺口。
- `逻辑内返回`：领域协议允许的拒绝、候选为空、请求材料返回或人读材料返回；必须保证结构不变化，且返回材料、日志、回执、显示或控制台输出不裁决机器事实。
