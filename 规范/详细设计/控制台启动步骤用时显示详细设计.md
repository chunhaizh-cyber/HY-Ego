# 控制台启动步骤用时显示详细设计

日期：2026-07-10

状态：已生成 / 与配套流程图范围一致 / START-TIME-S1 实施依据

## 1. 定位

在 `海中鱼巣/入口.cpp` 增加会话期人读启动计时器，显示各启动阶段用时和从进入 `main` 起算的累计用时。计时材料只存在于调用栈和控制台文本中，不进入日志、仓库、缓存、数据库或显示服务。

## 2. 当前事实

```text
入口已包含 <chrono>，但尚无启动阶段计时输出。
入口依次完成参数解析、运行日志入口、仓库与服务构造、世界树初始化、语素初始化、核心自检、SQL 审计、停止信号和运行入口分流。
控制面板窗口的 运行() 会阻塞到窗口关闭，因此入口只能准确声明“控制面板运行入口就绪”，不能把窗口存续时间计为启动时间。
```

## 3. 计时器

在匿名命名空间新增 `启动用时显示器`：

```text
时钟：std::chrono::steady_clock
总起点：构造计时器时记录
步骤起点：上一个步骤完成时更新
完成步骤(名称)：输出本步骤毫秒数和累计毫秒数
输出总用时(状态)：输出当前累计毫秒数，不改变步骤起点
```

输出通过局部 `std::ostringstream` 使用固定三位小数，避免修改 `std::cout` 的全局格式状态。步骤行输出完成后重新记录下一步骤起点，使观察器自身的控制台输出开销只进入累计总时间，不计入下一业务步骤的独立用时。

## 4. 输出格式

```text
[启动用时] <步骤>: <本步骤> ms | 累计=<累计> ms
[启动总用时] <状态>: <累计> ms
```

第一轮步骤：

```text
参数解析
运行日志入口
核心仓库与服务构造
世界树初始化
语素初始化
核心启动自检
SQL Server 结构统计审计投影
常驻停止信号安装
控制面板运行入口准备（窗口模式）
```

总用时状态：

```text
核心启动自检失败
自检模式完成
数据库自检模式完成 / 失败
常驻停止信号安装失败
控制面板运行入口就绪
无窗口常驻入口就绪
```

## 5. 调用边界

计时调用只插入既有步骤之后；不包裹、替换或重新排序现有领域调用。SQL 不可用、程序自检失败、信号安装失败和控制面板失败继续使用既有裁决与返回值。

## 6. 验收

| 编号 | 条件 |
| --- | --- |
| START-TIME-A1 | `--self-test-exit` 输出参数、日志、构造、世界树、语素、自检六个步骤 |
| START-TIME-A2 | 每个步骤同时输出非负步骤毫秒数和非负累计毫秒数 |
| START-TIME-A3 | 同一次启动的累计毫秒数不递减 |
| START-TIME-A4 | 自检模式输出一条 `[启动总用时] 自检模式完成` |
| START-TIME-A5 | 数据库模式输出 SQL 审计步骤和数据库自检总用时 |
| START-TIME-A6 | 计时代码不写日志、仓库、缓存、SQL 或业务状态，不参与任何条件判断 |

## 7. 完成边界

只可声明控制台已显示第一轮启动步骤用时。不得声明已经建立性能基线、启动性能已经优化、控制面板窗口显示耗时已精确测量或计时材料可作为机器事实。
