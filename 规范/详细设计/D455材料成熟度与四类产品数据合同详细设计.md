# D455 材料成熟度与四类产品数据合同详细设计

日期：2026-07-21

版本：v0.1

状态：正式详细设计 / DD-01 / 数据合同设计完成 / 代码未实现 / 不授权生产接线

## 1. 设计目标

本设计把正式规范中的两个独立维度落实为海中鱼巣可实现的强类型值式合同：

```text
材料成熟度轴：L0 -> L1 -> L2 -> L3

逻辑供给产品轴：
原始逐簇报告
稳定观察子集报告
扫描变化报告
目标跟踪报告
```

目标不是实现具体滤波、分割、跟踪或队列，而是先固定：

```text
产品身份
成熟度与产品类型组合
来源继承
观察范围
时间与时效
RGB-D / XYZ / 点云质量
局部特征质量
L3 稳定门
产品专用负载
逻辑内拒绝与内部逻辑错误
```

上述合同是 DD-02 报告队列与生产者生命周期、DD-03 任务域承接以及 DD-04 方法材料入口的共同上游。

## 5. 基础强类型

```text

L2跨轮关联

```

成熟度只表达当前材料达到的最高处理层级，不表达产品种类、队列、运行模式、质量总分或业务可用性。

### 5.2 观察运行模式

```text
enum class D455观察运行模式
未知
逐簇识别
扫描变化
目标跟踪
```

运行模式是等待项或订阅项的外显业务意图，不是相机全局状态。ROI、质量诊断、重捕获和补观察不得成为第四模式。

### 5.3 逻辑供给产品类型

```text
enum class D455逻辑供给产品类型

```

产品类型必须由强类型字段和 `std::variant` 等唯一负载分支共同成立；不得用自由文本、L0—L3 或消息类型替代。

### 5.4 结果分类

```text

有效
逻辑内拒绝
内部逻辑错误
```

验证结果至少包含结果分类和强类型原因。人读说明可附加，但不得参与裁决。

## 6. 身份、来源与范围结构

### 6.1 产品身份

```text
D455供给产品身份
- 报告编号：非零、在同一生产者世代内单调、不复用
- 报告版本：从 1 开始；已发布内容不可原地改写
- 生产者世代：区分设备重启或生产者重建
```

该身份只定位外设产品，不成为世界存在身份或任务身份。

### 6.2 受控材料引用

```text
D455受控材料引用
- 材料域编号
- 材料编号
- 材料版本
- 材料种类
- 来源批次号
- 来源摘要
```

材料种类至少区分：同步帧、点云、有效掩码、簇掩码、彩色局部图、深度局部图、跨轮关联材料、稳定性材料和基准材料。

引用不携带裸指针或 SDK 缓冲地址。DD-02 必须给出材料域的所有者和解析入口；引用不能自行证明目标仍存在或仍当前。

### 6.3 等待项和目标约束引用

产品头只携带外设侧可回显的不透明约束引用：

```text
D455等待项引用：等待项编号、版本
D455目标约束引用：目标材料编号、版本
```

产品不得直接写入或比较任务节点、需求节点或存在节点。DD-03 由任务域用权威等待项重新匹配任务和工作项。

### 6.4 观察范围

```text
D455观察范围材料
- 范围编号和版本
- 坐标系类型与坐标系版本
- ROI 左上、宽和高
- 可选范围掩码引用
- 可选目标约束引用
```

零面积 ROI、未知坐标系、掩码与图像尺寸不一致属于发布前逻辑内拒绝。已进入准备发布后读回出现上述矛盾属于内部逻辑错误。

### 6.5 时间窗口

```text
D455材料时间窗口
- 最早来源时间
- 最晚来源时间
- 宿主形成时间
- 最大允许年龄
```

时间必须同域或携带明确转换版本。形成时已经过期、时间倒置或最大允许年龄为零，不得发布。

## 7. L0—L3 成熟材料合同

### 7.1 L0 原始材料

L0 至少包括：

- 原始同步帧受控引用；
- 彩色、深度、左右红外来源布局；
- 点云材料说明；
- 深度单位、有效深度掩码和有效点掩码；
- 相机内外参、坐标系和位姿版本；
- 设备、批次、采集和宿主时间材料。

点云材料说明至少包含单位、参考坐标系、总点数、有效点数、点云引用和有效掩码引用。点云可以由原始深度派生，但必须保留来源深度引用和派生规则版本；不得覆盖原始深度。

### 7.2 L1 单轮处理材料

L1 在 L0 来源链上增加：

- 校正、对齐、滤波和分割规则版本；
- 本轮观察像素簇集合；
- 每簇掩码、平面轮廓、局部图和范围；
- 每簇每类特征的单轮质量和不可得原因；
- 未解释、低置信和不可观测区域掩码。

L1 不能声明跨轮稳定。没有完成分割时，可以形成只含 L0 来源和具名“尚未形成簇”的原始逐簇产品，用于质量门和诊断；不得伪造空簇为稳定观察子集。

### 7.3 L2 跨轮关联材料

L2 在 L1 上增加：

- 参与轮次和时间范围；
- 跨轮关联材料引用；
- 外设内部轨迹号及其生产者世代；
- 连续命中次数、缺失次数、复现离散度和稳定性评分材料；
- 特征级稳定、降级、不稳定或不可得状态；
- 当前仍未通过 L3 门的具名原因。

外设内部轨迹号只作跨轮回查，不能成为世界存在身份。

### 7.4 L3 稳定可交付材料

L3 必须同时满足：

```text
L0 来源完整且可解析
RGB-D 对齐状态明确
有效深度和有效空间坐标达到等待项要求
观察范围和坐标系闭合
材料在允许年龄内
跨轮参与数量达到规则门槛
连续命中和复现离散度达到规则门槛
每个准备交付的簇及特征都有局部质量状态
补全材料用途限制已保留
稳定规则版本、阈值来源和输入版本可回查
```

L3 只表示“允许任务域承接为业务方法运行材料”，仍不表示已确认存在、状态变化或目标可见事实。

## 8. 局部质量合同

### 8.1 质量等级

```text
enum class D455局部质量等级
未知
稳定可用
降级可用
不稳定
不可得
```

不得用一个整帧质量枚举覆盖全部局部簇和全部特征。

### 8.2 特征级质量材料

每项质量材料至少包含：

```text
局部对象引用
材料特征类型
质量等级
量化值或区间
阈值来源
规则版本
来源材料引用组
用途限制
不可得原因
```

材料特征类型至少覆盖对齐、深度有效率、空间坐标有效率、边界深度稳定、平面轮廓稳定、颜色稳定、尺寸稳定、位置稳定和跨轮复现。

未知、不可得或不稳定时不得填业务默认值。补全深度支撑的结果必须标记降级或不可得用途，不能独立通过 L3 存在确认相关门。

## 9. 四类产品负载

### 9.1 公共产品头

每类产品共同包含：

```text
产品身份
合同内容版本
来源外设身份材料
等待项引用
运行模式
产品类型
成熟度
来源材料引用组
观察范围
材料时间窗口
配置版本
质量规则版本
整体供料结果分类
```

整体供料结果只表达本次产品是否有可交付负载、空集合或具名降级，不覆盖局部质量状态。

### 9.2 原始逐簇报告

负载至少包含：

- L0 来源说明；
- 可选 L1 单轮簇集合；
- 可选 L2 跨轮候选稳定材料；
- 未解释、低置信和不可观测区域；
- 每簇掩码、局部图、平面轮廓和特征级质量；
- 尚未形成 L1 或尚未达到 L3 的强类型原因。

该产品只进入外设内部回查、任务域质量门、短期缓存和诊断，不能成为自我侧方法正式输入。

### 9.3 稳定观察子集报告

负载至少包含：

- 来源原始逐簇产品身份和受控引用；
- L3 稳定门证据；
- 通过稳定门的簇引用子集；
- 连续命中、复现离散度和空间锚定；
- 每簇每特征的稳定、降级、不稳定或不可得状态；
- 未通过簇的具名排除摘要。

不得复制整份原始逐簇材料。稳定子集为空可以作为“本轮稳定门已执行但无可承接簇”的合法空结果交任务域处理；不得交观察方法伪装成有效材料。

### 9.4 扫描变化报告

负载至少包含：

- 可比基准引用和基准版本；
- 本轮覆盖范围；
- 新增、消失候选、移动候选、尺寸候选和风险候选的强类型集合；
- 未覆盖集合与强类型原因；
- 当前 L3 来源和质量材料；
- 报告年龄和比较规则版本。

没有可比历史基准时不允许发布“扫描变化报告”。此时只能形成当前材料或结构化无基准结果，不能宣布变化。

### 9.5 目标跟踪报告

负载至少包含：

- 目标约束引用；
- 观察窗口；
- 跟踪状态：复现、降级复现、暂失、丢失或重捕获候选；
- 新空间坐标或明确空信息；
- 最近观测年龄；
- 特征复现离散度；
- 丢失原因和重捕获候选引用。

目标暂失或丢失是可发布的强类型结果，不能被静默舍弃。没有目标约束或观察窗口时不得形成目标跟踪产品。

## 10. 成熟度、模式和产品组合矩阵

| 产品类型 | 合法模式 | 合法成熟度 | 业务方法正式输入权 |
| --- | --- | --- | --- |
| 原始逐簇报告 | 逐簇识别 | L0、L1、L2 | 无，只到任务域质量门 / 回查 / 诊断 |
| 稳定观察子集报告 | 逐簇识别 | 仅 L3 | 任务域承接后只交观察方法 |
| 扫描变化报告 | 扫描变化 | 仅 L3 | 任务域承接后只交扫描方法 |
| 目标跟踪报告 | 目标跟踪 | 仅 L3 | 任务域承接后只交跟踪方法 |

固定拒绝：

```text
原始逐簇 + L3
稳定观察子集 + L0/L1/L2
扫描变化 + L0/L1/L2
目标跟踪 + L0/L1/L2
任何产品 + 未知模式或未知成熟度
识别方法直通任何外设产品
产品类型与 variant 负载分支不一致
```

原始逐簇来源即使参与了后续 L3 计算，其自身产品类型和内容仍保持 L0—L2 原始 / 中间供给身份；L3 交付另形成稳定观察子集或定向扫描 / 跟踪产品。

## 11. 来源继承和不可变规则

1. 每次成熟度提升必须保留直接输入材料引用、处理规则版本、时间范围和质量输出。
2. L1 不覆盖 L0，L2 不覆盖 L1，L3 不覆盖 L2；后层只引用前层。
3. 稳定观察子集必须引用来源原始逐簇产品，不复制掩码、局部图或原始帧。
4. 扫描变化和目标跟踪必须同时引用 L3 来源以及各自基准或目标约束。
5. 发布后的产品身份、类型、成熟度、来源、范围、时间和负载不可原地改写；新内容使用新报告版本或新身份。
6. 索引、摘要、缓存命中和显示名称不得代替完整来源读回。

## 13. 入口拒绝、逻辑内返回和内部逻辑错误

| 阶段 | 逻辑内拒绝 | 内部逻辑错误 |
| --- | --- | --- |
| 产品构造前 | 零身份、未知枚举、过期、范围无效、缺来源、组合矩阵不合法 | 不适用，尚未进入结构承载 |
| 成熟度提升 | 质量门未通过、跨轮不足、对齐或深度不满足、当前无稳定簇 | 已声明提升成功但缺直接来源、规则版本或读回证据 |
| 产品形成 | 合法空集合、无基准、目标约束缺失、特征不可得 | 类型字段与负载分支矛盾、L3 负载缺 L3 证据、来源链自循环 |
| 候选准备后 | 版本过期、发布窗口关闭可在写前拒绝 | 候选读回与输入产品身份、类型、成熟度、来源或负载不同 |
| 发布后读回 | 已过期、已撤销、普通读取目标不存在按 DD-02 协议处理 | 同一已发布身份出现两个不同内容、完整读回与发布内容不一致 |

内部逻辑错误必须阻断依赖路径并追根因，不能包装成低质量、无候选、等待或普通失败。

## 15. 后继边界

### 15.1 DD-02 承接

DD-02 负责：

```text
产品生产者生命周期
唯一设备控制域
产品候选准备、确认发布和精确撤销
强类型报告队列和四类只读视图
产品身份分配
材料域解析
时效、一次消费和不可舍弃事件
```

DD-01 不提前定义队列锁、物理分队列数量、消费游标或任务绑定状态。

### 15.2 DD-03 / DD-04 承接

DD-03 决定任务域如何匹配报告并形成已承接材料引用；DD-04 决定冻结工作包如何把稳定观察、扫描或跟踪材料交给对应方法。DD-01 只固定类型可区分和来源可验证，不授予消费权。

### 15.3 生产算法边界

具体校正、滤波、分割、点云、跨轮关联、扫描差异和跟踪算法可替换，但它们必须输出本设计的强类型合同。算法分数、缓存命中和轨迹 ID 不得绕过合同成为世界事实。
