# 运行期组合器与线程路由去令牌详细设计

日期：2026-07-14

状态：正式详细设计 / JY-336 / 对应 #270 / 不构成代码已实现声明

## 1. 目标

在 `#265-#269`、`#275` 已形成的隔离无令牌服务面之上，建立唯一运行期业务装配和线程侧强类型请求路由，使调用方向稳定为：

```text
线程 / 消息值式请求
-> 运行期业务请求路由
-> 运行期业务操作组合器
-> 已有领域组合器 / 无令牌业务服务
-> 服务专用数据操作层
-> 结构写入执行器 / 不透明会话
-> 四仓库
```

本设计只形成代码施工依据。它不迁移旧自我治理循环正文，不初始化系统角色，不发布生产运行期上下文，不删除兼容服务，也不恢复 `#214`。

## 2. 当前代码事实

### 2.1 已验证事实

1. `启动.运行期上下文.ixx` 当前只拥有协调器、接线和四仓库，`完整()` 固定为 `false`，宿主不能发布生产上下文。
2. `入口.cpp` 仍手工构造传统服务和四个历史初始化器，生产初始化仍由旧自我线程承担。
3. 四个 `初始化.*.ixx` 仍是 `ClInclude + #include` 兼容文件，不是 C++ 真模块。
4. `线程/路由.概念命名治理.ixx` 仍直接取得独占许可并向传统服务传递原始令牌；旧自我治理领域路由仍直接依赖传统服务和索引仓库。
5. `#265-#269`、`#275` 已形成七个服务专用数据操作对象、十四个无令牌业务服务和六个组合器；只有数据操作层导入结构写入执行器。
6. 新分层产物仍是隔离路径，既有生产调用尚未迁移。

### 2.2 不得写成当前事实的假定

```text
系统角色已经由新服务初始化。
运行期上下文已经拥有新服务装配。
旧自我线程或概念命名路由已经去令牌。
概念生命周期、统计缓存或安全删除已经完成新分层。
生产运行期上下文已经可以发布。
```

以上均只能由本计划或后继计划实现并验证后再声明。

## 3. 第一轮范围

### 3.1 纳入

1. 新建唯一 `运行期业务装配`，按固定顺序拥有已经形成的新分层对象。
2. 新建 `运行期业务操作组合器`，只封装六个既有组合器的八类强类型操作。
3. 新建线程侧 `运行期业务请求路由`，使用强类型变体请求 / 结果进行唯一分派。
4. 让 `运行期上下文` 拥有业务装配并可证明服务装配就绪。
5. 保持系统角色初始化未完成时上下文不可生产发布。
6. 建立独立自检，证明装配唯一、请求去令牌、分派守恒、拒绝不改结构和兼容路径不回退。

### 3.2 排除

```text

不迁移四个历史初始化文件。
不让入口切换到新运行期上下文。
不实现系统角色初始化或首次生产发布。
不实现真实动作调度、工作队列、线程创建或新的异步循环。

```

## 4. 运行期业务装配

### 4.1 新模块

```text
文件：海中鱼巣/装配.运行期业务.ixx
模块：海中鱼巣.装配.运行期业务
类型：生产真模块 / ClCompile
```

装配接收同一份 `结构事务接线`、主信息仓库、节点仓库、关系仓库、索引仓库和关系仓库编号。装配不拥有四仓库、协调器、宿主或租约；其生命周期由 `运行期上下文` 拥有。

### 4.2 成员矩阵

| 层 | 固定成员 | 数量 | 依赖边界 |
| --- | --- | ---: | --- |
| 内部原子材料 | `特征值服务` | 1 | 只供 `特征体系数据操作`，不向运行期路由公开 |
| 数据操作 | 存在场景、状态动态、特征体系、语素基础、轻量因果、概念图结构、需求任务方法 | 7 | 持有仓库 / 接线 / 执行器；不向线程公开 |
| 业务服务 | 存在、场景、状态、动态、特征、二次特征、基础信息、语素、轻量因果、概念图结构、需求、任务、方法、方法召回 | 14 | 无原始令牌公开面 |
| 业务服务补充 | `特征值服务` 不计业务公开服务；`方法` 已计入上行 | 0 | 防止把内部特征值能力误读为高级服务入口 |
| 既有组合器 | 存在场景、状态动态、特征状态、语素概念入口、概念结构发布、需求任务方法 | 6 | 只组合业务请求，不持事务能力 |
| 顶层组合器 | 运行期业务操作组合器 | 1 | 只封装六个既有组合器的八类入口 |

第一轮业务服务总数按当前正式类型计为十四个；旧 `#248` 的“十六个传统服务成员”矩阵撤回，不得把显示、统计或传统世界聚合服务凑入新装配。执行前若实际服务类型或构造依赖发生变化，按接口漂移退回设计，不由执行窗口临时增减。

### 4.3 构造顺序

```text
内部特征值服务
-> 存在场景数据操作
-> 状态动态数据操作
-> 特征体系数据操作
-> 语素基础数据操作
-> 轻量因果数据操作
-> 概念图结构数据操作
-> 需求任务方法数据操作
-> 按依赖顺序构造十四个业务服务
-> 六个既有组合器
-> 运行期业务操作组合器
```

所有数据操作必须引用同一接线和同一四仓库实例。`需求任务方法数据操作` 复用同一装配内的存在场景、状态动态数据操作；方法召回服务复用同一装配内的需求、任务、方法、特征和状态业务服务。不得为任一服务另建仓库、协调器或执行器所有权域。

### 4.4 完整性

`运行期业务装配::完整()` 只证明：

```text
七个数据操作均有效。
依赖对象均来自同一装配且生命周期有效。
运行期业务操作组合器可用。
公开面不返回仓库、接线、执行器、会话、许可或令牌。
```

它不证明系统角色、世界根、语素入口、需求根、概念根或方法登记根已经初始化。

## 5. 运行期业务操作组合器

### 5.1 新模块

```text
文件：海中鱼巣/领域/组合.运行期业务操作.ixx
模块：海中鱼巣.领域.组合.运行期业务操作
类型：生产真模块 / ClCompile
```

组合器只持六个既有组合器引用，不直接持业务服务、数据操作、执行器、仓库或事务能力。第一轮固定八类操作：

| 编号 | 强类型请求 | 委托入口 |
| --- | --- | --- |
| 01 | `已有场景创建并接纳实际存在请求` | `存在场景组合器::在已有场景中创建并接纳实际存在` |
| 02 | `创建状态并记录动态请求` | `状态动态组合器::创建状态并记录动态` |
| 03 | `创建槽位并发布初始状态请求` | `特征状态组合器::创建槽位并发布初始状态` |
| 04 | `创建语素概念入口请求` | `语素概念入口组合器::创建语素概念入口` |
| 05 | `发布概念结构请求` | `概念结构发布组合器::发布` |
| 06 | `创建需求并承接任务请求` | `需求任务方法组合器::创建需求并承接任务` |
| 07 | `召回并提交唯一选择请求` | `需求任务方法组合器::召回并提交唯一选择` |
| 08 | `完成任务并独立结算请求` | `需求任务方法组合器::完成任务并独立结算` |

组合器不增加第九类“任意服务调用”，不提供服务指针 / 引用逃生口，也不把日志文本、枚举整数或通用字典当作业务请求。

## 6. 线程侧请求路由

### 6.1 新模块

```text
文件：海中鱼巣/线程/路由.运行期业务请求.ixx
模块：海中鱼巣.线程.路由.运行期业务请求
类型：生产真模块 / ClCompile
```

### 6.2 请求与结果

请求采用封闭 `std::variant`，每个备选项直接使用第 5.1 节既有强类型请求。结果同样采用一一对应的封闭值式变体，并额外提供顶层路由状态：

```text
已分派
请求类型拒绝
请求载荷拒绝
业务逻辑内返回
内部不一致
```

顶层状态不能覆盖下层领域状态。线程必须同时保留具体值式结果，不能把“已分派”解释为业务成功。

### 6.3 入口检测

路由入口先检查：

1. 运行期业务操作组合器引用有效。
2. 请求变体处于八个已登记备选项之一。
3. 请求类型与结果分派表唯一对应。
4. 顶层可判断的零键、无效句柄、空关系组等明显载荷错误在分派前拒绝。

领域特有的类型、生命周期、授权、幂等和权威关系检查仍由既有组合器 / 业务服务负责。路由不得复制这些规则，也不得因下层拒绝而改写请求、补默认值或切回传统服务。

### 6.4 线程边界

本模块不创建线程、不启动异步任务、不持消息队列、不重试、不调度真实动作。它只提供可由后继线程调用的同步值式路由入口。线程不是动作来源；请求中出现的方法、任务或动态材料也不自动生成动作授权或世界事实。

## 7. 运行期上下文接线

`启动.运行期上下文.ixx` 单向 `import 海中鱼巣.装配.运行期业务`，成员顺序改为：

```text

-> 接线
-> 主信息仓库
-> 节点仓库
-> 关系仓库
-> 索引仓库
```

`#270` 不修改入口生产装配，不把候选期访问入口交给旧线程，也不让任何租约在上下文不完整时有效。

## 10. 非成功返回二分

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 请求变体无效、顶层参数不完整 | 逻辑内返回 | 分派前拒绝；结构不变化 |
| 下层返回入口拒绝、幂等冲突、许可竞争或版本漂移 | 逻辑内返回 | 原样保留具体领域状态；路由不重试或改写 |
| 系统角色初始化尚未形成 | 逻辑内返回 | 上下文不可发布；宿主保持空 |
| 数据操作有效性或同仓库身份不一致 | 追根因解决 | 候选不可用；停止后继 |
| 请求类型与结果类型错配 | 追根因解决 | 停止路由；不得返回伪成功 |
| 下层内部不一致、已发布后读回异常 | 追根因解决 | 沿下层根因路径停止；不得切回旧服务 |
