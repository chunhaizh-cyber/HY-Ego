# 运行宿主与多线程消息队列详细设计

编号：THREAD-D1
日期：2026-07-09
状态：小修后确认 / 文档治理 / 可作为后续线程代码实施切片依据 / 未改代码 / 未构建 / 未迁移旧函数

## 1. 定位

本详细设计承接用户新口径：性能优先，能并行且不破坏机器事实顺序、入口拒绝和领域服务边界的路径，默认设计为可并行。

本设计定义运行宿主线程、任务管理线程、任务工作线程、缓存统计线程、事件日志线程和外设采样材料线程的第一轮边界。线程通信采用结构化消息语义，并以有界队列承载。

本设计不直接实现 C++，不启动线程，不接真实外设，不声明自我线程、自我循环、自我苏醒、完整任务状态机、真实执行桥、完整缓存系统或事件日志持久化恢复完成。

## 3. 总体原则

1. 性能优先口径成立；实时性要求高时，能并行且不破坏机器事实顺序、入口拒绝、领域服务边界和写入复核的路径，默认设计为多线程或异步队列。
2. 线程只负责调度、搬运、等待和执行耗时计算，不成为动作来源。
3. 消息和队列只承载请求、材料、回执和调度信号，不承载机器事实。
4. 权威写入必须回到领域服务或仓库正式入口，重新复核句柄、版本、任务状态、任务方法关系、动作入口和发生时间戳。
5. 工作线程不得直接裸写节点仓库、主信息仓库、关系仓库或索引仓库。
6. 缓存统计线程只维护非权威缓存材料，缓存缺失、清空或过期不得改变业务事实。
7. 事件日志线程只写审计和恢复候选材料，不反向裁决运行期事实。
8. 外设采样线程第一轮只产生模拟或空采样材料消息，外部材料必须经材料准入或领域服务复核后才可进入后续结构。

## 4. 可并行判定条件

满足以下条件时，默认按多线程或异步队列设计：

```text
输入是只读快照、请求材料、回执材料、非权威缓存材料或审计材料。
当前步骤不直接形成权威机器事实写入。
同一任务内不存在必须保持先后顺序的生命周期转换或权威写入。
没有持有仓库锁、领域服务锁或跨仓库事务边界。
消息携带可复核的句柄、版本、发生时间戳、幂等键或批次号。
权威写入最终回到领域服务或仓库正式入口重新做入口拒绝。
句柄版本过期、重复消息、队列满载或消费者停止时可以拒绝、去重、延后或登记人读诊断，不留下半结构。
队列满载策略、优先级策略和停止收口策略已经在对应切片中定义。
```

出现以下情况时必须保持串行或单任务顺序通道：

```text
同一任务的创建、承接、选择方法、执行回执、完成 / 拒绝状态变化需要顺序裁决。
同一权威结构的写入、删除、重挂、状态变化、动态记录或需求结算存在先后依赖。
当前步骤依赖刚写入结构的立即读回复核。
需要跨仓库事务式隔离，且尚未有已确认事务边界。
无法证明并发不会改变机器事实顺序、入口拒绝结果或领域服务写入边界。
```

## 5. 线程角色

| 线程 | 允许职责 | 禁止职责 |
| --- | --- | --- |
| 运行宿主线程 | 启动、停止、心跳、调度周期消息、线程生命周期收口 | 不直接写业务事实，不把 Tick 当动作来源，不裁决任务完成 |
| 任务管理线程 | 读取待执行任务材料，生成任务执行请求消息，处理任务回执消息 | 不执行方法，不直接写动作动态，不绕过任务服务写状态 |
| 任务工作线程 | 消费任务执行请求，调用已确认领域服务入口，生成结果回执 | 不持久化执行成功 bool，不绕过任务方法关系，不直接写仓库 |
| 缓存统计线程 | 消费缓存刷新消息，重建非权威缓存和统计快照 | 不裁决需求满足、任务完成、方法成功或世界事实 |
| 事件日志线程 | 消费事件日志消息，追加写入审计和恢复候选材料 | 不作为当前事实来源，不修复结构，不阻断已完成权威写入 |
| 外设采样材料线程 | 产生外部材料消息和采样状态材料 | 第一轮不接 D455、体素、真实外设控制，不直接写世界事实 |

## 6. 消息类型

| 消息 | 来源 | 目标 | 语义 | 权威性 |
| --- | --- | --- | --- | --- |
| 调度周期消息 | 运行宿主线程 | 任务管理线程 | 触发一次任务候选读取 | 非事实 |
| 停止消息 | 运行宿主线程 | 所有线程 | 请求有序停止和 join | 非事实 |
| 任务执行请求消息 | 任务管理线程 | 任务工作线程 | 请求执行已具备结构材料的任务 | 非事实，请求材料 |
| 任务拒绝回执消息 | 任务管理线程 / 任务工作线程 | 任务服务入口候选 | 表达拒绝原因和材料快照 | 非事实，需服务复核 |
| 结果回执消息 | 任务工作线程 | 任务服务 / 需求服务 | 表达执行结果候选和实际状态材料 | 非事实，需领域服务复核 |
| 缓存刷新消息 | 领域服务或任务回执路径 | 缓存统计线程 | 请求重建非权威缓存 | 非事实 |
| 事件日志消息 | 仓库正式入口或领域服务 | 事件日志线程 | 记录已确认结构事件或拒绝事件 | 审计材料 |
| 外部材料消息 | 外设采样材料线程 | 材料准入服务或任务管理线程 | 表达外部输入材料 | 材料，不是事实 |

每条消息至少包含：

```text
消息编号。
消息类型。
来源线程角色。
来源服务或来源任务。
目标任务、方法、动作入口或材料句柄。
句柄版本。
发生时间戳。
优先级。
幂等键或批次号。
只读材料快照或材料句柄。
```

## 7. 句柄版本、幂等键和同一任务顺序裁决

消费者从队列取出消息后，必须重新复核消息内句柄和版本：

```text
句柄无效、已删除或版本过期：当前消息视为过期请求或过期材料，拒绝消费或返回拒绝回执；不得写成业务失败事实。
句柄仍有效但节点类型、关系类型、任务状态或方法关系不满足入口前置条件：由领域服务按入口拒绝处理，结构不得变化。
消息携带的只读材料快照只作复核材料，不作为当前权威事实来源。
```

幂等键冲突按以下规则裁决：

```text
同一幂等键、同一消息类型、同一来源、同一目标和同一材料摘要：允许去重，返回已有局部回执或忽略重复调度。
同一幂等键但材料摘要、目标、任务、方法或来源不同：视为幂等键冲突，拒绝本条消息并记录人读诊断；不得合并材料，不得写业务事实。
缺少幂等键或批次号的权威写入候选：不得进入异步写入复核路径，只能作为待补材料拒绝。
```

同一任务顺序必须独立裁决：

```text
同一任务的生命周期改变消息必须携带任务序号或进入同一任务的串行通道。
后序消息不得越过前序创建、承接、方法选择、执行回执、完成 / 拒绝等生命周期消息。
只读查询、非权威缓存刷新和人读审计材料可以并行，但不得改变任务权威状态或影响后续写入复核结果。
无法证明同一任务消息可重排时，默认不可重排。
```

## 8. 有界队列

第一轮建议实现通用有界消息队列，后续按性能数据替换局部队列实现。

| 队列 | 生产者 | 消费者 | 第一轮实现口径 | 满载策略 | 优先级策略 |
| --- | --- | --- | --- | --- | --- |
| 任务调度队列 | 任务管理线程 | 任务工作线程 | MPMC 有界阻塞队列，容量固定，满时返回局部失败 | 拒绝新调度请求或返回调度局部失败，不写任务失败事实 | 停止消息最高；已承接任务优先于候选扫描；同一任务按序号串行 |
| 任务回执队列 | 任务工作线程 | 任务管理线程或领域回执入口 | MPSC 有界队列，回执必须重新复核 | 不得丢弃已生成回执；满载时生产者返回局部阻塞 / 拒绝材料并登记人读诊断，依赖路径停止等待后续专项处理 | 停止消息最高；拒绝回执和结构变更回执高于普通观察回执；同一任务回执不可重排 |
| 缓存统计队列 | 领域服务和回执路径 | 缓存统计线程 | 可合并同类刷新，缓存刷新可丢弃过期消息 | 按缓存命名空间和幂等键合并；丢弃过期或重复刷新；不得影响权威结构 | 停止消息最高；结构统计重建高于命中统计；可重建低价值刷新最低 |
| 事件日志队列 | 仓库和领域服务 | 事件日志线程 | 不得阻断已确认业务结构；失败写人读诊断 | 满载时记录人读诊断并返回日志局部失败；不得回滚或改写已确认权威结构 | 停止消息最高；逻辑错误和已确认结构事件高于普通审计材料；重复低价值审计材料可按后续切片明确规则合并 |
| 外部材料队列 | 外设采样材料线程 | 材料准入服务或任务管理线程 | 第一轮仅模拟或空采样材料，真实外设另建专项 | 丢弃最旧未处理模拟 / 空材料或拒绝生产者；不得生成世界事实 | 停止消息最高；用户明确请求材料高于周期性空采样；过期模拟材料最低 |

队列满时不得自动写业务失败事实；只能返回局部失败、丢弃可重建的非权威缓存请求，或记录人读诊断。

停止消息必须能唤醒所有等待消费者；任何队列不得因满载吞掉停止消息。

## 9. 异步事件日志持久化声明边界

事件日志线程即使采用异步追加，也只能形成审计材料和恢复候选材料：

```text
事件日志入队成功不等于落盘成功。
事件日志落盘成功不等于当前运行期机器事实成立。
事件日志落盘失败不得改变、回滚、修复或裁决已经由领域服务 / 仓库正式入口确认的结构。
事件日志只能作为后续恢复专项的候选输入；恢复入口、回放规则、快照隔离和恢复拒绝矩阵必须另建并确认代码实施切片。
事件日志队列、文件 flush、人读诊断或运行日志不得作为业务状态来源。
```

## 10. 写入复核

任务工作线程的执行结果不得直接写为任务完成。必须按以下路径复核：

```text
结果回执消息
-> 任务服务复核任务有效、目标状态、任务方法关系和发生时间戳
-> 方法服务复核方法首、动作入口和规格材料
-> 状态服务 / 动态服务记录实际结果状态或实例动态
-> 因果服务只接受来源动态准入
-> 需求服务或需求结算记录入口复核目标状态和实际状态
```

复核失败时，拒绝后不得留下半任务、半状态、半方法关系、半动态或半因果引用。

## 11. 停止收口顺序

运行宿主停止时必须按以下顺序收口：

```text
1. 停止接收新的外部请求、调度周期消息和外部材料生产请求。
2. 向所有队列发送最高优先级停止消息，并唤醒所有等待消费者。
3. 先收口任务调度队列和任务回执队列：已取出的权威写入候选必须完成领域服务复核或登记为未完成候选材料；不得伪造成任务失败事实。
4. 再收口缓存统计队列：可重建的缓存刷新允许丢弃或合并为停止诊断，不影响权威结构。
5. 再收口外部材料队列：模拟或空采样材料允许丢弃；不得写世界事实。
6. 再收口事件日志队列：尽力写出已复制的审计材料；失败只登记人读诊断，不回滚业务结构。
7. 释放全部仓库锁、领域服务锁和队列内部锁之后，再 join 工作线程、缓存线程、事件日志线程和外部材料线程。
8. 最终停止报告只做人读说明，不写业务事实，不作为任务状态或需求状态来源。
```

## 12. 防死锁规则

禁止在持有仓库锁、领域服务锁、队列内部锁或跨仓库写入边界时执行以下操作：

```text
阻塞等待队列 push / pop。
等待条件变量、future、promise 或任意可能阻塞的异步结果。
join 线程。
等待外设、驱动、采样、文件系统慢 I/O 或用户回调。
日志落盘、事件日志 flush、文件 flush 或控制台 / 弹窗等可能反向阻塞的诊断输出。
调用可能反向进入仓库公开入口、领域服务入口或队列入口的回调函数。
```

允许顺序是：

```text
先复制必要材料。
释放仓库锁或领域服务锁。
再入队、等待、join、落盘或执行耗时计算。
权威写入重新进入领域服务 / 仓库正式入口时，只在入口内部使用短锁，并重新做入口拒绝。
```

## 13. 外设采样材料线程边界

本设计保留外设采样线程位置，但第一轮只允许模拟或空采样材料：

```text
采样线程启动。
产生外部材料消息。
材料带来源、时间戳、批次号和只读载荷。
材料进入准入服务或任务管理候选。
材料不得直接写存在、场景、状态、动态、需求、任务、方法或因果引用。
```

第一轮外设采样材料线程禁止：

```text
接 D455、双目相机、真实驱动、真实设备等待或真实采样 API。
验证真实外设样本。
触发外设动作或外设控制。
把空采样、模拟材料、采样状态、人读日志或队列回执写成世界事实。
```

以下能力不在第一轮：

```text
D455。
双目相机。
体素。
真实外设控制。
外设动作。
外设样本验收。
```

## 15. 验收矩阵

| 编号 | 验收项 | 通过标准 |
| --- | --- | --- |
| THREAD-A1 | 消息不承载事实 | 消息清空或重放不得改变权威结构 |
| THREAD-A2 | 队列有界 | 队列满时返回局部失败或丢弃可重建非权威请求 |
| THREAD-A3 | 线程不是动作来源 | 动态来源仍来自方法执行或领域服务入口 |
| THREAD-A4 | 工作线程不裸写仓库 | 依赖扫描不出现工作线程直接写核心仓库 |
| THREAD-A5 | 权威写入复核 | 回执进入任务 / 需求 / 状态 / 动态服务时重新做入口拒绝 |
| THREAD-A6 | 缓存非权威 | 缓存线程清空后需求、任务、方法事实不变 |
| THREAD-A7 | 事件日志不裁决 | 事件日志缺失或失败不改变运行期事实 |
| THREAD-A8 | 外设材料不入事实 | 外设材料线程不接 D455 / 体素 / 外设控制，不写世界事实 |
| THREAD-A9 | 停止收口 | 停止消息后线程可 join，无悬挂工作项被读成业务事实 |
| THREAD-A10 | 防死锁 | 持锁顺序和外部等待路径符合多线程防锁规范 |
| THREAD-A11 | 并行判定 | 每条异步路径均能说明可并行条件或单任务串行原因 |
| THREAD-A12 | 幂等和过期拒绝 | 句柄版本过期、幂等键冲突和同一任务乱序均被拒绝或串行化，且不写业务事实 |
| THREAD-A13 | 队列优先级 | 五类队列均定义满载策略、优先级策略和停止消息最高优先级 |
| THREAD-A14 | 异步日志边界 | 事件日志入队、落盘、flush 失败均不裁决当前机器事实 |

## 16. 完成声明边界

本详细设计小修确认后只能声明：


```text
运行宿主线程已实现。

```
## 中途非成功返回二分口径

本文件按 2026-07-09 硬规则修订：中途非成功返回只分为 `追根因解决` 和 `逻辑内返回`。

- `追根因解决`：前置条件已经满足，并进入创建、绑定、写关系、写状态、记录动态、结算、读回或结构承载后，结果不符合内部预期；必须停止依赖路径，定位根因，当前未证明完整回滚时登记事务隔离缺口或半结构隔离缺口。
- `逻辑内返回`：领域协议允许的拒绝、候选为空、请求材料返回或人读材料返回；必须保证结构不变化，且返回材料、日志、回执、显示或控制台输出不裁决机器事实。
