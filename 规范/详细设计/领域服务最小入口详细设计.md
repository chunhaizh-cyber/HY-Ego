# 领域服务最小入口详细设计

编号：020-D1
日期：2026-07-05
状态：已确认详细设计 / HR-001 第一轮领域服务入口

## 1. 定位

本详细设计对应：

```text
规范/000_项目规则总纲.md
规范/领域服务授权写入规范.md
规范/验收/最小闭环验收用例.md
```

领域服务负责把业务请求转为允许写入的结构变化。仓库只负责最小结构，领域服务负责业务准入、节点类型检查、生命周期检查和必要前置结构检查。

本文件只描述第一轮最小入口，不证明完整领域系统完成，不进入旧项目能力迁移。

## 2. 总体规则

第一轮领域服务入口必须满足：

```text
函数入口先排除无效句柄、错误节点类型、已删除节点和缺少前置结构。
拒绝路径只返回无效句柄、false、std::nullopt 或领域枚举结果。
拒绝路径不得写业务事实。
日志、弹窗和控制台输出只做人读诊断。
业务事实必须落到节点、关系、主信息值容器、状态节点、动态节点或因果引用节点。
```

目标领域服务分层：

```text
特征值服务
特征服务
存在服务
场景服务
状态服务
动态服务
二次特征服务
因果服务
世界服务
需求服务
任务服务
方法服务
语素服务
```

其中 `世界服务` 是基础信息聚合服务；特征值、特征、存在、场景、状态、动态、二次特征、因果是基础信息原子服务；需求、任务、方法是高级信息服务；语素服务只负责文本入口绑定。

当前第一轮代码已包含以下服务入口：

```text
世界服务
特征服务
特征值服务
存在服务
场景服务
状态服务
动态服务
二次特征服务
需求服务
任务服务
方法服务
因果服务
语素服务
```

当前第一轮拆分不代表目标服务分层已经完整完成；抽象状态 / 抽象动态治理、完整依赖注入治理和旧项目能力迁移仍未完成。

## 2.1 基础信息服务目标边界

基础信息服务边界以 `规范/基础信息服务分层规范.md` 为准。

| 服务 | 目标职责 | 当前实现状态 |
| --- | --- | --- |
| 特征值服务 | 只服务于特征系统的值承载，不作为通用值容器 | 第一轮已有最小 I64 写入 / 读取，权限边界待收紧 |
| 特征服务 | 管理特征入口 | 第一轮已有 `创建特征类型` |
| 存在服务 | 管理实际存在和虚拟存在 | 第一轮已拆出，世界服务创建存在时调用存在服务 |
| 场景服务 | 管理场景和场景内临时实例状态 / 实例动态 | 第一轮已拆出，世界服务创建场景和接纳存在时调用场景服务 |
| 状态服务 | 区分实例状态和抽象状态 | 第一轮已拆出，任务状态、需求结算状态和方法授权状态经状态服务创建 |
| 动态服务 | 区分实例动态和抽象动态 | 第一轮已拆出，动态记录经动态服务创建，因果服务不再直接创建动态 |
| 二次特征服务 | 管理关系性、程度性或复合判断前置特征 | 第一轮已拆出，特征服务创建二次特征时调用二次特征服务 |
| 因果服务 | 管理因果引用、证据和抽象候选 | 第一轮已有最小因果引用入口 |
| 世界服务 | 编排基础信息原子服务 | 第一轮已改为存在 / 场景聚合编排；基础信息通用节点仍保留兼容壳 |

高级信息服务必须通过基础信息服务取得基础能力：

```text
需求服务通过存在服务、场景服务、特征服务、特征值服务和状态服务工作。
任务服务通过存在服务、场景服务、特征服务、状态服务和动态服务工作。
方法服务通过存在服务、场景服务、特征服务、特征值服务、动态服务和因果服务工作。
```

语素服务只绑定已存在信息节点，不创建基础信息、高级信息或临时实例状态 / 动态。
