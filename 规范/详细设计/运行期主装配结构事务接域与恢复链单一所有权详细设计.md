# 运行期主装配结构事务接域与恢复链单一所有权详细设计

日期：2026-07-12

状态：正式依赖门控设计 / 绑定 #247-#249 / 等待 #246 实际接口复核

## 1. 定位与依据

本设计承接 #217 隔离事务合同和 #246 主装配 S0，建立唯一生产运行期上下文、四仓库接域、领域服务接线、失败清理和恢复链同所有者验收。流程图为 `流程图/20260712_运行期主装配结构事务接域与恢复链单一所有权流程图_v0.1.md` 及同名 HTML。

## 2. 当前事实与假定合同

当前事实：#217 仍在执行；当前入口主装配未接域；#246 未执行；恢复链代码未实现。假定合同：#217 提供正式值式接线、值式令牌和移动许可；#246 输出主装配对象寿命、构造点、删除 / 失败清理矩阵及恢复链复用 / 修订 / 替代裁决。

## 3. 唯一运行期上下文

上下文用途为持有唯一协调状态、四仓库、领域服务装配和发布身份。写入方仅为启动装配模块；线程、SQL、控制面板、快照和恢复只持只读租约。生命周期从启动候选构造开始，完整初始化后一次发布，停止时先停消费者再析构服务、仓库和协调状态。

## 4. #247 主装配接域

新建生产真模块 `启动.运行期上下文` 与自检真模块 `自检.运行期上下文`。同一协调器只生成一份正式接线，四仓库必须共享域、纪元和运行期状态。任何仓库未接域、重复接域、不同状态混装或上下文未完成即发布均写前拒绝；构造后身份矛盾归追根因。

## 5. #248 服务接线与失败清理

新建生产真模块 `装配.领域服务` 与自检真模块 `自检.领域接线`。服务只保存值式接线，调用期由顶层生产路由取得许可并下传同一令牌。#246 的删除 / 失败点按六类裁决：测试隔离、写前无结构、创建期补偿、已发布删除、候选丢弃、内部不一致清理。已发布结构删除必须走正式所有者 / 事务入口；内部不一致不能静默补偿。

## 6. #249 端到端验收

新建 `自检.主装配事务` 真模块，证明全新启动、正常读写、拒绝不变式、失败清理、快照冻结许可来源和恢复发布许可均引用同一种正式上下文身份。该切片只验收和窄幅修正 #247/#248 模块，不实现 HYSNAP01 或恢复算法。

## 8. 非成功二分

外部模式无效、接线材料缺失、候选未发布属于逻辑内返回；已经完成接域、创建、删除、清理或发布后身份、读回、计数不一致属于追根因解决。

## 9. 依赖与漂移

`#217 -> #246 -> #247 -> #248 -> #249`。#246 实际文件、上下文所有权或恢复裁决不一致时不得改代码，退回设计修订。#249 通过后才允许按 JY-277 解锁或正式修订恢复链。

## 10. 完成边界

只可声明单进程生产主装配已纳入统一结构事务合同。不得声明 ACID、跨进程、断电恢复、数据库恢复、自我苏醒或旧能力迁移完成。
