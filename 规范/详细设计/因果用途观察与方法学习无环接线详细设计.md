## 6. #282 用途事件与结构化结果效用

### 6.1 模块

新增五个真模块，预期阶段 850：

| 模块 | 责任 | 禁止能力 |
| --- | --- | --- |
| `材料.用途事件` | 请求、状态、拒绝原因、任务 / 需求效用分量、观察阶段、完整事件 DTO 的唯一声明所有者 | 仓库、许可、会话、服务引用、线程对象 |
| `算法.用途事件` | 完整回合匹配、正式结果映射、结构化效用形成和不变量复核 | 仓库读取、时间源、统计、持久状态 |
| `服务.用途事件` | 纯值入口编排和返回分类 | 数据操作、用途节点写入、方法学习、运行期接线 |
| `组合.用途事件` | 唯一直接消费 #225 组合器 DTO，抽取规范化结果来源并适配到纯值服务 | 仓库、事务、持久化、把任意候选事后配对 |
| `自检.用途事件` | Debug 隔离 A01-A16 与阶段 850 | 生产依赖、Release 登记 |

生产调用方向：

```text
用途事件组合器::形成用途事件候选
-> 把 #225 正式 DTO 适配为材料模块拥有的规范化结果来源
-> 用途事件业务服务::形成结构化结果效用
-> 用途事件算法
-> 材料.用途事件 值式结果
```

`材料.用途事件` 不得 import `组合.任务结果结算`，防止材料层反向依赖组合层。只有 `组合.用途事件` 可以 import #225 组合模块；它不得重复声明材料 DTO。

### 6.2 请求

材料层第一版请求固定为：

```text
用途来源冻结材料 冻结来源。
规范化任务结果来源材料 结果来源。
uint32 用途规则版本 = 1。
```

执行前用途来源冻结至少包含：完整因果概率候选、完整任务执行冻结请求、非零冻结键、用途来源规则版本、冻结任务序号和冻结时间戳。规范化结果来源至少包含完整回执、三种结果分类、任务 / 需求 / 选择 / 方法 / 动作 / 场景 / 主体 / 目标 / 实际状态 / 动态、目标与实际状态值、提交状态、可选结算状态、待结算提示和幂等 / 时间材料。

`组合.用途事件` 从 #225 请求 / 结果抽取规范化来源；不得由调用方用平行句柄覆盖。冻结来源中的任务执行冻结请求必须与结果回执携带的冻结请求完整相同。没有执行前冻结来源时逻辑返回；不能在结果产生后临时拿任意候选补配。

### 6.3 执行前冻结与结果匹配

算法必须：

1. 复核冻结键、规则版本、冻结时间戳、完整因果候选和完整任务执行冻结请求。
2. 复核新版结果回执协议和其完整冻结请求。
3. 复核任务结果属于三种正式收口且其目标裁决、提交状态、待结算提示与状态相符。
4. 用 `冻结任务执行请求相同` 确认执行前冻结与结果回执属于同一完整任务回合，并逐字段确认任务、任务序号、选择记录、方法和动作一致。
5. 冻结因果候选的历史机会证据不要求包含当前任务回合；它是执行前决策材料，不是当前结果。若候选碰巧包含当前回合，也不得据此倒推它参与了选择。
6. Q10000 和历史机会分类只复制为概率分量，不参与任务达成或需求结算裁决。

### 6.4 结构化结果效用 DTO

```text
用途事件规则版本。
执行前用途来源冻结、完整新版结果回执和同一冻结请求。
来源需求、任务、选择记录、方法、动作入口。
目标状态、实际状态、任务结果动态。
因果原因 / 结果完整结构键、因果规则版本、完整历史机会证据组。
机会数、正结果数、负结果数、Q10000。
正式任务结果状态。
任务达成分量：未达成 / 已达成。
需求结算分量：未形成 / 待结算 / 已满足。
观察阶段：尝试未达成 / 待结算 / 最终已结算。
```

不设置单一效用分数、成功 bool、权重、阈值、置信度或自动学习建议。

### 6.5 三种结果映射

| 结果 | 必须同时成立 | 输出 |
| --- | --- | --- |
| 未达成已转待重筹办 | 目标裁决未达成；任务迁移提交成功；无需求结算成功 | 尝试未达成；需求最终效用未形成 |
| 任务已完成待需求结算 | 目标裁决已达成；任务完成提交成功；待结算提示完整；需求结算未成功 | 待结算；不得作为最终效用 |
| 已完成并结算 | 目标裁决已达成；任务完成与需求结算均成功；结算材料已满足 | 最终已结算；需求已满足 |

状态与字段组合不一致属于正式 DTO 内部矛盾，必须返回内部不一致并追根因。

| 任务结果尚未收口或不是三种正式状态 | 逻辑内返回 | 不猜测结果 |

## 7. #283 权威用途观察结构承载与恢复边界 S0

#283 无代码阶段，不创建模块、节点类型、主信息格式、索引命名域、关系角色、数据操作层或恢复入口。它在 #282 与 #286 完成后只读扫描实际接口，形成以下事实矩阵：

```text
字段承载准入：#282 全部字段的来源、权威性、生命周期、现有合法承载和可重建性。
主信息准入：I64 可选槽容器、逐槽写入 / 读回、稀疏扩容、不可变所有权和红线冲突。
身份与索引：冻结键、完整任务回合、全局 uint64 索引、命名域 / 碰撞 / 完整比较和跨恢复身份。
关系与阶段：候选端点、顺序号、基数、有效 / 失效读回和当前阶段唯一性是否有现成 API。
事务发布：写前唯一、候选、精确撤销、确认、最后发布和并发单赢家是否可由现有会话证明。
容量与重建：嵌套向量规模、领域上限、未知版本 / 标签 / 缺槽处理和完整往返解码能力。
恢复边界：四仓库快照、未知节点类型拒绝、待结算定位和 #226-#230、#237-#239 唯一恢复链。
```
