## 5. 文件格式

### 5.1 段头

字段按以下顺序显式小端编码：

| 字段 | 类型 | 规则 |
| --- | --- | --- |
| 魔数 | 8 字节 | ASCII `HYEVSEG1` |
| 格式版本 | U32 | 固定 1 |
| 字节序标记 | U32 | 固定 `0x01020304` |
| 段编号 | U64 | 非零，必须等于配置 |
| 记录数 | U64 | 与实际帧数一致 |
| 首序号 | U64 | 空段为 0；非空段固定为第一帧序号 |
| 末序号 | U64 | 空段为 0；非空段固定为最后一帧序号 |
| 段字节数 | U64 | 与文件实际长度一致 |
| 段头校验 | U64 | 对前述段头字段计算 FNV-1a 64 |

任何保留字段、平台指针、线程编号、SQL 主键、控制面板字段或外设字段均不得进入版本 1 段头。

### 5.2 事件帧

每帧顺序：

| 字段 | 类型 | 规则 |
| --- | --- | --- |
| 帧总长度 | U32 | 含帧头、负载和帧校验；不得超过 8192 |
| 负载版本 | U32 | 固定 1 |
| 事件序号 | U64 | 从 1 开始严格递增，不允许间断或重复 |
| 结构事件时间戳 | U64 | 非零，来自调用方 |
| 事件类别 | U32 | 必须通过现有类别准入 |
| 写入结果 | U32 | 必须通过现有结果准入 |
| 仓库编号 | U64 | 原样值材料 |
| 节点编号 | U64 | 原样值材料 |
| 节点版本 | U64 | 原样值材料 |
| 主信息编号 | U64 | 原样值材料 |
| 主信息版本 | U64 | 原样值材料 |
| 关系编号 | U64 | 原样值材料 |
| 关系版本 | U64 | 原样值材料 |
| 索引键 | U64 | 原样值材料 |
| 入口名称字节数 | U32 | 0 拒绝，上限 512 |
| 摘要字节数 | U32 | 可为 0，上限 4096 |
| 入口名称 | UTF-8 字节 | 不含结尾 0 |
| 摘要 | UTF-8 字节 | 不含结尾 0 |
| 帧校验 | U64 | 对本帧除校验字段外全部字节计算 FNV-1a 64 |

`请求裁决运行期事实` 和 `请求恢复运行期结构` 不写入文件，因为入口只接受二者都为 false 的材料。读取后这两个字段固定还原为 false。

### 5.3 编码边界

```text
所有整数显式写入 1/4/8 字节小端序列。
所有长度先做上限与剩余字节检查，再分配内存。
所有 UTF-8 转换失败均拒绝，不用替换字符静默修正。
文件末尾必须恰好落在最后一帧末端；多余字节视为损坏。
读取失败不返回已解析前缀。
```

## 6. 追加流程

### 6.1 写前拒绝

在获取服务互斥锁后、任何文件写入前拒绝：

```text
配置段编号为 0。
最大段字节数小于空段头或大于第一轮 4 MiB 固定上限。
目标或临时路径不在日志/事件段目录。
事件材料未通过事件材料是否可写入。
结构事件时间戳为 0。
调用方预填结构事件顺序号。
入口名称或摘要 UTF-8 转换失败 / 超限。
既有段未知版本、损坏或序号不连续。
同名临时段已存在。
追加后超过单帧或单段上限。
```

这些分支是设计允许的材料拒绝，属于逻辑内返回。目标段字节必须保持不变。

### 6.2 序号分配

空段首帧分配序号 1；非空段分配 `末序号 + 1`。末序号为 `UINT64_MAX` 时返回段已满，不回绕。

调用方不得以节点编号、任务顺序号、消息编号或时间戳代替事件序号。事件序号只在同一段内有顺序意义，不跨段裁决业务顺序。

### 6.3 候选发布

```text
读取并严格验证目标段或构造空段。
复制原字节并追加新帧。
更新段头记录数、首末序号、段字节数和段头校验。
写同目录唯一临时文件。
flush、close 后重新打开临时文件。
完整解析，核对新增事件和所有段头字段。
使用 Windows 同卷替换 + 写穿语义发布目标段。
重新打开目标段，完整读回。
只有最终读回一致才返回成功。
```

第一轮采用完整段复制换取发布正确性；性能、增量提交标记和自动轮换后置，不得在实现时私自改为裸追加。

## 7. 只读流程

`读取事件段`：

```text
不存在目标文件 -> 返回空段候选，不创建文件。
存在目标文件 -> 读取完整字节并严格解析。
任一校验失败 -> 返回空事件组和明确拒绝原因。
全部通过 -> 返回事件材料值副本组。
```

只读入口不得：

```text
修补文件。
删除残留临时段。
跳过损坏帧。
返回有效前缀。
写入节点、主信息、关系、索引。
调用领域服务。
重放入口名称对应的函数。
```

## 8. 并发与生命周期

一个 `事件日志段服务` 实例持有一个私有互斥锁，追加与读取同锁串行。锁内不得调用领域服务、日志落盘、弹窗、外设、SQL 或用户回调。

第一轮不提供跨进程锁和同一段多实例写入。调用方必须确保每个段文件只有一个服务实例；测试必须显式验证两个线程通过同一实例追加时序号连续。

服务析构不自动修改段，不自动修复临时文件。残留临时段由后续治理工具或人工处理；本切片只报告冲突。

## 9. 非成功返回二分

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 配置无效、路径越界、材料无效、时间戳为 0、调用方预填序号 | 逻辑内返回 | 不创建或修改文件 |
| 既有外部 / 跨重启段未知版本、损坏、序号断裂 | 逻辑内返回 | 不追加、不修补、不返回部分事件 |
| 字段、帧或段容量超限 | 逻辑内返回 | 返回明确上限原因，不轮换 |
| 残留临时段存在 | 逻辑内返回 | 不覆盖，不触碰目标段 |
| 前置通过后临时文件写入 / flush / close 不及预期 | 追根因解决 | 停止依赖路径，不返回成功 |
| 临时段写成后严格读回不一致 | 追根因解决 | 目标旧段不替换，保留诊断候选 |
| 同卷替换返回异常 | 追根因解决 | 不声明目标已发布，核对目标旧段 |
| 替换成功后最终目标读回不一致 | 追根因解决 | 停止依赖路径，写逻辑错误诊断 |
| 成功结果中的序号、记录数或字节数与读回不同 | 追根因解决 | 不向后续恢复设计提供该结果 |

诊断日志和弹窗仍只做人读提醒，不可用于修复事件段或裁决事件有效性。
