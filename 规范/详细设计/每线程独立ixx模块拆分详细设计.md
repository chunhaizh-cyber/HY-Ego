# 每线程独立 ixx 模块拆分详细设计

版本：v0.1
日期：2026-07-10
状态：已确认 / 已实现 / 已验证

## 1. 设计目标

把当前集中在 `海中鱼巣/核心/运行宿主.h` 中的 6 个实际线程角色拆为 6 个独立 C++20 模块接口文件，并把共享消息协议和有界队列拆为不拥有线程的共享模块。

本设计固定以下结构要求：

```text
每个线程角色一个文件。
线程文件使用 .ixx。
同一个 .ixx 不得混放多个线程角色。
共享消息、队列和停止协议单独成模块，不得伪装成线程文件。
```

## 2. 当前代码事实

### 2.1 当前线程分布

`海中鱼巣/核心/运行宿主.h` 当前定义 4 个类，但实际持有 6 个 `std::thread`：

| 当前类 | 当前线程成员 | 问题 |
| --- | --- | --- |
| `运行宿主` | `宿主线程` | 与其余线程类、状态和结果结构混在同一头文件 |
| `任务线程壳` | `任务管理线程`、`任务工作线程` | 两个线程角色混在一个类和一个文件 |
| `缓存事件线程壳` | `缓存统计线程`、`事件日志线程` | 两个线程角色混在一个类和一个文件 |
| `外设采样材料线程壳` | `采样材料线程` | 与其他线程角色混在同一头文件 |

### 2.2 当前共享结构

```text
海中鱼巣/核心/运行消息队列.h
```

当前同时承载运行消息类型、线程角色、消息结构、队列结果和有界队列实现。它不拥有 `std::thread`，适合拆为消息协议模块和有界队列模块。

### 2.3 当前工程能力

```text
PlatformToolset = v145
LanguageStandard = stdcpp20
AdditionalOptions 包含 /utf-8
实施前 `海中鱼巣/线程/` 没有 .ixx，项目没有 C++ 模块接口编译项
项目另有 `海中鱼巣/领域/初始化.世界树.ixx`，但它按 `ClInclude` 和 `#include` 使用，不是本线程模块接口
```

因此实施时必须先证明最小线程 `.ixx` 模块可由当前工程编译和导入，不能把既有头文件式 `.ixx` 当作 C++20 模块能力证据。

## 3. 固定文件边界

### 3.1 线程模块

| 文件 | 唯一线程角色 | 允许持有的线程成员 | 禁止内容 |
| --- | --- | --- | --- |
| `海中鱼巣/线程/运行宿主线程.ixx` | 运行宿主线程 | 1 个运行宿主 `std::thread` | 任务、缓存、日志、外设线程 |
| `海中鱼巣/线程/任务管理线程.ixx` | 任务管理线程 | 1 个任务管理 `std::thread` | 任务工作线程和领域落账 |
| `海中鱼巣/线程/任务工作线程.ixx` | 任务工作线程 | 1 个任务工作 `std::thread` | 任务管理线程和任务完成裁决 |
| `海中鱼巣/线程/缓存统计线程.ixx` | 缓存统计线程 | 1 个缓存统计 `std::thread` | 事件日志线程和权威事实裁决 |
| `海中鱼巣/线程/事件日志线程.ixx` | 事件日志线程 | 1 个事件日志 `std::thread` | 缓存统计线程和恢复裁决 |
| `海中鱼巣/线程/外设采样材料线程.ixx` | 外设采样材料线程 | 1 个外设采样材料 `std::thread` | D455、体素、真实驱动和外设动作 |

### 3.2 共享模块

| 文件 | 用途 | 写入方 | 读取方 | 生命周期 | 验证方式 |
| --- | --- | --- | --- | --- | --- |
| `海中鱼巣/线程/运行消息协议.ixx` | 导出运行消息类型、线程角色、优先级、消息和轻量结果结构 | 线程模块通过构造消息写入调用方持有的材料 | 线程模块和入口验收 | 进程内类型定义 | 模块导入编译、消息字段自检、禁止机器事实扫描 |
| `海中鱼巣/线程/有界运行消息队列.ixx` | 导出有界队列和入队 / 出队 / 停止结果 | 各线程模块经正式队列入口写入 | 各线程模块经只读或出队入口读取 | 线程对象或编排对象生命周期内 | 满载、停止优先、幂等、版本、乱序和并发验收 |

共享模块不得定义线程类、不得持有 `std::thread`、不得调用领域服务写业务事实。

## 4. 模块命名与导入规则

模块名使用中文业务命名，第一轮建议：

```cpp
海中鱼巣.线程.运行消息协议
海中鱼巣.线程.有界运行消息队列
海中鱼巣.线程.运行宿主线程
海中鱼巣.线程.任务管理线程
海中鱼巣.线程.任务工作线程
海中鱼巣.线程.缓存统计线程
海中鱼巣.线程.事件日志线程
海中鱼巣.线程.外设采样材料线程
```

实施第一步必须验证 v145 对中文模块名、UTF-8 文件名和入口 `import` 的实际支持。若工具链失败，必须追根因并记录准确编译错误；不得静默改用英文业务模块名，也不得退回混合头文件。

## 5. 模块依赖方向

```text
运行消息协议
  -> 有界运行消息队列
  -> 六个线程模块
  -> 入口.cpp
```

附加规则：

```text
线程模块之间不得循环 import。
任务工作线程可以接收任务管理线程公开的队列引用或只读协作材料，但不得包含任务管理线程类定义。
入口或后续无线程编排模块负责组装多个线程对象；编排对象不得自行持有第七个业务线程。
线程模块不得 import 控制面板、SQL 数据库适配、D455、体素或外设驱动模块。
```

## 6. 生命周期与消费循环

每个线程模块必须独立拥有：

```text
线程状态
停止请求
线程已进入标记
唯一 std::thread 成员
启动入口
停止请求入口
join 收口入口
本线程唯一消费循环或等待循环
```

每个线程模块不得拥有：

```text
第二个 std::thread
其他线程角色的启动 / 停止状态
绕过领域服务的仓库写入口
以日志、控制台或显示结果裁决成功的逻辑
通过析构静默吞掉内部结构错误的修复逻辑
```

析构函数可以作为最后的 join 防泄漏边界，但正常运行必须显式执行停止请求和收口等待，并由验收证明线程均不可 join。

## 7. 任务与缓存事件拆分规则

### 7.1 任务管理与任务工作

`任务线程壳` 不再作为同时拥有两个线程的容器。拆分后：

```text
任务管理线程：接收和排队任务执行请求材料，不执行方法，不裁决任务完成。
任务工作线程：消费已准入请求并生成回执材料，不裸写仓库，不裁决任务 / 需求 / 方法结果。
```

任务调度队列和任务回执队列的所有权必须在代码实施时明确为独立编排所有权或单一线程所有权，禁止两个模块都宣称拥有同一队列生命周期。

### 7.2 缓存统计与事件日志

`缓存事件线程壳` 不再作为同时拥有两个线程的容器。拆分后：

```text
缓存统计线程：只消费缓存刷新材料，缓存不裁决业务事实。
事件日志线程：只消费事件日志材料，日志失败不改变运行期结构，日志不作为恢复事实。
```

缓存刷新队列和事件日志队列必须分别归属对应线程模块，不共享状态枚举或停止标记。

## 8. 旧文件收口

代码实施完成后必须满足：

```text
海中鱼巣/核心/运行宿主.h 不再定义任何线程类或 std::thread。
海中鱼巣/核心/运行消息队列.h 不再作为运行消息与队列的权威定义来源。
入口.cpp 使用 import 取得线程和协议类型，不得同时 include 旧定义形成双源。
```

旧文件可以删除；若因过渡期必须保留兼容壳，兼容壳不得定义线程类、不得重复定义消息或队列结构，并必须在同一切片内给出删除条件。默认方案是直接删除旧线程定义文件，避免双源。

## 9. 工程登记规则

`海中鱼巣.vcxproj` 和 `海中鱼巣.vcxproj.filters` 必须显式登记所有 `.ixx` 模块接口单元。模块依赖顺序由 MSBuild 扫描和模块元数据解析，不能依靠文件列表偶然顺序。

工程配置必须继续保持：

```text
C++20
/utf-8
Debug x64
v145
```

不得通过关闭一致性检查、降低语言标准或把 `.ixx` 当普通头文件包含来绕过模块编译错误。

## 10. 失败收口

| 阶段 | 非成功情形 | 收口分类 | 处理 |
| --- | --- | --- | --- |
| 模块探针 | `.ixx` 不能编译或入口不能导入 | 追根因解决 | 修正工程项、模块声明、导入顺序或工具链配置后重试 |
| 模块导出 | 导出类型不可达、重复定义或依赖循环 | 追根因解决 | 修正模块依赖与导出边界，不保留头文件双源 |
| 线程拆分 | 同一文件仍持有两个线程角色 | 追根因解决 | 继续拆分，不以兼容为由保留混放 |
| 构建 | Debug x64 失败 | 追根因解决 | 定位首个编译 / 链接错误并修正 |
| 运行 | 生命周期、消息或验收结果与既有行为不一致 | 追根因解决 | 定位具体线程模块、队列和入口调用链 |
| 请求材料为空或队列无消息 | 设计允许且不写结构 | 逻辑内返回 | 返回明确拒绝结果，不写业务事实 |
| 队列满载或已停止 | 设计允许且不写结构 | 逻辑内返回 | 局部拒绝，不修改权威结构 |

## 11. 实施顺序

```text
S0：建立一个最小运行消息协议模块并完成入口导入编译探针。
S1：建立运行消息协议和有界运行消息队列共享模块。
S2：拆分运行宿主线程和外设采样材料线程。
S3：拆分任务管理线程和任务工作线程，明确队列所有权。
S4：拆分缓存统计线程和事件日志线程，分离状态与停止标记。
S5：更新入口、项目文件和筛选器，移除旧头文件定义双源。
S6：执行构建、默认入口、停止收口、并发和静态归属验收。
```

S0 未通过时不得进入 S1-S6。S2-S4 每完成一组都必须先构建，避免一次性积累模块依赖错误。

## 12. 验收矩阵

| 编号 | 验收项 | 通过标准 |
| --- | --- | --- |
| THREAD-MOD-A1 | 模块探针 | v145 / C++20 / UTF-8 下 `.ixx` 编译并由入口导入成功 |
| THREAD-MOD-A2 | 一线程一文件 | 6 个线程角色各自只存在于一个指定 `.ixx` 文件 |
| THREAD-MOD-A3 | 不混放 | 任一线程 `.ixx` 不定义第二个线程角色，不持有第二个 `std::thread` |
| THREAD-MOD-A4 | 共享模块纯净 | 消息协议和有界队列 `.ixx` 不定义线程类、不持有 `std::thread` |
| THREAD-MOD-A5 | 旧定义清除 | `运行宿主.h` 和 `运行消息队列.h` 不再形成线程或协议双源 |
| THREAD-MOD-A6 | 工程登记 | 8 个 `.ixx` 均由项目和筛选器登记为模块接口单元 |
| THREAD-MOD-A7 | 构建 | Debug x64 构建 0 错误 |
| THREAD-MOD-A8 | 既有行为 | THREAD-S2 至 THREAD-S6 默认入口验收继续通过 |
| THREAD-MOD-A9 | 停止收口 | 6 个线程均可显式停止并 join，无残留可 join 线程 |
| THREAD-MOD-A10 | 事实边界 | 线程、缓存、日志和外部材料均不直接裁决或裸写机器事实 |
| THREAD-MOD-A11 | 禁止能力 | 不新增 D455、体素、真实外设驱动、自我循环或自我苏醒 |
| THREAD-MOD-A12 | 静态扫描 | 不存在同一 `.ixx` 多个 `std::thread` 成员或多个线程类定义 |

## 13. 完成声明边界

本设计和后续计划完成只能声明：

```text
线程文件边界已设计。
C++20 模块工程能力已验证或记录真实阻塞。
6 个当前线程角色已拆为独立 .ixx 模块。
既有线程壳验收在模块化后继续通过。
```

不得扩大声明为：

```text
完整真实线程调度完成。
真实闭环完成。
自我循环、自我苏醒或自我治理完成。
真实外设、D455 或体素接入完成。
旧鱼巢线程能力迁移完成。
```

## 14. 实施门禁

用户已确认“一线程一文件、不得混放、使用 `.ixx`”这一结构方向。修改 C++、删除旧头文件定义和改变工程模块结构仍需将对应代码实施计划登记到可执行队列后执行。
