## 1. 定位

本设计把安全删除拆为五个不能越级的切片：

```text
CONCEPT-S8-S1：删除准入候选、完整当前输入重建和删除后直接边纯值计算。
STRUCT-TXN-S1：结构事务域、端点冻结、跨仓库写集和统一读取隔离基础。
CONCEPT-S8-S2：节点 / 主信息 / 关系 / 索引、四类结构所有者、概念登记和统计缓存的原子清理适配。
CONCEPT-S8-S3：候选锁内复核、安全删除提交和活动快照单次发布。
CONCEPT-S8-S4：删除后固定点、审计、幂等和等价概念新句柄重建验收。
```

当前代码只具备分散单锁删除、值式固定点材料和概念图内部活动快照。本文描述的是预期正式契约，不得把它写成当前实现事实。

## 4. 删除候选与完整当前输入

### 4.1 请求

建议值式请求：

```text
概念删除候选请求
  目标概念
  预期活动版本
  候选数量预算
  直接边计算预算
```

预算只控制纯值计算，不授权缩小输入集合。预算不足返回待重算材料，不得用局部分量代替完整当前输入。

### 4.2 候选完备阶段

```text
删除候选完备阶段::未定义 = 0
删除候选完备阶段::领域影响已形成 = 1
删除候选完备阶段::结构写集已补全 = 2
```

S8-S1 只能形成阶段 1。只有 S8-S2 在同一结构事务域中补齐反向主键、所有者授权和清理包身份后，才能形成阶段 2。阶段值只描述 DTO 完整度，不是持久业务状态。

### 4.3 领域影响候选

```text
概念删除影响候选
  完备阶段
  基准活动版本
  目标概念完整句柄
  根类别
  节点记录身份副本
  主信息句柄
  生命周期阶段与当前关系句柄
  当前相关有效关系记录组
  当前支持数量
  当前外部引用数量
  受影响分量
  完整剩余签名组
  删除后直接边组
  删除后根可达摘要
  待补全结构所有者组
```

`待补全结构所有者组` 第一版至少包含索引反查、对应原子服务、概念图登记和统计缓存。它是明确门禁，不能被空数组解释为已经完整。

### 4.4 完整输入重建

概念图服务新增只读值式完整输入材料，不暴露内部快照引用：

```text
同一活动版本的四根。
全部当前活动概念与规范签名。
全部当前概念上下位、实例支持和定义材料关系。
全部当前生命周期材料。
目标的节点相关有效关系。
```

删除候选计算必须：

```text
确认输入均来自同一活动版本。
从完整概念 / 签名集合删除目标。
删除所有以目标为端点的概念内部关系候选。
把固定点算法内部的共享直接边计算与传递约简提取为纯值函数。
让现有固定点创建路径和删除候选路径复用同一函数。
对完整剩余集合重算直接边、无环和四根可达。
```

不得只把目标下位概念挂到一个猜测父节点，也不得扫描写后结构再修补。

## 5. 结构事务域与统一读取隔离

### 5.1 协调器

新增生产行为真模块：

```text
文件：海中鱼巣/核心/协调.结构事务.ixx
模块：海中鱼巣.核心.协调.结构事务
工程登记：ClCompile
调用方式：import
```

模块公开结构事务协调合同：

```text
结构事务协调器
  非零事务域编号
  共享 / 独占结构许可
  当前独占事务序号
  值式写集身份
```

许可采用 RAII 值类型，不可复制。许可本身只控制并发可见性，不承载业务事实。

STRUCT-TXN-S1-A01 至 A14 不得写入该生产模块或 `入口.cpp`，必须由 `海中鱼巣/核心/自检.结构事务.ixx`（`export module 海中鱼巣.核心.自检.结构事务`）独立承载并向中央 `海中鱼巣.自检.运行器` 登记。生产模块不得依赖自检模块。

### 5.2 仓库接线

节点、主信息、关系和索引仓库可选接入一个协调器。为保持已有隔离自检，未接协调器的仓库仍可执行既有单仓库能力，但不得参与安全删除。

参与安全删除的四仓库、概念图服务、对应原子服务和概念安全删除服务必须报告同一非零事务域编号；任一不一致都在取得独占许可前逻辑内拒绝。

### 5.3 普通公开读写

已接事务域时：

```text
所有公开仓库读取先取得共享结构许可，再取得本仓库共享锁。
所有普通仓库写入先取得共享结构许可，再做跨仓库端点复核，最后取得本仓库独占锁。
关系创建 / 重挂和索引绑定必须在共享许可持有期内完成节点有效性复核与写入。
概念图活动读取 / 写入和结构所有者删除相关入口同样参与共享许可。
```

同一公开调用链只取得一次结构许可。协调器提供不可复制的同域令牌；领域服务、关系仓库或索引仓库取得令牌后，把它显式传给下层令牌感知入口，下层只验证域身份并取得自己的锁，不得再次申请共享许可。无上游令牌的普通公开入口才自行取得共享许可。事务独占持有者同样不得重新申请共享许可；事务包提交入口接收同域独占令牌，验证后直接取得本仓库锁。

### 5.4 锁序

```text
结构事务共享 / 独占许可
-> 活动图锁_
-> 图写锁_
-> 节点仓库或主信息仓库或关系仓库或索引仓库中的一个锁
```

删除提交不同时持有多个仓库锁。全局独占许可已经阻止其它参与者读写，因此各包可按顺序取得单仓库锁并发布，避免新增仓库间 ABBA。

持有任何上述锁时禁止调用日志落盘、弹窗、持久化 I/O、外设等待、用户回调或未声明锁序的上层服务。

### 5.5 第一版原子性边界

第一版证明的是单进程内、所有参与者遵守同一事务域时的统一读取隔离：

```text
独占许可取得前：允许逻辑内拒绝，不写删除结构。
独占许可内准备阶段：重读、分配和构造包，不改变权威结构。
唯一提交点后：只执行已准备的无分配发布，不提供普通失败返回。
活动快照发布并完成结构读回后：释放独占许可。
```

不证明进程崩溃、内存损坏、断电或跨重启恢复。没有提交前准备证明时，不得进入提交点。

## 6. 完整写集与结构所有者

### 6.1 结构写集

```text
结构删除写集
  事务域编号
  基准活动版本
  目标节点及预期版本
  目标主信息及预期版本
  全部当前相关关系及预期版本 / 状态
  全部反向主键
  原子服务所有者授权
  生命周期关系登记项
  专用关系登记项
  新活动快照材料
  统计缓存清理键
```

写集必须是精确集合。缺项、额外当前关系、额外主键、生命周期变化或新增支持 / 引用都会使候选过期；不得在提交时临时扩展写集。

### 6.2 核心仓库适配

预期新增：

```text
节点 / 主信息显式审计读取。
关系按精确写集准备失效包和审计读回。
索引按节点反向读取全部主键，并在绑定 / 删除时维护双向映射。
节点、主信息、关系、索引各自的事务准备包和同域独占令牌提交入口。
```

准备包必须预先持有发布所需的新记录 / 新映射副本或已保留容量。提交入口只做域令牌、预期版本和包身份的内部断言，然后以无分配交换或确定性原地状态推进发布。

### 6.3 结构所有者授权

四类普通概念按节点类型映射到：

```text
存在 -> 存在服务
动态 -> 动态服务
关系 -> 二次特征服务
因果 -> 因果服务
```

对应服务只授权本类型、非根、退役目标，并形成值式所有者清理授权；不直接编排跨仓库提交。授权材料必须绑定目标节点、主信息和事务域。

概念图服务负责准备：

```text
生命周期登记清理包。
专用关系登记清理包。
不含目标的新活动快照。
活动版本加一和固定点上一候选图清理。
```

统计服务只准备按概念清理的非权威缓存键，不得参与删除准入或提交授权。

## 7. 安全删除编排

### 7.1 服务归属

新增 `概念安全删除服务` 作为领域聚合编排器。它可以依赖：

```text
结构事务协调器。
节点、主信息、关系、索引仓库的事务适配入口。
概念图服务。
存在、动态、二次特征、因果四个原子服务。
统计服务。
```

它不得替代四个原子服务裁决节点所有权，不得让 UI、SQL、日志或调用方直接提交仓库包。

### 7.2 锁内重读

取得独占许可后必须重读：

```text
目标仍是同一当前节点句柄且非根。
活动版本仍等于候选基准版本。
生命周期仍为同一退役关系句柄与版本。
当前有效支持仍为零。
全部外部有效关系仍为零。
全部相关关系和反向主键与写集完全一致。
所有者授权、概念登记和缓存键仍匹配。
删除后完整快照仍可由当前输入重建。
```

候选过期、新增支持 / 引用或版本变化是逻辑内返回；完整当前结构与已声明写集互相矛盾是追根因解决。

### 7.3 准备阶段

在权威结构不变化的前提下构造：

```text
关系失效包。
索引双向键删除包。
主信息删除包。
节点删除包。
四类所有者授权清理包。
概念图登记清理包。
统计缓存清理包。
删除后活动快照与活动版本。
```

所有容器分配、排序、去重、容量保留、版本上限检查和无环 / 根可达验证都必须在进入提交点前完成。

### 7.4 提交顺序

统一读取隔离下，外部读者不会看到中间态。第一版固定顺序：

```text
1. 发布关系失效包。
2. 发布索引主键删除包。
3. 发布主信息删除包。
4. 发布节点删除包。
5. 发布四类所有者、生命周期、专用关系和统计缓存清理包。
6. 最后单次发布新活动快照、活动版本并清除固定点上一候选图。
7. 完成内部读回后释放独占许可。
```

进入步骤 1 后不得返回普通失败。内部不一致属于逻辑错误；提交函数必须在设计上通过预构造与无分配发布排除该路径，而不是写后扫描修复。

### 7.5 回执

值式回执可包含：

```text
目标旧节点句柄与当前删除审计句柄。
旧主信息句柄与当前删除审计句柄。
关系审计句柄组。
已删除主键组。
原 / 当前活动版本。
删除后固定点摘要。
```

回执只汇总底层证据，不单独持久化为机器事实。

## 8. 删除后固定点与审计

### 8.1 结构读回

释放独占许可后，经普通公开入口证明：

```text
旧节点和主信息普通读取为空，显式审计为已删除且版本单调。
目标全部相关关系普通读取为空，显式审计完整且端点 / 类型保持。
索引反向主键组为空，旧主键不再返回旧节点。
四类所有者登记、生命周期登记、专用关系登记和按概念缓存均不含目标。
活动概念、签名、全图和四根树均不含目标。
剩余全部概念仍恰好归属一个根，无环，直接边等于删除候选重算结果。
```

### 8.2 幂等

同一旧候选重复提交时：

```text
节点 / 主信息 / 关系审计完整且索引 / 登记已清零 -> 返回已删除幂等材料。
不推进任何版本，不重复发布活动快照。
任一审计缺失或结构残留 -> 追根因解决，不把普通读取为空当成功。
```

### 8.3 等价概念重建

使用新的真实实例和原有概念候选输入路径形成同签名候选。固定点只能在当前活动图中查找可复用概念；已删除旧节点不在活动图，必须创建新节点编号 / 版本身份。旧关系审计和旧缓存不得阻止新关系形成。

## 9. 目标接口候选

以下均为假定契约，不是当前实现事实：

```text
概念图服务::读取概念删除影响候选(请求)
概念图算法::重算完整签名直接边(完整签名组, 预算)
海中鱼巣.核心.协调.结构事务::取得共享许可()
海中鱼巣.核心.协调.结构事务::取得独占许可(结构写集)

关系仓库::准备节点关系失效包(...)
索引仓库::读取节点主键组(节点)
索引仓库::准备节点主键删除包(...)

概念图服务::准备概念登记与活动快照清理包(...)
统计服务::准备概念缓存清理包(...)

```

实际命名可以按代码风格小修，但服务归属、值式材料、事务域、锁序、失败二分和完成边界不得漂移。

## 10. 逻辑内返回与追根因解决

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 无效句柄、四根、非活动概念、非退役、仍有支持或外部引用 | 逻辑内返回 | 不生成候选，不写结构 |
| 活动版本漂移、候选预算不足或允许的当前材料不足 | 逻辑内返回 | 返回拒绝 / 待重算值式材料，不写结构 |
| 事务域未接入或参与者域编号不同 | 逻辑内返回 | 不取得独占许可，不写删除结构 |
| 独占许可内发现候选过期、新增支持 / 引用或生命周期变化 | 逻辑内返回 | 释放许可，本次不写删除结构 |
| 旧候选已删除且全部审计与清零证据完整 | 幂等逻辑内返回 | 不推进版本或活动快照 |
| 完整当前输入产生环、根不可达、签名矛盾或固定点不一致 | 追根因解决 | 停止依赖路径，不用局部重挂修补 |
| 完整写集缺关系 / 主键 / 所有者 / 登记，或多出未声明当前项 | 追根因解决 | 提交前停止，禁止扫描补齐后继续 |
| 预构造包或删除后快照内部不完整 | 追根因解决 | 尚未进入提交点，丢弃包并停止依赖路径 |
| 进入提交点后任一包出现普通失败返回 | 追根因解决 | 说明预构造契约失效，不得释放成功材料或隐藏半结构 |
| 删除后普通读取为空但审计、反向索引或登记仍有残留 | 追根因解决 | 不声明删除完成，不由读取过滤兜底 |
| 重复删除审计不完整或版本继续推进 | 追根因解决 | 停止幂等依赖路径 |

## 11. 切片边界

### 11.1 CONCEPT-S8-S1

只允许纯值删除候选、完整当前输入和共享直接边重算。禁止修改仓库、事务、服务登记、活动快照或调用删除入口。

### 11.2 STRUCT-TXN-S1

只建立通用结构事务域、四仓库共享 / 独占许可、端点复核覆盖和写集壳。生产实现固定为 `协调.结构事务` 真模块；A01-A14 固定为 `自检.结构事务` 真模块并向中央运行器登记。入口只允许最小上下文组装和登记，禁止承载完整场景、并发循环或逐项输出。禁止概念删除、领域所有者清理、业务状态或恢复。

STRUCT-TXN-S1 直接依赖 #243 完成。执行前必须读取中央运行器、已有自检登记模式、四仓库和写入事务的实际接口；任一接口与假定契约不一致时，退回设计计划修订，不得在实施窗口临时改造计划范围。

### 11.3 CONCEPT-S8-S2

只补核心清理包、审计、反向索引、四类所有者授权、概念登记包和缓存包。禁止编排最终删除提交或发布删除后活动快照。

### 11.4 CONCEPT-S8-S3

只允许概念安全删除服务消费已补全写集，在统一隔离下提交并发布活动快照。禁止非零引用自动解绑、批量删除多个概念、崩溃恢复或旧能力迁移。

### 11.5 CONCEPT-S8-S4

只补完整后验审计、幂等、并发交错和等价概念新句柄重建验收；不得借验收扩大删除范围。
