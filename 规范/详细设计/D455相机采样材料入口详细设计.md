## 1. 设计目标

建立可独立验证的 D455 第一轮材料入口：

```text
RealSense SDK
-> D455 采集器
-> SDK 无关同步帧材料
-> 专用有界缓存
-> 标量外部材料消息
```

该入口只生产非权威材料，不接生产运行期组合器，不调用领域写服务，不解释图像内容。

形成时对应上游：


## 4. 协议模块

### 4.1 导出类型

`协议.D455采样材料` 至少导出：

```text
D455流类型
D455像素格式
D455时间戳域
D455采集器状态
D455采集拒绝原因
D455采集配置
D455设备身份材料
D455相机内参材料
D455相机外参材料
D455图像平面材料
D455同步帧材料
D455采集结果
D455帧来源
```

`D455帧来源` 是只暴露中文业务入口的抽象来源合同：

```text
采样一次(超时毫秒)
请求停止()
关闭()
读取状态快照()
```

模拟来源和真实采集器实现同一合同。上行桥只依赖 `D455帧来源`，不依赖具体 `rs2` 采集器。

### 4.2 采集配置

配置至少包含：

```text
可选设备序列号
RGB 宽、高、帧率、像素格式
Depth 宽、高、帧率、像素格式
左右 IR 宽、高、帧率、像素格式
采样超时毫秒
```

第一轮真实样本默认配置固定为：

```text
RGB：1280 x 720 / 30 fps / RGB8
Depth：848 x 480 / 30 fps / Z16
左 IR：848 x 480 / 30 fps / Y8 / 流索引 1
右 IR：848 x 480 / 30 fps / Y8 / 流索引 2
超时：有限正值，由真实样本入口显式给出
```

第一轮默认固定为 RGB8，并按每像素 3 字节复核步长和实际字节数；不得由调用路径隐式改成 BGR8 或猜测通道顺序。设备不支持该 profile 时逻辑内返回，不静默改用其它格式、尺寸或帧率。

### 4.3 图像平面材料

每路材料必须保存：

```text
流类型与流索引
像素格式
宽、高、每像素字节数、步长、实际字节数
源帧编号
设备时间戳微秒和时间戳域
只读字节组
相机内参材料
```

Depth 额外保存深度比例。同步帧材料还保存设备身份、宿主接收单调时间戳、批次号、标定版本摘要，以及 `Depth -> RGB`、`左 IR -> Depth`、`右 IR -> Depth` 三组 SDK 原始外参材料。第一轮不额外派生逆变换，不用代码合成外参。

四路材料不得共享指向 SDK 缓冲区的视图。`std::vector<std::byte>` 可作为复制后字节容器；帧身份仍由材料编号与版本表达，不能用 vector 地址或 shared_ptr 地址表达。

### 4.4 只读共享

缓存读取允许返回 `std::shared_ptr<const D455同步帧材料>` 式运行期只读快照，以避免整帧重复复制。该指针只作为运行期缓存引用：

```text
不得写入节点、主信息、关系、索引或持久化结构
不得作为长期身份或幂等键
不得比较地址裁决材料相等
缓存索引删除后，已取得的只读快照只允许完成当前读取
```

## 5. D455 采集器

### 5.1 SDK 隔离

`采集器.D455相机` 是唯一允许包含：

```cpp
#include <librealsense2/rs2.hpp>
```

的 D455 生产模块。所有 `rs2::*` 类型放在非导出实现或 PImpl 中。导出的类只出现协议模块类型和标准库所有权类型。

### 5.2 设备选择

打开前先枚举 D400 系列设备并复核产品名 / 产品 ID。规则：

1. 指定序列号时，只能选择该序列号对应的 D455；不存在或不是 D455 时拒绝。
2. 未指定序列号且只有一个 D455 时选择该设备。
3. 未指定序列号且有多个 D455 时拒绝歧义。
4. 只有其它 RealSense 型号时不得冒充 D455。

设备名称、序列号、固件版本和产品 ID 作为外部身份材料保留，不直接成为项目节点身份。

### 5.3 打开与采样

打开顺序：

```text
入口配置拒绝
-> 枚举并唯一选择 D455
-> 配置四路 profile
-> 启动 pipeline
-> 从实际 pipeline profile 读回设备和流配置
-> 读回与请求一致才进入运行中
```

`采样一次` 使用有限超时。取得 frameset 后分别提取 Color、Depth、IR1、IR2；任一路为空则丢弃本次候选，不发布半帧。复制前按实际 profile 计算最小步长和字节数，复制后再次复核。

第一轮不创建 `rs2::align`，不运行滤波，不改变帧内容或标定。

### 5.4 停止

`请求停止` 使后续采样尽快返回；`关闭` 调用 pipeline stop 并释放实现。析构可作为最终资源保护，但显式验收必须调用关闭并证明可收口。不得使用无限 `wait_for_frames`。

## 6. D455 帧材料缓存

### 6.1 配置与身份

缓存构造必须显式接收：

```text
最大条数
最大总字节数
单帧最大字节数
最大材料年龄
```

任一关键上限为零时拒绝构造或进入不可接收状态。材料编号使用非零 `uint64`，只递增不复用；材料版本从 1 开始。版本只表达该缓存身份的生命周期，不表达相机帧像素变化。

### 6.2 状态

缓存条目至少区分：

```text
未发布候选
已发布
已撤销 / 已过期（不再可查）
```

只有已发布条目能由普通只读入口按“材料编号 + 版本”解析。未发布候选只允许本次上行桥通过不透明候选能力复核和确认，不向普通读者暴露。

### 6.3 操作

公开能力至少为：

```text
准备帧材料候选
确认发布帧材料
精确撤销帧材料
按编号版本读取
丢弃过期材料
读取缓存快照
停止接收
```

缓存内部使用专用同步保护索引、条目和字节计数；不得在锁内调用 SDK、外部回调、日志、弹窗、领域服务或消息队列。

第一轮满载策略固定为拒绝新候选，不静默驱逐仍可读材料。调用方可先显式清理过期材料后重新请求；不得用旧材料冒充新采样结果。

## 7. D455 材料上行桥

### 7.1 输入

上行请求至少包含：

```text
非零消息编号
非零批次号
非零消息幂等键
目标运行线程角色
当前宿主时间戳
采样超时
```

目标角色第一轮只允许 `任务管理`，但返回消息仍只是待后续路由复核的材料通知，不形成任务或动作。

### 7.2 顺序

固定顺序：

```text
上行请求入口拒绝
-> 调用 D455帧来源::采样一次
-> 复核 SDK 无关材料完整性
-> 缓存准备未发布候选
-> 通过候选能力读回编号、版本、来源和字节计数
-> 构造标量运行消息
-> 确认缓存发布
-> 普通入口按同一编号和版本读回
-> 返回消息和只读材料引用
```

本轮不调用 `有界运行消息队列::入队`。后续生产组合器若入队失败，必须调用上行桥 / 缓存的精确撤销入口；该接线需要新的专项计划。

### 7.3 消息映射

| 运行消息字段 | D455 来源 |
| --- | --- |
| `消息编号` | 上行请求非零编号 |
| `类型` | `运行消息类型::外部材料` |
| `来源角色` | `运行线程角色::外设采样材料` |
| `目标角色` | 第一轮固定 `任务管理` |
| `目标材料` | 缓存材料编号 |
| `句柄版本` / `当前句柄版本` | 缓存材料版本 |
| `发生时间戳` | 宿主接收单调时间戳 |
| `幂等键` | 上行请求幂等键 |
| `批次号` | 同步帧批次号 |
| `材料摘要` | 设备、四路帧号、布局和标定的非权威摘要 |
| `只读材料说明` | `true` |
| `承载机器事实` | `false` |

不得使用摘要代替缓存读回，不得把设备序列号字符串塞入消息标量字段。

## 8. 自检设计

### 8.1 无硬件模拟自检

`线程/自检.D455采样材料.ixx` 自己拥有模拟帧来源和完整验收正文，入口只登记一次。至少覆盖：

| 编号 | 验收 |
| --- | --- |
| D455-A01 | 六个新文件的模块名、工程登记和生产 / 自检依赖方向正确 |
| D455-A02 | 完整四路模拟 frameset 可转成 SDK 无关材料 |
| D455-A03 | 缺 RGB / Depth / 任一 IR 均逻辑内返回且缓存不变 |
| D455-A04 | 零尺寸、错误步长、错误字节数、空标定、零时间戳在发布前拒绝 |
| D455-A05 | 缓存条数、总字节、单帧字节和年龄上限生效 |
| D455-A06 | 重复编号 / 错版本读取拒绝，已发布材料按编号版本只读 |
| D455-A07 | 内部读回不一致走追根因结果并精确撤销，无普通可读半条目 |
| D455-A08 | 运行消息只含材料引用，且 `承载机器事实=false` |
| D455-A09 | 消息材料引用可解析为刚发布的同一设备、批次和四路帧 |
| D455-A10 | 模拟来源停止后采样拒绝，缓存停止接收后不再写入 |
| D455-A11 | D455 模块不访问四仓库、领域服务、SQL、控制面板或体素 |
| D455-A12 | 既有 THREAD-S1 至 S6 和完整默认自检回归通过 |

模拟自检不包含真实 SDK 调用，因此无设备机器仍应可运行。

### 8.2 真实样本自检

`适配/自检.D455真实样本.ixx` 只有显式参数 `--d455-real-sample-exit` 才登记和运行。至少采集一组启动样本和若干连续样本，验证：

```text
唯一 D455 身份
请求 profile 与实际 profile
四路材料完整及布局
逐路帧号和设备时间戳原样保存
宿主接收时间戳单调
深度比例、内参和外参可读
缓存编号和消息引用闭合
有限等待、请求停止、关闭和析构均可收口
```

真实设备不可用属于该显式验收的逻辑内失败，不影响无硬件默认自检，但不能声明真实样本通过。

## 9. 入口与工程修改

### 9.1 入口

`海中鱼巣/入口.cpp` 只允许增加：

```text
一个模拟自检模块 import，以及 x64 SDK 启用构建中的真实样本模块条件 import
一个显式真实样本参数解析
一次 D455 模拟自检登记
只有参数出现时的一次真实样本自检登记
最小值式报告和退出码汇总
```

### 9.2 工程

`海中鱼巣.vcxproj` 和 `海中鱼巣.vcxproj.filters`：

1. 六个 `.ixx` 以 `ClCompile` 登记并归入现有源文件筛选器；真实采集器和真实样本模块可以按配置条件排除，但不得改为 `ClInclude`。
2. 新增 build-system 属性 `EnableD455RealSense`，默认关闭；默认构建仍编译协议、缓存、上行桥和模拟自检。
3. 只有 x64 且 `EnableD455RealSense=true` 时添加 RealSense include、lib、依赖库、源码可读条件宏和 DLL 复制。
4. Debug x64 链接 `realsense2d.lib` 并复制 `realsense2d.dll`。
5. Release x64 链接 `realsense2.lib` 并复制 `realsense2.dll`。
6. SDK 根只从 `$(VcpkgRoot)` 或 `$(VCPKG_ROOT)` 取得；不得把当前机器的 `D:\vcpkg` 绝对路径写入工程。
7. Win32 不得链接 x64 SDK，也不构建真实采集器和真实样本模块；模拟材料链继续可构建。
8. 未启用真实采集时传入 `--d455-real-sample-exit`，入口必须明确非零拒绝“当前构建未启用”，不得静默通过。
9. 不修改 `D:\vcpkg`，不安装或升级依赖。

SDK 路径不得散落在 C++。实施前优先复核可用的 MSBuild / vcpkg 属性；若只能使用机器绝对路径，必须退回设计确认可移植构建方案，不在执行窗口自行固化个人路径。

## 10. 非成功返回与内部不一致

| 阶段 | 逻辑内返回 | 追根因解决 |
| --- | --- | --- |
| 配置 / 设备 | 配置无效、无设备、多设备歧义、指定序列号不存在、profile 不支持 | 实际 profile 读回与 SDK 已确认启动配置自相矛盾 |
| 采样 | 有限超时、断开、候选四路不完整 | 完整候选存在但 SDK 数据大小、步长、标定或来源矛盾 |
| 缓存 | 超限、满载、过期、重复、错误版本普通读取 | 候选确认后同编号版本读回不一致、字节计数失真、精确撤销目标不一致 |
| 上行 | 上游逻辑内拒绝，因此不生成消息 | 消息已构造但其引用不能解析为刚发布材料 |
| 停止 | 未打开时停止可返回无变化结果 | 已请求停止但有限等待和关闭无法收口 |

追根因结果必须阻止本切片验收继续依赖该路径；容错宏开启时按公共逻辑错误日志和弹窗规则执行。日志和弹窗不参与缓存修复或成功裁决。
