# 节点仓库详细设计

编号：010-D1
日期：2026-07-05
状态：已确认详细设计 / 核心内核 P0 / 可作为第一轮代码实施依据

## 1. 设计目标

节点仓库负责节点身份、节点类型、主信息绑定和生命周期。节点仓库不是业务内容仓库，不承载对象属性、状态、任务结果、方法结果或因果解释。

节点仓库的第一轮目标是提供稳定节点句柄、最小节点记录、有效性判断、删除失效和类型读取能力，为关系仓库、索引仓库、主信息仓库和领域服务提供共同身份地基。

## 2. 规范承接

| 上层规则 | 本设计承接 |
| --- | --- |
| 结构闭环 | 节点只表达结构身份，不用日志、显示标题或临时文本承载机器逻辑 |
| 节点身份 | 稳定身份由仓库编号、节点编号、版本号共同判定 |
| 主信息边界 | 节点记录只绑定主信息句柄，不复制主信息字段 |
| 入口前置拒绝 | 创建、读取、删除和关系引用前先拒绝无效句柄、旧版本和已删除节点 |
| 中文命名 | 函数名、变量名、业务成员名使用中文 |

## 3. 结构设计

| 结构 | 用途 | 写入方 | 读取方 | 生命周期 | 验证方式 |
| --- | --- | --- | --- | --- | --- |
| `节点句柄` | 节点长期身份 | 节点仓库 | 领域服务、关系仓库、索引仓库 | 随节点记录生命周期 | 仓库编号、节点编号、版本号均有效 |
| `节点记录` | 节点最小记录 | 节点仓库 | 节点仓库、授权领域服务 | 创建到删除 | 删除后旧句柄不可读 |
| `节点编号` | 仓库内本地身份编号 | 节点仓库 | 节点仓库、持久化适配 | 仓库内只递增不复用 | `0` 无效，删除后不回收 |
| `节点类型` | 节点结构分类 | 领域服务创建入口 | 领域服务、关系仓库入口拒绝 | 创建后稳定 | 类型错误时写入口拒绝 |
| `主信息句柄` | 节点到主信息的绑定 | 领域服务创建入口 | 领域服务、主信息仓库 | 随节点记录生命周期 | 主信息句柄有效后才能创建节点 |
| `有效状态` | 节点生命周期状态 | 节点仓库 | 节点仓库、领域服务 | 创建到删除 | 删除后旧句柄失效 |
| `创建序号` | 调试、排序和审计材料 | 节点仓库 | 只读诊断和后续持久化 | 创建时写入 | 只递增，不作为业务事实裁决 |

节点记录第一版至少包含：

```text
节点编号
节点类型
主信息句柄
版本号
有效状态
创建序号
```

创建时间若工程环境已提供稳定时间源，可以作为非裁决审计材料加入；没有时间源时，不得用临时时间字符串承载机器事实。

## 4. 编号和版本语义

节点编号规则：

```text
类型语义：uint64
无效值：0
起始值：1
分配方式：仓库内单调递增
删除行为：只标记已删除并递增版本，不回收编号
稳定身份：仓库编号 + 节点编号 + 版本号
```

版本语义：

```text
创建版本号 = 1。
删除节点时节点记录状态改为已删除并递增版本。
普通业务值变化不递增节点版本。
任务推进、特征值变化、方法执行和需求结算不递增节点句柄版本。
业务变化必须由特征值、状态、动态、因果引用、任务或方法结构承载。
关系仓库写入时必须校验源节点和目标节点当前版本。
旧节点句柄不得重新命中删除后的新节点。
```

## 5. 节点类型边界

第一版允许由节点仓库承载的节点类型必须来自正式枚举或正式创建入口。现阶段核心范围包括：

```text
基础信息
语素入口
存在
场景
特征
特征值
需求
任务
方法
状态
动态
二次特征
因果引用
```

其中 `基础信息` 作为项目语义时表示基础信息类信息节点集合；作为 `节点类型::基础信息` 时只表示尚未拆入具体信息类型的通用基础信息节点。代码和规范不得把两者混用。

`高级信息` 不是节点类型，不得新增节点枚举。当前所谓高级业务角色只能由已有 `需求`、`任务`、`方法` 节点类型表达。

节点类型只表达结构分类，不表达业务事实成立。例如 `存在` 节点只能证明存在身份已登记，不能证明现实观察事实已经成立。

## 6. 函数设计

| 函数 | 功能 | 入口拒绝 | 读写结构 | 验证方式 | 状态 |
| --- | --- | --- | --- | --- | --- |
| `创建节点` | 创建指定类型节点并绑定主信息 | 主信息句柄无效、节点类型未定义、写入方未授权 | 写节点表 | 返回有效节点句柄，可读取 | 已实现（文档声明）/ 设计已确认 / 待复验 |
| `读取节点` | 按句柄读取有效节点快照 | 无效句柄、仓库编号不匹配、版本不匹配、已删除 | 读节点表 | 删除后旧句柄返回空 | 已实现（文档声明）/ 设计已确认 / 待复验 |
| `删除节点` | 标记节点删除并使旧句柄失效 | 无效句柄、版本不匹配、已删除 | 写节点状态和版本 | 删除后节点是否有效为 false | 已实现（文档声明）/ 设计已确认 / 待复验 |
| `节点是否有效` | 判断句柄当前是否命中有效节点 | 无 | 读节点表 | 无效、旧版本、已删除均为 false | 已实现（文档声明）/ 设计已确认 / 待复验 |
| `读取节点类型` | 读取有效节点的结构类型 | 节点无效、已删除、版本不匹配 | 读节点表 | 领域服务可据此拒绝错误类型 | 第一轮应具备 / 待复验 |
| `读取主信息句柄` | 读取节点绑定的主信息句柄 | 节点无效、已删除、版本不匹配 | 读节点表 | 特征服务可通过授权路径组织特征值内部入口访问值容器 | 第一轮应具备 / 待复验 |
| `仓库编号` | 返回当前仓库编号 | 无 | 读仓库编号 | 关系仓库可用于校验 | 已实现（文档声明）/ 设计已确认 / 待复验 |

## 7. 入口前置拒绝

节点仓库必须在入口拒绝：

```text
节点编号为 0。
仓库编号不匹配。
版本号不匹配。
节点编号不存在。
节点已删除。
主信息句柄无效。
节点类型未定义。
写入方没有创建该节点类型的权限。
```

入口拒绝只返回局部执行结果或无效句柄，不得先创建错误节点再由校验器修复。

## 8. 与其他仓库的边界

| 对象 | 交互方式 | 边界 |
| --- | --- | --- |
| 主信息仓库 | 创建节点前必须已有有效主信息句柄，或由领域服务组织创建顺序 | 节点仓库不解释主信息值 |
| 关系仓库 | 关系写入前读取节点有效性和版本 | 节点仓库不保存父子列表或语义关系 |
| 索引仓库 | 索引绑定前校验节点有效性 | 节点仓库不保存主键反查表 |
| 领域服务 | 通过正式入口创建业务节点 | 外部业务模块不得裸写节点记录 |

## 9. 禁止行为

```text
不得在节点记录中新增业务字段。
不得用节点类型替代领域事实。
不得复用已删除节点编号。
不得让指针成为长期节点身份。
不得先创建错误节点再由校验器修复。
不得让显示层或日志层写节点。
不得把 `高级信息` 作为节点类型。
```

## 10. 验收条件

设计确认后，代码复验至少覆盖：

```text
有效主信息可创建节点。
无效主信息不能创建节点。
未定义节点类型不能创建节点。
删除后旧句柄失效。
节点编号删除后不复用。
节点类型可被领域服务读取并用于入口前置拒绝。
关系仓库不能用已删除节点创建关系。
普通业务值变化不改变节点句柄版本。
```
