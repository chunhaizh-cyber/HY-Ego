## 1. 定位

本设计承接已经完成并验证的 CONCEPT-S1 至 S5，把后续范围拆为：

```text
S6A：名称反查、待命名请求、上下文临时指代和生命周期初态。
```

## 4. 权威结构

### 4.1 生命周期阶段

建议值式枚举：

```text
概念生命周期阶段::活跃 = 1
概念生命周期阶段::冷却 = 2
概念生命周期阶段::退役 = 3
```

`已删除` 不作为可指向状态节点的第四阶段。物理删除由节点句柄失效表达，避免在有效节点上同时保留“已删除”业务状态。

### 4.2 生命周期抽象状态登记

初始化服务通过状态服务创建三个抽象状态节点，并由概念图服务登记：

```text
生命周期阶段 -> 状态节点句柄
```

状态节点只承载阶段值，不承载发生时间戳、名称或概念身份。登记必须完整、互异、值一致，重复登记同一组幂等，冲突登记写前拒绝。

### 4.3 概念生命周期关系

新增关系类型：

```text
关系类型::概念生命周期
源节点 = 活动概念
目标节点 = 已登记生命周期抽象状态
多目标 = 否，同一源概念最多一个有效目标
写入方 = 概念图服务
读取方 = 概念图服务、显示只读适配
生命周期 = 随概念有效期；物理删除前必须失效
```

关系仓库只做关系类型准入和原子关系重挂；“源必须是活动概念、目标必须是已登记状态、唯一当前状态”由概念图服务封口。

### 4.4 活动图发布接线

S6A 完成后，完整发布必须满足：

```text
四根初始化活动图时均有活跃生命周期关系。
新创建普通概念在进入新活动快照前已获得活跃关系。
复用既有概念必须读回唯一生命周期。
候选发布失败时，本轮新增生命周期关系与概念节点按后缀逆序清理或读取隔离。
活动快照替换后不得出现“活动概念可读但生命周期缺失或多目标”的半结构。
```

首次为四根补生命周期时，概念图服务必须先保存关系数量快照并记录本轮新增关系句柄；四根未全部读回前不得发布“生命周期初始化完成”状态。任一根关系失败时逆序失效本轮已创建关系。已经创建但尚未登记、也未被任何概念关系引用的通用抽象状态节点不构成生命周期事实；内部创建失败仍按追根因解决并停止依赖路径。

## 5. 名称、待命名和临时指代

### 5.1 名称反查

语素服务建议新增：

```text
语素服务::读取概念名称入口组(概念) -> vector<节点句柄>
```

实现只沿 `语素概念追溯` 关系按目标反查有效语素入口，按完整稳定句柄排序并去重。不得读取文本决定概念身份，不得把 `语素对应信息` 的任意绑定都解释为名称。

### 5.2 待命名请求

建议 DTO：

```text
概念待命名请求材料
  概念
  根类别
  活动图版本
  原因（名称入口为空）
```

概念图服务只对当前活动概念返回该材料。已有至少一个有效名称入口时逻辑内返回空；概念无效、非活动或图版本不一致时写前拒绝。

### 5.3 临时指代

建议 DTO：

```text
概念临时指代材料
  表达上下文编号
  指代序号
  概念
  活动图版本
```

它只在一次表达上下文中有效，不写节点、主信息、关系、索引、缓存或日志。它不是中文代词词面；未来表达层可根据上下文把该令牌渲染为代词或指示表达。

## 6. 三类用途观察

### 6.1 用途类型

```text
概念用途类型::一般
概念用途类型::因果预测
概念用途类型::因果行动
```

一般用途适用于四类概念。因果预测和因果行动只接受活动因果概念；其它类别在统计写入前逻辑内拒绝。

### 6.2 非权威缓存

统计服务新增独立命名空间组合：

```text
缓存类型::概念用途观察
来源结构类型::概念
缓存值类型::用途观察
失效条件::权威结构变化 / 手动清空 / 规则版本变化
可持久化 = false
```

建议记录：

```text
概念用途观察材料
  命名空间
  概念
  用途类型
  观察次数候选
  最近观察时间戳
  累计结果效用候选
  可裁决业务事实 = false
```

一般用途只推进次数和最近时间，结果效用候选保持 0。预测 / 行动用途可接收调用方已经结构化计算的有符号效用变化；统计服务不解释来源文本，也不把正值升级为因果事实。

同一概念、用途类型和命名空间的时间戳不得倒退。缓存可清空；清空前后活动图、支持关系和生命周期关系必须不变化。

### 6.3 生命周期复核候选

建议规则材料：

```text
概念生命周期复核规则
  规则版本
  当前时间戳
  最近使用截止时间戳
  退役复核截止时间戳
```

建议候选结果枚举：

```text
保持活跃
建议冷却
建议退役复核
建议重新激活
材料不足
```

统计服务只返回用途观察；最终候选由概念图服务结合当前生命周期、活动图版本、实例支持和外部引用形成。候选仍不是转换许可。

## 7. 生命周期转换

### 7.1 状态机

允许：

```text
活跃 -> 冷却
冷却 -> 活跃
冷却 -> 退役
退役 -> 活跃
```

拒绝：

```text
活跃 -> 退役
退役 -> 冷却
同状态重复转换（作为幂等逻辑内返回）
根概念离开活跃
```

### 7.2 #210 零支持与候选换代前置

活跃到冷却的正式验收链固定为：

```text
选择当前活动普通概念，不得回退四根。
保留失效前 #195 生命周期复核候选作为旧版本值式材料。
通过 #210 正式入口逐条失效该普通概念的当前有效实例支持，直到当前有效支持为零。
每次真实失效推进活动版本，使失效前生命周期复核候选因版本约束过期；#210 清除固定点 `上一候选图_`，审计历史继续可读。
拒绝旧候选，按当前活动版本重新读取 #195 生命周期复核候选。
确认新候选属于同一真实普通概念，来源活动版本当前，零支持与用途观察材料完整。
```

#210 写前逻辑拒绝，或本轮结束时真实普通概念仍未达到零当前有效支持，属于逻辑内返回：本次被拒请求不写，已完成的合法支持失效与活动版本推进保持，不使用旧候选，生命周期关系不变。每条 #210 请求独立原子；S7 不新增跨多关系事务，也不因后续拒绝或并发新增支持回滚已经完成的合法失效。已真实达到零当前有效支持并推进活动版本后，旧生命周期复核候选仍被接受，或新候选的活动版本、零支持数量、用途观察材料与权威读取不一致，才属于追根因解决；禁止回退根、复用旧候选或裸删支持关系继续验收。当前候选仍显示外部引用或正向用途时，同样按既有逻辑内返回保持原生命周期。

### 7.3 权威复核

冷却或退役前必须重读：

```text
当前活动图仍包含该概念。
当前生命周期关系唯一且状态匹配。
不存在有效实例支持。
不存在概念图内部关系之外的有效引用。
用途候选与规则版本、活动图版本一致。
```

概念图内部关系指 `实例支持概念`、`概念上下位`、`概念定义材料` 和 `概念生命周期`。实例支持单独复核；其它类型的有效入边或出边均作为外部引用候选，包括语素、需求、任务、方法和表达相关正式关系。

S7 可为关系仓库补值式 `获取节点相关关系记录组(节点)` 只读入口，但不得让关系仓库裁决“有用”或生命周期。

### 7.4 原子转换

概念图服务持有生命周期写锁后：

```text
读取唯一当前关系记录。
复核状态机和权威材料。
调用关系仓库原子重挂该关系的目标状态节点。
读回新关系版本、唯一状态和活动概念身份。
发布值式转换结果。
```

不得先删除旧关系再创建新关系。第一轮不新增隐藏的进入时间字段；冷却期由调度方在规则版本和截止时间满足后提交复核请求，概念图服务仍重读最近用途材料。

### 7.5 重新激活

以下任一权威事件可形成重新激活依据：

```text
新增有效实例支持。
新增外部正式引用。
收到通过准入的有效用途观察并由概念图服务复核。
```

重新激活只重挂生命周期关系，不复制概念节点、不改变签名、不新增上下位关系。

## 8. 命名需求接线门禁

S6A 的待命名请求不是需求节点。当前需求服务创建完整需求需要主体、目标宿主、场景、目标特征、当前特征状态材料和目标状态；概念节点不是存在宿主，也没有“已命名”目标特征的正式映射。

因此 S6C 必须先只读复核：

```text
待命名请求如何进入自我治理协议且保持非权威请求性质。
命名目标应落在自我存在的哪个正式特征 / 状态，还是需要独立信息目标需求设计。
需求创建后如何关联目标概念而不把概念当存在宿主。
重复待命名请求如何复用同一未完成需求。
名称绑定后如何结算目标状态并停止重复请求。
```

在该映射形成专用流程图和详细设计前，禁止调用裸 `创建需求()` 冒充完整命名需求。

## 9. 安全删除门禁

S7 完成后只允许进入 S8-S0。S8-S0 必须复核：

```text
根、活动图、生命周期和所有入边 / 出边的实际读取接口。
直接下位概念兼容重挂与无环候选。
概念、生命周期关系、支持、定义和上下位关系的批量失效顺序。
节点与主信息协同失效、索引显式清理和等价概念重建。
事务、补偿或活动快照隔离是否足以防止可读半结构。
删除后固定点、旧句柄失效、无孤儿、无环和四根可达。
```

只有 S8-S0 正式记录证明范围足够，才能生成安全删除专用流程图、详细设计和代码计划。

## 10. 目标接口候选

以下是计划假定契约，不是当前实现事实：

```text
语素服务::读取概念名称入口组(概念)
概念图服务::登记生命周期状态(阶段, 状态节点)
概念图服务::读取概念生命周期(概念)
概念图服务::读取待命名请求(概念, 语素服务)
概念图服务::读取全部待命名请求(语素服务)
概念图服务::构造临时指代(概念, 上下文编号, 指代序号)
统计服务::记录概念用途观察(命名空间, 概念, 用途, 时间戳, 效用变化)
统计服务::读取概念用途观察(命名空间, 概念, 用途)
统计服务::清空概念用途观察(命名空间)

```

## 11. 逻辑内返回与追根因解决

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 无效、非活动概念或图版本不一致 | 逻辑内返回 | 不生成请求、不写统计或生命周期 |
| 已有名称 | 逻辑内返回 | 返回空待命名请求，名称关系不变化 |
| 非因果概念提交预测 / 行动用途 | 逻辑内返回 | 不写用途缓存 |
| 用途时间戳无效或倒退 | 逻辑内返回 | 原缓存不变化 |
| 统计候选不足、仍有支持或外部引用 | 逻辑内返回 | 保持原生命周期 |
| #210 请求被拒绝或真实普通概念仍未达到零当前有效支持 | 逻辑内返回 | 本次拒绝不写；已完成合法失效和活动版本推进保持，不使用旧候选，生命周期关系不变 |
| 根概念请求冷却 / 退役、非法状态跳转 | 逻辑内返回 | 关系与版本不变化 |
| 生命周期状态登记后读回不一致 | 追根因解决 | 停止 S6A 依赖路径 |
| 新活动概念发布后生命周期缺失或多目标 | 追根因解决 | 不发布完整成功材料，清理或隔离半结构 |
| 用途缓存前置通过后写入 / 读回不一致 | 追根因解决 | 停止 S6B 依赖路径 |
| #210 已推进活动版本但旧候选仍可接受，或新候选版本 / 零支持 / 观察材料不一致 | 追根因解决 | 停止 S7，禁止根回退、裸删或复用旧候选 |
| 生命周期关系重挂后目标或版本不一致 | 追根因解决 | 停止 S7 依赖路径，不继续召回 |
| 扫描把未实现命名需求或删除接口写成事实 | 治理逻辑错误 | 修订记录后才能生成后续计划 |
