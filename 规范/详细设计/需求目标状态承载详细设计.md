# 需求目标状态承载详细设计

编号：022-D1
日期：2026-07-05
状态：已确认详细设计 / DV-1 已验证 / S0 缺口 2

## 1. 定位

本详细设计用于修正当前 `需求服务` 把“需求目标状态”降格为需求节点主信息 I64 值容器的问题。

需求需要的是目标状态，不是 I64。I64 只允许作为第一轮状态服务和特征服务内部使用的状态值材料，不得作为需求目标的对外语义。

本设计不迁移旧项目需求主信息字段，不新增需求主信息目标字段，不声明需求系统完成。

## 2. 设计目标

第一轮只处理最小闭环中的目标状态满足规则：

```text
当前特征状态材料满足目标状态
第一轮数值材料暂按 当前状态材料 >= 目标状态值 判断
```

本轮不实现任意比较表达式、不实现需求树完整目标模型、不实现价值结算摘要。

## 3. 规范承接

| 上层规则 | 本设计承接 |
| --- | --- |
| MI-V1 值容器范围收束 | 需求、任务、方法不得把主信息仓库当通用值容器 |
| 特征值访问边界 | 只有特征服务可直接访问特征值服务；高级服务不得直接依赖特征值服务 |
| 高级服务依赖边界 | 高级服务只允许直接依赖存在、场景、特征、二次特征、状态、动态、因果服务 |
| 3100 / 5100 / 5110 需求正式合同 | 创建需求时必须已有目标存在、目标特征、当前特征状态材料、目标状态和验证方式 |
| 基础信息服务分层 | 需求服务通过特征服务读取当前特征状态材料，通过状态服务创建目标状态 |
| 入口前置拒绝 | 无效目标存在、无效目标特征、当前状态材料缺失或目标状态创建失败时不创建有效需求 |

## 4. 结构承载

既有目标状态需求路径继续使用现有节点类型和关系类型，不新增主信息字段。概念命名初始化基线另行追加关系类型 13，但不改变既有需求目标状态的模板关系语义。

| 结构 | 用途 | 写入方 | 读取方 | 生命周期 | 验证方式 |
| --- | --- | --- | --- | --- | --- |
| 需求节点 | 需求身份 | 需求服务 | 需求服务、任务服务 | 随需求生命周期 | 创建后节点类型为需求 |
| 目标存在节点 | 被需求改变或满足的存在主体 | 存在服务 | 需求服务、任务服务 | 随存在生命周期 | 创建需求入口检查节点类型为存在 |
| 目标特征节点 | 被需求判断的特征类型 | 特征服务 | 需求服务、方法服务 | 随特征生命周期 | 创建需求入口检查节点类型为特征 |
| 当前特征值节点 | 目标特征当前状态材料 | 特征服务 | 需求服务、方法服务经特征服务读取 | 随特征值生命周期 | 创建需求入口必须经特征服务读出当前状态材料 |
| 目标状态节点 | 需求目标状态承载 | 状态服务 | 需求服务 | 随需求目标生命周期；第一轮作为抽象目标状态 | 读取状态值等于第一轮目标状态材料 |
| 需求目标状态关系 | 需求到目标状态的模板关系 | 需求服务 | 需求服务 | 随关系生命周期 | `模板` 关系可回查目标状态节点 |
| 需求目标材料关系 | 需求到目标存在、目标特征、当前特征值的引用关系 | 需求服务 | 需求服务、任务服务 | 随关系生命周期 | `引用` 关系可按节点类型过滤回查 |
| 共享概念命名目标状态 | 两个根需求初始化完成后的命名目标值材料 | 需求初始化服务经状态服务创建 | 后续概念命名计划只读消费 | 随本次初始化结果；不属于任一根需求目标 | `读取状态值` 等于请求值且 `状态是否实例状态` 为假 |
| 需求目标概念关系 | 后续命名需求到目标概念的专用引用 | 后续概念命名领域服务 | 概念命名领域服务、概念图服务 | 随命名需求生命周期 | 关系类型固定为 13；不参与概念用途或生命周期活性计数，但仍阻断安全删除候选 |

第一轮关系口径：

```text
需求节点 -> 目标存在节点：引用
需求节点 -> 目标特征节点：引用
需求节点 -> 当前特征值节点：引用
需求节点 -> 目标状态节点：模板
```

使用 `模板` 关系承载目标状态，是为了避免与需求结算状态的 `引用` 关系混淆。后续若新增 `需求目标状态` 等专用关系类型，必须先更新 `规范/4010_子规范_统一仓库稳定句柄与通用关系索引边界.md`、`规范/3100_根规范_需求_20260720.md` 与 `规范/5100_子规范_需求下级规则_20260720.md`。

概念命名初始化只在两个根需求均完整形成后，通过状态服务创建一个值为 1 的共享抽象状态，并把句柄放入 `自我根需求初始化结果::概念命名目标状态`。该状态是后续命名请求的值式目标材料，不是第三根需求，不替代安全根或服务根各自的目标状态，也不是概念身份。无效世界树或根需求初始化失败必须在这次共享状态写入前返回；成功路径以共享状态创建和读回作为最后业务写入，不再追加扫描修复。

## 6. 创建流程

```text
输入：目标存在、目标特征、当前特征值、目标状态值材料
-> 检查目标存在为存在节点
-> 检查目标特征为特征节点
-> 通过特征服务读取当前特征状态材料，缺失则拒绝
-> 通过状态服务创建目标状态节点，目标状态不携带发生时间戳
-> 创建需求节点
-> 写需求到目标存在、目标特征、当前特征值的引用关系
-> 写需求到目标状态的模板关系
-> 返回需求节点
```

目标状态节点不是实例状态，不带发生时间戳，不作为需求结算状态。

需求结算状态仍由 `记录结算状态` 通过状态服务创建场景内实例状态，并带发生时间戳。

## 7. 结算流程

```text
输入：需求节点、目标特征节点、当前特征值节点
-> 检查需求节点有效且类型为需求
-> 检查需求引用了目标特征和当前特征值
-> 读取需求目标状态节点
-> 通过状态服务读取目标状态值材料
-> 通过特征服务读取当前特征状态材料
-> 当前特征状态材料满足目标状态：返回 已满足
-> 当前特征状态材料不满足目标状态：返回 未满足
-> 证据缺失：返回 证据不足
```

`需求结算结果` 仍是局部领域枚举，不得持久化为普通 bool。

## 8. 特征值访问边界

`特征值服务` 只作为特征系统内部值承载实现。

允许直接访问 `特征值服务` 的业务类：

```text
特征服务
```

高级服务可直接依赖的基础服务限定为：

```text
存在服务
场景服务
特征服务
二次特征服务
状态服务
动态服务
因果服务
```

需求、任务、方法服务不得直接包含、接收或调用 `特征值服务`。

## 9. 验收条件

代码实施后必须验证：

```text
有效目标存在、目标特征、当前特征状态材料和目标状态可创建需求。
需求节点主信息值容器不再保存目标状态。
需求通过模板关系可回查目标状态节点。
目标状态节点可通过状态服务读出第一轮目标状态材料，且不携带发生时间戳。
当前特征状态材料满足目标状态时结算为已满足。
当前特征状态材料不满足目标状态时结算为未满足。
无效目标存在拒绝且不创建需求。
无效目标特征拒绝且不创建需求。
当前特征状态材料缺失拒绝且不创建需求。
需求服务和方法服务不再直接依赖特征值服务。
MC-01 到 MC-11 不回归。
```
