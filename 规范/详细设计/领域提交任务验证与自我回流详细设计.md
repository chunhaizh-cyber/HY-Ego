# 领域提交、任务验证与自我回流详细设计

日期：2026-07-21

版本：v0.1

状态：现行详细设计；DD-05 P1；施工拆分为 DD-05F0、DD-05F、DD-05A、DD-05B；已按当前特征事务接口和三件套复核修订

## 1. 设计目标

本设计把方法实际结果收束为四层可施工合同：

```text
DD-05F0 特征批次当前值换代与同会话参与底座
-> 让特征领域形成可逆的批次候选，并向具名同层数据操作提供不透明参与合同

DD-05F 任务状态特征集合底座
-> 消费 DD-05F0，让生命周期阶段只作投影，任务完整状态由正式特征集合成立

DD-05A 通用领域提交、任务验证与结构化回流
-> 让原始执行回执先经唯一治理，再形成唯一强类型上行材料

DD-05B D455 工作包方法结果适配
-> 只读消费 DD-04 工作包和材料视图，把 D455 方法结果接入 DD-05A
```

四层共同回答：跨特征 / 任务结构如何同一事务发布、实际结果由谁提交、任务目标由谁验证、未达成和缺口如何收束、需求由谁复核、自我只能消费什么，以及外设材料为什么不能旁路进入自我邮箱。

## 3. 当前代码事实与复用边界

### 3.1 可复用底座

1. `线程/任务结果回执协议.ixx` 已承载任务、方法、动作、场景、实际状态、动态和来源冻结请求，但它只是非权威执行材料。
2. `线程/路由.任务结果治理.ixx::处理显式回执` 已能先复核回执协议，再调用窄组合器。
3. `领域/组合.任务结果结算.ixx` 已具备任务、需求、方法、动作、状态和动态的权威重读，以及未达成转待重筹办、任务完成和需求独立结算底座。
4. `领域/服务.任务.ixx` 和 `领域/数据操作.需求任务方法.ixx` 已具备任务生命周期、方法选择、实际结果和完成的正式写入路径。
5. `线程/路由.运行期业务请求.ixx` 已有任务结果治理请求 variant，可以继续作为显式同步调用入口。

### 3.2 当前差距

1. 任务权威材料仍主要由 `分层任务生命周期阶段`、生命周期关系和少量选择 / 结果关系组成；5200、5210 的固定任务特征类型、状态段和阶段特征矩阵尚无机器承载。
2. `特征业务服务` 是实例槽、模板和当前值关系的唯一公开写入方；现有 `特征体系数据操作` 已有同会话“创建槽位并发布一个初始状态”和特征值侧表事务参与者，但公开调用是独立提交，尚无候选任务宿主与多特征批次共同确认 / 发布的受控内部接口。
3. `任务结果治理路由` 与 `任务管理上行桥` 都调用 `任务工作线程::取出一条任务结果回执()`，形成两个竞争消费者。
4. `任务结果结算组合器` 在同一次调用中直接完成任务并提交需求结算，没有经过“任务域形成唯一上行材料 -> 自我请求 -> 需求域复核”的职责边界。
5. `需求业务服务::判定目标状态达成` 仍以裸状态值比较为主，没有完整经过类型、值域、版本和三态判断收束。
6. `自我治理消息协议.ixx` 仍允许外部材料适配入口形成观察材料准入请求；外设报告可能绕开任务工作包和正式方法结果治理。
7. `自我治理领域路由.ixx` 的固定安全根兼容路径仍会直接创建方法首、动作入口和选择关系；该路径不得提升为通用内部循环。

### 3.3 既有设计的使用方式

- `任务结果完成与需求独立结算详细设计.md` 只复用权威重读、目标裁决、待重筹办、任务完成和需求结算结构，不直接作为施工依据。
- `任务执行调度与强类型回执详细设计.md` 只复用强类型执行回执和冻结来源，不证明完整任务特征或回流已经形成。
- `自我治理循环详细设计.md` 是归档重定向，不得作为施工依据。
- 生产线程启动、连续调度、停止、恢复和完整装配属于 DD-06，不在本设计施工范围。

```text
-> 任务域读取已发布实际结果、任务目标和完整任务特征快照

-> 裁决目标达成、推进、禁止结果、验证和当前性

-> 形成唯一任务治理回流材料
-> 任务管理上行桥只消费治理后材料并投递自我治理邮箱
-> 自我请求需求域复核来源需求
```

独立环境事实变化可以直接成为自我下一轮状态重判触发，但不得伪造任务结果、任务完成或需求满足。任务绑定动作必须完整经过上面的任务验证主链。

## 5. DD-05F0：特征批次当前值换代与同会话参与底座

### 5.1 定位

现有特征数据操作已经支持单槽位初始值原子发布，特征值原始材料参与者已经能够登记多项新值，但尚缺：

```text
同一批次创建多个实例槽和初始值
同一批次为多个既有槽形成新当前值并失效旧当前关系
初始与换代混合批次的完整读回、撤销和最后发布
供具名同层数据操作参加同一事务的不透明批次合同
```

DD-05F0 只补通用特征原子能力，不写任务阶段、状态段或 55 类任务语义。

### 5.2 值式规格和内部参与包

特征业务服务形成不可变的候选宿主初始特征项规格、既有宿主当前值换代项规格和特征批次变更规格。每个初始项至少固定特征定义、槽位幂等主键、特征值幂等主键、原始类型和值、合法来源身份 / 版本；每个换代项还固定既有槽位、预期当前值关系 / 值 / 版本和新值身份。批次必须有幂等身份、规则版本、确定顺序和完整载荷摘要，但摘要不替代逐项语义比较。

特征体系数据操作在内部形成一个不可复制的不透明参与包：

```text
本批次规格
+ 新特征值原始材料参与者
+ 精确候选写集
+ 旧当前关系和撤销材料
+ 批次阶段
```

参与包只能在数据操作内部协作层被结构写入执行器使用；不得进入特征业务服务返回值，不得交给任务服务、组合器、线程、消息或外部调用方。

### 5.3 同层参与合同

特征数据操作是实例槽、模板、当前值关系和值材料的唯一结构写入者。具名同层数据操作只可提交已经由特征业务服务形成的批次规格，并在同一个结构写入会话中提供候选宿主；它不能直接创建特征槽、值节点、当前值关系或访问特征值服务。

```text
特征业务服务写前复核定义、值域、来源和批次语义
-> 同层数据操作取得唯一结构写入权并创建其领域候选宿主 / 结构
-> 特征数据操作在同一会话写多个槽位 / 新值 / 当前值换代候选
-> 特征值参与者准备原始材料
-> 会话与参与者完整确认
-> 最后发布并释放唯一写入权
```

原始 `结构写入会话`、执行器、事务令牌和参与者不出数据操作层。施工必须用具名友元、内部桥或等价编译期边界，把参与入口限制在登记的数据操作；不得建立领域公开“传会话”函数。

### 5.4 当前值换代

换代不原地覆盖旧值：创建新特征值节点与原始材料候选，建立新当前值关系，旧当前值关系只失效并保留历史。普通读取在统一可见点前只能看到旧完整值，发布后只能看到新完整值；不得观察双当前或无当前的中间组合。

精确同键同义批次读回既有发布结果；同键异义、预期关系 / 版本漂移或批次顺序不同均在第一写前冲突。完整前置后任一候选、读回、确认、撤销或发布异常属于内部逻辑错误。

### 5.5 最低验收

1. 多槽初始、多值换代和初始 / 换代混合批次均一次发布。
2. 任一写点失败按逆序恢复旧完整结构和侧表；撤销失败隔离事务域。
3. 统一可见点前普通读取只见旧值或未建立，发布后只见完整新值。
4. 同义幂等、异义冲突、版本漂移和重复槽 / 值逐项成立。
5. 服务公开签名不含会话、执行器、令牌、参与者、仓库或可变记录。
6. 核心执行器、仓库、特征值服务和关系类型零修改。

## 6. DD-05F：任务状态特征集合底座

### 6.1 机器承载

任务完整状态使用现有结构共同承载：

```text
任务节点和稳定句柄
+ 来源需求、运行态治理存在、场景、目标状态等结构锚点
+ 当前生命周期关系、版本和发生时间
+ 任务固定特征定义、实例槽位、当前特征值和来源关系
+ 当前业务状态段及状态段当前性
+ 方法开放特征挂载记录
= 当前任务状态特征快照
```

`任务状态特征快照` 只是一份值式读回投影，不新增第二套机器事实。枚举值只有在任务、特征定义、实例槽位、值节点、合法来源、版本、当前性和生命周期版本全部闭合时才有业务意义。

`任务状态段材料` 同样不是第二套状态权威；它只投影 5200 固定特征中的 `当前业务状态段` 和 `状态段当前性`，并同时携带其特征定义、槽位、值、来源和版本。只有这两个固定特征与该状态段要求的全部区分特征共同成立时，状态段材料才可读。

### 6.2 固定特征目录

任务领域建立 5200 第 4 章全部固定特征类型的稳定目录。每个目录项至少包含：

```text
固定特征类型
稳定特征定义身份
值种类和值域
允许来源角色
首次可出现阶段
允许当前的阶段集合
是否为目标阶段必需
是否允许空值
验证规则版本
```

固定特征目录由任务领域唯一解释；普通调用方不得临时创建任务业务特征类型。方法开放特征继续使用正式特征定义身份和挂载记录，不塞入固定枚举。

### 6.3 强类型值

固定特征值至少区分：

- 强类型分类值：承接角色、当前性、状态段、筹办结果、等待类型、执行结果、目标达成、验证、归因、失败和取消等；
- 单调编号：筹办轮次、等待版本、执行轮次、终止版本；
- 稳定身份和版本引用：活动路径、AND 组、来源事实、触发事实、冻结和方法内容版本；
- 有量纲数值：父权重、AND 份额、冻结预算、推进量、剩余差距和目标达成比例。

数值必须保留类型、单位或量纲、值域、来源和版本；不得以裸 `I64` 或显示文本代替。

### 6.4 原子迁移

DD-05F 必须消费 DD-05F0 已进入 main 的批次初始 / 当前值换代规格、不透明参与包和同层参与接口；不得在本切片重新实现通用特征候选或改变核心执行器。

特征和任务写入权分工固定为：

```text
任务业务服务
-> 裁决任务固定特征值、来源、目标阶段和业务状态段
-> 调用特征业务服务形成不可变批次规格
-> 调用需求任务方法数据操作提交任务领域请求

特征业务服务
-> 唯一公开复核特征定义、值域、宿主类型和初始值规格
-> 不接收或返回原始事务会话、参与者或候选能力

需求任务方法数据操作 + 特征体系数据操作
-> 只在数据操作内部协作层共享同一个结构写入会话
-> 任务数据操作只写任务专有结构
-> 特征数据操作只写实例槽、模板、当前值关系和值材料
-> 特征值侧表继续作为同一事务参与者
```

`特征体系数据操作` 提供只对具名任务数据操作开放的候选宿主批次写入桥；桥只接受特征业务服务已经形成的不可变规格和本次会话创建的任务宿主，不向任何领域服务、组合器、线程或消息暴露 `结构写入会话`、执行器或参与者。不得由任务数据操作复制特征槽位和值写入算法，也不得通过两个独立领域提交事后补齐。

任务首次创建固定为一个原子入口：

```text
写前形成 55 类固定特征目录引用和“已创建 / 承接初态”所需特征批次规格
-> 同一会话创建任务、来源关系、治理存在、场景和目标锚点
-> 同一会话创建初始生命周期状态与关系
-> 同一会话由特征数据操作创建初始实例槽、值、来源和承接初态状态段
-> 任务结构与特征批次共同确认
-> 特征值侧表参与者确认
-> 最后发布完整任务
```

禁止先发布裸任务，再调用“初始化任务固定特征”补写；旧任务若缺少完整特征只能读为结构不完整并进入具名迁移治理，不得运行期静默补默认值。

任务状态迁移改为：

```text
权威读取当前任务锚点、生命周期和完整特征快照
-> 复核预期当前生命周期关系、版本和特征快照版本
-> 按 5210 计算目标阶段必需、可选和禁止特征
-> 形成全部新特征值候选和目标业务状态段候选
-> 形成目标生命周期状态与迁移证据候选
-> 在同一不透明结构会话中完整读回候选
-> 最后发布新的生命周期关系和特征快照当前性
-> 旧生命周期、状态段和特征值保留历史
```

目标阶段必需特征缺失、禁止特征仍当前、来源不合法、版本不一致或值域错误，必须在第一写前拒绝。前置通过后候选创建、读回、撤销或最后发布异常属于内部逻辑错误。

### 6.5 公开入口

公开入口和内部协作入口分层如下；具体签名由施工时按当前接口机械落位：

```text
特征业务服务::形成候选宿主初始特征批次规格(任务特征值式请求)
任务业务服务::创建承接任务(请求中完整携带初始固定特征来源和值式材料)
读取任务状态特征快照(任务)
提交任务状态迁移(任务状态迁移请求)
复核任务阶段与特征集合(任务, 预期阶段)
读取任务状态段历史(任务)
```

`形成候选宿主初始特征批次规格` 只返回不可变值式规格，不执行提交；`创建承接任务` 是初始任务结构和初始特征的唯一公开提交入口。若为实现复用保留具名“初始化”函数，它只能是同一提交内部助手，不得成为对已发布裸任务的独立业务入口。

既有 `迁移生命周期` 不再允许仅凭目标阶段和幂等编号完成生产迁移。兼容调用必须补齐目标特征变更，或在执行前被拒绝并退回设计；不得在服务内部为未知调用者补默认业务特征。

### 6.6 最低验收

1. 已创建、已承接、筹办中、排队中、执行中、等待中、待重筹办、已完成、失败和取消十个阶段均按 5210 验证最低特征、可选特征和禁止特征。
2. 同阶段不同方法开放特征或值形成不同完整状态，不被阶段枚举合并。
3. 缺任一结构锚点、必需特征、来源、版本或当前性时读回为结构不完整，不以阶段兜底。
4. 精确重复迁移幂等读回；异义特征值、来源或版本冲突。
5. 迁移失败不留下可读半特征集；旧状态和旧特征完整保留审计。
6. 任务结果治理不得在没有完整执行中 / 结果待验证特征快照时进入任务完成判断。

## 7. DD-05A：通用结果治理与结构化回流

### 7.1 唯一原始回执消费权

`任务结果治理路由::处理工作线程一条回执` 是原始任务结果回执队列的唯一消费者。`任务管理上行桥` 删除对 `任务工作线程` 的依赖，不再调用 `取出一条任务结果回执()`。

显式同步调用可以继续调用 `处理显式回执`，但必须经过同一协议复核和同一组合器，不能平行实现第二套治理。

### 7.2 任务结果治理输出

任务结果组合器只返回一个强类型 `任务治理回流材料`。所有后继共同字段至少包含：

```text
回流材料身份和规则版本
来源任务、来源需求、方法运行身份、动作入口和执行轮次
来源冻结、任务状态特征快照和结果轮次版本
每项方法结果的唯一分类
事实提交等级
目标达成裁决、推进量、剩余差距、验证和禁止结果
任务域正式提交结果及提交后权威读回
后继类型和后继当前性
幂等键、去重键、发生时间和过期条件
```

后继类型决定附加必需证据：

| 后继类型 | 条件必需证据 |
| --- | --- |
| 完成待需求复核 | 已发布实际状态、恰一动作动态和可选因果引用；必须满足 5160 |
| 待重筹办 | 上一结果 / 当前性失效事实、旧冻结失效证据和重新触发来源 |
| 合法等待 | 等待对象、条件、合法生产者、触发合同和等待版本；可以没有可发布实际状态 |
| 结构化缺口 | 三组复合值及其来源证据；可以没有可发布实际状态或动作动态 |

待复核和不可提交的执行材料不得为了填满统一 DTO 伪造实际状态或动态；它们只按其合法后继的条件合同进入回流。

合法后继类型固定为：

| 后继类型 | 任务结构变化 | 上行用途 |
| --- | --- | --- |
| 完成待需求复核 | 任务域已经提交完整完成特征和结果 | 请求需求域复核；不携带“需求已满足”布尔值 |
| 待重筹办 | 任务域已经失效旧冻结并提交未达成 / 当前性失效特征 | 触发后继筹办，不提交需求结算 |
| 合法等待 | 任务域已经提交等待类型、条件、生产者和重新触发合同 | 供自我重判与调度，不冒充失败 |
| 结构化缺口 | 形成完整缺口候选，不直接创建需求 | 交需求统一准入形成、合并或拒绝 |

入口拒绝和内部逻辑错误不是上述业务后继。内部逻辑错误停止依赖路径，不投递“等待”或“缺口”消息。

独立世界事实变化不属于 `任务治理回流材料`。它使用独立 `世界事实变化触发材料`，至少包含事实稳定身份 / 版本、来源领域、状态 / 动态证据、发生时间、当前性、去重键和过期条件；不得填入虚假任务、需求、方法或执行轮次。该材料仍须经过与任务回流相同的消息封套完整性、同义去重、异义冲突和当前性验证后，才能触发自我状态重判。

### 7.3 方法输出与世界事实分层

方法输出状态、输出运行场景和执行报告先保留在动作运行作用域。只有对应实在域服务根据 8210 的来源、时间、范围、质量、版本和提交等级完成最后发布后，任务域才能把发布状态作为实际结果读取。

正常事实、异常事实和待复核材料分别保留；不可提交材料不得写世界事实。执行报告成功不替代实际回读，输出运行场景不直接成为世界当前场景。

### 7.4 目标验证

任务域按 5120 对每个结果唯一归类，然后使用正式特征、状态和判断收束服务比较任务目标与实际结果。`任务结果归类事实组` 不是临时 DTO：它是任务状态迁移中 `执行结果类型`、`附带结果路由状态`、`禁止结果状态` 等正式固定特征及其来源证据的强类型值式解释，逐项保留结果稳定身份、四类归类、来源方法、动作动态、输出场景、规则版本和当前性，并与本轮任务状态特征快照共同发布。任务治理回流材料只能读取该已发布事实组，不得重新在消息层分类。

必须区分：

```text
已达成
未达成但有推进
未达成且无推进
方向相反
材料不足
不可比较
版本过期
禁止结果同时成立
内部材料矛盾
```

任务只有在实际结果当前、目标已达成、验证通过、禁止结果已独立治理、完成当前性为当前、目标达成比例和完成来源完整时，才能提交已完成。方法成功、动作发生、来源需求满足和日志都不是完成证据。

### 7.5 派生需求三组复合值

“派生需求三件套”解释为三组复合值，而不是三个字符串：

```text
目标组 = 目标对象 + 目标特征 + 目标状态 + 目标合同版本
当前与偏差组 = 当前状态 + 结构化偏差 + 事实提交等级 + 当前性
来源证据组 = 来源任务 + 来源方法 + 动作运行身份 + 来源动作动态 + 结果轮次
```

同时携带父需求、活动路径、发生时间、规则版本、合法生产者和回流触发。缺父需求、目标不完整、父子目标重叠或循环、代码缺陷需求化都必须拒绝或按内部逻辑错误收口。

### 7.6 上行封套与生命周期

任务管理上行桥只接收已发布的 `任务治理回流材料` 或 `世界事实变化触发材料`，按各自合同验证完整性后形成非权威值式自我治理消息。封套至少具有：

```text
消息身份
材料类型；任务材料使用来源任务与结果轮次，世界事实材料使用事实身份与版本组成去重键
权威结构稳定引用和版本
发生时间、接收时间和过期条件
消息类型与目标服务
确认 / 已消费状态只作队列治理，不作业务事实
```

重复同义材料幂等；同键异义冲突；过期材料具名拒绝。消息本身不能变成任务、需求或世界事实。

### 7.7 需求复核

自我收到“完成待需求复核”后，只请求需求业务服务复核来源需求。需求服务必须按 5160 完整验证需求、目标合同、来源任务、实际状态、动态、可选因果和规则版本，并用正式判断收束服务裁决。

只有明确已满足才发布唯一需求正式结算记录。未满足不建结算记录；材料不足、不可比较、过期和冲突返回具名结果。任务域不得直接写需求，需求域不得回写任务。

### 7.8 旧旁路退出

1. `自我治理消息协议` 不再允许原始外设报告、外部材料适配入口或未治理任务回执进入自我邮箱。
2. 固定安全根兼容路由不再自动创建方法首、动作入口或方法选择。仍需保留的自检必须提供已由任务 / 方法领域形成的正式材料。
3. `非权威待需求结算提示` 若仍存在，只能转换为新的需求复核材料，不能直接触发结算。

## 8. DD-05B：D455 工作包方法结果适配

### 8.1 前置

DD-05B 只有在 #327 / DD-04 进入 `main`、DD-05A 进入 `main`，且设计窗口逐签名复核下列合同后才能施工：


### 8.2 唯一适配方向

```text

-> D455 方法动作运行
-> D455 实际回读与来源质量材料

```

结束不是与治理并行的旁支。只有 DD-05A 返回已经持久化且可权威读回的任务治理结果，适配层才按该结果调用工作包 / 材料结束入口。写前拒绝保持工作包未消费或按 #327 合同标记本次尝试拒绝；内部逻辑错误停止路径并隔离本次工作包，不得标记正常结束；重复同义治理只允许幂等读回同一个结束结果。

DD-05B 不消费物理报告队列、不取得 DD-02 消费权、不创建 DD-03 承接、不修改通用任务结果协议，也不承担生产线程装配。

### 8.3 三类结果

| 方法入口 | 实际结果 | 正式提交和任务材料 |
| --- | --- | --- |
| 稳定观察 | 已确认的存在 / 状态观察或空观察事实 | 按质量和范围提交正常、异常或待复核事实；空观察不伪造存在 |
| 扫描变化 | 前后可比较的变化或材料不足 | 形成状态、动态和结构化残差；材料不足进入待复核 / 等待 |
| 目标跟踪 | 当前目标坐标、暂失或丢失 | 坐标当前时提交跟踪状态；暂失 / 丢失保留空坐标和具名状态 |

每类结果都绑定工作包、方法运行身份、内容版本、动作入口、时间、范围、质量和当前性。D455 报告本身不进入自我邮箱。

## 9. 逻辑内结果、拒绝与内部错误

| 分类 | 示例 | 收口 |
| --- | --- | --- |
| 写前拒绝 | 身份、类型、权限、版本、任务特征快照、工作包或材料当前性不合格 | 零结构变化，返回具名拒绝 |
| 逻辑内结果 | 目标未达成、输入过期、材料不足、合法等待、外部事实变化、工作包过期 | 由任务域提交对应强类型后继 |
| 内部逻辑错误 | 完整前置后写入 / 读回 / 发布异常；同一回执双消费；执行期才发现冻结前结构缺口；同一任务出现互斥当前特征 | 停止依赖路径并追根因，不包装成等待、失败或需求 |

## 10. 并发和幂等

1. 原始回执队列只有任务结果治理路由一个消费者；任务管理上行桥无权读取原始回执。
2. 同一任务结果轮次由任务稳定身份、执行轮次、冻结版本和回执身份组成治理幂等键。
3. 同义重复读回既有任务治理结果；异义重复冲突；不能靠“另一线程得到相同结果”掩盖并发越权。
4. 不同任务可以并发治理；同一任务的状态迁移和完成提交由任务结构当前关系及版本串行。
5. 需求第一轮同一稳定身份至多一个已发布满足记录。
