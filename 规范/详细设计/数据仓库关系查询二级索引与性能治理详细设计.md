# 数据仓库关系查询二级索引与性能治理详细设计

日期：2026-07-13

状态：正式详细设计 / JY-353 从 `c5a2f0b` 实际接口复核通过 / #257 当前首个待执行 / 790 未实施

## 1. 目标

`规范/仓库与服务分层事务边界规范.md` 和 #262-#271 架构纠偏链已经完成。下一步基于新的无原始令牌业务调用路径建立可复现性能基线，再用关系仓库内部二级索引缩小关系查询候选集。令牌路径、批量读取、仓库分片和控制面板按需投影均后置并按指标裁决。

本设计不把当前六项显示分类解释成六棵物理树。概念结构按多根森林处理，并允许多父共享节点和图关系。

## 2. 当前代码事实

1. `关系仓库` 当前用 `unordered_map<关系编号, 关系记录>` 保存唯一权威主表，并由一把 `shared_mutex` 保护。
2. 创建重复检查、关系审计组、子节点、正向 / 反向关系组、节点相关关系、存在判断、有效数量和删除准备包均存在遍历整张关系表的路径。
3. `索引仓库` 当前是独立的主键到节点候选入口，不承载关系事实，也不适合承担关系写入同锁一致性。
4. 四个核心仓库各有独立读写锁；结构事务许可与活动许可表另有协调竞争路径。
5. 当前没有正式锁等待、扫描条数、吞吐、P95 或 P99 基线，不能声明运行饱和或分片必要。
6. 控制面板当前可生成六类完整结构，但六类只是显示分类；显示层不得成为事实源。

## 3. 权力与结构边界

```text
权威关系：关系表当前记录。
派生候选：关系仓库内部二级索引中的关系编号集合。
领域入口候选：通用索引仓库现有主键 / 命名空间索引。
机器裁决：领域服务 + 关系仓库对权威记录和端点的复核。
人读投影：控制面板按请求生成的值式树 / 森林 / 图材料。
```

不得为世界、概念、需求、任务、方法或显示分类建立互不相通的身份体系。不得把二级索引持久化结果、命中次数、性能报告或控制面板投影解释成机器事实。

## 4. S0 可复现基线

### 4.1 隔离夹具

建立只用于自检的隔离四仓库夹具，不接生产运行期上下文。固定规模为 1,000、10,000、100,000 条有效关系；每个规模至少包含：

```text
正向稀疏键和密集键
反向稀疏键和密集键
同顺序号多关系
多根概念森林
同类概念多父共享节点
非普通父子的图环
有效与已失效当前审计记录
重挂后同编号新版本关系
```

普通父子继续遵守当前单父与防环规则；多父和图环只能使用已允许相应语义的关系类型。

### 4.2 指标

每个样本预热 20 次、正式至少 200 次，报告中位数、P95、P99，不只记录平均值：

| 指标 | 定义 |
| --- | --- |
| 候选检查数 | 为回答一次查询实际检查的关系记录数量；S0 按当前代码路径与固定规模记录全表上界，S1 由隔离自检记录索引桶候选数 |
| 查询耗时 | 公开查询入口从调用到值式结果返回的单次耗时 |
| 等待代理值 | 并发场景调用耗时减去同规模单线程无竞争中位数；明确不是 `shared_mutex` 内部精确等待时间 |
| 吞吐 | 固定 10 秒窗口内完成且结果正确的操作数 |
| 写入代价 | 创建、失效、重挂、删除各自 P95 / P99 |
| 正确性摘要 | 与隔离全表参考模型逐关系句柄、版本、状态、端点、顺序对比 |

机器、编译配置、CPU、电源模式、线程数、随机种子、关系分布和提交号必须写入基线记录。不同机器的绝对耗时不得直接横向裁决。

## 5. S1 二级索引结构

二级索引作为 `关系仓库` 私有成员，与关系表使用同一把仓库锁：

| 索引 | 键 | 值 | 用途 |
| --- | --- | --- | --- |
| 有效正向索引 | 完整源节点句柄 + 关系类型 | 按顺序号、关系编号可稳定排序的关系编号候选 | 目标组、来源关系记录组、重复 / 唯一性检查 |
| 有效反向索引 | 完整目标节点句柄 + 关系类型 | 关系编号候选 | 来源组、目标存在、普通父子单父检查 |
| 有效节点相关索引 | 完整节点句柄 | 关系编号候选 | 节点删除影响面、节点相关关系读取 |
| 当前审计正向索引 | 完整源节点句柄 + 关系类型 | 有效或已失效当前记录的关系编号候选 | 当前版本审计组 |

键必须使用完整稳定句柄，不能只用本地节点编号。值只保存关系编号，不复制权威状态、端点或版本。审计索引只支持当前版本有效 / 已失效记录，不虚构旧版本历史链。

## 6. 查询流程

1. 入口先拒绝无效节点句柄和未定义关系类型。
2. 在关系仓库共享锁内按键复制关系编号候选。
3. 对每个编号读取关系表当前记录，复核关系编号、当前版本、类型、状态、源 / 目标完整句柄和可选顺序号。
4. 释放关系仓库锁后按现有事务语境复核端点有效性；不得让显示层或调用方自行解释端点。
5. 按现有契约以顺序号、关系编号确定性排序；节点组不得因索引容器顺序改变返回顺序。
6. 普通业务读取只返回有效关系；显式审计读取才可返回当前已失效记录。
7. 同一结构许可内发现索引候选与关系表不一致属于逻辑错误，停止后续实施或验收；不得自动全表扫描修复。

## 7. 同一提交边界

### 7.1 创建

写前用相关桶完成完全重复、单父、任务生命周期槽位和任务方法选择唯一性检查。先预备所有可能分配内存的新索引项，再在同一独占锁内发布主表和索引。任一步异常必须撤销本次已插入项和主表记录，不返回有效关系句柄；编号允许保持单调跳号。

### 7.2 失效

复核精确当前句柄后推进关系版本和状态；从三个有效索引移除，当前审计索引继续指向同一关系编号。重复失效保持现有幂等语义，不重复推进版本。

### 7.3 重挂

先按新端点执行全部入口拒绝和唯一性检查，再预备新端点索引项。提交时更新同一关系编号、推进版本、移除旧有效 / 审计键并发布新键。旧句柄失效，新句柄及新端点候选可读；任何失败不得留下新旧端点同时作为有效事实。

### 7.4 物理删除

只接受现有授权和精确当前句柄。先复核三个有效索引、审计索引与主表对应，再在同一锁内移除所有索引项和主表记录。普通删除继续要求独占令牌；S1 不改变事务 ABI 或删除授权。

## 8. 多根、多父与图关系

1. 二级索引不内置“整仓库只有一个根”假设。
2. 同一节点可在允许多父语义的关系类型中由多个源指向；每条关系以完整关系句柄独立保留。
3. 图关系可以成环；只有明确要求无环的普通父子或领域专用关系由原所有者拒绝成环。
4. 同一节点从多个根可达时，关系查询返回全部有效边；显示投影可以按完整节点句柄去重节点展示，但必须保留边和引用位置。
5. 完全重复关系继续按关系类型规则写前拒绝，不能用显示去重掩盖重复事实。

## 9. S1 验收阈值

正确性是硬门槛，性能收益不能抵消正确性失败：

```text
A. 全部查询结果与隔离全表参考模型逐项一致。
B. 创建、失效、重挂、删除后，主表和所有索引的一致性检查为 100%。
C. 100,000 关系稀疏夹具中，单次候选检查数不超过对应桶实际关系数，且不再与全表 N 成正比。
D. 相同机器 / 配置下，稀疏正向、反向、存在、节点相关查询 P95 均不高于 S0 的 50%。
E. 90% 读 / 10% 写固定并发场景吞吐不低于 S0 的 150%。
F. 纯写场景 P95 不得高于 S0 的 150%，且入口拒绝、版本和事务回归全部通过。
G. 小规模 1,000 关系完整自检总耗时不得出现超过 10% 的稳定回归；连续三轮超过即停止验收。
```

若 D-G 因测量抖动未稳定满足，保留正确实现但不得宣称性能验收通过，必须重复基线或退回设计裁决。

## 10. S2 证据门槛

S2 只有在 S1 后连续三轮满足任一条件时才允许改代码：

```text
取得 / 验证结构许可占目标查询 P95 的 20% 以上；
有效关系数量或多次单项读取占调用链 P95 的 20% 以上；
批量业务调用中同一许可内重复读取同一关系桶 8 次以上，且复制 / 复核成本占 P95 的 20% 以上。
```

未达到门槛时，S2 只生成“不实施”记录并完成。达到门槛时也只能优化已证明热点；不得改变核心事务 ABI、许可强度、同线程重入拒绝、普通删除独占或令牌不跨线程 / 队列保存规则。

## 11. S3 分片门槛

S3 只有在 S1 与必要的 S2 后，100,000 关系、至少 8 读取线程 + 2 写入线程的稳定负载连续三轮同时满足下列条件时才进入代码设计：

```text
关系仓库等待代理值占查询 P95 的 30% 以上；
增加线程后吞吐提升低于 20%；
热点集中在可稳定划分且不会破坏统一句柄 / 跨分区关系提交的键域。
```

未满足即记录“不分片”。满足后必须另行冻结分片键、跨分区写集、锁顺序、恢复和快照格式；本设计不直接授权分片代码。

## 12. 控制面板按需投影

控制面板后继只接收：显示分类、一个或多个当前概念根、关系类型白名单、深度、页大小和继续游标。默认不生成全量六类结构。投影必须：

```text
从领域服务或关系仓库只读入口取得值式候选；
复核关系句柄、版本、状态和有效端点；
对多根森林逐根生成，对共享节点保留引用边；
使用访问关系集合防止图环无限展开；
不写仓库，不回流索引，不缓存为机器事实。
```
