# 方法动作场景角色关系详细设计

日期：2026-07-15

状态：JY-347 / QR-183 已裁决 / #279 实施前详细设计 / 代码未实现

## 1. 定位

本文解决 `#224 / TASK-EXECUTION-S1` 暴露的结构冲突：动作入口输入场景和输出场景在第一轮都必须等于任务场景，但当前以普通 `引用` 写入顺序号 1 / 2 时，关系仓库按“类型 + 源 + 目标”拒绝第二条关系。

本设计新增专用关系：

```text
关系类型::方法动作场景 = 17
方向：动作入口方法节点 -> 场景节点
顺序号 1：输入场景角色
顺序号 2：输出场景角色
有效唯一键：关系类型 + 完整源动作入口句柄 + 顺序号
```

本文不改变 TE-D6，不把普通关系的全局唯一键改成包含顺序号，不实现任务执行调度、线程队列、状态动态写入或强类型回执。

## 2. 依据和当前事实

### 2.1 已验证当前事实

1. 当前 `关系类型` 已定义到 `任务方法选择 = 16`；关系 17 当前是两个自检中的未知关系负例。
2. 关系仓库只对 `任务生命周期证据` 和 `任务方法选择` 使用专用唯一性；其它关系按“类型 + 源 + 目标”拒绝有效重复。
3. 当前动作入口创建使用：

```text
引用(动作入口, 输入场景, 1)
引用(动作入口, 输出场景, 2)
```

4. 当前动作入口读取从普通 `引用` 关系组按顺序号 1 / 2 解释输入、输出角色，并返回两个完整关系证据。
5. `任务生命周期证据 = 15` 已证明专用角色关系可以按“同源 + 同顺序号”唯一，同时允许同一目标跨角色复用，且不改变普通引用语义。
6. 关系仓库现有创建、读取、审计、失效、严格删除和结构化关系写入入口足以承载新类型；不需要新增公开仓库 ABI。
7. 通用关系重挂当前会保持关系类型和顺序号但可更换端点。动作场景角色是不可变动作规格的一部分，不能沿通用重挂改变。

### 2.2 当前缺口

```text
没有可区分同一动作到同一场景的输入 / 输出角色关系类型。
关系 17 尚未进入枚举和关系仓库准入。
动作入口新写和读回仍使用普通引用槽 1 / 2。
未知关系负例仍固定为 17。
通用关系重挂尚未拒绝未来的动作场景角色关系。
```

## 4. 结构合同

### 4.1 枚举

```cpp
方法动作场景 = 17
```

数值 17 一旦进入代码、快照或后续恢复材料，不得重排、复用或改名复义。数值 18 及更大值继续作为当前未定义关系写前拒绝边界。

### 4.2 端点和角色

| 字段 | 合同 |
| --- | --- |
| 源节点 | `节点类型::方法`；领域数据操作还必须复核其为动作入口角色 |
| 目标节点 | `节点类型::场景` |
| 顺序号 1 | 输入场景角色 |
| 顺序号 2 | 输出场景角色 |
| 关系状态 | 有效时参与动作规格读取；已失效 / 已删除时只按既有审计合同读取 |
| 关系版本 | 创建为 1；失效、删除沿现有关系生命周期递增 |

关系仓库只承担稳定结构准入，不读取动作入口角色状态、稳定动作键或方法登记关系。上述业务语义仍归 `数据操作.需求任务方法`。

### 4.3 唯一性

关系 17 的有效唯一性固定为：

```text
方法动作场景 + 完整源动作入口句柄 + 顺序号
```

因此：

```text
动作 A -> 场景 X，槽 1：允许。
动作 A -> 场景 X，槽 2：允许。
动作 A -> 场景 X，槽 1 再次创建：拒绝。
动作 A -> 场景 Y，槽 1 再次创建：拒绝。
动作 B -> 场景 X，槽 1：允许。
```

唯一性检查必须在关系仓库写锁内完成。不得先写第二条再扫描修复。

## 5. 仓库入口设计

### 5.1 创建

现有 `创建关系` 和 `结构化创建关系` 继续作为统一入口，不新增关系 17 专用公开函数。关系仓库私有准入增加：

```text
关系类型上限追加到 方法动作场景 = 17。
顺序号只能是 1 或 2。
源节点当前类型必须是方法。
目标节点当前类型必须是场景。
同类型、同完整源句柄、同顺序号已有有效关系时拒绝。
```

创建成功后仍返回普通完整 `关系句柄`。调用方必须按句柄、版本、状态、类型、端点和顺序号读回，不得只用“存在某目标”证明角色完整。

### 5.2 读取和存在判断

现有按源和关系类型读取记录组足够：

```text
获取关系记录组(动作入口, 方法动作场景)
```

领域数据操作层必须确认：

```text
槽 1 恰好一条。
槽 2 恰好一条。
没有其它顺序号。
两个关系句柄、版本和端点均当前有效。
两个目标都为场景；两个目标允许相同。
```

通用 `存在关系(类型, 源, 目标)` 只能回答端点关系是否存在，不能区分输入或输出角色。机器逻辑不得用它替代角色记录组读回。

### 5.3 失效、删除和审计

关系 17 复用既有关系生命周期：

```text
失效后普通读取不可见，显式审计可读已失效当前版本。
删除后普通读取不可见，严格删除继续服务未提交写集收口。
旧关系失效或删除后，新的动作入口规格可使用新身份和新关系。
```

本文不新增恢复已失效关系或原地复活能力。

### 5.4 重挂

普通和结构化通用关系重挂都必须对关系 17 写前拒绝。原因：顺序号是动作规格角色，原地改端点会在不改变动作入口身份的情况下改变已发布规格。

如果后续需要变更动作输入或输出场景，必须先设计动作入口规格换代，创建新的动作入口身份并保留旧身份审计；不得在 #279 中临时实现。

## 6. 数据操作设计

### 6.1 新写入

`数据操作.需求任务方法::创建方法规格共用` 在动作入口分支中改为：

```text
写入动作入口身份、稳定动作键、角色状态和入口状态
-> 方法动作场景(新动作入口, 输入场景, 1)
-> 方法动作场景(新动作入口, 输出场景, 2)
-> 按完整关系证据分别读回
-> 完整规格匹配后请求提交
```

即使输入场景与输出场景相同，两条关系也必须分别存在，且关系句柄不同、角色顺序号不同。

### 6.2 新读回

动作入口读取必须同时扫描关系 17 和旧普通引用动作场景槽；只有确认关系 17 完全不存在时，才能进入旧结构只读兼容。出现以下任一情况必须返回内部不一致：

```text
只有槽 1 或只有槽 2。
同一槽多条有效关系。
出现顺序号 1 / 2 之外的关系 17。
目标不是当前有效场景。
关系 17 与旧动作普通引用槽混存。
```

完整读回继续填充现有值式 `输入场景关系` 和 `输出场景关系` 字段，不增加原始事务能力或仓库能力到业务服务公开面。

### 6.3 旧结构只读兼容

仅当动作入口完全没有有效关系 17 时，读取器可以按现行普通 `引用` 槽 1 / 2 读取旧结构。兼容路径必须满足：

```text
两个槽各恰好一条。
目标均为当前有效场景。
不存在关系 17。
只返回只读材料，不写关系 17，不删除旧引用，不建立双写。
```

普通引用现有唯一性决定旧结构无法表达同一目标跨两个角色，因此旧结构不得满足 #224 的 TE-D6 同场景双角色要求。#224 只能消费关系 17 的完整新结构。

## 7. 幂等、并发和失败收口

1. 同主键同完整动作入口规格命中已发布结构时返回精确幂等读回，不新增关系。
2. 同主键但输入 / 输出场景、稳定动作键、角色状态或入口状态不同，返回幂等冲突。
3. 同一动作源的槽 1 / 2 在关系仓库写锁内分别唯一；并发同规格只有一个提交者，其余经完整读回收口为幂等。
4. 数据操作在创建动作入口身份后发生任一写入或读回非预期，属于追根因解决；不得映射成普通输入拒绝。
5. 写入会话必须逆序撤销关系、索引、节点和主信息；失败后节点、关系和主键数量以及可读动作入口结构回到前态。
6. 无法用读回和数量快照证明无可读半结构时，停止 #279，登记待确认问题，不登记 785。

## 8. 调用路径

### 8.1 创建

```text
方法业务服务
-> 数据操作.需求任务方法
-> 结构写入执行器 / 不透明会话
-> 关系仓库结构化创建关系 17
-> 同会话完整读回
-> 请求提交
-> 方法规格值式结果
```

### 8.2 读取

```text
任务或方法业务服务
-> 数据操作.需求任务方法
-> 共享许可内按源读取关系 17 记录组
-> 角色槽、句柄、版本、状态和端点复核
-> 动作桥或方法规格值式材料
```
