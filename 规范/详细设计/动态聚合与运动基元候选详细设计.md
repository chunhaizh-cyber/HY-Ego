本详细设计只覆盖 FS-02-G 动态聚合和运动基元候选边界，不实现动态聚合算法，不实现运动基元识别算法，不接 D455、体素、外设、控制面板、SQL、WebView2 或线程。

## 2. 定位

动态聚合候选是对一组实例动态、状态变化片段和二次特征候选的只读归集材料。运动基元候选是动态聚合候选中的动作或变化片段候选，不是稳定动作结论、不是方法成功事实、不是因果结论。

候选只能作为后续专项输入：

```text
二次特征候选。
方法候选。
因果模板候选。
显示只读材料。
```

候选不得直接写入：

```text
需求满足。
任务完成。
方法成功。
稳定因果结论。
抽象动态长期结论。
动作入口可执行事实。
```

## 3. 输入材料

候选输入第一轮只允许来自当前已确认结构：

| 输入材料 | 来源服务 | 必须复核 |
| --- | --- | --- |
| 实例动态节点 | 动态服务 | 节点有效、类型正确、场景有效、存在主体有效、发生时间戳存在 |
| 动态材料视图 | 动态服务 | 被改变目标、前值、后值、主体、场景、时间戳完整 |
| 状态变化片段 | 状态服务 + 动态服务 | 前后状态同主体、同场景、同目标 |
| 二次特征候选 | 二次特征服务 | 组成项可追溯到状态或动态 |
| 来源动作入口 | 动态服务 / 方法服务已确认材料 | 来源动作为空或为动作入口角色节点 |

第一轮禁止输入：

```text
外设采样。
D455 帧。
体素占据位。
控制面板事件流水。
SQL 行。
线程事件。
日志标签。
自然语言显示文本。
```

## 4. 候选结构

| 字段 | 含义 | 第一轮要求 |
| --- | --- | --- |
| 候选编号 | 运行期候选局部编号 | 只作候选身份，不作为长期机器事实 |
| 候选类型 | 动态聚合候选或运动基元候选 | 必填 |
| 来源场景 | 候选所属场景 | 必须有效 |
| 来源主体 | 候选主体存在 | 必须有效 |
| 来源动态集合 | 参与候选的动态节点集合 | 必须非空，全部同场景 |
| 起始时间戳 | 候选片段开始时间 | 必须来自来源动态 |
| 结束时间戳 | 候选片段结束时间 | 必须不早于起始时间戳 |
| 被改变目标集合 | 候选涉及目标 | 来自来源动态材料 |
| 前后值片段 | 候选涉及的值变化 | 来自来源动态材料 |
| 二次特征候选 | 可选组成候选 | 只作候选材料 |
| 候选评分 | 可选 Q10000 或后续评分 | 不裁决事实 |
| 候选版本 | 候选规则版本 | 规则变化时旧候选失效 |
| 生成原因 | 人读说明 | 不承载机器逻辑 |

候选结构第一轮默认只读、可丢弃、可重建。是否持久化必须后续确认。

## 5. 动态聚合候选边界

动态聚合候选允许：

```text
按同一场景、同一主体和时间顺序归集动态片段。
按被改变目标集合形成候选序列。
按来源动作入口形成动作相关候选。
按二次特征组成项形成复合变化候选。
```

动态聚合候选禁止：

```text
自动创建抽象动态。
自动创建二次特征结论。
自动创建方法候选事实。
自动创建因果模板。
把候选评分写入需求、任务或方法事实。
```

聚合缺材料时只返回候选不足，不写半结构。

## 6. 运动基元候选边界

运动基元候选只表示一段可能可复用的变化片段。

第一轮候选条件：

```text
来源动态集合非空。
所有来源动态同场景。
所有来源动态具有发生时间戳。
候选主体可明确。
候选目标集合可读取。
前后值片段可追溯。
```

运动基元候选不得解释为：

```text
稳定动作。
本能动作。
动作入口实现。
方法可执行。
外设动作。
体素运动。
线程任务。
```

如后续需要把候选转为方法、动作入口或二次特征，必须另建专项并确认写入方、读取方、生命周期、验证方式和禁止扩大声明。

## 7. 候选清空和缺失边界

候选清空必须满足：

```text
不删除来源动态。
不删除来源状态。
不删除二次特征节点。
不改变因果引用。
不改变需求、任务、方法状态。
不改变动作入口身份。
```

候选缺失时：

```text
返回空候选或候选不足。
不得补写动态。
不得补写状态。
不得伪造来源动作。
不得写日志或显示文本作为候选事实。
```

## 8. 入口拒绝矩阵

| 编号 | 拒绝项 | 处理 |
| --- | --- | --- |
| FS02-DYN-01 | 来源动态集合为空 | 返回空候选，不写结构 |
| FS02-DYN-02 | 来源动态无效或错误节点类型 | 拒绝该候选，不写结构 |
| FS02-DYN-03 | 来源动态缺场景 | 拒绝该候选，不写结构 |
| FS02-DYN-04 | 来源动态缺存在主体 | 拒绝该候选，不写结构 |
| FS02-DYN-05 | 来源动态缺发生时间戳 | 拒绝该候选，不写结构 |
| FS02-DYN-06 | 来源动态缺目标、前值或后值 | 返回候选不足，不写结构 |
| FS02-DYN-07 | 来源动态跨场景混合 | 拒绝聚合，不写结构 |
| FS02-DYN-08 | 来源动态时间顺序无法确定 | 返回候选不足，不写结构 |
| FS02-DYN-09 | 来源动作不是动作入口角色节点 | 拒绝动作相关候选 |
| FS02-DYN-10 | 候选请求包含外设、体素、SQL 或显示文本材料 | 拒绝候选，不解释材料 |

拒绝后必须保持节点、主信息、关系、索引、状态、动态、二次特征、因果、需求、任务和方法结构不变化。

## 9. 与后续服务的关系

| 后续方向 | 可接收内容 | 仍需门禁 |
| --- | --- | --- |
| 二次特征服务 | 来源动态集合、状态变化片段、候选评分 | 需确认二次特征写入规则 |
| 方法服务 | 运动基元候选作为方法候选材料 | 需确认方法候选召回和条件结果对设计 |
| 因果服务 | 动态聚合候选作为因果模板候选材料 | 需确认因果模板和查询候选设计 |
| 显示只读视图 | 候选摘要和来源材料 | 需确认只读 DTO / 视图边界 |

候选材料不得自动推进到上述服务。任何写入都必须由对应服务专项确认。
