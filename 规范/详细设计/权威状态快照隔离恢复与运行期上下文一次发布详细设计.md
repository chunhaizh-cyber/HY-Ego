本设计取代旧“四仓库局部快照即可恢复”和“一个 #226 同时完成生产所有权、冻结、编码、发布”的施工假定，把权威持久化和启动恢复拆成前置事实复核、生产所有权、冻结材料、编码发布与恢复链：

```text

RECOVERY-S1：严格解析外部快照，按指定身份构造隔离四仓库候选。
RECOVERY-S2：恢复特征值侧表、系统角色、索引所有者候选和领域登记，重建概念活动图。

RECOVERY-S4：系统性验证失败不变式、墓碑防复活、并发冻结和隔离恢复循环。
```

当前 `仓库快照服务` 只有布尔式拒绝壳，四仓库没有全状态导出 / 指定身份导入；运行期上下文、完整全新候选首次发布、唯一宿主和租约类型已经存在，但生产入口仍构造另一套旧仓库 / 服务栈。新运行期上下文中没有概念活动段 9 的当前所有者。本文除“当前事实”小节外均描述预期正式契约，不得写成当前实现事实。

### 9.2 S3C 显式启动策略

```text

```

### 9.3 S3A 恢复中断任务收口

线程和队列尚未构造时，在隔离任务服务中扫描正式当前生命周期入口：

```text
已创建、已承接、筹办中、等待中、待重筹办：保持当前结构。
排队中：以结构化恢复中断证据迁移到待重筹办。
执行中：以结构化结果未知证据迁移到等待中。
已完成、失败、取消：保持终态。
```

恢复中断证据绑定快照生成序号、冻结事务序号和原生命周期关系 / 版本，只作本次正式任务迁移准入材料。不得从旧执行请求、消息或回执恢复动作，不得把执行中任务直接判失败 / 完成，也不得自动重试未知外部副作用。

### 9.4 S3A 跨段完整性屏障

发布前串行复核：

```text

有效节点到主信息、有效关系端点和历史墓碑端点规则。
特征原始材料、槽位唯一当前值和类型版本。
系统角色必需集合、稳定键所有者和索引集合。
方法 / 需求 / 任务 / 命名 / 删除实际登记。
世界根、自我、主场景、两个根需求和概念四根。
概念生命周期、签名、活动图和活动版本。
任务恢复中断收口和全部瞬态队列为空。
```

屏障生成值式结果和不可复制、不可伪造、绑定候选 / 快照 / 事务身份的移动式发布许可，不持久化第二份业务事实。外部材料不满足规则是逻辑拒绝；当前代码对已验证候选给出互相矛盾结果是追根因。没有该许可，S3B 宿主不得发布。

### 9.5 S3B 首次发布与 S3C 启动顺序

已有上下文时逻辑拒绝，不销毁旧上下文，不进入热替换。发布后发现混合上下文、角色漂移或租约身份不一致属于追根因，停止线程启动。

### 9.6 S3D 独立自检承载

生产和自检模块固定为：

```text

上下文.运行期.ixx

自检.恢复总成.ixx
```

全部新 `.ixx` 必须按 `ClCompile + export module + import` 形成真模块。三个领域自检模块承载场景、并发、故障注入和逐项输出；恢复总成只聚合值式报告。`入口.cpp` 只允许最小上下文、一次总成调用和总结果，不得保留恢复领域 Axx 正文。生产模块不得 import 自检模块。

## 10. RECOVERY-S4 失败不变式与防复活

### 10.1 防复活

```text
删除节点 / 主信息 / 关系恢复为墓碑，普通读取为空，审计读取完整。
恢复后的下一编号 / 创建序号沿快照高水位继续，旧身份不复用。
失效关系保持失效，不因端点有效而重建。
历史关系指向墓碑端点不进入普通图。
索引旧键按所有者键保留策略处理，不静默释放或重绑。
退役概念保持退役，已失效支持保持失效，物理删除概念不进入活动图。
任务终态不复活，排队 / 执行只走恢复中断迁移。
```

### 10.2 并发与确定性

```text
持续写入方与快照请求交错：每份快照必须对应一个完整冻结点。
同一冻结材料用 1 / 多线程编码得到完全相同规范字节和摘要。
同一快照用 1 / 多线程解析得到完全相同值式候选。
至少 50 轮生成、严格读回、隔离恢复和屏障循环。
候选失败不发布，正式快照提交前失败不破坏旧文件。
```

首版不做进程强杀、断电注入、磁盘扇区损坏、多进程并发写、在线热替换或跨版本迁移。这些能力必须另建专项。

## 12. 非成功返回二分

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 前置未完成或实际接口未冻结 | 依赖门控返回 | 不改代码，退回设计或等待前置 |
| 请求路径 / 版本 / 序号无效 | 逻辑内返回 | 不冻结、不写文件 |
| 临时文件创建、空间、权限或同卷替换前失败 | 逻辑内返回 | 清理本次候选，旧快照保持 |
| 当前运行期冻结材料互相矛盾 | 追根因解决 | 释放许可，不编码、不发布 |
| 当前写入方的临时文件无法被当前解析器读回 | 追根因解决 | 不替换旧文件，停止依赖路径 |
| 外部文件魔数 / 版本 / 长度 / 校验 / 字段不支持 | 逻辑内返回 | 丢弃候选，不回退全新 |
| 外部快照跨段身份 / 所有者 / 角色冲突 | 逻辑内返回 | 丢弃整个隔离候选，不修复 |
| 已严格验证导入包无法按指定身份精确读回 | 追根因解决 | 不发布，不调用普通创建入口兜底 |
| 全新模式检测到恢复目标或策略含混 | 逻辑内返回 | 不初始化、不删除快照 |
| 宿主已有上下文 | 逻辑内返回 | 不热替换，旧上下文保持 |
| 发布后出现新旧仓库 / 服务混合 | 追根因解决 | 停止线程启动，不把混合上下文当成功 |
