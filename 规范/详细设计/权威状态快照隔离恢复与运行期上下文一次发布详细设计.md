## 1. 定位

本设计取代旧“四仓库局部快照即可恢复”的施工假定，把权威持久化和启动恢复拆成五段：

```text
PERSIST-S1：统一冻结全部权威材料、冻结外规范编码、原子发布快照文件。
RECOVERY-S1：严格解析外部快照，按指定身份构造隔离四仓库候选。
RECOVERY-S2：恢复特征值侧表、系统角色、索引所有者候选和领域登记，重建概念活动图。
RECOVERY-S3A：完成恢复中断任务收口、跨段屏障和不可伪造发布许可。
RECOVERY-S3B：把恢复候选适配到 #247/#248 最终同一运行期上下文类型，并消费同一空宿主首次发布合同。
RECOVERY-S3C：完成显式启动分支及发布后线程、SQL 审计和只读控制面板接线。
RECOVERY-S3D：把恢复领域自检迁入独立真模块并执行 S3 端到端验收。
RECOVERY-S4：系统性验证失败不变式、墓碑防复活、并发冻结和隔离恢复循环。
```

当前 `仓库快照服务` 只有布尔式拒绝壳，四仓库没有全状态导出 / 指定身份导入，入口没有隔离运行期上下文。本文描述的是预期正式契约，不得写成当前实现事实。

### 2.1 已验证当前事实

```text
节点、主信息和关系仓库保留删除 / 失效记录，但公开读取只面向当前有效句柄。
```

### 6.2 冻结材料

取得 #217 同域独占许可后，按确定顺序逐个所有者复制：

```text
主信息仓库全部记录、仓库编号和下个主信息编号。
节点仓库全部记录、仓库编号、下个节点编号和下个创建序号。
关系仓库全部记录、仓库编号和下个关系编号。
索引仓库全部键值，并与所有者声明集合做精确集合相等复核。
特征值服务 I64 版本与 Vec 完整记录。
系统角色清单。
领域登记候选。
概念登记、活动候选和活动版本。
```

每次只取得一个所有者内部锁并复制到预先定义的值式容器；在同域独占许可下，其它参与写入方不能改变材料。不得同时持有两个仓库锁，不得在锁内排序、编码、分配无界大字节、打开文件、写日志或弹窗。

所有普通权威写入口必须先参与 #217 事务域。实际接口复核发现任一写入口可绕过事务域时，#226 保持门控并退回设计 / 上游修订，不能生成“多数一致”的快照。

### 6.3 冻结读回

释放许可前只做线性复杂度、有界分配的内部一致性复核：

```text
全部段绑定同一事务域和冻结事务序号。
仓库编号、高水位非零且高于已有身份。
节点主信息身份可在本次主信息材料中定位。
索引键集合与所有者声明集合精确相等。
侧表节点身份、系统角色和登记身份均能在本次仓库材料中定位。
概念活动版本非零，候选身份均在本次材料中。
```

当前运行期内部不一致属于追根因；不得把它编码成外部坏快照再交给恢复拒绝。

### 6.4 冻结外编码与发布

冻结材料不可变后，独立段可以并行规范排序、编码和计算段校验。所有并行任务结束后，单线程按段类型拼装段表、跨段摘要和总头。

发布顺序：

```text
1. 以 CREATE_NEW 创建本次临时文件。
2. 写入完整候选并 FlushFileBuffers。
3. 用当前严格解析器读回临时文件，逐项比较长度、摘要和全部值式材料。
4. 用 MoveFileExW(REPLACE_EXISTING | WRITE_THROUGH) 同卷替换固定正式文件。
5. 最终严格读回正式文件并返回发布结果。
```

步骤 4 是文件提交点。提交点前路径 / 权限 / 空间 / 临时文件冲突等允许失败为逻辑内返回，旧正式快照保持。当前写入方生成的临时文件无法被当前解析器读回属于追根因。步骤 4 后最终读回不一致同样追根因；不以日志宣称发布成功。

## 7. RECOVERY-S1 严格解析与隔离四仓库

### 7.1 解析阶段

恢复请求只读打开指定正式文件，依次验证：

```text
魔数、总头 / 段表 / 快照版本和显式小端。
总长度、段表边界、段数量、类型顺序和唯一性。
段偏移不重叠、不溢出、不越界。
每段校验和跨段摘要。
每类记录定长头、变长数组长度和记录数量。
所有整数转换、长度乘法和内存预算。
```

只有总边界固定后才能并行解析独立段。解析结果是不可变值式 DTO，不构造当前仓库，不调用领域服务，不写日志修复。

### 7.2 隔离仓库候选

新增恢复专用静态构造或内部构造器，不能复用普通 `创建*` 入口猜身份。导入顺序：

```text
主信息全部记录与精确高水位。
节点全部记录、仓库编号、精确高水位和创建序号高水位。
关系全部记录、仓库编号和精确高水位。
索引候选暂存，不绑定。
```

高水位必须非零并严格大于段内最大已用编号 / 创建序号，但恢复值必须使用快照中的原值，不能重新计算为最大值加一。记录版本和状态按当前支持枚举严格映射，未知数值拒绝。

### 7.3 端点规则

有效关系：

```text
来源和目标必须对应当前有效节点的精确完整句柄。
节点类型组合必须满足当前关系类型准入。
普通父子必须单父、无自身、无环。
```

失效 / 删除历史关系：

```text
端点仓库编号和本地编号必须存在于节点全记录段。
关系保存的端点版本必须非零，且不得大于该身份当前墓碑 / 当前记录版本。
端点可以已删除，不能要求普通有效读取成功。
历史关系不得重新进入普通关系索引、父子树、概念活动图或任务当前态。
```

### 7.4 精确读回

每个恢复专用导入包都在隔离对象内逐记录审计读回。外部快照字段无效在构造前逻辑拒绝；已经通过当前解析 / 校验 / 导入前置的包无法精确读回，属于实现内部不一致并追根因。当前运行期始终不变化。
