## 1. 目标与权威边界

本设计把现行正式规范转换为可施工的自我内部治理主链：

```text
根特征缺口候选
-> 安全事件、因素、排除和有效未复发证据维护
-> 分层安全值被动维护
-> 因果精确深度、严格五组和任务执行权限投影
-> 需求生长、OR 路径选择与 AND 权重分配
-> 主需求决议和任务意图
-> 任务按稳定身份取得唯一筹办执行权
-> 六类筹办结果闭合
-> 方法冻结和执行
-> 实际状态 / 动态 / 因果回读
-> 任务完成、需求满足、价值结算分别裁决
-> 用途观察、学习候选和独立晋级
-> 后继筹办回合读取不可变新代际
```

正式依据为 0050、3100—3300、4010、4020、4040、4070、4110、4210、4220、4310—4330、5100—5130、5160、5200—5250、5300—5340、6100—6150、7130、8100、8200、8210、8300、8310。

本设计不以线程、日志、返回码、缓存、读取投影或当前旧代码承载机器事实。自检只能形成缺口候选；需求、任务、方法、状态、动态、因果和价值分别由所属领域写入。

## 3. 主链行为

### 3.1 安全维护、分层与权限投影

1. 自我线程每个有效治理循环调用一次正式安全维护入口，但只传入当前事实截止版本和单调实际时间。
2. `SAFETY-C1` 先消费已经发布的新安全事件、复发和因素变化，完成主动结算，再读取因素验证排除并按有效未复发证据裁决被动维护；线程轮次不参与数值。
3. 安全事实事务只原子发布安全值、状态 / 动态、维护账和当前关系。发布成功后，`SAFETY-C2` 从一致性快照计算精确因果深度、严格五组和任务权限值式投影。
4. 无有效安全因果路径形成“未归层”；因果环、坏端点、版本漂移或本应存在却缺失的安全层材料形成具名拒绝或缺口，不按名称猜层。
5. 权限为零不删除需求、任务或筹办材料，只排除关联任务参加正常执行竞争；安全事件登记、观察和机器事实写入继续。

### 3.2 根特征缺口到任务意图

1. 读取唯一自我、内部世界、根定义、实例槽和当前值。
2. 缺项、版本漂移或不完整读取时返回具名拒绝；不能先创建需求。
3. 自检只形成缺口候选，需求领域按目标状态去重后建立或复用需求。
4. 每条 OR 路径继承完整父权重 `W`；AND 只属于一个确定 OR。当前 OR 路径中的确定 AND 组有 `n` 个共同成立成员时，每个成员按 `W / n` 平分；不可整除余量只按 5110 要求随分配版本冻结的确定性余量规则处理。分配版本或余量规则缺失时拒绝发布分配结果，禁止把 `n` 解释为 AND 组数量，也禁止自定义比例或“规范化份额”。
5. 主需求决议只选择当前治理输入，不改变其它需求的正式状态。
6. 任务领域消费值式意图，建立单目标任务及完整初始特征。

### 3.3 唯一筹办执行权与六类闭合

并发隔离键固定为任务稳定身份。不同任务可并行；同一任务只有取得当前筹办占用的路径才能读取任务、召回、选择和冻结。未取得者在召回前返回正常并发拒绝。

筹办必须闭合为以下一类：

1. 可执行冻结；
2. 子目标承接；
3. 能力缺口承接；
4. 合法等待；
5. 目标已完成；
6. 授权失效或取消。

“无确认路径”只是路径查找状态，最终必须归入上述六类。无候选方法不得自动创建方法首。执行期发现冻结前已存在的方法结构缺口，属于筹办漏检和内部逻辑错误。

恢复时旧运行代次的占用和冻结不得继承为执行权。隔离恢复候选必须先证明旧运行代次已经终止，并证明前轮外部动作已停止或已有实际回读；只有证明成立后，才原子失效旧占用及旧冻结当前性，以新恢复代次和新筹办轮次重新取得执行权。若不能证明，只能形成带动作核验生产者和重新触发条件的合法等待，不得取得新执行冻结或重复外部动作。

### 3.4 执行、回读和三个独立裁决

方法执行只消费已冻结任务授权、输入和方法代际。动作完成报告不是任务完成事实；必须读取正式状态、动态及必要因果证据，与任务目标、允许结果和禁止结果比较。

顺序固定为：

```text
方法运行结果
-> 实际事实回读
-> 任务领域裁决是否完成
-> 需求领域裁决是否建立已满足记录
-> 4320 / 6100 对应的正式价值结算入口独立结算预算与份额
-> 组合形成下一轮只读输入
```

需求正式满足与价值结算不能生成混合持久记录。任一前置不满足均为逻辑内返回；前置通过后的领域写入、联合读回或最后发布异常属于内部错误。

### 4.1 SAFETY-C1—C2 / v1 外部合同

`SAFETY-C1 / v1` 与 `SAFETY-C2 / v1` 的唯一完整 DTO、算法、事务和排序合同由 `规范/详细设计/分层安全值被动变化与因果任务执行优先级详细设计.md` 冻结，#377 提供隔离源码候选，#359 负责真实接线：

| SELF 合同 | 合法消费 | 禁止 |
| --- | --- | --- |
| SELF-C1 / #354 | 读取安全层、未归层 / 结构缺口和权限来源，形成需求路径与任务意图 | 自行计算或修改安全值、阈值、深度和权限 |
| SELF-C2 / #355 | 把当前权限来源版本作为任务选择 / 冻结材料并在派发前重读 | 把安全权限解释为任务授权、筹办占用或执行冻结 |
| SELF-C4 / #357 | 把新事件、主动结算和因素排除请求提交给各自正式入口，再由 SAFETY-C1 维护 | 任务完成直接改安全值或权限 |
| SELF-C6 / #359 | 每轮调用安全维护、从当前快照形成权限并由任务管理只读消费 | 在线程或调度器复制算法、反写安全值或用消息优先级冒充权限 |

SELF-C3 和 SELF-C5 默认不写安全值或权限；方法和学习只能消费正式授权及已经发布的事实。

## 5. 并发、停止与恢复

- 需求去重键：自我身份 + 目标状态稳定键 + 规则版本。
- 任务筹办键：任务稳定身份；不是根需求或方法身份。
- 方法登记键：方法稳定身份 + 内容版本；同一版本不可覆盖。
- 学习晋级键：来源方法代际 + 确定观察组 + 候选稳定键。
- 方法代际可见性：记录发布代次和来源任务回合，并逐任务读取当前冻结快照。来源任务产生代际的同一回合及任何发布前已冻结回合本轮不可见，结束后必须重新取得筹办权并权威重读；发布后建立或当前没有活动旧冻结的新任务，在首次取得筹办权并权威重读时可按条件召回。禁止使用一个全局“最早筹办轮次”。
- 运行消费者键：运行期身份 + 自我身份；同一运行期只允许一个治理消费者。
- 停止先停止新轮次，再收口当前动作、回读、任务状态和筹办占用；不得留下可读半结构。
- 恢复先在隔离候选仓校验身份、版本、关系、类型化记录和业务不变量，再重建索引，全部成功后原子发布。

## 6. 验证矩阵

| 级别 | 验证 |
| --- | --- |
| 静态 | 六份合同模块唯一；公开接口不暴露线程、锁、仓库或令牌；无跨域写权 |
