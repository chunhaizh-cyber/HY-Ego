# 语素系统详细设计

编号：004-D1
日期：2026-07-05
状态：第一轮详细设计 / 第一轮小能力代码已实现并验证 / 系统级完成禁止宣称

## 1. 设计目标

本详细设计对应 `规范/语素系统规范.md`，把规范中的用途、功能、实现方法和对外服务落到函数级设计。

目标是按项目级语素系统规范建立新项目语素服务的结构入口。旧项目 `语素类` 只能作为参考来源，不能决定本设计边界：

```text
词面材料
-> 语素准入
-> 语素入口节点
-> 目标节点绑定
-> 候选查询 / 冲突候选
-> 领域服务复判
```

## 2. 规范对应关系

| 规范条款 | 详细设计承接 |
| --- | --- |
| 主要用途 | 语素服务提供入口身份、准入、绑定、候选和只读显示服务 |
| 功能范围 | 第一轮函数和后续候选函数分开设计 |
| 实现方法 | 使用节点、关系、索引和只读候选包，不复制旧语素树 |
| 对外服务 | 所有函数收束在 `语素服务` |
| 准入规则 | `判断文本是否可作为最小词单元`、`创建概念入口` |
| 查询规则 | 后续 `按名称入口查节点`、`生成查询冲突候选包` |
| 显示规则 | 后续 `生成节点显示标题集合` |

## 3. 能力实现分层

### 3.1 总设计能力

语素系统详细设计覆盖以下总能力：

```text
语素入口身份
最小词单元准入
概念入口
实例入口
别名关系
有限候选查询
查询冲突候选
只读显示标题
```

### 3.2 当前代码已实现能力

当前代码已实现：

```text
创建语素入口
判断文本是否可作为最小词单元
创建概念入口
读取入口绑定目标
读取入口概念追溯
语素入口同一
```

当前实现要求调用方先提供已存在的对应信息节点。对应信息节点现阶段只能是长期基础信息类节点、需求、任务或方法。长期基础信息类节点包括基础信息通用节点、存在、场景、特征、抽象状态、抽象动态、二次特征、因果引用。场景内临时实例状态和实例动态不得直接作为语素入口对应信息节点。语素服务在函数入口排除不允许的节点后创建语素类型节点，写入语素入口到对应信息节点的引用关系，并可选绑定主键索引。

当前实现已检查第一轮最小词单元准入，可创建概念入口并写入概念追溯关系。当前实现不生成候选查询结果。

当前入口拒绝路径已接入项目统一容错检查宏。默认构建下宏关闭，拒绝路径只返回无效句柄；启用 `HY_EGO_ENABLE_FAULT_TOLERANCE_CHECK` 后，拒绝路径必须日志记录并弹窗提醒，但日志和弹窗不得参与任何机器事实。

当前状态按四态制理解：

```text
语素入口、最小词单元准入、概念入口、绑定目标读取、概念追溯读取、入口同一判断：
规范已确认 / 代码已实现 / 验证已通过 / 系统级完成禁止宣称。
```

### 3.3 第一轮已实现能力

第一轮已实现并验证：

```text
语素入口身份
最小词单元准入
概念入口
入口绑定目标读取
入口概念追溯读取
语素入口同一判断
```

暂不实现：

```text

查询冲突候选包

短语组合语义解析
```

这些暂缓项必须另开小切片，不能混入第一轮建设。

## 4. 结构设计

第一轮优先不新增独立语素记录结构，先使用已有节点、主信息、关系和索引。

| 结构 | 用途 | 写入方 | 读取方 | 生命周期 | 验证方式 |
| --- | --- | --- | --- | --- | --- |
| 语素入口节点 | 表达已准入入口身份 | 语素服务 | 语素服务 / 领域服务 | 随节点仓库生命周期 | 创建后能用节点句柄读取 |
| 对应信息节点 | 语素入口必须先对应的内部信息节点 | 基础信息服务 / 需求服务 / 任务服务 / 方法服务 | 语素服务 / 领域服务 | 随节点仓库生命周期 | 函数入口只接受长期基础信息类节点、需求、任务或方法 |
| 语素入口到对应信息节点关系 | 表达入口已绑定内部信息 | 语素服务 | 语素服务 / 领域服务 | 随关系仓库生命周期 | 关系写入入口拒绝无效源节点或无效目标节点 |
| 语素入口到概念节点关系 | 表达概念追溯 | 语素服务 | 语素服务 / 世界服务 | 随关系仓库生命周期 | 能从入口回查概念节点 |
| 语素入口到目标节点关系 | 表达绑定目标 | 语素服务 | 语素服务 / 领域服务 | 随关系仓库生命周期 | 能从入口回查目标节点 |
| 词面索引 | 从注册种子定位入口候选 | 语素服务 | 语素服务 | 可重建索引 | 同一最小词单元重复查到同一候选 |

若实现时发现仅用现有结构不足，必须先回到规范确认是否允许新增 `语素入口记录` 或 `语素绑定记录`。

## 6. 第一轮准入流程

```text
输入词面材料
-> 判断文本是否可作为最小词单元
-> 确认已有对应信息节点
-> 对应信息节点必须是长期基础信息类节点、需求、任务或方法
-> 读取或创建语素入口节点并绑定对应信息节点
-> 绑定概念节点
-> 写入入口到概念节点引用关系
-> 返回语素入口节点句柄
```

失败分流：

```text
空文本 / 短语 / 组合系统标签 / 未理解文本
-> 不创建正式语素入口
-> 返回待理解状态或失败原因
```

没有对应信息节点时：

```text
不创建正式语素入口
不创建悬空语素入口等待后补
返回失败状态或无效句柄
```

第一版若没有待理解候选结构，只能返回失败状态，不得伪造已持久化待理解候选。

## 7. 概念入口流程

```text
输入：词面、概念节点句柄
-> 词面准入
-> 概念节点有效性检查
-> 创建或读取语素入口节点
-> 写入入口到概念节点引用关系
-> 返回入口句柄
```

约束：

```text
概念入口只证明语言入口和概念节点绑定。
概念入口不证明现实存在成立。
概念入口不创建需求、任务、方法运行事实。
```

## 8. 函数与规范一致性要求

每个函数实现前必须检查：

```text
是否能找到对应规范条款。
是否写清功能、用途和使用范围。
是否明确输入、输出、读写结构和验证方式。
是否避免裸字符串、显示文本、日志或 SQL 投影参与机器判断。
是否通过语素服务封口，避免外部裸写仓库。
```

找不到对应规范条款的函数不得实现，必须先回到规范层补充或修订。

## 9. 反例

```text
输入 "红色杯子"。
如果没有语言解析结果和内部结构绑定，不允许创建正式语素入口。
```

```text
输入 "杯子"，创建概念入口。
该入口只能证明词面入口绑定概念节点，不能证明现实中有杯子存在。
```

```text
两个节点同名 "杯子"。
语素服务不得静默选择第一个作为事实；查询能力未实现前必须返回缺口或暂缓。
```

## 10. 验收条件

第一轮语素系统代码实施前，必须先确认：

```text
是否只实现语素入口身份、准入和概念入口。
是否允许词面文本只作为注册种子和显示来源。
是否暂缓别名、查询、显示和冲突包。
是否暂缓 SQL / 控制面板投影。
是否采用新封口函数优先，不复制旧函数体。
```

第一轮代码实施后，至少验证：

```text
Debug x64 构建通过。
启用 `HY_EGO_ENABLE_FAULT_TOLERANCE_CHECK` 的 Debug x64 构建通过。
默认构建运行时，语素入口拒绝路径不写容错日志、不弹窗。
已有长期基础信息类节点可生成语素入口。
已有需求、任务或方法节点可生成语素入口。
特征值节点不能生成语素入口。
场景内临时实例状态和实例动态不能生成语素入口。
短语不能生成正式语素入口。
概念入口能回查概念节点。
语素入口不裁决需求、任务、方法或世界事实。
```
