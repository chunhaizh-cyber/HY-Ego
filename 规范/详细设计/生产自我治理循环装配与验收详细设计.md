## 1. 目标与边界

本设计冻结 #359 对生产运行期、自我治理消费者、停止恢复和端到端验收的唯一共享装配所有权。线程只调度，不能成为动作来源或业务事实写入方；全部变化仍通过 SELF-C1—C5 及所属领域服务完成。

## 2. 生产装配

装配顺序固定为：

```text
建立或恢复运行期候选
-> 校验唯一自我、内部世界、节点、关系、类型化记录和版本
-> 重建索引与只读投影
-> 若存在旧筹办占用 / 执行冻结历史，隔离保留且不继承执行权
-> 证明旧运行代次终止且前轮动作停止或已有实际回读
-> 能证明：失效旧当前性并形成待重筹办触发
-> 不能证明：形成合法等待、动作核验生产者和重新触发条件，禁止新执行
-> 原子发布运行期
-> 绑定唯一自我治理消费者
-> 登记 SELF-C1—C5 提供者和最小本能方法集
-> 启动治理轮次调度
```

正常启动与恢复发布互斥。入口、工程、filters、统一运行器、自我线程治理处理器接线和完整验收只归 #359，#354—#358 不得修改共享装配面。

## 3. 治理轮次状态机

1. 读取当前运行期和唯一自我。
2. 若上一轮外部动作未能证明停止或完成实际回读，返回合法等待。
3. 消费根特征缺口和上一轮输入，形成主需求决议。
4. 任务管理为不同任务并发调度；同任务由任务身份唯一筹办占用隔离。
5. 六类筹办结果分别进入执行、子目标、能力缺口、等待、已完成或取消路径。
6. 执行路径完成实际回读、任务验证、需求满足和价值结算。
7. 4330 的尝试未达成与最终已结算观察分栏进入统计，待结算排除；候选经独立晋级后记录发布代次和来源任务回合。发布前已冻结回合本轮不可见，旧回合结束后重新取得筹办权并权威重读可见；发布后建立或没有活动旧冻结的新任务，在首次取得筹办权并权威重读时可按条件召回。
8. 形成下一轮输入并提交轮次结果；等待下一事件或继续轮转。

## 4. 停止与恢复

### 4.1 停止

- 先拒绝新治理轮次和新筹办占用。
- 等待或取消尚未开始的调度材料。
- 已开始的外部动作必须完成实际回读或形成不能重入的合法等待证据。
- 失效当前运行代次的筹办占用，完成事实和索引读回。
- 原子发布停止结果；停止后任何线程不得继续写业务事实。

### 4.2 恢复

- 在隔离候选仓读取节点、稳定主键、关系、领域类型化记录、历史事实和版本。
- 校验唯一自我、内部世界成员、实例特征槽、任务历史、方法代际和结算分账。
- 检测旧筹办占用和旧执行冻结历史，隔离保留但不继承执行权。
- 证明旧运行代次已经终止，并证明前轮外部动作已停止或已有实际回读；证明成立后才原子失效旧占用和旧冻结当前性，形成待重筹办触发。
- 不能证明前轮动作收口时，形成合法等待、动作核验生产者和重新触发条件，禁止取得新执行冻结或重复外部动作。
- 重建索引、状态/动态列表投影和下一轮输入。
- 任一缺项不发布半结构；全部成功后以新恢复代次原子发布。

## 6. 端到端验收

| 验收层 | 必须证明 |
| --- | --- |
| 装配 | 生产运行期唯一发布，治理消费者唯一，SELF-C1—C5 真实提供者接通 |
| 最小方法 | 最小清单逐项登记、召回和实际回读；被动结算方法不参加主动召回 |
| 任务并发 | 同任务唯一筹办，不同任务并行；六类结果完整 |
| 事实闭环 | 方法结果不冒充事实；状态、动态、因果正式提交并读回 |
| 三类裁决 | 任务完成、需求满足、价值结算分别成立 |
| 学习 | 候选经独立授权发布新代际，后继回合可见 |
| 多轮 | 连续至少十轮，覆盖无缺口、合法等待、子目标、能力缺口和完成 |
| 生存 | 待机、唤醒、故障收口、停止和恢复后继续下一轮 |
| 恢复 | 旧占用失效，索引重建，半结构不可见，未确认外部动作不重复 |

构建、自检和真实运行只在 #359 代码计划授权后执行。显示、日志或线程退出不能替代验收。
