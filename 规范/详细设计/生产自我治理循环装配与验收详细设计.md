## 1. 目标与边界

本设计冻结 #359 对生产运行期、自我治理消费者、安全 / 服务维护与任务权限路由、休眠唤醒、停止恢复和端到端验收的唯一共享装配所有权。线程只调度，不能成为动作来源或业务事实写入方；全部变化仍通过 SELF-C1—C5、SAFETY-C1—C2、SERVICE-C1—C3 及所属领域服务完成。

## 2. 生产装配

装配顺序固定为：

```text
建立或恢复运行期候选
-> 校验唯一自我、内部世界、节点、关系、类型化记录和版本
-> 重建索引与只读投影
-> 若存在旧筹办占用 / 执行冻结历史，隔离保留且不继承执行权
-> 证明旧运行代次终止且前轮动作停止或已有实际回读
-> 能证明：失效旧当前性并形成待重筹办触发
-> 不能证明：形成合法等待、动作核验生产者和重新触发条件，禁止新执行
-> 原子发布运行期
-> 绑定唯一自我治理消费者
-> 登记 SELF-C1—C5、SAFETY-C1—C2、SERVICE-C1—C3 提供者和最小本能方法集
-> 绑定自我线程每轮安全 / 服务维护路由、休眠最小入口和任务管理权限只读路由
-> 启动治理轮次调度
```

正常启动与恢复发布互斥。入口、工程、filters、统一运行器、自我线程治理处理器、安全 / 服务维护与权限路由、任务管理权限消费接线和完整验收只归 #359，#354—#358、#377 与 #379 不得修改共享装配面。

## 3. 治理轮次状态机

1. 读取当前运行期和唯一自我。
2. 若上一轮外部动作未能证明停止或完成实际回读，返回合法等待。
3. 新事件、复发、因素变化和因素排除先由各自合法领域发布，主动安全结算由 6100 发布；随后只调用一次 SAFETY-C1 只读消费已发布结果，重算残余约束并按实际有效时间裁决被动维护。
4. 新的人类需求只调用 SERVICE-C1 `建立或读取服务合同`，把合同与 30% 预支独立原子发布；生产循环只调用 SERVICE-C2 `结算服务生存治理批次`，按事实序处理合同终态 / 70% 余款或关闭、准备补回、到期未满足事件和跨过的完整秒，禁止在聚合入口重复结算预支。同秒主动安全值变化优先，`A=0` 保持终止。
5. 安全与服务事实分别原子发布后，先裁决 `A=0/1/>1` 生存控制态：`A=0/1` 的普通安全 / 服务任务权限均为零；只有 `A>1` 才从一致快照形成 SAFETY-C2 与 SERVICE-C3 权限，安全来源组合两层权限，独立服务来源使用服务根权限，安全兼服务来源不得重复取得服务权限。
6. 消费根特征缺口和上一轮输入，形成主需求决议；安全需求只读取正式因果层和权限来源，服务需求只读取有效合同和服务来源。
7. 任务管理先重读任务与权限当前性，再按确定性比较选择；零权限任务保留但不参加正常竞争。不同任务可并发，同任务由任务身份唯一筹办占用隔离。
8. 六类筹办结果分别进入执行、子目标、能力缺口、等待、已完成或取消路径。
9. 执行路径完成实际回读、任务验证、需求满足和分账结算；任务完成不得直接改安全值或服务值。
10. 4330 的尝试未达成与最终已结算观察分栏进入统计，待结算排除；候选经独立晋级后记录发布代次和来源任务回合。发布前已冻结回合本轮不可见，旧回合结束后重新取得筹办权并权威重读可见；发布后建立或没有活动旧冻结的新任务，在首次取得筹办权并权威重读时可按条件召回。
11. 形成下一轮输入并提交轮次结果；等待下一事件、进入休眠或继续轮转。

## 4. 停止与恢复

### 4.1 停止

- 先拒绝新治理轮次和新筹办占用。
- 等待或取消尚未开始的调度材料。
- 已开始的外部动作必须完成实际回读或形成不能重入的合法等待证据。
- 失效当前运行代次的筹办占用，完成事实和索引读回。
- 原子发布停止结果；停止后任何线程不得继续写业务事实。

### 4.2 恢复

- 在隔离候选仓读取节点、稳定主键、关系、领域类型化记录、分层安全值、事件 / 因素 / 排除、服务合同、结算阶段、准备归属、到期未满足事件、衰减段、服务值、生存安全根值、时间纪元、已消费证据游标、历史事实和版本。
- 校验唯一自我、内部世界成员、实例特征槽、任务历史、方法代际和结算分账。
- 检测旧筹办占用和旧执行冻结历史，隔离保留但不继承执行权。
- 证明旧运行代次已经终止，并证明前轮外部动作已停止或已有实际回读；证明成立后才原子失效旧占用和旧冻结当前性，形成待重筹办触发。
- 不能证明前轮动作收口时，形成合法等待、动作核验生产者和重新触发条件，禁止取得新执行冻结或重复外部动作。
- 重建索引、状态 / 动态列表、安全因果层、安全 / 服务权限只读投影以及下一轮输入；单调未复发时间和服务完整秒不得跨运行代次无证据续算。
- 任一缺项不发布半结构；全部成功后以新恢复代次原子发布。

## 6. 端到端验收

| 验收层 | 必须证明 |
| --- | --- |
| 装配 | 生产运行期唯一发布，治理消费者唯一，SELF-C1—C5 真实提供者接通 |
| 安全提供者 | SAFETY-C1—C2 真实提供者接通；安全事实先原子发布，权限只作当前快照值式投影 |
| 安全维护 | 新事件优先、有效未复发时间按实际证据累计、低位 / 高位回归边界和恢复不跨代次拼接 |
| 安全调度 | 严格五组四道门守恒、组 5 完整比较、零权限不派发且不删除任务 |
| 服务合同 | 唯一合同、默认 / 显式有效期、30% / 70%、到期未满足事件和准备补回 |
| 服务维护 | 完整秒批量等价、最长未满足、30 天门禁、有效活动最低 1 和可审计衰减段 |
| 生存控制 | `A=0/1/>1`、服务调节根回归、真实安全事件优先及服务值不改分层安全值 |
| 根权限 | `A<=L` 服务零权限、开区间释放、`A>=H` 安全 / 服务 1% / 99%，安全兼服务不重复 |
| 最小方法 | 最小清单逐项登记、召回和实际回读；被动结算方法不参加主动召回 |
| 任务并发 | 同任务唯一筹办，不同任务并行；六类结果完整 |
| 事实闭环 | 方法结果不冒充事实；状态、动态、因果正式提交并读回 |
| 三类裁决 | 任务完成、需求满足、价值结算分别成立 |
| 学习 | 候选经独立授权发布新代际，后继回合可见 |
| 多轮 | 连续至少十轮，覆盖无缺口、合法等待、子目标、能力缺口和完成 |
| 生存 | 服务值归零休眠、有效人类需求合同预支后安全门禁唤醒、待机、故障收口、停止和恢复后继续下一轮 |
| 恢复 | 旧占用失效，索引重建，半结构不可见，未确认外部动作不重复 |

构建、自检和真实运行只在 #359 代码计划授权后执行。显示、日志或线程退出不能替代验收。
