# 任务结果同义并发结算版本漂移幂等归并详细设计

日期：2026-07-19

状态：JY-477 / #314 待执行 / 纯窄纠偏

## 1. 定位

本设计补充 `任务结果完成与需求独立结算详细设计.md` 的并发幂等合同。它只修正需求结算数据操作在无写入竞争退出时的当前态归并，不改变任务结果回执、任务完成、需求结算机器结构、服务公开 ABI 或待结算提示的非权威性质。

## 2. 当前事实与根因

1. `提交需求结算` 写前已有正式结算时，会完整比较来源任务、实际状态、动态、因果、结论、幂等状态和时间，返回幂等读回或冲突。
2. 写前未结算、进入会话后若需求材料已被竞争者推进，代码把收口标为 `版本漂移`。
3. 当前 `版本漂移` 在重新读取需求之前直接返回；`并发占用` 却会重新读取正式结算并归并为幂等或冲突。
4. 因此同义并发调用是否得到幂等读回，取决于竞争者是在会话材料复核前还是状态型归属查询后发布。
5. #312 在冻结候选第 17 轮直接观察 TASK-RESULT-S1 A14/A18 失败；#313 累计 60 个完整样本未复现。二者共同证明现象具有调度敏感性，但未复现不证明稳定。

## 3. 默认裁决

| 编号 | 裁决 |
| --- | --- |
| TRC-D1 | `版本漂移` 不自动成功；只有读回到完整同义正式结算时才归并为 `幂等读回`。 |
| TRC-D2 | `版本漂移` 与 `并发占用` 共用一次权威需求、正式结算和来源任务读回。 |
| TRC-D3 | 当前仍未结算时保留 `版本漂移`；不在数据操作层循环、休眠或重新提交。 |
| TRC-D4 | 当前已有异义正式结算时返回 `幂等冲突`，不得覆盖或回滚。 |
| TRC-D5 | 当前多归属、缺字段、来源任务矛盾或写后不完整继续归为 `内部不一致`，走追根因解决。 |
| TRC-D6 | 不新增字段、枚举、模块、阶段、线程、锁、日志状态或公开入口。 |

## 4. 实现状态机

```text
会话内部不一致 -> 内部不一致
会话入口拒绝 -> 入口拒绝
会话未进入 -> 按结构失败映射
会话已请求提交 -> 当前态完整同义且结构已提交 -> 已提交
会话并发占用或版本漂移 -> 读取当前需求与来源任务
  -> 当前无正式结算且原因为版本漂移 -> 版本漂移
  -> 当前完整同义 -> 幂等读回
  -> 当前完整异义 -> 幂等冲突
  -> 当前结构不完整 -> 内部不一致
```

`正式结算匹配` 继续作为唯一完整同义比较器，不复制字段比较，不用哈希、时间戳或日志代替权威结构。

## 6. 非成功返回

版本漂移后当前仍未结算是逻辑内返回；完整同义正式结算是逻辑内完成；异义正式结算是逻辑内冲突；结构不完整、写后不一致和同版本矛盾是追根因解决。结构是否变化不能单独裁决分类。

## 7. 验收

1. SERVICE-DATA-S5 同义结算 100 路精确为 1 个 `已提交`、99 个 `幂等读回`、0 个 `版本漂移`、0 个其它状态。
2. 异义并发仍精确形成一个权威结算、同义幂等和异义冲突，不扩大成功集合。
3. TASK-RESULT-S1 A14/A18、Debug/Release 完整自检和 Debug 连续 20/20 通过。
4. 数据操作公开 ABI、服务文件、组合器、入口、工程、阶段 610/760 和机器结构差异为 0。

## 8. 完成边界

完成只证明同义并发需求结算的无写入版本漂移可按权威当前态归并，消除该分支的调度依赖。它不证明所有并发路径稳定，不直接完成 #310/#305，不实现持久化重试、恢复或迁移完成。
