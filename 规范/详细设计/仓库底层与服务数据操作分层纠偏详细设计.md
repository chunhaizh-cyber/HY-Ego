# 仓库底层与服务数据操作分层纠偏详细设计

日期：2026-07-13

状态：正式详细设计 / JY-316 / 代码尚未实施

## 1. 目标

在保留统一四仓库和结构事务安全语义的前提下，建立以下稳定分层：

```text
协调层：签发 / 释放许可，形成事务上下文。
仓库层：验证令牌、管理本仓库锁、维护通用结构不变量。
服务数据操作层：隐藏仓库和令牌，完成同一许可内精确多仓库提交。
领域服务层：业务准入、业务不变量、幂等和成功 / 失败语义。
应用 / 领域组合器：定义跨服务一次操作的事务边界。
```

本设计不把业务语义下沉仓库，不立即删除兼容入口，不执行暂停中的 `#214`，也不先做性能微调。

## 2. 当前偏差

1. 至少 15 个领域文件出现原始结构令牌，生产概念命名路由直接取得许可并向服务传递令牌。
2. 至少 21 个领域文件直接持有四仓库引用；业务、事务和结构写入职责混合。
3. 存在服务同时有无令牌写、带令牌候选写；场景服务只有无令牌写。
4. `创建虚拟存在` 和 `世界服务::创建场景存在` 的中途关系失败不能统一证明无残留。
5. 当前性能链会测量旧分层路径，应在架构纠偏完成后重建基线。

文本命中数量只作为 S0 定向证据，#262 仍需冻结逐函数 / 逐调用点矩阵。

## 3. 四仓库详细合同

### 3.1 主信息仓库

用途：主信息身份、生命周期、拓扑锚点和值容器。

入口必须拒绝错误仓库、编号 0、版本不符、无效状态和越权会话。第一轮不新增仓库记录级“未发布”状态；主信息候选创建后记录已经结构有效，候选能力和完整句柄只能留在同一独占写入会话的数据操作层内。确认只关闭撤销能力，不修改记录有效性。节点发布依赖主信息当前有效。

### 3.2 节点仓库

用途：节点身份、类型、主信息锚点、创建序号和生命周期。

创建前复核主信息当前有效；发布后节点与主信息必须双向一致。节点类型只做通用枚举准入，仓库不得判断“需求是否应该创建任务”等业务条件。

### 3.3 关系仓库

用途：权威树图关系。创建、失效、重挂和删除必须复核完整当前关系句柄及有效端点。普通父子单父 / 防环属于已确认通用结构约束；其它关系的业务唯一性由领域服务形成类型化请求。

关系二级索引后续仍位于关系仓库私有实现，只保存关系编号候选，并与关系表同提交；本轮架构纠偏不提前实现。第一轮也不新增通用关系候选类型，写入会话记录本次关系精确句柄并在最后发布前逆序撤销。

### 3.4 索引仓库

用途：索引记录和入口候选。索引命中后必须复核节点完整句柄、类型、生命周期和必要关系。索引不得成为最后发布事实，除非领域详细设计明确索引记录本身就是待发布对象；即使如此，业务事实仍需权威结构复核。第一轮不新增索引候选类型，写入会话记录本次绑定键并负责精确逆序撤销。

### 3.5 普通删除兼容裁决

主信息 / 节点普通删除已验证独占令牌；关系 / 索引现有删除入口仍只验证共享路径。`#263` 不直接改义旧签名，而是新增统一结构结果的独占严格删除入口，供 `#264` 后新数据操作层使用。旧关系 / 索引共享删除只标记为兼容清理入口，等待 `#271` 按调用点收口。

## 4. 统一操作结果

仓库和数据操作层使用结构化结果区分：

```text
已提交
幂等读回
入口拒绝
许可拒绝
版本漂移
候选已撤销
候选已确认
内部不一致
```

公开业务服务把这些结构结果映射为本领域结果，但不得暴露许可、锁或仓库内部错误细节。入口拒绝和业务失败属于逻辑内返回；入口通过后的内部不一致必须追根因。

## 5. 不透明写入会话

### 5.1 类型

新增真模块：

```text
海中鱼巣/核心/会话.结构写入.ixx
模块：海中鱼巣.核心.会话.结构写入
```

导出不可默认构造、不可复制、可移动的 `结构写入会话` 只读能力面。原始 `结构事务许可`、`结构事务令牌`、仓库引用和释放函数全部为模块私有。

### 5.2 执行器

新增真模块：

```text
海中鱼巣/核心/执行器.结构写入.ixx
模块：海中鱼巣.核心.执行器.结构写入
```

执行器持有结构事务接线，只接受类型化操作回调。写入执行器只能取得一次独占许可、构造写入会话并保证释放；共享许可只允许用于独立只读复核，不得构造写入会话。领域服务、线程、消息和组合器不能读取令牌。

### 5.3 数据操作访问

只有计划登记的 `数据操作.*` 模块可以调用会话的仓库操作能力。实现优先采用会话成员的窄结构方法，不导出“读取原始令牌”逃生口。会话内部维护本次主信息 / 节点候选、关系句柄和索引键的精确写集；关系 / 索引不建立候选类型。仓库继续保留原始令牌 ABI 作为内部兼容层，待所有调用迁移后再裁决可见性。

## 6. 服务专用数据操作层

每个服务族建立独立真模块，职责为：

```text
接收不透明会话
复核通用结构当前性
创建主信息 / 节点未发布候选
创建关系 / 索引
按精确句柄逆序撤销
执行最后发布和候选确认
返回值式结构结果
```

数据操作层不接收线程消息，不决定业务策略，不输出日志作为结果，不形成跨领域反向依赖。

## 7. 领域服务公开面

新业务服务真模块只导出：

```text
业务请求数据
业务结果数据
无原始令牌的业务入口
必要的值式只读入口
```

服务公开入口不得出现 `结构事务令牌`、`结构事务许可`、仓库引用或锁类型。业务服务可以通过只读端口获得值式当前材料，并在数据操作提交前由同一操作再次复核；许可内复核由数据操作层完成，业务语义仍由服务给出的类型化规则限定。

## 8. 跨服务组合器

组合器用于一个业务操作涉及多个服务时固定顺序和成功语义。组合器调用结构写入执行器，但只提供业务操作闭包，不读取会话内部令牌。参与服务通过内部类型化操作端口与数据操作层协作，不在公开入口之间传递会话。

第一轮组合器：`组合.存在场景.ixx`。后续按状态动态、特征语义、需求任务方法和运行期业务操作建立独立模块。

## 9. 存在场景垂直样例

### 9.1 新模块

| 文件 | 类型 | 公开职责 |
| --- | --- | --- |
| `领域/数据操作.存在场景.ixx` | 生产真模块 | 同会话执行存在 / 场景结构写入和精确撤销 |
| `领域/服务.存在.ixx` | 生产真模块 | 实际存在 / 虚拟存在业务准入和结果 |
| `领域/服务.场景.ixx` | 生产真模块 | 场景创建 / 接纳业务准入和结果 |
| `领域/组合.存在场景.ixx` | 生产真模块 | 创建并接纳的一次事务边界 |
| `领域/自检.存在场景分层.ixx` | 自检真模块 | 入口、失败注入、并发和无残留验收 |

既有 `存在服务.h`、`场景服务.h`、`世界服务.h` 保留为兼容入口；首轮只把明确调用点接到新服务，不直接删除旧方法。

### 9.2 实际存在

业务服务检查幂等键和请求形状；数据操作层创建主信息候选和存在节点候选，读回后以节点身份确认作为发布点，再确认候选。任一步失败逆序撤销。

### 9.3 虚拟存在

业务服务要求来源为当前需求、任务或方法。数据操作层同会话复核来源，创建主信息 / 存在候选和 `存在 -> 来源` 引用；来源关系是最后发布结构。关系失败不得留下存在。

### 9.4 场景

创建主信息候选和场景节点候选，读回场景类型和主信息锚点后确认。场景服务不直接持仓库。

### 9.5 接纳与组合创建

接纳已有存在时，归属关系是唯一写入和发布结构。创建并接纳时，存在候选、可选场景候选和归属关系必须处于同一会话；归属关系最后发布。失败后四仓库数量和可读结构回到前态。

## 10. 服务迁移批次

| 顺序 | 服务族 | 主要原因 |
| --- | --- | --- |
| 1 | 存在 / 场景 / 世界聚合 | 最小主信息、节点、关系垂直样例 |
| 2 | 状态 / 动态 | 具备时间戳、场景归属、前后值和候选撤销 |
| 3 | 特征值 / 特征 / 二次特征 | 特征值只服务特征系统，需固定内部数据操作边界 |
| 4 | 语素 / 概念 / 因果 / 世界聚合余项 | 多关系、生命周期、轻量因果和跨基础信息读取 |
| 5 | 需求 / 任务 / 方法 / 方法召回 | 依赖前述基础服务，业务授权和状态机复杂度最高 |
| 6 | 初始化 / 自我治理 / 概念命名 / 运行期组合器 | 移除生产路由取得并传递原始令牌 |
| 7 | 兼容路径收口 | 生产调用点归零后再私有化、弃用或删除 |

### 10.1 后续专项材料门禁

本详细设计足以直接支撑 `#263` 仓库合同、`#264` 会话执行器和已有专项流程图覆盖的 `#265` 存在场景样例。它对状态动态、特征体系、语义基础、需求任务方法和运行期组合器只给出服务族顺序，不替代各服务族代码逻辑流程图和详细设计。因此 `#266-#270` 当前只保留计划骨架，专项三件套形成并与 #262 调用点矩阵一致前不得执行；`#271` 同步等待这些正式产物和精确兼容入口清单。

现有 `#214/#215/#224/#225/#231-#235` 的原始令牌所有权合同，以及 `#248-#250` 与 `#270` 的运行期装配重叠，必须在对应专项设计中明确修订、归并或替代，不能靠依赖顺序默认化解。

## 11. 双路径治理矩阵

#262 必须为每个写入口登记：

```text
文件 / 函数
调用方
生产 / 自检 / 兼容 / 禁止
无令牌 / 带令牌 / 候选
当前许可所有者
当前最后发布点
当前失败清理
目标服务族
替代计划
删除门禁
```

在替代调用点、回归和失败注入没有全部通过前，兼容入口只能标记，不能删除。

## 12. 锁与许可

写入执行器每次业务操作只取得一次独占结构许可。仓库仍按自身锁管理短临界区，不透明会话不允许锁升级。需要同时持有多个仓库锁时继续遵守 `节点 -> 主信息 -> 关系 -> 索引`；优先用全域独占许可下的逐仓库短锁，禁止服务持锁调用上层服务。共享许可仅供只读执行，不得进入本设计的写入会话。

## 13. 失败收口

```text
许可前拒绝：结构不变化。
候选创建失败：撤销本次已建候选。
关系 / 索引失败：先删本次关系 / 索引，再撤销身份候选。
最后发布失败：逆序撤销全部本次未发布结构。
最后发布后读回异常：追根因停止，不静默回滚已发布事实。
撤销不能证明数量和可读结构回到前态：追根因停止后继计划。
```

## 14. 兼容与 ABI

四仓库带令牌 ABI 第一轮保留，作为写入会话内部适配目标。领域服务带令牌入口、无令牌写入口和服务候选类型按调用矩阵逐批迁移。不得一次性全仓删除，不得让新模块通过 `读取令牌()` 重新泄漏原始令牌。
