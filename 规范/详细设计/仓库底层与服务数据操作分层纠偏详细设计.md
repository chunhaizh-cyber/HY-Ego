# 仓库底层与服务数据操作分层纠偏详细设计

日期：2026-07-13

状态：正式详细设计 / JY-316-JY-320 / #263-#265 已实施验证 / #272 待执行 / #266 依赖门控

## 1. 目标

在保留统一四仓库和结构事务安全语义的前提下，建立以下稳定分层：

```text
协调层：签发 / 释放许可，形成事务上下文。
仓库层：验证令牌、管理本仓库锁、维护通用结构不变量。
服务数据操作层：隐藏仓库和令牌，完成同一许可内精确多仓库提交。
领域服务层：业务准入、业务不变量、幂等和成功 / 失败语义。
应用 / 领域组合器：定义跨服务一次操作的事务边界。
```

本设计不把业务语义下沉仓库，不立即删除兼容入口，不执行暂停中的 `#214`，也不先做性能微调。

## 2. 当前偏差

1. 至少 15 个领域文件出现原始结构令牌，生产概念命名路由直接取得许可并向服务传递令牌。
2. 至少 21 个领域文件直接持有四仓库引用；业务、事务和结构写入职责混合。
3. 存在服务同时有无令牌写、带令牌候选写；场景服务只有无令牌写。
4. `创建虚拟存在` 和 `世界服务::创建场景存在` 的中途关系失败不能统一证明无残留。
5. 当前性能链会测量旧分层路径，应在架构纠偏完成后重建基线。

文本命中数量只作为 S0 定向证据，#262 仍需冻结逐函数 / 逐调用点矩阵。

## 3. 四仓库详细合同

### 3.1 主信息仓库

用途：主信息身份、生命周期、拓扑锚点和值容器。

入口必须拒绝错误仓库、编号 0、版本不符、无效状态和越权会话。第一轮不新增仓库记录级“未发布”状态；主信息候选创建后记录已经结构有效，候选能力和完整句柄只能留在同一独占写入会话的数据操作层内。确认只关闭撤销能力，不修改记录有效性。节点发布依赖主信息当前有效。

### 3.2 节点仓库

用途：节点身份、类型、主信息锚点、创建序号和生命周期。

创建前复核主信息当前有效；发布后节点与主信息必须双向一致。节点类型只做通用枚举准入，仓库不得判断“需求是否应该创建任务”等业务条件。

### 3.3 关系仓库

用途：权威树图关系。创建、失效、重挂和删除必须复核完整当前关系句柄及有效端点。普通父子单父 / 防环属于已确认通用结构约束；其它关系的业务唯一性由领域服务形成类型化请求。

关系二级索引后续仍位于关系仓库私有实现，只保存关系编号候选，并与关系表同提交；本轮架构纠偏不提前实现。第一轮也不新增通用关系候选类型，写入会话记录本次关系精确句柄并在最后发布前逆序撤销。

### 3.4 索引仓库

用途：索引记录和入口候选。索引命中后必须复核节点完整句柄、类型、生命周期和必要关系。索引不得成为最后发布事实，除非领域详细设计明确索引记录本身就是待发布对象；即使如此，业务事实仍需权威结构复核。第一轮不新增索引候选类型，写入会话按当前 `严格删除主键(主键, 预期节点, 令牌)` 真实签名记录每次“主键 + 预期节点完整句柄”绑定对并负责精确逆序撤销。

### 3.5 普通删除兼容裁决

主信息 / 节点普通删除已验证独占令牌；关系 / 索引现有删除入口仍只验证共享路径。`#263` 不直接改义旧签名，而是新增统一结构结果的独占严格删除入口，供 `#264` 后新数据操作层使用。旧关系 / 索引共享删除只标记为兼容清理入口，等待 `#271` 按调用点收口。

## 4. 统一操作结果

仓库和数据操作层使用结构化结果区分：

```text
已提交
幂等读回
入口拒绝
许可拒绝
版本漂移
候选已撤销
候选已确认
内部不一致
```

公开业务服务把这些结构结果映射为本领域结果，但不得暴露许可、锁或仓库内部错误细节。入口拒绝和业务失败属于逻辑内返回；入口通过后的内部不一致必须追根因。

## 5. 不透明写入会话

### 5.1 类型

新增真模块：

```text
海中鱼巣/核心/会话.结构写入.ixx
模块：海中鱼巣.核心.会话.结构写入
```

导出不可默认构造、不可复制、不可移动的 `结构写入会话` 窄能力面。会话只能由执行器在取得许可的当前线程栈内直接构造，并以左值引用同步传给类型化回调；不得返回、转存、跨线程移动或延长到执行器调用之外。原始 `结构事务许可`、`结构事务令牌`、仓库引用和释放函数全部为模块私有。

### 5.2 执行器

新增真模块：

```text
海中鱼巣/核心/执行器.结构写入.ixx
模块：海中鱼巣.核心.执行器.结构写入
```

执行器持有结构事务接线，只接受已登记 `数据操作.*` 模块提交的类型化操作回调。写入执行器只能取得一次独占许可、构造写入会话并保证释放；共享许可只允许用于独立只读复核，不得构造写入会话。领域服务、线程、消息和组合器不能读取令牌，也不能在生产路径直接向执行器提交任意仓库操作回调。

### 5.3 数据操作访问

只有计划登记的 `数据操作.*` 模块可以调用会话的仓库操作能力；`#264` 的独立自检模块是第一轮唯一非生产例外。实现优先采用会话成员的窄结构方法，不导出“读取原始令牌”逃生口。会话内部维护本次主信息 / 节点候选、关系句柄和“主键 + 预期节点完整句柄”索引绑定对的精确写集；关系 / 索引不建立候选类型。仓库继续保留原始令牌 ABI 作为内部兼容层，待所有调用迁移后再裁决可见性。执行器即使导出为核心模块，也不得被领域服务、线程路由或组合器直接作为任意写回调入口使用，生产导入关系由计划和静态扫描约束。

### 5.4 #263 实际接口适配

`#264` 只消费 `#263` 已形成的十个独占结构化入口：主信息 / 节点候选创建、确认、撤销，关系结构化创建与严格删除，以及索引结构化绑定与严格删除。主信息 / 节点严格普通删除不属于本会话撤销路径。会话可以在模块私有实现中读取其自有许可的原始令牌并调用这些入口，但不得向回调、数据操作模块、领域服务或调用方返回令牌、许可或仓库引用。

`#264` 不修改四仓库、`结果.结构写入.h`、`结构事务接线.数据.h` 或 `协调.结构事务.ixx`。执行器直接消费现有 `取得独占许可`，会话直接消费现有结构化入口。若实现发现必须修改上述只读依赖才能闭合，按接口漂移退回设计，不得在执行窗口扩权。

候选可见性不能只靠“不导出句柄”证明。执行器必须要求接线有效，并以同一独占令牌调用四仓库；四仓库构造链和各结构化入口共同复核同一结构事务域。独占许可持有期间，其它线程的无令牌读取会在取得共享许可时等待，同线程无令牌读取会被重入门禁拒绝；同会话读回只能走会话内部令牌窄入口。许可必须晚于提交或撤销收口释放。若实测出现不经同域许可即可观察中途结构的读取旁路，停止 `#264` 并退回仓库级发布隔离专项。

## 6. 服务专用数据操作层

每个服务族建立独立真模块。数据操作模块私有持有本服务族所需的结构写入执行器引用和只读仓库引用，对业务服务只返回值式材料与值式结果。业务服务和组合器不得取得执行器、仓库、许可、令牌或会话。数据操作模块职责为：

```text
对上提供类型化业务数据操作入口和值式只读材料入口
在模块内部向结构写入执行器提交同步类型化回调
只在该回调内部接收不透明会话左值引用
复核通用结构当前性
创建主信息 / 节点未发布候选
创建关系 / 索引
按关系句柄和索引“主键 + 预期节点”绑定对逆序撤销
执行最后发布和候选确认
返回值式结构结果
```

数据操作层可在取得独占会话前读取节点类型、主键绑定和关系记录等值式材料，供业务服务完成写前裁决；进入会话后必须以同一完整版本句柄调用 `节点可读` 复核当前性。当前节点类型在同一节点生命周期版本内不可变，因此“写前值式类型材料 + 许可内完整句柄当前性复核”构成第一轮类型复核合同。会话不新增返回节点类型、任意仓库读取或原始令牌的逃生口。

数据操作层不接收线程消息，不决定业务策略，不输出日志作为结果，不形成跨领域反向依赖。只读仓库引用、执行器引用和会话均不得向业务服务、组合器或结果结构暴露。

## 7. 领域服务公开面

新业务服务真模块只导出：

```text
业务请求数据
业务结果数据
无原始令牌的业务入口
必要的值式只读入口
```

服务公开入口不得出现 `结构事务令牌`、`结构事务许可`、仓库引用或锁类型。业务服务可以通过只读端口获得值式当前材料，并在数据操作提交前由同一操作再次复核；许可内复核由数据操作层完成，业务语义仍由服务给出的类型化规则限定。

## 8. 跨服务组合器

组合器用于一个业务操作涉及多个服务时固定顺序和成功语义。组合器调用已登记的服务专用数据操作入口并提交业务操作，不直接导入结构写入执行器；数据操作层在内部向执行器提交类型化结构回调。参与服务通过内部类型化操作端口与数据操作层协作，不在公开入口之间传递会话。

第一轮组合器：`组合.存在场景.ixx`。后续按状态动态、特征语义、需求任务方法和运行期业务操作建立独立模块。

### 9.2 请求与结果

第一轮固定五类请求：

| 请求 | 必填字段 | 写前拒绝 |
| --- | --- | --- |
| 创建实际存在 | 非零全局幂等主键 | 主键为 0；主键已绑定错误类型、过期句柄或虚拟存在语义 |
| 创建虚拟存在 | 非零全局幂等主键、来源节点 | 来源不是当前需求 / 任务 / 方法；同键存在的来源不一致 |
| 创建场景 | 非零全局幂等主键 | 主键为 0；主键已绑定非场景或过期句柄 |
| 场景接纳已有存在 | 当前场景句柄、当前存在句柄 | 无效 / 过期句柄、错误类型、权威关系材料矛盾 |
| 在已有场景中创建并接纳实际存在 | 当前场景句柄、非零存在幂等主键 | 场景错误；主键映射冲突；同键身份是虚拟存在 |

公开结果至少区分：

```text
已提交
幂等读回
入口拒绝
幂等冲突
许可拒绝
版本漂移
内部不一致
```

结果只携带本操作必要的存在 / 场景 / 关系完整稳定句柄，不携带仓库引用、会话、许可、令牌、锁或未发布候选。结构结果映射为业务结果时，`内部不一致` 必须触发追根因路径，不得降级为普通入口拒绝。

### 9.3 幂等与关系重复

主键为当前统一 `uint64` 全局索引键，第一轮不新增领域命名空间。调用方必须提供非零全局唯一键：

```text
未绑定：允许进入写入会话。
已绑定到同类型、同语义当前身份：幂等读回。
已绑定到不同节点、错误类型、过期版本或不同虚拟来源：入口冲突拒绝，结构不变化。
```

实际存在的幂等读回必须证明不存在需求 / 任务 / 方法来源语义；虚拟存在的幂等读回必须证明来源关系唯一且与请求来源一致。场景只要求同键当前节点类型为场景。

关系结构化创建不提供重复关系幂等。场景接纳前先读取 `场景 -> 存在` 当前归属关系材料：同一当前关系已存在时由业务层返回幂等读回；不存在时才创建；同一端点出现多条、句柄版本矛盾或读回不可唯一解释时追根因停止。并发窗口中若写前读取为空、进入会话后创建被重复关系拒绝，执行器先完整撤销本次写集并释放许可，数据操作层再权威重读；只有读到同一当前归属关系才能映射为幂等成功，否则返回结构化失败或内部不一致。

### 9.4 五条固定写入路径

1. 创建实际存在：`幂等主键 -> 主信息候选 -> 存在节点候选 -> 主键绑定 -> 全部读回 -> 请求提交`。
2. 创建虚拟存在：`幂等主键 + 来源 -> 主信息候选 -> 存在节点候选 -> 主键绑定 -> 存在到来源引用 -> 全部读回 -> 请求提交`。来源引用是本操作最后形成的业务关系。
3. 创建场景：`幂等主键 -> 主信息候选 -> 场景节点候选 -> 主键绑定 -> 全部读回 -> 请求提交`。
4. 场景接纳已有存在：`已有场景 + 已有存在 -> 场景到存在归属 -> 关系读回 -> 请求提交`；写前已存在同一关系则直接幂等读回。
5. 在已有场景中创建并接纳实际存在：`已有场景 + 存在幂等主键 -> 创建缺失的实际存在身份和 / 或归属 -> 全部读回 -> 请求提交`。场景始终是已有身份；第一轮不得同时新建场景和存在。

第五条路径对应当前 `世界服务::创建场景存在(场景)` 的业务范围，但不改接该兼容入口。若幂等主键已经读回同一实际存在，只创建缺失归属；若归属也已存在，返回幂等读回。

### 9.5 分层调用与失败收口

固定调用路径：

```text
调用方
-> 存在业务服务 / 场景业务服务 / 存在场景组合器
-> 数据操作.存在场景的值式只读材料与类型化操作
-> 数据操作模块内部调用结构写入执行器
-> 一次独占许可内的不透明结构写入会话
-> 同会话读回
-> 提交确认或精确逆序撤销
-> 值式领域结果
```

业务服务和组合器不得直接导入执行器。数据操作层在会话外取得值式节点类型、主键和关系材料，业务层完成语义裁决；会话内以同一完整句柄调用 `节点可读` 复核当前性。若句柄版本漂移，必须在第一笔依赖该句柄的结构写入前拒绝，或由执行器撤销已形成的本次写集。

任何主信息、节点、主键、来源关系或归属关系写入不及预期，均由 #264 在许可释放前按索引、关系、节点候选、主信息候选逆序收口。收口后必须以数量快照和可读结构证明回到前态；无法证明时停止后继实施并登记断点。候选确认后出现内部不一致属于追根因解决，不得静默删除已确认结构。

### 9.6 第一轮排除

```text
不同时新建场景和存在。
不实现虚拟存在创建并接纳组合。
不收纳实例状态或实例动态。
不迁移既有生产调用，不删除或改义兼容入口。
不修改 #264 会话 / 执行器、四仓库、事务协调核心或统一结果头。
不恢复 #214，不夹带 #257 或后续性能优化。
```

## 10. 服务迁移批次

| 顺序 | 服务族 | 主要原因 |
| --- | --- | --- |
| 1 | 存在 / 场景 / 世界聚合 | 最小主信息、节点、关系垂直样例 |
| 2 | 状态 / 动态 | 具备时间戳、场景归属、前后值和候选撤销 |

## 12. 锁与许可

写入执行器每次业务操作只取得一次独占结构许可。仓库仍按自身锁管理短临界区，不透明会话不允许锁升级。需要同时持有多个仓库锁时继续遵守 `节点 -> 主信息 -> 关系 -> 索引`；优先用全域独占许可下的逐仓库短锁，禁止服务持锁调用上层服务。共享许可仅供只读执行，不得进入本设计的写入会话。

## 13. 失败收口

```text
许可前拒绝：结构不变化。
候选创建失败：撤销本次已建候选。
关系 / 索引失败：先删本次关系 / 索引，再撤销身份候选。
最后发布失败：逆序撤销全部本次未发布结构。
最后发布后读回异常或候选确认异常：追根因停止，不静默回滚已发布事实。
撤销不能证明数量和可读结构回到前态：追根因停止后继计划。
```

## 14. 兼容与 ABI

四仓库带令牌 ABI 第一轮保留，作为写入会话内部适配目标。领域服务带令牌入口、无令牌写入口和服务候选类型按调用矩阵逐批迁移。不得一次性全仓删除，不得让新模块通过 `读取令牌()` 重新泄漏原始令牌。
