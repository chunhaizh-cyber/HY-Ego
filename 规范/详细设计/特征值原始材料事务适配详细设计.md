# 特征值原始材料事务适配详细设计

日期：2026-07-14

状态：JY-328 修订正式详细设计 / QR-181、QR-182 已裁决 / #274 再次发布依据 / 未改代码

## 1. 目标

为新建特征值的初始 I64、VecI64、VecU64 原始材料建立与四仓库结构写集一致的提交、撤销和读取边界：

```text
特征体系数据操作层
-> 特征值原始材料事务参与者
-> 结构写入执行器与不透明会话
-> 四仓库写集 + 特征值服务内部侧表写集
-> 同一独占许可内准备、确认或撤销
```

本设计不建立特征业务入口，不修改已发布特征值，不迁移旧函数体，不把特征值侧表提升为核心仓库。

## 2. 当前代码事实

### 2.1 已有结构会话

`#272 / CORE-SESSION-S2` 已实现并验证：

```text
本会话主信息候选 I64 值写入与匹配
节点类型、节点主信息、I64 值和关系记录组同许可读取
主信息、节点、关系和索引完整写集读回
失败、异常和无提交决定时精确逆序撤销
```

当前会话没有服务内部侧表参与者，也没有公开“是否为本会话候选”的只读判断。JY-325 进一步确认：`请求提交()` 会把决定改为提交，而现有节点类型、节点主信息和 I64 读取均通过 `可继续写入()` 要求决定仍为未决定；因此这些普通入口在请求提交后必然拒绝。既有 #272 自检同时把“决定后普通读取拒绝”作为已验证合同，不能为 #274 直接放宽。

### 2.2 当前特征值原始材料

`特征值服务.h` 当前承载：

| 材料 | 当前权威承载 | 当前版本承载 |
| --- | --- | --- |
| I64 值 | 特征值节点主信息槽位 0 | `I64版本记录表_` |
| VecI64 | `Vec记录表_` | 同一 Vec 记录的 `容器版本` |
| VecU64 | `Vec记录表_` | 同一 Vec 记录的 `容器版本` |

`I64版本记录表_`、`Vec记录表_` 和 `原始值锁_` 均为 `特征值服务` 私有成员。现有写入口会先读取节点，再取得原始值锁，并在锁内访问主信息仓库；它们是兼容事实，不具备加入 `#272` 写集的候选或撤销合同。

### 2.3 真实断点

若 `#267` 直接在结构会话内调用现有特征值写入口：

```text
无令牌节点 / 主信息访问会触发同线程重入门禁；
侧表写入不在结构会话写集内；
结构失败后无法精确撤销 I64 版本或 Vec 记录；
持有结构许可后等待原始值锁，会与兼容路径形成锁序反转风险。
```

因此原 `#267` 必须先依赖本适配切片。

JY-325 的接口漂移不是参与者顺序错误，而是原设计把“普通会话只读入口”误当成了“提交准备能力”。本次修订必须建立独立能力，不能改写 `可继续写入()`，也不能把领域材料冻结职责下沉给执行器。

### 2.4 JY-327 确认故障验收断点

执行窗口的未提交草稿已证明 JY-326 主路径可以编译并完成一次完整自检，但 A09、A10 实际复用同一个准备拒绝布尔值。错误类型在参与者准备阶段即被拒绝，侧表从未插入临时记录，因此不能证明“参与者已准备后，会话确认失败时精确撤销”。

执行窗口未提交草稿中的通用参与者路径在参与者准备成功后立即调用 `会话.完成提交()`，两者之间没有合法观察或故障点；当前正式代码仍未实现该参与者路径。四仓库、令牌、候选和确认能力保持封闭是正确合同，不得为制造测试故障而放宽。本次只为计划中的拟议路径增加固定阶段隔离自检合同，不改变既有生产执行入口。

## 4. 通用事务参与者

在 `执行器.结构写入.ixx` 增加不含领域语义的抽象参与者合同。生命周期方法只允许执行器调用：

```text
准备提交(const 结构提交准备只读视图&)
完成提交()
完成撤销()
```

执行器的新重载固定顺序：

```text
取得一次独占许可
-> 执行业务回调
-> 会话无失败且已经请求提交
-> 执行器构造提交准备只读视图
-> 参与者准备提交
-> 会话完成提交
-> 参与者完成提交
-> 释放许可
```

任一回调、会话或参与者失败：

```text
参与者完成撤销
-> 会话完成撤销
-> 释放许可
-> 返回结构化失败
```

参与者不得获得可写会话、原始令牌、许可或仓库引用。视图只在执行器当前线程栈内同步存活；参与者不得保存其地址或引用。现有 `执行(回调)` 重载和 #264/#272 行为保持不变。

### 4.1 隔离自检确认故障合同

`执行器.结构写入.ixx` 在专用宏 `HY_EGO_ENABLE_STRUCTURE_COMMIT_FAULT_SELF_TEST` 下增加独立自检重载，语义固定为：

```text
执行确认前故障自检(业务回调, 通用参与者, 无参观察回调)
```

该重载复用正常的独占许可、会话、窄视图和参与者准备顺序。无参观察回调必须非空，只能在参与者 `准备提交` 返回成功后、调用 `会话.完成提交()` 前执行一次；回调不接收会话、参与者、窄视图、令牌、许可、仓库或候选。观察回调返回后固定制造结构化内部失败，不允许继续确认。

故障收口顺序固定为：

```text
参与者完成撤销
-> 证明本次 I64 版本 / Vec 记录按精确节点删除并释放内部锁
-> 会话完成撤销
-> 证明主信息 / 节点候选、关系和索引均回前态
-> 释放独占许可
-> 返回内部不一致自检结果
```

自检重载不得增加执行器数据成员、全局 / 静态故障开关、可变点位编号、延迟到下一次调用的状态或生产重载参数。生产二参 `执行(回调, 参与者)` 不接收故障回调，也不得调用自检重载。

### 4.2 编译和调用隔离

工程只在 `Debug|x64` 的预处理器定义中加入 `HY_EGO_ENABLE_STRUCTURE_COMMIT_FAULT_SELF_TEST`。执行器自检重载、特征值事务自检中的确认故障验收正文以及入口中的该自检 import / 调用 / 735 汇总均受同一宏控制；Release 不定义宏、不编译自检重载、不登记该隔离自检。自检 `.ixx` 在 Release 下仍必须保留合法 `export module` 编译单元，不能因宏关闭形成无模块声明的空文件。

允许出现故障符号的文件仅限：

```text
海中鱼巣/核心/执行器.结构写入.ixx
海中鱼巣/领域/自检.特征值原始材料事务.ixx
海中鱼巣/入口.cpp（仅宏条件 import / 调用 / 汇总）
海中鱼巣.vcxproj（仅 Debug|x64 宏）
```

`参与者.特征值原始材料.ixx`、提交准备只读视图、特征值服务和其它生产模块不得出现故障符号。Release x64 构建必须成功，并由静态扫描证明自检重载未形成。

## 5. 提交准备只读视图

在 `会话.结构写入.ixx` 导出通用 `结构提交准备只读视图`。它不可默认构造、不可复制、不可移动；构造权只授予 `结构写入执行器`，业务回调不能自行取得。视图只暴露：

```cpp
bool 主信息是本会话候选(主信息句柄 主信息) const;
bool 节点是本会话候选(节点句柄 节点) const;
std::optional<节点类型> 读取候选节点类型(节点句柄 节点) const;
std::optional<主信息句柄> 读取候选节点主信息(节点句柄 节点) const;
std::optional<std::int64_t> 读取候选I64值(主信息句柄 主信息, std::uint64_t 槽位) const;
```

视图每次调用都先走独立窄门禁：当前线程、会话阶段仍为写入中、决定为提交、无首次失败、尚未进入确认或撤销收口。节点类读取还必须先证明节点属于本会话候选；I64 读取必须先证明主信息属于本会话候选。无效句柄、错误版本、任意已发布结构、其它会话候选、跨线程、提交决定前或收口后均返回 `false` / 空值。

现有 `结构写入会话::读取节点类型`、`读取节点主信息`、`读取I64值`、`读取关系记录组` 和全部写入口不改签名、不改门禁；它们在请求提交或撤销后继续拒绝。视图不暴露关系读取、仓库、令牌、写入、确认或撤销能力，不改变写集读回标记，也不导出候选对象。

## 6. 特征值事务参与者

新增真模块：

```text
文件：海中鱼巣/领域/参与者.特征值原始材料.ixx
模块：海中鱼巣.领域.参与者.特征值原始材料
```

模块导出两个仅供特征体系数据操作层使用的能力：

```text
特征值原始材料事务参与者
特征值原始材料侧表访问器
```

`特征值服务.h` 只增加前置声明和友元，不新增公开写入口。参与者通过友元访问私有原始值锁和两张侧表。

### 6.1 参与者状态

参与者不可默认构造、不可复制、不可移动，只能绑定一个 `特征值服务` 实例。内部至少保存：

```text
待发布原始材料值式副本组
已插入 I64 版本记录的精确节点句柄组
已插入 Vec 记录的精确节点句柄组
非阻塞取得并保持到收口的原始值独占锁
未准备 / 已准备 / 已提交 / 已撤销阶段
```

第一轮数据操作可以登记一个或多个新特征值材料，但同一节点只能出现一次，所有材料原始版本必须为 1。

### 6.2 准备提交

执行器只在会话已经通过全部写集读回并请求提交后构造窄视图并调用参与者。参与者逐项复核：

```text
特征值节点和主信息句柄有效；
窄视图证明节点与主信息均属于本会话候选；
窄视图读取候选节点类型为 特征值；
窄视图读取候选节点主信息等于本次候选主信息；
I64 请求由窄视图读取候选主信息槽位 0 并等于预期值；
Vec 请求由窄视图证明候选主信息槽位 0 未建立；
侧表中不存在同节点的 Vec 或 I64 版本记录；
原始材料字段互斥、序列非空且不超过 4096。
```

复核完成后参与者不得再次调用窄视图，再使用 `try_to_lock` 取得原始值独占锁。未取得锁属于许可竞争，参与者不写侧表，执行器撤销结构会话。取得锁后插入本次临时记录并保持锁，不调用视图或仓库。

### 6.3 完成提交与撤销

会话完成提交后，参与者只改变自身阶段并释放内部锁。记录已经是正式特征值原始材料。

会话确认失败或执行器进入失败收口时，参与者按精确节点句柄和版本 1 删除本次插入记录，然后释放内部锁。删除目标缺失、重复、类型或版本不匹配属于内部不一致，不得扫描修复或删除其它记录。

## 7. 侧表只读访问器

访问器只读取特征值服务私有侧表，不读取节点、主信息、关系或索引。调用方必须已经持有共享结构许可并完成节点和关系读取。

读取顺序：

```text
数据操作层取得共享结构许可
-> 带令牌读取节点、主信息、关系和 I64 值
-> 访问器 try_lock_shared 原始值锁
-> 复制 I64 版本记录或 Vec 完整值和版本
-> 数据操作层组合并复核唯一原始类型
-> 释放侧表锁和结构许可
```

锁竞争时访问器返回明确的“许可竞争”，不得等待，不得在没有同一结构许可的情况下重读并拼接材料。返回值不包含内部指针或可写引用。

## 8. 锁序与并发

新路径锁序固定为：

```text
结构共享 / 独占许可
-> 非阻塞原始值共享 / 独占锁尝试
```

兼容路径可能仍沿 `原始值锁 -> 仓库入口` 访问。新路径不得在持有结构许可时等待原始值锁：

```text
尝试成功：在同一结构许可内完成读取或提交收口。
尝试失败：不写侧表；写路径撤销结构写集，读路径释放结构许可并返回许可竞争。
```

参与者取得原始值独占锁后不得调用四仓库。该规则防止新旧路径互等，不宣称兼容路径已经迁移。

## 10. 入口拒绝与追根因

逻辑内返回：

```text
无效句柄、未知类型、字段冲突、版本不是 1、空或超长序列；
节点或主信息不是本会话候选；
原始值锁竞争，且侧表尚未写入；
第一笔发布前发现版本漂移。
```

隔离自检固定故障不是逻辑内业务返回。它只用于证明准备成功后的收口顺序，预期返回 `内部不一致`；该结果不得进入生产重试、机器事实或完成声明。

追根因解决：

```text
完整请求进入准备发布后，会话候选、节点类型或主信息锚点不符合内部预期；
已请求提交后窄视图无法构造，或窄门禁发现线程、阶段、决定、失败状态不符合执行器合同；
侧表出现重复类型、重复版本或 I64 / Vec 并存；
会话确认失败后无法精确撤销本次侧表记录；
失败后数量或可读结构未回到前态。
```
