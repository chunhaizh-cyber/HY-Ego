# 特征值原始材料事务适配详细设计

日期：2026-07-13

状态：正式详细设计 / JY-324 / #274 施工依据 / 未改代码

## 1. 目标

为新建特征值的初始 I64、VecI64、VecU64 原始材料建立与四仓库结构写集一致的提交、撤销和读取边界：

```text
特征体系数据操作层
-> 特征值原始材料事务参与者
-> 结构写入执行器与不透明会话
-> 四仓库写集 + 特征值服务内部侧表写集
-> 同一独占许可内准备、确认或撤销
```

本设计不建立特征业务入口，不修改已发布特征值，不迁移旧函数体，不把特征值侧表提升为核心仓库。

## 2. 当前代码事实

### 2.1 已有结构会话

`#272 / CORE-SESSION-S2` 已实现并验证：

```text
本会话主信息候选 I64 值写入与匹配
节点类型、节点主信息、I64 值和关系记录组同许可读取
主信息、节点、关系和索引完整写集读回
失败、异常和无提交决定时精确逆序撤销
```

当前会话没有服务内部侧表参与者，也没有公开“是否为本会话候选”的只读判断。

### 2.2 当前特征值原始材料

`特征值服务.h` 当前承载：

| 材料 | 当前权威承载 | 当前版本承载 |
| --- | --- | --- |
| I64 值 | 特征值节点主信息槽位 0 | `I64版本记录表_` |
| VecI64 | `Vec记录表_` | 同一 Vec 记录的 `容器版本` |
| VecU64 | `Vec记录表_` | 同一 Vec 记录的 `容器版本` |

`I64版本记录表_`、`Vec记录表_` 和 `原始值锁_` 均为 `特征值服务` 私有成员。现有写入口会先读取节点，再取得原始值锁，并在锁内访问主信息仓库；它们是兼容事实，不具备加入 `#272` 写集的候选或撤销合同。

### 2.3 真实断点

若 `#267` 直接在结构会话内调用现有特征值写入口：

```text
无令牌节点 / 主信息访问会触发同线程重入门禁；
侧表写入不在结构会话写集内；
结构失败后无法精确撤销 I64 版本或 Vec 记录；
持有结构许可后等待原始值锁，会与兼容路径形成锁序反转风险。
```

因此原 `#267` 必须先依赖本适配切片。

## 3. 默认裁决

| 编号 | 裁决 |
| --- | --- |
| FV-TXN-Q1 | 第一轮只支持本会话新建特征值节点的初始原始材料，版本固定为 1。 |
| FV-TXN-Q2 | 不支持更新、覆盖或删除已发布特征值；同值重写和版本递增后置。 |
| FV-TXN-Q3 | 核心执行器只新增通用事务参与者生命周期，不导入特征值类型。 |
| FV-TXN-Q4 | 参与者准备发布时使用非阻塞独占锁尝试；侧表读取使用非阻塞共享锁尝试。锁竞争不等待，返回许可竞争并撤销。 |
| FV-TXN-Q5 | I64 值继续写入主信息候选；参与者只登记 I64 版本。Vec 值和版本继续写入特征值服务内部侧表。 |
| FV-TXN-Q6 | 参与者必须证明节点和主信息均属于本会话候选，不能把通用参与者变成既有结构修改后门。 |

## 4. 通用事务参与者

在 `执行器.结构写入.ixx` 增加不含领域语义的抽象参与者合同。生命周期方法只允许执行器调用：

```text
准备提交(结构写入会话&)
完成提交()
完成撤销()
```

执行器的新重载固定顺序：

```text
取得一次独占许可
-> 执行业务回调
-> 会话无失败且已经请求提交
-> 参与者准备提交
-> 会话完成提交
-> 参与者完成提交
-> 释放许可
```

任一回调、会话或参与者失败：

```text
参与者完成撤销
-> 会话完成撤销
-> 释放许可
-> 返回结构化失败
```

参与者不得获得原始令牌、许可或仓库引用。现有 `执行(回调)` 重载和 #264/#272 行为保持不变。

## 5. 会话候选只读能力

在 `会话.结构写入.ixx` 增加：

```cpp
bool 主信息是本会话候选(主信息句柄 主信息) const;
bool 节点是本会话候选(节点句柄 节点) const;
```

两者只查询当前会话已有候选容器，不访问仓库、不改变写集、不确认候选，也不导出候选对象。跨线程、会话结束、无效句柄或错误版本均返回 `false`。

## 6. 特征值事务参与者

新增真模块：

```text
文件：海中鱼巣/领域/参与者.特征值原始材料.ixx
模块：海中鱼巣.领域.参与者.特征值原始材料
```

模块导出两个仅供特征体系数据操作层使用的能力：

```text
特征值原始材料事务参与者
特征值原始材料侧表访问器
```

`特征值服务.h` 只增加前置声明和友元，不新增公开写入口。参与者通过友元访问私有原始值锁和两张侧表。

### 6.1 参与者状态

参与者不可默认构造、不可复制、不可移动，只能绑定一个 `特征值服务` 实例。内部至少保存：

```text
待发布原始材料值式副本组
已插入 I64 版本记录的精确节点句柄组
已插入 Vec 记录的精确节点句柄组
非阻塞取得并保持到收口的原始值独占锁
未准备 / 已准备 / 已提交 / 已撤销阶段
```

第一轮数据操作可以登记一个或多个新特征值材料，但同一节点只能出现一次，所有材料原始版本必须为 1。

### 6.2 准备提交

执行器只在会话已经通过全部写集读回并请求提交后调用。参与者逐项复核：

```text
特征值节点和主信息句柄有效；
节点与主信息均属于本会话候选；
会话读取节点类型为 特征值；
节点主信息等于本次候选主信息；
I64 请求的候选主信息槽位 0 等于预期值；
Vec 请求的候选主信息槽位 0 未建立；
侧表中不存在同节点的 Vec 或 I64 版本记录；
原始材料字段互斥、序列非空且不超过 4096。
```

复核后使用 `try_to_lock` 取得原始值独占锁。未取得锁属于许可竞争，参与者不写侧表，执行器撤销结构会话。取得锁后插入本次临时记录并保持锁，不调用仓库。

### 6.3 完成提交与撤销

会话完成提交后，参与者只改变自身阶段并释放内部锁。记录已经是正式特征值原始材料。

会话确认失败或执行器进入失败收口时，参与者按精确节点句柄和版本 1 删除本次插入记录，然后释放内部锁。删除目标缺失、重复、类型或版本不匹配属于内部不一致，不得扫描修复或删除其它记录。

## 7. 侧表只读访问器

访问器只读取特征值服务私有侧表，不读取节点、主信息、关系或索引。调用方必须已经持有共享结构许可并完成节点和关系读取。

读取顺序：

```text
数据操作层取得共享结构许可
-> 带令牌读取节点、主信息、关系和 I64 值
-> 访问器 try_lock_shared 原始值锁
-> 复制 I64 版本记录或 Vec 完整值和版本
-> 数据操作层组合并复核唯一原始类型
-> 释放侧表锁和结构许可
```

锁竞争时访问器返回明确的“许可竞争”，不得等待，不得在没有同一结构许可的情况下重读并拼接材料。返回值不包含内部指针或可写引用。

## 8. 锁序与并发

新路径锁序固定为：

```text
结构共享 / 独占许可
-> 非阻塞原始值共享 / 独占锁尝试
```

兼容路径可能仍沿 `原始值锁 -> 仓库入口` 访问。新路径不得在持有结构许可时等待原始值锁：

```text
尝试成功：在同一结构许可内完成读取或提交收口。
尝试失败：不写侧表；写路径撤销结构写集，读路径释放结构许可并返回许可竞争。
```

参与者取得原始值独占锁后不得调用四仓库。该规则防止新旧路径互等，不宣称兼容路径已经迁移。

## 10. 入口拒绝与追根因

逻辑内返回：

```text
无效句柄、未知类型、字段冲突、版本不是 1、空或超长序列；
节点或主信息不是本会话候选；
原始值锁竞争，且侧表尚未写入；
第一笔发布前发现版本漂移。
```

追根因解决：

```text
完整请求进入准备发布后，会话候选、节点类型或主信息锚点不符合内部预期；
侧表出现重复类型、重复版本或 I64 / Vec 并存；
会话确认失败后无法精确撤销本次侧表记录；
失败后数量或可读结构未回到前态。
```
