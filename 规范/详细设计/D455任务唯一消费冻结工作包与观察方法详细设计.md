## 1. 设计目标与依据

本设计重绑 DD-03、DD-04 到节点直接任务、方法和任务筹办结构，形成：

```text
单一报告队列等待项
-> 任务域唯一消费占用
-> 任务 / 工作项 / 消费窗口绑定的已承接材料引用
或：观察结果形成的识别派生需求材料
-> 需求统一入口 / 新任务承接
-> 同一任务唯一筹办执行权
-> 不可变 D455 方法工作包或识别任务工作包
-> 观察 / 识别 / 扫描 / 跟踪强类型结果
```

正式依据为 0050、3200、3300、4010—4050、5200—5250、5300—5340、6320、6340—6370、8100。外设报告只能由任务域正式承接；任务筹办可重复，但同一任务同一时刻只能有一个有效筹办占用。

同一报告跨窗口、任务和恢复代次的一次承接唯一键只能是 `D455报告身份`。窗口、任务和工作项只作准入与绑定，不得进入能够放宽唯一性的竞争键。

任务侧数据操作只通过下列内部参与包进入同一结构事务：

```cpp
class 任务外设材料承接参与包 {
public:
    virtual ~任务外设材料承接参与包() = default;
    virtual 节点直接身份结构写入事务参与者& 读取事务参与者() noexcept = 0;
    virtual D455承接结果 读取发布后结果() const = 0;
};
class 任务外设材料数据操作 {
public:
    virtual ~任务外设材料数据操作() = default;
    virtual 任务外设材料承接准备结果 准备任务外设材料承接(
        const D455承接规格& 规格,
        const D455报告队列项& 报告候选,
        const 任务当前性快照& 任务快照) = 0;
};
class 任务当前性只读提供者 {
public:
    virtual ~任务当前性只读提供者() = default;
    virtual 任务当前性读取结果 读取任务当前性(
        节点句柄 任务,
        std::uint32_t 预期任务版本) const = 0;
};
```

`任务外设材料承接服务` 构造注入：

```cpp
任务外设材料承接服务(
    D455报告消费提供者& 报告提供者,
    任务当前性只读提供者& 任务提供者,
    任务外设材料数据操作& 数据操作,
    节点直接身份结构写入执行器& 执行器);
```

其中 `D455报告消费提供者` 由 PER-C2 暴露内部消费候选、确认、撤销和读回；`任务当前性只读提供者` 只返回节点类型、任务 / 节点版本、当前筹办轮次和当前状态事实组的值式快照。`任务当前性状态` 只裁决本次读取结果，不代替任务完整状态事实。三个端口都不得暴露仓库、锁或事务令牌。执行器确认顺序固定为“报告消费参与者 -> 任务侧承接参与者”，撤销逆序，最后统一发布；发布后两个参与包必须分别读回 `已确认或幂等读回` 与 `已承接或幂等读回`。任一发布后读回不一致均为内部错误。

## 3. PER-C6—PER-C8 / v0.2

### 3.1 PER-C6 / v0.2：唯一消费与已承接引用

- 模块 / 文件：第 2 节 #366 五个精确文件；只读消费 #362 的 `海中鱼巣/线程/队列.D455观察供给报告.ixx`。
- DTO：等待工作项、消费窗口、承接规格、承接结果、不透明引用。
- 输入 / 输出：输入任务、工作项、消费窗口和当前报告候选；输出唯一已承接 D455 材料引用或结构化拒绝。
- ABI / 规则版本：导出 `inline constexpr std::uint32_t 任务外设材料承接ABI版本 = 2`；报告、承接、消费状态和生命周期版本必须同时读回。
- 提供 / 消费：#366 提供，#367 消费。
- 写读方：任务域数据操作唯一写；目标任务和 #367 只读 / 领取。
- 生命周期：从报告消费候选开始，到方法结束、取消、过期或隔离完成。
- 并发键：报告稳定身份；消费窗口只作绑定与当前性条件。一个报告跨全部窗口和代次只形成一个当前承接。
- 逻辑内返回：无匹配、错任务、错工作项、质量不足、过期、已承接。
- 内部错误：队列确认后承接记录无法发布或读回。
- 恢复：不恢复物理队列或分类索引。恢复候选必须含报告稳定身份、TTL / 当前性、外设配置版本、任务 / 工作项 / 消费窗口、消费状态版本及生产 / 消费代次；已承接项只恢复任务侧引用且不得重新消费，未承接且当前不可舍弃项以原报告身份进入新生产代次投递候选，过期、已消费和失效项按原状态收束。
- 漂移退回：队列消费签名、任务 / 工作项字段、承接唯一性或生命周期改变。

### 3.2 PER-C7 / v0.2：读取筹办许可与冻结工作包

- 模块 / 文件：第 2 节 #367 六个精确文件；任务筹办入口由 5230 对应实现提供。
- DTO：SELF-C2 筹办许可只读投影、D455 / 识别 / 扫描工作包冻结规格、三类强类型工作包和冻结结果；`识别派生需求材料`、`合法外设当前数据项` 与 `扫描输入材料` 的类型也唯一归属本合同模块，#369 只 import 后形成值，避免 #367 / #369 循环定义。
- 输入 / 输出：输入 SELF-C2 已发布的任务筹办许可投影以及一种当前材料引用；输出与方法入口严格匹配的不可变工作包。C7 不读取、不持有也不修改筹办占用实体。
- ABI / 规则版本：导出 `inline constexpr std::uint32_t 任务感知方法工作包ABI版本 = 2`；任务、筹办轮次、材料、方法和冻结版本共同裁决当前性。
- 提供 / 消费：#367 提供，#368 消费。
- 写读方：SELF-C2 唯一取得筹办权；#367 只读其已发布许可投影并唯一写感知工作包；方法执行只读。
- 生命周期：每个任务可在后继筹办轮次形成新工作包；同一轮次冻结后不可变。识别工作包属于观察结果派生的新任务回合；首次扫描工作包属于合法外设当前数据项派生的新扫描任务回合；二者都不是观察工作包的子调用。
- 并发键：任务身份。不同任务可并发；同一任务第二路径在方法召回前拒绝。
- 逻辑内返回：许可不存在、许可非当前、许可中的任务 / 筹办轮次与请求不一致、引用失效、方法 / 产品映射不匹配、任务取消。
- 内部错误：两路径同时进入同一任务召回；冻结后字段改变；前置通过后无法读回。
- 恢复：C7 只把旧工作包保留为历史只读记录，不恢复其当前性，也不失效、迁移或重建任何筹办占用。SELF-C2 / C13 的任务筹办恢复提供者完成旧运行代次与前轮动作核验并发布新恢复代次、待重筹办触发和新的许可投影后，C7 才能据该新许可冻结新工作包；没有新许可时只返回合法等待。
- 漂移退回：任务筹办接口、任务 / 方法直接身份、冻结版本或所有权改变。

### 3.3 PER-C8 / v0.2：四方法结果

- 模块 / 文件：第 2 节 #368 三个精确文件；不拥有后继需求、存在或世界事实提交。
- DTO：四类登记 / 入口规格、D455 / 识别 / 扫描任务工作包只读视图、四类结果候选和结构化逻辑内结果。
- 输入 / 输出：输入当前且入口匹配的强类型工作包；输出观察候选、唯一已有存在句柄 / 空句柄、扫描结果或跟踪结果。
- ABI / 规则版本：导出 `inline constexpr std::uint32_t 感知本能方法ABI版本 = 2`；通过本设计冻结的只读端口消费 SELF-C3 方法登记投影和 `PER-C7 / v0.2`。
- 提供 / 消费：#368 提供，#369、#370 消费。
- 映射：
  - 稳定观察子集 -> 观察；
  - 存在材料 + 已确定候选范围 + 已有特征引用 -> 识别；
  - 已有可比基准的扫描变化报告，或观察方法 / 提交入口已经形成的合法外设当前数据项 + 已确认存在集合 + 内部场景基准（首次允许无历史可比材料） -> 扫描；
  - 目标跟踪 -> 跟踪；
  - 原始逐簇 -> 禁止进入任何业务方法。
- 生命周期：方法调用内形成候选；观察结果若需识别，先形成派生需求材料并结束本方法回合，后继新任务再执行识别。
- 并发键：工作包身份；不得直接互调四种方法。
- 逻辑内返回：工作包失效、输入类型不匹配、无观察结果、识别空句柄、无变化、目标丢失。
- 内部错误：已冻结动作入口不存在、结果来源不对应工作包、同包产生冲突结果。
- 恢复：未提交候选可丢弃并重算；已提交结果按结果身份幂等读取，不从日志恢复。
- 漂移退回：新增 D455 产品直通识别入口、识别 / 扫描工作包来源或 ABI 改变、产品映射、结果字段或来源动作语义改变。
