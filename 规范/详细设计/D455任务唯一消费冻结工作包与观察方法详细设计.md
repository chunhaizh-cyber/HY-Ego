## 1. 设计目标与依据

本设计重绑 DD-03、DD-04 到节点直接任务、方法和任务筹办结构，形成：

```text
单一报告队列等待项
-> 任务域唯一消费占用
-> 任务 / 工作项 / 消费窗口绑定的已承接材料引用
或：观察结果形成的识别派生需求材料
-> 需求统一入口 / 新任务承接
-> 同一任务唯一筹办执行权
-> 不可变 D455 方法工作包或识别任务工作包
-> 观察 / 识别 / 扫描 / 跟踪强类型结果
```

正式依据为 0050、3200、3300、4010—4050、5200—5250、5300—5340、6320、6340—6370、8100。外设报告只能由任务域正式承接；任务筹办可重复，但同一任务同一时刻只能有一个有效筹办占用。

```cpp

结果<识别任务工作包> 冻结识别任务工作包(const 任务筹办占用& 占用, const 方法句柄& 方法, const 识别派生需求材料& 材料);
结果<扫描任务工作包> 冻结扫描任务工作包(const 任务筹办占用& 占用, const 方法句柄& 方法, const 扫描输入材料& 材料);
```

方法只读与自身入口匹配的工作包；不得访问物理队列、生产者、受控材料仓内部或其它方法入口。识别工作包只能来自结构化观察结果经需求统一入口、新任务承接和新筹办轮次后的冻结结果，不能由观察方法直接构造或调用。

## 3. PER-C6—PER-C8

### 3.1 PER-C6 / v0.1：唯一消费与已承接引用

- 模块 / 文件：第 2 节 #366 五个精确文件；只读消费 #362 的 `海中鱼巣/线程/队列.D455观察供给报告.ixx`。
- DTO：等待工作项、消费窗口、承接规格、承接结果、不透明引用。
- 输入 / 输出：输入任务、工作项、消费窗口和当前报告候选；输出唯一已承接 D455 材料引用或结构化拒绝。
- ABI / 规则版本：`PER-C6 / v0.1`；报告、承接、消费状态和生命周期版本必须同时读回。
- 提供 / 消费：#366 提供，#367 消费。
- 写读方：任务域数据操作唯一写；目标任务和 #367 只读 / 领取。
- 生命周期：从报告消费候选开始，到方法结束、取消、过期或隔离完成。
- 并发键：队列项身份 + 消费窗口；一个报告只形成一个当前承接。
- 逻辑内返回：无匹配、错任务、错工作项、质量不足、过期、已承接。
- 内部错误：队列确认后承接记录无法发布或读回。
- 恢复：不恢复物理队列或分类索引。恢复候选必须含报告稳定身份、TTL / 当前性、外设配置版本、任务 / 工作项 / 消费窗口、消费状态版本及生产 / 消费代次；已承接项只恢复任务侧引用且不得重新消费，未承接且当前不可舍弃项以原报告身份进入新生产代次投递候选，过期、已消费和失效项按原状态收束。
- 漂移退回：队列消费签名、任务 / 工作项字段、承接唯一性或生命周期改变。

### 3.2 PER-C7 / v0.1：筹办权与冻结工作包

- 模块 / 文件：第 2 节 #367 六个精确文件；任务筹办入口由 5230 对应实现提供。
- DTO：筹办占用、D455 / 识别 / 扫描工作包冻结规格、三类强类型工作包和冻结结果；`识别派生需求材料`、`合法外设当前数据项` 与 `扫描输入材料` 的类型也唯一归属本合同模块，#369 只 import 后形成值，避免 #367 / #369 循环定义。
- 输入 / 输出：输入任务筹办占用以及一种当前材料引用；输出与方法入口严格匹配的不可变工作包。
- ABI / 规则版本：`PER-C7 / v0.1`；任务、筹办轮次、材料、方法和冻结版本共同裁决当前性。
- 提供 / 消费：#367 提供，#368 消费。
- 写读方：取得筹办权的任务域路径唯一写；方法执行只读。
- 生命周期：每个任务可在后继筹办轮次形成新工作包；同一轮次冻结后不可变。识别工作包属于观察结果派生的新任务回合；首次扫描工作包属于合法外设当前数据项派生的新扫描任务回合；二者都不是观察工作包的子调用。
- 并发键：任务身份。不同任务可并发；同一任务第二路径在方法召回前拒绝。
- 逻辑内返回：筹办权占用、引用失效、方法 / 产品映射不匹配、任务取消。
- 内部错误：两路径同时进入同一任务召回；冻结后字段改变；前置通过后无法读回。
- 恢复：保留原任务、筹办轮次、占用版本、来源运行代次和历史当前性；必须先证明旧运行代次已终止，并证明前轮外部动作已停止或已有实际回读，再失效旧占用和旧冻结，发布新恢复代次与待重筹办触发，由新筹办轮次重新取得执行权。不能证明前轮动作已收口时只形成合法等待、动作核验生产者和重新触发条件，不得恢复旧执行权或重复发起外部动作。
- 漂移退回：任务筹办接口、任务 / 方法直接身份、冻结版本或所有权改变。

### 3.3 PER-C8 / v0.1：四方法结果

- 模块 / 文件：第 2 节 #368 三个精确文件；不拥有后继需求、存在或世界事实提交。
- DTO：四类登记 / 入口规格、D455 / 识别 / 扫描任务工作包只读视图、四类结果候选和结构化逻辑内结果。
- 输入 / 输出：输入当前且入口匹配的强类型工作包；输出观察候选、唯一已有存在句柄 / 空句柄、扫描结果或跟踪结果。
- ABI / 规则版本：`PER-C8 / v0.1`；消费 #356 `SELF-C3 / v0.1` 方法登记扩展合同和 `PER-C7 / v0.1`。
- 提供 / 消费：#368 提供，#369、#370 消费。
- 映射：
  - 稳定观察子集 -> 观察；
  - 存在材料 + 已确定候选范围 + 已有特征引用 -> 识别；
  - 已有可比基准的扫描变化报告，或观察方法 / 提交入口已经形成的合法外设当前数据项 + 已确认存在集合 + 内部场景基准（首次允许无历史可比材料） -> 扫描；
  - 目标跟踪 -> 跟踪；
  - 原始逐簇 -> 禁止进入任何业务方法。
- 生命周期：方法调用内形成候选；观察结果若需识别，先形成派生需求材料并结束本方法回合，后继新任务再执行识别。
- 并发键：工作包身份；不得直接互调四种方法。
- 逻辑内返回：工作包失效、输入类型不匹配、无观察结果、识别空句柄、无变化、目标丢失。
- 内部错误：已冻结动作入口不存在、结果来源不对应工作包、同包产生冲突结果。
- 恢复：未提交候选可丢弃并重算；已提交结果按结果身份幂等读取，不从日志恢复。
- 漂移退回：新增 D455 产品直通识别入口、识别 / 扫描工作包来源或 ABI 改变、产品映射、结果字段或来源动作语义改变。

## 4. 验证矩阵

| 类别 | 验证 |
| --- | --- |
| 唯一消费 | 同报告竞争仅一个承接；不同报告并发；只读视图不能消费 |
| 任务筹办 | 同任务仅一路进入召回；不同任务并发；筹办可形成新轮次 |
| 冻结 | 三类工作包全字段值式冻结；领取、取消、过期、重放有确定结果 |
| 映射 | 原始逐簇全拒绝；观察 / 跟踪只收匹配 D455 产品；已有基准扫描收扫描变化，首次扫描只收观察链已合法提交的当前数据项；识别只收新任务冻结的存在材料和确定候选范围 |
| 方法隔离 | 观察结果先形成结构化识别需求，再经需求 / 新任务 / 新筹办轮次进入识别；四方法零直接互调 |
| 边界 | 方法不读物理队列，不改承接记录，不互调，不直接写世界事实 |
| 错误 | 前置拒绝与内部错误具名区分；队列确认后无普通失败分支 |
| 集成 | #373 汇合真实提供者后完成编译、并发、自检和端到端验证 |
