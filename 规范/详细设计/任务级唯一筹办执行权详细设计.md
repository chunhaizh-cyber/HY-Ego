# 任务级唯一筹办执行权详细设计

日期：2026-07-21

状态：设计完成 / 代码未实现 / 可形成独立实施切片 / 不等于完整任务筹办闭环

## 1. 设计目标

本设计只解决一个基础并发合同：

```text
不同任务
-> 可以并发筹办

同一个任务
-> 同一时刻只有一个有效筹办占用和执行权
-> 未取得者在方法召回前拒绝
-> 只有持有当前执行权的路径可以召回、选择和形成执行冻结
```

本设计不把裸线程编号、互斥锁、生命周期枚举或事后幂等读回当作业务执行权。执行权来自任务结构中一次原子提交形成的当前筹办占用事实，并由任务身份、占用关系、筹办轮次、结构版本和当前性共同裁决。

本设计建立的是筹办执行权基础设施，不宣称六类筹办闭合结果、任务状态特征全集、D455 工作包或生产任务调度已经完成。

## 3. 当前代码事实

1. `数据操作.需求任务方法.ixx` 已使用当前 `任务生命周期` 关系、状态节点、版本和发生时间承载生命周期迁移，并在一次结构写入会话中失效旧关系、发布新关系。
2. `任务业务服务::迁移生命周期` 把 `已提交` 和 `幂等读回` 都纳入 `任务提交结果::成功()`；调用方只检查 `成功()` 时无法区分本路径是否赢得本轮唯一提交。
3. `方法召回业务服务::召回` 只验证任务处于 `筹办中` 且尚无当前选择，没有验证调用路径是否持有该任务当前筹办执行权。
4. `适配唯一建议` 和 `任务业务服务::提交方法选择` 绑定当前任务生命周期关系与版本，但没有把“本路径是原子取得占用的赢家”作为显式入口合同。
5. `需求任务方法组合器::召回并提交唯一选择` 直接进入召回、适配和选择提交，召回前没有取得任务级执行权。
6. `任务执行组合器::冻结任务执行请求` 只读取筹办中阶段和当前选择，未要求调用路径提交当前筹办执行权材料。
7. `任务执行调度路由` 在提交队列候选前形成冻结，因此可以在同一受保护筹办路径内接收执行权并完成冻结，消费阶段不需要继续持有筹办权。
8. 当前自检覆盖生命周期并发、幂等写入、召回排序和选择发布，但没有证明“同任务 50 路只有一条进入召回、其余 49 路在召回前拒绝”。
9. 当前代码没有完整的任务状态特征存储和六类筹办闭合结果实现；本设计不得用生命周期阶段补造该完成事实。

## 4. 结构映射裁决

### 4.1 当前筹办占用事实

本切片复用一次原子 `已承接 / 待重筹办 -> 筹办中` 生命周期提交产生的结构，不建立第二套进程内占用表：

```text
任务稳定身份
+ 当前筹办中状态节点
+ 当前任务生命周期关系
+ 当前生命周期版本
+ 当前记录有效性
+ 从生命周期历史确定的筹办轮次
-> 当前筹办占用事实
```

阶段枚举 `筹办中` 单独不构成执行权。只有本路径收到该次迁移的精确 `已提交`，并读回以上结构全部当前一致，才取得执行权。`幂等读回`、同阶段读回、事后发现已有选择或锁竞争成功都不授予执行权。

### 4.2 强类型执行权材料

新增值式结构：

```text
任务筹办执行权
- 任务稳定句柄
- 当前筹办状态节点
- 当前筹办占用关系
- 当前生命周期版本
- 筹办轮次
- 触发幂等材料编号
- 触发发生时间
```

完整性要求：全部身份和版本非零，任务处于筹办中，占用关系为该任务唯一当前生命周期关系，状态节点和触发材料与取得时读回一致，筹办轮次与生命周期历史一致。

执行权是结构事实的值式能力投影，不是第二权威记录。复制该值不产生新执行权；任一字段被修改、旧关系失效、阶段迁移或新筹办轮次形成后，旧值立即失效。

### 4.3 状态分类

```text
任务筹办执行权状态
- 已取得
- 并发拒绝
- 入口拒绝
- 当前性漂移
- 许可拒绝
- 版本漂移
- 内部不一致
```

`并发拒绝` 是正常前置返回，表示同一任务已有当前筹办占用；它不进入召回，也不生成任务失败、等待、需求或日志机器事实。

## 5. 原子取得规则

新入口 `取得任务筹办执行权` 固定执行：

```text
验证任务身份、触发幂等材料和发生时间
-> 权威读取当前任务
-> 只接受 已承接 或 待重筹办
-> 使用当前生命周期关系和版本形成筹办中迁移规格
-> 原子提交迁移
-> 精确状态必须为 已提交
-> 完整读回当前关系、状态、版本和历史轮次
-> 返回任务筹办执行权
```

分类规则：

1. 当前已经是 `筹办中`：返回 `并发拒绝`，不得调用迁移幂等读回取得权。
2. 两条路径从同一前态并发：只有结构提交状态精确为 `已提交` 的路径取得权；另一条根据权威读回返回 `并发拒绝` 或当前性漂移，不进入业务区间。
3. 迁移返回 `幂等读回`：不取得权。原赢家继续使用首次返回的权值；重试调用不重新授权。
4. 已进入排队、执行、等待、完成、失败或取消：按阶段合同拒绝，不发起召回。
5. 前置通过后出现双当前关系、历史轮次不一致、已提交但读回不是唯一当前占用：`内部不一致`，停止依赖路径并追根因。

## 6. 受保护入口

### 6.1 方法召回

`方法召回请求` 增加任务筹办执行权值式材料。`方法召回业务服务::召回` 在读取方法登记根或候选索引前，先调用任务服务复核：

```text
请求任务 == 执行权任务
当前阶段 == 筹办中
当前关系 / 状态 / 版本 == 执行权
当前筹办轮次 == 执行权
当前无已发布选择
```

缺执行权、旧执行权或错任务都在候选读取前拒绝。召回服务继续只形成非权威候选，不取得或释放执行权。

### 6.2 选择提交

召回结果保留本次执行权快照。`适配唯一建议` 只有在执行权仍当前时才形成选择提交请求；选择提交同时绑定：

```text
任务
筹办占用关系与版本
筹办轮次
方法生命周期关系与版本
排序和召回规则版本
来源候选数量
```

同一执行权重复读回同一选择只属于选择记录的幂等性，不为其它路径补发执行权。若另一条路径已经绕过门禁进入召回或选择，必须返回内部不一致并追查根因。

### 6.3 执行冻结

`任务执行调度请求` 携带执行权快照字段；它们只是传输材料，不成为第二权威。`任务执行组合器::冻结任务执行请求` 在读取方法规格和动作入口前复核同一执行权，并要求当前选择来源生命周期关系、版本和筹办轮次与执行权一致。

冻结成功后，`任务执行调度路由` 才准备队列候选并提交 `筹办中 -> 排队中`。该阶段迁移原子失效筹办占用关系，执行权随之失效。队列消费和方法执行不得继续使用筹办权。

### 6.4 组合器

`需求任务方法组合器` 增加受控筹办入口：

```text
取得执行权
-> 读取任务和需求材料
-> 方法召回
-> 适配唯一建议
-> 提交选择
-> 返回选择结果和仍当前的执行权
```

后继调用方必须把同一权值交给冻结入口。该组合入口只覆盖“已有唯一就绪候选”的基础路径；无候选、非唯一候选、等待、能力缺口、子目标、目标已完成和授权失效仍须由任务筹办闭合结果设计收束。本切片不得把这些结果包装成普通失败、待重筹办或无结构回退，也不得因此宣称完整筹办已实现。

## 7. 重复筹办与权值失效

任务可以重复筹办：

```text
前轮冻结失效或执行结果触发待重筹办
-> 待重筹办保持前轮历史
-> 新一轮原子进入筹办中
-> 新关系、新版本、新轮次
-> 新执行权
```

旧执行权在以下任一情况发生时失效：

- 当前筹办关系失效；
- 生命周期离开筹办中；
- 新筹办轮次形成；
- 任务身份、状态节点、关系或版本不匹配；
- 选择来源不再对应该占用；
- 任务结构读回内部不一致。

新一轮不得覆盖前轮关系、选择和冻结历史。执行权失效不删除历史结构。

## 8. 并发矩阵

| 场景 | 预期 |
| --- | --- |
| 同一任务 50 路同时取得权 | 1 路 `已取得`，49 路 `并发拒绝`；只有 1 路出现非空召回结果 |
| 同一任务重复调用相同触发材料 | 原赢家首次 `已取得`；重试不得用 `幂等读回` 再取得权 |
| 同一任务两路已进入召回 | 内部并发规则破坏，不按同义选择成功 |
| 不同任务同时筹办 | 均可取得各自执行权并进入召回 |
| 同一根需求产生的不同任务 | 均可并发，不以根需求串行 |
| 不同任务选择同一可复用方法 | 均可并发，不以方法身份串行 |
| 旧权值用于新轮次 | 当前性漂移，在候选或动作读取前拒绝 |
| 选择由当前权形成，冻结使用错权 | 冻结前拒绝，不形成队列候选 |
| 冻结后阶段迁移为排队中 | 原权失效，后续不能再次召回或冻结 |

## 9. 文件与所有权

本设计不新建代码文件，不修改工程登记和入口。结构、服务和调用点在现有模块内原地收窄：

| 所有者 | 责任 |
| --- | --- |
| `数据操作.需求任务方法.ixx` | 执行权值式材料、筹办轮次结构读回、选择与占用一致性 |
| `服务.任务.ixx` | 原子取得、当前性复核和状态映射 |
| `服务.方法召回.ixx` | 召回前执行权门禁、召回结果绑定 |
| `组合.需求任务方法.ixx` | 取得权后召回与选择编排 |
| `组合.任务执行.ixx` | 冻结前同权复核 |
| `线程/协议.任务执行请求.ixx` | 传输执行权快照字段，不裁决权威 |
| `线程/路由.任务执行调度.ixx` | 同权冻结、队列候选和排队迁移顺序 |
| 既有相关自检与兼容请求 | 迁移为显式取得权并新增并发验收 |

不得新增进程局部占用表、全局任务筹办大锁、线程编号字段、日志裁决、数据库锁或第二任务状态仓库。

## 10. 失败与内部错误边界

| 边界 | 正常前置返回 | 内部逻辑错误 |
| --- | --- | --- |
| 取得权 | 错身份、错阶段、已有占用、版本漂移 | 已提交后无唯一当前关系、轮次错乱 |
| 召回 | 无权、旧权、错任务、冻结后再召回 | 两条不同权值同时通过同任务门禁 |
| 选择 | 候选不唯一、方法合法失效 | 选择来源与权值不一致仍提交 |
| 冻结 | 权值失效、选择或方法合法漂移 | 错权冻结成功、冻结前既有结构缺口 |
| 排队迁移 | 队列容量或合法当前性变化 | 队列确认后阶段未迁移或占用仍当前 |

诊断只做人读。内部不一致必须停止受影响路径，不能改写为任务失败、等待、普通需求或同义幂等成功。

## 11. 验收

至少验证：

1. 同任务 50 线程只有一个取得权和一个进入召回；
2. 49 个失败结果在召回前形成，候选组、建议和选择均为空；
3. 不同任务、同根不同任务、同方法不同任务可以并发；
4. 精确 `已提交` 才授予执行权，`幂等读回` 不授予；
5. 召回、适配、选择和冻结逐步复核相同权值；
6. 排队迁移后旧权失效；
7. 新筹办轮次生成新权且历史仍可读；
8. 同任务双路径越过召回门禁被归类为内部不一致；
9. 既有任务执行、任务结果结算、自我治理和运行期兼容自检语义不漂移；
10. `入口.cpp`、工程文件、D455、控制面板、SQL 和核心仓库差异为零。
