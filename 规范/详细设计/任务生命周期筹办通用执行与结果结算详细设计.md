## 1. 定位

本设计把 #201 发现的任务系统断点拆为五个不可越级的实施切片：

```text
TASK-LIFECYCLE-S1：唯一当前态关系、生命周期版本和审计历史读回。
TASK-LIFECYCLE-S2：合法迁移矩阵、并发版本准入、终态封口和专用失败 / 取消证据。
TASK-PLANNING-S1：筹办材料、方法召回建议和任务服务独占选择提交。
TASK-EXECUTION-S1：冻结执行请求、管理 / 工作线程调度、受控执行和强类型非权威回执。
TASK-RESULT-S1：正式结果复核、目标比较、任务完成和独立需求结算。
```

当前代码只有历史实例状态、任意非零 I64 状态辅助、固定安全根样例、非权威线程消息和只读上行桥。本文描述的是预期正式契约，不得写成当前实现事实。

## 5. S1 唯一当前态与历史

### 5.1 关系与值式材料

关系固定为：

```text
关系类型::任务生命周期 = 14
方向：任务 -> 实例状态
有效数量：同一完整任务句柄恰好 1
顺序号：生命周期版本，从 1 严格加一
状态：当前关系有效，旧关系已失效
```

建议值式 DTO：

```text
任务生命周期当前材料
  任务
  来源需求
  任务虚拟存在
  场景
  当前关系
  当前状态节点
  当前阶段
  生命周期版本
  发生时间戳

任务生命周期历史项
  生命周期关系审计
  状态节点
  阶段
  版本
  发生时间戳
  是否当前
```

读取必须先按关系类型和完整任务端点取得有效当前关系，再读目标实例状态的主体、场景、状态值和时间。缺失、多条、版本为零、端点或状态材料不一致属于内部结构矛盾，不得返回“猜测当前态”。

### 5.2 创建发布

任务创建在 #217 同域独占结构许可中完成：

```text
任务节点和主信息
-> 任务虚拟存在与来源需求 / 场景承接关系
-> 已创建实例状态
-> 任务生命周期关系版本 1
-> 完整读回
-> 发布任务创建结果
```

进入第一项写入前的无效输入是逻辑内返回。进入写入后任一内部结果不及预期必须追根因；不得发布没有唯一当前态的任务，也不得靠入口扫描清理。

### 5.3 旧入口收口

`记录任务状态` 和 `记录任务生命周期状态` 收口为任务服务内部辅助，外部调用点改用正式迁移入口。既有普通 `引用` 历史在本切片不物理删除；新权威读取只看专用关系，历史读取可以明确标记兼容材料与正式换代材料。

## 6. S2 合法迁移

### 6.1 固定矩阵

| 当前阶段 | 允许目标 | 入口 / 证据 |
| --- | --- | --- |
| 已创建 | 已承接 | 任务服务确认来源需求承接配对完整 |
| 已承接 | 筹办中 | 筹办请求和当前需求目标材料完整 |
| 筹办中 | 排队中 | 冻结执行请求已成功进入正式私有队列，随后任务服务复核 |
| 排队中 | 执行中 | 工作线程已取得当前冻结请求，任务服务复核选择和版本 |
| 执行中 | 等待中 | 当前动作允许等待且有结构化等待证据 |
| 等待中 | 执行中 | 同一当前选择和可恢复执行证据仍有效 |
| 排队中 / 执行中 / 等待中 | 待重筹办 | 当前请求 / 方法过期、结果未达标或正式重筹办证据 |
| 待重筹办 | 筹办中 | 旧选择已失效、当前需求仍未结算 |
| 执行中 | 已完成 | 仅 TASK-RESULT-S1 的目标匹配结果入口 |
| 任一非终态 | 失败 | 专用失败证据和允许来源 |
| 任一非终态 | 取消 | 专用取消授权；执行中另需动作停止证据 |

除表中边外全部写前拒绝。已完成、失败、取消没有出边；已创建不能直接完成，筹办中不能直接执行，等待中不能直接完成。

### 6.2 请求与提交

```text
任务生命周期迁移请求
  任务
  预期当前关系
  预期生命周期版本
  目标阶段
  发生时间戳
  幂等材料编号
  来源任务 / 方法 / 动作 / 状态 / 动态证据（按入口选填）
```

提交顺序：

```text
写前验证请求形状和目标矩阵
-> 取得 #217 同域独占许可
-> 重读任务、当前关系、当前状态、来源需求、场景和主体
-> 重读专用证据
-> 创建新实例状态和版本 N+1 新关系
-> 使版本 N 旧关系失效

-> 返回值式结果
```

提交实现可以根据 #217 正式能力调整内部发布顺序，但对共享读者只允许看见换代前或换代后完整结构。进入唯一提交点后不得有普通失败返回；不做版本倒退回滚，不宣称跨重启原子恢复。

## 7. S3 筹办、召回与任务服务选择

### 7.1 实际接口门禁

#223 是 #209 的明确后继。执行前必须确认 #203-#208 已形成方法登记、规格、派生索引、召回、复判和确定性排序，#209 已按实际接口完成任务授权 / 选择接线只读扫描。若 #209 的正式授权形状不是本设计的双向需求承接配对，或建议 DTO 不含来源版本，#223 不得改代码，必须退回修订。

### 7.2 筹办请求

任务处于筹办中时冻结：

```text
任务完整句柄与生命周期版本
来源需求与目标状态
归属(需求 -> 任务)和引用(任务 -> 需求)配对关系
主体与场景
方法登记 / 规格 / 索引来源版本
允许的执行能力类别
```

方法召回链只返回值式建议。无候选、候选过期、条件不满足或语义并列均逻辑内返回，任务保持筹办中。不得用句柄顺序、容器遍历顺序或线程完成顺序打破语义并列。

### 7.3 选择提交

只有任务服务可提交：

```text
重读当前生命周期仍为筹办中
-> 重读双向授权配对
-> 重读唯一建议及其登记 / 规格 / 召回 / 复判 / 排序来源版本
-> 重读方法首和主动动作入口可调用材料
-> 写唯一当前任务方法关系
-> 正反读回任务、方法和来源版本
```

首次选择不自动迁移到排队中。待重筹办后的新选择必须先使旧选择关系失效并保留审计，再创建新当前选择；不允许两个有效选择并存。

## 8. S4 通用执行与强类型回执

### 8.1 冻结请求

新增版本化强类型请求候选：

```text
任务执行请求版本 = 1
任务执行冻结请求
  请求编号与幂等键
  任务、来源需求、生命周期关系和版本
  当前任务方法关系、方法首和方法登记版本
  方法规格版本、动作入口和动作规格
  主体、场景、目标状态
  创建时间、截止时间和尝试序号
```

普通 `运行消息` 只携带请求编号、类型和路由摘要作为信封。完整请求存放在任务管理线程持有的私有、有锁、有界强类型队列中；不得把完整身份压缩成裸节点编号后再猜回句柄。

### 8.2 入队与生命周期

```text
任务执行服务形成当前冻结请求
-> 管理线程尝试写私有队列
-> 满、停止、重复、过期：逻辑内返回，生命周期不变
-> 入队成功：任务服务重读并迁移 筹办中 -> 排队中
-> 工作线程出队：任务服务重读并迁移 排队中 -> 执行中
```

入队成功本身不是任务事实。若入队后生命周期复核发现请求已经过期，过期请求按值式结果丢弃，权威生命周期只能经任务服务的合法迁移处理。

### 8.3 受控执行

第一版适配表只登记已有正式领域入口，例如特征状态写入方法。适配项必须绑定方法登记、规格版本、动作入口类型和领域服务调用形状；它不是任意函数指针表，也不允许插件、脚本、SQL、外部进程、外设或用户回调。

工作线程只调用任务执行服务。实际状态和动作动态由方法服务或目标领域服务形成并完整读回。写后证据不完整属于追根因；允许等待或重筹办则返回专用材料并由任务服务迁移，不伪造成功回执。

### 8.4 强类型回执

回执至少包含：

```text
协议版本、请求编号、尝试序号
任务、生命周期关系和来源版本
任务方法关系、方法首、动作入口和规格版本
实际状态、动作动态、主体和场景
执行开始 / 完成时间和材料序号
```

协议、队列和上行桥只证明材料形状与非权威顺序。回执不能直接写任务结果、完成或需求结算。

## 9. S5 结果、完成与需求结算

### 9.1 正式结果治理

上行桥继续只读复核并投递治理消息。任务结果服务收到消息后必须再次读取：

```text
任务当前生命周期和版本仍匹配执行请求。
任务当前选择、方法登记、规格和动作入口仍匹配。
实际状态的主体、场景、值材料和来源证据完整。
动作动态的来源方法 / 动作、前后材料和时间完整。
回执编号、尝试序号和既有结果不存在冲突。
```

过期、精确重复或允许拒绝在写前逻辑内返回。同一当前版本不变但权威结构与已准入回执自相矛盾时追根因，不能把协议通过当成领域成功。

### 9.2 目标比较

状态服务新增结构化比较候选：

```text
状态目标比较结果
  已达成
  未达成
  材料不足
  内部不一致
```

比较基于目标状态定义、实际状态材料、主体和场景，不向任务服务暴露“比较两个裸 I64 即得业务语义”的快捷入口。未达成时保留状态与动态事实，不记录最终任务结果；任务按当前证据进入等待中或待重筹办。

### 9.3 结果与完成提交

目标达成后，在 #217 同域独占许可中：

```text
重读当前执行中关系和版本
-> 重读目标比较与动作证据
-> 写唯一任务实际结果关系
-> 经专用完成入口换代到已完成
-> 读回实际结果、完成关系和生命周期历史
```

精确重复读回既有完成材料，不创建第二结果或第二生命周期版本。同任务同版本的冲突结果是内部不一致。失败、取消任务不得被结果回执改成已完成。

### 9.4 独立需求结算

任务完成读回后才调用需求服务正式结算。结算入口重新核对来源任务、需求目标、任务实际状态、动作动态、主体和场景。结算暂不可执行或允许拒绝时保留任务已完成和动作事实，返回待结算材料并允许同证据重试；不得回滚任务结果，也不得把需求直接标记为满足。

## 10. 目标接口候选

以下均为假定契约，不是当前实现事实：

```text
任务服务::读取任务当前生命周期(任务, 状态服务)
任务服务::读取任务生命周期历史(任务, 状态服务)
```

| 结算暂不可执行 | 逻辑内返回 | 保留任务已完成，返回待结算材料 |
| 当前关系多条 / 缺失或版本 / 状态自相矛盾 | 追根因解决 | 停止依赖路径，不猜当前态、不扫描修复 |

| 任务完成写后不一致 | 追根因解决 | 停止需求结算；不伪造完成材料 |
