## 1. 定位

本设计把 #201 发现的任务系统断点拆为五个不可越级的实施切片：

```text
TASK-LIFECYCLE-S1：唯一当前态关系、生命周期版本和审计历史读回。
TASK-LIFECYCLE-S2：合法迁移矩阵、并发版本准入、终态封口和专用失败 / 取消证据。
TASK-PLANNING-S1：筹办材料、方法召回建议和任务服务独占选择提交。
TASK-EXECUTION-S1：冻结执行请求、管理 / 工作线程调度、受控执行和强类型非权威回执。
TASK-RESULT-S1：正式结果复核、目标比较、任务完成和独立需求结算。
```

当前代码只有历史实例状态、任意非零 I64 状态辅助、固定安全根样例、非权威线程消息和只读上行桥。本文描述的是预期正式契约，不得写成当前实现事实。

JY-292、JY-294、JY-296 已根据 #221 三轮实际接口复核补齐存在创建链、需求承接读取链与原子服务候选撤销能力；JY-297 只修订关系枚举负例的回归映射，不改变生产合同。第 5 节和第 15 节以这些修订为准。

## 5. S1 唯一当前态与历史

### 5.1 关系与值式材料

关系固定为：

```text
关系类型::任务生命周期 = 14
方向：任务 -> 实例状态
有效数量：同一完整任务句柄恰好 1
顺序号：生命周期版本，从 1 严格加一
状态：当前关系有效，旧关系已失效
```

建议值式 DTO：

```text
任务生命周期当前材料
  任务
  来源需求
  任务虚拟存在
  场景
  当前关系
  当前状态节点
  当前阶段
  生命周期版本
  发生时间戳

任务生命周期历史项
  生命周期关系审计
  状态节点
  阶段
  版本
  发生时间戳
  是否当前
```

读取必须先按关系类型和完整任务端点取得有效当前关系，再读目标实例状态的主体、场景、状态值和时间。缺失、多条、版本为零、端点或状态材料不一致属于内部结构矛盾，不得返回“猜测当前态”。

### 5.2 创建发布

任务创建在 #217 同域独占结构许可中完成：

```text
任务服务验证同域独占令牌并创建任务节点和主信息
-> 存在服务验证同一令牌并创建任务虚拟存在及来源引用
-> 任务服务写来源需求 / 场景 / 目标状态承接关系
-> 状态服务验证同一令牌并创建已创建实例状态
-> 任务生命周期关系版本 1
-> 完整读回
-> 发布任务创建结果
```

任务虚拟存在是任务运行特征与过程账的正式承载，不采用“复用需求主体 / 目标宿主而不创建虚拟存在”的替代合同。它必须由存在服务创建；任务服务不得直接写存在节点。

创建提交的正式顺序为：

```text
自检取得一次独占许可
-> 任务服务验证同一令牌
-> 需求服务以同一令牌重读需求有效性与承接材料
-> 需求、主体、宿主、场景和目标状态仍完整且相互一致
-> 创建任务主信息 / 节点
-> 存在服务创建任务虚拟存在
-> 状态服务创建已创建实例状态
-> 创建需求承接、任务生命周期版本 1 等关系
-> 完整读回
-> 发布任务创建结果
```

需求失效、承接材料为空或材料已漂移，属于第一笔任务写入前的逻辑内返回；不得用许可前快照继续创建。进入第一笔写入后才适用本节后述的许可内精确逆序清理。

存在服务令牌路径固定新增以下最小重载：

```text
创建实际存在(const 结构事务令牌&)
创建虚拟存在(节点句柄 来源节点, const 结构事务令牌&)
读取存在(节点句柄, const 结构事务令牌&) const
存在是否有效(节点句柄, const 结构事务令牌&) const
```

存在服务构造函数附加默认空的值式 `结构事务接线`；令牌写入口必须先验证独占令牌，再向主信息、节点和关系仓库下传同一令牌。普通无令牌构造和重载保持现有兼容语义，但不参加 #221 A09-A10 事务验收。`创建存在概念根` 不属于 #221 路径，不为本切片增加令牌重载。

进入第一项写入前的无效输入是逻辑内返回。存在 / 状态服务的令牌创建入口必须分别返回不可伪造的 `任务虚拟存在未发布候选` 与 `任务初态未发布候选`：类型不可默认构造、不可复制，只绑定创建它的服务实例、当前独占令牌身份和本次精确句柄组；只在调用栈存活，不进入长期 DTO、消息、仓库、日志或显示。

进入写入后任一内部结果不及预期必须在同一独占许可内追根因并按所有权逆序收口：任务服务先撤销自己创建的生命周期 / 承接关系，再把状态候选交回状态服务撤销，把虚拟存在候选交回存在服务撤销，最后撤销任务节点和主信息。任务服务不得直接删除状态 / 存在节点或其内部关系。完整读回成功后，任务服务分别调用原服务的候选确认入口，使撤销能力失效；确认只改变调用栈候选状态，不写机器结构。

同一候选在成功撤销后重复撤销返回幂等已撤销且不再写结构。首次撤销前发现候选句柄缺失 / 冲突、服务实例不匹配、令牌不匹配、候选已确认发布或只撤销部分结构，均进入追根因，不能冒充幂等。每层只清自己创建且候选记录的句柄，不扫描、不猜测、不撤销既有结构。不得发布没有唯一当前态的任务，也不得在释放许可后留下半任务、孤立存在或孤立状态。

### 5.3 旧入口收口

`记录任务状态` 和 `记录任务生命周期状态` 收口为任务服务内部辅助，外部调用点改用正式迁移入口。既有普通 `引用` 历史在本切片不物理删除；新权威读取只看专用关系，历史读取可以明确标记兼容材料与正式换代材料。

## 7. S3 筹办、召回与任务服务选择

### 7.1 实际接口门禁

#223 是 #209 的明确后继。#203-#208 已形成方法登记、规格、派生索引、召回、复判和确定性排序；#209 已确认双向关系结构存在，但公开双向授权、唯一当前态、当前选择关系 / 版本、换代审计和精确动作桥尚未形成。

执行 #223 前必须读取 #221 / #222 的实际任务生命周期接口，并逐项复核本节假定字段。若 #221 / #222 改变当前关系 / 版本形状，或任务服务自有 DTO 无法作为 `方法召回服务.h` 与 `方法服务.h` 的共同单向依赖，不得改代码，必须退回设计。执行窗口不得新增中立协议头、让任务服务反向 include 方法侧头文件或新增任务筹办服务来临时绕开。

### 7.2 双向授权与筹办材料

任务服务新增公开只读材料候选：

```text
任务承接授权材料
  任务
  来源需求
  需求目标状态
  需求到任务归属关系完整句柄 / 版本
  任务到需求引用关系完整句柄 / 版本
  当前生命周期关系完整句柄 / 生命周期版本 / 当前阶段
  任务虚拟存在、主体与场景
```

两条承接关系必须同时有效，分别以需求和任务为起点、汇合到同一任务 / 需求；任一关系缺失、多条、完整句柄 / 版本或端点自相矛盾时不得降级为单向授权。缺失是写前逻辑内返回，当前同版本结构矛盾进入追根因。

只有当前阶段精确为筹办中时，调用方才可把授权材料连同任务目标交给 #206-#208。方法召回链仍只返回完整值式建议。无候选、候选过期、条件不满足、语义并列以及没有独立权威来源的 `策略唯一` 均逻辑内返回，任务保持筹办中；不得用句柄、容器遍历或线程完成顺序打破并列。

### 7.3 无环扁平选择请求

任务方法选择请求固定由 `任务服务.h` 定义；不得嵌套 `方法选择建议材料`、候选组、复判材料或方法召回服务类型。方法召回侧使用既有单向依赖，在提交前重读完整 S6 批次和全部来源，只有 `唯一可建议` 可以适配为：

```text
任务方法选择请求
  规则版本
  幂等材料编号
  任务、来源需求、目标状态
  双向授权两条关系完整句柄 / 版本
  当前筹办生命周期关系 / 版本
  建议方法
  完整候选批次身份 / 签名
  派生索引发布版本
  方法登记关系 / 版本
  方法生命周期关系 / 版本
  结果规格关系 / 版本
  条件策略、条件规格和运行上下文来源身份 / 版本
  主动动作入口、归属方法和来源关系 / 版本
```

这里的“扁平”只表示不嵌套方法召回 DTO，不表示可以裁剪来源。方法召回侧负责语义复核并证明请求完整对应刚复核的整个建议；任务服务只理解稳定句柄、关系、版本、批次身份和本领域阶段，在提交边界结构性重读这些来源，不复制方法登记 / 规格语义，也不反向调用方法召回或方法服务。

### 7.4 选择提交与审计

只有任务服务可提交：

```text
验证任务自有扁平请求形状
-> 取得狭窄任务 / 同域写边界
-> 重读当前生命周期仍为请求中的筹办关系 / 版本
-> 重读双向授权配对两条关系完整句柄 / 版本
-> 重读完整批次身份、索引发布版本、方法登记 / 生命周期 / 规格 / 条件 / 上下文 / 动作来源
-> 处理既有当前选择的幂等、冲突或正式重筹办换代
-> 写唯一有效顺序号 30 引用(任务 -> 方法)
-> 正反读回选择关系完整句柄 / 版本、所属生命周期版本和全部来源
```

当前选择 DTO 至少返回任务、方法、选择关系完整句柄 / 版本、所属筹办生命周期关系 / 版本、双向授权材料和是否幂等。方法登记 / 规格 / 动作语义不复制进任务权威结构，由方法服务在动作桥读取时重读。关系顺序号固定为 30，只标识任务方法选择槽位，不承担选择版本；关系完整句柄 / 版本才是当前选择身份。

精确同请求幂等读回。同一生命周期版本已有不同方法时写前拒绝，不用新排序结果覆盖。只有 #222 已正式把任务从待重筹办换代回新的筹办中生命周期版本时，任务服务才可使旧选择失效并保留审计，再创建新当前选择；每条关系操作保持既有原子语义，不新增跨多关系回滚。多条有效当前选择、端点矛盾或写后唯一读回失败都必须追根因。

### 7.5 精确动作桥与下游接口

任务服务必须提供：

```text
读取任务承接授权材料(任务)
读取当前任务方法选择材料(任务)
复核当前任务方法选择仍为当前(当前选择材料)
读取任务方法选择审计(任务)
```

方法服务可单向包含任务服务以读取 `当前任务方法选择材料`；动作桥只接受该材料，并复核精确选择关系完整句柄 / 版本、任务、方法、所属生命周期版本，再由方法服务重读登记、生命周期、规格、动作入口归属和主动可调用来源。任意 task -> method 引用、失效选择、旧生命周期选择或只有方法节点的材料均不得通过。

首次选择不自动迁移到排队中。#214 只消费任务服务的双向授权和精确当前选择只读材料；#224 在冻结、入队、出队和执行前继续复核同一选择关系 / 版本，不得自行从任意引用重建选择。

## 8. S4 通用执行与强类型回执

### 8.1 冻结请求

新增版本化强类型请求候选：

```text

任务执行冻结请求
  请求编号与幂等键
  任务、来源需求和双向承接授权两条关系完整句柄 / 版本
  生命周期关系 / 版本和当前筹办阶段
  当前任务方法选择关系完整句柄 / 版本和所属生命周期版本
  方法首和方法登记 / 生命周期版本
  方法规格版本、动作入口和动作规格
  主体、场景、目标状态
  创建时间、截止时间和尝试序号
```

普通 `运行消息` 只携带请求编号、类型和路由摘要作为信封。完整请求存放在任务管理线程持有的私有、有锁、有界强类型队列中；不得把完整身份压缩成裸节点编号后再猜回句柄。

### 8.2 入队与生命周期

```text
任务执行服务重读双向授权和精确当前选择并形成冻结请求
-> 管理线程尝试写私有队列
-> 满、停止、重复、过期：逻辑内返回，生命周期不变
-> 入队成功：任务服务重读授权、选择关系 / 版本并迁移 筹办中 -> 排队中
-> 工作线程出队：任务服务重读同一精确选择并迁移 排队中 -> 执行中
```

入队成功本身不是任务事实。若入队后生命周期复核发现请求已经过期，过期请求按值式结果丢弃，权威生命周期只能经任务服务的合法迁移处理。

### 8.3 受控执行

第一版适配表只登记已有正式领域入口，例如特征状态写入方法。适配项必须绑定方法登记、规格版本、动作入口类型和领域服务调用形状；它不是任意函数指针表，也不允许插件、脚本、SQL、外部进程、外设或用户回调。

工作线程只调用任务执行服务。实际状态和动作动态由方法服务或目标领域服务形成并完整读回。写后证据不完整属于追根因；允许等待或重筹办则返回专用材料并由任务服务迁移，不伪造成功回执。

### 8.4 强类型回执

回执至少包含：

```text
协议版本、请求编号、尝试序号
任务、双向授权关系、生命周期关系和来源版本
精确当前任务方法选择关系 / 版本、方法首、动作入口和规格版本
实际状态、动作动态、主体和场景
执行开始 / 完成时间和材料序号
```

协议、队列和上行桥只证明材料形状与非权威顺序。回执不能直接写任务结果、完成或需求结算。

## 9. S5 结果、完成与需求结算

### 9.1 正式结果治理

上行桥继续只读复核并投递治理消息。任务结果服务收到消息后必须再次读取：

```text

实际状态的主体、场景、值材料和来源证据完整。
动作动态的来源方法 / 动作、前后材料和时间完整。
回执编号、尝试序号和既有结果不存在冲突。
```

过期、精确重复或允许拒绝在写前逻辑内返回。同一当前版本不变但权威结构与已准入回执自相矛盾时追根因，不能把协议通过当成领域成功。

### 9.2 目标比较

状态服务新增结构化比较候选：

```text
状态目标比较结果
  已达成
  未达成
  材料不足
  内部不一致
```

比较基于目标状态定义、实际状态材料、主体和场景，不向任务服务暴露“比较两个裸 I64 即得业务语义”的快捷入口。未达成时保留状态与动态事实，不记录最终任务结果；任务按当前证据进入等待中或待重筹办。

### 9.3 结果与完成提交

目标达成后，在 #217 同域独占许可中：

```text
重读当前执行中关系和版本
-> 重读目标比较与动作证据
-> 写唯一任务实际结果关系
-> 经专用完成入口换代到已完成
-> 读回实际结果、完成关系和生命周期历史
```

精确重复读回既有完成材料，不创建第二结果或第二生命周期版本。同任务同版本的冲突结果是内部不一致。失败、取消任务不得被结果回执改成已完成。

### 9.4 独立需求结算

任务完成读回后才调用需求服务正式结算。结算入口重新核对来源任务、需求目标、任务实际状态、动作动态、主体和场景。结算暂不可执行或允许拒绝时保留任务已完成和动作事实，返回待结算材料并允许同证据重试；不得回滚任务结果，也不得把需求直接标记为满足。

### 15.6 JY-294 需求承接材料实际接口修订

#221 第二次执行前复核确认，`任务服务::按需求创建任务` 在外层独占许可内首先调用无令牌 `需求服务::需求是否有效` 与 `读取需求承接材料`。两者会进入已接域节点 / 关系仓库并同线程重复取得结构许可。任务服务绕过需求服务直接读仓库违反领域服务边界；许可前值式快照又缺少正式版本复核合同，会把需求漂移转化为 TOCTOU。


### 15.7 JY-296 原子服务未发布候选能力修订
