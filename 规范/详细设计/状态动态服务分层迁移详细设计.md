# 状态动态服务分层迁移详细设计

日期：2026-07-13

状态：正式详细设计 / JY-323 修订 / #266 重新待执行施工依据 / #272、725 已完成且不得重复执行 / 未改代码

## 1. 目标

在 `#265` 已验证的分层模式上，为状态和动态建立新的无原始令牌业务服务面：

```text
调用方
-> 状态业务服务 / 动态业务服务 / 状态动态组合器
-> 数据操作.状态动态
-> #272 扩展后的结构写入执行器和不透明会话
-> 四仓库
```

本设计只形成五条隔离新路径，不修改既有 `状态服务.h`、`动态服务.h`，不迁移生产调用，也不删除兼容入口。

## 2. 当前事实与设计断点

### 2.1 当前状态服务

当前 `状态服务.h` 同时存在无令牌创建、带令牌候选创建和候选确认撤销路径。实例状态由以下结构承载：

```text
状态节点类型
主信息槽位 0：状态值
主信息槽位 1：发生时间戳
主信息槽位 2：幂等编号低 32 位
主信息槽位 3：幂等编号高 32 位
场景 -> 状态：运行期临时
状态 -> 存在：引用
状态 -> 场景：引用
```

抽象状态只使用长期状态节点和主信息值，不应携带运行期临时关系或发生时间戳。

### 2.2 当前动态服务

当前 `动态服务.h` 可使用状态服务创建完整前后状态，也保留一条直接创建裸值材料状态的兼容路径。实例动态由以下结构承载：

```text
动态节点类型
主信息槽位 1：发生时间戳
场景 -> 动态：运行期临时
动态 -> 存在：引用
动态 -> 场景：引用
动态 -> 被改变目标：因果来源，顺序号 1
动态 -> 前状态：因果来源，顺序号 2
动态 -> 后状态：因果来源，顺序号 3
动态 -> 来源动作：因果来源，顺序号 4，可选
```

JY-321 发现原设计的 `0 / 1 / 2 / 4` 与当前三组实例动态写入及 `读取动态材料` 的正式合同不一致。JY-323 裁决本批保持当前 `1 / 2 / 3 / 4` 权威合同；`#266` 不实施顺序号迁移，不增加兼容读取，也不处理历史结构迁移。

抽象动态只使用长期动态节点和主信息材料，不应携带发生时间戳或运行期临时关系。

### 2.3 必须前置的会话能力

`#272 / CORE-SESSION-S2` 已在现有会话上实现并验证候选 I64 值写集，以及同一独占许可内的节点类型、节点主信息、I64 值和按源节点 / 关系类型关系记录组读取。`#272`、阶段 725 已完成并归档，不得由 `#266` 重复实施；执行窗口仍须逐签名重读实际公开面。

## 3. 分层职责

### 3.1 数据操作层

新增 `状态动态数据操作`，它是本批生产路径中唯一允许导入结构写入执行器的模块。职责：

```text
提供值式主键、节点、状态和动态材料读取
把类型化状态 / 动态写入规格提交给执行器
在同一独占会话内复核已有节点、值和关系
创建候选主信息、节点、关系和主键绑定
完成同令牌读回、请求提交或精确撤销
映射结构写入结果，不做上层策略
```

数据操作层可以私有持有四仓库引用、事务接线和关系仓库编号，但不得向业务服务返回这些对象。它不导出原始令牌、许可、会话或候选。

### 3.2 状态业务服务

`状态业务服务` 负责：

```text
创建抽象状态
创建完整实例状态
读取抽象 / 实例状态值式材料
形成实例状态写入规格
裁决状态主键幂等和状态结构业务不变量
```

`实例状态写入规格` 至少包含主键、场景、主体、值、时间戳。规格只能由状态业务服务在完成入口检测后形成；组合器和动态业务服务不得自行拼装未经状态服务裁决的裸状态结构。

### 3.3 动态业务服务

`动态业务服务` 负责：

```text
创建抽象动态
使用两个已有完整实例状态记录状态变化动态
读取抽象 / 实例动态值式材料
裁决动态主键幂等、目标和来源动作角色
```

读取前后状态时必须调用状态业务服务的值式入口。提交写入后，数据操作层在同一独占会话内再次按状态结构合同复核；这属于状态动态共用数据操作层的当前性验证，不授予动态服务自行解释或创建状态结构的权力。

来源动作允许为空。非空时，数据操作层必须在同一独占许可内按以下权威合同复核，不能复制既有私有辅助函数的宽松结果作为事实：

```text
来源动作节点类型 = 方法
模板关系顺序号 0：恰好一个当前状态目标，状态值 = 5
模板关系顺序号 1：恰好一个当前状态目标，状态值在 1..6
两条模板关系、两个状态节点及其主信息均以完整当前句柄读回
缺失、重复、目标类型错误、句柄过期或值域错误均不满足来源动作合同
```

### 3.4 状态动态组合器

`状态动态组合器` 只承接“一次创建完整前后状态并记录实例动态”。组合器先调用状态业务服务形成两个状态写入规格，再把两个规格和动态请求交给状态动态数据操作，后者在一次独占会话内完成三项结构。

组合器不导入执行器，不接触仓库、令牌、会话或候选。

## 5. 公开请求与结果

### 5.1 状态请求

```cpp
struct 创建抽象状态请求 {
    std::uint64_t 幂等主键;
    std::int64_t 状态值;
};

struct 创建实例状态请求 {
    std::uint64_t 幂等主键;
    节点句柄 场景;
    节点句柄 主体;
    std::uint64_t 发生时间戳;
    std::int64_t 状态值;
};
```

### 5.2 动态请求

```cpp
struct 创建抽象动态请求 {
    std::uint64_t 幂等主键;
    std::int64_t 动态材料;
};

struct 记录已有状态变化动态请求 {
    std::uint64_t 幂等主键;
    节点句柄 前状态;
    节点句柄 后状态;
    节点句柄 被改变目标;
    节点句柄 来源动作;
};

struct 创建状态并记录动态请求 {
    std::uint64_t 前状态幂等主键;
    std::uint64_t 后状态幂等主键;
    std::uint64_t 动态幂等主键;
    节点句柄 场景;
    节点句柄 主体;
    std::uint64_t 前时间戳;
    std::uint64_t 后时间戳;
    std::int64_t 前状态值;
    std::int64_t 后状态值;
    节点句柄 被改变目标;
    节点句柄 来源动作;
};
```

`来源动作` 可为空。非空时必须按第 3.3 节的节点类型、模板槽唯一性、状态值和值式读回合同证明是动作入口角色。组合请求三个主键必须非零且互不相同。

### 5.3 结果

公开业务结果至少区分：

```text
已提交
幂等读回
入口拒绝
幂等冲突
许可拒绝
版本漂移
内部不一致
```

结果只携带本次状态、动态及必要关系的完整稳定句柄和值式材料，不携带仓库、令牌、许可、会话、候选、锁或指针。

## 6. 入口检测

所有公开入口必须在调用数据操作写入口前完成：

```text
幂等主键非零
组合路径三个主键互不相同
节点句柄完整且当前
场景类型为场景
主体类型为存在
实例时间戳大于 0 且不超过 int64 最大值
前时间戳不晚于后时间戳
被改变目标当前可读
来源动作为空，或满足：方法节点、模板槽 0 唯一当前状态值为 5、模板槽 1 唯一当前状态值在 1..6、完整关系和节点句柄可读
已有前后状态均为完整实例状态
前后状态主体一致、场景一致
```

错误输入属于逻辑内返回，必须在第一笔写入前结束。入口检测用于发现问题来源，不在函数内部为非法参数填默认值或自动修正结构。

## 7. 幂等合同

四类身份均使用调用方提供的非零全局主键：

```text
抽象状态
实例状态
抽象动态
实例动态
```

索引命中只提供候选。每次幂等读回必须复核完整节点句柄、节点类型、主信息值、时间戳、场景、主体和全部必要关系。

```text
未绑定：允许进入写入。
绑定到同类型、同值、同时间和同关系的完整当前事实：幂等读回。
绑定到错误类型、过期句柄、不同值、不同时间或不同关系：幂等冲突，结构不变化。
```

并发创建中，结构写入失败后必须先由执行器撤销本次写集并释放许可，再权威重读。只有重读到完全相同事实才映射为幂等成功。

## 8. 五条写入路径

### 8.1 创建抽象状态

```text
主键写前读取
-> 主信息候选
-> 槽位 0 写状态值
-> 状态节点候选
-> 主键绑定
-> 同令牌读回值、节点、主键
-> 确认提交
```

不得写槽位 1，不得创建运行期临时关系。

### 8.2 创建完整实例状态

```text
主键 + 场景 + 主体写前读取
-> 许可内复核场景和主体
-> 主信息候选
-> 槽位 0 写状态值
-> 槽位 1 写时间戳
-> 槽位 2 写幂等编号低位
-> 槽位 3 写幂等编号高位
-> 状态节点候选
-> 主键绑定
-> 场景到状态运行期临时关系
-> 状态到主体引用
-> 状态到场景引用
-> 同令牌完整读回
-> 确认提交
```

状态节点脱离任何一项值或关系都不是有效实例状态。

### 8.3 创建抽象动态

```text
主键写前读取
-> 主信息候选
-> 槽位 0 写抽象动态材料
-> 动态节点候选
-> 主键绑定
-> 同令牌读回值、节点、主键
-> 确认提交
```

不得写发生时间戳，不得创建运行期临时关系。

### 8.4 使用已有完整状态记录动态

```text
动态服务通过状态业务服务读取前后状态材料
-> 复核同主体、同场景、时间有序
-> 数据操作进入独占会话
-> 同许可重读两个状态完整材料
-> 复核目标和来源动作
-> 动态主信息候选，槽位 1 写后状态时间戳
-> 动态节点候选和主键
-> 场景到动态运行期临时关系
-> 动态到主体、场景引用
-> 动态到目标、前状态、后状态和可选动作的有序因果来源关系，顺序号固定为 1 / 2 / 3 / 4
-> 同令牌完整读回
-> 确认提交
```

动态发生时间戳固定等于后状态时间戳。

### 8.5 创建前后状态并记录动态

状态业务服务先形成两个 `实例状态写入规格`。数据操作在同一会话内：

```text
创建完整前状态
-> 创建完整后状态
-> 创建实例动态并引用这两个状态
-> 读回三项身份、值、时间和关系
-> 一次请求提交
```

任一步失败都撤销三项本次结构。动态服务和组合器不得调用“创建值材料状态”或自己创建状态节点。

## 9. 状态和动态材料读取

### 9.1 完整实例状态

必须唯一解释：

```text
节点类型 = 状态
槽位 0 有值
槽位 1 是合法时间戳
槽位 2 / 3 构成非零幂等编号
唯一场景运行期临时关系
唯一存在主体引用
唯一场景引用
两个场景句柄一致
```

缺失、重复或矛盾属于内部不一致，不得把半结构当作“未找到”。

### 9.2 完整实例动态

必须唯一解释：

```text
节点类型 = 动态
槽位 1 等于后状态时间戳
唯一场景运行期临时关系
唯一主体和场景引用
顺序号 1 恰好一个被改变目标关系，完整关系与端点句柄当前可读
顺序号 2 恰好一个前状态关系，完整关系与端点句柄当前可读
顺序号 3 恰好一个后状态关系，完整关系与端点句柄当前可读
顺序号 4 可选且最多一个来源动作关系；非空时满足第 3.3 节动作入口角色合同
前后状态完整、同主体、同场景、时间有序
动态场景和主体与状态材料一致
```

关系仓库是权威。索引、缓存、日志和显示材料不能替代上述复核。

## 10. 失败与追根因

```text
入口参数不合法：逻辑内返回，结构不变化。
写前主键冲突：逻辑内返回幂等冲突。
许可内句柄版本漂移：第一笔写入前返回版本漂移。
任一候选值、节点、关系、索引写入不及预期：内部不一致或结构化失败，禁止提交。
任一同令牌读回不一致：追根因并精确撤销。
撤销后存在半结构或数量增长：追根因停止，#267 及后继不得推进。
候选确认后发现不一致：不得静默删除已确认事实，记录逻辑错误并停止。
```

失败注入必须覆盖每个主信息值、节点、主键和关系写点。证明“无有效返回句柄”不够，还必须证明可读结构和数量回前态。

## 11. 并发

至少覆盖：

```text
同抽象状态主键竞争
同实例状态主键竞争
同抽象动态主键竞争
同实例动态主键竞争
组合路径三个主键竞争
已有状态在写前读取后发生版本漂移
```

同一事实只能有一个权威提交。其余调用只有在权威读回完全一致后才能返回幂等成功。不得用微基准牺牲入口前置拒绝、版本句柄、独占许可或事务收口。
