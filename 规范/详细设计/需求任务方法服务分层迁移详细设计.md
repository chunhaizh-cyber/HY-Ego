# 需求任务方法服务分层迁移详细设计

日期：2026-07-14

状态：正式详细设计 / JY-333 / #269 依赖门控施工依据 / 代码未实施

## 1. 定位

本设计把需求、任务、方法和方法召回从传统直接仓库 / 原始令牌路径重建为：

```text
无令牌业务服务
-> 服务专用数据操作层
-> 结构写入执行器
-> 一次独占许可内的不透明会话
-> 四仓库权威结构
```

第一轮只建立隔离新路径和独立自检，不迁移生产调用，不删除旧兼容入口。`#277 / DQ-169 / 757` 是关系换代硬前置；完成后仍须重新逐签名复核本设计假定接口。

## 3. 模块结构

本批新增七个真模块：

| 文件 | 职责 |
| --- | --- |
| `领域/数据操作.需求任务方法.ixx` | 唯一持四仓库和执行器的生产模块；执行类型化读写、精确撤销和权威重读 |
| `领域/服务.需求.ixx` | 需求创建、承接、父子、概念命名目标材料和结算业务准入 |
| `领域/服务.任务.ixx` | 任务承接、生命周期、授权、选择、结果和动作桥材料业务准入 |
| `领域/服务.方法.ixx` | 方法登记、角色、生命周期、规格、关系和动作入口业务准入 |
| `领域/服务.方法召回.ixx` | 非权威召回、复判、排序、建议和选择请求适配 |
| `领域/组合.需求任务方法.ixx` | 固定需求到任务、方法建议到任务选择、任务完成到独立需求结算边界 |
| `领域/自检.需求任务方法分层.ixx` | 隔离功能、故障、并发、兼容和完成边界验收 |

业务服务和组合器不持仓库、执行器、会话、许可或原始令牌。生产与自检依赖单向分离。

## 4. 通用结果与幂等

所有写入口返回统一风格的领域结果：

```text
已提交
幂等读回
入口拒绝
幂等冲突
许可拒绝
版本漂移
内部不一致
```

所有新建身份必须接受非零稳定幂等主键。一次操作需要多个新身份时使用类型化键组，不把调用方任意文本、日志、函数名或指针散列成机器主键。

写前按主键读取：

```text
结构完整且语义相同 -> 幂等读回。
主键已绑定到不同类型或不同语义 -> 幂等冲突。
同一语义出现多个主键 / 当前关系 -> 内部不一致。
未找到 -> 才进入写入会话。
```

## 5. 需求服务

### 5.1 完整目标状态需求

请求至少包含：

```text
需求幂等主键、目标状态幂等主键。
需求主体、目标宿主、场景。
目标特征、当前特征状态材料。
目标状态值材料。
```

需求业务服务先通过存在、场景、特征和状态公开值式入口形成规格。数据操作层在一次会话内创建目标抽象状态、需求节点、主键和以下关系：

| 角色 | 关系 | 顺序号 |
| --- | --- | ---: |
| 主体 | 需求 -> 存在 `引用` | 1 |
| 目标宿主 | 需求 -> 存在 `引用` | 2；自我同端点按既有反向角色合同处理 |
| 场景 | 需求 -> 场景 `引用` | 3 |
| 目标特征 | 需求 -> 特征 `引用` | 4 |
| 当前特征状态材料 | 需求 -> 特征值 `引用` | 5 |
| 目标状态 | 需求 -> 抽象状态 `模板` | 0 |

目标状态对外语义是状态节点，不是 I64。最后发布后必须读回完整承接材料。

### 5.2 概念命名需求

`#269` 只建立无令牌的概念命名需求创建 / 读取和权威 DTO，至少包含：

```text
需求、主体、目标宿主、场景、目标状态。
目标概念及完整目标关系。
服务根需求及完整普通父子发布关系。
当前满足状态。
```

公开读取只接收需求句柄，自行从权威关系派生目标概念，不接收调用方提供的“预期目标概念”。本批不绑定名称、不创建动作动态、不生成执行回执；`#214` 继续暂停。

### 5.3 需求父子

父子关系继续使用 `普通父子`，方向父需求到子需求。业务服务验证双方均为需求、子需求有目标状态、非自身后，数据操作层调用 `#277` 会话普通父子入口。单父、防环、首次挂载和重挂由关系仓库裁决。

### 5.4 需求结算

结算请求复核：需求目标状态、来源任务已完成、实际状态、场景、主体、动态证据和可选轻量因果引用。比较由状态业务口径形成，不能由调用方 bool 裁决。

结算状态及其来源关系在独立会话提交；任务完成已经成立时，结算允许拒绝只返回待结算材料，不回滚任务结果或生命周期。

## 6. 任务服务

### 6.1 创建承接壳

请求包含任务、任务虚拟存在和初始生命周期状态的非零幂等键，以及来源需求和时间戳。需求业务服务先返回完整承接材料；存在 / 状态业务服务形成虚拟存在和实例状态写入规格。

同一会话创建：

```text
任务节点。
任务虚拟存在。
已创建实例状态。
需求 -> 任务归属。
任务 -> 需求、虚拟存在、场景引用。
任务 -> 目标状态模板。
任务 -> 初始状态任务生命周期关系。
```

最后发布后读回双向承接授权、唯一当前生命周期和完整任务承接材料。

### 6.2 生命周期换代

沿用当前已验证状态机、幂等材料、迁移证据和终态拒绝。数据操作顺序：

```text
读取唯一当前生命周期和审计历史。
创建新实例状态及证据关系。
创建新任务生命周期关系，顺序号 = 旧版本 + 1。
同许可读回新关系和状态材料。
经 #277 可逆失效旧当前关系。
读回唯一当前和连续历史后请求提交。
```

重筹办进入筹办中前，旧方法投影和旧选择发布关系也必须经 `#277` 可逆失效并加入同一关系变更写集。

### 6.3 方法选择

方法召回只产生非权威建议。任务业务服务拥有选择提交：

```text
复核承接授权、筹办中当前态和完整来源版本。
复核建议方法当前登记、活跃生命周期、规格和动作角色。
创建不可变任务方法选择记录及值材料。
创建任务 -> 方法投影关系。
最后创建任务 -> 选择记录发布关系。
完整读回后提交。
```

同幂等键同请求幂等，不同请求冲突。任务服务不包含或信任非权威索引作为授权。

### 6.4 实际结果与完成基础入口

`#269` 提供无令牌的权威基础入口：绑定唯一实际状态，并在满足当前执行中状态和目标比较后迁移到已完成。它不接线程强类型回执，不生成动作动态。

`#224` 继续负责真实执行调度和强类型回执；`#225` 继续负责回执治理、结果提交与独立需求结算编排，但其底层服务合同必须改为复用 `#269` 新入口。

## 7. 方法服务

### 7.1 登记根和角色

保持已验证结构：方法登记根、共享活跃 / 失效抽象状态、方法首、条件、结果和动作入口均使用方法节点并由模板状态区分角色。所有新身份使用稳定键。

方法首创建可在同一会话形成方法节点、角色状态、虚拟存在、活跃生命周期和根归属关系；中途失败不得留下可召回登记项。

### 7.2 生命周期

方法生命周期继续使用固定模板顺序号和同一关系编号。失效 / 恢复经 `#277` 可逆重挂关系到共享状态，版本单调推进。失效方法仍可审计，但不能进入召回、选择或动作桥。

### 7.3 条件 / 结果 / 动作入口

保持当前角色、关系类型和顺序号合同：

```text
条件规格：目标特征、期望抽象状态和原始版本。
结果规格：目标特征、目标抽象状态和来源版本。
动作入口：方法角色值 5，入口状态值 1..6，输入 / 输出场景规格。
```

实例状态不得进入长期方法规格。动作入口只建立身份和只读桥材料，不执行动作或创建动态。

## 8. 方法召回

第一轮新分层保持：

```text
从任务与需求权威材料形成请求。
读取活跃方法登记和结果规格形成候选。
按当前特征槽位和原始版本复判条件。
按既有确定性规则排序并输出只读建议。
唯一建议才可适配为任务选择请求。
```

派生索引、候选、复判、排序和建议均非权威。第一轮新路径允许从权威登记和规格确定性重建候选，不要求修改旧 `方法召回索引.h`；若复用旧索引，命中后仍必须由新服务重读完整关系句柄、版本和端点。

## 9. 组合器

组合器固定三条边界：

```text
需求创建 -> 任务承接创建。
方法召回唯一建议 -> 任务服务选择提交。
任务结果 / 完成 -> 独立需求结算。
```

组合器只传递类型化请求和值式结果，不导入执行器。任务完成与需求结算不是一个回滚边界；真实方法执行、动作动态和回执生产不是本组合器职责。

## 10. 失败收口

```text
入口拒绝：第一笔写入前返回，结构和数量不变。
新写入失败：会话逆序撤销索引、关系、节点和主信息候选。
关系换代后失败：先经 #277 逆序恢复旧关系，再撤销本次新写集。
读回矛盾：内部不一致，停止后继动作桥、结果或结算产物。
释放许可后复核无法证明前态 / 当前态：登记断点，不登记 760。
```

不得扫描整仓修复、调用旧无令牌路径补偿、物理删除已发布业务事实或用日志 / 控制台承载失败状态。
