## 1. 设计目标与约束

本设计只冻结 #373 最终汇合的装配面和验收边界：

```text
真实 D455 设备独占
-> 采样、缓存、上行、成熟度与产品生产
-> 单一报告队列
-> 任务唯一消费与冻结工作包
-> 观察 / 识别 / 扫描 / 跟踪分属独立任务回合
-> 场景事实、状态动态和体素
-> 断流、停止、恢复候选、跨重启续跑
-> 真实硬件端到端验收
```

正式依据为 0050、4010—4070、6200—6370、8100、8110，以及存在、场景、状态、动态、任务和方法根规范。生产恢复必须服从 4070 的隔离恢复、索引重建和运行期上下文一次发布。

跨崩溃恢复遵守 6340 的现行裁决：物理队列和四索引不保存为权威队列快照；关键报告与消费治理事实按稳定身份持久为恢复候选；新运行期隔离复核并重建空物理队列、四索引和新生产 / 消费代次。

### 3.1 恢复提供者与所有权

#373 只编排，不接管各域写权。内部恢复端口固定为：

```cpp
class 权威结构恢复提供者;
class D455报告恢复提供者;
class 任务筹办恢复提供者;
class 存在体素恢复提供者;
```

四个端口分别返回不可复制的内部恢复参与包，并各自负责本域候选准备、确认、撤销和完整读回：

- 权威结构恢复提供者：节点、稳定主键、关系、领域类型化记录、版本、历史事实和索引；
- D455 报告恢复提供者：报告 / 消费治理事实、空物理队列、四索引和新消费代次；
- 任务筹办恢复提供者：旧占用历史、动作收口核验、新恢复代次与待重筹办触发；不继承执行权；
- 存在体素恢复提供者：存在体素模型、单元、消费记录和可重建空间索引。

`D455恢复编排器` 构造注入四个恢复提供者和 `运行期上下文宿主&`。确认顺序为权威结构 -> 报告治理 -> 任务筹办 -> 体素；撤销逆序；四域完整读回后才构造 `运行期上下文` 并调用现有 `运行期上下文宿主::尝试首次发布全新上下文(std::shared_ptr<运行期上下文>)`。#373 不直接失效任务占用、不直接改报告消费事实、不直接重写体素记录。

### 3.2 装配、生命周期和公开行为

`D455生产运行期装配` 构造注入真实 `D455帧来源` 工厂、PER-C1—C12 固定提供者、`运行期上下文宿主&`、`运行宿主线程&`、`自我线程&`、`任务管理线程&` 和 `任务工作线程&`。上下文内部可以持有指针 / 引用，但对外只返回 `D455运行期上下文身份` 和租约，不暴露可变容器、设备裸句柄、仓库、锁或事务令牌。

#359 固定结果完成且释放共享接线所有权后，#373 只取得第 2 节已登记共享文件的后继 D455 接线修改权，并以 #359 固定提交中的这些文件 blob 作为 S0 起点；#359 其它私有 provider、路由、任务请求和治理实现继续只读。所有权移交不改写 #359 固定提交，也不允许 #373 反改其私有合同。

公开行为固定为：

```cpp
D455设备占用结果 尝试取得D455设备占用(
    const D455设备身份材料& 设备,
    const D455运行规格& 规格);
D455生产运行期启动结果 启动D455生产运行期(
    const D455设备占用& 占用,
    const D455运行规格& 规格);
D455恢复装配结果 从D455恢复候选装配运行期(
    const D455设备占用& 占用,
    const D455运行规格& 规格,
    const D455恢复候选清单& 候选);
D455断流处理结果 处理D455断流(
    const D455运行期上下文身份& 上下文,
    const D455断流材料& 材料);
D455停止结果 停止D455生产运行期(
    const D455运行期上下文身份& 上下文,
    const D455停止规格& 规格);
D455端到端验收结果 验收真实D455生产闭环(
    const D455验收规格& 规格);
```

对应结果 DTO 都由具名稳定状态、可选上下文身份 / 恢复读回、失败验收项和 `bool 追根因` 组成，不使用 `结果<T>`。设备占用释放失败、上下文发布后服务版本不一致、停止后仍发布报告 / 事实以及半恢复上下文可见均为内部错误。

- 模块 / 文件：第 2 节精确文件集。
- DTO：设备占用、运行规格、生产 / 重连代次、运行上下文、恢复候选清单、停止规格、验收规格 / 结果。
- 输入 / 输出：输入真实设备身份、运行配置、PER-C1—PER-C12 固定实现和可选恢复候选；输出一次发布的完整运行期上下文、具名停止 / 恢复结果和真实硬件验收结果。
- ABI / 规则版本：导出 `inline constexpr std::uint32_t D455生产运行期ABI版本 = 2`；生产、消费、恢复、重连和筹办恢复代次分别版本化，不能由同一裸序号代替。
- 提供 / 消费：#373 提供，最终独立审计消费。
- 写读方：生产运行期唯一持有设备和装配写权；只读投影无控制权。
- 生命周期：设备占用从准入到完整停止；上下文一次发布；断流封闭旧生产代次，重连建立新代次。
- 并发隔离：
  - 物理设备身份 -> 唯一设备占用；
  - `D455报告身份` -> 跨窗口、任务和代次的唯一报告承接 / 消费；消费窗口只作准入绑定；
  - 任务身份 -> 唯一筹办执行权；
  - 场景 + 模型代次 -> 体素发布。
- 逻辑内返回：配置无效、设备占用失败、恢复候选不完整、断流、重连超窗、真实样本不满足验收。
- 内部错误：完整装配前置通过后上下文无法一次发布，或已经发布的服务句柄互相不一致。
- 恢复：
  1. 文件和数据库只提供快照、事件和候选；
  2. 在隔离区校验代次、身份、版本、结构和业务不变量；
  3. 重建全部可重建索引；
  4. 报告 / 队列候选不得直接变成世界事实；
  5. 已承接未完成项只恢复任务侧引用且禁止重新消费；
  6. 尚未承接且当前不可舍弃项以原报告身份进入新生产代次投递候选；
  7. 过期、已消费和失效项按原状态收束；
  8. 旧运行代次的筹办占用和执行冻结只由任务筹办恢复提供者保留历史，不继承执行权；
  9. 任务筹办恢复提供者证明旧运行代次终止且前轮真实外部动作已停止或已有实际回读后，失效旧占用，发布新恢复代次与待重筹办触发，由新筹办轮次重新取得执行权；
  10. 无法证明前轮动作已收口时，只形成任务侧合法等待、动作核验生产者和重新触发条件；
  11. 重建空物理队列与四索引；
  12. 全部恢复分支、索引和服务句柄校验成功后，随新生产 / 消费 / 恢复代次一次发布运行期上下文。
- 漂移退回：真实设备占用、队列恢复语义、运行上下文发布、断流 / 重连代次或验收级别改变。

## 5. 启停顺序

启动：

1. 验证配置、设备、标定和规则版本；
2. 原子取得设备占用；
3. 在隔离区装配 PER-C1—PER-C12 服务；
4. 如有恢复候选，按 4070 完整准入并重建索引；
5. 一次发布运行期上下文；
6. 启动采集和生产；
7. 开放任务消费。

停止：

1. 关闭新任务消费窗口；
2. 停止发布新产品；
3. 封闭采集 / 生产代次；
4. 对已进入事实事务和体素事务的候选完成确认或撤销；
5. 形成完整恢复候选；
6. 释放设备占用；
7. 销毁运行期临时缓存。

断流：

1. 封闭旧采集和生产代次；
2. 不删除已发布报告、已承接引用或已提交领域事实；
3. 已承接项只恢复任务侧引用且不重新消费；不可舍弃未承接项保留原报告身份形成新代次投递候选；其余按状态收束；
4. 合法重连时建立新重连 / 生产 / 消费代次，不续写旧代次，并从空物理队列与空四索引开始重建。
