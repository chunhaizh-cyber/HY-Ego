## 1. 设计目标与约束

本设计只冻结 #373 最终汇合的装配面和验收边界：

```text
真实 D455 设备独占
-> 采样、缓存、上行、成熟度与产品生产
-> 单一报告队列
-> 任务唯一消费与冻结工作包
-> 观察 / 识别 / 扫描 / 跟踪分属独立任务回合
-> 场景事实、状态动态和体素
-> 断流、停止、恢复候选、跨重启续跑
-> 真实硬件端到端验收
```

正式依据为 0050、4010—4070、6200—6370、8100、8110，以及存在、场景、状态、动态、任务和方法根规范。生产恢复必须服从 4070 的隔离恢复、索引重建和运行期上下文一次发布。

跨崩溃恢复遵守 6340 的现行裁决：物理队列和四索引不保存为权威队列快照；关键报告与消费治理事实按稳定身份持久为恢复候选；新运行期隔离复核并重建空物理队列、四索引和新生产 / 消费代次。

## 3. 运行期结构

| 结构 | 字段 |
| --- | --- |
| `D455设备占用` | 设备稳定身份、占用代次、标定版本、运行上下文身份、状态版本 |
| `D455生产运行代次` | 设备、启动序号、生产代次、开始 / 封闭时间、规则版本 |
| `D455重连代次` | 原生产代次、重连序号、设备 / 标定一致性、开始时间、结果 |
| `D455运行期上下文` | 采集器、生产者、受控材料域、单一队列、任务承接、方法、事实、体素服务的值式句柄集合 |
| `D455恢复候选清单` | 快照代次、权威节点 / 关系 / 类型化记录、索引重建输入、报告 / 消费治理恢复候选、任务筹办占用历史、前轮外部动作核验材料、完整性摘要 |
| `D455端到端验收结果` | 硬件身份、配置、样本范围、各合同结果、断流 / 重启结果、失败证据 |

裸线程编号、指针、日志行、界面状态或数据库连接不能承担这些身份。

建议行为：

```cpp
结果<D455设备占用> 尝试取得D455设备占用(const D455设备身份& 设备, const D455运行规格& 规格);
结果<D455运行期上下文> 装配D455生产运行期(const D455设备占用& 占用, const D455运行规格& 规格);
结果<D455运行期上下文> 从D455恢复候选装配运行期(const D455设备占用& 占用, const D455恢复候选清单& 候选);
结果<D455重连结果> 处理D455断流(const D455运行期上下文& 上下文, const D455断流材料& 材料);
结果<void> 停止D455生产运行期(D455运行期上下文& 上下文, const D455停止规格& 规格);
结果<D455端到端验收结果> 验收真实D455生产闭环(const D455验收规格& 规格);
```

## 4. PER-C13 / v0.1

- 模块 / 文件：第 2 节精确文件集。
- DTO：设备占用、运行规格、生产 / 重连代次、运行上下文、恢复候选清单、停止规格、验收规格 / 结果。
- 输入 / 输出：输入真实设备身份、运行配置、PER-C1—PER-C12 固定实现和可选恢复候选；输出一次发布的完整运行期上下文、具名停止 / 恢复结果和真实硬件验收结果。
- ABI / 规则版本：`PER-C13 / v0.1`；生产、消费、恢复、重连和筹办恢复代次分别版本化，不能由同一裸序号代替。
- 提供 / 消费：#373 提供，最终独立审计消费。
- 写读方：生产运行期唯一持有设备和装配写权；只读投影无控制权。
- 生命周期：设备占用从准入到完整停止；上下文一次发布；断流封闭旧生产代次，重连建立新代次。
- 并发隔离：
  - 物理设备身份 -> 唯一设备占用；
  - 队列项 / 消费窗口 -> 唯一报告消费；
  - 任务身份 -> 唯一筹办执行权；
  - 场景 + 模型代次 -> 体素发布。
- 逻辑内返回：配置无效、设备占用失败、恢复候选不完整、断流、重连超窗、真实样本不满足验收。
- 内部错误：完整装配前置通过后上下文无法一次发布，或已经发布的服务句柄互相不一致。
- 恢复：
  1. 文件和数据库只提供快照、事件和候选；
  2. 在隔离区校验代次、身份、版本、结构和业务不变量；
  3. 重建全部可重建索引；
  4. 报告 / 队列候选不得直接变成世界事实；
  5. 已承接未完成项只恢复任务侧引用且禁止重新消费；
  6. 尚未承接且当前不可舍弃项以原报告身份进入新生产代次投递候选；
  7. 过期、已消费和失效项按原状态收束；
  8. 旧运行代次的筹办占用和执行冻结只保留历史，不继承执行权；
  9. 证明旧运行代次终止且前轮真实外部动作已停止或已有实际回读后，失效旧占用，发布新恢复代次与待重筹办触发，由新筹办轮次重新取得执行权；
  10. 无法证明前轮动作已收口时，只形成任务侧合法等待、动作核验生产者和重新触发条件；
  11. 重建空物理队列与四索引；
  12. 全部恢复分支、索引和服务句柄校验成功后，随新生产 / 消费 / 恢复代次一次发布运行期上下文。
- 漂移退回：真实设备占用、队列恢复语义、运行上下文发布、断流 / 重连代次或验收级别改变。

## 5. 启停顺序

启动：

1. 验证配置、设备、标定和规则版本；
2. 原子取得设备占用；
3. 在隔离区装配 PER-C1—PER-C12 服务；
4. 如有恢复候选，按 4070 完整准入并重建索引；
5. 一次发布运行期上下文；
6. 启动采集和生产；
7. 开放任务消费。

停止：

1. 关闭新任务消费窗口；
2. 停止发布新产品；
3. 封闭采集 / 生产代次；
4. 对已进入事实事务和体素事务的候选完成确认或撤销；
5. 形成完整恢复候选；
6. 释放设备占用；
7. 销毁运行期临时缓存。

断流：

1. 封闭旧采集和生产代次；
2. 不删除已发布报告、已承接引用或已提交领域事实；
3. 已承接项只恢复任务侧引用且不重新消费；不可舍弃未承接项保留原报告身份形成新代次投递候选；其余按状态收束；
4. 合法重连时建立新重连 / 生产 / 消费代次，不续写旧代次，并从空物理队列与空四索引开始重建。

| 模拟 | 断流、重连、取消、过期、版本漂移、恢复拒绝 | 真实硬件 |
| 真实硬件 | 真实 D455 连续采集，四产品生成，任务唯一消费，观察事实和体素更新 | 模拟帧或构建成功 |
