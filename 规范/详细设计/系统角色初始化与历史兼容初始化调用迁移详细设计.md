# 系统角色初始化与历史兼容初始化调用迁移详细设计

日期：2026-07-14

状态：正式详细设计 / 绑定 #250 / JY-337

## 1. 定位

本设计承接 `#270 / 770` 已形成的唯一运行期业务装配，建立第一版新服务驱动的系统角色初始化，并把四个历史初始化 `.ixx` 从兼容 include 形态迁为隔离真模块。

两个目标必须同时成立，但权力严格分离：

```text
新系统角色初始化：决定未发布运行期上下文是否具备完整初始化材料。
历史兼容初始化模块：只维持当前旧入口、旧线程和历史自检的编译与行为回归，不参与新上下文完整性。
```

## 2. 已验证当前事实

1. `装配.运行期业务.ixx` 已拥有一个内部特征值服务、七个数据操作、十四个业务服务、六个组合器和八类顶层操作，但没有系统角色初始化或角色读回入口。
2. `启动.运行期上下文.ixx` 已拥有候选业务装配；`完整()` 固定为假，服务装配完整时宿主仍返回“缺少初始化完成材料”。
3. 四个历史初始化文件仍使用 `#pragma once + #include`，其中世界树和语素初始化直接接触仓库或传统服务；不能被新装配调用。
4. 旧语素初始化创建 18 个显示项；显示文本和类型名称不属于系统角色，也不得裁决机器身份。
5. 旧世界树初始化直接创建普通父子关系并把自我坐标写入主信息 I64 槽；新路径没有对应授权服务，不能复制该直写方式。
6. 当前有 11 个文件直接 include 至少一个历史初始化文件；工程仅把世界树、需求、概念图三个文件登记为 `ClInclude`，语素初始化未独立登记。
7. 当前新服务已提供创建基础信息、场景、实际存在、抽象状态 / 动态、特征定义 / 槽位 / 初值、二次特征、轻量因果、完整需求、方法登记根和概念根角色复核所需入口。
8. 当前新服务没有通用世界树普通父子发布入口，因此系统角色拓扑必须由专用数据操作层在不透明会话内承载，不能让初始化器直接写关系仓库。

本设计不把预期新增接口写成当前实现事实。

## 3. 范围与排除

### 3.1 本批范围

1. 定义 `系统角色清单版本 = 1`、显式稳定键参数和完整值式角色材料。
2. 新增系统角色专用数据操作层，负责稳定键预检、世界拓扑关系发布和最终权威读回。
3. 新增系统角色初始化器，只调用新业务服务、既有组合器和系统角色数据操作。
4. 运行期业务装配内部拥有系统角色数据操作和初始化器，不向外暴露服务、仓库、执行器、会话或令牌。
5. 运行期上下文在候选期接收一次初始化请求；只在完整读回后保存角色清单并使 `完整()` 成立。
6. 宿主在本批仍不真正发布；完整候选返回“首次发布尚未启用”，由 #249 后继接管。
7. 四个历史初始化文件原地转换为兼容真模块；全部当前 include 调用点只迁为 import。
8. 新旧初始化路径分别验收，禁止互相调用。

### 3.2 明确排除

```text
不把旧 18 个显示语素写入系统角色清单。
不迁移旧自我相对坐标 I64 槽；坐标必须等待特征体系专项。
不迁移旧自我线程初始化权力、Tick、启动正文或真实线程调度。
不发布运行期上下文，不生成有效租约，不切换生产入口。
不接入概念生命周期、活动快照、统计缓存、概念安全删除或恢复链。
不删除传统服务或兼容初始化类，不在 #271 前批量删除兼容路径。
不把名称、日志、显示、SQL 或控制台输出当作角色发现依据。
```

## 4. 系统角色清单版本 1

### 4.1 稳定值类型

新增单头值类型 `领域/系统角色清单.数据.h`。只声明枚举、参数、完整句柄和值式结果，不实现流程，因此无需 `.cpp`。

角色清单至少保存：

| 角色组 | 必需材料 | 权威复核 |
| --- | --- | --- |
| 世界锚点 | 世界根、自我场景、自我存在 | 主键唯一、节点类型、主信息有效 |
| 世界拓扑 | 世界根到场景普通父子、场景到自我普通父子、场景接纳自我归属 | 关系句柄当前有效、端点和类型精确匹配 |
| 安全根需求 | 特征定义、实例槽位、当前特征值、目标状态、根需求 | 特征当前值、需求主体 / 场景 / 目标状态完整读回 |
| 服务根需求 | 特征定义、实例槽位、当前特征值、目标状态、根需求 | 同上，且全部身份与安全组不同 |
| 概念四根 | 存在根、动态根、关系根、因果根 | 稳定键、完整句柄和预期节点类型复核 |
| 方法登记 | 方法登记根 | 根角色、活跃 / 失效生命周期状态完整读回 |

因果根由动态根作为来源准入证据形成轻量因果引用；关系根第一版由有序组成 `[存在根, 因果根]` 的二次特征承载。该组合符合当前二次特征长期组成项白名单，不把动态节点直接作为二次特征组成项，也不把单个因果引用解释为稳定因果结论。

### 4.2 初始化参数

参数显式携带所有创建身份所需稳定键和安全 / 服务两组初值、目标值。入口必须在第一笔写入前证明：

```text
清单版本等于 1。
全部稳定键非零且全局互异。
安全 / 服务当前值和目标值满足领域服务入口要求。
域、仓库和候选上下文有效。
同一键当前为空，或已绑定完全相同的预期类型和业务语义。
```

调用方不得传入原始令牌、许可、会话、仓库或任意服务引用。

## 5. 新系统角色初始化调用路径

```text
运行期上下文::初始化系统角色(值式参数)
-> 运行期业务装配::初始化系统角色
-> 系统角色初始化器
-> 新业务服务 / 既有组合器
-> 系统角色数据操作
-> 结构写入执行器 / 不透明会话
-> 四仓库
-> 版本 1 系统角色清单
-> 运行期业务装配完整读回
-> 候选上下文保存清单。
```

运行期业务装配增加系统角色数据操作和初始化器成员，但七个既有数据操作、十四个业务服务、六个组合器和八类线程路由不改义。系统角色初始化不是第九类线程业务请求，只允许启动期候选工厂同步调用。

## 6. 初始化顺序

### 6.1 写前预检

系统角色数据操作以共享许可一次读取全部稳定键。已占用键必须逐键匹配预期节点类型；错误类型、同键异义和版本漂移在第一笔写入前返回。当前仓库读回自相矛盾属于追根因。

### 6.2 世界锚点

```text
基础信息业务服务创建世界根。
场景业务服务创建自我场景。
存在场景组合器在已有场景中创建并接纳自我实际存在。
```

旧自我相对坐标不在本批创建。

### 6.3 两个根需求

每组按固定顺序：

```text
特征业务服务创建特征定义。
特征状态组合器创建实例槽位并发布版本 1 初始 I64 特征状态。
需求业务服务创建完整目标状态需求和抽象目标状态。
```

需求目标对外语义始终是目标状态；I64 只作初始特征状态和第一轮状态值材料。

### 6.4 概念四根

```text
存在业务服务创建实际存在根；不得把世界根扩成虚拟存在来源。
动态业务服务创建抽象动态根。
轻量因果业务服务创建因果根。
二次特征业务服务以有序组成 [存在根, 因果根] 创建关系根。
概念图结构业务服务按四个稳定键和句柄复核根角色组。
```

本批不创建生命周期状态、不登记活动快照、不绑定显示名称。

### 6.5 方法登记根

方法业务服务创建方法登记根、角色状态、活跃状态和失效状态，并按稳定键完整读回。

### 6.6 世界拓扑最后发布与清单读回

系统角色数据操作取得一次独占不透明会话：

```text
复核全部角色当前句柄、主键和节点类型。
复核场景接纳自我归属关系。
创建或幂等读回 世界根 -> 自我场景 普通父子关系。
创建或幂等读回 自我场景 -> 自我存在 普通父子关系。
拒绝同角色冲突父关系和同端点异顺序语义。
读回全部角色、关系和主信息。
请求提交。
```

只有该步骤成功且装配再次复核通过，才形成系统角色清单。清单本身只定位权威结构，不覆盖仓库事实。

## 7. 幂等、失败和并发

| 情况 | 分类 | 收口 |
| --- | --- | --- |
| 版本错误、零键、重复键、参数值不合法 | 逻辑内返回 | 第一笔写入前拒绝，结构数量不变化 |
| 已占用键类型或语义冲突 | 逻辑内返回 | 不覆盖、不删除、不重绑；上下文不完整 |
| 许可竞争 | 逻辑内返回 | 保留原服务状态；调用方显式重试 |
| 相同请求重复初始化 | 幂等读回 | 返回同一角色清单，结构数量不增长 |
| 多线程同参数初始化 | 并发幂等 | 一个完成，其余读回同一材料；不得形成两份清单 |
| 服务步骤已提交后后继逻辑内返回 | 未发布候选 | 已提交对象必须各自是稳定键可读的完整业务事实；不形成角色清单，不解释为系统初始化完成，可用同参数继续 |
| 最后拓扑会话失败 | 逻辑内返回或追根因 | 会话新关系精确撤销；角色清单不形成 |
| 前置通过后节点 / 关系 / 主信息 / 目标状态读回矛盾 | 追根因解决 | 停止后继，不扫描修复、不切换旧初始化路径 |

不得把“角色清单未发布”扩大声明为所有先前独立服务提交的物理数量完全不变。可声明的是：无非法半结构成为系统角色，宿主和租约保持空。

## 8. 运行期上下文状态

运行期上下文新增：

```text
初始化系统角色(参数)
读取系统角色材料()
系统角色完整()
```

`完整()` 由以下条件共同成立：

```text
结构核心完整。
服务装配完整。
角色清单版本为 1。
角色清单全部句柄有效、同仓库。
业务装配再次按权威结构读回一致。
```

本批不让宿主真正写入 `当前上下文_`。完整候选调用 `尝试发布` 时返回新增状态“首次发布尚未启用”，宿主保持空；#249 后继再启用唯一首次发布。

## 10. 文件分类

| 文件 | 分类 | 工程登记 | 调用方式 | 说明 |

## 12. 完成边界

完成后只可声明：第一版系统角色初始化和权威读回已在未发布上下文候选内实现；四个历史初始化文件已迁为隔离兼容真模块，当前 include 调用已改为 import。

不得声明：运行期上下文已发布、生产入口已切换、旧自我循环已迁移、历史初始化逻辑已删除、坐标系统已迁移、恢复链已实现或旧能力整体迁移完成。
