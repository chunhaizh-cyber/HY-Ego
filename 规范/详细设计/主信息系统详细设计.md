# 主信息系统详细设计

编号：005-D1
日期：2026-07-05
状态：待确认详细设计 / 第一轮边界草案

## 1. 设计目标

本详细设计对应 `规范/主信息系统规范.md`，把主信息系统的统一仓库、最小记录、值容器和字段准入门禁落到函数级设计。

第一轮目标不是迁移旧项目 `*主信息类`，而是确认：

```text
主信息仓库统一
主信息记录最小
业务字段不膨胀
具体信息按结构分流
字段新增先走规范和功能确认
编号使用 uint64 语义且不复用
```

## 2. 规范对应关系

| 规范条款 | 详细设计承接 |
| --- | --- |
| 主要用途 | 主信息仓库只提供身份、生命周期和值容器服务 |
| 功能能力分层 | 当前函数和后续待实现函数分开 |
| 实现方法 | 节点、关系、索引、特征、状态、动态、因果分流 |
| 对外服务 | 当前只暴露主信息仓库最小函数 |
| 字段准入规则 | 新增字段必须先有字段功能确认 |
| 分流规则 | 旧字段不直接进入主信息仓库 |

## 3. 当前结构设计

| 结构 | 用途 | 写入方 | 读取方 | 生命周期 | 验证方式 |
| --- | --- | --- | --- | --- | --- |
| `主信息句柄` | 主信息长期身份 | 主信息仓库 | 节点仓库 / 领域服务 / 主信息仓库 | 随主信息记录生命周期 | 有效句柄可读，删除后旧句柄失效 |
| `主信息记录` | 最小主信息记录 | 主信息仓库 | 主信息仓库 / 授权领域服务 | 创建到删除 | 记录编号、版本、状态一致 |
| `主信息编号` | 仓库内本地身份编号 | 主信息仓库 | 主信息仓库 / 持久化适配 | 仓库内只递增不复用 | `0` 无效，删除后不回收 |
| `仓库编号` | 区分不同主信息仓库 | 主信息仓库初始化 | 句柄比较 / 持久化适配 | 随仓库生命周期 | 句柄仓库编号不匹配时拒绝读取 |
| `值容器` | 承载最小数值 | 主信息仓库 / 授权领域服务 | 特征值服务 / 需求服务等授权入口 | 随主信息记录生命周期 | 写入 I64 后可读取 |
| `拓扑锚点编号` | 预留拓扑锚点 | 待确认 | 待确认 | 随主信息记录生命周期 | 后续单独确认 |

当前不新增以下结构：

```text
词主信息记录
语素入口主信息记录
基础信息主信息记录
需求主信息记录
任务主信息记录
方法主信息记录
```

## 4.1 编号设计

当前主信息编号设计：

```text
类型语义：uint64
无效值：0
起始值：1
分配方式：仓库内单调递增
删除行为：只标记已删除并递增版本，不回收编号
稳定身份：仓库编号 + 主信息编号 + 版本号
```

`std::int64_t` 只用于 `值容器` 和 `写入I64值` / `读取I64值`，不得用于主信息编号。

后续若出现长期运行容量问题，处理顺序必须是：

```text
先评估内存、索引、快照和持久化体量。
再建立仓库编号 / 分片 / 纪元 / 归档设计。
最后才进入代码实现。
```

不得通过复用主信息编号、把已删除编号重新分配给新记录、或把编号临时改成有符号类型来规避容量问题。

## 5. 字段功能确认门禁

任何新增主信息字段前，必须先形成字段功能说明：

```text
字段名
主要用途
写入方
读取方
生命周期
是否权威
是否可由关系 / 索引 / 特征 / 状态 / 动态 / 因果替代
入口前置拒绝条件
验证方式
禁止宣称
```

没有字段功能说明，不得进入代码实施。

## 6. 旧字段处理设计

第一轮设计结论：

```text
`词主信息类::词` 不迁移为主信息字段。
`语素入口主信息类::对应基础信息` 保持关系承载。
`语素入口主信息类::词性`、`信息入口类型`、`对应基础信息类型` 暂缓到语素候选和入口路由设计。
`主信息基类::名称`、`类型` 暂缓到语素入口关系和显示派生设计。
需求、任务、方法字段进入各自系统详细设计，不进入通用主信息详细设计。
特征、状态、动态、二次特征、因果字段进入对应系统详细设计，不进入通用主信息详细设计。
```

## 7. 示例分流

输入：

```text
这个杯子很漂亮
```

主信息系统只允许承载其中的最小值。例如高程度值可进入值容器。其他内容应分流：

```text
这个杯子 -> 指代解析和实际存在节点
杯子 -> 语素入口和概念节点
漂亮 -> 复合特征概念节点
很 -> 二次特征 / 程度修饰概念节点
整句来源 -> 语言材料或来源动态
评价事实 -> 状态 / 特征值 / 因果引用
```

不得新增 `漂亮`、`很漂亮`、`杯子很漂亮` 等主信息字段。
