## 1. 目标与权威边界

本设计把分层安全需求转换为两项可施工合同：

```text
SAFETY-C1 / v1
稳定直接因果图
-> 精确因果深度与五组只读投影
-> 安全事件、因素、排除和有效未复发证据
-> 分层安全值主动结算与被动维护
-> 状态、动态、因果和安全领域记录原子发布

SAFETY-C2 / v1
分层安全值与阈值快照
-> 1% / 99% 递归权限门控
-> 任务多安全需求取最高权限
-> 任务基础优先级合成
-> 任务管理线程确定性消费
```

正式依据为 0050、1140、1160、1170、1190、3100、3200、4210、4220、4310、4320、5110、5120、5210、5230、6100、6110、6120、6130、6140、6150、8100、8200、8300、8310。其中 6140 唯一规定分层安全值被动维护，6150 唯一规定因果精确深度、五组投影和任务执行优先级权限；本设计只能解释实现，不能反向覆盖规范。

本设计只覆盖安全需求。服务值仍按 6100、6120 的成功服务、时间衰减和休眠合同处理，不套用本设计的低位回升、高位回落或分层权限算法。

下列边界固定：

1. 安全事实与风险证据改变安全值；安全值和层级门控派生任务执行优先级权限；任务调度只消费权限。调度压力、线程拥堵、队列长度和算力不足不得反写安全值。
2. 权限是任务参与正常执行排序的影响系数，不是线程、时间、算力、任务数量或成功概率配额。
3. 权限为零不删除、取消、失效或改写需求与任务身份，不阻止任务筹办、材料接收、安全观察、事件登记或机器事实写入；它只排除普通执行竞争。
4. 自我线程每个有效治理循环最多调用一次安全维护入口。线程只是调度者；数值是否变化只由事实版本、单调实际经过时间和有效证据裁决。
5. 运行消息的低、普通、高、停止四档优先级只服务运行控制。停止消息优先规则不受 SAFETY-C2 覆盖，也不得承载安全任务权限。
6. 当前轻量因果引用、固定安全根兼容路由、FIFO 强类型任务请求队列和日志均不具备本设计要求的权威结构，不能作为已实现证据。

## 2. 合同、术语与不变量

### 2.1 合同版本

| 合同 | 所有者 | 消费者 | 不变量 |
| --- | --- | --- | --- |
| `SAFETY-C1 / v1` | 安全领域服务、因果领域服务及其专用数据操作 | 自我治理、需求、任务和安全审计 | 只从同一稳定因果图版本和正式安全事实形成层值 |
| `SAFETY-C2 / v1` | 安全权限投影服务 | 任务调度组合器、SELF-C1、SELF-C2 | 权限只读派生，来源版本完整，递归闭合为满刻度 |

### 2.2 核心术语

| 术语 | 精确定义 |
| --- | --- |
| 安全根结果 | 某个自我安全轴上的具名根结果状态；是因果路径终点，不是自由文本分类 |
| 具体安全结果 | 可由稳定状态、结果比较及其因果身份指向安全根结果的具名结果 |
| 精确因果深度 | 从具体安全结果沿“原因指向结果”的有效直接因果边到安全根结果的最少边数，类型为非负 I64 |
| 安全层 | 具名来源因果信息、目标安全结果和因果图版本共同形成的版本化维护投影；精确深度只是该投影的当前路径距离，不是持久身份 |
| 安全组 | 深度 1—4 分别映射到组 1—4，深度 `>=5` 映射到组 5；深度 0 不形成普通调度组 |
| 复发机会 | 当前场景中具名风险因素确实有可能再次产生可观察结果的版本化事实 |
| 感知覆盖 | 在确定时间区间内，正式观察能力足以发现该因素复发的版本化覆盖事实 |
| 有效未复发时间 | 复发机会、感知覆盖、场景/因素身份一致并且区间内无复发证据同时成立的区间并集长度 |
| 残余风险约束 | 历史安全事件中尚未被逐项验证排除的具名因素集合 |
| 安全权限 | 某安全层从收到权限中保留的任务执行排序影响系数 |
| 上层剩余权限 | 当前安全组按组内最严格释放倍率传递给下一更高安全组的权限 |

### 2.3 全局不变量

1. 图、层值、阈值、速率、事实、权限和任务关联都必须带稳定身份及规则版本；文本名称不参与裁决。
2. 每个安全层定义分别冻结 I64 的 `Vmin、Vmax、L、H、Rup、Rdown、Utime`，并满足 `Vmin <= V <= Vmax`、`Vmin <= L < H <= Vmax`、`Rup >= 0`、`Rdown >= 0`、`Utime > 0`。不同层的原始值不直接相加、平均或比较。
3. 6120 的 I64 `0..99` 物理安全根值不是分层安全值。某个分层定义只有具名复用该量纲并显式继承控制态语义时，`0`、`1` 才是其控制态，且被动维护不得跨入或跨出；其它分层定义不得套用这两个数值的根轴含义。
4. 普通被动维护只在本层合法值域内运行：低位以下最多回升到低位阈值，高位以上最多回落到高位阈值，中间闭区间保持原值。
5. 同一维护请求、相同输入版本和相同截止单调时间重复调用必须返回同义结果，不得重复累计时间或重复写值。
6. 没有有效路径的具体安全结果记为“未归层”，不得伪造深度；有环、坏端点、混合版本或图不一致时拒绝该图版本，不能降级成未归层。
7. 任何安全值变化都同时具备前值、后值、变化量、方向、时间区间、规则版本和因果来源；普通 bool、线程轮次或函数返回码不构成变化事实。

## 3. 权威结构和只读投影

### 3.1 领域类型化记录

| 记录 | 最小字段 | 写入方 | 读取方 | 生命周期 |
| --- | --- | --- | --- | --- |
| 稳定因果图快照 | 自我、根结果、图版本、节点集合版本、边集合版本、适用场景版本、发布时间 | 因果领域服务 | 安全分层服务 | 不可变版本；旧版保留 |
| 直接因果边记录 | 边身份、原因结果、结果结果、证据动态、适用场景、有效期、规则版本、记录状态 | 因果领域服务 | 因果图读取器 | 历史持久；失效不删除 |
| 安全层定义 | 自我、抽象定义、值域、低/高阈值、升/降速率、时间单位、规则版本 | 安全领域服务 | 维护和权限服务 | 定义换代，不原地改版本 |
| 安全层当前记录 | 维护族身份、来源因果、目标/最终结果、图版本、当前值状态、方向维护纪元、累计有效时间、最后处理事实游标 | 安全领域数据操作 | 安全服务、权限服务 | 当前值可换代；历史值保留 |
| 安全事件记录 | 事件身份、主体、层/结果、场景、发生时间、来源动态、严重度材料、规则版本 | 安全领域服务 | 维护和结算 | 历史持久；不覆写 |
| 事件因素记录 | 事件、因素身份、场景适用、复发判定合同、证据版本、记录状态 | 安全领域服务 | 维护、因果和观察 | 历史持久；新增因素形成新版本 |
| 因素排除记录 | 事件、因素、结论、验证方法、验证动态、适用场景、有效期、版本 | 安全领域服务 | 残余风险聚合 | 只有“验证排除”解除约束 |
| 未复发证据段 | 事件、因素、场景、运行代次、单调开始/结束、复发机会、感知覆盖、身份比较、复发结论、版本 | 观察/安全正式入口 | 安全维护 | 不可变时间段；重叠不重复累计 |
| 安全权限快照 | 来源图/层值/阈值/关联版本、组 1—5 收到/保留/释放权限、四道门和规则版本；`Q1+Q2+Q3+Q4+Q5=M` | 安全权限投影服务 | 任务调度 | 可重建只读投影；不存在五组之外的剩余池，不反写层值 |
| 任务安全关联 | 任务、来源安全需求、根结果、层身份或未归层标记、关联版本、当前性 | 任务/需求正式入口 | 权限投影和调度 | 随任务冻结换代；历史保留 |

“完整安全读取材料”只允许是上述记录按请求组装出的值式投影，不建立第二份无类型权威容器。索引只按自我、根结果、层、事件、因素、场景和任务加速定位，可由权威记录重建。

### 3.2 安全层身份

安全层维护族稳定键为：

```text
(自我稳定身份,
 本层安全目标或结果身份,
 最终具体安全结果身份,
 安全抽象定义身份,
 安全抽象定义版本)
```

某一时刻的投影单元在维护族之上追加：

```text
当前来源因果信息
+ 当前因果图版本
+ 当前安全值状态版本
+ 阈值和维护参数版本
+ 当前残余风险约束集合版本
```

精确深度和安全组都不是维护族持久身份组成部分。因果图换代时旧投影退出当前使用，新投影必须显式承接仍适用的安全值历史。深度 `0` 表示需求目标就是具体安全结果，只进入直接安全处置/主动结算路径，不建立普通任务调度组；深度 `7` 的投影进入组 `5`，但仍以精确深度 `7` 参与比较、读回和审计。

同一深度存在多个来源因果或安全结果时，各维护族保持独立当前值和阈值。6150 在同一组内按各层释放倍率的最小值门控上层，不能先平均层值。影响层值的主动结算先按 6100 的同源去重和安全根控制规则形成候选，不在本设计中以平均、计数或任务数量直接合成。

## 4. 稳定直接因果图和精确分层

### 4.1 输入准入

`读取稳定安全直接因果图` 必须一次读取并冻结：

```text
自我与安全根结果身份
图版本、节点集合版本和边集合版本
全部可达直接因果边及端点
每条边的证据动态、适用场景、有效期和记录状态
请求场景和事实截止版本
读取代次
```

准入要求：

1. 所有边均为正式直接因果，不接受轻量因果引用、概率候选、文本解释或只含分数的模板命中。
2. 原因端、结果端和证据端点都存在且当前可读；边版本、集合版本和图版本一致。
3. 适用场景与请求场景可比较；不适用边从投影中排除，但其存在不被删除。
4. 同一边身份不得出现异义端点；同一结构键的多版本只能取请求冻结版本下的唯一当前项。
5. 读取完成后重读图版本。前后版本不同、集合数量或内容哈希不一致时返回版本漂移，不形成部分图。

### 4.2 图方向、环检测与最短路径

直接边方向固定为：

```text
原因结果 --直接导致--> 结果结果
```

对一个需求目标结果 `X`，从 `X` 沿正向边搜索到一个或多个具体安全结果 `R`。`X` 与某个 `R` 是同一结果身份时深度为 `0`；`X` 直接导致 `R` 时深度为 `1`。

算法固定为：

1. 按稳定结果句柄和稳定边句柄升序建立邻接表。
2. 对冻结图中与根结果相关的全部节点执行有向环检测。发现自环、回边或共享分支形成的环时拒绝整个根结果投影，并返回构成环的最小确定性句柄序列。
3. 图无环后，从 `X` 执行确定性广度优先搜索，边数使用非负 I64。每次加一前检查 I64 溢出。
4. 到任一具体安全结果的最少边数构成候选深度；能够到达多个具体安全结果时取全体候选中的最小值，并保留全部取得该最小值的具体安全结果和来源路径。
5. 存在多条同深路径时，以“完整因果身份、边句柄序列、节点句柄序列”的规范序选择排序主路径；全部同深路径仍保留为可回读只读来源。
6. 搜索穷尽仍不能到达任何具体安全结果时，形成带图版本的“未归层”结果；未归层不是安全或不安全结论。

不得固定最大搜索深度，也不得为了五组展示截断搜索。恢复后必须按发布的因果图和索引重算投影，并与持久化图版本互证。

### 4.3 五组投影

```text
精确深度 = 0      -> 不形成普通调度组
精确深度 = 1..4   -> 安全组 = 精确深度
精确深度 >= 5     -> 安全组 = 5
```

| 安全组 | 包含精确深度 | 组内顺序 |
| --- | --- | --- |
| 1 | 1 | 精确深度、主路径句柄序列、层身份 |
| 2 | 2 | 同上 |
| 3 | 3 | 同上 |
| 4 | 4 | 同上 |
| 5 | 全部 `>=5` | 精确深度升序，再按主路径、层身份升序 |

组只是有界检索和正常任务调度路由。深度 0 的具体安全结果本体不形成调度组；任务不能因“直接处置”名称取得旁路权限，必须承接一项绑定预防、处置或阻断因果信息的来源需求，并由该因果信息到具体安全结果的一条当前有效直接边机械形成 `D=1、G=1`。权限递归、任务关联、并列比较和审计始终保留精确深度。

## 5. 安全事件、因素与有效未复发时间

### 5.1 事件和因素完整性

一个安全事件至少具备：

```text
事件稳定身份
自我和受影响安全结果/层
发生场景与发生时间
前状态、后状态或风险变化
来源观察/动作动态
已知具名因素集合及集合版本
尚未知因素标记
事实与规则版本
```

“未知因素标记”表示因素搜索尚未闭合，不能生成一个名为“未知”的可排除因素。新增已确认因素形成事件因素集合新版本；历史集合不改写。

因素排除使用三值结论：

```text
待验证
验证排除
验证未排除
```

只有带验证动态、适用场景、有效期和版本的“验证排除”从残余因素集合中移除该具名因素。待验证、未知、证据过期或验证未排除均继续保留残余风险约束。

### 5.2 有效时间段准入

时间本身不产生安全事实。时间段 `[开始, 结束)` 只有同时满足以下条件才计入：

1. `结束 > 开始`，二者来自同一运行代次内的单调时钟。
2. 自我在整个区间内处于与事件因素适用范围可比较的当前场景。
3. 该因素在该场景确实具有复发机会，且机会事实覆盖完整区间。
4. 感知监测在完整区间内有效，能够检测该因素的具名复发结果。
5. 事件因素、场景、感知目标和实际观察中的稳定身份一致，版本未漂移。
6. 区间内不存在该因素的复发事件，也不存在覆盖空洞、身份未知或观察结果待确认。

跨运行代次、系统休眠、时钟回退、恢复间隙或监测空洞不得直接拼接。只有另有正式外部时间证据并重新通过上述条件时，才可形成新的有效证据段。

### 5.3 去重与聚合

同一 `(事件, 因素, 场景, 证据版本)` 的有效区间先按开始时间、结束时间、证据身份排序，再做区间并集；重叠部分只计一次。

一个安全层的低位回升有效时间采用保守聚合：

1. 先收集该层全部当前残余事件；
2. 每个事件收集全部未被验证排除的具名因素；
3. 每个因素计算自上一处理游标后的有效区间并集长度；
4. 层的本轮有效未复发增量取所有残余因素增量的最小值；
5. 任一残余因素没有复发机会、没有有效覆盖、身份不可比或仍为未知因素时，本轮增量为零。

该最小新增有效时间同时供本轮低位回升或高位回落使用，不表示因素风险相等。高位回落除仍存在残余风险约束外，也必须具备本轮新增有效未复发时间；不能仅按物理时间或“没有收到消息”回落。场景或因素版本漂移时停止累计并要求重读。

多事件、多因素和多场景的确定性排序固定为：

```text
安全组升序
-> 精确深度升序
-> 事件发生时间升序
-> 事件稳定句柄升序
-> 因素稳定句柄升序
-> 场景稳定句柄升序
```

排序只保证重放一致，不改变风险强弱。主动安全结算需要聚合风险强弱时仍由 6100 的必要物理因素、风险余量、同源去重和安全根控制规则裁决。

## 6. 分层安全值维护状态机

### 6.1 速率和时间单位

每个安全层定义冻结两组非负 I64 速率和一个正 I64 时间单位：

```text
低位回升速率 Rup
高位回落速率 Rdown
维护时间单位 Utime
```

每个方向分别建立维护纪元，保存旧、新累计有效未复发时间 `Told`、`Tnew`。确定性公式为：

```text
累计应变化量 Q(T) = floor(R * T / Utime)
本轮新增变化量 Delta = Q(Tnew) - Q(Told)
```

计算使用至少 128 位中间值，首版曲线固定为分段线性。低于一个值单位的有效变化通过累计时间保留，不能强行变化 1，也不能由执行代码改为指数、循环次数步进或浮点近似。无实际值变化但累计时间或处理游标前进时，仍形成维护进度记录；它不是安全值变化动态。

### 6.2 同批输入优先顺序

新事件、复发、因素变化和排除证据必须先由各自合法领域完成准入并正式发布；安全维护入口不代写这些上游事实。安全评估在一个冻结的已发布事实批次内按以下顺序处理：

```text
冻结已发布的新事件、复发、因素和排除版本
-> 先处理新事件、复发和因素变化
-> 执行主动安全事实结算
-> 读取验证性排除并重算残余风险约束
-> 判定是否允许被动维护
-> 原子确认、读回并最后发布
-> 从发布后的同一一致性快照计算最终权限投影
```

新事件优先于被动维护。某层本批存在新事件时：

1. 先按已发布事实结算调整该层值；
2. 重置受影响事件、因素和方向的有效时间维护纪元；
3. 本批不对该层执行被动回升或回落；
4. 其它未受影响层仍可按自身版本维护。

全部具名因素均被验证排除且不存在未知因素时，解除该事件的残余风险约束，并按当前物理事实、状态、动态和因果重新结算；不得直接把安全值写到高阈值、值域上界或历史事件前值。

### 6.3 被动分支

设当前安全值为 `V`，低位阈值为 `L`，高位阈值为 `H`：

```text
V < L
-> 只消费有效未复发时间
-> V' = min(L, V + 低位整数变化量)

V = L
-> 保持原值

L < V < H
-> 保持原值，不储存将来跨区间后的变化额度

V = H
-> 保持原值

V > H 且仍有残余风险约束、存在新增有效未复发时间
-> 消费有效未复发时间
-> V' = max(H, V - 高位整数变化量)

V > H 且全部具名因素已验证排除、无未知因素
-> 不执行高位回落，转当前事实重结算入口
```

若该分层定义具名继承 `0`、`1` 控制态语义，控制态由定义的主动入口管理，被动维护不得跨入或跨出。其它定义不对数值 `0`、`1` 附加根轴语义。

低位回升的目的只是让已取得有效未复发证据的历史危险逐步退出过度占用；不能因为其它任务等待、队列拥堵或希望释放权限而提高安全值。高位回落只在残余因素未闭合时保持警觉，不证明风险已经发生。

### 6.4 重复、漂移和恢复

维护幂等键至少包含：

```text
自我、层身份、层值版本、事件/因素集合版本、
图版本、场景版本、有效证据区间及覆盖版本、维护方向、维护规则版本
```

同键同义返回已形成结果；同键异义写前拒绝。任何来源版本在准备后变化都撤销候选并返回正常版本漂移。准备成功后确认、读回或发布不一致属于内部逻辑错误。

恢复时只恢复已发布层值、事件、因素、排除、证据段、处理游标、方向维护纪元、累计有效时间和规则版本。单调时间证据不能跨运行代次直接拼接；恢复后只有新形成并通过完整准入的证据段可以增加累计有效时间。

## 7. 递归任务执行优先级权限

### 7.1 固定刻度、单层映射和组释放倍率

权限满刻度固定为：

```text
权限满刻度 M = 1,000,000
本层最小保留比例 = 10,000       // 1%
最大向上释放比例 = 990,000      // 99%
```

每个安全层先独立计算向上释放倍率 `R(V)`：

```text
V <= L:
    R(V) = 0

L < V < H:
    R(V) = floor((V - L) * 990,000 / (H - L))

V >= H:
    R(V) = 990,000
```

同一组包含多个当前有效安全层时：

```text
组释放倍率 R_g = min(组内全部当前有效安全层的 R(V))
```

必须保留全部取得最小值的限制层来源。同组层值可以有不同值域，只分别用自己的 `L/H` 映射为统一释放倍率，不能直接比较或平均原始值。

边界等号语义固定：

- `V <= L`：该层阻止本组向上释放；
- `V >= H`：该层允许本组向上最多释放 99%；
- 只有开区间 `L < V < H` 使用线性插值。

存在有效安全路径却缺维护单元、当前值、阈值或兼容版本时，本组释放倍率不可用，上层权限为零并形成具名结构缺口。空组不把权限赠给更高组；该组释放倍率按零处理，收到权限保持为本组未消费权限。

### 7.2 四道递归门控与五组闭合

组 1 到组 5 固定形成四道门控。设组 `g` 收到权限为 `A_g`、组权限为 `Q_g`：

```text
A_1 = M

对 g = 1..4：
    A_(g+1) = floor(A_g * R_g / M)
    Q_g = A_g - A_(g+1)

Q_5 = A_5
```

组 5 是终端汇总组，不再向不存在的“组 6”释放。全部乘法使用至少 128 位中间值，默认向下取整；余量保留在更低组，因此：

```text
0 <= Q_g <= 1,000,000
Q_1 + Q_2 + Q_3 + Q_4 + Q_5 = 1,000,000
```

不得在每层重新发行 100%，不得在递归后再做浮点归一。某组没有可执行任务时，其 `Q_g` 保持未消费，不重分配给其它组。

### 7.3 无安全深度、材料缺失与失效投影

目标不存在当前有效安全因果路径时形成“无安全深度”，按非安全任务规则处理，不伪装成组 0 或固定历史层。目标自称安全但缺少规范要求的路径时形成具名结构缺口。

安全路径存在但门控材料不完整时，受影响组释放倍率不可用、其上层权限为零；不得用历史显示值、其它层平均值、`I64_MAX` 或满权限兜底。因果图有环、坏端点或混合版本时拒绝受影响投影，旧投影只有仍满足自身有效期和全部来源当前性时才能继续使用。

## 8. 任务关联、基础优先级合成和确定性排序

### 8.1 任务权限

任务安全关联来自需求—任务、需求目标和安全层投影的正式关系。调度器不得按任务名称、方法名称、文本标签或队列来源猜测安全层。

一个任务关联多个安全需求时：

```text
任务安全权限 = max(全部当前有效来源需求的组权限 Q_g)
```

不得求和、平均或按关联数量放大。多个来源得到相同最高权限时，保留全部并列来源；排序主来源按需求稳定句柄、具体安全结果句柄、精确深度和规范化路径键确定。

无安全因果路径的任务按非安全任务规则处理，不消费或伪造安全组权限。权限为零的安全任务保持正式身份、筹办、冻结和材料接收能力，但不进入普通执行候选集合；安全事件响应任务同样必须经具名来源需求和 `D>=1` 的因果投影取得权限。只有不属于任务执行的系统停止控制消息按 8100 独立优先，不得把它解释为安全任务权限旁路。

### 8.2 基础优先级合成

任务领域提供非负 I64 版本化基础优先级 `B`。它只能来自 5200 定义的当前调度优先级及其正式来源版本，不等于结算预算，也不得由安全服务修改。

```text
任务有效排序值 = floor(B * 任务安全权限 / 1,000,000)
```

使用至少 128 位中间值。安全权限控制参与影响，基础优先级保持任务之间的正式排序材料；任何一项版本缺失都不补默认值。

### 8.3 调度顺序

任务管理线程每次消费前取得同一权限快照截止版本下的候选集合。组 1—4 的默认比较键为：

```text
任务有效排序值降序
-> 任务安全权限降序
-> 任务基础优先级降序
-> 任务创建序号升序
-> 任务稳定句柄升序
```

组 5 汇总全部精确深度 `>=5` 的任务。为避免把所有深层任务压成一个无语义桶，每个组 5 候选还必须冻结：

```text
当前分层安全值及其值域/阈值版本
归一化安全不足度
预计达到目标/阈值的时间，或无法预计时的有效时间证据及完整性
未处理后果严重度
因果成立强度、样本依据和规则版本
真实精确深度、规范化最短路径和稳定来源键
```

归一化安全不足度只在本层自身定义内计算：

```text
不足度 = clamp(floor((Vmax - V) * 1,000,000 / (Vmax - Vmin)), 0, 1,000,000)
```

`Vmax - Vmin` 必须为正，计算使用 checked wide integer。不同层只比较该统一倍率及其规则版本，不比较原始值。预计时间没有正式模型或材料不完整时不得补零或无穷哨兵，而是按“不可预计”排在具备完整预计时间之后，再比较有效时间证据完整性和截止版本。

因果证据序不生成综合置信度。适用范围、因果规则版本和证据截止版本先全部通过当前性门禁；之后先比较当前反例数升序，再比较有效基础正样本数降序。版本号不参与强弱大小，只参与当前性和稳定重放；计数与证据集合不一致时该候选形成具名缺口。

组 5 比较键固定为：

```text
任务安全权限降序
-> 未处理后果严重度降序
-> 归一化安全不足度降序
-> 可预计者优先且预计时间升序
-> 有效时间证据完整性/截止版本降序
-> 因果成立强度降序
-> 因果规则版本和证据版本规范序
-> 任务基础优先级降序
-> 并列最高权限来源中的最小精确深度升序
-> 规范化最短路径键升序
-> 来源需求稳定句柄升序
-> 任务创建序号升序
-> 任务稳定句柄升序
```

组 5 不能把精确深度统一改写为 5。排序完成后、真正迁移到执行中之前，必须重读任务当前性、冻结版本和权限来源版本；漂移时移除该候选并重算，不执行旧快照。

没有可执行任务时返回正常“无可执行任务”。权限全为零时不忙等、不取消任务，可等待事实、层值或权限版本变化唤醒。运行停止消息始终先于业务排序消费。

### 9.2 同会话参与者

1. 安全层类型化记录参与者；
2. 有效时间、方向维护纪元和残余约束读取进度参与者；
3. 当前特征值关系换代参与者；
4. 完整后状态参与者；

7. 可重建索引参与者。

固定过程为：

```text
冻结全部来源版本
-> 准备各参与者候选
-> 逐项写后读回
-> 按依赖顺序确认
-> 统一审计前值、后值、动态、因果和权限来源
-> 最后发布运行期可见代次
```

任一准备失败撤销已准备候选并逻辑内返回；确认后出现读回不一致、撤销失败或半发布属于内部逻辑错误，隔离受影响运行代次并追根因。

权限快照不是本事务参与者，也不持久化为第二份权威事实。安全事实事务最后发布成功后，权限服务从同一当前版本的安全值、阈值、因果图、需求和任务事实形成值式投影；调度冻结前再次复核来源版本。权限形成失败不能回滚已经合法发布的安全事实，只能使本轮调度没有可用权限投影；若纯值算法在合法前置下违反守恒或确定性合同，则隔离调度路径并追根因。

## 11. 当前代码差距和退役边界

### 11.1 因果差距

当前 `海中鱼巣/领域/服务.轻量因果.ixx` 和 `海中鱼巣/领域/因果服务.h` 只具备轻量因果身份、动态准入和候选材料；尚无稳定直接因果图快照、反向/正向完整边枚举、环检测、最短路径、精确深度和五组投影。`SAFETY-C1` 必须作为待实现提供者合同，消费者不得补造同名接口。

### 11.2 自我线程差距

当前 `海中鱼巣/线程/自我线程.ixx` 只在消息批次存在时逐条调用可选治理处理器；等待超时不会调用安全维护。`海中鱼巣/线程/自我治理领域路由.ixx` 仍使用固定当前值 `1`、目标值 `10` 的安全根兼容链。该路径必须由后继生产装配计划退役，不能扩写为本设计算法。

自我线程目标行为是每个有效治理循环恰好调用一次 `调度一轮安全维护`，包括没有业务消息但维护截止时间已到的循环。调度入口显式传入事实截止版本和单调时间；线程不读取或修改层值。

### 11.3 调度差距

当前 `任务执行调度请求` 不含安全需求关联、图版本、精确深度、安全组、权限值、权限来源版本或基础优先级合成证据。当前强类型任务队列按确认入队顺序消费，路由每次 FIFO 取一条请求。

后继实现必须形成候选集合排序或等价稳定优先队列，并在消费前重读权限版本。不得把 `有界运行消息队列` 的四档运行优先级改写成安全权限，也不得破坏停止消息优先和队列停止锁存。

### 11.4 事务差距

当前节点直接身份写入会话已具备节点、关系和索引候选的确认、逆序撤销及最后发布底座，但分层安全记录、状态、动态和因果参与者尚未统一接入。实现计划必须先形成参与者和同代次发布合同，再接自我线程和任务调度。

## 12. 错误、拒绝和无变化分类

| 情形 | 分类 | 结果 |
| --- | --- | --- |
| 无路径且图完整一致 | 逻辑内结果 | 形成“未归层”，不伪造深度 |
| 当前值在中间区间、有效时间为零、无整数变化量 | 逻辑内无变化 | 推进合法游标/累计时间或精确重复，不写值变化动态 |
| 权限为零、无可执行任务、队列停止 | 逻辑内结果 | 保留任务和事实；不执行普通候选 |
| 身份无效、版本漂移、证据过期、场景不可比、覆盖不完整 | 写前拒绝 | 不准备写候选 |
| 因果图有环、坏端点、混合版本或集合自相矛盾 | 结构拒绝 | 拒绝新投影；不得降级成未归层 |
| 新事件与被动维护同批 | 合法优先顺序 | 先事件结算；受影响层本批跳过被动维护 |
| 前置通过后写后读回不一致、确认失败、半发布、撤销失败 | 内部逻辑错误 | 隔离受影响路径并追根因 |
| 调度器反向请求提高安全值、线程轮次作为变化量 | 越权请求 | 写前拒绝并形成治理诊断 |

| SP-A06 | 有效未复发 | 缺复发机会、缺覆盖、场景不一致、身份不一致或出现复发均不累计 |
| SP-A07 | 时间去重 | 重叠区间、重复调用和同截止时间不重复累计 |
| SP-A08 | 低位回升 | 在本层值域内线性回升，累计时间、向下取整和封顶正确 |
| SP-A09 | 高位回落 | 有残余因素才回落到高阈值，不跌破；全部排除转主动重结算 |
| SP-A10 | 中间与控制态 | 中间闭区间无被动变化；具名继承的 0/1 控制态不被动跨越 |
| SP-A11 | 同批优先级 | 新事件 / 复发 / 因素变化先进入主动安全事实结算，再读取因素排除并重算残余约束，最后裁决被动维护；事件层本批不回归 |
| SP-A12 | 原子事务 | 任一候选失败全撤销；确认、读回和最后发布同代次 |
| SP-A13 | 单层权限边界 | `V<=L` 为 100/0，`V>=H` 为 1/99，开区间线性 |
| SP-A14 | 递归闭合 | 组 1—5、组内最小释放倍率和舍入余量下总和精确为 1,000,000 |
| SP-A15 | 多需求任务 | 取最高权限，不求和；同值主解释稳定 |
| SP-A16 | 任务排序 | 组 1—4 基础顺序稳定；组 5 完整比较权限、后果严重度、安全不足度、预计 / 时间证据、因果证据、基础优先级、真实深度、规范化路径、来源需求、任务创建序号与任务句柄 |
| SP-A17 | 零权限边界 | 不执行普通竞争，但任务、筹办、材料和观察不消失 |
| SP-A18 | 线程边界 | 每有效循环最多调用一次；循环次数和运行消息优先级不进入公式 |
| SP-A19 | 恢复 | 不跨运行代次拼接无证据时间；层值、证据、游标、维护纪元、累计时间和版本恢复一致 |
| SP-A20 | 服务隔离 | 服务值衰减、休眠和成功服务结算未被本算法改写 |

分级验证必须区分：

1. 纯值算法和静态合同；
2. 安全领域参与者的故障注入与原子撤销；
3. 因果图、状态、动态和安全记录的真实同会话接线；
4. 任务候选集合排序与零权限行为；
5. 自我线程生产装配、停止、恢复和连续运行；
6. 最终真实风险事件、感知覆盖和场景切换样本。

低级验证不得替代高级完成声明。

## 15. 完成声明边界

本文只冻结设计合同。当前不得声明：

- 稳定直接因果图、最短路径或安全分层已经实现；
- 安全值被动回升/回落已经生产运行；
- 任务管理线程已经按安全权限排序；
- 自我线程已经每轮调用安全维护；
- SELF-C1—C6、最小自我闭环或真实安全能力已经完成。

只有目标规范、流程图、知识图谱和叶子计划范围一致，计划索引登记精确计划版本与 blob，代码按计划实现并通过 SP-A01—A20 及生产接线验证后，才能按对应层级形成候选完成声明。
