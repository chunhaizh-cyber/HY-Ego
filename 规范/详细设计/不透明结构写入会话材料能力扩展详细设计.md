# 不透明结构写入会话材料能力扩展详细设计

日期：2026-07-13

状态：正式详细设计 / JY-320 / #272 施工依据 / 未改代码

## 1. 目标

在不改变四仓库权威边界、结构事务 ABI、独占许可强度和现有精确撤销顺序的前提下，为 `结构写入会话` 增加状态、动态及后续值式领域结构所需的最小材料能力：

```text
写入本会话主信息候选的 I64 值
同令牌读回并确认候选 I64 值
同令牌读取当前节点类型
同令牌读取当前节点主信息锚点
同令牌读取主信息指定 I64 槽位
同令牌按已知源节点和关系类型读取当前关系记录组
```

本设计不建立状态或动态业务语义，不把仓库开放给领域服务，也不把原始令牌从会话中导出。

## 2. 当前代码事实

### 2.1 已有能力

`#264 / SERVICE-DATA-S1` 已实现并验证：

```text
创建主信息候选
创建节点候选
创建关系
绑定主键
主信息可读
节点可读
关系可读
主键绑定匹配
请求提交 / 请求撤销
无提交决定、失败和异常时精确逆序撤销
```

会话不可默认构造、复制或移动，只能由执行器在当前线程栈内以左值引用同步传递。执行器每次只取得一次独占许可。

### 2.2 已确认缺口

当前状态和动态结构需要在主信息槽位中写入值、时间戳和幂等编号，并在同一独占许可内复核已有状态的节点类型、主信息锚点、I64 值和关系材料。现有会话只有“可读”布尔入口，不能表达这些材料。

若 `#266` 直接调用主信息、节点或关系仓库的带令牌入口，就必须让数据操作层取得原始令牌；若改用无令牌入口，则会触发同线程重入门禁或形成写前材料漂移。两者都违反 JY-316 至 JY-319 的分层合同。

## 3. 能力边界

### 3.1 候选值写入

新增：

```cpp
结构写入结果 写入候选I64值(
    主信息句柄 主信息,
    std::uint64_t 槽位,
    std::int64_t 预期值);

bool 候选I64值匹配(
    主信息句柄 主信息,
    std::uint64_t 槽位,
    std::int64_t 预期值);
```

`写入候选I64值` 只接受当前会话 `主信息候选组_` 中仍处于未发布阶段的完整句柄。既有主信息、其它会话候选、已确认候选、已撤销候选、错误仓库或错误版本一律在写入前拒绝。

值写入调用 `主信息仓库::写入I64值(主信息, 槽位, 值, 令牌)`。写入成功后登记：

```text
主信息完整句柄
槽位
预期值
已读回 = false
```

同一候选同一槽位第一轮只允许一次预期写入。重复同值可返回幂等读回，但不得重复加入写集；重复不同值必须入口拒绝，不能覆盖已经登记的候选值。

`候选I64值匹配` 必须使用同一会话令牌读回并精确比较。匹配后才把对应值写集标记为已读回。请求提交时，任一值写集未读回都必须拒绝提交并进入撤销收口。

### 3.2 通用结构材料读取

新增窄入口：

```cpp
std::optional<节点类型> 读取节点类型(节点句柄 节点) const;
std::optional<主信息句柄> 读取节点主信息(节点句柄 节点) const;
std::optional<std::int64_t> 读取I64值(
    主信息句柄 主信息,
    std::uint64_t 槽位) const;
std::vector<关系记录> 读取关系记录组(
    节点句柄 源节点,
    关系类型 类型) const;
```

这些入口只在当前所有线程、会话未结束且句柄有效时调用对应仓库带令牌读取。返回值是普通值式材料，不包含仓库引用、令牌、许可、锁、候选或可变指针。

关系记录组必须沿用关系仓库当前稳定顺序。它只召回当前记录材料；调用数据操作层必须根据关系仓库编号构造完整关系句柄，并用 `关系可读` 复核当前版本、端点、关系类型和顺序号。材料重复、端点矛盾或完整句柄无法读回属于内部不一致。

### 3.3 读入口的非成功语义

```text
调用前句柄无效、会话已经结束：逻辑内返回，不写结构。
有效句柄在取得许可前已经版本漂移：返回空材料或版本漂移，由数据操作层在第一笔写入前收口。
入口和当前性已经复核后，读取自有候选或刚取得的当前关系却失败：内部不一致，追根因。
```

会话层不把“空材料”解释为状态不存在、动态缺失或业务拒绝；业务解释留在服务专用数据操作层和领域服务。

## 4. 值写集与撤销

新增私有 `候选I64写入记录`：

```text
主信息
槽位
预期值
已读回
```

提交前完整读回条件扩展为：

```text
所有主信息候选已读回
所有节点候选已读回
所有关系写集已读回
所有索引写集已读回
所有候选 I64 值写集已读回
```

撤销顺序仍为：

```text
索引
-> 关系
-> 节点候选
-> 主信息候选
```

不增加单独值删除或旧值恢复。原因是值写入只允许本会话新建的主信息候选；撤销该主信息候选会物理移除其完整值容器。若实现需要修改既有主信息或为值建立补偿日志，说明范围已经漂移，必须停止并退回设计。

## 5. 入口拒绝

`写入候选I64值` 至少拒绝：

```text
跨线程调用
会话不在写入阶段
已请求提交或撤销
已有首次失败
无效主信息句柄
错误仓库或过期版本
主信息不属于本会话候选
同一候选同一槽位重复写入不同值
```

读取入口至少拒绝无效句柄、跨线程访问和已经结束的会话。读取入口不得改变写集或把空材料自动修补为新结构。

## 7. 调用路径

```text
数据操作层
-> 结构写入执行器
-> 一次独占许可
-> 结构写入会话
-> 候选值写入 / 当前结构材料读取
-> 同令牌完整读回
-> 请求提交或精确逆序撤销
-> 释放许可
-> 值式结构结果
```

只有已有或后续正式计划允许的 `数据操作.*` 模块可以在生产路径调用这些能力。领域服务、组合器、线程和消息处理器仍不得导入执行器或接触会话。

## 8. 并发与锁

不改变独占许可语义、事务域、仓库锁和既有锁序。会话不新增锁，不升级许可，不并行执行回调。其它线程的共享读取必须继续在独占许可释放前等待；同线程无令牌读取继续被重入门禁拒绝。

材料读取只发生在独占许可内，目的是复核当前结构，不是建立长生命周期快照。许可释放后返回的材料只能用于形成本次值式结果，不得作为后续写入授权。
