# 运行期上下文首次发布租约与恢复链单一所有权详细设计

日期：2026-07-14

状态：正式详细设计 / 绑定 #249、DQ-141 / JY-338 / 阶段 680 待实施

## 1. 定位

本设计承接 `#270 / 770` 的唯一运行期业务装配和 `#250 / 690` 的版本 1 系统角色初始化，只启用全新完整候选的一次首次发布、同身份租约和并发单赢家。

恢复链在本设计中只冻结所有权与调用方向：

```text
#226：从已发布正式租约取得快照冻结来源。
#229：恢复隔离候选通过跨段屏障后形成不可伪造发布许可。
#237：消费该许可，把恢复候选适配到同一上下文类型、同一宿主发布槽和同一租约类型。
```

本批不实现快照格式、恢复解析、恢复候选、恢复许可、恢复发布或启动恢复分支。

## 2. 已验证当前事实

1. `运行期上下文` 已唯一拥有结构事务协调器、正式接线、四仓库和运行期业务装配。
2. `运行期上下文::完整()` 当前要求结构核心完整、业务装配完整、版本 1 系统角色清单存在且完整，并由业务装配权威复核成功。
3. `#250` 已验证 21 个稳定键、系统角色完整句柄、场景归属、两条普通父子关系和方法登记材料；完整候选现在可以形成。
4. `运行期上下文宿主::尝试发布` 当前对完整候选仍返回 `首次发布尚未启用`，不会写入 `当前上下文_`。
5. `运行期上下文租约` 已持有 `shared_ptr<const 运行期上下文>`；宿主为空时租约无效，宿主持有完整上下文后其现有有效性合同可以成立。
6. 当前 `尝试发布` 只有三个自检调用点：运行期上下文自检、运行期组合分层自检和系统角色初始化自检；没有生产启动调用点。
7. 当前没有 HYSNAP01 正式实现、隔离恢复候选、恢复屏障、恢复发布许可、恢复适配或恢复启动分支。
8. `#226`、`#229`、`#237` 都把唯一运行期上下文、宿主和租约作为上游所有权合同，但各自仍有独立未满足依赖。

以上为 `9c97eee` 当前代码事实；本设计的新增接口和阶段 680 不是当前实现事实。

## 3. 范围与排除

### 3.1 本批范围

1. 把当前通用名 `尝试发布` 收窄并改名为 `尝试首次发布全新上下文`，公开语义只允许全新初始化形成的完整候选。
2. 在唯一宿主发布锁保护下，把首个完整候选的同一 `shared_ptr` 写入唯一 `当前上下文_` 槽并返回 `已发布`。
3. 已发布后拒绝任何重复或第二候选，不做在线替换。
4. `读取租约()` 继续复制宿主持有的同一 `shared_ptr<const 运行期上下文>`；多个租约必须指向同一上下文身份。
5. 更新三个现有自检调用点，新增独立 `自检.主装配事务` 真模块，登记阶段 680。
6. 用静态扫描和文档合同冻结 `#226/#229/#237` 的单一所有权边界。

### 3.2 明确排除

```text
不实现 HYSNAP01、文件编码、快照发布或快照解析。
不构造隔离四仓库、领域侧表、恢复系统角色或任务中断迁移。
不实现 #229 恢复发布许可或 #237 恢复候选适配。
不新增恢复专用上下文、宿主、租约、协调器或发布槽。
不切换入口生产对象，不选择全新 / 恢复启动模式。
不启动线程、SQL、控制面板、D455、外设或缓存消费者。
不修改四仓库、事务 ABI、系统角色初始化、运行期业务装配或领域业务服务。
不恢复 #214、#257，不执行 #271 兼容路径迁移。
```

## 4. 单一所有权合同

### 4.1 对象所有权

```text
全新候选构造方
-> shared_ptr<运行期上下文> 候选
-> 唯一运行期上下文宿主
-> 唯一 当前上下文_ 发布槽
-> shared_ptr<const 运行期上下文> 租约
-> 发布后消费者只读取得上下文身份和正式业务入口。
```

`运行期上下文` 仍不可复制。宿主不暴露 `当前上下文_` 的可写引用；租约不提供宿主写入权。候选的原始 `shared_ptr` 与宿主保存值可以共享同一对象，但只有宿主发布槽裁决“当前运行期”。候选调用方保留的指针不得成为第二发布槽或当前事实来源。

### 4.2 唯一身份

首次发布成功后必须同时成立：

```text
宿主不为空。
读取租约有效。
租约读取地址等于发布候选对象地址。
重复读取的全部租约地址相等。
租约读取的结构核心、业务装配和系统角色仍完整。
```

日志、阶段输出、显示文本或 `shared_ptr::use_count` 不裁决身份；身份只按宿主持有对象和租约读取对象比较。

## 5. 全新候选首次发布入口

目标入口：

```text
运行期上下文宿主::尝试首次发布全新上下文(
    std::shared_ptr<运行期上下文> 候选)
```

处理顺序：

```text
1. 候选为空 -> 无效候选。
2. 结构核心不完整 -> 结构核心不完整。
3. 业务装配不完整 -> 缺少领域服务装配。
4. 完整性复核失败 -> 缺少初始化完成材料。
5. 取得发布锁。
6. 宿主已有当前上下文 -> 已有当前上下文。
7. 在提交点前再次复核候选完整性；复核失败保持宿主为空。
8. 把同一候选 shared_ptr 移入 当前上下文_。
9. 返回 已发布。
```

步骤 7 是发布提交点前的最终复核，用于避免首次检查与写入之间的候选状态漂移。发布锁不得在持有期间调用日志落盘、弹窗、文件 I/O、外设、用户回调或线程等待。当前完整性复核只进入上下文和业务装配现有只读路径，不反向获取发布锁。

`首次发布尚未启用` 状态完成历史门控职责后应从当前公开发布结果中移除；不得保留一个永远不应返回的成功前占位状态掩盖遗漏。

## 6. 并发与租约

### 6.1 并发首次发布

多个线程可同时提交不同的完整全新候选。每个候选可以在锁外完成第一轮完整性检查，但唯一提交点必须由同一发布锁串行保护。

验收结果固定为：

```text
恰好一个调用返回 已发布。
其余完整候选返回 已有当前上下文。
宿主持有的对象等于唯一赢家候选。
落败候选不进入宿主，不影响赢家结构。
```

本批不引入 `std::async`、工作队列或常驻线程。并发自检可以使用已有自检允许的短生命周期 `std::thread`，生产模块不得新增线程。

### 6.2 租约

`读取租约()` 与发布共享同一宿主锁，只复制当前 `shared_ptr`。发布前默认租约无效；发布后所有租约有效并指向同一对象。租约释放只减少共享所有权，不清空宿主，不构成停止或热替换。

本批不新增“撤销发布”“清空宿主”或“重新发布”入口。停止顺序和宿主释放由后续启动 / 关闭专项定义。

## 7. 恢复链单一所有权

### 7.1 #226 快照冻结来源

`#249` 完成后，`#226` 只获得以下上游事实：已发布正式上下文存在，且可由租约定位同一接线、四仓库和业务装配所有者。它不因此自动可执行；仍须满足 #226 的权威侧表、删除、防复活、冻结和格式映射前置。

`#226` 禁止：

```text
从 #217 隔离自检组冻结“运行期快照”。
临时新建协调器、仓库组、宿主或租约。
把候选上下文或调用方保存指针当作当前运行期。
```

### 7.2 #229 恢复发布许可

`#229` 继续独占恢复屏障和不可伪造发布许可。许可必须绑定恢复候选身份、快照身份和事务身份，不可复制、不可序列化，也不由 #249 预造。

全新发布不需要恢复许可；恢复发布必须需要恢复许可。两条路径不能通过一个不区分来源的通用公开入口合并。

### 7.3 #237 恢复适配

`#237` 必须复用：

```text
同一个 运行期上下文 类型。
同一个 运行期上下文宿主 类型和对象。
同一个 当前上下文_ 发布槽。
同一个 运行期上下文租约 类型。
同一个协调状态与业务装配所有权。
```

未来恢复入口必须是只接受 #229 能力的专用窄入口或由宿主友元消费的不可伪造适配结果。它不得调用 `尝试首次发布全新上下文`，不得把恢复候选重新标成“全新”，也不得保留可绕过许可的通用 `尝试发布(完整候选)` 兼容重载。

## 8. 非成功返回二分

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 空候选、结构核心不完整、业务装配不完整、初始化材料缺失 | 逻辑内返回 | 宿主和结构不变化，不返回有效租约 |
| 宿主已有当前上下文 | 逻辑内返回 | 保持原上下文，不替换、不销毁、不比较候选内容 |
| 并发落败 | 逻辑内返回 | 返回已有当前上下文，落败候选不进入宿主 |
| 发布锁前后候选由完整变为不完整 | 逻辑内返回 | 提交点前拒绝，宿主保持空；登记调用方并发使用问题 |
| 前置通过后宿主持有对象与租约身份不一致 | 追根因解决 | 停止后继消费者，不伪造成功，不用日志或显示兜底 |
| 发布成功后出现第二发布槽、第二宿主或恢复绕过普通入口 | 追根因解决 | 停止依赖计划，退回设计修订所有权链 |

所有发布拒绝都不得写节点、主信息、关系、索引或领域侧表。数量快照用于证明发布入口无业务结构写入，不用于宣称完整恢复原子性。

## 11. 完成与后继边界
