## 1. 设计目标与依据

本设计建立从合法方法结果到场景事实、状态动态和体素模型的逐层提交链：

```text
稳定观察材料
-> 像素簇
-> 存在候选
-> 结构化识别派生需求材料
-> 需求统一入口 / 新任务 / 新筹办轮次
-> 识别任务工作包
-> 唯一已有存在句柄或空句柄
-> 空句柄按规则形成复验或新存在承接需求
-> 具名存在确认入口按 7100 创建并读回新实例（适用时）
-> 场景成员 / 观察事实提交
-> 状态 / 动态完整事实
-> 已确认存在新增的体素材
-> 体素融合 / 分解 / 父子模型
-> 场景体素快照与视觉先验只读投影
```

正式依据为 0050、1140—1170、4010—4070、4210、4220、6200—6250、6300—6330、6360、6370、7100。6210 约束当前值闭合和首次扫描基准，6220 约束场景认知工作区，6230 约束已确认存在的递归细分，6330 约束作用后的一阶改变与关系二次特征，7100 约束新实例身份、场景归属、概念归属、扩充与完整读回。

## 3. PER-C9—PER-C12

### 3.1 PER-C9 / v0.1：存在候选、识别和场景提交

- 提供 / 消费：#369 提供识别派生需求材料给 #367，提供观察事实与体素材来源给 #371、#373；#369 同时只读消费 #367 的工作包合同和 #368 的观察 / 识别结果合同。
- 模块 / 文件：第 2 节 #369 三个精确文件；`识别派生需求材料`、`合法外设当前数据项` 与 `扫描输入材料` 类型唯一由 #367 的 `海中鱼巣/线程/协议.任务感知方法工作包.ixx` 定义。#369 只 import 前两种类型并形成值，`扫描输入材料` 由 #367 基于合法当前数据项或已有基准扫描变化报告形成；#367 不反向 import #369，避免模块循环或双定义。现有需求、任务、存在、场景服务只读复用。
- 输入：第一段输入 PER-C8 观察结果、稳定材料引用、目标场景和来源动作；第二段输入后继新任务执行后形成的 PER-C8 识别结果；两段不在同一方法调用内连续执行。
- 输出：像素簇材料、存在候选、识别派生需求材料、唯一句柄 / 空句柄、新存在承接或复验需求材料，以及前置完整后已提交的观察事实句柄和与之绑定的合法外设当前数据项。
- ABI / 规则版本：`PER-C9 / v0.1`；观察结果、派生需求、识别工作包、识别结果和事实提交各自保留来源版本。
- 写读方：#369 只形成候选和结构化需求材料；需求统一入口、任务承接和筹办形成新回合；#368 识别服务只读识别工作包；具名存在确认 / 领域提交入口按 7100 创建新实例；场景 / 存在领域事务提交事实。
- 生命周期：材料和候选可失效；派生需求只在新任务承接后进入识别；已有或新建存在、关系和观察事实按领域版本持久。
- 并发键：候选身份用于识别；存在 + 场景 + 观察版本用于事实提交。
- 逻辑内返回：无合法簇、无唯一对应、空句柄、需补观察、需冲突复验、新存在承接等待、场景 / 类型 / 版本不匹配。
- 内部错误：两种方法直接互调；新实例创建前置通过后实例 / 场景 / 概念扩充无法完整读回；事实前置通过后场景成员或观察事实无法原子提交 / 读回。
- 恢复：只恢复完整节点、关系和类型化记录并重新准入；报告、队列或候选不能直接恢复为世界事实。
- 漂移退回：识别唯一性、需求 / 任务回流、7100 实例创建、场景成员关系、事实字段或来源动作改变。

### 3.2 PER-C10 / v0.1：状态动态与任务回流

- 模块 / 文件：第 2 节 #370 三个精确文件；复用现有状态 / 动态数据操作和服务，不建立第二套事实仓。
- 提供 / 消费：#370 提供，#373 消费。
- 输入：独立扫描任务返回的首次基准 / 扫描变化结果，或独立跟踪任务结果，以及权威主体和场景、时间、来源动作、前后版本。扫描 / 跟踪结果必须来自各自“结构化需求 -> 新任务 -> 新筹办 -> 匹配工作包”的独立回合。
- 输出：状态事实、动态事实和任务结果引用；三者分别裁决。
- ABI / 规则版本：`PER-C10 / v0.1`；状态 / 动态 DTO 服从 1160 / 1170，任务回流服从 `SELF-C4 / v0.1`。
- 写读方：状态 / 动态领域数据操作写；任务域只读结果引用并裁决任务。
- 生命周期：事实持久；当前状态是关系 / 索引投影，不复制权威数组。
- 并发键：主体 + 特征 + 场景 + 事实版本；动态按来源动作和发生次序附加。
- 逻辑内返回：主体未知、事实字段不完整、版本漂移、首次扫描无可比基准、变化未达到事实门。首次扫描只允许提交当前值 / 当前状态基准，不得形成“发生变化”的动态；只有可比基准和可验证差异同时成立时才形成变化动态。
- 内部错误：前置通过后状态 / 动态无法同事务读回，或任务结果引用事实不存在。
- 恢复：按节点、关系、类型化记录、版本和历史事实恢复；重建当前投影。
- 漂移退回：1160 / 1170 字段、关系 19 / 20、任务结果或动作来源语义改变。

### 3.3 PER-C11 / v0.1：体素融合与分解

- 模块 / 文件：第 2 节 #371 四个精确文件。
- 提供 / 消费：#371 提供，#372 消费。
- 输入：具有材料稳定身份、来源观察材料、发生时间、已确认新增事实 / 关系身份、`未消费` 状态和当前消费版本的观察体素材，以及场景、坐标系、融合 / 分解规格。
- 输出：新体素模型代次、材料消费记录、分解和父子模型关系。
- ABI / 规则版本：`PER-C11 / v0.1`；新增确认事实 / 关系身份、场景事实身份和发布版本必须可读。
- 写读方：体素服务唯一写；场景体素服务只读。
- 生命周期：模型代次与证据历史持久；工作缓存可清空重建。
- 并发键：场景 + 体素模型当前代次控制发布；观察体素材稳定身份 + 新增确认身份 + 消费版本控制一次消费，材料仅一次进入目标代次。
- 逻辑内返回：无融合点、越界、质量不足、坐标不兼容、重复消费、版本漂移。
- 内部错误：前置通过后父子模型、空间不变量、消费记录或发布读回不一致。
- 恢复：从完整场景、模型代次、材料稳定身份、来源观察材料、发生时间、证据、消费状态 / 版本和消费记录恢复，不从显示网格恢复。
- 漂移退回：空间坐标、单元身份、父子模型、消费幂等或发布规则改变。

### 3.4 PER-C12 / v0.1：场景体素与视觉先验

- 模块 / 文件：第 2 节 #372 三个精确文件。
- 提供 / 消费：#372 提供，#373 消费。
- 输入：已发布体素模型代次、场景和空间范围。
- 输出：只读场景体素快照、视觉先验和后继观察候选辅助材料。
- ABI / 规则版本：`PER-C12 / v0.1`；快照、模型代次、坐标 / 位姿和先验规则版本必须绑定。
- 写读方：体素快照服务读取；观察候选组合只消费，不反写事实。
- 生命周期：快照绑定模型代次；模型前进后旧快照仍是历史只读投影。
- 并发键：场景 + 模型代次 + 读取范围；只读无发布占用。
- 逻辑内返回：范围为空、模型不存在、代次过期、无先验命中。
- 内部错误：快照内容与绑定模型代次不一致，或先验覆盖权威观察结论。
- 恢复：从已发布模型重新生成快照和先验；禁止把先验作为事实恢复源。
- 漂移退回：模型读取、快照字段、先验用途或候选融合方向改变。

## 4. 事务与错误收口

1. 候选形成不取得事实身份。
2. 领域写入统一使用 4040 的不透明候选—确认—撤销—最后发布。
3. 外部输入不合法、无唯一对应、无融合点和版本漂移是逻辑内返回。
4. 前置通过后的写入、读回、发布、父子不变量或事实引用异常是内部错误。
5. 数据库、文件、报告队列和控制面板都不能直接发布世界事实。
6. 观察、识别、扫描和跟踪只能通过结构化结果 / 缺口、需求统一入口、新任务和新筹办轮次衔接。

## 5. 验证矩阵

| 类别 | 验证 |
| --- | --- |
| 分层 | 像素簇、存在候选、存在句柄、场景事实逐层不同类型 |
| 识别 | 观察与识别分属两个任务回合；确定范围内唯一句柄或空句柄；轨迹号和算法分数不能替代 |
| 新存在 | 稳定零候选只形成承接 / 复验需求；后继具名入口按 7100 创建实例、场景 / 概念关系并完整读回 |
| 事实 | 状态 / 动态必需字段逐项拒绝；关系 19 / 20 与类型化记录一致 |
| 首次扫描 | 无可比基准只建立当前值 / 状态基准；不得提交变化动态 |
| 事务 | 故障注入下无可读半事实；撤销逆序；最后发布后完整读回 |
| 体素 | 只消费已确认新增；重复、越界、质量差和版本漂移均不提交 |
| 投影 | 快照 / 先验可清空重建且不改变事实 |
| 恢复 | 恢复候选不直接成为事实；完整准入后一次发布 |
| 集成 | #373 以真实 D455 产品跑通场景事实、状态动态和体素代次 |
