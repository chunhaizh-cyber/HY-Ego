## 1. 设计定位

本设计把迁移大纲当前代码复核后的下一批限定为四段：

```text
S1 实例槽位当前值唯一性与统一原始值版本。
S2 真实内容哈希与非权威命中缓存。
S3 值域、稳态与候选比较。
S4 结构化序列化材料与恢复候选。
```

四段只补 FS-03 已经形成正式详细设计、且当前代码仍明确缺失的能力。它们不重复 #180 Vec 原始值基础，也不修改 CONCEPT-S2 至 S5 或 PANEL-TREES-S1 的目标领域服务；共同使用的 `入口.cpp` 只允许按队列串行追加本切片自检，禁止并行覆盖。

代码边界并非完全分离：#185 CONCEPT-S4 已在提交 `4cb0988` 完成，并在 `海中鱼巣/领域/特征服务.h` 建立 `概念特征约束材料` 与 `读取宿主概念特征约束组`；概念身份只消费特征定义和 I64 / Vec 精确值，实例槽位、当前值句柄和可选 Vec 容器版本不进入签名。#188 必须复核并保留该实际契约，把统一原始值版本作为兼容增强。若保留该契约必须修改 `概念图服务.h` 或其它 #188 禁止文件，则属于接口漂移，退回设计计划窗口，不得在执行时扩大范围。

## 3. 已验证当前事实

### 3.1 特征值服务

```text
I64 存在主信息 I64 槽位中。
VecI64 / VecU64 存在特征值服务运行期 Vec 记录表中。
Vec 写入首次版本为 1，同类型替换版本加 1。
I64 与 Vec 类型互斥。
读取原始类型已经存在。
读取容器版本对 I64 返回空，没有统一原始值版本。
```

### 3.2 特征服务

```text
实例特征槽位由“宿主归属槽位 + 槽位模板绑定特征类型”表达。
槽位到特征值使用归属关系。
读取宿主当前特征值遇到多个归属特征值时返回第一个，没有拒绝或追根因。
写入宿主 I64 值在无候选时创建特征值，有候选时原地改写。
特征服务已经封口 I64、VecI64、VecU64、原始类型和 Vec 版本读取。
```

### 3.3 统计与快照

```text
统计服务已经有特征值内容哈希和命中观察命名空间 DTO。
当前“内容哈希材料”明确标记尚未计算真实哈希。
统计服务没有按值版本保存、读取或清空真实哈希记录。
仓库快照服务只评估恢复请求，不生成特征值序列化段，也不写恢复结果。
```

## 5. S1 当前值唯一性与统一版本

### 5.1 实例槽位识别

实例槽位仍按现有结构识别：

```text
槽位节点类型 == 特征
并且存在 槽位 --模板--> 特征类型
并且存在 宿主 --归属--> 槽位
```

本批不为“实例槽位”新增节点类型、主信息字段或文本标记。

### 5.2 唯一当前值口径

对实例槽位读取全部 `归属` 目标后：

| 数量 | 处理 |
| --- | --- |
| 0 | 逻辑内返回“尚无当前值” |
| 1 | 返回唯一特征值句柄 |
| 大于 1 | 前置结构已存在歧义，触发追根因解决，不选择第一个 |

`读取宿主当前特征值` 应提升为公开只读入口或由公开材料入口封装，但仍只属于特征服务。

### 5.3 创建限制

`创建特征值(特征节点)` 保持兼容，但增加分层：

```text
普通特征类型节点：继续允许按现有规则创建多个归属值材料。
实例槽位节点：已有一个归属特征值时写前拒绝再次创建。
实例槽位已有多个归属值：追根因解决，不创建第三个值。
```

本批不删除已有值，不自动挑选保留值，不做数据修复。

### 5.4 统一原始值版本

新增值式材料：

```text
特征值原始材料
  特征值节点
  原始类型
  原始值版本
  I64 值或 Vec 值副本
```

版本规则：

```text
未写值时版本为 0 或无材料。
首次成功写入版本为 1。
每次成功写入版本加 1，包括同值重写。
写入拒绝、类型冲突或内部错误不增加版本。
版本达到 uint64 上限时写前拒绝。
```

Vec 现有 `容器版本` 兼容保留，并成为统一原始值版本的来源。I64 增加服务内部版本记录。不得把节点句柄版本当原始值版本。

### 5.5 S1 预期入口

```text
特征值服务::读取原始值材料(特征值节点)
特征值服务::读取原始值版本(特征值节点)
特征服务::读取实例槽位当前特征值(槽位节点)
特征服务::读取特征值原始材料(特征节点, 特征值节点)
```

原有 `读取特征值原始版本` 兼容入口可扩展为 I64 / Vec 均返回统一版本，不得破坏 #180 Vec 语义。

## 6. S2 内容哈希与非权威命中缓存

### 6.1 规范字节序列

哈希输入固定为：

```text
哈希规则版本（固定宽度小端）
原始类型枚举值（固定宽度小端）
元素数量（固定宽度小端；I64 为 1）
I64 / VecI64 元素先做 ZigZag64，再逐项固定宽度小端
VecU64 元素逐项固定宽度小端
```

规则版本 1 固定使用 64 位 FNV-1a：偏移基数 `14695981039346656037`，质数 `1099511628211`。不得直接对结构体内存、地址、`size_t`、本机字节序或编译器填充字节做哈希。

不把以下材料放进内容哈希：

```text
节点句柄。
特征句柄。
原始值版本。
名称、日志、显示文本。
命中次数或缓存序号。
```

因此不同节点的同类型同内容可得到同一哈希候选。身份与版本仍单独保留在材料中。

### 6.2 哈希材料

```text
特征值内容签名材料
  特征节点
  特征值节点
  原始类型
  原始值版本
  哈希规则版本
  内容哈希64
```

`内容哈希64` 为候选键，不是唯一事实。

### 6.3 缓存记录

统计服务新增进程内记录：

```text
缓存命名空间
特征节点
特征值节点
原始类型
原始值版本
哈希规则版本
内容哈希64
命中次数
最近命中序号
```

唯一缓存键：

```text
缓存命名空间 + 特征值完整句柄 + 原始值版本 + 哈希规则版本
```

原始值版本变化后旧记录自然失效，不覆盖为新事实。读取同哈希候选时只返回当前规则版本且句柄仍有效的值式记录。

### 6.4 原始值复核

新增特征服务入口：

```text
复核特征值内容相同(左特征, 左值, 右特征, 右值)
```

复核顺序：

```text
句柄与归属
-> 原始类型
-> 原始值版本读回
-> 完整 I64 / Vec 内容比较
-> 再次复核版本未漂移
```

同哈希但完整值不同返回“不相同候选”，不视为逻辑错误；缓存若把该候选裁决为相同则属于逻辑错误。

### 6.5 缓存非权威

必须验证：

```text
清空缓存前后原始值读取一致。
缓存缺失时仍能直接计算签名和比较。
命中次数变化不修改节点、主信息、关系、索引、需求、任务或方法。
未知哈希规则版本写前拒绝。
```

## 7. S3 值域、稳态与候选比较

### 7.1 输入

入口只接收：

```text
来源特征句柄。
显式特征值句柄组。
比较规则版本。
```

不得从日志、缓存命中次数、TreeView、SQL 行或线程消息猜测样本。

### 7.2 样本复核

所有样本必须：

```text
归属来源特征。
节点和主信息有效。
原始值已建立。
原始类型一致。
原始值版本可读。
读取前后版本一致。
```

样本重复句柄第一轮拒绝，避免用同一值重复计数伪造稳态。

### 7.3 精确比较

| 类型 | 第一轮比较 |
| --- | --- |
| I64 | 小于、等于、大于 |
| VecI64 | 完整序列等于或不等于；不定义大小顺序 |
| VecU64 | 完整序列等于或不等于；不定义大小顺序 |
| 混合类型 | 不可比，逻辑内返回 |

### 7.4 值域候选

I64 样本生成：

```text
最小值
最大值
样本计数
来源句柄组
来源版本组
规则版本
待复判状态
```

Vec 第一轮不生成顺序区间，只可生成“全部样本完整相同”候选。

### 7.5 稳态候选

最少 3 个不同特征值句柄。第一轮稳定判定只采用：

```text
全部样本原始内容完全相同 -> 稳态候选。
任一不同 -> 非稳态候选。
```

该判定只描述给定样本集合，不证明长期世界事实。后续若需要时间窗口、频率、容差或区间稳定，必须另建算法切片。

### 7.6 不写结构

S3 只返回值式候选材料，不创建：

```text
抽象状态。
抽象动态。
二次特征。
概念节点。
需求结算。
任务状态。
方法成功记录。
```

## 8. S4 序列化材料与恢复候选

### 8.1 材料结构

```text
特征值序列化材料
  格式版本
  节点类型枚举版本
  关系类型枚举版本
  仓库集合编号
  材料序号
  特征完整句柄
  特征值完整句柄
  原始类型
  原始值版本
  元素数量
  I64 值或 Vec 值副本
```

第一轮格式版本固定为 1；枚举版本使用本切片明确常量，不从文本读取。

### 8.2 生成入口

```text
特征服务::生成特征值序列化材料(仓库上下文, 特征节点, 特征值节点, 材料序号)
```

生成前复核归属、类型、版本和值；读取后再次复核版本，版本漂移则逻辑内返回“材料已过期”。

### 8.3 恢复候选

```text
特征服务::复核特征值恢复候选(目标仓库上下文, 序列化材料)
```

输出只包含：

```text
可作为恢复候选 / 已拒绝。
拒绝原因枚举。
规范化原始材料副本。
```

不调用任何写入入口。

### 8.4 拒绝矩阵

| 原因 | 分类 | 结构变化 |
| --- | --- | --- |
| 未知格式或枚举版本 | 逻辑内返回 | 无 |
| 仓库上下文不匹配 | 逻辑内返回 | 无 |
| 无效句柄、错误类型或错误归属 | 逻辑内返回 | 无 |
| 元素数量与内容不一致 | 逻辑内返回 | 无 |
| 原始类型和值字段冲突 | 逻辑内返回 | 无 |
| 版本为 0 或达到非法范围 | 逻辑内返回 | 无 |
| 缓存、命中、稳态、显示或 SQL 字段要求恢复为事实 | 逻辑内返回 | 无 |
| 前置复核通过后候选材料内部组装不一致 | 追根因解决 | 不发布候选 |

### 8.5 明确不做

```text
不生成文件字节流。
不写磁盘或数据库。
不创建、覆盖或删除节点。
不写主信息、关系或索引。
不恢复缓存。
不替换运行期仓库。
```

## 9. 并发与锁

### 9.1 特征值服务

统一原始值版本与对应值写入必须在同一特征值服务写锁内发布。读取材料在共享锁内复制完整值和版本，不返回内部引用。

### 9.2 特征服务

不得同时长期持有关系仓库锁和特征值服务锁。调用顺序固定为：

```text
先通过关系仓库公开只读入口复制归属句柄
-> 释放关系仓库内部锁
-> 调用特征值服务复制原始材料
-> 必要时再次读回关系和版本
```

### 9.3 统计服务

缓存记录使用自身锁；不得持有缓存锁调用特征服务或特征值服务。候选组先复制出锁，再由特征服务复核。

## 10. 逻辑内返回与追根因解决

### 10.1 逻辑内返回

```text
无效或错误类型输入。
特征值不归属特征。
槽位尚无当前值。
实例槽位已有一个值时再次创建。
未知哈希 / 比较 / 序列化规则版本。
样本少于 3、类型混合、不可比或候选为空。
序列化上下文不匹配或材料过期。
请求真实恢复、落盘或业务裁决。
```

必须发生在写前，或只返回值式候选且不产生可读半结构。

### 10.2 追根因解决

```text
实例槽位已有多个归属特征值。
前置通过后值写入成功但版本未递增。
类型和值字段内部冲突。
缓存清空或碰撞改变原始值比较结论。
候选组装后来源句柄、版本或样本数不一致。
恢复候选前置通过后材料内部不一致。
```

不得通过选择第一个、丢弃多余值、重建缓存或忽略错误继续推进。

## 13. 验证层级

每个切片执行时至少验证：

```text
Debug x64 构建。
默认自检入口对应 A01-A10。
连续运行和并发样例按切片风险执行。
高级服务直接依赖特征值服务扫描。
禁止文件和禁止能力扫描。
git diff --check。
tools/check_specs.py。
```

S4 只证明序列化材料和恢复候选，不得把构建通过扩大为持久化或恢复完成。

## 14. 完成声明边界

本详细设计生成后只能声明第二批接口、结构、依赖、拒绝、追根因和验收范围已经确定。

```text

旧鱼巢特征值能力已经迁移完成。
```
