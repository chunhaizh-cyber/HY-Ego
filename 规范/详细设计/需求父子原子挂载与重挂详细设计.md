## 1. 定位

本设计承接需求树第一轮只读材料与拒绝入口，把下一次真实结构变化严格限定为：

```text
创建首个需求父子关系。
把已有父需求原子重挂为另一个父需求。
读取唯一父需求和稳定排序的直接子需求组。
```

它不实现完整需求树算法，也不把阻塞权重、逻辑组织、方向签名、枚举目标或方法候选升格为本切片的机器事实。

## 4. 结构与生命周期

### 4.1 通用挂载结果

在 `关系仓库.h` 定义局部值式结果枚举，建议命名：

```text
节点挂载结果::已创建
节点挂载结果::已重挂
节点挂载结果::已在目标父
节点挂载结果::无效端点
节点挂载结果::拒绝自身
节点挂载结果::拒绝成环
节点挂载结果::内部不一致
```

用途：描述一次通用普通父子挂载请求的结果。写入方为关系仓库；读取方为需求服务和未来其它领域服务；生命周期仅限一次调用，不持久化为业务事实。

### 4.2 需求父子更新材料

建议值式 DTO：

```text
需求父子更新材料
  子需求
  旧父需求（可空）
  新父需求
  通用挂载结果
```

该 DTO 只证明本次关系写入或逻辑内返回结果，不表示需求满足、任务授权、方法成功或阻塞事实。

### 4.3 权威结构

唯一权威结构仍是关系仓库中的有效 `普通父子` 关系：

```text
源节点 = 父需求
目标节点 = 子需求
```

不得把父需求写入主信息字段、缓存、日志、控制面板文本或 SQL 行。

## 5. 目标接口

### 5.1 关系仓库

建议新增：

```text
关系仓库::挂载或重挂节点(节点, 新父节点) -> 节点挂载结果
```

规则：

```text
入口先拒绝失效端点和自身。
进入关系仓库独占锁后重新读取当前普通父关系。
当前父已经是目标父时返回幂等结果，不修改关系。
在同一锁内确认新父不位于节点的后代链。
无旧父关系时创建一条普通父子关系。
有旧父关系时原地修改源节点并推进关系版本。
写入后内部记录不符合预期时返回内部不一致并触发追根因检查。
```

现有 `重挂节点` 行为保持兼容，不在本切片改变其“只处理已有父关系”的语义。

### 5.2 需求服务

建议新增：

```text
需求服务::提交需求父子更新(子需求, 新父需求) -> 需求父子更新材料
需求服务::读取父需求(子需求) -> optional<节点句柄>
需求服务::读取子需求组(父需求) -> vector<节点句柄>
```

需求服务职责：

```text
验证两端均为有效需求。
验证子需求目标状态可读。
拒绝自身、当前同父和调用时可证明的成环。
只调用关系仓库通用入口，不裸改关系记录。
成功后读回唯一父需求和父节点直接子需求组。
子需求组按完整稳定句柄排序，返回副本。
```

## 6. 原子处理

### 6.1 首次挂载

```text
当前父为空
-> 独占锁内复核仍为空
-> 防环
-> 创建普通父子关系
-> 读回父节点等于新父需求
```

如果前置通过后创建关系仍不及预期，不得归为逻辑内返回，必须追根因解决。

### 6.2 重挂

```text
当前父为其它节点
-> 独占锁内复核当前有效关系
-> 防环
-> 原地修改关系源节点
-> 关系版本加一
-> 旧父子节点组不再含子需求，新父子节点组只含一次
```

不得先删除旧关系再创建新关系，避免产生可读的无父或双父中间态。

### 6.3 并发

两个线程同时给同一子需求提交不同父需求时，关系仓库独占锁串行裁决。最终只能存在一个有效普通父关系；后到请求可成为一次合法重挂，但不得出现双父、成环或悬空关系。

## 7. 入口拒绝

需求服务写前拒绝：

```text
子需求或父需求句柄无效。
任一端点不是需求节点。
子需求等于父需求。
子需求目标状态缺失、多目标或类型错误。
当前父已经是目标父需求。
调用时已能证明新父需求位于子需求后代链。
```

关系仓库通用入口继续拒绝：

```text
节点仓库中不存在或已删除的端点。
自身挂载。
锁内复核可证明的成环。
已有多个有效普通父关系的内部结构歧义。
```

拒绝前后节点数、关系数和索引数必须保持不变。

## 8. 逻辑内返回与追根因解决

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 无效句柄、错误节点类型、缺目标状态 | 逻辑内返回 | 写前拒绝，结构不变 |
| 自身挂载或成环 | 逻辑内返回 | 返回拒绝材料，旧父关系不变 |
| 当前父已是目标父 | 逻辑内返回 | 返回幂等材料，不推进版本 |
| 前置通过后首次关系创建失败 | 追根因解决 | 停止依赖路径，定位关系仓库根因 |
| 前置通过后重挂记录或版本不符合预期 | 追根因解决 | 不继续发布需求树材料 |
| 写入后父节点、子节点组或关系数量不一致 | 追根因解决 | 停止依赖路径，不以日志掩盖 |
| 发现同一子需求已有多个普通父关系 | 追根因解决 | 不选择第一个，不自动修复 |
