# NODE-TYPED-MIGRATION NT-P4 九段记录 ABI、严格恢复与旧格式拒绝详细设计

日期：2026-07-24

版本：v0.2

合同：`NT-C4/v0.4`

设计状态：正式施工设计；本设计冻结尚未实现的新格式，不证明任何快照已经生成或恢复。

## 1. 依据与版本裁决

- 0050、4010、4020、4040、4070、4170、4210、4220、4330、5160、5230、5300、7130、7140
- `规范/详细设计/NODE-TYPED-MIGRATION_NT-P4_统一快照恢复退役验收详细设计.md`
- `流程图/20260724_NODE-TYPED-MIGRATION_NT-P4_九段记录严格恢复与旧格式拒绝施工流程图_v0.1.md`

原 `NT-C4/v0.3` 只完整冻结容器头和节点段，没有覆盖 P3 新增的语义 / 因果 / 需求 / 任务 / 方法记录。它尚未实现，因此退出施工合同。`NT-C4/v0.4` 采用：

```text
魔数 = HZYCSNP2 = 48 5A 59 43 53 4E 50 32
容器格式版本 = uint16 2
段清单版本 = uint32 2
九段记录 ABI 版本 = uint32 1
固定小端
```

容器头仍为 104 字节、段头仍为 64 字节；字段偏移、SHA-256 计算方式和资源上限沿用现行 P4 设计，但容器格式版本、段清单版本和魔数按本节更新。

## 2. 基础编码

| 类型 | 编码 |
| --- | --- |
| `节点句柄` / `关系句柄` | 仓库身份 `uint64`、对象编号 `uint64`、版本 `uint32`，共 20 字节 |
| `节点稳定主键` | 命名域 `uint64`、键值 `uint64` |
| 可选项 | `uint8` 存在标志 0/1，存在后紧跟具名值 |
| 字节 / 字符串 / 向量 | `uint32` 数量后接元素；UTF-8 不隐式正规化 |
| 枚举 | 下表或正式规范固定的显式无符号宽度 |
| 有理数 | 分子 `uint64`、分母 `uint64`；分母非零且已约分 |
| 记录 | `uint16 子类型 + uint16 记录格式版本 + uint32 记录字节数 + 具名字段` |

序列化逐字段写入，不复制 C++ 内存、填充、`size_t`、指针、引用、锁或 vtable。

| 3 | 特征值类型化记录 | 特征值完整句柄、原始值版本 |
| 4 | 状态域记录 | 状态完整句柄、记录版本 |
| 5 | 动态域记录 | 动态完整句柄、记录版本 |

## 4. 记录 ABI

| 节点 | 节点编号 u64；稳定主键 16；节点类型 u16；节点版本 u32；记录状态 u8；创建序号 u64 |
| 关系 | 关系编号 u64；关系类型 u16；源句柄 20；目标句柄 20；顺序号 i64；关系版本 u32；记录状态 u8 |
| 特征值 | 特征值句柄 20；记录格式 u32；原始值版本 u64；记录状态 u8；原始类型 u32；按类型编码 I64 / VecI64 / VecU64 |
| 状态 | 状态句柄 20；记录格式 u32；记录版本 u64；记录状态 u8；发生时间 u64；可选接收时间；记录模式 u16 |

### 4.4 段 8—9

用途观察记录字段顺序：观察句柄、记录版本 u64、状态 u8、规则版本 u32、阶段 u16、机会数 / 正数 / 负数各 u64、Q10000 i64。

特征批次发布记录字段顺序保持为：批次身份两个 u64、业务规则版本 u64、账记录格式编码值 u64、状态 u8、宿主句柄、项目顺序 u64、定义 / 实例槽 / 新值 / 来源句柄、可选写前关系 / 值 / 原始值版本、发布后关系句柄。

节点直接新域的精确映射是：

```text
批次身份第一 u64 = 节点直接特征批次身份命名域 ABI，固定为 1
批次身份第二 u64 = 节点直接新域账内 特征批次身份.编号
业务规则版本 u64 = 新域批次记录的业务规则版本
账记录格式编码值 u64 = 新域 u32 记录格式版本的零扩展
状态 u8 = 1，仅表示该记录已经发布
项目顺序 u64 = 新域项目顺序号
```

两分量身份是规范化快照中的稳定 4170 身份，不反向扩写账内批次身份。第一分量是跨恢复不变的格式命名域，不能由调用方提供、从批次号推导或改写成来源运行期域身份；来源运行期域身份只存在于快照头和段来源。账记录格式编码值必须非零且不大于 `uint32` 上限；当前 4170 没有独立可变记录版本，不得凭空编码。候选记录不进入段 9，状态除 1 外均拒绝。

恢复装载时必须验证第一分量等于 1，只把第二分量恢复为新域账内批次身份；恢复再冻结仍输出同一命名域和批次编号，因此九段规范化权威载荷比较不受新运行期域身份变化影响。

项目种类不新增字节字段：写前当前关系、写前当前值和写前原始值版本三个可选字段全部不存在时唯一解释为初始；三者全部存在时唯一解释为换代；部分存在必须拒绝。

## 5. 历史、失效和跨段互证

当前、已失效和已删除记录都保存在所属段，以完整身份和版本排序；不得把历史压成数量或摘要。段 2 保留关系 0—23 的所有版本。

恢复候选至少互证：

1. 每个句柄目标存在且版本 / 类型匹配；
2. 稳定主键命名域、节点类型、高水位和历史不复用；
3. 关系端点、关系 0—23 角色、基数和有效 / 失效链；
4. 段 3—9 每项记录有合法节点或具名记录身份，记录状态和版本连续；
5. 状态 / 动态具名关系完整；
6. 五角色自我身份和固定世界拓扑完整；
7. 任务选择关系 16、任务生命周期 14 / 15、方法生命周期 23 与段 7 一致；
8. 概念关系 9—12 与段 6 签名一致；
9. 特征批次段与特征当前关系、历史换代和项目顺序一致。

任一缺项整体拒绝，不补默认对象、默认关系或空领域记录。

## 6. 恢复 DTO、结果与生命周期

`恢复材料包` 值式拥有九个具名不可变段。结果状态数值统一为：

```text
已形成=1
材料拒绝=2
资源失败=3
已有当前上下文=4
内部不一致=0x8000
```

拒绝原因固定为：魔数、版本、字节序、头长、段清单、段重复、段缺失、段顺序、超限、截断、尾随、摘要、记录帧、未知枚举、身份、关系端点、历史连续性、跨段互证、自我结构、旧运行占用。

隔离装载入口按节点→关系→段3→4→5→6→7→8→9顺序调用具名有界适配器。每个适配器只接收所属段并构造普通路径不可见的新仓；禁止公开注入可写仓库。

`节点直接运行期上下文候选` 独占全部隔离仓，可移动不可复制；验证成功后转成 `已验证运行期上下文候选`。后者只能调用一次：

```cpp
已验证运行期上下文候选&& 取得已验证运行期上下文候选() &&;
```

移动后源对象进入空状态；#352 只能把右值交给首次发布入口。解析、装载、重建、互证或再冻结失败时整个候选析构，当前宿主零可见。

## 7. 旧格式

当前可证明的旧持久格式支持集合为空：

```text
支持旧格式版本集合 = {}
```

`LEGACY-AUTH-MEM-V1`、`LEGACY-EVAL-V1`、未知字节和任何只有说明没有完整魔数 / 头 / 记录布局证据的材料都返回具名拒绝，成功迁移分支不可达。以后增加支持版本必须先独立冻结完整物理格式和逐字段无歧义映射，再提升本合同版本。

旧输入身份固定为来源类型 u16、字节长度 u64、SHA-256 32 字节。识别码：当前格式 1、已登记旧格式 2、未知 3、损坏 4；拒绝码：支持集合为空 1、未知版本 2、布局证据不足 3、字段不可唯一映射 4、历史不完整 5、跨段事实不完整 6。

拒绝结果必须证明输入字节与摘要不变、输出为空、当前宿主不变。只有未来证据完整版本才可输出当前格式，并强制交 #350 再解析、隔离装载、互证和再冻结。

## 8. 验证

验证包括九段逐字段金样、同值同字节、段 / 记录排序、未知值拒绝、全部上限、历史链、跨段悬空、五角色自我、旧占用失效、候选单次移动、析构零可见和空旧格式支持集合。
