# D455 任务域唯一消费与已承接材料引用详细设计

日期：2026-07-21

状态：DD-03 正式详细设计；代码未实现

设计包：DD-03

## 1. 设计目标

本设计把 DD-02 唯一交给任务域的一次性报告消费能力，收束为任务绑定、工作项绑定、等待合同绑定和消费窗口绑定的正式运行材料：

```text
DD-02 一个物理强类型报告队列
-> 任务域唯一消费能力
-> 权威任务、工作项和等待合同重读
-> 产品、范围、目标、质量和时效匹配
-> 任务侧未发布承接候选
-> DD-02 正式交接包
-> 原子发布任务已承接材料记录
-> 不透明已承接材料引用
-> DD-04 冻结工作包领取入口
```

本设计必须同时解决 CG-05 和 CG-06 的结构缺口：

1. 任务域有唯一、强类型、可调用的外设报告承接入口；
2. 工作项、等待项和消费窗口都有稳定身份、版本和当前性；
3. 一个报告只形成一个当前已承接记录；
4. 已承接引用只能由绑定任务和工作项读取或领取；
5. 无匹配、错任务、重复、过期和质量不足在 DD-02 确认前收口；
6. DD-02 确认后不再存在普通失败分支，结构矛盾必须隔离并追根因。

## 3. 当前代码事实与复用边界

1. `协议.任务执行请求.ixx` 的 `任务执行冻结请求` 只有调度请求、任务材料、方法材料、动作材料和一个目标状态值变化适配器，没有工作项身份、等待合同或外设材料引用。
2. `任务管理线程.ixx` 只维护通用运行消息队列和强类型任务执行请求队列；发送入口拒绝非任务执行请求，没有外设报告消费入口。
3. `任务工作线程.ixx` 只从任务管理线程取任务执行请求并形成强类型回执，不得轮询外设流。
4. `服务.任务.ixx` 当前只公开任务承接、生命周期迁移、方法选择、完成和权威读取；没有外设等待合同、工作项、消费窗口和已承接材料入口。
5. `数据操作.需求任务方法.ixx` 当前任务材料只有来源需求、运行态治理存在、场景、目标状态、生命周期、当前选择和实际结果，没有等待特征快照或外设材料记录。
6. 生产模块定向检索没有 `任务承接角色`、`等待类型`、`等待条件当前性`、`重新触发状态`、`等待版本` 等正式任务特征实现。DD-03 必须建立外设承接所需的最小任务等待特征快照，不能只看生命周期枚举。
7. `组合.任务执行.ixx` 和 `装配.运行期业务.ixx` 没有 D455 报告或已承接材料依赖；DD-03 不提前修改它们。
8. DD-01、DD-02 当前仍是待实施设计合同。它们的计划完成并集成前，DD-03 只能登记为依赖门控施工计划。

可复用边界：

- 复用 `节点句柄` 作为任务稳定身份；
- 复用 `任务业务服务::读取任务` 作为任务权威结构读回；
- 借鉴强类型任务执行请求候选的移动独占、确认和精确撤销形状；
- 不复用通用 `运行消息`、`有界运行消息队列` 或线程状态承载任务材料事实；
- 不修改现有任务生命周期、选择和执行协议来伪装已承接材料。

## 4. 总体架构裁决

### 4.1 一个任务域消费者，不等于一个裸线程

DD-03 新建 `任务外设报告承接组合器`。它在构造时移动取得 DD-02 的 `D455任务域消费权`，并绑定一个报告队列世代、一个任务外设材料数据操作实例和一个任务业务服务实例。

该组合器：

- 不可复制；
- 不公开消费权；
- 不记录任务管理线程编号；
- 用内部承接轮次互斥保护“匹配—预留—交接—发布”区间；
- 同一队列世代只允许一个有效实例；
- 后续由 DD-06 任务管理角色长期持有和调度。

线程只是调用载体。唯一消费事实由 DD-02 消费权和 DD-03 组合器身份共同成立。

### 4.2 任务运行态专用侧表

工作项、等待项、消费窗口和已承接材料是运行期任务治理事实，不是长期世界事实，也不复制任务目标、来源需求或生命周期。它们进入专用 `任务外设材料承接数据操作`：

```text
任务稳定句柄
+ 工作项稳定身份和版本
+ 等待特征快照和来源
+ 消费窗口身份、版本和当前性
+ 已承接材料记录、状态和来源报告
+ DD-02 交接包的唯一材料寿命
```

该侧表是正式机器结构，不是缓存命中、日志或显示材料。它在运行期上下文内权威；上下文世代结束后全部引用失效。后续持久化只能保存恢复候选，不得让文件或数据库直接恢复为当前承接事实。

### 4.3 模块分层

```text
线程.协议.任务外设材料承接
-> 只定义任务工作项、等待特征、窗口、引用、状态和值式结果

领域.数据操作.任务外设材料承接
-> 唯一拥有侧表、索引、候选、交接包和承接状态

领域.服务.任务外设材料承接
-> 读取任务权威材料，裁决登记、匹配、当前性和领取准入

领域.组合.任务外设报告承接
-> 唯一持有 DD-02 消费权，编排跨队列和任务侧发布

领域.自检.任务外设材料承接
-> 只用隔离任务和 DD-01 / DD-02 夹具验证全部边界
```

## 6. 强类型身份与状态

### 6.1 身份

```text
任务外设工作项身份
- 工作项编号
- 工作项版本
- 调度批次编号

任务外设等待项身份
- 等待项编号
- 等待项版本

任务外设消费窗口身份
- 窗口编号
- 窗口版本

任务已承接外设材料身份
- 承接记录编号
- 承接记录版本
- 运行期上下文世代
```

全部编号非零，版本从 1 开始，同一运行期上下文内不复用。任务本身继续使用 `节点句柄`，不得用工作项编号、等待项编号或线程编号替代任务身份。

### 6.2 工作项状态

```text
未定义
已登记待激活
当前活动
已挂起
已结束
已取消
已失效
```

工作项只表达调度和材料绑定范围，不替代任务生命周期、筹办占用或方法执行冻结。

### 6.3 等待项状态

```text
未定义
当前等待
已满足待重触发
已过期
已取消
已失效
```

### 6.4 消费窗口状态

```text
未定义
已打开
正在关闭
已关闭
已过期
```

一个窗口只能绑定一个任务、一个工作项和一个等待项。一次等待窗口最多正式承接一个产品；有界订阅窗口必须同时给出最大承接数量和截止时间，不允许无边界订阅。

### 6.5 已承接材料状态

```text
未定义
候选预留中
已承接待领取
工作包保留中
已进入工作包
已消费结束
窗口关闭失效
已过期
内部隔离
```

DD-03 实现到“已承接待领取”和领取候选能力；DD-04 使用领取入口形成冻结工作包并确认“已进入工作包”。普通读取不改变状态。

## 7. 工作项与等待特征合同

### 7.1 工作项

`任务外设工作项` 至少包含：

```text
身份和版本
绑定任务句柄
来源调度批次
创建时间和截止时间
状态及状态版本
最大活动等待项数
来源任务当前生命周期关系和版本
```

登记时必须权威读取任务完整承接结构，拒绝终态任务、任务版本过期和同身份异义内容。工作项不得复制来源需求、任务目标或当前方法。

### 7.2 等待特征快照

`任务外设等待特征快照` 是 5200 / 5210 外设承接所需的最小正式任务特征集合，至少包含：

```text
等待类型：外部事实或输入
等待条件当前性：未满足 / 已满足 / 已失效 / 待复核
重新触发状态：未触发 / 已触发 / 触发失效
等待版本
等待对象身份
合法生产者：D455 外设报告控制域
阻塞来源
重新触发条件
特征来源任务和来源版本
状态段当前性
```

不得只凭任务生命周期 `等待中` 建立等待项。登记时必须同时证明任务完整状态与该特征快照一致；当前代码尚未有通用任务特征系统，因此 DD-03 数据操作必须显式保存来源、类型、值、版本和当前性，不能压成一个 bool 或自由文本。

### 7.3 外设等待约束

`D455任务外设等待约束` 至少包含：

```text
等待意图：一次等待 / 有界订阅
期望 D455 产品类型和运行模式
可选来源设备身份约束
期望观察范围或目标约束
必需局部特征集合及允许质量状态
最大报告年龄
优先级和截止时间
是否允许命中缓存
最大正式承接数量
重复处置策略
约束规则版本
```

原始逐簇产品只能匹配任务域质量门、回查或诊断用途；它不能取得“可进入方法工作包”资格。稳定观察子集、扫描变化和目标跟踪产品在 DD-03 中只标记为“可供后继工作包复核”，最终方法映射由 DD-04 裁决。

## 8. 消费窗口和已承接引用

### 8.1 消费窗口

窗口打开前必须同时读回：

```text
任务仍当前且非终态
工作项当前活动
等待项当前等待
等待特征快照当前
时间窗口未过期
承接计数未到上限
```

窗口关闭后不得重新打开同一版本。需要继续等待时创建新窗口版本或新等待版本，旧记录保留历史。

### 8.2 不透明引用

`任务已承接外设材料引用` 只包含：

```text
运行期上下文世代
承接记录身份和版本
绑定任务句柄
工作项身份
消费窗口身份
```

引用不包含报告产品副本、材料引用组、交接包、裸指针、`shared_ptr`、队列地址或线程编号。读取必须回到服务并复核全部绑定身份和当前性。

### 8.3 内部记录

`任务已承接外设材料记录` 内部至少保存：

```text
不透明引用全部字段
等待项身份和版本
来源 D455 报告身份、产品身份、产品类型和运行模式
来源设备、范围、质量、时间和内容摘要
承接时间、有效截止时间和承接状态版本
重复裁决材料和匹配规则版本
DD-02 D455报告承接交接包
```

产品值和材料寿命只存在于交接包中，不复制到第二个权威容器。值式摘要只做重复候选和审计，命中后必须完整读回产品。

## 9. 确定性匹配规则

### 9.1 候选顺序

任务域每轮承接先冻结当前窗口快照，按下列顺序选择：

```text
等待优先级降序
-> 截止时间升序
-> 任务稳定句柄升序
-> 工作项身份升序
-> 等待项身份升序
-> 报告发生时间升序
-> 报告身份升序
```

报告头中的外设侧等待项引用只作候选提示，不能绕过任务侧权威等待项重读。一个报告同时匹配多个等待项时，只能按上述顺序选择一个。

### 9.2 固定匹配顺序

```text
任务和任务当前性
-> 工作项身份、状态和来源任务版本
-> 等待特征集合和等待项当前性
-> 消费窗口、数量和时间
-> 报告与产品值式合同
-> 来源设备和产品类型
-> 运行模式和用途边界
-> 观察范围或目标约束
-> 局部特征质量要求
-> 报告年龄和截止时间
-> 重复与缓存策略
```

前一步不成立时不得跳到后一步，也不得为了消费队列临时放宽等待约束。

### 9.3 四类产品边界

| 产品 | DD-03 可承接用途 | 是否可直接标记为方法材料 |
| --- | --- | --- |
| 原始逐簇报告 | 任务域质量门、来源回查、诊断 | 否 |
| 稳定观察子集报告 | 观察等待项；空稳定子集可作为结构化空结果 | 只标记待 DD-04 复核 |
| 扫描变化报告 | 扫描等待项；必须有基准和覆盖范围 | 只标记待 DD-04 复核 |
| 目标跟踪报告 | 跟踪等待项；暂失 / 丢失不可静默舍弃 | 只标记待 DD-04 复核 |

识别没有外设报告直通等待类型。DD-03 不创建方法身份，也不判断最终方法入口。

### 9.4 重复

1. 同一报告身份只能形成一个任务承接记录。
2. 同一报告身份、同一等待项的精确重试只能读回既有结果，不再次取得交接包。
3. 同一报告身份绑定不同任务、工作项或窗口属于错绑定，写前拒绝。
4. 不同报告身份但内容摘要相同只进入重复候选；必须完整比较产品身份、来源、时间、范围和负载后，按等待项重复策略承接、释放或具名终结。
5. 目标丢失、安全事件、外设错误和影响目标判断的结构化缺口不得按普通重复静默删除。

## 10. 跨队列与任务侧的提交顺序

### 10.1 固定顺序

```text
读取 DD-02 分类视图并取得报告身份
-> 用唯一消费权准备 DD-02 报告承接候选
-> 读取候选产品和材料元数据
-> 权威重读任务、工作项、等待项和窗口
-> 形成强类型匹配裁决
-> 在任务数据操作中预留未发布承接候选和全部容量
-> 精确读回任务侧候选
-> DD-02 确认正式承接，取得唯一交接包
-> 无失败移动安装交接包到已预留任务侧候选
-> 原子确认任务已承接记录和各身份索引
-> 发布后按不透明引用完整读回
-> 更新等待条件和重新触发状态
```

### 10.2 DD-02 确认前

确认前允许的正常结果：

- 无活动等待项；
- 任务、工作项、等待项或窗口已经过期；
- 产品、范围、质量、时效或用途不匹配；
- 报告已被其它合法轮次认领；
- 同任务精确重复读回；
- 任务侧候选容量不足；
- 当前性在准备前发生合法变化。

仍当前但不匹配时释放 DD-02 承接候选回 `已发布待承接`。可舍弃且已过期、明确重复或具名无命中的报告可以按 DD-02 规则终结；不可舍弃报告不得因无匹配而静默终结。

### 10.3 DD-02 确认后

`确认正式承接D455报告` 成功后，报告已离开活动队列，不能回滚成未消费。任务侧候选必须在确认前完成全部内存、索引和容量预留；交接包移动安装和任务侧发布必须为 `noexcept`、不分配、不可调用用户代码。

确认后若出现下列任一情况，属于内部逻辑错误：

```text
预留候选消失或代次变化
交接包无法安装
报告身份、产品身份或材料元数据与候选读回不同
同一报告出现第二个当前承接记录
记录已发布但交接包不在唯一记录中
索引、容量、状态或完整读回不一致
```

出现内部逻辑错误时把候选和交接包锁存在 `内部隔离` 记录中，停止该承接组合器依赖路径并追根因；不得析构交接包后伪装成普通丢失或等待。

## 11. 当前性、领取和窗口结束

### 11.1 当前性复核

每次读取或领取已承接引用必须验证：

```text
运行期上下文世代一致
承接记录身份和版本一致
任务句柄与记录绑定相同
工作项和窗口身份相同
记录状态允许读取或领取
产品仍在交接包内完整可读
有效截止时间未过
```

任务生命周期变化不自动删除历史记录，但会使不再满足合同的引用失效。历史记录只读，不得被新工作项重新激活。

### 11.2 工作包领取候选

DD-03 提供移动独占 `任务已承接材料领取候选`：

```text
准备领取
-> 状态：已承接待领取 -> 工作包保留中
-> DD-04 完成冻结后确认进入工作包
-> 冻结未形成时释放回已承接待领取
```

同一记录同时只允许一个有效领取候选。错误任务、错误工作项、错误窗口、错误产品用途或过期引用必须在状态迁移前拒绝。

### 11.3 窗口结束

窗口关闭时：

- 拒绝新承接候选；
- 释放未发布任务侧候选；
- `已承接待领取` 记录转为窗口关闭失效并释放交接包；
- `工作包保留中` 必须先释放或完成；
- `已进入工作包` 由 DD-04 工作包生命周期结束后回报消费结束；
- 历史身份、产品身份和强类型终结原因保留为有界审计材料。

窗口关闭、任务取消或运行期停止不能静默销毁处于内部隔离或不可舍弃产品状态的交接包。

## 13. 入口拒绝、逻辑内返回和内部逻辑错误

| 阶段 | 逻辑内拒绝 / 返回 | 内部逻辑错误 |
| --- | --- | --- |
| 工作项登记前 | 零身份、任务不存在、终态、版本过期、同身份异义 | 已登记记录完整读回与输入不同 |
| 等待项登记前 | 缺等待特征、无合法生产者、未知产品、无时效、无范围 / 目标 | 已声明当前等待但特征来源、版本或状态段不闭合 |
| 窗口打开前 | 工作项未活动、等待项非当前、时间过期、数量为零 | 同一窗口版本同时绑定两个任务或等待项 |
| 报告匹配 | 无报告、产品 / 范围 / 质量 / 时效不匹配、合法重复 | 同一冻结窗口快照产生两个不同最高优先匹配 |
| DD-02 承接候选 | 已认领、已过期、消费权无效 | 一个报告同时出现两个候选 |
| 任务侧候选 | 容量不足、当前性在确认前变化 | 候选读回身份、绑定或产品材料变化 |
| DD-02 确认后 | 无普通失败分支 | 交接包不能无失败安装、发布或读回不一致 |
| 已承接引用读取 | 引用过期、错任务、错工作项、错窗口、状态不允许 | 当前记录存在但交接包或索引缺失 |
| 领取 | 已有领取候选、用途待 DD-04 拒绝、窗口关闭 | 同一记录进入两个工作包 |
| 窗口关闭 | 有工作包保留中时逻辑内等待 | 静默销毁不可舍弃或内部隔离材料 |

内部逻辑错误必须按公共诊断规则写逻辑错误日志并在宏开启时弹窗，但日志和弹窗不修复结构、不释放隔离材料、不改变任务状态。

## 14. 并发和锁边界

1. DD-02 队列锁只保护报告活动记录、索引和交接包移动。
2. DD-03 数据操作锁只保护工作项、等待项、窗口、承接记录和任务侧索引。
3. 组合器承接轮次锁保护跨两侧的顺序，但不作为长期业务身份。
4. 不允许持有 DD-02 队列锁时调用任务业务服务或仓库。
5. 顺序固定为先读任务、再准备两侧候选、最后短时完成 DD-02 确认和任务侧无失败发布；不得形成队列锁—任务仓库锁反向依赖。
6. 同一报告最多一个承接候选、一个任务侧候选和一个已承接记录。
7. 同一已承接记录最多一个领取候选。
8. 不同任务、不同工作项的已承接材料读取和 DD-04 后续执行可以并发；不能因同源 D455 或同一根需求全部串行。

## 16. 后继边界

### 16.1 DD-04 承接

DD-04 只允许使用：

```text
任务已承接外设材料引用
-> 准备领取任务已承接外设材料
-> 只读产品值和来源材料
-> 复核任务、方法、产品和执行冻结
-> 确认进入一个冻结工作包
```

DD-04 不得取得 `D455任务域消费权`、`D455报告承接候选` 或 DD-02 分类视图读取权。识别方法没有外设材料领取入口。

### 16.2 DD-05 / DD-06 承接

- DD-05 使用方法结果和任务验证链，不接收原始外设报告消息。
- DD-06 把承接组合器装配给任务管理角色并连续调度；DD-03 不启动线程、不切生产入口。
- 自我线程只接收任务结果、缺口、派生需求或事实变化，不接收交接包和队列项。

### 16.3 完整任务特征系统边界

DD-03 只建立外设承接所需的最小等待特征快照，不宣称 5200 / 5210 全部固定任务特征和状态段已经代码实现。后续完整任务状态特征治理必须另行审计和实施；不得以 DD-03 侧表代替全部任务特征系统。
