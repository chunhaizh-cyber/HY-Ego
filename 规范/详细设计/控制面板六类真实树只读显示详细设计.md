# 控制面板六类真实树只读显示详细设计

编号：PANEL-TREES-D1

日期：2026-07-11

状态：详细设计已生成 / 支撑 CONCEPT-S1 至 S5 连续计划和 PANEL-TREES-S1 依赖门控计划 / 未改 C++ / 未构建 / 未运行程序

## 1. 设计定位

本设计承接用户“完善控制面板，所有树都显示出来”的明确需求，把当前只有世界树和语素树真实 DTO、其余四类只有空分类标题的控制面板，收束为六类真实领域结构的只读树形投影：

```text
世界树
概念树
语素树
需求树
任务树
方法树
```

“树”在本设计中是领域图的只读树形视图，不要求六类底层结构都变成普通单父树。控制面板不得为了显示方便新增业务关系、复制领域节点、扫描 SQL 行、写日志状态或把静态标题解释为真实根节点。

本设计同时固定执行连续性：先执行 CONCEPT-S1，再逐级执行 S2 至 S5，最后执行控制面板六树接线。概念图活动版本和抽象树投影未形成前，控制面板计划保持依赖门控，不允许先落“五棵真树加一个空概念根”。

## 3. 已验证当前事实

### 3.1 控制面板

1. `控制面板树分类` 已固定世界、概念、语素、需求、任务、方法六类；状态和动态只作详情。
2. `控制面板树视图材料` 当前只证明分类允许展示，没有真实节点结构。
3. `控制面板树结构材料` 当前只有单个 `根节点`，不能表达概念四根或多根需求、任务、方法。
4. `控制面板窗口快照` 和窗口构造函数只接收世界树、语素树两份结构材料。
5. `重建当前树视图` 只有分类 0 和 2 插入真实节点；概念、需求、任务、方法只调用 `插入分类树根`。
6. 当前刷新只重新读取总览和六类分类候选；世界、语素结构仍是构造时快照。
7. 当前窗口已有 TreeView 递归插入、展开、分类导航、节点详情和只读拒绝边界，可直接扩展。

### 3.2 当前真实结构来源

1. 世界树已有从世界根沿普通父子关系形成的真实 DTO。
2. 语素树已有语素类型根和 17 个初始化语素入口的真实 DTO；语素服务还可读取对应信息和概念追溯目标组。
3. 需求初始化结果明确提供安全根需求和服务根需求；需求服务可读取目标状态、主体、目标宿主、场景、当前特征状态和最近结算材料。
4. 任务服务可在给定任务句柄时读取来源需求、虚拟存在、场景、目标状态、创建 / 完成状态、实际结果和选择方法。
5. 方法服务可在给定方法首时读取虚拟存在、条件、结果、动作入口、来源任务、父方法、前置方法和后续方法。
6. 节点仓库没有“枚举全部节点”入口，控制面板不得为显示方便新增裸仓库全表扫描。
7. CONCEPT-S0 已确认当前没有概念图服务、三类专用关系、四根权威登记、活动图版本或抽象树只读投影。

## 5. 六类树的正式含义

| 分类 | 真实根 | 允许展开边 | 主要读取方 | 不得扩大声明 |
| --- | --- | --- | --- | --- |
| 世界树 | 世界树初始化结果中的世界根 | 普通父子关系 | 既有世界树 DTO 生成路径 | 不证明完整世界模型 |
| 概念树 | 概念图服务登记的存在、动态、关系、因果四根 | 活动版本内 `概念上下位` | 概念图服务 | 不把树投影视为单父权威结构 |
| 语素树 | 语素类型入口及正式初始化语素入口组 | 语素分类、对应信息、概念追溯的只读投影 | 语素服务及既有语素 DTO | 不把词面或显示文本当概念身份 |
| 需求树 | 安全根需求、服务根需求及后续正式根需求组 | 需求自身的目标、主体、场景、结算材料；承接任务仅作关系分支 | 需求服务、任务服务 | 不证明父子需求写入已实现 |
| 任务树 | 从根需求反向读取到的正式承接任务 | 来源需求、虚拟存在、场景、状态、实际结果、选择方法 | 任务服务 | 不用窗口状态裁决任务完成 |
| 方法树 | 由正式任务选择关系读取的方法首 | 条件、结果、动作入口、来源任务、父 / 前置 / 后续方法 | 方法服务 | 不执行方法，不把按钮当动作来源 |

需求、任务、方法树允许包含不同节点类型的关系叶项，因为它们展示的是该领域对象的真实结构投影。关系叶项必须携带目标节点稳定句柄和关系角色，不能只显示一段说明文字。

## 7. 只读 DTO 设计

### 7.1 节点项

`控制面板树节点材料` 继续作为会话期值式 DTO，但身份有效性从“显示名称和四个语素字段均非空”改为结构优先：

```text
节点句柄：必填，真实领域节点身份。
节点类型：必填，必须与节点仓库读回一致。
关系角色：必填，说明该项在当前投影路径中的语义。
名称语素入口组：可空，只作可读名称来源。
显示名称：可空或由显示层回退生成，不裁决身份。
重复投影：只说明同一节点在另一条有效路径已出现。
子节点组：当前路径下的值式子项。
```

不得把 `显示名称`、分类标题或关系角色字符串作为节点查重键。节点同一性只按完整节点句柄判断；投影项同一性按“分类 + 路径 + 节点句柄 + 关系角色”判断。

### 7.2 树结构材料

`控制面板树结构材料` 调整为：

```text
分类
状态
只读快照标志
来源版本或来源快照序号
真实根节点组
真实节点投影数量
```

一个分类可有多个真实根：

```text
概念树固定四个真实根。
需求树至少包含安全根和服务根。
任务树和方法树可包含多个由正式关系发现的根。
世界树当前一个根。
语素树当前一个语素类型根。
```

UI 分类标题不放进 `真实根节点组`，也不增加伪造节点句柄。

### 7.3 六树候选与窗口快照

新增逻辑结构：

```text
控制面板六类树候选材料
  固定六项树结构材料
  候选序号
  是否完整
  拒绝原因

控制面板窗口快照
  总览材料
  六类树结构材料组
  操作请求材料
  结构统计快照
  数据库审计查询
```

候选只存在于本次刷新调用。只有六项同时通过结构复核，窗口才替换 `六类树结构材料组`；任何单类逻辑内返回或追根因结果都不得留下部分新快照。

## 8. 根发现与读取路径

### 8.1 世界和语素

第一轮复用已经验证的世界树、语素树结构 DTO，不在本切片移动其生成算法。控制面板服务持有值副本，不持有可写仓库引用。

语素树后续展开可调用：

```text
语素服务::读取入口绑定目标组
语素服务::读取入口概念追溯组
```

关系目标只作为叶项或受控子项，不反向修改语素关系。

### 8.2 需求

需求根输入来自 `自我根需求初始化结果` 的安全根需求与服务根需求。控制面板服务逐根调用现有入口：

```text
读取需求树后续结构材料
读取需求承接材料
读取需求目标状态
读取需求主体
读取需求目标宿主
读取需求场景
读取需求当前特征状态材料
读取最近结算记录及结算证据入口
```

父子需求写入当前未实现，因此需求树第一轮不得伪造子需求关系。需求根及其真实目标、承接和结算分支已经构成可读需求结构投影。

### 8.3 任务

PANEL-TREES-S1 允许在任务服务新增只读入口：

```text
任务服务::读取需求承接任务组(需求节点, 状态服务)
```

该入口只使用 CONCEPT-S1 新增的关系来源枚举读取“引用目标为指定需求”的来源任务，并逐个调用现有 `读取任务承接材料` 复核，不返回非任务节点或承接壳不完整任务。

对每个任务继续读取：

```text
来源需求
虚拟存在
运行场景
目标状态
创建状态
完成状态
实际结果状态
选择方法
```

### 8.4 方法

方法根从任务服务正式 `读取任务选择方法` 结果去重得到，不扫描全部方法节点。对每个方法首调用：

```text
读取方法候选材料
读取方法关系材料
读取方法条件节点集合
读取方法结果节点集合
读取方法动作入口
读取动作输入规格与输出规格
```

父、前置、后续方法关系允许形成图。路径中重复遇到同一方法时返回“重复引用叶项”并停止该路径继续展开，不复制方法事实、不修复关系。

### 8.5 概念

概念树唯一入口固定为 CONCEPT-S5 正式产物：

```text
概念图服务::读取活动图版本()
概念图服务::读取全部概念根()
概念图服务::读取抽象树视图(根概念)
```

控制面板服务不得直接读取关系仓库中的 `概念上下位` 关系，也不得自己判断活动版本。四根的视图结果在概念分类下并列显示。

多父概念按每条有效路径重复投影；重复项携带同一概念句柄和重复投影标志。概念树路径出现环、跨根或活动版本外关系属于内部不一致，必须追根因，不得按普通重复引用静默截断后宣称图有效。

## 9. 服务与窗口边界

### 9.1 控制面板服务

控制面板服务负责：

```text
复核只读请求边界。
持有世界 / 语素已验证 DTO 值副本和需求根句柄组。
调用概念、需求、任务、方法、状态、语素服务公开只读入口。
构造六类完整候选快照。
返回结构化状态和拒绝原因。
```

控制面板服务不得：

```text
持有仓库可写引用。
调用任何领域写入口。
直接遍历节点仓库全表。
直接读取概念专用关系裁决活动图。
调用特征值服务。
用 SQL、日志、文本或显示标题补齐树节点。
```

### 9.2 显示服务

显示服务继续只做通用只读来源、写事实和文本裁决门禁。六树结构的领域读取归控制面板服务编排，不要求显示服务反向依赖全部领域服务。

### 9.3 控制面板窗口

窗口只接收控制面板服务和非树审计 / 统计材料。窗口构造函数不再分别接收世界树、语素树 DTO；刷新时统一调用 `生成六类真实树只读材料`。

窗口不得直接持有概念、需求、任务、方法、状态或语素服务引用。这样 UI 消息、按钮和 TreeView 通知不会成为领域调用来源。

## 10. TreeView 行为

### 10.1 同时显示六类

每次成功重建按固定顺序插入六个 UI 分类根：

```text
世界树
概念树
语素树
需求树
任务树
方法树
```

随后把每类 `真实根节点组` 插入对应分类根下。六个分类根默认展开；真实节点沿 DTO 递归展开。现有分类导航保留，点击导航只选择并滚动到对应分类根。

### 10.2 选中和详情

选中 UI 分类根时显示：

```text
分类名称
真实根数量
真实投影项数量
只读来源说明
```

选中真实节点时显示：

```text
完整节点句柄
节点类型
当前投影关系角色
可读名称组
是否重复投影
当前路径子项数量
```

分类说明和显示名称只做人读。详情面板不得生成需求满足、任务完成、方法成功或概念等价结论。

### 10.3 无名节点

有正式语素名称时显示名称。没有名称时显示：

```text
<节点类型> [仓库编号:节点编号:版本号]
```

该字符串不写入语素、节点、主信息、关系或索引，不作为后续查重、身份或命名完成证据。

## 11. 预算与确定性

第一轮固定显示预算：

```text
单类最大投影项：4096。
六类合计最大投影项：16384。
单路径最大深度：128。
```

排序规则：

```text
UI 分类顺序固定。
同一根组按完整节点句柄升序。
关系角色先按固定枚举顺序，再按目标完整句柄升序。
概念多父路径按上位路径完整句柄序列排序。
```

预算耗尽属于逻辑内返回：候选整体不发布，返回结构化超预算状态。不得显示截断结果并声称“所有树已显示”。后续若规模增长，需要独立懒加载 / 分页设计，不能静默改变本计划语义。

## 12. 快照发布与刷新

```text
读取当前有效窗口快照
-> 构造六类候选
-> 六类逐项结构复核
-> 完整性和预算复核
-> 保存上一份有效快照
-> 替换会话期六树快照
-> 重建 TreeView
-> 读回六分类及节点引用
```

若 DTO 已完整但 TreeView 插入失败，窗口尝试用上一份有效 DTO 恢复视图。恢复也失败时关闭窗口并报告追根因错误，不保持一个空白 TreeView 冒充成功。

快照发布只改变窗口会话材料，不是运行期业务事务；不得复用 `写入事务` 或修改仓库版本。

| PANEL-TREES-R02 | CONCEPT-S5 活动图未就绪 | 逻辑内返回 | 六树候选不发布，不显示空概念根 |

| PANEL-TREES-R04 | 合法类别当前没有任何真实根 | 逻辑内返回 | 该次完整六树候选不发布 |
| PANEL-TREES-R05 | 节点数或深度预算耗尽 | 逻辑内返回 | 不发布截断树 |
| PANEL-TREES-R06 | 无名概念或其他无名节点 | 正常成功 | 使用类型 + 句柄做人读回退 |
| PANEL-TREES-R07 | 方法等非概念关系路径重复 | 正常成功 | 生成重复引用叶项并停止该路径 |
| PANEL-TREES-R08 | 概念活动图出现环、跨根或版本外边 | 追根因解决 | 停止刷新，不发布候选 |
| PANEL-TREES-R09 | 有效根通过后领域读回自相矛盾 | 追根因解决 | 停止刷新并追查领域结构根因 |
| PANEL-TREES-R10 | 六类完整复核后 DTO 数量或分类不一致 | 追根因解决 | 不替换窗口快照 |
| PANEL-TREES-R11 | TreeView 插入或引用索引建立失败 | 追根因解决 | 恢复旧视图；恢复失败关闭窗口 |
| PANEL-TREES-R12 | 显示前后权威结构变化 | 追根因解决 | 判定显示路径越权并停止依赖链 |
