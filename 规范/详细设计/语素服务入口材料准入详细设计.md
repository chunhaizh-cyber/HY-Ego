## 1. 定位

本详细设计承接第 2 项“外部材料 / 语素请求准入”代码逻辑流程图及 2.1 到 2.5 子流程图。

本设计只定义语素服务如何把外部材料降权为语素请求、如何在入口排除非法材料、如何写入语素入口关系，以及如何向后续逻辑提供只读入口材料。

本设计不证明当前 C++ 已按本设计全部实现，不形成施工计划，不允许直接进入代码实施。

## 3. 设计对象

语素服务入口材料分为三类请求：

```text
只绑定已有信息节点。
创建概念入口。
追加已有语素入口关系 / 读取绑定组。
```

外部材料只能作为请求材料，不直接成为世界事实。

语素服务输出给其他逻辑的材料只能是：

```text
语素入口句柄。
语素对应信息关系和绑定目标组。
语素概念追溯关系和概念追溯组。
可选主键索引读回材料。
入口拒绝原因和来源追溯材料。
```

显示标题、语言命名、SQL 投影、控制面板材料和日志不参与语素入口机器事实写入。语言命名结果只能作为人读候选，不自动回流为语素文本。

## 4. 结构承载

| 材料 | 结构承载 | 写入方 | 读取方 | 权威性 |
| --- | --- | --- | --- | --- |
| 语素入口身份 | `节点类型::语素` 节点 + 主信息锚点 | 语素服务 | 后续领域服务 / 显示层只读 | 机器事实 |
| 语素对应信息 | `关系类型::语素对应信息` | 语素服务 | 后续领域服务 / 只读显示 | 机器事实 |
| 语素概念追溯 | `关系类型::语素概念追溯` | 语素服务 | 后续领域服务 / 只读显示 | 证据材料 |
| 可选主键 | 索引仓库主键到语素节点绑定 | 语素服务经索引仓库 | 调用方 / 只读验证 | 入口索引 |
| 文本准入状态 | `语素准入状态` 返回值 | 语素服务 | 调用方 | 拒绝材料 |
| 绑定目标组 | 关系仓库只读查询结果 | 语素服务 | 调用方 / 显示层只读 | 只读材料 |

概念追溯只表示追溯材料，不等于稳定概念裁决，不等于自然语言理解完成。

## 5. 服务入口矩阵

| 入口 | 类型 | 输入 | 输出 | 写结构 | 入口拒绝 |
| --- | --- | --- | --- | --- | --- |
| `判断文本是否可作为最小词单元` | 只读准入 | 文本 | `语素准入状态` | 否 | 空文本、空白、组合分隔符、长度超过 3 |
| `对应信息节点准入` | 只读准入 / 目标辅助 | 节点句柄 | 通过 / 拒绝 | 否 | 无效句柄、不允许类型、运行期实例状态 / 动态、特征值 |
| `概念追溯目标准入` | 只读准入 / 目标辅助 | 节点句柄 | 通过 / 拒绝 | 否 | 无效句柄、需求 / 任务 / 方法默认拒绝、存在 / 场景默认拒绝、特征值、实例状态 / 动态 |
| `创建语素入口` | 写入口 | 对应信息节点、可选主键 | 语素节点或无效句柄 | 主信息、语素节点、对应信息关系、可选索引 | 对应信息不准入、任一写入步骤失败、主键冲突 |
| `创建概念入口` | 组合写入口 | 文本、对应信息节点、概念追溯目标、可选主键 | 语素节点或无效句柄 | 创建语素入口并追加概念追溯 | 文本不准入、对应信息不准入、概念追溯目标不准入、创建入口失败 |
| `追加语素对应信息` | 写入口 | 语素入口、对应信息节点 | 成功 / 失败 | 对应信息关系 | 入口不是语素、目标不准入、关系已存在、写入不及预期 |
| `追加语素概念追溯` | 写入口 | 语素入口、概念追溯目标 | 成功 / 失败 | 概念追溯关系 | 入口不是语素、目标不准入、关系已存在、写入不及预期 |
| `读取入口绑定目标组` | 只读入口 | 语素入口 | 节点句柄组 | 否 | 入口不是语素则返回空集合 |
| `读取入口概念追溯组` | 只读入口 | 语素入口 | 节点句柄组 | 否 | 入口不是语素则返回空集合 |
| `读取入口绑定目标` / `读取入口概念追溯` | 兼容只读入口 | 语素入口 | 首个节点或空 | 否 | 入口不是语素或目标组为空 |

第一轮保留单目标读取函数只能作为兼容读，不得重新把语素对应信息或概念追溯解释成单值字段。

## 6. 文本准入设计

文本准入只判断文本能否作为第一轮最小词单元，不创建任何结构。

拒绝条件：

```text
空文本。
任一字符为空白。
任一字符为组合分隔符。
有效字符数大于 3。
```

第一轮组合分隔符集合以当前 `语素服务::字符是组合分隔符` 代码事实为准：

```text
/ \ - _ + , . ; : ， 。 ； ： 、
```

文本准入失败只返回拒绝状态，用于追溯非法材料来源。不得在语素服务内部裁剪、拆词、替换或兜底修复文本。

## 7. 2.2a 对应信息节点准入

对应信息节点用于 `语素对应信息` 关系。

允许范围：

```text
基础信息
存在
场景
特征
抽象状态
抽象动态
二次特征
因果引用
需求
任务
方法
```

状态和动态节点必须额外排除运行期临时目标关系。存在运行期临时关系的状态 / 动态视为实例状态 / 实例动态，不得作为语素入口对应信息。

必须拒绝：

```text
无效句柄。
特征值节点。
运行期实例状态。
运行期实例动态。
其他未确认节点类型。
```

对应信息节点必须是已有节点。语素服务不得为了绑定而自动创建基础信息、需求、任务或方法。

## 8. 2.2b 概念追溯目标准入

概念追溯目标用于 `语素概念追溯` 关系，第一轮默认比对应信息节点准入更窄。

允许范围：

```text
基础信息。
特征。
抽象状态。
抽象动态。
二次特征。
因果引用。
```

默认拒绝：

```text
需求。
任务。
方法。
存在。
场景。
特征值。
运行期实例状态。
运行期实例动态。
其他未确认节点类型。
```

如果后续确需需求、任务或方法作为高级语义追溯目标，必须另行确认详细设计，并明确：

```text
高级语义追溯目标的业务含义。
写入方和读取方。
拒绝路径。
不得新增“高级信息”节点类型。
不得绕过需求、任务或方法服务的领域边界。
```

## 9. 语素入口创建

`创建语素入口` 的目标写入顺序为：

```text
对应信息节点准入。
主信息仓库创建主信息。
节点仓库创建语素节点。
关系仓库创建语素对应信息关系。
可选主键索引绑定。
读回或最小验证。
返回语素入口句柄。
```

任一写入步骤失败，必须进入追根因解决收口：

```text
不返回有效语素入口。
不允许留下可被读取为有效业务事实的半结构。
已创建结构必须回滚、失效或被读取隔离。
必要时登记逻辑错误或待确认问题。
不得依赖日志、显示、扫描器或后续校验修复半结构。
```

当前索引仓库事实为：主键为 `0` 时拒绝绑定；主键已存在且指向同一节点时返回成功；主键已存在但指向其他节点时返回失败。因此新建语素入口绑定非零主键时，应把“主键已被其他节点占用”视为冲突，创建流程不得返回有效入口。

## 10. 概念入口组合流程

`创建概念入口` 是组合流程，不是“先由总览图调用 2.3 再追加 2.4”的拆分流程。

组合顺序：

```text
文本准入。
对应信息节点准入。
概念追溯目标准入。
创建语素入口。
追加语素概念追溯关系。
读回概念追溯组。
返回语素入口句柄。
```

任一准入失败或写入不及预期，必须返回空句柄并保持结构不变化。

概念追溯关系已存在时，不重复写入；创建概念入口场景中如果追溯关系已经存在，需要确认是否属于重复请求或内部逻辑异常，不能静默扩大为新事实。

## 11. 追加关系与读回验证

追加关系入口必须先验证语素入口：

```text
输入入口句柄必须存在。
入口节点类型必须是语素。
非语素入口返回失败或空集合。
```

关系类型白名单只允许：

```text
语素对应信息。
语素概念追溯。
```

按关系类型选择目标准入：

```text
语素对应信息 -> 2.2a 对应信息节点准入。
语素概念追溯 -> 2.2b 概念追溯目标准入。
```

重复关系处理：

```text
已有相同语素入口、关系类型和目标节点时，返回失败或重复拒绝。
重复拒绝不得写新关系。
重复请求不得被当作新事实。
```

写入后必须读回：

```text
追加对应信息后，读取入口绑定目标组必须包含目标节点。
追加概念追溯后，读取入口概念追溯组必须包含目标节点。
```

读回不符合预期时，视为逻辑错误：

```text
停止后续详细设计推进到施工计划。
登记断点问题。
追溯关系仓库、调用参数和入口句柄。
不得生成施工计划或代码实施候选。
```

## 13. 锁边界和公共函数边界

语素服务应顺序调用仓库公开入口，不在服务层同时持有多个仓库锁。

当前写入顺序涉及：

```text
主信息仓库
节点仓库
关系仓库
索引仓库
```

若后续为了事务化或追根因解决收口需要多仓库同时持锁，必须另建详细设计并遵守节点仓库 -> 主信息仓库 -> 关系仓库 -> 索引仓库的锁顺序规则；本详细设计不授权多仓库同时持锁。

本设计不新增公共函数工具层。若施工切片需要新增准入辅助函数，应优先作为 `语素服务` 内部辅助或对应仓库入口，不得作为全局公用函数迁移。

## 中途非成功返回二分口径

本文件按 2026-07-09 硬规则修订：中途非成功返回只分为 `追根因解决` 和 `逻辑内返回`。

- `追根因解决`：前置条件已经满足，并进入创建、绑定、写关系、写状态、记录动态、结算、读回或结构承载后，结果不符合内部预期；必须停止依赖路径，定位根因，当前未证明完整回滚时登记事务隔离缺口或半结构隔离缺口。
- `逻辑内返回`：领域协议允许的拒绝、候选为空、请求材料返回或人读材料返回；必须保证结构不变化，且返回材料、日志、回执、显示或控制台输出不裁决机器事实。
