状态：设计候选已形成；采用隔离节点直接新结构域；关系 19/20 规范前置已闭合；NT-P1 与 P2A 条件已冻结为待实现接口合同；代码许可读取叶子计划和任务状态台账

## 1. 目标

把状态和动态实例事实从当前的“节点绑定主信息、无类型 I64 槽和通用关系拼装”，迁移为：

```text
完整状态事实
= 状态节点
+ 关系 19：主体、场景、特征、值、来源存在
+ 状态域类型化记录

完整动态事实
= 动态节点
+ 关系 20：主体、场景、被改变目标、前状态、后状态、来源存在
+ 按动态种类成立的来源动作、来源低层动态、同源状态迁移动能
+ 动态域类型化记录
```

发生时间、接收时间、记录模式、动态种类、聚合规则版本等非拓扑值只进入所属领域类型化记录；全部对象联系只进入正式关系。场景、主体、状态和动态列表只由关系与索引形成可重建读取投影。

### 3.3 当前接口缺口

## 4. 正式结构设计

### 4.1 状态域类型化记录

建议新增值式类型 `状态域记录`，最小字段为：

| 字段 | 类型语义 | 约束 |
| --- | --- | --- |
| `所属状态` | 完整状态节点句柄 | 必须为本事务状态候选或当前有效状态 |
| `格式版本` | 无符号版本 | 初版为 1，未知版本拒绝 |
| `记录版本` | 无符号版本 | 初始为 1，零值拒绝，不静默覆写 |
| `记录状态` | 4010 记录状态 ABI | 新事实为有效 |
| `发生时间` | 无符号时间 | 非零；是本状态事实发生时间 |
| `接收时间` | 可选无符号时间 | 存在时不早于发生时间；缺失不以 0 冒充 |
| `记录模式` | 状态域强类型枚举 | 区分观察、动作结果、治理事实等已登记模式；未知值拒绝 |

记录不得保存主体、场景、特征、值、来源存在的句柄副本，不得保存主信息句柄、拆分主键、通用槽位、显示名或日志文本。

### 4.2 动态域类型化记录

建议新增值式类型 `动态域记录`，最小字段为：

| 字段 | 类型语义 | 约束 |
| --- | --- | --- |
| `所属动态` | 完整动态节点句柄 | 必须为本事务动态候选或当前有效动态 |
| `格式版本` | 无符号版本 | 初版为 1，未知版本拒绝 |
| `记录版本` | 无符号版本 | 初始为 1，零值拒绝 |
| `记录状态` | 4010 记录状态 ABI | 新事实为有效 |
| `发生时间` | 无符号时间 | 与后状态发生时间一致，不复制前后状态端点 |
| `动态种类` | 动态域强类型枚举 | 至少区分状态迁移动能、动作致变、观察/外部、聚合 |
| `聚合规则版本` | 无符号版本 | 原子动态为 0；聚合动态非零 |

记录不得保存主体、场景、目标、前后状态、来源动作、来源低层动态、同源迁移动能或来源存在句柄副本。

### 4.3 记录身份与存储

记录身份固定为：

```text
所属节点完整句柄 + 领域记录类型 + 记录版本
```

不新增跨领域通用记录编号或句柄。状态记录和动态记录可以由同一物理模块管理，但必须保持两个强类型集合、两个读取结果和各自字段校验，不得用变体字节包或整数槽把它们合并成通用语义。

建议结构：

- `状态域记录存储`：按完整状态句柄定位当前及历史记录版本；
- `动态域记录存储`：按完整动态句柄定位当前及历史记录版本；
- `状态动态类型化记录事务参与者`：只在结构写入会话内登记本次值式记录，完成准备、确认、发布和撤销；
- `状态动态类型化记录只读访问器`：在共享读边界返回不可变副本，不暴露锁或容器。

### 4.4 关系 19 状态事实角色

| 顺序号 | 角色 | 方向与端点 | 基数 | 验证 |
| --- | --- | --- | --- | --- |
| 1 | 主体 | 状态 -> 存在 | 恰一 | 主体当前有效 |
| 2 | 场景 | 状态 -> 场景 | 恰一 | 场景当前有效且与主体语境相容 |
| 3 | 特征 | 状态 -> 特征 | 恰一 | 特征当前有效 |
| 4 | 值 | 状态 -> 特征值 | 恰一 | 值当前有效且与角色 3 语义相容 |
| 5 | 来源存在 | 状态 -> 存在 | 恰一 | 产生、提交或观察材料的稳定存在 |

唯一键为“关系类型 19 + 完整状态源句柄 + 顺序号”。主体和来源存在可以合法指向同一存在，但仍是两条不同角色关系。

### 4.5 关系 20 动态事实角色

| 顺序号 | 角色 | 方向与端点 | 基数与组合 |
| --- | --- | --- | --- |
| 1 | 主体 | 动态 -> 存在 | 恰一 |
| 2 | 场景 | 动态 -> 场景 | 恰一 |
| 3 | 被改变目标 | 动态 -> 特征或特征值 | 恰一 |
| 4 | 前状态 | 动态 -> 状态 | 恰一完整状态 |
| 5 | 后状态 | 动态 -> 状态 | 恰一完整状态 |
| 6 | 来源动作 | 动态 -> 方法 | 动作致变恰一；普通迁移、观察、外部为零 |
| 7 | 来源低层动态 | 动态 -> 动态 | 原子为零；聚合至少一项且可多项 |
| 8 | 同源状态迁移动能 | 动态 -> 动态 | 动作致变恰一；其它为零 |
| 9 | 来源存在 | 动态 -> 存在 | 恰一 |

前后状态必须由关系 19 读回为同主体、同场景、同特征且时间有序。角色 3 必须与两端状态的特征或值语义一致。动作致变动态的角色 8 必须指向同主体、同场景、同端点的状态迁移动能。

## 5. 输入契约与调用语境

### 5.1 创建完整实例状态请求

新请求必须显式包含：

```text
状态稳定主键
主体
场景
特征
特征值
来源存在
发生时间
可选接收时间
记录模式
```

状态业务服务负责：

1. 拒绝零稳定主键、无效/旧版本句柄、时间越界和未知记录模式；
2. 通过节点身份只读合同复核主体、场景、特征、特征值、来源存在类型；
3. 通过特征业务服务的值式只读合同复核特征值属于目标特征且当前可用；
4. 形成不可变 `完整实例状态写入规格`，不把仓库、记录参与者或会话能力交给调用方。

### 5.2 创建完整动态请求

新请求必须显式包含：

```text
动态稳定主键
动态种类
主体、场景、被改变目标
前状态、后状态
来源存在
来源动作（按动态种类）
来源低层动态组（按动态种类）
同源状态迁移动能（按动态种类）
聚合规则版本
```

动态业务服务只能经状态业务服务读取完整前后状态投影。不得由动态数据操作用裸关系拼装并绕过状态领域解释。

### 5.3 特征值归属只读依赖

P2B 不得读取 P2A 的领域记录容器。代码派发前，P2A 或唯一共享接线切片必须冻结一个由特征业务服务拥有的值式只读合同，至少能机械返回：

```text
目标特征当前有效
目标特征值当前有效
特征值与目标特征的正式归属/值域关系成立
原始值版本当前（只作复核，不把原始值复制进状态）
```

该接口是 P2A -> P2B 的真实窄依赖；未发布时只门控 P2B，不冻结 P2C 或其它无关设计。

## 6. 写入过程

### 6.1 状态事实

```text
领域服务形成完整规格
-> 数据操作按稳定主键预读
-> 同键同义：幂等读回
-> 同键异义：写前冲突
-> 未找到：进入结构写入会话
-> 创建带直接稳定主键的状态节点候选
-> 登记状态域记录参与者
-> 创建关系 19 的角色 1—5
-> 绑定状态命名域索引候选
-> 同会话读回节点、记录、五角色和正反索引
-> 全部参与者确认待发布
-> 最后无失败发布并释放唯一写入权
-> 事务外组合读回完整状态投影
```

同义幂等比较必须覆盖节点稳定主键、关系 19 全部角色端点、状态记录模式和全部时间字段；不得只比较 I64 值或索引命中。

### 6.2 动态事实

```text
领域服务读取完整前后状态并形成动态规格
-> 数据操作按稳定主键预读
-> 同键同义或同键异义收口
-> 进入结构写入会话
-> 再读前后状态当前性及关系 19 完整性
-> 创建带直接稳定主键的动态节点候选
-> 登记动态域记录参与者
-> 创建关系 20 的必需角色和动态种类角色
-> 绑定动态命名域索引候选
-> 同会话读回记录、关系组合、端点和索引
-> 全部参与者确认并最后发布
-> 事务外组合读回动态及前后状态投影
```

### 6.3 状态到动态

新后状态发布后，按主体、特征、场景和时间查找相邻前状态。没有相邻前状态时返回具名结果“基准状态已发布、未形成动态”；该分支已经形成完整状态，不得返回裸失败。找到前状态后，先形成状态迁移动能；若同时存在合法方法动作，再另形成动作致变动态并通过关系 20 角色 8 引用同源迁移动能。

## 7. 读取投影

### 7.1 完整状态投影

```text
索引缩小状态候选
-> 回读节点直接稳定主键、类型、版本和记录状态
-> 读取关系 19，复核角色 1—5 各恰一
-> 读取状态域记录，复核所属节点、格式/记录版本和时间值域
-> 经节点直接特征服务复核角色 3/4 相容
-> 组合不可变状态值式材料
```

任何必需关系、记录或版本缺失都不得返回部分成功。索引未命中且权威结构无此身份可以返回未找到；索引命中悬空、角色缺失或记录不一致属于内部逻辑错误。

### 7.2 完整动态投影

```text
索引缩小动态候选
-> 回读动态节点
-> 读取关系 20 的角色组合
-> 经节点直接状态服务读取角色 4/5 的完整状态投影
-> 读取动态域记录并复核动态种类与角色 6—8 基数
-> 复核来源存在、目标语义、时间和同源证据
-> 组合不可变动态值式材料
```

### 7.3 列表和当前性

- 主体状态列表：按关系 19 角色 1 反向索引候选，再完整读回；
- 场景状态列表：按关系 19 角色 2 反向索引候选；
- 主体动态列表：按关系 20 角色 1 反向索引候选；
- 场景动态列表：按关系 20 角色 2 反向索引候选；
- 当前状态：在完整投影上按主体、特征、场景、时间、来源和版本选择。

这些列表和排序结果均为可丢弃投影；不得落为自我、存在或场景内部权威数组。

为供 P2C 形成自我读取投影，P2B 至少提供以下两个值式读取候选：

```text
状态业务服务::读取主体状态事实组(节点句柄 主体)
动态业务服务::读取主体动态事实组(节点句柄 主体)
```

两个入口内部只能分别使用关系 19 / 20 的主体角色反向索引缩小候选，再逐项执行完整事实读回；输出为本次调用期的不可变值式结果组，不返回持久列表、关系仓库、索引仓库、读取许可或事务能力。索引清空或重建不得改变返回语义。

## 8. 非成功返回与失败收口

### 8.1 逻辑内返回

以下分支必须强类型返回并保持相应结构边界：

| 分支 | 时点 | 结构变化 |
| --- | --- | --- |
| 无效身份、错类型、错版本、错时间、错来源、错角色组合 | 第一笔写入前 | 零变化 |
| 特征和值不相容 | 第一笔写入前 | 零变化 |
| 同键同义 | 写前权威读回 | 零变化，返回原身份 |
| 同键异义 | 写前权威读回 | 零变化，返回幂等冲突 |
| 二次复核发生规范允许的当前性漂移/许可竞争 | 本步骤第一笔写入前 | 零变化，返回具名结果 |
| 基准状态无相邻前状态 | 状态已经完整发布后 | 返回已发布状态身份，不创建动态 |

### 8.2 追根因解决

前置通过后出现节点、关系、领域记录、索引、参与者准备/确认、逆序撤销、最后发布或事务外完整读回异常，均为内部逻辑错误。处理固定为：

```text
停止新增写入
-> 统一可见点前按精确依赖逆序撤销
-> 读回证明事务前态
-> 无法证明则在释放唯一写入权前隔离事务域
-> 已发布异常保留证据，停止依赖路径
```

不得把内部缺关系、缺记录或端点漂移包装成候选为空、等待、普通需求、学习缺口或幂等成功。

## 9. 生命周期与版本

1. 状态和原子动态是历史事实，发布后不原地改写端点或时间；修正形成新事实节点。
2. 类型化记录初始版本为 1；若规范允许记录换代，必须生成新记录版本并使旧版本退出当前读取，历史仍可审计。
3. 关系失效按 4010 推进关系版本，旧关系句柄退出普通读取；不得原地恢复。
4. 索引清空、缓存失效、列表裁剪只影响定位和投影，不删除事实。
5. P4 快照必须保存节点、关系 19/20、领域记录及历史/失效版本；本叶子不实现恢复。

## 10. 分阶段兼容策略

P2B 采用“隔离新域入口新增、旧默认域全程只读、P3 建立新生产候选调用图、P4 一次切换并物理退役”的阶段策略：

1. P2B 新域入口只产生新结构；不同时写主信息槽和类型化记录，不被旧默认装配持有。
2. 旧 `创建实例状态请求`、旧 I64 状态/动态入口及现行 `服务.状态/动态`、`数据操作.状态动态` 在 P2B 中只读；其产物不得被新完整事实读取器接纳。
3. 新旧身份必须用节点直接稳定主键、记录存在性和关系 19/20 明确区分；禁止用“先读新、缺失再把旧当新”的静默回退。
4. P3 只在隔离新域形成逐调用点的新生产候选适配，不切换旧默认入口；P4 才统一修改默认装配、现行调用方和兼容头。
5. P3 必须为每个旧调用语境显式提供特征、特征值、来源存在和动态种类；无法推出时退回业务设计，不得编造默认节点。
