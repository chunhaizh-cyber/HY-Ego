## 3. 当前事实

| 事实项 | 当前承载 | 本设计使用方式 |
| --- | --- | --- |
| 动作入口角色 | `方法服务::登记动作入口`、`读取方法动作入口`、`按稳定动作键读取方法动作入口` | 作为动作执行桥请求材料的结构身份，不代表真实执行已经发生 |
| 本能动作注册材料 | `方法服务::本能动作注册材料是否有效` | 只校验动作入口、主动可调用状态、稳定动作键和输入 / 输出规格材料 |
| 动作动态证据 | `方法服务::记录动作动态证据`、`动态服务::记录实例动态` | 只在有效方法首和动作入口成立后记录证据，不写任务完成 |
| 轻量因果准入 | `因果服务` 现有来源动态准入口径 | 只把来源动态作为准入证据，不持久化来源动态关系，不生成稳定因果结论 |

## 4. 结构边界

| 概念 | 结构承载 | 写入方 | 读取方 | 生命周期 | 验证方式 |
| --- | --- | --- | --- | --- | --- |
| 动作执行桥请求材料 | 任务节点、方法节点、动作入口角色节点、稳定动作键、输入 / 输出规格场景 | 后续任务服务和方法服务读取组合，不新增事实字段 | 方法服务、后续执行桥候选 | 随任务选择方法和动作入口生命周期 | 缺任务方法关系、缺主动动作入口或规格缺失时拒绝 |
| 本能动作注册材料 | 动作入口角色状态、主动可调用状态、稳定动作键索引、规格场景关系 | 方法服务 | 方法服务 | 随动作入口生命周期 | `本能动作注册材料是否有效` 返回有效且读回规格 |
| 动作动态证据 | 实例动态节点、来源动作入口、来源方法关系候选 | 方法服务经动态服务 | 动态服务、因果服务、任务结果后续候选 | 单次动作证据生命周期 | 动态材料完整，来源动作有效，来源动态准入成立 |
| 拒绝与失败边界 | 局部返回结果和结构不变化 | 各领域入口 | 人读验收与后续调用方 | 单次调用 | 拒绝后不得写半结构；内部不及预期按逻辑错误追根因 |

## 5. 请求分流

```text
动作执行桥请求
-> 读取任务选择方法材料
-> 读取方法主动动作入口
-> 读取稳定动作键对应动作入口
-> 读取输入 / 输出规格
-> 形成候选请求材料
-> 不分派线程，不调用外部执行器，不写任务结果。

本能动作注册材料治理
-> 复核动作入口角色有效
-> 复核主动可调用状态
-> 复核稳定动作键命中同一动作入口
-> 复核输入 / 输出规格均可读
-> 只返回候选材料有效性，不注册函数指针。

动作动态证据联动
-> 真实方法写入路径或后续确认的被动提交路径成立
-> 复核来源方法和动作入口关系
-> 经动态服务记录实例动态
-> 因果服务只接受来源动态准入
-> 不写任务完成、需求满足或稳定因果结论。
```

## 6. 入口前置拒绝

后续代码实施切片必须先拒绝：

```text
无效任务节点。
缺少任务承接壳或任务选择方法关系。
无效方法首。
方法首无主动动作入口。
动作入口角色无效或已失效。
稳定动作键为 0、未命中或命中非本方法动作入口。
输入规格或输出规格缺失。
被动状态提交入口被普通执行桥请求当成主动动作。
被动整理入口被当成本能动作。
来源动作不是动作入口角色节点。
动态缺场景、存在主体、发生时间戳、被改变目标、前值或后值。
函数名、文本、日志标签、控制台命令、线程或外部输入被当作动作来源。
```

拒绝后不得写动作入口关系、稳定动作键索引、规格关系、特征状态材料、实例动态、来源方法关系、因果引用、任务结果或需求结算。入口拒绝已排除错误输入后，内部写入结果仍不符合预期时，必须作为逻辑错误追根因。

## 7. 服务边界

| 服务 | 允许 | 禁止 |
| --- | --- | --- |
| 任务服务 | 只提供任务授权、任务承接和任务选择方法材料 | 不分派线程，不直接写动作动态，不写需求结算 |
| 方法服务 | 管理动作入口、规格材料、本能动作注册材料和动作动态证据入口 | 不直接依赖特征值服务，不注册函数指针，不裁决任务完成 |
| 动态服务 | 记录和读取实例动态材料 | 不替代任务结果，不生成稳定因果 |
| 因果服务 | 接受来源动态准入证据 | 不持久化来源动态关系，不把单条因果引用解释为结论 |
| 日志 / 控制台 / 显示层 | 只做人读诊断或观察 | 不承载机器逻辑，不证明执行桥完成 |
