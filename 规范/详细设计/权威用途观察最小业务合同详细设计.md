## 5. 输入合同

### 5.1 正式入口候选

O1-D1 只接受 `用途事件结果` 作为未来纯值形成入口候选。当前 `用途事件规则版本 = 1` 必须先按以下唯一状态—具体原因—`事件` optional 矩阵分流：

| 状态 | 当前允许的具体拒绝原因 | `事件` | O1 收口 |
| --- | --- | --- | --- |
| 已形成 | 无 | 必须有 | 唯一可进入身份门禁的成功组合 |
| 入口拒绝 | 请求不完整、来源冻结不完整、结果尚未正式收口、非新版完整回执、来源冻结回合不一致 | 必须无 | 逻辑内入口拒绝，零结构变化 |
| 不支持 | 规则版本不支持 | 必须无 | 逻辑内不支持，不猜测兼容 |
| 内部不一致 | 正式结果材料矛盾、身份材料矛盾、算法不变量破坏 | 必须无 | 继续追根因并停止 |

`请求不完整` 是 `用途事件结果` 的安全默认拒绝态；`因果候选不完整` 只属于上游 `用途来源冻结结果` 当前入口，不是当前正式 `用途事件结果` 的合法原因。任何未列出的状态—原因配对、`已形成 + 无事件`、任一非成功状态携带事件，均为非法结果组合，属于内部不一致并追根因。成功事件一旦被接受，后续句柄、序号、来源、内容、阶段或投影矛盾不得再降格为普通入口拒绝。

### 5.2 输入字段分流

| 输入 | 进入候选合同 | 只做准入复核 | 明确排除 |
| --- | --- | --- | --- |
| `用途来源冻结材料` | 不直接整体进入 | 完整任务请求、冻结任务序号、因果候选、规则版本 | 冻结键、冻结时间戳、完整嵌套请求 |
| `规范化任务结果来源` | 正式结果分类；最终阶段的正式结算状态句柄 | 完整回执、目标裁决、任务提交、需求结算 | 完整回执、完整结算 DTO、待结算提示、请求参数 |
| `用途事件材料` | 九个固定来源 / 结果句柄；分栏效用 | 目标值 / 实际值与状态引用的一致性 | 两个值的第二权威副本 |
| `结构化结果效用` | 三个分栏结果、观察阶段、历史概率快照 | 历史概率身份和四项计数与冻结候选一致 | 完整机会证据组 |
| 可选既有纯值候选 | 完整值用于比较 | 只接受支持版本 | 哈希命中、索引命中或文本摘要 |

## 6. 语义身份

### 6.1 业务身份

未来用途观察的业务语义身份固定为：

```text
完整任务句柄 + 任务序号
```

完整任务句柄必须保留 `仓库编号 + 节点编号 + 版本号`；任务序号必须非零。未来纯值表示可以有独立格式版本，但格式版本只裁决解释兼容性，不改变业务身份字段。

### 6.2 事件、冻结请求、回执内嵌请求与回执互证

形成前必须逐项证明：

```text
事件.任务
= 冻结来源.任务执行请求.调度请求.任务
= 冻结来源.任务执行请求.任务材料.身份.节点
= 结果来源.完整回执.来源执行请求.value().调度请求.任务
= 结果来源.完整回执.来源执行请求.value().任务材料.身份.节点
= 结果来源.完整回执.句柄组.目标任务

冻结来源.冻结任务序号
= 冻结来源.任务执行请求.调度请求.任务序号
= 结果来源.完整回执.来源执行请求.value().调度请求.任务序号
= 结果来源.完整回执.来源任务序号
```

G2 还必须证明回执的 `来源执行请求` 存在，并与冻结来源中的完整 `任务执行请求` 逐字段相同；上述等式不能用回执摘要、哈希或单个编号替代。

### 6.3 不属于身份的字段

```text
冻结键与观察主键（旧字段名）
请求编号、幂等键、回执编号、批次编号
全部时间戳
节点编号单字段
因果完整结构键及其哈希
Q10000、机会数、正结果数、负结果数
方法、选择、动作、场景或主体
```

这些字段可以参与来源深检或内容比较，但不得补写、缩写或替代语义身份。

## 7. 不可变核心

O1 最小内容必须拆为“不可变核心 + 阶段负载”。不可变核心只含固定形状字段：

| 字段组 | 字段 | 语义 |
| --- | --- | --- |
| 来源业务引用 | `需求`、`选择记录`、`方法`、`动作`、`场景`、`主体` | 本次任务回合采用的固定来源引用 |
| 结果业务引用 | `目标状态`、`实际状态`、`动态证据` | 目标、实际结果和发生动态的固定引用 |
| 历史概率快照 | `因果完整结构键`、机会数、正结果数、负结果数、Q10000 | 观察形成时看到的非权威因果发生概率快照；不自动随未来统计变化 |

九个节点引用必须是完整有效句柄。历史概率身份必须由冻结来源中的完整 `因果概率候选` 通过 #287 正式入口重新形成，并与事件效用中的单一身份完整比较相同；四项计数必须逐项一致。

不可变核心不保存完整因果机会证据组。后继统计以完整因果键分栏重建，不反向修改本次核心。

## 8. 阶段负载

### 8.1 三种合法元组

| 观察阶段 | 正式结果 | 任务达成 | 需求结算 | 正式结算状态 |
| --- | --- | --- | --- | --- |
| 尝试未达成 | 未达成已转待重筹办 | 未达成 | 未形成 | 必须无 |
| 待结算 | 任务已完成待需求结算 | 已达成 | 待结算 | 必须无 |
| 最终已结算 | 已完成并结算 | 已达成 | 已满足 | 必须有；等于正式需求结算材料的当前结算状态 |

任何其它排列都属于阶段元组非法。最终已结算的正式结算状态必须从 `事件.结果来源.需求结算.材料.正式结算->结算状态.状态节点` 深检得到；`最近结算状态` 只作兼容投影，不得单独代替正式结算材料。待结算提示只负责调用期重试请求，不进入阶段负载，也不成为跨重启事实。

### 8.2 单向推进

同一语义身份只允许：

```text
待结算 -> 最终已结算
```

且必须保持不可变核心完全相同，新增的正式结算状态完整有效并与正式需求结算材料一致。

执行顺序固定为：先验证新候选自身属于三种合法元组；存在既有候选时，再验证既有候选自身合法；两者各自合法后才做身份、核心和阶段比较。已接受成功事件后，新候选最终阶段缺少正式结算材料、正式结算状态无效或材料互相矛盾，属于内部不一致并追根因，不进入同账比较。已发布既有候选自身非法同样追根因。以下禁止转换只在两个候选各自合法后裁决为同账冲突。

以下均禁止：

```text
尝试未达成 -> 任意其它阶段
最终已结算 -> 任意其它阶段
最终已结算 -> 待结算
待结算 -> 尝试未达成
待结算 -> 最终已结算但不可变核心变化
```

正式结算材料缺失、正式结算状态无效或材料不一致不属于“两个合法候选间的不允许转换”；它在候选合法性门禁已经归为内部不一致并追根因，不得再次进入同账冲突裁决。

## 9. 两类强类型结果

### 9.1 纯值形成结果

| 状态 | 原因类型 | 候选 |
| --- | --- | --- |
| 已形成 | 无 | 必须有完整候选 |
| 入口拒绝 | 上游 `用途事件结果` 是合法入口拒绝组合 | 必须无 |
| 不支持 | 上游是合法不支持组合，或目标纯值格式 / 用途 / 动态 / 因果规则版本不兼容 | 必须无 |
| 内部不一致 | 上游报告内部不一致、结果状态组合非法，或已接受成功事件后来源、身份、内容、阶段、投影仍矛盾 | 必须无 |

不得用普通 bool 同时表示入口拒绝、不支持和内部不一致。

### 9.2 同账比较结果

| 结论 | 条件 |
| --- | --- |
| 不同身份 | 完整任务句柄或任务序号不同 |
| 同账幂等 | 身份、不可变核心和阶段负载全部相同 |
| 同账允许推进 | 身份和不可变核心相同，既有为待结算，新值为合法最终已结算 |
| 同账冲突 | 新候选与可选既有候选各自合法后，身份相同但核心不同；或发生终态变化、逆向推进及其它不允许阶段转换 |
| 不支持 | 任一输入格式 / 规则版本不兼容 |
| 内部不一致 | 已发布层同一身份出现多个当前记录，或结构读回无法得到唯一结论 |

哈希只可召回比较候选，不能直接生成上述任何同账结论。

## 12. 容量与编码边界

O1-D1 只固定：

```text
语义身份、不可变核心和阶段负载的候选输出均为固定字段形状。
候选输出不得保留 vector、string、完整冻结请求、完整回执、完整结算 DTO、完整机会证据组或待结算提示；合法来源输入可以携带这些只读复核材料。
完整节点句柄和因果完整结构键是经批准的固定形状值对象，不属于被排除的来源业务 DTO。
任何未知格式或规则版本必须显式不支持。
```

O1-D1 不给出：

```text
规范编码布局和字节序
单记录规范字节上限
仓库总预算
物理槽位数量
恢复段编号、高水位或未知字段跳过规则
```

后继权威结构计划必须先测量并冻结这些数值，且在写入前拒绝超限材料。

## 13. 事务、最后发布与恢复证明义务

未来权威结构计划至少必须证明：

```text
1. 用途专用数据操作隐藏仓库与原始令牌。
2. 同一许可内完成候选、索引绑定、关系组、逐项读回和确认。
3. 同一语义身份零 / 一 / 多及并发单赢家在发布前复核。
4. 最后发布点唯一；读者在发布前看不到半结构。
5. 待结算推进最终不产生两个当前记录，历史仍可审计。
6. 每一种写后失败的撤销能力都有当前接口和故障注入证据。
7. 撤销失败按内部不一致收口，不静默继续。
```

当前没有上述完整证明。因此本设计不得写“任何写后异常都能精确撤销”。发生读回、确认、发布、撤销或恢复不符合合同，必须追根因并停止；只有正式事务合同已经逐项证明时，才能对相应路径声明精确撤销。

未来恢复只能原地进入 #226—#230、#237—#239 既有治理计划链；当前不预设格式名、段号、编码、导入 API、高水位、发布许可或第二恢复链。恢复默认不自动统计、学习、晋级、发布方法或执行动作。

## 14. 非成功返回二分

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 上游结果为 5.1 矩阵中的合法入口拒绝状态—原因—optional 组合 | 逻辑内返回 | 入口拒绝；零结构变化 |
| 上游结果为 5.1 矩阵中的合法不支持组合，或目标候选格式 / 规则版本不兼容 | 逻辑内不支持 | 不猜测、不降级 |
| 上游报告 5.1 矩阵中的内部不一致，或出现未列状态—原因配对 / 非法 optional 组合 | 追根因解决 | 内部不一致；停止后继 |
| 完整身份不同 | 正常分流 | 不同账项 |
| 同身份、同核心、同阶段 | 正常成功 | 同账幂等 |
| 同身份、同核心、待结算合法推进最终 | 正常成功 | 同账允许推进 |
| 新候选与可选既有候选各自合法后，同身份核心不同、终态变化、逆向或其它不允许阶段转换 | 逻辑内冲突 | 零结构变化 |
| 已接受成功事件后句柄无效、序号为零，或冻结 / 回执内嵌请求 / 回执 / 事件身份不一致 | 追根因解决 | 身份不变量破坏；停止后继 |
| 已接受成功事件中内容缺失，冻结、回执、结果、结算互相矛盾，阶段元组非法，或最终阶段缺少 / 矛盾正式结算材料 | 追根因解决 | 候选自身非法；内部不一致并停止后继，不进入同账比较 |
| 已发布既有候选的格式、身份、核心或阶段负载自身非法 | 追根因解决 | 已发布结构内部不一致；停止后继，不扫描选优 |
| 候选输出投影保留可变长集合或被排除来源业务 DTO | 追根因解决 | 投影合同破坏；停止后继 |
| 形成结果与已接受来源逐字段不一致 | 追根因解决 | 算法不变量破坏 |
| 已发布层同一身份出现多个当前记录 | 追根因解决 | 不扫描选优、不静默删除 |
| 写入开始后读回、确认、发布、撤销或恢复异常 | 追根因解决 | 停止后继；仅在正式证明覆盖的路径声明精确撤销 |
