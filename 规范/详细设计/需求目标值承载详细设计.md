# 需求目标值承载详细设计

编号：022-D1
日期：2026-07-05
状态：专项详细设计草案 / 待确认 / S0 缺口 2

## 1. 定位

本详细设计用于修正当前 `需求服务` 直接把 I64 目标值写入需求节点主信息值容器的问题。

本设计不迁移旧项目需求主信息字段，不新增需求主信息记录，不声明需求系统完成。

## 2. 设计目标

第一轮只处理最小闭环中的 I64 下限目标：

```text
当前值 >= 目标值
```

本轮不实现任意比较表达式、不实现需求树完整目标模型、不实现价值结算摘要。

## 3. 规范承接

| 上层规则 | 本设计承接 |
| --- | --- |
| MI-V1 值容器范围收束 | 需求、任务、方法不得把主信息仓库当通用值容器 |
| 特征值服务限制 | 特征值节点只保存特征系统当前值，不保存需求目标值 |
| 最小闭环设计 | 创建需求时必须已有目标存在、目标特征、当前特征值、目标值和验证方式 |
| 基础信息服务分层 | 需求服务通过存在、特征、特征值和状态服务取得基础能力 |
| 入口前置拒绝 | 无效目标存在、无效目标特征、当前值缺失或目标状态创建失败时不创建有效需求 |

## 4. 结构承载

第一轮使用现有节点类型和关系类型，不新增主信息字段。

| 结构 | 用途 | 写入方 | 读取方 | 生命周期 | 验证方式 |
| --- | --- | --- | --- | --- | --- |
| 需求节点 | 需求身份 | 需求服务 | 需求服务、任务服务 | 随需求生命周期 | 创建后节点类型为需求 |
| 目标存在节点 | 被需求改变或满足的存在主体 | 存在服务 | 需求服务、任务服务 | 随存在生命周期 | 创建需求入口检查节点类型为存在 |
| 目标特征节点 | 被需求判断的特征类型 | 特征服务 | 需求服务、方法服务 | 随特征生命周期 | 创建需求入口检查节点类型为特征 |
| 当前特征值节点 | 目标特征当前值 | 特征值服务 | 需求服务、方法服务 | 随特征值生命周期 | 创建需求入口必须能读出当前 I64 值 |
| 目标状态节点 | 需求目标值承载 | 状态服务 | 需求服务 | 随需求目标生命周期；第一轮作为抽象目标状态 | 读取状态值等于目标 I64 值 |
| 需求目标状态关系 | 需求到目标状态的模板关系 | 需求服务 | 需求服务 | 随关系生命周期 | `模板` 关系可回查目标状态节点 |
| 需求目标材料关系 | 需求到目标存在、目标特征、当前特征值的引用关系 | 需求服务 | 需求服务、任务服务 | 随关系生命周期 | `引用` 关系可按节点类型过滤回查 |

第一轮关系口径：

```text
需求节点 -> 目标存在节点：引用
需求节点 -> 目标特征节点：引用
需求节点 -> 当前特征值节点：引用
需求节点 -> 目标状态节点：模板
```

使用 `模板` 关系承载目标状态，是为了避免与需求结算状态的 `引用` 关系混淆。后续若新增 `需求目标状态` 等专用关系类型，必须先更新节点类型与关系类型枚举规范。

## 6. 创建流程

```text
输入：目标存在、目标特征、当前特征值、目标值
-> 检查目标存在为存在节点
-> 检查目标特征为特征节点
-> 通过特征值服务读取当前特征值，缺失则拒绝
-> 通过状态服务创建目标状态节点，状态值 = 目标值，目标状态不携带发生时间戳
-> 创建需求节点
-> 写需求到目标存在、目标特征、当前特征值的引用关系
-> 写需求到目标状态的模板关系
-> 返回需求节点
```

目标状态节点不是实例状态，不带发生时间戳，不作为需求结算状态。

需求结算状态仍由 `记录结算状态` 通过状态服务创建场景内实例状态，并带发生时间戳。

## 7. 结算流程

```text
输入：需求节点、当前特征值节点
-> 检查需求节点有效且类型为需求
-> 读取需求目标状态节点
-> 通过状态服务读取目标 I64 值
-> 通过特征值服务读取当前 I64 值
-> 当前值 >= 目标值：返回 已满足
-> 当前值 < 目标值：返回 未满足
-> 证据缺失：返回 证据不足
```

`需求结算结果` 仍是局部领域枚举，不得持久化为普通 bool。

## 8. 与旧函数的关系

当前代码中的以下函数是 S0 缺口来源：

```text
创建I64目标需求(std::int64_t 目标值)
结算I64目标(...)
```

后续代码切片必须二选一：

```text
删除外部可用路径，改用新函数名。
或保留旧函数名作为兼容壳，但内部不得再读写需求节点主信息值容器。
```

兼容壳若保留，必须明确调用新结构路径，并在函数说明中标记为兼容入口。

## 9. 验收条件

代码实施后必须验证：

```text
有效目标存在、目标特征、当前特征值和目标值可创建需求。
需求节点主信息值容器不再保存目标值。
需求通过模板关系可回查目标状态节点。
目标状态节点状态值等于目标 I64 值，且不携带发生时间戳。
当前值 >= 目标值 时结算为已满足。
当前值 < 目标值 时结算为未满足。
无效目标存在拒绝且不创建需求。
无效目标特征拒绝且不创建需求。
当前特征值缺失拒绝且不创建需求。
特征值节点不能被用来保存需求目标值。
MC-01 到 MC-11 不回归。
```
