## 1. 目标与边界

本设计冻结自我内部治理能够启动的最小本能方法集。方法不是名称、函数或线程分支，而是具有稳定身份、条件、目标结果、动作入口、内容版本、来源、可执行性和实际回读边界的正式对象。

无候选方法时不得自动创建方法首；被动提交类方法不参加主动召回。执行成功只表示动作入口完成，不直接表示任务完成、需求满足或价值结算。

## 2. 最小方法清单

| 稳定方法身份 | 类型 | 适用条件 | 目标结果与实际回读 | 动作归属 |
| --- | --- | --- | --- | --- |
| `本能.同步运行态` | 主动可召回 | 运行态投影与正式事实版本可比较 | 运行态投影与正式事实一致；回读版本 | 自我存在 / 运行服务 |
| `本能.改变自我实例特征` | 主动可召回 | 任务已冻结合法自我治理目标、槽位和允许值域 | 目标槽当前值达到冻结目标；回读状态与动态 | 特征、状态、动态领域 |
| `本能.进入待机` | 主动可召回 | 生存约束允许且当前非待机 | 待机状态事实成立 | 自我状态服务 |
| `本能.退出待机` | 主动可召回 | 唤醒条件成立且当前待机 | 活动状态事实成立 | 自我状态服务 |
| `本能.重置时序` | 主动可召回 | 任务明确给出可重置时序目标和范围 | 新时序版本发布并实际读回 | 时序所属领域 |
| `本能.结算合法任务结果` | 被动提交 | 已有任务验证结果、正式事实和结算前置 | 任务、需求与 4320 / 6100 对应价值结算正式入口分别返回结果 | 只协调三个正式入口，不拥有或跨域写入任何任务、需求、价值事实 |
| `本能.故障收口` | 主动 / 系统承接 | 具名故障影响当前治理路径 | 受影响路径隔离、占用收口、事实可读 | 任务与运行治理服务 |
| `本能.尝试学习` | 主动可召回 | 已有可解释内部因果样本，能回读动作主体、动作入口、条件、前后状态、结果特征和变化方向 | 形成候选方法结构、条件结果样本、参数或结构化缺口 | 只调用 5330 允许的候选入口，不登记 / 覆盖方法 |
| `本能.定向学习` | 主动可召回 | 外界信息、未归因现象，或动作主体 / 动作入口未知；候选范围、观察范围和值域已冻结 | 形成观察、确认、信息和方向材料或具名拒绝 | 只调用 5330 定向材料入口，不补已有内部动作方法 |
| `本能.练习方法` | 主动可召回 | 目标方法已有真实动作入口和既有条件 / 结果结构，练习场景、安全约束和参数值域已冻结 | 形成真实运行观察、参数探索、好用差距和新用途候选证据 | 方法执行 / 用途观察入口；不代写目标方法账 |

`改变自我实例特征` 只能处理已冻结的自我治理目标，不是任意业务目标的通用写入兜底。进入和退出待机是两个强类型动作，不用一个布尔值承担动作身份。

学习统计严格按 4330 分栏：尝试未达成与最终已结算可以分别进入确定统计，待结算观察不得进入效果差异；三类学习能力继续服从 5330 的输入分流，不能仅凭“已有观察组”互相替代。

## 3. 方法结构

每个登记规格必须回答：

| 项 | 设计 |
| --- | --- |
| 身份 | 方法稳定句柄、方法族、内容版本、父代或来源 |
| 承载结构 | 方法节点、条件关系、结果关系、动作入口稳定键、任务特征声明 |
| 写入方 | 方法领域唯一登记和发布 |
| 读取方 | 任务筹办、方法执行、用途观察 |
| 生命周期 | 内容版本不可变；历史代际保留；索引可重建 |
| 并发隔离 | 方法身份 + 内容版本；同义重放幂等，异义冲突拒绝 |
| 前置拒绝 | 条件、结果、动作、来源、作用对象、场景或版本不完整 |
| 内部错误 | 前置通过后发布、联合读回或动作绑定不一致 |
| 恢复 | 从方法、关系、类型化记录恢复，不自动登记或晋级 |
| 验证 | 方法清单完整、可召回边界明确、动作实际回读成立 |

## 4. 登记与执行顺序

```text
形成值式登记规格
-> 校验稳定身份、条件、结果、动作、来源和版本
-> 同键同义则返回既有代际
-> 同键异义则拒绝
-> 方法领域原子建立节点、关系和类型化记录
-> 完整读回后发布
-> 任务筹办按条件与结果召回
-> 冻结方法代际、任务授权和输入
-> 执行动作入口
-> SELF-C4 读取实际事实并独立验证任务目标
```

## 6. 验证

- 十个稳定方法身份逐项具备条件、结果、动作和版本。
- 重复登记同义收敛，异义拒绝；历史代际不可覆盖。
- 主动召回不返回被动结算方法。
- 改变特征只作用于冻结自我目标，不能写任意业务对象。
- 方法执行后必须进入实际回读；动作成功不能直接完成任务。
- 恢复不自动创建、选择、执行或晋级方法。
