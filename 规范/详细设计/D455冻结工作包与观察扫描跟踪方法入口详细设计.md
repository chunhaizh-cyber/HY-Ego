# D455 冻结工作包与观察扫描跟踪方法入口详细设计

日期：2026-07-21

状态：DD-04 P0—P6 已完成 / 当前代码事实已复核 / 代码未实现 / 依赖 DD-01—DD-03 与 TASK-PREP-RIGHT 正式实施和集成

## 1. 设计目标

本设计把 DD-03 已承接外设材料引用接入任务执行冻结，但不让通用任务冻结请求拥有 D455 产品、报告队列或材料寿命：

```text
既有任务执行基础冻结
+ DD-03 签发的不透明已承接材料引用
+ 产品到方法入口的唯一强类型映射
+ 方法执行内容快照与业务前置快照
+ 工作项、消费窗口和失效条件
-> 不可变任务外设方法冻结工作包
```

目标同时回答：

1. 冻结工作包如何只携带任务域签发的不透明引用，而不复制产品、交接包和受控材料；
2. 稳定观察子集、扫描变化、目标跟踪如何分别只进入观察、扫描、跟踪方法材料入口；
3. 原始逐簇报告为何只能留在任务域质量门、回查和诊断；
4. 识别方法为何不存在外设报告直通入口；
5. 工作包候选与 DD-03 领取候选如何在确认前可撤销、确认后无失败发布；
6. 冻结前已有的方法、动作入口、条件结果或产品映射缺口如何归为筹办漏检和内部逻辑错误；
7. 未派发、执行前失效、取消和运行期停止时如何具名结束材料寿命。

本设计只形成方法运行前的合法输入面，不执行观察、扫描或跟踪业务动作，不提交世界事实，不裁决任务完成。

## 3. 当前代码事实与复用边界

1. `协议.任务执行请求.ixx` 的 `任务执行冻结请求` 已携带调度、任务、方法、动作材料和适配器种类，但适配器只有 `任务目标状态值变化`。
2. `冻结任务执行请求完整` 已复核完整任务承接、筹办中阶段、当前方法选择、活跃方法、唯一完整动作入口、双向任务关系和关系版本；它硬编码目标状态值变化适配器。
3. `任务执行组合器` 构造时只接收一个稳定动作键，冻结时硬编码目标状态值变化适配器；当前性复核只作为类内私有步骤。
4. `任务执行组合器::执行任务目标状态值变化` 是既有唯一动作实现，不能被观察、扫描或跟踪复用。
5. `方法业务服务` 可以读取方法登记项及条件、结果、动作入口规格组；当前没有 D455 产品入口类型，也没有观察、扫描、跟踪入口隔离。
6. 当前方法结构没有一个单字段通用“方法内容版本”。现有可冻结证据是方法生命周期关系版本、条件 / 结果来源版本、规格节点与关系版本、动作入口关系版本和稳定动作键。
7. 当前任务执行冻结没有工作项、消费窗口、已承接材料引用、产品类型和方法业务前置快照。
8. `任务管理线程` 和 `任务工作线程` 只消费既有 `任务执行冻结请求`；本设计不修改它们。
9. 当前生产代码没有“已确认观察存在”专用公开结构，也没有观察、扫描、跟踪方法执行体。本设计不得伪造这些能力。
10. `任务执行调度与强类型回执详细设计.md` 的状态行仍写“代码未实现”，但 #224 / DQ-116 / JY-350 和阶段 600 已正式完成，属于文档状态滞后，不是代码缺失。

复用裁决：

- 复用现有基础冻结、任务 / 方法 / 动作材料和当前性复核，不复制第二套任务权威读取算法；
- 对现有适配器只做强类型窄扩展，不让目标状态值变化执行入口接受外设适配器；
- 外设材料放入独立工作包，不增加到通用 `任务执行冻结请求`；
- 方法执行内容版本在本包内用完整冻结证据组形成强类型复合快照，不在本切片建立通用 M0 方法代际系统；
- 已确认存在、扫描基准、目标凭证等业务前置使用不透明前置引用和值式版本绑定，正式生产者由 DD-05 复核接入。

## 4. 总体架构裁决

### 4.1 基础冻结与外设工作包分层

```text
任务执行冻结请求
-> 只冻结通用任务、方法、动作和调度材料

任务外设方法冻结工作包
-> 持有一个完整基础冻结
-> 持有一个 DD-03 不透明已承接引用
-> 持有产品 / 方法入口 / 前置 / 当前性元数据快照
-> 不持有产品负载、受控材料引用组、交接包、队列或消费权
```

通用冻结请求不成为 D455 专用协议。目标状态值变化和外设方法共同复用其任务、方法和动作当前性证明，但只有外设工作包可以携带 DD-03 引用。

### 4.2 任务执行适配器窄扩展

`任务执行适配器种类` 增加：

```text
D455稳定观察材料
D455扫描变化材料
D455目标跟踪材料
```

规则：

1. `任务目标状态值变化` 继续只允许进入 `执行任务目标状态值变化`；
2. 三个 D455 适配器只能进入 DD-04 工作包组合器，不能调用目标状态值变化执行入口；
3. 不新增 `D455原始逐簇材料` 适配器；
4. 不新增 `D455识别材料` 适配器；
5. 基础冻结完整性接受四个已知适配器，但具体产品、方法种类和工作包完整性由 DD-04 二次复核；
6. 未知适配器在形成冻结前拒绝；
7. 既有 `确认任务进入排队中` 和 `执行任务目标状态值变化` 必须显式拒绝三个 D455 适配器，不能仅依赖通用冻结完整性；
8. D455 工作包的排队和执行迁移只能由 DD-06 后继的工作包感知入口提交，DD-04 不借用既有目标状态值变化调度路由提前排队。

`任务执行组合器` 增加构造时的支持适配器参数，默认值保持 `任务目标状态值变化`，现有调用点不改语义。它还公开只读 `复核任务执行冻结当前性`，供 DD-04 在材料确认前复核；该入口不得写任务或方法。

### 4.3 方法材料适配表不是方法选择器

`任务外设方法材料适配器` 在自身构造时取得一组由 DD-04 固定、可审计且不可变的入口合同：

```text
方法首
方法执行内容快照版本
动作入口
稳定动作键
任务执行适配器种类
唯一合法 D455 产品类型
材料视图种类
入口合同版本
允许的业务前置种类集合
```

适配表只把已经由任务选择并冻结的方法动作入口映射到技术材料视图。它不得：

- 召回或选择方法；
- 创建方法首；
- 用产品类型替代方法条件、结果或动作入口；
- 修改方法长期能力；
- 允许调用方临时指定任意产品到任意方法。

同一方法、动作入口和稳定动作键只能有一个当前入口合同。相同键对应不同方法种类或产品类型属于装配内部错误。

入口合同不是由生产运行期调用方临时填写的业务配置，也不新增方法能力事实。它只能把既有方法条件、结果、动作入口和稳定动作键收窄到一个技术材料形状；构造完成后 DD-05 / DD-06 只能读取，不能增删、替换或绕过合同。

### 4.4 工作包候选与 DD-03 领取候选顺序

固定顺序：

```text
准备 DD-03 材料领取候选
-> 只读产品和值式来源元数据
-> 形成既有任务执行基础冻结
-> 回读完整方法条件、结果、动作入口和适配合同
-> 裁决产品、适配器、方法入口和业务前置唯一匹配
-> 在 DD-04 数据操作中预留未发布工作包候选及全部容量
-> 精确读回工作包候选
-> 再次复核基础冻结、材料引用和工作项当前性
-> DD-03 确认材料进入该工作包
-> noexcept 安装确认材料绑定并发布不可变工作包
```

DD-03 确认前任一正常拒绝都必须撤销工作包候选并释放领取候选回 `已承接待领取`。DD-03 确认后不存在普通失败；工作包候选已提前完成容量、索引和对象预留，安装与发布必须不分配、不调用用户代码且不抛异常。

### 4.5 方法执行内容快照

本包的 `方法执行内容快照` 至少冻结：

```text
方法登记项和生命周期关系版本
当前选择中的来源方法生命周期关系与版本
按稳定顺序排列的全部方法条件规格及来源版本
按稳定顺序排列的全部方法结果规格及来源版本
唯一动作入口及角色、状态、输入 / 输出场景关系和稳定动作键
方法材料入口合同及合同版本
适配器种类和产品类型
```

`方法执行内容快照版本` 是以上完整值式证据组的强类型复合版本。任一节点、关系、来源版本、动作键、适配合同或集合成员变化都生成不同版本并使旧工作包失效。它只解决 DD-04 冻结当前性，不宣称通用方法内容代际、学习晋级或 M0 已实现。

### 4.6 业务前置保持后继门控

DD-04 定义三类不透明业务前置引用：

```text
观察前置：任务场景、观察范围、期望特征集合和规则版本
扫描前置：自我内部场景、已确认存在集合、可比基准、覆盖集合和比较规则版本
跟踪前置：已确认目标存在、唯一凭证特征、观察窗口、范围和目标约束版本
```

引用必须包含类型、稳定身份、版本、合法生产者和当前性材料，但不复制世界事实。当前代码没有完整生产者时，生产接线保持门控；自检可以使用隔离夹具。不得用普通节点句柄、自由文本或一个 bool 伪装完整前置。

```text
失效条件集合
```

发布后所有字段不可原地修改。事实变化、方法变化、窗口变化或续订材料必须形成新工作包版本和新筹办轮次。

## 7. 三类产品到方法入口的唯一映射

| DD-01 产品 | 合法任务执行适配器 | 合法方法材料入口 | 最小业务前置 | 禁止 |
| --- | --- | --- | --- | --- |
| 原始逐簇报告 | 无 | 无 | 只由任务域质量门 / 回查 / 诊断读取 | 进入任何业务方法工作包 |
| 稳定观察子集报告 | `D455稳定观察材料` | 观察 | 任务场景、观察范围、期望特征集合、L3 与质量 / 时效当前 | 识别、扫描、跟踪 |
| 扫描变化报告 | `D455扫描变化材料` | 扫描 | 自我内部场景、已确认存在集合、基准、覆盖集合、比较和阈值来源 | 观察、识别、跟踪 |
| 目标跟踪报告 | `D455目标跟踪材料` | 跟踪 | 已确认目标、唯一凭证特征、观察窗口、范围和目标约束 | 观察、识别、全场扫描 |

空稳定观察子集可以作为任务域结构化空结果，但不能形成观察方法工作包。扫描无基准不得形成扫描变化工作包。目标暂失或丢失可以形成跟踪方法输入，但不得填造坐标。

识别只读取观察链已经形成的已有特征、候选范围和存在材料；DD-04 协议、适配器和组合器均不得出现识别外设产品入口。

## 9. 形成冻结工作包的固定流程

### 9.1 写前准入

请求至少提供：

```text
工作包身份和冻结轮次
任务执行调度请求
DD-03 不透明已承接引用
预期方法材料入口种类
完整业务前置快照
冻结、截止、验证、归因和禁止项版本
```

先拒绝零身份、错上下文、错任务、错工作项、错窗口、过期、未知入口、识别入口、原始逐簇入口、前置 variant 错配和截止时间无效。

### 9.2 领取和基础冻结

```text
DD-03 准备领取候选
-> 状态进入工作包保留中
-> 读取产品值式合同和来源元数据
-> 按入口种类选择构造时已冻结的任务执行适配器
-> 既有任务执行组合器冻结任务、方法和动作
```

基础冻结失败时释放领取候选。若任务仍处于合法可重筹办状态，返回强类型当前性或入口拒绝；若任务已经选择方法但方法、动作入口或结构缺失，则按第 13 节内部错误处理。

### 9.3 方法与产品复核

```text
读取方法登记项
-> 读取并稳定排序全部条件
-> 读取并稳定排序全部结果
-> 读取唯一动作入口
-> 读取唯一方法材料入口合同
-> 形成方法执行内容快照
-> 比较产品类型、适配器、入口种类和前置种类
```

观察 / 扫描 / 跟踪映射必须一一对应。调用方不能通过修改请求中的预期种类改变适配结果。

### 9.4 工作包候选和确认

```text
预留未发布工作包候选及索引容量
-> 精确读回候选全部字段
-> 复核基础冻结当前性
-> 复核 DD-03 领取候选、材料期限、工作项和窗口当前性
-> DD-03 确认材料进入具名工作包
-> noexcept 安装确认标记并原子发布工作包
-> 按工作包身份完整读回
```

确认前读回不同必须撤销两侧候选并追根因。确认后读回不同必须将工作包和材料绑定锁存在 `内部隔离`，停止依赖路径，不得释放材料伪装成普通失败。

## 10. 执行前当前性与失效

每次排队、领取执行或形成方法材料视图前必须验证：

```text
运行期上下文世代和工作包身份一致
任务、筹办轮次和调度请求一致
任务当前阶段允许该操作
当前方法选择仍指向冻结方法
方法生命周期、条件、结果、动作入口和适配合同仍匹配内容快照
DD-03 引用、工作项、窗口和产品绑定仍当前
业务前置引用、版本和合法生产者仍当前
产品和工作包均未过期
禁止项没有新命中
```

在冻结后发生的合法事实变化使工作包失效，并进入新筹办轮次或合法等待；不得在旧工作包上就地换产品、换方法、换前置或延长时间。

若执行期才发现冻结前已经存在下列缺口，属于筹办漏检：

```text
方法不存在或冻结前已失效
动作入口缺失或不唯一
条件 / 结果 / 条件结果规格在冻结前不完整
产品到方法入口映射缺失或歧义
扫描缺已确认存在、基准或覆盖合同
跟踪缺目标、凭证或观察窗口
工作包、任务、工作项或消费窗口绑定缺失
```

这些情况不得改写成任务失败、合法等待、普通材料不足或临时补造结构。

## 11. 工作包结束与材料寿命

工作包必须通过具名结束入口提交下列之一：

```text
正常方法消费结束
执行前当前性失效
任务合法取消
任务授权失效
工作包过期未派发
运行期有序停止
内部隔离待追根因
```

未派发、排队失败或执行前失效的已确认工作包不能把材料“退回队列”，因为 DD-02 报告已经正式承接；它必须结束 DD-03 材料消费并保留强类型原因。任务需要新材料时建立新等待项、窗口和承接记录。

析构函数只能释放未确认的本地候选；不得用析构静默结束已发布工作包、销毁不可舍弃材料或改变 DD-03 承接状态。

## 13. 入口拒绝、逻辑内返回和内部逻辑错误

| 阶段 | 逻辑内拒绝 / 返回 | 内部逻辑错误 |
| --- | --- | --- |
| 请求准入 | 零身份、错上下文、错任务 / 工作项 / 窗口、过期、未知入口 | 已发布同身份工作包内容与请求不同 |
| DD-03 领取前 | 引用失效、已有领取者、窗口关闭、用途不允许 | 同一承接记录同时进入两个领取候选 |
| 产品映射 | 原始逐簇、识别、产品与预期入口不符 | 同一当前动作入口存在两个不同合法映射 |
| 基础冻结 | 任务尚未达到筹办可冻结条件、当前性合法变化 | 已选择方法但方法、动作、条件 / 结果结构在冻结前缺失 |
| 业务前置 | 前置生产者尚未形成、引用过期或当前性变化 | 冻结前本应完整的扫描 / 跟踪前置缺失却已宣称可执行 |
| 工作包候选 | 容量不足、确认前当前性变化 | 候选完整读回与输入不同 |
| DD-03 确认前 | 任一逻辑内拒绝均撤销候选并释放领取 | 确认前无法精确撤销候选 |
| DD-03 确认后 | 无普通失败分支 | 安装、发布、索引、状态或读回不一致，进入内部隔离 |
| 执行前复核 | 冻结后事实合法变化、期限到达、授权取消 | 冻结前既有结构缺口被执行期发现 |
| 工作包结束 | 已结束幂等读回；内部隔离需等待人工根因治理 | 静默析构已发布包、丢弃不可舍弃材料或二次结束异义 |

内部逻辑错误按公共诊断规则写独立逻辑错误日志并在宏开启时弹窗，但诊断不修复结构、不改变任务状态、不释放隔离材料。

## 14. 并发与锁边界

1. 同一工作包身份最多一个候选、一个当前记录和一个状态链。
2. 同一 DD-03 已承接记录最多进入一个工作包。
3. 同一任务在筹办阶段仍由任务稳定身份的唯一筹办执行权串行；DD-04 不新增线程级锁替代它。
4. 不同任务、不同工作项的工作包可以并发形成和读取，即使使用同一可复用方法或同一 D455 设备来源。
5. DD-03 领取候选不持有 DD-02 队列锁；DD-04 不取得 DD-02 消费权。
6. 组合器不在持有工作包数据锁时调用任务、方法或 DD-03 服务。
7. 固定顺序是先取得 DD-03 领取候选，再完成外部只读与候选预留，最后短时确认两侧；不得形成任务仓库锁、DD-03 锁和工作包锁循环。
8. 工作包发布后只读；状态迁移使用独立版本和当前性，不原地改写冻结内容。
