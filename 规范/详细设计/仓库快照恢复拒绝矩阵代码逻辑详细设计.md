# 仓库快照恢复拒绝矩阵代码逻辑详细设计

更新时间：2026-07-08

状态：已生成 / 可直接生成待确认施工计划 / 未生成施工计划 / 未入队 / 未改 C++ / 未构建 / 未迁移旧函数

## 1. 设计定位

本设计承接第 18 项 `流程图/20260708_仓库快照恢复拒绝矩阵代码逻辑流程图_v0.1.md`，只定义后续恢复专项必须满足的拒绝矩阵和隔离恢复边界。

当前没有仓库快照服务、快照格式版本代码或恢复入口，本设计不允许恢复写入。

## 2. 输入入口

```text
节点快照
主信息快照
关系快照
索引快照
事件日志材料
快照头
格式版本
```

未有用户确认的恢复专项和施工计划时，所有恢复请求必须拒绝，当前运行期仓库保持不变。

## 3. 核心处理

```text
确认是否已有恢复专项许可
-> 读取快照头和格式版本
-> 复核版本、枚举和字段
-> 复核节点句柄、节点类型和主信息归属
-> 复核关系端点、关系类型、顺序号和唯一性
-> 复核索引键、索引目标和权威结构一致性
-> 构建隔离候选仓库
-> 执行最小闭环读回
-> 后续确认后才允许替换运行期仓库
```

任何失败都必须拒绝恢复，不能尝试兼容性猜测写入。

## 4. 结构承载

```text
候选快照材料：只作为恢复输入材料
隔离候选仓库：后续专项中的临时恢复承载
运行期仓库：未通过全部门禁前保持不变
恢复拒绝矩阵：版本、句柄、类型、关系、索引、读回门禁
```

数据库只可作为快照、事件日志、恢复和审计适配，不直接裁决机器事实。

## 5. 输出变化

成功路径在后续专项中只能输出隔离候选仓库通过读回的结果，并在另行确认后才允许替换运行期仓库。

失败路径必须丢弃候选仓库，当前仓库不变，不产生半恢复结构。

## 6. 当前代码差距

```text
无仓库快照服务。
无快照格式版本。
无恢复入口。
无隔离候选仓库恢复。
无替换门禁、恢复数量快照或恢复失败审计结构。
```

## 7. 验收矩阵

| 验收项 | 验证方式 | 通过口径 |
| --- | --- | --- |
| 未立项拒绝 | 无恢复专项许可时请求恢复 | 当前仓库不变 |
| 版本拒绝 | 快照版本或枚举不匹配 | 拒绝且不猜测兼容 |
| 句柄拒绝 | 悬空句柄或错误节点类型 | 不创建半节点 |
| 关系拒绝 | 端点缺失或关系冲突 | 不写半关系 |
| 索引拒绝 | 索引指向无效或覆盖关系事实 | 不用索引覆盖权威结构 |
| 隔离读回 | 候选仓库最小闭环失败 | 丢弃候选仓库 |

## 8. 后续边界

后续恢复能力必须先生成待确认施工计划，列出允许文件、禁止文件、隔离恢复策略、失败收口、读回验证和完成声明边界。
