# 需求结算单一正式合同语义收口详细设计

编号：DEMAND-SETTLEMENT-001-D2

日期：2026-07-16

状态：JY-369 目标设计 / #286 待执行 / 当前代码仅部分符合

## 1. 目标

把当前需求结算正式模块收口为：

```text
目标状态值 == 实际状态值 -> 允许形成唯一正式已满足结算。
目标状态值 != 实际状态值 -> 任务未达成，不完成既有通用组合入口（#225 新结果治理路径之外）的任务，不写需求最终结算。
同一需求零或一条正式结算；完整同义重复幂等读回，任一来源字段异义冲突。
读到第二条正式结算 -> 内部结构矛盾，追根因，不选择最近记录。
```

## 2. 当前事实与缺口

| 当前入口 | 已有事实 | 缺口 |
| --- | --- | --- |
| `需求业务服务::判定目标状态达成` | 使用值相等裁决 | 已符合目标 |
| `任务结果结算组合器::处理任务执行结果` | 未达成转待重筹办且不调用结算 | 只证明 #225 新组合路径 |
| `需求业务服务::提交独立结算` | 复核任务、目标和实际状态 | 不相等仍形成 `未满足` 并提交 |
| `需求结算写入规格` | 封闭结算结构写入规格 | 同时接受 `已满足 / 未满足` |
| `需求任务方法数据操作::提交需求结算` | 首条结算在同一结构会话写入 | 已有记录只比较结论，不比较完整来源 |
| `需求任务方法数据操作::读取需求_已许可` | 可读结算状态 | 多条时选择最近关系，仍接受 `未满足` |
| `需求任务方法组合器::完成任务并独立结算` | 组合任务完成和需求结算 | 先完成任务，后结算；未达成时可能形成错误终态 |
| `组合.运行期业务操作` | 公开转发上述组合入口 | 本设计不改公开签名，只验证转发仍进入收口后的实现 |

`需求服务.h` 的 `>= / 证据不足 / 未满足 / 最近记录` 是历史兼容路径，本设计不修改。它继续作为明确未收口边界，不能用于扩大 #286 完成声明。

## 3. 目标结构

正式结算仍由既有结构承载，不新增节点类型、关系类型或主信息字段：

| 结构 | 基数 / 值 | 写入方 | 读回和比较 |
| --- | --- | --- | --- |
| 需求到结算状态归属 | 每需求零或一条 | 需求专用数据操作层 | 两条及以上为内部矛盾 |
| 结算实例状态 | 值只能为 `已满足` | 状态数据操作同会话参与 | 比较幂等主键、场景、主体、发生时间和状态值；需求归属顺序号必须等于发生时间戳 |
| 来源任务引用 1 | 恰一条 | 需求专用数据操作层 | 完整句柄相等 |
| 目标状态模板 2 | 恰一条 | 需求专用数据操作层 | 必须等于需求当前目标状态 |
| 实际状态引用 3 | 恰一条 | 需求专用数据操作层 | 完整句柄相等 |
| 动态因果来源 4 | 恰一条 | 需求专用数据操作层 | 完整句柄相等 |
| 可选因果引用 5 | 零或一条 | 需求专用数据操作层 | 双方都空或完整句柄相等 |

`数据操作.需求任务方法` 新增并唯一拥有值式 `需求正式结算材料`，至少包含结算状态身份 / 幂等主键、场景、主体、发生时间、状态值、需求归属关系、来源任务、目标状态、实际状态、动态、可选因果及固定角色关系证据。`需求权威材料` 增加可选正式结算材料；既有 `当前结算 / 最近结算状态` 仅作为由该材料投影的兼容读字段，不再承担最近选择语义。

## 4. 服务入口

### 4.1 提交独立结算

`需求业务服务::提交独立结算` 在形成实例状态规格前必须完成：

```text
需求、来源任务、目标状态、实际状态、动态完整；可选因果引用只能全零或完整有效，部分句柄写前拒绝。
来源任务已完成且实际结果状态等于请求实际状态。
目标状态是需求唯一抽象目标；实际状态的场景 / 主体与任务场景 / 虚拟存在一致。
动态证据当前至少可读且节点类型为动态；更深动态语义继续由 #225 任务结果组合路径复核，本切片不扩权猜测。
目标状态值与实际状态值相等。
```

值不相等是设计允许的写前拒绝，返回入口拒绝且结构零变化；不得形成 `未满足` 实例状态规格。

### 4.2 完成任务并独立结算

`需求任务方法组合器::完成任务并独立结算` 必须先读取任务及其来源需求，调用 `判定目标状态达成`。只有裁决完整且为 `已达成` 才允许调用任务完成，再调用独立结算。未达成或材料不足时两个写入口都不调用。

## 5. 完整结构幂等

已有唯一结算时，数据操作层重建结算值式材料并逐项比较：

```text
结算状态幂等主键、场景、主体、发生时间戳、状态值。
来源任务、需求目标状态、实际状态、动态证据、可选因果引用。
需求与来源任务、任务实际结果状态、场景和主体当前性。
```

全部相同才返回 `幂等读回`。任一字段不同返回 `幂等冲突` 且结构零变化。哈希、结论、状态值或幂等主键单独相等都不足以裁决同义。

首次写入不能只依赖会话外预读。进入独占结构写入会话后，必须再次确认需求的状态型 `归属` 为零；若已被并发请求占用，本次不写候选，退出后重读 `需求正式结算材料` 并按完整同义 / 异义裁决。归属顺序号固定为结算发生时间戳。同义并发和异义并发都只能留下一个正式记录。

## 6. 单一正式记录读取

需求读取只接受零或一条符合结算角色的有效状态归属：

```text
零条 -> 当前结算为未结算。
一条且固定关系 1—5 基数 / 端点完整、值为已满足、归属顺序号等于发生时间 -> 读回唯一正式结算材料。
两条及以上、值为未满足、关系组不完整或目标类型异常 -> 内部不一致。
```

不得按时间戳、关系顺序号、关系编号或节点创建序号选择最近记录。

服务读取需求或提交结算时必须保留数据操作的失败分层。`内部不一致`、许可拒绝和版本漂移不得统一降级成普通入口拒绝；发现多记录或畸形正式结构时，读取结果和提交结果都必须可见为内部不一致并停止后继。

## 7. 非成功收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 无效句柄、零键、零时间、材料缺失、目标值不等 | 逻辑内返回 | 第一笔写入前结构零变化 |
| 已有唯一结算与请求任一字段不同 | 逻辑内幂等冲突 | 结构零变化 |
| 读到多条结算、关系组或状态值内部矛盾 | 追根因解决 | 不选择最近记录，不继续提交 |
| 进入写入后状态、关系、读回或确认不及预期 | 追根因解决 | 精确撤销未发布候选，不发布半结构 |

## 8. 自检映射

复用 `自检.需求任务方法分层` 与 `自检.任务结果结算`，不新增阶段：

```text
不相等直接调用服务 -> 拒绝且结构零变化。
不相等调用既有通用组合入口（#225 新结果治理路径之外） -> 任务不完成、需求不结算。
完整同义重复 -> 同一结算状态幂等读回。
同一来源任务但实际状态不等于任务权威实际结果 -> 写前入口拒绝且零变化。
另一个已完成来源任务连同其对应实际状态、动态形成完整有效异义元组 -> 已有正式结算时幂等冲突且零变化。
动态、可选因果、结算键或时间在其余前置合法时异义 -> 幂等冲突且零变化。
第二条结算或多记录读回 -> 内部不一致，不选最近记录。
并发完整同义请求 -> 单提交，其余幂等读回；两组异义并发 -> 仅一组可成为赢家，总计仍只有一条正式记录，另一组冲突。
全零可选因果可接受，完整有效可选因果可比较；部分句柄写前拒绝且结构零变化。
#225 三种正式任务结果治理回归不退化。
```
