# 0500 子规范 字段与结构新增准入约束

编号：0500
日期：20260722
状态：正式子规范
权威等级：正式规范

## 1. 定位

本规范是 0050 下的通用结构准入子规范，规定新增字段、结构、特征类型和只读投影前必须完成的归属判断。

直接上级规范：0050

直接依赖规范：无

相关边界规范：0300

## 2. 默认准入顺序

新增任何业务承载前，必须依次判断：

1. 是否只是已有对象的新身份。是则使用直接含稳定主键的节点与稳定句柄。
2. 是否表达对象可具有的属性。是则使用特征和特征值。
3. 是否表达对象在一个时点的复合情况。是则使用状态及其特征集合。
4. 是否表达前后状态变化。是则使用动态。
5. 是否表达两个或多个稳定对象之间的联系。是则使用关系或二次特征。
6. 是否表达为什么变化以及证据链。是则使用因果结构。
7. 是否表达目标、授权承接或可执行能力。是则分别使用需求、任务或方法。
8. 是否是关系无法表达、但所属领域必须长期保存的原始值。是则使用该领域唯一的类型化数据记录。
9. 是否只为加速查询。是则使用可重建索引或非权威缓存。
10. 是否只为显示、诊断或局部计算。是则只能作为非权威短生命周期辅助结构。

已有结构能够表达时，禁止新增平行业务字段或第二套事实结构。

## 3. 节点字段与领域类型化记录边界

节点记录的通用字段固定为：

```text
节点编号
稳定主键
节点类型
版本号
记录状态
创建序号
```

节点通用字段不得扩张为领域属性、拓扑锚点、通用值槽、外部协议字段、缓存或业务结论。正式拓扑进入关系；关系无法表达的必要原始值进入所属领域的类型化数据记录。类型化记录必须具名绑定所属对象、值类型、写入方、读取方、版本和恢复合同，不得以无类型数组、槽位号或通用联合容器承载跨领域语义。

表达对象属性、观测事实、统计事实、治理状态、判断结论、任务结果、方法结果、完成结论、风险结论或因果解释的字段，不得因为高频访问或实现方便进入节点通用字段或平行便利记录。

## 4. 特征类型准入

新增特征类型必须说明：

```text
特征类型身份
适用宿主
值域
合法写入方
来源材料或来源动态
当前性与版本
验证方式
与已有特征的区别
```

新增“是否、已经、可以、成功、明确、打开、连接、完成”等二态特征前，必须判断它是否是多个细粒度特征的综合结果。若是综合结果，必须展开组成特征、各自值域、收束规则、来源和规则版本；不得直接持久化为裸 bool。

方法向任务带入的开放业务特征必须由方法正式声明，并保留来源方法、内容版本、角色、激活原因、筹办轮次、是否必需和当前性；不得把所有方法业务特征封闭成任务全局枚举。

## 5. 新增结构准入

用于承载业务语义、状态集合、特征组合、需求准入、任务结果、方法结果、因果缺口或安全判断的持久结构默认禁止新增。

确需新增时，必须证明：

1. 现有节点、关系、领域类型化记录、索引、特征、特征值、状态、动态、因果、需求、任务和方法无法表达。
2. 新结构具有唯一明确职责，不复制已有权威事实。
3. 写入方、读取方、生命周期和事务发布边界已经明确。
4. 删除、失效、恢复和版本变化不会形成双重权威。
5. 对应对象根规范和子规范已经先行允许。

证明不足时不得新增。

## 5.1 新增根对象与根规范缺口准入

只有候选概念同时具有现有根对象无法表达的独立稳定身份、合法事实来源和生命周期，才可能形成新的根对象或根规范缺口。新增字段、代码模块、页面、算法、报告、缓存、协议消息、阶段名称或只读投影不自动构成根对象。

提出新的根对象前必须：

1. 复核 0050、全部现行根规范和直接相邻主题子规范。
2. 按本规范第 2 节与第 5 节证明现有节点、关系、领域类型化记录、索引、特征、特征值、状态、动态、因果、需求、任务和方法均无法承载。
3. 明确候选对象的定义、稳定身份、本项目结构承载、合法事实来源、写入方、读取方、生命周期、当前性、相邻边界和原则性禁止事项。
4. 证明把它作为既有对象的特征、状态、动态、关系、角色、模板或非权威辅助结构会产生明确语义错误，而不是实现不便。
5. 先完成冲突裁决和编号预留，再形成根规范正文并登记到 `规范/规范目录.md`；编号预留和候选清单本身不取得正式规范身份。

任一项不足时，应归入现有根对象或主题子规范，不得新增根规范。

## 6. 非权威辅助结构

局部运行期计算包、只读快照、显示投影、诊断结果、统计缓存和序列化中间结构可以存在，但必须同时满足：

```text
显式标注非权威
可从正式结构重建
不得反向写业务事实
不得跨域成为机器裁决依据
清空或过期不改变业务状态
```

## 7. 变更门禁

字段、结构或特征类型的新增、删除、改名和语义变化，必须先修订对应根规范与子规范，再进入机器实现。代码现状、性能需要、控制面板显示和数据库投影不得单独构成准入依据。
