# 鱼巢任务筹办与方法执行逻辑迁移规范

日期：2026-07-20

状态：正式规范 v0.1 / 第一批无冲突逻辑已迁入 / 三组源冲突分支门控 / 未改 C++

## 1. 定位

本规范把鱼巢正式任务、方法和内部循环规范中的业务逻辑迁入海中鱼巣。迁移只替换结构承载，不改变原有角色、顺序、条件、状态或失败语义。

正式源文件以 `规范/鱼巢正式规范迁移目录.md` 第 3 章为准，其中 `3100` 固定需求目标与满足证据上级边界，`5120` 固定单需求、单任务、单目标和多结果归类。旧代码、旧字段和旧函数只作结构映射证据。

## 2. 固定角色和权责

1. 需求主信息承载目标宿主、目标特征、目标状态、来源、任务承接和需求树治理；获取途径、日志摘要、显示文本和自由文本不得成为目标语义权威。
2. 需求满足必须通过实际状态变化、结算证据和任务反馈链判断，不得由任务返回码、线程状态或显示结果直接宣布。
3. 任务是需求的过程承接壳；一个任务只承接一个需求和一个目标特征类型，任务目标来自对应需求，任务不得另建第二套业务目标或多目标包。
4. 任务目标宿主、目标特征类型、当前状态、目标状态和满足关系必须分别来自对应需求的同义结构；任务缓存不得成为第二套目标定义。
5. 任务筹办和任务执行是任务管理生命周期普通函数，不注册为方法，不作为动作来源，也不得成为为解决自身而创建的子任务。
6. 任务管理侧负责任务筹办、执行编排和任务状态推进；自我线程只提供需求治理、优先级、生命周期事件和真实动作执行前许可，不得裸写任务状态。
7. 线程只是调度和执行载体，不是方法动作动态、状态事实或任务完成事实的来源。
8. 任务筹办不得内置安全、服务、观察、外设或学习等领域专用前置规则。前置关系只能来自需求目标、需求侧条件、因果、临时因果视图、方法条件、条件结果对或已入树子需求。
9. 任务筹办不得直接改需求树、递归执行子任务、执行学习链、直接执行方法或替父需求结算。

## 3. 任务筹办固定顺序

筹办顺序必须保持：

```text
读取任务及来源需求的正式目标语义
-> 判断任务是否需要筹办
-> 分开判断当前结构形态与目标语义
-> 复合目标优先分解；路径未明优先读取因果 / 条件路径
-> 只有普通可任务化叶子或必须直接执行的原子任务进入方法召回
-> 按方法结果能力完整遍历并填充候选集
-> 从候选集中选择当前方法
-> 复核方法状态、条件、结果、参数、禁止项和动作入口
-> 返回结构化就绪、等待、缺口、挂起或拒绝材料
```

不得因为当前没有子节点就把任务直接解释成普通叶子。AND 组、OR 组、方法路径组、因果子链组织节点和结构性缺口，必须先按目标语义展开或等待对应事实。

## 4. 方法召回与选择

1. 方法头只负责大范围召回，不直接证明可执行。
2. 方法查找的权威语义是按结果能力形成候选；方法名、动作名、函数名、显示文本和语素文本不得参与执行语义裁决。
3. 候选集必须先完整填充，当前方法只能从候选集中选择。不得边遍历边冻结首个命中，也不得用通用特征写入原语或学习方法硬编码兜底。
4. 首要排序依据是方法结果与任务目标的一致程度：目标宿主、目标特征类型、目标状态 / 值、满足关系和结果方向越一致越优先。
5. 结果一致程度相同时，再比较当前条件、参数绑定、禁止项、风险、副作用、预计效率、耗时、资源和事实样本。
6. 结果能力命中与当前可执行性是两个判断：条件或参数暂不满足可以使候选当前不可执行，但不能把结果不匹配的方法提升为更优选择。
7. 方法执行前必须以条件节点、结果节点、条件结果对、方法虚拟存在、参数绑定和动作入口复核，不得只凭方法头、方法投影或方法名称执行。

## 5. 筹办和执行回执

筹办不得只返回“找到 / 没找到”，执行不得只返回“成功 / 失败”。结构化材料至少要能区分：

```text
已就绪
等待子需求 / 子任务或外部事实
无路径 / 无既有候选
候选方法不可用
缺条件 / 参数 / 证据
方法结构或动作入口缺口
目标或来源材料非法
执行条件变化
执行产生可比较结果 / 无可比较结果
禁止结果、附带结果和风险结果
内部结构矛盾
```

无目标、目标宿主不可解析、目标特征缺失、来源需求非法、提交入口缺失或任务账无法写回，不得伪装成普通可生长缺口；应结构化拒绝或追根因。

筹办、执行和回执同步不得直接修改需求树。可生长缺口只形成带明确目标宿主和特征类型的材料，由正式需求提交入口裁决是否创建或复用子需求。

## 6. 执行与许可

1. 真实动作开始前，必须取得绑定当前工作项的一次性执行许可；工作线程未结束前，同一工作项不得重复申请。
2. 该许可只回答“本工作项当前能否开始真实动作”，不得替代任务筹办结果、任务状态、完成、失败、待重筹办或结算裁决。
3. 方法执行必须发生在任务授权、任务承接和任务选择方法均成立之后。
4. 执行必须重新复核当前任务、当前选择、方法登记 / 生命周期、条件、参数、动作入口和禁止项；过期材料不得继续执行。
5. 动作执行后先形成状态和动态事实，再由因果和结算路径消费。线程、日志或返回码不得冒充动作来源。

## 7. 完成、回流和结算

1. 任务完成只由任务自身目标状态和实际结果状态的可比较关系裁决。
2. 方法调用成功、结果比之前更好、日志显示成功或来源需求已经变化，都不能单独证明当前任务完成。
3. 实际结果必须区分当前任务目标结果、自我需要附带结果、自我不需要附带结果和自我禁止结果；四类事实分别入账，不得合并成单一布尔结果。
4. 动作执行成功但没有产生可比较目标结果时，只能记录“动作执行成功无可比较结果”及相应动作事实、推进或缺口，不得标记任务完成，不得更新目标符合 / 不符合账。
5. 当前任务目标结果必须继续比较结果值与目标值；固定值优先于区间，区间优先于方向，不得把“方向上变好”降格解释为固定目标已经达成。
6. 自我需要附带结果只进入其命中需求的账；自我不需要附带结果不触发当前需求满足；禁止结果和治理风险进入独立风险账，均不得混入当前任务目标完成统计。
7. 附带结果不能替代目标成功；目标不达成的方法，即使附带结果很多，也不能作为当前任务的有效方法。
8. 子任务完成不等于父任务完成。子任务对应的阻塞点形成可承接结果或本轮承接收束后，只触发父任务重新筹办 / 收束判断。
9. 父任务完成仍必须回到父任务自身目标和实际结果比较；历史子需求不得因为仍留在树上阻止父任务重筹办。
10. 父需求满足与价值结算由需求治理路径在吸收任务结果后独立裁决，不由任务筹办、任务执行或工作线程直接结算。

## 8. 任务级筹办并发门禁

在第 3 章业务筹办开始前，必须按任务稳定身份取得唯一筹办执行权：

```text
同一任务：同一时刻最多一条路径进入目标语义读取、方法召回、选择和冻结。
取得失败：在召回前正常退出，不参与候选、选择或冻结。
不同任务：允许并发，即使来自同一根需求或选择同一个可复用方法。
同一任务两条路径均越过门禁：内部并发规则破坏，必须追根因。
```

该门禁不改变第 3—7 章的业务顺序和分支，也不等同于第 6 章真实动作执行前的一次性工作项许可。具体承载结构留给后继结构设计，但不得持久化裸线程编号或把日志、锁对象本身当作任务身份。

## 10. 源规范冲突门控

### FISH-SPEC-CONFLICT-001：无候选时是否创建方法头

`5120` 第 6 条、`5210`、`5250` 部分条款和 `8200` 要求立即创建面向目标的方法头；`5230`、`5240` 及 `5250` 另一些条款明确禁止自动创建占位方法头，只允许挂起。当前不得实施这条分支。

### FISH-SPEC-CONFLICT-002：筹办回执写入权

`5230` 把候选、当前方法和执行回执同步定义为任务管理生命周期普通代码；`8200` 要求任务域提交本能方法写入任务状态、当前方法和候选集。当前只能继承两侧都不冲突的“线程不得裸写、状态推进必须正式提交、回执必须结构化”；具体写入分界冻结。

### FISH-SPEC-CONFLICT-003：执行期结构缺口

`5230` 把执行期才发现缺方法、缺动作入口、缺条件结果对视为筹办漏检 / 逻辑错误；`5300` 允许执行阶段输出对应原子需求。当前执行期结构缺口分支冻结；正常当前性变化、参数过期、条件变化和输出不足仍按无冲突规则处理。

## 11. #315 重做边界

`#315 v0.7` 及 A11 的“两线程都进入同一任务筹办并最终都返回完整冻结”不符合第 8 章，立即撤销。

后继 #315 设计至少必须证明：

```text
只有一条同任务路径能进入方法召回。
第二条路径在召回前返回筹办占用中或等价结构化拒绝。
第二条路径零候选、零选择、零冻结新增。
不同任务可以同时进入召回并选择同一方法。
鱼巢正式规范的任务形态判断、完整候选填充、选择排序和回执分类未被改变。
三组源冲突分支没有被 #315 偷渡裁决。
```
