# 7210 子规范 信息入口类型与信息存储树路由

编号：7210
日期：20260721
状态：正式子规范

## 1. 定位

本规范规定人类词性、信息入口类型、对象角色、领域存储入口和上下文路由之间的职责边界。

直接上级规范：2210。

直接依赖规范：2200。

相关边界规范：7200。

## 2. 四层分工

```text
人类词性
-> 把自然语言变成语言结构候选

信息入口类型
-> 把已理解候选变成项目内部对象入口

目标对象角色 / 主信息类型
-> 决定对象采用何种正式结构

领域存储路由
-> 决定由哪个领域服务、根和场景承载
```

人类词性负责候选，上下文负责消歧，信息入口类型负责路由，对象领域负责正式节点和事实。

## 3. 人类词性

名词、专有名词、动词、形容词、副词、介词、连词、数词、量词和句法标注都属于语言数字化结果。它们不直接决定：

- 节点属于哪个领域；
- 对象是概念还是实例；
- 对象是否为需求、任务、方法或世界事实；
- 对象采用哪种主信息结构；
- 对象是否可以正式发布。

外部分词标签必须先归一为受控中文人类词性值。

## 4. 信息入口类型

信息入口类型至少明确：

- 目标对象角色；
- 允许的主信息或领域对象类型；
- 默认领域入口、根抽象或场景；
- 是否为概念 / 模板入口或实例入口；
- 是否要求主体、时间、场景、任务、方法或关系上下文；
- 允许的对应信息和概念追溯角色；
- 路由规则版本。

入口类型可以包括存在概念、存在实例、特征模板、特征实例、状态模板、状态实例、动态模板、动态实例、二次特征、因果、场景、需求、任务、方法和语言记录等受控角色。只有系统已经明确需要独立机器对象时才新增入口类型。

## 5. 入口定义表

路由规则必须由统一入口定义表或等价强类型规则承载，不能散落在字符串判断、词性分支或显示层分支中。

每项定义至少包含入口类型、常见人类词性候选、目标对象角色、领域入口、默认根、上下文要求、概念 / 实例属性和版本。

入口定义表只产生合法路由候选，不直接创建对象，也不证明语言材料已经理解。

## 6. 人类词性到入口候选

人类词性和信息入口类型是一对多候选关系。例如：

| 人类词性 | 可能入口 |
| --- | --- |
| 名词 | 存在概念、特征模板、状态模板、场景概念、任务概念、方法概念 |
| 专有名词 | 存在实例、场景实例、任务实例、方法运行实例 |
| 动词 | 动态模板、动态实例、方法概念 |
| 形容词 | 特征模板、状态模板、特征实例 |
| 副词 | 二次特征、修饰关系 |
| 介词 | 关系模板或关系实例 |
| 连词 | 逻辑连接、因果或语言记录 |
| 数词 / 量词 | 数值、单位或特征值辅助候选 |

裸词通常偏向概念或模板；带主体、时间、场景、任务、关系和值绑定的表达才可能进入实例入口。该规律只形成候选，不能替代上下文确认。

## 7. 上下文消歧

消歧至少读取主体、时间、地点、场景、任务、方法、因果、关系、属性归属、值绑定和实例标识。

例如“位置”可以是特征模板；“B-1234 的位置”是实例特征入口；“当前位置为 101 机位”还包含主体特征和值对象关系。三者不能由同一词性节点直接混写。

同一词面可以形成多个语素入口。每个入口分别保存信息入口类型、上下文、对应信息和概念追溯。

## 8. 领域路由

概念和抽象对象进入相应领域的模板或抽象根；现实实例进入合法现实场景；内部推理或治理虚拟存在进入对应内部场景或领域治理入口；需求、任务和方法进入各自领域根；原始语言材料进入语言记录结构；临时解析信号不发布正式对象。

海中鱼巣中的“存储树”是领域根、场景归属、具名关系和索引的逻辑路由，不要求所有逻辑层级都采用同一种物理树结构或裸指针字段。

任务、方法和需求不是基础信息节点时，仍使用对应领域对象稳定句柄和具名关系，不得塞入“对应基础信息”单值字段。

## 9. 已理解门禁

信息入口类型只路由已经理解的入口。只有分词、人类词性和短语候选而未绑定内部对象时，必须保持未定义入口或语言候选状态。

概念入口必须绑定概念对象；实例入口必须绑定唯一实例对象并可追溯概念。路由表不裁决现实存在确认、任务满足、方法可执行或需求结算。

## 10. 标准流程

```text
接收自然语言
-> 形成语言记录、分词和人类词性候选
-> 读取上下文并生成信息入口候选
-> 确认已理解目标对象角色
-> 选择信息入口类型和路由版本
-> 请求目标领域查找或创建对象
-> 建立词、入口、对应信息和概念追溯关系
-> 完整读回并发布
```

任何一步证据不足都停留在语言候选层，不得先创建对象再靠事后扫描修正。

## 11. 词到词性的只读路由投影

语素服务必须提供不含仓库写入能力的值式材料，按以下结构完整读回：

```text
词身份与规范化词面
-> 受控人类词性身份与规范名称
-> 语素入口身份、信息入口类型、对应信息组、概念追溯组和路由版本
```

同一词可以出现多个词性，同一词性可以出现多个入口；投影必须保留完整集合和稳定身份，不得因分组、名称相同或显示方便而合并。规范名称用于人读，不能替代词性身份、入口身份或关系证据。

## 12. 禁止规则

1. 不得用人类词性直接决定节点类型、根或场景。
2. 不得把信息入口类型与人类词性合并为一个枚举或字段。
3. 不得把未理解材料按默认表直接落入存在、需求、任务或方法领域。
4. 不得用显示名称、文本长度或外部词性编码路由。
5. 不得把不同领域和生命周期对象混入一个裸引用字段。
6. 不得让入口定义表替代对象领域准入和事实裁决。

## 13. 验证边界

验证至少覆盖多词性候选、上下文消歧、多入口、概念 / 实例分离、领域路由、未理解拒绝和对象领域二次准入。
