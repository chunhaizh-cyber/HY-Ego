# 5100 子规范 需求下级规则

编号：5100
日期：20260720
状态：正式子规范

## 1. 定位

本规范展开需求的单目标、目标比较、角色、来源和当前满足合同。

直接上级规范：3100。

直接依赖规范：4130。

相关边界规范：3200、5110、5120。

## 2. 单目标合同

1. 一个需求只能绑定一个目标宿主、一个目标特征和一个目标状态合同。
2. 多目标必须由管理需求的直接子需求展开；每个子需求仍是单目标。
3. 一个任务只承接一个需求，所以任务权威目标仍是单目标；任务缓存不得成为第二套目标定义。
4. 逻辑组织语义不新增到需求节点通用字段或需求域平行记录。AND、OR、因果支撑和方法路径使用普通需求节点、强类型目标和正式路径关系表达。

## 3. 目标比较合同

1. 需求目标是目标状态。固定值、区间、枚举集合和方向关系由特征、状态或判断收束规则表达。
2. 需要方向比较时，正式比较关系只允许“大于、小于、等于”三种原子关系；大于等于、小于等于和可接受集合由组合规则表达。
3. 建立、刷新、取得、确认、明确、新鲜、改善、避免、补齐和验证不是比较关系，只能落入目标特征、目标值、生成类型、方法语义或派生原因。
4. 枚举目标只能用等于或等于集合，并且值必须属于特征定义值域。
5. 需求领域不直接比较原始值；它调用特征、状态或判断收束服务获得具名裁决。

## 4. 角色与任务化

1. 管理、执行、逻辑组织、路径支撑等角色由目标语义、路径关系、承接方式、活动状态和特征集合共同成立。
2. 有无子节点只说明拓扑，不得单独决定角色或是否可任务化。
3. 管理和逻辑组织需求不得被当作普通执行需求直接任务化；必须先按其展开、路径选择、聚合或结算规则处理。
4. 单次方法查找只面向已经通过任务化闸门的具体执行需求，不面向多目标管理包。

## 5. 来源与当前满足

1. 自检、观察、任务筹办、执行结果、方法能力、因果反推和结算等获取途径只作为提交证据与来源关系，不改变入树后的目标语义。
2. 当前满足必须读取当前事实和目标状态实时裁决，并记录来源、时间和版本。
3. 活动承接截止、历史满足或历史结算不得被解释为永久满足。
4. 目标宿主是特征挂载主体，不是“整体被改变”的自由文本对象。
5. 服务根需求的宿主仍是自我；服务对象只能在具有直接服务触发事实的上下文子需求中出现。

## 6. 结构映射与验证

需求节点及稳定句柄承载身份；正式关系承载主体、宿主、场景、特征、状态、父子和路径；状态及判断收束结构承载目标和值比较；来源动态和结算关系承载过程证据。

验证必须证明目标唯一、关系端点合法、目标状态完整、角色不是仅由拓扑推断、当前满足来自正式比较合同。
