# 5120 子规范 需求任务单目标与方法多结果归类

编号：5120
日期：20260720
状态：正式子规范

## 1. 定位

本规范定义需求和任务单目标合同，以及方法执行产生多结果时的分类、完成和结算边界。

直接上级规范：3100、3200、3300。

直接依赖规范：无。

相关边界规范：5110、5230、5300。

## 2. 单目标传递

```text
需求目标宿主 -> 任务目标宿主
需求目标特征 -> 任务目标特征
需求当前状态材料 -> 任务起始比较材料
需求目标状态 -> 任务目标状态
需求满足关系 -> 任务完成比较合同
```

任务可以保存冻结快照，但不得扩展成多目标包。复合目标必须在需求树和任务承接前拆成多个单目标子需求。

参数未闭合不得通过复制父业务目标制造参数子需求；必须形成明确的参数绑定、输入取得或方法结构缺口目标。

## 3. 方法结果分类

方法输出的每个当前、可验证结果必须归入且只归入下列一类：

| 类别 | 判定 | 去向 |
| --- | --- | --- |
| 当前任务目标结果 | 与任务目标宿主、特征和目标状态可比较 | 当前任务完成、推进和结算判断 |
| 需要的附带结果 | 不属于当前目标，但命中另一个当前有效需求 | 对应需求自己的满足检查和账本 |
| 不需要的附带结果 | 不命中当前目标或其它有效需求，也不触犯禁止项 | 普通事实或观察材料 |
| 禁止结果 | 命中安全、禁止项或风险约束 | 独立治理风险账和方法选择约束 |

动作执行成功但没有可比较目标结果，只能记录为动作事实，不得更新目标满足、结算或目标结果符合统计。

## 4. 完成与结算

1. 当前目标结果必须继续按目标状态和满足关系比较，不能只按特征类型命中。
2. 固定目标优先于区间，区间优先于宽泛方向；“方向变好”不能替代目标达成。
3. 附带结果不能冒领当前任务权重，也不能弥补当前目标未达成。
4. 目标达成和禁止结果可以同时成立，二者不得压成一个布尔值；目标结算与治理风险分别处理。
5. 每个结果必须保留来源方法、动作动态、输出场景、版本和当前性。

## 5. 结构映射与验证

任务目标由任务—需求、任务—目标状态等正式关系承载；方法结果由结果节点、结果规格、输出状态和动态证据承载；结果分类由任务领域的强类型归类事实承载。

验证必须覆盖无结果、单结果、多附带结果、禁止结果、目标特征命中但目标值未达成、目标达成同时出现禁止结果等场景。
