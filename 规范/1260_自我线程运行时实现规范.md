# 1260 自我线程运行时实现规范

## 1. 目标

本文用于固定当前 `自我线程模块.ixx` 的运行时实现边界，回答以下问题：

- 自我线程当前到底承担哪些运行时职责
- 主循环、动作线程、学习线程和延迟写回如何协同
- 队列、锁、结构调整和学习回流应如何分层
- 哪些属于自我线程运行时，哪些不属于

本文对应当前工程实现：

- `自我线程模块.ixx`
- `任务执行类.ixx`
- `任务调度类.ixx`


## 2. 定位

当前 `自我线程类` 的定位应固定为：

`任务运行时总控 + 心跳调度器 + 动作线程拥有者 + 学习线程拥有者 + 低频事实整理器`

它不是：

- 根约束定义器
- 宿主监督线程
- 后台前端服务线程
- 相机或外设运行时


## 3. 当前运行时车道

当前实现中，自我线程至少包含以下几条运行时车道：

- `主消息/心跳车道`
  - 处理主队列
  - 驱动根任务就绪判断
  - 决定是否把任务送入动作队列或学习队列
- `动作队列车道`
  - 推进普通任务和管理任务
  - 驱动根任务筹办器推进一步
- `学习队列车道`
  - 推进常驻学习任务
  - 执行 `采样 -> 更新 -> 验证 -> 提交 / 回滚`
- `延迟任务事实车道`
  - 暂存热点路径产生的任务事实
  - 在低频路径统一整理并写回
- `根任务结构调整车道`
  - 串行保护根任务结构调整与任务返回消息落地

这几条车道必须理解成：

- 同属一个运行时
- 但职责必须分层
- 不得因为“都在自我线程里”就互相直接越权改结构


## 4. 主循环职责

当前主循环的职责应固定为：

- 处理主队列消息
- 维持心跳驱动
- 刷新根任务的就绪与门控状态
- 触发动作线程、学习线程启动
- 协调低频整理与学习唤醒

主循环不应直接承担：

- 叶子步骤的具体执行
- 学习阶段内部每一步的细节
- 宿主重启监督


## 4.1 初始化阶段（新增硬约束）

`自我线程类::自我初始化()` 必须承担以下顺序化动作：

1. 初始化自我与基础特征承载。
2. 统一注册默认本能动作集（含方法首节点创建/复用与方法信息回填）。
3. 初始化初始需求与任务。
4. 刷新初始化完成状态并进入运行态。

必须固定：

1. 本能函数“首次注册”属于初始化阶段职责，不得延迟到任务执行/学习执行/周期维护中再补做。
2. 注册完成是“可执行前置条件”，未完成注册不得标记运行时初始化完成。
3. 注册不等于执行：初始化阶段不要求逐个执行本能函数本体。


## 5. 动作线程职责

当前动作线程入口的职责应固定为：

- 从执行队列中取任务
- 将任务置为 `执行中`
- 在结构调整锁保护下调用 `根任务筹办器_.推进一步`
- 接收 `结构_任务返回消息`
- 驱动需求状态刷新、叶子需求结算和调度器提交流程
- 对 `生成子任务` 的返回消息做直接入队

必须固定：

1. 动作线程只消费 `执行队列`。
2. 动作线程的结构改动入口，统一收束到 `推进一步 + 返回消息`。
3. 动作线程可以把事实排队到延迟写回车道，但不应在热点路径里做重整理。


## 6. 学习线程职责

当前学习线程的职责应固定为：

- 只消费 `学习队列`
- 特判常驻 `学习任务节点_`
- 推进学习阶段机
- 在更新阶段调用 `私有_尝试学习一步`
- 在提交阶段唤醒等待学习的方法和任务

当前学习线程应理解为：

- 已经能真正执行
- 但仍属于专用学习工作流
- 还不是完全并入普通叶子步骤执行器的最终形态

因此必须固定：

1. 学习线程内部的阶段推进，是学习专用运行时逻辑。
2. 学习线程的产出，主要是方法树更新、候选补全和等待任务唤醒。
3. 学习线程不直接替代任务调度器。


## 7. 延迟事实与低频整理

当前实现已经存在“动作线程只记最小事实，扩展整理移到低频路径”的口径。

因此应固定：

- 热路径允许只写最小回执、关键事实和最小方向特征
- 复杂投影、对齐、账本补齐和大规模同步，应由低频整理路径完成

延迟事实队列的职责应固定为：

- 承接热点路径不宜立刻落地的任务事实
- 在更低争用的时间点统一冲刷到任务虚拟存在


## 8. 本模块锁与顺序

当前模块至少存在以下锁：

- `故障锁_`
- `mtx_`
- `执行队列锁_`
- `学习队列锁_`
- `延迟任务事实锁_`
- `根任务结构调整锁_`

本模块的默认规则应固定为：

- 默认单锁化
- 能不双持就不双持
- `故障锁_` 视为独立故障账本锁，默认不与其它锁同持

若确实无法避免多锁同持，本模块的局部顺序应固定为：

1. `mtx_`
2. `执行队列锁_ / 学习队列锁_`
3. `延迟任务事实锁_`
4. `根任务结构调整锁_`

必须固定：

1. 队列等待时，只允许持有对应队列锁。
2. 持有 `根任务结构调整锁_` 时，不得再去等待条件变量。
3. `try_to_lock` 只能作为受限的拥塞退避手段，不得作为主正确性机制。


## 9. 启停与拥有关系

当前自我线程必须服从以下拥有关系：

- `自治宿主` 拥有 `自我线程`
- `自我线程` 拥有 `动作线程` 与 `学习线程`

因此必须固定：

1. 自我线程只负责启动和回收自己的直属子线程。
2. 动作线程和学习线程不得反向控制宿主。
3. 停机应遵守 [1230_多线程嵌套启停管理规范.md](D:\鱼巢\规范\1230_多线程嵌套启停管理规范.md)。


## 10. 自我线程的硬约束

1. 主循环负责“决定与派发”，动作线程和学习线程负责“消费与推进”。
2. 任何任务结构改动，都应尽量通过统一的返回消息和结构调整入口落地。
3. 热路径事实回写应最小化，重整理必须延后。
4. 学习线程是专用学习运行时，不得误当成普通动作线程的别名。
5. 任何队列等待都必须带谓词，并遵守 [1240_线程内部锁与条件变量使用规范.md](D:\鱼巢\规范\1240_线程内部锁与条件变量使用规范.md)。
6. 本模块的本地锁顺序必须可见，不能依赖“写代码的人自己记住”。
7. 线程生命周期状态变化和关键运行时状态变化必须形成日志。


## 11. 不负责的事项

本文不负责：

- 宿主监督与进程壳层协议
- IPC 快照格式与命令通道
- 相机设备采集循环
- 元层学习写回的实体关系定义
