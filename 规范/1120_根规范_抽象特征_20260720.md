# 1120 根规范 抽象特征

编号：1120
日期：20260720
状态：正式根规范

## 1. 根定位

抽象特征是特征类型、值域、叶子或复合形态以及抽象特征结构的权威本体。正式特征类型必须由稳定的特征定义身份及其结构关系表达，不是裸词面、语素入口、硬编码枚举或显示文本。

状态抽象统一回到抽象特征结构；不得另建与抽象特征平行的权威抽象状态树。

## 2. 本项目承载

海中鱼巣以“特征定义节点”承载抽象特征：

```text
节点类型为特征的定义节点及稳定句柄
+ 唯一非零主键和主信息身份
+ 抽象特征之间的普通结构关系或具名组成关系
+ 值域、模板条件和语义入口的正式关系
+ 用于定位的索引
```

特征定义节点与实例特征槽位可以复用 `节点类型::特征`，但必须由结构角色区分：定义节点不得带有“槽位到特征定义”的模板关系；实例槽位必须具有唯一宿主和唯一特征定义。节点类型相同不等于业务身份相同。

语素只通过正式信息关系充当名称入口、查找入口或初始化种子；机器判断、需求匹配、方法能力匹配和特征值解释必须回读特征定义节点身份。

## 3. 事实与边界

1. 抽象特征可以表达值域、复合组成、抽象上下位关系和状态模板条件。
2. 抽象特征不得承载实例当前值、业务完成结论、安全或服务画像、运行期评估结果。
3. 叶子与复合形态必须由正式结构关系成立，不得由名称后缀、显示类型或注释推断。
4. 名称入口和类型入口即使暂时指向同一语素，也只能解释为兼容入口，不证明正式类型定义已经完成。
5. 不得新增硬编码“特征类型总枚举”取代特征定义节点；默认初始化可以显式创建定义节点，但结果必须进入正式抽象特征结构。

## 4. 写入、读取与验证

抽象特征只能由特征领域服务创建、绑定和修改；数据操作层在不透明结构写入会话中完成节点、主信息、关系和索引的原子发布。特征值服务、高级业务服务、外设、显示层和普通调用方不得直接创建或改写抽象特征。

读取方必须通过特征领域服务取得稳定定义身份、结构关系和值域材料。验证至少回读：节点当前有效、类型为特征、主信息有效、主键唯一、定义角色成立、全部关系端点与版本当前。

## 5. 生命周期与当前性

抽象特征身份长期存在；定义内容变化必须保留版本和来源。语素入口变化、索引重建或显示名称变化不得改变抽象特征身份。定义失效或替换后，既有实例槽位和历史事实仍须能够回查原定义版本。
