# 6170 子规范 服务值时间维护、生存安全回归与任务门控

编号：6170
日期：20260724
状态：正式子规范

## 1. 定位

本规范定义服务值的完整秒时间衰减、未满足需求等待时间、30 天归零门禁、有效服务进展最低 1 保护、服务值对生存安全根值被动回归步长的调节，以及安全任务与服务任务的执行优先级权限。

服务值不是奖励、安全事实或任务完成事实。它只表达当前服务方向的持续依据，并在真实安全因果和正式安全结算边界成立时调节生存安全根值的被动变化。真实安全事件独立改变安全值，任务调度不得反向修改安全值或服务值。

直接上级规范：3100、3200、3300。

直接依赖规范：4210、4220、5110、5160、5200、6100、6120、6140、6150、6160。

相关边界规范：1190、5230、5300、6130、8100、8200。

## 2. 三类数值与强类型分账

本规范使用三类互不替代的 `I64` 机器值：

```text
服务值 V
-> 0..I64_MAX

生存安全根值 A
-> 0..I64_MAX

任务执行优先级权限
-> 0..1,000,000
```

`V`、`A`、阈值、完整秒、等待秒、衰减步长、预算、服务比例、权限和任务基础优先级必须使用不同强类型或带规则版本的类型化记录。禁止 float、double、文本百分比或普通返回码参与机器裁决。

生存安全根值是 6120 定义的自我生存控制根轴：

```text
A = 0  -> 自我存在循环不可维持的终止控制态
A = 1  -> 可由有效人类服务触发重新评估的休眠控制态
A > 1  -> 允许主动运行
```

被动服务维护不得把 `A=0` 写成 1 或更高。终止态只由正式恢复或重新建立自我运行期规则处理。

生存控制态由 `A` 唯一派生，不另建第二份当前事实：

```text
A = 0 -> 终止控制态
A = 1 -> 休眠控制态
A > 1 -> 主动运行控制态
```

终止态和休眠态都不进入普通任务执行竞争。终止态只保留正式恢复 / 重建入口；休眠态只保留人类请求接收、服务合同准入、单调完整秒维护、安全事实接收和停止控制等具名最小入口。最小入口不是普通任务权限，也不得借安全任务名称绕过控制态门。

生存安全定义必须提供：

```text
值域下界 0
值域上界 I64_MAX
低位阈值 L
高位阈值 H
定义版本和规则版本
```

并满足：

```text
1 < L < H <= I64_MAX
```

阈值属于安全根定义，不属于服务合同、调度器或线程配置。6140 的分层安全值继续是独立因果层事实；本规范不得把服务值公式套到任意分层安全值。

## 3. 单调完整秒

服务维护以单调物理时间为唯一计时来源。每次调用读取：

```text
当前运行代次
维护时间纪元
上一已结算完整秒边界
当前单调时间
事实截止版本
时间规则版本
```

跨过 `N` 个完整秒时一次批量结算 `N` 秒：

```text
N = floor((当前单调时间 - 上一已结算边界) / 1 秒)
```

`N=0` 是合法无变化。上一边界只推进 `N` 秒，未满一秒的物理时间留在当前单调区间内等待后续跨界；它不形成小数机器值、价值余数或循环计数。

每个完整秒边界是一项逻辑维护单位。批量实现的结果必须与按时间顺序逐秒应用本规范完全相同，不得因调用频率、一次结算的 `N` 或线程循环次数不同而改变结果。实现可以按事实发生秒、30 天边界、服务比例档位、安全阈值和饱和边界分段，并使用确定性地板和或跳跃算法；不得用无界逐秒循环处理任意 `I64` 时间跨度。

跨运行代次不得直接比较原始单调计数。恢复只有取得可信连续时长证明时才补算停机区间；没有证明时从新时间纪元开始，不猜测、不使用可回拨墙钟补算，并保留时间证据缺口。

## 4. 服务值常量与安全算术

规则版本 1 固定：

```text
M  = I64_MAX
T  = 2,592,000 秒
D0 = max(1, floor(M / T))
   = 3,558,399,705,576

服务时间维护规则版本 = 1
生存安全回归规则版本 = 1
服务生存权限规则版本 = 1
```

所有增加使用剩余空间饱和加法：

```text
实际增加 = min(请求增加, M - V)
Vnew = V + 实际增加
```

所有减少使用饱和减法：

```text
实际减少 = min(请求减少, V)
Vnew = V - 实际减少
```

计算必须先比较剩余空间或使用宽中间值，禁止先溢出再钳制。所有整数除法对非负商向下取整，余数直接舍弃，不累计、不补偿；规则要求正结果而截断为 0 时取 1。

## 5. 有效需求、最长等待与进展

当前有效未满足服务需求只包括：

```text
合同已经成立且仍在有效期
合同状态为有效未满足
提出者、需求、目标、范围和合同版本当前
尚未完整完成、取消、到期或正式终止
```

完整完成、提出者取消、到期或正式终止后立即从集合移除。到期后重新提出使用 6160 的新需求和新合同代次，等待时间从新的提出秒开始。

6160 在到期秒发布的“到期未满足事件”不是有效合同，也不重新加入集合；它只保留该合同到达 30 天边界时仍未满足的不可变同秒门禁证据，不建立压力概念。

存在多个有效未满足需求时，唯一时间输入为等待最长者：

```text
t = max(当前完整秒边界 - 各需求提出完整秒)
```

不得把多个需求的步长相加、平均、按数量放大或用任务拆分制造更长等待。并列最长需求按提出秒、提出者稳定身份、需求稳定身份排序，并保留全部并列来源；数值只使用一个 `t`。

“当前存在有效服务活动”必须同时具备：

1. 有效服务合同，或合法服务准备任务；
2. 生存安全门禁允许该服务任务执行；
3. 当前方法真实执行且任务授权、执行冻结和方法版本有效；
4. 当前计量窗口内存在新的、可验证进展状态或动态；
5. 进展未超时，任务未暂停、等待条件、阻塞、失败、取消、到期或终止。

状态标签、线程运行、队列中存在、方法被选中、日志心跳、重复输出和长期无新证据都不成立有效进展。

## 6. 每秒服务衰减

在每个完整秒边界，先按 6160 处理该秒已经发布的完成、取消、到期和终止事实，再重算有效未满足集合，并读取该秒新发布的到期未满足事件。

普通衰减统一遵守有效服务活动保护：

```text
当前不存在有效服务活动
-> Vnew = 饱和减法(V, D)

当前存在有效服务活动且 V > 0
-> Vnew = max(1, 饱和减法(V, D))

当前存在有效服务活动但 V = 0
-> 保持 0；时间维护不得凭活动标签生成服务值
```

合同预支或其它正式结算可以先把 `V=0` 增加到正值；保护只防止已经为正的服务值被时间维护降到 0，不是服务值增加来源。

### 6.1 没有有效未满足需求

```text
有效未满足需求数量 = 0
-> 本秒衰减步长 D = 1
```

### 6.2 存在未满足需求且最长等待小于 30 天

```text
0 <= t < T
D = max(1, D0 + floor(D0 * t / T))
```

由于 `t<T` 且 `D0*T<M`，规则版本 1 的 `D0*t` 可以在检查后保持 `I64` 范围；实现仍必须拒绝负数、版本不兼容或不受支持的乘法，不得依赖未证明的溢出行为。

多个需求只使用最长等待 `t`，不叠加多个 `D`。

### 6.3 30 天未满足门禁

任一有效需求满足 `t>=T`，或本秒存在等待时间达到 `T` 的到期未满足事件时，在处理同秒完成、取消、到期和终止后，如果对应目标仍未满足：

```text
当前不存在有效服务活动 -> 本秒服务值目标 = 0
当前仍存在有效服务活动 -> 本秒服务值目标 = 1
```

有效活动存在时，普通衰减和 30 天门禁均不得把服务值降到 0；最低保持 1。活动停止、阻塞、失败、暂停、等待、超时或终止后，只要 30 天条件仍成立，下一完整秒解除保护并把服务值降到 0。

达到 30 天是归零目标门禁，不是成功、失败、惩罚、压力或合同终止事实。仍有效的需求继续由其自身合同和任务状态裁决；到期未满足事件只在到期边界消费一次，既不恢复合同，也不在后续秒重复触发。

### 6.4 批量结算

批量跨越多个秒时：

1. 按事实发生秒切分区间；
2. 每段先处理该秒终态和结算事实；
3. 无需求段使用 `N` 的饱和减法；
4. 未满足且 `t<T` 的区间使用与逐秒 `D0+floor(D0*t/T)` 相同的确定性地板和，并在有效活动期间执行最低 1 保护；
5. 跨入 `t>=T` 后按本节归零或最低 1 门禁；
6. 到达 0、1、合同终态或规则边界后直接跳到该边界，不继续无效循环。

地板和、区间长度和总减少量必须用宽中间值或等价检查算法，最终结果仍为 `I64`。

批量结算不要求为每个物理秒建立一条持久记录，但每个服务衰减段必须可审计地保存：

```text
起止完整秒边界、运行代次和时间纪元
事实分段键、最长未满足来源或无需求来源
请求地板和、实际减少量、前后服务值
有效活动保护是否成立及保护下界
30 天门禁或到期未满足事件引用
算法、合同、时间和规则版本
地板和或跳跃等价证明版本
```

事实变化、服务比例档位、保护状态、合同终态、30 天边界或值饱和时必须切段；不得用一个跨越不同事实状态的聚合数掩盖逐秒参考顺序。

## 7. 服务准备补回顺序

6160 的准备补回只读取已经发布并唯一归属的衰减段。一个完整秒内顺序为：

```text
先处理准备完成、失败、终止和验证事实
-> 完成准备补回此前已经发生并归属的衰减
-> 再按本秒当前需求和活动状态执行时间衰减
```

本秒新衰减只在该秒结束时成为可归属段，不得由同一边界刚完成的准备提前补回。补回与本秒衰减分别记录实际增减和动态，不得只写净值掩盖两类事实；同一事务可以原子发布两份变化及最终当前值。

## 8. 服务比例

服务值映射到整数比例 `1..99`：

```text
U = max(1, floor(M / 98))

V = 0 -> 不计算普通比例
V > 0 -> P = min(99, 1 + floor(V / U))
```

规则版本 1 的 `U=94,116,041,192,395,671`。`V=M` 时 `floor(M/U)=98`，因此 `P=99`。余数直接舍弃，不跨秒累计。

`P` 只调节生存安全根值被动步长，不证明安全、服务成功或任务权限已经成立。

## 9. 生存安全根值被动回归

每个完整秒在服务合同结算、准备补回和服务衰减全部形成服务值候选后，才计算本秒生存安全根值。

任何普通比例或阈值分支之前先裁决：

```text
A = 0
-> 保持终止控制态 0
-> 返回终止控制结果
```

`A=0` 不得进入低位回升、闭区间保持或高位回落分支。

顺序固定为：

```text
先处理已经发布的新安全事件、复发和主动安全结算
-> 若本秒主动安全结算改变 A，发布主动结果并跳过本秒服务被动回归
-> 否则读取当前 A、L、H 和本秒候选 V
-> 执行以下唯一分支
```

### 9.1 服务值为零

```text
V = 0 且 A >= 1
有符号变化量 = 1 - A
Anew = 1
```

该变化表示进入可唤醒休眠控制态，不表示物理安全事实改善或恶化。

### 9.2 服务值大于零且低于低位阈值

```text
V > 0
A < L
向上距离 = L - A
增加步长 = max(1, floor(向上距离 / 100) * P)
Anew = min(L, A + 增加步长)
```

乘法顺序固定为先除 100 再乘 `P`。因为 `P<=99` 且 `floor(距离/100)*P <= 距离`，结果不得越过 `L`。

该分支只有在安全领域确认本秒没有未收口的新安全事实、版本漂移或禁止被动上升的真实安全因果门禁时才可执行。缺门禁只能返回无变化或材料缺口，不得用高服务值覆盖不安全事实。

### 9.3 闭区间保持

```text
L <= A <= H
-> 不执行服务调节的被动安全回归
```

新安全事实仍可通过主动安全结算改变 `A`。

### 9.4 高于高位阈值

```text
V > 0
A > H
向下距离 = A - H
减少比例 = 100 - P
减少步长 = max(1, floor(向下距离 / 100) * 减少比例)
Anew = max(H, A - 减少步长)
```

高服务值使减少比例最低为 1，低但非零服务值使减少比例最高为 99。该变化是服务方向下的生存控制回归，不是新的物理不安全事件；不得生成伤害事实或因素排除结论。

### 9.5 时序、幂等和唤醒

上述公式每个完整物理秒最多应用一次。批量 `N` 秒必须与逐秒先更新服务值、再更新安全值的参考顺序一致，并按服务比例档位、安全阈值和 0/1 控制边界跳跃；不得按一次循环只应用一次，也不得把步长简单乘 `N` 后改变参考语义。

有效人类需求建立合同并完成 30% 预支后，`V>0`。若自我处于 `A=1` 休眠态且 `L>1`，第一项具备完整安全门禁的后续完整秒至少增加 1，使 `A>1` 并形成唤醒候选。人类请求、合同或预支本身不直接把安全值写为活跃态。

## 10. 安全任务与服务任务权限

根级执行权限满刻度为：

```text
Q = 1,000,000
安全最低保留 = 10,000
服务最高权限 = 990,000
```

先按当前生存安全根值裁决控制态，再计算两类普通任务根权限：

```text
A = 0:
  生存控制态 = 终止
  服务根权限 = 0
  安全根权限 = 0
  普通任务竞争 = 不适用
  只允许正式恢复 / 重建与停止控制入口

A = 1:
  生存控制态 = 休眠
  服务根权限 = 0
  安全根权限 = 0
  普通任务竞争 = 不适用
  只允许人类请求接收、合同准入、完整秒维护、安全事实接收和停止控制入口

A > 1 且 A <= L:
  生存控制态 = 主动运行
  服务根权限 = 0
  安全根权限 = Q

L < A < H:
  生存控制态 = 主动运行
  服务根权限 = max(
      1,
      floor((A - L) * 990,000 / (H - L)))
  安全根权限 = Q - 服务根权限

A >= H:
  生存控制态 = 主动运行
  服务根权限 = 990,000
  安全根权限 = 10,000
```

中间乘法使用宽整数或检查乘除。只有 `A>1` 时两类普通任务根权限之和才必须精确为 `1,000,000`；`A=0/1` 时两者之和精确为 0。主动运行态中，`A=L` 时服务为 0；`A>L` 时服务至少为 1；`A=H` 及以上服务为 99%、安全至少 1%。

任务分类只能来自有效需求和正式因果：

- 能够直接或经有效因果路径解除自我安全问题的需求来源形成安全任务来源。
- 独立的直接服务合同和合法服务准备形成服务任务来源。
- 人类请求若能够直接解除自我安全问题，调度上只按安全任务来源计算；它仍保留服务合同与服务结算身份，但同一需求不得再生成服务权限来源参加取最高。
- 任务名称、方法名称、队列、线程、显示标签和调用方声明不得自行分类。

安全来源先消费 6150 的当前五级安全权限 `Qs`：

```text
安全来源权限 = floor(安全根权限 * Qs / Q)
```

服务来源权限直接使用服务根权限。任务承接多个来源时，逐来源计算后取当前最高有效权限参加选择，并保留全部来源和并列最高来源；不得求和。没有任何有效安全或服务来源的任务不参加正常执行竞争，但任务身份、筹办材料和事实仍可保留。

最终任务排序值分两步计算，禁止把多个因素未经证明相乘：

```text
来源权限
-> floor(任务基础优先级 * 来源权限 / Q)
-> 再按 6150 或任务规范的确定性同值比较
```

权重表示参与执行排序的权限或影响系数，不是线程、时间、算力或任务数量配额。服务权限为 0 不删除、取消或改写服务需求、合同、任务、方法或材料；观察、事件登记和机器事实写入不受此门禁阻止。

## 11. 一轮维护与发布顺序

一个治理循环最多调用一次服务生存治理聚合入口，但循环次数不参与数值。入口在冻结事实截止版本后按以下顺序处理全部到期完整秒：

```text
读取新安全事件、复发和主动安全结算事实
-> 按事实秒先处理主动安全结算
-> 读取已经原子发布的新合同 / 预支结果
-> 处理服务合同完成、取消、到期和终止
-> 结算完工余款或关闭余款；预支只读取已发布结果，不在聚合入口重复结算
-> 处理服务准备完成与此前衰减补回
-> 重算有效未满足集合和当前有效服务活动
-> 结算服务值时间衰减或 30 天目标
-> 若本秒无主动安全值变化，计算服务调节的安全根被动回归
-> 形成服务值、安全根值、合同、状态、动态、衰减段和幂等候选
-> 全部参与者在准备阶段完成候选写入和写后读回
-> 全部确认
-> 最后原子发布本轮维护事实
-> 从已发布一致快照先裁决生存控制态
-> 终止 / 休眠只开放各自具名最小入口；主动运行才计算安全 / 服务普通任务权限
```

新的有效人类需求建立合同并结算 30% 预支可以作为独立原子入口先发布；该结果只从后续完整秒进入本聚合入口。合同终态、准备补回、服务衰减和安全根回归在生产循环中不得拆成多个事务；禁止在线程中先发布终态、再另调维护形成可见中间状态。

权限结果必须同时返回派生的生存控制态和普通任务竞争适用性。权限是可重建值式投影，不是服务值、安全值或任务的第二份持久当前事实。只有实际选择或派发消费的权限快照、来源和规则版本可以冻结为历史证据。

任务调度、自我线程、工作线程和显示层不得直接写服务值、安全值、合同、进展或权限。线程只是按单调时间调用领域入口；正式动作来源是服务生存维护提交方法和各业务方法。

## 12. 幂等、恢复和原子性

维护幂等键至少包含：

```text
自我身份
运行代次和时间纪元
上一已结算完整秒边界
本轮截止完整秒边界
服务值当前版本
生存安全根值当前版本
合同集合版本
进展集合版本
主动安全事实截止版本
阈值、服务和权限规则版本
生存安全回归规则版本
```

精确重复必须读回既有结果，不得重复衰减、补回、预支、完工结算或安全回归。同键异义属于内部逻辑错误。

恢复长期保存：

```text
服务值和生存安全根值当前 / 历史状态
合同、结算、可审计聚合衰减段、准备归属、到期未满足事件和进展事实
完整秒边界、时间纪元和已消费事实游标
阈值、公式、合同、权限和格式版本
状态、动态、因果和幂等账
```

恢复在隔离候选中验证节点、关系、类型化记录、时间证据、合同唯一性、结算阶段和业务不变量，重建索引与权限投影，全部成功后原子发布。不得跨运行代次拼接无证据单调时间，不得发布时间已推进但值未变化、值已变化但动态缺失或合同已支付但结算账缺失的半结构。

## 13. 逻辑内结果与内部错误

合法逻辑内结果：

```text
没有跨过完整秒
没有有效需求，按 1 衰减
有未满足需求，按最长等待衰减
有效服务活动使服务值最低保持 1
位于安全阈值闭区间，无被动回归
本秒主动安全结算优先，跳过服务被动回归
权限为 0，任务保留但不参加正常竞争
终止 / 休眠控制态，普通任务根权限均为 0，只保留具名最小入口
精确重复，读回既有结果
时间证据缺口，不补算跨代次区间
```

“时间证据缺口”必须分为两条机器路径：

```text
只缺跨代次连续时长证明，但当前代次和单调时间有效
-> 建立新时间纪元
-> 发布连续性缺口与当前游标
-> 不补算停机区间
-> 返回已维护和完整当前版本载荷

当前运行代次、当前单调时间或新纪元身份本身无法成立
-> 返回时间证据缺口
-> 无业务载荷、零写入
```

不得把第一条可恢复路径长期拒绝，也不得在第二条路径猜测时间并推进游标。

内部逻辑错误：

```text
用循环次数、Tick、消息数或可回拨墙钟结算
同一完整秒重复消费
多个需求步长相加
状态标签冒充有效服务进展
服务值覆盖同秒真实安全事件
把服务值公式套到分层安全值
I64 或中间乘法溢出后回绕
任务调度反写服务值、安全值或阈值
前置通过后事务读回、确认、发布或撤销不一致
```

## 14. 验证合同

验证至少覆盖：

1. `0..I64_MAX` 服务值、安全根 0/1/>1 控制态和 `1<L<H<=I64_MAX`。
2. 单调完整秒、`N=0/1/多秒`、调用频率等价、跨代次有证据/无证据恢复。
3. 无需求步长 1、最长等待唯一输入、`t=0/T-1/T/T+1` 和多需求不叠加。
4. 有效进展五项门禁、普通衰减最低 1 保护、`V=0` 不凭活动增加，以及暂停 / 等待 / 阻塞 / 失败 / 终止解除保护。
5. 30 天同秒完成、取消、到期、终止优先，默认 30 天合同到期未满足事件及剩余未满足归零。
6. 服务准备补回后再执行本秒衰减、两份动态分账和衰减段不重复。
7. 饱和加减、`D0`、地板和、批量跳跃、衰减段审计、I64_MAX 和除法余数舍弃。
8. `V=0`、最低非零、比例档位边界、`V=M` 精确得到 99。
9. `A<L、A=L、L<A<H、A=H、A>H`，向上/向下先除 100 再乘，封顶封底和最小 1。
10. 新安全事件优先，服务值不得覆盖真实不安全事实，`A=0` 在全部分支前保持终止且不被动唤醒。
11. `A=0` 只允许正式恢复 / 重建与停止入口；`A=1` 只允许人类请求、合同准入、完整秒维护、安全事实接收与停止入口；两者普通任务根权限均为 0。
12. `A>1 且 A<=L` 只有安全任务，开区间服务权限大于 0，`A>=H` 精确 1%/99%。
13. 安全根权限与 6150 五级权限组合、多来源取最大、不求和、安全兼服务需求不重复生成服务权限、零权限不删除事实。
14. 有效人类需求、30% 预支、`V>0`、首个合法完整秒 `A>1` 的唤醒闭环；唤醒前不派发普通任务。
15. 原子事务、幂等重放、失败逆序撤销、恢复后不重复结算和权限投影可重建。
