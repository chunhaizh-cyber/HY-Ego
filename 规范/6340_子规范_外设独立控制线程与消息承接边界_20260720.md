# 6340 子规范 外设独立控制线程与消息承接边界

编号：6340
日期：20260724
状态：正式子规范

## 1. 定位

本规范规定每类外设、每个物理外设实例的独立控制域，外设命令、外设报告、任务域承接、事实提交和实时性舍弃边界。

直接上级规范：3200、3300

直接依赖规范：6330

相关边界规范：1160、1170、1180、1190、6320

## 2. 独立控制域

每个外设实例必须有独立控制域。每种外设应有独立外设控制线程或外设代理；高实时、高带宽、长时间占用或状态复杂的物理实例应独占 OS 线程。

低频简单外设可以共享线程池，但仍必须独立拥有：

- 外设稳定身份和驱动句柄；
- 命令队列和串行控制权；
- 状态机、状态缓存和错误状态；
- 报告队列、时效统计和丢弃统计；
- 可选私有学习或配置缓存。

“共享线程池”不等于“共享控制域”。同一物理外设不能被多个任务绕过控制域并发控制。

## 3. 线程职责分层

```text
外设控制域
-> 管理驱动、采集、控制、私有算法、连续状态和报告生产

任务域承接
-> 按任务、等待项、订阅项、优先级和时效匹配报告
-> 形成任务运行材料、结构化缺口或提交请求

领域正式提交
-> 裁决特征、状态、动态、存在关系和场景事实是否发布
```

外设线程负责“看、动、报”，任务域负责“接、匹配、分流”，相应领域服务负责“裁决并提交”。外设线程、任务管理线程和纯消息派生整理都不是世界事实或动作来源。

外设线程内部可以直接使用驱动适配器、状态机、缓存、队列、算法模块和私有学习模块。采集、校正、滤波、匹配、分割、轨迹规划、驱动重试和内部学习不得登记为自我可调用方法。

## 4. 外设命令合同

外设命令至少关联：

- 命令稳定身份；
- 来源任务和来源需求；
- 目标外设稳定句柄；
- 命令类型：控制、检查或使用；
- 目标宿主、目标特征类型、比较符和目标值；
- 优先级、截止时间和取消条件；
- 是否允许缓存结果；
- 参数材料及其版本。

比较符只允许大于、小于、等于。命令消息是线程通信对象，不是需求、任务目标、状态或世界事实；正式机器字段必须由强类型结构、关系和特征值承载。

控制类命令可以改变外设自身状态，但必须关联明确方法和动作动态。检查类命令原则上不启动、修复、切换主要模式或长期占用外设；需要改变主要工作状态时应形成控制类缺口。使用类命令取得结构化报告，不直接提交世界事实。

## 5. 外设报告合同

外设报告至少关联：

- 报告稳定身份和来源外设；
- 对应命令，可空；
- 来源材料、报告类型、逻辑供给产品类型、材料成熟度和发生时间；
- 输出特征集合、质量和适用范围；
- 配置版本、材料年龄和是否命中等待项；
- 错误、降级或不可观测原因。

通用报告类型可以包括控制回执、检查回执、使用回执、观察供给和错误报告。具体外设的逻辑供给产品和材料成熟度必须分别使用强类型字段，并服从该外设主题规范；双目相机的成熟度等级、产品分类和外显运行模式由 6350 唯一定义，本规范不重复枚举。

同一双目相机控制域的全部逻辑供给产品共用一个物理强类型报告队列。队列按 6350 给出的产品身份维护四个互斥、可重建的分类索引和只读视图；同一报告稳定身份只能进入一个产品索引，索引和视图不得复制材料、形成第二消费位置或独立承担机器事实。其它外设需要不同产品数时由其主题规范给出强类型产品集合，仍遵守“一个控制域一个物理报告队列、分类索引只作可重建定位投影”的原则。

报告、队列项、材料页和缓存都不是事实主体。它们只有经任务域承接并由合法领域入口提交后，才可以成为任务结果、特征、状态、动态或场景事实的来源材料。

## 6. 感知与作用外设

感知外设报告观察到的一阶特征材料和质量，不直接确认存在、关系或因果。作用外设可以直接改变目标存在或外设自身的一阶特征，但执行成功不等于任务目标达成。

作用闭环必须保持：

```text
动作前感知
-> 方法形成预测和允许误差
-> 作用外设执行
-> 动作后重新感知
-> 自我比较实际一阶特征
-> 计算关系二次特征
-> 形成归因候选或因果证据
```

相机、力传感器和编码器等只报告实际回读；关系、任务完成和因果由各自领域规则裁决。

## 7. 任务域承接

任务域是外设报告队列的唯一正式消费者。任务域承接至少执行：

1. 验证报告来源、完整性、版本和时效；
2. 匹配当前任务、等待项、订阅项和活跃需求；
3. 更新非权威最新报告缓存；
4. 按产品类型形成绑定任务和工作项的已承接外设材料引用、派生需求三件套、动作验证输入或结构化缺口；
5. 请求相应领域服务完成正式提交。

任务域不得直接改写存在、场景、特征、状态、动态、因果或价值事实。筹办与执行仍按任务域规范运行，外设报告不能绕过筹办成为当前方法或执行冻结。自我线程和普通任务工作线程不得绕过任务域直接读取报告队列；当前冻结方法只能读取本工作项已经承接的材料引用。

同一报告的一次承接以报告稳定身份作为唯一竞争键。消费窗口、任务、工作项、队列位置和消费代次只参与当前性、授权和绑定校验，不得与报告身份组成能够放宽唯一性的复合竞争键。任何两个窗口、任务或恢复代次都不得让同一报告稳定身份形成两个当前承接。

## 8. 实时性和舍弃

允许舍弃或只保留摘要的材料包括：

- 过期扫描变化报告；
- 重复报告和中间跟踪材料；
- 无等待项、订阅项或活跃需求命中的观察材料；
- 低优先级局部细分和配置学习样本。

不得静默舍弃：

- 安全事件；
- 当前任务所需回执；
- 状态跃迁；
- 跟踪目标丢失；
- 外设错误；
- 影响目标判断的结构化缺口。

处理策略和原因必须用强类型状态或枚举承载，例如正式承接、仅缓存、降级处理、摘要保留、已丢弃，以及过期、重复、无命中、队列积压等原因。说明文本只作人读诊断。

### 8.1 崩溃恢复与一次消费

物理报告队列和四类分类索引都是运行期传递结构，不作为长期权威队列快照保存，也不得在重启时按容器内存顺序原样恢复。需要跨重启保留的是能够重新准入的类型化报告材料和消费治理事实，至少包括：

```text
报告稳定身份、来源外设和来源材料引用
产品类型、成熟度、发生时间、TTL 和配置版本
是否属于不可静默舍弃类别
已经绑定的任务、工作项和消费窗口（如有）
消费状态、消费版本、生产代次、消费代次和处理结果
```

安全事件、当前任务所需回执、状态跃迁、跟踪目标丢失、外设错误和影响目标判断的结构化缺口，必须先形成可持久恢复的强类型报告或承接事实，再对运行期队列可见。普通无命中、已过期且未承接的高频观察材料可以不恢复；这只删除传递候选，不得删除已经发布的任务、状态、动态或其它领域事实。

新运行期恢复时按以下顺序处理：

```text
在隔离候选中读取报告与消费治理事实
-> 复核来源材料、TTL、外设/配置版本和任务/工作项当前性
-> 已正式承接者只恢复任务侧已承接引用，不重新消费报告
-> 尚未承接且仍当前的不可舍弃报告，使用原报告身份进入新生产代次的投递候选
-> 过期、已消费、来源失效或任务不再当前者形成强类型舍弃/收束结果
-> 重建一个空物理队列和四类索引
-> 全部校验通过后原子发布新生产代次和消费代次
```

恢复后的再次投递不是第二份报告事实；同一报告稳定身份在全部代次中至多形成一次正式承接。已承接但工作未完成的材料由任务领域按任务恢复合同继续治理，不能退回队列再次竞争。生产代次、消费代次和重连代次必须使用稳定结构与单调版本互证，裸线程号、进程号、队列位置和日志序号不得参与一次消费裁决。

任何恢复候选、重新投递或已承接引用仍只是材料或任务治理事实，不得因跨重启保存而升级为世界事实。出现同一报告双承接、已消费报告重新入队、四索引与物理队列不一致或半恢复队列对消费者可见，属于内部恢复逻辑错误。

## 9. 禁止规则

1. 外设线程不得直接写需求、任务、世界事实或安全、服务结算。
2. 自我侧不得直接消费高频外设原始流作为世界事实。
3. 任务域不得临时接管高实时驱动和外设连续状态。
4. 外设私有过程不得伪装成可召回方法。
5. 作用外设执行成功不得直接解释成需求或任务完成。
6. 感知报告不得直接解释成已确认存在或稳定因果。
7. 线程、消息、队列、日志和控制面板不得成为动作来源或业务状态载体。

## 10. 验证边界

验证至少覆盖每实例独立控制、同外设命令串行、报告时效、任务匹配、正式提交隔离、不可舍弃事件和线程非动作来源。

缺少真实外设、真实命令和报告、任务承接与正式事实提交证据时，不得宣称外设闭环已经实现。
