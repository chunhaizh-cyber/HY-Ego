# 最小闭环验收用例

编号：003-A1
日期：2026-07-05
状态：已建立验收用例 / 第一轮结构验收口径

## 1. 定位

本文件对应 `规范/最小闭环设计.md`，用于把“最小闭环通过”拆成可执行、可检查、可禁止扩大宣称的验收用例。

本验收用例只验证第一版内核能以结构方式完成一次需求、任务、方法和结果结算，不证明自我苏醒，不证明完整自我循环，不证明旧项目能力迁移。

## 2. 验收前置

进入本验收前，必须满足：

```text
主信息系统规范已确认。
主信息系统详细设计已确认。
节点仓库详细设计已确认。
关系仓库详细设计已确认。
索引仓库详细设计已确认。
写入事务详细设计已确认。
最小闭环设计已确认。
```

代码层面至少需要具备：

```text
Debug x64 构建通过。
节点、主信息、关系、索引、事务最小函数可调用。
世界服务、特征服务、特征值服务、需求服务、任务服务、方法服务、因果服务存在最小入口。
默认构建不启用 HY_EGO_ENABLE_FAULT_TOLERANCE_CHECK。
```

若代码尚未完成，本文件只能作为验收设计，不得声明最小闭环已经通过。

## 3. 验收样例固定输入

第一轮验收固定使用以下样例口径：

```text
自我存在：自我
现实场景：现实场景
目标特征：闭环能量值
初始 I64 值：1
目标 I64 值：10
比较方式：当前值 >= 目标值
验证方式：读取目标特征值结构，读取任务状态结构，读取需求结算结构，读取动态和因果引用结构
手写方法：把目标特征值写为 10
```

样例文本只作为人读说明和注册种子，不作为机器事实。

## 4. 验收分级

| 级别 | 名称 | 通过标准 | 禁止宣称 |
| --- | --- | --- | --- |
| A0 | 文档验收 | 本文件和相关规范存在，状态口径一致 | 不宣称代码已实现 |
| A1 | 构建验收 | Debug x64 构建通过 | 不宣称最小闭环通过 |
| A2 | 入口拒绝验收 | 无效句柄、错误类型、缺少前置结构在入口被拒绝 | 不宣称事后校验等于入口拒绝 |
| A3 | 结构变化验收 | 目标特征值、任务状态、需求结算、动态、因果引用均发生结构变化 | 不用控制台输出替代结构变化 |
| A4 | 闭环验收 | 创建需求后，经任务、方法、因果和结算路径完成结构闭合 | 不宣称自我苏醒、旧能力迁移或真实样本验证 |

## 5. 用例总表

| 用例编号 | 用例名称 | 输入入口 | 预期结构变化 | 必须拒绝 | 通过证据 |
| --- | --- | --- | --- | --- | --- |
| MC-00 | 文档和构建前置 | 构建项目 | 无业务结构变化 | 构建失败时不得进入闭环验收 | 构建结果、文档状态 |
| MC-01 | 种子准备 | 世界 / 特征 / 特征值服务 | 自我存在、现实场景、目标特征、初始特征值创建 | 无效主信息、未定义节点类型、无效关系 | 读取结构句柄和当前 I64 值 |
| MC-02 | 创建需求 | 需求服务 `创建需求` | 需求节点、目标值、目标关系、初始需求状态创建 | 目标存在无效、目标特征无效、当前特征值缺失、比较方式非法 | 读取需求目标和状态 |
| MC-03 | 任务承接 | 任务服务 | 任务节点创建并绑定需求，任务状态变化 | 无效需求、已结算需求、越权写入方 | 读取任务到需求关系和任务状态 |
| MC-04 | 方法选择 | 方法服务 | 手写方法入口被选中或绑定任务 | 无效任务、方法不可执行 | 读取任务到方法关系或方法执行记录；选中前不检查目标匹配 |
| MC-05 | 方法执行改值 | 方法服务 -> 特征值服务 | 执行前授权形成，方法直接把目标特征值从 1 改为 10 | 无效目标、无效特征值、执行前未授权写值 | 读取授权材料、改变前值和改变后值 |
| MC-06 | 动态记录 | 因果服务 | 动态节点新增，并记录被改变目标、改变前值、改变后值 | 缺少被改变目标、缺少前值、缺少后值 | 读取动态结构；来源方法 / 动作可为空 |
| MC-07 | 因果引用记录 | 因果服务 | 因果引用新增并指向来源动态 | 缺少来源动态 | 回查来源动态 |
| MC-08 | 需求结算 | 需求服务结算 | 需求状态由未满足变为满足，结算版本递增 | 当前值缺失、比较方式非法、证据缺失 | 读取需求结算结构 |
| MC-09 | 入口拒绝负例 | 各仓库和领域服务 | 不产生结构变化 | 无效句柄、已删除节点、错误类型、非法关系 | 读取前后结构数量不变 |
| MC-10 | 容错诊断边界 | 默认构建 / 宏开启构建 | 默认构建不写诊断事实；宏开启只做人读诊断 | 日志、弹窗参与机器事实 | 默认无弹窗；宏开启时日志和弹窗不影响结构 |
| MC-11 | 事务边界 | 写入事务 | 四个事务边界可读 | 失败后继续提交、伪造回滚成功 | 读取事务阶段和失败边界 |

## 6. 详细用例

### MC-00 文档和构建前置

输入：

```text
读取项目规范和详细设计。
执行 Debug x64 构建。
```

通过条件：

```text
主信息规范和主信息详细设计状态为已确认。
节点、关系、索引、写入事务详细设计状态为已确认。
最小闭环验收用例存在。
Debug x64 构建通过。
```

失败处理：

```text
文档缺失时，只能声明验收设计不完整。
构建失败时，不得进入最小闭环结构验收。
```

### MC-01 种子准备

输入入口：

```text
世界服务创建自我存在。
世界服务创建现实场景。
特征服务创建目标特征类型。
特征值服务写入自我目标特征当前值 = 1。
```

预期结构：

```text
存在节点有效。
场景节点有效。
特征节点有效。
特征值结构有效。
存在、场景、特征和特征值之间的必要关系有效。
当前 I64 值可读取为 1。
```

入口拒绝：

```text
无效主信息不能创建节点。
未定义节点类型不能创建节点。
无效存在不能写特征值。
无效特征不能写特征值。
```

### MC-02 创建需求

输入入口：

```text
需求服务创建需求。
目标存在 = MC-01 创建的自我存在。
目标特征 = MC-01 创建的闭环能量值特征。
目标值 = 10。
比较方式 = 当前值 >= 目标值。
验证方式 = 读取目标特征值结构。
```

预期结构：

```text
需求节点有效。
需求目标值可读取。
需求到目标存在和目标特征的关系有效。
需求初始状态为未满足或待处理。
```

入口拒绝：

```text
目标存在无效时拒绝创建需求。
目标特征无效时拒绝创建需求。
当前特征值不存在时拒绝创建需求。
比较方式非法时拒绝创建需求。
验证方式缺失时拒绝创建需求。
```

需求服务不得在 `创建需求` 内偷偷补建目标存在、目标特征或当前特征值。

### MC-03 任务承接

输入入口：

```text
任务服务根据需求创建任务。
```

预期结构：

```text
任务节点有效。
任务到需求的关系有效。
任务状态从未开始变为已创建或待执行。
```

入口拒绝：

```text
无效需求不能创建任务。
已删除需求不能创建任务。
已结算且不允许再执行的需求不能创建任务。
外部模块不能裸写任务节点和任务状态。
```

### MC-04 方法选择

输入入口：

```text
方法服务为任务选择手写方法。
```

预期结构：

```text
方法节点或方法入口有效。
任务到方法的关系有效，或任务执行记录能追溯到方法。
选中前不检查方法目标与需求目标是否一致。
```

入口拒绝：

```text
无效任务不能选择方法。
方法不可执行时不能绑定为执行方法。
方法目标匹配不作为选中前入口拒绝条件；目标写入条件在执行前授权阶段处理。
```

### MC-05 方法执行改值

输入入口：

```text
方法服务执行手写方法。
方法执行前必须已经形成写值授权。
方法服务可以直接请求特征值服务把目标特征值从 1 写为 10。
```

预期结构：

```text
目标特征值发生结构变化。
改变前 I64 值 = 1。
改变后 I64 值 = 10。
任务状态发生变化。
```

入口拒绝：

```text
无效目标存在不能写值。
无效目标特征不能写值。
无效特征值句柄不能写值。
执行前未授权时不能写值。
```

### MC-06 动态记录

输入入口：

```text
因果服务记录动作动态。
```

预期结构：

```text
动态节点或动态记录新增。
动态能指向被改变目标。
动态能读取改变前值和改变后值的材料或引用。
来源方法 / 动作可以为空。
```

入口拒绝：

```text
缺少被改变目标时拒绝创建动态。
缺少前后值材料时拒绝创建动态。
```

### MC-07 因果引用记录

输入入口：

```text
因果服务记录因果引用。
```

预期结构：

```text
因果引用新增。
因果引用能回查来源动态。
```

入口拒绝：

```text
缺少来源动态时拒绝。
```

### MC-08 需求结算

输入入口：

```text
需求服务读取目标特征值并结算需求。
```

预期结构：

```text
需求状态从未满足或待处理变为满足。
需求结算版本发生变化。
需求结算能追溯到当前值、目标值、比较方式和来源证据。
```

入口拒绝：

```text
当前值缺失时拒绝结算。
比较方式非法时拒绝结算。
来源证据缺失时拒绝结算。
无效需求不能结算。
```

### MC-09 入口拒绝负例

必须至少覆盖：

```text
无效节点句柄创建关系被拒绝。
已删除节点创建关系被拒绝。
无效主信息句柄写 I64 值被拒绝。
存在、场景、特征、抽象状态、抽象动态、二次特征、因果引用等长期基础信息类节点可以作为语素入口对应信息节点。场景内临时实例状态和实例动态不得直接作为语素入口对应信息节点；当前第一轮代码已通过状态服务、动态服务和 `运行期临时` 关系识别实例层，并在语素入口拒绝实例状态和实例动态。抽象状态和抽象动态当前仍只有最小创建入口，完整抽象治理待后续增强。
特征值节点默认不能作为语素入口对应信息节点，除非后续特征值系统规范另行确认。
目标特征值缺失时创建需求被拒绝。
显示层不能写业务事实。
```

通过条件：

```text
拒绝前后相关结构数量不变。
拒绝路径不写机器事实。
拒绝路径不是先写坏结构再由校验器发现。
```

### MC-10 容错诊断边界

默认构建：

```text
HY_EGO_ENABLE_FAULT_TOLERANCE_CHECK 未启用。
入口拒绝只返回失败或无效句柄。
不写容错日志。
不弹窗。
不产生机器事实变化。
```

宏开启构建：

```text
HY_EGO_ENABLE_FAULT_TOLERANCE_CHECK 启用。
入口拒绝路径可以记录日志并弹窗。
日志用于排错，弹窗用于人读提醒。
日志和弹窗只记录和提醒，不改变结构结果。
日志和弹窗不得参与需求满足、任务完成、方法成功或机器事实裁决。
弹窗意味着逻辑错误已经暴露，必须排除后才能继续推进依赖该路径的实现或验收。
```

### MC-11 事务边界

输入入口：

```text
种子准备事务。
需求触发事务。
方法执行事务。
结算事务。
```

预期结构：

```text
每个事务有明确开始和提交。
失败事务能读取失败阶段和失败原因。
已提交事务不能重复提交。
失败事务不能继续提交。
未实现回滚前，不宣称跨仓库原子性。
```

## 7. 最小闭环通过标准

只有同时满足以下条件，才能声明“最小闭环通过”：

```text
Debug x64 构建通过。
入口前置拒绝用例通过。
创建需求后，目标特征值发生结构变化。
任务状态发生结构变化。
需求结算状态发生结构变化。
动态结构新增。
因果引用结构新增。
因果引用能回查来源动态。
控制台输出只作为只读展示，不作为通过证据。
```

## 8. 禁止宣称

即使本验收通过，仍不得宣称：

```text
自我苏醒。
完整自我循环。
旧项目能力已迁移。
D455 外设已接入。
体素系统已接入。
控制面板已接入。
真实外设样本通过。
跨仓库完整 ACID 事务已完成。
```
