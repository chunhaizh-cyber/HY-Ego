## 4. 验收分级

| 级别 | 名称 | 通过标准 | 禁止宣称 |
| --- | --- | --- | --- |
| A0 | 文档验收 | 本文件和相关规范存在，状态口径一致 | 不宣称代码已实现 |
| A1 | 构建验收 | Debug x64 构建通过 | 不宣称最小闭环通过 |
| A2 | 入口拒绝验收 | 无效句柄、错误类型、缺少前置结构在入口被拒绝 | 不宣称事后校验等于入口拒绝 |
| A3 | 结构变化验收 | 目标特征值、任务状态、需求结算、动态、因果引用均发生结构变化 | 不用控制台输出替代结构变化 |
| A4 | 闭环验收 | 创建需求后，经任务、方法、因果和结算路径完成结构闭合 | 不宣称自我苏醒、旧能力迁移或真实样本验证 |

## 6. 详细用例

### MC-00 文档和构建前置

输入：

```text
读取项目规范和详细设计。
执行 Debug x64 构建。
```

通过条件：

```text
`规范/4020_子规范_主信息身份生命周期与字段边界.md` 已登记，且主信息系统详细设计状态为已确认。
节点、关系、索引、写入事务详细设计状态为已确认。
最小闭环验收用例存在。
Debug x64 构建通过。
```

失败处理：

```text
文档缺失时，只能声明验收设计不完整。
构建失败时，不得进入最小闭环结构验收。
```

### MC-01 种子准备

输入入口：

```text
世界服务创建自我存在。
世界服务创建现实场景。
特征服务创建目标特征类型。
特征服务写入自我目标特征当前状态材料 = 1。
```

预期结构：

```text
存在节点有效。
场景节点有效。
特征节点有效。
特征值结构有效。
存在、场景、特征和特征值之间的必要关系有效。
当前 I64 值可读取为 1。
```

入口拒绝：

```text
无效主信息不能创建节点。
未定义节点类型不能创建节点。
无效存在不能写特征值。
无效特征不能写特征值。
```

### MC-02 创建需求

输入入口：

```text
需求服务创建目标状态需求。
目标存在 = MC-01 创建的自我存在。
目标特征 = MC-01 创建的闭环能量值特征。
目标状态 = 状态服务创建的目标状态节点，第一轮状态值材料为 10。
满足方式 = 当前特征状态材料满足目标状态；第一轮暂以当前状态材料 >= 目标状态值判断。
验证读取路径 = 读取目标特征状态材料、任务状态、需求结算状态、动态和因果引用结构。
```

预期结构：

```text
需求节点有效。
需求目标状态可通过模板关系读取。
需求到目标存在和目标特征的关系有效。
需求初始状态为未满足或待处理。
```

入口拒绝：

```text
目标存在无效时拒绝创建需求。
目标特征无效时拒绝创建需求。
当前特征状态材料不存在时拒绝创建需求。
目标状态非法时拒绝创建需求。
满足方式未确认时拒绝创建需求。
验证读取路径缺失时拒绝创建需求。
```

比较方式和验证方式属于后续通用结算规则；第一轮 DV-1 固定为当前状态材料 >= 目标状态值。

需求服务不得在 `创建目标状态需求` 内偷偷补建目标存在、目标特征、目标状态或当前特征状态材料。

### MC-03 任务承接

输入入口：

```text
任务服务根据需求创建任务。
```

预期结构：

```text
任务节点有效。
任务到需求的关系有效。
任务状态从未开始变为已创建或待执行。
任务创建或待执行实例状态必须带发生时间戳，且可由状态服务读回。
```

入口拒绝：

```text
无效需求不能创建任务。
已删除需求不能创建任务。
已结算且不允许再执行的需求不能创建任务。
外部模块不能裸写任务节点和任务状态。
```

### MC-04 方法选择

输入入口：

```text
方法服务为任务选择手写方法。
```

预期结构：

```text
方法节点或方法入口有效。
任务到方法的关系有效，或任务执行记录能追溯到方法。
选中前不检查方法目标与需求目标是否一致。
```

入口拒绝：

```text
无效任务不能选择方法。
方法不可执行时不能绑定为执行方法。
方法目标匹配不作为选中前入口拒绝条件；目标写入条件在任务承接、任务方法关系和方法执行入口处理。
```

### MC-05 方法执行改值

输入入口：

```text
方法服务执行手写方法。
任务已经承接需求并选择方法。
任务到方法关系已经形成。
方法服务可以请求特征服务把目标特征状态材料从 1 写为 10。
```

预期结构：

```text
目标特征值发生结构变化。
改变前 I64 值 = 1。
改变后 I64 值 = 10。
任务状态发生变化。
任务完成实例状态发生时间戳可读。
```

入口拒绝：

```text
无效任务节点不能写值。
无效方法节点不能写值。
缺少任务到方法关系时不能写值。
无效目标特征不能写值。
特征值不归属目标特征时不能写值。
```

### MC-06 动态记录

输入入口：

```text
动态服务记录实例动态。
```

预期结构：

```text
动态节点或动态记录新增。
动态发生时间戳可读。
动态能指向被改变目标。
动态能读取改变前值和改变后值的材料或引用。
来源方法 / 动作可以为空。
```

入口拒绝：

```text
缺少发生时间戳时拒绝创建动态。
缺少被改变目标时拒绝创建动态。
缺少前后值材料时拒绝创建动态。
```

### MC-07 因果引用记录

输入入口：

```text
因果服务记录因果引用。
```

预期结构：

```text
因果引用新增。
来源动态只作为入口准入证据。
第一轮不持久化因果引用到来源动态的单条关系。
单个因果引用不得解释为稳定因果结论。
```

入口拒绝：

```text
缺少来源动态时拒绝。
```

### MC-08 需求结算

输入入口：

```text
需求服务读取目标状态和当前特征状态材料并结算需求。
```

预期结构：

```text
需求结算状态节点新增或变化。
需求结算状态发生时间戳可读。
需求结算能追溯到目标状态、当前特征状态材料、来源任务或来源证据。
第一轮满足规则为当前状态材料 >= 目标状态值。
```

入口拒绝：

```text
当前特征状态材料缺失时拒绝结算。
目标状态缺失时拒绝结算。
第一轮满足规则不可执行时拒绝结算。
来源证据缺失时拒绝结算。
无效需求不能结算。
```

### MC-09 入口拒绝负例

必须至少覆盖：

```text
无效节点句柄创建关系被拒绝。
已删除节点创建关系被拒绝。
无效主信息句柄写 I64 值被拒绝。
存在、场景、特征、抽象状态、抽象动态、二次特征、因果引用等长期基础信息类节点可以作为语素入口对应信息节点。场景内临时实例状态和实例动态不得直接作为语素入口对应信息节点；当前第一轮代码已通过状态服务、动态服务和 `运行期临时` 关系识别实例层，并在语素入口拒绝实例状态和实例动态。抽象状态和抽象动态当前仍只有最小创建入口，完整抽象治理待后续增强。
特征值节点默认不能作为语素入口对应信息节点，除非后续特征值系统规范另行确认。
目标特征值缺失时创建需求被拒绝。
显示层不能写业务事实。
```

通过条件：

```text
拒绝前后相关结构数量不变。
拒绝路径不写机器事实。
拒绝路径不是先写坏结构再由校验器发现。
```

### MC-10 容错诊断边界

默认构建：

```text
HY_EGO_ENABLE_FAULT_TOLERANCE_CHECK 未启用。
入口拒绝只返回失败或无效句柄。
不写容错日志。
不弹窗。
不产生机器事实变化。
```

宏开启构建：

```text
HY_EGO_ENABLE_FAULT_TOLERANCE_CHECK 启用。
入口拒绝路径可以记录日志并弹窗。
日志用于排错，弹窗用于人读提醒。
日志和弹窗只记录和提醒，不改变结构结果。
日志和弹窗不得参与需求满足、任务完成、方法成功或机器事实裁决。
弹窗意味着逻辑错误已经暴露，必须排除后才能继续推进依赖该路径的实现或验收。
```

### MC-11 事务边界

输入入口：

```text
种子准备事务。
需求触发事务。
方法执行事务。
结算事务。
```

预期结构：

```text
每个事务有明确开始和提交。
失败事务能读取失败阶段和失败原因。
已提交事务不能重复提交。
失败事务不能继续提交。
未实现回滚前，不宣称跨仓库原子性。
```
