# 0990 方法可靠度与谨慎度规则

## 1. 目标

本文用于把“方法可靠度”的作用边界收敛成一份与当前工程实现对齐的小规范。

本文回答的是：

- 方法可靠度到底衡量什么
- 方法可靠度是否决定“能不能试”
- 方法可靠度应如何影响试探粒度、检查强度与回退灵敏度
- 当前代码里哪些字段和特征已经在承载这套语义


## 2. 当前实现对齐结论

先固定以下结论。

### 2.1 方法可靠度不是硬过滤门

当前实现中，方法不会因为“历史成功率低”就直接被硬过滤掉。

现有代码更接近下面这条口径：

- `安全不过线` 才是硬阻断
- `历史成功率`、`成功次数 / 失败次数`、`风险成本修正` 主要用于评分、排序和调节试探姿势

这点已经体现在 [方法管理模块_v0.ixx](D:\鱼巢\方法管理模块_v0.ixx) 的方法打分逻辑里：

- 类型不匹配、标签完全不匹配会过滤
- 历史成功率只作为得分修正项
- 安全风险通过风险项参与控制

### 2.2 当前代码已经有三类直接承载点

#### A. 方法实体 / 绑定关系内容

以下键已真实存在：

```text
历史成功次数
历史失败次数
历史执行次数
历史成功率

成功次数
失败次数
总次数
成功率
```

它们已经落在：

- 方法实体内容
- 任务-方法绑定关系内容
- 学习写回与方法管理更新逻辑

#### B. 方法虚拟存在 / 评估原始量

当前实现中，方法节点与方法虚拟存在已经承载：

```text
节点_运行次数
节点_成功次数
节点_失败次数
节点_连续成功次数
节点_连续失败次数
```

这些值更接近“方法运行证据”。

#### C. 学习线程稳定性证据

当前实现已经存在：

```text
方法尝试状态
方法结果分裂次数
方法连续稳定次数
方法最近结果签名
```

它们已经可以表达：

- 方法是否还在观察中
- 结果是否发生漂移
- 结果是否连续稳定复现

### 2.3 当前规范不新增结构体字段

`v0` 不要求新增主信息结构成员。

方法可靠度相关语义，优先继续落在：

- 方法实体内容
- 绑定关系内容
- 方法虚拟存在特征
- 学习线程稳定性特征


## 3. 定义

方法可靠度定义为：

> `该方法在相近任务和相近场景下，使目标结果稳定出现的程度。`

这里最关键的不是“成功过没有”，而是：

> `结果会不会稳稳地来。`


## 4. 基本原则

### 4.1 方法存在即可进入尝试范围

只要方法存在，原则上就可以进入候选集。

方法可靠度不负责决定：

- 能不能进入候选
- 能不能被看见

### 4.2 安全值不过线才是真硬阻断

真正的硬阻断来自：

- 安全边界
- 禁止试探约束
- 明确失效或禁用关系

不是来自“可靠度偏低”本身。

### 4.3 可靠度是谨慎系数，不是通行证

可靠度越低，不是“不准用”，而是：

- 步子更小
- 检查更密
- 回退更快
- 影响范围更窄

所以：

> `可靠度不决定“用不用”，而决定“用得多谨慎”。`

### 4.4 紧迫度可以提高尝试意愿，但不能越过安全硬边界

控制顺序固定为：

```text
先看安全是否允许
再看服务值 / 需求紧迫度是否值得试
最后由可靠度决定试探强度
```


## 5. 最小计算规则

### 5.1 最小定义

`v0` 最小实现只看一件事：

> `目标结果是否稳定出现。`

### 5.2 最小计数口径

方法可靠度最小可写成：

```text
方法可靠度 = 稳定出现次数 / 尝试次数
```

其中：

- `尝试次数`
  - 该方法被真正用于执行的次数
- `稳定出现次数`
  - 目标结果命中任务完成判据
  - 没有触发严重回退
  - 结果没有明显漂移
  - 在近似条件下重复出现

### 5.3 Q10000 表示

统一建议写成：

```text
方法可靠度_Q10000 = 稳定出现次数 / 尝试次数 * 10000
```

例如：

```text
尝试次数 = 10
稳定出现次数 = 7
方法可靠度_Q10000 = 7000
```

### 5.4 与当前实现的对齐解释

当前代码尚未单列 `方法可靠度_Q10000` 字段。

因此 `v0` 对齐口径是：

- `历史成功率` 作为可靠度的第一层近似
- `方法连续稳定次数` 作为“是否稳定复现”的补证
- `方法结果分裂次数` 作为“结果是否漂移”的反证
- `方法尝试状态` 作为“是否还在观察中”的状态证据

也就是说：

> `在现有实现里，可靠度先由成功率承载，再由稳定次数和结果分裂次数修正。`


## 6. 控制作用规则

方法可靠度不负责“封门”，而负责调节下面 5 个控制点。

### 6.1 是否直接主用

- 高可靠：可直接作为主方法
- 中可靠：可主用，但应加检查
- 低可靠：仅在没有更好方法或需求紧迫时尝试
- 未知可靠：按低可靠处理

### 6.2 试探粒度

- 高可靠：一步可以稍长
- 低可靠：必须拆成更小步骤

### 6.3 检查强度

- 高可靠：检查可稀一些
- 低可靠：每步都验或高频验

### 6.4 回退灵敏度

- 高可靠：允许多走一步再判断
- 低可靠：稍有异常就应快速回退

### 6.5 路径扩展意愿

- 高可靠：可以进入较长执行链
- 低可靠：更适合短链、窄域、局部试探

可收成一句话：

> `越不可靠，越细步；越紧迫，越愿意试；越危险，越不能越线。`


## 7. 与安全值、服务值、需求值的关系

方法可靠度不是根值，它是根值的证据桥。

### 7.1 对服务值的回溯

结果稳定出现意味着：

- 目标更容易达成
- 服务完成概率更高

因此会形成：

```text
结果稳定出现
-> 服务达成概率上升
-> 服务值支持增强
```

### 7.2 对安全值的回溯

结果稳定出现也意味着：

- 波动更低
- 回退更少
- 可预测性更高

因此会形成：

```text
结果稳定出现
-> 风险和波动下降
-> 安全值支持增强
```

反过来同样成立：

- 结果不稳定 -> 服务支持减弱
- 结果不稳定 -> 安全支持减弱


## 8. 极简状态规则

本规则不要求建立复杂的“方法生命周期机”。

这里只固定 3 个极简语义状态：

```text
可尝试
待观察
禁用
```

### 8.1 可尝试

语义是：

- 方法存在
- 当前未触碰安全硬边界
- 可以进入候选范围

### 8.2 待观察

语义是：

- 可靠度不足
- 结果稳定性证据还不够
- 可以试，但必须更谨慎

与当前实现最接近的承载是：

- `方法尝试状态 = 未尝试 / 尝试中 / 待重新验证`

### 8.3 禁用

语义是：

- 触碰安全硬边界
- 当前上下文不允许再试

`v0` 不要求把它做成新的方法主状态枚举。

当前更稳的落点是：

- 控制阶段的硬阻断结论
- 任务/关系失效
- 当前轮调度中的禁用判断

### 8.4 与 0980 的关系

本规则中的“可尝试 / 待观察 / 禁用”是可靠度状态。

它不改写 [0980_原生学习兜底函数集规范（现有实现对齐版）.md](D:\鱼巢\规范\0980_原生学习兜底函数集规范（现有实现对齐版）.md) 里“原生 / 临时 / 候选 / 已验证 / 退役”那类方法来源与成长语义。

## 10. 最小更新规则

每次方法执行后，最小更新规则如下：

1. `尝试次数 + 1`
2. 若结果稳定出现，则 `稳定出现次数 + 1`
3. 重新计算 `方法可靠度_Q10000`

在当前实现尚未单列该字段前，可以先同步更新：

- `历史成功次数 / 历史失败次数 / 历史成功率`
- `方法连续稳定次数`
- `方法结果分裂次数`
- `方法尝试状态`

> `有方法就可试，可靠度只调谨慎度。`
