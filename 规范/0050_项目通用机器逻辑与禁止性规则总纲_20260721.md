# 0050 项目通用机器逻辑与禁止性规则总纲

编号：0050
日期：20260721
状态：正式上级规范

## 1. 定位

本规范只规定海中鱼巣全部机器逻辑共同服从的原则性规则。具体对象字段、特征类型、状态组合、业务流程和算法必须进入对应根规范或子规范，不在本文件展开。

## 2. 规范权威与实现语义原则

1. `规范/规范目录.md` 及其登记的生效规范是本项目机器逻辑的唯一正式定义来源。
2. 未经本项目规范目录登记的外部材料、先前工程材料、阶段记录、设计草案和程序行为，不取得本项目机器逻辑定义权。
3. 程序行为与生效规范不一致时，程序行为属于待治理差异，不得反向改变规范语义或作为继续扩展同类实现的依据。
4. 对象定义、事实来源、写入权、生命周期、合法分支、拒绝、内部逻辑错误或验证合同缺失或冲突时，禁止进入对应机器实现，必须先完成规范治理。
5. 正式规范闭环只证明机器逻辑定义已经建立，不证明相应程序能力已经实现、接通或通过验收。

## 3. 机器事实原则

机器事实必须由正式机器结构承载，并同时具备：

```text
稳定身份或明确归属
确定类型和值域
合法写入方
可追溯来源
生命周期、当前性或版本
完整发布事实
```

正式机器结构包括节点、稳定句柄、主信息、关系、索引、特征、特征值、状态、动态、因果、需求、任务和方法。

日志、控制台输出、注释、说明文档、临时文本、临时 JSON、显示字段、线程状态、锁状态、函数返回码、缓存命中和索引命中均不得独立裁决机器事实。

## 4. 身份与结构职责原则

1. 长期对象身份必须使用稳定句柄、稳定主键、编号和生命周期版本；裸指针、线程编号、锁地址、日志标签和显示文本不得承担长期身份。
2. 节点只提供对象身份与节点类型，不直接承载全部业务语义。
3. 主信息只保存身份附属值、拓扑锚点、值容器或明确标注的非权威缓存；可特征化事实不得复制成便利字段。
4. 关系表达稳定对象之间的正式联系，不得用普通父子关系替代全部业务关系。
5. 索引只加速定位，命中后必须回读权威结构；索引重建不得改变业务事实。
6. 特征表达对象可具有的属性类型；特征值表达该类型下的具体值材料。
7. 指针只能作为运行期临时引用或缓存，不得成为持久身份。

## 5. 特征、状态、动态与因果原则

1. 一项状态由宿主、特征类型集合、对应值集合、适用约束、来源和版本共同成立，不得由单个枚举、普通 bool 或显示名称替代。
2. 生命周期阶段只能作为粗粒度投影；完整业务状态必须回到对应特征集合。
3. 动态必须表达合法宿主的状态变化，并保留变化前、变化后、发生时间和来源动作。
4. 线程、调度器、日志、界面和外部消息不是动作来源；只有正式方法执行或领域服务写入可以成为动作来源候选。
5. 因果必须建立在合法状态和动态证据之上；单次来源引用、统计分数、解释文本和缓存命中不得直接成为稳定因果结论。
6. 综合二态判断必须保留组成特征、值域、收束规则、来源、版本和验证；不得把综合结论压成无来源 bool。

## 6. 需求、任务与方法原则

1. 需求表达目标状态；任务表达对一个确定目标的授权承接；方法表达可选择、可执行和可验证的能力。
2. 需求、任务和方法必须保持稳定身份分离，不得互相冒充。
3. 需求目标、任务状态和方法条件结果必须用正式特征、状态、关系和版本结构表达。
4. 方法执行成功、动作已经发生、日志显示成功、子任务完成或来源需求已满足，均不得单独证明任务完成。
5. 任务完成、需求满足和价值结算是三个独立裁决，必须分别保留依据和防重复边界。
6. 管理、执行、等待、筹办、完成和取消等业务角色或阶段不得只由有无子节点或单个枚举裁决。

具体需求路径与权重、任务特征及状态矩阵、筹办轮次、方法召回和结算规则分别进入 51xx、52xx、53xx 和相关子规范。

## 7. 写入权与事务原则

1. 外部调用方不得绕过领域服务直接写仓库。
2. 领域服务负责业务准入、业务不变量和结果语义；仓库负责通用结构不变量与本仓库原子读写。
3. 跨仓库写入必须在一个明确事务边界中完成；未发布候选不得被普通读取路径看见。
4. 对外发布必须是事务最后一步；提交前任一步失败都不得留下可读半结构。
5. 原始事务令牌、仓库锁和内部写会话不得穿透领域服务公开业务接口。
6. 写入口必须先完成身份、类型、权限、版本、关系和业务前置条件拒绝，再执行结构写入。

## 8. 并发原则

1. 并发隔离键必须是被竞争业务事实的稳定身份，不得用裸线程编号替代。
2. 同一业务对象上的唯一执行权必须由对象结构中的唯一有效占用事实裁决；互斥锁只保护内存访问，不承担业务状态。
3. 不同业务对象可以并发；不得因为来源相同、根需求相同或复用同一方法而错误串行化。
4. 两条路径越过唯一执行权同时进入同一对象的受保护业务区间，属于内部并发规则破坏，不得伪装成幂等成功。

## 9. 逻辑内结果与内部逻辑错误原则

1. 入口材料无效、身份失效、类型不符、权限不足、版本过期和业务前置条件不满足，应在写入前结构化拒绝。
2. 候选为空、合法等待、路径选择、目标已满足和授权已失效等协议允许结果，必须返回具名结构结果，不得只返回裸失败。
3. 前置条件通过后，内部创建、绑定、写关系、写状态、记录动态、结算、读回或发布结果不符合预期，属于内部逻辑错误。
4. 内部逻辑错误不得包装成普通需求、任务状态、学习缺口或等待继续运行；必须排除根因后才能恢复依赖路径。
5. 日志、弹窗、扫描器和事后校验只能诊断，不能修复结构或改变机器事实。

## 10. 外部材料、持久化与人读投影原则

1. 外部输入、外设采集、网络消息、文件和数据库记录首先都是材料，必须经正式准入与提交才能成为运行期事实。
2. 运行期仓库是当前机器事实来源；数据库和文件只承担快照、事件记录、恢复候选、审计和只读投影。
3. 恢复必须重新验证身份、版本、结构不变量和业务不变量，再通过正式入口发布。
4. 显示层、控制台和日志只能读取正式结构供人理解，不得写业务事实或参与业务裁决。
