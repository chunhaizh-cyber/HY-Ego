# 1180 根规范 二次特征

编号：1180
日期：20260722
状态：正式根规范

## 1. 根定位

二次特征属于特征体系，承载关系性、比较性、方向性和聚合性特征，是从状态、动态、任务、需求或其它正式事实派生的可回查判断对象。

## 2. 本项目承载

```text
二次特征节点及稳定句柄
+ 唯一非零稳定主键和当前节点句柄
+ 有序且角色明确的组成项关系
+ 二次特征定义、比较规则和规则版本
+ 输入事实、来源动态和有效范围关系
+ 可选宿主归属或场景事实目录关系
```

当前组合入口使用有序 `引用` 关系承载组成项。顺序只表达结构角色，不得靠调用参数位置以外的临时约定解释业务语义。

## 3. 事实与边界

1. 二次特征可以表达比较、方向、区间、覆盖、稳定性、归属冲突、候选唯一性和任务范围阈值等关系事实。
2. 二次特征不得替代一阶特征值、状态事实、动态事实或因果账本。
3. 自然语言关系理解结果不得直接写成权威二次特征。
4. 挂在存在特征结构中的二次特征表示该存在自身的派生或关系性实例特征；进入场景事实目录的二次特征通常表示场景中存在之间的关系事实。
5. 临时比较、筛选、评分、排序和一次性推导默认生成、使用后丢弃。只有后续事实、证据、因果样本、需求任务方法输入或审计需要回查时才允许持久化。

## 4. 写入、读取与验证

二次特征领域服务负责组成项、角色、规则、来源和持久化必要性的准入；数据操作层负责节点、二次特征域类型化记录、有序关系和索引的原子发布。普通调用方不得直接拼接关系形成二次特征。

验证必须证明组成项非空、稳定身份互不重复、顺序连续、角色和值域合法、输入事实当前、规则版本可用。判断型二次特征还必须满足 4160 的收束证据合同。

## 5. 生命周期与当前性

临时二次特征不进入权威事实仓库。持久二次特征必须声明适用范围、来源版本和失效条件；输入事实或规则版本变化后旧结论只能作为历史证据，不得继续作为当前结论。
