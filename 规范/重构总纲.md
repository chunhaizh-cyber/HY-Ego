# 重构总纲

编号：002
日期：2026-07-05
状态：阶段 0 正式规范

## 1. 定位

`海中鱼巣` 是 `鱼巢` 第二代全量重构实验工程。

本项目不复制旧项目目录、旧项目容器或旧项目显示链路，而是重新建立第一版内核协议。

第一版内核只追求：

```text
节点仓库
稳定句柄
主信息仓库
关系仓库
索引仓库
事务边界
领域服务
最小需求任务方法闭环
```

本规范只能证明重构边界和第一版结构口径已确定，不能证明代码已实现，不能证明旧项目能力已迁移。

## 2. 第一版保留范围

第一版保留以下核心概念：

```text
语素
基础信息类节点
存在
场景
特征
特征值
需求
任务
方法
状态
动态
因果引用
```

这些概念必须通过节点、主信息、关系、索引、特征、特征值、需求、任务、方法、状态、动态或因果引用承载。

## 3. 第一版暂缓范围

第一版暂缓以下能力：

```text
D455 外设
体素系统
控制面板
SQL 实时投影
复杂多线程
完整自我循环
历史旧项目兼容
真实外设样本验证
```

暂缓能力不得作为第一版验收条件，不得在最小闭环完成前提前接入。

## 4. 节点身份

节点长期身份固定使用：

```text
节点编号
节点句柄
版本号
```

节点句柄至少表达：

```text
仓库编号
节点编号
版本号
```

指针只能作为运行期缓存或临时访问材料，不能作为长期事实身份。

## 5. 主信息边界

主信息通过主信息仓库保存。

主信息长期身份固定使用：

```text
主信息编号
主信息句柄
版本号
```

主信息只允许保存：

```text
身份
拓扑锚点
值容器
明确声明的非权威缓存
```

以下内容不得直接扩散为主信息字段：

```text
对象属性
观测事实
统计事实
治理状态
判断结论
任务结果
方法结果
因果解释
```

这些内容应优先进入特征、特征值、状态、动态、二次判断结果、因果引用、需求、任务或方法。

新增主信息字段前必须写清：

```text
用途
写入方
读取方
生命周期
是否权威
验证方式
```

## 6. 关系边界

第一版关系不得混成单一父子关系。

关系类型至少区分：

```text
普通父子
归属关系
引用关系
索引关系
模板关系
运行期临时关系
因果来源关系
```

关系记录至少表达：

```text
关系编号
关系类型
源节点句柄
目标节点句柄
顺序号
版本号
有效状态
```

关系创建、删除、重挂和失效必须通过关系仓库或领域服务入口完成。

## 7. 机器逻辑结构化边界

所有机器判断必须走结构，不得用人读材料承载。

允许承载机器逻辑的结构包括：

```text
节点
主信息
关系
索引
特征
特征值
需求
任务
方法
状态
动态
因果引用
```

禁止承载机器逻辑的材料包括：

```text
日志
控制台输出
注释
说明文档
临时字符串
临时 JSON
显示标题
调试文本
```

## 8. 领域服务封口

所有业务写入必须经过领域服务或正式仓库入口。

第一版领域服务包括：

```text
语素服务
世界服务
特征服务
特征值服务
需求服务
任务服务
方法服务
因果服务
```

外部调用方不得直接写：

```text
节点仓库
主信息仓库
关系仓库
索引仓库
```

只读查询可以返回只读视图，不得暴露可写内部对象。

## 9. 持久化边界

第一版运行期核心以内存仓库为准。

数据库和文件持久化只作为：

```text
快照
事件日志
恢复材料
审计材料
只读查询材料
```

数据库不得直接裁决机器事实，不得绕过领域服务写运行期结构。

## 10. 显示边界

显示层、日志层和控制台输出只供人读。

显示层可以读取结构，但不得写业务事实。

显示内容不得作为需求满足、任务完成、方法成功或能力完成的机器判断来源。

## 11. 阶段 0 验收

阶段 0 只允许声明：

```text
重构总纲已建立
最小闭环设计已建立
第一版保留范围和暂缓范围已写入
节点身份、主信息、关系、结构化逻辑和显示边界已写入
```

阶段 0 不允许声明：

```text
代码已实现
核心内核已完成
最小闭环已跑通
旧项目能力已迁移
自我循环已完成
外设 / 体素 / 控制面板已接入
```
