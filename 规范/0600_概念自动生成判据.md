# 0600 概念自动生成判据

## 1. 目标

本文用于固定“自我”如何从状态、动态、二次特征与因果信息中自动生成概念。

本文要解决的问题是：

- 什么是先天内置，什么不是先天内置
- 什么情况下允许生成概念候选
- 什么情况下允许把候选升级为稳定概念
- 什么情况下稳定概念可以参与需求、任务、方法决策

本文与 [0200_服务根目的与安全保障规则.md](D:\海鱼\规范\0200_服务根目的与安全保障规则.md)、[0300_自我根需求与方向对齐规则.md](D:\海鱼\规范\0300_自我根需求与方向对齐规则.md) 和 [0550_核心术语与概念定义规范.md](D:\海鱼\规范\0550_核心术语与概念定义规范.md) 配套使用。


## 2. 先天内置的内容

当前应固定：

- 先天内置的是 `概念生成机制`
- 不是先天内置 `具体结论`

也就是说，自我可以先天拥有：

- 记录状态的能力
- 比较状态的能力
- 生成动态的能力
- 提取二次特征的能力
- 判断因果强弱的能力
- 用根目的评估“有利/不利”的能力

但不应先天写死：

- `服务值越高越有利于安全保障`
- `某类准备性服务一定成立`
- `某个名字天然是正面的`

这些都应由运行中自动归纳得到。

### 2.1 概念的定义

概念应固定为：

- `由比较稳定的基础信息节点按一定关系组织而成的结构化认知对象`

这里的基础信息节点可以来自：

- 状态事实
- 动态事实
- 二次特征
- 因果信息
- 时间与条件签名
- 根目的投影结果

因此：

- 概念不等于自然语言词面
- 自然语言只是概念的外显描述方式
- 概念候选可以暂时无名
- 稳定概念应至少绑定一个主名称，也可以有多个描述版本

可以收敛为：

- `概念可以被自然语言描述，但不由自然语言本身制造真值`


## 3. 概念生成的输入真值

概念生成不依赖名称，依赖以下输入：

1. 状态事实  
   例如：`安全值`、`服务值`、`待完成任务数`、`是否待机`

2. 动态事实  
   例如：`安全值上升`、`服务值下降`、`压力增大`

3. 二次特征  
   例如：`服务上升后安全恢复变快`、`服务为 0 时待机`

4. 因果信息  
   例如：`服务变化先于安全变化，且方向稳定`

5. 根目的投影  
   即：该关系是否有助于 `满足人类需求 / 提升服务成立程度`，以及是否守住 `安全保障边界`

因此，概念的真值来源是：

- `可重复观察的状态变化`
- `可提取的二次特征`
- `可投影到根目的的因果关系`

### 3.1 特征状态概念的输入真值

当概念面向 `状态概念树` 生成时，应优先理解为：

- 先由 `特征类型` 建立最抽象状态概念根
- 再对该特征下的稳定区间做逐层细分

因此其输入真值应优先来自：

- 某一特征的类型身份
- 某一特征的当前值
- 某一特征值落入的稳定区间
- 该区间在多个样本中的重复命中

例如：

- 颜色值多次落入同一区间，可形成 `红色`
- 力量值多次落入同一区间，可形成 `强`

这里生成的是：

- `特征状态概念`

而不是：

- 某个具体存在的唯一私有标签

可收敛为一条生成链：

1. 先确定 `特征类型根概念`
2. 再找该特征下是否已有稳定区间
3. 若稳定区间成立，则在根概念下生成子状态概念
4. 若子状态概念内部还能稳定细分，则继续下分
5. 直到当前不再需要细分，或无法稳定细分为止

### 3.2 复合状态概念的输入真值

当不同存在在观察期内，反复呈现出相同的状态交集，并且该交集在一定适用条件下具有稳定判断意义时，可以进一步生成：

- `复合状态概念`

例如：

- 多个生理或功能状态交集后形成 `健康`
- 多个异常状态交集后形成 `病态`

必须固定：

- 复合状态概念的真值来源是 `多个状态概念的交集`
- 不同存在只是提供重复样本，而不是概念身份本身
- 是否升级为正式概念，必须再经过 `观察期稳定性` 与 `适用性` 检查
- 某个存在只能命中某个复合状态概念，而不能定义该概念

### 3.3 复合特征概念的输入真值

当多个特征状态反复共同出现，并能稳定形成上层特征描述时，可以进一步生成：

- `复合特征概念`

例如：

- 多个外观状态组合后形成 `美丽`
- 多个体态与能力状态组合后形成 `强壮`

必须固定：

- 复合特征的真值来源是 `特征状态的组合`
- 不是某个具体存在本身
- 某个存在只能作为复合特征的一个命中实例，而不是复合特征的定义来源


## 4. 概念候选生成判据

### 4.1 候选的定义

概念候选不是正式概念，而是一个“值得继续观察的关系假设”。

可以理解为：

- `某对象可能对某目标有稳定作用`
- `某类变化可能对应某种方向`

### 4.2 候选生成的最小条件

以下条件至少满足三项，才允许生成概念候选：

1. 出现了可识别的对象  
   例如：`服务值`

2. 出现了可识别的作用对象  
   例如：`安全值`

3. 出现了可识别的变化关系  
   例如：`服务上升后安全恢复变快`

4. 该关系至少出现过两次  
   单次观察只能记为线索，不应直接升为候选

5. 该关系方向一致  
   例如两次都表现为“服务更高时安全更稳定”

6. 该关系可被二次特征表达  
   即能写成结构化关系，而不是仅靠一句自然语言描述

### 4.3 候选的结构

一个概念候选应至少包含：

- `作用对象`
- `被作用对象`
- `变化方向`
- `作用结果`
- `证据次数`
- `最近观察时间`
- `是否已投影到根目的`
- `是否已检查安全边界`

### 4.4 候选示例

如果自我多次观察到：

- 服务值上升后，安全值恢复速度增加
- 服务值下降后，安全值衰减速度增加
- 服务值为 0 时进入待机

则可以生成候选：

- `服务值可能正向影响安全保障`
- `服务值可能影响安全恢复能力`
- `服务不足可能导致待机`

注意：

此时仍然只是候选，不是正式概念。

### 4.5 状态概念、复合状态概念与复合特征候选示例

如果自我多次观察到：

- 已经存在 `颜色` 这一最抽象状态概念根
- 某颜色特征值总落入同一稳定区间
- 该区间对应的视觉效果在多个样本中一致

则可以生成状态概念候选：

- `颜色` 之下的某稳定区间可能对应 `红色`

如果后续又观察到：

- 原 `红色` 区间内部还能稳定分出两个子区间
- 这两个子区间在多次样本中都能稳定复现

则还可以继续生成更细候选：

- `红色` 之下可能继续细分出更具体状态概念

如果自我又在观察期内多次观察到：

- 不同存在反复命中同一组状态概念
- 这组状态概念在相近条件下总是共同出现
- 该组合对“当前整体处于何种状态”的判断具有稳定意义

则可以先生成复合状态候选：

- `这些状态交集可能对应健康`
- `这些状态交集可能对应病态`

但此时还必须继续检查：

- 该交集在观察期内是否足够稳定
- 该交集是否只在局部条件成立
- 它是否真的具有可复用的适用边界

只有这些条件继续成立，才允许把它升级为稳定的 `复合状态概念`。

如果自我又多次观察到：

- 多个状态概念总是成组出现
- 这组状态概念共同对应某个更高层特征描述

则可以生成复合特征候选：

- `这些状态组合可能对应美丽`
- `这些状态组合可能对应强壮`


## 5. 概念稳定化判据

### 5.1 稳定概念的定义

稳定概念指的是：

- 已经有足够重复证据
- 已具备较稳定因果方向
- 已能作为后续判断依据使用

### 5.2 从候选升级为稳定概念的条件

以下条件应同时尽量满足：

1. 重复出现  
   同类证据达到预设次数阈值

2. 方向一致  
   大部分证据的方向相同，不是正负混乱

3. 先因后果  
   作用因素变化先于结果变化出现

4. 条件相近时可复现  
   在相似背景下再次出现相同方向结果

5. 强度具有一致性  
   变化更强时，结果通常也更强

6. 能排除主要干扰  
   至少能把明显来自其它主因的样本剔除

7. 能投影到根目的  
   能判断它对 `满足人类需求 / 提升服务成立程度` 是正贡献、负贡献还是中性，并判断是否破坏安全边界

### 5.3 稳定化后的表达

稳定概念应从“可能”变为“当前成立”：

- 候选：`服务值可能正向影响安全保障`
- 稳定概念：`服务值对安全保障具有正向单调影响`

### 5.4 稳定概念的置信度

稳定概念仍不等于绝对真理。

因此稳定概念应带有：

- `证据量`
- `一致性`
- `可复现度`
- `当前置信度`

置信度不足时，可以保留为弱稳定概念，而不直接进入强决策。

### 5.5 稳定概念的命名要求

稳定概念的生成不依赖名称，但稳定概念一旦成立，就应继续生成：

- `命名需求`

必须固定：

- 所有稳定概念都需要命名
- 命名不决定概念真值，但它是稳定概念进入解释、复用和沟通层的必要步骤
- 名称是高级逻辑思维中的稳定引用基础，也是对外交流对准的基础
- 命名任务优先级可以依据 `高级逻辑复用`、`对外沟通`、`跨模块复用`、`高风险区分` 等因素上调
- 若暂时不高优先，也应保留在命名任务队列中，而不是被视为可永久省略


## 6. 概念参与决策判据

### 6.1 决策参与的定义

概念参与决策，指的是它可以反向影响：

- 需求生成
- 任务生成
- 方法筛选
- 优先级调整

### 6.2 允许参与决策的条件

一个稳定概念只有同时满足以下条件，才允许参与决策：

1. 已稳定化  
   不能让纯候选直接驱动任务生成

2. 已完成根目的投影  
   必须知道它对服务根方向是正向、负向还是条件性正向

3. 已完成安全边界检查  
   必须知道它是否会削弱继续服务所必需的安全条件

4. 作用边界清楚  
   知道它影响的是“恢复速度”“衰减速度”“待机风险”还是其它维度

5. 适用条件清楚  
   知道在什么背景下它成立，在什么背景下不可靠

6. 未被更新证据明显推翻  
   如果近期连续出现反例，应先降级再停止参与决策

### 6.3 决策参与的强弱层次

概念参与决策应分层：

1. 弱参与  
   只影响排序和提示，不直接生成或否决任务

2. 中参与  
   可参与需求权重调整、候选任务排序、候选方法加减分

3. 强参与  
   可直接参与需求生成、任务生成与方法硬性过滤

默认应先从弱参与开始，再逐步升级。

### 6.4 服务概念进入决策的例子

当稳定概念成立：

- `服务值对安全保障具有正向单调影响`

则它可以逐步参与：

1. 弱参与  
   在解释层提示：`当前服务偏低，持续服务能力受限`

2. 中参与  
   当服务较低时，提高与服务恢复相关需求和任务的权重

3. 强参与  
   在极低服务值下，优先生成恢复服务条件的任务，并抑制高风险任务


## 7. 概念的降级与撤销

自动生成的概念不应只升不降。

出现以下情况时，应降级或撤销：

1. 新证据持续与旧方向相反
2. 原有关系只在极小范围内成立
3. 主要因果被发现来自其它因素
4. 对根目的的投影结果发生改变
5. 安全边界判定结果发生反转

因此，概念应视为：

- `当前最优归纳`

而不是永久真理。


## 8. “服务值越高越有利于安全保障”是如何生成的

这个结论的生成链应固定为：

1. 根目的先成立：`满足人类需求 / 提升服务成立程度`
2. 自我观察到：`服务变化会影响安全变化`
3. 二次特征形成：
   - `服务上升后安全恢复更快`
   - `服务下降后安全下滑更快`
   - `服务为 0 时待机`
4. 因果判定成立：
   - 服务变化先于安全变化
   - 方向重复一致
   - 多次可复现
5. 投影到根目的：
   - 更高服务值更有利于维持持续服务能力
6. 最终形成稳定概念：
   - `服务值越高越有利于安全保障`

注意这里的“越高越有利”不是自然语言先验，而是：

- `对根目的净贡献更大，并且更稳地守住安全边界`


## 9. 本文件的约束结论

后续实现中应固定以下结论：

1. 自我先天拥有的是概念生成机制，不是具体概念结论。
2. 概念由比较稳定的基础信息节点构成，可被自然语言描述，但不等于自然语言本身。
3. 概念必须从状态、动态、二次特征和因果中自动归纳。
4. 状态概念树应先由 `特征类型` 建立最抽象状态概念根，再按稳定区间逐层细分。
5. 单一特征值区间可生成 `特征状态概念`，多个状态概念的稳定交集可进一步生成 `复合状态概念`，多个状态概念的稳定特征组合还可生成 `复合特征概念`。
6. 特征状态概念、复合状态概念与复合特征概念都不应直接绑定某个具体存在。
7. `健康`、`病态` 这类复合状态概念，必须额外通过 `观察期稳定性` 与 `适用性` 检查。
8. 单次观察最多产生线索，不能直接形成稳定概念。
9. 稳定概念必须满足重复、一致、先因后果、可复现、可投影到根目的，并通过安全边界检查。
10. 稳定概念进入决策前，必须先明确适用边界与决策强度。
11. 所有稳定概念都需要命名，因为名称是高级逻辑思维与对外交流对准的基础；命名任务优先级可以按高级逻辑复用、对外沟通、跨模块复用、高风险区分等因素调节。
12. 概念可以升级，也必须可以降级和撤销。
13. `服务值越高越有利于安全保障` 这类判断，必须先经过根目的投影后才能成立。
