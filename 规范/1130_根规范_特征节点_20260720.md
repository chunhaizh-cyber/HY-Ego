# 1130 根规范 特征节点

编号：1130
日期：20260722
状态：正式根规范

## 1. 根定位

特征节点表示某一合法宿主上的具体实例特征槽位，连接宿主、正式抽象特征、当前特征值、历史值和特征关系。实例特征不另建平行主信息类型。

## 2. 本项目承载

实例特征由以下结构共同承载：

```text
节点类型为特征的实例槽位及稳定句柄
+ 宿主到槽位的唯一归属关系
+ 槽位到特征定义的唯一模板关系
+ 槽位到当前特征值的唯一当前值关系
+ 历史值、状态和来源证据关系
+ 主键与定位索引
```

宿主必须是规范允许拥有特征的稳定对象。允许的宿主类型为存在、场景、需求、任务和方法；新增宿主类型必须先修订正式规范。

## 3. 事实与边界

1. 一个实例槽位必须引用一个正式特征定义；语素、枚举或显示名称不得替代该定义身份。
2. 同一宿主与同一特征定义在同一当前版本下只能有一个有效实例槽位。
3. 实例槽位可以引用当前值、历史值和相关状态证据，但不得复制特征值原始容器。
4. 特征解释必须来自特征定义、二次特征、状态、动态或因果证据，不得写入自由文本业务字段。
5. 仅凭语素或类型标签不能成立实例特征；正式写入必须先解析特征定义节点。

## 4. 写入、读取与验证

实例槽位和特征值关系只能由特征领域服务写入。特征领域服务负责宿主类型、定义身份、唯一槽位和值来源准入；数据操作层负责同一写入会话中的节点、主信息、关系、原始材料和索引发布。

需求、任务、方法及其它高级服务只能经特征领域服务读取宿主特征状态，不得直接访问特征值服务。有效读回必须同时证明宿主关系、模板关系、当前值关系及各端点版本一致。

## 5. 生命周期与当前性

槽位身份随宿主和特征定义长期可追溯。当前值替换不得改写历史事实；旧值转为历史后仍保留来源和版本。宿主、定义或值关系发生版本漂移时，旧读回不得继续冒充当前状态。
