# 语素系统规范

编号：004
日期：2026-07-05

## 1. 定位

本规范用于定义 `海中鱼巣` 第一版语素系统的项目级规则。

语素系统只负责把外部词面材料转成项目内部结构身份入口，不直接裁决世界事实、任务状态、方法可执行、需求满足或因果结论。

本规范从整个项目结构出发，约束语素入口、准入、绑定、候选查询、显示派生和跨领域服务使用边界。

旧项目语素材料只能作为参考来源和兼容差异材料，不决定本规范的结构边界。确认前不得据此实施代码。

## 2. 主要用途

语素系统的主要用途是给其他领域服务提供稳定语言入口。

具体用途：

```text
把词面材料变成语素入口身份
阻断未理解文本直接进入机器事实
把已理解入口绑定到概念节点或目标节点
为名称、类型、别名和查询提供候选入口
为显示层提供只读显示来源
把查询冲突显式交给上层领域复判
```

语素系统不是自然语言理解系统本体，不负责创建现实世界事实，不负责结算需求，不负责选择方法，不负责产生因果结论。

## 4. 实现方法

语素系统必须使用新项目结构实现，不复制旧项目双向链表语素树，不复制旧主信息字段布局，不复制旧 SQL 投影实现。

实现方法：

```text
语素入口身份 -> 节点
入口到概念节点 -> 引用关系
入口到对应信息节点 -> 引用关系
词面定位入口 -> 索引或可重建索引
候选查询 -> 运行期只读候选包
冲突结果 -> 运行期只读候选包
显示标题 -> 只读派生文本
```

主信息只允许保存身份、值容器或明确声明的非权威缓存。语素业务含义必须落到节点、关系、索引、特征、状态或候选结构中。

语素服务的入口拒绝不得依赖事后校验。需要诊断入口拒绝原因时，只能使用项目统一容错检查宏：

```text
HY_EGO_ENABLE_FAULT_TOLERANCE_CHECK
```

宏开启后可以日志记录和弹窗提醒；日志和弹窗只做人读诊断，不得作为语素入口、候选、查询、显示或任何机器事实来源。

## 5. 对外服务

语素系统对外只通过 `语素服务` 提供服务。

第一轮服务：

```text
判断文本是否可作为最小词单元
创建概念入口
读取入口绑定目标
读取入口绑定目标组
读取入口概念追溯
读取入口概念追溯组
追加语素对应信息
追加语素概念追溯
判断语素入口同一
```

后续候选服务：

```text
创建实例入口
绑定别名入口到节点
枚举指向节点的语素入口
按名称入口查节点
按类型入口查节点
按名称类型查节点
生成查询冲突候选包
生成节点显示标题集合
```

外部服务调用方只能消费：

```text
语素入口句柄
候选集合
冲突候选包
只读显示标题
准入状态
```

外部服务调用方不得直接写语素节点、主信息、关系、索引或显示文本。

## 6. 项目口径

语素系统在新项目中的稳定口径：

```text
语素入口只作为词面到结构身份的桥接
词面不是业务事实
已理解入口必须能绑定项目内部节点
概念入口必须能追溯概念节点
查询只能返回候选
显示只能只读派生
```

## 7. 改写口径

旧项目语素结构进入新项目时，必须服从以下兼容映射。本节只约束旧项目材料如何进入新项目，不限制语素系统规范本身的项目级范围。

| 旧项目概念 | 新项目承载 |
| --- | --- |
| 词节点 | 语素入口创建前的词面材料或非权威显示来源 |
| 语素入口节点 | 节点仓库中的语素节点 |
| 语素入口主信息字段 | 主信息只保存身份 / 值容器 / 非权威缓存；业务含义改入关系、索引、特征或候选结构 |
| 对应基础信息 | 语素入口节点到对应信息节点的引用关系 |
| 语素树遍历查询 | 语素服务只读候选查询 |
| SQL 语素投影 | 暂缓到显示和持久化阶段，只能作为诊断 / 查询材料 |

## 8. 禁止行为

以下行为在新项目语素系统中禁止：

```text
裸字符串直接参与机器判断
显示文本直接参与机器判断
SQL / 控制面板投影反向裁决机器事实
外部模块直接写语素主信息字段
外部模块直接遍历语素树形成正式查询契约
用别名列表、类型列表、义项列表或显示名字段承载机器事实
把查询候选静默选第一个作为事实
```

## 9. 暂缓能力说明

本节列出暂缓能力的原因，避免把未实现能力误认为第一版缺陷：

| 暂缓能力 | 暂缓原因 | 恢复条件 |
| --- | --- | --- |
| SQL / 控制面板语素投影 | 第一版运行期以内存结构为准，显示和持久化不裁决机器事实 | 显示层只读规范和持久化规范确认后 |
| 运行节点入口直达查询 | 容易绕过领域服务复判 | 查询候选包和领域服务复判契约确认后 |
| 短语组合语义解析 | 需要语言解析和组合语义设计 | 最小词单元、概念入口和实例入口稳定后 |
| 完整自然语言理解 | 超出语素入口职责 | 语言理解系统另行建立规范后 |
| 完整上位类型推导 | 需要类型关系和推导规则 | 类型关系规范确认后 |
| 显示标题统一接入控制面板 | 控制面板暂缓 | 显示层只读视图设计完成后 |

暂缓能力不得作为第一版语素系统验收条件。

## 10. 结构承载

第一版语素系统使用以下结构承载：

| 业务概念 | 项目承载结构 | 身份 / 值类型 |
| --- | --- | --- |
| 语素入口身份 | 节点 | 节点句柄 |
| 词面材料 | 外部输入材料或非权威显示来源 | 文本材料，不作为机器事实 |
| 对应信息节点 | 长期基础信息类节点、需求、任务或方法节点 | 节点句柄 |
| 语素入口对应关系 | 语素节点到一个或多个对应信息节点的引用关系 | 引用关系集合 |
| 已理解入口 | 语素节点 + 绑定关系 | 语素节点句柄、对应信息节点句柄、概念节点句柄 |
| 概念入口 | 语素节点 + 一个或多个概念节点引用关系 | 引用关系集合 |
| 实例入口 | 语素节点 + 实例节点引用关系 + 概念追溯关系 | 引用关系 |
| 别名 | 多个语素入口指向同一目标节点 | 引用关系 / 索引 |
| 查询候选 | 运行期只读候选包 | 候选节点句柄 + 命中来源 |
| 冲突候选 | 运行期只读候选包 | 候选集合 + 来源 |
| 显示标题 | 只读派生文本 | 非机器事实 |


读取方：

```text
语素服务
```

## 12. 准入规则

正式语素入口必须满足：

```text
输入材料是自然语言最小词单元，或已经被上层解析为可绑定入口。
入口创建前必须已有对应信息节点。
现阶段对应信息节点只能是长期基础信息类节点、需求、任务或方法；场景内临时实例状态和实例动态不得直接绑定。
入口必须绑定项目内部对应信息节点。
入口能追溯到概念节点，或明确登记为待理解候选。
准入结果不等于世界事实成立。
```

未理解文本、短语、组合系统标签和说明性长文本不得直接成为正式语素入口。没有对应信息节点时，不得先创建正式语素入口等待后补。

## 13. 查询规则

语素查询只能返回候选：

```text
按名称入口查候选
按类型入口查候选
按名称和类型交集查候选
枚举别名候选
生成冲突候选包
```

候选必须由对应领域服务复判。语素服务不得裁决需求满足、任务完成、方法可执行、世界事实成立或因果结论。

## 14. 显示规则

显示标题只读派生。

显示标题、临时标志、SQL 行文本和控制台输出不得写回：

```text
名称
类型
语素入口
节点
关系
特征
特征值
需求
任务
方法
动态
因果引用
```

## 15. 验证方式

规范确认后，代码实施至少需要验证：

```text
已有基础信息类节点时可创建语素入口。
已有需求、任务或方法节点时可创建语素入口。
特征值节点不能作为语素入口对应信息节点，除非后续特征值系统规范另行确认。
场景内临时实例状态和实例动态不能作为语素入口对应信息节点，只有抽象状态和抽象动态可以绑定。
没有对应信息节点时不能创建正式语素入口。
默认构建不生成容错日志、不弹窗；启用容错检查宏后，入口拒绝路径触发日志记录和弹窗提醒。
短语不能创建正式语素入口。
概念入口能回查概念节点。
同一语素入口可回查多个对应信息节点。
同一语素入口可回查多个概念追溯节点。
实例入口重复绑定第二实例时返回冲突或失败。
两个别名入口能指向同一目标节点并可反向枚举。
同名多候选返回候选集合，不静默选第一个。
显示标题变化不改变入口写入结果。
```
