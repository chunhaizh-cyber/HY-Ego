# 4020 子规范 主信息身份生命周期与字段边界

编号：4020
日期：20260721
状态：正式子规范

## 1. 定位

本规范定义主信息的稳定身份、记录生命周期、拓扑锚点、值容器和字段准入红线。主信息是节点身份的最小附属结构，不是各类业务字段的集合，也不是通用业务事实表。

直接上级规范：0050。

直接依赖规范：0500、4010。

相关边界规范：4030、4110、4130、4060。

## 2. 身份与记录

主信息稳定身份由完整主信息句柄承载：

```text
仓库编号 + 主信息编号 + 版本号
```

主信息记录只允许具有以下通用职责：

```text
主信息编号与生命周期版本
记录状态
节点引用的主信息锚点
最小拓扑锚点
受控原始值容器
明确登记且可重建的非权威缓存
```

主信息编号和拓扑锚点编号使用无符号 64 位身份语义；`0` 保留为无效。主信息编号在同一仓库内从非零值单调分配且永久不复用。主信息句柄的版本使用无符号 32 位身份语义，`0` 无效。

节点记录可以持有主信息句柄作为锚点。节点发布前必须复核该句柄属于目标仓库、版本当前且记录有效；主信息字段不得重复保存节点类型或用值槽替代节点身份。

## 3. 生命周期

主信息生命周期遵守：

1. 新记录取得非零主信息编号、初始非零版本和 `有效` 状态后，才可被节点候选引用。
2. 普通读取和写入只接受仓库编号、主信息编号、版本均匹配且记录状态为有效的句柄。
3. 删除推进记录状态与生命周期版本，使删除前句柄退出普通读取；原主信息编号永久保留，不得回收或复用。
4. 已删除主信息只允许由显式审计或恢复治理入口读取，不得作为新节点的当前主信息锚点。
5. 恢复不得把已删除记录原地改回有效；恢复合同只能重新准入历史候选，或以新编号形成新身份，绝不得复用原编号、让旧句柄恢复当前有效或绕过删除生命周期。
6. 主信息版本只表达生命周期当前性，不因普通值写入、缓存命中或显示更新而承担业务版本。
7. 节点删除、关系清理和主信息删除需要共同收口时，必须先冻结精确影响面并遵守统一事务规范；不得先删主信息留下仍可读节点锚点。

## 4. 拓扑锚点边界

拓扑锚点只为结构定位提供最小编号材料，不得替代正式关系。两个对象之间的归属、父子、引用、模板、语素对应、任务承接、方法选择或因果来源必须进入关系或对象专用结构。

拓扑锚点的非零值只有在具名用途、写入方、读取方、生命周期和恢复合同全部明确后才可写入。无法从锚点值还原对应正式结构时，不得把它作为机器裁决依据。

## 5. 值容器语义

主信息值容器只承载经对应对象规范授权的原始数值材料。当前通用槽位语义区分“未建立”和一个合法整数值；未建立不等于数值 `0`，不得用默认零伪造已有材料。

值容器允许的候选范围仅包括：

```text
对象规范明确要求的原始 I64 值
有格式版本和角色定义的固定数值材料
拓扑或身份无法直接表达但已证明需要随主信息保存的最小原始量
明确标注非权威、可确定重建且清空不改变业务事实的缓存值
```

值容器不自动赋予数值业务含义。相同槽位索引只有在同一节点类型、主信息格式版本和专用读取合同中才可解释；调用方不得跨对象复用槽位含义，也不得把槽位数量、是否非空或某个裸值直接解释为完成、满足、可执行、稳定或成功。

需要向量、结构化材料或节点句柄组时，必须优先进入特征值系统、关系或专用不可变结构。不得拆分指针、句柄或自由文本后塞入通用 I64 槽位来规避正式结构准入。

## 6. 字段准入与分流

新增或解释主信息字段 / 槽位前必须同时给出：

```text
所属对象和主信息格式身份
字段用途、数值类型和值域
槽位索引或结构角色
合法写入方和读取方
来源材料与业务结构的关系
是否权威以及为何不能由已有正式结构表达
生命周期、当前性和版本
更新、删除、序列化、恢复与未知版本拒绝
验证方式
```

任一项缺失时不得新增或解释该字段。通用分流固定为：

| 内容 | 唯一优先承载 |
| --- | --- |
| 对象身份与类型 | 节点和稳定句柄 |
| 两个或多个对象的联系 | 关系或二次特征 |
| 对象属性与具体值 | 特征、特征值和实例槽 |
| 时点复合情况 | 状态 |
| 前后变化 | 动态 |
| 原因、结果和证据 | 因果结构 |
| 目标、授权承接、能力 | 需求、任务、方法 |
| 查询加速 | 索引或非权威缓存 |
| 人读内容 | 文档、显示或日志 |

## 7. 主信息红线

下列内容不得成为主信息权威字段或通用值槽语义：

```text
对象属性和观测事实
名称、词面、显示标题或自由文本语义
语素多对应或概念追溯关系
状态、动态、二次特征或因果解释
需求目标、需求满足和价值结算
任务阶段、任务完成和任务结果
方法条件、方法结果、方法成功或可执行结论
缓存命中、统计画像、风险判断和治理结论
日志、控制台、SQL 或控制面板投影
```

不得因为读取频繁、布局方便、既有来源含有同类字段或存在兼容槽位，就把上述事实复制到主信息形成第二权威账。非权威缓存必须使用独立命名域和版本，能够从正式结构确定重建，清空或过期不得改变业务裁决。

## 8. 写入与读取权

主信息仓库只执行句柄、版本、状态、槽位和最小记录不变量。领域对象的主信息格式、槽位含义和业务准入由该对象的领域服务负责；多结构写入由服务专用数据操作在统一事务内完成。

外部调用方、显示、日志、控制台、SQL 投影和恢复材料不得直接写主信息业务值。领域服务也不得用主信息值容器绕过特征、状态、动态、关系、因果、需求、任务或方法服务的正式写入口。

读取主信息值后仍必须结合节点类型、格式版本、来源关系和对应对象规范复核。裸主信息记录或单个槽位不得独立成为业务结果。

## 9. 持久化与验证

主信息快照或恢复材料必须保留仓库编号、主信息编号、版本、记录状态、拓扑锚点、值容器空值位置和对应格式版本。遇到未知格式、槽位数量不符、必填槽为空、值域越界、旧句柄、节点锚点不一致或禁止字段语义时，必须在发布前拒绝。

验证至少覆盖：

1. `0` 编号 / 版本拒绝，主信息编号单调且删除后不复用。
2. 删除推进版本，旧句柄普通读取和写入均失败，审计读取不恢复当前事实。
3. 节点只能锚定当前有效主信息。
4. 未建立槽与合法零值可区分；越界槽、未知格式和值域错误在写前拒绝。
5. 需求、任务、方法、状态、动态、因果和多路关系不能仅凭主信息槽裁决。
6. 缓存清空和重建不改变权威结构。
7. 恢复材料逐项复核身份、状态、格式和槽位后才允许发布。
