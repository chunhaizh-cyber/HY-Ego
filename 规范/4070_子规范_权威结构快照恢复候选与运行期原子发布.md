# 4070 子规范 权威结构快照恢复候选与运行期原子发布

编号：4070
日期：20260722
状态：正式子规范

## 1. 定位

本规范统一规定运行期权威结构的快照保存、恢复候选隔离、全量互证、索引重建和运行期上下文一次性原子发布。它不规定具体文件格式、数据库产品、模块、类或函数。

直接上级规范：0050。

直接依赖规范：4010、4020、4040。

相关边界规范：4030、4060、7130、8300。

## 2. 长期保存范围

宣称完整权威结构快照时，必须保存同一快照代次下的：

```text
节点、节点稳定主键、完整句柄、版本和记录状态
全部有效、已失效和历史关系及其端点、角色、顺序和版本
各领域类型化数据记录、记录类型、所属节点、版本和状态
对象规范要求保留的历史事实、换代证据和不可变业务账
格式版本、快照代次、来源运行期身份和完整性校验材料
```

子存在、实例特征、状态和动态的“列表”，以及定位索引、缓存、摘要、显示投影和组合读取材料不是独立权威保存项。它们必须由节点、关系和领域类型化记录恢复后重建。快照可携带可丢弃加速材料，但不得因其存在而省略权威来源。

缺少失效关系、历史事实或必需领域记录的材料只能声明局部材料包，不得声明完整快照或完整跨重启恢复来源。

## 3. 恢复候选与代次隔离

文件、数据库、事件流和外部备份读取结果首先都是恢复候选。候选必须绑定唯一恢复批次和目标运行期域，在隔离环境中装载；当前运行期仓库、索引、投影和服务引用在最终发布前保持不变。

不得把不同快照代次、不同运行期域、不同格式版本或无法证明同源的增量片段拼成一个当前候选。需要事件重放时，必须从已通过准入的基线开始，按连续序号和明确规则重放；缺段、重复异义或顺序冲突均拒绝。

## 4. 全量恢复顺序

恢复顺序固定为：

```text
读取并校验格式、来源和快照代次
-> 装载节点、稳定主键、句柄、版本和记录状态到隔离候选
-> 装载有效、失效和历史关系
-> 装载各领域类型化数据记录和历史事实
-> 校验编号高水位、主键唯一性、关系端点和记录所属节点
-> 校验全部结构不变量和领域业务不变量
-> 从权威候选确定重建索引、列表和组合读取投影
-> 复核重建结果不产生新事实且与权威候选一致
-> 一次性原子发布完整运行期上下文
-> 通过正式服务按稳定身份完成发布后读回
```

任何阶段不得让普通读取者观察候选，也不得用当前运行期结构补齐候选缺项。索引重建、投影组合和缓存生成只能读取候选中的完整权威来源。

## 5. 拒绝与失败收口

下列任一情况必须在发布前拒绝：

```text
未知格式、未知记录类型、未知节点或关系 ABI
快照代次不一致、事件缺段或来源域不符
节点稳定主键为空、重复或与节点身份不一致
句柄、版本、记录状态或编号高水位非法
关系端点悬空、角色或基数不合法
领域记录缺失、错类型、错所属节点、错版本或值域越界
必需失效关系、历史事实或不可变业务账缺失
索引、列表或读取投影无法从权威候选确定重建
任一结构不变量或业务不变量互证失败
```

失败时销毁本批次恢复候选并保留原运行期上下文不变。禁止部分替换仓库、先发布节点后补关系、先发布索引后补事实、把缺项降级为默认值或用日志与扫描结果修复候选。

## 6. 原子发布与恢复后读回

只有全部候选、全部领域记录和全部重建索引已经验证完成，且发布步骤本身不再分配业务资源、不再执行可失败准入时，才可进入原子发布区。发布必须一次替换完整运行期上下文；服务、线程和普通读取者不得持有跨发布可变内部引用。

发布后按稳定身份通过正式服务读回节点、关系、领域记录和关键组合投影。读回不一致属于内部逻辑错误，不得回退到半旧半新上下文，也不得把“文件读取成功”“数据库连接成功”或“索引数量相同”解释为恢复成功。

## 7. 旧快照处理

旧快照若依赖已退出的通用附属信息、拓扑锚点或无类型值槽，只能采取以下之一：

1. 存在经正式设计确认的离线迁移器，能够无歧义地产生当前节点直接字段、正式关系和领域类型化记录，再把结果作为全新恢复候选重新准入；
2. 无法无歧义映射时明确拒绝，并保留只读审计能力。

不得在生产恢复入口中长期保留双轨权威、运行时猜测槽位含义或为兼容旧快照重新建立已退出结构。

## 8. 验证合同

验证至少覆盖：

1. 节点稳定主键为空或重复时恢复拒绝。
2. 节点、关系和领域记录分别合法但全量图互证失败时不发布。
3. 缺失失效关系、历史事实或必需领域记录时不能声明完整恢复。
4. 清空全部索引和列表投影后可从权威候选确定重建等价结果。
5. 恢复任一步失败后原运行期上下文、当前索引和服务读取结果保持不变。
6. 原子发布前普通读取不可见候选，发布后只能观察完整新上下文。
7. 旧无类型槽材料没有正式迁移映射时被拒绝。
8. 发布后按正式服务读回的身份、版本、关系和领域记录完全一致。
