# 0250 启动与退出健康态规则

## 1. 目标

本文用于把启动链、初始化链、健康链、故障链、退出链的判定收紧成统一规则，禁止再用“线程壳子还在”代替“系统真的活着”。


## 2. 适用范围

本文适用于：

- 自我线程
- 启动初始化链
- 调度主链
- 退出与销毁链
- UI / 日志 / 对外状态展示


## 3. 核心定义

### 3.1 已启动

`已启动` 的定义必须固定为：

- 实际执行线程已经进入主生命周期
- 线程处于 `初始化中 / 运行中 / 退出中 / 故障` 之一

必须明确：

- `已启动` 不等于 `初始化完成`
- `已启动` 不等于 `健康运行`
- `joinable()` 只能说明线程句柄存在，不能单独作为 `已启动` 判据


### 3.2 初始化完成

`初始化完成` 的定义必须固定为：

- 根需求已建立
- 根任务已建立
- 调度器已准备好
- 启动主链所依赖的核心对象已进入可工作态

必须明确：

- 只要上述关键条件缺一，就不得标记为 `初始化完成`
- 线程活着但主链未就绪，状态仍只能算 `初始化中`


### 3.3 健康运行

`健康运行` 的定义必须固定为：

- `运行状态 == 运行中`
- `初始化完成 == true`
- `自我核心未消亡`

收敛成判定式：

`是否健康运行() = 运行状态_ == 运行中 && 初始化完成_ == true && !自我核心_.是否已消亡()`

必须明确：

- `线程活着` 不是 `健康运行`
- `初始化完成` 也不是 `健康运行`
- 只要核心已消亡、关键链路异常、或运行状态已转故障，就不得继续对外宣称健康


### 3.4 故障态

`故障态` 的定义必须固定为：

- 初始化失败
- 调度失败
- 消息处理失败
- 周期维护失败
- 收尾失败
- 自我核心消亡
- 其他使主链失去可信运行能力的异常

故障态至少必须带出：

- `运行状态 = 故障`
- `最近故障摘要`
- 可见日志

必须明确：

- 故障态不是“线程还没退完”
- 故障态一旦成立，就不得继续对外展示为正常运行


## 4. 启动与退出强制规则

1. `是否正在运行()` 只表示线程真实活着，不表示系统就绪，不表示系统健康。
2. `是否初始化完成()` 必须独立判定，不得从线程存在性推导。
3. `是否健康运行()` 必须独立判定，不得从 `是否正在运行()` 或 `是否初始化完成()` 直接替代。
4. 对外状态展示至少必须包含：`线程活着 / 初始化完成 / 健康运行 / 最近故障`。
5. 退出链路必须走“先停止、再等待线程结束、最后销毁对象”，不得再出现“请求退出后对象失管”。
6. 对象销毁不得早于线程真实结束。


## 5. 禁止事项

以下做法必须禁止：

- 用 `joinable()` 直接对外宣称“系统运行正常”
- 用“线程对象非空”直接对外宣称“已启动完成”
- 用单个 `是/否` 文案概括自我线程整体健康态
- 请求退出后立刻 `release()` 造成对象失管


## 6. 落地要求

实现层必须至少提供以下对外接口：

- `是否正在运行()`
- `是否初始化完成()`
- `是否健康运行()`
- `最近故障摘要()`

UI、调试面板、日志诊断、自动巡检都必须基于这组状态，而不是只看线程对象是否存在。
