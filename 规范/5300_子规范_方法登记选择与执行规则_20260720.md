# 5300 子规范 方法登记选择与执行规则

编号：5300
日期：20260721
状态：正式子规范

## 1. 定位

本规范展开方法登记、角色、规格、能力、条件结果对、召回选择、运行事实、任务特征声明、动作动态和学习证据合同。

直接上级规范：3300。

直接依赖规范：1160、1170、1190、3200、4010、4020、4210。

相关边界规范：4220、5230、5310、5320、5340。

## 2. 方法结构角色

方法领域必须能够区分并验证：

| 角色 | 责任 |
| --- | --- |
| 方法登记根 | 管理方法登记项和当前生命周期集合 |
| 方法首 | 稳定方法身份、唯一治理虚拟存在和结构根 |
| 方法条件 | 输入、作用对象、目标特征、期望状态和来源版本 |
| 方法结果 | 输出、作用对象、目标特征、目标状态和来源版本 |
| 条件结果对 | 在确定条件下产生确定结果的样本、统计和稳定证据 |
| 动作入口 | 可调用能力、输入场景、输出场景、稳定动作键和当前状态 |

结果必须归属于具体方法并与适用条件保持可追溯关系。同一结果在不同条件下的统计不得混并。

动作入口的输入 / 输出场景使用 4010 锁定的关系类型 17。源必须是动作入口方法节点，目标必须是场景；顺序号 1 表示输入场景，顺序号 2 表示输出场景。同一完整源动作入口和同一角色至多一条有效关系，同一场景可以同时承担输入与输出两个不同角色。

方法领域服务是关系 17 的唯一业务写入与解释方。未知顺序号、错误端点类型或同源同角色第二条有效关系必须在写前拒绝；普通引用槽位即使同为 1 / 2，也不得替代专用关系 17 或据此裁决动作入口完整。

## 3. 方法治理虚拟存在

每个方法首拥有唯一治理虚拟存在，承载长期身份、能力、方法状态、允许召回/执行、任务特征声明、运行摘要、统计和学习证据。

治理虚拟存在不得承载某个任务的参数、授权、筹办结果、执行回执或本轮运行细账。不同方法必须以真实动作、目标、条件、结果或作用范围区分；语义完全等价时应复用或合并，而非只换主键重复建立。

## 4. 条件、结果和条件结果对

1. 条件项和结果项必须声明作用对象；需要保持同一对象时使用强类型绑定关系，不用文本绑定名作权威。
2. 条件和结果规格必须关联场景、目标特征、抽象状态、来源版本和当前性。
3. 方法可以有多个条件、多个结果和多组条件结果对；单个结果命中当前任务目标即可进入候选，其它结果仍按 5120 分类。
4. 条件或结果不可比较、作用对象不同、方向不同或规格版本不同的样本不得融合。
5. 候选条件结果对必须显式标记候选；只有真实运行反复验证后才能升格为稳定。

## 5. 方法状态和证据

方法状态必须是强类型方法特征，不使用“缺动作、缺条件、不可用”等自由文本作为状态值。结构缺口、运行次数、成功次数、连续成功、成熟度和条件结果对数量是判定证据或解释材料。

方法“可用”至少要求当前动作入口和最低执行规格成立；“好用”还要求稳定复现、质量和约束证据。具体值域、阈值和迁移矩阵必须在独立方法状态子规范形成后生效，不得由实现常量反向定义。

### 5.1 方法生命周期关系

方法生命周期使用 4010 锁定的关系类型 23。源必须是方法首，目标必须是主体为该方法唯一治理虚拟存在的完整实例状态事实；该状态仍须按 4210 具备场景、特征、值、发生时间和来源，不得以登记根共享抽象状态、枚举、普通 `模板` 关系或无类型槽代替。

同一方法恰有一条有效当前关系 23。顺序号从 `1` 连续递增并只表达方法生命周期版本；有效唯一键为“关系类型 23 + 完整方法句柄”，历史唯一键再加生命周期顺序号。迁移必须形成新的完整状态事实和新的关系 23，失效旧当前关系并保留审计；不得把同一关系重挂到另一个共享状态，也不得改写旧状态来伪造新版本。

方法领域服务及其专用数据操作是关系 23 的唯一业务写入与解释方。写前必须复核方法身份、唯一治理虚拟存在、生命周期特征和值域、前驱关系、连续顺序号、发生时间和来源；前置通过后创建、失效或读回异常属于内部逻辑错误。方法内容版本与关系 23 的生命周期版本保持分离，二者都必须可被 5230 的任务选择冻结精确读取。

## 6. 任务特征声明

方法必须声明任务执行它时需要挂载的特征类型，包括输入、参数、阶段、授权、安全、禁止项、验证、归因、失败处理和输出路由。

组合方法还必须声明跨方法协调、顺序、并行、回流和失败收束特征。任务只挂载这些正式声明，不得自行发明。

## 7. 召回与选择

1. 召回先按结果能力获取候选，再回读权威规格和生命周期复判。
2. 选择权属于取得任务筹办执行权的任务领域；方法服务和召回索引不提交任务选择。
3. 选择优先级依次是结果一致、条件与参数闭合、禁止和风险、运行质量与成本。
4. 无候选时不得创建方法首，不得使用通用写入、内核原语或名称匹配兜底。
5. 方法不可用或规格不完整时，只能形成具体缺口、等待或内部错误，不能伪装成已选择可执行方法。

## 8. 运行事实与返回账

每次真实方法运行必须具有独立运行身份，至少关联方法、任务或调用上下文、输入场景、输出场景、开始与结束时间、动作动态、执行摘要和当前版本。

方法治理虚拟存在只同步长期统计和最近摘要；本轮前后值、参数和输出留在运行事实及场景中。

动作运行成功不等于目标结果符合；结果符合/不符合统计只有在目标、输出和验证合同可比较时才能更新。

## 9. 动作动态和因果

主动动作真实调用方法时必须生成动作动态；被动状态提交由具名提交方法生成被动动作动态；非业务派生整理不生成主链动作动态。

方法未执行、条件不满足、只形成候选或只收取消息时不得伪造真实动作动态。线程不是动作来源。

状态变化线索可帮助发现前置和联动，但只有动作动态、条件、结果和复现证据共同成立时，才能支持完整动作因果和方法能力升格。

## 10. 学习和写入权

方法可以输出候选结构、样本和结构化缺口，但不能自行创建需求或任务。目标方法长期账只由目标方法自身执行路径或方法领域正式提交入口更新。

方法领域服务是登记、规格、条件结果对、动作入口、生命周期和长期账的唯一业务写入方；外部模块不得直接写仓库关系。

方法内容版本必须与生命周期版本、规格来源版本和任务选择快照保持身份分离。学习候选或晋级服务不得直接改写方法；只有方法领域服务可以在独立晋级通过后发布不可变的新代际方法身份。新代际不得原地覆盖来源方法，具体晋级、父代、可见性和同轮隔离由 5340 唯一规定。

## 11. 验证

验证必须覆盖角色端点、唯一治理虚拟存在、关系 23 当前唯一与连续历史、完整生命周期状态事实、生命周期和内容版本分离、索引复判、多条件多结果、不同条件下相同结果不混账、任务特征来源、动作动态来源、运行账分离和无候选边界。
