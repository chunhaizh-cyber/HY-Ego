# 仓库逻辑空间与领域树图结构规范

编号：仓库逻辑空间-001
日期：2026-07-06
状态：已确认项目规范 / 第一轮仓库逻辑空间与领域树图口径

## 1. 目的

本规范用于明确 `海中鱼巣` 第一版数据仓库和逻辑树图结构的组织口径。

项目第一版不按世界、语素、需求、任务、方法或特征值分别建立互不相通的物理仓库。运行期核心保持统一底层仓库，并在统一身份、统一关系、统一索引和领域服务边界上组织多棵逻辑树或逻辑图。

本规范只确认结构组织规则，不代表代码已经实现，不代表旧项目能力已经迁移，不代表数据库持久化、控制面板、外设或体素链路已经接入。

## 2. 分层口径

第一版仓库体系分为四层：

| 层级 | 责任 | 禁止事项 |
| --- | --- | --- |
| 物理层 | 统一节点仓库、主信息仓库、关系仓库、索引仓库和写入事务 | 不按每棵逻辑树复制一套不兼容仓库 |
| 身份层 | 统一句柄、仓库编号、本地编号和版本号 | 不为世界树、需求树、任务树或方法树另建身份体系 |
| 服务层 | 按领域服务分层封口写入和读取 | 不绕过领域服务直接写底层仓库形成业务事实 |
| 逻辑层 | 通过节点类型、关系类型、索引入口和服务入口组织特征值结构、世界图、语素入口图、需求树、任务树、方法树和动态因果证据图 | 不把索引、日志、显示文本或主信息字段当作树图事实源 |

## 3. 物理层统一规则

第一版物理核心只保留以下统一仓库：

```text
节点仓库
主信息仓库
关系仓库
索引仓库
写入事务
```

各逻辑结构不得自建以下不兼容物理仓库：

```text
世界节点仓库
语素节点仓库
需求节点仓库
任务节点仓库
方法节点仓库
特征值节点仓库
```

允许后续在容量、性能或持久化压力明确后，通过规范和详细设计增加仓库编号、分片仓库、空间编号、纪元编号、快照归档或冷热数据分层。此类扩展不得改变统一句柄语义，不得让某一逻辑树拥有独立事实裁决体系。

## 4. 身份层统一规则

所有长期对象身份统一使用：

```text
节点句柄
主信息句柄
关系句柄
索引句柄或索引记录编号
仓库编号
本地编号
版本号
```

统一身份规则：

1. `0` 保留为无效编号。
2. 同一仓库内编号只递增，不复用。
3. 稳定身份由仓库编号、本地编号和版本号共同判断。
4. 版本号只表达生命周期失效或身份重建，不表达普通业务值变化。
5. 业务变化必须进入特征值、状态、动态、任务、方法、因果引用或关系结构。
6. 指针只允许作为运行期缓存或临时引用，不得作为长期事实身份。

需求可以引用目标状态，任务可以引用需求和方法，方法可以引用条件材料或目标材料，语素入口可以引用多个信息节点。上述跨逻辑结构引用必须通过统一句柄和关系表达，不得创建跨树私有编号。

## 5. 树图承载规则

逻辑树和逻辑图的结构关系主要由关系仓库承载。

索引仓库只负责入口和候选召回，不承载树图事实，不裁决业务事实。

领域服务负责入口前置拒绝、写入边界、读取契约和业务语义，不得把写入权限下放给显示层、日志层、临时脚本或外部调用方。

统一口径如下：

```text
关系仓库表达树和图。
索引仓库帮助快速找到入口和候选。
领域服务决定能不能写、怎么写、怎么读。
节点、主信息、关系、状态、动态、因果引用和特征值结构承载机器事实。
```

禁止口径：

```text
索引 = 世界树
索引 = 需求树
索引 = 任务树
索引 = 方法树
主信息字段 = 需求目标
控制台输出 = 验证事实
日志内容 = 机器状态
```

## 6. 逻辑空间定位方式

第一版不新增物理仓库，不新增独立身份体系。

逻辑空间通过以下材料定位：

```text
节点类型
创建该节点的领域服务
根节点 / 入口节点关系
关系类型
索引命名空间
必要时的非权威空间标签
```

空间标签只允许作为诊断、候选召回或人读定位辅助，不得裁决业务事实，不得替代节点类型、关系类型、生命周期、入口前置拒绝或领域服务写入边界。

第一轮逻辑空间定位表：

| 逻辑空间 | 根 / 入口 | 主要定位材料 | 当前状态 |
| --- | --- | --- | --- |
| 特征系统 / 特征值结构 | 特征节点、特征值节点、特征状态材料入口 | 节点类型、特征服务创建路径、特征到特征值关系、特征索引命名空间 | 第一轮样例已验证；完整索引命名空间待实现 |
| 世界树 / 世界图 | 世界服务聚合入口、存在节点、场景节点 | 存在 / 场景节点类型、世界服务和原子服务创建路径、归属关系 | 第一轮样例已验证；完整世界根治理待后续 |
| 语素入口图 | 语素入口节点、概念入口节点 | 语素服务创建路径、语素对应信息关系、语素概念追溯关系、语素索引命名空间 | 多对应 / 多路追溯第一轮已验证 |
| 需求树 | 需求节点、需求目标状态关系 | 需求服务创建路径、需求到目标状态关系、需求到任务关系、需求索引命名空间 | 需求目标状态 DV-1 已验证 |
| 任务树 | 任务节点、任务承接需求关系、任务方法关系 | 任务服务创建路径、任务状态、任务到方法关系、任务索引命名空间 | 第一轮任务方法关系样例已验证；任务系统规范和详细设计已确认 / 未改代码 |
| 方法树 | 方法节点、任务方法关系、方法条件 / 结果材料 | 方法服务创建路径、方法节点类型、任务方法关系、方法索引命名空间 | 第一轮方法执行样例已验证；方法系统规范和详细设计已小修后确认 / 未改代码 |
| 动态 / 因果证据图 | 实例动态节点、抽象动态节点、轻量因果引用节点 | 动态服务和因果服务创建路径、动态材料、因果引用节点 | 发生时间戳和轻量因果引用第一轮已验证；可靠因果待后续 |

不得只靠索引命中、显示名称、日志文本、控制台输出或主信息裸字段推断某个节点属于哪个逻辑空间。

## 7. 索引边界

索引适合承载查找入口、候选召回和非权威定位材料。

第一版允许的索引用途包括：

| 索引用途 | 允许内容 | 裁决边界 | 当前状态 |
| --- | --- | --- | --- |
| 按主键查节点 | 主键到节点候选 | 仍需校验节点句柄、类型、生命周期和版本 | 当前已实现 uint64 主键索引基础能力 |
| 按语素查入口 | 语素入口候选、概念入口候选 | 多对应和多路追溯事实由关系仓库承载 | 规范确认待完整实现 |
| 按需求编号查需求 | 需求候选 | 需求目标、状态和结算事实由需求服务及结构承载 | 后续 |
| 按任务编号查任务 | 任务候选 | 任务承接、任务状态和任务方法关系由关系和状态结构承载 | 后续 |
| 按方法名称查方法 | 方法候选 | 方法可执行性由方法服务和任务方法关系裁决 | 后续 |
| 按特征标识查特征值候选 | 特征或特征状态材料候选 | 特征值读写必须经特征服务，特征值服务不得被高级服务直接访问 | 后续 |

索引读取结果必须被视为候选。调用方必须通过领域服务或仓库只读入口复核句柄有效性、节点类型、关系类型、生命周期和必要前置关系。

### 7.1 索引命名空间

统一索引仓库中，索引键必须带索引用途或命名空间。

不得把语素主键、需求编号、任务编号、方法名称、特征标识混在同一无类型主键空间中。

索引键建议包含：

```text
索引空间
键类型
键值
来源服务
```

索引键示例：

```text
语素:词面:杯子
需求:编号:1001
任务:编号:2001
方法:名称:写入特征状态
特征:标识:闭环能量值
```

第一轮仅实现 uint64 主键索引基础能力。领域命名空间索引属于后续能力，当前不得把后续索引用途宣称为已实现。

## 8. 关系边界

关系仓库是逻辑树和逻辑图的主要结构承载层。

第一版关系不得退化为单一普通父子关系。至少应区分以下关系语义：

```text
普通父子
归属关系
引用关系
索引关系
模板关系
运行期临时关系
因果来源关系
```

关系写入口必须在函数入口拒绝：

```text
无效源句柄
无效目标句柄
已删除句柄
错误节点类型
错误关系类型
违反该关系类型唯一性约束的重复关系
同一源节点、同一关系类型、同一目标节点的完全重复关系
未确认多目标语义时，同一源节点同一关系类型的多目标关系
悬空关系
越权写入方
缺少领域前置条件
```

已确认多目标语义的关系类型可以允许同源同类型多目标，例如语素多对应、多路概念追溯。读取时必须返回集合，不得静默选第一个目标作为事实。

节点移动统一视为普通父子关系重挂。通用移动逻辑只能落在关系仓库或后续确认的公共结构移动工具中；领域服务只做本领域前置拒绝后调用通用入口，不得重复实现移动算法。

## 9. 跨空间关系准入

逻辑空间之间可以互相引用，但不能任意写关系。跨空间关系必须由明确领域服务写入，并符合关系类型、节点类型、生命周期和前置关系要求。

第一轮跨空间关系准入表：

| 来源空间 | 目标空间 | 允许关系 | 写入方 | 说明 | 当前状态 |
| --- | --- | --- | --- | --- | --- |
| 语素入口图 | 世界图 / 基础信息 | 语素对应信息 / 引用 | 语素服务 | 支持一个语素入口对应多个信息节点，读取返回集合 | 第一轮已验证 |
| 语素入口图 | 概念 / 模板 | 语素概念追溯 / 模板 | 语素服务 | 支持多级、多路追溯，不恢复单值字段 | 第一轮已验证 |
| 需求树 | 特征系统 | 模板 / 引用 | 需求服务，经特征服务读取特征状态材料 | 表达需求目标状态、目标特征、当前特征状态材料 | DV-1 已验证 |
| 需求树 | 任务树 | 归属 / 引用 | 任务服务 | 任务承接需求，需求可枚举承接任务 | 任务系统规范和详细设计已确认 / 未改代码 |
| 任务树 | 方法树 | 引用 | 任务服务 / 方法服务 | 任务选择方法，方法执行权限来自任务授权、承接和任务方法关系 | 第一轮样例已验证 |
| 方法树 | 特征系统 | 引用 / 经特征服务写入 | 方法服务 | 方法执行不得直接访问特征值服务，必须通过特征服务请求修改特征状态材料 | DV-1 已验证 |
| 动态图 | 特征系统 | 因果来源 / 引用 | 动态服务 | 记录被改变目标、改变前值、改变后值和发生时间戳 | 第一轮已验证 |
| 因果证据图 | 动态图 | 来源动态准入证据 | 因果服务 | 轻量因果引用只用来源动态做入口准入检查，不持久化来源动态关系 | 第一轮已验证 |

未列入本表的跨空间写入必须先进入规范和详细设计，不得由实现时临时放开。

## 10. 逻辑空间

第一版统一底层结构上组织以下逻辑空间：

| 逻辑空间 | 主要承载 | 主要服务 | 说明 | 当前状态 |
| --- | --- | --- | --- | --- |
| 特征系统 / 特征值结构 | 特征、特征值、当前状态材料、历史状态材料、目标状态材料、特征到特征值的归属或引用关系 | 特征服务、特征值服务、状态服务 | 特征值服务只服务特征系统，只有特征服务可直接访问 | 第一轮样例已验证 / 完整增强待后续 |
| 世界树 / 世界图 | 存在、场景、场景成员、世界归属、基础信息类节点 | 世界服务、存在服务、场景服务 | 世界服务只做聚合编排，不绕过原子服务直接写基础信息结构 | 第一轮样例已验证 / 完整增强待后续 |
| 语素树 / 语素入口图 | 语素入口、概念入口、多对应、多路追溯、上位概念、概念模板候选 | 语素服务 | 多对应和多路追溯由关系仓库承载，不恢复单值主信息字段 | 多对应 / 多路追溯第一轮已验证 |
| 需求树 | 需求、目标状态、需求来源、需求结算、需求到任务关系 | 需求服务、状态服务、特征服务 | 需求目标是目标状态，不是 I64，不得塞回主信息通用字段 | 需求目标状态 DV-1 已验证 / 完整增强待后续 |
| 任务树 | 任务、任务状态、任务承接需求、任务授权关系、任务选择方法 | 任务服务 | 任务状态和承接关系由结构承载，不由线程、日志或控制台输出裁决 | 任务系统规范和详细设计已确认 / 未改代码 |
| 方法树 | 方法、方法入口、任务方法关系、方法条件、方法结果材料 | 方法服务、特征服务、动态服务 | 方法执行权限来自任务授权、任务承接和任务选择方法 | 方法系统规范和详细设计已小修后确认 / 未改代码 |
| 动态 / 因果证据图 | 实例动态、抽象动态、轻量因果引用、动态证据材料 | 动态服务、因果服务 | 轻量因果引用不等于稳定因果结论，单个因果引用不得扩大解释 | 第一轮样例已验证 / 可靠因果待后续 |

这些逻辑空间可以互相引用。跨空间引用必须使用统一关系和稳定句柄，不得复制对象，不得私建桥接事实表，不得用显示文本或日志拼接关系。

## 11. 领域服务封口

业务写入必须通过对应领域服务或正式仓库入口。

领域服务至少负责：

```text
入口前置拒绝
节点类型准入
关系类型准入
生命周期准入
越权写入拒绝
必要前置关系检查
结构写入编排
只读视图返回
验证读取路径定义
```

基础信息原子服务包括：

```text
特征值服务
特征服务
存在服务
场景服务
状态服务
动态服务
二次特征服务
因果服务
```

世界服务是基础信息聚合服务。需求服务、任务服务和方法服务是高级信息服务。语素服务是入口服务。

高级服务不得直接包含、接收或调用特征值服务。高级服务需要特征状态材料时，必须通过特征服务获得。

## 12. 主信息边界

主信息仓库不是旧项目主信息字段合集，也不是各逻辑树的事实字段表。

主信息允许保存：

```text
身份材料
生命周期材料
拓扑锚点
值容器
明确声明的非权威缓存
```

主信息不得直接承载：

```text
需求目标
任务结算
方法成功结论
世界事实裁决
语素多对应事实
因果可靠结论
显示层专用字段
日志诊断状态
```

凡是表达对象属性、状态变化、任务推进、方法执行、需求满足、因果解释或多路关系的内容，应优先进入特征、特征值、状态、动态、二次特征、因果引用、需求、任务、方法或关系结构。

## 13. 验证规则

本规范对应的验证不得只看日志、控制台输出、说明文字或索引命中。

验证必须读取以下结构之一或多项：

```text
节点记录
主信息记录
关系记录
索引候选及其复核结果
特征值结构
状态节点或状态材料
动态节点或动态材料
因果引用节点
需求节点及目标状态关系
任务节点及任务方法关系
方法节点及执行材料
```

完成声明必须区分：

```text
文档已生成
规范已确认
代码已实现
代码已编译
入口前置拒绝通过
最小闭环通过
旧能力迁移通过
真实外设样本通过
```

本规范确认后，只能声明为规范已确认，不得声明代码实现、入口拒绝、最小闭环、旧能力迁移或真实样本验证通过。

## 14. 风险和禁止扩大

必须防止以下风险：

1. 索引变成事实源。
2. 世界树变成万能树。
3. 主信息重新膨胀为旧字段合集。
4. 需求树、任务树和方法树拆成不相通的小体系。
5. 领域服务退化为薄转发，无法执行入口前置拒绝。
6. 关系仓库只剩普通父子关系，无法表达归属、引用、模板、运行期临时和因果来源。
7. 控制台输出、日志、说明文字或临时 JSON 被当作验证事实。
8. 特征值服务被需求、任务、方法绕过特征服务直接访问。
9. 逻辑空间被误解为物理拆仓，导致句柄体系分裂。
10. 根节点 / 空间入口缺失，导致逻辑树图只能靠约定或索引推断。

本规范不能作为以下完成证明：

```text
旧项目能力已迁移
自我循环已完成
自我苏醒完成
初步成熟完成
外设 / 体素 / 控制面板已接入
数据库持久化已接入
逻辑树图代码已经完整实现
```
