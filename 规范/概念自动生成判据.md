# 概念自动生成判据

## 1. 目标

本文用于固定“自我”如何从状态、动态、二次特征与因果信息中自动生成概念。

本文要解决的问题是：

- 什么是先天内置，什么不是先天内置
- 什么情况下允许生成概念候选
- 什么情况下允许把候选升级为稳定概念
- 什么情况下稳定概念可以参与需求、任务、方法决策

本文与 [自我根需求与方向对齐规则](D:\海鱼\规范\自我根需求与方向对齐规则.md) 配套使用。


## 2. 先天内置的内容

当前应固定：

- 先天内置的是 `概念生成机制`
- 不是先天内置 `具体结论`

也就是说，自我可以先天拥有：

- 记录状态的能力
- 比较状态的能力
- 生成动态的能力
- 提取二次特征的能力
- 判断因果强弱的能力
- 用根需求评估“有利/不利”的能力

但不应先天写死：

- `服务越高越好`
- `任务越少越安全`
- `某个名字天然是正面的`

这些都应由运行中自动归纳得到。


## 3. 概念生成的输入真值

概念生成不依赖名称，依赖以下输入：

1. 状态事实  
   例如：`安全值`、`服务值`、`待完成任务数`、`是否待机`

2. 动态事实  
   例如：`安全值上升`、`服务值下降`、`压力增大`

3. 二次特征  
   例如：`服务上升后安全恢复变快`、`服务为 0 时待机`

4. 因果信息  
   例如：`服务变化先于安全变化，且方向稳定`

5. 根需求投影  
   即：该关系是否有助于 `安全值最大化`

因此，概念的真值来源是：

- `可重复观察的状态变化`
- `可提取的二次特征`
- `可投影到根需求的因果关系`


## 4. 概念候选生成判据

### 4.1 候选的定义

概念候选不是正式概念，而是一个“值得继续观察的关系假设”。

可以理解为：

- `某对象可能对某目标有稳定作用`
- `某类变化可能对应某种方向`

### 4.2 候选生成的最小条件

以下条件至少满足三项，才允许生成概念候选：

1. 出现了可识别的对象  
   例如：`服务值`

2. 出现了可识别的作用对象  
   例如：`安全值`

3. 出现了可识别的变化关系  
   例如：`服务上升后安全恢复变快`

4. 该关系至少出现过两次  
   单次观察只能记为线索，不应直接升为候选

5. 该关系方向一致  
   例如两次都表现为“服务更高时安全更稳定”

6. 该关系可被二次特征表达  
   即能写成结构化关系，而不是仅靠一句自然语言描述

### 4.3 候选的结构

一个概念候选应至少包含：

- `作用对象`
- `被作用对象`
- `变化方向`
- `作用结果`
- `证据次数`
- `最近观察时间`
- `是否已投影到根需求`

### 4.4 候选示例

如果自我多次观察到：

- 服务值上升后，安全值恢复速度增加
- 服务值下降后，安全值衰减速度增加
- 服务值为 0 时进入待机

则可以生成候选：

- `服务值可能正向影响安全值`
- `服务值可能影响安全恢复能力`
- `服务不足可能导致待机`

注意：

此时仍然只是候选，不是正式概念。


## 5. 概念稳定化判据

### 5.1 稳定概念的定义

稳定概念指的是：

- 已经有足够重复证据
- 已具备较稳定因果方向
- 已能作为后续判断依据使用

### 5.2 从候选升级为稳定概念的条件

以下条件应同时尽量满足：

1. 重复出现  
   同类证据达到预设次数阈值

2. 方向一致  
   大部分证据的方向相同，不是正负混乱

3. 先因后果  
   作用因素变化先于结果变化出现

4. 条件相近时可复现  
   在相似背景下再次出现相同方向结果

5. 强度具有一致性  
   变化更强时，结果通常也更强

6. 能排除主要干扰  
   至少能把明显来自其它主因的样本剔除

7. 能投影到根需求  
   能判断它对 `安全值最大化` 是正贡献、负贡献还是中性

### 5.3 稳定化后的表达

稳定概念应从“可能”变为“当前成立”：

- 候选：`服务值可能正向影响安全值`
- 稳定概念：`服务值对安全值具有正向单调影响`

### 5.4 稳定概念的置信度

稳定概念仍不等于绝对真理。

因此稳定概念应带有：

- `证据量`
- `一致性`
- `可复现度`
- `当前置信度`

置信度不足时，可以保留为弱稳定概念，而不直接进入强决策。


## 6. 概念参与决策判据

### 6.1 决策参与的定义

概念参与决策，指的是它可以反向影响：

- 需求生成
- 任务生成
- 方法筛选
- 优先级调整

### 6.2 允许参与决策的条件

一个稳定概念只有同时满足以下条件，才允许参与决策：

1. 已稳定化  
   不能让纯候选直接驱动任务生成

2. 已完成根需求投影  
   必须知道它对安全根方向是正向、负向还是条件性正向

3. 作用边界清楚  
   知道它影响的是“恢复速度”“衰减速度”“待机风险”还是其它维度

4. 适用条件清楚  
   知道在什么背景下它成立，在什么背景下不可靠

5. 未被更新证据明显推翻  
   如果近期连续出现反例，应先降级再停止参与决策

### 6.3 决策参与的强弱层次

概念参与决策应分层：

1. 弱参与  
   只影响排序和提示，不直接生成或否决任务

2. 中参与  
   可参与需求权重调整、候选任务排序、候选方法加减分

3. 强参与  
   可直接参与需求生成、任务生成与方法硬性过滤

默认应先从弱参与开始，再逐步升级。

### 6.4 服务概念进入决策的例子

当稳定概念成立：

- `服务值对安全值具有正向单调影响`

则它可以逐步参与：

1. 弱参与  
   在解释层提示：`当前服务偏低，安全恢复能力受限`

2. 中参与  
   当服务较低时，提高与服务恢复相关需求和任务的权重

3. 强参与  
   在极低服务值下，优先生成恢复服务条件的任务，并抑制高风险任务


## 7. 概念的降级与撤销

自动生成的概念不应只升不降。

出现以下情况时，应降级或撤销：

1. 新证据持续与旧方向相反
2. 原有关系只在极小范围内成立
3. 主要因果被发现来自其它因素
4. 对根需求的投影结果发生改变

因此，概念应视为：

- `当前最优归纳`

而不是永久真理。


## 8. “服务越高越好”是如何生成的

这个结论的生成链应固定为：

1. 根需求先成立：`安全值最大化`
2. 自我观察到：`服务变化会影响安全变化`
3. 二次特征形成：
   - `服务上升后安全恢复更快`
   - `服务下降后安全下滑更快`
   - `服务为 0 时待机`
4. 因果判定成立：
   - 服务变化先于安全变化
   - 方向重复一致
   - 多次可复现
5. 投影到根需求：
   - 高服务更有利于安全接近最大值
6. 最终形成稳定概念：
   - `对安全而言，服务值越高越好`

注意这里的“越高越好”不是自然语言先验，而是：

- `对根需求净贡献更大`


## 9. 本文件的约束结论

后续实现中应固定以下结论：

1. 自我先天拥有的是概念生成机制，不是具体概念结论。
2. 概念必须从状态、动态、二次特征和因果中自动归纳。
3. 单次观察最多产生线索，不能直接形成稳定概念。
4. 稳定概念必须满足重复、一致、先因后果、可复现、可投影到根需求。
5. 稳定概念进入决策前，必须先明确适用边界与决策强度。
6. 概念可以升级，也必须可以降级和撤销。
7. `服务越高越好` 这类价值判断，必须先经过根需求投影后才能成立。
