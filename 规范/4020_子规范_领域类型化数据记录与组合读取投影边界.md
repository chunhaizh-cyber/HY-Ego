# 4020 子规范 领域类型化数据记录与组合读取投影边界

编号：4020
日期：20260722
状态：正式子规范

## 1. 定位

本规范定义关系无法表达的必要原始值如何进入所属领域的类型化数据记录，以及服务如何按需组合节点、关系和类型化记录形成完整读取投影。它禁止任何无类型通用值容器、通用槽位表或第二份持久化权威事实。

直接上级规范：0050。

直接依赖规范：0500、4010。

相关边界规范：4030、4040、4060、4070、4110、4210、7140。

## 2. 类型化记录身份

一项领域类型化数据记录必须同时具备：

```text
所属领域和记录类型
唯一所属节点的完整当前句柄
记录版本与格式版本
记录状态或当前性
该记录类型明确定义的值式字段
合法写入方和读取方
序列化、恢复和未知版本拒绝合同
```

记录身份由所属节点、领域记录类型和记录版本共同限定，不建立跨领域通用记录编号、通用附属句柄或任意槽位索引。不同领域可以使用独立类型和独立存储，但不得把实现容器、联合类型或字节数组提升为共同业务语义。

节点必须先以 4010 的直接稳定主键和完整句柄成立，类型化记录才可作为同一事务候选绑定。记录不得脱离所属节点成为独立业务对象，也不得替代节点身份。

## 3. 关系与原始值分账

正式拓扑全部由关系承载，包括归属、父子、成员、模板、引用、当前值、来源、主体、场景、前后状态和来源动作等对象联系。类型化记录不得复制关系目标句柄形成第二套拓扑。

类型化记录只保存关系无法表达、且该领域规范证明必须长期保留的原始值。若完整事实需要同时读取关系角色和原始值，服务在读取时组合两者；组合结果不是新的持久事实。

不得用以下方式规避分账：

```text
用整数槽位保存拆分句柄、关系端点或拓扑编号
用自由文本、JSON、二进制包或可选值数组承载跨领域语义
在节点通用字段中复制领域值、缓存、业务状态或显示材料
在领域记录中复制可由关系或其它权威记录确定重建的第二份事实
```

## 4. 领域最小记录合同

| 所属领域 | 类型化记录唯一职责 | 不得承担 |
| --- | --- | --- |
| 特征值 | `I64`、`VecI64` 或 `VecU64` 三者之一的原始值、原始值版本和必要容器版本 | 特征定义、槽位归属、当前值关系、业务解释 |
| 状态 | 发生时间、接收时间、记录模式及状态域明确允许的非拓扑原始材料 | 主体、特征、值、场景、来源关系的平行副本 |
| 动态 | 发生时间、聚合规则版本及动态域明确允许的非拓扑原始材料 | 主体、被改变目标、前后状态、场景、来源动作关系的平行副本 |
| 需求 | 需求本体必要的类型化目标、预算或版本材料，以需求根规范为准 | 任务、方法、状态或结算的副本 |
| 任务 | 授权承接、筹办轮次或任务本体必要的类型化材料，以任务根规范为准 | 方法能力、执行结果或需求满足的副本 |
| 方法 | 能力规格、内容版本或方法本体必要的类型化材料，以方法根规范为准 | 任务授权、任务完成或需求结算的副本 |
| 概念签名 | 签名规则版本、确定排序的原始值约束项和记录版本，以 7140 为准 | 定义节点、稳定目标、关系句柄、活动版本、登记或统计缓存 |
| 其它对象 | 仅限该对象根规范和主题子规范具名允许的必要原始值 | 通用槽位、跨领域事实或显示便利字段 |

状态和动态的“完整事实数据”按以下方式成立：

```text
完整状态事实 = 状态节点 + 主体 / 特征 / 值 / 场景 / 来源正式关系 + 状态域类型化记录
完整动态事实 = 动态节点 + 主体 / 被改变目标 / 前后状态 / 场景 / 来源动作正式关系 + 动态域类型化记录
```

任一组成缺失、版本不一致或关系端点失效，都不得返回完整事实。

## 5. 写入、换代与删除

领域服务负责业务准入和记录值域；服务专用数据操作在 4040 的同一事务中组织节点、关系、类型化记录和索引候选。拒绝必须发生在第一笔写入前；候选确认前必须读回记录类型、所属节点、版本、字段完整性和全部关联关系。

记录内容变化必须按所属领域规范推进记录版本或形成新记录，不得静默覆写后沿用旧版本。节点删除、关系失效和类型化记录退役共同发生时，必须按精确依赖逆序撤销或按已发布生命周期形成正式失效；不得留下有效节点无必需记录、有效记录无所属节点或新旧记录同时冒充当前。

## 6. 组合读取投影

领域服务可以返回“完整读取材料”“权威读取材料”“状态列表”“动态列表”或其它值式视图，但这些名称只表示一次读取结果：

```text
按稳定身份读取当前节点
-> 读取具名正式关系
-> 读取所属领域类型化记录
-> 复核全部版本、基数、端点和值域
-> 组合不可变值式结果
```

组合读取投影不得持久化为第二份权威记录，不得被外部调用方反向写入，不得因缓存、列表或摘要丢失而改变机器事实。索引和缓存只允许缩小候选；命中后仍须完成上述权威读回。

## 7. 持久化与恢复

长期材料必须保存领域记录类型、所属节点完整身份、记录版本、格式版本、记录状态和全部必需原始字段。恢复时类型化记录必须与节点和关系在隔离候选中互证；未知类型、未知格式、错所属节点、错版本、缺必填字段、值域越界或关系不完整均在发布前拒绝。

只保存通用整数槽、旧附属句柄或不可解释字节包的旧材料不能直接恢复为当前事实。旧材料只有经过具名离线迁移规则映射成当前节点、关系和领域类型化记录，并重新通过 4070 全量准入后才可作为候选；无法无歧义映射时必须拒绝。

## 8. 验证合同

验证至少覆盖：

1. 记录类型、所属节点、记录版本或格式版本无效时写前拒绝。
2. 领域记录不能保存关系端点、拆分句柄或跨领域槽位语义。
3. 特征值三种原始类型互斥，原始值版本与当前值关系一致。
4. 状态和动态缺任一必需关系或类型化记录时不能返回完整事实。
5. 组合读取投影清空、丢弃或重建不改变权威事实。
6. 同一节点的记录换代使旧版本退出当前读取且历史仍可审计。
7. 恢复候选的节点、关系和领域记录互证失败时不发布半结构。
