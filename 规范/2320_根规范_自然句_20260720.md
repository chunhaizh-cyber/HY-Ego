# 2320 根规范 自然句

编号：2320

日期：20260720

状态：正式根规范

## 1. 根定位

自然句是段落版本内有序的完整句级语言材料和解析锚点，承载原始文本、段落归属、句序、词出现序列和子句组成。自然句文本和解析结果都不是项目业务事实。

## 2. 正式结构承载

正式自然句至少由以下结构共同成立：

```text
自然句稳定身份与明确语言对象角色
+ 原始文本和语言归属
+ 唯一段落版本归属关系
+ 在段落中的顺序号
+ 有序词出现关系
+ 有序 / 有范围的子句组成关系
+ 解析版本、当前发布和来源证据
```

词出现关系必须区分词身份和本句中的一次出现。相同词在同一句中重复出现时，每次出现及其顺序都必须保留，不得因目标词句柄相同而去重。

## 3. 词序、入口候选与子句

1. 每个词出现至少引用一个当前可读词身份；若解析器已经给出人类词性或语素入口，只能作为带解析版本的候选或已确认语言入口关系保存。
2. 人类词性、语法角色、信息入口候选和模型置信度不直接成为世界事实。
3. 子句必须通过正式组成关系和词范围归属自然句；不得仅靠字符串切片或显示缩进维持长期身份。
4. 自然句修订后，词序、子句范围和解析候选必须绑定新版本，不得让旧范围继续指向已变化的文本。

## 4. 写入方、读取方与验证

语言领域服务负责自然句身份、段落归属、原始文本、词出现序列、子句组成、解析版本和发布。语素服务只读取已确认词 / 入口材料，不负责把整句直接升级为业务事实。

正式读回应互证自然句角色、唯一段落归属、句序、每个词出现的顺序与目标、子句范围和解析版本。词序读取必须保持稳定顺序和重复项。

## 5. 业务边界与禁止事项

解析出的指代、概念、特征、需求、状态、动态或因果必须经相应领域入口取得稳定身份和证据。

- 禁止把自然句、词序列、意图标签或模型摘要直接作为需求、任务状态、方法选择或因果结论。
- 禁止把同一词的重复出现折叠为一个关系后丢失顺序。
- 禁止把界面显示的句子树当作正式组成关系来源。
