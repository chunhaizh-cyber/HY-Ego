# 0960 四域方法树自动生长闭环规范 v0

## 1. 目标

本文用于把四域方法树的“自动生长”收成一条可执行闭环。

必须固定：

- 运行时先匹配本域方法树，不先调用兜底
- 匹配不到时，才进入本域兜底
- 本域兜底成功后，允许把成功路径写回本域方法树
- 写回后的新枝先作为 `运行期临时` 或 `本能` 来源存在
- 只有在相似任务上稳定复现后，才允许升格为 `反推` 或 `组合`
- 筹办树的自动生长，是任务层经验积累与智力进化的主要工程落点

本文直接回答：

- 四棵树在运行时如何匹配
- 匹配失败时如何发芽
- 发芽后如何记录稳定复现
- 何时升格、何时只保留临时枝

## 3. 运行时总闭环

四域方法树的自动生长，统一固定成以下 4 步：

```text
1. 用任务虚拟存在匹配本域树
2. 匹配不到 -> 走本域兜底
3. 兜底成功 -> 写回为临时枝
4. 稳定复现 -> 升格为候选枝或正式组合枝
```

可直接收成一句话：

```text
匹配本域树 -> 匹配不到就走本域兜底 -> 兜底成功就写回成新枝 -> 稳定复现后再升格
```

其中在任务控制语义上应继续固定：

```text
筹办不只是铺下一步路，也是把成功筹办经验沉淀进方法树、推动任务层智力进化的主域
```


## 4. 匹配规则

### 4.1 运行时主入参

四域方法树的运行时主入参仍然是 `任务虚拟存在`。

最小匹配上下文至少包括：

- 当前功能域
- 任务虚拟存在
- 当前方法首节点（如果有）
- 当前安全值 / 服务值 / 需求值 / 预算
- 最近结果稳定性、最近失败、风险变化、检查点状态

### 4.2 匹配单位

匹配单位固定为：

```text
首节点 -> 条件节点 -> 结果节点 -> 下一首节点
```

不允许引入第二套树边结构。

### 4.3 最小匹配流程

最小可执行匹配流程固定为：

```text
域根默认结果节点
-> 扫描其下所有候选首节点
-> 在候选首节点下扫描条件节点
-> 用条件主键命中某个条件节点
-> 命中后取对应结果节点
-> 返回命中的首节点作为建议执行首节点
```

默认条件节点和默认续接首节点只承担“占位骨架”和“默认挂载点”职责，不视为具体命中的成熟枝条。


## 5. 发芽规则

当本域匹配失败时，必须先走本域兜底。

若本域兜底在某个具体任务上走通一条局部路径，则允许立即把成功路径写回本域方法树。

写回后的最小结构固定为：

```text
父结果节点 -> 新首节点 -> 条件节点 -> 结果节点
```

### 5.1 挂载位置

写回位置固定为：

- 若已有“最接近的父结果节点”，则优先挂到该父结果节点下面
- 若没有更近的父结果节点，则挂到域根默认结果节点下面

### 5.2 动作句柄白名单

自动生长过程中，新枝的动作句柄只允许来自：

1. `枚举_动作句柄类型::本能函数ID`
2. `枚举_动作句柄类型::动作序列主键`

不允许长出第三种句柄类型。

### 5.3 初始来源

新枝初始来源必须可区分，不能一律写成 `组合`。

最小纪律：

- 本域兜底直接长出的新枝：`运行期临时`
- 本能直接沉淀的路径：`本能`
- 因果总结归纳的路径：`反推`
- 稳定复现后升格的成熟枝：`组合`


## 6. 结构动作

四域方法树的自动生长，只允许以下 5 类结构动作：

### 6.1 发芽

条件：

- 本域无命中路径
- 兜底成功一次

动作：

- 新增一组 `首 + 条 + 果`

### 6.2 抽枝

条件：

- 已有路径前半段常成功
- 后半段经常需要补同一步

动作：

- 在已有结果节点下面追加新的下一首节点
- 再给这个下一首追加条件节点与结果节点

### 6.3 分叉

条件：

- 同一首节点在不同条件下应走不同后续路

动作：

- 在同一首节点下增加多个条件节点

### 6.4 归并

条件：

- 多条枝条主干相同，只是条件或局部步骤不同

动作：

- 抽出共同首节点
- 把原枝条退成条件分支

### 6.5 修剪

条件：

- 长期不稳定
- 几乎不用
- 安全上已不再适合

动作：

- 不物理删除
- 只降权、隐藏或退到备用区


## 7. 稳定复现与升格规则

方法树成长的真正驱动力，不是“成功过一次”，而是“结果是否稳定复现”。

必须固定：

- 兜底成功让新枝出生
- 稳定复现让新枝长稳

最小升格阈值固定为：

- `3` 次稳定复现：达到候选枝阈值，可从 `运行期临时` 升格到 `反推`
- `5` 次稳定复现：达到正式组合枝阈值，可从 `反推` 或 `运行期临时` 升格到 `组合`

阈值后续允许配置化，但在未配置前，默认使用 `3 / 5`。


## 8. 四域边界

自动生长必须遵守“本域优先、本域兜底、本域沉淀”。

### 8.1 筹办树只吸收

- 新的细分方式
- 新的补条件方式
- 新的初始绑定方式
- 新的等待学习挂接方式
- 新的初始检查点布置方式

### 8.2 维护树只吸收

- 稳定的一步推进链
- 检查点插入模式
- 可靠度更新模式
- 继续维护 / 转纠偏 / 转收束 / 回筹办 的判别套路

### 8.3 纠偏树只吸收

- 替代方法模式
- 回退模式
- 补条件任务模式
- 直接收束模式

### 8.4 收束树只吸收

- 完成判定模式
- 失败收口模式
- 最终反馈模式
- 清理与归档顺序

禁止跨域混长。


## 9. 工程落地纪律

### 9.1 模块职责

自动生长相关能力按以下模块分层：

- `四域方法树匹配器_v0`：负责条件签名生成与树内匹配
- `四域方法树写回器_v0`：负责发芽、抽枝、分叉的最小写回
- `四域方法树生长判定器_v0`：负责稳定复现统计与来源升格
- `任务控制总控模块_v0`：负责把匹配、兜底、写回、升格串成运行时闭环

### 9.2 注册模块要求

[四域方法树注册模块_v0.ixx](D:\鱼巢\四域方法树注册模块_v0.ixx) 中的 `创建子首节点_` 必须允许显式携带 `枚举_方法沉淀来源`，不得把所有新枝写死为 `组合`。

### 9.3 运行时闭环要求

[任务控制总控模块_v0.ixx](D:\鱼巢\任务控制总控模块_v0.ixx) 必须至少完成以下闭环：

```text
匹配本域树
-> 若命中则优先复用
-> 若未命中则允许继续执行本域兜底
-> 本域执行成功后写回临时枝
-> 命中或写回后的方法枝都要累计稳定复现
-> 满足阈值后再升级来源
```


## 10. 最小验收标准

视为满足本规范的最小实现，应至少做到：

1. 域根默认结果节点下面可以新增非默认子枝
2. 本域匹配器可以区分“默认骨架”与“已沉淀具体枝条”
3. 写回器可以把成功路径回写为 `首 + 条 + 果`
4. 写回器可以保留 `运行期临时 / 本能 / 反推 / 组合` 的来源差异
5. 生长判定器可以按稳定复现次数完成最小升格


## 11. 一句话收束

> 四域方法树的自动生长，不是凭空发明新方法，而是在“本域匹配失败”时调用既有兜底函数，走通一条局部路径后，把它按 `首节点 + 条件节点 + 结果节点` 的结构写回对应域树里；先作为临时枝存在，再靠相似任务上的稳定复现逐步升格。
