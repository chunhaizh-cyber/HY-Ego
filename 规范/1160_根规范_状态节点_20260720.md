# 1160 根规范 状态节点

编号：1160
日期：20260722
状态：正式根规范

## 1. 根定位

状态节点是某主体、某特征在某一时间点或变化后的特征值事实切片。状态必须同时有主体、特征和值的确定语义；不得脱离主体成为裸值记录。

## 2. 本项目承载

实例状态由状态节点、稳定句柄、发生时间等状态域类型化记录，以及到场景、主体、特征、值和来源的正式关系共同承载；这些事实关系的专用类型、角色和基数由 4210 规定。场景到状态的运行期临时关系或事实目录关系只表达场景归属，不替代状态本体。

长期抽象状态若作为模板锚点存在，必须归属于抽象特征及其值域，不得形成与抽象特征平行的权威状态树。

## 3. 事实与边界

1. 实例状态必须具有存在主体、状态特征、状态值、发生时间、场景归属和可追溯来源；收到时间由相应子规范按材料来源规定。
2. 状态节点不得直接承载状态变化因果结论；变化进入动态、二次特征或因果结构。
3. 状态事实是零散事实切片，不得组织成状态树。
4. 状态不得普通挂入特征、存在或场景子结构并解释为历史集合；场景归属通过事实目录关系或索引表达。
5. 基础事实池或全局索引可以登记状态以支持主键查找和引用解析，但不得把登记入口解释成业务世界层级。
6. 显示状态、日志状态、任务生命周期枚举和函数返回值不得冒充世界状态事实。

## 4. 写入、读取与验证

状态领域服务负责抽象状态模板材料和完整实例状态的准入；实例状态写入必须先验证场景、主体、特征、值、时间和来源，再由状态数据操作在一个会话中发布节点、状态域类型化记录、关系和索引。

动态、需求、任务和方法服务只能读取完整状态材料，不得自行拼装裸状态。验证实例状态必须回读唯一主体、场景、特征、值、非零发生时间及全部关系当前性；缺少任何一项不得解释为完整状态。

## 5. 生命周期与当前性

状态事实创建后保持历史可追溯。所谓“当前状态”必须由宿主、特征、适用范围、时间、来源和版本共同选出，不能通过覆盖旧状态或单个枚举成立。状态当前性失效不等于历史事实删除。
