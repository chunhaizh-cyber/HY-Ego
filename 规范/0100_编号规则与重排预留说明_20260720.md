# 0100 编号规则与重排预留说明

编号：0100
日期：20260720
状态：正式编号规则
权威等级：正式编号规则

## 1. 命名格式

当前正式规范文件统一采用：

```text
编号_文档类型_主题_YYYYMMDD.md
```

示例：

```text
3200_根规范_任务_20260720.md
5210_子规范_任务状态特征集合与阶段关联_20260720.md
```

`规范目录.md` 作为固定入口不追加日期。未列入规范目录的其它材料不因使用四位编号而自动取得正式规范身份。

## 2. 编号原则

1. 编号是 `规范/规范目录.md` 使用的正式目录编号。
2. 同一逻辑对象保持稳定编号和上下级关系；底层承载统一使用海中鱼巣结构。
3. 编号跟随逻辑对象和规则主题，不跟随 C++ 文件名、类名、字段名或实现模块。
4. 同族对象优先放在相邻十位，保留中间编号供后续插入。
5. 规范尚未完成内容抽取时，可以在目录目标层级中预留编号，但不得伪装成正式规范。

## 3. 规范层级规则

```text
0050 项目通用上级规范
-> 对应 1xxx—3xxx 对象根规范
-> 对应 4xxx—8xxx 主题子规范
```

根规范只规定原则性合同：对象是什么、合法事实从哪里来、谁可以写、生命周期如何判定、什么绝对禁止。

子规范展开实现前必须明确的业务合同：特征类型、值域、状态组合、来源、版本、流程顺序、合法分支、拒绝、内部逻辑错误和验证方式。

不得把完整特征清单、状态矩阵、算法步骤或代码接口堆入 0050；也不得让子规范改变根规范的事实来源、写入权、身份边界、并发隔离、状态语义或完成判定。

## 4. 日期规则

1. 文件内容发生实质修改时，文件名日期更新为本次正式修订日期。
2. 只改错字、链接或格式，也必须写入修订记录，但可保留文件名日期。
3. 文件名日期变化后，必须同步更新规范目录、总索引、覆盖矩阵和全部正式引用。

## 5. 重排规则

1. 重排编号不得改变规范含义。
2. 重排必须保留旧编号到新编号的映射记录。
3. 重排必须同步更新文件名、规范目录、总索引、覆盖矩阵、冲突清单、缺失清单和正式交叉引用。
4. 重排期间旧编号文件不得与新编号文件同时作为权威规范。
5. 重排完成后，旧编号材料不得保留为并列权威入口；需要追溯时只保存编号映射和修订记录。
