# 8110 子规范 线程生命周期状态上报与控制面板线程信息

编号：8110
日期：20260720
状态：正式子规范

## 1. 定位

本规范规定项目线程创建、运行、堵塞、暂停、恢复、退出和故障的结构化上报、缓存、消费和控制面板只读显示边界。

上级规范：0050、8100。

## 2. 核心边界

```text
纯线程生命周期消息
-> 线程消息缓存
-> 控制面板线程信息模块

任务治理消息
-> 任务管理上行协议
-> 自我线程
```

线程信息只服务项目线程可见性、健康监控和诊断，不构成任务状态、需求满足、世界事实、动作动态、因果或价值结算依据。

## 3. 生命周期状态

线程消息至少可以表达：创建、启动中、运行中、堵塞或等待、暂停、恢复、停止请求中、收尾中、退出前、已退出、故障和异常退出。

这些是线程运行投影，不是业务状态。任务工作线程处于堵塞不能直接推出任务等待；线程退出不能直接推出任务完成或失败。

## 4. 发送责任

线程创建者发送创建消息。若系统线程 ID 只能在线程入口取得，创建者先用逻辑 ID 建立创建消息，线程入口再补发绑定系统线程 ID 的状态消息。

线程自身在生命周期或健康状态变化时发送状态消息，并在正常退出前发送退出前消息。线程崩溃、强制终止或未能上报时，由创建者或管理者检测回收并发送异常退出消息。

发送不得长期阻塞业务线程。控制面板不可用时，消息先进入中间缓存；缓存也不可用时记录独立诊断错误，线程仍按自身职责安全收口。

## 5. 消息最小结构

线程信息消息至少包含：

- 消息稳定身份、事件类型和发生时间；
- 线程逻辑 ID、可选系统线程 ID；
- 线程名称、用途、类别和所属模块；
- 创建者、线程池和工作项诊断引用；
- 前后生命周期状态和前后运行状态；
- 堵塞、暂停、健康和退出标记；
- 原因键和可选显示摘要。

线程逻辑 ID 是进程内稳定主键。系统线程 ID 只作诊断绑定。关联任务和工作项只作排错引用，不能推导业务状态。显示摘要不得参与路由、条件、因果或结算。

## 6. 消息缓存

每条消息必须以结构化材料完整写入临时文件或等价候选，再原子发布为正式未消费消息。文件或记录身份必须避免并发覆盖。

消费按发生时间和消息身份排序；同一线程的退出、故障和异常退出优先于迟到的旧运行消息。未确认消费的正式消息不得按时间清理。

已消费消息可以短期保留。创建、退出、故障、堵塞、暂停、不健康、具名原因以及关联任务或工作项的消息具有诊断价值，应先归档再清理。归档仍只是诊断材料，必须有容量和保留策略。

陈旧临时文件和无法解析的无效材料只有在保留期结束且排除正在写入后才可删除，并记录维护摘要或错误日志。业务线程不等待清理。

## 7. 控制面板消费

控制面板线程信息模块按逻辑 ID 查找或建立线程信息项，绑定系统线程 ID，更新名称、用途、模块、线程池、创建者、生命周期、运行状态、健康、最近事件、版本和时间。

创建、退出、故障和异常退出不得被普通状态覆盖。未知线程的状态先到时可以建立“创建消息未到”的占位；线程退出后到达的旧状态只能记为迟到消息。

线程信息表和页面都是只读运行投影，不得写任务、需求、世界事实、动作动态、派生需求或价值。

## 8. 页面显示

页面统计项目自身已登记线程总数、运行数、各状态数、已结束数和故障数。中栏显示完整项目线程列表，不用 OS 线程枚举补造项目线程，不用“等 N 项”隐藏条目。

详情至少显示用途、名称、逻辑 ID、系统线程 ID、创建者、模块、加载时间、运行时长、状态、最近事件、更新时间和故障或退出信息。

来源不足的字段显示“未记录”或“待线程生命周期消息接入”，不得用控制面板快照时间伪装线程创建时间。

## 9. 与自我和任务管理的边界

纯线程消息不得进入自我治理邮箱。任务执行尝试、实际结果、任务缺口、执行许可和观察承接只有在绑定具体业务对象并符合任务上行协议时，才可以由任务管理发送给自我线程。

任务管理界面可以为自己的工作线程上报线程状态，但不得把堵塞写成任务等待，不得充当所有线程的业务注册中心。

## 10. 验证边界

真实运行至少验证创建、逻辑 ID 和系统 ID 绑定、运行、堵塞、暂停、恢复、退出、异常退出、迟到消息、控制面板不可用和任务消息隔离。

日志和页面只能证明线程信息链，不得扩张为业务闭环完成。
