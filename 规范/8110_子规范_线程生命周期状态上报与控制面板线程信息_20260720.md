# 8110 子规范 线程生命周期状态上报与控制面板线程信息

编号：8110
日期：20260721
状态：正式子规范

## 1. 定位

本规范规定项目线程创建、运行、堵塞、暂停、恢复、退出和故障的最小消息合同、只读线程投影、上报责任和不裁决业务事实边界。

直接上级规范：8100。

直接依赖规范：无。

相关边界规范：1170、3200、8200。

## 2. 核心边界

```text
纯线程生命周期消息
-> 线程消息缓存
-> 控制面板线程信息模块

任务治理消息
-> 任务管理上行协议
-> 自我线程
```

线程信息只服务项目线程可见性、健康监控和诊断，不构成任务状态、需求满足、世界事实、动作动态、因果或价值结算依据。

## 3. 生命周期状态

线程消息至少可以表达：创建、启动中、运行中、堵塞或等待、暂停、恢复、停止请求中、收尾中、退出前、已退出、故障和异常退出。

这些是线程运行投影，不是业务状态。任务工作线程处于堵塞不能直接推出任务等待；线程退出不能直接推出任务完成或失败。

## 4. 消息最小合同

线程生命周期消息至少包含：

- 消息稳定身份、事件类型、发生时间和版本；
- 线程逻辑身份、可选系统线程 ID；
- 线程类别、所属模块和当前生命周期投影；
- 前一生命周期投影、原因键和当前性；
- 可选任务或工作项诊断引用。

线程逻辑身份是运行期投影主键，系统线程 ID 只作诊断绑定。任务和工作项引用只用于定位运行载体，不能推导任务状态、需求状态或业务结果；显示摘要、日志文本和名称也不得参与路由、条件、因果或结算。

## 5. 上报责任与发布边界

线程创建者负责发出创建事件；线程入口取得系统线程 ID 后可以补充诊断绑定。线程自身在生命周期变化时发出当前事件，并在正常退出前发出退出事件；无法自行上报的故障或异常退出由具名管理者根据实际回收结果补充诊断事件。

上报不得长期阻塞业务线程。消息只有在最小合同完整并发布后才能进入普通投影读取；候选、半消息、解析失败和未发布材料不得被读取方补造为正式线程投影。具体缓存介质、临时材料、归档、保留、清理和页面装配不属于机器逻辑合同，不由本规范规定。

## 6. 只读线程投影

线程信息读取方按线程逻辑身份、消息身份、发生时间和版本形成当前只读投影。创建、退出、故障和异常退出等不可逆生命周期事实不得被迟到的旧运行事件覆盖；状态先于创建事件到达时可以保留具名不完整投影，但不得伪造缺失来源。

投影可以提供线程类别、所属模块、逻辑身份、诊断绑定、当前生命周期、最近事件、发生时间、版本和故障原因等只读材料。具体页面区域、列名、字段排列、统计布局和缺省显示文案均不属于机器逻辑合同，不取得业务语义。

## 7. 与自我和任务管理的边界

纯线程消息不得进入自我治理邮箱。任务执行尝试、实际结果、任务缺口、执行许可和观察承接只有在绑定具体业务对象并符合任务上行协议时，才可以由任务管理发送给自我线程。

任务管理界面可以为自己的工作线程上报线程状态，但不得把堵塞写成任务等待，不得充当所有线程的业务注册中心。

线程投影、控制面板和诊断读取方不得写任务、需求、世界事实、动作动态、派生需求或价值，也不得把投影缺失、迟到或显示状态变成业务拒绝或业务完成依据。

## 8. 验证边界

真实运行至少验证创建、逻辑 ID 和系统 ID 绑定、运行、堵塞、暂停、恢复、退出、异常退出、迟到消息、控制面板不可用和任务消息隔离。

日志和页面只能证明线程信息链，不得扩张为业务闭环完成。
