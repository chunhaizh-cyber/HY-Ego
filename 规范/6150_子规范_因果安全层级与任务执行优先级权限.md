# 6150 子规范 因果安全层级与任务执行优先级权限

编号：6150
日期：20260724
状态：正式子规范

## 1. 定位

本规范定义安全需求如何按到具体安全结果的有效直接因果最短路径形成精确层级，如何映射为优先级组，以及各组如何递归取得任务执行优先级权限。

本规范只改变任务参加正常执行排序时的权限和影响系数，不改变需求、任务、方法、授权、结算预算、安全值或世界事实。

直接上级规范：1190、3100、3200。

直接依赖规范：4320、5110、5200、5210、6110、6140。

相关边界规范：6100、6120、6130、8100、8200。

## 2. 具体安全结果与有效直接因果边

具体安全结果是能够由正式状态、动态和安全根定义明确比较的安全结果身份。它必须绑定受影响自我、目标宿主、目标特征、结果状态或区间、适用场景、当前版本和安全根追溯关系。名称中含“安全”“生存”“维护”或其它领域词不构成具体安全结果。

从候选目标结果 `X` 指向结果 `Y` 的边只有同时满足以下条件，才是本规范可消费的有效直接因果边：

1. 边来自已发布的完整因果事实或当前适用的正式因果模板；
2. 原因角色为 `X`，直接结果角色为 `Y`，中间没有被隐藏或压缩的结果；
3. 状态、动态、证据、适用场景、主体、规则版本和当前性完整；
4. 边未被当前反证、排除、版本失效或适用范围拒绝；
5. 查询时使用的自我、场景和安全结果与边的适用锚点兼容。

派生置信分、显示路径、文本解释、历史候选、日志和缓存命中不得替代有效直接因果边。概率或置信只参与边是否达到现行适用门槛的裁决，不作为层级距离。

## 3. 精确因果深度

对需求目标结果 `X` 和具体安全结果 `R`，精确因果深度 `D(X,R)` 定义为：

```text
X 与 R 是同一结果身份                         -> D = 0，只表示结果本体，不形成安全因果层
存在 X 到 R 的当前有效直接因果路径             -> D = 最少直接因果边数量
不存在当前有效直接因果路径                     -> 无安全深度
```

若一个目标可以到达多个具体安全结果，取全部有效路径中的最短深度作为当前精确深度，并保留全部同深度具体安全结果、直接因果边集合和规范化路径来源。不得只保留一个无来源数字。

深度使用非负 `I64`，没有业务上限。图搜索不得因深度大于 5、固定数组容量、显示层级、超时偏好或预置类别而提前截断；组 5 汇总不改变精确深度。唯一技术边界是 `I64` 的可表示范围，超过该范围时属于表示能力内部错误，禁止钳制到 `I64_MAX`、5 或其它值。

路径计算必须在一个因果图、场景和规则版本快照内完成。使用确定性广度优先或等价最短路算法；同层扩展按完整因果身份和稳定句柄排序。遇到环时按本轮已访问的完整结果身份阻止重复扩展，但不得借环缩短距离。现行因果 DAG 若出现非法环，必须保留诊断并停止受影响投影。

## 4. 动态当前性和来源投影

精确深度是从当前因果事实计算的值式投影，不是需求、任务或因果节点上的永久字段。下列任一事实变化后必须重新计算：

```text
需求目标或目标状态版本
具体安全结果版本
直接因果边、证据或适用范围
当前场景或主体身份
事件因素排除或反证
安全规则版本
```

每个深度投影至少包含：

```text
需求身份和目标版本
具体安全结果身份
精确深度
全部同深度来源路径或可回读路径集合身份
因果图、场景和规则版本
形成时间和当前性
```

因果路径失效只使旧投影退出当前使用，不删除历史需求、任务或因果证据。没有路径时不得因需求名称或旧深度继续给予安全层级权限；目标自称安全但缺少应有路径时形成具名结构缺口。

## 5. 优先级组

优先级组使用 `I64`，只对 `D >= 1` 的安全因果层由精确深度唯一计算：

```text
优先级组 G = min(D, 5)
```

组语义为：

| 组 | 精确深度 |
| --- | --- |
| 1 | `D = 1` |
| 2 | `D = 2` |
| 3 | `D = 3` |
| 4 | `D = 4` |
| 5 | `D >= 5` |

`D = 0` 是具体安全结果本体，不属于五级安全因果层，也不得建立“第 0 优先级组”绕过来源因果和需求授权。任务不能以“直接修复”“应急”或自身名称把 `D = 0` 改写成组 1；它必须承接一项来源需求，该需求绑定的预防、处置或阻断因果信息到具体安全结果具有一条当前有效直接边，由同一投影机械得到 `D = 1、G = 1`。无法回读该来源因果、需求授权和路径版本时形成具名缺口，不参加正常安全任务排序。

组 5 只汇总执行权限，不截断、覆盖或删除精确深度。需求、任务、排序投影和审计材料必须继续保留真实 `D` 及其路径来源。

这不是固定的需求类别表。同一需求在因果、场景或目标版本变化后可以进入不同精确深度和组；变化必须形成新投影，不得改写旧投影。

## 6. 安全层到组门控输入

每个参与门控的安全层必须提供 6140 定义的当前安全值 `V`、低位阈值 `L`、高位阈值 `H`、当前值版本和阈值规则版本。不同值域先分别计算释放倍率，不直接比较原始安全值。

本轮组门控集合从当前候选安全需求的有效路径动态形成：路径上每个具名结果位置必须回到对应安全层维护单元，并按其到最终具体安全结果的精确深度进入相应组。不得预置固定的组成员或需求类别。路径应有中间安全层却无法回读维护单元时，属于门控材料不完整，不得跳过该层。

权重倍率固定为：

```text
M = 1,000,000
最低层保留倍率 = 10,000
向上一层最大释放倍率 = 990,000
```

单个安全层的向上释放倍率 `R(V)` 为：

```text
V <= L       -> R(V) = 0
L < V < H    -> R(V) = floor((V - L) * 990,000 / (H - L))
V >= H       -> R(V) = 990,000
```

因此：

- 安全值没有高于低位阈值时，上层取得零权限；
- 安全值从低位阈值向高位阈值上升时，本层保留倍率从 `1,000,000` 逐渐降到 `10,000`，向上释放倍率从 `0` 逐渐升到 `990,000`；
- 安全值达到或高于高位阈值时，本层仍保留收到权限的 1%，向上最多释放 99%。

一个组包含多个当前有效安全层时，组释放倍率取各层 `R(V)` 的最小值，并保留全部取得最小值的限制层来源。这样任一仍低于门槛的必要安全层都能阻止上层绕过。

存在安全因果路径却缺当前安全值、阈值、层身份或版本时，不得默认安全或默认释放。受影响组的释放倍率为不可用，上层执行权限为零，并形成具名材料或结构缺口；不得用 `I64_MAX`、历史显示值或其它层平均值兜底。

## 7. 五级递归门控与守恒

组 1 收到本轮完整安全执行优先级权限；从组 1 到组 5 依次经过四道释放门：

```text
组 1 -> 组 2
组 2 -> 组 3
组 3 -> 组 4
组 4 -> 组 5
```

设组 `g` 收到的可分配权限为 `A_g`，该组释放倍率为 `R_g`。计算固定为：

```text
A_1 = M

对 g = 1..4：
    A_(g+1) = floor(A_g * R_g / M)
    Q_g = A_g - A_(g+1)

Q_5 = A_5
```

`Q_g` 是组 `g` 的执行优先级权限倍率。全部中间乘法使用至少 128 位整数或等价检查乘除，所有值使用 `I64`。向下取整产生的余量保留在更低组，因此始终满足：

```text
0 <= Q_g <= M
Q_1 + Q_2 + Q_3 + Q_4 + Q_5 = M
```

不得在每层重新发行 100%，不得把空组权限临时赠予更高组，也不得因上层任务重要、等待时间长或队列拥堵反向增大释放倍率。

组内当前没有可执行任务时，该组权限保持为未消费权限，不重分配到其它组。全部组均无可执行任务时返回“当前无可执行任务”，不得制造任务、提高安全值或改变需求事实。

## 8. 需求到任务的权限来源

每项安全需求权限投影至少绑定：

```text
需求身份和目标版本
当前授权和活动路径版本
具体安全结果身份
精确因果深度和优先级组
有效最短路径来源
组权限倍率 Q
安全层值、阈值和规则版本
当前安全不足度
具体安全结果后果严重程度及版本
预计发生时间或时间证据状态及版本
因果成立证据计数和规则版本
```

需求仍按 3100、5110 保持自己的父权重、OR 路径、AND 份额和结算预算；这些值不等于本规范的执行优先级权限。

任务仍只有一个确定目标。按 3200，一个任务可以承接一个或多个当前有效、目标语义等价且共同授权该目标的来源需求；每条正式来源关系均保持自己的需求身份、路径、授权和当前性，不得合并成一条无来源汇总。目标不等价的需求不得借权限计算共用同一任务。

多需求任务逐项形成候选权限，取最高有效权限倍率作为任务权限：

```text
任务权限 Qtask = max(Qsource_1, Qsource_2, ...)
```

必须保留全部参与计算的来源，以及全部并列取得最高权限的来源；不得只留下最大数字。并列来源按需求稳定句柄、具体安全结果句柄、精确深度和规范化路径键确定顺序。

权限来源失效、路径变化或安全值版本变化时，旧权限投影退出当前调度资格，但任务、需求和历史投影继续保留。

## 9. 基础优先级与有效执行优先级

任务基础优先级 `B`、组权限 `Qtask` 和有效执行优先级 `E` 全部使用非负 `I64`。`B` 来自 5200 定义的当前任务基础优先级及其正式来源版本，不得由线程等待时长、队列位置或日志临时抬高。

计算固定为：

```text
E = floor(B * Qtask / M)
```

乘法使用至少 128 位中间值或等价检查乘除，向下取整，低于一个优先级单位时允许结果为零，不强行补 1。溢出、负值、来源版本不兼容或计算结果不可表示时不得钳制，受影响调度批次停止。

`Qtask = 0` 时，任务不参加正常任务执行优先级竞争，但不删除、取消、失败、改写或失效该任务。任务仍可筹办、接收材料、维护等待合同和保存事实；观察、事件登记、风险确认和机器事实写入不得因执行权限为零而停止。

执行优先级权限和有效执行优先级只决定正常调度排序，不表示线程、物理时间、算力、任务数量或完成概率配额，也不保证 99% 的执行时间。

## 10. 组 5 与全局确定性比较

调度只比较同一一致性快照中的当前候选。组 1—4 的默认比较顺序为：

1. `Qtask = 0` 的候选不进入正常竞争；
2. 有效执行优先级 `E` 较大者优先；
3. 权限倍率 `Qtask` 较大者优先；
4. 任务基础优先级 `B` 较大者优先；
5. 任务创建序号较小者优先；
6. 任务稳定句柄规范序较小者优先。

组 5 汇总全部 `D >= 5` 的任务。组 5 不把全部因素未经证明地连乘成一个 I64 总分，而使用以下可逐项审计的确定性比较材料：

```text
权限倍率 Qtask
当前安全不足度 U
预计发生时间或时间证据序 T
后果严重程度 C
因果成立证据序 K
任务基础优先级 B
并列最高权限来源中的真实精确因果深度 D
+ 规范化路径、来源需求和任务稳定顺序键
```

`U` 从来源层当前 `V、Vmin、Vmax` 归一为 `0..M` 的 I64 不足度，值越大表示当前安全满足度越低；不同原始值域不得直接比较。`C` 必须来自具体安全结果定义或正式后果事实的 I64 严重程度及版本。`T` 只在存在正式预计发生时间或时间证据时使用：正在发生 / 已到达优先，其次为预计剩余单调时间较短；不适用与证据不足必须分别标记，不能伪造时间。

`K` 只能由 1190 的当前基础正样本、当前反例、适用范围和规则版本形成值式证据序，不能新增持久置信度字段。因果适用范围、规则版本和证据截止版本必须先全部当前；随后按“当前反例数较少者优先，再按有效基础正样本数较多者优先”比较。版本号只裁决兼容与当前性，不作为强度数值；正样本或反例计数为负、与证据集合不一致或适用范围不可比时形成具名缺口。

组 5 的比较顺序固定为：

```text
权限倍率 Qtask 较大者优先
-> 后果严重程度 C 较大者优先
-> 当前安全不足度 U 较大者优先
-> 正式时间证据显示更紧迫者优先
-> 因果成立证据序 K 较强且版本当前者优先
-> 任务基础优先级 B 较大者优先
-> 并列最高权限来源中的最小真实精确深度较小者优先
-> 规范化最短路径键较小者优先
-> 来源需求稳定句柄较小者优先
-> 再使用任务创建序号和任务稳定句柄
```

其中后果严重程度、安全不足度、正式时间证据和因果成立证据均先于精确深度；因此精确深度只是次级信息，不得覆盖已成立的直接严重风险。某项材料按正式定义不适用时跳过该比较项并保留“不适用”来源；本应适用却缺失、版本不兼容或值非法时，受影响候选不得参加比较并形成具名缺口。不得把未知伪装成零、最大值或默认安全。

不得把组 5 中的精确深度统一写成 5，也不得按遍历到达顺序、容器地址、线程编号或哈希碰撞顺序裁决。

一个任务有多个并列最高权限来源时，用上述规则选择本次排序主来源，同时在冻结投影中保留全部并列来源；主来源选择不删除其它需求关系。

## 11. 一致性快照、冻结和重算

一次调度决议必须从同一一致性读取快照冻结：

```text
候选任务及授权、状态和基础优先级
任务关联需求及目标版本
有效直接因果图、场景和规则版本
具体安全结果与精确深度
安全层当前值、阈值和残余风险版本
后果严重程度、预计发生时间或时间证据、因果基础计数和适用版本
组释放倍率、递归权限和排序规则版本
```

冻结结果是绑定本次调度尝试的不可变值式权限投影，不是需求、任务或安全值的第二份权威事实。它至少包含输入版本、全部来源、`D`、`G`、`Qtask`、`U`、`T`、`C`、`K`、`B`、`E` 和确定性排序键。

在真实派发前，任一输入版本变化都使旧投影失效，必须重算。权限变化不自动改写已经运行中的任务状态；是否停止、取消或重筹办仍服从任务授权、执行冻结和安全保护的正式入口。下一次正常派发必须使用新权限。

## 12. 写入权与单向因果

```text
安全事件、风险证据
-> 6140 / 6100 裁决安全值
-> 本规范计算层级和执行优先级权限
-> 任务调度只读消费
```

任务调度资源紧张、任务等待过久、没有可执行任务、执行结果好坏和队列排序不得反向修改安全值、阈值、因果路径、残余风险或因素排除事实。

因果服务只提供当前路径和深度材料，不直接创建需求、任务或调度决议。自我线程可以每轮触发重算和选择，但线程不是安全变化或任务动作来源。任务管理只消费冻结权限，不写安全事实。

## 13. 逻辑内结果与内部错误

以下是合法逻辑内结果：

```text
目标没有安全因果路径，按非安全任务规则处理
安全路径存在但材料不完整，受影响上层权限为零并返回具名缺口
任务权限为零，不进入正常竞争
当前没有可执行任务
输入版本变化，旧冻结失效并重算
```

以下属于内部逻辑错误并停止受影响调度批次：

```text
精确深度被截断为 5 或固定最大层
有效最短路径没有来源或跨越版本快照
非法因果环参与距离缩短
缺安全层值或阈值却默认释放
递归权限总和不等于 M
整数溢出后钳制或以 I64_MAX 兜底
多需求任务只保存最大数字而丢失来源
权限为零被解释为删除、取消、失败或停止观察
调度结果反向写安全值或因果事实
```

## 14. 验证合同

验证至少覆盖：

1. `D=0` 不形成第 0 组；`D=1、2、3、4、5、6` 及远大于 5 的路径按严格五组映射，精确深度不被组 5 截断。
2. 多条路径、多个具体安全结果、同深度并列、路径失效、场景变化和非法环。
3. 无业务深度上限、`I64` 表示越界停止和固定类别零参与。
4. `V<L、V=L、L<V<H、V=H、V>H` 的释放倍率及等号语义。
5. 多安全层取最小释放倍率，缺层材料时不默认安全。
6. 五级四道释放门、1% 保留、99% 最大释放、整数舍入和五组总和恒等于 `1,000,000`。
7. 空组不越级重分配、全部无候选时返回空结果。
