# 0890 控制状态机与触发条件表

## 1. 目标

本文是 [0870_统一控制规则（安全服务双值核与自组织）.md](D:\鱼巢\规范\0870_统一控制规则（安全服务双值核与自组织）.md) 的工程化展开表。

当前版本不新增新的路径状态机和核心枚举骨架，而是把以下事项落成工程表：

- `控制.模式`
- `控制.方向`
- 现有状态枚举的最小迁移口径
- 筹办、维护、纠偏、收束的功能模式口径
- 保持、细分、延长、回退、收缩、学习修正等执行动作的触发条件
- 事件类型和审计最小字段


## 2. 当前落点

### 2.1 状态仍用现有枚举

统一继续使用：

```text
未定义 / 未启动 / 就绪 / 筹办中 / 排队中 / 运行中 / 执行中 / 等待中 / 挂起 / 完成 / 失败 / 取消 / 超时 / 无法执行 / 待重筹办
```

### 2.2 功能模式不是状态

`控制.模式` 是控制语义，不是新的状态枚举。

统一词表：

```text
筹办 / 维护 / 纠偏 / 收束
```

### 2.3 控制方向也不是状态

`控制.方向` 是控制朝向，不是生命周期状态。

统一词表：

```text
完成优先 / 稳定优先 / 修复优先 / 探索优先 / 收缩优先
```


## 3. 控制核表

| 字段 | 作用 | 类型建议 |
| --- | --- | --- |
| 控制.安全值 | 边界强度 | `0.000 ~ 1.000` |
| 控制.服务值 | 服务牵引 | `0.000 ~ 1.000` |

| 控制.方向 | 当前控制朝向 | 固定词表 |

## 4. 功能模式表

| 模式 | 进入条件 | 允许动作 | 禁止动作 |
| --- | --- | --- | --- |
| 筹办 | 新任务建立、旧路径失效后需重建、当前链尚未形成可执行下一步 | 细分、延长、插检查、初始化预算、主备方法初选 | 无安全约束的无上限铺路 |
| 维护 | 当前任务链已建立但需要保持健康和可执行性 | 保持、刷新绑定、补检查、局部修补、最小必要学习写回 | 无因重建整条任务链 |
| 纠偏 | 执行失败、风险升高、反馈安全变化为负、结果漂移 | 替换方法、回退、收缩、插恢复任务、补条件任务 | 沿失败链继续硬推 |
| 收束 | 结果已足够、收益不足以继续、或残留结构需要关闭 | 归并、结束、清尾、总结、学习写回、归档 | 无因继续扩张未完成支链 |

### 4.1 功能模式与双根偏重表

| 功能模式 | 安全值偏重 | 服务值偏重 | 典型目标 |
| --- | --- | --- | --- |
| 筹办 | 决定能立到什么边界 | 决定值不值得立、立多大 | 把任务立起来 |
| 维护 | 决定是否必须维持稳定 | 决定维持投入是否值得 | 把任务撑住 |
| 纠偏 | 决定是否必须立刻纠偏 | 决定纠偏后是否继续推进 | 把任务拉回来 |
| 收束 | 决定是否应尽快关闭不稳定残留 | 决定目标是否已经足够满足 | 把任务关得掉 |

### 4.2 功能模式与方法可靠度使用点

| 功能模式 | 可靠度主要用于 |
| --- | --- |
| 筹办 | 初选方法、决定路径长度、决定检查密度 |
| 维护 | 判断当前方法是否还稳、是否要刷新绑定或切换更稳方法 |
| 纠偏 | 判断当前方法是否失配、选择备用方法、决定回退幅度 |
| 收束 | 判断结果是否足够稳定、能否沉淀经验、是否写回方法经验 |


## 5. 现有状态迁移表

| 当前状态 | 允许迁移到 | 触发条件 | 禁止迁移 |
| --- | --- | --- | --- |
| 未定义 / 未启动 | 就绪 / 筹办中 / 取消 | 创建完成、立即筹办或取消 | 无准备直接到完成 |
| 就绪 | 筹办中 / 排队中 / 运行中 / 取消 / 无法执行 | 进入筹办、进入调度、直接推进或明确不可执行 | 无事件支撑直接到完成 |
| 筹办中 | 就绪 / 等待中 / 待重筹办 / 无法执行 / 失败 | 筹办成功、等待补条件、待后续重筹办、彻底无路或筹办失败 | 绕过筹办结论直接到完成 |
| 排队中 | 执行中 / 运行中 / 取消 / 超时 | 获得执行权、回到运行调度、取消或超时 | 无调度直接到完成 |
| 运行中 | 执行中 / 等待中 / 挂起 / 待重筹办 / 完成 / 失败 / 无法执行 | 开始执行、等待子任务/条件、挂起外援、阶段结束、完成、失败或无路可走 | 无结果支撑直接跳转 |
| 执行中 | 就绪 / 等待中 / 挂起 / 待重筹办 / 完成 / 失败 / 无法执行 | 步骤完成、等待补条件、挂起等待、转重筹办、任务完成、执行失败或无法执行 | 无执行结果直接回完成 |
| 等待中 / 挂起 | 就绪 / 排队中 / 待重筹办 / 取消 | 条件恢复、重新入队、等待结束后重筹办或取消 | 无恢复依据直接到完成 |
| 待重筹办 | 就绪 / 筹办中 / 无法执行 / 取消 | 重评估恢复、重新筹办、确认无路或取消 | 直接抹平成完成 |
| 完成 / 失败 / 取消 / 超时 / 无法执行 | 终结或归档处理 | 收束、归档、审计封口 | 回到执行中 |

必须固定：

- 不允许跳过 `执行中` 从 `就绪 / 排队中 / 筹办中` 直接到 `完成`
- 不允许无事件支撑的状态跃迁


## 6. 执行动作触发表

| 操作 | 主要触发条件 | 典型落地 | 必记字段 |
| --- | --- | --- | --- |
| 保持 | 当前链稳定、状态健康、只需维持可执行性 | 刷新状态和检查点，不扩张主结构 | 控制快照、系统.原因、控制.模式 |
| 细分 | `控制.需求值` 高、`特征.复杂度` 高、深度未超 `任务.最大深度` | 新子任务 + `分解` 关系 | 控制快照、系统.原因、来源.父任务标识 |
| 延长 | 需要补验证闭环、补安全检查、补恢复步骤 | 新后续任务或检查/恢复/总结任务 + `编排.顺序号` | 控制快照、系统.原因、编排.顺序号 |
| 插检查 | 稳定性不足、可靠度偏低、维护期需持续校验 | 新检查任务或检查节点 | 控制快照、系统.原因、目标.任务标识 |
| 替换方法 | 当前方法失配、备用方法更稳、恢复链要求切换 | 更新绑定关系或切换当前选中方法 | 控制快照、系统.原因、目标.方法标识 |
| 回退 | `反馈.失败原因` 非空、`反馈.安全变化` 为负、安全值过低 | 当前任务转 `待重筹办 / 失败 / 无法执行 / 等待中` + 替代方法或恢复任务 | 控制快照、反馈.失败原因、反馈.安全变化 |
| 收缩 | 安全值过低、`特征.波动性` 高、连续失败过多 | 停止细分、停备用扩张、保留主路径 | 控制快照、系统.原因、标签 |
| 归并 | 多条尾部分支已证明等价、可压缩临时结构 | 合并分支、冻结多余链路、保留主结论 | 控制快照、系统.原因、编排.分支编号 |
| 结束 | 完成判据满足、继续投入收益过低、或必须终止 | 关闭任务、释放资源、进入完成/取消/失效收口 | 控制快照、系统.原因、任务.完成判据 |
| 学习修正 | 反馈已形成且需要写回 | 经验实体 + `学习回写` 关系 + 权重更新 | 来源.反馈标识、目标标识、权重变化 |

### 6.1 功能域与执行动作的关系

必须固定：

- `筹办 / 维护 / 纠偏 / 收束` 是任务控制的功能模式
- `保持 / 细分 / 延长 / 插检查 / 替换方法 / 回退 / 收缩 / 归并 / 结束 / 学习修正` 是功能模式内部调用的执行动作

典型归属建议：

| 功能模式 | 常见动作 |
| --- | --- |
| 筹办 | 细分、延长、插检查 |
| 维护 | 保持、插检查、刷新绑定、局部修补 |
| 纠偏 | 替换方法、回退、收缩、插恢复任务 |
| 收束 | 归并、结束、学习修正、归档 |


## 7. 方法选择链表

| 步骤 | 输入 | 输出 | 硬约束 |
| --- | --- | --- | --- |
| 查询候选方法 | 当前任务、来源需求、现有绑定、反馈、经验 | 候选方法集合 | 只允许已注册固定方法 |
| 安全硬过滤 | 候选方法集合、控制.安全值 | 过线方法集合 | 不过线方法直接剔除 |
| 特征软评分 | 过线方法集合、特征字段、关系权重 | 排序结果 | 不得绕过安全过滤 |
| 方向修正 | 排序结果、控制.方向 | 修正后主备序列 | 方向只能修正，不得推翻硬边界 |
| 主备形成 | 修正后序列 | 主方法、备用方法 | 备用数量受风险与复杂度约束 |

最稳的评分口径：

```text
最终方法得分 =
    关系权重
  + 特征匹配分
  + 服务收益修正
  - 风险修正
```

## 9. 停止条件表

| 条件 | 类型 | 命中后允许动作 | 禁止动作 |
| --- | --- | --- | --- |
| `任务.最大深度` 已达上限 | 硬停止 | 维护 / 纠偏 / 收束 / 执行 | 继续细分 |
| `控制.安全值` 低于阈值 | 硬停止 | 纠偏 / 收束 / 终止 | 继续筹办扩张 |
| 连续失败过多 | 硬停止 | 纠偏 / 收束 / 终止 | 继续硬推 |
| `反馈.安全变化` 持续为负 | 硬停止 | 纠偏 / 收束 | 继续探索 |
| `特征.波动性` 过高 | 软停止趋硬 | 维护 / 收束 | 继续增加备用扩张 |

## 11. 收束

本文定义的是：

- `在不新增核心状态枚举的前提下，系统何时进入筹办、维护、纠偏、收束四类功能模式，以及细分、延长、回退、收缩、归并、结束等动作怎样落到现有任务状态和事件链里。`
