# 2200 根规范 词

编号：2200

日期：20260720

状态：正式根规范

## 1. 根定位

词是自然语言中经分界与规范化确认的最小词单元及其稳定语言身份。词面是词身份的语言材料，不是存在、特征、状态、需求、任务、方法或因果本体。

本规范中的“词”指可复用的词身份；某个词在文章、段落或自然句中的一次出现属于语言组成证据，不得与全局词身份混成同一事实。

## 2. 正式结构承载

正式词至少由以下结构共同成立：

```text
语素节点及稳定句柄
+ 明确的词角色材料
+ 规范化词面材料
+ 语言归属 / 规范化规则版本
+ 来源记录与当前发布版本
+ 词根到词的成员关系
+ 词到语素入口的组成关系
+ 精确查找索引或可互证的召回结构
```

允许复用 `节点类型::语素` 作为物理节点类型，但必须由正式角色材料区分词根、词和语素入口；不得根据显示名称、文本长度、所在层级或节点编号猜测角色。

词面必须作为可由语言 / 语素服务完整读回的规范化语言材料保存。只存在于初始化局部字符串、控制面板文本、日志、注释或临时 DTO 中的文字不构成词身份。

## 3. 身份、规范化与组成

1. 词身份由语言归属、规范化词面、规范化规则版本和稳定身份共同确定；显示文本相同但语言归属或规范化版本不同的材料不得直接判为同一词。
2. 同一规范身份的重复创建必须幂等读回；同一稳定键对应不同规范身份必须拒绝。
3. 外部编码、分词器标签、大小写变体和兼容字形只能作为来源材料；进入正式词身份前必须按具名规则规范化，并保留规则版本。
4. 词可以保留有序字 / 子语素组成关系；未形成字级正式结构时不得由显示层按字符串拆字伪造。
5. 自然句中的词序必须保留每次出现及顺序。同一词重复出现不得因词身份相同而去重。
6. 短语、组合系统标签、说明性长名称、未解析句段和只为显示产生的字符串不得直接登记为正式词；它们只能先进入语言记录或待理解候选。

## 4. 与语素入口的边界

词只承载语言身份，不直接绑定业务含义。词与项目内部信息的桥接必须经 2210 语素入口完成：

```text
词
-> 一个或多个语素入口
-> 对应信息 / 概念追溯
```

一个词可以有多个人类词性、多个信息入口类型和多个已理解含义。不得把这些差异压成词节点上的单个业务字段，也不得让词面直接决定信息类型、存储树或业务对象角色。

## 5. 写入方、读取方与验证

语言领域服务负责词面规范化、词身份创建、语言归属和词序材料；语素领域服务负责把已确认词身份接入语素入口。外部调用方不得绕过服务直接写节点、语言域类型化记录、关系或索引。

正式读回应至少验证：

```text
词节点当前可读
角色材料为词
规范化词面非空且编码合法
语言归属与规范化版本完整
词根成员关系唯一可读
词到语素入口关系端点和角色一致
索引命中后精确回读身份，而非只信任哈希或编号
```

内部写入完成后任一结构不能完整读回，属于逻辑错误并追根因；不得把半结构作为普通失败结果保留。

## 6. 禁止事项

- 禁止把词面、显示名、分词结果或人工标签直接作为业务事实。
- 禁止用编号、哈希、外部词性编码或控制面板文字替代可读词面材料。
- 禁止把短语或组合语义塞入单个词节点绕过正式组成关系。
- 禁止由控制面板、日志或说明文档补造机器中不存在的词、字或词性结构。
