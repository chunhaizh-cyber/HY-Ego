# 1140 根规范 存在节点

编号：1140
日期：20260720
状态：正式根规范

## 1. 根定位

存在节点承载具体存在、抽象存在、概念模板、概念集和内部世界入口，是存在类信息的统一身份对象。不得为抽象存在另建平行权威数据类型。

## 2. 本项目承载

```text
存在节点及稳定句柄
+ 主信息身份和唯一主键
+ 场景成员归属关系
+ 宿主到实例特征槽位的归属关系
+ 概念、内部世界和治理入口的具名关系
+ 实际存在或治理虚拟存在的来源关系
+ 定位索引
```

实际存在和治理虚拟存在可以复用存在节点结构，但两者的来源、允许归属和读取入口必须严格区分。需求、任务或方法的治理虚拟存在由对应领域对象独占，不是世界事实存在。

## 3. 事实与边界

1. 存在普通特征结构只承载该存在的一阶实例特征槽位，以及作为该存在实例特征入账的二次特征。
2. 子存在必须作为该存在内部世界场景的成员，不得直接成为父存在普通子节点。
3. 具体内部世界场景必须先归属于世界根的内部世界场景，再由宿主存在持有正式引用；不得挂入宿主所在现实场景冒充现实成员。
4. 状态、动态、因果、需求、任务和方法不得直接挂入存在普通特征结构。
5. 需求根、任务根和方法根属于治理入口关系，不是存在普通特征成员。
6. 分类证据、置信、版本、冲突、位置历史、位移和连续观测统计必须由特征、状态、动态、二次特征、场景或因果证据表达，不得复制为存在本体权威字段。
7. 高频读取可以建立服务访问函数或明确标注的非权威缓存；缓存不得参与机器裁决。

## 4. 写入、读取与验证

存在领域服务是存在身份、虚拟来源和存在专属结构的业务写入口；场景领域服务负责场景接纳；特征领域服务负责存在特征。任一服务不得越权直接写其它领域结构。

验证实际存在必须回读存在身份、主信息和允许的场景归属。验证治理虚拟存在还必须回读唯一来源关系，并证明它未进入世界根、现实场景、内部世界场景或场景普通成员投影。

## 5. 生命周期与当前性

存在的长期身份由稳定句柄、主键和版本共同成立。场景归属、概念归属或内部世界关系变化不得更换存在身份；关系变化必须保留来源和历史。治理虚拟存在的生命周期受其来源需求、任务或方法控制，不得由世界场景生命周期裁决。
