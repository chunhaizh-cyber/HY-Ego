# 6140 子规范 分层安全值被动变化与残余风险维护

编号：6140
日期：20260724
状态：正式子规范

## 1. 定位

本规范定义自我各安全结果层的安全值、低位阈值、高位阈值、残余风险约束、有效未复发时间和被动变化规则。它只裁决安全值的被动维护，不定义服务值衰减，不把任务调度压力反写为安全事实。

直接上级规范：1120、1130、1160、1170、1190。

直接依赖规范：4020、4210、4220、6100、6130。

相关边界规范：3100、3200、4320、6110、6120、6150、8100、8200。

## 2. 安全层维护单元

一个安全层维护族必须同时绑定：

```text
自我稳定身份
+ 本层安全目标或结果身份
+ 最终具体安全结果身份
+ 安全抽象定义身份和定义版本
```

同一维护族在某一时刻的安全层投影单元还必须绑定：

```text
当前来源因果信息
+ 当前因果图版本
+ 当前安全值状态
+ 低位阈值、高位阈值和维护参数版本
+ 当前残余风险约束集合
```

“层”由具名来源因果信息、目标安全结果和当前因果图版本共同形成投影，不由需求名称、固定需求类别、C++ 枚举、显示标签或预置安全因素表成立。安全目标到具体安全结果的精确因果深度及优先级组由 6150 动态计算；深度和组不是维护族的持久身份组成部分。因果图换代时旧投影单元退出当前使用，新投影单元必须显式承接仍适用的安全值历史，不得原地覆写或把旧值无来源复制为当前值。

安全层维护单元不得另造通用安全记录。当前值必须由自我实例特征槽、当前值关系和完整状态事实承载；阈值、速率、时间单位、舍入规则和规则版本只进入安全领域的类型化定义记录。事件、因素、场景和证据之间的拓扑只由正式关系承载。

## 3. I64 值域与阈值合同

本规范中的安全值、值域下界、值域上界、低位阈值、高位阈值、变化速率和规则版本均使用 `I64`。每个安全层定义必须冻结：

```text
值域下界 Vmin
值域上界 Vmax
低位阈值 L
高位阈值 H
低位回升速率 Rup
高位回落速率 Rdown
维护时间单位 Utime
规则版本
```

合法值必须满足：

```text
Vmin <= V <= Vmax
Vmin <= L < H <= Vmax
Rup >= 0
Rdown >= 0
Utime > 0
```

阈值和值必须使用同一安全定义、同一量纲和兼容版本。不得把不同安全层的原始值直接相加、平均或比较。

6120 定义的 `0..99` 自我物理安全根值不属于本规范的分层安全值，不得由本被动维护入口写入；它只按 6100 的物理安全事实和结算边界变化。某个分层安全定义若合法复用 `0..99` 量纲，仍必须使用独立状态和来源，不能成为物理安全根值的副本；其中 `0`、`1` 只有在该分层定义具名继承控制态语义时才保留，且不得由被动维护跨入或跨出。

## 4. 安全事件、因素和残余风险约束

一项安全事件必须能够回读事件身份、受影响自我、本层安全目标或结果、具体安全结果、发生场景、发生时间、来源状态和动态、当前版本及因素搜索状态。

事件相关因素必须逐项具备稳定身份、与事件的因果关系、适用场景、当前性和证据版本。因素状态至少区分：

```text
已确认仍相关
待验证
已验证排除
证据失效
```

“已验证排除”必须有能证明该因素在当前事件及适用范围内不再成立的状态、动态、观察覆盖、因果或处置结果。没有发现、暂时不可见、当前未复发、解释不足和未知因素均不等于已排除。

一个事件只在同时满足以下条件后解除残余风险约束：

1. 因素搜索已经按当前规则版本完成；
2. 全部具名相关因素均有当前有效的验证性排除事实；
3. 不存在未收束的未知因素范围；
4. 排除事实适用于当前自我、具体安全结果和场景范围。

任一条件不满足时，该事件仍是残余风险约束。约束存在不等于新的安全事件已经发生，也不得单独证明当前安全值应立即降低；它只决定高位被动回落是否继续成立，并限制“全部风险已解除”的声明。

## 5. 有效未复发证据

物理时间经过本身不产生安全事实。某一事件或因素的时间区间只有同时满足以下条件，才计入有效未复发时间：

1. 自我在该因素具有复发机会的场景范围内持续存在；
2. 当前场景与原事件或因素的适用场景身份可比较，且比较规则版本兼容；
3. 感知、监测或其它正式确认能力覆盖整个区间；
4. 事件和因素身份在区间起止处保持可比较，未发生不可解释换代；
5. 覆盖材料确认区间内没有同一事件或因素复发；
6. 起止时间来自单调实际时间，结束时间不早于开始时间。

循环次数、线程唤醒次数、日志数量、墙钟跳变和“没有收到消息”不得换算为有效未复发时间。

每个未解除约束的具名因素分别累计有效时间。因素搜索尚未完成时，未知因素范围只允许使用“同一安全事件可被有效探测且未复发”的事件级覆盖证据，不得伪造具名因素排除。多事件、多因素共同约束同一安全层时，本轮可消费的有效时间取全部当前约束项新增有效时间的最小值；任何约束项缺少复发机会、覆盖或身份可比性时，本轮新增有效时间为零。

不连续区间只能按各自有效长度相加；重叠区间不得重复累计。场景离开、覆盖中断、身份不可比或证据版本失效时立即截断当前区间，不得跨越缺口补算。

## 6. 被动变化分支

设当前安全值为 `V`，低位阈值为 `L`，高位阈值为 `H`。

### 6.1 低位回升

```text
V < L
+ 存在本轮新增有效未复发时间
-> 按低位回升速率逐渐增加
-> 最大只到 L
```

低位回升只表示当前有效未复发证据允许安全注意从历史事件中逐步释放到低位阈值，不证明事件因素已经排除，也不证明物理安全已经主动改善。其它任务需要注意、调度资源紧张或上层需求等待均不得提高安全值。

`V = L` 时不发生低位回升。

### 6.2 区间保持

```text
L <= V <= H
-> 不发生本规范定义的被动回升或回落
```

区间保持不阻止新安全事件、主动处置结果、因素排除或其它正式事实通过 6100 的安全结算边界改变安全值。

### 6.3 高位回落

```text
V > H
+ 至少一个相关历史事件仍有残余风险约束
+ 存在本轮新增有效未复发时间
-> 按高位回落速率逐渐降低
-> 最小只到 H
```

高位回落只表示尚未排除的潜在风险不允许安全层长期保持超过高位阈值的放松程度。它不是新的伤害事件、惩罚或服务值衰减。

`V = H` 时不发生高位回落。全部相关事件均已按第 4 章解除残余风险约束时，高位回落停止，并由主动安全结算入口按当前事实重新裁决；不得继续以历史事件压低安全值。

## 7. 线性变化、累计余数和舍入

每一方向使用同一确定性整数公式。设本方向速率为 `R`，自本方向维护纪元开始累计的有效未复发时间为 `T`：

```text
累计应变化量 Q(T) = floor(R * T / Utime)
本轮新增变化量 Delta = Q(Tnew) - Q(Told)
```

低位回升：

```text
Vnew = min(L, V + Delta)
```

高位回落：

```text
Vnew = max(H, V - Delta)
```

`R`、`T`、`Utime` 和结果均为非负 `I64`。乘法必须使用至少 128 位中间值或等价的检查乘除。舍入固定为对非负商向下取整；低于一个值单位的有效变化保留在累计时间中，不强行变化 1，也不因多次小区间调用丢失余数。

发生下列任一情况时，不得钳制、绕回或用 `I64_MAX` 兜底：

- 中间乘法或累计时间超出受支持表示范围；
- `Delta` 为负或与累计时间不一致；
- 加减结果越过值域；
- 参数版本、量纲或方向不兼容。

前置材料非法时写前结构化拒绝；前置已经通过后的算术或版本异常属于内部逻辑错误，整批不得发布。

## 8. 新事件、排除和被动维护的顺序

新安全事件、复发、因素变化和排除证据先由各自合法领域完成准入和正式发布；安全领域不得在本维护入口内代写这些上游事实。安全评估在一个冻结的已发布事实批次上严格按以下顺序：

```text
冻结已发布的新事件、复发和因素事实版本
-> 先处理新安全事件、复发和因素变化
-> 执行主动安全事实结算
-> 读取因素验证性排除并重算残余风险约束
-> 裁决是否允许被动维护
-> 形成被动变化候选
-> 原子发布本轮安全值、状态动态和维护账
```

新安全事件或复发事实优先于被动变化。同一批次存在新事件、复发或主动安全值结算时，先按新事实更新安全值并重置受影响事件、因素和方向的有效时间维护纪元；该批次不再用事件发生前的累计时间抵消新事件结果。

因素全排除解除残余约束后，必须按当前事实重新结算。未知因素不得借“未复发”或排除其它因素而升级为已排除。

安全值因主动事实进入、离开或跨越阈值区间时，旧方向维护纪元结束；位于 `[L,H]` 的时间不得储存为将来跨区间后的被动变化额度。

## 9. 幂等与原子发布

一次被动维护的幂等键至少包含：

```text
自我身份
安全层维护单元身份
当前安全值状态版本
残余风险约束集合版本
有效未复发证据区间及覆盖版本
阈值和维护参数版本
维护方向
规则版本
```

精确重复必须读回既有结果，不得重复累计时间、重复写状态或重复生成动态。任一相同幂等键对应不同结果属于内部逻辑错误。

正式发布必须在一个安全领域事务中完成：

1. 读回当前安全值、阈值、约束、证据和规则版本；
2. 形成新的累计时间、方向维护纪元和安全值候选；
3. 创建完整后状态和被动状态动作动态；
4. 关联前状态、后状态、场景、证据和正式提交方法；
5. 使旧当前值关系失效并建立唯一新当前值关系；
6. 回读全部结构和值；
7. 最后发布本轮维护记录和当前关系。

候选确认前不得被普通读取看见。失败必须逆序撤销；不得留下时间已推进但值未发布、值已发布但动态缺失、旧新值同时当前或残余约束已解除但历史被改写的半结构。

被动安全维护的动作来源是安全领域正式提交方法，动作语义固定为“被动动作”。自我线程只负责每轮调用评估，不是动作来源；每轮调用可以合法得到“无变化”结果。

## 10. 权力与边界

1. 安全事件、因素、场景、覆盖和排除事实由各自领域服务写入；安全领域服务只读取已发布事实并裁决安全值。
2. 任务调度、任务数量、执行队列压力和上层需求竞争不得写安全值、阈值、残余风险或有效未复发时间。
3. 权限为零不得阻止观察、事件登记、因素验证和机器事实写入。
4. 本规范不得直接套用于服务值。服务值衰减、成功服务和休眠继续由 6100、6120 规定。
5. 被动变化不得证明需求满足、任务完成、方法成功、因素排除或因果稳定。
6. 显示投影、日志和普通返回码不得承担当前安全值、残余约束或维护纪元的权威事实。

## 11. 逻辑内结果与内部错误

以下是合法逻辑内结果：

```text
位于阈值区间，无被动变化
没有新增有效未复发时间
残余约束已全部解除，不执行高位回落
累计变化不足一个值单位
精确重复，读回既有结果
```

以下属于内部逻辑错误并停止依赖路径：

```text
前置通过后缺当前值、阈值、事件、因素、场景或证据关系
同一因素同时有效地“仍相关”和“已排除”
把未知因素写成已排除
用循环次数或无消息时间累计有效时间
同一有效区间重复计时
新事件批次仍先执行被动回归
算术溢出后钳制或使用 I64_MAX 兜底
事务发布留下旧新当前关系或不完整状态动态
```

## 12. 验证合同

验证至少覆盖：

1. 安全层身份不依赖固定需求类别、显示名称或预置因素表。
2. 值域、低位阈值、高位阈值、速率和时间单位的 I64 合法性及版本互证。
3. `V<L`、`V=L`、`L<V<H`、`V=H`、`V>H` 六类边界。
4. 复发机会、场景可比、覆盖连续、身份可比和未复发五项证据逐项缺失时不计时。
5. 多事件、多因素、多场景取最小新增有效时间，重叠区间不重复。
6. 新事件优先、维护纪元重置、同批不抵消和因素全排除后解除约束。
7. 未知因素不冒充排除，搜索未完成时残余约束保留。
8. 累计余数、多次短区间与一次等长区间产生相同结果。
9. 向下取整、封顶到 `L`、封底到 `H` 和 checked wide integer 溢出拒绝。
10. 精确重复幂等、异义重复冲突、事务最后发布和失败零半结构。
11. 物理安全根值不被本入口写入；具名继承控制态语义的分层值不被动跨越 `0`、`1`。
12. 调度压力不能反写安全值，线程不成为动作来源，服务值不消费本算法。
