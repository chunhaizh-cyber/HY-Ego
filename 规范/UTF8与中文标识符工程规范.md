# UTF-8 与中文标识符工程规范

编号：018
日期：2026-07-05
状态：待确认规范 / 工程边界 P2

## 1. 定位

本项目要求函数名、变量名和类内业务成员名使用中文。为避免编码不一致导致构建和审查困难，源文件、文档和构建必须统一 UTF-8 口径。

## 2. 文件编码

```text
Markdown 文档使用 UTF-8。
C++ 源文件使用 UTF-8。
PowerShell 命令读取中文文件时显式使用 UTF8。
搜索中文路径时优先使用 rg。
```

## 3. C++ 边界

允许英文：

```text
C++ 关键字
标准库类型
第三方 API
编译系统字段
外部协议固定字段
```

禁止：

```text
为了方便新增英文业务变量名
为了显示临时文本新增机器逻辑字段
用乱码路径或乱码输出作为验证证据
```

## 4. 构建口径

Visual Studio / MSBuild 构建必须保持 UTF-8 源文件解析。当前项目命令优先使用：

```powershell
msbuild .\海中鱼巣.vcxproj /p:Configuration=Debug /p:Platform=x64 /m
```

## 5. 验收方式

```text
中文路径文件可读取。
中文标识符可编译。
构建输出无编码相关错误。
文档和代码中的中文不出现乱码。
```
