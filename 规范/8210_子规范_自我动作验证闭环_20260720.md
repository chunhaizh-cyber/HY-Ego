# 8210 子规范 自我动作验证闭环

编号：8210
日期：20260721
状态：正式子规范

## 1. 定位

本规范规定物理、内部治理、仓库和推理动作从目标、预测、执行、实际回读、验证、事实提交到纠偏和学习的统一闭环。

直接上级规范：1150、1160、1170、1180、1190、3200、3300。

直接依赖规范：4220、4320、6330。

相关边界规范：8200。

## 2. 总公理

事实来自对应实在域的实际回读，不来自目标、预测或执行报告。失败事实仍然是事实，必须被保存、解释和治理。

```text
目标态 G
-> 预测态 P
-> 执行事件 E
-> 实际回读 R
-> 比较 G、P、E、R
-> 事实提交等级
-> 任务治理、纠偏和学习
```

预测态只能存在于预测场景或私有值式模板。执行器返回“成功”只属于执行报告态。两者都不得直接写成事实态。

## 3. 状态分账

必须区分目标状态、预测状态、执行报告状态、原始回读状态、确认事实状态和待复核状态。

实际回读和预测冲突时保留实际回读，不得为了符合目标而丢弃、覆盖或改写。低置信回读可以进入待复核，但不能用预测值补齐事实。

## 4. 动作运行实例

每次动作必须有独立运行身份，至少关联：

- 来源任务、执行方法和动作入口；
- 目标对象、目标状态和允许误差；
- 动作前事实版本；
- 预测状态、依据、时间和容差；
- 执行开始、结束、指令或回调身份；
- 实际回读、来源域、时间、版本和置信；
- 目标残差、预测残差和执行报告偏差；
- 验证结论、提交等级和后继承接。

动作运行实例可以由自我内部世界中的专属运行存在和场景承载，但不得污染方法长期治理存在或世界事实对象。

## 5. 场景边界

目标场景和预测场景是模板或运行材料，禁止直接进入事实世界。实际回读场景保存对应实在域读取的状态；只有通过来源、时效、质量、范围、版本和验证门禁后，才可由领域服务提交为正常、异常或待复核事实。

D455 原始材料必须先经过观察材料、质量、对应和自我侧承接，不能直接写最终事实。

## 6. 统一实际回读

实际回读按实在域分别取得：

| 实在域 | 实际回读 |
| --- | --- |
| 物理外部域 | D455、IMU、编码器、力传感器等合法感知材料 |
| 内部治理域 | 方法执行后任务、需求、方法治理对象和运行存在的实际状态 |
| 仓库域 | 按稳定主键、关系、索引和版本重新读取的发布结构 |
| 推理域 | 可复核输入、步骤、输出和证据，而非预期结论 |

每个回读都必须带来源、时间、版本、范围和置信。动作声称改变内部对象但后置版本未推进，必须进入验证失败或待复核。

## 7. 预测器和执行器

预测器只能生成预测材料，必须保存预测依据、时间、容差和可验证特征。执行器必须保存运行身份、开始结束时间以及异步指令、版本或回调标识。

执行器返回值不构成事实。发送控制指令成功不能证明物理状态改变；领域函数返回成功不能证明治理对象已经形成目标状态。

## 8. 验证器

验证器至少比较：

- 目标与实际回读；
- 预测与实际回读；
- 执行报告与实际回读；
- 动作前后版本和状态；
- 来源、时间、范围和置信；
- 禁止结果和副作用约束。

比较必须使用明确特征类型、比较符、阈值和值域。每次验证形成结构化残差，不能只生成成功、失败或自然语言说明。

## 9. 事实提交等级

至少区分：

| 等级 | 处理 |
| --- | --- |
| 正常事实可提交 | 由对应领域服务提交正常事实 |
| 异常事实可提交 | 保留实际异常，不伪装正常 |
| 待复核 | 暂存并保持高置信旧事实，形成补材料合同 |
| 不可提交 | 禁止写事实，形成补观察或治理需求依据 |

所有提交事实方法都必须经过统一门禁。提交入口不是绕过验证的暗门。

## 10. 物理外部域

物理动作至少完成动作前感知、预测、作用执行、动作后感知和三层验证：外设自身反馈、外部感知回读、任务目标比较。

D455 材料必须标明坐标系、位姿版本、时间、深度质量和遮挡。关键动作需要连续稳定材料后才能提交最终事实。运动模糊、遮挡和低质量只能形成降级或待复核。

## 11. 内部治理、仓库和推理域

治理动作执行后必须从实际治理对象回读目标特征、约束、优先级、分配和版本。若规则约束使目标不可达，保留实际状态并形成目标或约束治理依据。

仓库写入后必须按稳定身份重新读取节点直接字段、领域类型化记录、父子或具名关系、索引和版本。读回与预期不一致属于内部结构错误，禁止用预测结构补写检查结果。

推理动作至少保存输入、规则或模型版本、关键步骤、输出和可复核证据。输出不符合目标时仍保存实际推理输出或异常推理事实。

## 12. 时间、版本和置信

动作必须保持单调时间序列：目标形成、预测、开始、结束、回读、验证和提交。内部对象和仓库动作必须保存前后版本；预期改变时，后版本应推进或存在明确变化动态。

实际回读必须有置信。低置信事实不能覆盖关键高置信事实，除非具名恢复策略允许并留下来源证据。

## 13. 并发和锁

采集、重计算、视觉解析、推理、匹配和验证应在锁外完成；只在最短提交段持有规定锁和结构事务会话。

禁止持有帧锁进行长时点云重建后再写场景，禁止持有消息队列锁等待外设回调，禁止持有世界结构写锁执行长时推理，禁止从后序锁域反向申请前序锁域。

## 14. 失败、残差和责任分配

动作未达目标时先保存真实回读和残差，再区分目标偏差、预测偏差、执行报告偏差、材料不足、约束阻挡和内部结构错误。

偏差不是被擦掉，而是分配给不同修正对象：动作补偿、方法可信度、预测模型、目标与约束、补观察或重新规划。不同偏差不得混用。

执行报告成功但实际失败，应降低报告或方法可信度；目标不可达时不得机械重复补偿，应形成调整目标、解除约束或重新规划依据。

## 15. 纠偏闭环

纠偏器只分析偏差并生成结构化后继承接，不得直接改事实、任务完成、需求满足、方法可靠度或模型参数。

任何补偿动作都必须建立新的动作运行身份，再次经过预测、执行、实际回读和验证。禁止开环纠偏。

方法可靠度和模型参数只能基于多次实际回读、残差、置信和结构化偏差缓慢更新，不能被单次异常大幅改写。

## 16. 与任务和方法的关系

任务治理不能只看执行成功或失败，而要读取事实提交等级、目标残差、验证结论、副作用和当前性。事实不可提交时任务不得完成；方法返回成功但目标未达成时任务不得完成。

方法选择应吸收验证成功率、报告可信度和预测命中率。经常报告成功而实际失败的方法必须降权。

动作验证结果只能作为任务域提交依据，不能裸写任务状态。派生需求至少携带目标对象、目标特征、目标状态、来源动作动态、结构化偏差和当前状态。

## 17. 禁止规则

1. 禁止把目标或预测直接写成事实。
2. 禁止把方法返回成功或指令发送成功当作事实改变。
3. 禁止无实际回读时用预测补写。
4. 禁止丢弃不符合期望的真实回读。
5. 禁止低质量外设材料提交为正常事实。
6. 禁止用执行报告单独完成任务或强化方法。
7. 禁止纠偏器直接改事实或模型。
8. 禁止在执行期补造冻结前缺失的方法结构。

## 18. 验证边界

至少建立一个物理动作和一个内部治理动作样板闭环，并验证目标、预测、执行、回读、残差、提交等级、任务不误完成和纠偏重新入链。

缺真实外设、真实回读和正式提交证据时，不得宣称物理动作闭环完成。
