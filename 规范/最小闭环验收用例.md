# 最小闭环验收用例

编号：014
日期：2026-07-05
状态：待确认验收用例 / 核心内核 P0

## 1. 目标

本用例把 `规范/最小闭环设计.md` 中的结构变化变成可执行验收目标。控制台输出只能展示结果，不能替代结构验收。

## 2. 种子准备验收

输入：

```text
世界服务创建自我存在
世界服务创建现实场景
特征服务创建目标特征类型
特征值服务写入当前值 1
```

必须产生：

```text
存在节点有效
场景节点有效
特征节点有效
特征值节点有效
特征值可读为 1
存在与场景、特征、特征值之间的关系可读
```

入口拒绝：

```text
无效主信息不能创建节点
无效节点不能创建关系
非信息节点或未确认值承载节点不能作为语素对应信息节点
```

## 3. 闭环触发验收

输入：

```text
需求服务创建需求：目标特征值达到 2
```

前置条件：

```text
目标存在已存在
目标特征已存在
当前特征值已存在
比较方式已定义
验证方式已定义
```

入口拒绝：

```text
目标存在无效 -> 拒绝
目标特征无效 -> 拒绝
当前特征值不存在 -> 拒绝
比较方式非法 -> 拒绝
验证方式缺失 -> 拒绝
```

## 4. 方法执行验收

必须产生：

```text
任务节点新增或任务承接关系新增
方法节点可被任务引用
目标特征值从 1 变为 2
动态节点新增
因果引用节点新增
动态能回查来源方法
因果引用能回查动态和被改变特征值
```

## 5. 需求结算验收

必须产生：

```text
需求结算结果为已满足
结算依据可回查目标特征值
结算不依赖控制台输出
结算不依赖日志或弹窗
```

## 6. 禁止通过条件

以下情况不得声明最小闭环通过：

```text
只有控制台输出，没有结构读取。
目标特征值未发生结构变化。
任务状态没有结构承载。
动态或因果引用缺失。
无效入口没有被函数入口拒绝。
依赖事后扫描修复结构。
```
