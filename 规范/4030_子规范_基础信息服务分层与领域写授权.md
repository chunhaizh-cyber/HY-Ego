# 4030 子规范 基础信息服务分层与领域写授权

编号：4030
日期：20260722
状态：正式子规范

## 1. 定位

本规范统一定义仓库、服务专用数据操作、基础信息原子服务、聚合服务、高级服务和入口服务的依赖方向与写入授权。它只规定层级权力和公开边界；各对象可写结构、业务状态、关系角色和结果语义由对应对象根规范与主题子规范唯一拥有。

直接上级规范：0050。

直接依赖规范：4010、4020。

相关边界规范：4040、4050、4070、4110、4210、7200、7210。

## 2. 分层模型

写入调用方向固定为：

```text
外部调用方 / 线程 / 消息
-> 应用或领域组合器
-> 领域服务公开业务入口
-> 服务专用数据操作
-> 不透明结构写入执行边界
-> 节点仓库、关系仓库、领域类型化记录参与者和索引仓库
```

各层职责不得倒置：

| 层级 | 唯一职责 | 不得承担 |
| --- | --- | --- |
| 仓库 | 句柄、版本、记录状态、端点、索引绑定和本仓库结构不变量 | 需求满足、任务完成、方法可执行等业务裁决 |
| 服务专用数据操作 | 隐藏仓库与事务细节，按类型化请求完成精确结构读写和结果转换 | 决定业务目标、授权、选择、满足或学习语义 |
| 领域服务 | 业务准入、领域不变量、幂等语义、结构请求和结果解释 | 签发许可、操作仓库锁、向调用方暴露原始会话 |
| 聚合服务 | 按业务顺序编排多个原子服务 | 绕过原子服务直接写其结构 |
| 应用 / 领域组合器 | 定义跨服务业务顺序和一次组合边界 | 直接写仓库或把原始事务能力交给线程 / 消息 |

显示、日志、控制台和 SQL 投影只能读取正式结构或人读材料。持久化层只能提供快照、事件和恢复候选；它们均不得调用写入口形成当前业务事实。

## 3. 基础信息服务分类

### 3.1 原子服务

基础信息原子服务为：

```text
特征值服务
特征服务
存在服务
场景服务
状态服务
动态服务
二次特征服务
因果服务
```

原子服务只拥有对应对象规范授予的创建、读取、变更和发布权。列入本表不表示它可以写其它对象，也不表示相应入口已经接通。

### 3.2 聚合服务

基础信息聚合服务为：

```text
世界服务
概念图服务
```

世界服务组织存在、场景、特征、状态、动态、二次特征和因果等原子服务形成世界结构；它不得自行复制这些原子服务的节点、关系或值写入算法。

概念图服务组织存在、动态、二次特征、因果、状态和语素名称入口等正式能力形成概念结构；概念身份、关系、生命周期和用途观察准入必须回到对应正式规范与专用数据操作。概念图服务不得创建需求、任务或方法，也不得以名称、缓存、日志、显示或 SQL 裁决概念身份。

### 3.3 高级服务与入口服务

高级服务为需求服务、任务服务和方法服务。它们分别拥有目标、授权承接和能力对象的业务准入，不存在独立的 `高级信息` 节点类型或通用高级信息仓库。

语素服务是语言入口服务，只把词和语素入口路由到已经由对应领域服务形成的长期信息。它不创建基础信息、高级对象或场景内实例状态 / 动态，也不把候选静默提升为事实。

## 4. 仓库与领域服务权力边界

仓库负责的最小结构包括：

```text
创建、读取、删除和审计节点 / 关系
读写和审计具名领域类型化数据记录
绑定、查询和删除索引候选
完整句柄、版本、状态和端点复核
普通父子挂载 / 重挂与通用关系读取
仓库内部原子不变量
```

仓库不得判断：

```text
需求是否满足
任务是否授权、完成或应重筹办
方法是否可选择、可执行或已学习
状态、动态或因果是否构成业务结论
语素候选是否对应真实信息
概念是否应命名、激活、退役或删除
```

领域服务负责在第一笔业务写入前完成身份、类型、权限、版本、生命周期、来源关系和对象业务前置条件复核；通过后只提交类型化结构请求。仓库结构写入成功不自动等于领域操作成功，领域结果必须按对象规范读回完整结构后形成。

对象具体写入矩阵不得集中复制到本规范。每个节点类型、关系类型、状态、动态、需求、任务、方法或因果对象的合法写入方，由其根规范和直接主题子规范唯一规定；本规范只裁决调用层级和越权路径。

## 5. 服务专用数据操作边界

跨节点、关系、领域类型化记录、索引或服务内部材料的写入必须经具名服务专用数据操作。该层负责：

```text
隐藏仓库引用、锁、许可、原始事务令牌和写入会话
接收领域服务已形成的类型化请求
在同一结构边界内执行通用前置复核和精确读写
返回稳定句柄、值式记录或结构化操作结果
保持候选、确认、撤销和最后发布顺序符合事务规范
```

数据操作不得解释需求满足、任务授权、方法选择、因果稳定或概念生命周期；不得向领域服务或调用方返回可变仓库引用、内部容器、锁、许可、原始令牌或写入会话。

单仓库操作只有在它不携带领域业务语义且仓库正式入口已经完整承担通用不变量时，才可由仓库内部直接完成。任何多仓库组合、业务唯一发布点或服务内部侧表参与均不得以“只有几步”为由绕过数据操作与统一事务边界。

## 6. 依赖方向

### 6.1 特征值封口

特征值服务只服务特征系统，只有特征服务可以直接访问。需求、任务、方法、世界、概念图、语素或其它高级服务需要特征状态和值材料时，必须通过特征服务取得值式结果或提交修改请求。

禁止把特征值服务、任一领域类型化记录或 I64 写入口扩张为全部业务对象的通用值服务。

### 6.2 聚合服务依赖

世界服务和概念图服务只能依赖基础信息原子服务的公开入口，以及本规范或其对象规范明确允许的专用只读 / 数据操作入口。聚合服务不得直接创建原子服务独占的节点或关系，也不得用仓库写入模拟原子服务。

聚合服务需要其它领域信息时优先取得值式只读材料。跨领域需要共同写入时，必须交给明确组合器和统一事务设计，不得让一个聚合服务取得另一个领域的永久写权。

### 6.3 高级服务依赖

需求、任务和方法服务可以依赖存在、场景、特征、状态、动态、二次特征和因果服务中与本对象规范相关的公开能力，但必须遵守：

1. 不直接访问特征值服务。
2. 不绕过状态 / 动态服务创建相应节点。
3. 不绕过另一个高级服务改写其对象事实。
4. 任务服务不得写方法长期能力；方法服务不得写任务专有回执或任务完成；需求服务不得执行方法。
5. 跨高级对象的读取使用专用值式入口；写入由该对象所有者服务完成。

### 6.4 入口服务依赖

语素服务只绑定已经存在且通过类型、生命周期和长期性复核的信息节点。语素对应信息、概念追溯和入口索引由语素服务按 2200、2210、7200、7210 写入；目标对象本体仍由其领域服务唯一写入。

场景内实例状态和实例动态不得直接绑定为长期语素入口。候选查询、索引命中、词面匹配或显示名称均不能替代目标对象复核。

## 7. 公开接口边界

领域服务公开业务入口只允许接收：

```text
业务请求
稳定句柄
不可变值式材料
具名幂等身份
结构化领域选项
```

公开入口只返回领域结果、稳定句柄、值式快照或结构化拒绝。禁止暴露、接收、返回、传递或比较：

```text
原始结构事务令牌
结构许可或仓库锁
仓库引用和内部容器
可变记录引用
写入会话、候选能力或撤销 / 确认控制权
```

线程和消息只是请求载体，不因持有对象句柄而取得写权。外部调用方不得决定仓库写入顺序、撤销顺序、确认点或最后发布点。

## 8. 越权判断与写前拒绝

是否越权至少依据：

```text
调用方服务身份
目标对象与节点 / 关系类型
对象根规范指定的写入方
直接依赖是否在允许方向内
是否绕过原子服务、对象所有者服务或专用数据操作
是否暴露原始事务、仓库或可变内部状态
所需前置身份、关系、生命周期和版本是否完整
```

下列路径必须在第一笔业务写入前拒绝：

- 外部、显示、日志、控制台或 SQL 投影直接写仓库。
- 持久化材料绕过当前规范准入直接恢复为事实。
- 世界或概念图聚合服务绕过原子服务写基础结构。
- 需求、任务或方法服务直接访问特征值服务。
- 一个高级服务直接改写另一个高级对象的专有事实。
- 语素服务创建目标业务对象或把候选裁决为事实。
- 领域公开入口暴露原始事务能力、仓库引用或锁。
- 新服务未登记依赖、写入对象、拒绝和验证合同即进入生产写路径。

拒绝不得产生节点、关系、领域类型化记录、索引、服务内部材料或业务状态变化。前置通过后的内部结构异常属于内部逻辑错误，必须按 0050 的共同错误原则停止新增写入并收口；具体结果分类由 4050 独立规定，不得回写成本规范的新权限例外。

## 9. 新增服务准入

新增服务或迁入旧能力前必须登记：

```text
服务身份与唯一职责
所属层级
公开请求和结果
可读对象与可写对象
允许直接依赖和禁止依赖
专用数据操作与事务边界
入口拒绝和内部逻辑错误
生命周期、幂等与版本
验证方式与禁止宣称
```

如果现有服务可以在不扩大职责的前提下承载新入口，应扩展现有服务；不得为显示、日志、SQL、临时兼容或旧函数搬迁新增具有业务写权的平行服务。新增依赖不得形成服务循环；真实跨域组合由上层组合器拆解并明确所有权。

## 10. 验证合同

实现验证至少证明：

1. 外部调用只能到达领域服务公开入口，生产路径没有仓库裸写和原始事务穿透。
2. 特征值服务只有特征服务直接访问；高级服务通过特征服务取得材料。
3. 世界和概念图聚合服务调用原子服务，不复制原子写入。
4. 需求、任务和方法服务不互相改写专有事实，只经值式入口读取或请求所有者服务写入。
5. 语素服务只绑定已存在长期信息，候选和实例状态 / 动态不会被提升为长期事实。
6. 每个写路径能够回指对象规范指定的唯一写入方和具名数据操作。
7. 越权或前置拒绝前后，相关仓库和服务内部材料均无变化。
8. 新服务具有完整登记且依赖图无环。

验证必须结合依赖扫描、公开签名复核、拒绝前后结构读回和目标对象验收；编译通过、函数可调用、日志成功或返回码为真均不能单独证明授权边界成立。
