# 7140 子规范 概念身份生命周期退役删除与活动投影分账

编号：7140
日期：20260722
状态：正式子规范

## 1. 定位

本规范统一定义四类概念的稳定身份、概念签名、关系 9—12、生命周期、退役、删除、历史审计以及活动图 / 登记 / 统计投影的分账边界。概念不是新的通用节点类型；存在、动态、关系和因果概念继续由各自对象身份承载。

直接上级规范：0050。

直接依赖规范：1140、1170、1180、1190、4010、4020、4030、4040、4050、4060、7100、7110。

相关边界规范：4070、4150、4330、7200。

## 2. 概念身份与四类根

当前概念结构只有四个固定类别根：

```text
存在概念根 -> 节点类型 3、存在身份命名域
动态概念根 -> 节点类型 10、动态身份命名域
关系概念根 -> 节点类型 13、二次特征身份命名域
因果概念根 -> 节点类型 11、因果身份命名域
```

每个根使用固定系统角色稳定主键、完整节点句柄和本类对象结构共同成立。四根永久活跃，不允许冷却、退役、删除或被普通概念替换。根登记数组、名称、索引命中、活动快照或显示树都不得单独证明根身份。

普通概念沿用所属对象节点类型和命名域。概念类别由节点类型、唯一类别根可达路径和本类对象正式结构共同确定；不得新增通用“概念节点”、概念类型字段、影子登记表或运行期枚举作为第二身份事实。

## 3. 概念签名类型化记录

不能从所属对象结构和关系确定重建的概念签名原始值，进入具名的概念签名类型化记录。每项记录至少包含：

```text
所属概念完整句柄
记录版本、格式版本和记录状态
签名规则版本
确定排序的约束项组：
  约束维度
  角色顺序号
  原始值类型
  I64 / VecI64 / VecU64 / 节点类型值
```

记录不得保存定义节点句柄、稳定目标句柄、关系句柄、活动版本、登记位置、索引物理键、统计值或通用任意槽。需要引用节点的约束由关系 11 承载，角色顺序号与类型化记录中的约束项互证。

概念签名发布后不可原地改义。签名实质变化形成新的概念稳定身份；旧节点、签名记录和关系保留审计。四个根的空签名由固定根身份确定重建，不再持久化第二份空记录。

## 4. 概念关系 9—12

### 4.1 实例支持概念关系 9

源为本类允许的实例对象，目标为当前概念；顺序号固定为 `1`。关系允许多对多，但同一完整源、目标不得重复。实例支持失效时保留关系历史；支持关系不能提升概念生命周期，也不能由活动快照替代。

### 4.2 概念上下位关系 10

源为直接上位概念，目标为直接下位概念；顺序号固定为 `1`。只允许同类别连接，允许一个下位概念有多个直接上位，整体必须为无环 DAG。每个非根当前概念必须且只能归入一个类别，并可到达该类别唯一根；跨根、成环、自环和无法到根必须在第一笔写入前拒绝。

关系 10 只保存直接边，不把传递闭包复制为权威关系。抽象树、祖先集、后代集和根可达摘要均为可重建投影。

### 4.3 概念定义材料关系 11

源为概念，目标为结构化定义或句柄型约束目标。顺序号必须为概念签名中已登记的非零约束角色；同一概念同一角色至多一个有效目标，同一目标可以在不同角色合法复用。类型化签名记录不得复制目标或关系句柄。

### 4.4 概念生命周期关系 12

源为概念，目标为三个固定生命周期状态之一：活跃、冷却、退役。目标状态身份必须是当前状态域允许的固定抽象状态；每个当前概念恰有一条有效当前关系。顺序号从 `1` 连续递增并只表达概念生命周期版本。

合法迁移为：

```text
活跃 -> 冷却
冷却 -> 退役
冷却 -> 活跃
退役 -> 活跃
```

迁移形成新的关系 12，失效旧当前关系并保留审计，不得重挂共享关系、改写旧目标或只更新登记数组。统计、用途观察和召回结果只能形成迁移建议；概念领域入口必须重读节点、签名、关系、生命周期和当前引用后裁决。

### 4.5 写入与解释权

概念图领域服务拥有概念角色、签名、关系 9—12 和生命周期的业务准入与组合编排权；所属对象节点和本域类型化记录仍由存在、动态、二次特征和因果各自领域服务及专用数据操作写入。概念图服务不得绕过对象领域写仓库，也不得用影子登记恢复缺失权威结构。

## 5. 活动投影、登记和统计

以下材料全部属于 4060 的非权威缓存或可重建投影：

| 材料 | 唯一性质 |
| --- | --- |
| 四根登记 | 固定根身份的定位投影 |
| 生命周期状态登记 | 三个固定状态身份的定位投影 |
| 专用关系登记 | 关系 9—11 的反向索引或调试投影 |
| 生命周期关系登记 | 关系 12 的反向索引或调试投影 |
| 活动快照和活动版本 | 节点、签名记录、关系和生命周期的派生活动投影及其代次 |
| 上一候选图、抽象树、主动召回组、待命名组 | 调用期候选或读取投影 |
| 生命周期复核候选、删除候选、直接边重算和根可达摘要 | 调用期值式材料 |
| 概念用途观察和统计缓存 | 无权威事件账时只从后续合法事件重新积累的非权威统计 |

这些材料可以清空、缺失和确定重建。清空后，概念身份、签名、上下位、生命周期、退役和删除裁决必须保持等价。投影与权威结构矛盾时必须丢弃或隔离投影并追根因，不得反向修改权威事实。活动版本只表达投影代次，不得成为概念身份、生命周期、签名版本或删除成立的权威版本。

## 6. 退役边界

退役不等于删除：

```text
退役
-> 节点、签名、上下位关系、对象本域记录和历史继续存在
-> 退出主动召回和普通活动投影
-> 仍可经正式生命周期迁移重新激活
```

退役前必须确认目标不是四根、当前生命周期允许迁移、签名和本域对象结构完整、当前引用符合退役规则。退役只改变关系 12 当前版本，不删除节点、签名、上下位或实例支持历史。

## 7. 删除准入和权威事务

删除表示概念不可逆退出当前结构。旧稳定身份、签名、关系和本域记录保留审计；以后形成等价概念必须取得新的稳定主键和节点身份，不得复用旧键或复活已删除节点。

### 7.1 写前拒绝

任一条件成立时必须在第一笔写入前拒绝：

```text
目标无效、跨域、节点类型或命名域错误，或不是当前概念
目标是四类根之一
生命周期不是退役
仍有当前实例支持关系 9
仍有概念结构之外的当前有效引用
节点、签名、本域记录、关系 9—12 或索引版本漂移
删除后剩余概念不能保持同类、无环和唯一类别根可达
无法形成全部替代直接上下位关系
写入权、事务域或任一对象所有者授权不成立
```

活动快照缺失、登记数组清空或统计缓存为空本身不得阻止删除，也不得改变删除裁决。

### 7.2 替代直接边

删除中间概念前，必须在移除目标后的权威关系 10 图上纯值重算替代直接边：对每个目标直接上位和直接下位组合，如果剩余图中不存在绕开目标的其它可达路径，则准备新的上位到下位关系 10；已有等义直接边则幂等复用。替代边必须与目标相关旧边的失效在同一权威事务发布，不得只写入活动快照或抽象树。

### 7.3 事务顺序

权威事务顺序固定为：

```text
取得唯一权威写入权
-> 重读节点、签名、本域记录、关系 9—12、全部外部关系和索引
-> 纯值重算删除后完整直接边、同类无环和根可达
-> 准备失效目标全部当前关系
-> 准备复用或创建新增直接关系 10
-> 准备签名及本域记录的删除历史
-> 准备索引移除和节点删除审计
-> 全部强类型参与者读回、确认
-> 无失败最后发布
-> 权威结构完整后验读回
```

节点必须最后退出当前普通读取。关系、签名和本域记录历史必须保留；稳定主键、节点编号和关系编号不得复用。确认前失败按 4040 精确逆序撤销；无法证明前态时隔离事务域。发布后权威读回异常属于内部逻辑错误。

## 8. 权威回执与投影后验报告

权威退役 / 删除回执只证明节点、签名、本域记录、关系、索引和历史事务结果；不得把活动快照、登记、召回组或统计缓存清理作为权威完成前提。

权威发布成功后再执行：

```text
失效活动图、登记投影、召回组和用途缓存
-> 从权威结构确定重建
-> 形成独立投影后验报告
```

投影更新失败属于发布后内部逻辑错误，停止投影依赖路径并追根因；不得回滚、否认或重复已经完成的权威删除。投影后验报告不得成为新的机器事实或任务完成依据。

## 9. 结果分类

| 分类 | 情况 |
| --- | --- |
| 入口拒绝 | 无效身份、错误类型、根概念、非退役、仍有支持 / 外部引用、命名域或权限不符 |
| 逻辑内结果 | 候选预算不足；取得写入权前竞争繁忙；权威版本变化导致候选过期；完整同义删除的幂等读回 |
| 内部逻辑错误 | 权威结构宣称完整但签名、关系或生命周期缺项；多生命周期、跨根、成环；替代直接边无法形成；确认 / 撤销 / 发布异常；发布后权威读回或投影确定重建不一致 |

内部逻辑错误不得降级为“活动快照不存在”“缓存未命中”或普通删除拒绝。

## 10. 持久化与恢复

长期材料保存概念所属对象节点、稳定主键、概念签名类型化记录、关系 9—12 当前及历史、本域记录历史、记录版本和删除审计。根登记、生命周期登记、活动快照、候选图、抽象树、召回组和统计缓存不作为恢复权威。

恢复隔离候选必须验证：

```text
四根固定身份和三阶段固定状态身份
每个概念恰一条当前生命周期关系
同类无环、唯一类别根可达
签名记录与关系 11 角色逐项一致
已删除概念无当前关系、当前索引或当前签名记录且历史完整
未知签名格式、未知命名域、悬空关系和缺失本域记录全部拒绝
```

通过后从权威候选确定重建全部登记、活动图、抽象树和召回投影；相同输入必须得到等价结果。用途统计没有正式事件账时恢复为空。全部权威恢复和所需投影预构造通过后，才可随完整运行期上下文一次发布；旧快照不能无歧义提取稳定主键、签名和关系角色时必须明确拒绝，不得发布半结构或回退全新。

## 11. 验证合同

验证至少覆盖：

1. 四根固定身份、永久活跃和根删除拒绝。
2. 关系 9 多对多、关系 10 同类多父 DAG / 唯一根可达、关系 11 签名角色、关系 12 当前唯一和连续历史。
3. 概念签名记录不保存节点 / 关系句柄、活动版本、统计或通用槽。
4. 清空全部登记、活动快照和统计缓存后，权威读取、生命周期和删除裁决等价。
5. 删除中间概念时替代直接关系 10 与旧边失效同事务发布。
6. 权威回执和投影后验报告分型；投影失败不回滚已发布权威删除。
7. 已删除身份不可复用，历史节点、签名、关系和本域记录仍可审计。
8. 快照恢复只消费权威结构，重建投影且不恢复无事件账统计历史。

验证必须通过正式节点、关系、领域记录和历史读回，不得用活动版本、登记数量、缓存命中、日志或显示树替代。
