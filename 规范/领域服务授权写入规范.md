# 领域服务授权写入规范

编号：017
日期：2026-07-05
状态：待确认规范 / 核心内核 P1

## 1. 定位

本规范定义哪些写入必须通过领域服务，避免外部模块直接写节点仓库、主信息仓库、关系仓库或索引仓库形成业务事实。

## 2. 授权原则

```text
仓库负责最小结构。
领域服务负责业务准入。
外部调用方只请求领域服务。
显示层只读。
日志层只读。
持久化层只保存快照、事件和恢复材料。
```

## 3. 第一版领域服务写入权

| 服务 | 可写范围 | 禁止 |
| --- | --- | --- |
| 语素服务 | 语素入口、语素对应信息、概念追溯 | 裁决世界事实、需求满足、方法可执行 |
| 特征值服务 | 特征系统内部值承载 | 作为需求、任务、方法的通用值容器，或被高级服务直接调用 |
| 特征服务 | 特征类型、特征关系和特征状态材料封口 | 写需求结算或任务状态 |
| 存在服务 | 实际存在和虚拟存在 | 写场景内状态、动态或因果结论 |
| 场景服务 | 场景和场景内临时实例状态 / 实例动态收纳 | 把临时实例状态 / 动态提升为长期事实 |
| 状态服务 | 实例状态、抽象状态和提升边界 | 无主体状态、无场景实例状态、直接长期化实例状态 |
| 动态服务 | 实例动态、抽象动态和提升边界 | 无主体动态、无场景实例动态、伪造方法动作来源 |
| 二次特征服务 | 二次特征入口和复合判断前置特征 | 写特征值或需求结算 |
| 因果服务 | 因果引用、证据和抽象候选 | 代替动态服务创建无主体动态 |
| 世界服务 | 基础信息聚合编排 | 绕过基础信息原子服务直接写基础结构 |
| 需求服务 | 需求节点和需求结算入口 | 直接执行方法，绕过特征 / 状态服务写基础结构，或直接调用特征值服务 |
| 任务服务 | 任务节点和任务承接关系 | 裸写方法长期能力，绕过存在 / 场景 / 特征 / 状态 / 动态服务写基础结构 |
| 方法服务 | 方法节点和执行入口 | 把线程当动作来源，绕过特征 / 动态 / 因果服务写结果，或直接调用特征值服务 |

## 4. 越权写入方判断

越权写入方至少包括：

```text
显示层写业务事实
日志层写业务事实
外部输入直接写世界事实
持久化层绕过领域服务恢复为当前事实
任务服务直接改方法长期能力
方法服务直接写任务专有回执
语素服务直接裁决候选为事实
世界服务绕过原子服务直接写基础信息结构
特征值服务被当作通用值容器
高级信息服务直接访问特征值服务
高级信息服务绕过基础信息服务创建存在、场景、特征、状态、动态或因果
临时实例状态 / 动态被直接作为长期语素入口对应信息
```

## 5. 验收方式

```text
外部模块不能直接写仓库形成业务事实。
领域服务入口会检查节点类型、生命周期和必要前置关系。
越权路径被入口拒绝或不存在公开 API。
```
