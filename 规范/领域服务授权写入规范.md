# 领域服务授权写入规范

编号：017
日期：2026-07-05
状态：待确认规范 / 核心内核 P1

## 1. 定位

本规范定义哪些写入必须通过领域服务，避免外部模块直接写节点仓库、主信息仓库、关系仓库或索引仓库形成业务事实。

## 2. 授权原则

```text
仓库负责最小结构。
领域服务负责业务准入。
外部调用方只请求领域服务。
显示层只读。
日志层只读。
持久化层只保存快照、事件和恢复材料。
```

## 3. 第一版领域服务写入权

| 服务 | 可写范围 | 禁止 |
| --- | --- | --- |
| 语素服务 | 语素入口、语素对应信息、概念追溯 | 裁决世界事实、需求满足、方法可执行 |
| 世界服务 | 基础信息、存在、场景 | 写任务状态、方法结果、因果结论 |
| 特征服务 | 特征类型和特征关系 | 写需求结算 |
| 特征值服务 | 特征值节点和值容器 | 裁决业务状态 |
| 需求服务 | 需求节点和需求结算入口 | 直接执行方法 |
| 任务服务 | 任务节点和任务承接关系 | 裸写方法长期能力 |
| 方法服务 | 方法节点和执行入口 | 把线程当动作来源 |
| 因果服务 | 动态和因果引用 | 伪造方法动作来源 |

## 4. 越权写入方判断

越权写入方至少包括：

```text
显示层写业务事实
日志层写业务事实
外部输入直接写世界事实
持久化层绕过领域服务恢复为当前事实
任务服务直接改方法长期能力
方法服务直接写任务专有回执
语素服务直接裁决候选为事实
```

## 5. 验收方式

```text
外部模块不能直接写仓库形成业务事实。
领域服务入口会检查节点类型、生命周期和必要前置关系。
越权路径被入口拒绝或不存在公开 API。
```
