# 领域服务授权写入规范

编号：017
日期：2026-07-05
状态：已确认规范 / 核心内核 P1 / 第一轮领域服务授权写入边界

## 1. 定位

本规范定义哪些写入必须通过领域服务，避免外部模块直接写节点仓库、主信息仓库、关系仓库或索引仓库形成业务事实。

本规范不取消仓库正式入口。仓库正式入口仍是核心内核的一部分，但只能负责最小结构写入和读取，不解释业务语义。业务事实必须由领域服务完成准入、编排、写入和读取封口。

## 2. 授权原则

```text
仓库负责最小结构。
领域服务负责业务准入。
外部调用方只请求领域服务。
显示层只读。
日志层只读。
持久化层只保存快照、事件和恢复材料。
```

任何业务写入口都必须先在函数入口排除：

```text
无效句柄
已删除句柄
错误节点类型
错误关系类型
越权写入方
缺少前置节点
缺少前置关系
不满足生命周期的对象
缺少发生时间戳
未确认多目标语义
```

拒绝路径不得写业务事实，不得依赖日志、弹窗、控制台输出或事后扫描器修正结构。

## 3. 仓库入口和领域服务入口边界

仓库正式入口只负责最小结构：

```text
创建 / 读取 / 删除节点
创建 / 读取 / 删除主信息
创建 / 读取 / 删除关系
绑定 / 查询 / 删除索引
普通父子关系重挂
基础句柄有效性和版本检查
```

仓库入口不得解释业务语义：

```text
不判断需求是否满足。
不判断任务是否完成。
不判断方法是否可执行。
不判断世界事实是否成立。
不判断语素候选是否为事实。
不判断动作入口是否具备真实可调用能力。
不判断需求结算记录是否权威完成。
```

领域服务负责业务写入编排和准入：

```text
检查节点类型。
检查生命周期。
检查来源关系。
检查领域规则。
检查跨服务依赖是否在白名单内。
调用仓库正式入口写入最小结构。
调用其它被授权领域服务写入对应结构。
提供专用读取入口，避免外部裸读关系误判事实。
```

外部调用方可以请求领域服务创建或读取业务结构，但不得绕过领域服务直接调用仓库入口形成业务事实。

## 4. 跨服务依赖白名单

第一轮高级服务只允许直接依赖以下基础信息服务：

```text
存在服务
场景服务
特征服务
二次特征服务
状态服务
动态服务
因果服务
```

特征值服务只服务于特征系统，只有特征服务可直接访问。需求、任务、方法、动作入口、需求结算、统计缓存等高级或后续服务需要特征状态材料时，必须通过特征服务获得。

| 调用方 | 允许直接依赖 | 禁止直接依赖 |
| --- | --- | --- |
| 需求服务 | 存在服务、场景服务、特征服务、状态服务、动态服务、因果服务、任务服务专用只读入口 | 特征值服务、仓库裸写、方法执行入口、缓存统计裁决 |
| 任务服务 | 存在服务、场景服务、特征服务、状态服务、动态服务、因果服务、方法服务专用只读入口 | 特征值服务、裸写方法长期能力、绕过需求服务改需求事实 |
| 方法服务 | 存在服务、场景服务、特征服务、状态服务、动态服务、因果服务、任务服务专用只读入口 | 特征值服务、任务专有回执裸写、绕过任务方法关系执行 |
| 世界服务 | 基础信息原子服务 | 绕过原子服务直接写基础信息结构 |
| 概念图服务 | 节点仓库只读、关系仓库正式创建 / 值式失效 / 审计 / 值式重挂 / 节点相关有效读取入口、存在 / 动态 / 二次特征 / 因果 / 特征 / 状态服务的专用读取或创建入口、语素服务名称反查入口、统计服务概念用途观察入口 | 特征值服务、需求 / 任务 / 方法服务、日志 / 显示 / SQL 裁决、绕过原子服务创建四类节点或生命周期状态 |
| 语素服务 | 只读目标节点、关系 / 索引授权入口、信息节点类型复核入口 | 创建基础信息、创建高级信息、创建临时实例状态 / 动态、裁决候选为事实 |
| 特征服务 | 特征值服务、存在服务、状态服务、关系 / 索引授权入口 | 写需求结算、任务状态、方法成功事实 |
| 显示 / 日志 / 控制台 | 只读结构或人读材料 | 任何业务事实写入 |
| 持久化恢复 | 快照、事件、恢复材料读取和重放所需的正式恢复入口 | 绕过领域服务裁决当前事实 |

后续新增服务必须先登记白名单，不得默认继承高级服务或基础信息服务的写入权。

## 6. 越权写入方判断

越权写入判断至少来自：

```text
调用方服务类型。
被写结构类型。
被写节点类型。
被写关系类型。
被写主信息槽或索引命名空间。
写入是否经过授权领域服务。
写入是否有必要前置关系。
写入是否在该服务白名单内。
写入是否绕过原子服务或专用读取入口。
```

越权写入方至少包括：

```text
显示层写业务事实
日志层写业务事实
控制台入口写业务事实
外部输入直接写世界事实
持久化层绕过领域服务恢复为当前事实
任务服务直接改方法长期能力
方法服务直接写任务专有回执
方法服务缺任务方法关系仍修改特征状态材料
需求服务直接调用特征值服务
任务服务直接调用特征值服务
方法服务直接调用特征值服务
语素服务直接裁决候选为事实
世界服务绕过原子服务直接写基础信息结构
特征值服务被当作通用值容器
高级信息服务绕过基础信息服务创建存在、场景、特征、状态、动态或因果
临时实例状态 / 动态被直接作为长期语素入口对应信息
非权威缓存被当作业务事实来源
```

第一轮不要求实现完整运行时权限系统，但必须通过以下方式形成可检查门禁：

```text
编译期依赖限制。
服务构造参数限制。
服务入口前置拒绝。
专用读取入口。
验收用例检查越权路径不存在或被拒绝。
```

## 7. 新增服务注册规则

新增领域服务、后续服务或迁移旧能力入口前，必须登记：

```text
服务名称
主要用途
可写结构
可读结构
允许直接依赖
禁止直接依赖
入口拒绝条件
验收方式
禁止宣称
```

不得为了显示、日志、控制台、SQL 投影、临时兼容字段或旧项目函数搬迁，新增绕过授权门禁的服务。

动作入口、需求结算、非权威缓存与统计等已确认专项若独立成服务，必须回填本规范的白名单和写入矩阵；在回填前只能按其已确认规范和详细设计中的服务边界执行。

## 8. 验收矩阵

| 编号 | 验收项 | 通过标准 |
| --- | --- | --- |
| AW-01 | 需求服务不得直接访问特征值服务 | 需求服务只能通过特征服务取得特征状态材料；直接包含、接收或调用特征值服务视为失败 |
| AW-02 | 任务服务不得直接写方法长期能力 | 任务服务只能写任务选择方法关系或调用方法服务只读入口；不能创建方法首、条件、结果或动作入口长期能力 |
| AW-03 | 方法服务缺任务方法关系不得改特征状态材料 | 无效任务、无效方法或缺少任务到方法关系时，特征状态材料不变化 |
| AW-04 | 语素服务不得裁决候选为事实 | 语素服务只能写入口对应信息、概念追溯或索引关系；不能把候选静默提升为世界事实、需求满足或方法可执行 |
| AW-05 | 世界服务不得绕过原子服务写基础结构 | 世界服务创建或聚合基础信息时必须调用对应原子服务；不得直接写存在、场景、特征、状态、动态、二次特征或因果结构 |
| AW-06 | 显示 / 日志 / 控制台不得写业务事实 | 这些入口只能读取结构或写人读材料；不能新增节点、主信息、关系、索引、状态、动态、任务、方法、需求结算事实 |
| AW-07 | 持久化恢复不得裁决当前事实 | 持久化层只能恢复快照、事件和审计材料；不能绕过领域服务把恢复材料直接裁决为当前需求、任务、方法或世界事实 |
| AW-08 | 拒绝后结构不变化 | 越权路径被拒绝后，节点、主信息、关系、索引、特征状态材料、状态、动态、因果、需求、任务和方法结构不发生业务变化 |
| AW-09 | 缓存不得参与业务裁决 | 非权威缓存缺失、清空或重建不得改变需求满足、任务完成、方法成功或结构事实 |
| AW-10 | 新增服务必须登记白名单 | 新增服务或旧能力迁移入口没有写入矩阵、依赖白名单和验收方式时，不得进入代码实施 |

验收可以按具体切片裁剪涉及仓库，但不得只检查返回值；必须检查拒绝前后相关结构没有业务变化。

## 9. 完成声明边界

本规范确认后只能声明：

```text
领域服务授权写入边界已确认。
仓库入口和领域服务入口边界已确认。
跨服务依赖白名单已确认。
服务写入矩阵和越权验收矩阵已确认。
```

不得声明：

```text
所有代码已经实现运行时权限系统。
所有领域服务越权测试已经完成。
任务系统 / 方法系统 / 动作入口 / 需求结算 / 缓存统计已经完整完成。
旧项目业务写入已经全部迁移。
显示层、日志层、持久化层已经接入完整只读治理。
```
