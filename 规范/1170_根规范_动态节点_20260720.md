# 1170 根规范 动态节点

编号：1170
日期：20260720
状态：正式根规范

## 1. 根定位

动态节点承载状态迁移和动作致变事实，是抽象动态、状态迁移账和动作动态账的统一事实对象。“变化”是动态抽象的最高兜底根名，“抽象动态”是匹配、筹办和规范表达使用的有效抽象单元；不得另设并列“动态模板根”。

## 2. 本项目承载

```text
动态节点及稳定句柄
+ 主信息中的抽象材料或发生时间
+ 场景和主体关系
+ 被改变目标关系
+ 前状态和后状态关系
+ 可选来源动作关系
+ 来源低层动态、层级、聚合方式和路径签名
+ 场景事实目录关系与定位索引
```

来源动作必须指向合法方法动作入口或领域服务动作事实。线程、调度器、外部输入、日志和界面不得成为来源动作。

## 3. 事实与边界

1. 动态必须保存前状态、后状态、主体、被改变目标、发生时间和来源；动作致变动态还必须保存合法来源动作。
2. 外部变化只能记录为观察动态或外部事件动态，不得伪装成自我方法动作。
3. 原子动态是零散事实切片，通过场景动态目录归属，不得挂入场景、存在、特征或状态普通子结构形成动态历史树。
4. 简单动态可以聚合成复杂动态，但聚合只能形成动态域内部的来源证据链，不改变原子动态的场景归属。
5. 基础事实池或索引登记不构成业务世界树层级或动态聚合层级。

## 4. 写入、读取与验证

动态领域服务负责抽象动态和完整状态变化动态的业务准入；必须先通过状态领域服务读取完整前后状态，并验证主体、场景、时间顺序、被改变目标和来源动作角色。数据操作层负责原子发布动态节点、主信息、关系和索引。

有效实例动态必须回读相同主体和场景下的前后状态、非零发生时间、被改变目标，以及与来源动作是否存在相一致的关系证据。内部结构缺口属于内部逻辑错误，不得包装成普通动态缺失。

## 5. 生命周期与当前性

动态事实创建后不可通过覆写改变历史。聚合版本、来源事实版本或动作身份变化时必须形成新版本或新聚合结果；旧动态仍可按稳定身份回查。动态索引裁剪只改变目录可见性，不删除动态事实。
