# 节点类型与关系类型枚举规范

编号：015
日期：2026-07-05
状态：待确认规范 / 核心内核 P1

## 1. 定位

本规范定义第一版允许的节点类型和关系类型边界。枚举只表达结构分类，不表达业务完成、状态结论或显示文本。

## 2. 节点类型

第一版允许：

```text
未分类
语素
基础信息
存在
场景
特征
特征值
需求
任务
方法
动态
因果引用
```

禁止：

```text
高级信息
名称字段节点
显示标题节点
SQL 投影节点
控制面板节点
临时日志节点
```

`高级信息` 不是节点类型，只是讨论中对需求、任务、方法等既有高级业务节点角色的简称。

## 3. 关系类型

第一版允许：

```text
普通父子
归属
引用
索引
模板
运行期临时
因果来源
语素对应信息
语素概念追溯
```

新增关系类型前必须说明：

```text
用途
写入方
读取方
生命周期
是否允许多目标
是否允许重复
是否可由已有关系表达
验证方式
```

## 4. 禁止行为

```text
不得用普通父子关系表达所有关系。
不得为显示方便新增节点类型或关系类型。
不得把需求满足、任务完成、方法成功做成节点类型。
不得把高级信息做成独立枚举。
```
