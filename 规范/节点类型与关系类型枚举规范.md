# 节点类型与关系类型枚举规范

编号：015
日期：2026-07-06
状态：已确认规范 / 核心内核 P1 / 第一轮节点类型与关系类型枚举边界

## 1. 定位

本规范定义第一版允许的节点类型和关系类型边界。枚举只表达结构分类，不表达业务完成、状态结论或显示文本。

枚举值一经进入代码、快照、事件日志、索引、持久化材料或跨版本恢复材料，不得重排、复用或改名复义。新增节点类型或关系类型只能追加新值，不得插入到既有枚举中间。删除旧类型只能废弃不用，不得把旧数值改作新语义。

## 2. 节点类型

第一版允许以下节点类型，当前代码事实见 `海中鱼巣/核心/句柄.h`。

### 2.1 节点类型数值锁定表

| 数值 | 节点类型 | 第一轮用途 | 备注 |
| --- | --- | --- | --- |
| 0 | 未分类 | 无明确类型或非法入口拒绝边界 | 不得作为业务节点正常落地目标 |
| 1 | 语素 | 语素入口节点 | 由语素服务写入 |
| 2 | 基础信息 | 基础信息通用节点 | 只作为尚未拆入具体基础信息类型时的通用承载 |
| 3 | 存在 | 存在身份 | 由存在服务或世界聚合服务经原子服务写入 |
| 4 | 场景 | 场景身份 | 由场景服务或世界聚合服务经原子服务写入 |
| 5 | 特征 | 特征类型或特征身份 | 由特征服务写入 |
| 6 | 特征值 | 特征系统内部值承载节点 | 只服务特征系统，不默认作为语素入口对应信息 |
| 7 | 需求 | 需求节点 | 高级业务节点，不是 `高级信息` 节点类型 |
| 8 | 任务 | 任务节点 | 高级业务节点，不代表任务完整承接壳已成立 |
| 9 | 方法 | 方法节点、方法角色节点或动作入口角色节点 | 高级业务节点，不代表方法可执行 |
| 10 | 动态 | 动态节点 | 需由动态服务区分实例动态和抽象动态 |
| 11 | 因果引用 | 轻量因果引用节点 | 不代表稳定因果结论 |
| 12 | 状态 | 状态节点 | 需由状态服务区分实例状态和抽象状态 |
| 13 | 二次特征 | 二次特征节点 | 由二次特征服务写入 |

### 2.2 基础信息类长期节点

`基础信息` 在语义上是基础信息类信息节点集合，不只等于 `基础信息` 单一节点类型。

第一轮基础信息类长期节点包括：

```text
基础信息通用节点
存在
场景
特征
抽象状态
抽象动态
二次特征
因果引用
```

其中 `状态` 和 `动态` 是底层节点枚举名。是否为抽象状态 / 抽象动态，不由枚举值单独裁决，必须由状态服务、动态服务、运行期临时关系和对应领域入口共同判定。

场景内实例状态 / 实例动态是临时零散节点，必须有存在主体、场景归属和发生时间戳，不得作为长期语素入口对应信息节点。抽象状态 / 抽象动态才可作为长期基础信息类节点绑定语素入口或跨场景复用。

`特征值` 是值承载节点，第一轮不得默认作为语素入口对应信息节点。若后续需要让语素直接指向某个值，必须先确认特征值系统规范和入口边界。

### 2.3 禁止节点类型

禁止新增或使用以下节点类型：

```text
高级信息
名称字段节点
显示标题节点
SQL 投影节点
控制面板节点
临时日志节点
任务完成节点
需求满足节点
方法成功节点
```

`高级信息` 不是节点类型，只是讨论中对需求、任务、方法等既有高级业务节点角色的简称。

### 2.4 枚举不裁决业务事实

以下判断不得只由节点类型得出：

```text
节点类型::任务 不代表任务已创建完整承接壳。
节点类型::方法 不代表方法可执行。
节点类型::需求 不代表需求已满足。
节点类型::状态 不代表状态已被确认长期有效。
节点类型::动态 不代表动态已进入长期抽象层。
节点类型::因果引用 不代表因果结论稳定可靠。
```

业务事实必须由领域服务、关系、状态、动态、因果引用、结算记录或对应结构共同裁决。

## 3. 关系类型

第一版允许以下关系类型，当前代码事实见 `海中鱼巣/核心/句柄.h`。

### 3.1 关系类型数值锁定表

| 数值 | 关系类型 | 第一轮用途 | 多目标 / 唯一性口径 |
| --- | --- | --- | --- |
| 0 | 普通父子 | 表达树形或层级父子 | 同一子节点同时只能有一个有效父节点 |
| 1 | 归属 | 表达对象归属于场景、存在、需求或容器 | 是否多目标由对应领域规范确认 |
| 2 | 引用 | 表达非拥有引用 | 默认由领域规范确认是否多目标；未确认前不得模拟列表字段 |
| 3 | 索引 | 表达索引项和目标之间的结构连接 | 第一轮优先由索引仓库管理，不裁决业务事实 |
| 4 | 模板 | 表达方法、任务、特征、需求目标等模板关系 | 多目标和唯一性由对应服务专用读取入口裁决 |
| 5 | 运行期临时 | 表达实例状态 / 实例动态等临时归属 | 不进入长期事实裁决，不作为长期语素绑定依据 |
| 6 | 因果来源 | 表达动态、前后值、目标、证据等来源材料 | 因果服务或动态相关服务写入和读取 |
| 7 | 语素对应信息 | 语素入口绑定对应信息节点 | 已确认同一语素入口可多目标 |
| 8 | 语素概念追溯 | 语素入口追溯概念节点 | 已确认可多级、多路目标 |
| 9 | 实例支持概念 | 实例节点支持一个或多个同类概念 | 同一实例可支持多概念，同一概念可被多实例支持；完全重复关系幂等 |
| 10 | 概念上下位 | 同类概念间的直接上位到下位边 | 允许下位多父；禁止跨类、自环和已知成环 |
| 11 | 概念定义材料 | 概念节点引用结构化定义材料 | 允许多目标；目标必须进入定义材料白名单 |

### 3.2 当前关系类型边界

```text
普通父子：只表达普通层级和通用移动重挂，不表达所有业务关系。
归属：用于拥有、承接或容器关系时，必须由领域规范说明源 / 目标类型和唯一性。
引用：用于非拥有引用时，外部调用方不得直接把裸引用裁决为业务事实。
索引：索引只服务定位、候选或加速；索引缺失不得改变权威结构事实。
模板：用于方法、任务、特征、需求目标等模板关系；读取必须由领域服务复核目标类型和唯一性。
运行期临时：用于实例状态 / 实例动态等临时归属；不得被长期语素入口当作抽象信息。
因果来源：用于动态证据、前值、后值、被改变目标和结算证据；轻量因果引用本身不写来源动态关系。
语素对应信息：已确认多目标，读取必须返回集合。
语素概念追溯：已确认多级、多路目标，读取必须返回集合或追溯图，不得静默选第一个。
实例支持概念：只由概念图服务写入；实例和概念必须类别兼容，概念根不得作为实例端。
概念上下位：只由概念图服务写入；源为直接上位、目标为直接下位，允许多父但必须保持同类无环。
概念定义材料：只由概念图服务写入；不得连接名称文本、日志、控制面板材料、SQL 行或线程消息。
```

关系仓库详细设计必须与本规范同步。若 `规范/详细设计/关系仓库详细设计.md` 对某关系类型的唯一性、多目标或读取规则有更具体约束，代码实施必须同时满足本规范和关系仓库详细设计；二者冲突时不得编码，必须先修正规范。

## 4. 新增类型准入流程

### 4.1 新增节点类型前必须说明

```text
用途
写入方
读取方
生命周期
是否可由已有节点类型 + 关系 / 状态 / 特征表达
是否允许作为语素入口对应信息节点
是否属于基础信息类、高级业务节点角色、运行期实例类或证据类
入口拒绝规则
验证方式
禁止宣称
```

不得为显示、日志、控制面板、SQL 投影、临时兼容字段或自然语言标题新增节点类型。

### 4.2 新增关系类型前必须说明

```text
用途
写入方
读取方
生命周期
是否允许多目标
是否允许重复
源节点类型和目标节点类型
是否可由已有关系表达
验证方式
禁止宣称
```

新增关系类型不得绕过关系仓库详细设计、领域服务授权写入规范和入口前置拒绝规则。

## 5. 禁止行为

```text
不得用普通父子关系表达所有关系。
不得为显示方便新增节点类型或关系类型。
不得把需求满足、任务完成、方法成功做成节点类型。
不得把高级信息做成独立枚举。
不得重排、复用或改名复义已经进入代码和持久化材料的枚举数值。
不得用枚举类型替代领域服务的业务裁决。
不得用运行期临时关系把实例状态 / 实例动态提升为长期基础信息。
```

## 6. 验收与完成声明

本规范确认只能证明：

```text
第一轮节点类型和关系类型边界已确认。
当前代码枚举数值已有文档锁定。
新增节点类型和关系类型已有准入门禁。
```

不能证明：

```text
代码已修改。
枚举持久化兼容已验证。
完整关系系统完成。
任务系统、方法系统、动作入口或需求结算系统完成。
旧项目能力已迁移。
```
