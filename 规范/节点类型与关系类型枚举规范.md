# 节点类型与关系类型枚举规范

编号：015
日期：2026-07-05
状态：待确认规范 / 核心内核 P1

## 1. 定位

本规范定义第一版允许的节点类型和关系类型边界。枚举只表达结构分类，不表达业务完成、状态结论或显示文本。

## 2. 节点类型

第一版允许：

```text
未分类
语素
基础信息
存在
场景
特征
特征值
需求
任务
方法
状态
动态
二次特征
因果引用
```

`基础信息` 在语义上是基础信息类信息节点集合，不只等于 `基础信息` 单一节点类型。当前基础信息类节点包括：

```text
基础信息
存在
场景
特征
状态
动态
二次特征
因果引用
```

`特征值` 是值承载节点，第一轮不得默认作为语素入口对应信息节点；若后续需要让语素直接指向某个值，必须先确认特征值系统规范和入口边界。

`状态` 和 `动态` 是当前节点枚举名称，后续必须由状态服务和动态服务区分实例层与抽象层。场景内临时实例状态 / 实例动态不得直接作为长期语素入口对应信息节点；抽象状态 / 抽象动态才可作为长期基础信息类节点。

禁止：

```text
高级信息
名称字段节点
显示标题节点
SQL 投影节点
控制面板节点
临时日志节点
```

`高级信息` 不是节点类型，只是讨论中对需求、任务、方法等既有高级业务节点角色的简称。

## 3. 关系类型

第一版允许：

```text
普通父子
归属
引用
索引
模板
运行期临时
因果来源
语素对应信息
语素概念追溯
```

新增关系类型前必须说明：

```text
用途
写入方
读取方
生命周期
是否允许多目标
是否允许重复
是否可由已有关系表达
验证方式
```

## 4. 禁止行为

```text
不得用普通父子关系表达所有关系。
不得为显示方便新增节点类型或关系类型。
不得把需求满足、任务完成、方法成功做成节点类型。
不得把高级信息做成独立枚举。
```
