# D455 相机外设材料规范

编号：D455-MATERIAL-01

日期：2026-07-13

状态：正式规范 v0.1 / JY-321 / 代码未实现

## 1. 定位

本规范定义 Intel RealSense D455 在 `海中鱼巣` 中第一轮进入工程的边界：

```text
D455 设备采集
-> SDK 无关的强类型帧材料
-> 有界运行期材料缓存
-> 只含材料引用的外部材料消息
-> 模拟自检与显式真实样本验收
```

D455 是外部材料来源，不是需求来源、动作来源或世界事实写入方。相机帧、设备状态、缓存、消息、日志和自检输出均不得直接裁决：

```text
存在是否成立
场景是否改变
特征或状态是否成立
需求是否满足
任务是否完成
方法是否成功
动作动态或因果引用是否形成
```

本规范只恢复 D455 相机材料入口专项，不恢复体素、视觉融合或外设本能动作迁移。

## 2. 当前事实与恢复依据

### 2.1 用户触发

`计划/计划索引.md` 规定 D455 / 体素 / 外设接入只有在用户重新明确提出后才能恢复。用户本轮已明确提出规划 D455 相机功能，满足建立独立 D455 材料入口专项的触发条件。

### 2.2 当前代码事实

```text
海中鱼巣/线程/外设采样材料线程.ixx
```

当前只承载模拟或空外部材料，文件头和自检均明确不接 D455、真实外设、体素或外设动作。本轮原地保留，不把 SDK 调用或真实采样循环塞入该文件。

```text
海中鱼巣/线程/运行消息协议.ixx
```

当前 `运行消息` 只有材料编号、版本、摘要、批次和时间戳等标量字段，适合传递“有一份材料可读取”的通知，不适合承载 RGB、Depth、红外、标定或 SDK 帧对象。本轮不扩大该协议。

当前工程尚未登记 RealSense 头文件、库和运行时 DLL。本机只确认已有 `D:\vcpkg` 的 `realsense2 2.56.3#1 x64-windows` 包，不能据此宣称项目已经接入 SDK 或相机已经可用。

### 2.3 旧能力证据

`实施记录/20260706_FSX_控制面板SQLD455体素外设排除项汇总记录.md` 中：

| 证据 | 本轮采用方式 | 本轮删除或后置方式 |
| --- | --- | --- |
| OF-202 D455 相机采集 | 保留“设备打开、采集帧、停止释放”的能力目标 | 不复制旧函数体，不迁移阶段耗时显示逻辑 |
| OF-203 双目相机适配 | 保留“SDK 输出先转换为材料”的边界 | 不迁移旧候选摘要、观察报告或旧接口层次 |
| OF-204 外设线程与队列 | 保留“有界材料、生命周期、停止收口”的约束 | 首轮不接生产线程，不复制旧外设线程和观察报告队列 |
| OF-205 体素与视觉融合 | 不采用 | 继续后置 |
| OF-206 外设本能动作 | 不采用 | 继续后置 |

本轮来源性质为“重新实现 / 新增约束”，只把旧函数当证据，不以旧函数作为迁移或代码单位。

## 3. 第一轮实施范围

第一轮只实现：

1. 枚举 D455，按可选序列号唯一选择设备；多设备且未指定序列号时拒绝猜测。
2. 以显式配置打开 RGB、Depth、左红外、右红外四路流。
3. 使用有限超时取得同一 RealSense frameset 中的四路帧。
4. 把帧字节、流格式、宽高、步长、帧号、设备时间戳、宿主接收时间戳、设备身份、深度比例、内参和外参复制为 SDK 无关值式材料。
5. 把完整帧材料放入专用有界缓存，只向现有运行消息协议输出材料编号、版本、批次、摘要和时间戳。
6. 提供无硬件模拟自检，以及只有显式参数才运行的真实 D455 样本验收。

第一轮明确排除：

```text
RGB / Depth 对齐
深度滤波、孔洞填补、点云、分割、跟踪、轮廓、双目粗距
IMU、自动曝光控制、发射器和激光功率控制
自动重连、热插拔和后台常驻采集
生产运行期组合器、现有外设采样线程改造和消息队列并发接线
存在 / 场景 / 特征 / 状态 / 动态 / 因果等世界结构写入
需求、任务、方法、外设本能动作、任务结果或需求结算
控制面板、SQL、ADO、WebView、录像、截图和分析包导出
体素、视觉融合和旧 D455 算法主链迁移
```

## 4. 帧材料合同

### 4.1 强类型材料

第一轮帧材料至少分为：

```text
D455设备身份材料
D455流配置
D455相机内参材料
D455相机外参材料
D455图像平面材料
D455同步帧材料
D455采集结果
```

公开业务类型、函数名、变量名和类内业务成员名使用中文；`RS2_*`、`rs2::`、像素协议名等外部固定标识可保留英文。

### 4.2 必需来源

一次可发布的 D455 帧材料必须同时包含：

```text
一帧 RGB
一帧 Depth
一帧左 IR
一帧右 IR
非空设备身份
非零材料编号、材料版本和批次号
每路明确的宽、高、步长、像素格式、字节数、帧号和设备时间戳
宿主接收时间戳
深度比例
与本批配置一致的内参和外参版本材料
```

任一路缺失、尺寸为零、步长不足、字节数与布局不一致、标定不完整或来源设备不一致时，不发布该帧材料。

四路帧号不强制相等，但必须来自同一 frameset，并分别保存真实帧号和时间戳。第一轮不得通过改写帧号或时间戳伪造同步。

### 4.3 SDK 隔离

`rs2::frame`、`rs2::frameset`、`rs2::pipeline`、SDK profile、SDK handle、`cv::Mat`、PCL 类型、裸指针和指向 SDK 内存的视图不得离开 D455 采集器模块。

采集器必须在下一次等待帧或释放 SDK 对象前完成字节复制。缓存和上行桥只读取 SDK 无关的不可变值式材料。

## 5. 有界缓存与消息合同

### 5.1 缓存

D455 完整帧不得写入 `运行消息`。专用缓存同时受以下上限约束：

```text
最大帧材料条数
最大总字节数
单帧最大字节数
最大材料年龄
```

容量为零、单帧超限、重复材料编号、缓存满载且不能按合同清理、过期材料或读取版本不一致均为逻辑内返回；不得写世界结构。缓存内部同步只保护材料一致性，不成为动作来源。

首轮缓存采用显式写入、只读副本读取、精确撤销和过期清理。不得在读者持有可写引用时淘汰，不得把裸指针作为材料身份。

### 5.2 上行消息

上行桥复用现有 `运行消息类型::外部材料`，只填写：

```text
消息编号
来源角色 = 外设采样材料
目标材料 = D455 缓存材料编号
句柄版本 / 当前句柄版本 = 材料版本
发生时间戳
批次号
材料摘要
只读材料说明 = true
承载机器事实 = false
```

`材料摘要` 只用于人读或快速不一致检查，不是帧内容、事实身份或业务裁决结果。

上行桥先完成材料准入，再在缓存中准备未发布候选，通过候选能力按同一编号和版本读回，构造消息后确认发布，最后用普通只读入口复核消息引用。候选读回、确认发布、普通读回或消息引用任一步不符合预期，必须精确撤销本次缓存材料并进入追根因解决；不得公开未发布候选，不得返回指向不存在材料的成功消息。

本轮上行桥只返回值式消息，不直接调用当前非线程安全的 `有界运行消息队列`，不建立生产线程接线。

## 6. 代码文件边界

第一轮固定建立 6 个真模块：

| 文件 | 模块 | 职责 |
| --- | --- | --- |
| `海中鱼巣/适配/协议.D455采样材料.ixx` | `海中鱼巣.适配.协议.D455采样材料` | SDK 无关配置、材料、枚举和结构化结果 |
| `海中鱼巣/适配/采集器.D455相机.ixx` | `海中鱼巣.适配.采集器.D455相机` | 唯一 RealSense API 边界；设备选择、有限等待、四路复制、停止释放 |
| `海中鱼巣/线程/缓存.D455帧材料.ixx` | `海中鱼巣.线程.缓存.D455帧材料` | 有界、版本化、只读的完整帧材料缓存 |
| `海中鱼巣/线程/上行桥.D455采样材料.ixx` | `海中鱼巣.线程.上行桥.D455采样材料` | 材料准入、缓存读回、轻量外部材料消息和精确撤销 |
| `海中鱼巣/线程/自检.D455采样材料.ixx` | `海中鱼巣.线程.自检.D455采样材料` | 无硬件模拟材料、拒绝、容量、过期、撤销和消息引用自检 |
| `海中鱼巣/适配/自检.D455真实样本.ixx` | `海中鱼巣.适配.自检.D455真实样本` | 显式真实 D455 设备、四路帧、标定、连续性和停止验收 |

六个文件均按 `ClCompile` 登记、包含匹配 `export module`，调用方只通过 `import` 使用。真实采集器和真实样本模块允许在非 x64 或未启用真实采集的构建中条件排除，但不得改为 `ClInclude`；生产模块不得依赖自检模块。

第一轮不建立传统 `.h/.cpp` 数据对。协议文件虽然导出值式结构，但同时承担外部材料协议和公共拒绝合同，按真模块建立；不得把 `.ixx` 按 `ClInclude` 登记或用 `#include` 引入。

## 7. 生命周期与入口拒绝

采集器状态至少区分：

```text
未打开
运行中
正在停止
已停止
打开失败
```

公开入口必须先拒绝：

```text
零宽高、零帧率或未知像素格式
零超时
空序列号格式
没有 D455
存在多个 D455 但未指定唯一序列号
指定序列号不存在
设备不支持所请求四路 profile
重复打开
未打开时采集
停止后继续采集
缓存上限无效
帧材料编号、版本、批次或时间戳无效
```

SDK 等待必须有明确超时，不得使用无法响应停止请求的无限等待。关闭和析构必须使 SDK pipeline 停止并释放，不能把 SDK 对象保存到缓存或消息。

## 8. 非成功返回二分

| 情况 | 分类 | 结构要求 |
| --- | --- | --- |
| 无设备、多设备未指定、序列号不存在、profile 不支持 | 逻辑内返回 | 不启动 pipeline，不写缓存，不生成消息 |
| 等待超时、设备断开、四路候选不完整 | 逻辑内返回 | 丢弃本次候选，停止或保持可明确重试状态，不发布材料 |
| 配置、材料编号、容量或版本在入口无效 | 逻辑内返回 | 入口拒绝，结构不变化 |
| 缓存满载、单帧超限、材料过期 | 逻辑内返回 | 不发布新消息；已有材料不被伪装为本次结果 |
| 前置已通过且 SDK 返回完整帧，但复制字节数、布局、标定或来源自相矛盾 | 追根因解决 | 停止依赖路径，不发布；宏开启时按公共容错规则记录和提醒 |
| 缓存写入成功后读回同一编号 / 版本不一致 | 追根因解决 | 精确撤销本次材料，停止依赖路径 |
| 已构造消息但其材料引用无法解析为刚发布材料 | 追根因解决 | 不返回消息，精确撤销本次材料 |
| 停止后仍有 SDK 等待不能收口 | 追根因解决 | 不声明生命周期验收通过，追查等待与所有权根因 |

逻辑内返回只产生结构化材料或拒绝结果；追根因路径不得被统一映射为普通“采集失败”后继续验收。

## 9. SDK 与工程边界

设计时确认的本机 SDK 事实：

```text
头文件：D:\vcpkg\installed\x64-windows\include\librealsense2\rs2.hpp
Debug x64：realsense2d.lib / realsense2d.dll
Release x64：realsense2.lib / realsense2.dll
版本：realsense2 2.56.3#1 x64-windows
```

这只是当前本机事实。实施前必须再次复核，不得修改 `D:\vcpkg`、安装包或扩大权限。SDK 缺失时停止实施并申请依赖安装授权。

工程新增 build-system 属性 `EnableD455RealSense`，默认关闭；真实采集构建必须同时为 x64 且显式启用。默认关闭时仍编译协议、缓存、上行桥和模拟自检，真实采集器与真实样本模块条件排除；未启用构建收到真实样本参数时必须明确非零拒绝，不得冒充通过。

SDK 根目录只通过 `$(VcpkgRoot)` 或 `$(VCPKG_ROOT)` 等构建属性提供，不得把 `D:\vcpkg` 写死进工程文件。Win32 不得误链 x64 库。运行时 DLL 必须由构建配置复制到对应输出目录，不能依赖偶然的系统 PATH。

## 10. 验收分级

### 10.1 无硬件自检

至少验证：

```text
四路完整模拟帧可写入、读回并生成轻量消息
缺任一路、零尺寸、错误步长、错误字节数、零编号和版本冲突均在发布前拒绝
缓存条数、总字节数、单帧字节数和年龄上限生效
消息只含材料引用，不含帧字节或 SDK 类型
缓存写后内部不一致可精确撤销
拒绝和撤销不写节点、主信息、关系、索引或领域结构
```

### 10.2 构建

```text
Debug x64 编译与链接通过
Release x64 编译与链接通过，或实施记录明确说明未验证原因
默认 `--self-test-exit` 回归通过
显式 D455 模拟自检通过
```

### 10.3 真实样本

只有连接真实 D455 并显式运行真实样本入口后，才可声明“真实 D455 样本通过”。至少验证：

```text
唯一设备身份可读
四路 profile 与实际帧元数据一致
连续多帧材料编号和宿主时间单调递增
每帧四路字节数与布局一致
深度比例和标定材料可读
有限超时与正常停止收口成立
缓存和消息仍不写世界事实
```

未连接设备、未运行真实样本或只通过模拟自检时，不得声明真实外设已接入并验证。

## 11. 完成声明边界

本规范和后续 D455-S1 计划完成后，按证据最多分级声明：

```text
D455 材料规范已确认
D455 采集与材料入口代码已实现
Debug / Release x64 已编译
模拟材料闭环已通过
真实 D455 样本已通过
```

不得扩大为：

```text
D455 已进入生产常驻运行
外设观察或视觉系统完成
体素、点云、分割、融合或跟踪完成
世界事实、存在、场景、状态或动态已由 D455 写入
本能方法、任务执行、方法学习或需求结算已接入
旧 D455 / 外设能力已经完整迁移
自我循环、自我苏醒或系统成熟完成
```

生产运行期接线、外设材料领域准入和视觉解释必须分别形成新的流程图、详细设计和计划，不得从本规范自动扩权。
