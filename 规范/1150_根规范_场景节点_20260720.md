# 1150 根规范 场景节点

编号：1150
日期：20260720
状态：正式根规范

## 1. 根定位

场景节点是世界事实、观察事实、输出事实和内部世界事实的容器身份，承载事实集合的归属、宿主和来源。场景不是任务回执容器，也不裁决需求满足或价值结算。

## 2. 本项目承载

```text
场景节点及稳定句柄
+ 主信息身份和唯一主键
+ 子场景结构关系
+ 场景到成员存在的归属关系
+ 场景到零散状态、动态和二次特征的事实目录关系或索引
+ 宿主、来源、时间范围和场景类型关系
```

关系仓库承载成员和事实归属，索引仓库只加速入口查找；索引命中后必须回读场景、事实节点和关系证据。

## 3. 事实与边界

1. 场景普通成员结构只允许子场景和成员存在。
2. 状态、动态和二次特征是零散事实，通过各自场景事实目录关系或索引归属，不得作为场景普通子节点。
3. 因果证据按因果根规范承载，不得作为自由说明节点挂入场景。
4. 需求、任务、方法、文章、显示摘要和任务回执不得成为场景普通成员。
5. 输出结果场景只承载真实结果事实，不承载“已成功”“已完成”等任务状态回执。
6. 临时比较、筛选、排序和一次性推导默认生成、使用后丢弃；只有满足二次特征持久化准入的关系事实才可进入场景事实目录。
7. 自我所在场景、输出结果场景和存在内部世界场景遵守同一成员边界，差异只由场景类型、宿主和来源关系表达。

## 4. 写入、读取与验证

场景领域服务负责场景身份、子场景和成员存在准入；状态、动态、二次特征领域服务分别负责产生本域事实，再由具名组合入口登记场景归属。显示层和普通调用方不得直接写场景结构。

有效读回必须证明场景身份当前、成员或事实节点类型合法、关系端点和版本一致。事实目录裁剪只能移除目录引用，不得删除事实节点或改变普通成员边界。

## 5. 生命周期与当前性

场景必须保留来源、适用时间范围和当前性。事实节点离开当前目录不等于事实被删除；历史事实仍须按稳定身份和来源回查。场景事实目录的固定容量和“新引用顶替旧引用”规则进入场景子规范，任何实现不得自行改变该合同。
