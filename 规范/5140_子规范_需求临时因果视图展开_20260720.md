# 5140 子规范 需求临时因果视图展开

编号：5140
日期：20260720
状态：正式子规范

## 1. 定位

临时因果视图是从一个完整需求目标向上游追溯条件、证据、促成路径、避免路径和方法就绪项的短生命周期解释结果。它不是新的权威需求树、因果树或机器账本。

上级规范：3100、1190、1195、5110。

## 2. 输入准入

展开前必须具有目标宿主、目标特征、当前状态材料、目标状态和满足关系。缺失任一权威目标材料时，不得展开；应进入目标补齐、拒绝或开发诊断。

可读取的正式来源仅包括：完整因果及其动态证据、需求侧条件、方法条件、条件结果对、已入树子需求和当前世界事实。不得按安全、服务、观察、外设或学习等领域名称写死前置链。

## 3. 因果追溯

1. 主动动作致变因果可以支持可任务化路径或方法候选。
2. 被动、提交、结算和状态变化因果只能提供证据、等待、条件或上游缺口；必须继续追溯其来源动态、来源因果或条件事实。
3. 追溯只能在找到主动动作致变因果，或明确停在事实、条件、证据、方法、规范或诊断缺口时收束。
4. 单次共现、概率候选、状态变化线索和轻量因果引用不得被扩大为确定路径。

## 4. 允许输出

临时视图只能输出：

- 可继续任务化的路径建议；
- 促成、避免、条件、证据、方法就绪、副作用或学习缺口草案；
- 不可任务化原因、等待条件或诊断材料。

输出必须保留目标、来源因果或动态、适用范围、当前性、生成版本和截断原因。它只是需求更新输入草案，必须交需求领域统一入口校验、去重、合并和裁决。

## 5. 路径与方法边界

1. 临时视图属于任务筹办中的路径分析材料，不替代任务级唯一筹办执行权和方法候选筹办。
2. 视图不得直接调用方法、创建任务、写需求树、改状态或提交结算。
3. 方法路径只说明某能力可能参与；真正候选必须按方法结果能力命中任务目标，并重新验证条件与动作入口。
4. 逻辑组织角色必须由目标特征定义和正式关系表达，不能按叶子/非叶子或显示名称猜测。
5. 子需求完成后仍要回查父目标当前是否达成，不得由路径结束直接宣称满足。

## 6. 安全、服务与观察边界

1. 未知场景不得直接解释为危险；安全路径必须有观察事实、风险因果和合法评估方法。
2. 服务值不足、自动衰减或无服务时间只可形成监听、机会检查、能力校准、进展核查、休眠或唤醒候选，不得伪造服务对象、服务成功或结算证据。
3. 服务不能使用“无负证据默认成功”；至少需要接收、反馈、外在状态改善、结果验证或因果归因。
4. 观察前置只能来自因果、需求条件或方法条件；相机预览、控制面板和只读快照不是需求满足证据。
5. 评估置信度、稳定度、优先级和成功率是临时推导，不能写成因果主信息权威字段。

## 7. 禁止事项

- 禁止持久化临时视图为并列权威树。
- 禁止生成不完整或仅含“改善、避免、明确”等口号的需求草案。
- 禁止任务管理侧自行指定层级权重；正式权重由需求领域按 5110 重算。
- 禁止把动作动态按成功/失败做根分类。
- 禁止把任务完成直接等于安全值、服务值或父需求变化。
- 禁止让未知、I64_MAX、无反馈或输出文本成为成功或满分证据。

## 8. 验证

验证必须覆盖主动因果、被动因果继续追溯、无因果路径、缺目标材料、路径截断、重复子需求合并、副作用并列输出以及草案未经提交不得成为正式需求。
