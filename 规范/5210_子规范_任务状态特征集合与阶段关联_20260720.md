# 5210 子规范 任务状态特征集合与阶段关联

编号：5210
日期：20260720
状态：正式子规范
权威等级：正式规范

## 1. 定位

本规范把 5200 定义的任务特征类型与任务生命周期阶段关联，规定每个阶段的最低结构前提、必需特征、可选特征和禁止激活特征。

生命周期阶段只是路由投影。本表是最低合同，不是用阶段枚举代替完整任务状态。

直接上级规范：3200。

直接依赖规范：5200。

相关边界规范：5220、5230。

## 2. 所有阶段共同前提

除入口拒绝外，任何有效任务阶段都必须能够回读：

```text
当前任务身份
来源需求
唯一运行态治理存在
运行场景
任务目标状态
当前生命周期关系、版本和发生时间
任务承接角色
目标当前性
授权当前性
当前业务状态段
状态段当前性
```

其中任一结构锚点缺失、重复或版本不一致，都属于结构不完整或内部逻辑错误，不得通过补写阶段枚举兜底。

## 3. 生命周期阶段与特征集合关联

| 阶段投影 | 最低必需特征 | 可选激活特征 | 禁止激活或必须为空 |
| --- | --- | --- | --- |
| 已创建 | 承接角色、目标当前性、授权当前性、当前业务状态段、状态段当前性 | 调度优先级、时限状态、路径和权重特征 | 筹办占用状态不得为有效占用；不得有当前方法、执行冻结、方法运行或实际结果 |
| 已承接 | 已创建全部；活动 OR 路径状态、所属 AND 组状态、继承父权重、AND 任务份额、冻结结算预算 | 调度优先级、时限状态、初次筹办触发类型 | 不得有执行冻结、执行轮次或方法运行；尚未筹办时不得伪造候选和当前方法 |
| 筹办中 | 已承接全部；筹办轮次、筹办触发类型、有效筹办占用、筹办读取当前性、路径查找状态、候选集合完备状态 | 方法选择当前性、方法特征契约当前性、条件/输入/参数/禁止项/验证/归因状态、筹办结果类型 | 不得有当前执行冻结、执行轮次、运行中方法或未停止的前轮动作 |
| 等待中 | 最近筹办轮次、筹办结果类型=子目标承接/能力缺口承接/合法等待之一；等待类型、等待条件当前性、重新触发状态、等待版本 | 历史候选和方法选择、缺失输入/条件/参数的开放方法特征、结算候选 | 筹办占用不得保持有效；执行冻结不得为当前；不得存在运行中方法。等待不得只由自由文本原因成立 |
| 排队中 | 当前方法选择记录；方法选择当前性=当前；方法特征契约当前性=已验证；前置条件=完整满足；输入=已闭合；参数=完整绑定；禁止项=未命中；验证和归因合同已形成；执行就绪=已就绪；执行冻结=当前 | 调度优先级、时限状态、方法声明的安全/结算/协调特征 | 不得有未满足等待条件、有效筹办占用、失效冻结、运行中方法或未定位结构缺口 |
| 执行中 | 排队中全部当前合同；执行轮次、执行输入当前性=当前、方法运行当前性=运行中、执行冻结当前性=当前 | 执行结果类型的进行中材料、开放输出/验证/归因特征 | 不得同时存在有效筹办占用；不得把冻结前已存在的缺方法、缺入口、缺条件结果对或目标缺失写成普通业务缺口 |
| 待重筹办 | 重新触发状态=已触发；触发来源关系；上一轮筹办或执行结果引用；原执行冻结=已失效；方法运行不再运行 | 历史候选、历史方法、结果和推进特征、等待解除证据 | 筹办占用在新一轮开始前不得有效；不得继续使用旧执行冻结；不得把本状态解释为上一轮“无法解决” |
| 已完成 | 实际结果状态；执行结果类型；目标达成裁决=已达成；验证裁决=验证通过；结果当前性=当前；目标达成比例；完成当前性=当前；完成来源和完成版本关系 | 本轮推进量、归因状态、附带结果路由、结算候选和结算状态 | 不得有有效筹办占用、运行中方法、当前执行冻结、未满足等待条件或待重筹办触发 |
| 失败 | 业务失败分类；失败证据当前性=当前；实际结果或合法失败条件；终止版本；方法运行不再运行 | 禁止结果状态、归因状态、附带结果路由、结算拒绝材料 | 内部逻辑错误、筹办漏检、无候选方法和单纯等待不得写成任务失败；不得有有效筹办占用或运行中方法 |
| 取消 | 取消当前性=取消完成；合法取消授权关系；终止版本；若取消前正在执行则动作停止确认=已停止 | 取消前历史候选、方法、结果和归因材料 | 不得有有效筹办占用、当前执行冻结或运行中方法；没有取消授权不得进入取消 |

## 4. 正式业务状态段

生命周期阶段之下至少区分以下业务状态段。状态段身份必须和表中区分特征同时成立；只写状态段名称仍不构成完整任务状态。

| 业务状态段 | 生命周期投影 | 区分该状态段的必需特征 | 禁止同时成立 |
| --- | --- | --- | --- |
| 承接初态 | 已创建 | 承接角色已形成；目标当前；授权当前；状态段当前 | 筹办轮次、有效筹办占用、当前方法、执行冻结 |
| 待初次筹办 | 已承接 | 活动 OR 路径和所属 AND 组已确定；权重、AND 份额和结算预算已冻结；筹办触发类型=初次承接 | 有效筹办占用、当前方法、执行冻结 |
| 筹办权已取得 | 筹办中 | 新筹办轮次；有效筹办占用；筹办读取当前性=当前；候选集合完备状态=未开始 | 运行中方法、当前执行冻结、另一有效筹办占用 |
| 路径形成中 | 筹办中 | 筹办权已取得全部；路径查找状态=形成中或需补齐；候选集合尚未完整 | 当前方法选择、当前执行冻结 |
| 候选形成中 | 筹办中 | 路径查找已得到可召回方向；候选集合完备状态=形成中 | 当前方法选择、当前执行冻结 |
| 候选已形成待选择 | 筹办中 | 候选集合完备状态=已完整；候选来源记录和版本当前；方法选择当前性=未选择 | 当前方法发布关系、执行冻结 |
| 方法已选择待契约复核 | 筹办中 | 当前方法选择记录当前；方法特征契约当前性=未挂载或待验证 | 执行就绪=已就绪、当前执行冻结 |
| 方法契约复核中 | 筹办中 | 方法特征已挂载；条件、输入、参数、禁止项、验证和归因合同逐项形成 | 当前执行冻结；未验证合同不得进入排队 |
| 等待子目标 | 等待中 | 筹办结果类型=子目标承接；等待类型=等待子目标；子目标、合法生产者和回流触发关系完整 | 有效筹办占用、当前执行冻结、运行中方法 |
| 等待能力补齐 | 等待中 | 筹办结果类型=能力缺口承接；等待类型=能力补齐；缺口宿主、缺口特征、合法生产者和回流触发完整 | 无定位的泛化学习需求、自动创建的方法首、当前执行冻结 |
| 等待条件或输入 | 等待中 | 等待类型=条件或输入；未满足条件/缺失输入集合、合法生产者、重新触发条件和等待版本完整 | 把目标语义不完整、缺方法入口等内部结构问题包装成等待 |
| 等待外部事实或授权 | 等待中 | 等待类型=外部事实、授权或时间；等待对象、来源、合法生产者、重新触发条件完整 | 自我臆测外部存在内在状态、当前执行冻结 |
| 可执行已冻结 | 排队中 | 方法选择、方法契约、条件、输入、参数、禁止项、验证、归因全部通过；执行就绪=已就绪；执行冻结=当前 | 有效筹办占用、未满足等待条件、结构缺口 |
| 执行动作中 | 执行中 | 执行轮次；执行输入当前；方法运行当前性=运行中；执行冻结当前 | 有效筹办占用、旧冻结、并发第二执行占用 |
| 执行结果待验证 | 执行中 | 方法运行已结束；实际结果已形成且当前；执行结果类型已确定；目标达成、推进和验证裁决尚未全部收束 | 任务完成、结算提交、用动作成功代替目标达成 |
| 目标未达成待重筹办 | 待重筹办 | 目标达成裁决=未达成；结果当前；上一执行轮次引用；重新触发状态=已触发；旧冻结失效 | 继续执行旧方法运行、把未达成直接写成失败 |
| 当前性失效待重筹办 | 待重筹办 | 目标、授权、路径、方法、输入或世界事实至少一项当前性失效；触发来源明确；旧冻结失效 | 继续使用旧冻结、覆盖上一筹办轮次 |
| 任务完成 | 已完成 | 目标达成=已达成；验证通过；实际结果当前；完成当前性=当前；目标达成比例和完成来源完整 | 有效等待、待重筹办、运行中方法、当前执行冻结 |
| 业务失败终止 | 失败 | 合法业务失败分类；当前失败证据；终止版本；方法运行已结束 | 内部逻辑错误、无候选、筹办漏检、普通等待 |
| 授权取消终止 | 取消 | 合法取消授权；取消完成；终止版本；执行过则动作停止已确认 | 无授权取消、未停止动作、有效筹办占用 |

同一业务状态段仍可由方法开放特征继续细分。例如两个任务都处于“方法契约复核中”，但所需输入、验证或安全特征类型不同，它们是不同完整状态；状态段名称只负责路由共同处理逻辑。

## 5. 筹办闭合结果与阶段去向

| 本轮筹办结果 | 必须同时形成 | 阶段去向 |
| --- | --- | --- |
| 可执行冻结 | 当前方法选择、方法特征挂载、输入/条件/参数/禁止项/验证合同完整和当前冻结 | 排队中 |
| 子目标承接 | 明确子目标、承接关系、合法生产者和回流触发 | 等待中 |
| 能力缺口承接 | 明确缺口宿主、缺口特征类型、合法生产者和回流触发 | 等待中 |
| 合法等待 | 等待对象、等待条件、合法生产者、重新触发条件和等待版本 | 等待中 |
| 目标已完成 | 当前实际结果、逐项满足证据和验证通过 | 已完成 |
| 授权失效 | 失效授权事实；可恢复时形成重新授权等待，不可恢复且有取消授权时形成取消 | 等待中或取消 |

候选方法为空只能形成“合法等待”或已正式定位的“能力缺口承接”，不得自动创建方法首，不得进入失败，也不得返回无结构的上层回退。

## 6. 重复筹办

以下事实可以触发新筹办轮次：

```text
子任务或子目标回流
等待条件满足
执行后目标仍未达成
当前方法、条件、参数或输入版本变化
活动路径或权重变化
任务授权变化
世界状态变化使旧冻结失效
```

新一轮必须增加筹办轮次，建立新的筹办占用和读取版本，并把上一轮状态段、候选、方法选择和冻结保留为历史。不得覆盖上一轮记录。

## 7. 特征一致性规则

1. 阶段迁移前必须验证目标阶段的必需特征全部存在且取值合法。
2. 目标阶段禁止激活的特征仍为当前时，不得迁移阶段。
3. 方法开放特征可以增加业务状态细分，但不得取消本表最低必需特征。
4. 相同阶段下，方法开放特征、值、来源或版本不同，属于不同业务状态段。
5. 生命周期枚举、状态节点、控制面板显示和日志不得替代本表的特征集合复核。
6. 筹办占用是运行治理事实，只在筹办中允许有效；它不替代任务业务状态，也不以线程编号表达。
