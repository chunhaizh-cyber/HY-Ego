# 4170 子规范 特征批次发布记录与幂等账

编号：4170
日期：20260724
状态：正式子规范

## 1. 定位

本规范定义一次特征批次发布如何形成专用、不可变、可完整语义比较的幂等记录。该记录只证明一组实例特征槽、当前值换代和值来源已经在同一事务中共同发布，不是新的特征值、状态、动态、任务结果或世界事实。

直接上级规范：1130、2100。

直接依赖规范：0500、4010、4020、4030、4040、4110。

相关边界规范：4050、4060、4140、4160、5200。

## 2. 结构身份与唯一职责

`特征批次发布记录` 是特征领域内部的专用不可变幂等账。它具有非零强类型批次身份和非零受支持规则版本，并绑定一次已经发布的完整有序项目组。

该记录使用特征领域专用的权威侧账承载，不复用特征值原始材料记录，不伪装成特征值节点、动态节点、状态节点或其它业务对象，也不把批次字段写入槽位、值节点或宿主的领域记录。

侧账的唯一职责是：

```text
按批次身份定位已经发布的完整请求语义和结果引用
+ 支持同键同义的历史重放读回
+ 支持同键异义的冲突拒绝
+ 为批次成员、顺序、来源和换代前后证据提供审计引用
```

侧账不得成为实例特征当前值的第二权威。当前槽位和值仍只由宿主关系、模板关系、当前值关系、特征值原始材料及其版本共同裁决。

同一个运行期结构域内只能有一个承担本规范职责的特征批次发布账。节点直接新域与旧默认域在最终切换前属于两个隔离运行期域：各自只能解释本域服务实际发布的批次，不能双写、双读、拼接、择优或用一域账补另一域缺项。节点直接新域的完整快照只读取节点直接特征数据操作实际提交的本域账；旧默认域账只由旧域审计、离线迁移或最终切换治理处理。

账内强类型批次身份只需在所属运行期域内唯一。节点直接新域快照需要形成两分量稳定身份时，规范化投影固定使用：

```text
节点直接特征批次身份命名域 ABI = uint64 1
+ 账内强类型批次身份
```

命名域 ABI 是跨恢复保持不变的格式常量，不能替代快照头中的来源运行期域身份；来源运行期域身份仍只来自 4040 冻结许可的真实见证。该投影不反向改写账内批次身份，也不建立第二批次权威。

### 2.1 结构准入证明

现有正式结构不能等义承载本记录：节点和稳定句柄只表达对象身份；关系不能独立保存一个批次的完整有序项目、写前证据和发布后引用；特征、特征值、状态和动态表达对象属性、时点情况或变化，不表达一次事务发布的幂等审计语义；索引和摘要只能加速定位，不能成为权威账；特征值原始材料记录只承载值本体，扩用会把值本体与批次业务语义混成双重权威。把这些材料写入节点通用字段或其它领域记录又会违反 4020 的分账边界。

因此专用侧账只承载“某一特征批次已经按完整语义共同发布”的从属记录，并满足 0500 的准入条件：

```text
唯一写入方：特征体系数据操作
唯一业务解释方：特征业务服务
读取方：幂等预读、写权内重检、发布后权威读回和恢复治理
生命周期：随批次发布建立，发布后不可变；不参与当前值裁决
事务边界：与槽位、值、关系和原始材料在同一 4040 事务共同发布
失效 / 恢复：普通业务不删除；恢复材料重新准入，不能原地覆盖
无双权威：当前值始终由正式槽位、值、关系和原始材料裁决
```

该记录虽有强类型批次身份，但没有独立于特征批次的目标、动作、世界事实来源或业务生命周期，只是 1130 / 2100 对象结构的一次发布证据；把它提升为新根对象会错误制造第二个特征事实本体。因此本规范是特征主题子规范，不建立新根对象。

## 3. 记录内容

一条已发布记录至少保存：

```text
强类型批次身份
非零侧账记录格式版本
受支持规则版本
候选或已发布宿主的完整稳定句柄
按非负连续顺序号排列的全部项目
每项的初始或换代种类
特征定义完整句柄
实例槽位完整句柄
新特征值完整句柄
合法来源完整句柄
换代项的写前当前关系完整句柄、写前当前值完整句柄和原始值版本
发布后的新当前关系完整句柄
```

侧账记录格式版本只解释记录字段形状和恢复兼容性；业务规则版本只解释批次语义和比较规则，二者不得合并、互相替代或用零值表示“默认”。

完整句柄必须作为类型化字段保存，不得拆成通用 I64 槽位。原始 I64、VecI64 或 VecU64 值只保存在特征值正式结构中；发布记录通过新值句柄回读，不复制原始值形成第二份值本体。

调用方不得提供用于裁决的载荷摘要。实现可以从强类型完整有序字段确定计算运行期摘要，但它只能作为可清空、可重建的非权威筛选缓存；无论摘要相同或不同，最终同义、冲突、成功和失败都必须由完整逐项语义及正式结构互证裁决，摘要无需成为已发布记录或业务结果字段。

## 4. 来源和版本

来源版本默认就是来源稳定句柄中的版本号，不再保存第二个裸版本字段。服务准入和第一写前必须共同证明：

1. 已发布来源句柄当前有效，且记录中的完整句柄版本与当前记录一致；或
2. 来源是同一事务内的具名候选，完整句柄属于本会话候选、候选类型合法，并与本批次宿主或登记的同层候选来源一致。

若某来源对象另有独立内容版本，必须引用该对象正式结构已经定义的版本材料；不得临时增加一个无承载来源版本。来源节点与值、槽位或批次的证据联系仍由正式来源关系和完整句柄共同证明。

## 5. 幂等和冲突

同一批次身份再次提交时必须先读取专用侧账，并分类为：

| 读取结果 | 收口 |
| --- | --- |
| 未找到 | 继续执行全部写前准入 |
| 找到且完整有序语义相同 | 权威重读槽位、值、关系和来源后返回已发布结果 |
| 找到但任一规则版本、宿主、项目数、顺序、种类、定义、槽位、新值、来源或换代写前证据不同 | 幂等冲突，零写入 |
| 许可竞争、版本漂移、格式未知或读取不完整 | 返回对应拒绝或内部逻辑错误，不得伪装成“已存在”或幂等冲突 |

同义判断必须比较完整有序项目语义和正式结构读回。批次身份相同但项目顺序不同属于异义；只有全部记录和全部已发布结构互证成立时才允许返回历史成功。

事务外预读只用于快速收口已发布历史，不授予继续写入。预读为“未找到”后，执行路径必须先取得覆盖本批次全部结构的核心唯一写入权，并在第一笔结构写入前再次读取侧账：同义则不形成候选并释放写入权后返回历史结果；异义零写冲突；许可、版本、格式或读取异常保持各自分类；只有锁内重读仍为未找到时才能形成候选。批次记录参与者准备阶段还必须在侧账独占占用内作最终竞争互证。

## 6. 同事务发布

批次发布记录必须成为 4040 统一事务中的独立内部材料参与者，并与特征值类型化原始材料参与者、节点、关系和索引共同收口：

```text
事务外预读未找到
-> 取得覆盖全部参与结构的核心唯一写入权
-> 第一结构写前再次读取侧账并完成同义 / 异义 / 异常分类
-> 仍未找到才形成未发布的结构候选和侧账候选
-> 特征值原始材料参与者和批次发布记录参与者按登记顺序准备并完整读回
-> 结构会话完成确认
-> 全部参与者按登记顺序确认待发布
-> 任一失败按依赖逆序撤销全部已准备参与者和结构写集
-> 全部确认后在同一最后发布区关闭撤销能力
-> 释放唯一写入权后普通读取才能取得结构与批次记录
```

结构执行边界必须支持同一事务登记多个具名参与者。登记顺序、确认顺序和逆序撤销顺序必须确定；不得把两个参与者拆成两个事务，也不得把一个参与者的成功当作全部参与者成功。

## 7. 写入权、读取权和生命周期

特征业务服务唯一解释批次身份、规则版本、完整请求语义、来源和值域；特征体系数据操作唯一形成侧账候选并把专用参与者加入同一事务。其它数据操作只能通过已授权的不透明同层合同消费完整规格，不得直接写、删、覆盖或伪造批次记录。

已发布记录不可原地修改。实例当前值后续再次换代时形成新的批次记录；旧批次记录、旧值和失效关系继续作为审计历史，但不参与当前值裁决。

本规范不授权普通业务删除批次记录。快照、恢复或清理必须保留格式版本、完整批次身份、规则版本、有序项目和全部完整句柄；恢复候选需要重新复核引用结构和版本后才能发布。未知格式、悬空句柄、项目缺项、顺序不连续或引用结构不一致时不得恢复为可读幂等账。

### 7.1 同冻结许可全量读取

节点直接新域的冻结提供面必须消费统一冻结协调方已经持有的 4040 不透明冻结许可，并一次返回：

```text
同一冻结见证
全部已发布特征值类型化记录副本
全部已发布本域特征批次记录副本
两仓真实结构版本与记录数量
```

两个记录组只能作为一个原子值式结果共同存在或共同为空；不得用两个独立可选结果表达部分成功。冻结提供面属于服务专用数据操作的有界内部只读入口，不进入特征领域公开业务服务，不暴露记录仓、内部容器、锁、事务许可或参与者。

锁序固定为“事务域统一冻结许可 -> 特征值类型化记录仓共享锁 -> 本域特征批次记录仓共享锁”。提供面在同一许可内读取见证、结构版本和数量，复制全部已发布记录，再次读取并复核；不得用分别取得的两个普通读取许可或事后比较最大批次编号冒充同代次。批次账没有可推导高水位时，不得从最大批次编号伪造高水位。

本域冻结记录必须保留运行期真实字段：账内批次身份、记录格式版本、业务规则版本、宿主和完整有序项目。发布状态只能从已发布记录形成规范化 `已发布` 投影；候选记录不得进入冻结材料。项目种类由写前关系、写前值和写前原始值版本三个可选字段全部为空或全部存在唯一判定，不能出现部分存在。

## 8. 禁止事项

- 禁止在特征值原始记录、实例槽位、宿主或来源对象领域记录中增加批次身份、规则版本、摘要、顺序、来源版本或拆分句柄槽。
- 禁止把特征值原始材料侧表扩成批次业务账。
- 禁止只凭索引命中、摘要命中、节点数量、普通 bool 或日志返回历史成功。
- 禁止把许可失败、读取失败或不完整记录归类为幂等冲突。
- 禁止发布记录复制原始特征值或成为当前值第二权威。
- 禁止一个参与者发布后再以另一个事务补写批次记录。
- 禁止节点直接新域快照读取旧默认域账，或把两域批次记录合并成一个 4170 段。
- 禁止冻结提供者自行取得第二事务域冻结权、回显调用方提供的域 / 代次，或从最大批次编号伪造高水位。

## 9. 验证合同

验证至少覆盖：

1. 多槽初始、多槽换代和混合批次均形成一条完整有序记录。
2. 同键同义读回同一记录和同一已发布结构；同键异义零写冲突。
3. 顺序变化、规则版本变化、来源变化和任一换代写前证据变化均不能返回旧成功。
4. 摘要相同但逐项不同仍冲突；清空摘要缓存不改变幂等结果。
5. 已发布来源和同会话候选来源分别通过完整句柄版本复核。
6. 许可竞争、版本漂移、未知格式和不完整记录保持各自分类。
7. 每个结构写点、每个参与者准备 / 确认点和最后发布前失败均逆序撤销为零可读半结构。
8. 批次字段在特征值原始记录、槽位、宿主和来源对象领域记录中零新增；句柄没有被拆分保存。
9. 快照或恢复遇到悬空引用、顺序缺口或未知格式时不发布。
10. 未支持规则版本在第一写前具名拒绝；单参与者旧事务入口和多参与者事务入口使用同一收口算法且旧回归不漂移。
11. 节点直接新域全量冻结只读取本域唯一账；旧域同号批次不能进入、覆盖或补齐新域结果。
12. 同一冻结许可下两组记录、见证、结构版本和数量共同自洽；许可变化、错域或部分材料进入内部逻辑错误。
