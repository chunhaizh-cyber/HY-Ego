# 8100 子规范 自我线程与任务管理线程权责边界

编号：8100
日期：20260721
状态：正式子规范

## 1. 定位

本规范规定自我线程、任务管理界面、任务工作线程、需求域、任务域和方法域之间的编排与写入权边界。

直接上级规范：1170、3100—3300。

直接依赖规范：5100—5330、6340。

相关边界规范：8110、8200。

## 2. 总原则

```text
自我线程
-> 管“为什么做、做哪个、优先做什么”

任务管理线程
-> 管“任务按什么正式状态合同推进到哪一步”

任务工作线程
-> 执行一个任务工作包，不拥有业务事实写入权

领域服务和方法动作入口
-> 提交需求、任务、方法和世界事实
```

线程是执行载体和调度者，不是动作来源。主动方法执行或领域正式提交才是动作动态的来源候选。纯消息搬运、回执整理和线程调度属于非业务派生处理，不登记为方法，不生成主链动作动态。

## 3. 自我线程职责

自我线程可以：

- 扫描和整理需求；
- 读取根特征和自我当前状态；
- 生成、估权和选择当前主需求；
- 形成任务发起、调度、取消、挂起、重筹办和权重调整意图；
- 对真实外部动作工作项给出一次性执行许可；
- 读取任务结果证据并由需求域入口复核需求满足；
- 触发合法结算和下一轮需求重判。

自我线程不得直接写任务状态、当前方法、候选集合、执行冻结、任务运行态治理存在或任务结果。自我线程发送的是事件、意图、材料或许可，不是目标任务阶段值。

## 4. 任务管理职责

任务管理界面和任务调度层负责：

1. 承接任务发起意图并请求任务领域创建任务；
2. 维护任务工作包、调度队列和工作项身份；
3. 按任务正式状态合同触发筹办、执行、等待、重筹办、取消和结果验证；
4. 调用任务领域提交入口推进任务状态；
5. 把任务结果、缺口、等待、执行许可请求和需求满足证据上报自我线程；
6. 按 6340 已承接材料合同，把任务域消费入口返回的材料引用纳入对应工作项调度；
7. 把纯线程生命周期材料发送到控制面板线程信息模块。

任务管理线程不得直接写需求、存在、场景、特征、状态、动态、因果或价值事实，也不得成为任务状态动作动态的来源。外设报告队列的唯一消费权、匹配和已承接材料边界由 6340 唯一规定；任务管理层只调度其正式返回，不复制报告或另行裁决世界事实。

## 5. 任务工作线程职责

任务工作线程每次只推进一个绑定任务和工作项的闭合工作包。它可以调用任务筹办、方法执行、结果验证和任务域提交入口，并返回结构化工作结果。工作包需要外设材料时，只能接收符合 6340 合同的已承接材料引用，不得自行轮询外设高频流或报告队列。

工作线程不得裸写任务状态、需求树或世界事实，不得把线程运行、堵塞、暂停和退出映射为任务业务状态，也不得以自身线程编号表达任务筹办或执行占用。

## 6. 任务筹办和执行

任务筹办是任务领域内建最小方法，不进入普通方法候选，也不建立独立持久化筹办回执。它在取得任务稳定身份下唯一筹办执行权后直接形成闭合结果；未取得者必须在召回前拒绝。

筹办只按需求目标、活动路径、因果、方法条件、条件结果对、授权、任务特征和当前事实找路径、找方法、找条件。不得把安全、服务、观察、外设或学习领域规则硬编码进筹办器。

无候选方法时不得自动创建方法首。筹办必须形成合法等待、子目标、能力缺口、目标已完成或授权失效等闭合结果。

执行只能消费当前冻结。执行期发现冻结前已存在的方法结构缺口属于筹办漏检和内部逻辑错误。

## 7. 执行前许可

任务管理准备派发真实外部动作时，向自我线程提交绑定工作项的执行许可请求。自我线程返回允许、禁止或挂起等待。

同一工作项未结束前不得重复申请。许可只决定本工作项能否开始真实动作，不裁决筹办、排队、等待、重筹办、完成、失败、取消或结算状态，也不替代任务授权和执行冻结。

## 8. 任务和需求状态分离

任务完成属于任务域；需求满足属于需求域。标准链：

```text
任务方法形成实际结果
-> 任务域验证任务目标并提交任务完成
-> 任务管理上报任务结果和需求满足证据
-> 自我线程请求需求域复核目标
-> 需求域提交需求满足或保持未满足
-> 合法结算入口处理价值
```

方法返回成功、任务完成回执或线程退出都不能直接写需求已满足。

## 9. 消息边界

自我到任务管理的合法消息包括任务发起、调度、取消、挂起、重筹办、权重调整、事实更新唤醒、父任务等待条件变化、结算完成和执行许可结果。

任务管理到自我的合法消息必须绑定具体需求、任务、工作项或执行尝试，包括任务创建结果、实际结果、任务缺口、合法等待、需求满足证据、派生需求三件套和执行许可请求。

外设报告到达本身不是自我治理消息。只有 6340 已承接材料经正式领域处理后形成的任务结果、结构化缺口、派生需求三件套或事实提交结果，才可以按对应消息合同进入自我线程。

纯线程创建、运行、堵塞、暂停、恢复、退出和健康状态只能进入控制面板线程信息模块，不得进入自我治理邮箱。

## 10. 写入权矩阵

| 领域 | 编排者 | 正式写入者 | 禁止跨写 |
| --- | --- | --- | --- |
| 需求 | 自我线程 | 需求领域服务和正式提交方法 | 任务管理不得写需求树 |
| 任务 | 任务管理线程 | 任务领域服务和状态提交入口 | 自我、工作线程不得裸写 |
| 方法 | 方法治理与学习流程 | 方法领域服务和动作入口 | 任务不得改方法能力事实 |
| 世界事实 | 具体业务方法与世界聚合 | 对应基础信息领域服务 | 线程和消息不得写事实 |

## 11. 禁止规则

1. 自我线程不得直接指定或写入新任务状态。
2. 任务管理不得直接写需求或世界事实。
3. 任务工作线程不得用自身状态代替任务状态。
4. 任务筹办不得直接写需求或硬编码领域前置关系。
5. 线程、队列、调度器和回执整理不得成为动作来源。
6. 纯线程状态不得成为自我治理事实。

## 12. 验证边界

验证至少覆盖任务发起、任务创建、唯一筹办占用、执行许可、工作项单次派发、任务结果上报、需求复核和线程信息隔离。
