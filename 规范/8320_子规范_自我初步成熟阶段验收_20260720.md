# 8320 子规范 自我初步成熟阶段验收

编号：8320
日期：20260720
状态：正式子规范

## 1. 定位

本规范规定自我苏醒后，从内部因果缺口形成学习需求，在内部信息不足时形成有目标、有安全门控、可回写的外界探索需求，并用结果推进原需求的初步成熟验收。

上级规范：0050、1190、1195、3100—3300、4310、6100—6130、8210、8310。

## 2. 核心定义

```text
初始化
-> 内部循环建立
-> 自我苏醒
-> 因果补完
-> 学习需求
-> 外界探索需求
-> 初步成熟
```

初步成熟不是“能够看到外界”，而是“能由内部需求说明为什么必须取得外界信息，并把结果用于自身因果和方法”。这里的说明必须由结构化缺口、目标、门禁和回写事实成立。

## 3. 前置条件

初步成熟必须建立在 8310 的苏醒门槛之后，并已经具备连续内部循环、任务承接和执行、实际回读、需求复核、合法结算、下一轮重判、待机及自检治理。

前置未满足时，只能称内部循环建设、苏醒建设或因果补完候选阶段。

## 4. 因果补完

自我需要回答本轮需求为何出现、任务为何未达目标、方法为何有效或无效、状态为何变化以及结果如何影响安全和服务。

最小因果证据至少关联前置状态、动作或动态、结果状态、受影响需求和安全 / 服务结果。第一阶段先保留具体证据动态样本或轻量因果引用；只有多个相似样本稳定且规则门禁满足后，才可沉淀因果模板。单个因果引用不能升级为稳定因果结论。

## 5. 学习需求

当前需求无法由已有方法满足、任务缺必要条件、方法结果不稳定、状态变化无法解释或因果链缺关键节点时，可以形成具名学习缺口。

学习需求必须包含目标宿主、目标特征、当前状态、目标状态、满足关系、父需求、活动路径、权重来源和合法生产者。不得只写“任务失败”或“需要学习”。

学习类型至少可以区分状态、因果、方法、对象、边界和风险。学习首先查内部因果、历史任务、方法结果、已有状态和记忆。

## 6. 外界探索需求

只有同时满足以下条件，学习需求才能派生外界探索需求：

- 内部信息不足；
- 原需求仍当前有效；
- 外部信息可能降低具名不确定性；
- 探索风险可接受；
- 存在可筹办的观察、询问、采样或其它探索路径。

探索需求至少明确探索对象、探索目标、期望信息、风险约束和完成条件。禁止空泛“探索外界”。

合法示例包括确认对象当前状态、取得任务缺失条件、验证动作是否导致结果或判断外部反馈是否支持方法。

## 7. 安全门控

探索前必须判断是否降低安全、破坏当前内部循环、覆盖核心记忆、产生不可恢复状态，以及是否有停止、待机或替代方案。

风险不可控时，探索需求转为安全评估、低风险观察或补充条件需求，不得直接执行。探索任务仍需正常筹办、执行冻结和动作验证。

## 8. 结果回写

探索结果不能只形成一次性文本。合法结果必须经对应实在域实际回读和领域提交，进入外部对象状态、任务结果、方法验证证据、因果证据样本、学习需求和探索需求的复核结算。

对象引用使用稳定句柄和具名关系，不保存裸指针。外设原始材料必须经观察承接，不直接进入因果或世界事实。

## 9. 八条验收标准

1. 自我能结构化指出缺少何种信息，而不只报告失败。
2. 未知能形成完整原子学习需求并归属原需求。
3. 学习需求能形成任务并由合法学习方法承接。
4. 自我先查内部信息，内部足够时不生成外界探索。
5. 每个探索需求都有对象、目标、期望信息、风险和完成条件。
6. 探索经过安全、可恢复和动作验证门禁。
7. 探索结果进入正式事实、因果证据和需求复核，不停留在文本。
8. 探索结果改变下一轮原需求：继续推进、确认不可行并换路，或发现风险并生成安全需求。

## 10. 最小成熟闭环

```text
内部需求
-> 任务未达目标或信息不足
-> 具名因果缺口
-> 学习需求
-> 内部学习任务
-> 内部信息仍不足
-> 有目标的外界探索需求
-> 安全门控和探索任务
-> 实际外界信息回写
-> 因果证据补完
-> 原需求继续、换路或重判
```

直接服务探索仍受服务触发事实和因果链准入约束；无直接服务触发时不得凭服务值缺口虚构服务对象和外界探索目标。

## 11. 不合格情况

没有内部需求驱动的随机探索、只在外部输入后被动响应、能学习但不能判断何时需要外部信息、取得信息却不回写、以及未知即绕过安全门控直接探索，均不属于初步成熟。

## 12. 验证边界

验收必须证明探索需求由内部因果缺口自动形成，目标和完成条件明确，风险门禁通过，实际外部结果进入世界事实和因果证据，并改变原始需求的下一轮判断。
