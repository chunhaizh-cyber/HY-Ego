## 1. 定位

方法系统负责表达“通过什么能力或动作可以推进某个目标状态”。

方法不是任务本身，也不是底层特征值写入原语。任务负责承接需求，方法负责提供可选择、可执行、可学习、可复用的能力结构。方法执行必须发生在任务授权、任务承接和任务选择方法之后。

本规范只建立方法系统第一轮边界，覆盖主信息迁移 S4 中登记的 MI-M1、MI-M3、MI-M4、MI-M5。MI-M2 动作句柄由 `规范/动作入口规范.md` 承接。

本规范不迁移旧 `方法主信息类` 字段，不恢复旧方法主信息 variant 结构，不声明方法系统代码已完成。

## 4. 方法节点角色

底层节点类型第一轮不新增：

```text
节点类型::方法
```

方法首、方法条件、方法结果第一轮不是新的底层节点类型，而是方法节点上的方法角色。实现时可通过方法角色状态、方法角色关系或后续确认的角色结构承载。

第一轮方法角色状态承载口径：

```text
方法角色状态使用抽象状态节点，不携带发生时间戳。
方法节点必须通过 `模板` 或方法服务确认的角色关系绑定唯一角色状态。
读取方法角色时，必须由方法服务读取该角色状态关系并复核状态值。
外部调用方不得直接按裸 `引用`、`模板` 或状态节点值裁决方法角色。
```

第一轮方法角色：

| 角色 | 含义 | 第一轮口径 |
| --- | --- | --- |
| 方法首节点 | 方法能力入口和候选召回锚点 | 可建立方案，代码待确认 |
| 方法条件节点 | 方法执行所需条件、参数规格和条件判定索引 | 可建立方案，代码待确认 |
| 方法结果节点 | 方法可产生的结果状态、结果方向和结果样本锚点 | 可建立方案，代码待确认 |
| 条件结果对 | 条件节点与结果节点之间经运行验证后的配对证据 | 后续增强，不进入第一轮最小实现 |
| 根写入原语 | 底层写入能力，不是普通可选方法 | 只能作为方法系统内部能力标记，不作为任务候选首选 |

如果后续需要新增 `方法首`、`方法条件`、`方法结果` 等专用节点类型，必须先修订节点类型与关系类型枚举规范，不得在代码中临时扩展。

## 5. 方法虚拟存在

方法虚拟存在是方法长期能力基底，不是世界事实存在。

规则：

```text
必须由存在服务创建。
来源必须是有效方法节点。
不得保存某个任务的专有参数、筹办回执、执行回执、临时缺口或任务运行账。
可以被多个任务虚拟存在引用为能力基底。
不得由任务服务绕过方法服务或存在服务直接改写。
不得写回主信息裸字段作为权威。
```

方法长期能力、方法状态、可用性、成熟度、运行统计和学习结果应通过方法虚拟存在关联到特征、状态、动态、二次特征或因果证据。任务本轮运行材料必须进入任务虚拟存在或方法运行实例结构，不得污染方法虚拟存在。

## 6. 条件场景和结果场景

方法条件场景和方法结果场景是方法规格容器，不是现实场景。

规则：

```text
方法条件场景保存条件、参数模板、条件判定索引和条件样本。
方法结果场景保存预期结果模板、结果状态样本、结果方向和结果初始状态引用。
任务执行的真实输入场景、执行期临时结果场景和输出回执场景必须另行承载。
运行实例可以在确认后归并为方法条件节点、方法结果节点或条件结果对，但不能直接把临时运行场景当作长期方法规格。
```

方法条件场景和结果场景的创建、读取、归并和状态变化必须经方法服务及基础信息服务完成，不能由控制台、日志、显示层或任务模块裸写。

第一轮方法条件节点和方法结果节点采用创建时自动绑定方法首的口径：

```text
创建方法条件节点时，必须同时创建方法条件角色节点、写入方法条件角色状态、绑定条件场景，并写入方法首节点到条件节点的模板关系。
创建方法结果节点时，必须同时创建方法结果角色节点、写入方法结果角色状态、绑定结果场景，并写入方法首节点到结果节点的模板关系。
未绑定到有效方法首的方法条件节点或方法结果节点，不得被读取为有效方法条件或有效方法结果。
```

## 7. 方法状态和策略状态

第一轮方法状态建议使用状态结构表达，不使用普通 bool 持久化。

方法状态候选：

| 状态 | 含义 |
| --- | --- |
| 待方法动作 | 缺少语义动作或动作入口 |
| 待可执行入口 | 有方法语义但缺可调用入口 |
| 待条件节点 | 缺少条件规格 |
| 待结果节点 | 缺少结果能力 |
| 待条件结果对 | 缺少已验证的条件结果配对 |
| 可用 | 方法结构足以被任务选择并尝试执行 |
| 好用 | 方法有足够复现证据，优先级可提高 |

策略状态候选：

```text
允许自动召回
允许默认执行
允许候选查找命中
根写入原语
```

这些策略不得作为裸 bool 业务事实保存。需要持久表达时，应落为方法状态、特征状态、二次特征判断或策略关系；运行期临时缓存只能作为非权威缓存，清空后不得改变方法事实。

## 8. 方法服务依赖

方法服务是高级信息服务，只允许直接依赖：

```text
存在服务
场景服务
特征服务
二次特征服务
状态服务
动态服务
因果服务
```

方法服务不得直接依赖特征值服务。方法执行需要读取或改写目标特征状态材料时，必须经特征服务。

精确动作桥是跨高级服务只读例外：方法服务可以单向依赖 `任务服务.h` 中的 `当前任务方法选择材料`，只用于复核任务、方法、`任务方法选择 = 16` 记录关系 / 节点、顺序号 30 方法投影完整句柄 / 版本和所属生命周期完整身份。该依赖不得写任务事实、不得直接解释选择记录主信息槽位、不得把任意任务到方法引用解释为选择，也不得形成任务服务反向包含方法服务的头文件环；方法登记、规格、生命周期和动作入口语义仍由方法服务自己重读。

方法召回侧是非权威索引发布版本与完整排序批次当前性的唯一复核方。适配任务选择请求前必须调用现有只读建议来源复核入口；任务服务不反向依赖方法召回服务或方法召回索引。索引发布版本只作为选择记录中的来源审计材料，不能单独授权、失效选择或替代方法登记 / 生命周期 / 规格 / 条件 / 上下文 / 动作来源的权威版本复核。

本方法系统规范确认不覆盖 MI-M2 动作入口。动作入口必须以 `规范/动作入口规范.md` 和 `规范/详细设计/动作入口详细设计.md` 的确认结果为准。方法系统第一轮不得把动作句柄、函数名、文本或控制台说明当作动作身份。
