# 7100 子规范 存在概念与实例创建最小闭环

编号：7100
日期：20260721
状态：正式子规范

## 1. 定位

本规范规定概念存在、抽象存在和实例存在的统一结构、概念归属、语言入口、实例创建、记忆扩充和可追溯归类边界。

直接上级规范：1140。

直接依赖规范：1120、1130、1150、1160、1170、1180、1190、2100、2200、2210。

## 2. 统一存在结构

存在只有一种权威身份结构：直接含稳定主键的存在节点及稳定句柄。概念存在、抽象存在和实例存在不建立平行身份类型，也不通过“存在形态”枚举互斥区分。

```text
概念存在 / 抽象存在
= 归入“存在”根抽象链概念侧的普通存在节点

实例存在
= 归属于现实场景、内部世界场景或其它合法实例场景的普通存在节点
```

“存在”是存在类信息的最高根抽象名。其它历史容器名不得成为并列最高抽象。

## 3. 概念和实例用途

概念存在首先服务两类最小闭环：

1. 作为自然语言概念词的已理解信息锚点；
2. 作为创建实例时扩充基础记忆的模板。

抽象存在还可服务分类、归纳、约束、匹配和因果模板适用范围，但它仍是普通存在节点，不是事实账本。

实例存在是现实场景、内部世界或合法运行场景中的具体对象。实例可以先于概念形成，也可以在后续归类过程中增加概念关系。

## 4. 概念归属

实例可以对应多个概念存在。海中鱼巣以具名概念归属关系承载概念集合，并提供受控索引用于快速匹配：

```text
实例存在稳定句柄
-> 一个或多个概念归属关系
-> 概念存在稳定句柄
```

第一个概念可以被投影为默认概念，但默认概念不得替代完整概念关系集合。删除或替换默认概念不得误删其它概念归属。

概念集合和索引只作为第一阶段归类入口、快速查询和模板初配依据。需要判断归类来源、置信、版本、冲突、可撤销性或是否仍然成立时，必须回读二次特征、状态、动态、来源场景和因果证据链。

置信度、来源、状态、命中特征和冲突特征不得复制到存在本体或概念集合本体中。

## 5. 唯一性

同一“存在”根抽象链概念侧中，同一规范名称、类型和父概念路径的概念存在应唯一。确保概念时必须先精确查找，命中相同身份则幂等读回，稳定键同义冲突则拒绝。

实例允许同名多实例。创建实例默认建立新身份，是否复用必须由场景、主键、来源和合法识别或匹配规则显式裁决，不能仅按名称全局复用。

## 6. 语言入口

概念词语素入口绑定概念存在；实例词语素入口绑定实例存在，并通过概念归属关系追溯概念存在。

```text
“飞机”概念入口 -> 存在 / 飞机
“B-1234”实例入口 -> 场景实例 / B-1234 -> 概念 / 飞机
```

专名是否指向实例必须由具有场景、身份或实例上下文的调用方显式确认，语素层不得猜测。一个实例可以有多个入口别名，但默认显示名称入口必须唯一。

## 7. 实例创建闭环

```text
取得一个或多个当前有效概念存在
-> 创建新的实例存在身份
-> 建立合法场景归属
-> 建立概念归属关系
-> 按顺序扩充概念基础记忆
-> 完整读回实例、场景、概念和扩充结果
```

实例创建入口必须一次性排除无效概念、错误场景、概念与实例角色混淆、重复稳定键和越权写入。结构发布后不能完整读回属于内部逻辑错误。

## 8. 概念记忆扩充

第一阶段至少可以把概念上的抽象特征定义实例化为实例特征槽位。需求、任务、方法、能力、用途和行为模板不是概念归属关系；后续需要时，只能由各自领域规则作为创建实例的扩充内容建立。

多个概念的特征扩充按概念加入顺序合并。重复判断依次使用相同抽象特征引用、相同特征类型、相同规范名称和类型。

合并规则：

1. 实例已有特征优先，不被概念默认值覆盖；
2. 多个概念提供同一特征时，先加入的概念优先；
3. 实例已有槽位无值且概念有合法默认值时可以补值；
4. 第一阶段只补缺，不进行动态继承、冲突集或复杂置信治理。

候选窗口、统计、观察时间和其它运行痕迹不得从概念复制到实例。指向场景实例、临时动态、当前任务或运行对象的引用不得作为概念默认值复制。

## 9. 实例在先、概念在后

实例可以先独立存在，概念也可以预先建立。后续归类过程可以为已有实例追加概念关系，并按明确授权选择是否补充缺失记忆。

这种认识顺序不禁止工程预置常用概念，但预置概念仍只具有词义锚点、模板、分类和约束权力，不能因为预置就成为现实事实。

## 10. 集合表达

业务集合优先由概念关系、特征和查询条件表达，不建立“集合存在形态”。例如可用机位由“概念包含机位、占用状态为空闲、可分配状态满足”联合查询。

只有集合自身需要稳定命名、调度或因果身份时，才可以建立普通存在节点；成员关系仍用正式关系或特征表达。

## 11. 抽象存在边界

抽象存在本体只保存身份、上位概念、下级概念或实例索引以及可选定义场景。它不得保存动态历史、安全评估、伤害记忆、当前风险、当前价值、是否可控或是否满足需求。

```text
抽象存在：表达“是什么类”
抽象特征：表达“可观察或比较什么”
因果模板：表达“为什么可能影响”
具体状态、动态、场景和因果证据：表达“实际发生什么”
评估方法：表达“本轮如何判断”
```

## 12. 禁止规则

1. 不得新增抽象存在平行身份类型、通用附属记录或存在形态枚举。
2. 不得把能力、需求、任务、方法、用途或行为模板塞入概念归属关系。
3. 不得因同名自动复用实例。
4. 不得把概念索引或镜像特征当作可追溯归类证据的替代。
5. 不得把动态、安全、风险和价值账写入抽象存在本体。
6. 不得让语素入口直接确认实例或概念归属。

## 13. 验证边界

验证至少覆盖统一存在结构、概念唯一、实例同名多身份、多概念归属、实例创建、特征补缺、运行引用不复制和归类证据回查。
