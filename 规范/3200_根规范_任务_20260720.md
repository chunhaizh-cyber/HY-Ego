# 3200 根规范 任务

编号：3200
日期：20260720
状态：正式根规范

## 1. 根定位

任务是对一个确定需求目标的授权承接对象。任务负责在自己的稳定身份下保存来源需求、目标、授权、运行上下文、筹办、方法选择、执行和结果事实；任务不是需求本身，也不是方法本身。

一个任务只承接一个确定任务目标。同一根需求可以合法产生多个不同任务；这些任务保持独立身份、独立运行态和独立筹办执行权。

## 2. 本项目承载

任务在海中鱼巣中由以下结构共同承载：

```text
任务节点及稳定句柄
任务到来源需求的正式关系
任务唯一运行态治理存在
任务运行场景关系
任务目标状态关系
任务生命周期状态及其版本、时间和历史关系
任务方法候选、选择和当前性记录
任务特征及方法特征挂载记录
任务实际结果状态和证据关系
任务授权、路径、权重和结算关系
```

“任务虚拟存在”在本项目中是任务节点内部唯一的运行态治理存在，不是世界事实存在。每个任务必须独占自己的运行态治理存在；不同任务即使来源于同一需求或选择同一方法，也不得共享。

任务身份、来源需求、目标状态、运行场景、当前方法和实际结果等已经由稳定节点或正式关系承载的事实，不得再复制成并列权威特征。任务特征只承载可特征化的治理事实和方法声明事实。

## 3. 任务状态原则

任务状态不是单个生命周期枚举。任务当前有效状态由以下内容共同成立：

```text
结构前提
+ 必需任务特征类型集合及其值
+ 可选任务特征类型集合及其值
+ 禁止激活任务特征类型集合
+ 来源和生产者
+ 当前性与版本
```

已创建、已承接、筹办中、排队中、执行中、等待中、待重筹办、已完成、失败和取消只允许作为生命周期阶段投影。阶段投影必须能够回读对应状态特征集合；不得只写枚举值就宣称任务已经形成该业务状态。

同一阶段可以有多个不同业务状态段。相同特征类型集合取不同值、来源或版本时，也属于不同任务状态。

## 4. 任务特征原则

任务完整特征集合由三类来源形成：

```text
任务领域固定治理特征
+ 来源需求锚定的任务特征
+ 当前激活方法正式声明的任务业务特征
```

任务不得临时发明无来源的独有业务特征。方法带入的特征必须保留来源方法、方法内容版本、特征角色、激活原因、筹办轮次、是否必需和当前性。

方法切换不得删除旧方法特征和值；旧特征转为停用或历史，新方法特征经完整挂载和验证后激活。

## 5. 任务筹办原则

任务筹办是任务领域内建最小方法，由待筹办任务直接触发。它不进入普通方法召回，不创建为解决自身的子任务，也不建立独立持久化筹办回执。

任务筹办可以重复进行。每一轮必须重新读取当前任务、来源需求、目标、活动路径、授权、方法及其版本，并直接形成本轮闭合结果：

```text
可执行冻结
子目标或能力缺口承接
合法等待
目标已经完成
授权失效或取消
```

无候选方法时不得自动创建方法首，也不得返回无结构的“无法解决、退回上层”。需要等待、子目标或能力缺口时，必须同时形成对应对象、合法生产者、重新触发条件和当前状态。

## 6. 筹办并发原则

筹办并发隔离键是任务稳定身份。同一任务同一时刻只能有一个有效筹办占用和执行权；未取得者必须在方法召回前正常拒绝。

筹办执行权由任务结构中的唯一有效占用事实裁决，禁止持久化裸线程编号。若两条路径已经同时进入同一任务的召回、选择或冻结，属于内部并发规则破坏，必须追根因；不得把双筹办形成的同义选择伪装成正常成功。

不同任务可以并发筹办，即使它们来源于同一根需求或选择同一可复用方法。

## 7. 执行、完成与错误原则

执行只能使用当前有效筹办冻结。冻结后方法代际、路径、授权、条件、输入或世界事实发生变化时，旧冻结失效，任务进入新筹办轮次。

执行期才发现冻结前已经存在的缺方法、缺动作入口、缺条件结果对、目标语义不完整或任务结构缺失，属于筹办漏检和内部逻辑错误；不得包装成普通需求、等待或任务失败。

任务完成只能由任务实际结果满足任务自身目标状态证明。方法返回成功、动作已经发生、子任务完成、来源需求已满足或日志显示成功均不能单独证明任务完成。

任务失败只允许表达合法业务失败条件已经命中并形成终止证据；内部逻辑错误不得写成任务失败。任务取消必须有合法取消授权，执行中的取消还必须有动作停止证据。
