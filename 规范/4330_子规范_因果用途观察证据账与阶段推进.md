# 4330 子规范 因果用途观察证据账与阶段推进

编号：4330
日期：20260721
状态：正式子规范

直接上级规范：1190。

直接依赖规范：4010、4220、4310、4320、5160。

相关边界规范：3200、3300、5230、5300、5340。

## 1. 定位

本规范定义因果完整结构键、用途观察权威证据账、同账阶段推进和非权威统计消费边界。

用途观察只记录某一任务回合已经冻结的选择、动作、结果和正式结算如何共同发生。它不证明稳定因果，不把概率等同于效用，不把统计或学习候选等同于方法事实，也不得反写产生该观察的本轮选择、方法或动作。

## 2. 因果完整结构键

因果完整结构键由以下业务组成共同成立：

```text
身份格式版本
+ 原因动态的完整模式身份与动态规则版本
+ 结果动态的完整模式身份与动态规则版本
+ 原因 / 结果有序角色
+ 因果规则版本
```

每个动态模式身份必须能够回读目标主体、目标特征、变化前状态、变化后状态、来源动作是否存在及其稳定身份、动态规则版本，以及适用时的二次角色。原因与结果不得排序后交换；交换两种角色形成不同结构键。

两个结构键只有在格式、原因完整身份、结果完整身份和因果规则版本全部相同时才相同。稳定哈希只用于选择候选桶；哈希命中后仍须逐项全字段比较。哈希碰撞、名称相同、概念相同、节点类型相同或单个概率相同均不得合并事实。

身份材料不完整、原因结果非法相同或版本不在显式兼容表时，不形成可召回结构键；不得以默认值、宽松版本判断或哈希值补齐。

## 3. 用途观察身份与不可变核心

每条用途观察以“完整任务句柄 + 非零任务序号”形成唯一业务身份。任务序号是任务领域正式发布且可回读的本任务回合身份分量，只在该任务内部解释；裸线程编号、时间戳、日志序号、冻结哈希和因果哈希都不能替代它，也不得把任务序号解释为方法代际。

每条已发布用途观察使用 4010 锁定的节点类型 15 承载其长期权威账项，并只由用途观察业务服务经专用数据操作创建。节点 15 不等于稳定因果，不作为长期语素入口，也不得由通用节点写入方按类型名称自行创建。

观察不可变核心必须通过稳定业务角色完整关联：

```text
来源需求
+ 任务方法选择记录
+ 来源方法
+ 实际动作入口和动作动态
+ 发生场景
+ 作用主体
+ 目标状态与实际状态
+ 同源动态证据
+ 观察形成时采用的因果完整结构键
+ 观察前冻结的机会数、正结果数、负结果数、Q10000 及其规则版本
```

观察前概率材料只保存当时证据快照，用于解释和后继统计，不裁决效用、因果稳定或方法好用。完整请求、回执、日志、显示文本、缓存、统计结果以及目标 / 实际状态值的第二权威副本不得进入观察本体。

## 4. 用途观察证据角色关系

用途观察的权威业务角色使用关系类型 18，源必须是节点类型 15 的完整当前句柄。顺序号固定为：

| 顺序号 | 角色 | 允许目标 |
| --- | --- | --- |
| 1 | 任务 | 本观察身份中的任务 |
| 2 | 需求 | 来源需求 |
| 3 | 选择记录 | 任务方法选择记录 |
| 4 | 方法 | 来源方法 |
| 5 | 动作 | 动作入口方法 |
| 6 | 场景 | 发生场景 |
| 7 | 主体 | 作用存在 |
| 8 | 目标状态 | 状态 |
| 9 | 实际状态 | 状态 |
| 10 | 实际动态 | 动态 |
| 11 | 原因被改变目标 | 原因动态合同允许的完整当前目标 |
| 12 | 原因来源动作 | 动作入口方法 |
| 13 | 结果被改变目标 | 结果动态合同允许的完整当前目标 |
| 14 | 结果来源动作 | 动作入口方法 |
| 15 | 正式结算状态 | 5160 已发布结算记录中的状态；只在最终阶段存在 |
| 30 | 尝试未达成阶段 | 与角色 9 相同的实际状态 |
| 31 | 待结算阶段 | 与角色 9 相同的实际状态 |
| 32 | 最终已结算阶段 | 与角色 15 相同的正式结算状态 |

角色 1—14 是不可变基础证据；角色 15 只在最终已结算阶段形成。每条当前观察必须在 30、31、32 中恰有一个有效阶段角色。唯一允许的阶段结构变更是把 31 失效并在同一原子提交中增加 15 和 32；30、32 不得转换，31 不得转回 30。

关系 18 的有效唯一键是“关系类型 + 完整源观察句柄 + 顺序号”。同源同角色不得存在第二条有效关系，不同角色允许复用同一目标；因此 9 与 30 / 31、15 与 32 必须能够共存。未知顺序号、错误端点、阶段零条或多条、非法阶段转换必须在写前拒绝；前置通过后的同类异常属于内部逻辑错误。通用关系重挂、删除或普通重复键不得绕过本专用角色合同。

## 5. 形成顺序与写入权

生产顺序固定为：

```text
任务取得筹办执行权并冻结方法选择、内容版本和用途来源
-> 执行正式动作并形成动作动态与结果状态
-> 任务结果按任务目标收束
-> 用途观察业务服务按任务结果形成尝试未达成或待结算账项
-> 需求满足时由 5160 形成正式结算记录
-> 用途观察业务服务把同一待结算账项推进为最终已结算
```

尝试未达成不进入需求结算；已经具备正式结算记录的最终结果可以在一次提交调用链中直接形成最终账项，但语义顺序仍必须先有任务结果、再有需求正式结算、最后形成用途观察最终阶段。

用途观察业务服务是观察身份、不可变核心、阶段和证据关系的唯一业务写入方，并经专用数据操作完成原子发布。任务、方法、结算、统计、学习、线程、显示层和外部调用方不得直接改写观察账。

第 N 回合结果不得改变第 N 回合已经冻结的任务选择、方法内容版本、动作入口或动作事实。N 只表示业务发生顺序，不得拿任务序号冒充方法代际。

## 6. 阶段负载

同一用途观察身份只允许下列三类阶段负载：

| 观察阶段 | 任务目标结果 | 需求结算 | 必要证据 |
| --- | --- | --- | --- |
| 尝试未达成 | 未达成并已由任务领域收束 | 无正式结算 | 目标、实际状态和本轮动作 / 动态证据 |
| 待结算 | 任务目标已达成 | 尚未发布正式结算 | 目标达成判断及同源结果证据 |
| 最终已结算 | 任务目标已达成 | 5160 的正式 `已满足` 记录 | 完整正式结算身份、目标、实际状态和证据一致性 |

阶段负载必须保留任务目标结果、需求结算状态、观察阶段、规则版本和阶段证据。最终阶段必须消费 5160 已发布且与不可变核心一致的正式结算记录；普通 bool、任务完成枚举或结算 DTO 文本不得替代。

## 7. 同账推进、幂等与冲突

新候选和既有记录必须先分别通过格式、身份、不可变核心、阶段负载和证据合法性验证，再执行同账比较：

```text
异完整任务句柄或异非零任务序号 -> 不同观察账项。
同身份 + 同不可变核心 + 同阶段负载 -> 同账幂等，读回既有记录。
同身份 + 同不可变核心 + 待结算推进为合法最终已结算 -> 同账允许推进。
同身份 + 异不可变核心 -> 同账冲突，零结构变化。
尝试未达成或最终已结算发生阶段变化、最终逆向待结算、待结算转尝试未达成或其它转换 -> 同账冲突。
同一身份出现多个已发布当前记录 -> 内部不一致，停止并追根因。
```

待结算推进只增加最终结算阶段证据，不得修改不可变核心。已发布既有记录自身非法、最终候选缺少正式结算、结算材料与核心矛盾或前置通过后写入 / 读回 / 发布不一致，均属于内部逻辑错误，不进入幂等或冲突比较。

因果键召回、观察身份和观察同账是三层不同裁决：因果哈希只能找到候选；观察身份只由任务及其回合裁决；只有身份相同才比较核心与阶段。任何一层都不得替代下一层。

## 8. 非权威统计与学习消费

统计只能从已发布、结构合法的用途观察重建，并遵守：

1. 按完整因果结构键、适用范围和统计规则版本分组；
2. 尝试未达成与最终已结算分栏，待结算不进入效果统计；
3. 样本数、正负结果数和派生比例分别保留，不压成单一好坏分、置信度或效用；
4. 同一输入集合和规则版本产生确定排序、确定去重和确定输出；
5. 清空、重建或丢弃统计不改变观察、任务、需求、因果或方法事实；
6. 统计、概念标注和候选排序只能作为 5340 的学习材料，不能自动晋级方法或创建新代际。

概念映射只能形成零、一或多项非权威标注。单项映射也不得替代完整因果结构键；多项不得按名称、类型、哈希或首项猜选。

## 9. 非成功收口

| 场景 | 分类 | 收口 |
| --- | --- | --- |
| 原始任务回合、完整因果键、冻结来源或结果材料缺失 | 逻辑内返回 | 零结构变化，不补默认值 |
| 格式或规则版本不受支持 | 逻辑内不支持 | 不猜测兼容，不降级读取 |
| 因果哈希未命中或桶内无完整相同键 | 正常召回未命中 | 不形成观察身份或同账结论 |
| 不同任务 / 回合身份 | 正常分流 | 建立或读取各自账项 |
| 同账同义或合法待结算推进 | 正常成功 | 幂等读回或发布阶段推进 |
| 同账异核心或非法阶段转换 | 逻辑内冲突 | 零结构变化 |
| 已发布材料非法、多当前记录、正式结算矛盾或事务结果异常 | 内部逻辑错误 | 停止依赖路径并追根因 |

## 10. 验证

验证至少覆盖完整结构键有序角色、版本拒绝、哈希碰撞后全字段比较、观察身份分离、不可变核心、三阶段合法形状、同义幂等、待结算单向推进、异核心与逆向冲突、多当前记录追根因、统计分栏与可重建，以及本轮结果不能反写本轮选择和动作。
