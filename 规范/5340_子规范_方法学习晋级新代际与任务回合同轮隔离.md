# 5340 子规范 方法学习晋级新代际与任务回合同轮隔离

编号：5340
日期：20260721
状态：正式子规范

直接上级规范：3300。

直接依赖规范：4330、5230、5300、5330。

相关边界规范：1190、3200、4310、5160。

## 1. 定位

本规范定义从权威用途观察形成非权威方法学习候选、经独立晋级授权发布不可变新方法代际，并保证新代际不反写产生它的任务回合。

本链只允许单向推进：

```text
已发布用途观察
-> 可重建统计和确定观察组
-> 非权威学习候选与确定差异
-> 独立晋级复核和授权
-> 方法领域发布不可变新代际
-> 后续任务筹办重新召回、选择和冻结
```

统计、学习候选、线程或一次成功都不能直接证明方法好用、修改现有方法、创建任务或授权动作。

## 2. 方法内容快照与观察组

学习候选必须以动作执行前冻结的方法完整内容快照为来源。快照至少包含：

```text
来源方法稳定身份
+ 独立方法内容版本
+ 确定排序的全部条件规格
+ 确定排序的全部结果规格
+ 完整动作入口规格
+ 输入 / 输出场景规格
+ 来源生命周期和适用范围
```

条件或结果规格各自的来源版本、任务选择时生命周期、任务回合编号和日志版本都不能替代方法内容版本。

来源观察必须来自 4330 已发布权威账，按完整观察身份确定排序、去重，并逐项复核身份、阶段、方法内容版本、适用范围和当前性。待结算观察不得进入效果差异；冲突、非法或过期观察不得由统计层静默挑选或修补。

## 3. 学习候选性质与差异合同

学习候选是值式、非权威、可清空重建的材料，只能由学习候选服务从完整方法快照、确定观察组和 5330 允许的学习能力形成。候选必须保留来源方法、内容版本、观察组完整身份、候选规则版本、适用范围和确定差异。

差异只能按正式方法结构角色和完整规格裁决：

| 差异 | 裁决 |
| --- | --- |
| 新增 | 新规格存在，来源版本没有对应正式角色规格 |
| 保留 | 正式角色和完整规格相同 |
| 移除 | 来源规格存在，候选明确证明新版本不再具有 |
| 替换 | 同一正式角色存在唯一旧规格和唯一新规格，且完整内容不同 |
| 未知 | 材料不足，无法确定新增、移除或替换 |
| 冲突 | 同一角色多映射、同身份异内容或正式材料互相矛盾 |

正式角色、方法内容版本或完整观察组不足时必须输出未知或冲突，不得按概率、单一分数、哈希、名称、任务序号或向量首项猜测差异。

尝试学习、定向学习和练习继续服从 5330：它们只形成各自允许的候选、观察或缺口，不能越权创建方法首、动作入口、稳定条件结果对或新代际。

## 4. 独立晋级复核与授权

晋级必须由独立晋级服务复核，至少同时确认：

1. 来源方法身份和内容版本仍是候选声称的版本；
2. 来源观察仍有效，统计能够从同一观察集合确定重建；
3. 候选完整，全部差异可确定，没有未知或冲突；
4. 任务或治理对象提供了明确、当前且范围匹配的晋级授权；
5. 晋级规则版本受支持，安全、禁止项、动作入口和验证合同通过；
6. 同一来源版本、观察组和候选未被同义重复消费，也不存在异义晋级冲突；
7. 新代际的全部条件、结果、动作和适用范围可以形成自足的完整方法结构。

晋级服务只输出强类型晋级授权或拒绝，不直接写仓库、方法关系、生命周期或当前选择。统计线程、用途观察服务、学习候选服务和任务执行体均不得代替晋级服务授权。

材料缺失、来源版本漂移、授权失效、规则不支持、未知差异或冲突必须在发布前结构化拒绝；前置通过后出现候选自相矛盾、重复当前授权或发布读回异常，属于内部逻辑错误。

## 5. 不可变新代际

只有方法领域服务可以消费有效晋级授权并发布新代际。新代际必须同时具有：

```text
新的稳定方法身份
+ 来源方法 / 父代的正式代际关系
+ 独立方法内容版本与代际身份
+ 完整条件、结果、条件结果证据和动作入口
+ 适用范围、生命周期、验证和来源观察关系
+ 发布版本与当前性
```

新代际是可独立召回和完整复核的方法事实，不是差异补丁、统计行或旧方法上的可变覆盖。来源方法及其历史证据继续保留审计；不得原地修改来源方法身份、内容版本或历史运行账来伪造晋级。

父方法、前置方法、后续方法和任务序号只表达各自业务关系，不能充当父代、代际号或新方法身份。发布新代际也不自动使其可执行；后续召回仍须按 5300 回读当前生命周期、完整规格、动作入口、条件和风险。

## 6. 任务回合同轮隔离

任务回合在动作前按 5230 冻结任务、筹办轮次、方法身份、方法内容版本、动作入口、条件、输入、授权、路径和验证合同。该冻结一旦进入执行，本轮形成的观察、统计、学习候选、晋级授权和新代际都不得：

- 改变本轮已冻结的方法或动作；
- 事后替换本轮选择记录或结果归因；
- 把新代际结果回写成来源方法本轮已有能力；
- 在本轮执行中绕过重新筹办直接使用新代际；
- 以同义候选或相同结果把两代方法合并为一项事实。

新代际只对后续任务回合可见。后续回合必须重新取得该任务的筹办执行权，权威重读方法登记和内容版本，重新召回、复核、选择并形成新的执行冻结。若相关方法版本在旧冻结执行前已经变化，旧冻结按 5230 失效，而不是就地切换到新代际。

不同任务可以在各自回合并发复用同一来源方法或后继代际；隔离键仍是各任务稳定身份及其正式回合，不得因共享方法而错误串行化任务。

## 7. 幂等、冲突与恢复边界

同一来源方法版本、同一确定观察组、同一候选规则和同一完整差异形成学习候选稳定键：

```text
同键同义 -> 候选或晋级请求幂等读回。
同键异差异、异授权范围或异目标代际 -> 冲突，零方法结构变化。
同一父代和代际身份出现多个已发布当前方法 -> 内部不一致，停止并追根因。
```

学习候选、统计和召回索引可以清空重建；晋级授权和已发布方法代际是权威业务事实，必须保留来源与版本。恢复材料只能作为候选重新准入；恢复过程不得自动晋级、自动发布新代际、自动选择方法或执行动作。

## 8. 写入与消费权

| 环节 | 唯一业务权力 | 禁止 |
| --- | --- | --- |
| 用途观察 | 4330 的用途观察业务服务 | 不写方法结构 |
| 统计 / 学习候选 | 统计与学习候选服务 | 不形成晋级事实 |
| 晋级授权 | 独立晋级服务 | 不直接写方法仓库 |
| 新代际发布 | 方法领域服务及专用数据操作 | 不改任务冻结或来源方法历史 |
| 后继选择 | 取得任务筹办执行权的任务领域 | 不由学习或方法服务代选 |

## 9. 验证

验证至少覆盖：

- 完整方法内容快照与规格来源版本、任务序号分离；
- 观察组确定排序、去重、当前性复核和待结算排除；
- 新增、保留、移除、替换、未知和冲突六类差异；
- 候选非权威、独立晋级授权和方法服务唯一发布；
- 新代际使用新稳定身份、保留父代且不原地覆盖；
- 本轮结果不改变本轮冻结，新代际只在后续回合重新筹办后可见；
- 同义幂等、异义冲突、多当前代际追根因和恢复不自动晋级。
