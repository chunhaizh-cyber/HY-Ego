## 2. 本能方法自身环境

1. 本能方法是自我的最小可执行动作；进入核心逻辑前，必须先确保自己作为方法的正式结构完整存在。
2. 自身环境至少必须能解析方法稳定身份、动作入口、方法运行宿主、条件规格、结果规格、本轮运行宿主和长期运行账。只存在函数、方法头或动作句柄时只能处于待补齐状态，不得作为叶子任务可执行动作。
3. 方法条件规格是参数设置、实参匹配和值域判断的权威；输入包只提供本轮实参来源，不得替代条件规格。
4. 筹办可以用方法头和能力签名召回、预筛；执行必须回到条件规格完成本轮绑定，条件匹配成功后才可进入核心逻辑。
5. 方法首节点、缓存句柄和名称都不能代替规格确认；执行前必须重新解析并复核当前方法、生命周期和规格版本。
6. 本轮实际输入、可执行参数、临时结果和输出回执必须与方法条件 / 结果规格容器分离。
7. 确保当前学习方法自身环境时不得修改目标方法；当前方法环境与目标方法环境、当前方法规格与目标方法规格、当前方法运行账与目标方法运行账必须分开。
8. 目标方法运行次数、成功次数、连续成功、状态和稳定性只能由目标方法自身的正式执行 / 归并路径更新。
9. 自身关键结构缺失是方法系统自举失败或逻辑错误；规格尚未就绪、当前上下文不可用、参数绑定失败和条件不满足必须分开返回，不得合并成目标方法缺口或静默成功。

## 3. 本能函数固定阶段与输入输出

本能函数必须按固定阶段组织：

```text
确保自身方法环境
-> 绑定并复核本轮实际输入
-> 执行或匹配核心逻辑
-> 形成输出结果、运行事实和结构化回执
```

1. 确保阶段不得因为节点已经存在而提前返回；绑定或执行失败仍必须在已经形成有效本轮运行宿主时进入输出阶段。
2. 缺输入不得猜测、默认补齐、按名称推断或继续执行；缺输出不得把函数运行成功解释为任务完成或需求满足。
3. 执行语义必须来自特征类型、值域、区间、抽象特征关系、限制包和方法能力包，不能来自方法名、动作名、函数名、语素或显示文本。
4. 可召回方法必须有正式能力规格。召回只回答候选，不执行方法、不生成最终实参；执行时不得用上位抽象能力替代最终具体输入。
5. 输入条件、搜索限制、候选范围、方法搜索根、缓存身份和来源任务必须可分辨；缓存只作运行期加速，不能成为长期任务语义主键。
6. 改变特征值等底层先天逻辑不能作为任意目标的通用语义候选；必须由能够解释目标变化的正式语义方法调用。
7. 执行前值必须从目标宿主的实例特征当前值读取；抽象特征只定义维度、范围和细化链，不承载实例当前值。
8. 执行差值或目标执行后值必须有明确来源并满足约束；二者同时存在时必须相互一致。缺省补零、从输入包伪造当前值、把“不等于零”压成单一区间或发生整数溢出均不得继续执行。
9. 功能结果、执行状态、结构写入结果、动作动态、方法运行账和任务 / 需求结果必须分账。一个混合返回结构不得同时裁决这些语义。
10. 海中鱼巣允许把鱼巢裸指针返回替换为稳定句柄和强类型值式回执，但必须保留“返回本轮运行身份、具体结果另由输出结构承载”的业务分离。

## 4. 运行事实、验证与统计

1. 每次执行必须形成可追溯的本轮运行事实，至少包含方法、输入、输出、开始 / 结束时间、执行摘要、失败原因和来源任务关系。
2. 动作执行成功后只产生条件结果样本和运行账；执行阶段不得直接宣布结果符合、方法可用、条件结果对稳定或需求满足。
3. 条件结果样本必须经过实际结果验证和方法归并入口，才可以更新符合 / 不符合计数和稳定性。
4. 方法长期运行账可以记录次数、动作执行成功 / 失败、累计 / 平均耗时和最近运行身份；本次执行前后值不得成为方法本体固定属性。
5. 返回成功只证明执行器完成了其操作，不证明现实结果达到目标。任务完成、需求满足和服务 / 安全结算必须另读实际状态与正式证据。

## 5. 练习、学习与方法补齐

1. 练习不是从零生成方法结构。只有目标方法已有动作节点、可调用执行入口和可引用条件结果结构时，练习才可完善参数、样本和稳定性统计。
2. 只有方法头、缺动作、缺执行入口时，练习只能输出结构化缺口、阻塞阶段和建议后继能力；不得自建动作、方法头、伪造试运行或直接创建需求。
3. 随机练习可以在正式限制范围内生成未出现过的候选实参，但必须标记未验证；不能用随机值补普通执行缺参。
4. 候选实参、候选区间和候选条件结果对只有在实际执行和多次验证支持后才能升级；单次成功、执行返回码或文本评价不得稳定化。
5. 学习入口必须区分当前学习方法、目标方法、学习材料、练习执行和目标方法归并，不得把两套运行账、虚拟存在或条件 / 结果场景混用。
6. 学习缺口只能形成候选、结构化回执或治理输入草案；是否创建需求、任务、方法或动作仍由对应领域正式入口裁决。
7. 方法执行前自检发现结构缺失时不得在执行阶段补造；受 `FISH-SPEC-CONFLICT-003` 影响的“执行期结构缺口是否直接形成原子需求”继续冻结。
