# 6350 子规范 双目相机外设独占观察线程

编号：6350
日期：20260720
状态：正式子规范

## 1. 定位

本规范规定双目相机作为高频感知外设的独占观察线程、内部材料层、三种外显观察运行模式、报告质量和对自我暴露边界。

上级规范：0050、1130—1190、2100、3200、3300、6300—6340。

## 2. 独占观察线程

双目相机应作为独立外设控制域独占观察线程，长期管理采集、校正、对齐、滤波、分割、跨轮关联、重捕获、质量诊断、配置学习、缓存和报告生产。

该线程是一只持续供料的“眼睛”，不是理解世界的主体。它不得直接成为存在、场景、需求、任务、状态、动态或价值结算的写入者，也不得成为方法动作动态的来源。

相机内部能力不要求逐项建成可召回方法。RealSense、OpenCV、SLAM、TSDF、OctoMap、VOS 和其它工程算法只服务外设内部稳定供料，不改变自我侧正式承接对象。

## 3. 材料层级

相机线程必须区分：

```text
L0 原始彩色、深度、点云和采集元数据
-> L1 单轮校正、滤波、分割和簇材料
-> L2 跨轮关联、质量与稳定性材料
-> L3 可交付的稳定外设观察材料和报告
```

L0、L1 只用于缓存、诊断和回查。单轮质量通过不等于跨轮稳定。只有来源、对齐、深度有效性、范围、质量、时效和跨轮稳定合同同时成立的 L3 材料，才允许由任务域承接。

L3 仍不是已确认观察存在。相机内部轨迹号、帧号、簇号、ROI 和掩码句柄只作材料定位，不能成为世界存在身份。

## 4. 三种外显观察运行模式

相机对任务域只公开三种高层观察运行模式：

| 模式 | 直接回答 | 主要报告 |
| --- | --- | --- |
| 逐簇识别 | 当前范围有哪些稳定观察像素簇、哪些区域未解释或不可观测 | 观察像素簇、簇掩码、局部图、稳定特征和质量摘要 |
| 扫描变化 | 已解释范围相对基准有哪些新增、消失、移动、尺寸或风险候选变化 | 变化候选、覆盖范围、未覆盖解释、质量和时效 |
| 目标跟踪 | 指定目标材料是否仍可复现及其新空间位置 | 跟踪状态、空间坐标、丢失原因和重捕获候选 |

三种模式不是相机唯一全局状态。实现必须允许多个等待项或订阅项按优先级共享采样预算。工作模式可以另表达曝光、分辨率、深度质量、低延迟或指定 ROI 等物理配置，但不得用来代替观察运行模式。

ROI、补观察、质量诊断、局部细分、重捕获、稳定特征取值和配置学习只能是等待项参数、报告字段或内部策略，不得新增为第四类外显模式。

## 5. 等待项和调度

相机等待项至少关联：

- 来源需求或任务；
- 目标范围或目标存在材料；
- 期望特征类型集合；
- 最大报告年龄和质量要求；
- 优先级、截止时间和是否允许缓存；
- 可选 ROI、掩码、目标摘要和内部策略提示。

高优先级安全跟踪和服务目标跟踪可以抢占采样预算；已解释场景的扫描变化低频保留；无等待项命中的高频材料只缓存、摘要或丢弃。已完成区域不得无条件重复完整观察。

若所有可执行观察改善路径已用尽，且不可观测原因已经明确，可以结束本轮，不得无限重试。

## 6. 报告内容

逐簇识别报告至少包含：

- 观察范围、材料集和时间；
- 观察像素簇集合和簇 ID 掩码；
- 未解释、低置信和不可观测区域掩码；
- 每簇平面轮廓、彩色与深度局部图的受控引用；
- 每类特征的稳定、降级可用、不稳定或不可得状态；
- 质量、时效和不可得原因。

扫描变化报告至少包含基准版本、本轮覆盖范围、新增或变化候选、高风险未知区域、未覆盖原因、报告年龄和质量。没有历史基准时只形成当前材料，不得报告状态变化。

跟踪报告至少包含目标种子、观察窗口、跟踪状态、最近观测年龄、新空间坐标或空信息、特征复现离散度、丢失原因和重捕获候选。跟踪目标必须已经有可复验稳定身份材料。

## 7. 掩码、局部图和质量

局部图是人工复核、递归细分、跨轮比较和配置学习证据，不是存在真值。掩码直接参与簇生成、像素归属、子轮廓提取、三维建模和可观测单位统计，必须用受控句柄和状态承载，不得降为显示附件。

相机可以提交掩码和局部质量，但可观测单位对应摘要、存在对应闭合、存在候选和是否确认存在仍由自我侧方法裁决。

不得用一个整轮“优质”或“不可用”状态覆盖所有局部对象和所有特征。每个候选、区域和特征类型必须分别表达稳定、降级、不稳定或不可得；不可得时不得填默认值，必须保留具名原因。

## 8. 自我侧承接边界

```text
相机线程提交 L3 报告
-> 任务域按等待项、任务和时效承接
-> 自我侧观察方法形成观察像素簇和存在候选材料
-> 识别、扫描或跟踪按各自规范读取正式材料
-> 领域服务提交特征、状态、动态或场景事实
```

相机线程不直接生成存在候选，不确认世界存在，不写当前状态，不创建需求，也不直接满足任务。观察、识别、扫描和跟踪不得互相直接调用，缺口经需求统一入口和任务筹办重新路由。

## 9. 私有学习边界

相机线程可以学习稳定采集配置、阈值候选、滤波参数和重捕获策略，并优先用于后续相似等待项。学习输出只能是带来源、样本范围和版本的配置候选或质量摘要。

私有学习不得改写需求、任务、存在、场景或方法能力事实，也不得把外设内部结论写成世界真值。

## 10. 禁止规则

1. 不得把单轮簇或外设轨迹 ID 作为已确认存在。
2. 不得把“帧内无遗漏”解释成语义识别或世界对象闭合。
3. 不得把视觉背景作为存在归属类型。
4. 不得把补观察、质量诊断和配置学习扩张为外显运行模式。
5. 不得用模拟材料、夹具或显示结果证明真实相机链路。
6. 不得把不稳定或不可得特征伪造成稳定特征。
7. 不得由相机线程直接写任务、世界事实、状态动态或价值。

## 11. 验证边界

验证至少覆盖独占控制、L0—L3 分层、三模式报告、局部质量、时效与抢占、材料身份隔离、任务承接和真实连续样本。

缺少真实 D455 连续材料和正式自我侧承接时，不得宣称跨轮身份稳定、存在确认、语义识别或场景认知完成。
