# 2300 根规范 文章

编号：2300

日期：20260720

状态：正式根规范

## 1. 根定位

文章是文档级语言材料身份和解析根，承载原始文档、来源、版本及有序段落组成。文章是语言材料和解析锚点，不承载业务结论。

## 2. 正式结构承载

正式文章至少由以下结构共同成立：

```text
4010 节点类型 16 与文章身份命名域形成的稳定身份
+ 标题、原始文本、作者、来源等语言材料
+ 语言归属、材料格式和来源版本
+ 当前发布版本及历史版本关系
+ 文章到段落的有序组成关系
+ 可选的文本世界场景解析锚点关系
+ 身份、来源和顺序索引
```

不得用语素、基础信息、场景或显示节点冒充文章。文章复用 4010 的通用节点物理结构，但必须使用文章节点类型、文章身份命名域、正式文章角色材料和语言专用服务边界。

## 3. 组成、顺序与版本

1. 一个段落版本必须且只能归属一个文章版本。
2. 段落顺序由有序关系、顺序号和版本共同表达，不依赖内存指针链、容器当前位置或文件行号。
3. 同一文章内段落顺序号必须唯一；重复段落文本仍是不同组成项，不得按文本去重。
4. 已发布文章修订必须形成新版本并保留来源关系；不得原地改写历史原文后继续沿用旧版本身份。
5. 对应文本世界场景只表示解析进入世界层的锚点；该关系本身不证明场景真实、状态成立或需求存在。

## 4. 写入方、读取方与验证

语言领域服务负责文章身份、原始材料、来源版本、段落组成和当前发布；其它领域只能读取语言材料或提交待裁决解析结果。

正式读回应互证文章角色、语言材料版本、来源、段落归属、顺序唯一性和当前发布版本。索引只用于召回，命中后必须精确回读文章身份与来源。

## 5. 业务边界与禁止事项

文章内容触发治理链时，必须经文章解析、词、语素入口及对应领域提交入口形成正式结构。

- 禁止把标题、路径、作者、来源、原文或摘要直接作为业务身份和机器结论。
- 禁止把文章内容直接写成需求、任务、动态、方法、状态或因果。
- 禁止把控制面板、日志或说明文档中的文章文字反向写成语言身份。
