# 0000 规范梳理总索引

编号：0000
日期：20260720
状态：正式规范索引

## 1. 定位

本文件是海中鱼巣机器逻辑规范集合的总入口。`规范/规范目录.md` 逐项列出权威文件；本文件规定编号分段、层级和完整性口径。

## 2. 编号分段

| 编号段 | 用途 |
| --- | --- |
| 0000—0999 | 总索引、上级总纲、编号、覆盖、冲突、完整性和通用结构规则 |
| 1000—3999 | 逻辑对象根规范 |
| 4000—4999 | 基础信息、特征、动态和因果子规范 |
| 5000—5999 | 需求、任务和方法子规范 |
| 6000—6999 | 安全、自我、观察、外设和视觉子规范 |
| 7000—7999 | 存在、语素和信息入口子规范 |
| 8000—8999 | 运行治理、内部循环和阶段验收子规范 |

## 3. 层级

```text
0050 项目通用上级规范
-> 1xxx—3xxx 对象根规范
-> 4xxx—8xxx 主题子规范
```

根规范只写对象定义、稳定身份、事实来源、写入权、生命周期、相邻对象边界和原则性禁止事项。子规范展开可机械判断的特征、值域、状态、流程、分支、错误和验证合同。

## 4. 完整性

当前正式目录登记 85 份编号规范：

- 7 份目录和通用规则；
- 22 份对象根规范；
- 56 份主题子规范。

相邻未占用编号是同族扩展预留，不代表当前对象或逻辑缺失。新增规范必须具有独立规则主题，不能只因新增代码文件、字段或显示页面而取得编号。

## 5. 使用规则

1. 正式权威入口是 `规范/规范目录.md`。
2. 0050 只写共同原则，具体规则进入对应根规范或子规范。
3. 同一对象的下级规范不得改变上级事实来源、身份、写入权、并发隔离和完成语义。
4. 数据结构统一使用海中鱼巣的节点、稳定句柄、主信息、关系、索引、特征、状态、动态和领域服务结构。
5. 实现和人读材料不得反向覆盖规范。
