# 4060 子规范 非权威缓存统计失效与确定重建

编号：4060
日期：20260721
状态：正式子规范

## 1. 定位

本规范规定非权威缓存、统计和派生观察材料的身份、来源、写入、读取、失效、清空与确定重建规则。它们只服务候选缩小、性能、去重、调度材料、诊断和人读观察，不裁决机器事实。

直接上级规范：0050。

直接依赖规范：4050。

相关边界规范：0500、4010、4050、4330、5160。

## 2. 非权威身份与准入

一项缓存或统计材料只有同时具备下列合同，才允许存在：

```text
缓存或统计类型
权威来源类型和稳定身份
来源内容版本或生命周期版本
生成规则身份和规则版本
缓存值类型
合法写入方和读取方
失效条件、清空范围和重建策略
是否允许持久化及恢复时的重新准入规则
```

上述组合构成类型化缓存命名空间。命名空间的显示字符串、日志标签、函数名或存储路径不得替代其正式身份，也不得成为权威对象身份。

新增缓存前必须证明它不复制第二套权威事实，且清空、缺失、过期或重建不会改变节点、主信息、关系、索引、特征、特征值、状态、动态、因果、需求、任务和方法的合法结果。证明不足时不得新增。

## 3. 允许用途与权威来源

允许的通用用途包括：

| 用途 | 权威来源 | 命中后的必需复核 |
| --- | --- | --- |
| 内容签名或去重候选 | 完整原始值、内容版本和生成规则 | 回读原始值或权威值式材料比较 |
| 索引命中、读取次数或最近观察 | 正式查询事件或权威事件账（若存在） | 不参与业务事实裁决 |
| 结构统计与分布 | 当前权威仓库结构 | 需要事实时回读目标结构 |
| 领域摘要或只读投影 | 领域权威记录及其版本 | 回读领域记录，摘要不反写 |
| 候选排序、复核建议或诊断 | 具名权威材料和确定规则 | 由拥有裁决权的领域服务重新准入 |

具体领域缓存的来源、值域和消费边界由对应对象规范补充。因果用途观察与方法学习材料由 4330 规定，需求结算摘要只能在 5160 的权威结算记录成立后形成；本规范不反向定义这两个领域事实。

以下材料不得作为缓存权威来源：

```text
日志、弹窗、控制台和显示文本
临时 JSON、HTML、截图或报表
线程、锁和调度器状态
未经准入的外部输入、外设报告或数据库记录
另一个无法追溯到权威结构的缓存结果
```

## 4. 写入与读取规则

缓存写入必须由拥有权威来源读取权的具名服务提交，写前复核命名空间、来源身份、来源版本、规则版本和值域。显示层、日志层、控制台、数据库投影和外部调用方不得反向写入缓存。

同一缓存身份的一次更新必须形成完整值，不得让读取方看到部分累计、混合版本或跨规则结果。计数、序号、时间和有符号累计越界时必须在改变缓存前拒绝；并发更新不得丢失已接受的同身份观察。

缓存读取必须允许以下三态而不混淆：

```text
当前有效缓存材料
缺失或已失效，允许按策略重建或重新积累
缓存内部身份、版本或结构不一致
```

前两者按领域协议返回材料或材料不足；第三种属于内部逻辑错误，不得伪装成普通未命中。缓存命中只缩小候选或提供派生材料，需要事实时必须回读权威来源。

## 5. 失效规则

出现下列任一变化时，受影响缓存必须失效或进入新命名空间：

```text
权威来源身份失效、内容变化或生命周期版本推进
组成关系、来源集合或当前性变化
生成规则、比较规则或统计口径版本变化
运行期域、恢复版本或导入来源变化
缓存声明的期限、容量代际或手动清空条件成立
领域规范规定的其它失效事件
```

旧命名空间中的材料不得冒充新来源或新规则下的当前材料。失效只改变缓存可用性，不得修改、删除、补写或失效权威来源。

缓存缺失或失效只能带来性能下降、候选减少、需要重建或诊断材料不足；不得改变同一权威输入下的业务裁决。若某业务路径离开缓存就无法正确裁决，说明缓存已经被错误提升为权威结构，必须停止该路径并治理根因。

## 6. 清空与确定重建

清空必须按完整缓存身份或具名命名空间执行，只删除非权威材料。清空前后权威结构、业务状态、任务与方法结果、需求满足和结算事实必须保持不变。

可重建投影的确定重建顺序固定为：

```text
取得当前权威来源和值式版本
-> 按命名空间声明的唯一规则计算
-> 形成新缓存候选
-> 再次复核来源身份与版本未漂移
-> 原子替换对应缓存材料
```

相同权威输入、来源版本和规则版本必须得到等价缓存结果。重建不得从旧缓存互相推导，不得根据日志、显示、名称或历史成功猜测缺失事实，也不得在重建过程中修复权威结构。

只记录运行期事件且没有权威事件账的观察统计，清空后只能确定恢复为空材料并从后续合法事件重新积累；不得宣称已经重建被清空的历史累计。若存在正式事件账，才可按具名规则从该账重放，并保留账本版本与重建范围。

来源在重建过程中漂移时，新候选必须丢弃或重试，不得发布混合版本结果。无法读取完整权威来源时返回材料不足；规范要求来源必然可读却实际不可读时，按 4050 进入内部逻辑错误。

## 7. 持久化与恢复

缓存默认可以丢弃。只有对应领域规范明确允许、命名空间包含完整来源与规则版本并且恢复后可重新准入时，才可以持久化。

从文件或数据库载入的缓存首先是恢复候选，不是当前缓存事实。必须重新验证：

```text
来源身份和生命周期仍有效
来源内容版本一致
规则版本仍受支持
值类型、范围和完整性合法
当前域允许该缓存被消费
```

任一项不成立即丢弃候选并按本规范重建或保持缺失，不得因为载入成功而反向改变权威结构。

## 8. 禁止事项

1. 禁止用缓存命中证明节点、关系、状态、动态、因果、需求、任务或方法事实存在。
2. 禁止用命中次数、最近命中、排序分数或统计多数裁决重要性、成功、满足、完成或安全结论。
3. 禁止把结构统计用于自动删除、补写、重挂或修复权威结构。
4. 禁止把缓存字段塞入主信息权威值容器，或为显示便利新增平行事实字段。
5. 禁止让一个缓存从另一个缓存推导出新的业务事实。
6. 禁止把缓存清空、重建成功、命中变化或统计报告当作业务动态来源。

## 9. 验证合同

| 验证项 | 必须证明 |
| --- | --- |
| 清空不改事实 | 清空前后涉及的权威结构和领域裁决等价 |
| 缺失不改裁决 | 关闭或移除缓存后，同一权威输入仍得到同一业务结果 |
| 命中需复核 | 缓存命中后回读权威来源，伪命中和过期命中不能成为事实 |
| 失效隔离 | 来源或规则版本变化后旧命名空间不再作为当前材料 |
| 确定重建 | 相同来源和规则得到等价结果，来源漂移不发布混合版本 |
| 事件观察边界 | 无权威事件账时清空后只重新积累，不伪造历史恢复 |
| 恢复重新准入 | 持久化候选在版本或来源不符时被丢弃且不改权威结构 |
| 内部不一致分类 | 缓存结构矛盾进入内部逻辑错误，不降级为普通未命中 |
