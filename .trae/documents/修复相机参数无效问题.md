# 修复相机参数无效问题

## 问题分析

通过分析代码，我发现相机参数无效的根本原因是：

1. **虚拟相机外设类**：已经正确设置了 `深度内参.有效 = true` 和 `相机.有效 = true`，代码逻辑正确。

2. **相机写入工作流模块**：在处理帧时会创建原始场景帧的共享指针，并将其赋值给每个存在观测的 `原始场景帧` 字段，代码逻辑正确。

3. **存在提取类**：在 `从点簇增强提取存在观测` 方法中，没有设置 `原始场景帧` 字段，这是问题所在！

4. **三维场景管理类**：在 `生成世界融合轮廓` 时，会检查 `obs.原始场景帧` 是否存在，然后检查 `帧.相机.有效` 和 `帧.深度内参.有效`，代码逻辑正确。

## 修复方案

### 方案1：修改存在提取类，确保设置原始场景帧
在 `存在提取类.ixx` 中，修改所有创建 `结构体_存在观测` 的方法，确保它们能够接收并设置 `原始场景帧` 字段。

### 方案2：修改三维场景管理类，增强错误处理
在 `三维场景管理类.ixx` 中，修改 `生成世界融合轮廓_可能回退` 方法，增强错误处理，当 `原始场景帧` 为空时给出更明确的错误信息。

### 方案3：修改存在观测结构，添加相机状态冗余存储
在 `基础数据类型.ixx` 中，修改 `结构体_存在观测`，添加相机状态的冗余存储，以便在 `原始场景帧` 不存在时也能使用相机参数。

## 推荐方案

**方案1**：修改存在提取类，确保设置原始场景帧

### 具体修改步骤

1. **修改存在提取类.ixx**：
   - 在 `从点簇增强提取存在观测` 方法中添加 `std::shared_ptr<const 结构体_原始场景帧> 原始场景帧` 参数
   - 在方法内部设置 `out.原始场景帧 = 原始场景帧`
   - 修改 `从点簇增强列表提取观测` 方法，添加相应参数并传递给 `从点簇增强提取存在观测`
   - 修改 `从父点簇增强提取子存在观测` 方法，添加相应参数并传递给 `从点簇增强提取存在观测`

2. **修改相机写入工作流模块.ixx**：
   - 在调用 `从点簇增强列表提取观测与轮廓观测` 方法时，传递 `原始场景帧` 参数

3. **修改存在提取类.ixx** 中的 `从点簇增强列表提取观测与轮廓观测` 方法：
   - 添加 `std::shared_ptr<const 结构体_原始场景帧> 原始场景帧` 参数
   - 在方法内部为每个生成的 `存在观测` 设置 `原始场景帧` 字段

## 预期效果

修复后，所有生成的 `存在观测` 都会正确设置 `原始场景帧` 字段，包含有效的相机参数和深度内参，从而解决世界融合轮廓生成失败的问题。
