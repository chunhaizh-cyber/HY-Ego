# 候产房机制指标添加计划

## 目标

为候产房机制添加 4 个指标：
1. **出生延迟**：从第一次进入候产房到创建存在的帧数/毫秒
2. **误产率**：出生后在 N 帧内被幽灵清理、或与已存在高度重叠而被合并的比例
3. **漏产率**：真实存在持续出现但一直没出生的比例（可用 trackId 或外部标注评估）
4. **重复产**：同一真实物体被生出多个存在（典型症状：候产房聚类/去重不稳）

## 实现方案

### 1. 扩展候产房簇结构

**修改点**：
- 在 `候产房簇` 结构体中添加 `首次进入时间_us` 字段
- 添加 `首次进入帧号` 字段

```cpp
struct 候产房簇 {
    std::vector<候产房观测> 观测列表;
    std::uint32_t 命中次数;
    std::uint64_t 最后更新时间_us;
    std::uint64_t 首次进入时间_us; // 新增
    std::uint64_t 首次进入帧号;     // 新增
    std::uint64_t 最后命中帧; // 最后命中的帧序号
    // 簇的特征
    Vector3D 平均位置;
    Vector3D 平均尺寸;
    std::size_t 平均轮廓块数;
};
```

### 2. 添加出生跟踪结构

**修改点**：
- 新增 `出生跟踪` 结构体
- 新增 `出生跟踪表` 存储出生的存在信息

```cpp
struct 出生跟踪 {
    存在节点类* 存在节点;
    std::uint64_t 出生时间_us;
    std::uint64_t 出生帧号;
    std::uint64_t 首次进入时间_us;
    std::uint64_t 首次进入帧号;
    std::int64_t 跟踪ID;
    bool 已被清理;
    bool 已被合并;
    std::uint64_t 被清理时间_us;
    std::uint64_t 被清理帧号;
};

std::unordered_map<存在节点类*, 出生跟踪> 出生跟踪表_;
```

### 3. 添加统计结构

**修改点**：
- 新增 `候产房统计` 结构体
- 新增 `候产房统计` 实例

```cpp
struct 候产房统计 {
    std::uint32_t 总出生数 = 0;
    std::uint32_t 误产数 = 0;
    std::uint32_t 漏产数 = 0;
    std::uint32_t 重复产数 = 0;
    double 平均出生延迟_ms = 0.0;
    std::vector<double> 出生延迟列表_ms;
};

候产房统计 候产房统计_;
```

### 4. 修改候产房管理函数

**修改点**：
1. **加入候产房**：记录首次进入时间和帧号
2. **检查出生**：计算出生延迟，添加到跟踪表
3. **清理过期观测**：统计漏产情况
4. **聚类观测**：检测重复产情况

### 5. 添加指标计算和日志输出

**修改点**：
- 新增 `计算候产房指标` 函数
- 新增 `输出候产房统计` 函数
- 在适当的时机输出统计信息

```cpp
void 计算候产房指标() {
    // 计算误产率
    double 误产率 = 候产房统计_.总出生数 > 0 ? (double)候产房统计_.误产数 / 候产房统计_.总出生数 : 0.0;
    
    // 计算漏产率
    double 漏产率 = 候产房统计_.总出生数 + 候产房统计_.漏产数 > 0 ? 
        (double)候产房统计_.漏产数 / (候产房统计_.总出生数 + 候产房统计_.漏产数) : 0.0;
    
    // 计算重复产率
    double 重复产率 = 候产房统计_.总出生数 > 0 ? (double)候产房统计_.重复产数 / 候产房统计_.总出生数 : 0.0;
    
    // 计算平均出生延迟
    if (!候产房统计_.出生延迟列表_ms.empty()) {
        double 总和 = 0.0;
        for (double delay : 候产房统计_.出生延迟列表_ms) {
            总和 += delay;
        }
        候产房统计_.平均出生延迟_ms = 总和 / 候产房统计_.出生延迟列表_ms.size();
    }
    
    // 输出统计信息
    日志::运行f("[三维场景管理] 候产房统计：总出生数={}, 误产率={:.2f}%, 漏产率={:.2f}%, 重复产率={:.2f}%, 平均出生延迟={:.2f}ms", 
                 候产房统计_.总出生数, 误产率 * 100, 漏产率 * 100, 重复产率 * 100, 候产房统计_.平均出生延迟_ms);
}
```

### 6. 实现幽灵清理和合并检测

**修改点**：
- 在幽灵清理函数中检测并更新出生跟踪状态
- 在存在合并时检测并更新出生跟踪状态

## 具体实现步骤

1. **修改候产房簇结构**：添加首次进入时间和帧号字段
2. **实现出生跟踪**：添加出生跟踪结构和跟踪表
3. **添加统计结构**：实现候产房统计结构体
4. **修改候产房管理函数**：更新相关函数以支持指标计算
5. **实现指标计算**：添加指标计算和日志输出函数
6. **集成到主流程**：在适当的时机调用统计函数

## 预期效果

- **出生延迟**：能够准确记录和统计从首次进入到出生的时间
- **误产率**：能够检测并统计出生后很快被清理或合并的情况
- **漏产率**：能够检测并统计持续出现但未出生的情况
- **重复产**：能够检测并统计同一物体被多次出生的情况

通过这些指标，可以更好地评估和优化候产房机制的性能，提高存在创建的准确性和稳定性。
