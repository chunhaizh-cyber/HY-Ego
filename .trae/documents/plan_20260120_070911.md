# 候产房观测清理问题修复计划

## 问题分析

**核心问题**：候产房观测被过早清理，导致无法达到出生阈值（连续3帧）。

**具体原因**：
1. **时间窗口过小**：候产房最大时间设置为 1000000 微秒（1秒）
2. **出生阈值过高**：需要连续3帧出现
3. **清理逻辑问题**：基于固定时间窗口清理，不考虑实际帧率

**影响**：
- 观测刚攒起来就被清掉
- 出生条件永远达不到（或极难达到）
- 系统无法稳定创建新存在

## 修复方案

### 1. 调整候产房时间窗口

**修改点**：
- 将 `候产房最大时间_us` 从 1000000（1秒）增加到 3000000（3秒）
- 理由：给予足够时间收集连续3帧观测，即使在低帧率情况下

### 2. 优化清理逻辑

**修改点**：
- 修改 `清理过期候产房观测` 函数
- 不仅基于时间，还要考虑簇的命中次数和年龄
- 对于接近出生阈值的簇，适当延长保留时间

### 3. 调整出生阈值逻辑

**修改点**：
- 考虑添加基于时间的出生条件
- 对于在较长时间内稳定存在的簇，即使帧数不足3帧也允许出生

### 4. 改进观测加入逻辑

**修改点**：
- 确保每次添加观测时正确更新簇的 `最后更新时间_us`
- 优化聚类逻辑，减少误聚类导致的观测分散

## 具体修改

### 1. 修改候产房参数

```cpp
// 将候产房最大时间从1秒增加到3秒
std::uint64_t 候产房最大时间_us = 3000000; // 3秒窗口
```

### 2. 优化清理逻辑

```cpp
void 清理过期候产房观测() {
    const std::uint64_t now_us = 结构体_时间戳::当前_微秒();
    
    候产房_.erase(
        std::remove_if(候产房_.begin(), 候产房_.end(),
            [&](const 候产房簇& cluster) {
                std::uint64_t 时间差 = now_us - cluster.最后更新时间_us;
                
                // 对于接近出生阈值的簇，延长保留时间
                if (cluster.命中次数 >= 2) {
                    return 时间差 > 候产房最大时间_us * 2; // 延长一倍时间
                }
                
                // 普通簇使用标准时间窗口
                return 时间差 > 候产房最大时间_us;
            }),
        候产房_.end()
    );
}
```

### 3. 优化出生条件判断

```cpp
// 在检查候产房出生函数中
if (cluster.命中次数 >= 出生阈值_K || 
    (cluster.命中次数 >= 2 && (now_us - cluster.最后更新时间_us) > 2000000)) { // 2秒内有2帧也允许出生
    // 处理出生逻辑
}
```

## 预期效果

- **观测保留时间**：从1秒增加到3秒，给予足够时间收集连续观测
- **出生成功率**：提高连续3帧观测的收集成功率
- **系统稳定性**：减少因时间窗口过小导致的观测丢失
- **适应性**：更好地适应不同帧率的场景

## 注意事项

- 保持时间窗口不要过大，避免内存占用过高
- 确保修改后的逻辑对高帧率场景仍然有效
- 监控系统性能，确保不会因保留时间延长而导致性能下降

## 验证方法

- 运行系统并观察候产房观测的生命周期
- 检查日志中"候产房观测过期，清理"的频率
- 验证连续3帧观测是否能够成功触发出生
- 测试不同帧率下的系统表现
