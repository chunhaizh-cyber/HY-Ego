# HY-Ego（鱼巢）

`HY-Ego` 是一个以“自我线程 + 世界树 + 任务/需求/方法 + 因果归纳”为核心的 C++20 实验工程。  
当前仓库按“仅源码”策略维护，便于代码同步、调试与二次开发。

## 仓库内容

- 主要语言：`C++20`（含大量 `.ixx` 模块）
- 解决方案：[`鱼巢.sln`](./鱼巢.sln)
- 工程：
  - [`鱼巢.vcxproj`](./鱼巢.vcxproj)：控制面板主工程
  - [`鱼巢Headless.vcxproj`](./鱼巢Headless.vcxproj)：Headless 宿主
  - [`鱼巢_外设_相机.vcxproj`](./鱼巢_外设_相机.vcxproj)：相机外设宿主

## 快速构建

推荐环境：

- Windows 10/11
- Visual Studio 2022
- MSVC v143
- Windows SDK 10.0
- x64

### 方式一：Visual Studio

1. 打开 [`鱼巢.sln`](./鱼巢.sln)。
2. 选择 `Debug|x64` 或 `Release|x64`。
3. 构建对应工程并运行。

### 方式二：MSBuild

```powershell
msbuild .\鱼巢.sln /p:Configuration=Debug /p:Platform=x64
```

## 当前架构分层

### 1. 基础数据环境

负责初始化全局基础表与环境：

- 语素
- 特征值
- 特征类型
- 语言环境

统一入口位于 [`语言环境类.ixx`](./语言环境类.ixx)。

### 2. 世界骨架

负责建立全局世界结构：

- 世界根
- 六大世界：现实 / 文本 / 想象 / 记忆 / 推理 / 虚拟
- 概念域根
- 因果根

世界树与因果树是两棵独立的树：

- 世界树承载场景、存在、特征、状态等基础世界对象
- 因果树承载抽象因果模板与因果树节点

### 3. 自我实体

当前自我实体语义为：

- `世界根 -> 现实世界 -> 自我初始场景 -> 自我`
- `自我 -> 自我内部世界`

其中：

- 自我是现实世界中的真实存在
- 虚拟世界保留作独立用途，不再充当自我内部世界
- 自我所在场景是现实世界中的当前场景

### 4. 自我线程

自我线程当前定位为：

- 消息处理线程
- 心跳推进线程
- 最基础本能函数的运行入口

自我线程负责轻量、必执行的周期逻辑，例如：

- 定时衰减服务值
- 风险安全回归
- 自我所在场景的场景安全度回归

重执行逻辑应尽量由工作线程承担，而不是长期阻塞自我线程主循环。

### 5. 因果系统

当前因果系统分两层：

- 实例因果：记录具体一次条件、动作、结果
- 抽象因果：对多次实例因果进行归纳和模板化

默认显示口径为：

```text
条件 + 动作 = 结果
```

## 当前初始化顺序

当前推荐的初始化顺序是：

1. 初始化基础数据环境
2. 初始化世界骨架环境
3. 初始化自我
4. 初始化需求 / 任务 / 方法
5. 启动自我线程与工作线程

其中：

- 世界骨架初始化负责建立世界树骨架与因果根
- 自我初始化负责建立自我相关特征类型与默认特征值

## 自我特征当前值约定

当前值统一挂在 `世界树.自我指针` 下的特征快照中读取，不再以状态列表反推。

当前关键自我特征包括：

- 安全值
- 服务值
- 物理安全
- 风险安全
- 待机状态
- 时序正向步长
- 时序反向步长
- 服务时序衰减步长

默认初始化语义：

- 安全值：`I64` 最大值的 `25%`
- 服务值：`I64` 最大值的 `50%`
- 物理安全：`I64` 最大值
- 风险安全：`I64` 最大值
- 默认步长：`1`

## 场景安全度

当前每个场景默认拥有一个特征类型 `安全度`，默认值为 `I64` 最大值的一半。

当前运行语义中：

- 只更新自我所在场景的 `安全度`
- 该变化属于现实世界场景因果
- 它表示自我对现实场景的主动赋值与持续修正

## 运行时状态与动态

当前实现中，运行时高频变化数据尽量不直接依赖临时世界树节点壳，而是优先使用主信息值存储：

- 高频运行时动态优先写入场景的 `运行时动态列表`
- 实例因果优先保存状态主信息快照
- 动态在生成因果时主要作为临时输入，不再长期持有临时动态节点引用

这样做的目的：

- 降低高频创建/删除临时节点带来的泄漏风险
- 减少悬空指针问题
- 让运行时层和长期结构层职责更清楚
