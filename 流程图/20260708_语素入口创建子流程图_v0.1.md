# 2.3 语素入口创建子流程图

更新时间：2026-07-08

## 依据

```text
海中鱼巣/领域/语素服务.h
海中鱼巣/入口.cpp
```

## 说明

本子流程表达 `语素服务::创建语素入口` 的代码逻辑。它只在对应信息节点准入通过后写结构。

## 流程图

```mermaid
flowchart TD
    A["输入：对应信息节点 / 可选主键"] --> B["2.2 对应信息节点准入"]
    B --> C{"准入是否通过"}
    C -- "否" --> R["返回无效句柄"]
    R --> R1["结构不变化：不创建主信息 / 语素节点 / 关系 / 索引"]
    C -- "是" --> D["主信息仓库创建主信息"]
    D --> D1{"主信息是否创建成功"}
    D1 -- "否" --> Z["失败收口：不返回有效入口"]
    D1 -- "是" --> E["节点仓库创建语素节点"]
    E --> E1{"语素节点是否创建成功"}
    E1 -- "否" --> Z
    E1 -- "是" --> F["关系仓库创建语素对应信息关系"]
    F --> F1{"对应信息关系是否创建成功"}
    F1 -- "否" --> Z
    F1 -- "是" --> G{"主键是否非 0"}
    G -- "是" --> H{"主键是否已存在或绑定失败"}
    H -- "是" --> Z
    H -- "否" --> H1["索引仓库绑定主键到语素节点"]
    G -- "否" --> I["跳过索引绑定"]
    H1 --> S["返回语素入口句柄"]
    I --> S
    Z --> Z1["回滚或失效已创建结构 / 登记逻辑错误或待确认问题"]
    Z1 --> Z2["不得留下可被读取为有效业务事实的半结构"]
```

## 关键边界

```text
创建入口前必须先排除不可绑定节点。
语素入口写入结构为主信息、语素节点、语素对应信息关系和可选主键索引。
本流程不解析文本、不创建概念追溯、不自动创建对应信息节点。
任一写入步骤失败时，必须失败收口，不返回有效语素入口。
主键索引按唯一主键处理；主键已存在或索引绑定失败时不得返回“入口有效但索引失败”的半结构，除非后续详细设计显式改口径。
当前图给出目标失败收口规则；若当前代码未完整实现回滚或失效，详细设计必须列为缺口。
```
