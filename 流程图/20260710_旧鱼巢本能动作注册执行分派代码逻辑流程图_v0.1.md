## 说明

本图只提取旧本能动作注册、可调用检查、执行分派、动作线程和动作验证报告的代码逻辑。它不迁移旧函数指针注册表、不接真实动作线程、不接外设动作，只形成后续动作入口增强和动作验证专项依据。

## 流程图

```mermaid
flowchart TD
    A["旧本能动作链入口"] --> B["本能方法类：默认种子 / 动作名称词 / 方法首"]
    B --> C["本能动作管理类：注册动作ID和函数指针"]
    C --> D{"动作登记是否有效"}
    D -->|否| R1["逻辑内返回：拒绝无效登记材料"]
    D -->|是| E["读取方法首 / 条件场景 / 输入参数场景"]
    E --> F["本能动作类：场景模式匹配和条件参数检查"]
    F --> G{"可调用检查是否通过"}
    G -->|否| R2["逻辑内返回：不执行、不写动作动态"]
    G -->|是| H{"当前是否允许真实函数分派"}
    H -->|否| R3["逻辑内返回：输出动作注册材料候选"]
    H -->|是| I["调用旧本能动作函数"]

    I --> J{"执行结果是否进入结构承载"}
    J -->|否| X1["追根因解决：不得只靠运行码或日志证明成功"]
    J -->|是| K["候选承载：任务结果状态 / 动作动态 / 需求结算记录"]
    K --> L["动作验证报告：目标 / 预测 / 实际 / 偏差 / 置信度"]
    L --> M{"报告是否被当作权威事实"}
    M -->|是| R4["逻辑内返回：报告摘要只能转结构后使用"]
    M -->|否| N["非权威摘要或后续动作验证材料"]

    C --> O["动作线程类：注册表 / 队列 / 启停 / 异常日志"]
    O --> P{"线程是否作为动作来源"}
    P -->|是| R5["逻辑内返回：线程不是动作来源"]
    P -->|否| Q["线程只可作为后续调度壳候选"]

    K --> RV["最小读回验证：动作入口、任务方法关系、输入输出规格、动态证据、结果状态"]
    RV --> S{"读回是否一致"}
    S -->|是| T["输出动作入口后续增强证据包"]
    S -->|否| X2["追根因解决：停止后续依赖"]
```

## 关键边界

```text
1. 本图不迁移旧函数体、旧动作句柄字段、函数指针注册表或动作线程。
2. 动作身份必须由动作入口角色节点、稳定动作键和方法动作入口关系承载。
3. 线程、日志、运行码、文本动作名和摘要文本不得成为动作来源或方法成功事实。
4. 动作验证报告若进入新项目，必须转为动作动态、任务实际结果状态、需求结算记录或非权威缓存材料。
5. 真实分派、取消、超时、重试和外设动作必须另建施工计划。
```
