## 说明

本图表达一次业务写入从外部请求到仓库提交的正式权力链。原始结构事务令牌只允许存在于协调层、写入会话和仓库内部，不进入领域服务公开入口、线程消息或外部调用方。

## 流程图

```mermaid
flowchart TD
    A["外部调用方 / 线程 / 消息提交业务请求；不携带结构令牌"] --> B["应用或领域组合器识别一次跨服务业务操作"]
    B --> C["领域服务公开入口检查业务参数、身份、类型、生命周期和幂等语义"]
    C --> C1{"业务准入是否通过"}
    C1 -- "否" --> L1["逻辑内返回：拒绝材料；仓库结构不变化"]
    C1 -- "是" --> D["组合器调用已登记数据操作入口；数据操作层向执行器请求一次独占不透明会话"]
    D --> E["结构事务协调层签发独占许可并负责最终释放；共享许可只读"]
    E --> E1{"许可是否有效"}
    E1 -- "否" --> L2["逻辑内返回：未进入写入；结构不变化"]
    E1 -- "是" --> F["服务专用数据操作层接收不透明会话和类型化结构请求"]
    F --> G["四仓库复核同域独占令牌；独占结构许可持续阻塞其它线程共享读取"]
    G --> H["主信息 / 节点候选 -> 即时关系 / 索引；记录候选、关系句柄和索引主键 + 预期节点完整对"]
    H --> H1{"中途结构结果是否符合预期"}
    H1 -- "否" --> R["按本次精确写集逆序撤销：索引绑定对 / 关系句柄 -> 节点候选 -> 主信息候选"]
    R --> R1{"结构和数量是否回到前态"}
    R1 -- "否" --> X1["追根因解决：停止后续计划，不扫描修复"]
    R1 -- "是" --> L3["返回领域失败结果；不发布业务事实"]
    H1 -- "是" --> I["创建领域详细设计指定的唯一最后发布结构"]
    I --> I1{"最后发布是否成功"}
    I1 -- "否" --> R
    I1 -- "是" --> J["同一会话读回主信息、节点、关系 / 索引和领域结构"]
    J --> J1{"读回是否完整一致"}
    J1 -- "否" --> X2["追根因解决：结构已发布，不用候选撤销掩盖错误"]
    J1 -- "是" --> K["确认候选；确认只关闭撤销能力"]
    K --> M["协调层释放许可；数据操作层返回结构化结果"]
    M --> N["领域服务解释业务成功并返回稳定句柄 / 领域结果"]

    P1["无令牌兼容写 / 带令牌写 / 候选写"] --> P2["先登记调用点、生产 / 自检 / 兼容分类和替代顺序"]
    P2 --> P3["按存在场景 -> 状态动态 -> 特征 -> 语义基础 -> 需求任务方法迁移"]
    P3 --> P4["生产调用点归零后才裁决私有化、弃用或删除"]

    Q1["关系二级索引、批量读取、计数、分片"] --> Q2["架构纠偏完成后重新建立基线并按证据门控"]
```

## 非成功返回二分

```text
逻辑内返回：业务入口拒绝、许可未取得、兼容路径尚未迁移；结构不变化。
追根因解决：入口通过后仓库结果异常、撤销无法回到前态、最后发布后读回不一致、同一会话内候选与权威结构不一致。
```

## 关键边界

```text
协调层签发 / 释放许可，不解释业务。
仓库验证令牌和锁，不解释需求 / 任务 / 方法语义。
领域服务负责业务准入，不暴露或比较原始令牌。
数据操作层隐藏仓库和令牌，负责精确提交与撤销，但不裁决业务。
跨服务组合器定义一次业务事务边界，外部调用方不携带令牌。
```
