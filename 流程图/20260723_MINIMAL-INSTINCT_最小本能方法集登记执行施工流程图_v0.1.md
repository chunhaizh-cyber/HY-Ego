## 身份与边界

本图冻结 `SELF-C3 / v0.1`。它是正式施工设计图，但不证明代码已实现。方法必须有稳定身份、条件、结果、动作、内容版本和回读边界；无候选时不得自动创建方法首。

```mermaid
flowchart TD
    A["形成最小本能方法值式登记规格"] --> B{"身份、条件、结果、动作、来源和内容版本完整？"}
    B -- "否" --> R1["逻辑内拒绝：不得建立名称方法或空方法首"]
    B -- "是" --> C{"同稳定键已有方法代际？"}
    C -- "同义" --> D["返回既有不可变代际"]
    C -- "异义" --> R2["逻辑内冲突：拒绝覆盖"]
    C -- "无" --> E["SELF-C3：方法领域建立方法节点、关系和类型化记录"]
    E --> F{"完整结构和动作绑定读回一致？"}
    F -- "否" --> X1["内部错误：撤销未发布写入或隔离代际"]
    F -- "是" --> G["原子发布方法代际并重建召回索引"]
    D --> H["任务筹办按冻结条件和目标结果召回"]
    G --> H
    H --> I{"方法属于主动召回集合？"}
    I -- "否：被动结算" --> J["仅由已验证任务结果协调任务、需求与 4320 / 6100 正式价值结算入口"]
    I -- "是" --> K["冻结任务授权、方法代际、输入、场景和动作入口"]
    K --> L{"动作前置条件仍成立？"}
    L -- "否" --> R3["逻辑内拒绝：不写半结构"]
    L -- "是" --> M["执行强类型动作入口"]
    M --> N["读取正式状态、动态及必要因果证据"]
    N --> O{"实际结果达到任务冻结目标？"}
    O -- "否" --> P["返回未达成和残差；不直接完成任务"]
    O -- "是" --> Q["提交方法实际结果给 SELF-C4"]
    J --> Q
    M -. "前置合法后动作发布或读回异常" .-> X2["内部错误：停止依赖路径并追根因"]
```

## 关键边界

1. 最小集包含同步运行态、改变自我实例特征、进入待机、退出待机、重置时序、结算合法任务结果、故障收口、尝试学习、定向学习和练习方法。
2. 改变自我实例特征仅作用于已冻结自我治理目标。
3. 被动结算方法不进入主动召回。
4. 方法动作成功不等于任务完成；实际回读进入 SELF-C4。
5. 恢复不自动登记、选择、执行或晋级方法。
6. 尝试学习、定向学习和练习严格按 5330 的不同输入前提分流；4330 的待结算观察不得进入效果统计。
