语素服务入口材料总览代码逻辑流程图

更新时间：2026-07-08

说明

本图是第 2 项总览图，只表达语素服务如何把外部材料降权为语素请求、如何选择服务入口、最终向其他逻辑输出哪些入口材料。详细实现拆为 2.1 到 2.5 五张子流程图。

流程图

flowchart TD
    A["外部材料：词面文本 / 对应信息节点 / 概念节点 / 可选主键"] --> B["降权为语素请求材料"]
    B --> C{"请求类型"}
    C -- "只绑定已有信息节点" --> D["2.2 对应信息节点准入"]
    C -- "创建概念入口" --> E["2.1 文本准入"]
    E --> F["2.2 概念节点与对应信息节点准入"]
    D --> G{"准入是否通过"}
    F --> G
    G -- "否" --> R["拒绝：输出原因 / 来源追溯 / 结构不变化"]
    R --> R1["不创建主信息 / 不创建语素节点 / 不写关系 / 不写索引"]
    G -- "是" --> H["2.3 创建语素入口"]
    H --> I{"是否需要概念追溯"}
    I -- "是" --> J["2.4 追加概念追溯"]
    I -- "否" --> K["2.5 读回验证"]
    J --> K
    K --> L["输出服务材料"]
    L --> L1["语素入口句柄"]
    L --> L2["语素对应信息关系 / 绑定目标组"]
    L --> L3["语素概念追溯关系 / 概念追溯组"]
    L --> L4["可选主键索引"]
    L --> M["只读方：后续领域逻辑 / 显示层只读 / 人读观察"]
