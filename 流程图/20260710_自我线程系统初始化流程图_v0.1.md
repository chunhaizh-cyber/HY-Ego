## 说明

本图只定义自我线程启动阶段的第一轮系统初始化。初始化成功后，自我线程进入等待停止或后续治理接线的驻留态；本图不宣称自我治理 mailbox、真实任务调度、自我循环或自我苏醒完成。

## 流程图

```mermaid
flowchart TD
    A["入口构造仓库、领域服务和初始化服务"] --> B["构造自我线程与显式初始化参数"]
    B --> C{"依赖、稳定键、当前值和目标值参数是否有效"}
    C -- "否" --> L1["逻辑内返回：拒绝启动，不创建线程，不写结构"]
    C -- "是" --> D["启动自我线程"]
    D --> E["自我线程入口标记为初始化中"]
    E --> F{"首笔写入前是否已收到停止请求"}
    F -- "是" --> L2["逻辑内返回：线程正常停止，结构不变化"]
    F -- "否" --> G["调用语素初始化服务，建立基础语素信息"]
    G --> H{"语素初始化结果和读回是否符合预期"}
    H -- "否" --> R1["追根因解决：标记初始化失败，停止依赖路径，程序不得进入常驻态"]
    H -- "是" --> I["调用世界树初始化服务，建立世界根、场景和自我存在"]
    I --> J{"世界树句柄、父子关系、归属关系和坐标读回是否符合预期"}
    J -- "否" --> R2["追根因解决：标记初始化失败，停止依赖路径，程序不得进入常驻态"]
    J -- "是" --> K["调用 初始化.需求.ixx 初始化两个根需求"]
    K --> K1["经特征服务建立安全值、服务值定义和自我当前特征材料"]
    K1 --> K2["经语素服务绑定安全值、服务值概念语素和显式稳定键"]
    K2 --> K3["经需求服务和状态服务创建安全根需求、服务根需求及抽象目标状态"]
    K3 --> M{"两个根需求、目标状态和完整承接材料是否可读"}
    M -- "否" --> R3["追根因解决：标记初始化失败，停止依赖路径，程序不得进入常驻态"]
    M -- "是" --> N["一次性发布自我线程初始化结果快照"]
    N --> O{"初始化期间是否锁存停止请求"}
    O -- "是" --> P["线程正常停止并由入口收口等待"]
    O -- "否" --> Q["进入驻留等待态，等待停止或后续治理接线"]
```

## 关键边界

```text
初始化开始后的停止请求只锁存，不中断三个结构阶段，避免发布半初始化快照。
任何内部写入或读回不及预期都归为追根因解决，入口不得继续进入控制面板或常驻运行。
初始化.需求.ixx 只做领域服务编排，不直接写节点、主信息、关系或索引仓库。
两个根需求是安全根需求和服务根需求；需求目标由抽象目标状态承载，I64 只作第一轮状态值材料。
不初始化任务、当前主任务、当前主方法、本能方法、外设、SQL、缓存事实或显示事实。
```
