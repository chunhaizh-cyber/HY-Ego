## 说明

本图用于补齐旧全局共享函数、双向链表模板、可解析引用、值容器和序列化 / 深拷贝辅助的归属判断。它不是迁移旧双向链表核心结构的许可，只给后续新增公共工具或仓库内部辅助时使用。

## 流程图

```mermaid
flowchart TD
    A["旧公用函数或辅助结构候选"] --> B{"是否触碰领域结构"}
    B -->|不触碰| C["纯通用工具候选：字符串、数值、时间、格式化"]
    B -->|触碰节点 / 主信息 / 关系 / 索引| D["归属核心仓库或领域服务"]
    B -->|触碰特征 / 状态 / 动态 / 需求 / 任务 / 方法 / 因果| E["归属对应领域服务"]
    B -->|旧双向链表模板| F["核心结构排除：不得作为新项目核心容器迁移"]
    B -->|序列化 / 深拷贝| G["恢复或快照辅助候选，必须隔离候选仓库"]

    C --> H{"是否已有同类工具"}
    H -->|是| R1["逻辑内返回：复用既有工具"]
    H -->|否| I["可进入公用函数按需建立门禁"]

    D --> J{"是否绕过领域服务写结构"}
    E --> J
    J -->|是| R2["逻辑内返回：不得建立全局共享写入口"]
    J -->|否| K["归属到仓库内部辅助或领域服务私有函数"]

    F --> R3["逻辑内返回：只提取规则，不复制旧链表模板"]
    G --> L{"恢复失败是否会污染当前仓库"}
    L -->|是| X1["追根因解决：恢复必须先隔离再验收"]
    L -->|否| M["快照 / 恢复辅助候选"]

    I --> RV["最小读回验证：调用前后权威结构不变或按归属入口变化"]
    K --> RV
    M --> RV
    RV --> S{"归属和读回是否一致"}
    S -->|是| T["输出公用函数归属证据包"]
    S -->|否| X2["追根因解决：不得留下越权共享入口"]
```

## 关键边界

```text
1. 纯通用函数才可进入公用函数候选。
2. 触碰领域结构的共享逻辑必须归属对应仓库或领域服务。
3. 旧双向链表模板不作为海中鱼巣核心结构迁移。
4. 已加锁辅助只能由同仓库内部持锁路径调用，不得暴露为跨服务公共入口。
5. 序列化 / 恢复辅助必须证明失败不污染当前仓库。
```
