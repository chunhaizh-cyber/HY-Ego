## 说明

本图描述 JY-369 目标合同的收口施工顺序，不把目标写成当前实现事实。当前 #225 任务结果组合路径已使用值相等裁决并在未达成时转待重筹办，但公开 `提交独立结算` 仍可写 `未满足`，数据操作只按结论幂等，需求读取仍会从多条状态归属中选择“最近结算”，既有通用组合入口 `完成任务并独立结算`（#225 新结果治理路径之外）也会先完成任务再结算。

## 流程图

```mermaid
flowchart TD
    A["公开任务结果或独立结算请求"] --> B{"句柄、幂等键、时间戳和证据是否完整"}
    B -- "否" --> LR1["逻辑内返回<br/>第一笔写入前结构零变化"]
    B -- "是" --> C["重读需求、来源任务、目标状态、实际状态和证据身份"]
    C --> D{"正式材料是否完整且相互一致"}
    D -- "允许的材料缺失或当前性不满足" --> LR2["逻辑内返回<br/>不补默认材料"]
    D -- "权威读回内部矛盾" --> RC1["追根因解决<br/>停止后继，不用最近记录或缓存修补"]
    D -- "一致" --> E{"目标状态值是否等于实际状态值"}
    E -- "否" --> LR3["逻辑内未达成<br/>不完成既有通用组合入口任务<br/>(#225 新结果治理路径之外)<br/>不写需求最终结算"]
    E -- "是" --> F{"当前正式结算记录数量"}
    F -- "零条" --> G0{"进入独占结构会话后<br/>再次确认状态型归属仍为零"}
    G0 -- "已被并发提交占用" --> J
    G0 -- "仍为零" --> G["只形成已满足实例状态规格<br/>归属顺序号=发生时间戳<br/>同一会话写完整证据关系"]
    G --> H{"全部写入和完整读回是否一致"}
    H -- "否" --> RC2["追根因解决<br/>精确撤销未发布候选"]
    H -- "是" --> I["最后发布唯一正式结算"]
    F -- "恰一条" --> J["比较完整结构<br/>幂等主键 + 时间 + 来源任务 + 目标/实际状态 + 动态 + 可选因果"]
    J -- "完整同义" --> ID["幂等读回原结算"]
    J -- "任一字段异义" --> CF["幂等冲突<br/>结构零变化"]
    F -- "两条或更多" --> RC3["追根因解决<br/>不得选择最近记录"]
    K["既有通用组合入口<br/>完成任务并独立结算<br/>(#225 新结果治理路径之外)"] --> L["完成任务前先调用值相等裁决"]
    L --> E
```

## 关键边界

```text
1. 第一轮唯一可持久化正式结论为已满足；未满足只表示本次任务结果未达成，不是需求最终结算节点。
2. 幂等必须比较完整 `需求正式结算材料`，不能只比较结论、状态值、主键哈希或最近时间；可选因果句柄只能全零或完整有效。
3. 第二条正式结算是内部结构矛盾，不得按时间、关系编号或创建序号选择一条继续。
4. 首次写入在独占结构会话内再次确认状态型归属为零；并发占用后退出写入并重读，按完整材料裁决幂等 / 冲突。需求到结算状态的归属顺序号必须等于结算发生时间戳。
5. 组合.需求任务方法必须在任务完成前复核值相等；未达成时任务和需求结算都保持结构零变化。
6. 组合.运行期业务操作只继续转发收口后的组合入口，本切片不修改其公开签名。
7. 需求服务.h 的 >=、证据不足和最近记录属于历史兼容路径，本切片不修改，也不得据此宣称全部兼容路径已经治理。
8. 写入后不一致属于追根因解决；日志、控制台、缓存和显示不得修补结算事实。
```
