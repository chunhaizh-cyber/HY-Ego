## 说明

本流程把旧数据库导入改为信息逻辑迁移。旧 `鱼巢` 只提供候选定义、服务逻辑和证据，不提供可直接复制的运行期机器事实。第一轮只在隔离候选仓库构建最小样本，不替换当前运行期仓库，不写 SQL Server。

## 流程图

```mermaid
flowchart TD
    A["用户确认无旧数据库"] --> B["冻结鱼巢只读证据快照"]
    B --> C["读取规范、说明书、初始化代码、定义表和既有 FS 逻辑包"]
    C --> D["提取定义候选、关系候选、服务逻辑候选和排除材料"]
    D --> E["逐项分类：保留、改写、暂缓、拒绝、待裁决"]
    E --> F{"本批是否有已确认候选"}
    F -- "否" --> G["逻辑内返回：记录空批次或待裁决项"]
    F -- "是" --> H["入口预检：来源指纹、稳定键、文本、目标服务和依赖"]
    H --> I{"前置条件是否满足"}
    I -- "否" --> J["逻辑内拒绝：不创建候选结构"]
    I -- "是" --> K["在隔离候选仓库经领域服务构建"]
    K --> L{"内部写入和读回是否符合预期"}
    L -- "否" --> M["追根因解决：丢弃整套隔离候选仓库"]
    L -- "是" --> N["生成节点、主信息、关系、索引和语义映射证据矩阵"]
    N --> O{"是否已有独立运行期导入许可"}
    O -- "否" --> P["保存迁移证据，等待后续实施切片"]
    O -- "是" --> Q["按后续确认事务入口导入内存仓库"]
    Q --> R["读回运行期结构并核对幂等、版本和拒绝矩阵"]
    R --> S["生成新项目仓库快照和事件审计材料"]
    S --> T["后续持久化适配写入 SQL Server"]
    U["旧数据库不存在"] -. "不作为来源" .-> B
    V["旧日志、缓存、显示文本、指针和旧编号"] -. "拒绝成为机器事实" .-> E
```

## 关键边界

```text
1. 迁移源是鱼巢的规范、代码定义、初始化顺序和服务逻辑证据，不是旧数据库。
2. 函数只作证据采集单位，服务逻辑包才是迁移确认单位。
3. 旧指针、旧节点编号、运行日志、缓存、显示文本和旧数据库字段不得直接成为新项目事实。
4. 第一轮只建隔离候选仓库；运行期导入、快照恢复和 SQL 持久化必须分别另有确认计划。
5. 前置材料不满足时逻辑内拒绝；进入候选结构构建后出现非预期结果必须追根因并丢弃整套候选仓库。
```
