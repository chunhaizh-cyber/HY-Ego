## 说明

本模板用于把旧应用逻辑迁移为服务调用编排。旧函数事实只作为证据节点，不作为最终代码单位。

## 流程图

```mermaid
flowchart TD
    A["旧函数事实证据"] --> B["提取应用逻辑意图"]
    B --> C["整理输入材料"]
    C --> D{"前置拒绝是否通过"}
    D -- "否" --> R["拒绝路径：结构不变化"]
    D -- "是" --> E["映射到领域服务入口"]
    E --> F["调用服务操作函数矩阵"]
    F --> G{"是否需要写结构"}
    G -- "否" --> H["返回只读材料或候选"]
    G -- "是" --> I["经领域服务写节点 / 主信息 / 关系 / 索引"]
    I --> J{"是否记录动态 / 因果"}
    J -- "否" --> K["输出结构变化和读取结果"]
    J -- "是" --> L["动态服务记录实例动态"]
    L --> M["因果服务记录轻量因果引用"]
    M --> K
    H --> K
    R --> N["验收：拒绝后节点 / 关系 / 索引数量不变"]
    K --> O["验收：输入入口 / 核心处理 / 结构承载 / 输出变化 / 调用路径 / 验证方式"]
    O --> P["生成批次级代码实施包候选"]
```

## 关键边界

```text
函数事实不是迁移单位；服务逻辑包才是迁移确认单位。
线程不是动作来源。

需求目标是目标状态，不是 I64。
特征值服务只由特征服务直接访问。
```
