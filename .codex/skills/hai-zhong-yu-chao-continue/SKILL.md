# 海中鱼巣继续路由

## 核心规则

把裸 `继续` 解释为：

```text
继续当前窗口已经拥有的角色和任务
```

不得解释为：

```text
执行仓库里写着的下一步
切换到窗口交互记录中的目标角色
消费下一个队列项
进入另一个 worktree
开始构建或集成
```

采用固定路由：

```text
裸继续
-> 识别当前窗口类型
-> 核对聊天声明与仓库登记是否一致
-> 只沿当前角色继续
-> 需要其它角色时完成本角色交接并派发独立窗口
```

## 窗口识别

按以下证据识别，不凭“下一步”字段猜测：

1. 用户最近一次明确声明，例如“这是计划窗口”“当前是执行窗口”。
2. 当前任务标题、既有任务对象和当前工作目录。
3. `项目记忆/窗口交互记录.md` 中当前所有者，而不是目标窗口或下一步。
4. `项目记忆/并行工作树登记表.md` 中当前路径、分支、worktree 编号和窗口所有者。
5. Git 当前分支、HEAD、dirty state 和目标计划 / 实施记录。

用户明确声明与仓库登记冲突时，只做只读核对并说明冲突。不得先执行再补登记，也不得用聊天中的裸 `继续` 完成角色切换。

仍无法唯一识别时，保持只读并询问窗口类型；不得修改文件、构建、提交、推送、创建 worktree 或消费队列。

## 按类型继续

### 设计窗口

允许：

- 继续当前规则、流程图、详细设计、计划、计划索引、任务队列和设计治理。
- 读取正式实施 / 集成回执并同步状态、归档、依赖和后继路线。
- 完成本角色验证、精确提交和非强制推送。
- 在正式登记与推送后，向独立执行或集成窗口派发任务。

禁止：

- 消费执行队列或修改 C++ / 工程 / 自检。
- 运行 C++ 构建、自检、连续验证或集成 cherry-pick。
- 因“下一步是执行 / 集成”而把当前窗口切成执行 / 集成窗口。
- 进入已登记任务 worktree 代替执行窗口工作。

若当前窗口已被用户声明为计划 / 设计窗口，后继执行或集成必须交给独立窗口。裸 `继续` 只继续设计工作。

### 执行窗口

允许：

- 继续本任务 worktree 唯一分配的计划和当前机械阶段。
- 按允许文件修改代码 / 工程 / 自检和执行专属记录。
- 运行计划验证，精确提交并只推送同名任务分支。

禁止：

- 修改计划、计划索引、任务队列、流程图、详细设计或中央治理。
- 消费未分配计划、切到 main、直接推送 `origin/main`。
- 完成或漂移后自行切成设计 / 集成角色。

完成或退回时只形成正式执行回执，状态停在 `分支完成待集成`、`执行完成待设计同步` 或 `接口漂移待设计修订`，然后交回对应窗口。

### 集成窗口

允许：

- 继续当前已登记集成批次的提交核对、固定顺序汇集、集成验证和专属集成记录。
- 仅在门禁全部通过时按登记方式推送集成分支，并由授权的主集成角色非强制发布 main。

禁止：

- 发明或修改业务语义、接口、计划和中央设计状态来消除冲突。
- 修改任务实现来让测试通过。
- 将“任务分支完成”直接写成“计划已完成”。

集成完成只到 `已集成待设计同步`；设计归档仍交回设计窗口。

### 只读监控 / 复核窗口

只继续读取 Git、计划、队列、实施记录、代码事实和验证证据并汇报。不得修改文件、运行构建、创建 worktree、暂存、提交、推送或入队。

## 角色切换门禁

裸 `继续` 永远不构成角色切换指令，即使单窗口串行双角色模式已经启用。

同一物理窗口只有同时满足以下条件才能切换：

1. 用户明确要求当前物理窗口切换到目标角色，不是只说 `继续`。
2. 当前角色已经完成允许的记录、验证、提交和非强制推送。
3. 计划索引、任务队列、窗口交互和工作树登记已正式写明目标角色与生效提交。
4. 推送后重新读取 Git、正式路由、目标计划、最新回执和实际接口。

默认双物理窗口模式下，不切换当前窗口；使用独立任务承接执行或集成。仓库中的“目标窗口 / 角色”和“下一步”只表示交接目的地，不表示当前窗口已经变成该角色。

## 中断恢复

若上一轮被用户中断或工具报告进程可能仍在后台：

1. 先检查当前角色启动的构建、测试或辅助进程是否仍在运行。
2. 用户已经暂停、质疑角色或改变方向时，停止本窗口启动的相关进程。
3. 核对 worktree、分支、HEAD、dirty state、已完成证据和未完成门禁。
4. 只按当前窗口角色报告续点；不得自动恢复被中断的其它角色工作。

## 裸继续的最小输出

执行动作前先用一句话说明：

```text
当前识别为 <窗口类型>，本轮继续 <该角色任务>，不切换角色。
```

若识别结果和用户预期不一致，先停在只读状态纠正，不边执行边解释。
