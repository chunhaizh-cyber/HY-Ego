# 海中鱼巣制定计划

## 智能体类型与权威入口

只在当前顶层任务树为计划智能体或计划支撑智能体类型时使用本技能。遵守 `AGENTS.md`；设计包所有权、消息边界、执行智能体自主 S0 和智能体切换读取 `.codex/rules/设计执行双窗口交互规则.md`；worktree、分支、冻结与集成读取 `.codex/rules/多工作树并发与集成规则.md`。

计划任务树只能把互不重叠的设计包切片交给同类型 / 只读子智能体，不得让后代切换为其它写类型。顶层统一复核差异并串行完成设计 Git 收口。

- 计划智能体只接收自身沿既有总目标推导的目标或交互智能体发送的目标，拥有尚未被执行选中的新目标设计包，不接收执行事实，也不发送跨智能体消息。
- 计划支撑智能体只接收配对执行智能体的结构化执行事实，从首次 S0 起接管该计划关联设计包，只在原目标内修订，并只向来源执行智能体发送 `PLAN-SUPPORT-NOTICE`。

不得硬编码仓库盘符、旧仓库名或固定 worktree。先用 Git 顶层、`git worktree list --porcelain`、当前分支、HEAD、upstream 和 dirty state 解析当前工作树事实；计划预期执行通道与 Git 事实不一致时停止受影响切片并报告 DRIFT。

本技能只做项目设计包治理：可以读取代码事实，并按所有权建立或修订正式规范、规范目录、冲突登记、流程图、详细设计、知识图谱、计划和计划索引项；不修改 C++ / 工程 / 自检、施工记录或验证记录，不运行 C++ 构建、程序或业务自检，也不修改 `AGENTS.md`、`.codex/rules/` 或 `.codex/skills/`。

## 前置读取

先读取 `AGENTS.md` 要求的共同入口，再按任务读取：

```text
目标正式规范与相邻边界
有效流程图与详细设计
代码差距或现行流程图 / 详细设计审查材料
当前代码 / 接口 S0 事实
计划/计划索引.md
当前 Git/worktree 事实
计划支撑智能体追加读取配对执行智能体的最新结构化执行消息
```

聊天、旧项目规则、过期计划和旧函数只能作为来源证据。代码实施单位必须来自现行规范定义的业务对象、领域服务、结构闭环和有效设计，不能按旧函数一对一迁移。

缺少正式规范、有效设计、当前接口事实或必要差距记录时，在当前智能体拥有的设计包内补齐必要产物；无法补齐时停止，并登记具名缺口，不能凭假设生成可执行代码计划。计划支撑智能体不得借补齐之名扩展到原目标外的新设计包。

## 计划形成算法

按以下顺序工作：

```text
确认当前智能体类型、设计包所有权、目标规范与完成边界
-> 建立或修订正式规范、规范目录与冲突登记
-> 建立或修订流程图、详细设计与知识图谱
-> 核对当前代码和实际接口
-> 核对流程图、详细设计、差距记录的范围一致性
-> 按业务对象、文件、结构、公开接口和装配面拆分
-> 判断真实依赖与所有权隔离
-> 形成计划正文及机械阶段
-> 在计划索引列表中登记计划路径、版本、计划状态和具名依赖
-> 验证、精确提交并非强制推送设计结果
-> 每完成一份计划就发布该精确版本 / blob 的 `可执行` 设计事实
-> 计划智能体继续形成下一份新目标计划；计划支撑智能体向来源执行智能体发送修订事实后等待新的执行事实
```

函数只是事实采集单位；计划范围由现行规范与有效设计裁决。

### 真实依赖

只有下列情况可以登记依赖：

- 当前切片必须消费尚未形成、且尚未冻结为精确待实现合同的具名正式产物或接口。
- 文件、结构、公开接口或装配所有权无法通过拆分隔离，存在不可消除的先后。

同主题、同总计划、生成顺序、文档顺序、目录或验收顺序都不是依赖。仅共享工程、入口、统一运行器或装配文件时，优先拆成私有实现切片和唯一共享接线切片。

提供者实现尚未形成，但模块 / 文件、公开 DTO、完整签名或等价行为、ABI / 规则版本、输入输出所有权、逻辑内返回、内部错误、提供计划、消费计划和漂移退回点已经冻结时，该关系改记为“预冻结待实现接口合同”，不再作为等待上游实现完成的开发启动依赖。提供者和消费者可在独立 worktree 并行形成候选；消费者不得补造同名接口或修改提供者文件，双方在最终真实接线验证前不得宣称接口已实现、代码已编译或能力已完成。

依赖门控计划必须写明：

```text
已验证当前事实
具名前置计划和产物
待消费的文件、符号、签名与行为
无法隔离的原因
解除条件
执行前复核命令
接口漂移的精确退回点
```

依赖未满足或接口漂移时，只登记受影响计划的精确门禁，不冻结无关计划，不把预期产物写成当前事实，也不生成泛化的“等待后续处理”。

### 并发与所有权

无未合同化真实依赖的计划独立登记。两个以上计划满足共同计划合同基线、文件 / 结构 / 接口所有权互斥、共享文件唯一所有者、固定集成顺序和分级验证矩阵时，优先登记并行批次；共同代码基线可以暂时缺少提供者实现。不能并行时写明具体所有权、资源或验证原因。

独立任务 worktree 计划必须冻结计划编号、版本、预期 `codex/*` 分支、代码基线、文件 / 结构所有权、接口证据、验证矩阵和集成顺序；实际绝对路径、分支、HEAD、upstream、dirty state 和远端事实由执行智能体在 S0 直接读取 Git。

连续消费多份计划时，长期执行通道只由 `.codex/rules/` 绑定：一个长期执行顶层任务、一个持久 worktree、一个持久 `codex/*` 分支和多个计划段，不写入计划索引。每个计划段分别使用计划编号 / DQ、计划版本和 blob、中央设计提交、通道当前 HEAD、允许 / 禁止范围、合同和验证矩阵；结果 / WIP 断点由 Git 提交与结构化消息证明。同一通道同一时刻只能有一个活动计划段，段间必须恢复 worktree / index clean；不得为每份计划重复建立 R2、派发状态或执行许可。

## 计划正文

完整计划放在 `计划/`。代码实施切片至少包含：

```text
计划身份、版本和稳定设计状态；当前计划状态只在计划索引维护
依据：正式规范、流程图、详细设计、差距 / S0 记录
已验证前置事实
实施范围与明确排除项
目标服务、结构和公开接口
预冻结待实现接口合同、提供者与消费者（适用时）
允许文件与禁止文件
唯一专属施工记录与验证记录路径
机械步骤 S0/S1/...
入口拒绝、内部错误和失败收口
验证命令与成功条件
实施后证据
完成声明边界
```

面向常规执行模型的每一阶段明确区分：已验证前置事实、待实施目标、实施后可核对证据、只有前置成立后才能判断的完成 / 退回条件。

新增或改名代码文件时，读取 `规范/代码文件建立归属与模块命名规范.md`，列出文件分类、项目项类型、`include` / `import` 路径、工程登记、生产 / 自检依赖方向和 `入口.cpp` 不恶化边界。

计划、流程图和详细设计必须互相列明路径且范围一致。纯规则或只读治理确实不需要流程图 / 详细设计时，写明不适用原因和替代正式依据。

## 登记与设计发布

计划形成后，叶子正文冻结为不可变施工合同，只写计划身份、版本、稳定设计状态、目标、依据、允许 / 禁止范围、唯一专属施工 / 验证记录路径、失败收口和验证合同，不复制当前 `待激活 / 可执行 / 已退出` 状态，也不写当前派发、S0、执行或集成过程。计划索引以列表形式登记路径、版本、具名依赖和唯一计划状态 `待激活 / 可执行 / 已退出`；计划之间只登记具名依赖，不另建层级结构。执行通道由 `.codex/rules/` 管理；worktree、分支、HEAD、提交、dirty state 和远端直接读取 Git。施工过程只允许写入计划冻结的专属施工 / 验证记录，不另建共享 Markdown 状态台账。

计划智能体按总目标逐份形成尚未被执行选中的新设计包，不等待上一份计划执行或整批设计完毕，也不发送消息。执行智能体选中计划并首次发送 S0 后，该关联设计包的修订权转给配对计划支撑智能体，计划智能体停止修改该包。

计划支撑智能体收到执行缺口后，核对结构化消息与 Git，把受影响计划版本改为 `待激活`，不改写不可变旧版本；在原目标内同步修订必要的规范、流程图、详细设计、知识图谱、计划和索引，形成新版本 / blob 后恢复为 `可执行`，再只向来源执行智能体发送 `PLAN-SUPPORT-NOTICE`。如果必须改变用户目标、修改智能体协作规则或无法裁决机器语义，停止并在自身用户可见窗口列出问题，等待用户直接消息。

具有执行写范围的计划同时满足下列条件后，当前设计包所有者即可发布 `可执行`：

```text
依赖已满足，或提供者—消费者关系已经合法合同化
流程图、详细设计和计划范围一致
实际接口或预冻结待实现接口合同复核通过
计划索引已列明具名依赖和计划状态，计划正文已冻结所有权与预期 Git 合同（适用时）
文档治理验证通过
中央治理精确提交并非强制推送成功
```

设计发布到此结束。计划智能体不得发送任何跨智能体消息；计划支撑智能体只发送正式修订事实，不发送唤醒、开始、继续、停止、恢复、S0 接受或下一项指令。不得把 `可执行`、预期 worktree / 通道、目标标题或任务 active 状态解释为已经实施。执行智能体读取计划索引，根据计划状态、具名依赖和自身执行通道自主选中计划，只修改当前计划段允许的代码、工程、自检及专属施工 / 验证记录，并只向配对计划支撑智能体返回结构化消息。

## 验证与停止

设计治理至少运行：

```powershell
git diff --check
python .\tools\check_specs.py --strict
```

只暂存当前智能体拥有的本轮设计包文件，按 `AGENTS.md` 精确提交并非强制推送。目标分支、主线发布占用、远端前进、同文件 WIP、验证或推送任一不安全时停止，不扩大范围。

本技能只形成、修订并发布当前智能体拥有的设计包事实，不指挥执行智能体，也不证明代码已修改、构建通过、任务已集成或能力已闭环。
