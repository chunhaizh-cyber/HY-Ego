# 海中鱼巣执行

## 角色与执行通道入口

只在当前顶层任务树为执行角色时使用本技能。遵守 `AGENTS.md`；角色 S0、完成 / 退回消息读取 `.codex/rules/设计执行双窗口交互规则.md`；worktree 与 Git 身份读取 `.codex/rules/多工作树并发与集成规则.md`。

不得硬编码仓库或 worktree 路径。用 Git 顶层、`git worktree list --porcelain`、当前分支、HEAD、upstream、dirty state、远端事实和 `.codex/rules/` 中的执行通道合同解析独立任务 worktree 或长期执行通道，并确认目标计划 blob、路径、分支、计划段起点和所有者闭合。

本技能只消费当前执行顶层任务实际所在的独立任务 worktree，或 `.codex/rules/` 为当前分支固定的长期执行通道。不得夺取其它执行任务、其它通道、其它 worktree 或其它所有权，也不得把裸 `继续` 解释为角色切换。长期执行通道不是多计划并发写区：同一时刻只能有一个活动计划段，每段单独冻结计划编号 / DQ、计划版本与 blob、中央设计提交、段起点、允许 / 禁止范围、接口合同和验证矩阵。

当前计划正常收口，或计划明确允许的代码 WIP 已形成具名断点提交且 worktree / index 恢复 clean 后，可以自主重读计划列表，并根据计划状态和具名依赖检查其它可执行计划，再按 `.codex/rules/` 与 Git 核对自身执行通道；不等待设计窗口派发、确认冻结、接受 S0 或选择下一项。不为每份计划重建 R2、顶层任务、worktree、分支或执行许可。

执行任务树的后代只能继承执行角色或只读；不得承担设计或集成写角色。多个计划允许切片确实可隔离时才使用执行 / 只读子智能体，并冻结唯一所有权；Git 暂存、提交和推送由指定智能体串行收口。

## S0

开始写入、构建或运行前：

```text
读取 AGENTS.md 要求的共同入口
-> 读取计划索引中的计划列表、每份计划的计划状态和具名依赖，并读取拟选中的叶子计划
-> 从 `.codex/rules/` 与 Git 读取执行通道身份
-> 读取关联规范、流程图、详细设计，以及实际代码接口或计划登记的预冻结待实现接口合同
-> 核对允许 / 禁止文件、依赖产物或合同版本、提供者 / 消费者所有权、验证矩阵和完成边界
-> 核对当前任务树角色、计划版本与 blob、计划状态、执行通道 / worktree、分支、upstream、当前 HEAD、dirty state 和远端事实
-> 核对当前通道没有第二活动计划段，并把当前 HEAD 固定为本段唯一起点
-> 核对本段允许 / 禁止范围不与已收口段、其它通道或当前活动所有权冲突
-> 返回 PASS 或具名 DRIFT
```

首次接管独立任务或长期通道，以及长期通道开始每一个新计划段时，计划索引、实际计划、设计材料、实际接口或预冻结合同和 Git 身份必须闭合；被选计划的计划状态必须为 `可执行`，具名真实依赖必须已满足。执行窗口自行完成 S0，PASS 后直接写入，不等待设计窗口接受。合同并行计划中，提供者实现尚不存在属于计划内当前事实，不得仅因此返回 DRIFT；执行者必须确认自己只按合同形成候选，且不修改接口所有者或共享接线文件。

长期执行通道的计划段起点等于选择该计划时的通道当前 HEAD。第一段从通道建立基线开始；后续段从上一已收口段的结果 / WIP 断点提交开始。通道 HEAD 可以累计前进，因此不得要求后续计划段 HEAD 仍等于通道建立基线，也不得把正常累计前进误报为基线漂移。

## 执行算法

```text
从当前独立任务或长期执行通道自主选择一份 `可执行` 且真实依赖已满足的计划
-> 核对没有第二活动计划段，把通道当前 HEAD、计划 blob 与白名单冻结为本段 S0
-> 复核具名前置产物与实际接口，或预冻结接口合同
-> 按计划机械阶段实施最小闭合切片
-> 只修改允许的代码 / 工程 / 自检
-> 运行计划明确授权的验证
-> 精确提交一个计划段结果，或按计划许可提交一个具名 WIP 断点
-> 非强制推送当前独立任务分支或长期执行通道分支
-> 向设计窗口返回结构化的完成、漂移或失败事实与计划修订请求
-> worktree / index 已恢复 clean：结束当前活动计划段并自主重读计划索引
-> 存在其它无关、真实依赖已满足的可执行计划：以当前累计 HEAD 建立下一计划段，重新 S0，PASS 后直接继续
-> 不夺取其它任务 / 通道 / worktree / 所有权，不自行集成
```

代码计划必须明确允许文件、禁止文件和验证命令。新增文件或修改 `入口.cpp` 时，还必须符合 `规范/代码文件建立归属与模块命名规范.md` 及计划中的文件分类 / 装配合同。

前置已满足后，内部结构写入、绑定、读回、发布或验证不符合预期时，按计划的内部逻辑错误路径停止并追根因；不得擅自改为普通失败或扩充计划。

## 允许写入

只允许当前选中计划列出的代码、工程和自检文件。任何低位计划、旧交互规则或历史实施习惯列出的 Markdown 路径都不再构成执行写权或完成必需项。

禁止修改或移动：

```text
AGENTS.md
**/*.md
规范/**
流程图/**
规范/详细设计/**
计划/**
计划/计划索引.md
```

计划未明确允许的文件一律禁止。无关 WIP 不得修改、暂存、stash、清理或带入提交。

## 依赖、漂移与退回

计划状态不是 `可执行`、具名真实依赖未满足、未合同化前置产物尚未进入目标代码基线、当前 Git 分支与计划执行通道不符，或计划 blob 不能与当前计划和本段中央设计提交闭合时，不执行。长期通道中尚未被选中的计划不形成活动计划段；选择时才冻结当时的累计 HEAD。合同并行计划只要求同版本预冻结接口合同可由目标计划 blob 证明，不要求提供者实现已经存在。`可执行` 只表示该精确施工合同可供选择；执行窗口仍须自行完成 S0，但不需要设计窗口另行许可。

实际接口任务按计划逐项比较文件、符号、签名与行为；合同并行任务逐项比较合同版本、模块 / DTO / 签名 / ABI / 所有权 / 返回错误和提供者映射。发生任一情况时立即停止扩大范围：

- 实际接口与计划假定契约不一致。
- 预冻结合同缺项、版本不一致、消费方需要补造接口或修改提供者独占文件。
- 需要修改禁止文件或中央治理文件。
- 通道建立基线、累计父链、当前计划段起点、所有权、worktree、分支或远端身份漂移。
- 有效流程图 / 详细设计缺失、撤回或不能覆盖计划范围。
- 发现需要新的业务裁决。

通过结构化消息写明计划编号 / DQ、计划版本与 blob、执行通道或独立任务 worktree、计划段起点、实际接口、具名差异、已改文件、验证、结果或 WIP 断点提交、dirty state、建议设计动作和完成边界。不得在仓库内新建或修改断点 Markdown，不得替设计窗口修订计划、改计划状态、解除依赖或修改计划索引。

受影响计划在 DRIFT 后保持停止。只有计划明确允许保留代码 WIP 时，才能在白名单内形成具名 WIP 断点；能够使 worktree / index 恢复 clean 后，执行窗口可以立即自主检查其它所有权隔离、无真实依赖的 `可执行` 计划，不等待设计回复。无法合法撤回或提交且工作区仍 dirty 时，整个长期通道停在当前计划段，不得 stash、覆盖现场或带入下一段。

代码差异验证后只提交并推送当前独立任务分支或长期执行通道分支，再发送退回消息。推送失败时保留本地事实并如实报告，不宣称设计窗口已经取得正式回执。

## 完成与 Git 收口

完成时执行计划规定的全部任务候选级验证；构建和程序运行只有计划明确授权时才可运行。合同并行计划明确把完整构建和运行后置时，不得自行运行，也不得把候选完成表述为编译或能力通过。至少检查：

```powershell
git diff --check
git diff --cached --check
```

只暂存当前计划段允许的代码、工程和自检文件；一个结果 / WIP 提交只能属于一个计划段。提交后非强制推送当前 `codex/*` 独立任务分支或长期执行通道分支。任何 Markdown 都不得进入 index。禁止 force、merge、rebase、cherry-pick、把 main 拉入执行分支或直接推送 `origin/main`。

回执只能说明执行续点、建议将当前计划状态改为 `待激活`、记录接口漂移或记录计划段候选完成；执行窗口不得直接修改计划索引。独立任务提交或长期通道计划段提交不等于进入 main，也不授权当前执行任务创建集成 worktree、修改设计治理文件或消费其它任务 / 通道所有权。长期通道最终汇集必须提供通道终端提交和各计划段起点—结果提交映射，不得把累计分支整体解释成一个不可分计划结果。

## 停止条件

当前计划完成时先回执，在 worktree / index clean 后自主重读当前通道所有权内是否存在其它 `可执行` 计划；没有候选才停止。无法解决、需要修订、用户暂停、角色 / 规则冲突、依赖未满足、身份漂移、验证失败或计划要求的环境不可用时停止受影响计划并回执；只有形成合法 clean 断点后才能检查其它无真实依赖计划。平台轮次结束但未命中完成 / 退回条件时，只发送精确执行续点并保持当前活动计划段，不得切换下一段。

不得用本技能声明计划归档、主线集成、完整业务闭环、旧能力等价迁移、外设接通、自我循环、自我苏醒或成熟验收。
