# 海中鱼巣实施前审查

## 角色与入口

严格服从仓库根目录 `AGENTS.md`。用 Git 顶层、目标计划与当前 worktree 的 Git 事实解析路径、分支、HEAD、提交、dirty state、角色和文件范围，不硬编码仓库路径或主分支名。

本技能是只读实施前证据审查：

```text
用户目标
-> 当前本地事实
-> 规范 / 详细设计 / 已确认计划
-> 允许文件 / 禁止文件 / 验证方式
-> PASS / DRIFT 审查回执
```

`PASS` 只表示本次所列证据未发现漂移；不替执行窗口选择计划、形成活动计划段或解除依赖。

## 专用步骤

1. 解析 Git 顶层并读取 `AGENTS.md`。
2. 按目标限载读取：

```text
计划/计划索引.md
规范/规范目录.md
目标正式规范、有效流程图与详细设计、现行计划、最新结构化消息或 Git 断点提交
```

3. 当前事实缺失或过期时先调用只读 S0 事实扫描。
4. 逐项核对：

   - 目标是否由当前本地事实支撑；
   - 规范、流程图、详细设计、计划及计划索引中的计划状态与具名依赖是否完整且范围一致；
   - 允许 / 禁止文件、结构与公开接口所有权是否明确；
   - dirty state 是否污染切片；
   - worktree、分支、HEAD、计划版本、执行通道和所有者是否与计划合同及 Git 事实一致；
   - 依赖、共享装配面、集成顺序和验证矩阵是否无冲突；
   - 入口拒绝、失败收口、验证命令和完成边界是否可机械执行。
5. 任一缺失或漂移统一返回 `DRIFT`，并用子类说明 `缺事实 / 缺规范 / 缺设计 / 缺计划 / 计划状态不符 / 所有权冲突 / 基线漂移 / 脏树污染 / 待用户裁决`。

## 输出

```text
审查对象：
当前事实是否足够：
当前工作树 / 分支 / 基线是否匹配：
权威依据：
允许修改范围：
禁止修改范围：
现有可复用入口：
主要风险：
审查结论：PASS / DRIFT（子类）
证据接收角色：
正式角色下一步：
```

`PASS` 回执必须列明所核对的提交、worktree、计划版本、计划状态、执行通道和文件范围；执行窗口仍须根据计划列表自主选中计划并在开工前完成 S0。`DRIFT` 只退回受影响对象，不越权修改设计治理材料。

## 停止边界

- 本技能不得修改文件、构建、运行程序、暂存、提交、推送、修改计划状态、解除依赖、裁决完成或取得活动计划段 / 发布占用。
- 不以旧函数体、旧字段、日志、README、控制台或注释补齐缺失依据。
- 计划合同、Git 基线、文件 / 结构 / 接口所有权或集成合同缺失时必须返回 `DRIFT`，不得批准并行执行。
- 用户要求立即修改时，先把回执交给当前正式设计或执行角色；本技能不自动切换角色。
